课程总论

GPU 性能工程的核心不是“把代码搬到显卡上”，而是把并行度、数据移动和硬件资源占用放进同一张账本。

Overview
建议学习对象

会写 Python 或 PyTorch，想看懂推理或训练系统里 kernel、算子融合、显存带宽和吞吐瓶颈的人。

前置知识

需要基本线性代数、张量形状、矩阵乘法和 Transformer 注意力结构；不要求已经会写 CUDA。

学完以后

应能读懂简单 CUDA/Triton 内核，解释常见性能指标，并判断一个 AI 算子的第一优化方向。

推荐路径

先建立“怎么判断瓶颈”的语言，再学“怎么写更接近硬件的算子”。

Route
01-03｜硬件与内存直觉

理解线程、warp、block、SM、全局内存、共享内存和寄存器之间的关系。

先回答：程序为什么没有把 GPU 跑满？
04｜测量优先

建立 profile、计时、吞吐、带宽、占用率和 Roofline 之间的联系。

先定位瓶颈，再谈优化手段。
05-06｜Triton 实作入口

用块级张量编程理解 matmul、softmax、layernorm 的数据移动模式。

把“并行”落到程序索引和内存访问。
07｜注意力与系统衔接

从 FlashAttention 看到分块、重算、显存占用和真实推理吞吐之间的取舍。

为推理引擎内核课程做准备。

单元设计

每个单元都有存在理由；学习页已完成，这里给出边界、顺序和讲义入口。

7 Units
01

GPU 执行模型与性能语言

存在理由：先把 SM、warp、block、occupancy、latency hiding 这些词变成一套判断语言，否则后面所有优化都会像经验规则。

打开学习页
02

CUDA 线程层级与内核边界

存在理由：CUDA 的 grid、block、thread 决定了工作如何映射到硬件；这一单元让学习者能读懂最小 kernel 的执行边界。

打开学习页
03

内存层次、访存合并与带宽

存在理由：AI 算子经常不是算不动，而是数据搬不动；本单元解释 global、shared、register、cache 对性能的影响。

打开学习页
04

测量、Roofline 与瓶颈定位

存在理由：没有测量就无法判断优化是否真实有效；本单元把计时、带宽、FLOPS 和 profile 结果连起来。

打开学习页
05

Triton 编程模型与块级思维

存在理由：Triton 是理解现代 AI kernel 的低门槛入口；本单元把 program id、block pointer、mask 和自动调参讲清楚。

打开学习页
06

矩阵乘、Softmax 与 LayerNorm

存在理由：这三个算子覆盖计算密集、带宽密集和归约模式，是理解深度学习性能工程的最小案例组。

打开学习页
07

注意力优化与工程边界

存在理由：注意力把显存、带宽、数值稳定和分块算法放在一起，是从单算子走向 LLM 推理内核的桥。

打开学习页

资料边界

这门课只承诺建立公开资料上的学习路径，不替代官方文档、源码阅读和真实硬件实验。

Boundary
不覆盖

不系统讲 GPU 微架构细节、驱动开发、完整编译器后端、厂商库内部实现。

需要实测

所有性能结论都依赖硬件型号、数据形状、精度、驱动和库版本；学习页制作时应附带可复现实验。
