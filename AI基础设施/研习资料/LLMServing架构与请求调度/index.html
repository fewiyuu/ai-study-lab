LLM Serving 架构与请求调度

8 个单元 AI 基础设施 Ray Serve LLM vLLM SLO 请求调度

Serving 不是“把推理引擎包成 HTTP 服务”。真正的问题是：请求不断进来时，系统如何在延迟、吞吐、成本和可用性之间做可解释的选择。

建议起点：先画出一次请求经过网关、队列、引擎和流式响应的路径，再讨论路由、伸缩和观测。

课程总论

这门课把 LLM Serving 看成一套生产系统：推理引擎只是核心算力节点，周围还需要协议、队列、路由、弹性、隔离和故障处理。

Overview
建议学习对象

想部署 LLM 服务、设计网关与调度策略、或评估 vLLM/Ray Serve/TensorRT-LLM 上线方案的人。

前置知识

需要了解 LLM 推理生命周期、HTTP/流式响应、基本队列和服务可观测性；最好先学现代 LLM 推理引擎内核。

学完以后

应能把一个 LLM 服务拆成入口、调度、执行、观测和容量规划，并解释每层的关键指标。
