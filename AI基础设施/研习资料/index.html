AI 基础设施方向

模型不是只在 API 里运行，它还要占显存、排队、通信和服务。

这条路线把大模型放回真实机器和真实流量里看：GPU kernel 为什么慢，KV cache 为什么吃显存，推理引擎怎样调度请求，分布式训练怎样切参数和梯度，Serving 架构怎样把模型能力接进产品。

CORE

GPU 性能工程负责底层计算直觉；现代 LLM 推理引擎内核负责把一次请求拆成 prefill、decode、KV cache 和调度。

SCALE

再看扩展和服务

分布式训练系统解释模型如何跨卡训练；LLM Serving 架构解释模型如何接进真实流量、网关、监控和成本约束。

推荐路径

从性能直觉到上线架构。

AI 基础设施不是单纯运维。它回答的是：同一个模型，为什么有人跑得慢、贵、不稳定，而有人能把它接进产品。

01

GPU 性能工程

先学会看 kernel、显存带宽、算子融合和 profiling 结果。

02

现代 LLM 推理引擎内核

继续理解 PagedAttention、KV cache 和 batching 如何影响吞吐与延迟。

03

分布式训练系统

回到训练侧，看参数、梯度、优化器状态、激活和通信怎么切分。

04

LLM Serving 架构与请求调度

最后把推理引擎接进服务链路，处理路由、限流、缓存、监控和成本。

如果只想快速开始：先读 GPU 性能工程。如果眼下的问题是部署和并发，直接读 LLM Serving 架构与请求调度；如果问题是训练爆显存或多卡效率，读 分布式训练系统。
