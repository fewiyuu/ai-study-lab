Agent Skills 研习资料

按顺序学习四个知识块：先建立 Skill 的定位，再写结构和触发，随后处理脚本资源，最后做评估、安全和跨平台发布。

01 技能的定位与渐进披露

学完后，你应该能判断一个任务该不该做成 Skill，并能解释为什么 Skill 的价值不是“多写一段提示词”，而是把可复用流程放进智能体运行环境。

02 SKILL.md 结构与触发

学完后，你应该能写出一个能被客户端发现的最小 Skill，并能审查 name、description、allowed-tools、compatibility、references 和 assets 的常见错误。

03 脚本资源与上下文经济

学完后，你应该能设计一个适合智能体调用的脚本接口，并能解释为什么 stdout、stderr、退出码、--help 和 dry-run 会影响智能体下一步行动。

04 评估、安全与跨平台发布

学完后，你应该能为一个 Skill 设计最小 eval 集，区分 should-trigger 与 should-not-trigger，写出输出断言，并说明发布到其他客户端前要检查哪些兼容性问题。
