## Lesson Boundary

- 本页只讲持久执行和检查点：恢复点、事件历史、Workflow / Activity 分工、人工等待、恢复证据。
- 本页不展开幂等实现、重试策略、结构化输出或追踪后端接入的完整工程细节；这些内容分别留给 03 / 04 / 05 / 06。
- 贯穿锚点建议固定为“研究助理生成报告并等待确认后发送”，因为它同时包含长任务、人工等待和外部副作用，但本页只处理“怎么从中断中继续”。
- 如果学习者已经能说清“恢复点在哪里、哪些状态必须保存、哪些动作会被重放”，就可以去看 03；如果还说不清这些，先留在本页把状态边界补齐。

## Gaps And Guardrails

- 不补新的网络考据，只使用课程地图、原始单元页和上面列出的公开资料名。
- 如果某个状态还不能准确分层，先标成“恢复所需信息不足”，不要硬塞成一个统一的 checkpoint。
- 本页的目标是把“中断后继续”讲清楚，不是把所有可靠性术语一次讲完。
