Agent 基础与工程

这门课回答一个实际问题：当模型不只要“回答”，还要“做事”时，应该怎样把它放进一个可控、可追踪、可停止的系统里。课程从 ReAct 循环起步，再往工程架构、设计模式、评测安全和 Multi-Agent 判断推进。

课程主线

overview

这门课在解决什么

它不是在堆 Agent 名词，而是在把“模型能调用工具”推进到“模型能在约束下持续做事”。这里最先要想清楚的是：谁来决定下一步、谁来执行动作、谁来保存事实、谁来拦住越权。

一旦这四件事说不清，后面的规划、记忆、评测、恢复和多 Agent 拆分都会变成空转概念。

学完这门课之后

你应该能读懂一条 Agent 轨迹，知道它为什么停、为什么回头、为什么要加权限检查，也能分清一个系统是在走固定流程，还是在让模型动态决定下一步。

placement

公开来源

课程的第一层直觉来自 ReAct: Synergizing Reasoning and Acting in Language Models。工程边界、工具、状态、权限和停止条件的讲法，参考了 你不知道的 Agent：原理、架构与工程实践。

为什么放在这里

这门课属于 Agent 方向里的起步主干，所以放在 `Agent/研习资料/Agent基础与工程` 下面。它先把循环讲透，再把循环放进工程语境里，正好接后面的 Context Engineering、Claude Code 工作流、Harness、工具契约和可靠性。

适合与前置

prereq

适合谁先读

适合已经知道大模型会“生成文本”，但还不清楚它怎么被接进工具、状态和执行边界的人。也适合已经在做 Agent，但总觉得系统“会跑却不好控”的人。

开始前最好知道什么

先知道基本的 LLM 调用、简单工具调用和最基础的状态概念就够了。这里不要求你先会复杂规划、多 Agent 编排或评测框架；这些会在后面的单元里再展开。

学习路线

route

阶段一：先把循环看懂

01 ReAct 与 Agent 循环。先知道思考、动作、观察和停止条件怎样连成一轮。

过渡：先能看懂一条轨迹，后面才谈系统设计。

阶段二：把循环放进系统

02 Agent 工程架构、03 Agent 设计模式。开始处理规划、工具、状态、权限和常见组织方式。

过渡：循环懂了以后，下一步是把它放进稳定结构里。

阶段三：把系统收口成可验收的东西

04 评测追踪与安全、05 MultiAgent 系统设计。开始关心怎么验收、怎么追踪、什么时候值得拆成多个角色。

过渡：当系统能跑之后，才轮到验证和扩展。
