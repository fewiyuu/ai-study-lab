课程入口

Claude Code 工作流

这门课按 Agent 运行机制拆 Claude Code：先跑通本地最小 Agent，再看消息历史、工具调用、规划、子 Agent、上下文压缩、计划看板、权限、Hook、记忆、恢复、调度和 MCP 扩展。前半段让你理解“它怎么开始做事”，后半段让你理解“它怎么持续做事”。

24 节课 从启动到扩展 Agent 工作流

课程主线

COURSE MAP

这门课在讲什么

它讲的是一个代码 Agent 如何从“接到任务”走到“在本地把事做完”，再走到“可恢复、可审计、可扩展的系统”。最核心的线索不是某个模型名，而是任务、工具、状态、权限和反馈怎么连起来。

学完之后

你应该能解释一个代码 Agent 的执行链路，知道哪里是入口、哪里是工具、哪里是状态边界，也能判断后续问题该归到哪一层。

全部小节

UNITS

01 本地启动最小 Agent

foundation

建立第一层直觉：Agent 怎么从任务进入模型、再变成本地动作。

进入学习页

02 消息历史与 Agent 循环

foundation

补上对话历史如何累积、循环如何继续。

03 工具调用让 Agent 做事

foundation

进入工具调用本身，理解动作请求和执行反馈。

04 多工具分发与专用工具

practice

看模型如何在多个工具之间做分配。

05 行动之前先规划

practice

从“立刻做”过渡到“先想清楚再做”。

06 子 Agent 与任务委托

practice

开始拆分任务，让子 Agent 承担局部工作。

07 Skill 渐进式披露

practice

理解能力包如何按需展开，而不是一次全塞进上下文。

08 上下文压缩与无限会话

practice

看长期会话如何保住有效信息。

09 任务持久化与计划看板

system

把暂时性的对话变成可持久追踪的任务。

10 后台任务与通知队列

system

处理长任务、后台执行和通知返回。

11 Agent 团队与消息总线

system

把多个 Agent 放进同一协作结构里看。

12 团队关机协议与 FSM

boundary

让协作系统在该停的时候正确停下。

13 计划审批协议与决策权

boundary

明确谁能改计划、谁能放行。

14 自主 Agent 与看板轮询

boundary

看 Agent 如何在看板上自主推进。

15 GitWorktree 任务隔离

boundary

用独立工作树隔离并行任务。

16 可审计任务追踪

boundary

把运行轨迹变成可复查的证据。

17 HarnessEngineering 总览

advanced

先建立运行外壳的整体视角，再往细部下钻。

18 消息规范化

advanced

把输入消息整理成稳定结构。

19 权限系统

advanced

处理 Agent 可做什么、不能做什么。

20 Hook 系统

advanced

看外部事件如何挂进执行链路。

21 记忆系统

extension

整理长期记忆的读写边界。

22 错误恢复

extension

让失败不至于把任务直接打断。

23 定时调度

extension

把执行从即时调用扩展到时间驱动。

24 MCP 外部系统扩展

extension

把前面的能力接到外部系统和工具协议上。
