多智能体系统

这门课只回答一个工程问题：什么时候该把任务拆成多个智能体，什么时候单智能体更稳、更便宜、更容易验收。

先单智能体 识别约束 估算协调成本 做小范围实验

先从哪里进

如果你还不确定 Agent 基线怎么跑，先读 01；如果你已经遇到上下文污染，再接 02；如果你在考虑并行与专业化，读 03；如果你要落到验证边界，读 04。

为什么放在这里

PLACEMENT

课程位置

这个主题放在 Agent 研习资料下面，是因为它接的是运行循环、上下文工程和工具契约之后的分岔问题：当系统已经会做事时，再判断要不要拆成多个智能体。

这门课解决什么

它不教你把角色命名得更像团队，而是教你看见真实瓶颈：上下文污染、独立搜索空间、工具和领域专业化，以及这些变化带来的协调成本。

先学什么

PREREQ
你最好先会

知道 Agent 是怎样在工具、状态和反馈里持续工作的；如果这还不稳，先去看 Agent 基础与工程。

你会遇到的分叉

一边是上下文过载，一边是可以并行探索或专业化分工的任务；两边的处理方式完全不同。

读完这门课之后

你应该能说明是否需要多智能体，并把判断落成基线、实验、指标和回退条件。
