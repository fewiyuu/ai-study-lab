## What The Course Index Establishes

课程首页把这门课排成一条清晰路线：先讲工具调用的契约思维，再到函数调用基础、结构化输出与 JSON Schema，随后进入 MCP 工具规范、错误与回退、BFCL 评测和可维护工具面设计。

对本单元最重要的是这几条约束：

- 先把工具当成可执行合同，而不是按钮列表
- 本单元只讲工具边界、输入、输出、权限和失败语义，不展开 JSON Schema、MCP 协议和评测细节
- 下一单元会继续处理工具描述、选择与分发

## What The Old Page Already Covers

旧页已经有一条可用的教学骨架，核心内容没有偏：

- 工具调用需要共同约束名称、用途、输入、输出、权限和错误
- 结构化结果比自然语言更容易继续消费
- 副作用和权限边界必须明说
- 失败语义是接口的一部分
- 评测样例要覆盖真实失败，而不只看一次成功
- 一个日历创建例子可以把这些点串起来

1. 先看工具调用为什么是一份合同
