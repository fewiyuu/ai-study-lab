## What The Course Index Establishes

课程首页把这门课排成一条清晰路线：先讲工具调用的契约思维，再进入本单元的函数调用基础，随后是结构化输出与 JSON Schema，接着是 MCP 工具规范、错误与回退、BFCL 评测和可维护工具面设计。

对本单元最重要的是这几条约束：

- 本单元要讲清楚工具描述、工具选择和宿主分发三件事各自负责什么
- 本单元要把“看见工具名”与“真的执行工具”拆开
- 本单元先不展开 JSON Schema、MCP 协议、错误回退和评测细节，那些会在后面的单元回到台面

## What The Old Page Already Covers

旧页已经有一条可用的教学骨架，核心内容没有偏：

- 工具名是路由键，不是展示标题
- 描述负责告诉模型什么时候该用、什么时候不该用
- 参数 schema 负责把模型要填写的内容收紧
- 宿主程序要把调用候选变成受控执行，不能直接信任模型输出
- 返回结果要服务下一步推理，不能只写一句“完成”
- 一个日历创建例子可以把描述、选择、确认和结果回填串起来
- 一个订单/政策工具组可以把“查询、写入、拒绝”三类动作分开看

## What To Keep

本单元重构时要保留的不是旧 HTML 壳，而是旧页的教学顺序：

1. 先看工具名和描述为什么会影响选择
2. 再看参数 schema 怎样把输入收紧
3. 然后把调用候选交给宿主程序做校验、确认、权限和分发
4. 最后看结果回填怎样支持下一步
