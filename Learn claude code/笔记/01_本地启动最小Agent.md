> [!abstract] 核心主线
> 本节的主线是把 `learn-claude-code` 的最小 Agent 在本地跑起来。用户输入任务后，本地程序把消息发给模型；模型决定下一步是回复文本还是请求工具；本地程序执行允许的工具，并把结果写回上下文。Hello 示例不是为了学会打印字符串，而是为了看到 Agent 从“生成文本”走向“在本地造成可检查的副作用”。

## 概念精讲

### Agent 不是模型本身

普通聊天模型只生成文本。Agent 多了一层本地运行时：它负责维护消息历史、提供工具定义、执行工具调用、收集结果，并决定是否继续下一轮。模型仍然重要，但真正创建文件、执行命令、读取目录的是本地程序。

判断句：看到 Agent “创建文件”时，不要说“模型直接改了磁盘”，应该说“模型提出工具调用，本地程序执行了这个调用”。

### 三个配置项

| 配置项 | 准确含义 | 错了会怎样 |
|---|---|---|
| API Key | 调用模型服务的身份凭证 | 常见 401、unauthorized、鉴权失败 |
| Base URL | 兼容接口的请求地址 | 请求发错地方，连接失败或命中网页 |
| Model | 实际调用的模型标识 | model not found 或 bad request |

> [!tip] 兼容接口的意思
> “Anthropic 兼容接口”说的是请求协议兼容，不是服务商只能是 Anthropic。只要平台提供兼容接口，就可以按项目要求配置 Key、Base URL 和 Model。

### 当前工作目录会影响结果

Agent 在本地创建文件时通常使用相对路径。你在哪个目录启动脚本，生成文件就可能落在哪个目录下。第一节做 Hello 示例时，不只要看终端输出，还要确认 `hello.py` 出现在哪里，并手动运行一次。

## 最小可操作样例

```bash
cd learn-claude-code
python --version
python s01_agent_loop.py
```

输入任务：

```text
实现一个 hello.py 文件，内容是打印 Hello World，并运行它。
```

复验：

```bash
ls
python hello.py
```

如果找不到文件：

```bash
find . -name "hello.py"
```

## 关键术语

| 术语 | 含义 | 容易混淆点 |
|---|---|---|
| Agent | 模型加本地运行时和工具系统 | 不等于单独的大模型 |
| Tool use | 模型请求本地执行某个工具 | 请求不等于已经执行 |
| Tool result | 本地工具执行后的结果 | 必须回写给模型，后续才能继续判断 |
| Base URL | API 请求入口 | 不是模型名，也不是控制台网页 |
| 工作目录 | 命令当前所在目录 | 会影响相对路径和生成文件位置 |

## 应用与迁移问答

> [!question] 换一个模型平台时要改什么？
> 重新确认 API Key、Base URL 和 Model。不要只换 Key，因为不同平台的接口地址和模型名也可能不同。

> [!question] 为什么作者从 PowerShell 切到 Git Bash？
> Shell 会影响命令写法、环境变量设置方式和路径表现。切换 shell 不是核心知识点，但提醒你排查问题时要把 shell 差异算进去。

> [!question] 为什么第一节不急着讲工具调用源码？
> 第一节先建立入口：本地程序能启动，能连接模型，能通过最小任务产生可检查的文件。工具调用的细节在后续消息循环和工具章节再拆。

## 练习 / 盲区复盘

> [!failure] 容易卡住的地方
> - 把 Base URL 和 Model 写反。
> - 在错误目录启动脚本，找不到生成文件。
> - 只看 Agent 输出“完成”，没有检查真实文件。
> - 以为兼容 Anthropic 接口就只能使用 Anthropic 官方服务。
> - 跑通 Hello 后直接跳过 S01，导致后面读消息循环时没有入口感。

## 费曼解释润色版

> [!quote]
> 最小 Agent 的工作方式可以这样讲：用户在命令行输入一个任务，本地 Agent 把这条消息发给模型。模型不会直接修改电脑，它只会返回文本，或者请求调用某个工具。本地程序收到工具请求后，在当前工作目录里执行允许的动作，比如创建 `hello.py` 并运行它。执行结果再被写回对话，模型才能知道刚才做得对不对。验收这个示例时，不能只看终端说成功，还要到目录里检查文件内容，并用普通 Python 命令手动运行一次。

## 易错卡片

> [!warning] 误区：模型直接创建了文件
> 正确说法：模型提出工具调用，本地程序执行后才会产生文件。

> [!warning] 误区：Key 对了就一定能跑
> 正确说法：Key、Base URL、Model 三项都要和当前服务商匹配。

> [!warning] 误区：Hello 示例太简单，可以跳过
> 正确说法：Hello 示例包含消息、模型、工具、本地副作用和复验，是后续源码阅读的入口。

> [!warning] 误区：终端输出成功就是验收完成
> 正确说法：还要检查生成文件的位置、内容和手动运行结果。
