线性模型：从假设函数到损失曲面

这一页先把线性模型的最小闭环钉住：输入 x 经过 w 和 b 变成预测 \hat y，预测和标签的差先变成残差，再压成一个标量损失。你会看到同一个小 batch 在“前向、算 loss、看曲面”三个位置上分别长什么样。梯度下降放到下一讲，这一页只负责把假设函数、损失函数和偏置项看明白。

学习目标

先定边界

这页先解决什么

把输入、参数、预测、残差和损失串起来；看见一张参数表时，知道哪个位置是模型，哪个位置是目标，哪个位置只是教学用扫描。

先别追的东西

梯度怎么推出来、学习率怎么选、为什么会走到局部最小，这些放到下一讲。这里先把损失曲面看成地图，不急着学走法。

目标 读到什么算懂了 能马上检查什么
假设函数 知道 y_hat = wx + b 里谁是输入，谁是参数。 看见一行数据就能写出最小模型。

概念骨架

先看整条线

这页的对象很少，关系却很紧。输入 x 先进入线性模型，模型给出预测 \hat y，预测和标签 y 的差变成残差 e，残差再压成损失 \mathcal{L}。只要你把这条线看清楚，后面读训练脚本时就知道该盯哪一层。

对象 角色 从哪里来 下一步去哪
x 输入 数据集或 batch 进入前向计算
w, b 参数 初始化或上一步更新后 决定预测的斜率和截距
\hat y 预测 模型前向 进入 loss
e 残差 \hat y - y 进入平方和平均
\mathcal{L} 损失 残差汇总 给下一讲的更新方向提供起点

核心机制

把公式拆开

线性模型做的事很直接：把输入和参数拼成预测，再把预测和标签的差压成一个能比较的数。平方误差的作用也很直接，正负误差不会互相抵消，大误差会被放得更重。

$$ \hat y_i = w x_i + b,\qquad e_i = \hat y_i - y_i,\qquad \mathcal{L} = \frac{1}{n}\sum_{i=1}^{n} e_i^2 $$

\hat y_i 是第 i 个样本的预测，e_i 是残差，\mathcal{L} 是均方误差。这里的 n 是当前 batch 里的样本数。

步骤 读什么 写什么 错了会看到什么
前向 x、w、b \hat y 输出和输入维度对不上，或预测方向不对
残差 \hat y、y e 把预测误差符号看反，后面判断全偏
loss e 标量 \mathcal{L} loss 还留着向量形状，后面不好接 backward
曲面 w、b、\mathcal{L} 参数空间里的高低关系 把扫描当训练，误以为枚举就是优化

手算追踪

同一个 batch 往下走

下面这组样本会在整页里重复出现：x=[1,2,3]，y=[2,4,6]。你先拿 w=1,b=0 看一次，再把参数换成 w=2,b=0，最后补一个 b=1 的情况。这样可以直接看出斜率和截距分别在改什么。

数据批次 x, y

读自数据集或 DataLoader。这里先只看同一个 batch，别把样本换来换去。

预测 \hat y

由当前参数算出。它只负责输出，不负责修正自己。

标量 loss \mathcal{L}

把一批样本的残差压成一个数，后面才能比较和优化。

损失曲面 参数空间

把 w 和 b 当坐标，就能看见高低起伏和低谷位置。

同一批样本的手算流程python
x = [1, 2, 3]
y = [2, 4, 6]

def predict(w, b):
    return [w * xi + b for xi in x]

pred_10 = predict(1, 0)
pred_20 = predict(2, 0)
pred_21 = predict(2, 1)
参数 预测 残差 MSE 读到的结论
w=1,b=0 [1,2,3] [-1,-2,-3] 14/3 斜率还没对上。
w=2,b=0 [2,4,6] [0,0,0] 0 这批数据正好落在这条线。
w=2,b=1 [3,5,7] [1,1,1] 1 截距往上平移了整条线。

偏置项的作用

看见截距

如果只有 y=wx，模型就会被锁在过原点的直线上。对 y=2x+1 这种关系来说，光调 w 不够，b 必须能把整条线整体抬高一格。

输入 x w=2,b=0 w=2,b=1 差别
0 0 1 只有带偏置时，x=0 的位置才不会被锁死在 0。
1 2 3 偏置把整条线整体上移。
2 4 5 斜率不变时，截距直接改输出的基线。

这件事和模型容量有关

线性模型的表达能力有限。数据如果呈现弯曲关系，或者需要更复杂的形状，单靠加样本不会让它自动变强。它能学到的，只是这条线能覆盖的那一部分规律。

损失曲面

把参数当坐标

把 w 和 b 当成平面上的坐标，\mathcal{L}(w,b) 就是一张地形图。高处说明这组参数让误差更大，低谷说明这组参数更接近目标。课程里画扫描图，是为了让你先看见地形，再学怎么往低处走。

$$ \mathcal{L}(w,b) = \frac{1}{n}\sum_{i=1}^{n}(wx_i + b - y_i)^2 $$

这里 w 和 b 是自变量，x_i 和 y_i 来自当前 batch。把它们代进去以后，\mathcal{L} 就给出这组参数对应的高度。

先看切片，再看整张图

读这类曲面时，常先固定一个参数，只看另一条切片。比如先把 b 固定在 0，看 w 从 1 到 2 时 loss 怎么掉下来；再把 w 固定在 2，看 b 从 0 到 1 时 loss 为什么又抬高。这样你会先看清斜率方向上的变化，再看截距方向上的平移。多维参数里，机器不会替你把两条方向分开解释，人脑先分开看，才更容易判断是谁在拉高误差。没有这一步，参数扫描图很容易被当成一张只会报高低的装饰图。

b \\ w w=0 w=1 w=2
b=0 56/3 14/3 0
b=1 35/3 5/3 1
方式 怎么做 代价 在这页里的角色
参数扫描 把几个候选 w,b 都算一遍 参数多时会很快变慢 帮助你看见损失曲面
梯度下降 沿着 loss 下降的方向一步一步改 需要下一讲里的导数和学习率 是真正的训练方法

为什么要平方

误差方向先别抵消

如果直接把残差相加，正负误差会互相抵消，最后你可能看见一个很小的数，却不知道模型其实偏得很厉害。把残差平方后再平均，就能保住误差大小，也能让大偏差被更重地惩罚。

处理方式 会发生什么 适合什么场景 这页为什么选它
直接相加 正负误差可能抵消 不适合当训练目标 会把明显的偏差掩盖掉
取绝对值 不会抵消，但在 0 附近不如平方顺手 有些稳健统计会用 这页不选它，只当对照
平方后平均 大误差被放大，小误差更平滑 回归入门和线性模型最常见 最容易和损失曲面连起来

一个很容易漏掉的错觉

如果你只看“总误差接近 0”，却不看每个样本的残差方向，就会误以为模型已经很好。实际上，可能只是正负抵消了。平方误差把这个错觉先挡住，让训练信号更稳定。

同一组残差的三种处理python
residuals = [-1, 2, -3]

sum_raw = sum(residuals)
sum_abs = sum(abs(e) for e in residuals)
sum_square = sum(e * e for e in residuals)

print(sum_raw, sum_abs, sum_square)
处理结果 数值 你会怎么读
直接相加 -2 看不出正负抵消后到底有多歪。
绝对值求和 6 能保住大小，但不如平方和那样顺着曲面分析。
平方求和 14 大误差被放大，适合拿来继续做优化。

为什么先用 MSE

回归任务的第一把尺子

这里先用 MSE。它把连续值误差变成一个稳定、可比较的标量。房价、温度、长度这类任务里，更关心的是离目标差了多少。MSE 让这个距离直接进入训练信号。

任务 输出长什么样 更自然的 loss 为什么
房价预测 连续数值 MSE / 其他回归损失 关注预测值离真实值有多远。
二分类 类别概率 交叉熵一类损失 关注类别判断是否对、概率是否合理。
线性回归入门 单个连续输出 MSE 误差结构最直观，最容易和参数曲面连起来。

别把“损失小”直接理解成“任务都对了”

在回归里，loss 小只能说明预测接近标签，不能说明模型已经理解了真实规律。它可能只是刚好贴上了这批数据。下一讲学梯度下降时，你会继续看到这一点：优化的是损失，不是“理解”。

同一个 batch 用不同 loss 的语义差text

回归视角：看预测值离标签有多远
分类视角：先问这组输出是不是一个类别标签
线性模型入门：先把距离问题看清，再谈更新方向

标签尺度先检查

先看单位，再看 loss

线性模型看起来简单，但数值尺度一乱，训练行为就会很怪。比如房价单位是万元，标签却写成元，预测差距会被放大一百倍；或者标签本来就很大，平方后 loss 会一下子涨得很快。这个时候先查尺度，不要先怀疑模型突然坏了。

检查项 你要看什么 如果不对，症状是什么
标签单位 元、万元、米、厘米有没有混用 loss 远大于预期，参数更新很躁
输入范围 x 是否比标签大很多倍 预测一开始就飘得很远
输出范围 模型初始预测是否和标签同量级 曲线能动，但像在另一个尺度上跑

这里的修法先别复杂化

尺度问题先别急着上正则化、换优化器或者换更深的网络。先把标签和输入的单位对齐，再看 loss 是否回到合理区间。很多“训练不稳”其实是输入预处理和标签单位出了错。

打印一轮最该看的数python
print("x min/max:", x.min().item(), x.max().item())
print("y min/max:", y.min().item(), y.max().item())
print("pred min/max:", pred.min().item(), pred.max().item())
print("loss:", loss.item())
打印结果 你先怀疑什么 下一步怎么查
loss 大得离谱 单位混了，标签尺度偏了 回到数据预处理和标签来源
pred 和 y 差几个数量级 初始化和尺度不匹配 先检查输入范围，再看学习率
loss 小但结果怪 只贴住了训练集，泛化还没看 把验证集一起打印

PyTorch 对应物

把公式搬到代码里

nn.Linear(1, 1) 本质上就是一个可学习的 wx+b。在 PyTorch 里，它管理的是参数张量和前向关系，公式只是它的外层写法。当前 batch 要先整理成 [batch, feature] 的形状，这样模型才能按列读输入。

线性模型的一轮 PyTorch 训练python
import torch
from torch import nn

x = torch.tensor([[1.0], [2.0], [3.0]])
y = torch.tensor([[2.0], [4.0], [6.0]])

model = nn.Linear(1, 1)
criterion = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

pred = model(x)
loss = criterion(pred, y)
optimizer.zero_grad()
loss.backward()
optimizer.step()
对象 读什么 写什么 如果用错会怎样
nn.Linear x 和当前参数 \hat y 输入维度不对时，前向直接卡住
nn.MSELoss pred 和 y 标量 loss loss 形状不对时，后面接不上 backward
SGD 参数上的梯度 更新后的参数 忘了 zero_grad() 会把上一轮梯度带进来
optimizer.zero_grad() 历史梯度 清掉上一轮残留 参数会被旧梯度污染，方向不干净

形状先看这一条

这里的 x 要写成 [3,1]。对 nn.Linear(1,1) 来说，最后一维是特征维，前面的维度是 batch 维。这个分法会一直沿用到后面的多维特征和更复杂网络。

读代码顺序

先找对象，再找顺序

拿到一段陌生的线性回归代码时，不要先盯优化器参数。先把这五个对象找出来：输入 x、标签 y、模型 model、损失函数 criterion、优化器 optimizer。它们一旦排对顺序，很多看起来很像的 bug 其实都能很快定位。

先找什么 判断它的作用 常见误读 正确检查点
x / y 数据从哪里来 把标签当输入，或把输入当标签 确认 x 和 y 的形状、dtype 和批次维度
model 谁负责前向 把前向和训练混在一起 看 model(x) 读了什么、写了什么
criterion 谁负责评价 把 loss 当成预测本身 看它是否把预测和标签压成标量
optimizer 谁负责更新 以为 backward 会直接改参数 确认 zero_grad -> backward -> step 的顺序
一段可读的最小训练脚本python
for x, y in loader:
    pred = model(x)
    loss = criterion(pred, y)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

如果顺序错了，会看到什么

如果 zero_grad() 放晚了，上一轮的梯度会继续留着；如果 step() 放在 backward() 前面，优化器拿到的是空梯度；如果把 criterion 写成了分类损失但标签还是连续值，loss 数字可能还能出来，方向却已经不对了。很多训练曲线“看起来在动”，其实只是状态没有完全坏掉。

代码行 读取 写入 / 变更 如果错位
pred = model(x) 输入和参数 预测 输出和输入维度不匹配
loss = criterion(pred, y) 预测和标签 标量 loss 标签类型错，loss 语义错
loss.backward() loss 参数 grad 图断了，grad 为空
optimizer.step() 梯度 更新参数 参数不动，训练像假跑

故障实验室

把症状和修法对上

下面这三个故障都能在线性模型里直接看见。它们分别对应输入、loss 语义和优化器顺序。你遇到问题时，不要先大改结构，先把这三个故障按症状排一下。

RuntimeError: mat1 and mat2 shapes cannot be multiplied
先看 `x` 有没有按 `[batch, feature]` 组织，再看 `nn.Linear` 的输入维度。

矩阵乘法报错 输入形状 先看 x.shape 和 nn.Linear 的输入维度
loss 数字能出来但不对 任务类型或标签语义 先确认是回归还是分类，再看 loss 选型

如果你已经在写 PyTorch 作业，第一步是打印一轮 x.shape、y.shape、pred.shape 和 loss。这个顺序能把大多数假问题先筛掉。

训练与验证

别只看训练集

线性模型在训练集上看起来很好，不代表它在新数据上也一样好。训练集 loss 低，只说明参数贴住了当前 batch；验证集 loss 低，才说明这条线在没见过的数据上也还能站住。这里最常见的误判，就是把训练集的下降当成整体进步。

位置 看什么 说明什么 错看会怎样
训练集 当前 batch 的 loss 参数是否贴住已见样本 只看到下降，会误以为模型已经学会泛化
验证集 没见过的样本 loss 这条线能不能迁移到新数据 如果一直不看，过拟合会被拖到后面才发现
训练和验证各看一次python
train_pred = model(train_x)
train_loss = criterion(train_pred, train_y)

val_pred = model(val_x)
val_loss = criterion(val_pred, val_y)

这个差别为什么重要

如果训练集 loss 一直降，验证集却很高，说明模型可能只是记住了这批数据的局部形状。线性模型容量低，这种情况不一定会特别严重，但它仍然会发生，尤其是数据尺度、噪声或标签质量有问题的时候。

现象 先怎么读 下一步怎么做
训练 loss 降，验证 loss 不降 先怀疑过拟合或数据分布不一致 回看标签尺度、噪声和划分方式
两个 loss 都很高 先怀疑模型太简单或 loss 选错 回到本页的损失选择和偏置项

练习时要怎么用这个区分

做完这页的题，不要只看自己会不会算 w=2,b=0 的 MSE。再问自己一次：如果换成验证集，这条线还会不会一样好。这个问题会直接把你往下一讲的优化行为推过去。

常见误区

看症状就能回查

下面这些问题都和这页的主线直接有关。看到症状时，先按表里的方向回去看，不要先换模型、换优化器或者把学习率乱调一通。

误区 现场症状 立刻修法
把 MSE 当分类准确率 连续值任务被说成“对不对” 先确认任务是回归还是分类，再选损失。
漏掉偏置 b 模型被固定在过原点的直线上 先看数据是否需要截距，再决定是否保留 b。
把参数扫描当训练 在小网格里找到低谷就停下了 把扫描当演示，不要把它当成真正的优化算法。
只看训练集 loss 训练集很好看，验证集却很差 把验证集一起看，先分清欠拟合还是过拟合。
把 loss.backward() 当成更新参数 以为 backward 结束就训练完了 记住 backward 只写梯度，真正改参数的是 optimizer.step()。
