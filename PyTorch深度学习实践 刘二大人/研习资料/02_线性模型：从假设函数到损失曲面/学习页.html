线性模型：从假设函数到损失曲面

这一页先把线性模型的最小闭环钉住：输入 x 经过 w 和 b 变成预测 \hat y ，预测和标签的差先变成残差，再压成一个标量损失。你会看到同一个小 batch 在“前向、算 loss、看曲面”三个位置上分别长什么样。梯度下降放到下一讲，这一页只负责把假设函数、损失函数和偏置项看明白。

学习目标

先定边界

这页先解决什么

把输入、参数、预测、残差和损失串起来；看见一张参数表时，知道哪个位置是模型，哪个位置是目标，哪个位置只是教学用扫描。

先别追的东西

梯度怎么推出来、学习率怎么选、为什么会走到局部最小，这些放到下一讲。这里先把损失曲面看成地图，不急着学走法。

目标读到什么算懂了能马上检查什么
假设函数知道 y_hat = wx + b 里谁是输入，谁是参数。看见一行数据就能写出最小模型。
损失函数知道残差如何变成一个标量。能说明为什么 loss 必须压成单个数。
参数曲面知道 w 和 b 变了，loss 也会跟着变。能区分教学用扫描和真正训练。

概念骨架

先看整条线

这页的对象很少，关系却很紧。输入 x 先进入线性模型，模型给出预测 \hat y，预测和标签 y 的差变成残差 e，残差再压成损失 \mathcal{L}。只要你把这条线看清楚，后面读训练脚本时就知道该盯哪一层。

核心机制

把公式拆开

线性模型做的事很直接：把输入和参数拼成预测，再把预测和标签的差压成一个能比较的数。平方误差的作用也很直接，正负误差不会互相抵消，大误差会被放得更重。

\hat y_i 是第 i 个样本的预测，e_i 是残差，\mathcal{L} 是均方误差。这里的 n 是当前 batch 里的样本数。

手算追踪

def predict(w, b):

pred_10 = predict(1, 0)

损失曲面

把参数当坐标

把 w 和 b 当成平面上的坐标，\mathcal{L}(w,b) 就是一张地形图。高处说明这组参数让误差更大，低谷说明这组参数更接近目标。课程里画扫描图，是为了让你先看见地形，再学怎么往低处走。

这里 w 和 b 是自变量，x_i 和 y_i 来自当前 batch。把它们代进去以后，\mathcal{L} 就给出这组参数对应的高度。

为什么要平方

误差方向先别抵消

如果直接把残差相加，正负误差会互相抵消，最后你可能看见一个很小的数，却不知道模型其实偏得很厉害。把残差平方后再平均，就能保住误差大小，也能让大偏差被更重地惩罚。

一个很容易漏掉的错觉

如果你只看“总误差接近 0”，却不看每个样本的残差方向，就会误以为模型已经很好。实际上，可能只是正负抵消了。平方误差把这个错觉先挡住，让训练信号更稳定。

同一组残差的三种处理python

residuals = [-1, 2, -3]

sum_raw = sum(residuals)
sum_abs = sum(abs(e) for e in residuals)
sum_square = sum(e * e for e in residuals)

print(sum_raw, sum_abs, sum_square)
处理结果数值你会怎么读
直接相加-2看不出正负抵消后到底有多歪。
绝对值求和6能保住大小，但不如平方和那样顺着曲面分析。
平方求和14大误差被放大，适合拿来继续做优化。

为什么先用 MSE

回归任务的第一把尺子

这里先用 MSE。它把连续值误差变成一个稳定、可比较的标量。房价、温度、长度这类任务里，更关心的是离目标差了多少。MSE 让这个距离直接进入训练信号。

别把“损失小”直接理解成“任务都对了”

在回归里，loss 小只能说明预测接近标签，不能说明模型已经理解了真实规律。它可能只是刚好贴上了这批数据。下一讲学梯度下降时，你会继续看到这一点：优化的是损失，不是“理解”。

标签尺度先检查

先看单位，再看

线性模型看起来简单，但数值尺度一乱，训练行为就会很怪。比如房价单位是万元，标签却写成元，预测差距会被放大一百倍；或者标签本来就很大，平方后 loss 会一下子涨得很快。这个时候先查尺度，不要先怀疑模型突然坏了。

标签单位

元、万元、米、厘米有没有混用

loss 远大于预期，参数更新很躁

这里的修法先别复杂化

尺度问题先别急着上正则化、换优化器或者换更深的网络。先把标签和输入的单位对齐，再看 loss 是否回到合理区间。很多“训练不稳”其实是输入预处理和标签单位出了错。

打印一轮最该看的数python

print("x min/max:", x.min().item(), x.max().item())
print("y min/max:", y.min().item(), y.max().item())
print("pred min/max:", pred.min().item(), pred.max().item())
print("loss:", loss.item())
打印结果你先怀疑什么下一步怎么查
loss 大得离谱单位混了，标签尺度偏了回到数据预处理和标签来源
pred 和 y 差几个数量级初始化和尺度不匹配先检查输入范围，再看学习率
loss 小但结果怪只贴住了训练集，泛化还没看把验证集一起打印

PyTorch 对应物

把公式搬到代码里

nn.Linear(1, 1) 本质上就是一个可学习的 wx+b。在 PyTorch 里，它管理的是参数张量和前向关系，公式只是它的外层写法。当前 batch 要先整理成 [batch, feature] 的形状，这样模型才能按列读输入。

代码
import torch
from torch import nn

x = torch.tensor([[1.0], [2.0], [3.0]])
y = torch.tensor([[2.0], [4.0], [6.0]])

model = nn.Linear(1, 1)
criterion = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

pred = model(x)
loss = criterion(pred, y)
optimizer.zero_grad()
loss.backward()
optimizer.step()

形状先看这一条

这里的 x 要写成 [3,1]。对 nn.Linear(1,1) 来说，最后一维是特征维，前面的维度是 batch 维。这个分法会一直沿用到后面的多维特征和更复杂网络。

读代码顺序

先找对象，再找顺序

拿到一段陌生的线性回归代码时，不要先盯优化器参数。先把这五个对象找出来：输入 x、标签 y、模型 model、损失函数 criterion、优化器 optimizer。它们一旦排对顺序，很多看起来很像的 bug 其实都能很快定位。

故障实验室

把症状和修法对上

下面这三个故障都能在线性模型里直接看见。它们分别对应输入、loss 语义和优化器顺序。你遇到问题时，不要先大改结构，先把这三个故障按症状排一下。

这里的操作桥

如果你已经在写 PyTorch 作业，第一步是打印一轮 x.shape、y.shape、pred.shape 和 loss。这个顺序能把大多数假问题先筛掉。

训练与验证

别只看训练集

线性模型在训练集上看起来很好，不代表它在新数据上也一样好。训练集 loss 低，只说明参数贴住了当前 batch；验证集 loss 低，才说明这条线在没见过的数据上也还能站住。这里最常见的误判，就是把训练集的下降当成整体进步。

代码
train_pred = model(train_x)
train_loss = criterion(train_pred, train_y)

val_pred = model(val_x)
val_loss = criterion(val_pred, val_y)

练习时要怎么用这个区分

做完这页的题，不要只看自己会不会算 w=2,b=0 的 MSE。再问自己一次：如果换成验证集，这条线还会不会一样好。这个问题会直接把你往下一讲的优化行为推过去。

23_comparecomparison

Q: 23. 对比“直接相加、绝对值、平方”三种处理，哪一种最适合这页的回归入门？为什么？

参考答案 / 评分要点

。最适合的是平方后平均，因为它不会让正负误差互相抵消，还会把大偏差放大，让优化信号更明显。直接相加会把方向抵消掉，无法真实反映偏差；绝对值虽然也能保住大小，但这页要和损失曲面、梯度下降继续衔接，平方更方便后面理解“曲面高低”和“下降方向”。
