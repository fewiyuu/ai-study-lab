# 核心机制 {#mechanism}
<!-- tag: 把公式拆开 -->

线性模型做的事很直接：把输入和参数拼成预测，再把预测和标签的差压成一个能比较的数。平方误差的作用也很直接，正负误差不会互相抵消，大误差会被放得更重。

$$
 \hat y_i = w x_i + b,\qquad
 e_i = \hat y_i - y_i,\qquad
 \mathcal{L} = \frac{1}{n}\sum_{i=1}^{n} e_i^2
 $$
 `\hat y_i` 是第 `i` 个样本的预测，`e_i` 是残差，`\mathcal{L}` 是均方误差。这里的 `n` 是当前 batch 里的样本数。

错了会看到什么

`\hat y`

把预测误差符号看反，后面判断全偏

参数空间里的高低关系

# 损失曲面 {#surface}
<!-- tag: 把参数当坐标 -->

把 `w` 和 `b` 当成平面上的坐标，`\mathcal{L}(w,b)` 就是一张地形图。高处说明这组参数让误差更大，低谷说明这组参数更接近目标。课程里画扫描图，是为了让你先看见地形，再学怎么往低处走。

$$
 \mathcal{L}(w,b) = \frac{1}{n}\sum_{i=1}^{n}(wx_i + b - y_i)^2
 $$
 这里 `w` 和 `b` 是自变量，`x_i` 和 `y_i` 来自当前 batch。把它们代进去以后，`\mathcal{L}` 就给出这组参数对应的高度。

## box info
### 先看切片，再看整张图
读这类曲面时，常先固定一个参数，只看另一条切片。比如先把 `b` 固定在 `0`，看 `w` 从 `1` 到 `2` 时 loss 怎么掉下来；再把 `w` 固定在 `2`，看 `b` 从 `0` 到 `1` 时 loss 为什么又抬高。这样你会先看清斜率方向上的变化，再看截距方向上的平移。多维参数里，机器不会替你把两条方向分开解释，人脑先分开看，才更容易判断是谁在拉高误差。没有这一步，参数扫描图很容易被当成一张只会报高低的装饰图。

# 为什么要平方 {#squared-error}
<!-- tag: 误差方向先别抵消 -->

如果直接把残差相加，正负误差会互相抵消，最后你可能看见一个很小的数，却不知道模型其实偏得很厉害。把残差平方后再平均，就能保住误差大小，也能让大偏差被更重地惩罚。

处理方式
会发生什么
适合什么场景
这页为什么选它


直接相加
正负误差可能抵消
不适合当训练目标
会把明显的偏差掩盖掉


取绝对值
不会抵消，但在 0 附近不如平方顺手
有些稳健统计会用
这页不选它，只当对照


平方后平均
大误差被放大，小误差更平滑
回归入门和线性模型最常见
最容易和损失曲面连起来

## box warn
### 一个很容易漏掉的错觉
如果你只看“总误差接近 0”，却不看每个样本的残差方向，就会误以为模型已经很好。实际上，可能只是正负抵消了。平方误差把这个错觉先挡住，让训练信号更稳定。

同一组残差的三种处理python

```python
residuals = [-1, 2, -3]

sum_raw = sum(residuals)
sum_abs = sum(abs(e) for e in residuals)
sum_square = sum(e * e for e in residuals)

print(sum_raw, sum_abs, sum_square)
```

处理结果
数值
你会怎么读


直接相加
`-2`
看不出正负抵消后到底有多歪。


绝对值求和
`6`
能保住大小，但不如平方和那样顺着曲面分析。


平方求和
`14`
大误差被放大，适合拿来继续做优化。

# 为什么先用 MSE {#loss-choice}
<!-- tag: 回归任务的第一把尺子 -->

这里先用 MSE。它把连续值误差变成一个稳定、可比较的标量。房价、温度、长度这类任务里，更关心的是离目标差了多少。MSE 让这个距离直接进入训练信号。

任务
输出长什么样
更自然的 loss
为什么


房价预测
连续数值
MSE / 其他回归损失
关注预测值离真实值有多远。


二分类
类别概率
交叉熵一类损失
关注类别判断是否对、概率是否合理。


线性回归入门
单个连续输出
MSE
误差结构最直观，最容易和参数曲面连起来。

## box info
### 别把“损失小”直接理解成“任务都对了”
在回归里，loss 小只能说明预测接近标签，不能说明模型已经理解了真实规律。它可能只是刚好贴上了这批数据。下一讲学梯度下降时，你会继续看到这一点：优化的是损失，不是“理解”。

# 标签尺度先检查 {#scale-check}
<!-- tag: 先看单位，再看 loss -->

线性模型看起来简单，但数值尺度一乱，训练行为就会很怪。比如房价单位是万元，标签却写成元，预测差距会被放大一百倍；或者标签本来就很大，平方后 loss 会一下子涨得很快。这个时候先查尺度，不要先怀疑模型突然坏了。

检查项
你要看什么
如果不对，症状是什么


标签单位
元、万元、米、厘米有没有混用
loss 远大于预期，参数更新很躁


输入范围
`x` 是否比标签大很多倍
预测一开始就飘得很远


输出范围
模型初始预测是否和标签同量级
曲线能动，但像在另一个尺度上跑

## box warn
### 这里的修法先别复杂化
尺度问题先别急着上正则化、换优化器或者换更深的网络。先把标签和输入的单位对齐，再看 `loss` 是否回到合理区间。很多“训练不稳”其实是输入预处理和标签单位出了错。

打印一轮最该看的数python

```python
print("x min/max:", x.min().item(), x.max().item())
print("y min/max:", y.min().item(), y.max().item())
print("pred min/max:", pred.min().item(), pred.max().item())
print("loss:", loss.item())
```

打印结果
你先怀疑什么
下一步怎么查


`loss` 大得离谱
单位混了，标签尺度偏了
回到数据预处理和标签来源


`pred` 和 `y` 差几个数量级
初始化和尺度不匹配
先检查输入范围，再看学习率


`loss` 小但结果怪
只贴住了训练集，泛化还没看
把验证集一起打印

# PyTorch 对应物 {#bridge}
<!-- tag: 把公式搬到代码里 -->

`nn.Linear(1, 1)` 本质上就是一个可学习的 `wx+b`。在 PyTorch 里，它管理的是参数张量和前向关系，公式只是它的外层写法。当前 batch 要先整理成 `[batch, feature]` 的形状，这样模型才能按列读输入。

线性模型的一轮 PyTorch 训练python

```python
import torch
from torch import nn

x = torch.tensor([[1.0], [2.0], [3.0]])
y = torch.tensor([[2.0], [4.0], [6.0]])

model = nn.Linear(1, 1)
criterion = nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.1)

pred = model(x)
loss = criterion(pred, y)
optimizer.zero_grad()
loss.backward()
optimizer.step()
```

对象
读什么
写什么
如果用错会怎样


`nn.Linear`
`x` 和当前参数
`\hat y`
输入维度不对时，前向直接卡住


`nn.MSELoss`
`pred` 和 `y`
标量 loss
loss 形状不对时，后面接不上 backward


`SGD`
参数上的梯度
更新后的参数
忘了 `zero_grad()` 会把上一轮梯度带进来


`optimizer.zero_grad()`
历史梯度
清掉上一轮残留
参数会被旧梯度污染，方向不干净

## box info
### 形状先看这一条
这里的 `x` 要写成 `[3,1]`。对 `nn.Linear(1,1)` 来说，最后一维是特征维，前面的维度是 batch 维。这个分法会一直沿用到后面的多维特征和更复杂网络。

<!-- tag: 先找对象，再找顺序 -->

拿到一段陌生的线性回归代码时，不要先盯优化器参数。先把这五个对象找出来：输入 `x`、标签 `y`、模型 `model`、损失函数 `criterion`、优化器 `optimizer`。它们一旦排对顺序，很多看起来很像的 bug 其实都能很快定位。


## box warn
### 如果顺序错了，会看到什么

# 训练与验证 {#generalization}
<!-- tag: 别只看训练集 -->

线性模型在训练集上看起来很好，不代表它在新数据上也一样好。训练集 loss 低，只说明参数贴住了当前 batch；验证集 loss 低，才说明这条线在没见过的数据上也还能站住。这里最常见的误判，就是把训练集的下降当成整体进步。

位置
看什么
说明什么
错看会怎样


训练集
当前 batch 的 loss
参数是否贴住已见样本
只看到下降，会误以为模型已经学会泛化


验证集
没见过的样本 loss
这条线能不能迁移到新数据
如果一直不看，过拟合会被拖到后面才发现

训练和验证各看一次python

```python
train_pred = model(train_x)
train_loss = criterion(train_pred, train_y)

val_pred = model(val_x)
val_loss = criterion(val_pred, val_y)
```

## box warn
### 这个差别为什么重要
如果训练集 loss 一直降，验证集却很高，说明模型可能只是记住了这批数据的局部形状。线性模型容量低，这种情况不一定会特别严重，但它仍然会发生，尤其是数据尺度、噪声或标签质量有问题的时候。

现象
先怎么读
下一步怎么做


训练 loss 降，验证 loss 不降
先怀疑过拟合或数据分布不一致
回看标签尺度、噪声和划分方式


两个 loss 都很高
先怀疑模型太简单或 loss 选错
回到本页的损失选择和偏置项

# 常见误区 {#mistakes}
<!-- tag: 看症状就能回查 -->

下面这些问题都和这页的主线直接有关。看到症状时，先按表里的方向回去看，不要先换模型、换优化器或者把学习率乱调一通。

误区
现场症状
立刻修法


把 `MSE` 当分类准确率
连续值任务被说成“对不对”
先确认任务是回归还是分类，再选损失。


漏掉偏置 `b`
模型被固定在过原点的直线上
先看数据是否需要截距，再决定是否保留 `b`。


把参数扫描当训练
在小网格里找到低谷就停下了
把扫描当演示，不要把它当成真正的优化算法。


只看训练集 loss
训练集很好看，验证集却很差
把验证集一起看，先分清欠拟合还是过拟合。


把 `loss.backward()` 当成更新参数
以为 backward 结束就训练完了
记住 backward 只写梯度，真正改参数的是 `optimizer.step()`。

### 24_generalize diagnosis

**Q:** 24. 为什么训练集 loss 下降，不代表模型已经学会泛化？

Explain: 训练集 loss 下降，只能说明模型在当前见过的数据上拟合得更好，不代表它会在新数据上同样表现好。泛化要看验证集，因为验证集更接近“没见过的数据”。如果训练 loss 和验证 loss 差距越来越大，就要怀疑模型只是记住了训练样本的局部形状，而没有学到稳定规律。

### 22_square choice

**Q:** 22. 为什么残差通常要先平方再平均？

- [ ] 只是为了让数字变大
- [ ] 防止正负误差抵消，并放大大偏差
- [ ] 这样就不需要标签了

Answer: 防止正负误差抵消，并放大大偏差

### 23_compare comparison

**Q:** 23. 对比“直接相加、绝对值、平方”三种处理，哪一种最适合这页的回归入门？为什么？

Explain: 最适合的是平方后平均，因为它不会让正负误差互相抵消，还会把大偏差放大，让优化信号更明显。直接相加会把方向抵消掉，无法真实反映偏差；绝对值虽然也能保住大小，但这页要和损失曲面、梯度下降继续衔接，平方更方便后面理解“曲面高低”和“下降方向”。

### 17_debug checklist

**Q:** 17. 训练时参数不动，你会先打印哪 4 个值？

Explain: 至少应打印 x.shape 、 y.shape 、 pred.shape 、 loss.item() ，再补一条参数或梯度信息更稳，比如某个参数的 grad 是否为空。这样可以把“输入错”“loss 语义错”和“更新没发生”区分开。只说“调学习率”不算检查。

### 18_symptom diagnosis

**Q:** 18. 如果训练能跑，但参数像没动，你应该优先怀疑哪一步？

Explain: 应先怀疑 optimizer.zero_grad() 、 loss.backward() 、 optimizer.step() 的顺序，确认梯度是否真的写到了参数上，优化器是否真的读取了这些梯度。其次再看学习率是否过小、输入是否正确、loss 是否真的是标量。如果只看损失数值而不查梯度，很多“假训练”看不出来。

### 8_order sequence

**Q:** 8. 把一次训练 step 的顺序写出来，并说明每一步读什么、写什么。

Explain: 顺序应该是：前向读 x 和当前参数，写出 \hat y ；loss 读 \hat y 和 y ，写出标量 \mathcal{L} ； optimizer.zero_grad() 清掉上一轮残留梯度； loss.backward() 把梯度写到参数的 grad 上； optimizer.step() 读取梯度并更新参数。少掉任何一步，训练链路都会断。

### 11_scan synthesis

**Q:** 11. 为什么把参数扫描画成损失曲面有用，但不能直接当训练方法？

Explain: 参数扫描的价值是把 (w,b) 和 loss 的关系直接画出来，让你看见低谷和高处，知道模型为什么会朝某些方向变好或变差。它不能直接当训练方法，因为参数一多，候选组合会爆炸，完整枚举很快就不可行。真正训练要靠下一讲的梯度下降，沿着局部信息一步一步走。

- [ ] 批量读取

### 13_linear code-reading

**Q:** 13. 为什么 `nn.Linear(1,1)` 和 `y = wx + b` 是同一件事？

Explain: nn.Linear(1,1) 学的是一个输入维度到一个输出维度的仿射变换，内部对应的就是权重 w 和偏置 b 。当前 batch 要组织成 [batch, feature] ，所以 x=[1,2,3] 在代码里通常写成 [[1.0],[2.0],[3.0]] ，也就是 [3,1] 。模型输出仍然是 [3,1] ，每一行对应一个样本的预测。这样一来，公式和框架里的对象是一一对应的。

### 14_feynman synthesis

**Q:** 14. 用 6-8 句话解释：为什么这页要排在梯度下降前面？

Explain: 答案至少要说明：这页先把 \hat y = wx + b 的结构讲清楚；再把预测和标签之间的差压成 loss；然后把 loss 画成参数空间里的地形；这样到了下一讲，学习率和更新方向才有落点。还要点出 nn.Linear 、 MSELoss 和 SGD 的位置关系，否则后面的训练脚本只会像一串 API 名字。
