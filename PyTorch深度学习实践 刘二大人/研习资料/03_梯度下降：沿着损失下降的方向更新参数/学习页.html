第 03 讲 · 梯度更新入口

梯度下降：沿着损失下降的方向更新参数

这一讲只讲一件事：当梯度已经算出来以后，参数怎样沿着损失下降的方向走。我们沿用上一讲的同一组锚点样本 x=[1,2,3]、y=[2,4,6]，先看方向，再看步长，再看 batch 的折中；梯度从哪里来，留到下一讲接上。

课程地图 上一讲：线性模型 下一讲：反向传播

读完这一页你应该能做什么

  • 能用 w_{t+1} = w_t - eta * grad 解释一次参数更新。
  • 能判断学习率太小、太大时会出现什么训练症状。
  • 能区分 batch、mini-batch、SGD，并把它们接到 PyTorch 训练循环里。

先抓主线

把梯度用起来

上一讲已经把线性模型和损失曲面放在一起看了。这一讲接着回答：当你已经站在曲面上的某一点时，下一步往哪里走，走多远，按几张样本来估计方向。这里先不追梯度是怎么被算出来的，只看梯度怎样被用来更新参数。

先看方向

梯度给的是“往哪边更陡”。更新时要反着走，才会往低处去。方向错了，训练会越走越高。

再看步长

学习率控制每次迈多大步。它不改方向，只改步子大小。太小会慢，太大容易震荡。

最后看 batch

batch 选得越小，方向越吵；越大，方向越稳，但每一步也更贵。mini-batch 是常用折中。

链路位置

先看路径

这一页接在“线性模型与损失曲面”后面，只回答更新这一段：loss 已经有了，参数怎样被推向更低的位置。梯度的来源、链式法则和 backward() 放到下一讲再接。

当前点 w_t, b_t

这是更新前的参数位置。你可以把它看成损失曲面上的一个坐标。

梯度 grad / ∇L

梯度告诉你“哪边更陡”。更新时必须反着走，不然就是往更高处去。

学习率 eta

学习率只负责把步子缩小或放大，不改方向本身。

下一轮参数 w_{t+1}, b_{t+1}

真正的训练动作在这里：把当前参数沿着反方向挪到新的位置。

步骤 读什么 写什么 错一步会看到什么
当前点 当前参数和当前 loss 不改参数，只标出位置 不知道自己站在曲面的哪一块
梯度方向 当前梯度的符号和大小 决定往上还是往下走 方向写反，loss 只会越改越大
学习率 eta 缩放一次移动的距离 太小几乎不动，太大会越过低谷
参数更新 当前参数和梯度 写出下一轮参数 参数看起来在训练，实际上没变

更新公式

方向和步长分开看

梯度下降的核心公式很短：当前参数往梯度的反方向走一步，步长由学习率控制。这个“减号”不能当成装饰；它表示参数要朝 loss 下降的方向走。

$$ \theta_{t+1} = \theta_t - \eta \nabla_{\theta}\mathcal{L}(\theta_t) $$

\theta_t 是当前参数，\eta 是学习率，\nabla_{\theta}\mathcal{L} 是当前点上的梯度。把 \theta 换成 w 或 (w,b)，这条式子都还成立。

先读符号，再读动作

梯度给方向，学习率给步长，更新公式把这两件事合成一次参数移动。只要记住这三层，就不会把“方向”和“速度”混在一起。

反向走一步

如果梯度是正的，说明当前参数往大了走会让 loss 变大，那更新后就应该变小；如果梯度是负的，更新后就应该变大。

手写一轮更新python
w = 1.0
b = 0.0
grad_w = -28 / 3
grad_b = 0.0
eta = 0.1

w_next = w - eta * grad_w
b_next = b - eta * grad_b
print(w_next, b_next)
当前梯度 下一步参数应该怎样动 在代码里怎么判断
正 参数往小的方向走 w -= eta * grad 后会减小
负 参数往大的方向走 w -= eta * grad 后会增大
接近 0 先别急着大步移动 可能已经接近局部最优，或者梯度估计太弱

同一条梯度，不同 eta 会发生什么

先看一维切片

这一节把 w=1 的锚点当作固定起点，只改学习率，不改梯度方向。这样可以直接看见：学习率这个“超参数”会变成参数每次移动的实际距离。把它调小，步子短；把它调大，步子长；如果长得离谱，就会跨过低谷，跳到另一边去。

这里用的梯度仍然是上一节的切片结果：grad_w = -28/3。因为梯度是负的，所以参数应该往大的方向走。问题不在“往哪走”，而在“每次走多远”。同一条梯度线，eta=0.01、eta=0.1、eta=0.5 的结果完全不同。

eta 更新后 w 和谷底的关系 你会看到什么
0.01 1.0933... 只挪了一小步 下降会很稳，但可能慢得明显
0.1 1.9333... 接近谷底 这一轮已经明显朝正确方向移动
0.5 5.6666... 一脚跨过谷底 下一轮很可能在另一边反弹回来
1.0 10.3333... 严重过冲 loss 往往会跳得很难看，训练很容易发散
同一梯度下扫一遍 etapython
w = 1.0
grad_w = -28 / 3

etas = [0.01, 0.1, 0.5, 1.0]
for eta in etas:
    w_next = w - eta * grad_w
    print(f"eta={eta:.2f}, w_next={w_next:.4f}")

真正要记的是判断方式

这里的数值只是为了把步长差异看清楚。实际训练里梯度会随着参数变化而变化，不能把 eta 固定成某个“最好”的数字就结束。更稳的判断方式，是看 loss 曲线是否平稳下降，参数是否在合理范围里移动。

锚点追踪

同一组样本看三次

这里继续沿用上一讲的锚点样本 x=[1,2,3]、y=[2,4,6]，先把 b 固定成 0，只看 w 这一条切片。这样可以把“loss 的高度”和“梯度往哪指”看得更清楚。

$$ \mathcal{L}(w) = \frac{14}{3}(w-2)^2,\qquad \frac{d\mathcal{L}}{dw} = \frac{28}{3}(w-2) $$

这个切片只是在这一组样本上把 b=0 先固定住。你可以把它当作一条一维坡道：w=2 是谷底，w=1 和 w=3 在谷底两边。

当前点 loss grad eta=0.1 时下一步 读出来的结论
w=1, b=0 14/3 -28/3 w=1.9333... 梯度为负，所以参数应该往大的一侧走。
w=2, b=0 0 0 w=2 已经到谷底，当前切片上不需要再挪。
w=3, b=0 14/3 28/3 w=2.0666... 梯度为正，所以参数应该往小的一侧走。
同一组样本的更新演示python
def loss(w):
    return (14 / 3) * (w - 2) ** 2

def grad(w):
    return (28 / 3) * (w - 2)

w = 1.0
eta = 0.1
print(loss(w), grad(w), w - eta * grad(w))

学习率

只改步长，不改方向

学习率控制“每次迈多大步”；方向仍由梯度符号决定。同一个梯度，如果 eta 太小，参数会慢得像挪不动；太大，参数会跨过低谷，在两边来回震荡，甚至直接发散。

学习率状态 常见现象 先看什么 下一步怎么调
太小 loss 缓慢下降，参数几乎不动 步子是否太短 先试着放大 2 到 5 倍
合适 loss 平稳下降，曲线方向一致 方向和步长都正常 继续观察，不要频繁改动
太大 loss 震荡、来回跳、甚至越训越差 是否跨过低谷 先降学习率，再看数据尺度
同一个梯度，不同 etapython
w = 1.0
grad = -28 / 3

for eta in [0.01, 0.1, 1.0]:
    w_next = w - eta * grad
    print(eta, w_next)

批量策略

同一公式，不同估计

梯度下降的公式不变，变的是“梯度从多少样本上估出来”。全量 batch 最稳，但最贵；SGD 最便宜，但噪声最大；mini-batch 在两者中间，是深度学习里最常见的折中。

策略 每次看多少样本 方向稳定性 代价 常见用途
batch gradient 全部训练样本 最稳 每一步最贵 数据不大、想要平滑曲线时
SGD 一个样本 最吵 每一步最便宜 教学演示、理论分析、极端在线场景
mini-batch 一小批样本 介于两者之间 性价比最好 绝大多数深度学习训练

全量 batch 的代价

因为数据一大，更新一步就要把全部样本都看完，代价太高。训练会变慢，实验也不灵活。

单样本 SGD 的代价

因为方向太吵，训练曲线容易抖。噪声有时有帮助，但噪声太大时，收敛会更难看。

mini-batch 的关键价值在于“够稳，又不至于太贵”。如果 batch 太小，你看到的梯度更像局部噪声；如果 batch 太大，你虽然得到更平滑的方向，却要付出更高的计算成本。对于大多数深度学习任务，真正需要的是一个能持续前进、又能接受训练代价的折中点。

现象 好处 坏处 适合怎么想
batch 很小 每一步便宜，噪声可能帮你摆脱平坦区域 方向更乱，loss 曲线更抖 把它看成“带噪的方向估计”
batch 适中 方向和成本比较平衡 没有极端的小 batch 那么便宜，也没有全量 batch 那么稳 这是最常见的训练折中
batch 很大 方向平滑，日志容易读 每一步都很贵，更新频率低 更像精细扫描，不像日常训练

数据尺度会把 eta 放大或缩小

单位不对，步长也会跟着不对

同一个 eta，放在不同的数据尺度上，训练感觉会完全不同。输入、标签、特征单位如果相差很大，梯度也会被一起放大或缩小，于是你明明没改学习率，更新步子却看起来完全变了。很多“怎么突然不收敛了”的问题，最后都能追到这个地方。

拿锚点样本做一个最直接的对照就够了：x=[1,2,3]、y=[2,4,6] 是正常尺度；如果你把目标改成 y=[200,400,600]，但参数还停在 w=1,b=0，那残差和梯度都会大很多。同样是 eta=0.1，这时一步可能就太猛了。

数据状态 梯度体感 同一个 eta 的表现 先检查什么
输入和标签都在正常数量级 更新幅度通常比较平衡 loss 往下走得比较平稳 先看更新方向和 batch
标签单位突然放大 100 倍 梯度也会被拖大 很容易跨谷，loss 乱跳 先看单位或归一化是否出了问题
特征尺度差很多 某些参数会主导梯度 训练偏向某些维度，曲线难看 先看输入是否需要标准化
打印尺度差异python
x_small = [1, 2, 3]
y_small = [2, 4, 6]
y_large = [200, 400, 600]

print("small:", max(y_small) - min(y_small))
print("large:", max(y_large) - min(y_large))

别把“eta 太大”和“尺度太大”混成一个问题

两者都会让 loss 震荡，但修法不一样。eta 太大，先降学习率；数据尺度太大，先查单位、归一化和输入预处理。你如果只记住“调小一点”，很容易把真正的问题藏起来。

现场排查顺序

看到 loss 乱跳时，先不要只盯着 eta。第一步看输入和标签的单位有没有突然变大；第二步看特征之间是否差几个数量级；第三步再看是否需要降学习率或调 batch。这样查，能把“步子太大”和“数据把梯度放大了”分开。

如果单位没问题，降 eta 后曲线通常会变稳；如果单位或尺度有问题，只降 eta 可能只是把训练拖慢。更稳的做法是先修数据尺度，再重新观察同一个学习率下的参数移动幅度。

一轮训练步骤到底读写什么

把顺序和状态钉住

到这里，方向、步长和 batch 已经分开了。下一步要把它们接回真正的训练循环。训练代码最容易出问题的地方，通常落在每一行代码读了什么、写了什么、该不该建图、该不该清旧梯度。顺序一旦错，训练看起来仍然在跑，但参数和梯度会悄悄跑偏。

下面这段最小训练循环，只保留本讲最关心的动作。它的作用是让你把“前向、清零、反向、更新”四步和各自的状态变化一一对上。真正的排错，往往就是从这几行开始的。

最小训练循环python
for x, y in loader:
    optimizer.zero_grad()
    pred = model(x)
    loss = criterion(pred, y)
    loss.backward()
    optimizer.step()
代码行 读取什么 写入什么 如果放错顺序会怎样
optimizer.zero_grad() 上一轮残留的 grad 把旧梯度清空 上一轮梯度混进这一轮，方向会歪
pred = model(x) 当前 batch 的输入和参数 模型预测 如果输入形状不对，前向就接不上
loss = criterion(pred, y) 预测和标签 标量 loss loss 如果没压成标量，后面不好反向
loss.backward() 标量 loss 把梯度写到参数的 grad 上 没有这一步，step() 读不到新梯度
optimizer.step() 参数上的 grad 更新参数值 参数看起来在训，其实一动不动

手动更新和优化器更新是一回事

如果不用优化器，手写版本就是“读 grad，然后在 no_grad 下把参数减掉一小步”。优化器只是把这套动作封装成了 step()。你一旦能把这两种写法对上，就不会把 backward() 和 step() 混为同一个动作。

动作 读什么 写什么 错了以后最容易看到的症状
optimizer.zero_grad() 上一次留下来的 grad 把旧梯度清空 两个 batch 的方向叠在一起，更新越来越难解释
loss.backward() 标量 loss 和前向时保存的计算图 把本轮梯度写到参数的 grad 上 grad 还是空的，或者残留上一轮的值
optimizer.step() 参数当前值和它们对应的 grad 真正改参数 训练日志在跑，参数却没有变化
with torch.no_grad() 当前参数值 只做更新，不把这一步接进计算图 手动更新被错误记录进图里，后面反向会变乱

把这张账本和上一讲连起来看，链路就完整了：前向先建立图，backward() 负责把梯度写出来，step() 负责改参数，zero_grad() 负责把上一轮的残留清掉。你如果在排错时只盯着“loss 有没有下降”，很容易忽略真正该看的地方其实是梯度是不是被正确写入、清空和读取。

曲面不只一个谷底

只抓这一讲需要的直觉

上一讲已经告诉你，\mathcal{L}(w,b) 可以被看成一张地形图。这一讲要补一块更实际的直觉：真实地形不一定只有一个漂亮的碗，还可能有平坦区域、鞍点和多个局部最优。你不需要在这一讲证明它们的性质，只需要知道梯度下降是局部方法，它只看当前位置附近的斜率。

这意味着两件事。第一，你现在看到的梯度，只代表当前这个点附近的方向，不能代表整张地图的终极答案。第二，mini-batch 的噪声有时会让参数从平坦地带抖出去，但它也会带来更大的抖动。训练更像一边走、一边看局部地形。

地形 梯度通常怎样 训练时体感 你现在该怎么理解
局部最优 附近斜率接近 0 参数会慢下来 先别急着把它当全局答案
平坦区域 梯度很小 loss 下降慢，像卡住 先查学习率，再查 batch 和数据尺度
鞍点 有的方向像上坡，有的方向像下坡 曲线可能看起来不稳定 这不等于算法坏了，只是地形复杂

这一点和本讲的关系

你现在不需要追求“找到最优解”的宏大表述，只需要知道：梯度下降是靠局部信息前进的，所以更新质量、学习率和 batch 质量，都会直接影响它能不能稳当地往下走。

学习率过大时，训练为什么会跨谷

一条完整的故障链

把锚点样本继续固定在 x=[1,2,3]、y=[2,4,6]，从 w=1 出发。如果 eta 很小，参数会一点一点往谷底靠；如果 eta 太大，参数会直接越过 w=2，跑到另一边。下一轮梯度的方向会随之反转，于是参数又被推回来。你看到的就是 loss 上下跳，曲线很难平滑下降。

这就是“跨谷”最常见的样子。它在日志里很直观：w 一次走太远，loss 明明已经快接近最低点，却因为步子过大把自己甩到了更差的位置。很多初学者以为“只要方向对就行”，实际上方向对不代表步长就对。

要把这件事看实，你只需要做一个极小的打印：连续记下三次更新前后的 w 和 loss。若 eta 合适，数值会朝谷底靠近；若 eta 太大，w 会在谷底两侧来回摆，loss 也会跟着跳。下面这张表就是一条很短但很完整的诊断链。

迭代 当前 w 下一步后 w loss 走势 说明
第 0 步 1.0 5.6666... 先下降，再很可能反弹 步子太大，直接越过谷底
第 1 步 5.6666... 可能被推回另一侧 继续乱跳 梯度方向因为越界而反转
第 2 步 另一侧的更大值 仍然摆动 没有稳定收敛迹象 该先降学习率，先别急着换更大模型
打印三步就能看出过冲python
w = 1.0
grad_w = -28 / 3
eta = 0.5

for step in range(3):
    loss = (14 / 3) * (w - 2) ** 2
    print(f"step={step}, w={w:.4f}, loss={loss:.4f}")
    w = w - eta * grad_w
    grad_w = (28 / 3) * (w - 2)

如果你看到这种日志

loss 在两个值之间来回晃，参数每次都越过谷底，先别急着换损失函数。先把 eta 降下来，再看是否还会跨谷。

这和 batch 太小不一样

batch 太小会让梯度更吵，但不一定跨谷；学习率太大则是每一步都走得太远。两者都会抖，但诊断点不同。

练习钩子

拿你自己的训练脚本，连着打印 3 到 5 个 batch 的 w、loss 或者某个参数范数。如果数值一直在两边跳，先降学习率；如果只是慢，先别急着下结论。

PyTorch 对应物

把手写更新接回训练循环

手算和 PyTorch 里做的是同一件事：先拿到梯度，再用梯度改参数。区别只在于，手写代码直接把参数减掉一小步，优化器则把这件事封装起来；而 with torch.no_grad() 是告诉框架，这一步只是更新，不需要进计算图。

手写更新和优化器更新python
w = torch.nn.Parameter(torch.tensor(1.0))
b = torch.nn.Parameter(torch.tensor(0.0))
eta = 0.1

# 手写更新
with torch.no_grad():
    w -= eta * w.grad
    b -= eta * b.grad

# 训练循环里的标准写法
optimizer.zero_grad()
loss.backward()
optimizer.step()
动作 读什么 写什么 如果顺序错了会怎样
optimizer.zero_grad() 上一轮残留的梯度 把旧梯度清掉 上一轮梯度会混进这一轮
loss.backward() 当前 loss 把梯度写到参数的 grad 上 没有梯度，step() 就没东西可读
optimizer.step() 参数上的 grad 更新参数值 训练看似在跑，参数却不动
with torch.no_grad() 当前张量值 只做更新，不建图 更新动作被错误地挂进计算图

先看症状，再看下一步

训练一旦不顺，先别急着换模型。先看现象属于哪一类，再回去查学习率、方向、batch 和更新顺序。下面这个小面板把四种常见症状拆开了。

症状 最先怀疑什么 下一步检查
loss 几乎不动 学习率太小，或者梯度太弱 先把 eta 放大一点，再看 grad 是否接近 0
loss 震荡或发散 学习率太大 先把 eta 降下来，再看数据尺度
参数看起来在训，但方向不对 更新方向写反了 确认是否少了减号，或者把 grad 当成了更新值
曲线很抖 batch 太小，估计噪声太大 先试 mini-batch，再观察曲线平滑度 
症状：loss 缓慢下降，但参数几乎不动。
先查：学习率是否太小。
下一步：先把 eta 放大 2 到 5 倍，再看 3 到 5 个 batch。

常见误区

把最容易错的地方先钉住

这几类错最常见，也最容易在训练日志里装成“看起来正常”。先认清它们，后面排错会快很多。

误区 现场表现 立即修法
把梯度当更新 以为 grad 本身就是参数变化 记住梯度只给方向，真正更新要乘 eta，还要反着走
学习率越大越好 训练曲线抖动、越训越差 先降 eta，再看数据尺度和 batch 大小
忘了 batch 的代价 一味追求全量或单样本 按数据量和稳定性选择 mini-batch
只看训练集 loss 训练看起来不错，验证集却很差 把验证集一起看，别让训练曲线替你做结论
顺序写反 step() 提前或 zero_grad() 放错位置 先清旧梯度，再反向，再更新

10. 为什么 optimizer.zero_grad() 通常要放在 backward() 前面？

答案要提到：梯度默认会累积，清零放在前面才能让每个 batch 单独更新。

参考答案 / 评分要点

PyTorch 里的梯度默认会累积，不会自动覆盖上一轮的值。把 zero_grad() 放在 backward() 前面，才能保证这一次 batch 只看当前梯度，不被上一轮残留影响。若是故意做梯度累积，才会把清零延后；但那是有意设计，不是默认写法。

15. 看到跨谷日志后，你会按什么顺序排查？

至少说出先降学习率、再看 batch 和数据尺度。

参考答案 / 评分要点

推荐顺序是：第一步先降学习率，因为跨谷最常见的原因就是步子太大；第二步看 batch 大小，确认是不是梯度太吵；第三步看数据尺度，避免因为输入或标签数值过大把梯度放大。若只说“调小一点试试”还不够，至少要能把优先级说出来。

16. 同一梯度下，eta=0.01 和 eta=1.0 的区别是什么？

要说清步长、跨谷和收敛速度三层意思。

参考答案 / 评分要点

eta=0.01 会让每一步更短，通常更稳，但可能慢；eta=1.0 会把每一步放得很大，更容易跨过低谷，loss 也更容易震荡或发散。好答案还会把“同一梯度、不同步长”的关系说清楚，而不是只说“大”和“小”。
