先抓主线

从更新倒回梯度来源

梯度下降需要梯度，但真正写训练代码时，你通常不会手写每个导数。PyTorch 会在前向传播时记录张量运算，等你对标量 loss 调用 backward()，它再沿着这张图把梯度传回参数。

阶段读什么写什么 / 改什么出错时的信号

前向先建计算图

forward

反向传播依赖前向传播留下的依赖关系。只看数值时，y_hat 是一个数字；从自动求导看，它还带着“我由 x * w + b 算出来”的记录。后面算 loss 时，这条记录继续接到 (y_hat - y)^2 上。

$$ y\_hat = xw + b,\quad e = y\_hat - y,\quad L = e^2 $$

这里的符号都能直接对应代码：x,w,b 先生成预测，预测和标签生成误差，误差平方生成标量 loss。

Le 2(-2.5) 26.25e

链式法则怎样接上整条路径

local

一个参数通常不会直接变成 loss。比如 w 先影响 y_hat，y_hat 影响误差 e，e 再影响 L。链式法则做的事，就是把每一段“局部影响”乘起来。

$$ \frac{\partial L}{\partial b} = \frac{\partial L}{\partial e} \cdot \frac{\partial e}{\partial y\_hat} \cdot \frac{\partial y\_hat}{\partial b} = 2e \cdot 1 \cdot 1 $$

代入 e=-2.5、x=2，得到 dL/dw=-10，dL/db=-5。

局部导数

每个运算只需要知道自己的局部导数。比如 y_hat=xw+b 对 w 的局部导数是 x，对 b 的局部导数是 1。

反向传播就是这件事：每个节点拿到上游梯度，再乘自己的局部导数，把结果传给前面的节点。网络变深以后，规则没有变，只是路径更多。

把一个 backward 手算到底

trace

下面这张表按真实反向顺序走一遍。这里沿用前向阶段已经保存的中间值：x=2、e=-2.5，不重新跑前向。

L=e^21dL/de = 2e = -5-5传给 e

e=y_hat-y-5de/dy_hat=1-5传给 y_hat

y_hat=xw+b 到 w-5dy_hat/dw=x=2-10w.grad

y_hat=xw+b 到 b-5dy_hat/db=1-5b.grad

代码
x = 2.0
y = 5.0
w = 1.0
b = 0.5

y_hat = x * w + b # 2.5
e = y_hat - y # -2.5
loss = e ** 2 # 6.25

dL_de = 2 * e # -5.0
de_dyhat = 1.0
dyhat_dw = x # 2.0
dyhat_db = 1.0

dL_dw = dL_de * de_dyhat * dyhat_dw # -10.0
dL_db = dL_de * de_dyhat * dyhat_db # -5.0

这段算顺了，后面的 loss.backward() 就只是把同一件事自动化，而且能处理更大的图。

同一个参数走多条路时，梯度要相加

branch

前面的例子只有一条主路径，适合入门，但真实计算图经常会分叉。一个参数可能同时影响两个中间量，这两个中间量再一起影响 loss。反向传播遇到这种结构时，会把从不同分支传回来的梯度加起来，再写到同一个参数的 .grad。

$$ z = wx + (w + b),\quad L = (z-y)^2 $$
$$ \frac{\partial L}{\partial w} = \frac{\partial L}{\partial z} \cdot \left( \frac{\partial a}{\partial w} + \frac{\partial c}{\partial w} \right) = 2(z-y)(x+1) $$

如果 x=2、y=6、w=1、b=0.5，那么 z=3.5，z-y=-2.5，所以 dL/dw = 2(-2.5)(2+1) = -15。

w -> a -> z -> La = w xda/dw = x = 22(z-y) 2 = -10

w -> c -> z -> Lc = w + bdc/dw = 12(z-y) * 1 = -5

合并到 w.grad两条路径都来自同一个叶子参数x + 1 = 3-10 + -5 = -15

代码
import torch

x = torch.tensor(2.0)
y = torch.tensor(6.0)
w = torch.tensor(1.0, requires_grad=True)
b = torch.tensor(0.5, requires_grad=True)

a = w * x
c = w + b
z = a + c
loss = (z - y) ** 2
loss.backward()

print(w.grad) # tensor(-15.)，来自两条路径的贡献相加
print(b.grad) # tensor(-5.)

这个机制能解释很多训练日志里的现象：一个参数的梯度会把所有影响 loss 的路径贡献合并起来。调试时只盯一条路径，很容易漏掉一部分来源，最后把梯度大小或符号看错。

排查梯度和手算不一致时

先查是不是漏了一条路径，或者把 batch 里多个样本的贡献当成了单个样本。排查顺序可以固定下来：第一步打印参与 loss 的所有中间量；第二步画出同一个参数到 loss 的所有路径；第三步确认 loss 的 reduction 是 mean 还是 sum；最后再比较每条路径的局部导数。这样能把“梯度不对”拆成几个能检查的小问题。

在后面的网络里，同一个权重会被很多样本、很多输出位置共同使用。它的 .grad 往往是多条路径、多条样本贡献合并后的结果。理解这一点后，你看到梯度比单样本手算值小一半或大几倍时，先检查平均方式、广播和共享路径，再去看学习率。

为什么同一张图通常只反向一次

graph

backward() 需要前向时保存下来的中间值。普通训练里，一次反向之后这些中间值通常就可以释放，内存也会回收一部分。这和“每个 batch 通常只反向一次”的节奏是对齐的。

状态中间值会发生什么适合场景风险

同一张图两次 backward 的边界复制

实际排错时，先分清是“图被释放了”还是“梯度没清干净”。前者看的是重复 backward 的报错，后者看的是 .grad 里是不是残留了上一轮的值。两者症状相近，但检查点完全不同。

PyTorch autograd 对应到哪里

code

PyTorch 里只有需要求梯度的张量才会被重点追踪。对参数来说，常见做法是创建时写 requires_grad=True，或者把它们放进 nn.Parameter。当这些张量参与可求导计算时，框架会在结果张量上挂上 grad_fn，说明它有来路。

代码
import torch

x = torch.tensor(2.0)
y = torch.tensor(5.0)
w = torch.tensor(1.0, requires_grad=True)
b = torch.tensor(0.5, requires_grad=True)

y_hat = x * w + b
loss = (y_hat - y) ** 2

print(y_hat.grad_fn) # AddBackward0 等，说明 y_hat 有计算来路
print(loss.grad_fn) # PowBackward0 等，说明 loss 也在图上

loss.backward()
print(w.grad) # tensor(-10.)
print(b.grad) # tensor(-5.)

对象 / 属性作用本例状态常见误读

requires_grad=True告诉 autograd 这个张量需要梯度w,b 打开给数据 x,y 乱开梯度

grad_fn记录结果由哪个可求导运算产生y_hat,loss 有把它当成最终梯度值

.grad保存反向传播写入的梯度w.grad=-10,b.grad=-5以为所有中间张量默认都有

backward()沿图传播梯度写入叶子张量以为它直接更新参数

叶子张量和状态账本

leaf

叶子张量可以先用一个实际规则记住：你直接创建并要求梯度的参数，通常就是叶子张量。它们是优化器要管理的对象，backward() 会把梯度累积到它们的 .grad 上。

w是是无w.grad

b是是无b.grad

y_hat否由 w,b 传来有默认不保留 .grad

loss否是有作为反向入口

中间张量为什么默认不保留 .grad

retain_grad

前向里每多留一个中间张量的梯度，内存就多占一份。训练时真正需要更新的是叶子参数，所以 PyTorch 默认只把梯度写到叶子张量的 .grad 里。中间张量可以重新算，叶子参数才是优化器要读的状态。

对象默认是否保留 .grad什么时候会有值代价典型用途

只在调试时保留中间梯度复制

这段代码只说明一件事：中间张量的梯度并非不能看，而是默认不保留。把它临时打开以后，才能沿着更细的路径定位哪里传错了值。

为什么常见 loss 是标量

scalar

backward() 需要一个清楚的起点。训练里最常见的起点是一个标量 loss，因为它代表“这批样本在当前参数下有多差”。如果 loss 还是向量，PyTorch 需要你额外提供每个元素的权重，也就是反向传播的初始梯度。

代码
pred = torch.tensor([2.0, 4.0, 7.0], requires_grad=True)
target = torch.tensor([2.0, 5.0, 6.0])

loss_each = (pred - target) ** 2 # shape: [3]
# loss_each.backward() # 常见情况下会报错：loss 需要先变成标量

loss = loss_each.mean() # shape: []
loss.backward() # 可以直接从标量目标反向传播

写法输出形状能否直接 backward()适合场景

reduction='none'每个样本一个 loss通常不直接用要做样本权重、排查异常样本

mean()标量可以最常见的训练入口

sum()标量可以希望 loss 随样本数累加时

梯度会累积，清零是必要步骤

grad

PyTorch 的 .grad 默认会累积。第二次 backward() 会把新梯度加到旧梯度上，不会自动覆盖。这一设计支持梯度累积等用法；在普通 mini-batch 训练里，如果你忘了清零，下一次更新会混进上一批数据的梯度。

代码
w = torch.tensor(1.0, requires_grad=True)

loss1 = (2 * w - 5) ** 2
loss1.backward()
print(w.grad) # tensor(-12.)

loss2 = (3 * w - 7) ** 2
loss2.backward()
print(w.grad) # tensor(-36.)，第二次 backward 会叠加到第一次上

时间点w.grad 初始状态动作w.grad 结果训练含义

第 1 个 batch 前None 或 0backward()当前 batch 梯度正常

第 2 个 batch 前没清零上一批梯度backward()旧梯度 + 新梯度更新方向被污染

第 2 个 batch 前清零0 或 Nonebackward()当前 batch 梯度普通训练循环的预期

把 backward 接回训练循环

operation

第三讲已经用梯度更新参数。这一页补上的就是梯度怎样产生、存在哪里。到第五讲开始用 nn.Module、loss 函数和优化器时，下面这段顺序就是最小模板。

代码
for x, y in loader:
 pred = model(x) # 前向：建立本轮计算图
 loss = criterion(pred, y) # 标量目标

 optimizer.zero_grad() # 清掉上一轮 .grad
 loss.backward() # 写入本轮梯度
 optimizer.step() # 读取梯度并更新参数

代码行读取写入 / 改变如果漏掉

pred = model(x)输入和当前参数预测与计算图没有可反向的图

loss = criterion(pred, y)预测和标签标量 loss没有优化目标

zero_grad()参数已有 .grad清空旧梯度新旧梯度混在一起

loss.backward()loss 和计算图参数 .grad优化器没有新梯度可用

optimizer.step()参数和 .grad参数数值只算梯度，参数不动

故障实验：看症状先定位哪一步

failure

反向传播相关错误大多来自状态没交接好。下面这几个症状可以直接映射到检查点。

.grad is Nonerequires_grad 和叶子状态参数没有被 autograd 跟踪创建参数时开启梯度，或使用 nn.Parameter

loss 波动异常zero_grad() 位置梯度累积混入旧 batch每个 batch 前清旧梯度

grad 有值但参数不动optimizer.step()只算梯度，没更新参数确认 step 被调用，且优化器拿到参数

非标量 backward 报错loss.shapeloss 没有 reduce先 mean() 或 sum()

这里先不追的内容

boundary

这一页把主线收在普通一阶梯度上。retain_graph=True、二阶梯度、hook、自定义 autograd.Function、复杂优化器内部状态都先放一边。同一张计算图通常反向一次后会被释放；如果你真的要对同一张图反向多次，才会遇到 retain_graph=True 这类选项。
