多维特征输入：矩阵形状与前馈网络

上一讲还是单特征线性模型，这一讲把输入换成一行多列特征。关键不在“层数变多”，而在 `Linear` 只看最后一维，上一层输出多少列，下一层就要接多少列。

这一页先把 batch 维、特征维、`in_features` / `out_features` 和 `forward` 的数据路径钉住。卷积、序列、flatten、BatchNorm、Dropout 先放到后面。

课程地图 上一讲：06 逻辑斯蒂回归 下一讲：08 加载数据集

读完先做这三件事

  • 把 `[N, 8]`、`Linear(8, 6)`、`Linear(6, 4)`、`Linear(4, 1)` 串成一条形状链。
  • 看到 `mat1 and mat2 shapes cannot be multiplied` 时，先查上一层 `out_features` 和下一层 `in_features`。
  • 能说清为什么多层网络要把非线性放在层与层之间。

学习目标

先定边界

这页先解决什么

把一行多列特征送进前馈网络之前，先认清哪一维是样本数，哪一维是特征数。只要这条线看顺了，后面的层间对齐就不用猜。

先别追什么

这里不展开卷积、序列、reshape、BatchNorm、Dropout，也不讲更复杂的损失选择。它们会在后面的单元里回来。

术语 先抓住什么 常见误解
batch 维 一次送进模型的样本数 把它当成特征列数
特征维 每个样本的输入列数 把它和 batch 维混在一起
in_features 这一层接收多少输入列 以为它是样本总数
out_features 这一层产出多少新特征 以为它等于类别标签本身
forward 数据从输入走到输出的路径 把它当成层名堆叠清单

概念骨架

先看整条线
输入 batch [32, 8]

32 是样本数，8 是每个样本的特征列数。`Linear` 先认最后一维。

第一层 [32, 6]

`Linear(8, 6)` 把 8 列变成 6 列，batch 维不动。

非线性 [32, 6]

`ReLU` 改数值，不改形状。它给后面的层留出非线性空间。

第二层 [32, 4]

`Linear(6, 4)` 只能接 6 列输入。前后不接，后面就会报形状错。

输出层 [32, 1]

最后一层常收成 1 个分数或 logit，后面再交给 loss 或决策规则。

$$ x \in \mathbb{R}^{N \times d_{in}}, \qquad y = xW^\top + b, \qquad y \in \mathbb{R}^{N \times d_{out}} $$

N 是 batch size，d_in 是输入特征数，d_out 是输出特征数。这里最重要的事只有一件：Linear 改最后一维，前面的 batch 维保留。

形状账本

读什么 / 写什么 / 哪里会坏
步骤 before after 读什么 写什么 错了会怎样
x [32, 8] 进入第一层前 8 列特征 送给 Linear(8, 6) 第一层 in_features 写错，后面就接不住
linear1 [32, 8] [32, 6] 最后一维 8 6 个新特征 mat1 and mat2 shapes cannot be multiplied
relu1 [32, 6] [32, 6] 上一层的激活值 同样形状的新数值 如果删掉，中间还是线性，层数只是摆设
linear2 [32, 6] [32, 4] 6 列输入 更窄或更宽的表示 前一层输出列数不等于 6，就断链
linear3 [32, 4] [32, 1] 4 列隐藏表示 1 个最终分数 输出宽度和任务 / loss 不对齐
先记这一句 线性层之间的对齐关系很朴素：上一层的 out_features 就是下一层的 in_features。真正出错时，先看这里，先别去改优化器。

具体病例

一个样本，三次变形

先看一行样本

糖尿病这类表格数据在这一讲里可以先想成一行 8 列数值。一个样本不是一张图，也不是一段序列，它先只是 8 个特征排成的一行。

如果把 32 个样本一起送进模型，输入就从 [8] 变成 [32, 8]。前者是单样本，后者是一个 batch。这里最容易弄混的地方，不是模型，而是“这一维到底是在数样本，还是在数特征”。

先看什么，后看什么

先看最后一维是不是 8，再看第一层是不是 Linear(8, 6)。如果这两件事对不上，后面再多写几层也没有用。

等第一层对了，再看第二层是不是接 6，第三层是不是接 4。前面接得上，后面才有资格谈激活函数和任务输出。

位置 shape 你该盯什么 这一行错了会发生什么
单样本 x_i [8] 这是 8 个特征，不是 8 个样本 如果把它当 batch，后面 reshape 和 loss 都会乱
一个 batch [32, 8] 32 是样本数，8 是特征数 如果把 32 写进 in_features，第一层就写歪了
第一层输出 [32, 6] 列数从 8 变成 6 下一层必须接 6 列，否则矩阵乘法接不上
激活后 [32, 6] 形状不变，数值变了 如果激活放错位置，模型还在跑，但表达力会变弱
第二层输出 [32, 4] 这是中间表示，不是最终答案 如果把 4 写成 8，最后一层的输入就乱了
最终输出 [32, 1] 这一层的宽度要和任务对应 如果任务是二分类或回归，输出宽度写错，loss 入口就会跟着出问题
调试时先打印 shapepython
def forward(self, x):
    print("input:", x.shape)
    x = self.linear1(x)
    print("after linear1:", x.shape)
    x = torch.relu(x)
    print("after relu1:", x.shape)
    x = self.linear2(x)
    print("after linear2:", x.shape)
    x = torch.relu(x)
    print("after relu2:", x.shape)
    x = self.linear3(x)
    print("output:", x.shape)
    return x
调试习惯比背答案更有用 真要查 bug 的时候，先打印 x.shape。不要急着改网络深度，也不要先怀疑 optimizer。大多数 shape 错误都能在第一行打印里看出来。

forward 路径

数据怎么走

forward 不是把层名排成一列，而是写清楚数据怎么从输入走到输出。每一行都要能回答两个问题：这一行读什么，下一行接什么。

多层前馈网络python
import torch
import torch.nn as nn


class DiabetesFFN(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear1 = nn.Linear(8, 6)
        self.linear2 = nn.Linear(6, 4)
        self.linear3 = nn.Linear(4, 1)

    def forward(self, x):
        x = torch.relu(self.linear1(x))
        x = torch.relu(self.linear2(x))
        return self.linear3(x)
代码行 它在做什么 如果写错会怎样
self.linear1 = nn.Linear(8, 6) 第一层接 8 列输入，输出 6 列表示 如果把 8 写错，模型一开始就和数据对不上
x = torch.relu(self.linear1(x)) 先线性变换，再加非线性 如果把 ReLU 放错位置，形状可能还对，表达力会变差
self.linear2 = nn.Linear(6, 4) 第二层只接前一层留下的 6 列 如果还写成 8，就会和上一层输出断开
return self.linear3(x) 最后收成 1 个输出分数 如果任务是二分类或回归，这一层的宽度要和任务对齐

为什么要把激活放在层之间

如果中间没有非线性，多个线性层连起来仍然等价于一个线性层。层数变多了，能表达的东西却没变多，这就是很多人第一次写前馈网络时会踩的坑。

隐藏层和输出层不是一回事

隐藏层负责重组表示，输出层负责收束到任务需要的宽度。输出层要不要接 sigmoid，取决于你后面用什么 loss 和决策规则。

代码与例子

把同一个 batch 走完

这一节只盯一组锚点数据。只要你能跟着它把每一层的 shape 写出来，后面的题就不会只是在背词。

同一个 batch 的形状推导python
x = torch.randn(32, 8)        # 32 个样本，每个样本 8 个特征
h1 = model.linear1(x)         # [32, 6]
h1 = torch.relu(h1)           # [32, 6]
h2 = model.linear2(h1)        # [32, 4]
h2 = torch.relu(h2)           # [32, 4]
y_hat = model.linear3(h2)     # [32, 1]
位置 形状 一句话解释
x [32, 8] 输入 batch 的最后一维是特征列
h1 [32, 6] 第一层改写特征列数
h2 [32, 4] 第二层继续收缩表示
y_hat [32, 1] 最后收成单个输出分数

正确连接

Linear(8, 6) → Linear(6, 4) → Linear(4, 1)

上一层输出多少列，下一层就接多少列。这个顺序对了，shape 才会一路通。

错误连接

Linear(8, 6) → Linear(8, 4)

上一层只留下 6 列，下一层却还等 8 列，矩阵乘法会直接报错。

特征尺度也会影响训练 这不是 shape 错误，但很常见。某些列数值特别大、某些列特别小，优化就会先盯住大尺度特征，训练速度和稳定性都会受影响。标准化不是为了让数据好看，是为了让不同特征更像在同一个台阶上说话。

排错链

先症状，后原因

这一讲最常见的报错不是“模型学不会”，而是“层接不上”。只要你知道先看哪一层、打印哪一个 shape，很多问题都能在 loss 之前被截住。

症状 大概率原因 先打印什么 怎么改
mat1 and mat2 shapes cannot be multiplied 上一层输出列数和下一层 in_features 没接上 x.shape、每层 in_features / out_features 把下一层 in_features 改成上一层输出列数
第一层就报尺寸不对 把 batch size 当成特征数，或者把特征数写错 输入 batch 的最后一维 先改输入列数，再改第一层宽度
模型能跑，但 loss 一直怪 特征尺度太不均匀，或者输出宽度和任务不符 pred.shape、target.shape、各列范围 先标准化，再对齐任务输出宽度
多层网络看着“像在工作” 中间没有放非线性，多个 Linear 叠成了线性 中间激活的位置 把 ReLU / Sigmoid 放回层与层之间
错误写法与修正python
# 错误：第二层还在等 8 列
self.linear1 = nn.Linear(8, 6)
self.linear2 = nn.Linear(8, 4)

# 修正：第二层接第一层的输出 6 列
self.linear1 = nn.Linear(8, 6)
self.linear2 = nn.Linear(6, 4)
先改哪里 先改 in_features，不是先改激活，也不是先改 loss。层间维度错了，后面所有判断都会被误导。

误区诊断

错因 / 后果 / 修法

把 32 当成特征数

后果：in_features 会写错，第一层从入口就断了。

修法：先看输入最后一维，32 是 batch size，8 才是特征数。

只看层名，不看上一层输出

后果：下一层的 in_features 和前一层 out_features 对不上。

修法：把每一层写成 [N, in] → [N, out]，从上到下检查一遍。

连续线性层中间不加激活

后果：多个线性层叠起来仍然是线性，层数变多了，表达力没长多少。

修法：把 ReLU 这类非线性放在中间层之间。

输出宽度和任务不对齐

后果：回归、二分类、多分类会各自要求不同的输出宽度和 loss 入口。

修法：先定任务，再定最后一层输出宽度。

特征尺度差太大

后果：训练可能变慢，甚至显得很不稳定。

修法：先做标准化或归一化，再看模型宽度要不要调。

把后面的 flatten、卷积、序列问题塞进这里

后果：现在看起来像都在讲“输入”，其实是不同的输入结构。

修法：先只练二维表格输入，后面再回到更高维的张量。

边界

这一页先到这里

先别追的东西

这一页只停在二维输入和前馈网络。图片输入怎么 flatten 成一行向量、序列模型怎么处理时间步、BatchNorm 和 Dropout 怎么插进结构里，先放到后面的单元。到 08 讲你会看到 batch 怎么从 Dataset / DataLoader 来；到 10 和 12 讲，你会再碰到更高维输入。

输入类型 常见形状 这一页关心什么 先不要混进来
二维表格 [N, d] 最后一维是特征列 卷积核、时间步、通道
三维序列 [N, T, d] 每个时间步的特征怎么传 这里还不讨论隐藏状态
四维图片 [N, C, H, W] 先明白 batch，后面再谈卷积 这里还不讨论卷积核和池化

输入结构对照

先分清二维、三维、四维

二维表格是这一讲的主角

这一讲最该先熟的是 [N, d]。N 是 batch size，d 是特征列数。只要你是把 CSV、结构化指标、传感器读数这类数据送进全连接层，优先问的就是“最后一维是多少”。

表格输入的好处也很直接：每个样本的列数固定，`Linear` 读最后一维就能工作。你不需要先把它想成图片，也不需要先去管时间顺序。先把列数对齐，训练循环就能往下走。

为什么要把别的输入先放旁边

三维序列和四维图片不是这一讲的主体，但它们会拿来做边界。序列关心时间步，图片关心通道和空间位置。它们和二维表格都带 batch 维，可是单样本结构完全不同。

先把二维表格吃透，再去看更高维输入，人的脑子会省很多事。你就不会把“最后一维是特征列”误记成“所有输入都只看最后一维的数值意义”。

输入类型 典型 shape 这一层最该看什么 对前馈网络意味着什么
二维表格 [N, d] 最后一维 d 直接送进 Linear(d, ...)
三维序列 [N, T, d] 时间步 T 和每步特征 d 如果硬接全连接层，先想清楚是保留时间步还是先做汇总
四维图片 [N, C, H, W] 通道 C 和空间位置 H, W 通常要先把结构交给卷积，而不是直接拿 Linear 处理
同一批输入在三种结构里的样子python
# 二维表格
table_x = torch.randn(32, 8)

# 三维序列
seq_x = torch.randn(32, 10, 8)

# 四维图片
img_x = torch.randn(32, 3, 64, 64)

print(table_x.shape)  # [32, 8]
print(seq_x.shape)    # [32, 10, 8]
print(img_x.shape)    # [32, 3, 64, 64]
这一节的用法 如果你以后看到新的输入结构，先问自己三件事：它的单样本长什么样、batch 维在哪、最后一维是不是可以直接喂给 Linear。这三个问题答不清，就先不要硬套这一讲的做法。

参数账本

形状变了，参数也会跟着变

先算一个具体网络

对这条网络来说，参数量不是“感觉上很多”这么简单。`Linear(8, 6)` 有 8×6 + 6 个参数，`Linear(6, 4)` 有 6×4 + 4 个参数，`Linear(4, 1)` 有 4×1 + 1 个参数。

这三个数加起来是 48 + 6 + 24 + 4 + 4 + 1 = 87。你只要改了输入特征数，这个总数也会跟着变。前面那一层最敏感，因为它直接读原始特征。

为什么要关心参数量

参数量会影响模型容量、训练速度、显存占用，也会影响过拟合风险。层宽不是越大越好，层宽太小又装不下足够的表示。

这一讲不要求你调参，但你应该知道：看 shape 不只是为了不报错，也是在看模型到底准备了多少表示空间。

层 参数公式 本页例子里的数量 如果输入从 8 列改成 10 列
Linear(8, 6) 8×6 + 6 54 变成 10×6 + 6 = 66
Linear(6, 4) 6×4 + 4 28 不变，因为它接的是上一层的 6 列输出
Linear(4, 1) 4×1 + 1 5 不变，除非你把最后一层宽度也改掉
总数 把每层参数加起来 87 变成 99
快速数参数python
def linear_params(in_features, out_features):
    return in_features * out_features + out_features

total = (
    linear_params(8, 6)
    + linear_params(6, 4)
    + linear_params(4, 1)
)
print(total)  # 87
这个账本要带着走 以后你看到网络结构图，不只要看“接不接得上”，还要看“每一层准备了多少参数”。这能帮你判断模型会不会太窄，也能帮你看清输入列数变了以后，为什么最前面那一层一定要改。

输入列数改了以后

从 8 列改到 10 列

先做一个具体改动

如果原来这份表有 8 个特征，后来又加了 2 个新特征，最前面的输入层必须跟着改。新的输入不再是 [32, 8]，而是 [32, 10]。这时第一层不能继续写 Linear(8, 6)，因为第一层真正读的是输入列数。

你可以把这件事理解成“门口换了门宽”。门宽变大了，门口的第一块板子也得跟着换。后面的层如果只接第一层输出，就不需要因为输入列数变化而全改一遍。

改动顺序

先改数据读取，让单样本变成 [10]；再改第一层 in_features；最后检查第一层到后面几层的列数有没有继续接上。只改模型不改数据，或者只改数据不改模型，都会在前向里撞墙。

步骤 原来 新增 2 列后 先改哪一处 错了会看到什么
输入样本 [8] [10] 数据读取和特征拼接 输入 shape 和模型入口对不上
第一层 Linear(8, 6) Linear(10, 6) 最先改 in_features mat1 and mat2 shapes cannot be multiplied
第二层 Linear(6, 4) Linear(6, 4) 通常不用动 如果动了，往往是前一层输出也一起变了
输出层 Linear(4, 1) Linear(4, 1) 先看任务有没有变 只要任务没变，输出宽度一般不用跟着改
改输入列数后的修正版python
class DiabetesFFN(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear1 = nn.Linear(10, 6)  # 输入从 8 列改成 10 列
        self.linear2 = nn.Linear(6, 4)
        self.linear3 = nn.Linear(4, 1)

    def forward(self, x):
        assert x.shape[-1] == 10
        x = torch.relu(self.linear1(x))
        x = torch.relu(self.linear2(x))
        return self.linear3(x)
这一段要能直接用 真遇到新列时，先改数据，再改第一层，再跑一批样本看 x.shape。这条顺序如果反了，很多人会把时间花在“层数是不是不够”这种假问题上。

故障演练

从报错回到层宽

把这一讲学透，不是为了背出 Linear(8, 6)，而是为了在代码真的坏掉时，能一眼看出坏在哪一层。下面这个小故障最常见：数据已经变成 10 列，模型还停在 8 列。

观察点 现场 你该怎么判断 下一步
数据 x.shape = [32, 10] 输入已经多了 2 列 检查第一层是不是还写成 Linear(8, 6)
模型 self.linear1 = nn.Linear(8, 6) 第一层入口没改 把 8 改成 10
报错 mat1 and mat2 shapes cannot be multiplied 矩阵乘法接不上 先别看损失，先把第一层改对
修复后 Linear(10, 6) -> Linear(6, 4) -> Linear(4, 1) 第一层接住新输入，后面沿着 6 列继续走 重新跑一批样本，确认 shape 一路通到底
故障与修复python
# 故障现场
x = torch.randn(32, 10)
model.linear1 = nn.Linear(8, 6)

# 修复
model.linear1 = nn.Linear(10, 6)

# 再检查
with torch.no_grad():
    y = model(x)
    print(x.shape, y.shape)
把诊断顺序记住 这类问题的顺序很固定：先看输入列数，再看第一层 in_features，最后才看后面的层和任务。你要是先去改激活或者 loss，通常只是在绕远路。

练习时先拿一条真实样本，把 x.shape、第一层的 in_features、报错信息和修正后的网络连起来，再回到第 11、13 题验证自己有没有把路径说清楚。
