多维特征输入：矩阵形状与前馈网络

理解多维特征如何进入线性层，掌握 batch 维、特征维、层间维度匹配和多层网络 forward 的形状推导。

这节课抓一件事

从单特征线性模型扩展到多特征、多层网络。

  1. 先看对象之间的数据流。
  2. 再看每一步的形状、类型和状态变化。
  3. 最后用练习题检查自己能不能诊断错误。

概念骨架

先搭主线
序列→编码→隐藏状态→GRU/RNN→最后状态→分类
先看输入

确认数据形状、标签形式和 dtype。许多训练错误在进入模型前已经发生。

再看变换

逐层追踪输出形状和语义，别只看 API 名字。每一层都应该能回答“它把什么变成什么”。

最后看目标

确认 loss 的输入约定、输出是否是标量，以及评估指标是否对应真实任务。

判断句：如果你说不清某一行读入什么、输出什么、改变什么状态，就先不要急着调参。

训练主线：数据 → 模型预测 → 标量 loss → backward 写梯度 → optimizer.step 更新参数 → eval 检查泛化。

讲义

慢读这一部分

1. 多维特征先改变输入形状

真实任务很少只有一个输入特征。糖尿病预测这类数据通常有多个指标，每个样本是一行，每列是一个特征。把数据送进模型前，你要知道张量形状通常是 `[batch_size, num_features]`。

`batch_size` 表示一次送进模型的样本数，`num_features` 表示每个样本有多少个特征。线性层关心的是最后一维，也就是特征数。`nn.Linear(8, 6)` 表示每个样本有 8 个输入特征，输出 6 个新特征。

你现在应该能看到一个二维张量，分清哪一维是样本数，哪一维是特征数。

读到这里，你应该能：拿到一段代码后先检查输入、输出和层间维度是否对齐。

2. 线性层本质是矩阵乘法

对多维输入来说，线性层不再是简单的 `x*w`，而是矩阵乘法加偏置。输入 `[N, in_features]` 乘上权重转置后得到 `[N, out_features]`。batch 维 `N` 会保留，特征维会被线性层改写。

这就是为什么层与层之间必须维度对齐。上一层输出多少特征，下一层的 `in_features` 就应该是多少。写错会报形状不匹配，或者更隐蔽地让网络结构不符合你的设计。

你现在应该能从一串 `Linear` 层推导每一层输出形状。

读到这里，你应该能：用这一小节的判断规则去检查相关代码。

3. 多层网络需要非线性激活

如果只是把多个线性层连续相乘，中间没有激活函数，整体仍然等价于一个线性变换。多层网络真正变强，是因为在线性层之间插入 sigmoid、ReLU 等非线性激活。

课程里用 sigmoid 演示前馈网络。现代实践中，隐藏层更常用 ReLU 或其变体，因为 sigmoid 在深层网络中更容易出现梯度变小。输出层是否用 sigmoid 取决于任务和 loss。

你现在应该能说明：多层不是为了看起来复杂，而是为了通过非线性组合表达更复杂关系。

读到这里，你应该能：用这一小节的判断规则去检查相关代码。

4. forward 写的是数据路径

`forward` 不是简单把层罗列出来，而是定义数据从输入到输出的路径。每一步的输出都会成为下一步输入。你需要保证形状对齐，也要知道激活函数放在哪里。

例如 `x = self.sigmoid(self.linear1(x))` 先做线性变换，再把结果压到 0-1。后面接 `linear2` 时，`linear2` 的输入特征数必须等于 `linear1` 的输出特征数。

你现在应该能逐行解释一个多层 `forward`：这一行读入什么形状，输出什么形状，为什么下一层能接上。

读到这里，你应该能：区分样本读取、batch 组织和模型训练三件事。

5. 特征尺度会影响训练

多维特征常有不同单位和范围。一个特征可能在 0-1，另一个可能在几百。未经处理的尺度差异会影响梯度大小和优化稳定性，让某些特征在训练中占据过大影响。

课程重点是结构，但真实项目里通常要做标准化或归一化。标准化不是为了让数据好看，而是让优化过程更稳定，让不同特征在相近尺度上参与训练。

你现在应该能在训练不稳定时想到检查输入特征尺度，而不是只改网络层数。

读到这里，你应该能：把训练 step 拆成前向、loss、清梯度、反向和更新。

6. 从单层到多层，调试重点转向 shape

多层网络的 bug 很多来自 shape。输入列数、隐藏层宽度、输出维度、标签形状，每一处都要对上。打印形状是最有效的调试方法之一。

不要害怕在 `forward` 里临时打印 `x.shape`。等结构稳定后再删掉。复杂网络中，形状推导是读代码的基本功。

你现在应该能用表格写出一个网络的层、输入形状、输出形状和激活函数。

读到这里，你应该能：用这一小节的判断规则去检查相关代码。

完整知识链路

从单特征线性模型扩展到多特征、多层网络。 它仍然落在同一条 PyTorch 主线上：先准备能被模型读取的数据，再让模型输出符合任务的张量，用合适的损失函数得到标量 loss，反向传播写入梯度，优化器更新参数，最后在评估模式下检查结果。换任务时，最先变化的是输入形状、模型输出含义和 loss 输入约定。

术语表

别混词
术语直觉与定义常见误解
batch 维一次送入模型的样本数量维度。误以为它是特征数。
特征维每个样本包含的输入变量数量。和 batch 维混淆。
隐藏层位于输入和输出之间的可学习变换层。以为层越多一定越好。
激活函数给网络引入非线性的函数。认为多层线性层本身就足够非线性。

易混概念对照

概念准确含义不要混成
batch size一次处理多少样本不是每个样本有多少列
in_features每个样本输入特征数不是样本总数
out_features该层输出的新特征数不是类别标签本身

代码与例子

逐行读语义

读代码时先标出输入、输出和状态变化。能解释这些，才算真正看懂。

多维输入的三层网络示例代码
class DiabetesModel(nn.Module):
    def __init__(self):
        super().__init__()
        self.linear1 = nn.Linear(8, 6)
        self.linear2 = nn.Linear(6, 4)
        self.linear3 = nn.Linear(4, 1)

    def forward(self, x):
        x = torch.relu(self.linear1(x))
        x = torch.relu(self.linear2(x))
        return self.linear3(x)

误区诊断

错因 / 后果 / 修法

混淆 batch 和 feature

后果：Linear 的 `in_features` 写错。

修法：看输入最后一维。

层间维度不连续

后果：矩阵乘法形状报错。

修法：上一层 out_features 等于下一层 in_features。

多层线性不加激活

后果：整体仍等价于一层线性。

修法：隐藏层之间加入非线性激活。

忽略特征尺度

后果：训练可能慢或不稳定。

修法：对输入做标准化或归一化。

输出维度和任务不符

后果：loss 和标签对不上。

修法：回归/二分类/多分类分别检查输出维度。

把训练循环当成背诵模板

后果：模板能跑不等于能诊断。每一行都要知道它读什么、写什么、改变哪个状态。

修法：把每行标成前向、loss、清梯度、反向或更新。

只看 loss 下降

后果：loss 下降只能说明优化目标变小，还要检查数据、标签、输出形状和评估指标是否对应任务。

修法：同时检查验证集指标和预测样例。

混用训练和评估状态

后果：`model.train()` 和 `model.eval()` 会影响 Dropout、BatchNorm 等层，评估时还要配合 `torch.no_grad()`。

修法：训练和评估写成两个清楚的函数。
