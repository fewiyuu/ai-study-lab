# 概念骨架 {#map}
<!-- tag: 先搭主线 -->

NCHW→卷积→通道→池化→展平→分类头

## box info
**先看输入**
确认数据形状、标签形式和 dtype。许多训练错误在进入模型前已经发生。


**再看变换**
逐层追踪输出形状和语义，别只看 API 名字。每一层都应该能回答“它把什么变成什么”。


**最后看目标**
确认 loss 的输入约定、输出是否是标量，以及评估指标是否对应真实任务。
判断句：如果你说不清某一行读入什么、输出什么、改变什么状态，就先不要急着调参。

**无 padding、stride=1 时**：H_out = H_in - kernel_size + 1
通道数由 `out_channels` 决定，池化通常只改变高宽。

# shape 追踪 {#trace}
<!-- tag: 先算再写 -->

| 步骤 | 读什么 | 写什么 | 先查什么 |
| --- | --- | --- | --- |
| 输入 | [N,1,28,28] | 单通道图像 | batch 和通道是否对齐 |
| conv1 | [N,10,24,24] | 10 个特征图 | kernel_size 是否和高宽变化一致 |
| pool1 | [N,10,12,12] | 缩小空间 | 池化只改高宽，不改通道 |
| conv2 | [N,20,8,8] | 更厚的特征图 | 下一层 in_channels 是否接上 |
| flatten | [N,320] | 喂给 Linear | 展平后的长度是否和 fc 对上 |

## box info
### 读法
先把输入写成 `[N,C,H,W]`，再沿着卷积、池化、展平、Linear 依次写输出。只要你能把每一步的 shape 算出来，很多报错还没跑到训练就能被发现。

## box warn
### 故障点
最常见的错，是池化后还按原尺寸算 `Linear`，或者把下一层卷积的 `in_channels` 继续写成 1。先看 shape，再看参数名，修法会快很多。

## box soft
拿这一页做练习时，你可以把每层输出都写在草稿纸上：`[N,1,28,28] -> [N,10,24,24] -> [N,10,12,12] -> [N,20,8,8] -> [N,320]`。这条链路写顺了，后面换成别的输入尺寸也知道该从哪一层重新算。

# 讲义 {#lecture}
<!-- tag: 慢读这一部分 -->

## box soft
### 1. CNN 先解决全连接丢空间结构的问题
全连接 MNIST 会把 `[1,28,28]` 展平成 784 个数。这样能训练，但原来相邻像素的空间关系被打散了。图像任务里，局部结构很重要，例如边缘、角点、笔画和纹理都出现在局部区域。
卷积层直接在二维空间上滑动卷积核。它不需要把图像一开始就拉直，而是在局部窗口里提取特征。这样模型能更自然地利用相邻像素之间的关系。
你现在应该能解释：CNN 为了保留并利用图像的空间结构。
读到这里，你应该能：用这一小节的判断规则去检查相关代码。

## box soft
### 2. PyTorch 图像输入默认是 NCHW
PyTorch 的 `Conv2d` 常见输入形状是 `[N, C, H, W]`。`N` 是 batch size，`C` 是通道数，`H` 和 `W` 是高度和宽度。MNIST 灰度图通道数是 1，彩色 RGB 图像通道数是 3。
很多教材或图片库会以 HWC 表示图像，即 `[H, W, C]`。进入 PyTorch 卷积前要转成 CHW，再由 DataLoader 组成 NCHW。通道维放错会导致形状报错或模型学到错误含义。
你现在应该能看到 `[64,1,28,28]` 并读成 64 张单通道 28×28 图像。
读到这里，你应该能：用这一小节的判断规则去检查相关代码。

## box soft
### 3. 卷积层会改变通道数和空间尺寸
`nn.Conv2d(in_channels, out_channels, kernel_size)` 中，`in_channels` 必须和输入通道数一致，`out_channels` 是卷积核组数，也就是输出特征图通道数。卷积核大小、padding、stride 会影响输出高宽。
例如 MNIST 输入 `[N,1,28,28]` 经过 `Conv2d(1,10,kernel_size=5)`，如果不 padding，空间尺寸会从 28 变成 24，输出是 `[N,10,24,24]`。通道变成 10，是因为有 10 个卷积核组在提取不同特征。
你现在应该能用 `in_channels`、`out_channels`、`kernel_size` 解释卷积层输出为什么会变。
读到这里，你应该能：用这一小节的判断规则去检查相关代码。

## box soft
### 4. 池化减少空间尺寸，不改变通道数
池化层常用于降低特征图的空间尺寸，减少计算量，也让特征对局部小位移更稳。`MaxPool2d(2)` 通常会把高宽减半，例如 `[N,10,24,24]` 变成 `[N,10,12,12]`。
注意池化一般不改变通道数。它是在每个通道内部做下采样，不会把 10 个通道变成 20 个通道。通道变化主要由卷积层的 `out_channels` 决定。
你现在应该能区分卷积和池化：卷积学特征并可改通道，池化压缩空间尺寸。
读到这里，你应该能：用这一小节的判断规则去检查相关代码。

## box soft
### 5. 卷积后接全连接要算展平维度
CNN 最后做分类时，常把卷积和池化后的特征图展平，再送入全连接层。这里最容易错的是 `Linear` 的输入维度。你必须根据前面每个卷积和池化后的形状，算出 `channels * height * width`。
如果前面两次 5×5 卷积和 2×2 池化，MNIST 尺寸可能从 28 到 24 到 12，再到 8 到 4。若最后通道是 20，展平维度就是 `20*4*4=320`。这就是很多示例里 `Linear(320, 10)` 的来源。
你现在应该能手算一个 CNN 分类头的输入维度，而不是把 320 当成神秘数字背下来。
读到这里，你应该能：拿到一段代码后先检查输入、输出和层间维度是否对齐。

## box soft
### 6. CNN 训练循环仍然没变
换成 CNN 后，训练循环仍然是 DataLoader 提供 batch，模型前向输出 logits，CrossEntropyLoss 计算 loss，zero_grad、backward、step 更新参数。变的是模型内部结构和输入形状，不是训练主线。
这也是课程反复强调四步流程的原因。你可以换线性层、卷积层、池化层，但数据、模型、损失、优化、评估这条链不变。
你现在应该能把 CNN 放回通用训练链路里，不会因为多了卷积层就忘记 loss 和 optimizer 的分工。
读到这里，你应该能：把训练 step 拆成前向、loss、清梯度、反向和更新。

## box soft
### 完整知识链路
从展平图像转向利用空间结构的卷积网络。 它仍然落在同一条 PyTorch 主线上：先准备能被模型读取的数据，再让模型输出符合任务的张量，用合适的损失函数得到标量 loss，反向传播写入梯度，优化器更新参数，最后在评估模式下检查结果。换任务时，最先变化的是输入形状、模型输出含义和 loss 输入约定。

# 术语表 {#terms}
<!-- tag: 别混词 -->

| 术语 | 直觉与定义 | 常见误解 |
| --- | --- | --- |
| NCHW | PyTorch 卷积常用图像 batch 形状：样本数、通道、高、宽。 | 和 HWC 混淆。 |
| 卷积核 | 在局部窗口上提取特征的可学习权重。 | 误以为它只是在压缩图片。 |
| 特征图 | 卷积层输出的多通道空间表示。 | 把它当成单张普通图片。 |
| 池化 | 对每个通道做空间下采样。 | 误以为会改变通道数。 |

### 易混概念对照

| 概念 | 准确含义 | 不要混成 |
| --- | --- | --- |
| 卷积 | 提取局部特征并可改通道数 | 不是分类器全部 |
| 池化 | 压缩空间尺寸 | 通常不学参数 |
| 全连接头 | 把特征映射到类别 logits | 不保留空间结构 |

# 代码与例子 {#examples}
<!-- tag: 逐行读语义 -->

读代码时先标出输入、输出和状态变化。能解释这些，才算真正看懂。

基础 CNN 结构示例代码

```python
class CnnNet(nn.Module):
 def __init__(self):
 super().__init__()
 self.conv1 = nn.Conv2d(1, 10, kernel_size=5)
 self.pool = nn.MaxPool2d(2)
 self.conv2 = nn.Conv2d(10, 20, kernel_size=5)
 self.fc = nn.Linear(320, 10)

 def forward(self, x):
 x = self.pool(torch.relu(self.conv1(x))) # [N,10,12,12]
 x = self.pool(torch.relu(self.conv2(x))) # [N,20,4,4]
 x = x.view(x.size(0), -1)
 return self.fc(x)
```

# 误区诊断 {#pitfalls}
<!-- tag: 错因 / 后果 / 修法 -->
<!-- layout: grid2 -->

## box warn
### 把 NCHW 写成 NHWC
**后果：**Conv2d 读错通道维或直接报错。
**修法：**确认输入是 `[N,C,H,W]`。

## box warn
### conv2 的 in_channels 写错
**后果：**第二个卷积层接不上第一个输出。
**修法：**上一层 out_channels 等于下一层 in_channels。

## box warn
### 池化后还按原尺寸算 Linear
**后果：**全连接层输入维度报错。
**修法：**逐层计算 H、W。

## box warn
### 认为池化会改通道数
**后果：**形状推导错误。
**修法：**池化通常只改 H、W。

## box warn
### 忘记展平
**后果：**Linear 收到 4D 特征图。
**修法：**进入全连接前 `view(x.size(0), -1)`。

## box warn
### 把训练循环当成背诵模板
**后果：**模板能跑不等于能诊断。每一行都要知道它读什么、写什么、改变哪个状态。
**修法：**把每行标成前向、loss、清梯度、反向或更新。

## box warn
### 只看 loss 下降
**后果：**loss 下降只能说明优化目标变小，还要检查数据、标签、输出形状和评估指标是否对应任务。
**修法：**同时检查验证集指标和预测样例。

## box warn
### 混用训练和评估状态
**后果：**`model.train()` 和 `model.eval()` 会影响 Dropout、BatchNorm 等层，评估时还要配合 `torch.no_grad()`。
**修法：**训练和评估写成两个清楚的函数。

# 完整样例 {#worked-example}
<!-- tag: 一条完整教学链 -->

## box soft
### 完整样例：一张 28×28 图像怎么走到分类头
卷积网络最容易乱的地方，NCHW 和通道数。拿 `[32, 1, 28, 28]` 这一批图像来说，Conv2d 先读输入通道，再决定输出通道和空间尺寸。你看到的“特征图换了一种组织方式”。
池化层的任务更窄：它主要缩小高宽，不负责改类别数。很多报错都来自把池化和卷积混成一类操作，结果把通道数和空间尺寸一起算错。真正该盯的是每一步后张量的 shape，而不是层名字本身。
到了分类头，所有特征图都要先摊平。`view(x.size(0), -1)` 的意思保住 batch，把剩下的空间和通道合成一条特征轴。这样 `Linear` 才能继续读。
如果把图像先展平再送进 Conv2d，空间结构会被抹掉；如果忘了改 `in_channels`，第二层卷积会直接读错输入。读 CNN 时，先把 shape 账本写对，再去看特征语义，顺序不能反。

## box info
### 输入和输出怎么对齐
先把输入 shape、标签 dtype、输出维度和训练目标写成同一张账本，后面再看 API。

## box warn
### 最常见的失配
这类页最常见的问题输入和输出的约定写反。先查 shape，再查 dtype，最后才查 loss。

对象shape / 语义要读什么常见失配输入[32, 1, 28, 28]灰度图像批次先看通道是 1卷积后[32, 10, 24, 24]10 个特征图高宽缩小 4池化后[32, 10, 12, 12]更粗的空间信息通道不变展平后[32, 1440]Linear 输入把空间和通道折叠

最小可执行片段复制

```python
import torch

x = torch.randn(32, 1, 28, 28)
x = conv1(x)
x = pool(x)
x = x.view(x.size(0), -1)
logits = head(x)
print(x.shape, logits.shape)
```

### 故障现场
如果你看到训练能跑但结论不稳，先回到这个样例把 logits、标签和 shape 再核一次。

# 故障链 {#bridge-chain}
<!-- tag: 从症状回到接口 -->

## box soft
### 第二条教学链：把 shape 账本记到分类头
CNN 的调试，第一步永远“每一层后的 shape 有没有按预期变化”。只要输入是 `[N, C, H, W]`，那就先看 `C` 有没有被卷积层接住，再看 `H` 和 `W` 有没有被 kernel、stride、padding 改掉。高宽的变化能不能解释通，是判断网络结构是否接上的最快办法。
很多人会把池化层当成“也能改通道”的东西，这会把整条链路直接算歪。池化的职责很窄，它只负责压缩空间维；卷积才是把通道从 1 变成 10、从 10 变成 20 的地方。等你把这两个动作拆开，再去读 `conv -> relu -> pool -> flatten -> linear`，每一步就都有明确目的，不会再看成机械堆层。
对于分类头来说，`view(x.size(0), -1)` 把空间轴交给 Linear 的那一步。你可以把它理解成“把一块特征图整理成一条特征向量”。如果这一步做错，后面的线性层就会拿错输入，错误信息常常表现为维度不匹配，而不是语义报错。
在真正的图像任务里，你最好养成一个小习惯：每一个 block 后都打印一次 shape，至少在第一次实现时这样做。先确认通道和空间尺寸，再确认分类头，再看 loss。这个顺序能避免最常见的误判，也能让你读陌生 CNN 时更快分辨哪些地方是在提特征，哪些地方只是在降尺寸。

症状你先查什么为什么先查它修法卷积前[N, 1, 28, 28]原始图像先看通道是否接对卷积后[N, 10, 24, 24]特征图变厚通道和空间同时改变池化后[N, 10, 12, 12]更粗的格局只压缩高宽展平后[N, 1440]喂给 Linear把卷积输出整理成向量

故障排查片段复制

```python
def forward(self, x):
 print("input:", x.shape)
 x = self.conv1(x)
 print("after conv1:", x.shape)
 x = self.pool(x)
 print("after pool:", x.shape)
 x = x.view(x.size(0), -1)
 return self.fc(x)
```

# 一致性补充 {#consistency-note}
<!-- tag: 把接口写稳 -->

## box info
### 学习时的落点
这一段把训练、评估、调试和迁移时应该固定住的检查顺序收束成一张表。
卷积网络最值得练习的，把每一步的张量形状写明白。你拿到输入以后，先记住它是 `[N, C, H, W]`，然后沿着卷积、池化、展平、Linear 的顺序去看。只要这条顺序不乱，CNN 的代码通常就很好读。
调试 CNN 时，最有效的动作往往打印中间 shape。你要知道 Conv2d 改的是通道和空间，MaxPool2d 主要改空间，Linear 才看最后的特征轴。如果这三类动作被你混成一类，报错信息会很像，但修法完全不同。
很多人第一次写 CNN，会在分类头前面漏掉展平。这个错误的本质忘了把卷积输出整理成向量。把特征图摊平这一动作看成一个接口转换，你就会更容易理解为什么 `view(x.size(0), -1)` 这行不能省。
还有一个容易漏掉的点，是不同卷积层的通道必须接上。第一层输出 10 个通道，第二层就要读 10 个通道；如果第二层还写成 1，模型就像把上一层的结果当成另一种数据看。这个错经常在报错中直接体现为通道数不匹配。
做题时你可以按三个层次回答：先说 shape 怎么变，再说每一步的职责，最后说哪里最容易错。这样写出来的答案比只写一句‘卷积提特征、池化降尺寸’更能拿来真正调试。
如果你想把 CNN 的知识真的用起来，最好每次先在纸上写一遍 `NCHW -> Conv -> Pool -> Flatten -> Linear`。这个小动作能把很多模糊的直觉变成可检查的顺序，也会让你在读开源代码时更快找到 block 的边界。
卷积并不是单纯把图像变小，它更像是在一条固定格式的张量账本上，连续改写通道和空间尺寸。你只要把每一步写成‘输入 shape -> 输出 shape -> 负责什么’，就会发现很多看似复杂的 CNN 其实只是同一种模式重复。
如果训练结果一直怪，先回到最前面看输入是不是灰度还是 RGB，通道数有没有对上，最后一层是不是和类别数一致。这个从头到尾的检查，比在中间层里盲目试错更靠谱。

场景先看什么为什么下一步输入[N, C, H, W]先看通道别把 batch 和通道混了卷积改通道 / 高宽核大小和 stride输出 shape 要算清池化压缩高宽下采样不负责改变类别数Linear最后一维分类前向量忘记展平就会卡住

收束后的检查脚本复制

```python
x = torch.randn(16, 1, 28, 28)
print("start", x.shape)
x = self.conv1(x)
print("conv1", x.shape)
x = self.pool(x)
print("pool", x.shape)
x = x.view(x.size(0), -1)
logits = self.fc(x)
print("logits", logits.shape)
```

### 1 选择

**Q:** 1. PyTorch Conv2d 常见输入形状是哪一种？

- [ ] [N,C,H,W]
- [ ] [N,H,W,C]
- [ ] [C,N,H,W]

Answer: [N,C,H,W]
Explain: 按 PyTorch 约定。

### 2 判断

**Q:** 2. MaxPool2d(2) 通常会改变通道数。

Explain: 错误。 MaxPool2d(2) 通常只在每个通道内部做下采样，把高宽减小；通道数由卷积层的 out_channels 决定，池化不会把 10 个通道变成别的数量。

### 3 填空

**Q:** 3. RGB 图像的输入通道数通常是 ______。

Explain: 通常是 3。灰度图像常写成 [N,1,H,W] ，RGB 图像有红、绿、蓝三个通道，所以第一层卷积的 in_channels 应改成 3。

### 4 短答

**Q:** 4. `[64,1,28,28]` 经过 `Conv2d(1,10,5)` 无 padding 后形状是什么？

Explain: 形状是 [64,10,24,24] 。batch 仍是 64，输出通道由 out_channels=10 决定；无 padding、kernel size 为 5 时，高宽从 28 变成 28-5+1=24 。

### 5 短答

**Q:** 5. `[N,10,24,24]` 经过 `MaxPool2d(2)` 后形状是什么？

Explain: 形状是 [N,10,12,12] 。 MaxPool2d(2) 把 24x24 的空间尺寸减半到 12x12，batch 和通道数保持不变。

### 6 短答

**Q:** 6. `x.view(x.size(0), -1)` 在 CNN 里做什么？

Explain: 它保留 batch 维度 x.size(0) ，把后面的通道、高、宽合并成一个特征向量。也就是把 [N,C,H,W] 变成 [N,C*H*W] ，这样才能送进 Linear 。

### 7 短答

**Q:** 7. 为什么第二个卷积层 `in_channels` 要等于第一个卷积层 `out_channels`？

Explain: 因为第一层卷积输出的张量会成为第二层卷积的输入。若 conv1 的 out_channels=10 ，那么进入 conv2 的通道数就是 10， conv2.in_channels 也必须是 10。

### 8 诊断

**Q:** 8. 报错说 expected input to have 10 channels, but got 1，可能哪里写错？

Explain: 常见错因是第二个卷积层的 in_channels 写成了 10，但实际送进去的张量仍只有 1 个通道，或者 forward 顺序把 conv2 直接接到了原图上。先打印进入 conv2 前的 shape，再让 conv2.in_channels 和上一层输出通道一致。

### 9 诊断

**Q:** 9. Linear 输入维度写错，通常如何排查？

Explain: 在进入 Linear 前打印展平前 shape，例如 [N,C,H,W] ，再计算 C*H*W 是否等于 Linear(in_features,...) 。如果中间改了卷积核、padding、stride 或 pool，分类头的输入维度也要跟着重算。

### 10 诊断

**Q:** 10. 把图像先展平再送 Conv2d，会有什么问题？

Explain: Conv2d 需要看到 4D 的 [N,C,H,W] ，因为卷积核要沿高宽滑动并在通道上读局部区域。先展平会把空间结构摊成一维，卷积层既拿不到 H/W 维度，也无法学习局部图像模式。

### 11 场景

**Q:** 11. 把 MNIST 换成 RGB 32×32 图片，第一层 in_channels 该怎么改？

Explain: 第一层应从 Conv2d(1,...) 改成 Conv2d(3,...) 。同时 32x32 会改变后续高宽，若分类头写死了 in_features ，还要重新打印展平前 shape 并更新 Linear 的输入维度。

### 12 场景

**Q:** 12. 为什么卷积适合图像？

Explain: 图像里的局部像素有空间关系，卷积核可以在局部窗口里提边缘、纹理等模式；同一组权重在不同位置共享，能减少参数并保留平移相关的结构信息。全连接一开始就把空间摊平，通常更难利用这些局部规律。

### 13 匹配

**Q:** 13. 匹配 Conv / Pool / Linear：提特征、降尺寸、分类输出。

Explain: Conv 主要提局部特征并改变通道语义； Pool 主要降低高宽、保留更粗粒度信息； Linear 接收展平后的特征向量，输出每个类别的分数。

### 14 短答

**Q:** 14. 特征图和原图有什么区别？

Explain: 原图的通道通常是可见颜色或灰度值；特征图是卷积核计算出的中间表示，通道数可能变多，每个通道对应某类局部模式的响应。它仍保留空间位置，但语义已经从像素值变成了特征响应。

### 15 短答

**Q:** 15. CNN 相比全连接处理图像的优势是什么？

Explain: CNN 的优势在于保留空间结构、使用局部连接和权重共享。它不用为每个像素位置单独学一套全连接权重，而是让同一个卷积核在整张图上找相似模式，因此参数更少，也更适合图像的局部规律。

### 16 短答

**Q:** 16. 为什么卷积层参数比全连接少？

Explain: 卷积层的参数量由 out_channels * in_channels * kernel_h * kernel_w 决定，和图像上有多少个空间位置无关，因为同一个卷积核会在不同位置重复使用。全连接层通常要为每个输入位置到每个输出单元都学习独立权重。

### 19 费曼

**Q:** 19. 用 6-9 句话解释：CNN 如何把一张 MNIST 图像变成十类 logits。

Explain: 一张 MNIST 图像进入模型时通常是 [N,1,28,28] 。第一层卷积读取 1 个输入通道，用多个卷积核生成多张特征图。激活函数保留非线性响应，池化把高宽缩小，让后续层看到更粗的空间结构。后面的卷积继续把低级边缘组合成更高层特征。进入分类头前， view 或 reshape 会把 [N,C,H,W] 展平成 [N,C*H*W] 。 Linear 根据这个向量输出 10 个 logits。训练时 CrossEntropyLoss 直接读取 logits 和类别标签，不需要先手动 softmax。
