# 时间步追踪 {#trace}
<!-- tag: 沿 hidden 走一遍 -->

| 步骤 | 读什么 | 写什么 | 为什么要看 |
| --- | --- | --- | --- |
| 输入序列 | [batch, seq_len, input_size] | 每步向量 | 先把维度分清 |
| RNNCell | x_t 和 h_{t-1} | h_t | 单步更新隐藏状态 |
| RNN | 整段序列 | 所有时间步输出和最后 hidden | 把循环封装起来 |
| 最后 hidden | 整段信息压缩 | 分类头输入 | 序列分类常用它 |

## box info
读法
`seq_len` 是时间步数，不是 batch；`hidden_size` 是状态长度，不是类别数。先把这三个词分开，再读代码。

# 讲义 {#lecture}
<!-- tag: 慢读这一部分 -->

## box soft
### 1. RNN 先解决顺序依赖问题
全连接网络和 CNN 主要处理固定形状输入。序列数据的特点是顺序有意义，比如一句话、一个名字、时间序列。第 1 个字符和第 5 个字符不是可随便打乱的列，它们的位置影响整体含义。
RNN 的基本想法是逐步读序列。每个时间步接收当前输入和上一步隐藏状态，输出新的隐藏状态。隐藏状态像模型对已经读过内容的压缩记忆。
你现在应该能解释：RNN 沿时间步不断更新隐藏状态。
读到这里，你应该能：用这一小节的判断规则去检查相关代码。

## box soft
### 2. 隐藏状态是序列信息的载体
隐藏状态 `h_t` 保存到当前时间步为止的信息。第 t 步会读取 `x_t` 和 `h_{t-1}`，生成 `h_t`。如果任务只需要整个序列的分类结果，常用最后一个隐藏状态做分类。
隐藏状态不是标签，也不是 loss。它是模型内部表示，会随着序列输入一步步变化。不同 batch、不同层数、不同方向的 RNN，隐藏状态形状也不同。
你现在应该能把 `input_t`、`hidden_prev`、`hidden_next` 三者的关系讲清楚。
读到这里，你应该能：沿时间步追踪输入、隐藏状态和最终输出。

## box soft
### 3. RNNCell 展示单步，RNN 封装整段序列
`nn.RNNCell` 只处理一个时间步，你需要自己写循环，把序列按时间步喂进去。`nn.RNN` 则一次接收整段序列，内部帮你循环，返回所有时间步输出和最后隐藏状态。
RNNCell 适合教学，因为它把时间展开过程暴露出来。真实项目里更常用 `nn.RNN`、`nn.GRU` 或 `nn.LSTM`，因为它们封装更完整，也更高效。
你现在应该能区分：Cell 是一步，RNN 是一串时间步。
读到这里，你应该能：沿时间步追踪输入、隐藏状态和最终输出。

## box soft
### 4. 形状约定是 RNN 最大门槛之一
PyTorch RNN 默认输入形状是 `[seq_len, batch, input_size]`。如果设置 `batch_first=True`，输入形状就是 `[batch, seq_len, input_size]`。很多报错都来自这两个约定混用。
隐藏状态形状通常是 `[num_layers * num_directions, batch, hidden_size]`。如果是单层单向 RNN，就是 `[1, batch, hidden_size]`。这和输入的 batch 维必须一致。
你现在应该能在写 RNN 前先决定是否使用 `batch_first=True`，并据此检查输入和 hidden shape。
读到这里，你应该能：拿到一段代码后先检查输入、输出和层间维度是否对齐。

## box soft
### 5. one-hot 输入高维稀疏，后续会引入 embedding
字符或词不能直接进入神经网络，需要先变成数值向量。最朴素方法是 one-hot：词表有多大，向量就有多长，目标字符位置为 1，其余为 0。
one-hot 容易理解，但维度高、稀疏，而且不同字符之间没有可学习的相似性。下一讲会用 embedding 把离散符号映射成低维稠密向量。
你现在应该能说明：one-hot 是把类别符号变成向量的起点，但不是处理自然语言的最终方案。
读到这里，你应该能：拿到一段代码后先检查输入、输出和层间维度是否对齐。

## box soft
### 6. 序列分类与逐步输出的区别
有些序列任务每个时间步都需要输出，例如语言模型预测下一个字符；有些任务只需要整个序列一个输出，例如名字分类。两类任务使用 RNN 的方式不同。
如果只做序列分类，可以取最后隐藏状态接线性层；如果每步都要预测，就要对每个时间步输出做映射并计算损失。不要把所有 RNN 任务都写成同一种输出形式。
你现在应该能判断一个序列任务需要“每步输出”还是“整段输出”。
读到这里，你应该能：沿时间步追踪输入、隐藏状态和最终输出。

## box soft
### 完整知识链路
从固定长度图像转向按时间展开的序列数据。 它仍然落在同一条 PyTorch 主线上：先准备能被模型读取的数据，再让模型输出符合任务的张量，用合适的损失函数得到标量 loss，反向传播写入梯度，优化器更新参数，最后在评估模式下检查结果。换任务时，最先变化的是输入形状、模型输出含义和 loss 输入约定。

# 代码与例子 {#examples}
<!-- tag: 逐行读语义 -->

读代码时先标出输入、输出和状态变化。能解释这些，才算真正看懂。

RNNCell 手写时间循环示例代码

```python
cell = nn.RNNCell(input_size=4, hidden_size=8)
hidden = torch.zeros(batch_size, 8)

for t in range(seq_len):
 x_t = sequence[:, t, :] # batch_first=True 的手写序列
 hidden = cell(x_t, hidden)
```

nn.RNN 封装整段序列示例代码

```python
rnn = nn.RNN(input_size=4, hidden_size=8, batch_first=True)
output, hidden = rnn(sequence)
# output: [batch, seq_len, hidden_size]
# hidden: [num_layers, batch, hidden_size]
```

错误示例：混用形状约定错误示例

```python
# 错误示例
rnn = nn.RNN(input_size=4, hidden_size=8) # 默认不是 batch_first
sequence = torch.randn(32, 10, 4) # 这是 [batch, seq, feature]
output, hidden = rnn(sequence) # RNN 会把 32 当 seq_len
```

# 误区诊断 {#pitfalls}
<!-- tag: 错因 / 后果 / 修法 -->
<!-- layout: grid2 -->

## box warn
### 混用 batch_first
**后果：**RNN 把 batch 和 seq 读反，结果或 shape 都不对。
**修法：**明确设置并按约定准备输入。

## box warn
### 把 hidden 当输出类别
**后果：**隐藏状态还需要接分类层或解码层。
**修法：**按任务决定如何映射输出。

## box warn
### RNNCell 不写时间循环
**后果：**只处理了一个时间步。
**修法：**Cell 外部手动 for 循环。

## box warn
### 忽略 hidden 初始形状
**后果：**层数、方向、batch 不匹配。
**修法：**按 `[layers*directions,batch,hidden]` 准备。

## box warn
### 把序列打乱成普通特征
**后果：**顺序信息丢失。
**修法：**保留时间步维度。

## box warn
### 把训练循环当成背诵模板
**后果：**模板能跑不等于能诊断。每一行都要知道它读什么、写什么、改变哪个状态。
**修法：**把每行标成前向、loss、清梯度、反向或更新。

## box warn
### 只看 loss 下降
**后果：**loss 下降只能说明优化目标变小，还要检查数据、标签、输出形状和评估指标是否对应任务。
**修法：**同时检查验证集指标和预测样例。

## box warn
### 混用训练和评估状态
**后果：**`model.train()` 和 `model.eval()` 会影响 Dropout、BatchNorm 等层，评估时还要配合 `torch.no_grad()`。
**修法：**训练和评估写成两个清楚的函数。

# 完整样例 {#worked-example}
<!-- tag: 一条完整教学链 -->

## box soft
### 完整样例：RNN 一步一步读序列
RNN 最容易弄混的就是时间步和 batch。`batch_first=True` 时，输入形状通常写成 `[B, S, F]`，其中 `B` 是样本数，`S` 是时间步数，`F` 是每步的特征数。只要这三个符号不乱，RNN 代码就好读很多。
`nn.RNNCell` 只负责一小步：它读当前 `x_t` 和上一步 `h_{t-1}`，吐出新的 `h_t`。如果想读完整个序列，就要把它放进一个 for 循环。这个循环模型真正“沿时间展开”的地方。
序列分类通常只取最后一个 hidden 送进线性层。这个 hidden 里已经压进了前面所有时间步的信息，所以它更像摘要，而不是最终类别。真正的类别预测，还是要靠后面的分类头。
如果你把 `[B, S, F]` 和 `[S, B, F]` 互换了，RNN 会把 batch 和时间步读反，后续 hidden 形状、loss 对齐和评估逻辑都会跟着乱掉。先定约定，再写代码，是这讲最值钱的习惯。

## box info
### 输入和输出怎么对齐
先把输入 shape、标签 dtype、输出维度和训练目标写成同一张账本，后面再看 API。

## box warn
### 最常见的失配
这类页最常见的问题输入和输出的约定写反。先查 shape，再查 dtype，最后才查 loss。

输入序列[B,S,F]每步向量先分清 batch 和 seq单步 hidden[B,H]当前记忆来自上一时间步循环输出T 次更新逐步累计Cell 不会自己展开分类头[B,C]类别 logits最后一个 hidden 接 Linear对象shape / 语义要读什么常见失配

最小可执行片段复制

```python
import torch

hidden = torch.zeros(batch_size, hidden_size)
for t in range(seq_len):
 x_t = sequence[:, t, :]
 hidden = cell(x_t, hidden)
logits = head(hidden)
print(hidden.shape, logits.shape)
```

# 故障链 {#bridge-chain}
<!-- tag: 从症状回到接口 -->

## box soft
### 第二条教学链：把时间步和 batch 一起看
RNN 类题最容易错的地方，是你以为自己在看序列，实际上代码在看 batch。只要把 `[B, S, F]` 和 `[S, B, F]` 搞反，后面的 hidden、loss 和评估就会全部错位。学会在写代码前先选定一种输入约定，比记住任何一个 API 细节都重要。
`RNNCell` 和 `RNN` 的区别，可以用一句话说明白：前者只处理一个时间步，后者帮你把整段序列展开。这个区别会直接影响你有没有写循环。你如果只调用一次 Cell，等于只读了一个 token；你如果让 RNN 接整段序列，才是在读上下文。
序列分类和序列标注也要分开看。序列分类只需要最后一个 hidden；标注任务要每个时间步都出结果。两类任务的损失函数、输出 shape 和训练循环都不同。如果不先分任务，后面所有“为什么维度不对”的问题都会变成一团。
因此，这页最值得带走的一张时序账本：输入读哪一维、时间步怎么滚动、hidden 怎样传递、最后取哪一步做分类。把这张账本写出来，你再看任何 RNN 代码，都能知道是哪一层在读什么、哪一层在改什么。

症状你先查什么为什么先查它修法输入[B,S,F]序列批次先分清 batch 和时间步Cell 更新[B,H]一步状态只读当前 x_t 和上一步 hidden整段 RNN[S,B,H] 或 [B,S,H]所有时间步要先定 batch_first分类头[B,C]类别 logits只取最后 hidden 时常见

故障排查片段复制

```python
hidden = torch.zeros(batch_size, hidden_size)
for t in range(seq_len):
 x_t = sequence[:, t, :]
 hidden = cell(x_t, hidden)
if sequence.dim() != 3:
 raise ValueError("expect [B, S, F]")
logits = head(hidden)
```

# 一致性补充 {#consistency-note}
<!-- tag: 把接口写稳 -->

## box info
### 学习时的落点
这一段把训练、评估、调试和迁移时应该固定住的检查顺序收束成一张表。
RNN 的好处在于，它把‘顺序’这件事变成了显式的时间步更新。你只要把输入写成 `[B, S, F]`，后面每一轮循环都只做一件事：把当前 token 和上一步 hidden 合起来，得到下一步 hidden。这个过程非常机械，但正因为机械，才特别适合拿来排查 shape。
很多人看到 `hidden` 会下意识把它当成最终分类结果，这是不对的。hidden 只是压缩后的中间状态，真正的类别还要经过线性层。这个区分很重要，因为你一旦把摘要和答案混起来，训练和推理的接口就会一起乱。
RNNCell 和 RNN 的区别，也可以用调试视角理解。Cell 适合你想手动看每个时间步发生了什么；RNN 适合你已经确定输入约定，想直接让框架帮你展开。如果你读不懂某个序列模型，先把它拆回 Cell + for 循环，往往就能看清楚。
在序列分类里，最后一个 hidden 常常最有用，因为它已经把前面所有时间步的信息压缩过了。可是一旦任务变成序列标注，你就不能只看最后一步，而要看每个时间步的输出。任务一换，输出头和损失函数就会跟着换。
最容易出错的地方，还是 batch_first。你要么统一使用 `[B, S, F]`，要么统一使用 `[S, B, F]`，不要在一页里来回换。只要输入约定统一，hidden 的 batch 维也就自然能对齐。
如果你想真正掌握这一讲，最好自己写一次手动循环，再写一次 `nn.RNN` 的版本，比较两者的输入和输出。这样你会发现，所谓‘神经网络的循环’，其实就是同一条状态链被重复调用多次。

场景先看什么为什么下一步输入[B,S,F]时间步在中间batch_first 要统一Cell单步更新x_t + hidden只读一个时间点RNN整段展开返回序列摘要别把它和 Cell 混了输出头[B,C]类别 logits最后 hidden 再接 Linear

收束后的检查脚本复制

```python
hidden = torch.zeros(batch_size, hidden_size)
outputs = []
for t in range(seq_len):
 x_t = sequence[:, t, :]
 hidden = cell(x_t, hidden)
 outputs.append(hidden)
stacked = torch.stack(outputs, dim=1)
logits = head(stacked[:, -1, :])
```

### 4 短答

**Q:** 4. `[32,10,4]` 在 batch_first=True 下分别表示什么？

Explain: 在 batch_first=True 下， [32,10,4] 分别是 batch=32、seq_len=10、input_size=4。也就是一次喂 32 条序列，每条 10 个时间步，每个时间步有 4 个特征。

### 5 短答

**Q:** 5. 单层单向 RNN 的 hidden 形状通常是什么？

Explain: 通常是 [1,batch,hidden_size] 。第一维是层数乘方向数，单层单向就是 1；第二维对应 batch；第三维是隐藏状态长度。

### 6 短答

**Q:** 6. 手写 RNNCell 循环中，为什么每一步都更新 hidden？

Explain: 因为 RNN 的“记忆”就存在 hidden 里。每个时间步读取当前输入和上一步 hidden，生成新的 hidden；如果不更新，就无法把前面字符或时间点的信息传到后面。

### 7 短答

**Q:** 7. `output` 和 `hidden` 的区别是什么？

Explain: output 通常包含每个时间步的输出，例如 batch_first 下是 [batch,seq_len,hidden_size] ； hidden 是最后保留下来的隐藏状态，形状含层数/方向数，常用于整段序列分类。

### 8 诊断

**Q:** 8. RNN 默认输入却喂 `[batch,seq,feature]`，会有什么问题？

Explain: 如果没有设置 batch_first=True ，PyTorch RNN 默认把输入读成 [seq_len,batch,feature] 。直接喂 [batch,seq,feature] 会把 batch 和序列长度的位置读反，导致语义错乱或 hidden shape 对不上。

### 9 诊断

**Q:** 9. 隐藏状态 batch 维和输入 batch 不一致，会怎样？

Explain: 会触发 shape 错误。hidden 的 batch 维必须和当前输入 batch 一致，否则每条序列无法对应自己的隐藏状态。常见原因是最后一个 batch 较小但复用了上一批的 hidden。

### 10 诊断

**Q:** 10. 把名字字符顺序打乱再训练，可能损失什么信息？

Explain: 会损失顺序信息。RNN 正是通过时间步顺序更新 hidden 来捕捉前后关系；把字符顺序打乱后，模型看到的拼写模式、前后缀和局部组合都被破坏。

### 11 场景

**Q:** 11. 股票日序列和房屋特征表，哪个更自然适合 RNN？为什么？

Explain: 股票日序列更自然适合 RNN，因为日期顺序本身携带信息，今天的状态可能依赖前几天。房屋特征表通常是一组静态字段，没有天然时间步，直接用普通 MLP 或树模型更常见。

### 12 场景

**Q:** 12. 名字分类是每步输出还是整段输出？

Explain: 名字分类通常是整段输出：一个名字对应一个国家或类别。你可以读完整个字符序列后取最后 hidden，再接分类头；不需要每个字符都输出一个国家。

### 13 匹配

**Q:** 13. 匹配 input_size / seq_len / hidden_size：单步向量长度、时间步数、隐藏向量长度。

Explain: input_size 是单个时间步的输入向量长度； seq_len 是时间步数； hidden_size 是 RNN 内部隐藏向量的长度，也是输出特征维度的核心部分。

### 14 短答

**Q:** 14. RNNCell 和 RNN 的区别是什么？

Explain: RNNCell 只处理一个时间步，需要你手写循环并维护 hidden； RNN 接收整段序列，内部帮你沿时间步循环，直接返回所有时间步 output 和最后 hidden。

### 15 短答

**Q:** 15. 为什么 RNN 适合序列数据？

Explain: RNN 适合序列数据，因为它按时间步读取输入，并用 hidden 保存到目前为止的上下文。后面的输出可以依赖前面的信息，这比把每个时间点当成彼此独立更符合文本、语音和时间序列。

### 16 短答

**Q:** 16. one-hot 的问题是什么？

Explain: one-hot 维度高且稀疏，字符或词之间没有可学习的相似性：任意两个不同符号看起来都一样远。后面的 Embedding 会把离散 id 映射成可训练向量，缓解这个问题。

### 19 费曼

**Q:** 19. 用 6-9 句话解释：RNN 如何沿时间步处理序列。

Explain: RNN 把序列拆成多个时间步，每一步读取当前输入 x_t 和上一时刻 hidden。新的 hidden 会带着目前为止的上下文继续传给下一步。用 RNNCell 时，你要手写这个循环；用 nn.RNN 时，模块内部帮你循环。输入维度顺序要和 batch_first 设置一致。 output 保存每个时间步的结果， hidden 保存最后状态。做整段分类时，通常取最后 hidden 接 Linear。调试时先打印输入、hidden、output 的 shape。
