## 一、总体印象

这套学习系统达到了 **"扎实草稿到好课程之间"** 的水平。在 Agent 工程、系统协议、GPU 性能等**高信息密度领域**，页面质量接近可发布标准；但在**批量生产的边缘课程**（如 PyTorch 系列）中，存在明显的模板化缺陷。整体架构设计优秀，但执行层面的"逐页打磨"环节有明显滑坡。

---

## 二、做得好的地方（强信号）

### 1. 教学架构设计非常成熟

- **shape-tracer 和 terminal-lab 被高频率、高质量地使用**。Triton 页面用 shape-tracer 把 "pid → offsets → mask → 计算 → 写回" 的数据流可视化；Multi-Agent 页面用四闸门 shape-tracer 展示拆分决策流程。这些组件不是装饰，而是承担了核心教学职能。
- **"角色账本"（ledger）和 "读写表" 是工程类课程的教学利器**。MCP 页面用表格清晰划分 host/client/server/tool/result 的读/写/错/查职责，把抽象的协议分层变成可检查的对象。
- **边界控制意识强**。几乎所有样本都有明确的 "本页不追的内容" / "先别追" 章节，符合 quality-rubric 中"保护学习者不掉进兔子洞"的要求。Multi-Agent 页面甚至用表格列出"现在先掌握 / 先不展开 / 之后接到哪里"。

### 2. 来源支撑规范且公开友好

- **Agent 和 MCP 页面的来源引用非常扎实**。Multi-Agent 页面引用了 Anthropic 三篇工程文章 + OpenAI Agents SDK 说明 + 中文工程博客，且都标注了访问日期（2026-06-21）。
- **MCP 页面**明确区分了官方教程、API 文档和 BFCL 评测集，没有编造来源或混用未经验证的二手信息。
- 来源引用格式统一，使用公开链接而非私人路径，符合 quality-rubric 的"公开可读性"要求。

### 3. 练习设计有诊断意识

- **题型混合做得不错**：选择、判断、填空、排序、匹配、代码阅读、诊断、场景迁移、费曼解释、检查清单、协议设计等题型都有覆盖。
- **开放题有脚手架**：例如 Multi-Agent 的 "设计一个 3-Agent 方案" 题，要求按 "lead / researcher / verifier" 格式填写，并追问 "为什么值得拆"，而不是放任学习者漫无目的地写。
- **Triton 页面的失败实验室（terminal-lab）**把整除、尾块越界、max 归约 other 三个场景做成可切换的终端输出，这是**把诊断练习嵌入教学章节**的优秀实践。

### 4. 交互组件真正服务于教学

- **Triton 页面的二维 tile 可视化**有滑块控制 M、N、pid_m、pid_n，实时展示当前 program 负责的绿色 tile 和越界的黄色 mask。这是"让机制比静态文字更容易检查"的典范。
- **Claude Code 页面的 todo 状态切换**（合法计划 / 两个进行中 / 空任务）让学习者直接看到校验通过 vs 失败的区别。
- 进度条、复制按钮、代码高亮、行号、公式渲染等基础交互齐全且稳定。

### 5. 共享页面壳层整体同步良好

- 4/5 的样本严格使用 `study-page-v2` 共享壳层（`data-page-shell="study-page-v2"`），导航结构、进度条、导语块、章节卡片、代码样式、练习分组、导出控件高度一致。
- 课程链接（上一单元 / 下一单元 / 课程地图）在大多数页面中正常工作。
- 多页课程使用共享本地资产（`../assets/study-page.css`、KaTeX、Prism），没有重复内联大量 CSS/JS，维护性较好。

## 三、不足与提高空间（按严重程度排序）

### 🔴 严重问题 1：PyTorch 课程练习参考答案严重模板化（质量事故）

**位置**：`10_卷积神经网络基础` 练习区第 2~20 题  
**症状**：**17 道开放题的参考答案全部是同一句话**：

> "先把 NCHW、通道、高宽、展平和分类头的 shape 串起来，再点出卷积和池化的分工。"

这道题出现在：判断题（MaxPool2d 通道数）、填空题（RGB 通道数）、短答题（shape 推导）、诊断题（Linear 维度排查）、场景题（MNIST 换 RGB）、匹配题（Conv/Pool/Linear）、面试题（CNN 优势）、清单题（调试 5 项检查）、代码修正题（conv2 修正）、费曼解释题（MNIST 到 logits）。

**后果**：
- 学习者点击 "展开反馈" 后，发现答案与题目无关，直接摧毁信任。
- 这违反了 quality-rubric 的"红旗"条款："用练习数量替代诊断价值"、"学习者能答完所有题，却仍不知道自己对不对"。
- 更违反了 study-page-quality 的核心要求："开放题应包含参考答案、评分要点"、"反馈应命名误解和下一步检查动作"。

**根因**：这是**批量生成时代理未逐题替换参考答案占位符**的典型症状。说明 PyTorch 系列页面在 "厚第一版，一次打磨" 环节被跳过，或验证器未覆盖参考答案内容检查。

**修复优先级**：最高。这批页面的练习区目前处于"不可用"状态。

### 🔴 严重问题 2：Claude Code 工作流页面壳层完全脱离共享模板

**位置**：`05_行动之前先规划`  
**症状**：
- 没有使用 `study-page-v2` 共享壳层，而是**全部内联 CSS/JS**（411 行中前 137 行是 `<style>`，后 120 行是 `<script>`）。
- 使用了完全不同的视觉风格：纸色背景（`#d8d4cc`）、网格底纹、深灰色侧边栏（`#232320`）、柠檬黄强调色（`#d7ef66`）、衬线字体标题。
- **缺少"返回课程地图"链接**，也没有上下单元导航。
- 导航标题使用英文（"Todo planning"），与其他课程中文导航不一致。

**后果**：
- 这直接违反了 quality-rubric 的"红旗"："未经用户明确指示，出现一次性配色、无关深色页面壳层、杂志导语或仪表盘布局"。
- 学习者从其他 Claude Code 页面跳转到此页时，会感觉自己进入了另一个网站。
- 如果这套课程需要统一维护侧边栏或全局样式，此页将成为"孤儿"，需要手动同步。

**根因**：可能是早期实验页面、子代理未加载共享模板契约、或用户曾要求特殊风格但未被隔离标记。

**修复优先级**：高。要么迁移到共享壳层，要么明确标记为例外并隔离维护。

### 🟡 中等问题 3：Triton 页面存在渲染错误和过度冗余文本

**位置**：`05_Triton 编程模型与块级思维`  
**症状**：
- 性能参数章节中，`<code>BLOCK_SIZE</code>` 被错误渲染为 `&lt;code&gt;BLOCK_SIZE&lt;/code&gt;`（出现在正文第 292~293 行）。这是 HTML 实体编码错误，说明内容在写入时经过了双重转义。
- "失败实验室" 下方的 "尾块链路" 章节文本**过度冗长**：用 5 段话反复讲同一件事（尾块诊断），最后一段甚至出现了 3 个 "p" 标签连续堆叠同一个观点（"块大小不是单调的"）。

**后果**：
- 渲染错误影响可读性，技术学习者会怀疑页面其他代码的准确性。
- 冗长文本稀释了信息密度，违背了 study-page-quality 的"反填充规则"："不要仅为满足数量而添加低信息量章节"。

**修复优先级**：中等。渲染错误必须修；冗长文本需要编辑瘦身。

## 四、系统性问题分析

### 1. "批量生成 → 逐页打磨" 的链条断裂

SKILL.md 明确强调："厚第一版，一次打磨"、"批处理脚本可以帮助创建第一版草稿，但最终质量来自页面级编辑审阅和直接 HTML 打磨"。

从 PyTorch 页面 17 道题共享同一参考答案的现象来看，**批量生成后的逐页编辑环节被跳过了**。这很可能是由于：
- 工作代理在并行处理多个页面时，未按 SKILL.md 要求"逐页阅读、编辑和验证"。
- 验证脚本（如 `validate_study_page_pipeline.py`）可能只检查了结构标记（DOM、data-export-context、导航锚点），而未检查**内容质量**（参考答案是否匹配题目、文本是否重复、是否有过渡冗余）。

### 2. 页面壳层一致性检查存在盲区

Claude Code 页面完全脱离共享壳层，说明：
- 要么 `validate_study_page_pipeline.py` 的 `--stage final` 未检查 `data-page-shell="study-page-v2"` 标记；
- 要么该页面是在验证流程之外生成的（如早期实验页面，后续未迁移）。

### 3. 领域质量不均

从抽样来看，**Agent 工程、系统协议、工具调用**等页面的质量明显高于**编程基础课程**（PyTorch、Claude Code 工作流）。这可能是因为：
- 前者有更强的作者领域知识或更充分的来源材料；
- 后者被视为"基础课"，在资源分配上被低估，导致批量生成后未得到同等打磨。

## 七、结语

这套 learning-kit 学习系统展现出了**优秀的教学设计框架和工程化能力**——shape tracer、terminal lab、角色账本、读写表、边界控制等组件都是中文技术学习资料中少见的精品。Agent 和系统协议类课程已经接近**可对外发布的水平**。

然而，"批量生成 + 共享模板"的模式在**执行末端出现了明显的质量控制滑坡**：参考答案的模板化、页面壳层的偶发逃逸、基础课程的重复冗余，都指向同一个问题——**逐页打磨环节被跳过了**。

用 SKILL.md 自己的话说："页面不是因为由生成器产出或共享有效模板就算完成。" 再给这批页面做一次**面向内容的编辑审阅**（而非仅结构验证），它们的整体质量可以从 "扎实草稿" 跃升到 "好课程" 甚至 "可发布课程"。

**最大的价值已经建好了；现在需要的是最后一公里的耐心。**
