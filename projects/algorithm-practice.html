算法刷题可视化

用交互式可视化展示经典算法执行过程，支持单步追踪、复杂度对比和自定义输入。

需要JavaScript · Canvas · 数据结构基础 加分TypeScript · D3.js · 测试框架

项目目标

GOALS

核心目标：实现一个网页，能可视化展示经典算法的执行过程，帮助理解和调试算法思路。

为什么做这个：算法面试不仅考你会不会写，还考你能不能讲清思路。可视化工具是展示理解深度的最佳方式。

预期产出：可交互的网页 + 算法实现库 + 演示视频。

前置知识

PREREQUISITES
  • 算法（数组、链表、树、图、排序、搜索）
  • JavaScript 基础（DOM 操作、事件、Canvas）
  • HTML/CSS 基础（布局、动画）

实施步骤

STEPS

MVP 阶段（1-2 周）

  • 实现排序可视化：冒泡、选择、插入、快速排序
  • 支持单步执行：点击"下一步"看算法每步的变化
  • 支持自定义输入：用户输入数组，看算法执行
  • 添加基础样式：颜色区分比较、交换、已排序区域

增强阶段（2-3 周）

  • 添加树结构可视化：BST 插入、删除、遍历（中序/前序/后序）
  • 添加图算法可视化：DFS、BFS、Dijkstra
  • 添加动态规划：最长公共子序列、背包问题
  • 添加复杂度对比：同一问题的不同算法，时间对比

进阶阶段（1-2 周）

  • 添加代码同步：可视化旁边显示对应代码，高亮当前行
  • 添加录制功能：把算法执行过程录成 GIF 或视频
  • 添加 LeetCode 集成：输入题号，自动加载测试用例
  • 添加移动端适配：手机也能查看基本可视化

技术栈建议

STACK
Core: Vanilla JS + Canvas 2D UI: 纯 CSS / Tailwind (可选) Animation: requestAnimationFrame Data: 算法实现纯 JS，不依赖外部库 Deploy: GitHub Pages / Vercel

验收标准

CHECKLIST
  • 至少支持 5 种排序算法的可视化
  • 支持单步执行和自动播放两种模式
  • 支持用户自定义输入数组（长度 2-100）
  • 树结构可视化支持插入、删除和遍历动画
  • 代码高亮与可视化同步，能看清当前执行到哪一行
  • README 包含算法列表、使用说明和截图

面试常见问题

INTERVIEW

Q1: 快速排序的最坏情况是什么？可视化怎么展示这个过程？

Q2: 动画帧率怎么控制？如果数据量很大，怎么保证不卡顿？

Q3: Dijkstra 和 BFS 在可视化上有什么区别？

Q4: 如果用户输入了很大的数组（10000 个元素），你的可视化怎么处理？

Q5: 你最喜欢的算法可视化是哪一个？为什么？
