AI 全栈平台

端到端 AI 平台实战项目：模型管理、API 网关、用户系统、计费监控。

需要 FastAPI · React · PostgreSQL · Docker

项目目标

GOALS

核心目标：搭建一个类似 OpenAI API 平台的完整产品：用户注册、模型管理、计费、监控。

为什么做这个：全栈面试最看重端到端能力。从数据库设计到 API 到前端到部署，展示完整的系统思维。

预期产出：可运行的完整产品 + 架构文档 + 部署指南。

前置知识

PREREQUISITES
  • LLM 基础与应用工程（API 调用、模型管理）
  • 工程基本功（Git、Linux、Docker）
  • Web 开发基础（HTTP、REST、数据库设计）

实施步骤

STEPS

MVP 阶段（2-3 周）

  • 设计数据库：用户、API Key、模型、请求日志、计费记录
  • 实现后端 API：用户注册/登录、API Key 管理、模型列表
  • 实现模型代理：接收用户请求，转发到 LLM 后端，返回结果
  • 实现前端：用户仪表盘、API Key 页面、用量统计

增强阶段（2-3 周）

  • 添加计费系统：按 token 计费、余额管理、充值记录
  • 添加限流：按 API Key 的速率限制和配额管理
  • 添加请求日志：完整记录每次请求的输入、输出、耗时、费用
  • 添加多模型支持：GPT、Claude、本地模型统一管理

进阶阶段（2-3 周）

  • 添加缓存：相同请求命中缓存，减少成本
  • 添加监控：QPS、延迟、错误率、成本 dashboard
  • 添加 CI/CD：GitHub Actions 自动测试和部署
  • 添加安全：API Key 轮换、请求审计、异常检测

技术栈建议

STACK
Backend: FastAPI + SQLAlchemy + PostgreSQL Frontend: React + Tailwind CSS Cache: Redis (请求缓存 + 限流) Auth: JWT + OAuth2 (GitHub/Google 登录) Deploy: Docker + docker-compose + Nginx Monitor: Prometheus + Grafana (可选)

验收标准

CHECKLIST
  • 用户能注册、登录、创建和管理 API Key
  • API 请求能被正确转发到 LLM，并记录日志
  • 计费系统准确：按 token 计费，余额实时更新
  • 限流生效：超过配额时返回 429，不超时不影响
  • 有监控面板：展示 QPS、延迟、成本趋势
  • README 包含架构图、数据库设计、API 文档和部署指南

面试常见问题

INTERVIEW

Q1: 如果 LLM 后端挂了，你的平台怎么保证用户体验？

Q2: API Key 泄露了怎么办？怎么快速撤销？

Q3: 计费按 token 计算，如果模型不返回 token 数量怎么办？

Q4: 缓存命中率怎么提升？哪些请求可以缓存，哪些不能？

Q5: 如果用户投诉账单不对，你怎么排查？
