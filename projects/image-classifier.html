图像分类系统

基于 PyTorch 实现完整的图像分类流程：数据增强、模型训练、推理部署和 Web demo。

需要PyTorch · Python · 基础 CNN 知识 加分Flask · ONNX · Docker

项目目标

GOALS

核心目标：训练一个图像分类模型，部署为 Web 服务，支持上传图片并返回分类结果。

为什么做这个：深度学习面试的基础项目。从数据到模型到部署的完整闭环，是展示工程能力的标准范式。

预期产出：训练脚本 + 模型权重 + Web 服务 + 项目文档。

前置知识

PREREQUISITES
  • PyTorch 深度学习实践（CNN、DataLoader、训练循环）
  • Python 基础（文件操作、HTTP 服务）

实施步骤

STEPS

MVP 阶段（1-2 周）

  • 选择数据集：CIFAR-10 或自定义数据集（如垃圾分类）
  • 实现数据加载和预处理：resize、normalize、train/val split
  • 实现 CNN 模型：ResNet-18 或 MobileNet（轻量）
  • 训练模型并保存最佳权重（验证集准确率最高）

增强阶段（2-3 周）

  • 添加数据增强：随机裁剪、翻转、颜色抖动
  • 添加学习率调度：StepLR / CosineAnnealing
  • 添加 TensorBoard 监控：loss、准确率、学习率曲线
  • 添加模型对比：不同架构的准确率-速度权衡

进阶阶段（1-2 周）

  • 导出 ONNX 模型，实现跨平台推理
  • 用 Flask 搭建 Web 服务，支持上传图片、返回分类结果
  • 添加 Grad-CAM 可视化：展示模型关注的区域
  • Docker 化部署，支持一键启动

技术栈建议

STACK
Framework: PyTorch 2.x + torchvision Model: ResNet-18 / MobileNet-V2 Data: CIFAR-10 / 自定义数据集 Monitor: TensorBoard Deploy: Flask + ONNX Runtime Container: Docker

验收标准

CHECKLIST
  • CIFAR-10 验证集准确率 > 85%（ResNet-18 基线）
  • Web 服务能上传图片并返回 top-3 分类结果
  • 有完整的训练曲线和超参数记录
  • 有 Grad-CAM 可视化，能解释模型决策
  • ONNX 模型能在 CPU 上推理（不依赖 GPU）
  • README 包含数据集说明、训练命令、API 文档

Q2: 数据增强对最终准确率提升有多少？你怎么选增强策略？

Q3: 如果训练集和测试集分布不一致，怎么处理？

Q4: ONNX 和原始 PyTorch 模型在精度上有什么差异？

Q5: Grad-CAM 的原理是什么？它能解释所有类型的模型吗？
