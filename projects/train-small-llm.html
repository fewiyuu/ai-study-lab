训练小型语言模型

从 tokenizer 到训练循环，完整复现 GPT-2 级别小模型的实战项目。

需要 PyTorch · Python · GPU

项目目标

GOALS

核心目标：从零开始训练一个 GPT-2 级别的小模型（约 100M 参数），在文本生成任务上验证效果。

为什么做这个：训练项目是大模型面试的核心差异化因素。能讲清训练闭环的人，比只懂调 API 的人更有竞争力。

预期产出：训练脚本 + 模型 checkpoint + 训练报告 + 生成 demo。

前置知识

PREREQUISITES
  • PyTorch 深度学习实践（Module、Loss、Optimizer、DataLoader）
  • Transformer 论文精读（自注意力、位置编码、解码器）
  • 从零训练小语言模型（数据、训练循环、评测）

实施步骤

STEPS

MVP 阶段（2-3 周）

  • 实现 BPE tokenizer：训练自己的小词汇表（~10k 词）
  • 实现 GPT-2 架构：嵌入层、Transformer 块、输出头
  • 实现训练循环：数据加载、前向传播、反向传播、梯度裁剪
  • 在小数据集（如中文维基百科子集）上训练，验证能收敛

增强阶段（2-3 周）

  • 添加数据清洗：去重、过滤低质量文本、控制长度分布
  • 添加训练监控：loss 曲线、学习率衰减、梯度范数
  • 添加评测：perplexity、生成质量人工评估、下游任务零样本
  • 添加 checkpoint 管理：定期保存、resume from checkpoint

进阶阶段（1-2 周）

  • 添加 LoRA 微调：在预训练基础上用少量数据做任务微调
  • 添加多卡训练：用 DDP 或 FSDP 扩展到多 GPU
  • 添加推理优化：KV Cache、量化（INT8）、采样策略
  • 编写完整训练报告：数据分布、训练配置、收敛分析、生成示例

技术栈建议

STACK
Framework: PyTorch 2.x Tokenizer: tiktoken (参考) / 自实现 BPE Data: HuggingFace Datasets / 自建语料 Monitor: Weights & Biases / TensorBoard Training: 单卡 A100 / RTX 4090 (24G 显存可跑 100M 模型) Deploy: HuggingFace Transformers (加载推理)

验收标准

CHECKLIST
  • 模型能生成连贯的中文/英文文本（至少 50 tokens 不跑题）
  • 训练 loss 从初始值下降到收敛（验证集 loss 不再下降）
  • 有完整的训练日志：每 step 的 loss、学习率、梯度范数
  • 能实现 checkpoint 保存和恢复训练
  • 有 LoRA 微调示例：在特定任务上提升性能
  • README 包含架构说明、训练命令、评测方法和生成示例

面试常见问题

INTERVIEW

Q1: 为什么你的模型用 100M 参数而不是 1B？这个决策依据是什么？

Q2: 训练数据怎么清洗？怎么判断一段文本质量高低？

Q3: loss 不下降时，你排查的顺序是什么？

Q4: LoRA 的 rank 怎么选？太小会怎样，太大会怎样？

Q5: 如果训练时显存爆了，你第一步做什么？
