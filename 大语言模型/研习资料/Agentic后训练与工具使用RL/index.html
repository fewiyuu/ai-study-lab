Agentic 后训练与工具使用 RL

这门课关心的不是“模型会不会吐出一个 function call”，而是它在工具选择、参数构造、多步规划和环境闭环里各自卡在哪一层。

如果你已经会一点工具调用，但还不知道怎么把它推进到可训练、可评测、可上线的 agent 工作流，从 01 开始最稳。

8 个单元 工具调用 环境反馈 交互式 RL

先读 01，确认能力层级；再看 02 的接口边界；然后按 03-08 的顺序进入轨迹、评测、环境和安全。

基于哪些资料

public sources
工程接口

OpenAI Agents SDK 文档。支撑工具、handoff、guardrails、tracing 和运行边界的理解。

训练环境

TRL 与 OpenEnv 资料。支撑状态化环境、reward 和 episode 结构的理解。

工具使用论文

ReAct、Toolformer、ToolBench / ToolLLM、API-Bank。支撑行动、观察和多步评测的主线。

RL 衔接

verl 与推理训练资料。用来对照可验证奖励和交互式奖励的差别。

建议学习对象

start here if

适合对象

做智能体应用、工具调用、自动化工作流、API 任务评测，或者想把模型从单次调用推进到多步任务的人。

前置知识

后训练基础、函数调用或工具调用接口、基础强化学习词汇、评测和日志追踪概念。

学完能做

能设计一个安全的工具使用训练或评测环境，说明轨迹、奖励、失败模式和上线边界。

推荐路径

roadmap
阶段一｜接口与安全

先看工具 schema、运行时、handoff、guardrails 和 tracing。第 01-02 单元把“模型能调什么”和“调的时候怎么不出事”讲清楚。

阶段二｜轨迹与监督

再看 ReAct、Toolformer、ToolBench 和 API-Bank。第 03-05 单元讲行动、观察、数据构造和多步评测。

阶段三｜环境与上线

最后看 SFT、OpenEnv、沙盒和副作用控制。第 06-08 单元把状态化训练、奖励和上线安全串起来。

单元设计

8 units
01foundation

工具使用的能力层级

把格式遵循、工具选择、参数构造、多步规划和环境适应拆成可诊断的层级，避免把 function calling 当成终点。

进入学习页
02foundation

工具 schema、运行时与安全边界

看工具接口、handoff、guardrails 和 tracing 怎么把调用变成可执行、可约束、可审计的系统。

进入学习页
03practice

ReAct：推理与行动交替

看 thought / action / observation 如何交替出现，以及为什么观察会改变下一步计划。

进入学习页
04practice

Toolformer 与自监督工具学习

看少量示例怎样扩展成工具使用数据，理解自监督工具学习的生成思路。

进入学习页
05assessment

ToolBench / API-Bank 与任务评测

看多步任务成功率、参数正确性和环境反馈怎样进入评测，而不只看文本相似度。

进入学习页
06advanced

从 SFT 到环境强化学习

看工具轨迹何时足够监督微调，何时必须进入状态化环境和奖励闭环。

进入学习页
07advanced

TRL OpenEnv 与交互式训练环

看 observation、action、reward、done 如何组成训练环，理解环境训练的工程入口。

进入学习页
08extension

沙盒、副作用与上线评测

看哪些工具只能进沙盒、哪些必须只读、哪些要加审计和回退，避免把训练直接带进生产。

进入学习页

资料边界

boundary

这门课不直接提供连接生产系统的训练方案，也不建议在没有沙盒、审计和回退的情况下让模型调用有副作用的工具。

OpenAI Agents SDK、TRL OpenEnv、verl 和工具论文都会继续演进；具体 API、环境接口和训练脚本请以当前公开文档为准。

如果你只想先掌握工具调用的最小闭环，先读 01 和 02；如果你想看完整训练链，再按 03-08 往后读。
