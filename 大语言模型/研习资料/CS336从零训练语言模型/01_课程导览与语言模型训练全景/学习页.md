<!-- tag: roadmap -->

# 为什么先学这个 {#why}
<!-- tag: motivation -->
<!-- layout: grid2 -->

## box soft
### 一个常见误会
很多人一开始只会把大模型当成一个 API：输入 prompt，拿到输出，觉得理解到此为止。真正卡住的时候，问题却常常出在输入表示、训练目标、损失对齐、评测方式或者系统瓶颈上。只看接口，很难判断错在什么地方。

## box warn
### 这门课想解决什么
它不是教你背架构名词，而是训练你从现象倒推机制：哪个对象变了，哪个步骤出错了，哪一层该先看。这样一来，遇到训练异常、低效、偏移或评测失真时，你知道先去哪里查。

# 训练栈 {#stack}
<!-- tag: mechanics -->

把语言模型训练压成一句话：把文本改写成“预测下一个 token”的问题，然后反复前向、算 loss、反向、更新。下面这条链路是本课程最重要的骨架。

**TEXT**原始文本、清洗、切分、采样。
**TOKENIZER**把文本转成 token id。
**BATCH**把 token 序列切成上下文窗口。
**MODEL**Transformer 输出每个位置的 logits。
**LOSS**next-token cross entropy 衡量差距。
**OPTIMIZER**用梯度更新参数。
**EVAL**看验证集和任务评估是否真的变好。

最小训练闭环示意复制

```python
for batch in data_loader:
    token_ids = tokenizer.encode(batch.text)
    x, y = make_next_token_batch(token_ids)
    logits = model(x)
    loss = cross_entropy(logits, y)
    loss.backward()
    optimizer.step()
    optimizer.zero_grad()
    eval_periodically(model, valid_data)
```

步骤读什么写什么出错信号


Tokenizer文本token id后续长度和词表都不对
Batchtoken 序列x / y 训练对next-token 对齐错位
Modelxlogitsshape 不对或 future leakage
Losslogits / y标量 lossloss 维度或 flatten 错误
Optimizergradients参数与 state更新后数值不变或偏移

逐行看这段代码时，不要把它当成模板。先盯住对象：`tokenizer.encode` 改的是输入表示，`make_next_token_batch` 改的是训练目标，`cross_entropy` 把预测错误写成可优化的数，`optimizer.step` 才真正改参数。你只要能说出每行读写了什么，后面读真实代码就不会只剩“会跑”和“不会跑”两种感觉。

# 哪些知识会迁移 {#transfer}
<!-- tag: transfer -->

机制会迁移attention、MLP、loss、optimizer 和并行方式都在读写张量，只是规模变了，代价变了，形状和约束没有消失。
判断顺序会迁移先问“哪个对象变了”，再问“这个变化在更大规模下是否还成立”。这个顺序比记某个数值更值得带走。
调试思路会迁移训练异常时，先分到表示、目标、更新、系统或评测，再决定去看哪一层。这个分流动作以后会一直用。

要带走的先别外推的为什么


读写对象的顺序某个具体数值数值受数据和规模条件影响更大
训练栈分层看问题单一 API 视角API 会遮住表示、目标和系统成本
先定位再优化一上来就调参数先分清是数据、模型还是系统问题

### 5 路径选择

**Q:** Q5. 为什么 tokenizer 要排在 Transformer 训练前面学？

Explain: Transformer 不直接读取自然语言字符串，它读取 token id。Tokenizer 决定文本如何被切分、id 如何映射、上下文长度里能装多少内容，进而影响 batch、embedding、loss 对齐和训练成本。所以先学 tokenizer，是为了知道后续模型和训练循环到底在处理什么对象。

### 6 场景诊断

**Q:** Q6. 场景：你训练出一个小模型，但验证 loss 和生成效果都不稳。先问哪三个问题？

Explain: 可以先问：数据和 tokenizer 是否让样本分布或 token 边界异常；x/y shift、mask、loss、optimizer 是否写错；验证集和生成设置是否能代表目标任务；小模型规模是否不足以支撑你期待的现象。
