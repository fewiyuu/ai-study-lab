STANFORD CS336 / COURSE MAP

从调用模型，退回到亲手训练模型。

CS336 的价值在于把语言模型训练栈拆开看清楚：文本如何变成 token，token 如何进入 Transformer，训练循环如何用 loss 和 optimizer 改参数，系统、数据、评测、后训练又怎样影响最终模型。前沿模型细节可以后补，训练栈的主线要先立住。

本课程地图先锁定前三个学习页：课程全景、Tokenizer 与数据表示、最小 Transformer LM 训练闭环。后续再扩展系统、scaling、数据、评测和 alignment/RL。

本轮制作范围

本轮先开放 3 个入口：导览页帮你建立全景，tokenizer 和训练循环负责把“语言模型从零训练”的前两步讲实。目标是完整但不臃肿，能直接自学。

这门课讲什么

OVERVIEW
Mechanics：机制

能说清 token、embedding、attention、MLP、loss、optimizer、batch、checkpoint 这些部件怎样连成训练循环。

Mindset：工程心态

训练语言模型包含写网络，也包含在数据、算力、吞吐、稳定性、评测之间做资源分配。

Intuitions：可迁移直觉

小模型实验不能完全代表前沿模型，但能训练你识别哪些决策是架构机制，哪些是规模、数据和系统约束带来的。

适合谁与前置知识

PREP
适合谁学

适合已经会调用大模型、写过一点 Python，想知道语言模型到底怎样从文本、数据、模型和优化器一步步训练出来的人。

需要先会什么

最好熟悉 Python、基本矩阵运算、概率里的条件概率和交叉熵；不要求先会 CUDA 或分布式训练。

推荐路线

roadmap
01 / 导览训练栈全景

先看清这门课的重心：从数据表示一路追到参数更新、系统代价和评测边界。

02 / 表示Tokenizer

把文本变成模型能处理的整数序列，同时理解长度、词表和边界。

03 / 模型最小 Transformer LM 闭环

把 token 序列变成 logits，再用 next-token loss 和 optimizer 更新参数。

04 / 系统训练吞吐

理解显存、通信、并行、kernel 和 profiling 为什么会限制训练。

05 / Scaling实验规律

用小实验看 compute、data、model size 的取舍。

06 / Data数据配方

从“更多文本”进入过滤、混合、去重、质量和评测集污染。

07 / Alignment后训练

把预训练模型接到指令、偏好、RL 和评测闭环。
