Harness Engineering

基于 OpenAI 文章《Harness engineering: Leveraging Codex in an agent-first world》制作的 5 块自学资料。主线是：当 Codex 这样的智能体能持续执行工程任务，人类工程师需要设计它工作的环境、证据、约束和收敛机制。

建议按顺序学习。每块是一个独立单文件 HTML，包含讲义、例子、误区诊断和练习记录导出。

01

从写代码转向设计环境

人类掌舵，Agent 执行

  • 解释为什么“让模型再努力一点”不是稳定的工程方法。
  • 说清 harness 的组成：文档、工具、测试、反馈和权限。
  • 把一个大目标拆成可执行、可验收、可复用的能力块。
02

让应用状态变成 Agent 能读的证据

UI、日志、指标和追踪

  • 解释应用可读性和普通可观测性的区别。
  • 设计一个让 Agent 复现 UI 问题的最小反馈回路。
  • 把日志、指标和 trace 转成可用于验收的证据。
03

把仓库知识写成 Agent 能读的地图

AGENTS、README、模板和脚手架

  • 判断哪些仓库知识应该写进 AGENTS 或 README。
  • 为一个目录设计 Agent 友好的入口说明。
  • 区分文档规则、代码模板、脚手架和测试约束的适用场景。
04

把工程品味变成可执行约束

架构、审查和不变量

  • 解释品味和可执行约束之间的关系。
  • 把一条抽象架构偏好改写成 Agent 可遵守的不变量。
  • 设计审查标准，区分必须阻止的问题和可以接受的风格差异。
05

高吞吐之后，要管理熵

并行、自治、清理和收敛

  • 解释高吞吐为什么会带来系统熵。
  • 设计多个 Agent 并行工作时的队列、范围和审查规则。
  • 识别重复能力、过期文档、测试噪声和未收敛分支。
