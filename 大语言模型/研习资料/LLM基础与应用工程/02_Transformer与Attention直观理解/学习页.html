这一页先抓住什么

  • attention 先算相关性，再按权重汇总 value。

先看这层为什么值得学

目标

你要练出的判断

Transformer 这章最容易学歪的地方，是把术语背得很熟，却仍然分不清一层 block 里哪部分在跨 token 读信息，哪部分只是在每个位置自己加工。这一页要做的，就是把这两件事拆开。你看到一个 token 时，先问它在读谁，再问它把读到的信息怎么混回自己的表示里。

如果你能把 attention 说成“每个位置给所有候选位置打分，再从对应的 value 里取回信息”，后面的内容就能顺着接上：为什么 decoder 需要 causal mask，为什么 softmax 要沿 key 轴，为什么输出还是长度不变的一串向量，以及为什么它不会直接变成答案文本。

先查什么

学完这一单元，先盯住四个检查点：QK^T 的形状、softmax 的轴、mask 的位置、最后乘回去的是 V 还是 K。只要这四个点错一个，代码可能照样跑，语义却已经变了。

这些检查点也会决定你怎么看调试日志：如果输出还像一个正常 tensor，注意力也未必是对的；如果图上看起来“很关注某个词”，也不能直接说明它做了正确的读取。要把观察点放回 query 行、key 列和 value 汇总这三件事上。

读完就能做的事

  • 用一句话解释 self-attention 的读取方式。
  • 用形状读法解释 `Q @ K.transpose(-2, -1)`。
  • 说清 causal mask 和 padding mask 的差别。
  • 写出一个不会把 softmax 轴搞错的最小实现。

先抓主线：一层 block 到底做了什么

总览

把“读上下文”放回一条链路里

一串 token 先进入注意力子层。每个位置都先形成自己的 query、key、value，然后 query 和所有 key 做相似度计算，得到一张“我该从谁那里拿多少信息”的表。这个表经过缩放、mask 和 softmax 以后，变成真正的读取权重，再去加权汇总 value。结果仍然是一串向量，只是每个位置混入了上下文。

后面的 FFN 不负责跨 token 通信，它负责对每个位置自己的表示做非线性加工。残差连接把原表示保留下来，LayerNorm 把数值范围稳住。整层 block 的目标，是让表示经过几轮小步骤后越来越适合下一层继续处理。

attention 直觉：一行向量怎样读同一句里的其他位置

直觉

先从“相关性表”开始读

把一句话切成 N 个 token 之后，每个位置都会对同一句里的所有位置算一遍相关性。结果是一张 N × N 的表：第 i 行表示第 i 个 token 在更新自己表示时，准备从哪些位置读取信息、各读多少。这里的“读”，指的是把对应位置的 value 混到自己的向量里。

softmax 的作用，是把一行分数变成可以解释的比例。比例之和约等于 1，意味着这一行把注意力分给了若干个候选位置。缩放项 sqrt(d_k) 的作用，是避免点积太大，让 softmax 一开始就被极端值压扁。mask 的作用，是把不该看的位置从候选集合里排出去。

从这个角度看，attention 的输出仍然是一串向量，只是每个位置带上了上下文。把它当成一轮按权重重写表示的步骤更合适。解释时先看 query 行，再看 key 列；单独拿热力图下结论，常常会看偏。

切换 query 看它在读谁

输入 X [B, N, D]

这是进入注意力层之前的 token 表示，每个位置一行向量。

Q / K / V [B, H, N, Dh]

同一批输入经过三组投影后，分别变成读取条件、匹配索引和被汇总内容。

分数 [B, H, N, N]

每个 query 对同一序列里的每个 key 打分，得到一张“我该读谁”的表。

权重 [B, H, N, N]

softmax 把分数变成比例。比例变化的方向，会直接体现 mask 和轴是否正确。

输出 [B, H, N, Dh]

每个 query 位置从 value 里拿回上下文信息，准备在多头合并后回到 [B, N, D]。

$$ \mathrm{Attention}(Q,K,V)=\mathrm{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right)V $$

这里的 QK^T 是“相关性分数”，sqrt(d_k) 是缩放项，softmax 把一行分数变成读取比例，最后乘回 V 得到新的表示。公式不长，但每个符号都对应一个可以检查的动作。

query 位置 高权重 key 这行在读什么 容易看错的地方
“模型” “写代码” 当前位置在拿最后一个 token 的 value 来补全当前语境 这里不等于“模型”这个词自己变成答案
“用” “模型” 当前位置在借前一个语义块，补足动作对象 高权重未必代表词义强，只是当前 query 的需要更高
“我” “我 / 喜欢” 当前位置可能更偏向保留前缀语气和主语结构 权重图只是混合比例，不能直接当因果解释

Q、K、V：三种投影视角

QKV

为什么同一输入要过三组投影

self-attention 里，输入 X 通常先经过三组不同的线性投影，分别得到 Q、K、V。它们来自同一批 token 表示，但用途不同。Q 代表当前位置提出的读取条件，K 代表每个位置提供给别人匹配的索引，V 代表真正会被汇总回来的内容。把这三件事分开，模型就能学会“用一类特征判断相关性，用另一类特征提供内容”。

如果把 Q、K、V 想成三段不同文本，很容易把投影层和输入来源混成一件事。更准确的说法是：同一输入经过三套不同参数的线性层，得到三种不同的视角。对于 encoder-decoder attention，decoder 的 Q 会去读 encoder 的 K/V；在 self-attention 里，三者通常都来自同一侧的输入。

形状和语义要一起看

对象 常见形状 在做什么 如果看错会怎样
X [B, N, D] 原始输入表示 容易把 batch、长度和隐藏维度混起来
Q / K / V [B, H, N, Dh] 拆成多头后的读取条件、匹配索引和内容 如果 head 维或 Dh 维理解错，后面矩阵乘法就会错位
scores [B, H, N, N] 每个 query 对每个 key 的分数 softmax 轴错时，语义会从“读谁”变成“谁被读”
out [B, H, N, Dh] 每个 query 从 value 汇总到的新表示 如果最后一步乘错对象，输出 shape 可能对，语义却不对

多头的作用多头把同一层里的读取视角拆开，让不同头学到不同的相关性模式；最后再把这些头合并回去，回到统一的隐藏宽度。

逐行拆解：QK^T 怎么得到“每个位置对每个位置”的分数
import torch

B, H, N, Dh = 2, 4, 5, 16
x = torch.randn(B, N, H * Dh)

q = x.view(B, N, H, Dh).transpose(1, 2)
k = x.view(B, N, H, Dh).transpose(1, 2)
v = x.view(B, N, H, Dh).transpose(1, 2)

scores = q @ k.transpose(-2, -1)          # [B, H, N, N]
weights = torch.softmax(scores / (Dh ** 0.5), dim=-1)
out = weights @ v                          # [B, H, N, Dh]

这段代码最值得先看的，是三件事：第一，Q 和 K 先在最后两维上做内积，所以 K 必须转置到 -2, -1；第二，softmax 必须沿最后一维做，因为每一行是在所有 key 上分配权重；第三，最后乘回去的是 V。只要这三步位置错了，代码就算能跑，语义也已经变了。

块结构：attention 之外还要加工和稳定

块结构

attention 负责通信，FFN 负责加工

attention 把不同位置的信息互相传递，FFN 则在每个位置内部继续加工。可以把它理解成两步：先让 token 之间“开会”，再让每个 token 在自己这一行里“整理笔记”。如果只保留 attention，模型会缺少逐位置的非线性变换；如果只保留 FFN，模型就不会跨 token 交流。

残差连接的意义，是给输入保留一条直通路。LayerNorm 的意义，是把每个位置的数值范围稳住，让下一层看到的分布不要过于飘。现代大模型常见的是 Pre-LN 结构，但这一页不把不同变体展开成一整章，只抓住“先通信、再加工、再稳定地传下去”这一条线。

几个容易混的角色

组件 主要职责 常见误解
attention 跨 token 读取信息 以为它只是“挑重点词”
FFN 对每个位置做逐点非线性加工 以为它也在做 token 间通信
residual 保留原输入的通路 以为它和 LayerNorm 是同一件事
LayerNorm 稳定数值分布 以为它负责屏蔽未来 token

先把 block 看完整attention 只是第一步，后面的 FFN、残差和归一化一起把表示稳住，深层网络才更容易训练。

最小结构：一个 decoder block 的顺序
def decoder_block(x, causal_mask):
    attn_out = self_attention(x, mask=causal_mask)
    x = layer_norm(x + attn_out)      # 残差保留原输入，norm 稳住分布

    ffn_out = feed_forward(x)
    x = layer_norm(x + ffn_out)       # FFN 对每个位置继续加工
    return x

这段伪代码省掉了 dropout、多头拆分和参数细节，但顺序已经足够说明逻辑：先让位置之间通信，再对每个位置单独加工，最后把两步的结果稳稳传给下一层。把这个顺序记住，比死背一串模块名更有用。

mask 与生成：为什么 decoder 不能看未来

mask

causal mask 挡住未来 token 的读取

自回归模型的目标，是根据已经出现的前缀去预测下一个 token。也就是说，在训练和生成时，当前位置只能看见当前之前的 token，不能提前偷看后面还没生成出来的 token。如果让它看到未来，训练就会变成作弊，推理时又没有这些未来信息，条件就对不上。

causal mask 的做法，是在注意力分数里把不允许读取的位置压成极小值，然后再做 softmax。这样未来位置在权重里几乎变成 0。顺序很重要：先 mask，再 softmax。先 softmax 再置零，表面上也像挡住了，归一化方式却已经变过了。

看一眼 mask 矩阵

[case-1]
scores shape  = [B, H, N, N]
mask before softmax
future keys   -> -1e9
softmax(axis=-1)
row sum       -> about 1
result        -> each query reads only allowed keys

观察点：第 3 个位置只能看前 3 个位置，不能看第 4、5 个未来位置。padding mask 的作用，是把补齐的空位从读取集合里移开。

$$ \text{masked\_scores} = \text{scores} + \text{mask} $$

mask 通常在非法位置写入一个很小的负数，softmax 之后这些位置的权重接近 0。重点在于它是否真的在 softmax 之前介入了分数。

mask 类型 挡住什么 常见症状 下一步先看什么
causal mask 未来 token 生成时像偷看了答案，训练和推理不一致 看 mask 是否在 softmax 前加入
padding mask 补齐位 无效位混进了权重分配，短句和长句表现不一致 看 padding 是否被当成普通 token 处理
错误的轴 挡错了维度 shape 正常，但语义反了 看 softmax 到底沿哪一维做

代码验收：先查轴、顺序和 mask

代码

先查这四件事

第一，Q、K、V 是否来自正确的输入，头维是否拆开。第二，K 有没有转置到最后两维，能不能真正形成 query 对 key 的两两打分。第三，缩放是否用了 sqrt(d_k)。第四，mask 是否在 softmax 前加入，softmax 是否沿最后一维做。

很多实现看起来都能输出同样的 shape，但语义已经变掉了。比如把 softmax 放到 query 轴，模型就在回答“每个 key 被多少 query 读取”，而不是“每个 query 应该读哪些 key”。这种错不一定马上报错，反而更容易埋进训练里。

错误示例：softmax 维度放错

错误示例：沿 query 维度归一化
scores = q @ k.transpose(-2, -1)
weights = torch.softmax(scores, dim=-2)   # 错：这里不是 key 维度
out = weights @ v

这段代码可能不报错，因为矩阵尺寸还能对上。问题在于每个 query 没有在所有 key 上分配读取比例，注意力图会看起来正常，语义却已经反了。下一步先把 softmax 移到最后一维，再看 query 行和 mask 是否一致。

正确示例：mask、缩放和 softmax 顺序
def scaled_dot_product_attention(q, k, v, mask=None):
    head_dim = q.size(-1)
    scores = q @ k.transpose(-2, -1)
    scores = scores / (head_dim ** 0.5)

    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)

    weights = torch.softmax(scores, dim=-1)
    out = weights @ v
    return out, weights
检查项 应该看到什么 常见错误
K 的转置 最后两维互换后才能做点积 直接相乘，虽然有时 shape 还能碰上，但语义不对
缩放 除以 sqrt(d_k) 用 hidden size、序列长度或根本没缩放
mask 顺序 先挡非法位置，再 softmax 先 softmax 再置零，归一化已经被改过
输出来源 最后乘的是 V 把 K 当成内容回写，混淆了索引和内容

误区诊断：最容易看偏的五个地方

诊断

五个最常见的理解偏差

  1. 把 attention 看成“只看重要词”会漏掉 query 行和 key 列的关系；更稳的说法是，当前位置按相关性读取上下文。
  2. Q、K、V 来自同一输入 X 的三组投影，不是三份独立文本；混了这点，读 self-attention 代码时就会把参数和内容看反。
  3. 只盯 shape 不盯轴，容易在 [B, H, N, N] 里忽略 softmax 的方向；轴一错，语义就反了。
  4. 把 mask 放在 softmax 后，等于先把分布算完再删位；归一化方式已经被改过。
  5. LayerNorm 负责稳定数值分布，不负责时间可见性；mask 和 LayerNorm 处理的是两件事。

出错时先看什么

症状 最先怀疑 先看哪里 为什么先看这里
输出 shape 正常，但行为怪 softmax 轴或 mask 顺序 注意力分数和归一化那一行 shape 能对上，不代表语义对上
未来 token 还能被读到 causal mask 未生效 softmax 前的分数 事后置零并不能恢复正确归一化
长序列训练不稳 缩放缺失或数值过大 点积分数的尺度 softmax 很容易在极端值下变尖
注意力图“很漂亮”但解释不通 把可视化当因果解释 回到 query 行和 value 汇总 权重图只能说明混合比例，不等于原因

7. 为什么要除以 sqrt(d_k)？

回答时至少说出：点积会随维度变大、softmax 容易过尖、缩放让分数尺度更稳、训练更不容易抖。

参考答案 / 评分要点

好的答案会说明：如果不缩放，Q 和 K 的点积在维度变大时更容易变成大数，softmax 结果会过于尖锐，几乎把权重压到少数几个位置上，导致梯度和训练都更不稳定。除以 sqrt(d_k) 的作用，是把 logits 的尺度拉回一个更可控的范围，也不会直接跟序列长度或 hidden size 绑定。

10. causal mask 挡住什么？padding mask 又挡住什么？

作答时至少说出“未来 token / 补齐位置”“训练和生成的时间顺序”“两种 mask 在目的上不同”这三层。

参考答案 / 评分要点

causal mask 挡住的是未来 token，它保证当前位置只能读到当前之前的前缀，不会在训练时偷看未来。padding mask 挡住的是补齐位置，它不关心时间顺序，只是把空位从读取集合里排出去。好的答案还会说清：一个是时间约束，一个是数据对齐；两者的目的不同，不能混成同一件事。

12. 输出 shape 正常，但注意力行为不对时，先检查哪几项？

先挑最接近根因的项，不要把所有可能原因都勾满。好的顺序通常是：转置、轴、缩放、mask。

13. 用 6-8 句话解释：Transformer 如何让 token 读取上下文并更新表示；再说明如果 mask 放在 softmax 后，会发生什么。

参考答案 / 评分要点

高质量答案会先把流程串起来：输入 X 先生成 Q/K/V，Q 和 K 计算 scores，scores 先缩放再加 mask，然后沿 key 轴做 softmax 得到权重，最后用权重对 V 加权求和得到新的表示。之后这个表示再经过残差、LayerNorm 和 FFN 继续加工。关于失败后果，答案要指出：如果把 mask 放在 softmax 后，非法位置已经参与了归一化，读未来的约束不再是原来的语义，训练和生成的条件也会被悄悄改掉。
