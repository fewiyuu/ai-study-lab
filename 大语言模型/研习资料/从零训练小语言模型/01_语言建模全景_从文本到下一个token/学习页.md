# 先把术语钉住 {#terms}
<!-- tag: terms -->

这几件事最容易混在一起：token、输入、标签、分数、概率、损失。看见它们时，先问“它是表示、目标、分数，还是损失”。

术语
它是什么
不要和什么混
为什么重要


token
模型实际看到的离散符号
不要直接等同于“词”
它决定序列长度、词表大小和训练成本


x
输入序列，负责给上下文
不要和 y 混成同一列
它告诉模型“看到什么以后再预测”


y
真实下一个 token
不要当成模型输出
它是监督信号，决定 loss 怎么算


logits
每个位置对词表中每个 token 的分数
不要直接当概率
它要先经过 softmax 才能解释成分布


softmax
把分数变成概率分布
不要和 loss 混在一起
它让“正确答案的概率”能被比较


交叉熵
衡量正确 token 被分到多大概率的损失
不要把它理解成聊天质量分数
它只在 base 目标里看下一位 token 的预测质量

## box
### token 不等于词
一个 token 可能是一个字、一个词、一个子词片段，甚至是标点。你如果把 token 当成“单词”，训练成本和上下文长度的估算都会偏掉。

## box soft
### logits 还没变成概率
logits 只是分数。只有经过 softmax，才能把“谁更像正确答案”变成概率分布，再去和真实标签比较。

$$
 L = - \frac{1}{T} \sum_{t=1}^{T} \log p_{\theta}(y_t \mid x_{\le t})
 $$
 这里的 `T` 是错位后序列的长度，`x` 是输入，`y` 是对应位置的真实下一个 token，`p_{\theta}` 是模型对正确答案的概率。只要正确 token 的概率上升，`-log` 就会下降，loss 也会跟着变小。

公式里最关键的方向：模型先产出分数，再把分数变成概率，再把真实标签拿来比较。训练就是沿着这个方向，把正确 token 的概率慢慢推高。

# 从单句到 batch，形状怎么变 {#batch-shape}
<!-- tag: shape -->

单句很好懂，但训练不会只看一句。实际训练时，会把多条样本打包成 batch，这时你要同时盯住两个轴：batch 维 B 和时间步维 T。B 表示一次喂给模型多少条样本，T 表示每条样本里有多少个位置。

如果样本长度不一样，通常会补 padding，再配一个 mask。mask 的作用很朴素：把填充位置排除在损失之外。这样做以后，模型不会把空位当成真 token 学进去。

对象
典型形状
读什么
写什么
错了会怎样


单条 x
[T]
一条 token 序列
当前位置之前的上下文
如果直接把整句塞进去，y 就没法错位


单条 y
[T]
同一条 token 序列
后一位真实 token
如果没错位，模型学到的是复制，不是预测


batch x
[B, T]
多条样本打包后的输入
整个 batch 的前文
如果 T 没统一，后面的矩阵运算会乱掉


batch y
[B, T]
多条样本打包后的标签
每个位置的下一 token
如果 y 对不齐，loss 会看见错误监督


logits
[B, T, V]
x 和参数
每个位置对词表 V 的分数
最后一维不是 V，交叉熵就没法按词表算


mask
[B, T]
哪些位置是真数据
把 padding 位排除掉
padding 被算进 loss，会把训练带偏

$$
 L = \frac{1}{\sum_{b,t} m_{b,t}} \sum_{b=1}^{B}\sum_{t=1}^{T} m_{b,t}\, \bigl(-\log p_{\theta}(y_{b,t}\mid x_{b,\le t})\bigr)
 $$
 `m_{b,t}` 是 mask。它等于 1 时，这个位置参与平均；它等于 0 时，这个位置被忽略。这样写的好处是，长度不一样的样本也能放在一起训练，而且 padding 不会污染 loss。

## box soft
### batch 里的平均不是装饰
如果你不把有效位置平均掉，长样本会比短样本更重。训练看起来还在算 loss，但不同长度的样本已经不在同一尺度上比较了。

## box info
### 为什么这里要看形状
shape 不是形式主义。它直接告诉你哪一维是样本、哪一维是时间、哪一维是词表。三个维度错一个，训练就会在不该错的地方报错，或者更糟，悄悄学偏。

# padding 和 mask 为什么要一起看 {#padding-mask}
<!-- tag: mask -->

当 batch 里的句子长短不一样时，通常要把短句补到同样长度。补出来的 PAD 只是占位符，不是真 token。如果你让它参与 loss，模型会被迫去“预测空位”，训练就会朝错误方向偏一点。

最稳妥的做法，是把每个位置都标一个 mask：真实位置记 1，填充位置记 0。这样，loss 只在真实 token 上累积，padding 位不会影响平均值。这个逻辑看起来简单，但它正是很多训练脚本的隐形坑。

样本
x
y
mask
说明


样本 A
我 / 喜 / 欢 / 预测
喜 / 欢 / 预测 / token
1 / 1 / 1 / 1
完整样本，所有位置都参与 loss


样本 B
语 / 言 / PAD / PAD
言 / 模 / PAD / PAD
1 / 1 / 0 / 0
后两位是补位，只用于对齐 batch 形状

带 mask 的 loss 片段
复制

```python
loss_per_pos = cross_entropy(logits.reshape(-1, V), y.reshape(-1), reduction="none")
mask_flat = mask.reshape(-1).float()
masked_loss = (loss_per_pos * mask_flat).sum() / mask_flat.sum()
```

这段代码只做了一件事：先算出每个位置的损失，再用 mask 把补位去掉，最后按有效位置数平均。只要你记住这一点，看到 “padding” 和 “loss” 同时出现时，就知道先检查 mask，而不是先怀疑模型结构。

## box warn
### 典型故障
如果短句和长句混在一起训练，而你没有 mask，loss 往往会被 PAD 拉低或拉高，曲线看起来不稳定，但根因有效位置数算错了。

## box info
### 先查哪一项
先查 y 里有没有把 PAD 当真答案，再查 mask 有没有和 y 同步，再查 loss 平均时是不是只除以有效位置数。这个顺序通常比改模型参数更快。

# 从 logits 到概率，再到 loss {#softmax-lens}
<!-- tag: softmax -->

这一步是很多初学者第一次真正卡住的地方。logits 只是分数，真正可比较的是 softmax 之后的概率分布。只要你把这个转换看懂，loss 的数字就不再神秘。

假设某个位置上，模型对三个候选 token 的 logits 分别是 2.0、1.0、0.0。softmax 会把它们变成一组加起来等于 1 的概率。因为指数会放大差异，logit 高一点的 token 会拿到更多概率，但不会直接等于 1。

1.43


C
0.0
0.09

手算 softmax 的思路
复制

```python
import math

logits = [2.0, 1.0, 0.0]
exps = [math.exp(x) for x in logits]
total = sum(exps)
probs = [x / total for x in exps]

print([round(p, 3) for p in probs])
```

$$
 p_i = \frac{e^{z_i}}{\sum_j e^{z_j}}, \qquad L = -\log p_{\text{correct}}
 $$
 这里的 `z_i` 是 logits，`p_i` 是概率。正确答案的概率越高，`-log` 越小，loss 也越小。训练就是不断调参数，让正确 token 的 `z_i` 相对更高一点。

## box warn
### 常见误读
如果你把 logits 当成概率，看到 2.0、1.0、0.0 这种数时会误以为“最大值就是答案”。其实真正比较的是 softmax 之后的分布，而不是原始分数本身。

## box info
### 训练里真正被推动的是什么
被推动的正确 token 相对其他 token 的位置。这个相对关系一旦看懂，很多 loss 曲线的变化就能解释得通。

# loss 曲线怎么读 {#curve-reading}
<!-- tag: diagnosis -->

训练时最常见的误解，是把“loss 在变小”直接等同于“模型在变强”。更稳的读法是：先看训练 loss 的方向，再看验证 loss 是否跟上，最后看两者之间的距离有没有越来越大。

如果训练 loss 降得很快，但验证 loss 一直不动，通常数据切分、目标错位或过拟合在说话。如果训练和验证一起下降，而且下降幅度接近，说明模型大概率真的在学通用模式。

曲线现象
更像什么
下一步先查哪一类问题


训练 loss 持续下降，验证 loss 也下降
大体正常
继续看样本质量和收敛速度


训练 loss 下降很快，验证 loss 不动
过拟合、数据泄漏或监督写错
先查训练 / 验证分流，再查 input / target


两条曲线都很抖
batch 太小、mask 不稳或学习率不合适
先查 batch、mask 和学习率，而不是先加层数


训练 loss 几乎不变
模型没在学，或者梯度链路断了
先查监督对齐，再查 backward / step 有没有真的执行

## box soft
### 为什么要同时看两条曲线
训练 loss 只告诉你“模型对已见数据的拟合程度”，验证 loss 才告诉你“它有没有在陌生数据上保持预测能力”。只看一条，容易把记忆当能力。

## box info
### 这页的收尾动作
以后你读训练日志时，先问三件事：目标对了吗，mask 对了吗，训练和验证分开了吗。只要这三个问题没答清，先别急着改模型规模。

# 最常见的几种错法 {#mistakes}
<!-- tag: debug -->

这页最值得记住的不是定义，而是排错顺序。先查对齐，再查维度，再查训练和验证是否分开，最后才怀疑模型不够大。

症状
最可能的原因
先查什么


loss 一直不降
x / y 错位了，或者标签被打乱了
先看 input 和 target 有没有差一位


模型学会复制当前 token
输入和目标用了同一个切片
先看构造监督那一行


shape 报错
logits 的最后一维没对齐词表大小
先看 V 和 loss 输入维度


训练集很好，验证集很差
数据分流不干净，或者记住了重复样本
先看训练 / 验证分离


输出看起来像在说话，但不稳定
base model 还没经过指令微调或后续对齐
先分清 base 能力和聊天能力

## box warn
### 先查顺序
看到异常时，先看监督信号有没有对齐，再看 logits 的最后一维，再看训练 / 验证分流。顺序一乱，排错就会绕远路。

## box info
### 别把 base model 当聊天模型
这一页讨论的是 next-token objective。聊天能力、指令跟随和偏好对齐，会在后面的单元和后训练章节里再回来。

### 5 debug-checklist

**Q:** 先查什么

- [ ] 检查 input 和 target 是否错开一位
- [ ] 检查 logits 的最后一维是不是词表大小
- [ ] 检查训练集和验证集有没有混在一起
- [ ] 先把模型层数翻倍

Explain: 正确顺序是先查输入 / 目标错位、再查 logits 维度、再查数据分流。先翻倍模型层数通常只会把问题藏起来。

### 6 synthesis

**Q:** loss 与能力

Explain: 好的答案至少要提到三点：第一，交叉熵下降意味着模型给正确下一位 token 分到更高概率；第二，这只是在优化 next-token objective，还没有覆盖指令跟随、偏好对齐和对话格式；第三，聊天能力通常还要依赖后续的微调、偏好优化和解码设置。只要把这三层分开，答案就够了。
