# 块内机制 {#mechanism}
<!-- tag: mechanism -->

最小 decoder-only Transformer 的一层 block 里，读写顺序其实很固定：先归一化，再做 causal attention；attention 的输出回到原形状后做残差相加；然后再归一化，再做 MLP，再次残差相加。位置编码、权重绑定和输出头都要和这条顺序对齐。

$$
 \begin{aligned}
 D &= C / H \\
 a_0 &= \mathrm{Emb}(input\_ids) + \mathrm{Pos}(0:T-1) \\
 a_{\ell} &= a_{\ell-1} + \mathrm{Attn}_{\ell}(\mathrm{Norm}_1(a_{\ell-1})) \\
 x_{\ell} &= a_{\ell} + \mathrm{MLP}_{\ell}(\mathrm{Norm}_2(a_{\ell})) \\
 logits &= \mathrm{Norm}_f(x_L) \, W_{out}^{\top}
 \end{aligned}
 $$
 `C` 是模型宽度，`H` 是 head 数，`D` 是每个 head 的维度，`L` 是 block 层数，`W_out` 是输出头权重。这个式子里的每个符号都能在代码里找到落点。

## box soft
### 权重绑定怎么理解
很多最小实现会把输入 embedding 和输出头绑在同一张权重表上。读 token 和写 token 用的是同一套词表空间，既省参数，也让输入端和输出端的词表语义保持一致。你现在应该能做什么：看到 `logits = x @ self.token_emb.weight.T` 时，知道它是一种明确的词表对齐决定。

<!-- interactive: ['shift-demo'] → keep as HTML -->

$$
 D = C / H,\qquad
 Q,K,V: [B,T,C] \rightarrow [B,H,T,D],\qquad
 scores: [B,H,T,T]
 $$
 `D` 是每个 head 的维度。`T` 进到 attention 后会平方增长到 `[B,H,T,T]`，所以长上下文最先贵的通常就是 scores 这一块。

`[B,H,T,T]`

### 先记住一条成本线
`T` 对 attention 的成本是平方级，`V` 对输出头的成本是线性级，`C` 会同时影响 embedding、attention、MLP 和输出投影。你现在应该能做什么：看到一个配置，就能先判断贵在哪里。

# 最小代码 {#code}
<!-- tag: code -->

下面这段是教学用 PyTorch 版本。它保留了最小前向路径：embedding、causal attention、SwiGLU MLP、pre-norm、残差、输出头绑定。它没有尝试覆盖所有现代变体，只负责把结构写清楚。

mini_transformer_lm.py复制

```python
import math
import torch
import torch.nn as nn
import torch.nn.functional as F

class RMSNorm(nn.Module):
 def __init__(self, dim, eps=1e-6):
 super().__init__()
 self.eps = eps
 self.weight = nn.Parameter(torch.ones(dim))

 def forward(self, x):
 scale = torch.rsqrt(x.pow(2).mean(dim=-1, keepdim=True) + self.eps)
 return self.weight * x * scale

class CausalSelfAttention(nn.Module):
 def __init__(self, dim, n_heads, block_size):
 super().__init__()
 assert dim % n_heads == 0
 self.n_heads = n_heads
 self.head_dim = dim // n_heads
 self.qkv = nn.Linear(dim, 3 * dim, bias=False)
 self.proj = nn.Linear(dim, dim, bias=False)
 mask = torch.tril(torch.ones(block_size, block_size))
 self.register_buffer("mask", mask.view(1, 1, block_size, block_size), persistent=False)

 def forward(self, x):
 B, T, C = x.shape
 q, k, v = self.qkv(x).chunk(3, dim=-1)
 q = q.view(B, T, self.n_heads, self.head_dim).transpose(1, 2)
 k = k.view(B, T, self.n_heads, self.head_dim).transpose(1, 2)
 v = v.view(B, T, self.n_heads, self.head_dim).transpose(1, 2)
 scores = (q @ k.transpose(-2, -1)) / math.sqrt(self.head_dim)
 scores = scores.masked_fill(self.mask[:, :, :T, :T] == 0, float("-inf"))
 att = F.softmax(scores, dim=-1)
 y = att @ v
 y = y.transpose(1, 2).contiguous().view(B, T, C)
 return self.proj(y)

class SwiGLUMLP(nn.Module):
 def __init__(self, dim, hidden_dim):
 super().__init__()
 self.up = nn.Linear(dim, hidden_dim, bias=False)
 self.gate = nn.Linear(dim, hidden_dim, bias=False)
 self.down = nn.Linear(hidden_dim, dim, bias=False)

 def forward(self, x):
 return self.down(F.silu(self.gate(x)) * self.up(x))

class Block(nn.Module):
 def __init__(self, dim, n_heads, block_size, mlp_ratio=4):
 super().__init__()
 self.norm1 = RMSNorm(dim)
 self.attn = CausalSelfAttention(dim, n_heads, block_size)
 self.norm2 = RMSNorm(dim)
 self.mlp = SwiGLUMLP(dim, int(mlp_ratio * dim))

 def forward(self, x):
 x = x + self.attn(self.norm1(x))
 x = x + self.mlp(self.norm2(x))
 return x

class MiniTransformerLM(nn.Module):
 def __init__(self, vocab_size, dim, n_layers, n_heads, block_size):
 super().__init__()
 self.block_size = block_size
 self.token_emb = nn.Embedding(vocab_size, dim)
 self.pos_emb = nn.Embedding(block_size, dim)
 self.blocks = nn.ModuleList([Block(dim, n_heads, block_size) for _ in range(n_layers)])
 self.norm = RMSNorm(dim)

 def forward(self, input_ids):
 B, T = input_ids.shape
 if T > self.block_size:
 raise ValueError("sequence is longer than block_size")
 pos = torch.arange(T, device=input_ids.device)
 x = self.token_emb(input_ids) + self.pos_emb(pos)[None, :, :]
 for block in self.blocks:
 x = block(x)
 x = self.norm(x)
 logits = x @ self.token_emb.weight.T
 return logits

model = MiniTransformerLM(
 vocab_size=16000,
 dim=256,
 n_layers=4,
 n_heads=4,
 block_size=128,
)
input_ids = torch.randint(0, 16000, (2, 128))
logits = model(input_ids)
print(logits.shape) # [2, 128, 16000]
```

代码位置
读入什么
写出什么
去掉会怎样


`self.token_emb(input_ids)`
`[B,T]` token id
`[B,T,C]` token 向量
模型只能看到整数编号，无法进入连续表示空间


`self.qkv(x).chunk(3, dim=-1)`
`[B,T,C]`
三份 Q/K/V
attention 没有发问、匹配和读取三类角色


`scores = scores.masked_fill(...)`
`[B,H,T,T]` scores
未来位置为 `-inf`
模型训练时会偷看后文，语言建模目标被破坏


`y = y.transpose(1, 2).contiguous().view(B, T, C)`
`[B,H,T,D]`
`[B,T,C]`
transpose 后内存不连续，后续 view 可能报错或读错布局

变体

Learned position embedding
保留在最小实现里，位置直接加到 token 向量上
看 `self.pos_emb(pos)` 这类位置表
最容易理解，适合先建立位置和 token 的分工


RoPE
先不进最小版

QK-Norm
先不进最小版

FlashAttention
先不追 kernel 级实现
看同一条 attention 公式怎么换成高效算子
会把“结构怎么读”提前变成“性能怎么跑”


KV cache
留给生成和推理单元
看自回归生成时 key/value 怎样复用历史状态
会把训练前向和生成前向混到一起


权重绑定
保留
看输入 embedding 和输出 head 是否共用同一张表
如果不讲清楚，输出头会像一个独立模块一样漂着

# 实现一致性检查 {#consistency}
<!-- tag: check -->

这页看起来讲的是结构，实际上还在训练你做一件更重要的事：让形状、公式和代码互相咬住。只要有一处对不上，模型就可能“能跑”，但语义已经跑偏。下面这张表就是你写最小实现时的手工检查单。

检查点
应当一致的地方
最容易出错的位置
一眼能看到的症状


符号闭合
`C`、`H`、`D`、`V`、`B`、`T` 的定义和使用要一致
公式、代码注释、练习题的维度写法
公式看懂了，代码却对不上，或者反过来


位置逻辑
position embedding 只加一次，或者 RoPE 只走 attention 路径
embedding 初始化和 attention 前处理
位置重复注入，生成和训练口径不一致


形状闭环
attention 和 MLP 最后都回到 `[B,T,C]`
输出投影、down projection、残差相加前
残差加不回去，或者只能靠硬 reshape 修补


内存布局
`transpose` 后再 `view` 时要保证连续
attention 输出重排
运行时布局报错，或者得到看似正确的错结果


接口边界
`forward(input_ids)` 只返回 logits，不偷偷算 loss
模型类和训练循环的分工
模型本体和训练逻辑搅在一起，后面没法复用

## box info
### 你现在应该能做什么
拿到一份最小 Transformer 实现，你应该先检查符号、形状、位置注入和输出接口是否闭合，再去看性能细节。这个顺序能让你先抓住“对不对”，而不是先被“快不快”带偏。

### 9 解释

**Q:** Q9 为什么在 `transpose(1, 2)` 之后要补 `contiguous()`，再做 `view(B, T, C)`？

Explain: transpose 会改变张量的步幅，张量常常不再是连续布局； view 依赖可直接重解释的连续内存，如果不先补 contiguous() ，就可能报错或按错误布局重排。把这点说清，就说明你知道这里布局安全的必要步骤。
