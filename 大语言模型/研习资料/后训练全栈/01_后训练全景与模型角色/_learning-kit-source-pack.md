## 来自旧页的现有教学骨架

旧页已经包含这些可复用主题：

- 学习目标：看懂流水线、识别角色边界、建立后续语言
- 概念骨架：`base -> SFT -> policy/reference -> reward/judge -> DPO/RLHF -> 评测/发布`
- 术语与误区：`base model`、`chat model`、`policy`、`reference`、`reward model`、`judge`、`reward hacking`
- 角色速查：谁更新、谁冻结、谁打分、谁做发布前检查
- 核心讲解：后训练在解决什么问题、`base` 到 `SFT`、`SFT` 到 `policy/reference`、`reward` 与 `judge` 的位置差别、三条常见路线
- 案例：DPO 配方里的对象、GRPO 训练环里的对象、同一个模型换角色
- 常见错误：把 `base` 当 `chat`、让 `reference` 跟着更新、把 `judge` 分数当真值、只看最终分数不看角色链
- 练习：18 题，分三组，覆盖角色识别、流程排序与诊断、迁移应用

## 适合搬进新页的关键判断

- `reference model` 在多数 DPO / RLHF 配方里应该冻结
- `policy model` 是被更新的对象
- `reward model` 服务训练内环
- `judge / evaluator` 服务训练外环或发布前检查
- 同一个 checkpoint 可以在不同阶段承担不同职责，但职责切换后，读写关系就变了
