SFT 训练配置与快速实验

本页把上一单元已经看懂的样本边界，翻成一份能跑的 SFT smoke test：先定显存和 token 预算，再定 batch、packing、LoRA、学习率、日志和 checkpoint，最后用最小实验检查配置是否自洽。读完这一页，你应该能先把训练计划写成可解释的参数表，再考虑放大数据量。中间会穿过一轮锚点样本走查，把配置、长度、输出和保存放到同一条路径上看，避免把参数名背熟了，却还是不知道哪一步先坏。

学习目标

start

本页的目标让你能把一份 SFT 配置拆开，看出每个旋钮在控制什么。

如果你能把一份小实验配置读成“样本怎么进来、token 怎么装、哪些参数被训练、什么时候留日志”，你就已经能开始跑 smoke test 了。

先抓主线

big

这页的主线很简单：把上一单元已经验证过的样本边界，翻译成一份能跑的小实验。训练器不需要你先想“模型会不会变强”，它先要你回答“这份配置能不能稳定地把数据喂进去，把参数更新出去，并把结果留下来”。

拿一个常见的小实验当锚点更好记：1 张 24GB GPU、一个 0.5B 到 1.5B 的 chat model、3000 条短指令数据。这个规模不追求极限效果，先追求三个东西：能跑、能解释、能回滚。只要这三件事成立，后面才有资格放大。

读信号

loss、样本和 checkpoint 一起看

最小配方

短样本很多

packing、max_length、gradient_accumulation_steps

想要任务样式变化

use_peft、lora_r、target_modules、learning_rate

先确认数据和模板，再看 LoRA 有没有接到正确层。

容量不够时看起来像没学，容量过大时又会把风格推歪。

$$ B_{\text{eff}} = D \times B \times A,\quad T_{\text{step}} \approx B_{\text{eff}} \times L $$

先看 B_eff，再看 L。B_eff 决定每次更新看多少样本，L 决定这些样本平均有多长。packing 改的是 L，gradient accumulation 改的是 B_eff，它们不在同一层。

batch 与显存

math

batch 的问题，表面上是在问“每次喂多少样本”，实际上是在问“每次优化器更新到底看了多少 token”。这个问题如果只看 per_device_train_batch_size，会漏掉梯度累积和序列长度。

D 是设备数，B 是每卡 micro batch，A 是 gradient accumulation steps。这个式子只数样本，不数 token。

L 是处理后的平均序列长度。开 packing 以后，L 往往更接近窗口上限；关掉 packing 时，L 更接近原始样本长度的平均值。

例子：1 张卡、micro batch=1、accum=8、平均长度约 1024

effective_batch = 8

tokens_per_step ≈ 8192`

如果你算出来的 token 量和显存表现对不上，优先检查是不是把 padding、packing 或平均长度估错了，而不是急着改学习率。

packing

efficiency

packing 的目标只有一个：把短样本之间的空白压下去，让 GPU 少算 padding。它解决的是效率，不是质量。坏样本、错模板、漏标签，packing 一个都修不了。

LoRA 与学习率

capacity

LoRA 解决的是“改多少参数”和“用多大步子”这两个问题。r 决定可训练容量，target_modules 决定哪几层真的被接上可训练分支，learning_rate 决定每次更新迈多大步。

learning_rate当前数据规模和 LoRA 容量。每次参数更新的步幅。太小像没学，太大容易把风格推歪。先看 loss 曲线和少量样本输出。

日志与 checkpoint

observability

训练不只是更新参数，还要留下证据。没有日志和 checkpoint，失败后你只能猜。这个部分要做的事很简单：让训练过程可读、可回滚、可对比。

排障顺序

terminal

当训练出问题时，最省时间的顺序通常先看症状，再把问题收回到最小检查点。下面这个小型失败实验就是为了训练这个顺序。

[error] CUDA out of memory in first forward pass

first check: per_device_train_batch_size, max_length, gradient_checkpointing

next check: did you turn packing on before single-sample labels were correct?

[warn] loss curve almost flat for 200 steps

first check: learning_rate too small or labels mostly -100

next check: did you accidentally freeze every trainable module?

[warn] sample output repeats prompt or role markers

first check: dataset format and template

next check: assistant_only_loss / completion_only_loss against the actual sample shape

[warn] checkpoint writing takes longer than training step

first check: save_steps, save_total_limit, output_dir on slow disk

next check: are you saving full weights when adapter checkpoints are enough?
