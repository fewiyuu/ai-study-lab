课程总论

后训练把会续写文本的模型，调整成能完成任务、遵守偏好和适应人类交互的模型。

Big Picture

后训练的主线不是某一个算法，而是一条数据、目标函数、训练稳定性、评测和发布约束共同组成的链条。

课程先从监督微调开始，建立指令数据、chat template、loss mask 和评测的基础；再进入偏好数据与 DPO，把“哪个回答更好”转成可优化信号；最后进入 PPO/GRPO 等强化学习路线，理解奖励、采样、KL 约束和训练稳定性的关系。

这门课强调全栈视角：数据格式、训练命令、算法直觉、评测口径和失败模式必须一起看。只会跑一个 trainer，不等于理解后训练。

02

SFT 数据、模板与 loss mask

存在理由：监督微调的质量很大程度取决于数据格式和哪些 token 被训练，必须先学会看样本。

打开学习页
03

SFT 训练配置与快速实验

存在理由：用 TRL 和 CLI 把 batch、packing、LoRA、学习率、日志和 checkpoint 串成可复现实验。

打开学习页
04

偏好数据与奖励建模

存在理由：偏好优化从人类或模型比较中学习，必须理解 pairwise 数据、标注噪声和 reward hacking 风险。

打开学习页
05

DPO、ORPO 与直接偏好优化

存在理由：这类方法是后训练常用入口，适合学习 reference model、隐式奖励和过优化边界。

打开学习页
06

PPO、GRPO 与 RLHF 训练环

存在理由：强化学习阶段引入在线采样、奖励和 KL 约束，是理解推理训练和工具使用训练的桥梁。

打开学习页
07

评测、判别器与回归检查

存在理由：后训练很容易提升某项能力同时伤害其他能力，必须建立自动评测和人工抽检的组合。

打开学习页
08

发布边界与安全回退

存在理由：模型上线前要检查幻觉、拒答、隐私、领域偏差和回退策略，不能只看 leaderboard 分数。

资料边界

后训练工具迭代很快，课程重点放在可迁移结构。

Boundary

本课程不承诺覆盖 TRL 的每个 trainer、每个参数或每个新算法；具体 API、默认参数和推荐配置应以当前官方文档与仓库为准。

模型安全、偏好标注和人类反馈涉及组织策略与伦理判断，本课程只提供技术学习框架，不替代真实发布审查。
