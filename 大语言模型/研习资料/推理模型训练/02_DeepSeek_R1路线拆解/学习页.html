术语边界

先直觉后定义
术语直觉解释精确定义常见误解
Cold-start先给模型少量高质量示范，让它知道推理答案该怎么写。在 RL 前用人工或筛选得到的少量长推理样本做监督微调，改善可读性、格式和初始探索质量。以为 cold-start 是主要能力来源；它更像启动器，不是最终的推理能力放大器。
RL让模型自己多试几种解法，答对的方向被加强。在可验证任务上采样回答，用准确性奖励、格式奖励等信号更新策略模型，使高奖励回答概率上升。以为 RL 必须依赖人类逐条评价；R1 路线强调自动可验证奖励。
SFT把已经筛出来的好解法变成稳定教材。Supervised Fine-Tuning，用输入和目标输出对训练模型。R1 路线里 SFT 用于冷启动，也用于把高质量推理和通用数据混合整理。以为 SFT 和 RL 二选一；实际常常交替使用，解决不同问题。
Distillation让小模型模仿强模型写出的推理过程和答案。用强模型生成或筛选出的样本训练小模型，使小模型学习强模型的输出分布、推理格式和部分能力。以为蒸馏等于复制全部能力；小模型仍受容量、数据覆盖和训练目标限制。

路线总览

R1-Zero

1. R1-Zero：先证明纯 RL 能激发推理行为

R1-Zero 的关键意义证明在合适的可验证任务和奖励下，基础模型可以通过强化学习出现更长的推理、反思和自我检查行为。它把注意力从“必须先大量人工标注思维链”转向“能不能用可判分任务直接优化推理行为”。

但纯 RL 也带来问题：可读性可能差，输出格式可能漂移，语言混杂，训练出的行为不一定适合普通用户直接使用。因此 R1-Zero 是路线的科学信号，不是完整产品配方。

判断句

如果一个阶段主要回答“可验证奖励能不能让模型自己学会探索推理”，它更接近 R1-Zero；如果它同时关心可读性、通用能力和部署可用性，它更接近 R1。

0

基础模型已有语言、知识和基本解题能力。没有这个底座，RL 只是在低质量策略上放大噪声。

1

Cold-start SFT给模型少量高质量长推理示范，改善可读性和格式，让后续探索更容易落在可训练区域。

2

Reasoning RL在数学、代码等可验证任务上采样多个回答，用准确性和格式等信号强化高奖励轨迹。

3

SFT 整理筛选高质量推理样本，并混合写作、问答、工具等通用数据，修复只会推理题的偏科问题。

4

对齐与安全继续优化帮助性、无害性和交互质量，避免模型只追求可验证分数而牺牲用户体验。

5

蒸馏小模型把强推理模型产生的数据用于训练较小模型，让能力以更低推理成本落地。

四个核心阶段

分工表

Cold-start：降低探索难度

没有 cold-start 时，模型直接进入 RL，可能会把奖励钻到奇怪格式上：答案也许能被判对，但推理文本难读、格式不稳定，甚至出现语言混杂。cold-start 的作用是先把“推理轨迹应该像什么”教给模型，让它学会清楚展开、反思、检查和给出最终答案。

这里要区分两个概念：能力是能否解题，轨迹形态是如何表达解题过程。cold-start 主要稳定轨迹形态，也给 RL 一个更好的初始策略。

RL：把可验证奖励变成能力放大器

RL 阶段的核心是可验证奖励。数学题可以比对最终答案，代码题可以跑测试，逻辑题可以用规则检查。模型每次对同一题采样多个候选，系统给每个候选奖励，训练算法再提高高奖励候选的概率。

这也是后面 GRPO、rollout、优势估计要解决的问题：不是只知道“哪个答案对”，还要知道“在同一批候选里哪个更好、好多少、更新会不会太猛”。本页只建立路线，算法细节会在后续单元展开。

对一道题采样 G 个回答：{y1, y2, ... yG}

用 verifier 得到奖励：r1, r2, ... rG

训练目标：提高相对更高奖励回答的概率，同时用 KL 等约束避免模型偏离过快。

SFT：把探索结果整理成稳定能力

RL 会产生大量轨迹，但不是每条都适合继续训练。SFT 在后半段的作用，是把高质量推理轨迹筛出来，和非推理类通用数据一起训练。这样做有两个好处：一是把 RL 学到的解题模式固化下来；二是补回聊天、写作、事实问答等通用能力，避免模型变成只会做竞赛题的狭窄系统。

Distillation：把强模型能力迁移给小模型

蒸馏用强模型产出的推理数据训练小模型。对资源有限的团队来说，这是很重要的路线：直接训练大规模 RL 昂贵，但用强模型生成或筛选数据做 SFT，能让较小模型获得相当一部分推理表现。

不过蒸馏有边界。小模型并没有自动获得强模型的全部搜索深度、知识覆盖和泛化能力；它学到的是数据中出现过、容量能承载、训练目标能稳定吸收的部分。

四阶段对照

Cold-start解决“起步太乱”。输入是少量高质量推理样本，输出是更可读、更稳定的初始策略。

RL解决“能力如何放大”。输入是可验证任务和奖励，输出是更会探索和验证的推理策略。

SFT解决“能力如何整理”。输入是筛选轨迹和通用数据，输出是更稳、更可用的模型。

Distillation解决“能力如何迁移”。输入是强模型数据，输出是推理能力更强的小模型。

观察顺序：cold-start 让 RL 更容易开始，RL 产生新能力，SFT 稳定和混合能力，distillation 把能力转给更便宜的模型。

例题

路线判断

例题 1：为什么“只做 SFT”不等于 R1 路线？

假设你有 100 万条数学题思维链，直接拿它们 SFT 一个模型。这个模型可能会学会更像推理模型的输出格式，也可能在覆盖到的题型上提升，但它没有经历“自己采样、被 verifier 判分、根据奖励调整策略”的过程。

route-checkconcept
只做 SFT：
题目 → 标准推理轨迹 → 模型模仿

R1 风格 RL：
题目 → 模型采样多个轨迹 → verifier 判分 → 奖励驱动策略更新

关键差异：
SFT 学的是“给定教材的模仿”；
RL 学的是“在可判分环境中，哪些探索行为更可能成功”。

所以 SFT 是 R1 路线里的重要工具，但不是路线的全部。它能稳定表达，也能整理 RL 产物；真正把“试错-奖励-更新”引入训练闭环的是 RL。

例题 2：为什么“只看最终答案”会误判推理训练？

如果一个模型在训练集答案上得分高，但推理过程充满不可读跳步，或者在换一种表述的题上快速崩溃，这不能说明路线成功。R1 类训练要同时观察答案正确率、推理长度分布、格式稳定性、跨 benchmark 泛化和 verifier 是否被钻空子。

观察项说明风险

最终答案是否被标准答案或测试通过可能过拟合题库或答案格式

推理轨迹是否有分解、检查、回溯可能只是写长，不是真的验证

格式约束是否能输出可解析答案格式奖励太强会压过解题能力

泛化评测新题型、新数据集是否仍提升只学会 benchmark 套路

常见错误

诊断清单

错误 1：把长 CoT 当作目标本身

更长的推理文本只是可能出现的行为，不是最终目标。目标是更可靠地解决可验证问题。只奖励长度会鼓励水字数。

错误 2：把 cold-start 当作全部秘密

冷启动能改善初始策略，但 R1 路线的关键突破在于可验证奖励驱动的 RL，以及后续整理、对齐和蒸馏。

错误 3：把蒸馏当作完整复现

蒸馏可以产出强小模型，但它更多是在模仿强模型数据，并不等于复现了大模型的 RL 训练过程。

错误 4：忽视 verifier 的漏洞

奖励函数就是训练方向。如果答案解析、单元测试或格式奖励有漏洞，模型会学习利用漏洞，而不一定学习真实推理。

最重要的修法

描述任何 R1 风格方案时，都用四问检查：训练前有没有冷启动或等价初始化？RL 的任务是否可验证？RL 后有没有筛选和 SFT 整理？能力是否通过蒸馏或评测迁移到目标模型？

一个最小实验配方

如果你想自己试一个迷你版本，可以用 20 道能自动判对的数学题，先手写一轮 cold-start 风格示范，再让模型对每题采样 4 个候选，最后用正确答案打分。先看 reward 和未见题是否同步上升，再看长度和格式有没有开始投机。这个配方不追求性能，追求的是把“阶段顺序”看清楚。

一旦你能在这个小实验里分辨出“答案更对”和“只是更会写”，你对 R1 路线的理解就已经从概念层转到操作层了。

迷你闭环

把一题走完

同一题怎么把整条路跑通

拿题目 2 + 3 * 4 = ? 做最小例子。它简单到能手算，但足够把 R1 路线的关键动作露出来：先有一条像样的示范，再让模型自己采样，再用 verifier 判分，最后把高质量轨迹送回 SFT。你看的“这条路径能不能被稳定复用”。

如果 cold-start 缺席，模型可能一开始就把步骤写乱；如果 RL 缺席，模型只会模仿教材，不会在反馈里变好；如果 SFT 缺席，RL 产出的好轨迹会散掉；如果蒸馏缺席，能力就停在大模型上，没法往小模型迁移。

题目 → 采样 4 条回答 → verifier 判分 → 选高分轨迹 → 回写 SFT 数据 → 再次训练策略。

这个闭环的重点每个环节都只做一件事：cold-start 负责定形，RL 负责放大，SFT 负责整理，distillation 负责迁移。

手工判分表

候选回答奖励原因
A先算 3×4=12，再算 12+2=14高答案对，顺序清楚，适合回收进 SFT。
B从左到右算成 20低算术规则错了，说明策略没有学会正确约束。
C写了很多步骤，最后答 14中答案对，但如果长度奖励太强，模型可能偏向冗长。
D只回 14中高最终答案对，但解释不足；是否保留取决于后续训练目标。

失败会怎么冒出来

如果 verifier 只看最后数字，候选 C 和 D 可能一起被抬高；如果长度项太强，模型会更像 C；如果 SFT 只吃 A 这一类样本，小模型会学到“推理模板”，却不一定学到稳定判题能力。

所以这个闭环把“探索、判分、整理、迁移”拆成四个互相配合的动作。你看懂这个闭环后，再回头看论文里的术语，就会知道每个词是在改哪一个状态。

自己换一道能自动验证的题，按“采样 4 条、判分、选样本、回写数据”写一遍，不要先追公式。能把这个流程写顺，你就已经能独立判断一个训练方案是不是只会喊口号。

再跑一遍：12 - 4 / 2

这道题比前面的加法多了一个优先级问题，刚好能看出 verifier 不是只管“有没有答案”，还要管“算式有没有按规则走”。正确轨迹应先算除法，再做减法，得到 10；如果模型把它从左到右算成 4，奖励就应该压下去。

正确轨迹：
12 - 4 / 2
= 12 - 2
= 10

如果你在这一题上只奖励最终数字，不看中间步骤，就很容易把“碰巧答对”和“真的会算”混在一起。R1 路线真正要放大的，是能稳定复用规则的那种行为。

把练习接上来

你现在可以自己选一道能自动判定的题，先写出一个好轨迹和一个坏轨迹，再给它们标高低奖励。这个动作看似朴素，但它是后面 rollout、优势估计和 GRPO 的起点：没有手工判分，就很难判断模型究竟学到了什么。

如果你能把“题目、候选、奖励、回写样本”四件事连起来说清楚，就已经能读懂后续单元为什么要先讲 reward，再讲采样，再讲优化稳定性。

Q: 诊断：只用大模型生成 50 万条 CoT 后 SFT 小模型，这算完整 R1 复现吗？

参考答案 / 评分要点

。不是完整复现。它更像蒸馏或整理阶段，因为缺了在线 RL 的采样、奖励和策略更新。更完整的回答还会补一句：如果没有 verifier 和奖励环，模型没有经历“自己试、自己被判分、自己调整”的过程。

6迁移

Q: 迁移：如果任务是开放式写作，为什么不能直接照搬数学 RL 奖励？

参考答案 / 评分要点

。开放式写作没有像数学那样稳定的唯一答案，奖励如果照搬“正确性优先”会变成单一模板或过度迎合某个评分器。更好的回答会说清：开放式任务更需要多维反馈、人工偏好、风格约束和安全约束。

7综合解释

Q: 综合解释：为什么 R1 后半段还需要 SFT 和对齐？

参考答案 / 评分要点

。RL 往往会把模型推向能刷分的方向，但用户真正要的是稳定、可读、可控、通用的回答。SFT 和对齐负责把刷分后学到的解题行为重新整理成可用行为，并把安全与交互问题压回可接受范围。

8设计

Q: 设计：给一个小型 R1 风格实验，写出任务、奖励、阶段和评测。

参考答案 / 评分要点

。答案不需要像论文那样长，但要有完整闭环：任务必须可验证，奖励必须直接连到答案，采样要说明每题多候选还是多题覆盖，评测要用未污染集合检查泛化。只写“用 GRPO 训练”不够。

9code reading

Q: 读下面这个奖励设定，最主要的问题是什么？

参考答案 / 评分要点

它没有验证正确答案，容易奖励格式和长度投机。正确答案是 A。奖励主导信号是长度和标签，真正的正确性没有被验证。更完整的回答还会补一句：这类 reward 最容易把模型推向模板化和长度投机，而不是解题能力。

10scenario-transfer

Q: 如果 reward 上升了，但未见题评测没有同步提升，第一步先查什么？

参考答案 / 评分要点

先查 verifier、训练题和评测题是否污染，奖励是否对准最终答案。正确答案是 B。先查奖励和数据边界，后查采样和优化器。这个顺序的好处是能先排除“方向错了却被更快地放大”的问题。

阅读检查

state

拿同一题 2 + 3 * 4 做例子，你可以很快看出每个阶段到底在读什么、写什么、容易坏在哪。这个检查比背概念更有用，因为它把路线变成了可追踪的状态变化。

日志实验台

failure

如果你怀疑实验在“看起来变好，实际没变强”，先把日志拆成三种最容易错的情况：奖励跟着长度跑、奖励跟着格式跑、评测和训练题太像。下面这个小实验台就是为了让这种差异一眼可见。

如果评测没有同步提升

先查 verifier、训练题和评测题是否污染，再查任务是不是过于单一，最后看模型是不是只在同一种题型上学会了局部套路。

同题追踪

one

拿同一题 2 + 3 * 4 来看，路线就不再是抽象名词。基础模型先给出一个答案方向，cold-start 把思路写得更稳，RL 用奖励把真正能被验证的候选推上去，SFT 和蒸馏把这些轨迹整理成更小、更稳、更便宜的模型。你只要盯住同一个题，就能看出每一阶段到底改了什么。

四段解读

Base 阶段的价值是提供语义和格式底座。你在这里确认模型有没有足够的基础能力去承接后面的试错。

Cold-start 的价值是把轨迹做得像能被训练的样子。它不负责把能力“凭空创造出来”，它只是把 RL 之前的起步姿态拉回可读、可学的位置。

RL 的价值是把“能被验证的成功”放大。它让模型自己采样、自己犯错、自己被奖励拉回正确方向，真正改变的是偏好分布，不是表面长度。

整理与迁移

SFT 和蒸馏做的是收束工作。前者把高质量轨迹重新揉回到通用对话和稳定解题里，后者把同样的轨迹变成更小模型可承受的训练数据。

如果这一步做坏了，常见症状“只会在训练题上做题”。这就是为什么路线后半段必须把对齐、数据覆盖和评测一起看。

这段追踪最重要的结论很简单：同一题在不同阶段的“输出形态”可以完全不同，但真正值得保留的是可验证成功率、稳定的表达和跨题型的泛化，而不是某一种固定话术。

失败后果

如果你把这条线看成“长思维链会自动带来强推理”，就会错过最关键的因果：R1 的变化来自训练信号的顺序和边界，不是来自某种神奇模板。长度可以是副产物，但不能当目标；格式可以帮起步，但不能当答案；蒸馏可以降成本，但不能替代在线 RL。

所以你以后看到一个“R1 风格”方案，先问自己这三个问题：有没有冷启动来稳轨迹？有没有可验证奖励来放大成功？有没有 SFT / 蒸馏把能力收回来？这三问比记任何缩写都更值钱。
