提示工程学习笔记

这门课把提示工程拆成五段：先写清任务结构，再用样例和格式稳住输出，然后把推理、搜索、分解与抽象接进来，最后再把知识增强和多模态一起接起来。

它适合已经会写基础 prompt，但还想继续把任务写稳的人。每一页都围绕一个可检查的对象：提示结构、示例、推理链、触发词、检索证据和非文本输入。

01 基础 02 示例与指令 03 推理提示 04 搜索、分解与抽象 05 知识增强与多模态

先从哪里进

如果你刚开始学，先走 01。已经会写结构化提示的人，可以直接从 03 看推理链，再接 04 和 05。

课程概览

big picture

这门课在学什么

它不只教“把一句话写顺”，而是教你判断任务缺什么：缺结构、缺样例、缺推理、缺触发模式、缺外部知识，还是缺图表和数学输入的处理方式。

学完之后能做什么

你可以把一个模糊请求拆成可执行流程，写出可检查的 prompt，判断什么时候该改模板、什么时候该换样例、什么时候该引入检索或多模态输入。

这门课基于什么

主要来自课程里的本地整理材料，再配合公开的提示工程指南和相关论文。每一页都尽量把原文里的方法，翻成能直接检查的流程。

放在这里为什么合适

这个课程目录把提示工程从基础写法一路连到复杂流程，适合把零散方法连成一条线，而不是把每个技巧都当成孤立知识点。

推荐路线

COURSE MAP

Phase 1

先把任务结构和提示骨架写稳，知道指令、上下文、输出格式和约束分别在做什么。

Phase 2

再看少样本、零样本和指令优化，理解示例和格式怎么改变输出稳定性。

Phase 3

最后把推理、搜索、分解与抽象、知识增强和多模态接上，理解复杂任务怎样从一段 prompt 变成一套流程。

01 基础 -> 02 指令与示例 -> 03 推理提示 -> 04 自动提示优化 -> 05 知识增强与多模态

推理型提示：CoT、Self-Consistency与生成知识

把检索、知识生成、主动标注和多模态输入接到提示工程里，理解它如何变成系统能力。
