第3讲：剖析大型语言模型

这一讲的重点不是继续讲“LLM 会回答”，而是把回答拆进内部：token 先变成 embedding，再经过多层 Transformer 变成 contextualized representation，最后通过 LM head 和 softmax 回到 token 分布。你读完后，应该能把一段输出异常先定位到表示层、attention 层、FFN，还是最后的解码层。

先定检查点

goal

P03 这讲到底在做什么

这讲把前一节已经“会说话”的 LLM 往内部拆：输入如何进入模型、每层表示怎样改变、最后怎么回到词表分数。它的价值在于给后面讲可解释性、实验观察和故障排查铺一张地图。

如果只记住“attention 很重要”会不够。你需要知道它在一层里读什么、写什么，FFN 负责什么，LM head 为什么又把隐藏状态投回词表。

本页只要求你掌握这些

  • 把一个 token 序列追到 embedding、hidden state、logits 和 next token。
  • 说明 representation 为什么会随层数变化，尤其是同形词在不同上下文里会分开。
  • 知道 attention、FFN、LM head 各自在哪一步读写信息。

现在你应该能做什么：看到一个中间层张量时，先问它是“表示”、还是“候选分数”，而不是先背术语。

全流程地图

trace

先把一轮前向写成同一条链。它的骨架很简单：token 先变成 embedding，Transformer 层一层层改写表示，最后 LM head 把最终表示映射到词表分数，softmax 之后才是可以采样的概率。

2embedding tabletoken id初始 hidden state模型看到的起点偏了，representation 从一开始就失真。

术语边界

terms

这几个词先分开

术语这一讲怎么用别误会成

representation某一层里 token 的连续向量表示。词义标签本身。

contextualized representation带上下文的表示，能分化同词异义。固定不变的词向量。

logits词表里各 token 的原始分数。概率。

logit lens把中间层表示投回词表，看它像什么候选 token。真正的最终答案。

patch scope在某个层或某个位置上替换 / 观察表示。已经完整解释模型全局行为。

先知道，先别钻

这一页会碰到 residual、layer norm、multi-head、causal mask，但先不要把注意力分头、KV cache、训练目标和长上下文优化一起追进去。它们都重要，只是这一讲先要把“表示怎样被逐层改写”讲清楚。

表示怎样漂移

representation

同一个 token，先相同后分化

一开始，模型只拿到 token id，所以同一个 token 在不同句子里会先走到同一个 embedding 附近。真正开始分化的是后面的层：attention 会把别的位置的信息拉进来，FFN 会把当前位置的特征重新组合，于是 contextualized representation 逐层变得更像“此时此地的 apple”，而不是词典里孤立的 apple。

这也是为什么你不能只看第一层就判断模型懂不懂。早层更像把词放进坐标系，后层才是在坐标系里把词变成语境对象。

一张 state / read / write 表

位置读什么写什么常见症状

embeddingtoken id初始向量切词错时，从起点就歪。

attention别的位置表示混合后的上下文表示长距离依赖没接上。

FFN当前位置表示局部特征变换后的表示特征没被提炼，层间变化太平。

LM head最终表示logits候选 token 排序错。

$$

z = W_{\text{lm}} h + b

$$

这里 h 是某一层的 hidden state，W_{\text{lm}} 是 LM head 的投影矩阵，z 是 logits。这个式子告诉你：中间表示要先经过线性投影，才会回到词表空间。

attention matrix 怎么读

matrix

先看方向，再看强度

attention matrix 的一个格子表示“当前位置会从哪个位置读多少信息”。对角线附近常常很强，因为当前位置通常会保留自己的信息；往左看时，模型在因果 mask 下只能读前文，不能偷看未来。

这张图不是解释器本身，它只是告诉你信息从哪里被拉进来。真正发生了什么，还要看 Q、K、V 各自带了什么，以及 residual 把什么保留下来。

部件读什么写什么排错时看点

Q当前位置的查询需求用于匹配的向量模型现在想从哪里找信息。

K每个位置的可匹配特征供别人来搜的向量哪些位置容易被选中。

V真正要汇入的内容被加权汇总的信息最后被搬回来的到底是什么。

FFN 在做什么

ffn

attention 负责“读”，FFN 负责“改写”

可以先把 attention 理解成跨位置的信息搬运，把别的 token 读进来；FFN 更像在每个位置内部做特征重组和非线性变换。两者串起来，模型才既能看上下文，又能把读到的信息加工成更适合下一层使用的表示。

所以如果某一层 attention 已经对齐到“apple”，FFN 可能会进一步把它推向“水果”或“公司”这类更有任务意义的表示。它不是附属层，是表示分化里真正干活的一半。

一眼看顺序

模块主要作用错位症状

attention在 token 间搬运信息该看的前文没进来。

residual保留原始表示信息被覆盖得太狠。

FFN对单点表示做非线性加工表示没有变得更可分。

$$

h' = W_2 \, \sigma(W_1 h + b_1) + b_2

$$

h 是当前位置表示，W_1 和 W_2 是两层线性变换，\sigma 是非线性。这个式子先不要求你背推导，只要记住：FFN 会把当前位置的特征重新拉伸、压缩和分隔。

Hugging Face 路线

example

先拿可打印对象

这一页的最小实作路线不追求训练，而是追求观察：先打印 hidden states，看不同层的表示如何变化；再打印 attentions，看谁在读谁；最后把中间层表示投到词表，看它更像什么 token。

观察 hidden states / attentions复制

from transformers import AutoModelForCausalLM, AutoTokenizer

model_name = "Qwen/Qwen2.5-0.5B-Instruct"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForCausalLM.from_pretrained(model_name)

text = "The apple is green"
inputs = tokenizer(text, return_tensors="pt")
outputs = model(**inputs, output_hidden_states=True, output_attentions=True)

print(len(outputs.hidden_states))
print(outputs.hidden_states[-1].shape)
print(outputs.attentions[-1].shape)

这段代码怎么看

对象它负责什么先看哪里

tokenizer把文字变成 input_ids，再把 id 还原回 token。切词和长度。

model根据 input_ids 计算 hidden_states / attentions / logits。最后一层表示和形状。

lm_head把 hidden state 投到词表空间。logits 排序和 top token。

如果这一步打印出来的 hidden state 形状和你想的不同，先不要怀疑“模型坏了”，先确认 batch、seq 和层数。很多问题只是输入维度没有对齐。

logit lens 和 patch scope

lens

先读曲线，再读结论

logit lens 的核心动作很简单：拿某一层的 hidden state，直接接到 LM head 上，看这一层如果提前承担输出职责，它会更像哪些 token。这个动作的价值把中间表示变成一条可读的变化曲线。你可以看到层数越深，表示是怎样从一个比较散的词面状态，慢慢收敛成更明确的候选分布。

用 apple 做例子最顺手。浅层时，logit lens 常会看到一些很通用的 token：the、is、fruit、apple 本身，或者和上下文还没有完全区分开的词。到了更深层，同一个 token 在水果语境里会更偏向食物相关分布，在公司语境里会更偏向品牌或组织相关分布。这个变化说明 representation 在层间被逐步重写的对象。

这也解释了为什么只看最后输出不够。最后一层当然能告诉你模型选了什么，但 logit lens 能告诉你它从哪一层开始像这样想了。对学习来说，这比单点答案更有用，因为你可以顺着层数追到“什么时候开始分叉”。

patch scope 把观察变成测试

如果 logit lens 是“读”，patch scope 就是“测”。做法是拿另一条样本在某个层、某个位置的表示，替换当前样本对应的位置，再看最终 logits 会不会跟着变化。这样你就不只是在看模型像什么，而是在问：这个位置的表示，是不是后续决策真的依赖的证据。

例如，先准备一句水果语境里的 apple，再准备一句公司语境里的 Apple。如果把水果句子的某层表示 patch 到公司句子的对应位置，最后分布明显往水果词汇倾斜，说明那一层表示确实参与了后面的决策；如果几乎没变，就说明信息可能还没有被写进关键路径，或者真正起作用的是别的层、别的位置。

这个方法很适合排查误判。很多人看到某层 attention 很强，就直接说“模型在这里想到了答案”；但真正的因果测试要看 patch 之后输出变不变。可解释性不是看起来合理就够了，关键是局部改动能不能带来稳定的输出变化。

所以这两种方法应该一起看：logit lens 负责给你方向感，patch scope 负责给你证据感。前者告诉你表示正在往哪里走，后者告诉你这条路有没有真的被后层使用。

方法读什么写什么看见什么常见误判

logit lens某层 hidden state投到词表空间这一层当前更像哪些 token把中间投影当成最终答案

patch scope另一条样本的局部表示替换目标层 / 位置最终 logits 是否被推着改把一个位置的影响夸大成全局解释

layer compare不同层的 top-k 变化跟踪分布收敛representation 何时开始稳定只看一层，把动态过程看成静态结果

causal test被 patch 之前的路径观察后续输出变化哪些表示真的被后层用到了只看相似度，不看因果效应

logit lens 伪代码复制

def probe_layer(model, input_ids, layer_idx, top_k=5):
 outputs = model(
 input_ids=input_ids,
 output_hidden_states=True,
 output_attentions=False,
 )
 hidden = outputs.hidden_states[layer_idx]
 logits = model.lm_head(hidden)
 probs = logits.softmax(dim=-1)
 top = probs[:, -1, :].topk(top_k, dim=-1)
 return top.indices, top.values

这段伪代码做的事很朴素：选一层，取它的 hidden state，接上 LM head，看最后一个位置最像哪些 token。它不是最终推理流程，但足够让你看到中间表示向词表空间投影后的样子。

如果层间 top-k 变化很快，说明表示还在重组；如果到了后几层 top-k 已经稳定下来，说明模型离输出结论更近了。这个变化曲线比单点判断更有信息量。

实操时，先在两个上下文里各跑一次，再只改一个层位去 patch。你会看到：同一个 token 的表示在不同层里既有相似处，也有决定性的差异。可解释性真正有用的地方，就是把这些差异变成可验证的实验。

操作桥

bridge

下一步怎么动手

如果你已经能把这一讲讲清楚，下一步最值得打开的是一个能导出 hidden states 和 attentions 的最小模型脚本。先选一个小模型，打印一条短句的层间输出，再把最后一层和中间层都接到 LM head 上看 top-k 变化。这个练习会把“我大概懂了”变成“我真的看见了”。

操作顺序建议很固定：先只跑 tokenizer，确认切词和长度；再跑 model 的前向，拿到 hidden states 和 attentions；接着只对某一层做 logit lens；最后再尝试 patch scope。一次只改一个变量，你才能知道变化到底从哪儿来。

失败时先看什么

现象先查哪里为什么

hidden state 形状不对batch / seq / hidden_size输入维度没对齐，后面都不准。

attention 看起来全空是否真拿到了 output_attentions有时只是没有打开输出开关。

logit lens 没变化层号和最后一位是否取对错层或错位置会直接把曲线看平。

patch 以后输出不动patch 的位置是否和目标样本对齐错位比无效更常见。

一个完整的错位样本

假设你跑的是 The apple is green，却发现模型最后偏向的是“company”而不是“fruit”。先不要把这件事直接解释成“模型把 apple 认错了”，而是按层看：切词是否把 apple 拆散了，embedding 是否和你想的一样，前面几层的 representation 是否已经开始向水果或公司语境分叉，LM head 读出的 logits 是否真的把水果词汇压低了。

如果 logit lens 在中间层已经能看到水果词汇，但最终输出还是公司词汇，问题更可能在后面几层、残差路径或解码参数。如果中间层从头到尾都像公司语境，那就说明错误更早，可能是上下文、数据提示或 attention 没把语境拉进来。这个判断很重要，因为它决定你下一步该改 prompt、改层，还是改解码。

这就是这页真正想训练你的动作：问“错是在哪一步被写进去的”。只要你能把这一步说清楚，后面的可解释性方法、调试方法和实验方法就都接得上。

层位你会看到什么怎么解释下一步

tokenizerapple 没被拆散输入表示先没问题。继续看 embedding。

中间层logit lens 已经偏向 fruit表示在朝水果语境收敛。继续看更深层。

更深层top-k 又偏向 company后层把表示改写到别的方向。看 residual、FFN 和 lm_head。

最终输出采样结果选了 company解码和最终分数把前面的趋势放大了。调 temperature / top-p，再复跑一次。

这个回放的关键，训练你把一条输出轨迹拆成可追踪的局部事实。等你能把“水果语境先出现、公司语境后覆盖”这类变化说清楚，后面看可解释性论文时，你会知道作者到底是在读表示，还是在做因果测试。

最小实验记录

如果你准备真的跑一遍，只要记四件事：输入句子、取的是哪一层、logit lens 的 top-k 里先出现了什么、最终输出是什么。再补一句话说明中间层和最终层是否朝同一个方向收敛。这个记录足够让你复盘一次实验，而不用回头翻整段输出日志。

字段写什么为什么

输入句子例如 The apple is green固定锚点，方便比较。

观察层layer 3 / layer 8 / layer 12知道变化从哪里开始。

写完这四项，你就能把一次实验从“跑过了”变成“看懂了”。后面无论是换层、换样本还是换解码参数，都可以继续沿着这张记录表做对照。
