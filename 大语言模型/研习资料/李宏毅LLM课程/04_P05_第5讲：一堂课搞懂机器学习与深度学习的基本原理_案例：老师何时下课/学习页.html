机器学习：从资料中找出函数 F

前几节一直把语言模型写成函数 F：输入半句话或上下文，输出下一个 token 的分布。P5 开始回答更根本的问题：这个 F 是怎么被找出来的？答案就是机器学习：用资料定义目标，在候选函数集合里找一个表现最好的函数。

课程不用语言模型做例子，而用一个很好懂的 regression 任务：输入一份投影片，预测李宏毅老师会讲多久。这个例子会带你看清 loss、feature、parameter、gradient descent、validation、overfitting 和 deep learning 的基本关系。

定义 loss→定义模型范围→优化参数→验证泛化

  1. 用 3+1 步解释机器学习训练流程。
  2. 区分 loss、evaluation metric、training data、validation data、test data。
  3. 解释 gradient descent、learning rate、batch size、epoch 和 shuffle。
  4. 判断 validation loss 高时，该从资料、模型范围、优化或过拟合哪里排查。

概念骨架

SKELETON

机器学习不是魔法

它做的是从候选函数里找一个 loss 小的函数。语言模型只是函数特别大、资料特别多、输出特别复杂。你现在应该能做什么：把“训练 AI”翻译成“找参数”。

深度学习是更大的函数集合

线性模型只能画直线，神经网络能用 ReLU 拼出复杂曲线。它不是脱离机器学习的另一个世界，而是把候选函数范围扩大。你现在应该能做什么：解释为什么模型更大不保证验证更好。

验证集是模拟考

训练 loss 低只说明它会做见过的题。validation 低才说明它可能泛化。test 是最后大考，不能拿来反复调参。你现在应该能做什么：区分 training、validation、test 的角色。

术语表：从投影片到函数

TERMS

这是一个 regression 任务

输入 X 是一份投影片，输出 Y 是这堂课会讲多久，单位可以是分钟。输出是一个数字，这类任务在机器学习里叫 regression，也就是回归。

这个任务故意和语言模型无关，是为了让你看到：机器学习的本质不是“文字接龙”，而是“用资料找函数”。同样的 3+1 步以后会用在语言模型训练上，只是输入输出和函数集合更复杂。

投影片不能直接喂给普通函数，要先变成 feature

函数能处理的是数字。你可以把投影片变成一串 feature，例如页数、总字数、标题长度、是否出现 learning、每页平均字数。feature 是你给模型看的线索。

feature 不是随便越多越好。好的 feature 来自 domain knowledge，也就是你对任务的理解。比如总字数和页数高度相关，可能不如“每页平均字数”有信息量。

对象本节含义例子
X模型输入一份投影片
Y模型输出上课时长
feature把输入转成数字线索页数、字数、平均字数
parameter训练要找的未知数w、b、神经网络权重
model候选函数集合所有 y=wx+b 的直线
ŷ = f(x) = w · x + bŷ 是预测时长，x 可以先只用投影片页数。w 表示每页大约讲多久，b 表示固定开场或额外时间。

讲义正文：机器学习训练的 3+1

LECTURE

第一步：先定义什么叫好

机器学习不是一开始就知道答案，而是先规定“给我一个函数 F，我要怎么判断它好不好”。上一讲评估模型时，我们把模型输出和标准答案比较，算出一个分数；训练里的第一步也类似，只是通常把分数写成越小越好的 loss。

在预测上课时长的例子里，训练资料包含过去课程的投影片和真实时长。把投影片丢进某个函数，得到预测时长，再和真实时长比较。每一堂课都算一次误差，最后平均起来，就得到这个函数在训练资料上的 loss。

L = (1/N) Σᵢ (ŷᵢ - yᵢ)²这里使用 mean square error。ŷᵢ 是预测时长，yᵢ 是真实时长，平方会避免正负误差互相抵消，也会更重罚大错。

training data 不只是拿来“喂模型”的材料，它还定义了 loss 的形状。若训练资料来自“机器学习 2021”，而目标是预测“生成式 AI 导论”，课程类型、投影片密度和讲课节奏都可能不同，训练出来的函数就容易偏离真正任务。

第二步：划出候选函数集合

投影片本身不能直接进入普通数学函数，所以要先变成 feature。最小做法是只看页数 x₁，然后假设时长和页数大致成正比：y=w₁x₁+b。这不是一个具体函数，而是一整个函数集合；每一组不同的 w₁,b 都对应一条不同直线。

这个集合就是 model 在本节里的主要含义。人类的 domain knowledge 会在这里进入训练过程：你认为页数、总字数、平均字数、是否出现 learning 这些线索有用，就可以把它们设计进函数范围。范围太小会表达不了真实关系，范围太大则更容易记住训练资料里的偶然细节。

候选集合 = { f(x)=w₁x₁+b | w₁,b 可变 }如果再加入总字数 x₂，就可以扩成 f(x)=w₁x₁+w₂x₂+b。参数越多，候选范围越大，也越需要验证。
模型选择能表达什么主要风险
只用页数时长随页数线性增加忽略内容密度
加总字数页数和文字量共同影响时长feature 相关性高
加平均字数每页内容浓度影响讲解时间更复杂，更需验证
神经网络用 ReLU 拼出非线性关系优化困难，可能过拟合

第三步：在候选集合里找 loss 最低的函数

有了 loss，也有了候选函数集合，训练就变成 optimization：找一组参数 w*、b*，让 loss 尽量小。两个参数的小例子可以暴力试很多组合，算出每组参数的 loss，再找最低点。课程里的例子会得到类似“每页约 1.67 分钟，固定额外时间约 4.85 分钟”的可解释参数。

真实模型可能有海量参数，不能暴力穷举。更通用的方法是 gradient descent。直觉上，你站在 loss surface 的某个位置，gradient 告诉你哪边更陡、往哪边走 loss 会下降；然后用 learning rate 控制步子大小，反复更新参数。

θ ← θ - η · ∇L(θ)θ 是所有参数组成的大向量，η 是 learning rate，∇L 是 loss 对每个参数的偏微分组成的 gradient。

learning rate 太大，可能跨过谷底或震荡；太小，loss 下降很慢。gradient descent 也可能停在 local minimum、saddle point，或者因为 gradient 很小让你误以为已经没有进展。神经网络训练时，PyTorch、TensorFlow 这类框架会自动计算 gradient，但你仍然要知道它在优化什么。

batch、epoch、shuffle 是训练节奏

完整 loss 要扫过所有训练资料。资料很大时，每次更新都看完所有资料太慢，所以会把训练资料切成 batch，每次用一个 batch 估计 loss 和 gradient，然后更新参数。一个 batch 里的资料越少，更新方向越抖；资料越多，方向越稳定，但单次计算更重。

所有训练资料都被看过一次，叫一个 epoch。若有 1000 笔资料，batch size 是 100，一个 epoch 会更新 10 次参数。每个 epoch 重新 shuffle，可以避免同样的样本永远组成同一个 batch，让每批资料更接近整体分布。

learning rate、batch size、epoch 数都属于 hyperparameter，也就是训练前由人设定的控制项。它们不是模型从资料里学出来的 parameter，却会强烈影响训练能否顺利下降。

术语一句话直觉常见错法
batch一次用来估计 loss 的一小包资料。以为 batch 越小一定越好。
epoch所有训练资料都被看过一次。以为 epoch 越多越好。
SGDbatch size 极小时的随机梯度下降。忽略更新方向会很抖。
shuffle每轮重新打乱样本。固定 batch 组合导致训练偏。
hyperparameter训练前人设的数值。把它和 parameter 混淆。

加一：validation 是训练后的模拟考

走完前三步，只能说明你在 training data 上找到了一个 loss 低的函数。它能不能用于新课程，要拿 validation set 检查。课程例子里，用机器学习 2021 训练，training loss 约 240；拿来验证生成式 AI 课程时，validation loss 升到 1000 多。这不是简单的“模型烂”，而是提示训练资料定义的目标和实际任务不匹配。

换成生成式 AI 导论 2024 当训练资料后，training loss 约 71，validation loss 约 122，明显更接近。validation 不好时，要回头看前三步：资料是否对、函数范围是否足够、optimization 是否真的找到低 loss、模型是否过度迎合训练资料。

deep learning 是更大的函数范围

如果真实关系不是直线，线性模型范围就太小。课程给的直觉是：复杂曲线可以用很多小线段逼近，而小线段可以用很多个 max(0, wx+b) 这样的 ReLU 山坡函数拼出来。

y = b + Σᵢ cᵢ · max(0, wᵢx + bᵢ)这是一层神经网络的雏形。输入乘权重、加 bias、过 activation，再组合输出。

多个 feature 组成向量，权重组成矩阵，bias 组成向量。输入乘矩阵、加 bias、过 ReLU，得到 hidden layer；重复多层，就成为 deep learning。神经网络范围更大，所以有机会表达复杂函数，但也更难优化、更容易过拟合。

核心直觉

机器学习就是在一大群候选函数里找一个 loss 小的。feature 决定函数能看见什么，model range 决定候选函数有多丰富，optimization 决定能不能找到好参数，validation 决定这个好是不是只对训练资料成立。

易混点

parameter 是训练会改的数，例如 w,b；hyperparameter 是训练前由人设定的数，例如 learning rate；loss 用来训练；metric 用来评价；validation 用来调参，test 用来最后验收。

Worked example：手算一次 MSE

两节课真实时长分别是 100 和 80 分钟，模型预测 90 和 120。误差是 -10 和 40，平方后是 100 和 1600，MSE 是 850。第二个大错主导了 loss，所以优化会优先把那一类大错压下来。

最小例子：把 3+1 步翻成代码

EXAMPLE
正确示例：定义 MSE lossloss
def mean_square_error(y_pred, y_true):
    diff = y_pred - y_true
    return (diff ** 2).mean()

这对应步骤一：先定义什么叫好。随机预测会得到很大的 loss，完美预测 loss 为 0。

正确示例：线性模型model range
def linear_model(x1, w1, b):
    return w1 * x1 + b

这对应步骤二：所有不同的 w1 和 b 组成候选函数集合。暴力穷举时，就是把很多组 w1,b 都拿来算 loss。

正确示例：手写 gradient descent 轮廓optimization
w1, b = 1.0, 15.0
lr = 0.0001
epochs = 100

for epoch in range(epochs):
    y_pred = linear_model(x_train, w1, b)
    grad_w = (2 * (y_pred - y_train) * x_train).mean()
    grad_b = (2 * (y_pred - y_train)).mean()
    w1 = w1 - lr * grad_w
    b = b - lr * grad_b

这对应步骤三：计算 gradient，按 learning rate 更新参数。真实深度学习会让框架自动算 gradient，但你要知道它在帮你做什么。

实作自检

  • 先跑简单线性模型，得到一个 baseline。
  • 神经网络 training loss 如果比线性还高，优先怀疑优化失败。
  • validation loss 比 training 高很多，优先怀疑过拟合或资料分布不一致。
  • 调 learning rate、batch size、epoch 时，保留验证记录。

错误诊断：从 loss 数字回推问题

DIAGNOSIS

看 train / validation loss

诊断建议

训练资料和验证任务分布不一致时，训练看似可行，验证会明显变差。

常见问题和修法

现象优先怀疑修法
training 低，validation 高过拟合或资料分布不匹配换训练资料、降低模型复杂度、加验证监控
training 高，validation 也高模型范围太小或优化没做好加 feature、换模型、调 learning rate
神经网络比线性 training loss 高optimization failure先修训练设置，而不是盲目加大模型
validation 越调越好，最终 test 差过度迎合 validation保留 private test，减少反复刷验证

核心判断

训练低、验证高：优先怀疑过拟合或数据分布不匹配。训练高、验证也高：可能模型范围太小或优化失败。训练低、验证低、测试高：可能过度迎合验证集或测试分布变了。

数据集用途不能做什么
training定义 loss，更新参数。用它证明泛化能力。
validation调资料、模型范围、超参数、early stopping。无限次调到完美后当最终结论。
public test有限次外部反馈。当成无限验证集刷榜。
private test最终大考。看完后再改模型。
