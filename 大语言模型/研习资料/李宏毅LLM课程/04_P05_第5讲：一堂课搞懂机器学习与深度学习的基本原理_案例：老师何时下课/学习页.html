术语表：从投影片到函数

TERMS

这是一个 regression 任务

输入 X 是一份投影片，输出 Y 是这堂课会讲多久，单位可以是分钟。输出是一个数字，这类任务在机器学习里叫 regression，也就是回归。

这个任务故意和语言模型无关，是为了让你看到：机器学习的本质“用资料找函数”。同样的 3+1 步以后会用在语言模型训练上，只是输入输出和函数集合更复杂。

投影片不能直接喂给普通函数，要先变成 feature

函数能处理的是数字。你可以把投影片变成一串 feature，例如页数、总字数、标题长度、是否出现 learning、每页平均字数。feature 是你给模型看的线索。

feature 不是随便越多越好。好的 feature 来自 domain knowledge，也就是你对任务的理解。比如总字数和页数高度相关，可能不如“每页平均字数”有信息量。

X模型输入一份投影片

Y模型输出上课时长

feature把输入转成数字线索页数、字数、平均字数

parameter训练要找的未知数w、b、神经网络权重

model候选函数集合所有 y=wx+b 的直线

讲义正文：机器学习训练的 3+1

LECTURE

第一步：先定义什么叫好

机器学习先规定“给我一个函数 F，我要怎么判断它好不好”。上一讲评估模型时，我们把模型输出和标准答案比较，算出一个分数；训练里的第一步也类似，只是通常把分数写成越小越好的 loss。

在预测上课时长的例子里，训练资料包含过去课程的投影片和真实时长。把投影片丢进某个函数，得到预测时长，再和真实时长比较。每一堂课都算一次误差，最后平均起来，就得到这个函数在训练资料上的 loss。

L = (1/N) Σᵢ (ŷᵢ - yᵢ)²这里使用 mean square error。ŷᵢ 是预测时长，yᵢ 是真实时长，平方会避免正负误差互相抵消，也会更重罚大错。

training data 不只是拿来“喂模型”的材料，它还定义了 loss 的形状。若训练资料来自“机器学习 2021”，而目标是预测“生成式 AI 导论”，课程类型、投影片密度和讲课节奏都可能不同，训练出来的函数就容易偏离真正任务。

模型选择能表达什么主要风险

第三步：在候选集合里找 loss 最低的函数

有了 loss，也有了候选函数集合，训练就变成 optimization：找一组参数 w、b，让 loss 尽量小。两个参数的小例子可以暴力试很多组合，算出每组参数的 loss，再找最低点。课程里的例子会得到类似“每页约 1.67 分钟，固定额外时间约 4.85 分钟”的可解释参数。

真实模型可能有海量参数，不能暴力穷举。更通用的方法是 gradient descent。直觉上，你站在 loss surface 的某个位置，gradient 告诉你哪边更陡、往哪边走 loss 会下降；然后用 learning rate 控制步子大小，反复更新参数。

θ ← θ - η · ∇L(θ)θ 是所有参数组成的大向量，η 是 learning rate，∇L 是 loss 对每个参数的偏微分组成的 gradient。

learning rate 太大，可能跨过谷底或震荡；太小，loss 下降很慢。gradient descent 也可能停在 local minimum、saddle point，或者因为 gradient 很小让你误以为已经没有进展。神经网络训练时，PyTorch、TensorFlow 这类框架会自动计算 gradient，但你仍然要知道它在优化什么。

术语一句话直觉常见错法

加一：validation 是训练后的模拟考

走完前三步，只能说明你在 training data 上找到了一个 loss 低的函数。它能不能用于新课程，要拿 validation set 检查。课程例子里，用机器学习 2021 训练，training loss 约 240；拿来验证生成式 AI 课程时，validation loss 升到 1000 多。这提示训练资料定义的目标和实际任务不匹配。

换成生成式 AI 导论 2024 当训练资料后，training loss 约 71，validation loss 约 122，明显更接近。validation 不好时，要回头看前三步：资料是否对、函数范围是否足够、optimization 是否真的找到低 loss、模型是否过度迎合训练资料。

最小例子：把 3+1 步翻成代码

EXAMPLE

正确示例：定义 MSE lossloss复制

`def mean_square_error(y_pred, y_true):

diff = y_pred - y_true

return (diff ** 2).mean()`

这对应步骤一：先定义什么叫好。随机预测会得到很大的 loss，完美预测 loss 为 0。

正确示例：线性模型model range复制

`def linear_model(x1, w1, b):

return w1 * x1 + b`

这对应步骤二：所有不同的 w1 和 b 组成候选函数集合。暴力穷举时，就是把很多组 w1,b 都拿来算 loss。

for epoch in range(epochs):

y_pred = linear_model(x_train, w1, b)

grad_w = (2 (y_pred - y_train) x_train).mean()

grad_b = (2 * (y_pred - y_train)).mean()

w1 = w1 - lr * grad_w

b = b - lr * grad_b`

这对应步骤三：计算 gradient，按 learning rate 更新参数。真实深度学习会让框架自动算 gradient，但你要知道它在帮你做什么。

实作自检先跑简单线性模型，得到一个 baseline。神经网络 training loss 如果比线性还高，优先怀疑优化失败。validation loss 比 training 高很多，优先怀疑过拟合或资料分布不一致。调 learning rate、batch size、epoch 时，保留验证记录。

错误诊断：从 loss 数字回推问题

DIAGNOSIS

看 train / validation loss

诊断建议训练资料和验证任务分布不一致时，训练看似可行，验证会明显变差。

现象优先怀疑修法

核心判断训练低、验证高：优先怀疑过拟合或数据分布不匹配。训练高、验证也高：可能模型范围太小或优化失败。训练低、验证低、测试高：可能过度迎合验证集或测试分布变了。

training定义 loss，更新参数。用它证明泛化能力。

validation调资料、模型范围、超参数、early stopping。无限次调到完美后当最终结论。

public test有限次外部反馈。当成无限验证集刷榜。

private test最终大考。看完后再改模型。
