# 总诊断框架：先分清是哪种失败 {#skeleton}
<!-- tag: MAP -->
<!-- layout: grid2 -->

这一讲最重要的直觉是：模型效果不好，不等于“过拟合”。如果 deep network 的 training loss 连 linear model 都比不过，它连训练资料都没拟合好，这叫 optimization 问题。此时该想的是 optimizer、learning rate、初始化、残差连接、normalization 等让训练更容易的办法。
反过来，如果 training loss 已经很低，但 validation loss 仍然很高，这才是典型的 overfitting。此时继续换 optimizer 可能只会让 training loss 更低，validation 不一定变好。你要考虑 dropout、更多代表性资料、data augmentation、CNN 这类缩小函数范围或改善泛化的办法。
因此，技巧的分类“病因 / 药效”匹配。训练神经网络像调一个复杂系统，第一步问：我现在想修的是 training loss，还是 validation gap？

## box soft
### 判断句
**training loss 高：**模型在训练集上也做不好，优先怀疑 optimization 或模型范围太小。
**training loss 低、validation loss 高：**模型记住训练集但泛化差，优先处理 overfitting。
**validation 本身不可信：**先检查资料切分、测试分布、指标定义，不要急着换模型。

改 Step 1：loss
换 cross entropy、加入额外资料、加入 unlabeled data、加入 L2 regularization，本质上都在改你要最小化的目标。


改 Step 2：函数范围
CNN、residual connection、normalization、dropout 影响模型能选择哪些函数，或让这个范围更适合训练。


改 Step 3：optimization
Gradient descent、AdaGrad、RMSProp、momentum、Adam、learning rate schedule 都在改找参数的路径。

# 术语表：先把名字放准位置 {#terms}
<!-- tag: TERMS -->

这一讲的术语很多，但它们不是平级清单。读术语时先问：它在三步骤里改了哪里？主要是在降低 training loss，还是在缩小 training / validation 的落差？下面这张表就是后面讲义的查阅入口。

| 术语 | 直觉解释 | 精确定义 | 容易误解的地方 |
| --- | --- | --- | --- |
| Optimization | 让模型在训练资料上真的学得动。 | 在 loss 和模型范围固定后，用参数更新规则寻找较小的 training loss。 | training loss 高时不要先喊 overfitting；那通常是训练都没训练好。 |
| Generalization | 训练集学会后，换到没见过的资料仍然做得好。 | 模型在 validation / test distribution 上维持低 loss 或高 metric 的能力。 | training loss 更低不必然代表 validation 更好。 |
| AdaGrad | 常见大梯度的方向少走一点，常见小梯度的方向多走一点。 | 按每个参数过去梯度平方累积量缩放有效 learning rate。 | 它会记住全部历史，旧梯度可能让后期步长过小。 |
| RMSProp | 只让近期梯度更有发言权。 | 用指数移动平均估计梯度平方尺度，再缩放更新量。 | 它反应更快，但 learning rate 太大时仍会震荡。 |
| Momentum | 方向不要只看当前一步，也保留过去的惯性。 | 把过去梯度方向加权平均成动量项，用它更新参数。 | 动量不是保证找到全局最低点，只是让训练不被局部小坡度轻易卡住。 |
| Adam / AdamW | Adam 同时看方向历史和尺度历史；AdamW 再把权重衰减处理得更干净。 | Adam 结合一阶矩和二阶矩估计；AdamW 使用 decoupled weight decay。 | Adam 常改善 optimization，AdamW 的 weight decay 更偏向帮助 generalization。 |
| Dropout | 训练时随机遮掉部分神经元，让模型不要太依赖个别路径。 | 训练阶段随机置零部分 activation，评估阶段使用完整网络并做尺度处理。 | 它会让训练更难；training loss 已经下不去时不该优先加。 |
| CNN | 把影像的局部性和平移共享写进模型。 | 通过 receptive field 和 parameter sharing 限制连接方式与参数数量。 | CNN 是缩小函数范围，不是“层数更酷”。任务没有局部共享结构时要谨慎。 |
| Normalization | 让中间数值尺度更稳，训练路径更顺。 | 对输入或隐藏层输出做均值、方差或范数尺度调整。 | 它也可能影响泛化，但主线是让 deep network 更容易优化。 |
| Cross entropy | 分类时给梯度一个能走的路。 | 比较 one-hot 标签分布和 softmax 概率分布的可微 loss。 | Accuracy 是评估指标，不适合直接拿来做梯度下降目标。 |

# 完整讲义：把技巧放回三步骤 {#lecture}
<!-- tag: LECTURE -->

字幕从一个很实用的问题开始：听到一种新的训练技巧时，怎么判断它该不该用？答案回到上一讲的三步骤。先定义 loss，再定义模型范围，最后用 optimization 找参数；训练完再用 validation 判断是否真的泛化。下面每个技巧都沿着这条线读。

Optimizer：先理解 vanilla gradient descent
Optimization 的任务是：loss 和模型范围都定好后，在参数空间里找一组参数 θ，让大写 L 尽量小。最朴素的方法是 vanilla gradient descent，也就是“没有加特殊口味”的梯度下降。它每一步只看当前位置的 gradient，然后沿着让 loss 下降的方向更新参数。

 θt+1 = θt - η · gt，其中 gt = ∇θL(θt)
 θ 是参数，η 是 learning rate，g 是当前位置 loss 对参数的梯度。


这个式子看起来很简单，但麻烦正藏在 η 里。learning rate 太大，参数可能第一步就飞到山谷对岸，数值爆掉；learning rate 太小，走一百步还像原地踏步。更麻烦的是，不同参数方向的地形可能完全不同：某个方向坡很陡，需要小步；另一个方向坡很平，需要大步。
vanilla gradient descent 给所有参数同一个 learning rate，这就像让所有人在不同路况上用同一个步长跑步。Optimizer 的一大类思想，就是不要只看“此刻的梯度”，而要利用过去的梯度记录，估计每个方向应该走多大、该不该保留动量。

learning rate0.045

学习率不是越大越好，也不是越小越稳；它要和地形尺度匹配。


伪代码：optimizer 的位置training loop
`for batch in train_loader:
 pred = model(batch.x)
 loss = loss_fn(pred, batch.y)
 loss.backward() # 算出每个参数的 gradient
 optimizer.step() # 用某种规则更新参数
 optimizer.zero_grad() # 清掉旧梯度，准备下一轮`
在 PyTorch 里，换 optimizer 往往只是换一个英文单字；但这不代表它只是“玄学参数”。你需要知道这个单字背后的假设，才知道什么时候该换、换了以后该看什么曲线。

AdaGrad / RMSProp：给不同方向不同步长


AdaGrad 的直觉
AdaGrad 观察过去每个维度的 gradient。如果某个维度过去经常出现很大的 gradient，说明这个方向地形陡，learning rate 应该变小；如果某个维度 gradient 通常很小，说明这个方向地形平，learning rate 可以相对变大。
它的做法是累积过去梯度平方，再用这个累计量去除 learning rate。平方会去掉正负号，所以 AdaGrad 关心的是 magnitude，也就是“这个方向通常有多陡”，而不是“该往正方向还是负方向走”。


 σt,i = √(g0,i2 + g1,i2 + ... + gt,i2)

 θt+1,i = θt,i - η · gt,i / σt,i
i 表示第 i 个参数维度。σ 越大，该维度实际步长越小。


AdaGrad 的强假设是：过去梯度大小能预测未来梯度大小。但地形可能像香蕉一样弯曲，同一个方向前面很陡，后面又变平。AdaGrad 把历史全部累积起来，旧梯度和新梯度权重差不多，于是 learning rate 调整不够及时。


RMSProp 的改法
RMSProp 不再把所有历史梯度平方一视同仁，而是给最近的梯度更大影响力。它用一个 α 控制旧资讯和新资讯的比例，让 σ 能跟着地形变化更快调整。

 σt2 = α · σt-12 + (1-α) · gt2
α 越大，历史记忆越长；α 越小，越重视最近梯度。


要注意的代价
RMSProp 反应更快，但也可能更不稳定。字幕里的 banana-shaped surface 例子显示，RMSProp 可以比 AdaGrad 走得更接近最低点，但 learning rate 设太大时也更容易震荡甚至乱飞。


方法记住什么历史主要目标常见误用

AdaGrad所有过去梯度平方的累积让每个参数有不同有效学习率以为它能自动解决所有 learning rate 问题
RMSProp带衰减的梯度平方平均让学习率尺度更及时适应地形把 learning rate 开太大，导致震荡

Momentum / Adam：方向也要看历史
AdaGrad 和 RMSProp 主要处理“每个方向应该走多大步”。Momentum 处理的是另一个问题：如果当前位置 gradient 很小，训练不一定该停。真实物体从山坡滚下来有动量，不会到平地就瞬间停住；参数更新也可以保留过去方向的惯性。

 mt = β · mt-1 + (1-β) · gt

 θt+1 = θt - η · mt
m 是动量项，会考虑梯度的正负方向；β 控制历史方向保留多少。


Momentum 会把过去梯度按方向加起来，所以它可能帮助你穿过 saddle point，甚至翻过浅的 local minimum。它不是保证找到 global minimum 的魔法，但它让“当前位置梯度很小”不再等于“马上停下”。
Adam 可以看成把 RMSProp 和 momentum 合在一起：RMSProp 提供梯度平方尺度 σ，momentum 提供方向 m。更新时，用 σ 调整步长，用 m 取代当前梯度方向。真实 Adam 还包含 bias correction 等细节，这一讲为了看懂主线，先抓住“尺度 + 方向”这个核心。


Adam 的一句话
Adam = 用 RMSProp 类似的方式估计每个方向该走多大，再用 momentum 类似的方式估计整体该往哪里走。
它常被当作 default optimizer，但 default 不等于永远正确。它主要强化 optimization，不保证 validation 一定更好。


 简化 Adam：θt+1 = θt - ηt · mt / σt
分母 σ 只看大小；分子 m 保留方向；ηt 可以随时间变化。

Learning rate schedule：η 也可以随时间变
很多训练不会把 η 固定不变，而是使用 schedule。常见策略是先 warm up，再 decay。warm up 让学习率从小变大，给 optimizer 一点探索地形、收集 m 和 σ 估计的机会；decay 则让学习率逐步变小，避免在谷底两侧一直震荡，让参数慢慢着陆。

什么时候换 optimizer？
当 training loss 压不下去，而你相信模型范围应该足够大时，换 optimizer 或 schedule 才像对症下药。
如果问题是 validation gap，换 Adam 可能让训练集更漂亮，泛化却更糟。

Dropout、初始化与 Pretrain：不只优化路径


Dropout
Dropout 在训练时随机丢掉一些神经元，validation / test 时再全员上阵。它给训练增加难度，迫使模型不要太依赖某几个神经元。
**主要帮助：**generalization。加 dropout 后 training loss 往往会更高，因为训练真的变难了。training loss 本来就压不下去时，不该先加 dropout。


Initialization
初始化决定参数从哪一个位置开始下坡。不同起点可能走向不同 basin；即使 optimizer 很强，好的初始化也可能让训练更稳定、更快。
字幕中的例子显示，只改初始化尺度，例如参考输入维度调整随机权重，loss 就可能从七八降到一甚至更低。


Pretrain
Pretrain 是先让模型学一个相关任务，再把学到的参数当成目标任务初始化。它可被看成更好的 initialization。
**神奇点：**它通常同时帮助 optimization 和 generalization，因为起点已经带有有用结构。


初始化还有一个更深的直觉：即使存在很多 training loss 都为零的 global minimum，它们的泛化能力也可能不同。平坦 basin 对训练集和验证集的小差异更不敏感；尖锐谷底只要 validation surface 稍微平移，loss 就可能大幅变坏。因此“找到最低点”还不够，找到什么样的低点也重要。
示例：技巧选择不要混淆病因decision snippet
`if train_loss_is_high:
 try_optimizer_or_lr_schedule()
 check_initialization()
 inspect_architecture_for_trainability()
elif train_loss_is_low_and_val_loss_is_high:
 try_dropout_or_weight_decay()
 improve_data_or_augmentation()
 inspect_validation_split()
else:
 verify_metric_and_test_distribution()`

CNN：用 domain knowledge 缩小函数范围
第二步是定义可选函数集合，也就是模型架构。一个函数范围太小，可能根本不包含好函数；范围太大，又容易 overfitting。好的架构借助 domain knowledge 划出一个“刚好包含好函数，又排除大量坏函数”的范围。


CNN 是这个思想的经典例子。图片输入对神经网络来说是巨大的向量：一张 1000×1000 的 RGB 图片有 300 万个数。如果第一层 fully connected，每个神经元都看整张图，参数量会极其夸张。但人类知道，早期视觉 detector 通常只需要看局部 pattern，例如鸟嘴、眼睛、爪子。
于是 CNN 加入 receptive field：每个神经元只看图片的一小块。它还加入 parameter sharing：同一个 pattern 可能出现在左上角，也可能出现在中间，用同一组参数去识别即可。这两件事都强制模型少选一些无用函数。


CNN 不是万能结构
CNN 对图片有效，是因为图片有局部性和平移共享的特性。语音也常能用，是因为有相似的局部模式。不是所有任务都适合 CNN；架构来自任务特性，不来自术语流行度。


CNN 设计直觉它缩小了什么主要帮助

Receptive field每个神经元只守备局部区域局部外的连接被强制设为零减少参数，改善泛化
Kernel size局部窗口大小，例如 3×3基础 pattern 的观察尺度让局部检测可控
Stride / padding窗口移动步幅与边界补零控制覆盖密度和输出尺寸保持空间结构
Parameter sharing不同位置共用同一组 filter同类 detector 不重复学习减少过拟合


从函数集合角度看，CNN 是 fully connected layer 的限制版。只要 fully connected layer 的部分参数恰好为零、部分参数恰好共享，它也能模拟 CNN；但直接训练那么大的范围很容易乱选。CNN 把人类对影像的理解写进架构，使搜索范围更小、更对题。

Residual 与 Normalization：架构也会影响 optimization


为什么深层网络难训练
如果参数靠近输出层，稍微改一下，输出和 loss 可能立刻变化，gradient 较容易算出有效值。但如果参数靠近输入层，中间经过很多层，微小变化可能传到最后几乎消失，这叫 gradient vanishing；也可能在某些路径上突然放大，这叫 gradient exploding。
真正难点不是“gradient 小”本身，因为学习率调大可以部分处理。难点是 gradient 有时很小、有时很大，使得 learning rate 设大设小都不合适。


 普通层：hl+1 = Wl hl

 残差层：hl+1 = Wl hl + hl
skip connection 像给信息和梯度开了一条高速路。


Residual connection，也就是 skip connection，会把输入直接加到输出上。这样低层参数的变化更容易一路影响最终输出，训练深层网络更不容易被中间层吞掉。它改的是架构，也就是函数范围；但它带来的主要好处是 optimization，让 loss surface 更容易走。


Normalization
Normalization 把每一层输出限制在某个范围内，例如均值接近 0、方差接近 1。输入资料的标准化可以看成最外层的 normalization；BatchNorm、LayerNorm 则把这个思想放到网络内部。


它为什么有用
每层数值尺度比较稳定时，learning rate 更容易调，optimization 更顺。由于它也给函数范围加了限制，所以有时也会稍微帮助 generalization，但主目标通常还是让训练更容易。

分类 loss：为什么不能直接优化 accuracy
上一讲用 regression 例子，loss 和 evaluation metric 可以都用 MSE。但分类问题不同。分类真正关心的是 accuracy：预测类别对不对。问题是 accuracy 对参数的小变化通常没有反应，只要最高分的类别没变，accuracy 就不变，gradient 几乎处处为零。


如果把 accuracy 当 loss，参数小改 0.01，模型分数可能稍微变了，但预测类别没变，所以 loss 没变，gradient 算出来是 0。梯度下降就不知道往哪里走。你真正需要的是一个可微、平滑、又和分类目标相关的训练目标。
Cross entropy 就承担这个角色。模型先输出每个类别的 score，经 softmax 变成概率分布；正确答案也写成 one-hot 分布。两个分布越接近，cross entropy 越小。


 softmax(z)i = exp(zi) / Σj exp(zj)


 CE(y, p) = - Σi yi log pi
y 是正确答案分布，p 是模型输出分布。


对象用途能否直接做梯度下降风险

Accuracyvalidation / test 时真正关心的指标通常不适合几乎处处无梯度
Cross entropy训练分类模型时常用 loss适合训练 loss 和测试指标不完全一致
Hinge loss 等其他 loss可定义不同训练偏好视定义而定需要理解其假设和目标


Cross entropy 改的是 Step 1：loss definition。它的主要好处是让 optimization 可以进行；它不天然改善 generalization，甚至因为训练目标和测试指标不同，会引入额外 gap。所以分类训练时要同时看 cross entropy 和 accuracy，不要只盯单一数字。

资料也是 loss 的一部分
Loss 通常是对训练资料求和或平均。你改变训练资料，其实也改变了 loss 的定义。Overfitting 的一个直接原因是：training data 不能代表 validation / test 的真实分布。最粗暴也常有效的办法是收集更多有代表性的资料。


更多资料
更多代表性资料通常改善 generalization，因为训练 loss 更接近真实世界 loss。但它不会让 optimization 更容易，反而可能更难：你要同时满足更多样本。


资料必须代表测试分布
如果训练资料本身和测试资料有系统偏差，一直收集同样偏的资料仍然缘木求鱼。资料量不是万能，分布更关键。


Data augmentation：不能收集，就合理制造
Data augmentation 是用规则制造额外训练样本。例如图片分类中，鸟左右翻转仍然是鸟，模糊后也仍然是鸟；语音辨识中，可以调音量、拉伸长度，甚至用语音转换增加变化。它的核心变换后 label 仍然合理。
如果任务是判断鸟头朝左还是朝右，左右翻转就不能保留原 label；如果任务是辨认说话人，把女声转成男声后仍标成原说话人，也可能制造错误资料。augmentation 可不可用，取决于你对任务的理解。


Mixup
Mixup 不只改输入，也改 label。把猫图和狗图按 0.5 / 0.5 混合，label 也变成 50% 猫、50% 狗。模型学到的是在输入之间平滑过渡，而不是只记住训练点。


适用时机
Data augmentation 主要帮助 generalization。training loss 压不下去时，加更多变形资料通常不会救 optimization，只会让训练目标更复杂。

Unlabeled data、L2 与 AdamW：把偏好写进 loss
Semi-supervised learning
很多时候，标注资料贵，无标注资料便宜。Semi-supervised learning 会把 unlabeled data 也放进 loss。因为没有正确答案，你必须加入假设。例如“世界非黑即白”：模型对无标注样本的输出分布应该尖锐，entropy 越小越好；或者“物以类聚”：相近样本应该输出相近分布。

 Ltotal = Llabeled + λ · Lunlabeled
λ 控制无标注资料项的影响力。它通常让 generalization 更好，但 optimization 更复杂。


Entropy 假设
如果没有 label，至少希望模型很确定地把样本分到某一边。输出分布越平均，entropy 越大；越尖锐，entropy 越小。


Graph 假设
把相近样本连成图，相连样本的输出分布应该接近。难点是“相近”怎么定义，这仍然依赖任务知识。


L2 regularization 与 weight decay
除了资料项，loss 也可以写入人类对参数的偏好。L2 regularization 偏好参数接近 0，背后的直觉是更平滑、更简单的函数往往更不容易过拟合。这个偏好不能保证数学上永远等于“简单”，但实践中常有用。

 R(θ) = Σi θi2，L' = L + λR(θ)

 weight decay 直觉：每次更新前先把 θ 稍微缩小一点，例如 θ ← 0.999θ
 L2 regularization 和 weight decay 是同一思想的两种讲法，但和 Adam 结合时要小心实现方式。

AdamW 可以理解为 Adam 加上更合适的 weight decay。和 Adam 相比，它主要强化 generalization，而不是让 optimization 更强；但和最朴素的 gradient descent 相比，它仍然保留 Adam 的优化能力。

# 例题讲解：先判病因，再选技巧 {#example}
<!-- tag: EXAMPLE -->
<!-- layout: grid2 -->

## box soft
### 例题 1：深层网络比线性模型还差
**现象：**同一批训练资料上，linear model 的 training loss 能降到很低，deep network 的 training loss 反而压不下去。
**判断：**这不是典型 overfitting。深层网络的函数范围理论上通常包含线性函数，若连训练集都拟合不好，优先怀疑 optimization、初始化、learning rate 或架构可训练性。
**下一步：**先检查学习率曲线、换 Adam / RMSProp、使用 warm up 或 decay、换初始化尺度，再考虑 residual connection 与 normalization。现在你应该能看到 training loss 高时先救训练过程。

## box
### 例题 2：训练集漂亮，验证集掉队
**现象：**training loss 已经很低，training accuracy 很高，但 validation loss 降不下来。
**判断：**这是 generalization 问题。继续把 optimizer 调得更强，可能只会让训练集更漂亮。
**下一步：**优先检查资料切分和测试分布，再考虑 dropout、weight decay / AdamW、data augmentation、mixup、更多代表性资料或更合适的架构限制。现在你应该能按 validation gap 选择泛化手段。

错误示例：看到 validation 差就立刻加 dropoutdiagnosis

```text
if validation_loss_is_high:
 add_dropout()
 train_longer()

# 少了关键检查：
# 1. training loss 是否已经低？
# 2. validation split 是否代表测试分布？
# 3. metric 和 loss 是否在看同一件事？
```

这个错误的根源是把所有坏结果都叫成 overfitting。修法是先看 training loss 与 validation loss 的组合：training loss 高，先救 optimization；training loss 低而 validation 高，再救 generalization。

核心直觉
训练技巧一套排障顺序。先看训练集能不能学会，再看验证集能不能跟上，最后才决定要改优化器、改架构、改资料还是改正则化。没有诊断就加技巧，常会把问题变复杂。


术语边界
**optimization** 解决“训练集也学不好”；**generalization** 解决“训练集好、验证集差”；**regularization** 是限制模型别太迎合训练集；**normalization** 是稳定数值尺度，不等于正则化。


Worked example：三组 loss 怎么判
A：train 2.1、valid 2.3，先怀疑模型太弱或优化没到位；B：train 0.1、valid 1.8，优先查过拟合、资料切分和 augmentation；C：train 0.2、valid 0.25、线上 2.0，说明验证集没有代表真实分布，需要重做评估资料。

| 技巧 | 改哪一步 | 主要帮助 | 适用症状 |
| --- | --- | --- | --- |
| AdaGrad / RMSProp / Adam / schedule | Step 3 optimization | Optimization | training loss 压不下去 |
| Dropout | 训练时改变网络使用方式 | Generalization | training 低、validation 高 |
| Initialization / pretrain | Step 3 起点 | 两者都可能帮助 | 训练不稳、不同 seed 差异大，或想迁移已有结构 |
| CNN | Step 2 函数范围 | Generalization | 任务有局部性、共享结构，例如图像或语音 |
| Skip connection / normalization | Step 2 架构 | Optimization 为主 | 深层网络难训练、梯度尺度不稳 |
| Cross entropy | Step 1 loss | Optimization | 分类任务需要可微训练目标 |
| 更多资料 / augmentation / mixup | Step 1 loss 的资料项 | Generalization | 过拟合、训练资料不够代表真实分布 |
| Semi-supervised learning | Step 1 loss 加无标注项 | Generalization | 标注少、无标注多，且假设合理 |
| L2 / weight decay / AdamW | Step 1 或 Step 3 的等价视角 | Generalization | 模型过度依赖复杂参数、validation gap 大 |
