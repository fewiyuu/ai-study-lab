图像与声音生成：从接龙到 Diffusion / Flow Matching

这一讲不是重新细讲 diffusion，而是回答一个更大的问题：文字模型靠“接下一个 token”做得很好，图片、声音、视频能不能也这样做？如果能，为什么后来又要把 diffusion、flow matching 这条路线接回来？

直觉先放在这里：多媒体生成最开始要解决的是“生成单位是什么”。离散 token 让图片和声音可以像文字一样接龙；连续 token 提高了还原品质，却把“一个输入对应多个正确答案”的问题暴露出来，于是需要生成模型来输出分布，而不是只输出一个平均向量。

本块学完能做什么

  1. 解释为什么像素、采样点直接接龙可行但低效。
  2. 区分语音、图片、视频 token 化的形式和损失。
  3. 说明 MaskGIT、MUSE、VAR 解决的是“生成顺序/层级”的问题。
  4. 说清离散 token 和连续 token 的取舍。
  5. 用自己的话讲出 generation head 为什么是两条路线的交汇点。

概念骨架：本讲的主线

SKELETON

这节课的主线可以压成一句话：先把图像、声音、视频变成 token，让它们可以被语言模型式的接龙模型处理；再发现离散 token 会限制品质，于是改用连续 token；连续 token 又不能直接用 MSE 接龙，因为它会学到多个正确答案的平均值；最后用 diffusion / flow matching 这类生成模型做 generation head，让每一步接龙都能从一个分布中采样。

文字接龙→ 图片/声音 token→ 自回归生成→ 生成顺序优化→ 离散 token 瓶颈→ 连续 token→ 分布生成→ generation head

第一层问题：单位

文字天然有 token。图像可以按像素、patch 或语义 token 切；声音可以按采样点或语音 codec token 切；视频还要加上时间维度。

第二层问题：顺序

文字从左到右很自然，图片不一定。模型可以按 raster order 生成，也可以像 MaskGIT 那样先猜全部，再逐步保留少量较可靠的位置。

第三层问题：分布

连续 token 的目标不是唯一向量，而是一片可能答案。diffusion / flow matching 的价值，就是把“输出一个点”改成“通过采样输出一个分布中的点”。

关键术语：先把词表稳住

TERMS

这节课的术语很多，但它们不是散的。先记住一条线：media token 负责把图片、声音、视频变成模型能接龙的单位；autoregressive model 负责按条件继续生成；diffusion / flow matching 负责从连续分布里采样；generation head 把“接龙”和“采样”放在同一个系统里。

术语直觉解释精确定义或边界常见误解
Tokenizer把原始图片、声音或视频压成 token 的模型。通常是训练出来的压缩器，不是简单切块规则。以为 tokenization 一定无损。
Detokenizer把 token 还原成可听、可看的信号。还原质量受 token 信息量和训练损失共同限制。以为 detokenizer 可以凭空补回所有丢失细节。
Discrete token用编号表示一个媒体片段。模型输出的是有限类别的概率分布。只看训练方便，忽略码本瓶颈。
Continuous token用实数向量表示一个媒体片段。表达力更强，但目标空间是连续分布。直接用 MSE 当成唯一正确答案来学。
RVQ多组 token 分层补残差，提高语音表示力。Residual vector quantization，每一组补前一组没表达好的部分。以为 200 + 200 只是 400 种表达，忘了组合数会相乘。
VQ-VAE把 autoencoder 中间表示量化成码本编号。常用于学离散 latent token。把它当普通 autoencoder，忽略中间离散化。
MaskGIT先猜很多位置，再保留少量可靠 token。用 mask-and-predict 的迭代方式生成图像 token。以为它不是自回归路线的一部分。
Flow matching学习一个方向场，把简单分布推向数据分布。模型看到中间状态和时间，输出移动方向。把它和 diffusion 当成完全无关的两件事。
Generation head接在大 Transformer 后面的小型生成模型。用上下文向量作条件，多步生成下一个连续 token。以为整个大模型每一步都要反复跑完整 diffusion。

完整讲义：两条生成路线为什么会合流

LECTURE

1. 从文字接龙走到图像、声音和视频

课程开头展示了几类现在已经常见的生成能力：文字生成图片、在原图上改出新内容、跨语言复制说话人的音色、歌词生成音乐、从图片生成带声音的视频。这些例子共同说明一件事：模型输出已经不只是一串文字，而是各种连续信号。图片是二维像素，声音是一维波形，视频是一串带时间轴的图片。

如果完全照搬语言模型的直觉，图片也可以“接龙”。PixelRNN / PixelCNN 曾经逐像素生成图片；声音也可以“接龙”，WaveNet 可以逐采样点生成语音。问题是单位太细。16 kHz 的声音一秒有 16000 个采样点，一张图片也有成千上万像素；每生成一个最小单位都要调用模型，序列太长就会拖垮训练和推理。

所以这节课不是问“图片、声音能不能接龙”，答案是可以；真正的问题是“用什么单位接龙”。像素和采样点是原始单位，表达完整但太长；token 是压缩后的单位，表达会损失一部分细节，但更适合被 Transformer 处理。看到一个模型说用语言模型生成图片或声音时，先判断它生成的是像素、波形，还是 tokenizer 产生的媒体 token。

2. Tokenization：把连续信号变成可接龙单位

图片和声音的 token 化不像文字切词。文字 tokenization 通常可以相对稳定地还原原文；媒体 tokenization 更像有损压缩。你把原始信号交给 tokenizer，得到一串或一组 token；再把 token 交给 detokenizer，希望还原结果和原信号足够像。这里的“像”不是单一标准，后面会分成数值像、感知像和分布像。

语音常见做法是 codec token。课程提到 RVQ，也就是 residual vector quantization：一组 token 不够表示复杂语音，就用多组 token 叠起来。第一组表示主要信息，第二组补第一组没表达好的残差，后面继续补。它像进位制：每组只有 200 种选择，两组组合起来不是 400 种，而是 200 × 200 种。

图片 token 通常是二维网格。早期 DALL-E 式做法可以把每个 8 × 8 小块映射到一个离散 token，例如在 8192 个可能 token 中选一个。视频 token 再多一个时间维度，一个 token 可能同时代表一小块空间和几个 frame 的片段。判断一个媒体 tokenizer 时，不只看 token 数量，还要看它覆盖的是空间、时间还是语义摘要。

3. 自回归路线：把图片和声音也当成 token 接龙

一旦有了 token，图片、声音、视频生成就可以使用语言模型的办法：根据 prompt 和已经生成的 token，预测下一个 token。差别只在 token 的定义从文字变成图片小块、语音片段或视频片段。课程提到一些论文标题直接写着 “language model beats diffusion” 或 “autoregressive model beats diffusion”，就是为了强调这种路线不只是玩具，确实能生成高质量图像或视频。

这条路线的吸引力很直接：Transformer 处理序列很成熟，扩模型、扩数据、扩上下文的方法都有现成经验。P9 里提到某些工作甚至不再为图像特制架构，只把 LLaMA 式结构拿来生成图像 token，也能得到竞争力很强的结果。

图片

把图片压成二维 token，再拉直成序列。模型可以从左到右、从上到下生成，也可以用更灵活的顺序。

视频

把空间加时间一起压缩，一个 token 可能覆盖一个小区域和若干 frame。VideoGPT 早期就用 discrete latent + Transformer 做视频生成。

声音

把声音变成一维 token 序列，文字后面可以接语音 token。第十讲和作业会进一步处理语音 token。

不过，自回归路线也有两类压力。第一是长序列：图片、视频、声音 token 数量仍然可能很大。第二是 token 瓶颈：如果 tokenizer 压缩太狠，detokenizer 再强也只能根据残缺信息还原，画质、音质或细节会被上限锁住。

4. 生成顺序：图片不一定要从左上角画到右下角

文字从左到右有天然顺序，图片没有。最朴素的做法叫 raster order：按行扫描，从左到右、从上到下生成 token。但人画图常常先画主体，再补背景和细节。模型也可以不固定顺序。

MaskGIT 的想法是：训练时随机 mask 掉一些图片 token，让模型根据文字和剩下的 token 还原被遮住的位置。推理时先放一整排 mask，让模型一次猜所有位置；但第一次全部同时生成会缺少协调，可能到处都想画同一个头。于是只保留少量较可靠的 token，其他盖回去再猜。反复几轮后，图片逐渐完整。

一般自回归：一次产生 1 个 token，顺序固定。
MaskGIT 式生成：一次尝试产生所有 token，但每轮只保留 K 个，再继续修正。
关键不是“每轮谁被保留一定要最优”，而是让少量已确定内容约束后续内容，避免所有位置各画各的。

层级生成：先草稿，再细节

MUSE 先生成较低分辨率图片，再把小图扩展成高清图；VAR / visual autoregressive model 则把从小图到大图也看成一种接龙。视频生成也可以先生成粗糙、低帧率或低分辨率版本，再逐步细化。

容易混淆的点

MaskGIT 不是传统“左到右接龙”，但它仍常被放在自回归大家族里，因为它仍然是逐步决定 token，只是每一步可以决定多个位置，顺序由模型和当前置信度共同塑造。

5. 训练 tokenizer 时，什么叫“还原得像”

Tokenizer 和 detokenizer 的训练核心是：原始信号经过压缩再还原后，要和原始信号像。但“像”有三种层次，课程用声音举例，图像也一样适用。

相似度怎么量优点问题
表面上的像直接比较采样点、像素或向量距离，例如 MSE。简单、可微、容易训练。和人类感知可能差很远。声音平移一个采样点，人听不出，MSE 可能很大。
感知上的像用识别模型的 hidden representation 比距离。更接近人听起来、看起来是否相似。依赖代理模型，代理模型不一定覆盖全部感知。
模型难以分辨的像训练 discriminator 区分真信号和还原信号。能逼还原结果更像真实分布。训练更复杂，类似 GAN 式博弈。

课程里的答案很朴素：小孩子才问选哪一种，大人全都要。现实系统常常把 regression loss、perceptual loss、adversarial loss 混合起来，让还原同时满足低层数值、感知语义和真实感。

6. 连续 token：更强的表达力，也带来新的训练难题

离散 token 会限制画质和音质。比如一小块图像只用 8192 个编号之一表示，细节一定会被压掉。模型再大、数据再多，如果 detokenizer 拿到的 token 本身信息不足，上限仍然被 tokenizer 卡住。连续 token 的想法是：一个 token 不再是整数编号，而是一个向量，向量每个维度是实数。这样表达力大得多，还原结果可以更接近原图或原声。

但连续 token 不能直接照搬文字模型的训练。文字模型输出的是一个离散 token 的概率分布：看到“李宏毅是”，后面既可以接“教授”，也可以接“演员”，模型可以给两个词不同概率，然后采样其中一个。连续 token 如果用 MSE 训练，麻烦来了：同一个文字“奔跑的狗”可以对应草地上的狗，也可以对应城市里的狗。两个目标向量都对，但它们的平均值往往不是一张合理的图。

MSE 训练连续 token 的典型问题：
若正确答案可能是 zA 或 zB，模型用一个向量 ẑ 同时靠近两者，最优解可能接近平均值 (zA + zB) / 2。
在离散 token 中，平均体现为概率分布；在连续向量中，平均可能变成不自然的混合样本。

7. 生成模型与 Flow Matching：让模型输出分布

diffusion、GAN、VAE、normalizing flow、flow matching 这些方法外表不同，深层目标相似：从一个容易采样的 source distribution，例如高斯分布，变成真实数据所在的 target distribution。也就是说，它们不是只输出一个向量，而是通过“采样 + 变换”产生一整片可能样本。

Flow matching 的直觉尤其适合本讲：从 source 分布中采样一个点 x0，模型在每个时间 t 告诉它应该往哪个方向走。走很多小步后，原本简单分布里的点会被移动到目标分布附近。这个“方向场”就是 vector field。

训练 flow matching 的核心样本：
xt = (1 - t) x0 + t x1
模型看到 (xt, t)，学习输出方向 v = x1 - x0。
x0 来自 source distribution，x1 来自 target distribution。

8. 两条路线交汇：generation head

现在可以把两条路线接起来了。自回归模型负责“根据文字和已经生成的内容，决定下一步应该生成什么条件”；生成模型负责“在这个条件下，从可能答案分布中采样出一个连续 token”。这就是 generation head 的核心。

如果直接把整个巨大 Transformer 当作 flow matching 或 diffusion 的网络，每生成一个连续 token 都要跑很多 iteration，成本会很可怕。generation head 的改法是：大 Transformer 只跑一次，输出一个上下文向量；后面的小模型接收这个上下文向量和从高斯分布采样的噪声，迭代多次，把噪声变成最终连续 token。多轮迭代只发生在小 head 上，而不是反复跑大 Transformer。

为什么说是交汇

接龙模型解决序列条件：第几个 token、前面生成了什么、文字 prompt 是什么。生成模型解决连续分布：同一个条件下有许多合理答案，不能用一个平均向量糊过去。generation head 把两者各自擅长的部分拆开。

仍然存在的成本问题

即便 head 很小，每个连续 token 也可能需要多轮迭代。如果一张图有很多 continuous token，总成本仍然高。因此 2025 年附近很多研究会追问：能不能减少 iteration，甚至一步生成？课程提到 rectified flow、energy model 等方向。

这个结构还可以和 MaskGIT、VAR 等自回归技巧叠加。被 mask 的位置不再预测离散编号，而是预测连续向量；每个连续向量再交给 diffusion head 或 flow matching head 生成。于是你会看到各种标题：autoregressive image generation with continuous tokens、video generation with continuous tokens、text-to-audio with generation head。

例题讲解：把机制落到最小流程

EXAMPLE

例题 1：一张图片如何走完整个生成流程

问题：给模型一句 prompt “一只奔跑的狗”，它不是直接把每个像素都吐出来，而是先在 token 空间里生成。最常见的流程是：训练 tokenizer 把目标图片压成 token；训练生成模型预测这些 token；推理时生成新 token；最后 detokenizer 把 token 还原成图片。

判断重点：如果 token 是离散编号，模型输出的是每个编号的概率；如果 token 是连续向量，模型要处理的是一片可能向量，而不是一个固定答案。

例题 2：为什么 MaskGIT 不一次保留全部结果

第一次把所有 mask 都填掉时，每个位置都只看到文字条件，彼此之间还没有足够约束。左边 token 可能想画头，右边 token 也可能想画头，结果就会出现多个局部都合理、整体不协调的图片。每轮只保留少量结果，是为了让后续位置能被已经确定的结构约束。

判断重点：MaskGIT 的关键不是“并行生成一定更好”，而是“先给全局一个猜测，再用少量可靠 token 建立上下文”。

伪代码：先压缩，再接龙，再还原pipeline
# 训练 tokenizer / detokenizer
tokens = tokenizer(raw_image_or_audio)
recovered = detokenizer(tokens)
loss = similarity(raw_image_or_audio, recovered)

# 训练生成模型
prompt = "一只奔跑的狗"
target_tokens = tokenizer(target_image)
model.learn_to_predict(prompt, target_tokens)

# 推理
new_tokens = model.generate(prompt)
new_media = detokenizer(new_tokens)
伪代码：自回归骨架 + 小型生成头generation head
context = transformer(prompt, previous_continuous_tokens)
noise = sample_normal_vector()

for t in generation_steps:
    direction = generation_head(noise, context, t)
    noise = update(noise, direction)

next_token = noise
previous_continuous_tokens.append(next_token)

这段伪代码里，大 Transformer 负责读 prompt 和已经生成的 token，输出上下文；小型 generation head 负责多步更新噪声。删掉 generation head，就只剩“输出一个向量”的回归问题；把 head 扩成整个大模型，又会让每个 token 的多步采样成本变得很高。

核心直觉

文字生成的难点是“下一个符号是什么”；图像和声音还多一个难点：一个位置可能有很多合理的连续答案。离散 token 把问题变回分类，连续 token 则需要 diffusion / flow matching 这类生成模型来表达分布。

易混点

tokenizer 负责压缩媒体，detokenizer 负责还原；autoregressive 说明按已生成内容预测下一步；diffusion / flow matching 说明如何从噪声走向目标分布；generation head 是把分布生成放进每个 token 步的小模块。

Worked example：为什么 MSE 会平均出怪结果

同一个文字 prompt 下，下一块图像既可能是“狗头朝左”，也可能是“狗头朝右”。如果连续 token 直接用 MSE 回归唯一向量，模型可能学到两者平均，结果像模糊的中间状态。flow matching 的目标是从噪声采样出其中一个合理答案，而不是输出平均值。

误区诊断：读论文时先问四件事

DIAGNOSIS

先问生成单位

它生成的是像素、波形、离散 token、连续 token，还是 latent 表示？单位决定序列长度、还原上限和训练目标。

再问生成顺序

它是 raster order、random order、MaskGIT 式迭代、从低分辨率到高分辨率，还是多种顺序叠加？这决定推理步数和全局协调能力。

最后问分布怎么处理

如果目标是连续向量，它如何避免 MSE 平均问题？是 diffusion head、flow matching head，还是其他生成头？

常见误区、后果和修法

  • 把 tokenizer 当成无损切分，会高估 detokenizer 的补细节能力；修法是先看重建质量和 token 数量。
  • 以为自回归只能从左到右，会看不懂 MaskGIT 和层级生成；修法是把“顺序”单独拿出来问。
  • 看到 continuous token 就直接用 MSE，会得到多个正确答案的平均样本；修法是检查是否用了生成头。
  • 把 diffusion 和 flow matching 当成完全无关，会错过它们都在做分布变换的共同点；修法是从 source 到 target 的角度看。
  • 只比较生成效果不看 iteration，会忽略真实推理成本；修法是问每个 token、每张图、每秒声音要跑几次模型。
错误示例：把连续 token 当成唯一答案回归diagnosis
# 错误思路：同一个 prompt 只回归一个平均向量
target = average(all_valid_continuous_tokens_for_prompt)
pred = transformer(prompt)
loss = mse(pred, target)

# 后果：多个合理样本被压成一个不自然的混合点

读到新的图像、声音或视频生成论文时，把它拆成“表示单位、生成顺序、分布生成、推理成本”四个问题。只要这四个问题问清楚，论文标题里再多新名词，也能知道自己该往哪一层追问。
