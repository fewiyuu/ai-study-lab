13_P14｜作业3：从 Logit Lens 到 Patchscopes

这份作业表面上是十题多选题，真正要补的是一块可解释性基础：语言模型不是只在最后一层才“突然知道答案”。每一层都会产生一个 hidden state，也就是这一层对当前 token 的内部表示。我们可以把它投到词表空间、把某类行为抽成向量，或把它移植到另一个模型里请模型解释。

概念骨架

MAP

共同对象：hidden stateTransformer 每一层都会为每个 token 产生一个向量。这个向量不是人类可直接阅读的文字，但它包含模型到这一层为止整合出的信息。判断句：只要问题在问“模型某一层知道了什么”，第一反应就应该找对应 token、对应 layer 的 hidden state。

三种读法Logit Lens 把 hidden state 直接送过 LM head，看它像不像某个 next token。Function Vector 把一批有共同行为的 hidden state 相减，得到能注入行为的方向。Patchscopes 把 hidden state 放到另一个解释 prompt 里，让语言模型自己说出它编码了什么。判断句：投影看 token 倾向，相减看行为方向，patch 看可读解释。

作业边界这份作业不要求提交代码，但选择题里有两题依赖 sample code 的观察结果。你需要取得 Llama-3.2-1B-Instruct 和 AdvBench 资料集授权，填入 Hugging Face token，用 GPU 跑 Colab，再观察特定层数下的 top token 和拒绝效果。判断句：没有实际跑样例时，层数相关题目容易靠猜。

术语表

TERMS
术语直觉解释准确说法常见误解
hidden state某一层对某个 token 的内部记号。Transformer block 输出的向量，维度等于模型隐藏维度，后续层会继续读取和改写它。把它当成一个确定的人类语义标签；实际上它是分布式表示。
LM head把模型内部向量转回词表分数的投影头。通常是最后用于计算 vocabulary logits 的线性层或 tied embedding 矩阵。以为只有最后一层能接 LM head；Logit Lens 正是把中间层也拿来投影。
Logit Lens给模型每一层戴一个“词表眼镜”。把中间层 hidden state 通过最终 LM head 得到 pseudo-logits，再看 top token。把早期层的乱码 top token 当成稳定解释；早期层未必已经对齐到词表空间。
Function Vector把一类行为压成一个可注入的方向。从一组激活中抽取能触发某种函数或行为的向量，可加到模型内部表示上影响输出。以为任意层注入都有效；实际效果往往只在特定层或位置明显。
refusal vector让模型倾向拒绝的行为方向。课程样例用“会拒绝 prompt”的平均 hidden state 减去“不会拒绝 prompt”的平均 hidden state 得到。以为只收集有害 prompt 就够了；不减去普通 prompt 会混入主题、语气等无关信息。
Patchscopes把一个模型的内部表示拿给另一个上下文解释。将 source model 某层某 token 的 hidden representation patch 到 target model 的提示位置，让 target model 生成可读解释。把它理解成普通问答；关键在于被解释对象中间层表示。
source / target model被观察的模型和负责解释的模型。source model 提供 hidden state；target model 接收转换或插入后的表示并生成答案。以为两者必须完全相同；Patchscopes 可以用更强模型解释较小模型，但要处理表示空间差异。
Hugging Face token访问受限模型和资料集的钥匙。Colab 中登录 Hugging Face 后才能下载 gated model 或 gated dataset。把 token 当作作业答案共享；这既不安全，也可能违反授权要求。

读完这一段，你应该能看到一个题目就判断：它是在考“投影到词表”、 “抽取行为方向”，还是“换一个上下文解释表示”。

讲义主线

LECTURE

1. 为什么要看中间层

语言模型给出一个 token 之前，会经过很多层 Transformer。最后一层输出 logits，我们能直接看到模型认为下一个 token 是什么；中间层就没那么直观。问题是，中间层并不是空白的。模型可能在某些层已经知道主题、实体或答案，只是还没有把这些信息整理成最终词表分布。

作业里的首都例子就是这个思路：prompt 问的是 Poland 的首都，早期层 top token 可能还是括号、字母或不相关片段；到某些中后层，top token 开始出现 Poland；再往后才稳定出现 Warsaw。这个变化说明模型的表示逐层从形式、主题、实体关系走向最终答案。

读到任何“模型在想什么”的说法，都要先问它观察的是什么对象。观察最终输出，只能知道结果；观察每层 hidden state，才有机会看到结果形成的过程。你现在应该能做的是：把“最终答案”和“中间表示”分开说，不把两者混成同一个东西。

2. Logit Lens：把中间层投到词表

Logit Lens 的做法很直接：拿某一层某个 token 的 hidden state，送进模型最后用来预测词表的 LM head，于是得到一组 pseudo-logits。分数最高的 token，就是“如果这一层现在必须回答，它最像会接什么”。这不等于真实最终输出，但它提供了一个可比较的视角。

它适合回答两类问题。第一，答案或主题什么时候开始出现？比如从哪一层开始 top token 变成 Poland，后来才变成 Warsaw。第二，模型的表示是否逐层变得更接近最终预测？如果某些层的 top token 完全无法解释，不能急着说模型“什么都不知道”，因为早期 hidden state 可能还没有对齐到最终词表空间。

课程样例中，Llama-3.2-1B-Instruct 最多观察到第 16 层；你可以改 target layer，看同一个 prompt 在不同层 top-5 token 的变化。你现在应该能做的是：看到一张每层 top token 表，能说出哪个阶段是无关片段、哪个阶段开始抓到主题、哪个阶段接近答案。

  1. Function Vector 的直觉如果模型在某些 prompt 下稳定表现出同一类行为，例如拒绝有害请求，那么这些样本的内部激活里可能共享某个方向。把这个方向抽出来，再加到另一个 prompt 的 hidden state 上，就可能诱导模型表现出相同行为。这里的“函数”模型内部可触发的一种过程或能力。你现在应该能做的是：把“行为向量”理解成一种内部方向，而不是一段显式规则。
  2. 为什么要相减只平均会拒绝 prompt 的 hidden state 不够，因为里面不只有拒绝行为，还混着主题、语气、句长、安全词等信息。课程样例再收集一批不会拒绝的 prompt，把两边平均激活相减，目标是抵消普通 prompt 也有的背景成分，留下更接近拒绝行为的方向。判断句：如果没有 non-refusal baseline，抽出来的向量就可能“这批资料的共同风格”。
  3. 注入不一定处处有效把 refusal vector 加到模型内部表示上，不代表每层都会让模型拒绝。不同层负责的信息组织不同，有些层太早，行为还没形成；有些层太晚，输出分布已接近定型。作业题会让你观察：在让模型回答 2+2 时，哪些层注入拒绝向量会让它拒绝，哪些层仍然回答 4。你现在应该能做的是：解释“层数”为什么会影响干预效果。
  4. Patchscopes 的动机Logit Lens 在前面层常会给出难解释的 token。难解释不代表信息不存在，只是“直接投到词表”这个读法可能不合适。Patchscopes 换了一个问法：把 source model 的某层 hidden state 放到 target model 的解释提示中，让 target model 生成一个可读答案。判断句：Patchscopes 让另一个上下文把这个表示翻译出来。
  5. CEO 例子的含义论文例子里，source prompt 类似 “Amazon's former CEO attended Oscars”。研究者取 CEO 这个 token 在某层的 hidden state，经过转换后 patch 到 target prompt 的问号位置。target model 输出 Jeff Bezos，说明 source model 在那一层的 CEO 表示里已经编码了“Amazon 前 CEO 是 Jeff Bezos”这一信息。你现在应该能做的是：说清楚 Patchscopes 的证据链，不只背“它能解释 hidden state”。
  6. Colab 不是可选摆设这份作业不用提交代码，但 sample code 不是装饰。Logit Lens 要你看 top-5 token 随 layer 变化；Function Vector 要你看拒绝向量加到不同层后的输出差异。选择题如果问具体层数或观察结果，必须以课程提供模型和参数为准，随便换模型会让答案变掉。判断句：模型 ID、资料集、target layer 和注入位置都属于实验条件。

完整知识链路

把本节串起来，可以这样讲：语言模型每层都会产生 hidden state。Logit Lens 把这些 hidden state 直接送过 LM head，看每层像是在预测什么 token；Function Vector 把多组激活相减，抽出类似拒绝这样的行为方向，再测试注入到不同层是否改变输出；Patchscopes 则把 hidden state patch 到另一个模型或提示里，让它生成可读解释。三者都在观察内部表示，但读法不同：一个看词表分数，一个看行为方向，一个看跨上下文解释。你现在应该能拿到一个新可解释性方法时，先问它读的是哪一层、哪个 token、用什么方式把向量变成人能理解的证据。

最小例子

EXAMPLE

下面把三个方法压成最小可读结构。目的看懂每一步读哪个向量、写哪个结果、删掉后会坏在哪里。

代码
prompt = "The capital of Poland is"
inputs = tokenizer(prompt, return_tensors="pt").to(model.device)

with torch.no_grad():
 outputs = model(**inputs, output_hidden_states=True)

target_layer = 12
last_token_state = outputs.hidden_states[target_layer][0, -1, :]
logits = model.lm_head(last_token_state)
top_ids = logits.topk(k=5).indices
print(tokenizer.convert_ids_to_tokens(top_ids))

为什么这段容易错

第一行和第二行必须用两类 prompt。只用 harmful prompts 会把“有害请求的主题特征”和“拒绝行为”混在一起。第三行做相减，是为了让普通 prompt 也有的语言风格、长度和背景信息尽量抵消。最后的 generate_with_added_vector 是干预：把向量加回某层某位置，让模型输出被这个方向推一下。

如果换层后效果消失，不代表拒绝向量不存在，而是说明这一层未必是最容易影响拒绝行为的位置。你现在应该能做的是：解释为什么作业会问“哪几层有效”，而不是只问“有没有拒绝向量”。

最小示例：Patchscopes 的数据流text

source prompt: Amazon's former CEO attended Oscars.
source token: CEO
source layer: 取 CEO 位置某一层的 hidden state

transform: 把 source hidden state 转成 target model 可接收的表示

target prompt: cat -> cat
 135 -> 135
 hello -> hello
 ? -> [patched hidden state]

target output: Jeff Bezos

读这个例子时别跳过证据链

Patchscopes 的结论不是“模型回答了 Jeff Bezos，所以它懂了”。证据链更细：被 patch 的对象是 source model 在 CEO 这个 token、某一层的 hidden state；target prompt 是一个解释模板；target model 输出 Jeff Bezos，说明那个 hidden state 中已经包含能被解释成 Jeff Bezos 的信息。

如果 source 和 target 模型不同，中间还需要 transform，让表示空间能接上。你现在应该能做的是：用“source 表示 → transform → target prompt → 解释输出”四步复述 Patchscopes。

误区诊断

DIAGNOSIS

常见坑

  1. 把 Logit Lens 的 top token 当成模型真实最后答案。修法：说清它是把中间层 hidden state 经过 LM head 得到的 pseudo-logits，只是观察视角。
  2. 看到早期层 top token 很怪，就说模型早期层没有信息。修法：承认早期层可能未对齐词表，可换 Patchscopes 等方法解释。
  3. 只平均会拒绝 prompt，不设置不会拒绝 prompt 的 baseline。修法：用两组平均 hidden state 相减，减少主题和风格噪声。
  4. 以为 refusal vector 加到任何层都有效。修法：实际测试不同层，因为行为方向只在某些层和位置更容易影响输出。
  5. 把 Patchscopes 当成普通问答。修法：指出被问的source model 的某层 hidden representation。
  6. 随便改模型 ID 再回答测验层数题。修法：作业题以课程指定 Llama-3.2-1B-Instruct 和样例参数为准。
