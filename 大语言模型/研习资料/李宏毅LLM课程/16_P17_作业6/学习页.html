概念骨架

MAP

任务：十分类图片识别模型看到一张角色图片，要输出十个角色之一。training set 和 validation set 有 label，testing set 没有 label。最终 prediction file 应该有 500 笔 id 与预测类别，对应 testing set 的 500 张图片。判断句：这是 classification，不是预测连续分数，所以评估看 accuracy 而不是 RMSE。

CNN 为什么适合图片图片有 spatial locality：邻近像素关系更强；也有 translation invariance：同一个局部形状移动位置后仍然代表类似特征。CNN 用小 kernel 在局部区域扫过图片，既能抓局部特征，又比把整张图摊平成 MLP 少很多参数。判断句：CNN 的优势来自参数共享和局部感受野，不只是“层数更多”。

训练与提交训练时用 data augmentation 增加变化，用 convolution / ReLU / pooling / linear layers 做分类，用 CrossEntropyLoss 和 optimizer 更新参数。validation 用来保存表现最好的 model weights，inference 时对 test set 生成 prediction file 上传 Judge Boy。判断句：validation 不更新参数，testing 没有 label，只能生成提交。

讲义主线

LECTURE

1. 图像分类：把图片映射到类别

这次作业的目标很直接：给模型一张 MyGO 或 Ave Mujica 角色图片，让它预测十个主要角色中的哪一个。输入是 RGB 图片 tensor，输出是 10 维 logits，每一维对应一个角色。最后选 logits 最大的类别作为 prediction file 里的预测结果。

和上一节回归不同，这里不预测连续分数，而是做 classification。训练时 label 角色编号。loss 也从 MSE/RMSE 换成 CrossEntropyLoss，评估重点变成 accuracy：预测类别和真实类别是否一致。

你现在应该能做的是：把一张图片进入模型后的路径说清楚：resize/augmentation → tensor → CNN feature extractor → linear classifier → class prediction。

2. 为什么不用巨大 MLP 直接摊平图片

如果把 128×128×3 的图片摊平成向量，再接一个很大的 linear layer，参数量会爆炸。字幕里的例子中，一个 MLP 第一层就可能有 2 亿多个可训练参数，模型大小接近 776 MB，而且效果还不一定好。

CNN 的核心是局部连接和参数共享。一个 3×3 kernel 会在整张图片上滑动，使用同一组参数提取不同位置的局部特征。这样参数量和图片大小不再一对一爆炸。例子里的 CNN 只有两万多个参数，模型大小约 0.11 MB，却能比巨大 MLP 更适合图片分类。

你现在应该能做的是：解释 CNN 为什么“更小但更适合图片”，而不是只说它是图像任务常用架构。

  1. Data augmentation资料只有 1578 张训练图，容易不够。augmentation 用现有图片制造合理变化，例如 crop、水平翻转、颜色扰动、模糊。选择增强时要问：它会不会改变角色类别？如果不会，就可能帮助模型学到更稳的特征。判断句：增强是扩大训练变化，不是凭空增加新角色。
  2. Convolution 参数nn.Conv2d(in_channels, out_channels, kernel_size, stride, padding) 是本节核心。RGB 图片第一层通常 in_channels=3；out_channels 决定输出多少张 feature map；kernel size 决定局部窗口大小；stride 决定滑动步长；padding 决定边缘补多少。判断句：改 convolution 后，必须检查输出 shape 能不能接下一层。
  3. Padding 与 stride 的取舍不加 padding 时，卷积会让空间尺寸变小；加一圈零 padding 可以让某些 kernel/stride 下输出尺寸更容易控制。stride 大会减少计算量，但可能略过细节；stride 小保留更多信息，但计算更重。判断句：空间尺寸变化不是小事，它会影响 flatten 后的 linear input dimension。
  4. Pooling 与 ReLUReLU 把负值压成 0，引入非线性；pooling 在局部窗口里保留最大值或平均值，降低空间尺寸和计算量。max pooling 常用于保留最强特征，average pooling 更像平滑汇总。判断句：pooling 通常不学习参数，但会改变 feature map 尺寸。
  5. CrossEntropyLoss 与 accuracy分类模型输出 10 维 logits，不需要先手动 softmax 再给 CrossEntropyLoss。训练时 loss 用于更新参数；validation 时 accuracy 用于看真实分类表现。训练 loop 和回归类似，但指标和 label 形式不同。判断句：loss 下降不一定等于 accuracy 一定同步提高，两个都要看。
  6. Baseline 与技巧sample code 直接跑可拿基本分；微调 hyperparameter 或换模型可过更高 baseline。可尝试 data augmentation、调整 CNN layer、使用 pretrained weights、ensemble、dropout、test-time augmentation，但不一定需要全部用上。判断句：技巧越多不等于越稳，先根据 validation curve 判断瓶颈。

完整知识链路

把本节串起来，可以这样讲：作业6 用 CNN 做十分类图像识别。先把图片 resize 成固定大小，并在训练时做不改变角色身份的 augmentation；再用卷积层抓局部特征，用 ReLU 和 pooling 形成更稳定、更小的 feature map；随后 flatten 接 linear classifier 输出十个角色的 logits；训练时用 CrossEntropyLoss 和 optimizer 更新参数，validation 时看 loss 和 accuracy 并保存 best weights；最后对 testing set inference，生成 500 笔 id/prediction 的文件上传 Judge Boy。想提高 baseline，要在模型大小、augmentation、CNN 深度、hyperparameter 和泛化之间取平衡。

CNN 作业排查模板

  1. 先查数据：图片是否 resize 到统一尺寸，label 是否对应 0-9 类别，train/valid/test 是否分开。
  2. 再查 shape：每层 Conv2d 的 in_channels 是否接上前一层 out_channels，pooling 后 flatten 维度是否匹配 linear。
  3. 再查训练：train loop 有 zero_grad、backward、step，valid loop 没有 step。
  4. 最后查提交：500 个 id 顺序不乱，prediction 是类别编号，文件不是手动改出来的。

worked example：validation 低于 training

如果训练准确率 95%，validation 只有 55%，先不要继续加深模型。更合理的排查是：augmentation 是否破坏角色身份、模型是否过大、epoch 是否过多、validation set 是否分布不同。修法可能是减小模型、加 dropout/weight decay、调弱增强或回到 best validation weights。

最小例子

EXAMPLE

这一节的最小例子不追求把作业代码完整重写，而是把最容易出错的三件事摆清楚：卷积层的输入输出 shape、参数量为什么比 MLP 小、训练循环里每一步到底在更新什么。

代码
# 输入是一批 RGB 图片：batch_size x channel x height x width
x = torch.randn(32, 3, 128, 128)

# 3 个输入通道，9 个输出通道，2x2 kernel，stride=1，padding=1
conv = nn.Conv2d(in_channels=3, out_channels=9, kernel_size=2, stride=1, padding=1)
y = conv(x)

print(y.shape) # torch.Size([32, 9, 129, 129])

# 不含 bias 的参数量：3 * 9 * 2 * 2 = 108
# 若有 bias，再加 out_channels = 9

为什么这个结构能用于作业

输入图片被 resize 到 128 x 128。第一段 MaxPool2d(2) 把宽高降到 64 x 64，第二段再降到 32 x 32；最后有 64 个通道，所以 flatten 后长度是 64 32 32。分类头输出 10 个 logits，对应 MyGO / Ave Mujica 的 10 个角色类别。

BatchNorm2d 用来稳定每层特征的数值分布，ReLU 加入非线性，MaxPool2d 降低空间尺寸并保留局部强响应。这里的重点能解释：卷积负责抽局部特征，池化负责压缩位置细节，线性层负责把特征投到类别分数。

如果你把第二层卷积改成 nn.Conv2d(16, 64, 3)，模型会报错，因为前一层输出通道是 32，不是 16。如果你删除一次 pooling，最后的 flatten 长度会变成 64 64 64，分类头的输入维度也必须跟着改。

训练 step：分类任务的 loss 和 accuracypython

criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

model.train()
for images, labels in train_loader:
 images, labels = images.to(device), labels.to(device)
 logits = model(images) # shape: [batch, 10]
 loss = criterion(logits, labels) # labels 是类别编号，不是 one-hot

 optimizer.zero_grad()
 loss.backward()
 optimizer.step()

model.eval()
correct, total = 0, 0
with torch.no_grad():
 for images, labels in valid_loader:
 logits = model(images.to(device))
 pred = logits.argmax(dim=1)
 correct += (pred.cpu() == labels).sum().item()
 total += labels.size(0)
accuracy = correct / total

训练循环的检查点

CrossEntropyLoss 直接吃 logits，不需要你先手动 softmax；它内部会处理 log-softmax 和负对数似然。如果你先 softmax 再喂进去，数值稳定性和梯度都会变差。labels 应该是 0-9 的类别编号，而不是十维 one-hot 向量。

zero_grad -> backward -> step 是一次参数更新的顺序：先清空旧梯度，再根据当前 batch 反向传播，最后让优化器更新参数。validation 只评估泛化表现，所以要用 model.eval() 和 torch.no_grad()，不能在 validation loop 里调用 optimizer.step()。

误区诊断

DIAGNOSIS

常见坑

  1. 把 data augmentation 当作越多越好。错因：没有检查增强是否改变标签语义；后果：模型学到错误样本，例如过度裁切导致角色关键特征消失；修法：只保留不改变类别身份的 crop、flip、color jitter、blur 等操作，并用 validation 比较。
  2. 改了 Conv2d 后不检查 shape。错因：只看层数，不看通道和宽高；后果：下一层 in_channels 或 Linear 输入维度不匹配；修法：每改一段就打印 tensor shape 或手算输出尺寸。
  3. 手动 softmax 后再喂 CrossEntropyLoss。错因：误以为 loss 需要概率；后果：梯度和数值稳定性变差；修法：让模型输出 logits，直接传给 CrossEntropyLoss。
  4. validation loop 里还在更新参数。错因：没有区分 train 和 eval；后果：验证分数不再是干净评估，还可能浪费显存；修法：validation 使用 model.eval()、torch.no_grad()，不调用 backward 和 step。
  5. 认为模型越大一定越好。错因：忽略数据量只有 1578 张 train 图；后果：训练集准确率升高但 validation/test 变差；修法：用 validation accuracy、dropout、weight decay、early stopping、pretrained weights 共同判断。
  6. 最后手动改 prediction file。错因：把提交文件当成随手可调的表格；后果：id 顺序、类别编号或行数出错，500 张 test 图无法正确评分；修法：由推理脚本稳定导出 id/prediction 两列，再检查行数和类别范围。
