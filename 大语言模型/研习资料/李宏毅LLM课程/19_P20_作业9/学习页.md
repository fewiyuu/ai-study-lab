# 概念骨架 {#skeleton}
<!-- tag: MAP -->

Diffusion：从噪声到图片forward process 往干净图片加 noise；reverse process 从 noise 逐步 denoise。模型真正学的是在给定 step 和条件时预测噪声。
Text-to-image：加上文字条件如果只给 step 和 noisy image，模型只能去噪。加上 text prompt 的 condition 后，模型就能朝“河边毛绒娃娃”“海边背包”等目标生成。
Customization：指定对象身份普通 prompt 很难精确描述“我家的这只狗”。客制化通过参考图片和 special token，让模型把某个 token 绑定到指定对象特征。

# 术语表 {#terms}
<!-- tag: TERMS -->

| 术语 | 直觉解释 | 准确说法 | 常见误解 |
| --- | --- | --- | --- |
| text-to-image | 用文字生成图片。 | 模型接收 prompt，生成与文字描述对齐的图像。 | 以为 prompt 越长越一定能描述出指定对象。 |
| forward process | 把干净图片逐步弄脏。 | 对图片逐步加入 noise，构造训练需要的 noisy image / noise pair。 | 以为训练只需要最终纯噪声，不需要中间 step。 |
| reverse process | 从噪声一步步还原图片。 | 从高 noise step 开始，多次 denoise 得到 clean image 或 clean latent。 | 以为模型一步直接生成完整图片。 |
| noise predictor | 预测这一步要移除的噪声。 | 输入 noisy image / latent、step 和 condition，输出预测 noise。 | 以为模型直接输出去噪后的图片；实际常预测 noise 更稳定。 |
| latent diffusion | 在压缩表示里去噪。 | 先把图片表示成 latent，在 latent space denoise，再用 decoder 还原成图片。 | 以为 diffusion 一定在像素空间做所有计算。 |
| special token | 代表指定对象的新词。 | 例如 new1 ，训练后模型把它和 reference object 的视觉特征绑定。 | token 不放进 prompt，模型就能自己知道对象是谁。 |
| Custom Diffusion | 通过训练学会指定对象。 | 用 reference images 微调部分参数，让 new1 指向目标对象。 | 只改 prompt 不训练，也叫 Custom Diffusion。 |
| BLIP-Diffusion | 不训练，直接用图像编码器给对象条件。 | predictor / image encoder 把 reference image 编成模型可识别的条件 token。 | 它不需要任何 reference image；其实 reference image 是条件来源。 |
| guidance scale | 文字条件影响多强。 | 越高越强迫模型听 prompt，但太高可能颜色怪异或图像失真。 | 越高越好，最高一定最符合 prompt。 |
| image alignment | 生成图像像不像参考对象。 | 评估目标对象身份、外观特征是否保留。 | 只要符合场景就算好，不看对象像不像。 |
| text alignment | 生成图像是否符合 prompt 场景。 | 评估姿态、环境、动作、物品等文字条件是否满足。 | 只要对象像就算好，不管 prompt 要求。 |

你现在应该能做什么：看到一张生成图，分别说出它的 image alignment 和 text alignment 可能高还是低。

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 1. Diffusion 为什么从 noise 讲起
Text-to-image 的目标是“输入文字，输出图片”。Diffusion model 的做法从 noise 开始，一步步 denoise。编号越大的 step 通常 noise 越严重；越接近 step 1，图片越干净。
训练时我们可以从干净图片出发，主动加 noise，得到每个 step 的 noisy image 和真实 noise。这样就构造出了 input / output pair：输入 noisy image 和 step，目标是预测当时加进去的 noise。

## box
### 2. 为什么预测 noise
视频强调，实际模型内部常做的是 noise prediction。模型先预测当前图里的 noise，再把预测的 noise 从 noisy image / latent 中扣掉，得到更干净的结果。
经验上，让模型预测噪声通常比直接吐出去噪图片更好训练。你可以把它理解为：模型每一步只解决“这一步哪里是噪声”，而不是一次性承担“整张图该长什么样”的所有责任。

3. Text guidance 的作用没有文字条件时，模型只知道要变干净，不知道要生成什么。加入 text prompt 后，每一步 denoise 都有方向：毛绒娃娃、海边背包、河边场景等条件会影响最终图像。这也是 Task 1 prompt 向量操作能成立的原因：操作发生在模型内部的 condition / latent 表示上，不是在像素上直接相加相减。
4. Latent diffusion 更省计算现代 diffusion 往往不直接在 pixel space 里做所有去噪，而是在 latent space 操作。图片先被编码成 latent representation，denoise 后再通过 decoder 还原成图片。你不需要在作业里推完整公式，但要记住：中间表示模型可处理的向量或张量。
5. Customization 为什么难“生成一只柯基”很容易，“生成我这只柯基穿西装、在水里游泳”很难。即使用很长 prompt 描述外观，模型也未必能抓住指定对象身份。所以 customization 要把 reference image 引入模型，让一个 special token 代表这个对象。之后 prompt 里写 `new1 dog`，模型就应该生成像参考狗的狗。
6. Custom Diffusion：训练新 tokenCustom Diffusion 会用 reference images fine-tune 模型，让 `new1` 学会对应对象特征。instant prompt 很重要，例如参考图是黑狗，就把训练 prompt 写成 “photo of a new1 black dog”。如果训练 prompt 加了修饰，inference prompt 也要做相应调整，否则训练时学到的条件和生成时使用的条件不一致。
7. BLIP-Diffusion：用 predictor 免训练BLIP-Diffusion 的路线不同：它不要求你重新 fine-tune 模型，而是用 pretrained predictor / image encoder 把 reference image 编码成模型可识别的条件。好处是省训练时间，且因为预训练时见过很多 reference image，泛化能力可能更好。作业里主要调 inference steps 和 guidance scale。
8. 双指标评分与提交Judge Boy 用 GPT-5 mini 评估生成结果，既看 reference object 像不像，也看 prompt 场景是否符合。两个指标都要过 baseline，单一指标高不够。Task 2 有两个方法，各要生成 30 张图片并上传对应 zip。不要改 zip 文件名；Judge Boy 每天 3 次提交，临近截止可能排队。

## box info
### 完整知识链路
本节可以这样串：diffusion 先通过 forward process 学到不同 step 的噪声，再通过 reverse process 从 noise 逐步 denoise；text-to-image 在每一步加入 prompt condition；customization 进一步加入 reference object，让模型不仅符合文字场景，还保留指定对象身份。Custom Diffusion 靠训练 special token，BLIP-Diffusion 靠 pretrained predictor 编码 reference image。最后结果用 image alignment 和 text alignment 双指标评估。你现在应该能做什么：看到一个生成失败案例，判断它是对象不像、场景不符、guidance 太强，还是训练 prompt / inference prompt 不一致。

# 最小例子 {#example}
<!-- tag: EXAMPLE -->

下面用最小伪代码把 P20 的三个关键结构摆出来：去噪循环、Custom Diffusion prompt、两种 alignment 检查。

示例 1：diffusion denoising looppython

```text
latent = torch.randn(shape) # 起始 noise
text_cond = text_encoder("a new1 dog swimming in the water")

for step in scheduler.timesteps:
 predicted_noise = noise_predictor(latent, step, text_cond)
 latent = scheduler.step(predicted_noise, step, latent)

image = decoder(latent)
```

你现在应该能做什么：指出 `step`、`text_cond`、`predicted_noise` 分别在 reverse process 中扮演什么角色。

示例 2：Custom Diffusion 的 prompt 一致性python

```text
# 训练时 reference object 是一只黑狗
instance_prompt = "a photo of a new1 black dog"

# 推理时也保留对象类别和 new1，不要只写场景
prompts = [
 "a photo of a new1 black dog in a suit",
 "a photo of a new1 black dog swimming in the water",
]
```

## box soft
### 逐行理解
`new1` 是对象身份，`black dog` 是辅助描述。训练和推理都保留这些线索，模型更容易同时抓住“这只狗”和“新场景”。如果推理时只写 “a dog in a suit”，模型可能退回生成普通狗。

示例 3：双指标验收python

```text
def pass_customization(image_alignment, text_alignment, image_th, text_th):
 return image_alignment >= image_th and text_alignment >= text_th

print(pass_customization(8.8, 6.0, 6.5, 8.5))
# False：对象像，但 prompt 场景不够对齐。
```

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->
<!-- layout: grid2 -->

## box redsoft
### 常见坑
1. 以为 diffusion 一步生成图片。错因：忽略 reverse process；后果：看不懂 Task 1 的中间图；修法：按 step 观察 noise 逐渐减少。
2. 把 prompt 向量运算理解成像素相减。错因：不知道操作发生在 condition / latent 表示上；后果：NTU COOL 观察题选错；修法：回到 latent diffusion 和 text condition。
3. Custom Diffusion prompt 不含 `new1`。错因：忘记 special token 是对象身份；后果：模型只能生成普通类别；修法：训练和推理 prompt 都保留 `new1`。
4. 训练 prompt 加了 cat/dog，推理 prompt 却没加。错因：训练推理条件不一致；后果：对象身份不稳；修法：推理时同步保留关键修饰。
5. 只追求 image alignment。错因：只看对象像不像；后果：场景不符合 prompt，text alignment 不过线；修法：每张图同时检查对象和场景。
6. guidance scale 开太高。错因：以为越听 prompt 越好；后果：颜色怪异、图像失真；修法：参考建议不要超过 20，并实际观察结果。
7. zip 文件名或图片数量错误。错因：手动改动输出文件；后果：Judge Boy 上传失败或评分异常；修法：使用 sample code 生成的两个 zip，每个对象 30 张图片。
