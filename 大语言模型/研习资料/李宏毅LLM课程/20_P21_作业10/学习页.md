# 概念骨架 {#skeleton}
<!-- tag: MAP -->

Tokenizer：把语音离散化Mimi 输入 24 kHz 音讯，按时间切成 frame，经 convolution、transformer、linear projection，再用 VQ / RVQ 量化成多层 token。Detokenizer：把 token 还原成语音如果 token 保留了足够资讯，detokenizer 重建出来的音讯就会接近原音。PESQ、听感和 transcript 是三种不同视角。Backbone：用 token 生成新语音CSM 1B 接收 text 和 audio prompt，输出新语音。它使用 Mimi token，所以前两段观察会直接影响你理解第三段生成。

| 1 | Mimi tokenizer / speech tokens | token 层、VQ/RVQ、UMAP 情绪分布 | 1-6 |
| 2 | Mimi detokenizer | 重建品质、shuffle token、Alice/James token 交错 | 7-10 |
| 3 | CSM backbone LLM | speech LLM 如何生成新音讯与 voice cloning | 11-12 |

# 术语表 {#terms}
<!-- tag: TERMS -->

| 术语 | 直觉解释 | 准确说法 | 常见误解 |
| --- | --- | --- | --- |
| speech generation | 输入一段语音或条件，输出有关系的新语音。 | 输入 speech sequence，输出与输入存在任务关系的 speech sequence。 | 以为任意输出音档都算生成；输出必须服务翻译、对话、换声等任务。 |
| cascade model | ASR、LLM、TTS 串起来。 | 先把语音转文字，再由文字模型回答，最后把文字转成语音。 | 以为文字能完整保留语气、情绪、音色和笑声。 |
| Mimi codec | 语音的 tokenizer 加 detokenizer。 | 共同训练 encoder、quantizer、decoder，让音讯可离散化并可重建。 | 以为 tokenizer 和 detokenizer 是互不相关的两个工具。 |
| VQ | 从 codebook 里找最近的 token。 | Vector Quantization，把连续 embedding 映射到最近的 codebook vector。 | 以为 token id 的数字大小有连续语义。 |
| RVQ | 第一层不够，再量化剩下的误差。 | Residual Vector Quantization，逐层量化 residual，让离散表示更接近原 embedding。 | 以为层数越多只是重复，没有补新的资讯。 |
| coarse / fine token | 粗层抓大结构，细层补细节。 | 较前层常保留较粗的语义或声学结构，后层继续补 residual 细节。 | 以为任意一层都保留同样资讯。 |
| UMAP | 把高维 token embedding 画成二维点图。 | 非线性降维工具，尽量保留局部结构，便于观察聚类趋势。 | 把二维距离当成原空间的精确距离。 |
| PESQ | 客观比较重建音质。 | 比较 reference audio 与 reconstructed audio，分数约 -0.5 到 4.5，越高越接近。 | 完全取代听感；它只回答相似度的一部分。 |
| Whisper transcript | 把音讯转写成文字来辅助判断内容。 | 用 ASR 看重建后说的内容是否接近原句。 | 以为 transcript 能说明音色和情绪是否保住。 |
| CSM | 使用 Mimi token 的 speech LLM。 | 可输入 text + audio prompt，生成带 voice cloning 特性的语音。 | 以为它只是普通 TTS，不需要 speech tokens。 |

读完术语表，先能把“语义内容”“情绪/音色”“重建音质”分开说。P21 的很多选择题就卡在这三件事被混成一件事。

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 1. 为什么不满足于 ASR + LLM + TTS
字幕里的例子是，一个人用生气或讽刺的语气说“你真厉害”。ASR 只会留下文字“你真厉害”。这句话单独看可以是称赞，也可以是讽刺；一旦情绪和语调被丢掉，后面的 LLM 很可能按普通称赞来回答，TTS 再把回答读出来，系统就错过了原始语音里的关键信号。
cascade model 的问题中间的文字瓶颈太窄。文字擅长表示语义内容，却不擅长保留说话人、音色、笑声、停顿、情绪强度。P21 要你观察 speech token model，是因为它尝试把语音先变成离散 token，让模型在 speech token domain 里处理较完整的语音资讯。
判断句：如果一个系统先把语音压成 transcript，再进入语言模型，它就可能丢掉 transcript 以外的声音线索。

## box
### 2. Mimi codec 的任务
Mimi 是 codec：前半段 tokenizer 把音讯变成 tokens，后半段 detokenizer 把 tokens 重建回音讯。它们在训练时一起优化，让输入音讯和输出音讯尽量接近。
作业不要求你训练 Mimi，只用已经训练好的模型观察它的行为。你要看的token 里面到底保住了什么。能保住内容、情绪、音色，detokenizer 才有机会重建出像原音的结果；保不住的部分，后面再怎么解码也补不回来。
判断句：遇到 codec 题，先问 token 是否足以让 detokenizer 重建原音，而不是只问 token id 长什么样。

3. 从音讯到二维 token 表
Mimi 输入固定 24 kHz 的音讯，再按时间切成 frame。字幕中提到 frame 率是 12.5 Hz，也就是沿时间轴抽出一段段表示。每个 frame 经过 convolution、transformer 和 linear projection 后，得到连续 embedding。
文字 token 通常是一维序列；Mimi 的 speech tokens 更像二维表。横向是时间 frame，纵向是 quantizer layer。你看到的某个 token id，必须同时知道它属于哪一层、哪个时间片，才有解释意义。
判断句：如果只把 speech token 当成一串像文字一样的 id，就会看不懂后面的 shuffle、split index 和 layer 观察。


4. VQ / RVQ 怎样把连续向量离散化
VQ 的动作很朴素：先有一个训练好的 codebook，输入一个 embedding 后，找离它最近的 codebook vector，然后用对应 token 表示它。这样做能把连续向量变成离散 token，但一定会留下误差。
RVQ 继续处理这个误差。第一层量化后，原 embedding 与选中 vector 之间有 residual；第二层再对 residual 找最近 token，后面层继续重复。层数越多，离散表示理论上越接近原 embedding，但结构也更复杂。
判断句：VQ 回答“第一步选哪个 token”，RVQ 回答“剩下的误差怎样逐层补”。


5. coarse 与 fine 不等于好坏
字幕里把第 0 层称为 VQ 层，后面的 residual 层从第 1 层往后排。Mimi 可用到 32 层，作业会让你观察第 0、6、16、31 层。通常越后面的层越 fine，补更细的 residual；前面的层更 coarse，可能承载较粗的语义或声学结构。
coarse/fine 不是“前层差、后层好”的简单排序。前层可能对内容轮廓更重要，后层可能让音色或细节更接近原音。作业的交错实验正是为了让你听出不同层换成 Alice 或 James 后，结果怎么变。
判断句：说某层“有用”时，要说明它对内容、音色、情绪、清晰度中的哪一项有影响。


6. UMAP 只帮你看趋势
Section 1 的第 6 题用情绪数据集。每个音档已有情绪标签，例如 amused、angry、disgusted、sleepy、neutral。流程是把音档送进 Mimi tokenizer，取某一层 token 对应的 embedding，再沿时间 frame 做平均，得到固定长度的向量。
UMAP 把这些高维向量投到二维。不同颜色的点如果容易分开，说明这一层的表示可能带有可区分的情绪信息；如果挤在一起，就不能轻易说它能分辨情绪。但 UMAP 不是精确分类器，二维图上的远近不能直接等同于原始高维距离。
判断句：UMAP 可以支持“看起来有聚类趋势”，不能单独支持“这一层必然能准确分类”。


7. Detokenizer、PESQ 与 transcript 分别看什么
Section 2 才把 tokens 解回音讯。第 7-8 题比较中文、英文、笑声、音乐的重建质量。语音质量带主观性，所以作业引入 PESQ：输入 reference audio 和 reconstructed audio，分数越高表示越接近原始音讯。
PESQ 不是全部答案。它能帮你减少纯听感的摇摆，但听感仍能发现一些指标没覆盖的现象。第 10 题还提供 Whisper transcript，因为交错 token 后，音色可能像某个人，但内容是否接近原句要靠 transcript 辅助判断。
判断句：PESQ 看音质相似，Whisper transcript 看文字内容，主观听感补充情绪、音色和自然度。


8. shuffle 和 split index 在检查层的功能
第 9 题把四层 tokens 随机打乱，你需要尝试不同排列再听重建结果。四层共有 4! = 24 种排列，最坏情况下要系统试完。只听两三个结果就下结论，很容易错过正确顺序。
第 10 题把 Alice 和 James 两段同长度音档都 token 化成八层，再按 split index 交错。例如前四层来自 Alice，后四层来自 James，然后 detokenizer 重建。红蓝层的切换让你观察前后层对说话人、内容、清晰度的影响。
判断句：shuffle 检查层顺序，split index 检查不同层携带哪类语音资讯。

## box
### 9. CSM 才是在生成新语音
前两段做的是 reconstruction：输入一段音讯，变成 token，再尽量还原同一段音讯。Section 3 在 tokenizer 和 detokenizer 中间加入 CSM 1B。它接收 text 和 audio prompt，输出新的语音，不只是把原音讯还原回来。
字幕里的示例是给 speaker one 和 speaker two 的 audio prompt，再给一句 text prompt，如让 speaker one 说 “Are you ok”。模型需要从 audio prompt 学到说话人的声音条件，再按 text prompt 生成没有直接出现过的新音讯。这就是 voice cloning 式的 speech generation。
判断句：reconstruction 问“原音能不能回来”，generation 问“给定条件能不能产生新的目标音讯”。

## box
### 10. Colab 的运行顺序很重要
Colab 先有环境准备：检查 GPU、安装套件、下载资料。Section 1 会载入 Mimi，并定义取得 Mimi tokens 的核心函数。字幕特别提醒，这个函数后面还会用到；如果前面没跑，Section 2 和 Section 3 会缺少必要对象。
CSM 还需要 Hugging Face 上的授权与 token。生成结果有随机性，偶尔不好听不代表机制错误；按作业要求多生成几次，再结合输出音档和题目选项判断。P21 表面上只是 12 道选择题，实际是在训练你把观察结果转成可解释的判断。
判断句：跑作业时先确认环境、Mimi 载入、取 token 函数、待填写代码格和输出观察都完成，再回去选答案。

## box info
### 完整链路
从头串起来：cascade speech model 会把语音先压成文字，容易丢失情绪和音色。Mimi codec 用 VQ/RVQ 把语音变成多层离散 tokens，并用 detokenizer 尝试重建。UMAP 帮你观察某层 token embedding 是否呈现情绪聚类，PESQ 和 Whisper transcript 帮你判断重建后的音质和内容。最后 CSM 作为 backbone speech LLM，在 speech token domain 里生成新的语音。读完这一段，你应该能拿到一个语音生成系统，判断它是 cascade 路线还是 speech token 路线，并说出可能保留或丢失哪些资讯。

# 最小例子 {#example}
<!-- tag: EXAMPLE -->

下面的伪代码不追求和 Colab 完全同名，只用来把三个核心动作讲清：取得 Mimi token、重建音讯、用 CSM 生成新语音。

示例 1：取得多层 speech tokenspython

```text
audio = load_audio("angry_sentence.wav", sample_rate=24000)
tokens = mimi_tokenizer.encode(audio, num_layers=8)

# tokens shape 类似 [layers, frames]
layer0 = tokens[0] # VQ / coarse token
layer7 = tokens[7] # later RVQ / fine detail
```

读这段代码时看两件事：`sample_rate=24000` 对应 Mimi 的输入规格，`[layers, frames]` 对应语音 token 的二维结构。删掉层维度，后面的 layer 分析就没法做。

示例 2：重建并用 PESQ 比较python

```text
reconstructed = mimi_detokenizer.decode(tokens)
score = pesq(reference_audio=audio, degraded_audio=reconstructed)

print(score) # 越高代表重建越接近 reference
```

这里的 reference 是 tokenizer 前的原始音讯，degraded 是 detokenizer 重建后的音讯。PESQ 高只能说明两者更接近，不能单独证明情绪、说话人和自然度都保住。

示例 3：CSM speech generationpython

```text
audio_prompt = [
 ("speaker_1", "unhappy", speaker1_audio),
 ("speaker_2", "and sad", speaker2_audio),
]
text_prompt = "speaker_1: Are you ok?"

new_audio = csm.generate(audio_prompt=audio_prompt, text_prompt=text_prompt)
```

CSM 这里根据 speaker 条件和文字条件生成一句新语音。判断它是否成功，要同时听声音是否像、内容是否对、结果是否自然。

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->
<!-- layout: grid2 -->

## box redsoft
### 常见坑
1. 把 speech token 当文字 token。错因：忽略 layer/frame 二维结构；后果：看不懂 Mimi 输出；修法：按时间 frame 和 RVQ layer 分开解释。
2. 以为 ASR 文本保留全部语音资讯。错因：只看语义内容；后果：忽略情绪、音色、笑声等副语言信息；修法：比较 cascade 和 speech token model。
3. UMAP 图过度解读。错因：把二维距离当精确距离；后果：从图上读出不存在的强结论；修法：只把它当观察聚类趋势的工具。
4. 只靠主观听感回答重建题。错因：忽略 PESQ / transcript；后果：不同人判断不一致；修法：客观指标为主，听感为辅。
5. 第 9 题只试几个排列就下结论。错因：没意识到四层有 24 种排列；后果：可能错过正确重建；修法：系统枚举或记录每次排列。
6. CSM 生成一次不好就判定失败。错因：忽略随机性；后果：误选答案；修法：多生成几次，结合题目要求观察。
7. 把 reconstruction 和 generation 混用。错因：没有看中间是否有 speech LLM 生成新 tokens；后果：把 Section 2 和 Section 3 的任务说反；修法：先判断输出是否是同一段音讯。
