P23 核心讲义

第1讲：生成式 AI 是什么

这一讲先把词摆正。人工智慧先当成一个目标，生成式 AI 关注的是复杂、有结构、不能靠穷举列完的对象。接下来就看三个层次怎么连起来：输出空间为什么大，机器学习怎样从资料里找函数参数，语言模型又怎样把整段生成拆成一连串下一个 token 的判断。

读完本页你应该能做什么

  • 先判断一个任务是有限选项选择，还是复杂对象生成。
  • 能把 f(x)=ax+b 这样的函数例子，翻成“参数、模型、训练资料、测试”的机器学习说法。
  • 能用“next-token 预测”解释为什么语言模型看起来在生成整段文字。
  • 能用一张状态台账说明训练和推论时读什么、写什么，哪里最容易出错。

先看这节课要解决什么

goals

这一讲只做三件事

  1. 把人工智慧、生成式 AI、机器学习、深度学习放回各自层次。
  2. 用一个粗算例子说明，为什么“生成一篇文章”和“小列表选答案”是两类问题。
  3. 把语言模型的生成过程拆成“预测下一个 token”的连续步骤。

现在你应该能做什么：先判断一个词是在讲目标、问题类型，还是解决手段。

边界先讲清

这页先不展开 Transformer 细节、训练技巧、多模态、RAG 和安全对齐。它们会在后面的课程里回来。这里先把最基本的判断框架站稳，否则后面每个新词都会漂。

先抓主线

overview

人工智慧是目标

在这一讲里，AI 先当成一个目标：让机器做出人们会认为有智能的事情。这个目标可以用很多手段达到。

生成式 AI 是任务类型

它关注的是生成复杂、有结构、无法穷举的对象。文章、图像、语音、翻译，都会碰到这种大输出空间。

机器学习是方法

机器学习把问题写成函数，再从资料里找参数。深度学习则是用神经网络来表示这种函数。

词先这样理解这节课里常见的判断句
人工智慧目标，用来描述要做成什么样的能力。先问它要展现哪种能力。
生成式 AI生成复杂结构对象的一类任务。先看输出是不是有限标签。
机器学习从资料中找函数参数。先看有没有“训练资料”这件事。
深度学习用神经网络做机器学习。先把神经网络当函数，不先神秘化。

规模感：为什么生成任务会变大

scale

先看一个粗算

如果每个位置大概有 1000 个常用字可选，100 个位置连起来，就是 1000^100。这是一个粗略数量级估算，用来建立感觉：组合空间的增长是指数级的。

有了这个感觉，很多判断就会变直观。比如“生成一篇 100 字文章”和“在两个标签里选一个答案”属于两类问题，前者要在极大的组合空间里维持结构，后者只是在有限选项里做选择。

三个常见大小

  • 分类：少量标签，空间小，重在边界清晰。
  • 翻译：输出字词很多，但还要符合目标语言结构。
  • 写作 / 画图：输出空间更大，还要兼顾内部一致性。

这就是为什么这节课一开始要从“复杂、有结构、不能穷举”说起。概念没摆正，后面的模型名词都会被误读成“越大越高级”。

核心机制

mechanism

从 f(x)=ax+b 看参数

中学函数里，a 和 b 是未知数。给几组输入输出之后，人可以反推它们。机器学习做的事就是把这类“找未知数”的过程自动化，只是实际模型远比直线复杂得多。

参数例子
# 资料告诉我们：
# x=4 -> y=5
# x=2 -> y=-1

def f(x):
    return 3 * x - 7

print(f(1))  # -4

1000100 = (103)100 = 10300

这句只想传达一个事实：生成一篇 100 字的东西，组合空间大到不能穷举。

训练、测试、推论分别做什么

阶段读什么写什么错了会怎样
训练训练资料更新参数模型学不到规律，或学到错误规律。
测试 / 推论新输入产出预测把训练样本拿来测试会高估效果。
模型当前参数输入到输出的映射把模型想成静态物件，会看不懂它怎么变。

这一步最重要的是方向：训练时参数会被改，测试时参数保持固定。很多评估错误都从这里开始。

现在你应该能做什么：看到一个系统时，分清它是在读资料找参数，还是在拿新输入做推论。

状态台账

ledger
对象进入这一阶段前这一阶段做了什么错了会出现什么现象
训练资料已经准备好提供输入输出对应关系资料少、资料错、覆盖不够，会直接带偏参数
模型参数未确定被训练过程调整参数没学好，测试时输出就会飘
测试输入训练时没见过喂给已训练模型如果把训练样本当测试样本，结果会虚高
next-token 输出只有前文预测下一步 token如果一步选错，错误会接回输入继续放大

这张台账的价值在于，它把“看不见的状态变化”说成了可以检查的对象。你不必先知道所有算法名词，只要知道哪一步读什么、写什么、会把什么状态改掉，就能先排出很多错误。

现在你应该能做什么：看到一个流程时，先标出它读的是什么、写的是什么、以及错误会从哪里冒出来。

文字接龙怎么把大问题拆小

语言模型按顺序预测下一个 token。把“下一步会出现什么”重复很多次，就会得到完整回复。

x 输入
台湾最高的山是哪座玉山
y 目标
最高的山是哪座玉山

你可以把它理解成：每一步都在做一个有限选择，但整串选择拼起来，就像在生成。

判断路线

route

第一步：先看输出

先看输出空间：有限标签就往分类去想，复杂对象就往生成去想。这个判断比“用了什么模型”更早。

第二步：再看训练

有没有训练资料？有没有参数要被找出来？如果有，这件事就已经进入机器学习的语言了。这里再去分深度学习，才有意义。

第三步：最后看生成策略

如果任务是生成，再问它是一次性输出，还是逐步预测下一个 token。一步一步接龙，才是这页真正要你抓住的生成策略。

输出空间 → 训练资料与参数 → 生成策略

这条路线的用途是检查顺序。先看输出，再看训练，再看生成方式，很多术语就不会互相串台。

操作桥：拿三个任务做一次完整判断

workflow

为什么要再做一次完整判断

前面已经把概念分开了。真正能用时，要把它们连成一条判断链：先看输出是什么，再看资料怎么来，最后看生成是一次完成还是一步一步来。

这三问看起来朴素，却能挡掉很多误判。比如，垃圾邮件侦测看起来也有输出，但它输出的是有限标签；中文翻英文看起来像换词，但它必须让目标语言成立；写短诗看起来像聊天，但它要从开放空间里不断组织新结构。三者的难点完全不同。

如果这一步没做对，后面所有模型名词都会被你放到错误的框里。你会以为“用了神经网络”就一定是生成，也会以为“输出很长”就一定比分类更高级。真正稳的判断，不靠气势，靠顺序。

同一条任务链，三种结果

任务输出空间训练资料生成方式常见错判
垃圾邮件侦测有限标签邮件 + 标签选一个类别把“有输出”误当“生成”
中文翻英文大而有结构双语对齐语料逐 token 生成把翻译误当字词替换
写短诗更大、更开放诗歌或文本语料逐 token 生成只盯风格，不看结构
判断伪代码
def classify_task(output_space, has_training_data, generation_step):
    if output_space == "finite_labels":
        return "classification"
    if has_training_data and generation_step == "next_token":
        return "generation"
    return "needs_more_context"

print(classify_task("finite_labels", True, "pick_one"))
print(classify_task("structured_text", True, "next_token"))

判断错了，后面会怎么坏

  • 把分类当生成，训练、评估、解释都会说偏，后面再看模型能力会失真。
  • 把生成当分类，会低估输出空间，误以为模型只是在“多几个标签里选”。
  • 把翻译当成单纯替换词，会漏掉语法、语序和目标语言的结构约束。

把它变成你自己的检查表

以后碰到一个新任务，就按这三步写一张最小判断表：第一列写输出空间，第二列写训练资料，第三列写生成方式。只要这张表能填出来，你就已经比“先猜模型名”更接近问题本身。

这张表也能帮你复习别的章节。以后看到评估、对齐、图像生成、工具调用，你还是可以先问这三件事，只是“输出空间”会换成结构、状态、资源或者权限。

现在你应该能做什么：面对一个新任务，先写出判断表，再决定它更像分类、翻译还是生成。

从课堂句子读出判断轴

reading

把一句话拆成三层

这一讲里有不少句子在给你判断轴。比如“生成 100 字文章不能靠穷举”这类话，真正要你抓的是组合空间；“一步预测下一个 token”真正要你抓的是顺序生成；“AI 是目标”真正要你抓的是任务边界，而不是某个具体技术名词。

如果你把这些句子当成口号，就会很快忘；如果你把它们当成判断轴，就能拿去读别的章节。你会发现，后面的评估、对齐、图像生成、工具调用，虽然场景变了，判断顺序还是差不多：先问输出是什么，再问过程怎么跑，再问错误会在哪一步冒出来。

这页把定义和判断轴放在一起，重点是让你能把一句话拆成可检查的信号。只要这个动作做顺了，你就能自己把课程里的句子变成判断表，而不是死记术语。

句子 -> 判断表

听到的句子先抓的信号应该落到哪里如果抓错
生成 100 字文章不能靠穷举组合空间规模感 / 生成把生成误看成小选择
模型要从资料里找参数训练关系机器学习只记模型名，不知道参数从哪来
一步预测下一个 token顺序生成语言模型 / 自回归以为它一次产出完整答案
AI 是目标任务边界概念层把目标和方法混在一起
拆句小工具
def read_sentence(sentence):
    if "穷举" in sentence or "组合" in sentence:
        return "scale / generation"
    if "找参数" in sentence or "训练" in sentence:
        return "machine_learning"
    if "token" in sentence or "接龙" in sentence:
        return "language_model"
    if "AI" in sentence:
        return "task_boundary"
    return "needs_manual_reading"

print(read_sentence("生成 100 字文章不能靠穷举"))
print(read_sentence("一步预测下一个 token"))

如果把句子读错，会连着坏什么

你会把判断落在错误层次上，后续的例子、练习和评估方式也会跟着错。最常见的后果是：本来应该先写输出空间，你却先写模型名；本来应该先讲训练关系，你却先讲架构细节。这样看似讲了很多，实际上没有把问题闭合。

这一页后面的练习已经在防这个错，但最省力的做法，是你自己先把句子拆成判断轴，再决定该往哪一层放。

练习入口

你可以从课程里的任意一句话开始练，例如“生成一篇文章”“用文字接龙生成回复”“训练资料帮模型找参数”。每次只做一件事：先圈出输出空间，再写一句判断，然后再去想下一步该看什么。这个动作看起来小，但它能把阅读、记忆和做题连起来。

现在你应该能做什么：把一条课堂句子拆成信号、归类、判断和后续动作四步。

排错示例：把任务放回正确层次

diagnosis

一个真实会发生的误判

假设你在做课程整理，看到一段系统说明：“输入一封邮件，输出垃圾 / 正常。”如果只看“输入”和“输出”这两个词，很容易把它放进生成式 AI。把输出空间写清楚以后，任务就很明确：这是有限标签选择。

这类误判会在后面引发连锁反应。你会开始期待模型产生长解释，期待训练数据像作文语料一样组织，甚至会把评估指标也写错。这个任务更像分类，重点在边界是否分得清，而不是是否会写。

这节课的判断重点是输出空间怎么长出来的。一旦这个问题答错，后面所有对模型能力的描述都会跟着偏。

追踪表：从任务到症状

环节应该看到什么错判后会看到什么怎么改回来
任务定义有限标签或大结构对象把“有输出”当成生成先写输出空间，再谈模型
训练资料邮件 + 标签、双语语料、文本语料把所有任务都想成同一种数据回到任务类型重新选语料
输出方式选类别或逐 token 生成把所有输出都解释成“创造”分清选择和生成两条路线
评估症状分类看准确率，生成看结构和可读性指标写在错误层次上先决定任务，再决定指标
把判断写成代码
def route_task(task):
    if task.output_space == "finite_labels":
        return {
            "route": "classification",
            "check": "accuracy",
            "warning": "不要用生成式标准去要求它写长解释",
        }
    if task.output_space == "structured_text":
        return {
            "route": "generation",
            "check": "fluency + structure",
            "warning": "不要把它误当成标签选择问题",
        }
    return {
        "route": "needs_more_description",
        "check": "clarify task boundary",
        "warning": "先补任务定义，再选模型",
    }

如果还把它判错，会继续坏在哪里

  • 评估指标会错位，生成任务被拿准确率去量，分类任务被拿长文本质量去量。
  • 训练语料会选错，任务本来只需标签，结果却被当成长文本学习。
  • 解释方式会错位，后来你会用同一套说法去解释翻译、分类、写作，所有边界都被磨平。

把排错变成练习

你可以拿身边的三个例子来练：一个邮件分类器、一个翻译器、一个写作助手。先给每个任务写一张表，表里只放四列：输出空间、训练资料、输出方式、评估方式。只要你能填出这四列，就说明你已经把任务放回了正确层次。

这一段和前面的判断路线配合起来，就能把“看见一个任务就乱猜”变成“先归类，再决定怎么学、怎么评估、怎么解释”。

现在你应该能做什么：拿一个任务说明，先写出错判会从哪里开始，再写出该怎么把它拉回正确层次。

失败演练：把翻译任务放回正确层次

failure-lab

具体案例：一句中文翻成英文

把“我今天要交作业”翻成英文时，系统要输出一句符合英文语法的句子。它要一步步组织词序、时态和搭配。这个任务和垃圾邮件侦测都带有输入和输出，但处理对象不同。

如果你把翻译写成标签选择，输出就会被压成“正确 / 错误”一类的结果，根本看不见语法和语序；如果你把它理解成整句一次成型，你又会忽略它背后其实是一串逐 token 的预测。真正有用的看法，是把翻译放进生成式任务里，再看它生成的每一步要读什么、写什么。

步骤读什么写什么如果这一步错了
输入层原句含义翻译目标会把句意翻歪
组织层上下文和语法词序与搭配会生成像字典拼贴的句子
输出层前文已经生成的 token下一个 token会把错误继续接回去

一个小代码桥

逐步翻译伪代码
def translate_step(prompt, prefix):
    next_token = model.predict_next(prompt + prefix)
    return prefix + next_token

prompt = "我今天要交作业"
prefix = ""
for _ in range(4):
    prefix = translate_step(prompt, prefix)
print(prefix)

这段伪代码的重点是判断顺序：每次都要拿“原句 + 已生成前缀”去预测下一步。它说明翻译和文字接龙共享同一种节奏，只是输入语言和目标语言不同。

如果你把它误判成分类，第一步就会把问题缩成有限标签，后面永远不会得到真正的句子；如果你把它误判成一次性输出，你又看不见前缀如何影响后续 token。两种错法都会让你在练习里写出很空的解释。

失败后果

错层次的后果很具体：评估会用错指标，训练资料会选错形式，解释时也会把翻译说成“字词替换”。到了真正要复述课程的时候，你会发现自己能背一些词，却说不清楚为什么翻译和分类分属不同问题。

练习接点

你现在可以直接拿三种任务各写一句：垃圾邮件侦测、中文翻英文、写短诗。每句都先标输出空间，再写生成方式，再补一句可能的失败后果。这样写一轮，你会更容易看出这一讲真正关心的是“怎么判断任务”。

现在你应该能做什么：看到翻译任务时，先把它放回生成链，再去看词序、语法和前缀如何影响后续 token。

常见误区

mistakes

先别掉进这五个坑

  1. 把 AI 视作一个目标。
  2. 只看“有输出”，不看输出空间。
  3. 把神经网络先想成人脑，反而看不见参数和函数。
  4. 把训练和测试混在一起，评估就失去意义。
  5. 把语言模型看成一步步预测下一个 token 的过程。

为什么这些坑会影响后面

如果你把分类和生成混在一起，后面看生成策略就会失焦；如果你把训练和测试混在一起，后面看评估就会乱；如果你把神经网络神秘化，后面学 Transformer 只会记词，不会看机制。

现在你应该能做什么：听到一个任务时，先检查它的输出空间、训练方式和生成方式，再决定它属于哪一类。

历史线索

history

不是今天才出现

这页强调的生成式思路并不是近几年才有。翻译本身就是很早就存在的生成任务，结构化学习也早就尝试过把复杂输出拆成可学习的步骤。今天真正变化的是模型规模、训练方式和应用体验。

所以这里有个很实用的区分：概念可以早就存在，流行度和能力可以在某个阶段突然上来。把这两件事混在一起，会误以为“生成式 AI”是一个新造出来的词。

3. 用一句话区分生成和分类。

参考答案要抓“输出空间”这个判断轴。

参考答案 / 评分要点

分类是在有限选项里选一个答案；生成是在大得多、带结构的输出空间里创造对象。要点是把“有限选项 / 复杂结构”说出来，不要只说“一个是选，一个是写”。

解释数量级

4. 为什么 100 字文章不能靠穷举解决？

把“组合空间太大”说清楚，最好顺带提到 10^300 这个数量级。

参考答案 / 评分要点

粗略假设 100 个位置、每个位置有 1000 个常用字可选，组合数就是 1000^100，也就是 10^300。这个数量级远大于能穷举的范围，所以生成不是“列出所有候选再挑一个”这么简单。

参考答案 / 评分要点

训练阶段读训练资料并更新参数；测试和推论阶段参数应保持固定，只把新输入送进模型看输出。流程上可以说成：资料进入训练，参数被写入模型，测试时再用这组参数处理新输入。

复述机制

20. 用 5 到 7 句话解释 next-token 预测为什么能生成整段文字。

参考答案 / 评分要点

一段好答案要讲出：语言模型每次只预测下一个 token；每一步面对的是有限候选；预测出的 token 会接回输入，成为下一步的上下文；连续很多步以后，就能拼出整段回复。最后最好补一句，错误也会被接回去，影响后续生成。
