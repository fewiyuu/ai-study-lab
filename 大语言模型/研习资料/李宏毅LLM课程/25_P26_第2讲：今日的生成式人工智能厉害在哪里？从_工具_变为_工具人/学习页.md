# 概念骨架 {#skeleton}
<!-- tag: MAP -->

旧形态：工具工具有明确固定功能。翻译系统只负责翻译，评估时主要看翻译质量。
新形态：工具人通才模型没有固定功能，必须靠指令指定任务。它可以翻译、写代码、读文件、画图、搜索、客制化。
新问题：边界能力越全面，越难评估，也越需要安全防御、提示设计和模型调整方法。

| 比较项 | 单功能工具 | 通才工具人 | 学习判断 |
| --- | --- | --- | --- |
| 功能 | 固定，如翻译 | 由 prompt 决定，如翻译、写作、分析、执行代码 | 先写清任务，再期待输出 |
| 输入 | 通常是特定格式 | 自然语言、文件、图片、工具结果等 | 输入质量影响很大 |
| 评估 | 单指标或窄任务 | 多任务、多标准、用户偏好不一致 | 不要只看某个榜单 |
| 风险 | 较集中 | 幻觉、有害内容、越狱、偏见、过度拒答 | 有用性和安全性要一起看 |
| 改进方式 | 改工具或换工具 | 改 prompt、给资料、用工具、微调模型 | 先改输入，必要时改模型 |

# 术语表 {#terms}
<!-- tag: TERMS -->

| 术语 | 直觉解释 | 准确说法 | 常见误解 |
| --- | --- | --- | --- |
| 工具 | 只做一类事的系统。 | 功能固定、输入输出范围窄的应用，例如传统翻译系统。 | 以为所有 AI 都应该先问“它有哪些按钮”。 |
| 工具人 | 能按指令帮你做很多事的助手。 | 课程里的比喻，指无固定单一功能、可按 prompt 执行多种任务的生成式 AI。 | 以为它真的像人一样理解和负责。 |
| 通才模型 | 什么都能试着做一点。 | 跨任务、跨输入类型、可组合工具的模型或系统。 | 以为通才就等于所有任务都很强。 |
| GPT-3.5 | 较基础的 ChatGPT 版本。 | 能对话和生成文字，也能写代码，但工具和多模态能力有限。 | 以为试过 3.5 就能断言所有 LLM 做不到。 |
| GPT-4 | 功能更全面的代表。 | 课程示例中可读文件、读图、搜索、写并执行代码、画图、用工具、客制化。 | 以为它没有边界或永远正确。 |
| 客制化 | 让模型带上专属资料或角色。 | 通过 GPTs 等方式设定用途、说明、知识文件和工具，让通用模型服务特定场景。 | 以为上传资料后就不需要验收。 |
| 能力评估 | 判断模型到底会什么。 | 对通才模型进行多任务、多维度、多偏好的系统测试。 | 拿一个任务超过某模型，就说全面超过。 |
| 幻觉 | 模型努力回答时说错或编造。 | 生成看似合理但事实不可靠的内容。完全拒答可避免犯错，但也失去有用性。 | 以为零幻觉模型就是最好模型。 |
| 安全防御 | 防止模型输出不该输出的内容。 | 让模型拒绝脏话、歧视、违法、有害请求等，但防御可能被绕过，也可能过度保守。 | 以为加了防御就不会被诱导。 |
| prompt engineering | 设计输入，让模型给出更接近目标的输出。 | 通过清楚指令、格式约束、额外资讯、例子等巧妙设计 prompt。 | 以为只是背咒语。 |
| 开源模型 | 可以拿到并调整的模型。 | 例如 Llama 等，使用者可以在一定条件下调整参数或微调。 | 以为开源就一定容易训练。 |
| 参数调整 | 直接改变模型内部。 | 通过训练或微调改变模型参数，使同样输入得到更符合需求的输出。 | 以为只会修好目标问题，不会引入副作用。 |

现在你应该能做什么：判断一个 AI 系统是在“固定功能工具”模式，还是“通才助手”模式。

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 1. 旧生成式 AI 像工具
Google 翻译也可以算生成式 AI，因为它输入一段文字，输出另一种语言的文字。但它的功能非常明确：翻译。你不需要告诉它“请翻译”，因为整个系统的目的就是翻译。
这种系统像工具。工具有固定用途，评估也相对明确：翻译好不好、错词多不多、语序顺不顺。用户对它的期待范围比较窄，所以系统边界也比较清楚。
现在你应该能做什么：用 Google 翻译解释什么是单功能生成式 AI。

## box
### 2. 新生成式 AI 像工具人
ChatGPT 也能翻译，但你只输入一句中文，它不会自动翻译，因为它不知道你的任务是什么。你必须说“请把以下句子翻译成英文”，它才会把同一段模型能力用于翻译。
这就是通才模型的新形态：没有固定单一功能，任务由 prompt 决定。它可以帮你写作业、写报告、写代码、读文件、读图片、画图、搜索和客制化。它更像“能听指令处理任务的助手”，而不是一个只有一个按钮的工具。
现在你应该能做什么：解释为什么“你想让它做什么”比“它能做什么”更接近正确问法。

3. ChatGPT 是代表，不是生成式 AI 全部课程列出 Gemini、Microsoft Copilot、Anthropic Claude 等模型或系统，说明通才化不是 ChatGPT 独有现象。ChatGPT 只是目前最具代表性的例子。更准确地说，ChatGPT 是语言模型应用的一种；语言模型又只是生成式 AI 的一种。图像、语音、视频生成也属于生成式 AI。用 ChatGPT 举例是为了说明趋势，不是把全部生成式 AI 缩成一个产品。现在你应该能做什么：把产品名、语言模型、生成式 AI 三层分开。
4. GPT-4 展示了“工具人”的全面性课程对比 GPT-3.5 和 GPT-4：GPT-4 可以读文件、读图片、做网络搜索、写代码并执行、画图、调用工具、客制化 GPTs。文字云例子很典型：模型先列能力，再写代码生成文字云，发现中文字体显示失败后解释原因，再根据用户上传的中文字体文件修正。这个例子展示的是“任务闭环”：理解需求、生成代码、执行代码、观察错误、解释原因、请求或使用额外文件、修正结果。它不再只是生成一段文字，而是在多个能力之间切换。现在你应该能做什么：把 GPT-4 的能力理解成多工具协作，而不是“更会聊天”。
5. 但全面能力不等于无所不能老师提醒：不要试了 GPT-3.5 就断言 AI 做不到某事，最好先试更强模型。但这不代表 GPT-4 就是神。它依然可能失败、误判、受上下文限制、受工具限制，也可能在复杂任务中给出不可靠结果。正确态度是：扩大想象空间，但保留验收意识。通才模型值得尝试，但每个任务仍要看输入、工具、评价标准、风险和失败后果。现在你应该能做什么：既不低估强模型，也不把强模型当保证正确。
6. 通才能力带来“模型在想什么”的研究问题当模型能处理很多任务，人们自然会问：它内部到底表示了什么？课程提到研究者分析 Llama 心中世界地名的位置关系，发现模型对一些地理结构有相当程度的内部表示。但对台湾细粒度地名，普通 Llama 表现混乱；读过更多繁体中文资料的台德模型可能更了解台湾。这说明模型“知道什么”与训练资料有关，也说明模型内部表示可以被研究、比较和解释。现在你应该能做什么：理解可解释性问题为什么随着通才模型变重要。
7. 通才模型很难评估翻译工具只需评估翻译。通才模型不同，用户可能要求它做任何奇怪的事。课程用“请说哈哈哈 100 次”比较 Gemini、台德、GPT-3.5：一个努力做但超过次数，一个做了一半觉得没意义，一个直接拒绝。哪一个最好？没有标准答案。有人觉得努力执行最好，有人觉得拒绝无意义任务最好，有人觉得适度完成再提醒最好。通才模型评估困难，就在于“好”的定义会随任务、用户和价值偏好变化。现在你应该能做什么：看到榜单或宣传时，问它测了哪些任务、哪些标准。
8. 不犯错和有用之间有张力如果模型想完全不犯错，最简单的方法是永远拒答：“我无法回答”。这样几乎没有幻觉，但也几乎没用。模型会犯错，往往是因为它尝试帮助你解决问题。这不代表我们要容忍错误，而是要理解评估时不能只追求零错误。一个有用模型需要在“愿意尝试”和“知道何时拒绝”之间取得平衡。过度自信会危险，过度拒绝也会失去价值。现在你应该能做什么：把幻觉问题放进有用性和安全性的平衡里看。

## box info
### 9. 安全防御也变复杂
生成式 AI 不再从固定选项中选择，它可以生成任何文本，因此可能输出抄袭、脏话、歧视、违法建议等有害内容。模型通常有防御能力，会拒绝直接不当请求。
但防御可能被绕过。课程示例中，直接让 GPT-3.5 说脏话会被拒绝；让它扮演“喜欢说脏话的乡民”，就可能绕过限制。另一方面，模型为了政治正确可能过度保守，甚至在图像生成中产生新的争议。安全持续对抗和取舍。
现在你应该能做什么：解释为什么通才模型的安全比单功能工具更难。

## box
### 10. 人还能做什么：两条路线
当工具人能做很多事，人类下一步有两条路线。第一条是改变自己：模型参数固定时，通过更好的 prompt、额外资料、格式约束和问题拆解，让同一个模型给出更好的输出。这就是 prompt engineering。
第二条是训练自己的模型：如果现有模型无法满足需求，可以用开源模型或已有模型做微调，调整参数，让同样输入得到更合适输出。但调整参数像给模型做手术，可能解决一个问题，也可能引入副作用。
现在你应该能做什么：先判断应该改输入，还是已经需要改模型。

11. prompt engineering 不是背咒语prompt 是给模型的输入。engineering 的重点是巧妙设计：你要告诉模型任务、目标、限制、背景和输出格式。模型作为固定函数，同一个输入给同一个模型会有相对固定的行为；改变输入，就可能改变输出。把 prompt engineering 说成“人与 AI 沟通的艺术”有道理，但不要神秘化。它本质上是把需求写得更清楚、把上下文补得更完整、把评价标准嵌进输入。现在你应该能做什么：把“帮我做一下”改成含任务、资料、格式、限制的 prompt。
12. 训练模型是更重的选择如果你无法通过 prompt 达成目标，或者需要长期、稳定、领域化行为，就可能考虑训练或微调自己的模型。开源模型如 Llama 让这条路成为可能。但调参不是无风险。模型是巨大函数，改变某部分参数可能影响许多看不见的能力。你以为修好了某个行为，可能让别的任务退化。这也是为什么后续课程会专门讲训练、微调和遗忘问题。现在你应该能做什么：说明为什么“直接训练自己的模型”不是默认第一步。
完整知识链路这一讲把生成式 AI 的爆火解释为形态变化：从 Google 翻译这类单功能工具，转向 ChatGPT / GPT-4 这类无固定功能的通才助手。通才模型能读文件、读图片、写并执行代码、画图、搜索和客制化，但也因为能力太广而难评估、难防御、难保证不犯错。用户要学会两种应对：先通过 prompt engineering 改变输入、提供资料和约束；如果仍不够，再考虑训练或微调模型。你现在应该能做什么：面对一个 AI 任务，先问“我能否通过更好的输入让现有模型完成”，再问“是否值得改模型”。

# 最小例子 {#example}
<!-- tag: EXAMPLE -->

这一讲的最小例子是：同一个模型，不同 prompt 会让它表现得像不同工具。

示例 1：同一句输入，任务不明确text

```text
输入：
這份作業今天截止，我還沒開始。
```

如果只给这句话，通才模型不知道你要翻译、改写、安慰、规划还是写邮件。它可能自由猜测任务。单功能翻译工具不会有这个问题，因为功能已经固定。

示例 2：把通才模型指定成翻译工具text

```text
请把下面这句话翻译成自然英文。
要求：
- 保留焦虑但口语的语气
- 不要扩写

句子：
這份作業今天截止，我還沒開始。
```

加入任务、语言、语气和限制后，模型才知道自己现在要扮演“翻译工具”。这就是从通才能力中取出具体功能。

示例 3：把通才模型指定成计划助理text

```text
我有一份作业今天截止，还没开始。
请帮我做 3 小时内的执行计划。
输出格式：
1. 先列必须完成的最小交付
2. 再列 30 分钟一个阶段的时间表
3. 最后列提交前检查清单
```

同一句原始信息，换一个 prompt，模型就变成计划助理。工具人同一个基础模型可以被 prompt 引导到不同任务。

错误示例：只看单项评测就宣布全面超过diagnostic

```text
某模型在“哈哈哈 100 次”任务上比 GPT-3.5 更听话，
所以它已经全面超过 GPT-3.5。
```

错因是把单任务表现当成全面能力。后果是误判模型适用范围。修法是看多任务评估、任务分布、失败案例和用户偏好。

现在你应该能做什么：把一个模糊需求改写成明确 prompt，并说明它让模型扮演了哪个“工具”。

精修直觉：工具到工具人，是控制方式变了传统工具的功能写在产品里，用户只给材料；通才模型的功能常写在 prompt 里，用户要同时给材料、目标和限制。它需要你把当前功能临时定义出来。输入越像任务说明书，模型越像具体工具；输入越像一句情绪，模型越容易猜错任务。
术语快辨：改输入 vs 改模型改输入是 prompt engineering：不动模型参数，通过指令、资料、范例、格式约束改变输出。改模型是训练或微调：直接改变参数，让模型长期带上新行为。前者轻、快、可回退；后者重、贵、要评估副作用。默认先改输入，再考虑改模型。
Worked example：同一句话三种任务原句“这份作业今天截止，我还没开始。”如果 prompt 是“翻译成英文”，模型扮演翻译工具；如果 prompt 是“制定三小时计划”，它扮演计划助理；如果 prompt 是“写一封请求延期邮件”，它扮演写作助理。结论：通才模型的能力不是自动展开的，任务边界由 prompt 显式指定。

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->
<!-- layout: grid2 -->

## box redsoft
### 常见坑
1. 把 ChatGPT 当成生成式 AI 全部。错因：混淆产品和技术类别；后果：忽略其他模型和模态；修法：分清产品、语言模型、生成式 AI。
2. 问“它能做什么”后就停住。错因：仍把它当固定工具；后果：错过通才用法；修法：先说你想让它完成什么任务。
3. 试过 GPT-3.5 就断言强模型做不到。错因：忽略版本差异；后果：低估现有能力；修法：用更强模型或工具链验证。
4. 单任务超过就说全面超过。错因：评估维度太窄；后果：误选模型；修法：看多任务、多标准和失败案例。
5. 认为模型拒答越多越安全越好。错因：只看零错误；后果：模型失去有用性；修法：平衡愿意尝试和知道拒绝。
6. 把 prompt engineering 当咒语。错因：不理解输入设计；后果：只背模板不解决问题；修法：写清任务、资料、限制和格式。
7. 轻率微调模型。错因：以为改参数只会修目标问题；后果：引入能力退化或副作用；修法：先改输入，必要时再训练并评估。

### 易混概念
概念 A概念 B区别工具工具人前者功能固定，后者由 prompt 指定任务。ChatGPT语言模型前者是应用，后者是底层模型类别。语言模型生成式 AI语言模型主要处理文字，生成式 AI 包含文字、图像、语音等。能力全面所有任务都强全面表示可尝试很多任务，不代表每个任务都可靠。改 prompt改参数前者改变输入，后者改变模型内部。安全拒答过度拒答前者避免有害输出，后者可能让模型无法完成合理任务。
