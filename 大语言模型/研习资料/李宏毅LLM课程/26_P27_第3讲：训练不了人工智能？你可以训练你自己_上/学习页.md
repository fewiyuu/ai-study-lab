# 概念骨架 {#skeleton}
<!-- tag: MAP -->

模型不变参数没有更新，模型本身没有被训练。你能动的是输入：任务描述、上下文、资料、范例、输出格式和检查要求。
输入变清楚咒语可能让模型更认真推理；前提和资料让它少猜；范例让它知道任务规则和输出样子。
效果有边界同一句 prompt 对旧模型和新模型可能效果不同。in-context learning 只存在于本轮上下文，不会把知识写进参数。

| 策略 | 解决什么问题 | 典型做法 | 边界判断 |
| --- | --- | --- | --- |
| Prompt trick | 模型没有展开推理、没有解释、没有按预期努力。 | 要求 step by step、先解释再给结论、强调任务重要性。 | 不是万能，对模型版本和任务很敏感。 |
| 自动找咒语 | 人不知道哪种提示最能控制目标模型。 | 用强化学习或让模型自己提出 candidate，再在任务上测试。 | 人看不懂的咒语多对旧模型有效，强模型可能只会困惑。 |
| 补前提 | 问题有歧义，模型不知道语境。 | 说明身份、地区、目标读者、判断标准，例如“从台湾人的角度”。 | 前提给错会把模型带向错答案。 |
| 补资料 | 模型缺事实知识或资料过时。 | 把论文、文件、表格、规则放进上下文，让模型基于资料整理。 | 它能用资料，不代表永久记住资料。 |
| 给范例 | 模型不知道任务格式、标签含义或特殊规则。 | few-shot 示例、类别定义、输入输出样例。 | 强模型更可能跟随范例；弱模型可能只学到选项表面。 |

# 术语表 {#terms}
<!-- tag: TERMS -->

| 术语 | 直觉解释 | 准确说法 | 常见误解 |
| --- | --- | --- | --- |
| prompt | 你交给模型的全部输入。 | 包括任务指令、背景、资料、范例、格式要求和限制条件。 | 以为 prompt 只是一句命令。 |
| 新人助理 | 有能力，但不了解你的人。 | 课程中的使用模型直觉：LLM 有一般知识，却不知道你的私有背景和隐含前提。 | 以为模型答错就一定是智力不够。 |
| Chain of Thought | 叫模型一步步想。 | 通过类似 Let's think step by step 的提示，让模型生成中间推理过程后再回答。 | 以为这句话对所有模型、所有任务都强。 |
| rationale / explanation | 先说明理由，再给答案。 | 要求模型解释评分、判断或分类理由，可能提高某些任务与人类判断的一致性。 | 以为解释写得长就一定正确。 |
| emotional prompt | 给模型情绪压力。 | 例如告诉模型“这件事对我的职业生涯很重要”，在一些论文实验中会影响准确率。 | 以为可以稳定压榨出更强能力。 |
| principled instruction | 被系统测试过的提示原则。 | 研究者验证网络上流传的提示传说，例如直接说要做什么通常比说不要做什么更有效。 | 以为礼貌、小费、处罚等都是普遍定律。 |
| context | 模型本轮能看到的文字。 | 当前输入窗口里的所有内容，包含对话历史、资料、范例和问题。 | 以为模型看过一次就永久会了。 |
| 前提 | 答案成立的语境。 | 身份、地区、目标读者、定义方式、评价标准等影响回答的条件。 | 不给前提却要求模型猜中自己的意思。 |
| zero-shot | 不给范例，直接做。 | 只凭任务指令要求模型完成新输入。 | 以为没有范例就一定不能做。 |
| few-shot | 先给几个样例。 | 在 prompt 中提供若干输入输出范例，让模型按同样规则处理新样本。 | 以为范例越多越好，不检查一致性。 |
| in-context learning | 在当前上下文里临时学。 | 模型参数不变，只根据输入里的范例、资料和规则改变本轮输出。 | 把它误解成真正训练或微调。 |
| model parameter | 模型内部的权重。 | 训练会改变参数；本节所有方法都不改变参数。 | 以为模型读过教科书后参数就被改写了。 |

现在你应该能做什么：看到一个改 prompt 的技巧时，先判断它是在补推理过程、补前提、补资料，还是补范例。

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 1. 这节课为什么叫“训练你自己”
老师开场先提醒：本节没有任何模型被训练，也不是教某个特定任务的固定 prompt 模板。原因很直接：今天的语言模型已经很会理解自然语言，你通常不需要记一套神秘格式，只要把需求讲到另一个人能看懂，模型也多半有机会看懂。
真正需要训练的是使用者。你要训练自己识别模型缺什么：它是没展开推理，还是不知道你的前提？它是缺事实资料，还是不知道输出格式？如果缺的是这些输入信息，就不该立刻说“模型不行”，而应该先把上下文补齐。
现在你应该能做什么：把“写一个好 prompt”理解成任务澄清，不是背咒语。

## box soft
### 2. 在线新人助理：这节课最重要的心法
把 LLM 想成在线新人助理很有用。它“像人”，因为拥有一般知识、语言理解能力和一定推理能力；它“新人”，因为第一次认识你，不知道你的出生年月、偏好、文件内容、课程要求、地区语境，也不知道你脑中没有说出口的条件。
所以很多失败上下文失败。你问“NTU 是什么缩写”，模型可能默认南洋理工大学；你从台湾大学语境问，它才会回答国立台湾大学。新人助理不是读心术系统，你不交代前提，它只能根据世界上的常见分布去猜。
现在你应该能做什么：当模型答非所问时，先问“我有没有把新人需要知道的背景说出来”。

3. 神奇咒语：CoT 为什么让人兴奋Chain of Thought 的典型写法是 `Let's think step by step`。在较早模型上，它能显著提高数学题正确率：直接回答可能只有较低正确率，要求模型一步步思考后会明显改善。直觉上，这是让模型把中间推理写出来，减少直接跳到答案时的错误。但你不能把 CoT 当护身符。新版本模型可能即使没被要求也会“认真想”，所以同一句提示带来的增益会变小。课程中提到，旧版本 GPT-3.5 加 step-by-step 进步明显，新版本本身正确率已经提高，再加提示只剩小幅改善。现在你应该能做什么：遇到多步推理任务时可以试 CoT，但要用实际样本验证效果。
4. 解释答案：让模型先给理由另一种提示是要求模型解释自己的答案。课程用作文批改举例：如果模型先说明文章好在哪里、不好在哪里，再给分数，评分结果可能更接近人类老师。这里的关键把判断依据显式化，让输出不只是一个孤零零的分数。解释也有边界。较早研究发现，让 GPT-3 或更早模型解释并不一定提高准确率。解释写得像样，不代表理由真实可靠；它可能只是把错误答案包装得更有说服力。因此解释提示适合搭配验收标准，而不是替代验收。现在你应该能做什么：要求模型输出“理由 + 结论”，再检查理由是否真的支持结论。
5. 情绪勒索和提示传说课程提到有论文测试“这件事对我的职业生涯很重要”这类情绪提示，在一些模型和任务上会提升表现。还有研究系统验证网络上的提示传说：礼貌未必有用；直接告诉模型“要做什么”通常比说“不要做什么”更有效；小费、处罚、无偏见要求等可能影响输出。这些结果很有趣，但不能变成迷信。它们说明模型对输入文字很敏感，却不说明存在一套永远有效的咒语。你要把这些提示当成可测试的候选策略，而不是当成定律。现在你应该能做什么：看到“万能 prompt”时，问它在哪个模型、哪个任务、哪个版本上测过。
6. 自动找咒语：让模型或强化学习帮你试如果不知道哪句提示最好，可以更系统地找。课程举例：用强化学习训练另一个语言模型，让它专门生成 prompt，并用目标任务的反馈来更新它。实验目标是让 GPT-3 回答更长，结果找到人类看不懂的“喂喂喂……”式咒语，让平均回答长度超过直接要求“回答长一点”。这说明 prompt 也可以被优化和排名。同一个任务、同一个模型，不同 prompt 会产生差异。今天也可以直接问强模型：“如果我要你解数学题，什么提示会让你表现更好？”再把候选提示逐个测试。现在你应该能做什么：把 prompt 当成可实验对象，而不是一次写完就认命。
7. 人类看不懂的咒语，对强模型未必有用“喂喂喂”这种提示对 GPT-3 有效，不代表对 GPT-3.5 之后的模型有效。更强的聊天模型更像正常对话对象，你给它乱码，它可能只会说看不懂。模型越接近人类交流习惯，越不容易被奇怪 token 稳定操控。这也解释为什么“咒语文化”会逐渐降温。早期模型更脆弱，某些提示会像暗门；后来的模型本身更会推理、更会遵循任务，提示工程就从找暗号，转向清楚说明目标、资料、格式和评价标准。现在你应该能做什么：优先写人能看懂的 prompt，再考虑特殊技巧。
8. 补前提：NTU 为什么会答成两种学校模型回答错误，有时只是因为问题本身有歧义。NTU 可以是台湾大学，也可以是南洋理工大学。你不说语境，模型会按它认为更常见的语境回答。你说“从台湾人的角度”，答案就变成国立台湾大学；你说“从新加坡人的角度”，答案就变成南洋理工大学。前提不是装饰，它决定答案空间。地区、身份、读者、行业、课程规则、公司内部定义都会影响正确答案。对新人助理来说，前提就是任务入口。现在你应该能做什么：把模糊问题改写成带身份、地区或标准的问题。
9. 补资料：模型不会的事实，可以放进上下文语言模型不是无所不知。课程让 GPT-4 整理 GPT-1、GPT-2、GPT-3 的参数量和训练资料，参数量较容易答，但训练资料细节可能缺失。解决方法把相关论文直接给它读。当 GPT-4 拿到 GPT-1、GPT-2、GPT-3 论文后，就能从资料中整理更准确的表格。这里真正的能力是“基于给定资料阅读和汇总”，不是“凭空知道一切”。在学习和工作中，把文件交给模型，让它按你的表格结构整理，往往比单纯问记忆更可靠。现在你应该能做什么：当问题依赖具体文件时，把文件内容纳入上下文，并要求模型引用依据。
10. 给范例：few-shot 让模型知道任务形状如果模型不知道情感分析是什么意思，可以给几个输入输出范例：今天天气真好 → 正面；今天运气真差 → 负面；这朵花真美 → 正面；我累了 → 负面。模型看到范例后，更容易知道你要的是标签判断，而不是聊天、解释或改写。这叫 in-context learning。名字里有 learning，但这里没有训练；模型参数没有改变。改变的是输入里多了范例，模型在当前上下文中按这些范例继续完成文字接龙。现在你应该能做什么：为陌生任务提供 3-5 个一致的输入输出样例。
11. 早期模型可能没有真正读懂范例2022 年有研究故意把范例标签反过来：本来正面的句子标负面，本来负面的句子标正面。如果模型真的读懂范例，新题也应该跟着反转。结果当时一些模型没有明显跟随错误范例，说明它们可能只是从范例中知道“答案有正面和负面两个选项”，并没有仔细学习映射规则。这提醒我们：few-shot 有效不等于模型理解了所有细节。它可能只是更清楚任务类型、输出格式或候选标签。你的范例如果有特殊规则，还要额外提醒模型仔细读定义。现在你应该能做什么：给范例后，检查模型是否真的跟随规则，而不是只套常识。
12. 更强模型真的更会跟随上下文2023 年的研究用更强模型测试，发现大模型更可能跟随错误范例。这里的“答错”反而代表它读懂了上下文，因为实验故意要求它反常识地映射。课程还举 GPT-4 新闻分类：把“政治类”定义成产业经济消息，把“财经类”定义成国内政治消息，模型一开始仍按常识选财经；但提醒它“类别定义可能与一般定义不同，请仔细读”，它就能按定义选政治。结论：强模型更有能力读复杂上下文，但你仍要把异常规则显式标出来。越反常识的规则，越需要提醒模型不要用默认常识覆盖题目定义。现在你应该能做什么：面对反常规定义时，在 prompt 里显式要求按本题定义，不按常识定义。
13. Gemini 1.5 与 Kalamang：长上下文里的临时学习课程提到 Gemini 1.5 的技术报告：Kalamang 是使用者很少、网络资料稀缺的语言，模型裸考几乎不会翻译。研究者把文法书和字典放进上下文后，模型突然能做一定质量的翻译。这个例子非常适合理解 in-context learning。模型在“前面有整本教科书”的上下文中，根据教科书继续生成翻译。下一次如果不再提供教科书，它仍然不会翻译。现在你应该能做什么：区分“本轮看着资料会做”和“以后不看资料也会做”。
完整知识链路本节把 prompt engineering 从“背神奇咒语”拉回到“训练使用者的任务表达能力”。模型参数固定时，你仍然可以通过输入改变输出：用 CoT 或解释提示让模型展开推理，用情绪或原则化指令测试模型敏感性，用强化学习或模型自荐寻找候选 prompt，用前提消除歧义，用资料补足事实，用范例说明任务规则。所有这些都发生在上下文里，不改变模型内部参数。你现在应该能做什么：模型答错时，先诊断缺的是推理过程、前提、资料还是范例，再决定怎样改 prompt。

# 最小例子 {#example}
<!-- tag: EXAMPLE -->

下面三个例子把本节的策略落到可直接改写的 prompt 上。重点看每次补了哪一类信息。

示例 1：模糊问题改成带前提的问题text复制

```text
模糊：
NTU 是什么的缩写？

补前提：
请从台湾大学学生的语境回答：NTU 是什么的缩写？

再换前提：
请从新加坡大学生的语境回答：NTU 是什么的缩写？
```

第一句让模型猜语境。第二句和第三句把地区身份写出来，答案就应该分别靠近国立台湾大学和南洋理工大学。判断句：如果一个缩写、术语或评价标准有多个含义，先补前提，再怪模型。

示例 2：few-shot 让模型知道标签和格式text复制

```text
任务：判断句子的情绪，只输出「正面」或「负面」。

范例：
今天天气真好 -> 正面
今天运气真差 -> 负面
这朵花真美 -> 正面
我累了 -> 负面

新句子：
这次作业终于交完了 -> ?
```

这里的范例告诉模型两件事：任务是情感分析，输出格式是一个标签。它没有改变模型参数，只是在当前上下文中给出可模仿的规则。判断句：few-shot 的第一价值是定义任务和格式，第二价值才是让模型学习特殊映射。

示例 3：反常识类别要显式提醒text复制

```text
类别定义：
政治类：报道产业经济、企业投资、金融普惠的消息。
财经类：报道政府部门、政党选举、公共政策的消息。

请特别注意：上面的类别定义可能和一般用法不同。
分类时必须按本题定义，不要按常识定义。

新闻：
某 AI 公司宣布扩大投资并推出新一代模型。

输出：政治类 / 财经类
```

如果不提醒，模型可能被常识牵走；提醒后，它更可能按当前题目定义作答。判断句：当你的规则和常识冲突时，必须把“按本题定义”写进输入。

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->

误区 1：CoT 是万能咒语错因：把某篇论文或某个旧模型结果当通用定律。后果：在新模型或不适合任务上浪费注意力。修法：用自己的任务样本做 A/B 测试。
误区 2：礼貌一定提升表现错因：把人际沟通习惯直接投射到模型。后果：prompt 变长却没有补充任务信息。修法：优先写清目标、资料、约束和输出格式。
误区 3：解释越长越可信错因：把流畅理由当证据。后果：错误答案被包装得更像真的。修法：检查理由是否引用了给定资料、是否支持结论。
误区 4：不给前提却要求模型猜中错因：忘了模型是新人，不知道你的地区、身份和隐含标准。后果：模型按常见语境回答，你觉得它答错。修法：补“从谁的角度、按什么定义、给谁看”。
误区 5：把资料交给模型就不用验收错因：以为有资料就不会读错。后果：摘要、表格或引用可能漏项、误读或混合来源。修法：要求列出处、保留不确定项、抽查关键字段。
误区 6：范例前后不一致错因：few-shot 样例标签混乱或格式不统一。后果：模型不知道该模仿哪条规则。修法：删掉矛盾样例，保留少量高质量样例。
误区 7：以为 in-context learning 真训练了模型错因：被 learning 这个词误导。后果：以为模型下次还能无资料完成同一任务。修法：记住参数没变，知识只在本轮上下文中可见。
误区 8：反常识规则不提醒错因：以为模型会自动压过常识、严格按定义做。后果：新闻分类、评分规则、公司内部标签会被常规定义覆盖。修法：明确说“定义可能不同，请按上面定义”。
误区 9：先训练模型，不先改输入错因：把所有失败都归因于模型能力不够。后果：成本上升，还可能引入新副作用。修法：先诊断输入问题；prompt、资料、范例都不够时，再考虑训练路线。
