# 概念骨架 {#skeleton}
<!-- tag: MAP -->

产出端：两篇作文英文作文参考 TOEFL 独立写作，约 300 字，agree / disagree 类型；中文作文参考学测国文写作第二部分，以「缝隙的联想」写约 500 字繁体中文文章。
评估端：AI 批改助教课程在 DaVinci 平台建立两个官方批改助教，英文和中文分别对应。文章必须交给正确语言的助教批改，下载 JSON 才能上传。
提交端：过程可追溯NTU COOL 要求填写是否使用 AI、使用什么模型、prompt、最终文章、对话分享链接，并上传官方批改结果。后台会比对 JSON，不可手动修改。

| 阶段 | 你要做什么 | 产物 | 最容易错的地方 |
| --- | --- | --- | --- |
| 文章撰写 | 用 AI 生成、自己写，或 AI 生成后人工修改。 | 英文约 300 字；繁中约 500 字。 | 没有保存 prompt、忘记说明修改过程、字数和语言不符合要求。 |
| 文章批改 | 在 DaVinci 安装并使用对应语言的官方批改助教。 | 批改回复和可下载 JSON 文件。 | 中文文章用英文助教批、旧对话重复批、没有下载满意结果。 |
| 作业提交 | 在 NTU COOL Question 2 填答并上传 JSON。 | 问答、文章、分享链接、JSON。 | 上传格式错、改 JSON、只保留旧版本、超过截止时间。 |
| 诚信检查 | 不抄袭、不共享 prompt 或批改结果、不交他人账户结果。 | 可被后台对照的个人作业记录。 | 以为“AI 生成”就不需要个人过程记录。 |

# 术语表 {#terms}
<!-- tag: TERMS -->

| 术语 | 直觉解释 | 准确说法 | 常见误解 |
| --- | --- | --- | --- |
| AI 生成作文 | 让模型帮你写初稿。 | 用 ChatGPT、Gemini、DaVinci 等生成式模型根据主题、字数、语气和结构要求输出文章。 | 以为直接复制第一版就最稳。 |
| 人工修改 | 对 AI 初稿做二次编辑。 | 可以修改模型输出，也可以自己写后让模型润色；提交时要说明你怎样使用模型。 | 以为改过就不用保留原 prompt。 |
| DaVinci / 达哥平台 | 课程指定的模型平台。 | 联发科提供的平台，含不同模型、会话风格和额度限制；本作业也在其中使用官方批改助教。 | 以为任何平台评分都能替代官方批改。 |
| conversation style | 输出风格控制。 | 字幕提到 precise、neutral、creative。写作时建议 creative，因为作文需要多样表达。 | 以为 creative 等于可以忽略题目要求。 |
| AI 批改助教 | 官方评分器。 | 课程在 DaVinci 建立的 assistant，结合官方批改 prompt 与学生文章，输出等级/分数和批改结果。 | 把自己开的普通 GPT 对话当正式分数。 |
| system prompt | 批改助教的评分说明书。 | 官方把题目、评分标准、评分步骤、范文或等级要求写进提示词，再与学生文章合并输入模型。 | 以为批改模型只是随便读文章打分。 |
| JSON 批改结果 | 可上传的正式结果文件。 | 从 DaVinci 官方批改助教下载的结构化文件，NTU COOL 需要接收这个格式。 | 手动改分数或改文件内容。 |
| NTU COOL | 作业提交系统。 | 需要填写模型使用情况、prompt、文章、对话链接，并上传 JSON。上传次数不限，但只保留最新结果。 | 以为只上传文章就算完成。 |
| 额度 | 平台每天可用量。 | DaVinci 生成与批改都会消耗额度；字幕说文章批改大约每天十次，GPT-4 消耗远高于 GPT-3.5。 | 反复刷新批改，最后没额度提交。 |
| 迟交 | 超过截止时间提交。 | 字幕明确说不接受任何迟交。具体日期以当期课程公告为准。 | 以为上传次数不限就能截止后补交。 |
| 作业诚信 | 结果必须属于你。 | 禁止抄袭、共享 prompt 和批改结果、提交非本人 DaVinci 账户结果、修改 JSON。 | 以为 AI 作业没有抄袭问题。 |

现在你应该能做什么：把“生成作文平台”“官方批改助教”“提交系统”三件事分开，不再混成一个按钮。

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 1. 作业目标：练 AI 工作流
这次作业叫“都是 AI 的作文比赛”，因为写作和批改都可以交给 AI。学生要产出两篇文章，并使用官方 AI 助教给文章评分。这个设计把前一讲的 prompt engineering 落到真实任务：你要写出能让模型生成好文章的指令，还要理解模型批改为什么需要评分 prompt、语言匹配和正式提交文件。
你可以自己写，也可以让 AI 写，还可以让 AI 写完后自己修改。课程允许多种方式，重点是你必须诚实说明使用过程，并把需要的记录提交上去。
现在你应该能做什么：说明这次作业考的是“生成 + 评估 + 可追溯提交”，不是单纯作文。

## box soft
### 2. 两篇作文：英文一篇，繁中一篇
第一篇是英文作文，参考 TOEFL 独立写作，题型是 agree or disagree，约 300 字。你需要围绕题目表达立场、给出理由和例子。英文作文的重点不是炫词，而是论点清楚、结构稳定、语言可读。
第二篇是中文作文，参考学测国文写作第二部分，题目是「缝隙的联想」，约 500 字繁体中文。它更像主题联想写作，需要把题目中的意象转成观察、经验、论述或抒情。无论用 AI 还是自己写，都要确保题目、语言和字数要求没有跑掉。
现在你应该能做什么：为两篇文章分别列出语言、题型、字数和主题要求。

3. Prompt 不只是题目，还要有约束字幕给了几类 prompt 示例：直接告诉模型主题；加上更详细要求，例如段落数、语气、字数、写作风格；或者使用角色扮演，例如告诉模型“你是一位优秀写作者”。这些都是可试的起点，但好的 prompt 应该包含题目、语言、字数、结构、语气和不可偏离的要求。如果你只写“写一篇缝隙的联想”，模型会自由发挥。自由发挥不一定错，但容易出现字数不足、语言不符、题旨漂移或结构松散。把约束写清楚，后续批改才更可能稳定。现在你应该能做什么：把一句短 prompt 扩成可执行的写作规格。
4. ChatGPT / Gemini：要保留对话记录助教介绍了 ChatGPT 的常用功能：新建对话、输入讯息、修改输入、复制回复、重新生成、分享整段对话链接。作业提交时，如果使用 ChatGPT 或 Gemini 这类能分享对话链接的平台，应该把分享链接上传到 NTU COOL。如果你和模型来回互动很多轮，分享链接比手动复制全部 prompt 更可靠。若没有链接，至少要简要说明使用过程和关键 prompt。重点是让助教能理解文章是怎样生成或修改出来的。现在你应该能做什么：写作时就保存 prompt 和链接，而不是交作业前回忆。
5. DaVinci 写作：注意风格与额度DaVinci 平台有多个模型和会话风格。字幕提到 precise 会让同一输入更稳定，creative 会让输出更多样，neutral 介于中间。写作文时助教建议使用 creative，因为作文通常需要更自然和多样的表达。但 DaVinci 有每日额度限制。生成文章和批改文章都会消耗额度；GPT-4 的消耗比 GPT-3.5 高很多。你要把额度当作资源规划：先用低成本方式探索 prompt，再把成熟版本拿去正式批改。现在你应该能做什么：在写作迭代前规划模型、风格和尝试次数。
6. 官方批改：用指定 assistant，不是随便问 GPT过去作文需要专业老师批改；本作业改成由 AI 批改助教自动评分。官方批改助教DaVinci 上安装的 assistant。它背后有官方批改 prompt，包含题目、标准、步骤、等级要求和可能的范文示例。官方会把批改 prompt 和你的文章合并后喂给模型，让模型输出批改结果和分数。因此，自己在 ChatGPT 里复制一段评分标准让模型打分，可以作为练习参考，但不能替代官方批改结果。现在你应该能做什么：解释为什么“普通模型评分”和“官方批改 JSON”不是同一件事。
7. 语言必须匹配：英文助教批英文，中文助教批中文本作业有两个批改助教，名称开头都是类似 GenAI homework two assistant，后缀区分语言。英文文章要用英文批改助教，中文文章要用中文批改助教。如果把中文文章丢给英文助教，或把英文文章丢给中文助教，批改结果会被视为无效，拿不到该文章分数。这评分器和标准错配。现在你应该能做什么：正式批改前先检查“文章语言”和“assistant 语言”是否一致。
8. 每次批改都开新对话助教特别提醒：每重新评估一篇文章，都要开启新的对话窗口。原因是旧对话可能残留前一次文章、批改结果或模型状态，影响下一次评分。新对话能让每次批改更干净。批改会消耗额度，字幕说大约每天十次文章批改额度。看到满意结果时要及时下载 JSON，避免后面误操作、额度耗尽或找不到对应版本。现在你应该能做什么：建立“新对话批一次、满意就下载”的批改习惯。
9. NTU COOL 不是只收文件，还要收过程说明提交时需要完成 Question 2。系统会问你是否使用 AI 生成文章、用了哪些模型、怎样使用模型、生成文章用的 prompt、最终文章内容。如果你用 ChatGPT 或 Gemini 等平台，还要提供分享链接。英文部分和中文部分题型类似，都要填过程和上传对应的 DaVinci 批改结果 JSON。上传次数不限，但只保留最新版本；这意味着你可以修正，但也可能不小心用错误版本覆盖正确版本。现在你应该能做什么：在提交前用清单核对每篇文章的 prompt、文章、链接和 JSON。
10. 评分结构：3 分提交，7 分作文字幕说明作业 2 占整体成绩 10 分。其中 3 分来自 NTU COOL 上的问答提交，7 分来自两篇文章的批改结果。因为有英文和中文两篇文章，所以每篇最多 3.5 分。批改助教会给文章分数或等级，再根据投影片上的区间转换成作业分数。字幕没有保留完整区间，因此实际换算要以课程投影片、LMS 或助教公告为准。你在本页需要记住的是结构：提交完整性和文章质量都会影响总分。现在你应该能做什么：解释为什么“文章分数高”但“NTU COOL 没填完整”仍会丢分。
11. 时间点与迟交字幕中给出的该轮课程时间是：NTU COOL 提交截止为 2024-03-14 23:59，成绩最晚公布时间为 2024-03-28 23:59。这是视频所讲课程轮次的历史时间点，不应直接当作你当前作业的有效日期。真正重要的规则是：不接受迟交。无论当期截止日期是哪一天，都要在系统时间前完成问答和 JSON 上传。不要把“上传次数不限”误解成“截止后也能补”。现在你应该能做什么：把截止日期写成绝对时间，并提前完成最终上传。
12. 诚信规则比技巧更重要作业明确禁止任何形式的抄袭，不要分享自己的 prompt 和批改结果给别人，不要提交不是由自己 DaVinci 账号得到的批改结果，也不要尝试手动修改 JSON。官方会把上传结果和 DaVinci 后台数据比对。违规后果很重：第一次违反，该次作业零分且学习总成绩乘 0.9；第二次违反，课程成绩为 F。老师和助教保留修改规则的权利。也就是说，不要把这次作业当成“骗过评分器”的游戏。现在你应该能做什么：区分合法迭代文章和违规修改/共享结果。
完整知识链路作业 2 把生成式 AI 的使用变成可评分流程。你先用 prompt 驱动模型生成或修改两篇作文，再用官方 DaVinci AI 批改助教按指定标准评分，下载 JSON，最后在 NTU COOL 填写模型使用过程、prompt、文章、链接和批改结果。评分由提交完整性和两篇作文质量组成，同时受额度、语言匹配、版本保存和诚信规则约束。你现在应该能做什么：从空白页开始，按步骤完成一份可追溯、可提交、不会因格式或规则失误无效的作业。

# 最小例子 {#example}
<!-- tag: EXAMPLE -->

下面展示“可提交工作流”长什么样。写 prompt 时要让模型知道题目、语言、字数、结构和风格；批改时要用官方助教。

写作 prompt：英文 agree / disagreetext复制

```text
Write an approximately 300-word TOEFL independent writing essay.
Task type: agree or disagree.
Topic: [paste the official English topic here]
Position: I agree.
Structure: introduction, two body paragraphs, conclusion.
Tone: clear, formal, and suitable for an academic writing task.
Requirements:
- Give concrete reasons and examples.
- Do not invent personal facts that sound too specific.
- Keep the essay around 300 words.
```

这个 prompt 比“帮我写英文作文”更稳，因为它写清楚题型、立场、结构、语气和字数。你可以生成多版，再自己检查论点是否重复、例子是否空泛。

写作 prompt：繁中「缝隙的联想」text复制

```text
請以「縫隙的聯想」為題，寫一篇約 500 字的繁體中文文章。
寫作方向：
- 從生活中的縫隙出發，延伸到人與人、理想與現實，或自我成長。
- 文章要有明確主軸，不要只堆砌意象。
- 語氣自然，有畫面感，但不要過度華麗。
- 結尾要回扣題目。
```

中文作文更容易出现“华丽但空”的问题，所以 prompt 里要加主轴、意象和回扣题目的要求。生成后仍要人工检查繁体、字数和题旨。

提交清单：每篇文章都要留痕checklist复制

```text
每篇文章提交前检查：
1. 最终文章是否符合语言、题目和字数要求？
2. 是否记录了使用的模型？
3. 是否保留了生成文章的 prompt？
4. 如果平台支持，是否保存了对话分享链接？
5. 是否用正确语言的官方批改助教评分？
6. 是否下载了未修改的 JSON 批改结果？
7. NTU COOL 中是否上传的是最新且正确的 JSON？
```

判断句：只要缺少“过程记录”或“官方 JSON”，这份作业就不完整，即使作文看起来写得很好。

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->

误区 1：用普通 GPT 打分当正式结果错因：混淆练习评分和官方批改。后果：没有有效 JSON，NTU COOL 无法认定正式成绩。修法：必须用 DaVinci 官方批改助教。
误区 2：中文文章用英文助教批改错因：只看 assistant 名称开头，没看语言后缀。后果：批改结果无效。修法：英文对英文助教，中文对中文助教。
误区 3：旧对话里反复批不同文章错因：图方便，不开新窗口。后果：上下文残留影响批改。修法：每篇、每次正式批改都开新对话。
误区 4：满意结果没有立刻下载错因：以为平台会自动保留你想要的版本。后果：找不到对应 JSON 或被后续版本覆盖。修法：看到满意结果立即下载。
误区 5：手动修改 JSON错因：以为结构化文件只是普通文本。后果：后台比对异常，可能触犯规则。修法：上传原始下载文件，不手动编辑。
误区 6：只提交文章，不交过程错因：把作业理解成传统作文。后果：NTU COOL 问答分丢失，过程不可追溯。修法：保留模型、prompt、文章、链接和 JSON。
误区 7：额度无限刷新错因：把 AI 评分当免费无限尝试。后果：正式批改前耗光额度。修法：先人工检查，再用批改次数做关键迭代。
误区 8：共享 prompt 或批改结果错因：觉得 prompt 不算答案。后果：违反作业规则，影响成绩。修法：独立完成，问题去讨论区或 TA 邮箱问。
误区 9：把历史日期当当前截止错因：没有区分视频字幕中的课程轮次。后果：误判真实作业期限。修法：本页记流程，日期以当期 LMS 公告为准。
