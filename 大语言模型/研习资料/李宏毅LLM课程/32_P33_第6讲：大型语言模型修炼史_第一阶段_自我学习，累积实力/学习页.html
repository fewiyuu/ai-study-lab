第一阶段：自监督地学会文字接龙

前面的课程一直在使用已经训练好的大型语言模型：怎么提示它、怎么让它反省、怎么让它使用工具、怎么让多个模型合作。从这一节开始，问题反过来：这些模型最初是怎么练出来的？第一阶段的答案很直接，也很容易被误解：它们不是一开始就被教“请回答问题”，而是在海量文本上学习“给定前文，下一个 token 应该是什么”。

这一步叫自监督学习。网页、书籍、论坛、百科、代码和各种文字都可以被整理成接龙训练资料。模型通过最佳化找出数十亿甚至上千亿参数，让自己越来越会预测下一个 token。它会学到语言知识，也会从文本里吸收世界知识；但第一阶段训练出的 GPT-3 这类模型仍然很不受控，常常知道很多，却不稳定回答人的问题。

网页文字→清洗去重→接龙训练资料→最佳化参数→会预测 token→还不会稳定听指令

学完后要能做四件事

  1. 解释语言模型如何把句子变成“前文 → 下一个 token”的训练样本。
  2. 区分参数、超参数、初始参数、训练、推论和最佳化。
  3. 说明训练失败、训练成功但测试失败、overfitting 分别是什么意思。
  4. 解释为什么 GPT-1/2/3 的第一阶段训练能积累知识，却还不能稳定按指令回答。

概念骨架

MAP

对象：语言模型是函数

输入是未完成句子，输出是下一个 token 的预测。模型内部有大量参数，参数决定它怎么接龙。

过程：训练就是找参数

训练资料给出正确接法，最佳化根据这些样本寻找参数。找完参数后，用模型回答新输入叫推论。

风险：会接龙不等于会听话

第一阶段只从文本里学下一个 token。它能学到语言和世界知识，但没有被明确教成稳定问答助手。

层次本节问题要抓住的判断常见错法
文字接龙模型到底学什么？所有阶段都在学 token 预测，只是训练资料不同。以为第一阶段学知识，第二阶段才学语言。
训练机制参数怎么来？参数由最佳化自动找出；人主要设超参数、资料和训练策略。把参数和超参数混成一个词。
泛化问题训练成功为什么还会测试失败？模型只满足训练资料，可能学到颜色这类错误捷径。用人的常识替模型脑补。
资料来源训练资料从哪里来？网页文字可以自动构造大量样本，但必须清洗、过滤、去重和控制品质。以为“资料越多越好”没有副作用。
历史演进GPT-1/2/3 做到了什么？规模和资料量一路增加，知识更丰富，但第一阶段输出仍不受控。以为 GPT-3 已经等同 ChatGPT。

现在你应该能做什么：用“资料、参数、最佳化、推论、泛化”这几个词，把第一阶段训练讲成一条完整链路。

超参数训练机器的设置。训练前由人或搜索流程设定的选项，例如学习率、batch size、训练步数、优化器设置等。听到“调参数”就以为是在手动改模型权重。
最佳化根据资料找参数。optimization。通过算法让模型参数更符合训练资料的过程。以为最佳化一定会成功。

测试 / 推论拿训练好的参数来用。testing / inference。参数固定后，输入新资料，让模型输出预测。以为推论还会继续学习参数。
overfitting训练题会了，新题不会。模型很好地符合训练资料，却在未见过的测试资料上表现差。以为训练集表现好就代表模型真的学懂。
初始参数训练开始时的参数起点。最佳化从这组参数出发搜索；不同初始参数可能得到不同结果。以为初始参数随便设都没影响。
from scratch从随机参数开始训练。不使用已有模型权重，通常随机初始化参数后开始训练。以为从零训练只是不看旧资料。
自监督学习资料自己产生答案。self-supervised learning。利用原始文本本身构造监督信号，例如前文预测后文。以为完全没有人工介入，连清洗都不需要。
语言知识知道句子怎样才通顺。关于语法、搭配、句式、词序等语言规律的知识。以为语言知识等于全部知识。
世界知识知道世界事实和规律。关于物理、社会、常识、历史、专业事实等内容的知识。以为 300 亿 token 一定足够覆盖世界。

现在你应该能做什么：解释“调参数”在日常机器学习语境里多数指调超参数，而不是手动填写上亿个参数。

讲义主线

LECTURE

1. 所有阶段都在学文字接龙，只是资料不同

这节课先纠正一个常见误解：语言模型修炼分三个阶段，不代表三个阶段在学完全不同的本领。底层训练目标仍然围绕文字接龙，也就是给定前文预测下一个 token。不同阶段真正变化的是训练资料的形态，以及人类如何把想要的行为写进资料里。

第一阶段使用大量原始文本，模型主要学“前文之后通常出现什么”。后续阶段会加入人类写好的问答、偏好比较等资料，让模型更懂如何按指令行动。判断句：如果你听到“预训练、监督微调、RLHF”，先问它们各自喂给模型的训练资料是什么。

现在你应该能做什么：用一句话说明“阶段不同，训练资料不同，但都和 token 预测有关”。

2. token：模型不是一次吐出整段答案

语言模型生成答案时，不是一次性写完整段，而是一步一步产生 token。为了教学简单，字幕中把一个中文字当一个 token；真实系统里，一个中文字可能被切成多个 token，英文单词也可能被拆成词片段。token 是模型看到和生成文本的基本单位。

这一点很重要，因为训练样本也是按 token 构造的。句子“人工智慧真神奇”可以拆成“人 → 工”“人工 → 智”“人工智 → 慧”“人工智慧 → 真”等多个训练样本。每个样本都在告诉模型：看到这个前文，下一个 token 应该是什么。判断句：语言模型的训练数据不只是完整句子，而是从句子里切出的许多前文-后文对。

现在你应该能做什么：把一个短句拆成至少三个接龙训练样本。

3. 模型是函数，参数决定函数怎么接龙

这一节把语言模型写成函数 F。输入是未完成句子，输出是后面可能接的 token。因为文字接龙极其复杂，这个函数不能只靠几条规则写出来，它需要数十亿甚至更多参数来表示。

参数不是人手工写的规则。人准备训练资料，设定训练流程，最佳化算法根据资料自动寻找这些参数。训练结束后，参数固定下来，模型就可以在新输入上做推论。判断句：参数是模型通过训练学到的行为载体，不是训练资料本身。

现在你应该能做什么：区分“模型记住的参数”和“训练时喂给它的文本资料”。

4. 训练和推论：一个是找参数，一个是用参数

训练 training / learning 是寻找参数。训练时，模型看到许多前文和正确下一个 token，最佳化算法不断调整参数，让模型更符合训练资料。推论 inference / testing 是参数已经找好后，输入新前文，让模型产生预测。

这两个阶段的状态不同。训练会改变参数，推论通常不改变参数。你在网页上问 ChatGPT 问题，它一般是在推论；你用大规模资料更新模型权重，才是在训练。判断句：如果参数没有被更新，你通常是在使用模型，而不是训练模型。

现在你应该能做什么：看到一个操作时判断它是训练还是推论。

5. 超参数：训练机器怎么运行

字幕把最佳化想成一台机器。你把训练资料丢进去，它运转后吐出参数。但这台机器启动前要先设定一些东西，这些设置叫超参数。比如学习率、batch size、训练轮数、优化器选择、随机种子等都属于训练流程的设置。

平常说“做深度学习就是调参数”，很多时候其实是在调超参数。模型参数数量太大，不可能由人一项一项填写；人能调的是训练流程设置。不同超参数会影响训练是否稳定、是否收敛、是否过拟合、需要多少算力。判断句：参数是训练结果，超参数是训练前或训练中人为设定的控制项。

现在你应该能做什么：听到“调参数”时追问对方指的是模型权重还是训练设置。

6. 训练失败：最佳化没找到符合资料的参数

训练不是按下按钮就一定成功。你可能设置了不合适的超参数，训练过程震荡、发散或卡住，最后得到的参数连训练资料都不能很好拟合。字幕中的例子是：训练资料说“人工智”后面应该接“慧”，但模型学完后没接对。

遇到训练失败，常见做法是调整超参数再试。为什么训练大模型需要算力？一方面是每次训练本身成本巨大，另一方面是失败后要多次实验。助教在作业里会帮你设好超参数，所以你可能暂时感受不到这件事的痛苦。判断句：训练失败指训练集本身都没学好，不是泛化问题。

现在你应该能做什么：把训练失败和 overfitting 区分开。

7. overfitting：训练成功，但测试失败

字幕用猫狗分类解释 overfitting。训练资料里两只猫都是黑的，两只狗都是黄的。模型可能找到一个看似符合训练资料的规则：黑色就是猫，黄色就是狗。它在训练集上全对，所以训练成功；但遇到黄色猫时，就会错判成狗。

这说明机器不会自动知道“颜色不重要，外形才重要”。最佳化只关心参数是否满足训练资料，不会主动使用人类常识。要减少这种错误捷径，一个方法是增加训练资料多样性，比如加入黄色猫、黑色狗，迫使模型不能只靠颜色。判断句：训练集表现好只能说明模型符合已见资料，不能保证它学到合理规律。

现在你应该能做什么：用“训练成功、测试失败、错误捷径”解释 overfitting。

8. 初始参数与 from scratch：从哪里开始找

最佳化不是在所有可能参数里凭空跳到答案，而是从一组初始参数出发逐步寻找。初始参数会影响最终找到的参数。若初始参数离好解较近，训练可能更容易得到想要的结果；若初始参数完全随机，就叫 from scratch，从零开始训练。

好的初始参数可以看成先验知识：模型在正式学习任务前已经具备某些能力。问题是，第一阶段之前去哪找好参数？如果没有已有模型，就只能随机初始化，然后靠海量文字和算力训练出基础能力。判断句：初始参数不是训练资料，但它会影响训练起点和最后可能到达的位置。

现在你应该能做什么：解释为什么“从零训练大模型”比“微调已有模型”困难得多。

9. 语言知识和世界知识：接下一个 token 需要两种能力

要接对下一个 token，模型先需要语言知识。比如“这个人突然就”后面可以接“跑”，也可以接“飞”，但很难接一个完全不合语法的词。语言知识包括语法、搭配、词序和句式，研究中相对少一些资料就可能学到不少。

但文字接龙还需要世界知识。“水的沸点是摄氏”后面接“100 度”不只是语法正确，更依赖物理知识。世界知识还分层：低气压下水的沸点会改变，50 度也可能在特殊条件下合理。字幕强调，世界知识复杂而多层次，早期研究发现 300 亿个文字看起来仍不够。判断句：通顺不等于正确，语言知识不能替代世界知识。

现在你应该能做什么：举一个“语法通顺但事实错误”的接龙例子。

10. 自监督学习：从原始文本自动制造题目

第一阶段能做大，是因为文字资料很容易自动转成训练样本。只要有句子，就能构造“前文 → 下一个 token”的题目，不需要人工逐条标注答案。这个方式叫 self-supervised learning，自监督学习。它不是没有监督信号，而是监督信号来自资料自身。

比如“人工智慧真神奇”这句话自己就提供了多个答案：看到“人”应接“工”，看到“人工智”应接“慧”。网络上有海量文本，所以训练资料几乎可以不断扩张。判断句：自监督学习的“答案”藏在原始资料内部，不需要人另写标签。

现在你应该能做什么：解释为什么网页文本能直接变成语言模型训练资料。

11. 网页资料需要清洗：不是随便爬就结束

虽然网页文字很多，但直接拿来训练会有问题。字幕引用 DeepMind 训练 Gopher 的例子：需要过滤色情、暴力等有害内容；去掉 HTML 标记；保留有用符号，例如表情符号；控制资料品质；去除重复资料；还要做 testing set filtering 保证实验严谨性。

重复资料尤其危险。字幕提到有论文发现某段婚礼公司业配在网页资料中重复 6 万多次。若不去重，模型可能学到不合理偏好，动不动生成类似广告。资料品质分类器也常被用来提高高品质文本的影响力，例如更像维基百科或教科书的内容可能被看得更多。判断句：资料多只是起点，资料分布和清洗会直接改变模型行为。

现在你应该能做什么：列出训练网页语料前至少四种清洗动作。

12. 资料授权与伦理：能看到不等于一定能用

字幕提到 New York Times 起诉 OpenAI 和 Microsoft 的新闻，作为训练资料授权争议的例子。法律问题这里不展开，但它提醒我们：第一阶段依赖海量文字，一旦“能爬到就拿来训”，就会碰到版权、隐私、付费内容、数据来源透明度等问题。

从学习角度，你需要知道资料不是中性的。资料来源决定模型吸收什么事实、偏见和风格；授权方式决定模型训练是否可持续；清洗策略决定哪些内容被放大或压低。判断句：训练资料不是背景杂音，它是模型能力和风险的来源之一。

现在你应该能做什么：说明为什么资料治理是大模型训练的一部分。

13. GPT-1、GPT-2、GPT-3：天资和努力一起放大

字幕用两个维度解释模型变大：参数量代表模型复杂度，可以粗略看作“天资”；训练资料量代表模型读过多少东西，可以粗略看作“努力”。GPT-1 在 2018 年只有约 1.17 亿参数，训练资料大约 700 本书；GPT-2 在 2019 年有 15 亿参数，用约 40GB 文本；GPT-3 在 2020 年有 1750 亿参数，用约 580GB 文本，约 3000 亿 token。

规模变大后，模型确实知道更多，能力也变强，但当年的结果并没有立刻让所有人相信“文字接龙就是人工智能”。GPT-2、GPT-3 在一些问答评测上仍然不算特别惊人。判断句：模型规模和资料量重要，但它们只解决“累积实力”的一部分，不自动解决“按人类意图使用实力”。

现在你应该能做什么：用 GPT-1/2/3 的演进说明参数量和资料量的共同作用。

14. GPT-3 的不受控：知道很多，但不知道你要它回答

字幕演示了 GPT-3 早期版本 DaVinci 002 的感觉。问它“什么是 machine learning”，它有时能回答，有时会反过来生成新的问题，好像在续写网页或论坛内容。你加上 “Q:” 和 “A:” 这类格式，它才更容易按问答方式输出。问 “LOL” 时，如果写一个“1.”，它可能就明白你想要多个释义。

这不是因为 GPT-3 什么都不知道，而是因为第一阶段训练没有明确教它“用户问问题时应该回答”。它只学过网络文本后面常接什么，而网络上“台湾最高的山是哪座山”后面可能接选项、评论邀请、广告或别人的问答格式。判断句：第一阶段模型有知识，但缺少稳定调用知识的指令行为。

现在你应该能做什么：解释为什么早期 GPT-3 对 prompt 格式特别敏感。

完整知识链路

P33 的完整链路是：语言模型是一个输入前文、输出下一个 token 的函数；训练就是用大量“前文 → 正确下一个 token”的样本，通过最佳化找到参数；超参数控制训练过程，初始参数决定搜索起点；训练可能失败，也可能训练成功但测试失败，即 overfitting；第一阶段靠自监督学习把网页文本变成海量训练资料，但资料需要清洗、去重、品质控制和授权治理；GPT-1/2/3 通过扩大参数和资料累积语言知识与世界知识，却仍然不会稳定按人类指令回答。你现在应该能做什么：解释为什么第一阶段像“累积内力”，下一阶段还需要人类指导模型如何使用能力。

最小例子

EXAMPLE

这些例子把字幕中的训练概念落到可读流程。重点不是代码能直接训练 GPT，而是看懂资料如何变成学习信号。

示例 1：把句子拆成接龙训练样本python-ish
sentence = ["人", "工", "智", "慧", "真", "神", "奇"]

training_examples = [
  (["人"], "工"),
  (["人", "工"], "智"),
  (["人", "工", "智"], "慧"),
  (["人", "工", "智", "慧"], "真")
]

每个样本都问同一类问题：看到前文，下一个 token 是什么。模型不是被人逐条解释“人工智慧是什么意思”，而是在大量类似样本中逐渐学会语言规律和世界知识。

示例 2：参数、超参数和推论的边界workflow
hyperparameters = {
  "learning_rate": 0.0001,
  "batch_size": 1024,
  "training_steps": 1000000
}

parameters = random_initialize()
parameters = optimize(parameters, training_data, hyperparameters)

# inference: parameters stay fixed
next_token = model_predict(parameters, prompt="水的沸点是摄氏")

超参数控制训练怎么跑，参数是训练后得到的结果。最后一行只是使用参数预测，不会自动把新问题学进模型。

示例 3：错误示例：只看训练集就宣布学会diagnosis
训练资料：
  黑猫 -> 猫
  黑猫 -> 猫
  黄狗 -> 狗
  黄狗 -> 狗

模型学到的捷径：
  if color == "黑": return "猫"
  if color == "黄": return "狗"

训练集：全对
新图片：黄色猫 -> 被误判成狗

这就是 overfitting 的直觉：模型满足了训练资料，却没有学到能迁移到新资料的合理规律。判断模型时必须看没见过的测试资料。

精修直觉：第一阶段是在“攒可用规律”

预训练不是把互联网塞进数据库，也不是教模型背标准答案。它把海量文本切成“前文 -> 下一个 token”的题目，让模型参数逐渐吸收语言规律、事实搭配、代码模式和常识线索。它攒出了能力，但还没有明确学会“用户问我时，我该按指令回答”。

术语快辨：训练失败 vs overfitting

训练失败是训练资料本身都没拟合好，例如看到“人工智”仍接不出“慧”；overfitting 是训练集表现很好，但新资料失败，例如只学到“黑色就是猫”。前者常看优化、超参数和训练稳定性；后者常看数据多样性、测试集和错误捷径。

Worked example：一句话变成四道题

“人工智慧真神奇”可以变成“人 -> 工”“人工 -> 智”“人工智 -> 慧”“人工智慧 -> 真”。如果模型每一步都把正确 token 的概率调高，长期看就学会了很多接续模式。换成网页级别的海量文本，模型会积累语言知识和世界知识；但如果网页重复、低质或未清洗，也会把偏见和噪声一起学进去。

误区错因后果修法

        
以为 token 一定等于一个中文字。把教学简化当成真实实现。无法理解不同模型的 token 计费、长度限制和切分差异。记住 token 是模型的文本单位，具体切法由 tokenizer 决定。

        
把参数和超参数混在一起。没有区分训练结果和训练设置。听不懂“调参数”到底在调什么。参数由训练学出，超参数由训练前设置或搜索。

        
认为训练集全对就代表模型学懂。忽略泛化能力。模型可能只学到颜色、格式、重复词等捷径。用测试集和更多样资料检查。

        
用人的常识替模型脑补。以为模型会自动知道哪些特征重要。低估 overfitting 和资料偏差。看训练资料给了模型什么限制，而不是看人觉得什么合理。

        
以为自监督学习完全不需要人工。只看到标签自动生成，忽略资料治理。有害内容、HTML 噪声、重复广告和低品质文本会污染模型。加入清洗、过滤、品质分类、去重和测试集过滤。

        
以为资料越多一定越好。忽略重复、偏见、授权和分布问题。模型可能放大垃圾内容、广告片段或法律风险。关注资料来源、比例、质量和授权。

        
以为 GPT-3 已经等于 ChatGPT。把第一阶段预训练和后续对齐混淆。无法解释早期 GPT-3 为什么很不受控。区分“知道很多”和“稳定按指令回答”。

        
以为 prompt 格式永远同等重要。把 GPT-3 早期模型的经验泛化到所有新模型。过度迷信格式，忽略模型已被后续阶段训练得更会听指令。根据模型代际和训练方式判断 prompt 敏感度。

训练失败是训练资料都没拟合好；overfitting 是训练资料拟合好，但新资料表现差。

        
语言知识 vs 世界知识
语言知识让句子通顺，世界知识让内容符合事实和条件。模型要接对 token，两者都需要。

        
知识积累 vs 指令跟随
第一阶段能让模型吸收知识，但不保证它知道“用户问我问题时应该回答”。
