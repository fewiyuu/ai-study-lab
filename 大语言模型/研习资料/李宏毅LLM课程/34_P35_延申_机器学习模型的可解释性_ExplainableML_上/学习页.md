# 概念骨架 {#skeleton}
<!-- tag: MAP -->

动机：答案正确仍可能靠错线索模型可能利用数据偏差、背景文字、文件格式等捷径。解释的第一用途是暴露这些“看起来会了”的假象。
对象：解释一次判断或整体概念local explanation 解释某个输入为什么得到某个输出；global explanation 问模型整体认为什么叫猫、马或某类对象。
方法：解释也有噪声和误导热力图、梯度、attention、probe 都只能提供证据片段。解释结果本身也要被检查，不能看着顺眼就信。

| 问题 | 对应方法 | 能看见什么 | 最容易错解 |
| --- | --- | --- | --- |
| 模型看输入哪一块？ | Occlusion、Saliency Map、SmoothGrad、IG。 | 哪些像素、区域或 token 对输出影响大。 | 把热力图当绝对真相。 |
| 模型内部如何处理输入？ | 隐藏层降维可视化。 | 表示是否逐层变得更适合任务。 | 二维投影看起来分开就过度解释。 |
| Attention 能说明什么？ | Attention map。 | 某些 token 或位置之间的权重关系。 | 以为 attention 永远等于解释。 |
| 某层是否含某种资讯？ | Probing classifier。 | embedding 是否可被简单模型读出词性、实体等属性。 | 忽略 probe 太弱或太强带来的误判。 |
| 解释给谁看？ | 法律、医疗、客户、开发者、自我检查。 | 解释目标会改变方法选择。 | 以为好的解释只有一种标准。 |

现在你应该能做什么：看到一张解释图时，先问它解释的是哪一次输入、用的什么方法、想说服谁、有什么失败模式。

# 术语表 {#terms}
<!-- tag: TERMS -->

| 术语 | 直觉解释 | 准确说法 | 常见误解 |
| --- | --- | --- | --- |
| Explainable ML | 让模型给理由。 | 为原本难以理解的模型提供可检查、可沟通的解释方法。 | 以为解释一定等于模型真实思考过程。 |
| Interpretable | 模型本来就较容易看懂。 | 模型结构本身较透明，例如简单线性模型、小决策树。 | 以为所有决策树都天然容易解释。 |
| Black box | 只知道输入输出，不清楚内部。 | 难以直接理解内部决策机制的模型，如大型深度网络。 | 以为黑箱一定不能用。 |
| Local explanation | 解释这一次判断。 | 给定一个具体输入，说明哪些部分导致当前输出。 | 用一个样本的解释代表整个模型。 |
| Global explanation | 解释模型整体概念。 | 不针对单张图，而是问模型整体认为什么特征构成某类。 | 以为 global 一定比 local 更可靠。 |
| Occlusion | 遮住一块，看输出变不变。 | 修改或删除输入 component，观察输出变化来估计重要性。 | 以为遮挡没有引入分布外输入问题。 |
| Saliency Map | 输入重要性热力图。 | 通常基于输出对输入 component 的梯度，显示哪些位置影响输出。 | 以为亮点越多解释越好。 |
| SmoothGrad | 把多张带噪热力图平均。 | 对输入加入多种噪声，计算多张 saliency map 后平均，减少视觉噪声。 | 以为更平滑就一定更真实。 |
| Integrated Gradients | 沿路径累计梯度。 | 从 baseline 到真实输入的路径上累积梯度，缓解局部梯度为零的问题。 | 以为按一下就不用理解 baseline。 |
| Hidden representation | 模型中间层表示。 | 输入经过某层后得到的向量，可用于观察模型如何逐层转换资讯。 | 把降维图形当完整内部机制。 |
| Attention | 位置之间的权重。 | 模型在某层某头中，对不同 token 或位置分配的权重。 | 以为 attention weight 总能解释因果。 |
| Probing | 用探针测某层含什么资讯。 | 训练一个小模型读取 embedding，看能否预测词性、实体、语音属性等。 | 忽略 probe 本身训练好坏和能力大小。 |

现在你应该能做什么：区分“模型本身可解释”和“给黑箱模型补一个解释方法”。

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 1. 为什么答对还不够：神马 Hans 的教训
神马 Hans 能用马蹄敲出数学题答案，看起来像会算术。后来发现，它并没有理解数学，而是观察围观者的细微反应，知道什么时候该停。这个故事放到机器学习里，就是模型可能得到正确答案，却靠了完全错误的线索。
模型在测试集上表现高，不代表它学到人类希望的规则。它可能在数据里找到捷径：背景、文件格式、水印、采集设备、标注偏差。可解释性的第一价值，是让我们检查模型是不是在“读旁观者表情”。判断句：准确率回答“有没有答对”，解释回答“为什么答对”。
现在你应该能做什么：用 Hans 的故事说明为什么需要解释模型。

## box soft
### 2. 真实场景需要理由，不只是心理安慰
银行贷款、医疗诊断、法律判案、自驾车急刹，这些场景的错误成本很高。模型如果拒绝贷款、判断病症、建议假释或突然刹车，人们需要知道理由。理由不只是给用户看的说明书，也影响责任归属、合规审查和后续改进。
在开发模型时，解释还可以帮助我们修正系统。过去深度学习常靠调 learning rate、改架构、换超参数来试错。如果能知道模型为什么犯错，也许能更有方向地改数据、改模型或改训练流程。判断句：可解释性既是信任问题，也是调试问题。
现在你应该能做什么：为一个高风险 AI 场景说明为什么必须给理由。

3. 不要因为深度模型难解释就只用简单模型线性模型容易解释，因为每个 feature 的 weight 可以直接看；小决策树也容易解释，因为路径上的问题和答案清楚。但简单模型通常能力有限，无法处理复杂图像、语音、语言任务。深度模型强，却难解释。字幕用“路灯下找钥匙”的故事提醒：不能因为路灯下看得见，就坚持只在路灯下找。真正有用的模型可能在黑暗处，我们要做的是扩展照明范围，让强模型也能被检查。判断句：可解释性让强模型变得可检查。现在你应该能做什么：反驳“黑箱模型都不该用”这个过度结论。
4. Decision Tree 也不总是解释性的终点决策树看起来很透明：每个节点问一个问题，走到叶节点得到判断。问题是，现实中好用的常常random forest 或 gradient boosting 这类很多树的组合。500 棵树一起投票，就很难直接说清最终判断来自哪条规则。这说明“模型类别”不等于“解释一定简单”。小树可解释，大森林未必；线性模型可读，但特征工程复杂时也可能难懂。判断句：解释性要看具体模型规模、结构和使用方式，不只看名字。现在你应该能做什么：解释为什么 random forest 不像单棵决策树那么直观。
5. 好解释没有唯一标准：它常常是给人看的理由Explainable ML 的目标不像 accuracy 那样容易放进 leaderboard。什么才是最好的 explanation？字幕给出一个很有现实感的观点：好的解释往往是让目标对象能接受的理由。这个对象可能是客户、老板、监管者、医生，也可能是开发者自己。这不代表可以编理由糊弄人，而是提醒我们：解释目标必须明确。给患者看的解释、给工程师调试用的解释、给法院审核的解释，不会完全一样。判断句：解释质量要结合使用者、任务风险和验证方式判断。现在你应该能做什么：说明为什么 Explainable ML 很难像分类准确率那样排名。
6. Local vs Global：解释这张图，还是解释“猫”这个概念Local explanation 问的是：给定这张图片，模型为什么说它是猫？它看的是眼睛、耳朵、脸，还是背景？Global explanation 问的是：对这个分类器整体而言，什么样的图会被当成猫？它在理解模型内部的类别概念。两者回答不同问题。local 适合审查某次决策；global 适合理解模型整体偏好。不能用一张图的 local 热力图代表整个模型，也不能用 global 概念忽略某次具体误判。判断句：解释前先确定问题是“为什么这一次”还是“模型整体学了什么”。现在你应该能做什么：给同一个猫分类器分别提出一个 local 问题和一个 global 问题。
7. Occlusion：遮住一块，看模型信心怎么变最直接的 local 方法是遮挡。把灰色方块移到图片不同位置，观察模型输出“博美狗”或“轮胎”的概率变化。如果遮住狗脸后概率大跌，说明狗脸区域对判断重要；如果遮住背景没有影响，说明背景不是关键。这个方法直观，但也有边界。灰色方块本身可能制造不自然输入；遮挡大小会影响结果；被遮住后模型行为可能与真实缺失不同。判断句：occlusion 能暴露重要区域，但它测的是“遮住后输出变化”，不是模型内心独白。现在你应该能做什么：解释遮挡法如何判断图片局部重要性。
8. Saliency Map：用梯度画输入重要性Saliency map 通常看输出分数对输入 component 的梯度。影像里 component 可以是 pixel 或区域；文字里可以是 token。梯度大的位置表示：这个位置稍微改变，输出可能变化较大，于是被画得更亮。宝可梦 vs 数码宝贝例子很有警示性。模型测试准确率很高，但 saliency map 显示它关注背景角落。后来发现宝可梦多是 PNG，数码宝贝多是 JPG，背景差异成了捷径。另一个真实 benchmark 里，马的图片左下角水印文字被模型当成重要线索。判断句：saliency 的价值暴露模型是否看错地方。现在你应该能做什么：用背景捷径解释为什么高准确率模型仍可能不可靠。
9. SmoothGrad：平滑热力图，但别把平滑当真相原始 saliency map 往往噪声很多。SmoothGrad 的做法很简单：给原图加入不同随机噪声，分别计算 saliency map，再把多张图平均。这样亮点会更集中，视觉上更干净。字幕也提醒：更平滑、更让人舒服，不一定代表更真实。可能模型确实用了背景草地，SmoothGrad 让图变得好看后，反而让人忽略问题。判断句：SmoothGrad 是降噪方法，不是解释正确性的保证。现在你应该能做什么：说明 SmoothGrad 如何做，以及为什么仍需谨慎。
10. Integrated Gradients：解决局部梯度可能为零的问题只看局部梯度会出问题。字幕用大象鼻子解释：鼻子越长越像大象，但长到一定程度后，再加长一点，模型输出几乎不变。此时局部梯度接近零，saliency 可能误以为鼻子不重要。Integrated Gradients 的思路是从 baseline 到真实输入沿路径累计梯度，而不是只看当前点附近一小步。这样能捕捉“从没有鼻子到长鼻子”的整体影响。这里不要求推导公式，但要知道它试图补 saliency 的短板。判断句：局部变化小，不代表这个特征整体不重要。现在你应该能做什么：用大象鼻子例子解释为什么只看梯度会误判。
11. Hidden representation：看模型中间层怎样整理资讯除了问输入哪部分重要，我们还可以问模型内部如何处理输入。字幕用语音识别例子：原始 acoustic feature 中，不同人说同一句话看起来差异很大；经过网络多层处理后，同一句话的表示逐渐排列在一起，说明模型把“说话人差异”压低，把“内容”抽出来。因为中间层向量可能有 100 维甚至更多，人眼难以看懂，所以会用降维方法投到二维观察。这种图能提供直观线索，但投影本身也会扭曲结构。判断句：隐藏层可视化能帮助观察趋势，但不能单独证明完整机制。现在你应该能做什么：解释为什么语音模型中间层可能逐渐去掉说话人特征。
12. Attention：有时有解释力，但不能神化Transformer 里的 attention 很容易被拿来解释，因为它显示一个 token 对其他 token 的权重。某些例子中，attention 确实能显示代词、实体或句法关系；课程作业也会挑比较明显的例子。但文献里有争论：Attention is not Explanation，也有人反驳说 Attention is not not Explanation。意思不能把 attention weight 自动等同因果解释。判断句：attention 可以作为线索，不是最终判决书。现在你应该能做什么：回答“attention 能不能解释模型”时给出有条件的答案。
13. Probing：用探针问某层有没有某种资讯如果你想知道 BERT 某层 embedding 里有没有词性资讯，可以训练一个小分类器，让它读取 embedding 预测 POS tag。如果 probe 正确率高，说明这些 embedding 中可以读出词性相关资讯。也可以训练 NER probe 看是否含人名、地名等实体资讯。Probing 的关键风险在 probe 本身。正确率低，可能是 embedding 没有资讯，也可能是 probe 没训好；正确率高，也可能是 probe 太强，自己学了复杂规律，而不是 embedding 本身清楚编码。判断句：probe 的结论必须连同 probe 能力一起报告。现在你应该能做什么：说明为什么 probing 结果不能直接当作模型机制真相。
14. Probe 不一定是分类器：语音还原的例子字幕最后讲了一个更有趣的 probe：把语音识别模型某层 embedding 接到 TTS 模型，试着还原原始声音。如果经过前几层后还原出的声音保留说话人特征，说明该层还保存不少 speaker 信息；如果经过后几层后还原声音听不出是谁讲的，说明模型可能逐渐抹去了说话人，只保留内容。另一个例子是含钢琴噪声的语音。通过 CNN 层后噪声仍在，通过第一层 LSTM 后钢琴声明显变小，说明噪声过滤可能发生在 LSTM 开始处。判断句：probe 可以设计成分类、还原、生成等多种任务，只要它能测试某层表示中有什么信息。现在你应该能做什么：举出一个非分类 probe 的用途。
完整知识链路P35 的主线是：可解释性为了检查模型的理由是否可靠。模型答对可能靠错误捷径，因此高风险场景、合规审查和模型调试都需要解释；简单模型可解释但能力有限，强模型难解释但不应被放弃；解释可以是 local 或 global；输入重要性可用 occlusion、saliency、SmoothGrad、Integrated Gradients；内部处理可看 hidden representation、attention 和 probing；但每一种解释方法都有失败模式。你现在应该能做什么：面对一个解释结果，既能读懂它，也能怀疑它。

# 最小例子 {#example}
<!-- tag: EXAMPLE -->

下面把解释方法压缩成可读流程。重点是每个方法到底测量了什么。

示例 1：Occlusion 的最小逻辑python-ish复制

```text
original_score = model(image)["dog"]

for patch in sliding_windows(image):
 masked = cover(image, patch, color="gray")
 new_score = model(masked)["dog"]
 importance[patch] = original_score - new_score

# score drop is large -> this patch matters for current decision
```

这测的是“遮住后输出下降多少”。如果遮住狗脸分数大跌，狗脸对这次 dog 判断重要。

示例 2：SmoothGrad 的最小逻辑python-ish复制

```text
maps = []
for i in range(100):
 noisy_image = image + random_noise(scale=0.1)
 maps.append(saliency(model, noisy_image, target="dog"))

smooth_map = average(maps)
```

它把多张带噪图的 saliency 平均，让稳定出现的重要区域保留下来，随机噪声被抵消。平滑不是正确性的证明。

示例 3：Probing 的最小逻辑workflow复制

```text
frozen_model = BERT
embedding = frozen_model.layer_8(sentence)

probe = train_classifier(
 input=embedding,
 target=part_of_speech_tags
)

if probe_accuracy is high:
 layer_8 representation contains information useful for POS prediction.
```

probe 准确率高只能说明“可以从这里读出某种资讯”。它不直接证明模型在原任务中一定用这项资讯做决策。

精修直觉：解释是证据，不是审判书可解释性要回答“模型为什么这么判断”，但多数方法只能给证据片段。热力图说明输入变化和输出变化有关，attention 说明权重分配，probe 说明某层表示可读出某种信息。它们都能帮助排查捷径，却不能自动等于模型真实因果机制。
术语快辨：local、global、interpretablelocal explanation 解释一次具体判断，global explanation 解释模型整体概念；interpretable model 是模型结构本身较容易读，例如小决策树；explainable method 是给黑箱模型补解释工具。一个 local 热力图不能代表整个模型，一个可解释模型也不保证表现足够好。
Worked example：热力图发现背景捷径如果猫狗分类器准确率很高，但 saliency map 总是亮在背景水印上，下一步做对照：去掉水印再测、换背景再测、遮住动物主体再测。如果去掉水印后分数大跌，说明模型可能学到数据捷径。这个例子训练的是“读懂解释结果，也要验证解释结果”。

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->

| 误区 | 错因 | 后果 | 修法 |
| --- | --- | --- | --- |
| 模型准确率高，就说明理由正确。 | 只看输出，不看决策线索。 | 模型可能靠水印、背景、文件格式等捷径。 | 用 local explanation 检查输入重要区域。 |
| 黑箱模型都不该用。 | 把可解释性和模型能力对立起来。 | 放弃强模型，退回能力不足的方法。 | 改进解释方法，让强模型也能被检查。 |
| 决策树一定容易解释。 | 只想到小树，忽略森林和复杂树。 | random forest 的整体规则仍然难懂。 | 按具体模型规模和组合方式评估解释性。 |
| 热力图好看就代表解释好。 | 把视觉舒适当成真实性。 | SmoothGrad 可能掩盖背景捷径。 | 用遮挡、反事实、数据检查交叉验证。 |
| 局部梯度小就说明特征不重要。 | 忽略饱和区。 | 可能误判大象鼻子这类关键特征。 | 考虑 Integrated Gradients 等路径累计方法。 |
| Attention 权重就是因果解释。 | 把权重关联当成模型真正原因。 | 过度解读某些 attention head。 | 把 attention 当线索，配合扰动和其他证据。 |
| Probe 准确率低就证明没有资讯。 | 忽略 probe 可能没训练好。 | 低估某层表示。 | 报告 probe 设置、能力、基线和训练质量。 |
| Probe 准确率高就证明模型使用了这项资讯。 | 混淆“可读出”和“被原任务使用”。 | 过度解释内部机制。 | 结合 causal intervention 或任务行为验证。 |

Local vs Globallocal 解释一次判断，global 解释模型整体概念。不要用一个 local 结论代表全部。
解释 vs 真实机制解释方法通常给的是证据，不是模型完整思考记录。越复杂的模型越要多证据交叉。
可接受 vs 可验证解释要让人能接受，也要尽量能验证。只让人舒服的解释不够。
