第二阶段：人类老师怎么点拨语言模型

第一阶段的模型已经在大量文本中学过复杂的文字接龙规律，但它不一定知道“用户问问题时应该怎么回”。第二阶段要做的是 instruction fine-tuning：把人类准备的指令与答案整理成训练样本，让模型把原本散落在参数里的能力，用成更接近人类期待的回答方式。

学这一节，不要只背“监督微调”四个字。你要能解释为什么训练样本要标出 user 和 AI，为什么只靠少量人类标注资料会学到奇怪规则，为什么 pretrain 参数是第二阶段成功的关键，以及 adapter、LoRA、FLAN、InstructGPT、LLaMA 这些名字各自处在什么位置。

概念骨架

MAP

第一阶段留下了什么

pretrain 让模型在大量文本中学会复杂的接续规律。它可能知道“台湾最高的山是玉山”“世界最高的山是圣母峰”，也可能掌握很多语言、格式和事实的关联。

但预训练目标通常只是把文字接下去。模型会接话，不等于它知道自己正在扮演一个回答用户问题的助手。

第二阶段补了什么

instruction fine-tuning 用“指令，答案”样本教模型：当输入来自用户，后面应该接 AI 的回应。这个阶段属于 supervised learning，因为训练资料里有人工或外部系统提供的目标答案。

判断句：如果样本没有把角色和答案边界标清，模型会分不清该继续用户的话，还是该开始回答。

为什么不是从零教

人类标注资料很贵，数量远远比不上网络语料。如果只用少量标注样本从随机参数开始训练，模型只要找到能满足训练集的简单规则就够了，不一定学到真实语义。

第二阶段要靠第一阶段的参数作初始点，让优化过程从一个已经懂很多模式的位置出发。

BERT / GPT 早期

用预训练模型做各任务的 fine-tuning，常见做法是打造多个专才。

FLAN / T0

收集大量任务做 instruction tuning，尝试让一个模型完成更多任务。

InstructGPT / LIMA

强调真实用户问题和高质量资料，资料量不必无限大。

LLaMA 之后

开源预训练模型让更多团队能基于 LoRA、adapter 和指令资料训练自己的模型。

术语表

TERMS
术语直觉解释准确说法容易误解的地方
instruction fine-tuning用指令和答案点拨模型。把预训练模型继续放在有目标答案的指令资料上训练，使它更会遵守用户指令。不是把模型从零训练成助手，也不是简单加一条 prompt。
supervised learning有老师给标准答案的学习。训练样本包含输入与目标输出，优化过程让模型输出更接近目标答案。“有老师”不等于老师给了足够多、足够好的世界知识。
pretrain先在大量资料上练内功。用大规模通用语料训练模型，使参数中保留语言、事实、格式和推理模式。预训练模型有能力，但不一定会稳定按照用户意图回答。
fine-tuning在已有参数上小幅调整。以预训练参数为初始参数，在较小、较针对性的资料上继续优化。如果更新过猛或资料很差，微调也可能破坏原有能力。
user / AI 标记告诉模型哪段是人说的，哪段该由助手接。训练样本中的角色边界，让同一段文字在不同上下文下有不同目标输出。去掉角色标记后，模型可能把自问自答和助手回答混在一起。
adapter在原模型旁边加少量可训练部件。冻结大部分原始参数，只训练新增的小规模参数，减少算力并限制偏离。adapter 不是一种固定结构，而是一类参数高效微调方法。
LoRA常见的 adapter 做法。Low-Rank Adaptation，通过低秩矩阵更新近似权重变化，训练参数量远小于全量微调。LoRA 不是提升模型知识的魔法，它仍然依赖基础模型和资料质量。
专才一个模型专门做一个任务。为翻译、编辑、阅读理解等任务分别 fine-tune 出不同模型。专才在自己任务上强，不代表能理解用户所有意图。
通才一个模型学很多任务。把翻译、摘要、编辑、问答等大量 instruction 资料混在一起训练，期待任务间迁移。通才也需要高质量任务分布，不是任务越杂越好。
Self-Instruct让强模型帮忙造指令资料。先让模型生成任务和输入，再让模型生成答案，用这些资料训练另一个模型。有质量、授权、同质化和竞争条款风险。

读完这一段，你应该能把“预训练、指令微调、LoRA、通才”放进同一条训练链路里。

讲义主线

LECTURE

1. 指令资料长什么样

人类老师要先想出用户可能问的问题，再准备一个合适答案。比如用户问“台湾最高的山是哪座？”目标答案是“玉山”。对语言模型来说，这不是一张问答卡片，而是一段要继续接下去的文本：user: 台湾最高的山是哪座？ AI: 后面应该接 玉山，再接结束符。

这个设计把“聊天”改写成模型熟悉的文字接龙任务。模型看到 user 的内容和 AI 的起始标记，就知道接下来应该生成 AI 的回答，而不是继续模仿用户自言自语。你现在应该能做的事：给任意一个问答例子补上角色标记和目标输出。

2. 为什么要标出 user 和 AI

同一句文字在不同角色边界下，正确接续可能完全不同。如果没有标记，台湾最高的山是哪座？玉山 可能表示 AI 已经回答完，下一步该输出结束；也可能表示用户在自问自答，AI 可以回应“是的”。

角色标记不是装饰，而是训练目标的一部分。它让模型知道哪部分是条件，哪部分是要学习的答案。判断句：如果一条 instruction 资料无法说清“谁在说话、模型该接哪里、何时结束”，它就不是稳定的训练样本。

3. 只靠人类标注为什么不够

人类标注资料很贵。就算收集上百万笔对话，对理解世界仍然很少。模型训练时只关心找到一组满足训练资料的参数，不会自动按人类觉得合理的方式归纳。

如果训练集中只有“台湾最高的山，玉山”，模型可能学到“看到最就答玉山”这种简单规则。它在训练集上对，却在“世界最深的海沟”这类新问题上崩掉。

4. pretrain 参数为什么关键

pretrain 后的参数已经记录了很多复杂规律。以它作为初始参数，微调时就比较不容易跑到“看到最就答玉山”这种离谱参数区。

这不是说预训练保证正确，而是说优化起点更好。模型原本可能已经知道圣母峰、马里亚纳海沟等事实，instruction fine-tuning 只是在教它如何把这些能力按用户指令调出来。

5. fine-tuning 为什么叫微调

第二阶段不是把模型推倒重练，而是在第一阶段参数附近继续优化。初始参数决定了优化从哪里开始，也影响最后会落到哪个解。

如果你担心参数偏离太多，可以使用 adapter 或 LoRA：冻结原模型大部分参数，只训练少量新增参数。这样既省算力，也能让新模型保留更多基础能力。

6. adapter 与 LoRA 的实际意义

对普通学习者来说，全量更新一个大模型通常算不动。LoRA 这类方法把训练目标缩小到少量参数，使免费或低成本算力也能做一部分优化。

判断句：如果你看到作业或项目要求用 LoRA，不要只理解为“某个库参数”，它是在解决大模型参数太多、全量训练太慢的问题。

7. 举一反三来自哪里

字幕里提到多语言 BERT 的例子：模型在多语言语料上预训练后，只教它某一种语言的阅读理解任务，它可能迁移到另一种语言的同类任务。

这说明 instruction fine-tuning 的效果不只来自标注样本本身，也来自预训练阶段已经学到的表示。少量指令像钥匙，真正的能力储备在预训练参数里。

8. 专才和通才是两条路线

专才路线为每个任务训练一个模型，比如翻译模型、编辑模型、阅读理解模型。BERT 时代常见的做法就是把预训练模型 fine-tune 成不同任务的专才。

通才路线把很多任务合在一起教同一个模型。FLAN、T0、InstructGPT、PaLM instruction tuning 都在探索这条路。目标不是让模型背所有模板，而是让它看到很多任务后学会新的任务形式。

完整知识链路

本节可以这样串起来：第一阶段 pretrain 让语言模型从大量文本中学到复杂接续规律，但它不一定知道如何扮演一个听指令的助手。第二阶段用 instruction fine-tuning 把用户指令和目标回答整理成 supervised learning 资料，明确 user 和 AI 的角色边界，让模型在预训练参数附近学会更稳定地回应人类请求。因为人类标注资料少，模型不能从零学世界知识，所以 pretrain 参数、资料质量和微调方式都很重要。adapter 和 LoRA 让微调更省算力，也限制模型偏离基础能力。沿着这条线，研究者先打造任务专才，后来尝试用 FLAN、T0、InstructGPT 等方法打造通才；LLaMA 开放后，Alpaca、Vicuna 等项目让更多团队能基于开源预训练模型和指令资料做自己的模型。

课程线索

内容线索

课程线索一：从“会接话”到“会回答”

字幕一开始强调，第一阶段让模型累积了很多实例，却“不知道使用方法”。这句话的学习价值在于区分能力储备和行为对齐。模型可能有知识、会写通顺文本，但没有经过指令微调时，用户问问题它未必按对话助手的方式答。

你现在应该能做的事：看到一个基础模型乱接话时，不急着说“它没知识”，先判断它是知识不足，还是没有学会指令格式。

课程线索二：资料格式决定训练目标

老师用“台湾最高的山是哪座？”和“你是谁？”说明，问答样本会被转成文字接龙格式。关键不是例子本身，而是训练样本必须明确角色、答案和结束位置。

你现在应该能做的事：检查一份 instruction dataset 时，先看是否有稳定的 role 字段、是否把用户输入和 assistant 输出分开。

课程线索三：少量标注会诱发捷径

“看到最就答玉山”是一个典型捷径规则。它提醒我们：模型只要能降低训练损失，就可能选择人类没想到的归纳方式。训练资料少、覆盖窄时，这种风险更高。

你现在应该能做的事：评价微调结果时，不只看训练样本是否答对，还要拿相邻但不同的问题检查泛化。

课程线索四：资料质量比数量更硬

InstructGPT 用真实用户问题，LLaMA 相关工作提到约 2.7 万笔高质量资料，LIMA 甚至讨论约 1000 笔资料也可能产生不错效果。这里的重点不是“越少越好”，而是指令资料要贴近真实使用场景。

你现在应该能做的事：比较两个 instruction dataset 时，问它们是否真实、多样、有清楚答案，而不只看样本数量。

核心概念精讲

WORKED

核心直觉：第二阶段是在教“该怎么回答”

预训练模型已经会从大量文字里接下一个 token，但它不知道所有输入都应该被当成用户请求。Instruction fine-tuning 的作用，是把“用户说什么”和“助手应该怎样接”整理成训练信号，让模型把已有知识以更像助手的方式调出来。

你现在应该能做什么：解释为什么一个会背事实的预训练模型，还需要指令微调才能成为聊天助手。

易混边界：Prompt 技巧不会更新参数

Prompt 是推理时临时给模型看的上下文；instruction fine-tuning 是训练时改变模型参数或 adapter 参数。二者都能影响回答风格，但发生位置不同：prompt 改的是当次输入，微调改的是模型以后遇到类似输入时的默认行为。

你现在应该能做什么：看到一个改进方案时，先判断它是在改输入、改数据，还是改参数。

Worked example：角色标记怎样变成训练目标

样本 user: 把 Good morning 翻译成中文。 assistant: 早安。 在自回归训练中会被拆成许多下一 token 预测。模型看到 assistant: 之前的上下文后，目标是依次预测“早”“安”“。”和结束位置。若去掉 user 与 assistant，模型就不清楚“早安”是用户自答、助手回答，还是一段普通文本。

你现在应该能做什么：为任意问答样本标出条件、目标输出和结束边界。

最小例子

EXAMPLE

下面用一个小样本看懂第二阶段的训练格式。重点不是代码库，而是每一行代表什么训练信号。

正确示例：把问答转成 instruction fine-tuning 样本jsonl
{"messages":[
  {"role":"user","content":"台湾最高的山是哪座？"},
  {"role":"assistant","content":"玉山。"}
]}
{"messages":[
  {"role":"user","content":"请把 Good morning 翻译成中文。"},
  {"role":"assistant","content":"早安。"}
]}

逐行拆解

role: user 是条件的一部分，告诉模型这段话来自用户。role: assistant 后面的内容是训练目标，模型要学习在这种上下文后接出这个回答。

如果训练时采用自回归目标，模型会逐 token 学习“玉”“山”“。”以及结束位置。它不是只记住整句答案，而是在每一步学习当前上下文下应该接什么。

错误示例：角色边界丢失

台湾最高的山是哪座？玉山。 这样的文本没有告诉模型“玉山”是用户自答，还是助手回答。下一步该接结束、赞同、解释，都会变得含混。

判断句：instruction data 的格式错了，后面再调学习率或 LoRA rank，也很难补回目标定义的混乱。

概念示例：adapter / LoRA 只训练少量新增参数python
base_model = load_pretrained_model("llama-like-base")
freeze(base_model.parameters())

lora = attach_lora_adapters(
    base_model,
    target_modules=["q_proj", "v_proj"],
    rank=8
)

train(lora.trainable_parameters(), instruction_dataset)

读完例子，你应该能解释：预训练模型提供初始能力，instruction_dataset 提供点拨方向，LoRA 让训练只改少量参数。

误区诊断

DIAGNOSIS

五个常见坑

  1. 把 instruction fine-tuning 当成 prompt 技巧。错因是混淆训练时更新参数和推理时提供上下文；修法是看模型参数有没有被继续优化。
  2. 以为人类标注越多越好。错因是只看数量；后果是大量模板化问题可能让模型远离真实用户需求；修法是检查真实性、多样性和答案质量。
  3. 忘记 user / AI 标记。错因是把聊天资料当普通文本；后果是模型不知道该继续谁的话；修法是保持稳定角色格式。
  4. 以为 LoRA 会自动让模型变强。错因是把训练方法当成资料质量；后果是低质量资料也会被学进去；修法是先看基础模型和数据，再谈参数效率。
  5. 拿开源模型随便接 Self-Instruct 资料就上线。错因是忽略授权、偏差和重复生成；后果是质量不稳或踩到使用条款；修法是做来源审计和人工抽检。
