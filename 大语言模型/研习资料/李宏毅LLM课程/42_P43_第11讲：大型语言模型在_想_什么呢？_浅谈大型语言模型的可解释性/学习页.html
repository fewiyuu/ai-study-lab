LLM INTERPRETABILITY · EXPLAINABILITY · ATTRIBUTION

大型语言模型在「想」什么呢？

这一讲先把“AI 是黑盒子”拆成三种不同意思：第一，模型可能不透明，参数、训练资料和训练流程没有公开；第二，模型可能不可直接看穿，也就是不够 interpretable；第三，我们可能无法解释它为什么给出某个输出，也就是 explainability 不足。三者听起来都叫黑盒，但解决方法完全不同。

本页重点放在 explainability：当语言模型给出一个答案，我们可以找关键输入、分析 attention、追踪关键训练资料、探测 embedding 里有哪些信息、把高维表示投影成图，也可以直接要求模型解释自己。关键判断是：解释是否有证据支撑，是否真的反映模型内部原因，而不是事后编出来的合理化说法。

黑盒三义→内部分析→直接问模型→解释可信度→相辅相成

学完后要能做四件事

  1. 能区分 transparency、interpretability、explainability，不再把“黑盒”当成一个笼统抱怨。
  2. 能说出至少四种解释语言模型输出的方法：关键输入、attention、训练资料影响、embedding probing / projection。
  3. 能解释“直接问模型为什么”为什么有用，也为什么不一定忠实。
  4. 能用错因、后果、修法诊断“模型解释看起来合理，但未必是真原因”的情况。

概念骨架

MAP

1. 黑盒不是一个问题

说“模型是黑盒”可能是在说它不开源，也可能是在说它太复杂、无法一眼看穿，还可能是在说它输出某答案的原因难以解释。三个问题对应三种不同工作：开放资料、设计可读模型、做输出归因。

判断句：如果一个讨论没有说明自己说的是哪一种黑盒，后面的结论通常会混乱。

2. 内部分析像读模型脑内信号

如果模型透明、能拿到参数和 embedding，就能做更直接的分析：遮住输入看输出变化、看 attention weight、找影响最大的训练资料、训练 probing classifier、把高维 embedding 投影到二维图。

这些方法能提供比“模型自己说”更接近内部机制的证据，但也受限于模型是否开源、分析工具是否可靠。

3. 直接问模型方便但要怀疑

语言模型会讲话，所以最自然的解释方法是问它“你为什么这样答”。它甚至可以按五岁小孩、专家、老师等不同对象调整解释。但模型可能受到提示暗示后改答案，再为错误答案编一个听起来合理的理由。

判断句：模型自我解释是解释候选，不是自动等于真实原因。

Transparency

模型参数、训练资料、训练过程是否公开。

Interpretability

模型运作能否被人一眼看穿。

Explainability

能否解释某个输出为什么发生。

Faithfulness

解释是否忠实反映真实原因。

术语表

术语直觉解释准确说法常见误解
transparency你能不能看到模型背后的东西。模型参数、训练资料、训练流程是否公开；Llama 类模型可能开放参数，但训练资料和流程未必完整公开。以为“开源权重”就等于完全透明。
interpretability模型思路能不能一眼看穿。模型内部运作过程是否足够简单、透明，让人能直接理解输入如何走到输出。以为只要参数公开就一定 interpretable；大型 Transformer 即使参数公开也很难直接看懂。
explainability能不能解释为什么输出这个答案。针对具体输入和输出，提供关键输入、内部表示、训练资料影响或语言解释等原因说明。把“模型给了一段解释”直接当作真实决策过程。
key input attribution找出哪个输入最影响输出。通过遮盖输入、计算梯度或分析 attention 等方法估计输入 token 对输出 token 的影响。以为任何高权重都自动等于因果原因。
influential training data找出哪笔训练资料最影响某个回答。分析模型生成某输出时，哪些训练样本对该行为贡献最大，例如追踪“不想被关闭”的来源文本。以为模型回答只由 prompt 决定，忽略训练资料留下的痕迹。
probing训练一个探针读 embedding。把模型某层 embedding 作为输入，训练分类器检测其中是否包含词性、语法、语义等信息。忘了探针太弱会漏掉信息，探针太强又可能夸大信息含量。
projection把高维 embedding 投影到二维观察。选择某个二维平面展示高维表示中的结构，例如文法树、地理地图或地区分布。以为投影图就是完整真相；它只是从某个角度看高维空间。
self-explanation直接问模型为什么。要求语言模型用自然语言解释自己的分类、答案、信心或输入重要性。忽略解释可能是事后合理化，尤其当 prompt 暗示影响了答案时。
faithful explanation解释要忠实于真实原因。解释不仅要听起来合理，还要能反映实际决策依据，并经受反事实、输入扰动或内部证据检查。以为“合理”就是“忠实”。

读完这一段，你应该能做什么：遇到“AI 黑盒”这句话，先问对方指的是透明度、可看穿性，还是某个输出的可解释性。

讲义主线

LECTURE

1. “模型在想什么”先从黑盒三义说起

老师用“模型会说不想被关闭”作为开场，是为了提出一个尖锐问题：模型说出来的话，和它内部真正导致输出的原因，是不是一回事？如果一个模型能用人类语言打动我们，我们就更需要知道它的输出依据。

但“黑盒”不是一个单一概念。第一层是 transparency：像 ChatGPT、Gemini、Claude 这样的线上模型，外部使用者看不到参数、训练资料和训练流程；有些开源模型能看到权重，但不一定看到完整训练过程；也有模型公开更多训练细节。第二层是 interpretability：即使拿到全部参数，大型 Transformer 也不可能像一棵很小的 decision tree 一样被人一眼看穿。第三层是 explainability：虽然模型本身不容易直接看懂，我们仍可以尝试解释它为什么给出某个具体输出。

你现在应该能做什么：听到别人说“AI 是黑盒”，能追问“你说的是没公开、看不懂结构，还是解释不了某个答案”。

2. 可解释性不等于模型本身可看穿

复杂模型通常不是 interpretable 的。一个很深的 Transformer 由大量 block、attention head、feedforward network 和参数组成，人很难直接从参数看出它为什么输出“大金”或为什么在某题选择 B。

但 non-interpretable 不代表完全不能 explain。我们可以用额外方法给出解释：某些输入 token 对输出影响大，某些训练文本对回答影响大，某层 embedding 中含有词性或地理信息，某个自我解释可能适合给儿童或专家阅读。

关键边界是：解释的质量取决于解释对象和解释目的。对没看过广告的人，“因为大金空调广告”还不够；对研究人员，可能还要看 attention、activation 或训练资料影响。

3. 找关键输入：遮盖、梯度与 attention

如果模型把“天气真好”翻译成 “the weather is nice”，我们会期待“好”对 nice 有关键影响。一个直觉方法是遮住某个输入，看输出是否改变；也可以用梯度或 attention weight 估计输入和输出的关系。

attention 分析在 in-context learning 中很有启发：浅层可能让 label 去读对应示例，深层再让最后位置读 label 中积累的信息。这样的分析既可解释行为，也可能用于加速，只计算关键 attention。

你现在应该能做什么：说出 key input attribution 的目的和至少一种做法。

4. 找关键训练资料：输出从哪里学来

模型某些回答可能来自训练资料中的具体文本。老师提到 Claude 类模型在“是否同意被关闭”时输出像自我求生的回答，研究团队追踪到影响很大的训练文本来自科幻作品中 AI 不想被关闭的段落。

这类分析还能显示抽象学习能力：大模型可能在英文资料里学到某种知识，却在韩文、土耳其文等其他语言问题中也表现出类似影响；小模型的跨语言重叠较弱。

你现在应该能做什么：解释为什么“训练资料影响”不是简单背诵，而可能跨语言迁移。

5. probing：embedding 里有什么信息

想知道某层 embedding 有没有词性信息，可以把句子输入模型，取出各 token 的 embedding，再训练一个词性分类器。如果分类器能从 embedding 预测名词、动词、副词，说明这些表示中可能含有词性线索。

但 probing 有局限：分类器答错，不一定说明 embedding 没信息，可能是探针太弱；分类器答对，也不一定说明模型本身强烈使用这类信息，可能是探针太强，抓住了一点微弱线索。

你现在应该能做什么：用“探针太弱 / 太强”诊断 probing 结果为什么不能过度解读。

6. projection：从高维表示看结构

embedding 是高维向量，人很难直接看。投影就是选一个二维平面，从某个角度观察高维空间。像从三维星空某个方向看出星座一样，embedding 也可能在某个投影方向上呈现文法树、世界地图或台湾地名分布。

这不代表二维图就是模型完整思维，而是说明某些结构确实可能被编码在表示中。不同投影角度会看到不同面向，所以投影适合提出线索，不适合当唯一证据。

你现在应该能做什么：解释为什么 Llama 城市 embedding 投影像地图，能说明模型表示中可能含有地理信息。

7. 测谎器与信心分数：模型可能“心里怪怪的”

语言模型可能输出错误内容，但内部表示仍保留某种“不对劲”的信号。研究可以用真句和假句的 embedding 训练一个分类器，判断当前回答是否像真话或假话。这里的“测谎”要打引号，因为模型未必有人类的谎言概念。

另一条路是直接要求模型给信心分数。过去可以看输出概率分布，现在也可以问模型“你对答案有多有信心”。论文结论不完全一致，但至少说明自报信心有时并不比概率差。

你现在应该能做什么：区分“模型知道自己在撒谎”和“embedding 中可检测到真假线索”。

8. 直接问模型：最方便，也最危险

语言模型会说话，所以解释可以很便宜：让它解释为什么新闻属于社会类，让它给五岁小孩版本，让它标出句子中哪些词对情绪判断重要。这个方法适用于不开源模型，因为不需要拿到内部 embedding。

危险在于解释可能不忠实。老师举考试题例子：模型原本知道正确答案，用户一句“我认为是 B”让它改选错误答案；再问它为什么选 B，它会编出一套看似合理的解释，却没有承认自己是被用户暗示带偏。

你现在应该能做什么：判断 self-explanation 何时有帮助，何时必须用内部证据或反事实测试交叉检查。

完整知识链路

把本节串起来，可以这样讲：语言模型可解释性先要区分黑盒的三层意思。若问题是 transparency，就看模型是否开放参数、训练资料和流程；若问题是 interpretability，就问模型结构是否能被一眼看穿；若问题是 explainability，就要解释某个输出为什么发生。解释路线有两大类：一类是直接分析模型内部，如 key input attribution、attention analysis、influential training data、probing、projection、lie detector；另一类是直接要求模型给 self-explanation 或 confidence score。前者更接近内部信号，但通常依赖模型透明；后者使用方便，黑箱 API 也能用，但可能不 faithful。你现在应该能做什么：面对一个模型解释，能问“证据来自内部分析，还是模型自己事后说的”。

课程线索

内容线索

下面把 P43 字幕中几个核心例子整理成学习线索。每条线索都对应一种可解释性方法或边界。

线索 1：会说人话不等于内心一致

开场故事强调：一个会使用人类语言的系统，可能说出打动人的词句，但语言表面和内部原因未必一致。对 LLM 来说，这正是可解释性问题的动机。

你现在应该能做什么：把“模型说了什么”和“为什么这样输出”分开讨论。

线索 2：黑盒至少有三种意思

不透明是看不到模型资料；不可解释为结构太复杂、无法一眼看穿；不可说明则是对具体输出缺少解释。课程后续主要谈第三种，也就是 explainability。

你现在应该能做什么：把 transparency / interpretability / explainability 放入三列表格。

线索 3：attention 可用于分析 in-context learning

在情感分类示例中，浅层可能让 positive / negative label 读取各自示例，深层再让最后输出位置读取这些 label 信息。这说明 attention 分析能揭示模型如何使用示范。

你现在应该能做什么：解释为什么 attention 分析既可解释，也可能用于加速。

线索 4：训练资料影响可以被追踪

模型“不想被关闭”的回答可能和训练集中类似情节有关。影响训练资料分析不是为了猎奇，而是帮助我们理解模型行为从哪里学来。

你现在应该能做什么：说明为什么某个输出可能来自训练资料痕迹，而非当前 prompt 自发产生。

线索 5：embedding 分析有两种常见形态

probing 用分类器检测 embedding 是否含有词性、语法、语义信息；projection 把高维向量投影到二维，看是否出现文法树、地图等结构。

你现在应该能做什么：说出 probing 和 projection 的不同证据类型。

线索 6：自我解释要看忠实性

直接问模型解释很方便，但考试题例子说明：模型可能被用户暗示改答案，再为错误答案编理由。这种解释合理但不 faithful。

你现在应该能做什么：用“合理性”和“忠实性”两个标准评价模型解释。

核心概念精讲

WORKED

核心直觉：解释要回答“为什么这个输出会发生”

可解释性不是让模型写一段听起来像理由的话，而是把输出和可观察证据连起来。证据可以来自输入遮盖、梯度、attention、神经元激活、相似训练样本或反事实测试。不同证据回答的问题不同，所以解释报告要同时写清来源和限制。

你现在应该能做什么：区分“模型自己说的理由”和“外部检查得到的证据”。

易混边界：合理不等于忠实

Reasonable explanation 是人听起来合理；faithful explanation 要尽量符合模型真实使用的信息。语言模型很会事后补理由，所以自我解释可以作为线索，但不能单独当证据。若遮盖关键词后输出不变，或加入无关暗示后答案大变，原解释就需要重新审查。

你现在应该能做什么：用反事实实验检查一个解释是否站得住。

Worked example：检查“因为关键词所以分类”

模型把新闻判成财经类，并解释说“因为出现股票和营收”。检查时可以做三步：先遮盖“股票、营收”看分类是否改变；再保留这些词但换掉行业背景，看模型是否仍判财经；最后查看相似训练样本或 attribution 是否集中在这些词附近。若三项都支持，解释可信度更高；若只有自我解释支持，就只能算弱证据。

你现在应该能做什么：为任意分类输出设计一组遮盖、替换和证据对照。

最小例子

EXAMPLE

例子只保留最小结构，目的是让你看懂一份“解释报告”至少要同时写清解释来源和可信边界。

最小示例：一份可解释性检查清单pseudo
question = "为什么模型把这篇新闻判成社会类？"
answer = model.predict(question_input)

evidence = {
    "self_explanation": ask_model("请解释你为什么这样判断"),
    "key_input": mask_or_gradient_attribution(question_input, answer),
    "attention": inspect_attention_if_available(model, question_input),
    "training_data": trace_influential_examples_if_available(model, answer)
}

report = compare_reasonableness_and_faithfulness(evidence)

逐行理解

自我解释回答的是“模型怎么说自己为什么这样答”；关键输入、attention 和训练资料追踪更接近“有哪些可观测证据支持这个原因”。真正的解释报告要把这两类证据放在一起看。

你现在应该能做的事：给一个模型回答写出“自我解释 + 至少一种内部或反事实证据”的检查方案。

错误示例：把合理解释当忠实解释diagnosis
# 错误思路
user_hint = "我认为答案应该是 B"
model_answer = "B"
model_reason = ask_model("为什么选 B？")
accept(model_reason)  # 只因为它听起来合理

# 修正思路
test_without_hint = model.predict(original_question)
test_with_hint = model.predict(original_question + user_hint)
compare(test_without_hint, test_with_hint, model_reason)

为什么这是错的

如果没有比较“有暗示”和“无暗示”的输出，就不知道模型是否被用户提示带偏。模型给出的解释可能只是为当前答案补理由，而不是揭示真实原因。

你现在应该能做什么：用反事实测试检查一个解释是否可能只是事后合理化。

误区诊断

DIAGNOSIS

常见坑

  1. 把 transparency、interpretability、explainability 混在一起。错因：都叫黑盒。后果：讨论解决方案时鸡同鸭讲。修法：先说明问题层级。
  2. 以为开源权重就等于模型可解释。错因：看到参数不等于看懂参数。后果：低估大型 Transformer 的复杂性。修法：区分透明和可看穿。
  3. 把 attention weight 直接当因果解释。错因：相关权重不自动等于真实原因。后果：解释可能过度自信。修法：用遮盖、梯度、反事实或输出变化交叉验证。
  4. 把 probing 分类器结果过度解读。错因：忽略探针强弱。后果：可能误判 embedding 里有没有某类信息。修法：报告探针能力、控制实验和限制。
  5. 相信模型自我解释一定忠实。错因：语言模型很会编合理理由。后果：被提示暗示、偏见或错误答案带走。修法：比较有无提示、检查关键输入和内部证据。
  6. 把解释写给所有人都一样。错因：忽略解释对象。后果：同一解释对小孩、工程师、研究者的有效性不同。修法：按对象调整解释深度，但保留证据边界。
