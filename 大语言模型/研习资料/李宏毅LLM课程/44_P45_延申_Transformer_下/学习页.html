44_P45｜（延申）Transformer (下)

P44 讲 encoder 怎样把输入序列变成一排向量；P45 讲 decoder 怎样把这排向量变成输出序列。以语音辨识为例，encoder 读入声音讯号，decoder 从 BOS 这个开始符号出发，每一步输出词表上的 probability distribution，选出一个 token，再把这个 token 当成下一步输入，直到输出 END 停止符。

概念骨架

MAP
  1. decoder 负责生成输出序列encoder 已经把输入变成一排向量，decoder 要根据这些向量产生文字、字母、语音帧等输出。AR decoder 每次只产生一个 token，再把自己的输出喂回下一步。判断句：如果解释 decoder 时没有提 BOS、上一步输出和 END，就还没有讲清生成过程。
  2. decoder 有两种路线auto-regressive decoder 一步一步生成，质量通常较稳，但速度受输出长度限制；non-autoregressive decoder 一次并行生成整句，速度快、长度可控，但性能通常更难追上 AR。判断句：NAT 的快来自并行生成，不是因为它更懂语言。
  3. 训练和推理不完全一致训练时常用 teacher forcing，把正确答案前缀喂给 decoder；推理时 decoder 只能看到自己过去产生的 token。这个 mismatch 叫 exposure bias，会导致一步错步步错。判断句：如果训练只让 decoder 看正确前缀，测试时遇到错误前缀会很脆。

auto-regressive decoder一个 token 一个 token 接龙。每一步把先前输出当作输入，产生下一个 token 的分布，直到输出 END。以为它一次输出完整句子。

vocabularydecoder 能输出的单位清单。输出向量长度等于词表大小，每个维度对应一个字、词、subword 或特殊符号。以为输出向量直接就是文字。
masked self-attention只看已经生成的左边。decoder 的 self-attention 在位置 t 只能参考 1 到 t 的输入，不能参考未来 token。把它和 encoder 的完整 self-attention 混淆。
cross attentiondecoder 向 encoder 查资料。Q 来自 decoder，K/V 来自 encoder；decoder 用当前状态去抽取输入序列信息。以为 decoder 完全靠自己历史输出，不看 encoder。
NAT一次并行生成整句。non-autoregressive decoder 可吃一排 BOS 或长度提示，一步产生完整输出。以为 NAT 既快又天然比 AR 准；实际上常要很多技巧追性能。
teacher forcing训练时把正确答案前缀喂给 decoder。用 ground truth token 作为 decoder 输入，使每一步分类训练更稳定。忘了推理时没有正确答案可喂，产生 exposure bias。
cross entropy每一步分类的训练目标。让 decoder 输出的 probability distribution 接近正确 token 的 one-hot vector。以为最小化 cross entropy 必然最大化 BLEU。
beam search保留多个候选路径近似找好序列。比 greedy decoding 多看几条可能路径，但不穷举所有组合。以为 beam search 对所有生成任务都更好；创造性任务可能更需要随机性。

讲义主线

LECTURE

1. AR decoder 怎样一步步生成

语音辨识例子里，encoder 先把声音讯号变成一排向量。decoder 的工作是产生“机器学习”这串文字。它一开始不会凭空输出，而是先吃一个特殊 token：BOS，表示“请开始”。

decoder 输出的长度等于 vocabulary size 的向量。经过 softmax 后，这个向量是一个分布：每个中文字、subword 或特殊符号都有一个概率。概率最高的 token 被选为当前输出，例如先输出“机”。然后“机”会被当作下一步输入，和 BOS 一起帮助 decoder 继续输出“器”。

这个过程一直持续，直到 decoder 输出 END。END 和 BOS 一样是特殊符号，专门用来告诉模型“句子结束”。你现在应该能做什么：把 decoder 的生成过程解释成“上一步输出进入下一步”的文字接龙。

2. masked self-attention 为什么必要

encoder 可以一次读完整个输入序列，所以它的 self-attention 可以让每个位置看见所有位置。decoder 不一样：生成第二个 token 时，第三个、第四个 token 还不存在，不能偷看未来。

masked self-attention 的规则是：产生第 t 个位置的表示时，只能看 1 到 t 的输入。产生 b1 只能看 a1；产生 b2 只能看 a1、a2；产生 b3 才能看 a1、a2、a3。这个 mask 让训练时的并行计算仍符合推理时的自回归顺序。

你现在应该能做什么：把 encoder self-attention 和 decoder masked self-attention 的差异讲清楚。

  1. cross attention 是 encoder-decoder 桥梁decoder 不能只看自己过去输出，它还要读 encoder 处理过的输入。cross attention 的做法是：query 来自 decoder 当前状态，key 和 value 来自 encoder output。decoder 用自己的 Q 去和 encoder 的 K 算 attention 分数，再用这些分数加权 encoder 的 V，抽取当前生成步骤需要的信息。语音辨识中，这就像 decoder 生成某个字母或中文字时，把注意力对准对应时间段的声音讯号。你现在应该能做什么：用一句话说明“Q 来自 decoder，K/V 来自 encoder”。
  2. END 决定序列何时停seq2seq 的输出长度不能从输入长度直接算出来，所以 decoder 必须学会停止。做法是在词表里加入 END 特殊符号，训练时要求正确输出序列之后再输出 END。如果没有 END，decoder 就会像推文接龙一样一直接下去。BOS 和 END 可以用同一个符号或不同符号实现，关键是它们分别只在开始输入和结束输出时承担角色。你现在应该能做什么：解释为什么 decoder 的 vocabulary 里除了普通 token，还要有特殊符号。
  3. NAT：一次生成整句non-autoregressive decoder 不像 AR 一步一步接龙，而是一次吃一排 BOS 或长度提示，同时产生整句。输出长度可以由额外 classifier 预测，也可以给一个长度上限，再用 END 截断右侧内容。NAT 的优势是平行化，输出 100 个 token 也可以一个步骤生成；在语音合成中还可以显式控制输出长度，让说话速度变快或变慢。但 NAT 的性能通常不如 AR，需要许多技巧弥补。你现在应该能做什么：说清 NAT 快在哪里，以及难在哪里。
  4. 训练：每一步都是分类训练语音辨识时，需要声音讯号和正确文字。若正确答案是“机器学习 END”，训练会要求 decoder 在看到 BOS 时输出“机”，看到 BOS+机 时输出“器”，依此类推，最后输出 END。每一步的目标 token 都可以写成 one-hot vector；decoder 输出的是 softmax 分布。训练最小化每一步分布和正确 one-hot 之间的 cross entropy，总和越小越好。你现在应该能做什么：把 decoder 训练解释成一连串 vocabulary 分类问题。
  5. teacher forcing 与 exposure bias训练时，decoder 的输入通常是正确答案前缀，这叫 teacher forcing。它让训练稳定，因为每一步都在干净上下文中学预测下一 token。推理时没有正确答案可喂，decoder 只能吃自己刚刚输出的 token。如果前一步错了，后续输入就变脏，可能一步错步步错。这种训练和推理输入分布不一致，就是 exposure bias。schedule sampling 的思路是训练时偶尔给 decoder 一些错误或模型生成的 token，让它学会处理不完美前缀。你现在应该能做什么：解释 teacher forcing 为什么方便，也为什么带来风险。
  6. 解码和技巧：copy、guided attention、beam searchcopy mechanism 让模型从输入复制词汇，适合人名、罕见词和摘要中直接沿用原文。guided attention 用在语音辨识或语音合成时，鼓励 attention 从左到右移动，避免漏念或漏听。beam search 比 greedy decoding 保留更多候选路径，可能找到整体概率更好的句子。但它不是万能：答案明确的任务如语音辨识常受益；需要创造力的 sentence completion 或 TTS 反而可能需要随机性，否则会重复、僵硬或听起来不自然。你现在应该能做什么：根据任务类型判断是否应该用 beam search 或加入随机性。

完整知识链路

把本节串起来，可以这样讲：Transformer 的 encoder 处理输入序列，decoder 负责产生输出序列。AR decoder 从 BOS 开始，每一步通过 masked self-attention 读取已生成前缀，通过 cross attention 用自己的 Q 去读 encoder 的 K/V，再经过 feedforward 和 softmax 输出词表上的概率分布。输出 token 会被喂回下一步，直到 END 停止。训练时用 teacher forcing 给正确前缀，最小化每一步 cross entropy；但推理时只能看自己的输出，因此会有 exposure bias。为了让 seq2seq 更可靠，还会用 copy mechanism、guided attention、beam search、schedule sampling，甚至用 RL 处理 BLEU 这类不可微评估目标。你现在应该能做什么：把 decoder 的推理和训练分开讲，并指出它们哪里不一致。

核心概念精讲

WORKED

核心直觉：Decoder 用已有前缀一步步生成未来Decoder 的任务是生成输出序列。第一个输出依赖 BOS 和 encoder 信息；第二个输出依赖 BOS、第一个输出和 encoder 信息；如此重复直到 END。Masked self-attention 保证当前位置不能偷看未来 token，cross attention 让 decoder 能读输入侧的信息。你现在应该能做什么：画出 BOS -> 第 1 个 token -> 第 2 个 token -> END 的生成链。

易混边界：训练前缀和推理前缀不同训练时常用 teacher forcing，把正确答案前缀喂给 decoder，所以每一步都在干净上下文里学下一个 token。推理时没有正确答案，只能把模型自己刚生成的 token 放回前缀；一旦前面错了，后面就会在错误上下文上继续生成，这就是 exposure bias 的来源。你现在应该能做什么：解释为什么训练 loss 很低的模型，实际生成仍可能连锁出错。

Worked example：语音辨识生成“机器学习”Encoder 先把音讯特征压成 encoder_out。Decoder 从 BOS 开始，第一步通过 cross attention 听输入，选择“机”；第二步前缀变成 BOS, 机，再选择“器”；后面依次生成“学”“习”和 END。若第一步误生成“几”，后面可能继续生成“何学习”之类错误，因为后续前缀已经变了。你现在应该能做什么：说明 beam search 为什么保留多个候选前缀，能降低早期单步选错的风险。

最小例子

EXAMPLE

下面用伪代码把 AR decoder 的推理和训练分开写。注意：推理看自己的输出，训练看正确答案前缀。

代码
encoder_out = encoder(audio_vectors)
prefix = [BOS]

while True:
 h = decoder.masked_self_attention(prefix)
 ctx = decoder.cross_attention(query=h[-1], key_value=encoder_out)
 probs = softmax(output_layer(ctx))
 y = choose_token(probs)
 if y == END:
 break
 prefix.append(y)

为什么要小心

teacher forcing 让训练清楚稳定，但推理时 prefix.append(gold) 这一步会变成 prefix.append(model_output)。这就是训练和测试不一致的来源。

误区诊断

DIAGNOSIS

常见坑

  1. 以为 decoder 一次输出完整句子。错因：忽略 AR 生成。后果：看不懂 BOS、END 和错误传播。修法：一步一步画出 prefix 变化。
  2. 把 masked self-attention 和 encoder self-attention 混淆。错因：都叫 self-attention。后果：不知道为什么 decoder 不能看右边。修法：记住 decoder 未来 token 尚不存在。
  3. 以为 decoder 不需要看 encoder。错因：只关注自身历史输出。后果：看不懂 cross attention。修法：Q 来自 decoder，K/V 来自 encoder。
  4. 忘了训练时 teacher forcing 与推理时自回归不同。错因：只看训练流程。后果：忽略 exposure bias。修法：分别写训练 prefix 和推理 prefix。
  5. 以为 beam search 对任何任务都更好。错因：把最高概率路径等同于人类偏好的输出。后果：开放式生成可能重复僵硬。修法：按任务是否有明确答案选择 greedy、beam 或 sampling。
  6. 以为 cross entropy 最小就等于 BLEU 最高。错因：混淆训练目标和评估指标。后果：模型选择可能不符合实际评估。修法：validation 按任务指标评估，必要时考虑 RL 风格优化。
