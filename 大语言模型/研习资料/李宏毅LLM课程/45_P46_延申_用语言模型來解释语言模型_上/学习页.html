P46 · Interpreting A Language Model With A Language Model

用语言模型解释单个神经元

这一讲介绍 OpenAI 的一个可解释性实验：拿 GPT-2 里的某个神经元，记录它在不同 token 上的 activation，再请 GPT-4 根据这些激活轨迹写出自然语言解释。解释写完并不算结束，还要让模型按照这段解释去“扮演”那个神经元，看预测出的激活值和真实激活值有多接近。

学完后，你应能读懂这个流程里的三个角色：被解释的 GPT-2 神经元、负责写解释的 explainer、负责按解释预测激活的 simulator；也能说明为什么很多神经元解释不出来，以及 explanation score 低时不能直接说“方法没用”。

本节要拿下的能力

  1. 判断一篇可解释性工作到底在解释什么对象：单个神经元、注意力头、整层表示，还是最终输出。
  2. 把 activation 轨迹整理成语言模型能读的输入，并说明为什么常要过滤零值 token。
  3. 用“解释 → 模拟 → 比对真实激活”的链路评价一个解释是否可靠。
  4. 分清成功解释、牵强解释和无法用普通语言描述的神经元行为。

对象先定清楚

这里不是解释 GPT-2 为什么回答某句话，也不是解释整层网络。它解释的是单个神经元：它看到哪些 token 或上下文时会输出较大的数值。

核心直觉：先把“要解释谁”缩小到一个可观察的数值单元，否则后面的解释、验证和打分都会失去对象。

解释来自样本

GPT-4 看不到神经元的权重，也不会直接打开模型内部。它拿到的是一批句子及每个 token 后面的激活值，然后根据这些模式猜测功能。

易混点：这不是读取模型内部真相，而是从行为样本归纳假设；样本偏了，解释也会偏。

评价要靠复现

自然语言解释听起来合理还不够。好的解释应该能作为脚本，让 simulator 对新句子的激活值做出接近真实神经元的预测。

核心直觉：解释不是一句漂亮描述，而是一份“能不能指导别人复现行为”的说明书。

判断句：如果一个解释只能描述几个高亮词，却不能预测新样本上什么时候亮、什么时候不亮，它还只是观察摘要，不是稳定解释。

术语表

TERMS
术语直觉解释准确说法容易错的地方
神经元网络内部的一个数值单元。在某一层中输出一个标量 activation 的计算单元；这里关注的是 GPT-2 某层某编号神经元。把神经元当成一定有清楚语义的“概念开关”。多数神经元未必能被单句解释。
Activation神经元“亮”了多少。给定输入 token 时，神经元输出的数值。OpenAI 的提示中会把它归一化到 0 到 10。以为 activation 高就等于模型最终答案一定受它控制。它只是内部局部信号。
Subject model被拿来观察的模型。实验里主要是 GPT-2。它读很多句子，产生真实神经元激活轨迹。混同 subject model 和 explainer。写解释的不一定是被解释的模型。
Explainer负责看轨迹、写解释的模型。GPT-4 根据 token 与 activation 的对应关系，用自然语言描述神经元可能在检测什么。把 explainer 的文字当事实。它也可能猜错、说得过宽或编出模糊解释。
Simulator按解释扮演神经元的模型。给它一段解释和一个新句子，让它预测最后一个 token 或每个 token 的 activation。score 低时只怪解释差，忽略 simulator 本身也可能不擅长按脚本预测数值。
零值过滤把一直不亮的 token 先拿掉。由于大量 token 的 activation 是 0，提示中常把非零 token 单独再列一次，帮助模型抓住触发条件。过滤后忘了负例，导致解释过宽，例如把 not all 误解释成 all。
Explanation score解释能复现激活的程度。比较 simulator 根据解释预测的 activation 与 subject model 真实 activation 的接近程度，通常压到 0 到 1。把 0.15 当成完全失败。它可能说明很多单神经元本来就难以用语言描述。

读完这些术语后，你可以做一件事：看到一个神经元解释实验时，先标出“谁被解释、谁写解释、谁验证解释”。这三个角色标错，后面所有判断都会歪。

讲义主线

LECTURE

1. 可解释性不是一个单一技术

课程一开始先提醒：可解释性 AI 不是固定的一招。你想解释最终答案，可以看证据、注意力或归因；你想解释模型内部，可以看神经元、注意力头、层表示或子网络。不同对象需要不同方法。

OpenAI 这篇工作把问题收得很窄：给定 GPT-2 的一个神经元，观察它什么时候 activation 高，然后猜它在检测什么。这个目标很小，但也正因为小，才可以做“写解释后再验证”的闭环。

判断时先问对象，而不是先问技术名。若对象是单个神经元，你期待的是“它在什么输入条件下亮”；若对象是最终回答，你期待的是“哪些输入证据影响了输出”。两者不能混在一起。

2. 先收集激活轨迹

实验先让 subject model 读大量句子。对每个句子，记录目标神经元在每个 token 后面的输出数值。为了让 GPT-4 更容易阅读，数值会被归一化到 0 到 10，数值越大表示该神经元越“兴奋”。

单看一个高亮 token 往往不够。课程里提到一个专门预测 and 的神经元：它在读到 in、have、how 时亮，但真正的规律要看下一个 token 都是 and。这说明解释神经元时不能只盯着当前位置，还要看上下文关系。

读到这里，你应能把一条样本写成“token 序列 + 每个 token 的 activation”，并说明哪个位置真正提供了解释线索。

3. Prompt 如何写

GPT-4 需要先知道任务背景：研究者正在分析神经网络里的神经元；接着告诉它输入格式：每个 token 后面有一个 0 到 10 的 activation；最后要求它解释该神经元在寻找什么。

只给完整句子常会让模型被大量 0 干扰，所以还会把非零 token 再列一次。这个步骤不是美化输入，而是在帮 explainer 把注意力放到真正触发神经元的位置。

判断句：若提示里没有说明数值含义，GPT-4 很可能把 activation 当成普通标注或词频，解释就会跑偏。

4. 成功例子长什么样

有些神经元确实能被自然语言抓住。课程里提到“正确神经元”：看到 property、right 等和把事情做对有关的词时会亮。GPT-4 能把它概括为“寻找与正确、恰当、做对有关的词或片语”。

另一个例子是比喻神经元。它在类似 like water mixed with water 这样的比较句中被激活，GPT-4 能把它解释成侦测比喻或比较结构。这里的规律比单词更抽象，但仍然可以用人类语言描述。

判断句：成功解释通常同时满足两个条件：它能覆盖多个正例，也能说出为什么某些相似但不满足条件的句子不该亮。

5. 失败例子为什么难

有些神经元的高亮样本看起来毫无规律。课程里有一个例子，GPT-4 只给出“general concept、title、partial turns”之类很宽泛的说法，几乎等于没解释。后来人工分析发现它可能和前面有错字或罕见词有关，但也不是完全稳定。

还有一个更抽象的神经元，会在序列规律被破坏时亮：前面是数字、总统名单或重复模式，后面突然出现不合规律的项。这个现象对人类也不直观，GPT-4 解释失败并不奇怪。

判断句：当解释只能列出一堆不相干标签，不敢给触发条件，也不能排除反例时，它更像猜测。

6. 解释必须被评分

自然语言解释容易让人误判：一句话听起来合理，不代表它真能描述神经元。OpenAI 的做法是让 GPT-4 按这段解释扮演神经元。比如解释说某神经元检测 -ing 动词，就给它 I am、I am running、I am running to 这样的句子，让它预测最后一个 token 的 activation。

然后把 simulator 预测的数值和 GPT-2 真实神经元输出比对。越接近，explanation score 越高。这个 score 不评价文字是否漂亮，只评价这段解释能不能支持数值预测。

判断句：解释力分数关注“能否复现行为”，不是“解释是否读起来顺”。

7. 分数透露出的趋势

课程里提到两个大趋势。第一，较小模型中的单个神经元有时更容易解释；较大模型里，很多功能可能分散在一组神经元中，单看一个神经元就不明显。第二，同一模型里，低层神经元通常更容易解释，高层神经元更抽象，也更难被短句概括。

实际分数也很克制。那个“正确神经元”的解释分数约 0.42，而一些很难解释的神经元只有 0.14 左右；GPT-4 平均解释分数约 0.15，人类解释平均也只到 0.18 左右。这个结果不该被读成“GPT-4 太弱”，也可能说明多数单个神经元没有清晰的人类语言功能。

判断句：如果人类解释也只比模型略好，就要考虑对象本身是否适合被单神经元、单句子解释。

完整链路可以这样复述：先从 GPT-2 选一个神经元，收集它在许多 token 上的 activation；把高亮轨迹整理成 prompt 给 GPT-4，让它写出神经元功能解释；再让 GPT-4 或另一个模型按解释预测新句子的 activation；最后用真实 activation 比对预测，得到解释力分数。这个流程的强处是可验证，弱处是它仍受样本选择、语言表达能力和 simulator 能力限制。

讲义线索

线索

这里保留字幕里的核心材料，并把口语内容整理成适合回看的线索。

动机

课程从“用 AI 解释 AI”切入，但马上把问题缩小：不是解释整个语言模型，而是解释 GPT-2 某一层某个神经元什么时候会 activation 高。

输入格式

OpenAI 的 prompt 会告诉 GPT-4：下面给的是 token 和 activation，数值范围被归一化到 0 到 10。模型需要根据这些轨迹说出神经元在寻找什么。

零值处理

李宏毅提到自己复现时也观察到：只给完整句子效果不够好。把 activation 为 0 的 token 去掉，再给模型看一次，通常能得到更清楚的解释。

成功案例

课程展示了“正确/恰当”神经元、比喻神经元、重复 pattern 神经元。这些例子能被 GPT-4 找到较稳定的触发条件。

失败案例

错字或罕见词神经元、规律破坏神经元都很难。GPT-4 可能给出很宽泛、很含糊的解释，人类也未必一眼看懂。

评估结论

解释力分数显示：少数神经元有清楚解释，多数神经元的解释分数很低。人类解释平均分也不高，说明问题不只是 GPT-4 不会写。

最小例子

EXAMPLE

下面的例子保留研究流程，不追求复现 OpenAI 的完整实验。关键是看懂每一步读什么、写什么、验证什么。

最小示例：从激活轨迹到解释评分python
subject_model = "GPT-2"
target_neuron = ("layer_31", "neuron_892")

examples = collect_activations(
    model=subject_model,
    neuron=target_neuron,
    texts=corpus,
    normalize_to=(0, 10)
)

prompt = format_for_explainer(
    examples=examples.top_by_activation(),
    include_nonzero_tokens_again=True
)

explanation = gpt4_explainer(prompt)

new_text = "I have an idea, and it may work."
simulated = gpt4_simulator(explanation, new_text)
actual = run_subject_neuron(subject_model, target_neuron, new_text)

score = compare_activation(simulated, actual)

逐行拆解

collect_activations 读取真实 GPT-2 的目标神经元输出，写出一组 token 到数值的记录。它不生成解释，只负责把行为样本拿出来。

format_for_explainer 把记录变成 GPT-4 能读的文本，并额外列出非零 token，减少无关 0 值干扰。少了这一步，模型容易被长句里的大量零值淹没。

gpt4_simulator 不再写解释，而是按解释预测新文本上的 activation。最后的 compare_activation 才是判断解释是否有用的地方。

错误示例：只看高亮词就下结论

如果几个样本里 all 都高亮，就直接说“该神经元检测 all”，可能漏掉前面的 not。这类解释太宽，遇到 All students arrived 可能就错。

修法是加入负例和对照句：既看会亮的输入，也看相似但不该亮的输入。一个解释能排除反例，才更接近神经元真实行为。

Worked example：手算一个简化解释分数

假设真实神经元在四个 token 上的 activation 是 [0, 8, 9, 1]，simulator 按解释预测为 [1, 7, 8, 2]。四个位置的绝对误差分别是 1, 1, 1, 1，平均误差是 1。如果把 0 到 10 的量尺粗略转成分数，可写成 1 - 1/10 = 0.90。

真实论文会用更正式的相关性或解释力指标，不会只用这个玩具公式。但这个小例子说明了评分的直觉：它比较的是“预测曲线是否跟真实 activation 同步”，不是评价解释文字是否顺耳。你现在应该能拿一组真实值和预测值，说明分数高低来自哪里。

误区诊断

DIAGNOSIS

六个常见坑

  1. 把解释对象说大了。本节解释单个神经元，不是解释整个 GPT-2 或最终答案。修法：先写清 layer 与 neuron 编号。
  2. 只看正例。高亮样本只能说明“可能会亮”，不能说明“只在这里亮”。修法：补相似负例。
  3. 把零值过滤当成真相过滤。去掉零值只是方便模型阅读，不代表零值样本没价值。修法：写解释时保留反例意识。
  4. 把 GPT-4 的解释当权威。它会给出宽泛解释，甚至不知道自己在说什么。修法：用 simulator 和真实 activation 比对。
  5. 误读 explanation score。分数低不一定只说明解释写错，也可能是神经元难以语言化或 simulator 不会扮演。修法：拆开三种可能。
  6. 以为每个神经元都对应一个概念。大模型里很多功能可能由一组神经元共同承担。修法：必要时从单神经元转向神经元组或子空间。
