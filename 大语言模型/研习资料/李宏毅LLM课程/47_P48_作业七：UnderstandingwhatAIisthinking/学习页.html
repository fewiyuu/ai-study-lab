47_P48｜作业七：Understanding what AI is thinking

这次作业问的是一个很实际的问题：模型给出正确答案时，它到底是抓住了关键线索，还是刚好猜中？在医疗、法律这类高风险场景里，只给结果不够，还要能说明依据。解释不是装饰，它决定我们能不能信任、排查和改进模型。

概念骨架

MAP

为什么要解释模型答对一道题，可能是抓到关键证据，也可能只是输出分布刚好撞上正确答案。解释要回答“为什么会产生这个结果”，尤其是在错误代价很高的场景里。判断句：如果一个解释不能指向输入中的证据，它最多只是事后说法。

怎么量化解释feature attribution 会把目标 output token 固定住，再估计每个 input token 对它的贡献。结果常画成 saliency map：一个格子就是某个输入 token 对某个输出 token 的重要性分数。判断句：问分数时，先确认目标输出是哪一行，再确认输入 token 是哪一列。易混点：同一个 input token 对不同 output token 的分数可以完全不同，所以不能说 America “全局重要性是 0.308”。

怎么读自然语言解释LLM 可以直接用自然语言解释预测理由，也可以被要求给每个 token 打 importance score。自然语言容易读，但它不自动等于真实内部机制，还会受 prompt、随机性和输出格式影响。判断句：模型说得顺，不代表它的 attribution 就对。

术语表

TERMS
术语直觉解释准确说法常见误解
model explanation说明模型为什么给出某个预测。解释可以来自输入归因、注意力、梯度、探针、样例或模型生成的文字理由。把“看起来合理的文字理由”当成真实内部过程。
feature attribution给输入里的每个 token 分配重要性。固定一个目标输出，估计不同 input token 对这个目标的贡献。不指定目标 output token，就直接比较所有分数。
saliency map把 attribution score 画成表格或热力图。通常一侧是 input token，另一侧是 output token，格子里的 score 表示贡献大小。看错行列，把别的输出 token 的分数拿来答题。
gradient-based attribution看目标分数对输入表示有多敏感。选出目标 token 的 logit，反向传播，依据梯度估计各输入 token 的贡献。以为梯度分数就是唯一真相，忽略方法假设和实现细节。
attention attribution用 attention 权重观察模型关注哪里。在 Transformer-based model 中，attention weight 可作为一种输入重要性线索。把 attention weight 直接等同于因果解释。
Inseq作业用来跑解释方法的工具包。给它模型、attribution 方法和输入，它可以产生 saliency map 等分析结果。只会复制代码，不知道三个必要输入各是什么。
sentence completion模型继续补全句子。生成后面的 token 时，前面已经生成的 token 也会进入下一步输入前缀。以为原始 prompt 之后生成的 token 不会影响后续 token。
LLM explanation让语言模型用人能读懂的话解释预测。可以要求模型给分类结果、解释理由，或模拟 attribution 给 token 打分。只要模型解释得流畅，就默认它可靠。

读完这一段，你应该能看着一张 attribution 图说清楚：解释的是哪个输出、参考了哪些输入、分数能说明什么、不能说明什么。

讲义主线

LECTURE

1. 为什么模型解释重要

课程一开始强调：模型预测出正确答案，不代表它真的聪明。它可能抓到了关键线索，也可能只是碰巧猜中。这个区别平时容易被忽略，因为我们常常只看最终答案。

在医疗和法律场景里，问题更尖锐。模型不能只给“阳性”“阴性”或“该胜诉”“不该胜诉”这类结果，还要给出支撑判断的理由。解释能帮助人检查模型有没有看错重点，也能帮助开发者发现模型弱点，再进一步改进。

判断句：只要任务有风险或需要追责，就不能只看 output，要看 output 为什么会出现。

2. 解释不是一种东西

字幕里给了两个例子。图像分类模型说图片是猫，explanation 要回答“为什么它觉得这是猫”。评论分类模型说 review 是 positive，explanation 可以指出它看到了 fantastic、terrific 这类正向词。

这两个例子看起来都叫解释，但形式不同：一个可能是图上的热区，一个可能是文字理由。这次作业也沿着这条分界展开：先用数值方法量化 input token 的重要性，再用 LLM 产生自然语言解释或模拟 token 打分。

判断句：解释的形式可以不同，但都要回答“哪个输入证据支持哪个输出”。

  1. gradient-based字幕中的例子是输入 “The first president of America is”，模型生成 GEORGE。要解释 GEORGE，就先从输出 logits 中挑出 GEORGE 对应的 logit。接着把这个 logit 反向传播回输入表示，观察每个 input token 对目标 logit 的影响。这里的关键先固定目标输出。你现在应该能做的事：看到题目问某个输出 token 时，先找到它对应的那一行。
  2. attentionattention 方法常见于 Transformer-based model。它直接利用模型内部的 attention weight，观察生成 output 时，模型把权重分给哪些 input token。这很方便，也很容易误读。attention weight 可以当作一种线索，但它不保证就是完整因果解释。作业中可以按图答题，但复述时要保留这个边界。你现在应该能做的事：说出 attention attribution 有用在哪里，也说出它不能被夸大成什么。
  3. Inseq 的三个输入字幕说 Inseq 用起来很直接：需要模型、分析方法和 input。模型决定要解释谁，方法决定用 gradient-based 还是 attention，input 决定要分析哪句话或哪段文本。输出通常是 saliency map。它把 output token 和 input token 之间的重要性关系画出来。每个格子的 score 都有方向：某个输入对某个输出。你现在应该能做的事：拿到代码时指出 model、method、input 分别在哪里。
  4. machine translation第一个任务场景是机器翻译。输入英文句子 “The first president of America is George Washington”，模型输出中文翻译。saliency map 左侧或下方对应 input token，上方或纵向对应 output token。字幕举了数字：生成“美国”这个输出时，America 的重要性分数可以是 0.308。作业题也可能问：生成 Washington 时，America 对应的 score 是多少，示例里是 0.181。你现在应该能做的事：不要凭印象答“America 很重要”，而要定位到指定 output token 的那一格。
  5. sentence completion第二个任务场景是句子补全。这里有一个很容易错的点：生成后面的词时，前面已经生成出来的词也会成为下一步的输入前缀。比如原始输入只有 “The first president of America is”。当模型生成到 Washington 时，前一步生成的 George 已经被放进上下文，所以 George 也会出现在 Washington 的 input prefix 里。你现在应该能做的事：判断某一步生成时，哪些 token 已经进入了输入，而不是只看最初 prompt。
  6. LLM explanation第二大任务用自然语言解释。第一种做法很直接：要求模型给出预测，再解释为什么这么预测。评论分类例子中，模型可以说因为 fantastic、terrific 是正向词，所以判断为 positive。第二种做法更接近 feature attribution：用 prompt 要求 LLM 给每个 token 一个 importance score，再输出分类结果。由于 ChatGPT 有随机性，格式可能不稳定；如果没有按要求输出，作业建议多试几次。你现在应该能做的事：把“解释是否好懂”和“解释是否可靠”分开判断。

完整知识链路

把 P48 串起来，可以这样讲：模型解释先从“答对不等于懂”开始。数值路线固定一个 output token，用 gradient-based 或 attention 方法估计 input token 的重要性，再通过 saliency map 回答具体格子的分数。自然语言路线让 LLM 说明预测理由，或模拟 attribution 给 token 打分。读结果时不要只看解释是否顺耳，要检查目标 token、输入前缀、分数、格式和结论是否彼此一致。

提示词示例：让 LLM 模拟 attributionprompt

请判断下面评论是 positive 还是 negative。
同时给每个 token 一个 0 到 1 的重要性分数，并解释分数最高的 token 为什么影响分类。

Review: "The movie was fantastic, but the ending was too long."
输出格式：
prediction:
token_scores:
explanation:

读这个 prompt 时要注意什么

它让模型同时做分类、打分和解释。若输出缺少 token_scores，或者分数和解释对不上，就不能直接当作有效结果。字幕提醒 ChatGPT 有随机性，格式不对时需要多试几次；但多试为了得到可检查的格式。

Worked example：三步读 saliency map

题目问：“生成华盛顿时，America 的重要性是多少？”第一步，先找目标 output token，找输出行“华盛顿”。第二步，在这一行里找 input token “America” 所在列。第三步，读交叉格，示例中是 0.181。

如果题目改问“生成美国时，America 的重要性”，目标 output token 变成“美国”，同一列的交叉格就变成 0.308。你现在应该能用“目标行、输入列、交叉格”三个词解释任何 saliency map 读数题。

误区诊断

DIAGNOSIS

常见坑

  1. 只看模型答对，没有检查它依据哪个输入。修法：要求解释指向具体 input token 或证据。
  2. 读 saliency map 时看错 output 行。修法：先定位目标 output token，再读对应 input token 的格子。
  3. 把 0.308、0.181 这类 score 当成全局重要性。修法：说明它们只属于某个 output-input 配对。
  4. 做 sentence completion 时忘记已生成 token 会进入前缀。修法：逐步列出当前生成位置之前的全部 token。
  5. 把 attention weight 当作绝对因果。修法：把它说成一种解释线索，不说成完整证明。
  6. ChatGPT 输出格式乱了还直接作答。修法：重试或调整 prompt，直到 prediction、token_scores、explanation 都可检查。
  7. 只复制 Inseq 代码，不知道 model、method、input 各是什么。修法：运行前把三者标出来。
