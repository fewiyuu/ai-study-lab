# 概念骨架 {#skeleton}
<!-- tag: MAP -->

输出后补救模型回答已经生成，再用事实查核、有害词检测、警告提示或防火墙做二次检查。它的价值是多一道关，不是把模型变成真理机器。核心直觉：防火墙像出口安检，只能拦下可识别的风险，不能证明货物从源头就没有问题。判断句：如果系统只能说“网络上有人这样写”，它给的是支持证据，不是最终事实。
偏见被测出来偏见不是只靠直觉争论。可以把输入中的性别、种族、国家、名字或职业换掉，再比较模型输出的情绪、排序、代词和立场是否明显变化。判断句：同一份履历只改名字却得到不同排名，就要怀疑模型利用了不该影响结论的社会线索。
来源被追踪AI 生成文本侦测有两条路：事后训练分类器，或生成时主动加水印。前者怕模型更新和写作风格变化，后者怕改写、同义替换和攻击。判断句：侦测分数只能当证据的一部分，不能单独判定一个人是否违规。

资料层训练资料可能已有偏见，也可能缺少可靠事实来源。
训练层参数学习、RLHF、红队奖励都会改变模型倾向。
生成层采样概率、拒答规则和水印规则影响具体输出。
使用层事实查核、履历筛选和文本侦测都发生在真实应用里。

# 术语表 {#terms}
<!-- tag: TERMS -->

| 术语 | 直觉解释 | 准确说法 | 常见误解 |
| --- | --- | --- | --- |
| 幻觉 | 模型说得像真的，但内容不一定是真的。 | 语言模型在证据不足、任务过宽或分布外情境中生成不受事实支持的陈述。 | 把流畅、具体、语气肯定当成事实可靠。 |
| 事实查核 | 把模型回答拿去找外部证据。 | 从回答中抽取可检验陈述，转成查询，再用检索结果判断是否有支持证据。 | 以为“搜得到类似句子”就等于绝对正确。 |
| FActScore | 把长回答拆成更小的事实来检查。 | 一种细粒度事实评估思路：抽取原子事实，检索证据，再判断每个事实是否被支持。 | 忘了抽事实、改写问题、检索和判断本身也可能出错。 |
| 防火墙 | 放在模型和用户之间的安全层。 | 输出后处理模块，可做事实查核、有害内容检测、警告、改写或拦截。 | 把防火墙当成根治方案，而不是降低风险的额外关卡。 |
| 偏见 | 模型对某些群体、职业或立场有系统性倾向。 | 在只有敏感属性或社会线索变化时，输出评价、排序、情绪或立场发生稳定差异。 | 以为反刻板印象就不是偏见；只要方向和人类直觉不同，也仍可能是偏见。 |
| 红队 | 专门找系统漏洞的人或模型。 | 在 LLM 安全中，红队可以生成刺激输入，让被测模型暴露偏见、歧视或其他不安全行为。 | 以为红队只是在骂模型；它本质上是系统化测试。 |
| AI 文本侦测器 | 判断一段话像不像机器写的工具。 | 通常用机器文本和人类文本训练分类器，输出概率、分数或可疑标记。 | 把侦测器结果当作司法证据，忽略误判和模型版本漂移。 |
| 水印 | 藏在生成文字里的暗号。 | 生成时改变某些 token 的采样概率，让知道规则的检测器能从统计模式中识别来源。 | 以为水印肉眼可见，或以为改写不会破坏水印。 |

读到这里，你已经可以把术语放进“风险在哪里、怎么检测、哪里会失效”这三个问题里。

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 1. 为什么先讲“亡羊补牢”
大型语言模型经过预训练、指令微调和偏好对齐后，仍然会讲错话。课堂里的例子很直接：让 GPT-4 推荐 LLM 安全综述论文，它可能给出看似学术、格式完整的标题，但拿去 Google 搜不到。问题不在于模型“故意骗人”，而在于它擅长生成像答案的文本，不等于每个事实都经过检索和验证。
“亡羊补牢”的想法是：既然不能保证模型永远不犯错，就在模型和人之间再放一层检查。这个安全层可以做事实查核、有害词检测、毒性评分、警告提示，甚至直接拦截或改写回答。它像部署系统里的第二道关口，但它只是在降低风险。
判断时要记住位置：模型先给出回答，安全层再处理回答。它不是重新训练模型，也不是让模型从此没有幻觉。你看到任何“输出后查核”方案，都要问：它查的是什么陈述，依赖什么证据，查核模块本身会不会犯错。

## box
### 2. 事实查核不是“找得到网页就真”
Gemini 这类平台会把回答中的句子标成绿色或红色：绿色表示网络上找得到类似支持，红色表示找不到。这个设计有帮助，因为它提醒用户不要无条件相信回答。但它也有边界。网络上有类似说法，不代表事情一定对；网络上找不到，也不代表事情一定错。
FActScore 一类方法把流程拆得更细：先从模型回答里抽出需要检查的原子事实，再把事实改写成查询，拿去搜索资料，最后判断是否被证据支持。每一步都合理，也每一步都可能错。抽事实可能漏掉关键句，改写问题可能改变意思，搜索结果可能本身不可靠，最后的判断常常又交给另一个语言模型。
课堂里“两个李宏毅”的例子尤其重要。模型把演员李宏毅和教授李宏毅的资料混到一起，每个链接或单句都可能是真的，但合成后的个人介绍是错的。这种错误不是单句查核能完全解决的，它要求系统识别实体一致性：这些事实是不是属于同一个人、同一个时间、同一个上下文。

3. 用替换测试看偏见HELM 一类评测会把输入中的敏感词换掉，例如把“男”换成“女”，或者换不同种族、地区、国家相关词，再比较模型输出。如果只是敏感属性变了，回答的情绪分数、评价方向或建议强度却大幅变化，就说明模型可能有偏见。这里讨论的是“偏见是否存在”，不是立刻判断每种偏见是否应该被修正。课堂特意把这两件事分开：检测偏见是技术问题，是否处理以及怎么处理，还牵涉公平性、社会规范和应用场景。
4. 红队让模型暴露问题强模型常会避开敏感回答，直接问不一定测得出偏见。红队的做法是让另一组人或另一个模型专门寻找触发输入。在 LLM 场景里，可以训练一个红队语言模型，让它生成能诱导被测模型说出偏见、歧视或不安全内容的提示。训练思路类似奖励学习：红队输出一句话，被测模型回答，如果回答越偏见，红队得到的 reward 越高。这样红队模型会逐渐学会找漏洞。你现在要能分清：被测模型不是在被训练，红队模型才是在被优化来找问题。
5. 履历、名字和 embedding彭博社的虚拟履历实验把同一份履历换上八个不同名字，让模型为金融分析师、HR、软件工程师等岗位排序。因为模型有随机性，实验需要重复很多次。结果显示，同样履历只因名字不同就可能排名不同，说明模型把名字背后的族群、性别或地区线索带进了判断。把名字丢进模型得到 embedding 后再投影到二维平面，也能看到不同地区或族群名字会聚成不同群。聚类本身不是错误，因为名字确实带有统计信息；问题是这些信息被用于履历筛选时，可能让无关因素影响决策。
6. 职业与政治倾向职业刻板印象可以从代词和称谓里看出来。让模型给幼稚园老师写反馈，它可能默认用女性代词；给建筑工人写反馈，它可能默认男性代词。医生这类职业可能相对中性，技师或建筑工人则容易出现强性别倾向。政治倾向也能被测。直接问模型是否支持某个政治命题，它常会两边都说，避免表态。给出额外约束，例如“回答只能从这些选项中选一个”，模型就可能稳定选择某些立场。这个结果提醒我们：把 LLM 用在政治、政策或舆论相关任务时，不能假设它没有立场。
7. 减偏有四个入口偏见可能来自资料，也可能来自训练过程、生成过程或输出后的使用方式。资料层可以做清理、重采样和标注检查；训练层可以调整目标函数、加入公平性约束或偏好资料；生成层可以修改输出概率，让某些偏见表达不那么容易被采样；输出后可以加防御层，对回答做检测、警告、改写或拒绝。这些入口没有一个是万能的。资料清理可能删掉真实差异，训练约束可能牺牲其他能力，生成时改概率可能让语言变僵，输出后改写可能掩盖问题来源。减偏根据任务风险选择组合。
8. AI 文本侦测与水印侦测 AI 文本的朴素思路是收集机器生成句子和人类生成句子，训练分类器找差异。对单一模型、单一版本，它有机会做得不错；一旦模型改版、提示方式变化、人类写作风格受 AI 影响，分类器就会变得不稳。另一条路是在生成时加水印。简化例子是把 token 分成红组和绿组，在特定位置提高某一组 token 的概率。人读起来不一定看得出差异，但检测器知道规则后，可以统计生成文本是否符合这套偏好。水印也会被改写、同义替换或重采样削弱。

## box
### 完整知识链路
这一讲可以串成一条安全链：语言模型会生成流畅但可能错误的内容，所以输出后要有事实查核和防火墙；模型在训练资料和训练目标中吸收社会线索，所以要用替换测试、红队、履历排序、职业代词和政治测验把偏见测出来；AI 生成文本进入论文和审稿场景后，分类器和水印可以提供来源线索，但都不能单独作最终判断。你现在应该能拿一个真实 LLM 应用，标出它在哪些位置需要查事实、查偏见、查来源。

# 安全线索 {#source}
<!-- tag: 线索 -->

下面把字幕里的关键例子整理成适合回看的线索。每条都对应讲义里的一个判断点。

幻觉例子强如 GPT-4，推荐安全综述论文时也可能编出查不到的论文标题。结论：LLM 不是搜索引擎，答案需要外部证据。能做的判断：看到文献、人物经历、法律条文这类事实型输出，先查来源，不被格式和语气骗过。
Gemini 事实查核Gemini 会用 Google 搜索结果给句子标记支持或存疑。绿色代表有类似资料背书，红色代表找不到支持。结论：查核结果是风险提示，不是绝对判决。能做的判断：绿色也可能引用错实体，红色也可能只是资料少。
实体混合让模型“完整介绍李宏毅”，它可能把演员李宏毅和教授李宏毅的信息合并。单句可能都对，合起来却错。结论：长回答要检查实体一致性。能做的判断：当回答跨越多个来源时，检查每条事实是不是属于同一个对象。
偏见检测把输入中的性别、族群、国家或名字换掉，再比较输出情绪、排序和立场。结论：偏见可以通过受控变量测试暴露出来。能做的判断：如果只改敏感属性就改变结果，模型可能在用不该用的线索。
履历和政治同一份履历换名字后多次排序，模型会表现出族群和性别偏好；政治测验加上强制选项后，模型也会显示倾向。结论：应用到招聘和政治任务时要特别谨慎。能做的判断：高风险应用不能只看一次回答，要用重复实验和对照组。
审稿与水印研究者用分类器分析会议审稿意见，ChatGPT 发布后部分领域疑似 AI 生成比例上升；另一类方案是在生成时给 token 选择加统计水印。结论：来源追踪有线索，但不等于零误判。能做的判断：侦测器和水印都要配合人工复核、上下文证据和申诉机制。

# 最小例子 {#example}
<!-- tag: EXAMPLE -->

下面两个最小例子把课堂里的流程压成可检查步骤。读代码时看三件事：输入是什么，中间判断是什么，输出凭什么可信。

最小示例：事实查核防火墙pseudo

```text
answer = llm("请推荐几篇 LLM 安全综述论文")
claims = extract_atomic_claims(answer)

for claim in claims:
 query = rewrite_as_search_query(claim)
 evidence = search_web(query)
 verdict = judge_support(claim, evidence)
 show_badge(claim, verdict)

# badge 表示“有无支持证据”，不是“事实已经被证明”。
```

## box soft
### 逐行理解
`extract_atomic_claims` 负责把长回答拆成小事实，例如论文标题、作者、年份。`rewrite_as_search_query` 把事实转成检索问题。`judge_support` 再看证据是否支持原句。最容易漏的地方是：拆事实、改写查询和判断支持都可能由另一个 LLM 完成，因此也可能幻觉。
你现在可以检查一个查核系统：它是否只给“有网页背书”，还是会处理实体混合、来源可信度和证据冲突。

最小示例：简化版水印pseudo

```text
tokens = split_vocabulary_into_red_and_green()

for position in range(max_length):
 probs = llm_next_token_probs(context)
 if position % 2 == 0:
 probs = boost(probs, group="green")
 else:
 probs = boost(probs, group="red")
 token = sample(probs)
 context += token

# 检测器知道分组和规则后，统计文本是否偏向这套选择模式。
```

## box warn
### 这个例子能说明什么
水印轻微改变生成时的 token 概率。人读起来通常仍然通顺；检测器看的是统计模式。它的边界也很清楚：如果文本被大幅改写、翻译、重采样，原来的统计模式可能被破坏。

## box info
### Worked example：一段回答怎样过三道安全检查
假设模型回答：“A 论文在 2023 年提出某个安全框架，作者是 X 和 Y。”第一道检查把它拆成三个原子事实：论文是否存在、年份是否正确、作者是否正确。第二道检查检索证据，发现论文存在且年份正确，但作者列表没有 Y。第三道检查把结论写成“部分支持：论文和年份有证据，作者 Y 缺少支持”。
这个例子说明：事实查核的好输出指出哪一条事实有证据、哪一条没有证据、证据是否指向同一个实体。你现在应该能把一段长回答拆成可查的原子事实，而不是被整段文字的流畅度带着走。

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->
<!-- layout: grid2 -->

## box redsoft
### 常见坑
1. 把事实查核当作绝对真理。错因：混淆“有支持证据”和“事实成立”。后果：实体混合、错误来源和过期信息仍会漏过。修法：检查证据质量、实体一致性和反证。
2. 只看单个样例就判断模型有无偏见。错因：忽略模型随机性和样本覆盖。后果：可能把偶然输出当成稳定倾向。修法：做对照组、重复采样和统计比较。
3. 把反刻板印象当作没有偏见。错因：只看方向是否符合常见偏见，不看是否由敏感属性驱动。后果：模型仍可能在招聘、排序和推荐中不公平。修法：回到受控变量测试。
4. 认为红队就是让模型说难听话。错因：把系统化漏洞搜索理解成普通挑衅。后果：测试覆盖不足，也无法形成可复现样本。修法：定义攻击目标、奖励信号和失败类型。
5. 用 AI 侦测分数直接处罚学生或审稿人。错因：忽略分类器误判、模型版本变化和人类写作风格变化。后果：误伤真实作者。修法：把侦测结果当线索，配合上下文证据。
6. 以为水印不可破坏。错因：只看到生成时嵌入，没有考虑改写攻击。后果：检测器在翻译、润色、同义替换后失效。修法：说明水印强度、攻击模型和检测阈值。

## box
### 易混概念
概念 A概念 B区别
