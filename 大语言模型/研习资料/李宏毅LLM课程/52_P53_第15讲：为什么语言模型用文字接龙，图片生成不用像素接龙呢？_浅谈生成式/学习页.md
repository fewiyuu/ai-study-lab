# 概念骨架 {#skeleton}
<!-- tag: MAP -->

同一个生成问题文字、图片、声音表面不同，底层都可以拆成有限单位。给模型一个 condition，它要决定这些单位如何排列，最后形成句子、图像或声音。判断句：如果你能指出“基本单位是什么”，就已经抓到生成问题的第一层结构。
两种生成节奏自回归是按部就班，前一个单位生成后再生成下一个单位；非自回归是齐头并进，多个位置同时生成。前者更容易保持局部一致，后者更快。判断句：看到“必须等上一步结果”的流程，就优先想到自回归。
影像为什么特殊图像和声音的单位数量太多。逐像素或逐 sample 生成虽然概念上可行，真实使用时会被等待时间拖垮。于是影像生成常把并行、压缩和多阶段组合起来。判断句：当单位数量达到百万级，生成策略的核心问题通常先变成速度。

文字一句话是一串 token。语言模型每步预测下一个 token，再把新 token 放回上下文。
图片一张图有大量 pixel。1024×1024 约一百万像素，逐个生成会很慢。
声音一秒声音有上万 sample。一分钟语音可能超过百万个采样点。
折中先压缩或分阶段，再用并行模型补细节，减少等待同时保住一致性。

读完这一段，你应该能把“为什么不用像素接龙”回答成数量、并行和一致性三件事，而不是只说“因为图片复杂”。

# 术语表 {#terms}
<!-- tag: TERMS -->

| 术语 | 直觉解释 | 准确说法 | 常见误解 |
| --- | --- | --- | --- |
| 生成式人工智能 | 根据条件产生复杂结构。 | 给定 condition 后，学习数据分布并生成文字、图像、声音等结构化对象。 | 以为它只是从资料库取一个现成答案。 |
| Token | 文字模型处理的基本片段。 | 由 tokenizer 和 vocabulary 决定，可是字、词的一部分或特殊符号。 | 把 token 等同于一个汉字或一个英文单词。 |
| Pixel | 图片里的一个颜色格子。 | 图像在空间网格上的基本显示单位，每个像素有颜色值。 | 以为逐像素生成只是“多一点循环”，忽略百万级步骤。 |
| Sample | 声音波形上的一个采样点。 | 音频按采样率离散化后的数值点，例如 22kHz 表示每秒约 22000 个点。 | 以为一分钟声音只是一小段文字级别的长度。 |
| 自回归生成 | 一步一步接下去。 | Auto-regressive generation，每一步以前面已生成结果为条件产生下一个单位。 | 以为它一定只用于文字；图像和声音也可以自回归，只是常常太慢。 |
| 非自回归生成 | 很多位置同时生成。 | Non-autoregressive generation，多个输出位置在同一次或同一阶段中并行决定。 | 以为并行必然质量更好；它常遇到一致性问题。 |
| 多峰性问题 | 同一个条件下有多个合理答案，模型各位置可能想的不一样。 | Multimodality 在这里指生成分布有多个模式，独立并行生成时可能混合不同模式。 | 和“多模态输入输出”混为一谈。 |
| 潜在向量 | 给生成过程补一份全局想法。 | 随机采样的 latent vector，为并行生成提供一致的隐含条件。 | 以为它是噪声，没有语义作用。 |
| Autoencoder | 先压缩，再还原。 | Encoder 把图像或声音压成潜在表示，decoder 再从潜在表示重建原对象。 | 把它当成普通降低解析度，忽略神经网络学习到的表示。 |
| Diffusion | 从噪声逐步变清楚。 | 把生成拆成多次去噪，每一步让样本比前一步更接近目标分布。 | 以为 diffusion 是一口气直接画出图片。 |

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 1. 生成问题先看“基本单位”
课程先把视角从文字跳出来。生成式 AI 要产生的是复杂而有结构的对象：一段话、一张图、一段声音。复杂的意思可能性多到无法穷举；有结构的意思是各部分不能随便拼，句子的词序、图片的空间关系、声音的时间波形都要协调。
这些对象又都能拆成有限单位。文字是 token，图像是 pixel，声音是 sample。语言模型每次从词表里选 token；图像里的每个像素有颜色；声音按采样率变成一个个数值点。这个拆法让“生成”可以被模型化：给定 condition，决定这些单位应该是什么、应该怎样排列。
现在你应该能做的事：看到一个生成任务，先写出 condition、基本单位和输出对象，再讨论模型方法。

## box
### 2. 自回归为什么适合文字
文字接龙的正式名称是 auto-regressive generation。语言模型读到输入后，先生成第一个 token；再把第一个 token 接回输入，生成第二个 token；如此重复，直到完整回答结束。每一步都依赖前面已经生成的内容。
这个机制慢，但很自然地保持了局部一致。比如“李宏毅是演”后面更可能接“员”；如果第一步生成“教”，后面就会切换到“授”。模型不需要一开始同时决定所有位置，因此比较不容易把不同答案混在同一句里。
判断句：如果一个任务的后一个单位强烈依赖前一个单位，并且单位总数不算夸张，自回归往往是稳妥选择。

## box warn
### 3. 像素接龙和 sample 接龙概念上可行
课程中特别提醒：图片不是不能用接龙生成。早期的图像版 GPT、PixelRNN/PixelCNN 类思路，都可以一个像素一个像素地产生。声音也可以逐 sample 生成，WaveNet 就是典型例子。它们说明“接龙”不是文字专属。
问题在数量级。1024×1024 的图片约有 1048576 个像素，逐像素生成相当于为一张图做一百多万步。22kHz 的一分钟语音约有 132 万个 sample。用接龙方式生成一句较长文字你都会等待，让模型生成一张图或一分钟声音时走百万步，真实体验很难接受。
现在你应该能做的事：用数量级解释方法选择，而不是只说“图像模型不适合语言模型方法”。

## box info
### 4. 非自回归用并行换时间
非自回归的想法很直接：不要等第一个像素生成后才生成第二个像素，而是把很多位置同时生成。只要硬件能并行计算，总等待时间就会大幅下降。图像尺寸通常预先固定，例如 512×512 或 1024×1024，所以模型知道要填多少位置。
不过并行不等于总运算量消失。它主要减少的是等待链条：原来必须排队的一百万步，现在可以被分批或同时处理。对用户来说，时间体验差异很大；对机器来说，可能还是花了很多计算资源。
判断句：非自回归适合“位置数量很多、输出长度或形状已知、可以并行算”的场景。

## box redsoft
### 5. 多峰性让并行结果变糊
非自回归的主要问题各位置可能各自脑补。输入“画一只奔跑的狗”，合理答案有很多：向左跑的白狗、向右跑的黑狗、草地背景、街道背景。并行生成时，如果不同位置参考同一个条件却各自选择不同模式，就可能把多个答案叠在一起。
文字例子更好看清楚：输入“李宏毅是”，第一个位置可能觉得“演”概率最高，第二个位置独立判断时可能觉得“授”或“手”概率高。并行抽样可能得到“演手”这种不通顺组合。自回归先选“演”，下一步就会在“演员、演艺、演过”等后续里继续，因此更容易保持一致。
这里的 multimodality “同一个条件下有多个合理生成模式”。这两个词容易撞名，答题时要说清语境。

## box soft
### 6. 潜在向量给并行过程统一想法
一种补救方式是给模型一个额外的随机向量。它让所有位置共享同一份隐含设定：这只狗向左、白色、背景是草地。这样每个位置在并行生成时，不只看文字 condition，也看同一个 latent vector。
你可以把 latent vector 理解成“这次样本的全局草稿”。VAE、GAN、flow-based model、diffusion 等方法形式不同，但常见共同点是：生成前会有一个随机采样的潜在表示，模型用它来确定这次生成的具体版本。
判断句：如果同一个 prompt 能生成很多不同图，而每次图内部还要保持一致，就需要某种全局随机因素来固定这次的选择。

## box
### 7. 压缩表示让接龙长度变短
另一种补救是不要在原始像素空间里接龙。先训练一个 encoder 把图片压缩成人看不懂、但 decoder 能还原的表示。比如把 16×16 个像素压成一个 latent 位置，原来一百万像素就可以变成几千个潜在单位。
这不是普通降分辨率。普通缩图会直接丢掉很多细节；autoencoder 的 encoder 和 decoder 是神经网络，它们学习怎样把重要结构压进去，再从压缩表示重建。人不一定看得懂压缩结果，但 decoder 看得懂。
现在你应该能做的事：解释为什么“生成 latent 再 decode”能让自回归步骤减少，同时还保留最终高清输出的可能。

## box
### 8. 多阶段生成把难题拆小
影像生成还常把一次困难生成拆成多阶段：先生成低解析度草稿，再逐步提升解析度；或者从几乎全是噪声的图开始，一步一步去噪到清晰图。每一步只需要修正一点点，比一次性从文字直接生成完整高清图更容易保持一致。
Diffusion 就可以这样理解：每个版本都比前一个版本更少噪声，最终变成干净图。它看起来是非自回归的多次使用，也带有某种层级式自回归味道：逐版本推进。
判断句：当你看到“先粗后细、先噪声后清晰、逐步修补”的流程，重点阶段设计怎样限制脑补空间。

## box soft
### 完整知识链路
把这一节串起来：生成式 AI 给定 condition 后，要把有限基本单位组合成复杂对象。文字单位数量相对可控，而且前后依赖强，所以自回归文字接龙很成功。图像和声音也能接龙，但像素和 sample 数量很快到百万级，逐步等待不可接受。非自回归能并行生成，速度好，却会因为同一条件有多个合理答案而产生多峰性问题。影像生成于是引入 latent vector 固定本次脑补，用 autoencoder 把原始像素压成短得多的潜在表示，再用多阶段或 diffusion 逐步细化。你现在应该能用“速度、质量、一致性”三轴解释不同生成策略的取舍。

下面是从 P53 字幕中整理出的课程材料。它们把口语内容整理成适合回看的学习线索。

线索 1：跳出文字课程从语言模型转向更一般的生成式 AI：文字、影像、声音都是复杂对象，都由有限基本单位构成。这个转向让后面的策略比较有了共同坐标。你现在应该能说明：为什么先讲 token、pixel、sample。
线索 2：接龙不是文字专属图像可以逐像素生成，声音可以逐 sample 生成。早期图像生成和 WaveNet 说明这种做法不是幻想，问题主要出在速度和长度。你现在应该能避免“图片不能接龙”的绝对说法。
线索 3：百万级单位改变方法选择1024×1024 图片约一百万像素，一分钟 22kHz 声音约 132 万采样点。逐单位生成会把等待时间拉到难以使用。你现在应该能用数量级而不是感受解释慢在哪里。
线索 4：并行生成会各自脑补非自回归把多个位置同时生成，但同一个 prompt 对应多个合理答案。不同位置如果选择不同答案模式，图像可能糊、文字可能不通顺。你现在应该能解释多峰性为什么影响质量。
线索 5：压缩表示不是普通缩图Encoder 把图片压缩到潜在表示，decoder 再还原。这个表示人未必看得懂，但模型能用它减少要生成的长度。你现在应该能区分 autoencoder 和简单降低解析度。
线索 6：逐版本推进多阶段生成可以从低解析度到高解析度，也可以从有噪声到无噪声。Diffusion 的直觉就是把一个难题拆成多步去噪。你现在应该能把 diffusion 放回 AR/NAR 折中的框架里。

# 最小例子 {#example}
<!-- tag: EXAMPLE -->

下面的伪代码帮助你读懂三种生成节奏。注意每一步读取什么、写入什么、以及哪里可能出错。

自回归：每次只写下一个 tokenpseudo

```text
context = ["李", "宏", "毅", "是"]
while not finished:
 next_token = model.predict_next(context)
 context.append(next_token)

# 特点：后一步能看到前一步的选择；慢，但较容易保持句子一致。
```

非自回归：多个位置同时写pseudo

```text
condition = "画一只正在奔跑的狗"
positions = all_pixels_or_patches(image_size)
outputs = model.predict_all_positions(condition, positions)

# 风险：不同位置可能各自选择不同版本的狗，最后混成不协调的图。
```

混合策略：先固定全局想法，再逐步细化pseudo

```text
condition = encode_text("一只白狗在草地上向左奔跑")
latent = sample_latent_vector(seed)
draft = generator.create_compressed_plan(condition, latent)

for stage in refinement_stages:
 draft = stage.refine(draft, condition)

image = decoder.reconstruct(draft)
```

## box soft
### 读代码的顺序
先问每段代码的“输出单位”是什么。第一段是 token，第二段是 pixel 或 patch，第三段是压缩表示再解码成图。输出单位决定生成步数，也决定可能的错误类型。

## box warn
### 删掉会怎样
如果删掉 latent，不同位置更容易各自脑补；如果删掉 refinement，只靠一次并行输出可能粗糙；如果删掉 decoder，压缩表示不能变成用户能看的图。

## box info
### Worked example：用数量级判断该不该接龙
假设文字回答有 200 个 token，自回归要走约 200 步；一张 1024×1024 图片若逐像素生成，要走 `1024 × 1024 = 1,048,576` 步；一分钟 22kHz 音讯若逐 sample 生成，要走 `22,000 × 60 = 1,320,000` 步。
这三个数字说明：接龙在原理上都可行，但等待链条长度完全不同。文字的 200 步还可以接受，图像和声音的百万步就会让工程系统优先考虑并行、压缩表示或多阶段生成。你现在应该能遇到新任务时先估算“基本单位数量”，再讨论生成策略。

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->
<!-- layout: grid2 -->

## box redsoft
### 常见坑
1. 说“图片不能接龙”。错因是把工程上不常用误认为原理上不可行；修法是改成“可以，但逐像素太慢”。
2. 只用“图片复杂”解释方法差异。错因是缺少数量级；修法是算出像素或 sample 的步数。
3. 把非自回归等同于质量更好。错因是只看到并行速度；后果是忽略多峰性和局部不一致。
4. 把本节的多峰性和多模态模型混在一起。错因是英文词相近；修法是说明这里指多个合理生成模式。
5. 把 latent vector 当成没有意义的随机噪声。错因是没看到它固定本次样本的全局设定；后果是解释不了同一 prompt 的多样输出。
6. 把 autoencoder 当成缩小图片。错因是只看压缩结果；修法是说明 encoder/decoder 学的是可重建的潜在表示。
7. 以为 diffusion 只是“慢慢画”。错因是忽略去噪阶段；修法是说清从噪声到清晰图的逐步条件生成。

## box
### 易混概念对照
概念 A概念 B区别
自回归非自回归前者一步依赖一步，后者多个位置并行决定。
PixelPatch / LatentPixel 是原始显示单位；patch 或 latent 是为了让模型更容易处理的压缩单位。
多峰性问题多模态输入前者说同一条件有多个生成模式；后者说模型能处理文字、图像、语音等模态。
一次并行生成多阶段并行生成一次并行快但容易粗糙；多阶段把每步任务变小，常能提升一致性。
做题时先写“错因”，再写“后果”，最后写“修法”。只给结论通常查不出理解问题。
