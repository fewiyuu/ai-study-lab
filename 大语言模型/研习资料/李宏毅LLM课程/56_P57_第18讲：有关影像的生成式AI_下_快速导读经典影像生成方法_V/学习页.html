56_P57｜经典影像生成方法速览：VAE、Flow、Diffusion、GAN

影像生成的难点不只是“网络会不会画图”，而是同一段文字可以对应很多张合理图片。输入“一只奔跑的狗”，它可能是草原上的哈士奇，也可能是城市里的柴犬。训练资料里这些图片都配着相同或相近的文字，模型如果只看文字，就会不知道该学哪一种细节。

概念骨架

MAP

共同问题训练资料里的文字说明经常不完整。同一句 prompt 对应多张真实图片时，模型不能只靠“正确答案”学习，因为答案本来就不唯一。判断句：只要一个任务有多种合理输出，就要问模型如何处理那些没被输入明说的变量。

共同工具经典方法都会引入某种隐藏资讯：VAE 把图像压成 code，Flow 把图像和 noise 做可逆转换，Diffusion 把图像逐步加噪再学习去噪，GAN 用判别器把“像不像”变成训练信号。到这里，你可以把“噪声”先理解成模型用来填补细节的数值向量，不要急着把它看成无用杂讯。

共同边界这些方法不是互斥标签。GAN 可以挂在 VAE、Flow 或 Diffusion 后面当评价器；Diffusion 也常和 Transformer 结合。比较方法时要先说明任务、训练稳定性、采样速度和生成质量。

术语表

TERMS
术语直觉解释准确说法常见误解
文字条件用户给模型的描述，例如“一只奔跑的狗”。它是生成的条件，但通常不是完整的监督答案。以为 prompt 写了主题，图片细节就被唯一决定了。
潜变量 / Noise模型用来补齐细节的一串数值。可以来自 Encoder 的输出，也可以在生成时从简单分布随机采样。把 noise 当成纯粹无意义的干扰。
Auto Encoder先压缩，再还原。Encoder 从输入抽取 code，Decoder 根据 code 重建输入，训练目标是让输出接近输入。以为 Encoder 抽出的资讯一定是人能读懂的文字。
VAE让潜空间可采样的 Auto Encoder。训练时学会从图像抽潜变量，生成时可采样一个潜变量交给 Decoder。只记住“加噪”，忘了它是在让中间空间更适合生成。
Flow-based model把图像和潜变量互相转换。要求 Decoder 可逆，所以反函数可当 Encoder 使用。以为它只是把 VAE 换了名字。
Diffusion model从随机噪声慢慢去噪成图。训练一个 Denoise 模块；训练时自己给干净图加噪，推理时反复去噪。以为它一步直接生成清晰图像。
GAN让生成器接受另一个模型的评价。Generator 产图，Discriminator 判断图像和条件是否匹配，两者交替训练。问“GAN 和 Diffusion 谁更强”却不说明 GAN 可作为外挂评价器。
Latent Action从画面变化里猜出来的控制动作。互动式影像生成可用动作抽取模型推断“可能按了哪个按钮”，再用该动作预测下一帧。把 latent action 当成真实记录到的人类按键。

讲义主线

LECTURE

1. 为什么文字生图会让模型无所适从

一张图片包含太多没有写进文字的变量。训练资料里同样写着“一只奔跑的狗”，图片可能是一只哈士奇在草原上跑，也可能是一只柴犬在城市里跑。对模型来说，这一个输入对应很多合理答案。

如果训练时硬把同一个文字条件对应到不同图片，模型会在细节上被拉向不同方向。它可能知道“狗”和“奔跑”是什么，却不知道该把品种、背景、姿态、光照选成哪一种。生成结果出现混杂或模糊，常常监督讯号本来就不完整。

判断句：看到文字生图任务时，先问“哪些细节没有被文字说明”。能列出这些缺口，就能理解为什么经典生成模型需要潜变量、噪声、采样或判别器。

2. Auto Encoder 的基本想法

Auto Encoder 先训练一个 Encoder 读入图片和文字，抽出文字没描述到的资讯；再训练一个 Decoder 读取文字和这些抽出的资讯，尽量还原原图。训练目标很直接：输入一张训练图片，经过抽取和重建后，输出越接近原图越好。

这里的“资讯抽取”不一定输出人类能读懂的句子。它通常是一串数值向量。第一维可能和狗的品种有关，第二维可能和背景有关，也可能没有这么整齐的人类语义。只要 Decoder 读得懂，并能用它还原图片，这串向量就完成了它的工作。

判断句：Encoder 的输出给 Decoder 使用的内部 code。你以后看到潜变量时，要问它帮助 Decoder 补了哪些输入没有说出的细节。

  1. VAE：生成时把细节交给采样训练时 Encoder 能看到真实图片，所以它可以抽出“文字没说的资讯”。但真正生成时，图片还没出现，Encoder 没有东西可读。VAE 的处理方式是：把这些潜在资讯看成可采样的向量。用户只给文字时，模型先从某个分布随机取一个潜变量，再让 Decoder 根据文字和潜变量生成图片。判断句：VAE 的随机性不是装饰，它负责决定“这次脑补成哪一种合理图片”。
  2. Flow：只训练一个可逆 DecoderFlow-based model 和 VAE 很像，也需要在图像与潜变量之间来回转换。差别在于 Flow 强迫 Decoder 必须可逆。可逆意味着 Decoder 有一个反函数。反函数可以把图片变回潜变量，所以不需要另外训练一个 Encoder。判断句：Flow 的重点“从 noise 到图像的转换可以反过来算”。
  3. Noise 不是没用的东西课程里提到把臭脸向量减掉、把笑脸向量加上，输出的人脸会笑得更明显。这个例子说明潜变量或 noise 里有图像属性。它可能控制品种、背景、表情、姿态、光线。虽然我们未必知道每一维的准确意义，但可以通过向量运算观察某些方向是否对应稳定变化。判断句：如果改动潜变量后输出只改变某个属性，说明这个方向携带了可操作的生成资讯。
  4. Diffusion：反复做一个小任务Diffusion model 的 Decoder 也从 noise 走向图片，但它不尝试一步完成。它训练一个 Denoise 模块，每次只把有噪图片变干净一点。训练资料可以自己造：把干净图片逐步加噪，得到“有噪图”和“较干净图”的配对，然后教 Denoise 模块逆着走。推理时从随机噪声开始，重复去噪数百次或上千次，最后得到清晰图像。判断句：Diffusion 的能力来自多次小修正；它的成本也来自多次小修正。
  5. Diffusion Transformer：Transformer 做去噪在影像模型里，图片或影片可以被切成 patch。Diffusion Transformer 让 Transformer 读取有噪 patch、文字条件和时间步，输出更干净的 patch。Sora 这类影片模型被课程拿来说明这一点：生成影片时在 patch 表示上反复去噪，最后再还原成影像。判断句：这里的 Transformer 在每一步负责“去掉一部分噪声”。
  6. GAN：把评价变成训练信号GAN 引入 Discriminator。它读图像和文字条件，判断两者是否匹配、图像是否像真实样本。Generator 不再只追某张唯一真图，而是学习产生能让 Discriminator 给高分的图片。Generator 和 Discriminator 交替训练：一个不断产出更像真的图，一个不断学会更细的判断标准。它很像额外挂上的评价器，可以和 VAE、Flow、Diffusion 结合。判断句：GAN 的核心用可学习的评价函数替代单一标准答案。

完整知识链路

整节可以串成这样：文字条件不完整，所以模型需要额外资讯来决定图像细节。Auto Encoder 在训练时从真实图片抽出这些资讯，VAE 在生成时随机采样这些资讯，Flow 用可逆变换让图像和潜变量互相转换，Diffusion 通过反复去噪从 noise 走到图像，GAN 用 Discriminator 给 Generator 提供“像不像、配不配”的学习信号。到了互动式影像生成，缺失资讯从“图像细节”变成“用户按了什么动作”，于是课程又把 Auto Encoder 的思路迁移到 latent action：从前后画面里推断动作，再用画面和动作预测下一帧。你现在应该能解释每种方法把“未被输入明说的资讯”放在了哪里。

最小例子

EXAMPLE

下面的伪代码把本节四种方法压成可读流程。重点不在语法，而在每一步读什么、写什么、缺什么。

代码
text = encode_text("一只奔跑的狗")
image = load_training_image()

latent = encoder(image, text) # 抽出文字没写出的细节
rebuild = decoder(text, latent) # 用文字和细节重建原图
loss = distance(rebuild, image) # 让重建图接近训练图

sampled_latent = sample_normal() # 真实生成时没有原图，只能采样
new_image = decoder(text, sampled_latent)

容易改错的地方

如果把 sampled_latent 删除，生成会退化成“同一段文字总是同一张图”的倾向，很难覆盖多种合理细节。如果把 loss 理解成必须还原唯一标准答案，也会误解这节课：训练图是监督信号，但生成任务允许多种合理输出。

判断句：训练时潜变量来自真实图片，生成时潜变量来自采样；这两句话能分清，VAE 的核心就稳了。

最小示例：Diffusion Transformer 去噪python

patches = sample_noise_patches()
text = encode_text("雪地里的一只猫")

for step in range(1000, 0, -1):
 cleaner = transformer_denoise(patches, text, step)
 patches = update(patches, cleaner, step)

image = patch_decoder(patches)

再看 Diffusion

patches 一开始全是噪声。transformer_denoise 每次只让它更干净一点；重复很多次后，patch 才能被还原成图像。这解释了课程中反复出现的说法：Diffusion 的痛点是去噪次数很多，研究趋势之一是用更少步骤得到清晰图。

你现在应该能判断：如果有人说 Diffusion 是“一步画完整图片”，这句话不准确。

Worked case：同一句 prompt 怎样产出三张合理狗图

任务是给“一只奔跑的狗”生成三张彼此不同但都合理的图片：一张草地哈士奇、一张街道柴犬、一张海边金毛。核心判断“未被文字说明的品种、背景、姿态和光线由谁补上”。

方法流程例子你要检查的点

VAE固定文字条件，每次采样不同 latent，再由 decoder 生成不同狗图。训练时 latent 来自真实图，生成时 latent 来自采样；不要说生成阶段还能从目标图抽细节。

Flow从简单噪声采样 z，经可逆变换得到图像；训练时真实图可反算回 z。看它是否能反向算密度，而不只是能生成好看的图。

Diffusion从三组不同噪声 patch 出发，在同一文字条件下反复去噪，得到三张图。多样性来自初始噪声和采样路径，代价来自多步去噪。

GANgenerator 产图，discriminator 判断图像是否像真实狗图且是否匹配文字。它提供的是可学习评价信号，训练稳定性和模式覆盖要单独检查。

结论：四种方法都在处理“文字没说完”的问题，只是补信息的位置不同。你现在应该能做的事：看到新的图像生成论文时，先找它把未说明细节放在 latent、noise、可逆变换、去噪步骤，还是判别器反馈里。

误区诊断

DIAGNOSIS

常见坑

  1. 把 noise 当成无用杂讯。错因：只看名字，没有看它在生成时补细节。后果：无法解释表情、背景、风格如何变化。修法：把 noise 看成潜变量的一种来源。
  2. 认为 VAE 生成时还能用 Encoder 读原图。错因：混淆训练和推理。后果：说不清为什么要采样潜变量。修法：训练时有原图，生成时只有文字条件。
  3. 把 Flow 说成“多训练一个 Encoder”。错因：漏掉可逆约束。后果：无法区分 Flow 和普通 Auto Encoder。修法：说明反函数直接承担 Encoder 的角色。
  4. 认为 Diffusion 一步生成图像。错因：忽略多步 Denoise。后果：看不懂为什么采样速度是研究重点。修法：讲清 forward process 加噪与 reverse process 去噪。
  5. 问“GAN 和 Diffusion 谁赢”但不说明比较对象。错因：把 GAN 当成只能独立存在的模型。后果：忽略 GAN 可以作为评价训练框架挂到其他生成器上。修法：先定义任务、质量、速度和训练稳定性。
  6. 把 latent action 当作真实按键记录。错因：没看动作抽取模型的来源。后果：误判互动模型的数据要求。修法：说明它是从前后画面变化中推断出来的隐藏动作。
