P58｜延申：Variational Auto-encoder / VAE

这一节回答一个生成模型里的老问题：如果普通 autoencoder 已经能把图像压成 code 再还原，为什么还要多一个 VAE？核心原因是，普通 autoencoder 只保证训练样本对应的点能被还原，不保证潜空间中间的空白区域有意义。你随便从潜空间取一个点，decoder 可能输出不像真实图像的东西。

概念骨架

MAIN
  1. 普通 autoencoder 的问题

encoder 把图像压到一个 code，decoder 再从 code 还原图像。训练目标只要求某些训练点可还原，却没有要求两个训练点之间、周围或随机采样到的位置都能产生合理图像。

判断句：如果一个模型只学会把已见样本重建回来，还不能说明它的潜空间适合生成。

  1. 正则项的必要性

如果只最小化重建误差，模型会把 variance 学成接近 0。这样采样又退回普通 autoencoder：没有真正的噪声，也没有连续可用的潜空间。

KL 正则把 q(z|x) 往 N(0, I) 拉，避免每个样本把自己的分布缩成针尖。

术语表

VOCABULARY

Autoencoder先压缩再还原。由 encoder 和 decoder 组成，训练目标通常是让输出接近输入。以为会重建就一定会生成好样本。
Latent space压缩后的隐藏空间。模型用低维或结构化向量 z 表示输入背后的因素。把 latent vector 当作一定可解释的人工特征。
μ / mean分布中心。encoder 输出 q(zx) 的均值向量，表示该输入在潜空间中的中心位置。把 μ 当作最终唯一 code，忽略采样。
σ / variance允许扰动的大小。encoder 输出方差或 log variance，再经过变换保证为正。觉得 variance 越小越好，因为重建会更稳。
Reparameterization把随机性挪到外部噪声。先采样 ε ~ N(0, I)，再令 z = μ + σ·ε，使梯度能穿过 μ 和 σ。以为直接从 q(zx) 抽样也不影响反向传播。
KL divergence两个分布差多远。D KL (q(zx)p(z)) 衡量 encoder 产生的分布与先验 p(z) 的差异。把 KL 当作普通欧氏距离，忽略方向性。
ELBO难算目标的下界。log p(x) 的 evidence lower bound，最大化它相当于同时提升重建项并缩小分布差距。只背公式，不知道它为什么能替代原目标。
Conditional VAE告诉模型要生成哪一类。在 encoder 或 decoder 里加入条件 y，使模型在保留风格变量的同时控制类别。以为条件一加上去就能保证生成质量。

讲义主线

LECTURE

一、孤立点能重建，不代表中间点有意义

课程先用月相举例：假设满月和弦月分别被普通 autoencoder 编到一维 code 上的两个位置。训练只告诉 decoder：“这个点要变回满月，那个点要变回弦月。”至于两个点中间的位置，训练并没有给过明确约束。

我们心里会期待中间点解码成“介于满月和弦月之间的月相”，但神经网络没有义务这么做。中间区域可能输出奇怪图像，也可能突然跳到另一种形状。问题不在 decoder 不够聪明，而在训练目标没有要求潜空间连续、平滑、可采样。

你现在应该能判断：看到“从 autoencoder 的 latent space 随机取点生成图像”这类说法时，要先问这些点是否被训练目标覆盖过。

二、VAE 用加噪把点扩成区域

VAE 的关键动作是：对每个输入，不只训练一个 code，而是训练围绕这个 code 的一片区域。encoder 输出 μ 和 σ 后，模型从常态噪声中采样 ε，并得到 z = μ + σ·ε。decoder 必须从这个 z 还原原图。

如果满月的 code 周围一圈都要解码回满月，弦月的 code 周围一圈都要解码回弦月，那么两片区域重叠时，decoder 会被迫在误差之间折中。它更可能学到连续变化的月相，而不是在中间区域乱跳。

判断句：VAE 的“variational”味道不只是加一点随机性，而是把每个样本对应成一个可学习分布。

$$

z = μ(x) + σ(x) · ε,    ε ~ N(0, I)

μ 和 σ 由 encoder 产生；ε 是外部噪声；z 是交给 decoder 的 latent code。

$$

三、为什么 variance 不能让模型自己随便学

如果训练目标只有 reconstruction error，最省事的做法是让 σ 变得极小。噪声一消失，模型就能像普通 autoencoder 一样稳定重建训练样本。这样训练误差小了，但生成能力没有被修好。

因此 VAE 需要另一个约束：让 q(z|x) 不要远离标准常态先验 p(z)=N(0,I)。课程里提到的额外项可以直观看作对 variance 和 mean 的限制。σ 太小会被罚，μ 离 0 太远也会被罚。这样潜空间不会变成一堆互不相干的尖点。

你现在应该能做的事：当别人说“只要重建误差低，VAE 就训练好了”，你可以指出它还要检查 KL 项，否则 latent space 可能不可采样。

四、最常见的损失写法

实现里通常让 encoder 输出 mu 和 logvar。用 logvar 是为了让网络输出可以是任意实数，再通过 exp 得到正的 variance 或 standard deviation。

损失常写成两部分：重建损失负责“像原图”，KL 损失负责“潜空间可采样”。两者的比例会影响生成质量和重建清晰度。KL 太弱，潜空间散；KL 太强，重建可能变糊。

$$

Loss = Reconstruction(x, x̂) + DKL( qφ(z|x) || N(0, I) )

如果使用最小化视角，重建误差和 KL 都是要压低的项；如果使用最大化 ELBO 视角，符号会反过来。

$$

五、生成图像其实是在估计 p(x)

课程把一张图像看成高维空间中的一个点。如果图像是 20×20 像素，它就可以看作 400 维向量 x。生成模型想做的事，是估计训练资料背后的 p(x)：像宝可梦的图像概率高，乱七八糟的图像概率低。只要知道 p(x)，就能从高概率区域采样图像。

Gaussian mixture model 的想法是：复杂分布可以由许多高斯分布加权混合出来。先选第几个高斯，再从那个高斯采样 x。这像是给每个样本分一个离散 cluster。

但真实图像不只属于一个类别。一个手写数字可以同时有粗细、倾斜、弯曲程度、书写风格等属性。VAE 用连续向量 z 表示这些属性，比离散 cluster 更像 distributed representation。

六、decoder 是从 z 到 x 分布的函数

VAE 假设先从简单先验 p(z)，通常是 N(0,I)，采样一个 z。然后 decoder 根据 z 产生 p(x|z) 的参数。也就是说，每个 z 都对应到 x 空间里的一个分布。

这和有限个高斯混合不同：z 是连续的，所以可能的分布有无穷多个。神经网络负责学习从 z 到这些分布参数的复杂映射。即使 p(z) 很简单，经过强大的 decoder 后，得到的 p(x) 仍然可以很复杂。

$$

pθ(x) = ∫ pθ(x|z) p(z) dz

z 是连续变量，所以这里是积分。θ 是 decoder 的参数，它决定 z 如何映射到 x 的分布。

$$

七、真正想最大化的是 log p(x)

如果能直接最大化训练资料的 log likelihood，我们就能找到更好的生成模型。但 p(x)=∫p(x|z)p(z)dz 通常不好直接算。课程接着引入一个新的分布 q(z|x)：给定 x，估计它可能来自哪些 z。

q(z|x) 在实现里就是 encoder。它和 decoder 的方向相反：decoder 是给 z 产生 x 的分布，encoder 是给 x 产生 z 的分布。引入 q 以后，可以把 log p(x) 拆成一个可优化的 lower bound 加一个 KL gap。

八、ELBO 的两项含义

第一项是期望重建 log likelihood：从 q(z|x) 采样 z，然后希望 decoder 给原图 x 高概率。实现里常用 MSE 或 binary cross entropy 近似这一项。

第二项是 KL：让 q(z|x) 接近 p(z)。如果 q 和真实 posterior p(z|x) 越接近，lower bound 和 log p(x) 的距离越小。训练时同时学习 encoder 与 decoder，就是为了让这个下界更有用。

$$

log pθ(x) ≥ Eqφ(z|x)[ log pθ(x|z) ] - DKL( qφ(z|x) || p(z) )

右边就是 ELBO。最大化它，等价于提高重建概率并压低 q(z|x) 与先验 p(z) 的距离。

$$

Worked case：一张手写 7 的概率账本

假设训练样本 x 是一张手写数字 7。VAE 不直接说“这张图来自某个确定 code”，而是让 encoder 给出 qφ(z|x)：看过这张 7 以后，哪些潜变量 z 比较可能解释它。先验 p(z)=N(0,I) 则表示还没看任何图片前，模型希望从哪里采样。

概率对象直觉在 ELBO 里的作用

qφ(z|x)encoder 看见这张 7 后给出的潜变量分布。提供可采样的 z，让 decoder 练习从附近区域重建同一张图。

pθ(x|z)decoder 拿到某个 z 后，给这张 7 多高概率。重建项要提高它；否则生成出来不像原图。

p(z)训练结束后真正采样会用的标准常态分布。KL 项把 q 往这里拉；否则随机采样会落到 decoder 没学过的位置。

ELBO直接算 log p(x) 太难，于是优化一个可算下界。一边奖励“这个 z 能重建 x”，一边惩罚“这个 q 离先验太远”。

如果只追重建，encoder 可能把每张训练图塞到互不相连的尖点里；训练 loss 好看，但从 N(0,I) 随机采样会乱。若 KL 太强，所有图都被压回标准常态，decoder 分不清细节，重建会糊。你现在应该能做的事：看到 VAE loss 时，把它翻译成“重建账”和“采样账”两本账，而不是只背 ELBO 公式。

符号别被正负号绊倒

论文常写“maximize ELBO”，代码常写“minimize loss”。如果把最大化目标改成最小化损失，重建项和 KL 项的符号会转换。不要只记“加 KL”或“减 KL”，要先确认当前表达是在最大化还是最小化。

你现在应该能做的事：看到 VAE 公式时，先标出哪一项鼓励重建，哪一项约束潜变量分布，再判断符号是否和优化方向一致。

最小例子

EXAMPLE

最小 VAE 训练片段

下面的代码只展示语义，不追求完整可运行。关键是看清数据流：encoder 不直接返回 z，而是返回 μ 和 log variance；随机噪声通过 reparameterization 进入 z；decoder 从 z 重建 x；loss 同时包含重建和 KL。

逐行拆解：VAE training step复制

mu, logvar = encoder(x) # q_phi(z|x) 的参数
std = (0.5 * logvar).exp() # 保证标准差为正
eps = torch.randn_like(std) # 外部常态噪声
z = mu + std * eps # reparameterization
x_hat = decoder(z) # p_theta(x|z) 的输出

recon = mse_loss(x_hat, x) # 让输出接近输入
kl = -0.5 * (1 + logvar - mu.pow(2) - logvar.exp()).sum()
loss = recon + beta * kl # beta 可控制 KL 强度
loss.backward()
optimizer.step()

删改后果

删掉 eps 和 std，模型就退回“用 μ 当 code”的普通 autoencoder，潜空间附近区域没有被训练。删掉 KL，模型会为了重建把每个样本分布推到自己舒服的位置，训练后从 N(0,I) 采样可能落不到 decoder 熟悉的区域。

把 logvar.exp() 写错，可能得到负 variance 或数值爆炸。把 beta 调得过大，decoder 可能收到的信息太少，重建会模糊；调得过小，潜空间会散乱。判断代码时不要只看能否跑完，要看它是否保留了 VAE 的两条约束。

Conditional VAE

conditional VAE 会把条件 y 交给模型，例如告诉 decoder “要生成数字 3”。latent z 更偏向风格、粗细、倾斜等属性，条件 y 负责类别控制。这样可以用一个样本的风格去生成同风格的其他数字。

VAE 的生成缺点

VAE 常用像素级误差衡量重建。一个像素错在笔画末端和错在关键结构上，MSE 可能看起来差不多，但人眼感受完全不同。因此 VAE 生成的图像容易偏平均、偏模糊。

为什么引出 GAN

GAN 不直接要求生成图像和数据库某张图一模一样，而是让 generator 试图骗过 discriminator。它更贴近“看起来像真的”这个目标，但训练很不稳定，loss 也不总能反映图像质量。

误区诊断

DIAGNOSIS

常见误区集中修正

  1. 误区：VAE 只是 autoencoder 加随机噪声。修正：它还要让 q(z|x) 接近先验，使潜空间能被采样。
  2. 误区：variance 越小越好。修正：variance 太小会让模型退回孤立点重建，生成时不可用。
  3. 误区：KL 只是防 overfitting 的普通 L2。修正：KL 约束的是整个分布，不只是参数大小。
  4. 误区：重建越清晰，生成越好。修正：重建质量和采样质量有张力，要同时看潜空间分布。
  5. 误区：GAN 一定比 VAE 好。修正：GAN 图像可能更锐利，但训练信号不稳定，模式崩塌也常见。

错误示例：把 VAE 写回普通 Autoencoder

下面这类代码表面上能重建，但失去了 VAE 的核心：没有采样，也没有 KL 约束。它最容易骗过训练 loss，却在生成阶段暴露问题。

错误示例：漏掉 reparameterization 与 KL复制

z = encoder(x) # 只得到一个固定 code
x_hat = decoder(z)
loss = mse_loss(x_hat, x)
loss.backward()

修法是让 encoder 输出分布参数，用 ε 采样 z，并把 KL 项加入损失。判断这类代码时，先找 mu、logvar、eps 和 kl 是否同时出现。

现象：采样图像乱可能原因是 q(z|x) 没被拉向先验，训练时 decoder 没见过从 N(0,I) 采样来的区域。先检查 KL 是否缺失或权重过低。

现象：重建图像糊可能原因是 KL 过强、latent 容量太小，或像素级损失鼓励平均答案。先区分“重建糊”和“采样乱”这两类问题。

现象：GAN loss 好看但图像怪这对抗训练的评估困难。GAN 需要 generator 与 discriminator 保持竞争，不能只盯一个 loss。
