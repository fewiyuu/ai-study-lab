# 概念骨架 {#skeleton}
<!-- tag: MAP -->

1. 生成任务：从向量到对象
GAN 要训练的是一个生成器。最基本版本里，你从一个简单分布抽出随机向量 `z`，把它丢进 Generator，输出一张图片、一段句子或其他结构化对象。图片在网络里一个很长的向量；每个维度对应像素颜色，重新排列后才成为可看的图片。
判断句：如果你不能说清 `z`、`G(z)` 和图片像素向量之间的关系，就还没真正进入 GAN 的训练问题。


2. 对抗训练：从评价到反馈
Discriminator 也是 neural network。它吃一个完整对象，输出一个分数，分数越高表示越像真实资料。训练时，真实样本要高分，Generator 产生的假样本要低分；随后固定 D，让 G 产生能让 D 给高分的样本。
这里的“骗过”只是拟人化说法。真正发生的是梯度从 D 的输出一路回传到 G 的参数，让 G 朝“让 D 分数更高”的方向更新。


3. 大局观：为什么 GAN 有用
逐像素相似度只能告诉你每个 component 是否接近目标，却不一定懂 component 之间的关系。手写数字多一个孤立像素，逐像素误差可能很小，但整体看起来就不像人写的数字。D 看到的是完整图片，因此更容易学到“眼睛、嘴巴、线条、区域关系”这些整体约束。
GAN 的关键把 G 的反馈从局部差距换成了一个有整体视角的评价函数。

你现在应该能做什么：看到 GAN 图示时，能指出真实样本从 database 来，假样本从 `G(z)` 来，D 的训练目标是区分二者，G 的训练目标是让 D 给假样本高分。

# 术语表 {#terms}
<!-- tag: TERMS -->

术语直觉解释准确说法常见误解
Generator / G从 latent vector 生成对象的网络。一个函数 `G(z)`，输入通常来自 Gaussian 或 uniform distribution，输出图片、句子等结构化对象。以为 G 直接“懂图片”，忽略输出本质是高维向量。
Discriminator / D给完整对象打真实性分数的网络。一个函数 `D(x)`，输入对象，输出 scalar，常被限制在 0 到 1 之间表示真实概率或真实性分数。以为 D 会用语言告诉 G 哪里错；实际是通过梯度反馈。
Adversarial trainingG 和 D 轮流改进。固定 G 训练 D 区分真假，固定 D 训练 G 让假样本得高分，两个 objective 交替更新。以为二者真的是互相“作战”；其实是同一个优化流程的两部分。
Latent code / z控制生成结果的隐藏向量。低维或中维随机向量，被 G 映射到高维对象空间；某些方向可能对应头发长度、颜色、姿态等特征。以为每一维一定能被人工解释；实际可解释性不保证。
Batch size一次训练取多少真实样本和随机向量。从 database sample `N` 张真实图，同时 sample `N` 个 `z` 生成假图，`N` 常见如 32、64。以为 batch size 只是速度参数；它也会影响训练稳定性。
AutoEncoder先压缩再还原。Encoder 把图片变 code，Decoder 从 code 重建图片；Decoder 可被看成一种 Generator。以为重建 loss 低就代表随机生成质量好。
VAE给 code 加噪声，让 Decoder 更稳定。Encoder 输出均值和 variance，从分布 sample 后送入 Decoder，并用额外约束避免 variance 退化为 0。以为 VAE 一定比 GAN 清晰；通常 VAE 更稳但可能更模糊。
Structure learning输出有结构的对象。输出可以是 sequence、matrix、graph、tree；重点在 component 之间的关系。只评估单个 component，忽略整体一致性。
Argmax generation用 D 直接找最高分对象。理论上可找 `argmax_x D(x)` 生成样本，但高维图片空间几乎无法穷举。以为 D 会评价就自然会生成。
FID score衡量生成图片像不像真实图片的指标。FID 越小通常代表生成分布越接近真实分布；课程指出 GAN 对参数敏感，结果 range 很大。只看单张好图，不看分布与稳定性。

读完这一段，你应该能把“G 生成、D 评分、交替训练、整体反馈”四个词连成一句话，而不是分别背定义。

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 从随机向量到可控特征
课程一开始先把 GAN 放在“让机器生成东西”的任务里。最基本的生成器没有文字条件、也没有图片条件，只吃一个随机向量。这个设置看起来不实用，因为你并不知道随机向量具体代表什么；但它是理解 GAN 训练的最干净版本。之后的 conditional generation 才会把文字、图片或标签作为可控条件输入。
图片生成时，G 的输出是一个高维向量。假设图片有很多像素，每个像素又有颜色通道，那么输出维度会非常长。把这个向量重新排成图片，就得到头像。课程用二次元人物头像举例：改变 latent vector 的某些维度，可能让头发变长、颜色变蓝、嘴巴张开。这说明 G 学到的不只是像素表，而是从 latent space 到图片特征的连续映射。
你现在应该能做什么：看到一张由 GAN 生成的脸，能把它拆成“抽样 z → G 输出高维向量 → reshape 成图片 → 某些 z 方向影响视觉特征”。

## box
### Discriminator 函数给分
字幕里用了老师和学生、猎物和天敌的类比来帮助理解，但要马上回到技术对象。D 不会真的对 G 说“你少画了嘴巴”，它只输出一个 scalar。训练 D 时，真实图片被标成高分，G 产生的图片被标成低分。D 学到的判别标准会越来越细：一开始也许只看是不是彩色，后来会看有没有眼睛、嘴巴、线条是否协调。
当 D 固定后，G 与 D 可以接成一个大网络：前半段是 G，后半段是 D，输入 `z`，输出一个分数。训练 G 时只更新前半段参数，让最后的分数变大。必须固定 D，否则模型可以通过改 D 的最后一层让分数变大，而不是让图片真的变好。
判断句：训练 G 时，D 是评分器和梯度来源；训练 D 时，G 是当前假样本生产器。二者角色会在每个 iteration 中切换。

字幕线索 1：技术地位视频开头引用了 GAN 在无监督学习和深度学习中的影响，并提到大量 GAN 变体。学习时不用记住动物园里的每个名字，先掌握基础版的 G/D 结构，后续 LSGAN、WGAN、BGAN 才有位置放。
字幕线索 2：二次元头像真实 database 提供正样本；随机初始化的 G 先产生噪声；D 学会给真实图高分、假图低分；随后 G 逐渐产生更像头像的图。更新次数越多，眼睛、嘴巴、颜色、整体轮廓逐步出现。
字幕线索 3：结构化学习输出图片、句子、翻译结果都属于 structure learning。困难点所有 component 的关系要协调。GAN 的 D 正是从完整对象上给反馈。

完整知识链路：GAN 把生成问题拆成“快速提出候选”和“整体评价候选”。Generator 负责从 latent code 快速生成对象，但单靠逐像素重建容易只学表象；Discriminator 负责检查完整对象是否像真实资料，但单靠它直接生成会碰到高维 argmax 难题。交替训练把两边优点接起来：G 代替手写 argmax 产生强负样本，D 给 G 提供带大局观的训练信号。

# GAN 线索 {#source}
<!-- tag: 线索 -->

这一节字幕最核心的操作流程可以写成两个阶段。每个 iteration 先更新 Discriminator，再更新 Generator。下面公式保留原始 GAN 的直觉版本；课程也提醒，后续会讲更好的 objective，所以这里重点是看懂每一项在做什么。

$$
训练 D：maximize
 1N
Σ log D(x)
 +
 1N
Σ log(1 - D(G(z)))
 
 训练 G：maximize
 1N
Σ log D(G(z))
$$

## box
### 阶段 A：固定 G，只训练 D
1. 从真实 database 抽 `N` 张图，作为 positive examples。
2. 从简单分布抽 `N` 个 `z`，通过当前 G 生成 `N` 张假图，作为 negative examples。
3. 更新 D，让真实图的 `D(x)` 变大，让假图的 `D(G(z))` 变小。
这一步像训练二分类器或回归器。重要的是负样本来自当前 G，而不是固定的随机噪声；因为太弱的负样本会让 D 学到很粗糙的判别标准。

## box
### 阶段 B：固定 D，只训练 G
1. 重新抽一批 `z`，送进 G 得到假图。
2. 把假图送进固定的 D，得到真实性分数。
3. 只更新 G，让 `D(G(z))` 越大越好。
实现时可以把 G 和 D 接成一个大网络。D 的参数被冻结，梯度穿过 D 回到 G。这样 G 学到的是“如何产生让当前 D 认为真实的图片”，而不是直接追逐某一张 target 图片的像素。

伪代码：一次 GAN training looppython

```text
for real_batch in dataloader:
 z = sample_noise(batch_size, dim)
 fake_batch = G(z).detach()
 d_loss = -mean(log(D(real_batch)) + log(1 - D(fake_batch)))
 update(D, d_loss) # 只改 Discriminator

 z = sample_noise(batch_size, dim)
 fake_batch = G(z)
 g_loss = -mean(log(D(fake_batch)))
 update(G, g_loss) # 固定 Discriminator，只改 Generator
```

你现在应该能做什么：给出任意 GAN 代码时，先检查 D step 有没有 detach/freeze G，G step 有没有 freeze D，以及真实样本和假样本是否在同一个 batch 逻辑里被比较。

# 比较与原理 {#example}
<!-- tag: COMPARE -->
<!-- layout: grid2 -->

## box
### 为什么 Generator 不能只靠 AutoEncoder 自学
课程提出第一个问题：既然 Generator 是从 code 到图片的函数，为什么不能直接用 AutoEncoder 的 Decoder 当 Generator？确实可以。Encoder 把图片压成 code，Decoder 从 code 重建图片。训练完成后，拿 Decoder 输入随机 code，就能生成图。
问题在于训练资料只覆盖有限 code。Decoder 见过 `a` 能生成左斜的 1，见过 `b` 能生成右斜的 1，但输入 `(a+b)/2` 时不保证会得到正常的正面 1。VAE 通过给 code 加 noise、约束 variance，让 Decoder 对没见过的 code 更稳定；但 VAE 的重建目标仍常偏向平均化，图片可能比较糊。
更深的问题是逐像素 loss 的评价不等于人的整体评价。一个孤立黑点可能只差一个 pixel，却破坏数字结构；笔画稍微变长可能差多个 pixel，但整体仍像人写的数字。GAN 引入 D，就是为了让 G 的反馈不只来自逐点距离。

## box
### 为什么 Discriminator 不直接生成
D 的优势是大局观。完整图片送进 D 后，它可以用卷积 filter 检查是否有孤立像素、眼睛是否成对、线条是否连贯。它比逐个 component 生成更容易判断 component 之间的 correlation。
但用 D 直接生成，要解的是 `argmax_x D(x)`：在所有可能图片中找一个让 D 给最高分的图片。高维图片空间太大，穷举所有像素组合几乎不可行。传统 graphical model 或 energy-based model 会对函数形式做强假设，才有机会求解；假设越强，表达能力越受限。
GAN 的妙处是让 G 来近似这个 argmax。G 学着产生 D 会给高分的样本，于是不用手写搜索算法；同时 D 又能不断用 G 生成的强负样本修正自己的弱点。

方法生成方式优势风险
AutoEncoderDecoder 从 code 重建图片训练直观，重建目标清楚随机 code 区域不一定稳定，容易只学 reconstruction
VAE从连续分布 sample code 后 decodelatent space 更平滑，训练相对稳图片可能模糊，最优视觉质量常不如 GAN
Discriminator-only求 `argmax_x D(x)`有整体评价能力高维搜索困难，需要强假设
GANG 生成候选，D 给整体反馈有机会生成清晰样本，能学强负样本训练敏感，G/D 失衡时不稳定

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->
<!-- layout: grid2 -->

## box
### 常见坑与修法
1. **把 GAN 当成普通二分类器。** 错因：只看 D，不看 G 的反向训练。后果：讲不清为什么假样本会越来越像真样本。修法：每次都写出 D step 和 G step。
2. **用固定随机噪声当负样本。** 错因：负样本太弱。后果：D 只学到“噪声不像图片”，遇到差一点的头像也可能给高分。修法：让当前 G 产生 hard negative examples。
3. **训练 G 时忘记固定 D。** 错因：把大网络整体都更新。后果：分数可以靠改评分器变大，图片不一定变好。修法：G step 中 freeze D，只让梯度穿过 D。
4. **只用逐像素 loss 判断生成质量。** 错因：把局部距离当整体合理性。后果：孤立像素、模糊平均、结构不协调。修法：引入整体评价或分布级指标。
5. **以为 GAN 一定比 VAE 好。** 错因：只看最好样本。后果：忽略 GAN 的参数敏感和训练失败。修法：同时比较稳定性、最优质量、FID、失败样本。
6. **以为生成图不在训练集就一定没过拟合。** 错因：生成任务不像普通 train/test 分类那样直接。后果：可能只是记住局部模式。修法：扩大资料、检查最近邻样本、看分布覆盖。

## box
### 易混概念对照
概念 A概念 B区别
GeneratorDecoderDecoder 可以作为 Generator，但 AutoEncoder 的训练目标是重建，不等于 GAN 的对抗反馈。
DiscriminatorClassifierD 可以像分类器训练，但它的输出还会成为 G 的训练信号。
Pixel loss整体真实性Pixel loss 看局部差距，整体真实性看 component 关系和分布。
VAE 稳定GAN 清晰VAE 参数结果更集中，GAN 最好结果可能更清晰但 range 更大。
Argmax 搜索Generator 近似前者直接优化图片空间，后者学习一个从 z 到高分样本的映射。
你现在应该能做什么：看到“GAN 不稳定”这句话时，不只回答“调参数”，还要检查 D/G 能力平衡、负样本强度、目标函数、batch size、资料量和评估指标。
