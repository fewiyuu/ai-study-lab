59_P60｜延伸：Flow-based Method

这一节讲的是 flow-based generative model，也常叫 normalizing flow。它的想法很直接：从简单分布抽一个噪声向量 z ，用可逆函数 G 变成复杂样本 x ；因为变换可逆，我们又能把真实样本 x 送回 z ，精确计算它在模型分布下的 likelihood。

讲义主线：最大似然、变量替换与 Flow 目标

LECTURE

1. 最大似然：训练目标先摆正

训练时手里有真实样本 x_1,...,x_N。Maximum likelihood 的目标是让这些真实样本在模型分布下的概率密度越大越好。直觉上，如果真实人脸照片在 p_G 下都很可能出现，模型分布就更像真实人脸分布。更正式地说，最大化资料的 log likelihood 等价于让 p_data 和 p_G 的 KL divergence 变小。

这里的难点在 p_G(x_i)。如果 G 是任意神经网络，模型分布很复杂，通常不知道怎样精确算一个给定 x_i 的密度。

Jacobian 是多变量函数的一阶导数矩阵。假设 F 输入二维向量 z=(z_1,z_2)，输出二维向量 x=(x_1,x_2)。Jacobian 把每个输出对每个输入的偏微分收起来：行对应输出 component，列对应输入 component。

举一个线性例子：x_1=z_1+z_2，x_2=2z_1。那么 J_F=[[1,1],[2,0]]。如果 F 有反函数，F^{-1} 的 Jacobian 和 F 的 Jacobian 互为 inverse；它们的 determinant 也互为倒数。这件事让 G 和 G^{-1} 的密度公式能互相转换。

3. 变量替换公式：密度为什么要除以体积变化

$$ pX(x)=pZ(F-1(x))·det JF-1(x) $$

一维时，这个式子就是 p_X(x)=p_Z(z)|dz/dx|。二维时，|dz/dx| 被 Jacobian determinant 的绝对值取代。字幕里用蓝色小正方形和绿色菱形解释：蓝色方块的概率质量 p_Z(z)·Δz_1·Δz_2 等于绿色菱形的概率质量 p_X(x)·面积。

Flow 的生成方向是 z -> x=G(z)；训练时拿到真实样本 x_i，先用反函数算 z_i=G^{-1}(x_i)，再计算它在简单 prior 里的密度，以及反函数带来的体积变化。

第一项希望真实样本被送回的 z_i 落在 prior 高密度区。如果 prior 是标准正态分布，原点附近密度高，所以它会鼓励 G^{-1} 把样本往原点附近送。第二项防止模型把所有样本都压到同一个点。若所有输入都映射成零向量，Jacobian 会接近零矩阵，determinant 为 0，取 log 后变成负无穷。

Flow 线索：字幕里真正留下的四条线

线索

原字幕从生成模型对比讲起，临时补线性代数，再落到具体 layer。整理这节时，最重要的是把口头讲解收束成四条适合回看的线。

数学前置

Jacobian 记录局部偏微分；determinant 的绝对值记录局部体积变化；change of variable 把体积变化接到 density 上。

架构代价

为了可逆和可算 determinant，G 不能是任意网络。输入输出维度要一致，layer 要特殊设计。

实际层与应用

Coupling layer 通过复制一半变量换来容易反解和三角 Jacobian；Glow 用可逆 1x1 convolution 混合 channel。应用包括人脸 latent interpolation 和语音合成。

可逆给 x 能唯一找回 z。维度不同通常就不可能双射。

反函数好算训练要不断从真实样本走回噪声空间。

determinant 好算不能每次都对巨大 Jacobian 做昂贵运算。

表达力靠堆叠单层受限，就把多层 compose 成一条 flow。

最小例子：一维缩放与 coupling layer

EXAMPLE

一维缩放先建立密度直觉

假设 z 在 [0,1] 上均匀分布，高度是 1。令 x=2z+1，原来长度为 1 的区间会变成 [1,3]，长度变成 2。总概率还必须是 1，所以新分布高度变成 1/2。

你现在应该能判断：变换把空间拉宽时，密度会降低；把空间压窄时，密度会升高。

Flow 的承诺

Flow 不绕开 p_G(x)，而是强迫 G 有足够好的结构：给定 x 可以算回 z=G^{-1}(x)，而且能算出密度变化比例。

Coupling layer 是 NICE、RealNVP 等 flow model 的典型设计。把输入向量 z 切成两半：前半段直接复制到输出，后半段用前半段算出的 scale 和 shift 做 affine transformation。这里的 f 和 h 可以是复杂神经网络，因为取 inverse 时不需要反过来解它们，只要正向再算一次即可。

$$

x1:d=z1:d

β, γ=f(z1:d), h(z1:d)

xd+1:D=β ⊙ zd+1:D + γ

$$

正确示例：coupling layer 的正向和反向Python-like

def forward(z_a, z_b):
 beta = f(z_a) # f 可以很复杂，只要求输出维度对齐
 gamma = h(z_a)
 x_a = z_a
 x_b = beta * z_b + gamma
 log_det = sum(log(abs(beta)))
 return x_a, x_b, log_det

def inverse(x_a, x_b):
 z_a = x_a
 beta = f(z_a)
 gamma = h(z_a)
 z_b = (x_b - gamma) / beta
 inv_log_det = -sum(log(abs(beta)))
 return z_a, z_b, inv_log_det

它的 determinant 也好算。Jacobian 的左上角是 identity，右上角是 zero，整体 determinant 只看右下角；右下角又是 diagonal，determinant 就是对角线元素相乘，也就是 β 的元素相乘。左下角再复杂都不用管。

Worked case：训练一个样本时怎样算 likelihood

假设一个二维 flow 的生成方向是 x_1=2z_1、x_2=0.5z_2。它是可逆的：z_1=x_1/2、z_2=2x_2。训练时拿到真实样本 x=(2,1)，不要从 z 开始生成，而是先反推 z=G^{-1}(x)=(1,2)。

步骤计算对象含义

  1. 反向映射z=(1,2)看真实样本在简单 prior 空间里落在哪里。
  2. prior 分数log p_Z(1,2)如果 z 离标准正态中心太远，这一项会低。
  3. 体积修正J_{G^{-1}} = [[0.5,0],[0,2]]，|det|=1这个变换一维拉宽、一维压窄，总面积不变，所以密度不用额外升降。
  4. 合成分数log p_X(x)=log p_Z(z)+log |det J_{G^{-1}}(x)|真实样本的 likelihood 同时取决于 z 的 prior 密度和局部体积变化。

如果把第二个缩放从 0.5 改成 0.25，反向 Jacobian 的 determinant 会变成 2，log |det| 会增加；这表示生成方向把空间压得更窄，数据空间密度会被抬高。你现在应该能做的事：给任何简单 flow 写出训练方向、prior 项和 log determinant 项，而不是只画 z -> x。

交互观察：体积变化怎样进入 log likelihood

VIZ

局部方块被变换成平行四边形蓝色小方块经过线性近似后变成绿色平行四边形。面积变大，密度就要变小；面积变小，密度就要变大。

x-space

面积 Δz1Δz2面积乘上 |det J|

Glow 让 flow-based model 再次受到注意，其中一个关键层是 invertible 1x1 convolution。对图像来说，每个 pixel 有 RGB 三个 channel。1x1 convolution 在每个位置上用同一个 3×3 矩阵 W 混合这三个 channel：输入是三维向量，输出还是三维向量。

观察 determinant 对 log likelihood 的影响这里用一维缩放做直觉练习。缩放越大，空间越被拉开，密度高度越低；log determinant 项会记录这个变化。

判断句：flow 不是只让样本变漂亮，它还要把每一步空间缩放带来的密度变化记到账上。

误区诊断：哪些说法一听就要停下来

DIAGNOSIS

常见坑

  1. 把本节的 normalizing flow 和其他带 flow 的方法混用。修法：回到 G^{-1}、Jacobian determinant、change of variable 三个信号。
  2. 以为可逆就够了。修法：还要检查 inverse 是否好算、determinant 是否好算。
  3. 把 z 设成低维 latent。修法：flow 为了双射，通常让 z 和 x 维度相同。
  4. 只看生成方向，不看训练方向。修法：训练时真实样本要走 x -> z，并记录 log determinant。
  5. 每层 coupling 都复制同一半。修法：交替 mask 或加入 channel mixing。
  6. 把 1x1 convolution 当普通卷积。修法：这里的核心是每个位置上的通道混合矩阵要可逆，determinant 要能写出来。

易混概念

概念 A概念 B区别可逆determinant 好算可逆只保证能回去；determinant 好算才保证训练目标能便宜计算。低维 latent等维 latentGAN/VAE 常用低维 latent；flow 为了可逆，通常让 z 和 x 维度相同。生成方向训练方向生成用 z -> x；训练真实样本 likelihood 时常用 x -> z。
