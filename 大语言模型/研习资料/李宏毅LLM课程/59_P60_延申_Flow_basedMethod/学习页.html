Normalizing Flow / Flow-based Generative Model

Flow-based Method：把生成变成可逆的变量替换

这一节讲的是 flow-based generative model，也常叫 normalizing flow。它的想法很直接：从简单分布抽一个噪声向量 z，用可逆函数 G 变成复杂样本 x；因为变换可逆，我们又能把真实样本 x 送回 z，精确计算它在模型分布下的 likelihood。

李宏毅老师先把它放在 auto-regressive、VAE、GAN 旁边比较，再补 Jacobian、determinant、change of variable。读完这页，你要能把这些数学对象接回生成模型，而不是只把它们当线性代数公式。

读完你要能做什么

  1. 说清 flow 为什么能直接最大化 likelihood。
  2. 解释 Jacobian determinant 为什么会出现在密度变换里。
  3. 判断一个 flow layer 是否满足可逆、易算 determinant 两个条件。
  4. 用 coupling layer、Glow 和 1x1 convolution 说明常见架构设计。

概念骨架：从简单噪声到精确密度

SKELETON

生成模型要做的事，是学出一个分布。训练资料来自真实分布 p_data，模型通过生成器 G 定义出另一个分布 p_G。好的生成器不是只要能吐出一张看起来不错的图，而是要让 p_G 尽量接近 p_data。

Flow-based method 的骨架是：先从容易计算的 prior 抽样，例如标准正态分布；再通过一串可逆变换把噪声搬到数据空间；训练时走反方向，把真实样本送回 prior 空间，同时用 Jacobian determinant 记录每一步空间体积被拉伸或压缩了多少。

简单噪声 z 可逆 G 复杂样本 x 精确密度 pG(x) 最大似然训练

Auto-regressive

一次生成一个 component，语音这类有时间顺序的数据很自然。缺点是生成慢；图片这种对象还会遇到顺序选择的问题。

VAE / GAN

VAE 优化 likelihood 的下界，不是原始 likelihood。GAN 的视觉质量强，但 generator 和 discriminator 的目标彼此拉扯，训练不稳。

Flow

把简单分布经过可逆变换变成数据分布。它能写出精确 likelihood，但要求输入输出维度相同，并且每层都要可逆、好算 determinant。

术语表：把对象叫准

TERMS

这一节的术语容易互相缠在一起。先把每个词放回它负责的位置：目标函数、变量变换、局部体积、可逆层、应用模型。

术语直觉解释精确定义常见误解
Normalizing flow把简单噪声一步步流成复杂样本。一串可逆且 Jacobian determinant 可计算的变换，用 change of variable 精确计算样本密度。把所有带 flow 的生成方法都当成同一个训练框架。
Maximum likelihood让真实样本在模型下更可能出现。最大化 Σ log p_G(x_i)，等价于让模型分布更接近数据分布的一种训练目标。只看生成图片好不好看，不看真实样本的 density 能不能算。
Jacobian多维函数的局部变化表。把每个输出 component 对每个输入 component 的偏微分排成矩阵。只把它当求导练习，不知道它在描述局部空间变化。
Determinant局部面积或体积被放大几倍。方阵对应的有向体积缩放因子；密度公式里用绝对值。忘记取绝对值，或把它当成和输入位置无关的常数。
Change of variable坐标换了，概率质量不变，密度要跟着体积改。p_X(x)=p_Z(F^{-1}(x))|det J_{F^{-1}}(x)|。只写点的映射，漏掉 Jacobian determinant。
Invertible layer能正走也能反走的一层。给定输出可以唯一还原输入，并能计算反向的 log absolute determinant。以为普通 CNN/RNN 直接就能作为 flow 层。
Coupling layer一半复制，一半用另一半控制缩放和平移。NICE/RealNVP 常用的可逆层，Jacobian 呈三角块结构，determinant 好算。嫌复制一半没做事，忽略它是为了可逆和可算。
Glow / 1x1 convolution用可逆小矩阵混合每个像素的通道。Glow 中的 invertible 1x1 convolution 对每个像素乘同一个可逆矩阵，整体 determinant 是小矩阵 determinant 的多次方。把它当普通卷积，没检查矩阵是否可逆。

讲义主线：最大似然、变量替换与 Flow 目标

LECTURE

1. 最大似然：训练目标先摆正

先把生成器看成一个分布定义器。我们从一个简单分布抽出 z，例如标准正态分布，再送进 G 得到 x=G(z)。虽然 z 的分布很简单，经过神经网络后，x 形成的 p_G 可以很复杂。

训练时手里有真实样本 x_1,...,x_N。Maximum likelihood 的目标是让这些真实样本在模型分布下的概率密度越大越好。直觉上，如果真实人脸照片在 p_G 下都很可能出现，模型分布就更像真实人脸分布。更正式地说，最大化资料的 log likelihood 等价于让 p_data 和 p_G 的 KL divergence 变小。

maximizeL(G)=Σi=1N log pG(xi)
这里的难点在 p_G(x_i)。如果 G 是任意神经网络，模型分布很复杂，通常不知道怎样精确算一个给定 x_i 的密度。

2. Jacobian 与 determinant：局部空间怎么被拉伸

Jacobian 是多变量函数的一阶导数矩阵。假设 F 输入二维向量 z=(z_1,z_2)，输出二维向量 x=(x_1,x_2)。Jacobian 把每个输出对每个输入的偏微分收起来：行对应输出 component，列对应输入 component。

JF(z)=[∂x1∂z1∂x1∂z2;∂x2∂z1∂x2∂z2]

举一个线性例子：x_1=z_1+z_2，x_2=2z_1。那么 J_F=[[1,1],[2,0]]。如果 F 有反函数，F^{-1} 的 Jacobian 和 F 的 Jacobian 互为 inverse；它们的 determinant 也互为倒数。这件事让 G 和 G^{-1} 的密度公式能互相转换。

Determinant 是把一个方阵变成一个数。二维里，[[a,b],[c,d]] 的 determinant 是 ad-bc。这不只是考试公式，它的几何意义是面积缩放倍率；三维里对应体积缩放倍率；更高维里仍然表示局部体积的缩放比例。取绝对值是因为方向可以翻转，但密度和体积都不能是负数。

3. 变量替换公式：密度为什么要除以体积变化

设 x=F(z)，并且 F 可逆。我们想知道 p_Z(z) 和 p_X(x) 的关系。核心约束只有一句：同一小块概率质量不会因为坐标变换凭空增加或减少。小块面积变大，高度就要变小；小块面积变小，高度就要变大。

pX(x)=pZ(F-1(x))·det JF-1(x)

一维时，这个式子就是 p_X(x)=p_Z(z)|dz/dx|。二维时，|dz/dx| 被 Jacobian determinant 的绝对值取代。字幕里用蓝色小正方形和绿色菱形解释：蓝色方块的概率质量 p_Z(z)·Δz_1·Δz_2 等于绿色菱形的概率质量 p_X(x)·面积。

4. Flow 目标函数：真实样本怎样被送回噪声空间

Flow 的生成方向是 z -> x=G(z)；训练时拿到真实样本 x_i，先用反函数算 z_i=G^{-1}(x_i)，再计算它在简单 prior 里的密度，以及反函数带来的体积变化。

log pG(xi)=log pZ(G-1(xi))+logdet JG-1(xi)

第一项希望真实样本被送回的 z_i 落在 prior 高密度区。如果 prior 是标准正态分布，原点附近密度高，所以它会鼓励 G^{-1} 把样本往原点附近送。第二项防止模型把所有样本都压到同一个点。若所有输入都映射成零向量，Jacobian 会接近零矩阵，determinant 为 0，取 log 后变成负无穷。

Flow 线索：字幕里真正留下的四条线

线索

原字幕不是按教科书章节写的，而是从生成模型对比讲起，临时补线性代数，再落到具体 layer。整理这节时，最重要的是把口头讲解收束成四条适合回看的线。

为什么引入 flow

Auto-regressive 慢，VAE 优化 lower bound，GAN 训练不稳定。Flow 的卖点是直接优化 likelihood，并且不用对抗训练。

数学前置

Jacobian 记录局部偏微分；determinant 的绝对值记录局部体积变化；change of variable 把体积变化接到 density 上。

架构代价

为了可逆和可算 determinant，G 不能是任意网络。输入输出维度要一致，layer 要特殊设计。

实际层与应用

Coupling layer 通过复制一半变量换来容易反解和三角 Jacobian；Glow 用可逆 1x1 convolution 混合 channel。应用包括人脸 latent interpolation 和语音合成。

可逆给 x 能唯一找回 z。维度不同通常就不可能双射。
反函数好算训练要不断从真实样本走回噪声空间。
determinant 好算不能每次都对巨大 Jacobian 做昂贵运算。
表达力靠堆叠单层受限，就把多层 compose 成一条 flow。

最小例子：一维缩放与 coupling layer

EXAMPLE

一维缩放先建立密度直觉

假设 z 在 [0,1] 上均匀分布，高度是 1。令 x=2z+1，原来长度为 1 的区间会变成 [1,3]，长度变成 2。总概率还必须是 1，所以新分布高度变成 1/2。

你现在应该能判断：变换把空间拉宽时，密度会降低；把空间压窄时，密度会升高。

Flow 的承诺

Flow 不绕开 p_G(x)，而是强迫 G 有足够好的结构：给定 x 可以算回 z=G^{-1}(x)，而且能算出密度变化比例。

Coupling layer 是 NICE、RealNVP 等 flow model 的典型设计。把输入向量 z 切成两半：前半段直接复制到输出，后半段用前半段算出的 scale 和 shift 做 affine transformation。这里的 f 和 h 可以是复杂神经网络，因为取 inverse 时不需要反过来解它们，只要正向再算一次即可。

x1:d=z1:d
β, γ=f(z1:d), h(z1:d)
xd+1:D=β ⊙ zd+1:D + γ
正确示例：coupling layer 的正向和反向Python-like
def forward(z_a, z_b):
    beta  = f(z_a)      # f 可以很复杂，只要求输出维度对齐
    gamma = h(z_a)
    x_a = z_a
    x_b = beta * z_b + gamma
    log_det = sum(log(abs(beta)))
    return x_a, x_b, log_det

def inverse(x_a, x_b):
    z_a = x_a
    beta  = f(z_a)
    gamma = h(z_a)
    z_b = (x_b - gamma) / beta
    inv_log_det = -sum(log(abs(beta)))
    return z_a, z_b, inv_log_det

它的 determinant 也好算。Jacobian 的左上角是 identity，右上角是 zero，整体 determinant 只看右下角；右下角又是 diagonal，determinant 就是对角线元素相乘，也就是 β 的元素相乘。左下角再复杂都不用管。

Worked case：训练一个样本时怎样算 likelihood

假设一个二维 flow 的生成方向是 x_1=2z_1、x_2=0.5z_2。它是可逆的：z_1=x_1/2、z_2=2x_2。训练时拿到真实样本 x=(2,1)，不要从 z 开始生成，而是先反推 z=G^{-1}(x)=(1,2)。

步骤计算对象含义
1. 反向映射z=(1,2)看真实样本在简单 prior 空间里落在哪里。
2. prior 分数log p_Z(1,2)如果 z 离标准正态中心太远，这一项会低。
3. 体积修正J_{G^{-1}} = [[0.5,0],[0,2]]，|det|=1这个变换一维拉宽、一维压窄，总面积不变，所以密度不用额外升降。
4. 合成分数log p_X(x)=log p_Z(z)+log |det J_{G^{-1}}(x)|真实样本的 likelihood 同时取决于 z 的 prior 密度和局部体积变化。

如果把第二个缩放从 0.5 改成 0.25，反向 Jacobian 的 determinant 会变成 2，log |det| 会增加；这表示生成方向把空间压得更窄，数据空间密度会被抬高。你现在应该能做的事：给任何简单 flow 写出训练方向、prior 项和 log determinant 项，而不是只画 z -> x。

交互观察：体积变化怎样进入 log likelihood

VIZ

局部方块被变换成平行四边形

蓝色小方块经过线性近似后变成绿色平行四边形。面积变大，密度就要变小；面积变小，密度就要变大。

小方块足够小时，复杂函数在这块附近可以近似成线性变换。Jacobian 告诉你每个方向怎么变，determinant 的绝对值告诉你小体积整体变成原来的几倍。

判断句：只要是密度变换，不要只看点被搬到哪里，还要看这个点附近的一小块空间被拉伸了多少。

Glow 让 flow-based model 再次受到注意，其中一个关键层是 invertible 1x1 convolution。对图像来说，每个 pixel 有 RGB 三个 channel。1x1 convolution 在每个位置上用同一个 3×3 矩阵 W 混合这三个 channel：输入是三维向量，输出还是三维向量。

xp=W zpfor each pixelp
det J=det(W)H·W

这里用一维缩放做直觉练习。缩放越大，空间越被拉开，密度高度越低；log determinant 项会记录这个变化。

1.50
区间长度1.50
密度高度0.67
log |det|0.41

判断句：flow 不是只让样本变漂亮，它还要把每一步空间缩放带来的密度变化记到账上。

Glow 的 demo 可以做 latent space interpolation：把两张脸送进 G^{-1} 得到两个 z，在 z 空间平均，再送回 G，得到两者之间的脸。也可以用“笑脸平均向量减不笑平均向量”找出让脸变笑的方向。语音生成里也有 Parallel WaveNet、WaveGlow 这类把 flow 思想用到快速合成的模型。

误区诊断：哪些说法一听就要停下来

DIAGNOSIS

常见坑

  1. 把本节的 normalizing flow 和其他带 flow 的方法混用。修法：回到 G^{-1}、Jacobian determinant、change of variable 三个信号。
  2. 以为可逆就够了。修法：还要检查 inverse 是否好算、determinant 是否好算。
  3. 把 z 设成低维 latent。修法：flow 为了双射，通常让 z 和 x 维度相同。
  4. 只看生成方向，不看训练方向。修法：训练时真实样本要走 x -> z，并记录 log determinant。
  5. 每层 coupling 都复制同一半。修法：交替 mask 或加入 channel mixing。
  6. 把 1x1 convolution 当普通卷积。修法：这里的核心是每个位置上的通道混合矩阵要可逆，determinant 要能写出来。

易混概念

概念 A概念 B区别
可逆determinant 好算可逆只保证能回去；determinant 好算才保证训练目标能便宜计算。
低维 latent等维 latentGAN/VAE 常用低维 latent；flow 为了可逆，通常让 z 和 x 维度相同。
生成方向训练方向生成用 z -> x；训练真实样本 likelihood 时常用 x -> z。
