# 概念骨架 {#skeleton}
<!-- tag: MAP -->

训练方向：自己制造噪声标签
从资料库拿一张干净图片，随机采样一组噪声，把噪声按某个 timestep 加到图片里。因为噪声是我们自己加的，所以目标答案不需要人工标注。
判断句：看到训练题时，先问“模型输入的是哪一张 noisy image，目标输出是哪一组噪声”。


生成方向：同一个模型反复用
推理时先采样一张纯噪声图，然后用同一个 denoise module 从大 timestep 走到小 timestep。每一步都预测当前图里的噪声，再把它扣掉一部分。
判断句：如果解释里没有 timestep，模型就不知道现在该处理“很脏的图”还是“快干净的图”。


文字条件：不是最后贴标签
text-to-image 不是先无条件生成图，再把文字描述贴上去。文字描述会作为条件进入每一步 noise predictor，引导模型删掉不符合文字的噪声成分。
判断句：如果训练资料只有类别标签，没有具体描述，就很难学会细粒度 prompt 的对应关系。

**FORWARD****x0 → xt**训练时加噪，知道真实噪声。
**REVERSE****xT → x0**生成时去噪，只能靠模型预测噪声。
**CONDITION****c**文字、类别或其他控制条件。

# 术语表 {#terms}
<!-- tag: TERMS -->

术语直觉解释准确说法常见误解
DDPM最经典的一类去噪扩散模型。Denoising Diffusion Probabilistic Model，用加噪过程定义训练任务，再学习反向去噪。以为它只是“把噪声滤镜调低”的图像处理算法。
forward process把干净图片一步步弄脏。训练时按预设噪声日程，把采样噪声加入 x0，得到不同 timestep 的 xt。以为 forward 是模型生成图片的方向。
reverse process从纯噪声一步步还原图片。推理时从 xT 开始，多次调用去噪网络，逐步得到更接近数据分布的样本。以为它一步就输出最终图。
noise predictor负责猜“当前图里哪些部分是噪声”。输入 noisy image、timestep 和可选条件，输出预测噪声 εθ。以为它直接输出干净图片。
timestep告诉模型现在噪声有多严重。一个表示加噪/去噪阶段的编号或嵌入，常从大步数走向小步数。把 timestep 和训练迭代 step 混成同一件事。
文字条件告诉模型要往哪类图像去。text embedding 或 caption 条件，和 xt、t 一起输入 noise predictor。以为 prompt 只在最后筛选生成图。
图文配对资料一张图配一段描述，让模型学文字和视觉的对应。text-to-image 训练通常依赖大量 image-caption pairs，资料规模会直接影响语义覆盖。以为只有图片或只有类别标签就足够支持复杂文字生图。

# 讲义主线 {#lecture}
<!-- tag: LECTURE -->
<!-- layout: grid2 -->

## box
### 1. 为什么从噪声开始
扩散模型的生成入口一组从 normal distribution 采样出来的随机数。要生成 256×256 的图片，就需要同样尺寸的向量或张量，再把它看成一张“全是杂讯”的图。
这个设定的好处是起点很简单：随机噪声容易采样。难点被转移到“如何一步步删掉不该留下的噪声”。李宏毅在课里用了雕像的说法：图像好像已经藏在噪声里，模型要做的是把不属于目标图像的部分拿掉。
这说训练后的反向过程会把随机样本推向训练资料分布。你看到的是一个概率生成过程，不是一次确定性的修图。

## box
### 2. 同一个 denoise module 为什么还要吃 timestep
生成过程会反复调用同一个 denoise module，但每一步的输入状态不同。刚开始的图几乎全是噪声，后面几步已经能看出轮廓。对同一个网络来说，这两种输入需要不同的处理强度。
所以模型除了看 noisy image，还要知道现在处在哪个 timestep。编号大，代表噪声重；编号小，代表接近结束。实际系统常把这个编号转成 embedding，再交给网络使用。
少了 timestep，模型会被迫用同一种策略处理所有噪声程度。它可能在早期去噪不够，在后期又把细节删坏。以后看到 diffusion 结构图，先检查输入里有没有 `x_t` 和 `t`。

$$
xt = √ᾱt · x0 + √(1 - ᾱt) · ε，ε ~ N(0, I)
 εθ(xt, t, c) ≈ ε
 第一行表示训练时怎样把干净图 x0 加噪成 xt；第二行表示模型学习预测我们加进去的噪声。c 是可选条件，例如文字描述。
$$

3. 为什么预测噪声，而不是直接输出干净图
直接让网络从一张 noisy image 输出干净图当然也可以尝试，但它相当于让模型在每一步都“画出一张带细节的图”。预测噪声更像在回答“当前这张图里哪部分该被删掉”。
课程里强调，这两件事难度不同。生成一张带噪声的猫，几乎要求模型已经会画猫；预测噪声则把任务拆得更局部、更稳定。多数 DDPM 论文因此选择训练 noise predictor。


4. 训练资料如何自动造出来
noise predictor 需要成对资料：输入是 noisy image 和 timestep，目标是对应噪声。这个标签不用人工标，因为训练时的噪声本来就是我们随机采样后加进去的。
具体做法是：从资料库取干净图，采样噪声，按某个 timestep 加进去。你知道加进去的 ε 长什么样，于是它就成了 ground truth。接下来就和一般神经网络训练一样，最小化预测噪声与真实噪声之间的误差。


5. forward 和 reverse 不要反着记
forward process 又叫 diffusion process，是“把资料扩散成噪声”的训练构造过程。reverse process 才是生成时“从噪声回到图像”的采样过程。
判断时可以看方向：如果起点是干净训练图，通常是在讲 forward；如果起点是纯噪声，通常是在讲 reverse。这个区分会影响你读公式、读代码和读流程图。


6. 文字怎么进入 diffusion
如果只训练无条件生图，模型只需要从噪声生成“像资料集的图片”。要做 text-to-image，就要让每一步去噪都知道文字要求。于是 noise predictor 的输入从 `(x_t, t)` 变成 `(x_t, t, text)`。
训练也要跟着变化：每张图片需要有对应文字描述。把图片加噪后，模型看 noisy image、timestep 和 caption，目标仍然是预测噪声。文字条件不改变“预测噪声”这个任务，只改变模型应该朝哪种图像分布去去噪。


7. 资料规模为什么重要
课程里把不同资料规模放在一起比较：几万张图片只能支持很有限的作业练习；ImageNet 约百万张图片，但更多是类别标签，不是细致 caption；LAION 级别的图文配对资料达到数十亿张，才让模型见过大量对象、风格、语言和描述方式。
这也解释了为什么一些影像生成系统能理解中文、日文或名人名字：训练资料里可能出现过这些语言和对应图像。这里也带来隐私和版权边界，模型能生成名人脸，往往意味着资料分布里有足够多相关样本。


8. 完整链路
把 P61 串起来：DDPM 先定义一个可控的加噪过程，让干净图在不同 timestep 变成不同程度的 noisy image；因为噪声由我们采样，所以训练标签天然存在。网络学习 `noise predictor` 后，推理时从随机噪声出发，按 timestep 逐步预测并扣掉噪声。若要文字生图，就把文字条件也输入每一步 noise predictor，并用图文配对资料训练这种对应关系。
你读完这一段，应该能把“训练时加噪”和“生成时去噪”分开讲，也能指出文字条件参与的是每一步去噪，而不是最后给图片贴标签。

# 精讲例子 {#workthrough}
<!-- tag: WORKED -->
<!-- layout: grid2 -->

## box soft
### 直觉：模型学的不是“修图滤镜”
Diffusion 的直觉可以这样抓：先设计一个可控的“弄脏过程”，把真实图片变成不同程度的 noisy image；再训练模型学会在任意噪声程度下指出“这张图里哪些成分像是后来加进去的噪声”。生成时从纯噪声出发，模型每一步都把样本往训练资料分布里推一点。
所以随机噪声不是本来就藏着某张猫图。更准确的说法是：训练后的反向过程把一个随机样本，逐步改造成“像训练资料且符合条件”的样本。

## box warn
### 术语边界：五个符号不要混
- `x0` 是干净训练图片，不是模型输出的默认目标。
- `xt` 是在第 `t` 个噪声阶段得到的 noisy image。
- `ε` 是我们自己采样并加进去的真实噪声。
- `εθ` 是 noise predictor 对噪声的预测。
- `c` 是条件，例如 caption、类别或其他控制信息。
最容易混的是 `timestep` 和 `training step`：前者描述图片当前有多脏，后者描述参数更新到第几次。

$$
worked example：一张“猫在花园里”的训练图片
 取 x0 = 干净猫图，采样 t = 700，采样 ε，算出 x700。模型看见的是 (x700, 700, “a cat in a garden”)，训练目标是预测 ε，而不是直接背出原图。
 这一步的监督标签来自“我们自己加了什么噪声”，所以不需要人工给每个像素标注答案。
$$

训练怎么读
如果模型预测的 `εθ` 和真实 `ε` 很接近，就说明它知道当前 noisy image 里哪些部分应该被删。loss 下降，不代表它已经会一步画出最终图，而是表示它更会做局部去噪判断。


生成怎么读
推理时没有干净图，也没有真实噪声标签。系统从 `xT` 开始，反复调用同一个 predictor：输入当前 `xt`、当前 `t` 和文字条件，预测噪声，再更新到更干净的状态。


错误答案长什么样
“模型先生成一张随机猫图，最后再按 prompt 贴上花园标签”是错的。prompt 不是最后筛选器，它作为条件参与每一步去噪；如果训练资料没有细致 caption，模型就很难稳定响应细节 prompt。

你现在应该能做什么：给任意一个 diffusion 结构图标出训练输入、训练目标、推理起点、timestep 和文字条件进入的位置。

# 课程线索 {#source}
<!-- tag: SOURCE -->

下面把课程中的口头说明整理成可复习线索。重点是把例子接回流程，去掉口语重复。

线索 1：DDPM 是主参考
课程说明以 Denoising Diffusion Probabilistic Model 为主要例子，并指出 DALL·E、Imagen、Stable Diffusion 等影像生成系统都与这一类思路有关。
复习点：DDPM 是理解扩散模型的入口，不等于所有现代系统的完整实现。


线索 2：生成从纯噪声开始
推理时先从高斯分布采样与目标图片尺寸相同的向量，把它当作噪声图片，再从大编号 timestep 一路去噪到小编号 timestep。
复习点：大 timestep 通常对应更严重的噪声，小 timestep 更接近最终图片。


线索 3：denoise module 里实际是 noise predictor
去噪模块预测当前图里的噪声，再把这份噪声从输入图中扣掉。课程强调多数论文选择这种目标，是因为预测噪声比直接生成干净结果更容易训练。
复习点：题目问“模型输出什么”时，优先答预测噪声，不要顺手写成最终图片。


线索 4：训练资料由加噪过程产生
从资料库拿干净图片，自己加随机噪声，记录 timestep 和真实噪声。于是输入与目标成对出现，noise predictor 可以用监督学习训练。
复习点：噪声标签不是人工画出来的，是采样时天然知道的。


线索 5：文字生图需要图文对
要让模型听懂 prompt，就需要图片和文字成对的资料。只有类别标签的数据集不等于详细描述；大规模图文资料能提供更多对象、语言和风格的覆盖。
复习点：文字条件进入训练和推理的每一步去噪。


线索 6：资料规模影响能力边界
课程用 7 万张作业资料、ImageNet 级别资料和 LAION 5.85B 图文资料作对比，说明好看的文字生图背后通常有极大规模的训练资料。
复习点：能力不是只靠算法名字，资料规模、caption 质量和语种覆盖也很关键。

# 最小例子 {#example}
<!-- tag: EXAMPLE -->

这个例子用伪代码表示 DDPM 训练和采样的最小结构。它不追求可直接运行，目的是让你看懂每行在读什么、写什么。

训练示例：学习预测噪声python

```text
image = sample_clean_image(dataset) # x0：资料库里的干净图
t = sample_timestep(1, T) # 现在要模拟哪一个噪声程度
noise = sample_normal_like(image) # ε：我们自己采样的真实噪声
noisy_image = add_noise(image, noise, t) # xt：按 timestep 加噪后的图

pred_noise = noise_predictor(noisy_image, t, text=None)
loss = mse(pred_noise, noise) # 训练目标：预测加进去的噪声
loss.backward()
optimizer.step()
```

采样示例：从噪声逐步生成图像python

```text
x = sample_normal(shape=(3, 256, 256)) # xT：纯噪声起点
for t in reversed(range(1, T + 1)):
 pred_noise = noise_predictor(x, t, text="a cat in a garden")
 x = denoise_step(x, pred_noise, t) # 用预测噪声更新到更干净的状态

image = decode_or_clip(x) # 最后得到可显示的图片
```

## box soft
### 逐行拆解
`sample_timestep` 决定这次训练样本的噪声严重程度；`sample_normal_like` 产生真实噪声；`add_noise` 得到模型输入；`mse` 让模型输出贴近真实噪声。删掉 timestep，模型就少了“现在有多脏”的信息。

## box warn
### 错误示例：把目标写成干净图
如果误把 `loss = mse(pred_image, image)` 当作本节主线，就会把 DDPM 的典型噪声预测目标讲偏。P61 讲的主流 DDPM 解释围绕 noise predictor。

# 误区诊断 {#diagnosis}
<!-- tag: DIAGNOSIS -->
<!-- layout: grid2 -->

## box redsoft
### 常见坑：错因、后果、修法
1. **把 forward process 当成生成方向。** 后果是读流程时会把训练和推理说反。修法：干净图加噪是 forward，纯噪声去噪才是 reverse。
2. **说 denoise module 直接输出干净图。** 后果是忽略本节核心的 noise predictor。修法：先说预测噪声，再说用预测噪声更新当前图。
3. **漏掉 timestep。** 后果是无法解释同一个模型如何处理不同噪声程度。修法：把 `x_t` 和 `t` 一起写进输入。
4. **以为文字条件只在最后筛图。** 后果是讲不清 text-to-image 如何按 prompt 生成。修法：文字条件进入每一步 noise predictor。
5. **把类别标签当成详细 caption。** 后果是高估资料能支持的 prompt 细节。修法：区分 label、caption、图文配对规模。
6. **只看单张生成图判断模型好坏。** 后果是被随机种子、prompt 偶然性和挑图影响。修法：固定 prompt 集、多个 seed、同时看语义一致性和图像质量。
