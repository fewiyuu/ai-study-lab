Transformer 的核心，是让 token 互相读取

大语言模型里的 Transformer 不是一个神秘黑箱。它反复做一件事：让每个位置根据当前上下文更新自己的表示。attention 负责从其他位置取信息，前馈网络负责对每个位置的表示做非线性加工，残差和归一化让很多层堆起来后仍能训练。

这页先讲直观链路，不追求完整推导。你需要能读懂公式里的 QK^T、softmax、mask、加权求和，也要能说清 Transformer 为什么容易并行，以及它为什么在长序列上会变贵。

token 向量→ 自注意力读取上下文→ 残差 + LayerNorm→ FFN 加工→ 下一层表示

学完后要能做四件事

  1. 用一句话解释 attention：每个位置按相关性读取其他位置的信息。
  2. 区分 Q、K、V：Q 提出读取条件，K 参与匹配，V 提供被汇总内容。
  3. 能解释 causal mask 为什么挡住未来 token。
  4. 看到最小 attention 代码时，能检查转置、softmax 维度、缩放和 mask 顺序。

整体骨架：一个 block 里发生了什么

MAP

概念：attention 是读上下文

每个 token 位置都有自己的表示。attention 让这个位置根据相关性去读取其他位置的 value，然后汇总成新的表示。它不是只看邻居，也不是只看自己。

判断句：看到 attention，就问"当前 query 在读哪些 key 对应的 value"。

边界：Transformer 不等于 attention

attention 是 Transformer block 的一部分。完整 block 还包含线性投影、多头合并、残差连接、LayerNorm 和 FFN。只会背 attention 公式，还不足以读懂模型结构。

判断句：解释 Transformer 时，至少要说 attention 和 FFN 两段。

最小工程动作：先查形状

读代码时先确认 [B, N, D] 是否一致。attention 权重常是 [B, heads, N, N]，输出要回到 [B, N, D]。形状不清，后面很容易把维度意义说错。

判断句：能跑不代表 softmax 维度正确。

attention 直觉：一行向量怎样读取其他行

ATTN

从"相关性表"开始理解

假设一句话被切成 N 个 token。self-attention 会为每个位置计算它和其他位置的相关性，得到一个 N × N 的表。第 i 行表示第 i 个 token 在读取整句时，分别给每个 token 多少权重。

softmax 把原始分数变成每行和为 1 的权重。权重不是最终答案，只是这次混合时的比例。权重乘以 V 后，每个位置都会得到一个新的向量，这个向量带上了它读取到的上下文信息。

为什么 Transformer 比 RNN 更容易并行？RNN 要按时间步一个接一个处理，后一个位置依赖前一个位置的隐藏状态。self-attention 可以一次计算所有位置之间的相关性，所以训练时更容易并行。代价也明显：N 越长，N × N 关系表越大。

到这里，你应该能说清：attention 的输出仍然是一串向量，只是每个位置的向量已经混入了上下文。

试一试：切换 query 看读取对象

Attention(Q, K, V) = softmax((QKT) / sqrt(dk)) V

这个公式可以分四步读：先用 Q 和 K 做点积得到匹配分数；再除以 sqrt(d_k) 稳住分数尺度；然后对每一行做 softmax 得到读取权重；最后用权重对 V 做加权求和。公式短，但每一步都有明确语义。

Q、K、V：不是三份文本，而是三种投影视角

QKV

为什么要拆成三套向量

输入 X 经过三组不同线性层，得到 Q、K、V。它们来自同一批 token 表示，但用途不同。Q 是当前位置提出的读取条件，K 是每个位置暴露出来用于匹配的索引，V 是真正被汇总回来的内容。

如果只用同一份向量同时做匹配和输出，模型表达会更受限。拆成 Q/K/V 后，模型可以学到"用一种特征判断相关性，用另一种特征提供内容"。例如在代码补全里，当前位置可能用语法需求去匹配函数定义，但真正需要汇总的是参数名、返回值或上下文变量。

常见错法是把 Q、K、V 解释成三段不同文本。self-attention 里它们通常来自同一个输入序列，只是经过不同线性投影。encoder-decoder attention 才会出现 decoder 的 Q 去读取 encoder 的 K/V。

易读错的形状

对象常见形状语义
X[B, N, D]B 条样本，每条 N 个 token，每个 token D 维
Q/K/V[B, H, N, Dh]分成 H 个头，每个头宽度 Dh
scores[B, H, N, N]每个 head 中，每个 query 对每个 key 的分数
output[B, N, D]多头合并后回到原隐藏宽度

多头 attention 不是复制多个模型。它是在同一层里让不同头学习不同的读取模式。

逐行拆解：QK^T 得到每个位置对每个位置的分数python
import torch
B, H, N, Dh = 2, 4, 5, 16
q = torch.randn(B, H, N, Dh)
k = torch.randn(B, H, N, Dh)
v = torch.randn(B, H, N, Dh)

scores = q @ k.transpose(-2, -1)   # [B, H, N, N]
weights = torch.softmax(scores / (Dh ** 0.5), dim=-1)
out = weights @ v                       # [B, H, N, Dh]

关键行是 k.transpose(-2, -1)。如果不转置，最后两个维度无法形成 query 对 key 的两两打分。softmax 沿最后一维做，因为每个 query 要在所有 key 位置之间分配读取比例。最后乘 V，输出仍保留每个 query 位置，只是内容被上下文更新。

Transformer block：attention 之后还要加工和稳定

BLOCK

残差连接解决什么

残差连接就是把模块输出和原输入相加。它让信息有一条更直接的通路，也让深层网络训练更稳。没有残差时，很多层连续变换可能让早期信息丢得太快，梯度传播也更困难。

LayerNorm 用来稳定每个位置的特征分布。它不是激活函数，也不是 dropout。它通常和残差配合使用，让下一层看到的数值范围更可控。不同模型会有 Pre-LN 或 Post-LN 的结构差异，现代大模型常见 Pre-LN。

FFN 是逐位置的前馈网络，常见结构是线性升维、激活、线性降维。attention 负责跨 token 混合信息，FFN 负责在每个 token 内部加工特征。两者配合，才构成一个完整 block。

三组角色边界

容易混的概念区别
attention / FFNattention 跨位置混合；FFN 对每个位置单独加工。
residual / LayerNormresidual 保留原输入通路；LayerNorm 调整数值分布。
multi-head / multi-layermulti-head 是同一层内多个读取视角；multi-layer 是多层 block 堆叠。
hidden size / head dimhidden size 是合并后的总宽度；head dim 是每个头的宽度。
最小结构：一个 decoder block 的读法python
def decoder_block(x, causal_mask):
    attn_out = self_attention(x, mask=causal_mask)
    x = layer_norm(x + attn_out)      # 残差保留原信息，norm 稳定数值

    ffn_out = feed_forward(x)
    x = layer_norm(x + ffn_out)       # FFN 对每个位置继续加工
    return x

这段伪代码省略了多头拆分、dropout 和具体参数，但顺序足够读懂主线。第一段让位置之间通信，第二段对每个位置的表示再加工。删掉 attention，模型缺少跨 token 读取；删掉 FFN，模型的非线性变换能力会弱；删掉残差和归一化，深层堆叠会更不稳。

mask 与生成：为什么 decoder 不能看未来

MASK

causal mask 挡住什么

自回归语言模型按"预测下一个 token"训练和生成。当前位置只能基于前面已经出现的 token 预测后面的 token。如果训练时让它看见未来 token，它就会学到作弊路径，推理时这些未来信息不存在，训练和推理条件就对不上。

causal mask 通常在 softmax 前加到 scores 上，把未来位置设成很小的负数。这样 softmax 后，未来位置权重接近 0。顺序很重要：先 mask 再 softmax。先 softmax 再置零，会破坏每行权重归一化，也可能让实现出现隐蔽偏差。

encoder-only 模型通常不需要 causal mask，因为它们的任务允许双向看整句，例如分类或填空式预训练。decoder-only 生成模型需要 causal mask，因为每一步只能用已知前缀预测下一个 token。

看一眼 mask 矩阵

观察点：第 3 个位置可以看 1、2、3，不能看第 4、5 个未来位置。

masked_scores = scores + causal_mask
weights = softmax(masked_scores, dim = last_dimension)

这里的 causal mask 不是把 token 从序列里删掉，而是在注意力分数里挡住非法读取。输出序列长度仍然不变，每个位置仍然会产生表示，只是它不能从未来位置拿信息。

最小代码：能验收 attention 实现

CODE

正确实现要查四个点

第一，Q/K/V 是否来自正确输入，形状是否拆成了 head。第二，计算 scores 时 K 是否转置到最后两个维度。第三，缩放是否使用 head_dim，而不是 hidden_size 或序列长度。第四，mask 是否在 softmax 前加入，softmax 是否沿 key 维度。

只看输出 shape 不够。错误的 softmax 维度也可能输出同样形状，但语义已经变了。比如沿 query 维度 softmax，会变成"每个 key 被多少 query 读取"的归一化，不是"每个 query 读取哪些 key"。

错误示例：softmax 维度错

错误示例：沿 query 维度归一化python
scores = q @ k.transpose(-2, -1)
weights = torch.softmax(scores, dim=-2)  # 错：这里不是 key 维度
out = weights @ v

这段代码可能不报错，因为形状仍然能对上。问题是每个 query 没有在所有 key 上分配读取比例。修法是沿最后一维 softmax：dim=-1。

正确示例：mask、缩放和 softmax 顺序python
def scaled_dot_product_attention(q, k, v, mask=None):
    head_dim = q.size(-1)
    scores = q @ k.transpose(-2, -1)
    scores = scores / (head_dim ** 0.5)
    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)
    weights = torch.softmax(scores, dim=-1)
    out = weights @ v
    return out, weights

顺序不要随便调。缩放放在 softmax 前，mask 也放在 softmax 前。最后一维是 key 位置，所以 softmax 沿最后一维。返回 weights 方便调试，但不要把它当成完整解释模型行为的证据。

误区、边界和易混概念

DIAGNOSIS

五个常见误区

  1. 把 attention 当成"只看重要词"。错因是忽略每个位置都有一行权重；后果是说不清输出序列；修法是按 query 行解释。
  2. 把 Q、K、V 当成三段不同文本。错因是混淆投影和输入来源；后果是读 self-attention 代码时理解错；修法是说明它们通常都由 X 线性投影得到。
  3. softmax 维度写错。错因是只追形状；后果是权重语义变了；修法是沿 key 维度归一化。
  4. mask 放在 softmax 后。错因是把置零当成屏蔽；后果是概率归一化不稳；修法是先把非法 scores 设为极小值再 softmax。
  5. 认为 Transformer 没有顺序信息。错因是看到并行计算就忽略位置编码；后果是解释模型时漏掉位置；修法是说明 token 向量还要加入或使用位置信息。

三组易混概念

概念准确区别判断方法
attention score / attention weightscore 是 softmax 前分数；weight 是 softmax 后权重。看是否已经归一化、每行是否约等于 1。
causal mask / padding maskcausal mask 挡未来；padding mask 挡补齐位置。看它屏蔽的是三角区域还是 pad token。
encoder / decoderencoder 常双向读输入；decoder 自回归生成，通常有 causal mask。看任务是理解整段输入，还是逐步生成下一个 token。
attention / 解释性权重显示混合比例，不等于严格因果解释。不要只凭一张权重图判断模型为什么这样答。
