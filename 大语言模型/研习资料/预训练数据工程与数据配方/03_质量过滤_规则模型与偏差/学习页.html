质量过滤

质量过滤：规则、模型与偏差

这一页只处理过滤前半段：先把质量信号看清，再决定用规则过滤还是分类器过滤，最后把阈值和偏差审计补上。等你能说出一条样本为什么被留、为什么被删、删掉后会偏向什么，这一单元才算真正接住了后面的去重和配方。

课程地图上一单元下一单元

读完这一页，你应该能做什么

  • 能把质量信号分成规则可见、语义可见和需要人工复核三类。
  • 能说明规则过滤、分类器过滤和阈值选择分别改了什么。
  • 能按分桶删除率、来源覆盖和样本上下文做一轮偏差审计。

学习目标

boundary

先认信号

这一页先判断文本里有哪些信号能直接看见：长度、乱码、字符比例、语言混杂、模板痕迹和重复片段。

再认机制

规则过滤负责可解释、可快速批处理的问题；分类器过滤负责语义质量判断；阈值负责把分数变成保留、删除和复核动作。

最后认偏差

任何过滤器都会改变数据分布。你需要能看出删掉了什么、对谁删得更多、上下文是否被误伤，以及这会把训练预算推向哪里。

概念骨架

trace

这页的主线只有一条：先看输入里能被检查的信号，再选择规则或模型，再把输出变成保留、删除、降权或复核。中间每一步都应该留下能回看的证据，不然过滤就只剩一个黑盒决定。

document signals 读取长度、字符组成、语言线索、模板痕迹和来源字段。

这里回答的是“这段文本看起来像什么”。如果信号层都不稳，后面的模型和阈值只会放大噪声。

rule pass 读取 blocklist、格式约束、长度阈值和占比规则。

这里写出可解释的删除条件。规则适合处理显式噪声，但也最容易误杀长尾内容。

classifier score 读取已标注样本、特征和模型输出分数。

这里处理“看起来像高质量但很难手写规则”的样本。分数不是真值，它只是在某个标注定义下的近似判断。

keep / drop / review 读取规则命中、模型分数和阈值。

这里把分数变成动作。阈值一改，预算、留存比例和偏差都会改，所以它从来不只是技术参数。

层 读取什么 输出什么 失败信号
信号层 长度、字符比例、语言线索、来源字段 可见的质量特征 把噪声、乱码和模板都混成一个“低质”标签
规则层 blocklist、格式约束、阈值 显式删除或保留 规则写得太硬，误杀长尾知识
分类器层 人工标注、特征、模型参数 质量分数 分数看起来稳定，实际继承了标注偏差
动作层 规则命中、分数、阈值 keep / drop / review 只看整体留存率，不看分桶偏差

质量信号

signals

先看信号，意思是先看文本里真正能被稳定检查的东西。长度太短、字符分布极端、乱码、模板痕迹、语言混杂、重复片段和明显的 blocklist 命中，这些都属于能直接观察的信号。

最先看的四类信号

  • 结构信号：段落数、标题层级、列表和引用是否完整。
  • 字符信号：非字母数字比例、标点密度、重复字符和乱码。
  • 语言信号：语言识别分布、混语比例、脚本混杂。
  • 来源信号：域名、路径、站点类型和抓取批次。

容易误读的信号

短文本不一定差，长文本也不一定好；非字母数字比例高不一定是噪声，也可能是代码、公式或表格。信号的作用是提示你去看上下文，而不是替你直接下结论。

信号 常见触发 优先动作 可能误伤
过短 标题、导航、评论碎片 标记为低信息密度 短知识卡、目录、代码注释
乱码 解码失败或字符集错配 回查编码和抓取记录 含特殊符号的正式文本
模板痕迹 页眉、页脚、推荐阅读、导航 进入规则过滤候选 带重复结构的教学页
语言混杂 多语言站点、代码片段、引用 拆分来源或降权 技术文档里的合法英文术语

规则过滤

rule-based

规则过滤适合处理能清楚写出来的问题：长度阈值、blocklist、字符占比、格式约束、域名白名单和模板结构。它的优势是快、便宜、可解释；它的代价是写得太粗时很容易把真正有价值的长尾文本一起删掉。

rule_config.yaml 
filters:
  - name: min_length
    keep_if: text_length >= 180
  - name: blocklist
    delete_if_any:
      - "登录"
      - "广告"
      - "推荐阅读"
  - name: punctuation_ratio
    review_if: punctuation_ratio > 0.35
  - name: language_gate
    keep_if: lang_confidence >= 0.80
actions:
  low_confidence: review
  hard_hit: drop
  pass: keep
case 1: text_length=42, blocklist_hit=1, action=drop
规则 读取什么 输出什么 失败信号
长度阈值 文本字符数 keep / drop / review 把短但重要的标题、注释或代码解释删掉
blocklist 关键词和上下文窗口 硬删除或人工复核 词命中但语境正常的样本被误删
字符占比 标点、数字、非字母数字比例 异常标记 代码、公式、表格被当成噪声
语言门控 语言识别和置信度 保留、降权或拆分 多语技术文档被粗暴切碎

分类器过滤

classifier

规则处理的是“说得清”的问题，分类器处理的是“很难一条条写规则”的质量判断。教育性、回答完整性、信息密度、语义连贯性和事实性，常常更适合让分类器给分，但前提是你知道它是按什么标注标准训练出来的。

分类器链条

标注定义决定什么叫“高质量”，训练样本决定模型学到什么，输出分数决定后续怎么切阈值。三者里任何一个偏了，分数都不再只是“客观评价”。

你需要记录的东西

至少要记：标注来源、标签定义、正负样本构成、训练版本、分数含义和哪个阈值对应哪个动作。没有这些字段，分数就没法回查。

classifier flow 
document -> feature extraction -> quality classifier -> score
score >= retain_threshold   -> keep
score between bands         -> review
score < reject_threshold    -> drop
环节 读取什么 写什么 失败信号
标注定义 质量标准、写作目标、样本边界 可训练标签 同一句文本在不同标注者手里标准不一致
训练样本 人工标注和筛选规则 模型参数 模型学会了标注风格，不是质量本身
推理分数 新 document 质量分 整体 AUC 好看，边界桶很差
动作映射 分数和阈值 keep / drop / review 阈值一改，留存和偏差一起变

阈值与留存

threshold

阈值就是把分数变成动作的分界线。更高的阈值会更严，留下来的样本更少，但也更容易把边缘知识一并删掉；更低的阈值会更宽松，保留更多样本，但也更可能把噪声放进训练预算。

$$ r_t = \frac{n_{\text{keep},t}}{n_{\text{total},t}},\qquad \Delta r = r_{t_2} - r_{t_1} $$

这里的 r_t 是第 t 个分桶的留存率，n_{\text{keep},t} 是保留下来的样本数，n_{\text{total},t} 是该分桶总数。看阈值时，不能只看整体留存率，还要看各分桶的 \Delta r 是否明显倾斜。

调阈值的直觉

把阈值想成保守程度：阈值上调，模型更难通过；阈值下调，模型更容易通过。真正要比较的不是“哪个数更大”，而是每个数背后损失了哪些内容。

人工复核放在哪里

边界分数段最值得复核。它们最容易出现模型不确定、标注歧义和规则冲突，也最能帮助你改进阈值而不是单纯改模型。

偏差审计

audit

偏差审计的核心问题很直接：删掉了什么，删得有没有分布性，剩下来的内容是否还代表原始语料。最有用的审计不是一个总删除率，而是一组分桶、来源和上下文的删除率。

审计维度 应该看什么 高风险信号 下一步
来源分桶 域名、站点类型、语言 某些域名被系统性删得更多 回看规则是否偏向特定站点结构
内容分桶 主题、长度、格式、符号密度 代码、数学或对话被连带删掉 把规则拆成更细的条件
上下文分桶 被删样本前后文 命中词语义正常，但上下文被误判 增加语境判断或改用复核
保留动作 keep / drop / review 比例 复核队列长期挤满边缘样本 调整阈值或重写规则

为什么总删除率不够

总删除率只能告诉你删了多少，不能告诉你删的是谁。一个看起来“更干净”的语料，可能只是把某类内容、某些来源或某些写作风格系统性挤掉了。

案例推演

cases

案例 1：短页面为什么先走规则

一个只有标题和导航的页面，通常不需要分类器先判断教育性。长度、模板痕迹和 blocklist 就足够说明它更像噪声，先走规则更快也更省算力。

案例 2：教育性分类器为什么要看阈值

同一批样本里，阈值越高，保留下来的往往越像“课堂式文本”；但如果阈值太高，代码说明、例子、反例和边缘领域文档也会跟着掉出去。

案例 3：blocklist 命中为什么要看上下文

词命中不等于内容有害。句子可能在解释、引用、比较或教学。只要上下文还给了它正常语义，直接删除就会把真正的教学语境一起删掉。

这一页只讲过滤决策本身。去重、去污染和泄漏控制会在下一单元继续处理“同一批文本为什么不能重复进入训练预算”这个问题。

案例回查

audit

这一段把前面的机制放到一个完整回查案例里。假设一批教育性网页里，有一类段落总被删得更多，原因不是分类器本身，而是规则先碰到了导航词、页脚词和模板结构。你要做的不是立刻调大阈值，而是把命中的信号、动作和后果拆开看。这个案例的价值在于，它能把“结果”重新翻译成“机制”，让你知道删掉的不是一个抽象的低质文本，而是一条有具体来源、具体上下文和具体触发条件的样本。

案例设定

样本来自公开网页抽样。它们都带有稳定正文，但其中一部分页面的页眉、目录和推荐阅读比正文更显眼。规则层把这些块当成噪声删了以后，留下的正文数量看起来少了，教育性分类器的平均分却反而更高。这个变化很容易被误读成“更干净了”，其实它可能只是把结构更复杂、上下文更多的内容先删掉了。

第一轮判断

这里最容易犯的错，是看到分数提高就认为过滤更好。实际上，分数变高可能只是因为最复杂的长尾内容被先删掉了，留下来的只剩更像课堂文本的样本。这个时候你要回看的是命中的规则和被删样本的上下文，而不是先改训练目标。你现在应该能做的，是把这一类样本的来源、段落位置和命中信号记到同一张表里，再决定下一步是复核、改规则还是重算阈值。

步骤 读取什么 写什么 错误时的症状
先看规则命中 blocklist、长度、字符占比 删除原因 把 blocklist 误当成内容质量结论
再看被删上下文 前后句、段落位置、标题关系 语义上下文 教学解释、引用或例子被误删
然后看分桶 来源、主题、格式、语言 删除分布 某类站点或内容被系统性削掉
最后再调阈值 分数段和留存率 keep / drop / review 阈值调整掩盖了真正的规则问题

回查时的错误顺序

不要先问模型是不是太小，也不要先问阈值是不是太高。先把命中的规则、上下文和分桶拉出来，确认到底是规则写得太粗、上下文没看见，还是阈值把边界样本推走了。只有这一步稳，后面的改动才有依据。

边界说明

boundary

这一页先不展开的内容

先不讲去重、近似重复、污染检测、训练泄漏、分词器版本和混合配方；这些都需要下一单元和后面的数据配方单元接着处理。你现在只需要把过滤层的读写对象看准。

这页的判断标准

如果一个样本被删，你能说出是哪个信号、哪条规则、哪个阈值或哪个分类器分数触发的，就说明这页学到位了。说不出来，就还没从“结果”退回到“机制”。

操作桥接

bridge

这一段把前面的判断链翻成一次可执行的人工回查。假设你拿到一批样本日志：每条日志都列出 source、命中的规则、分类器分数、阈值和最终动作。你发现某个来源的保留率比别的来源低很多，但这不代表它一定更差。真正要做的是把同一批日志放在一起，看看它们到底是被规则删掉、被分数压掉，还是因为阈值太严而进了复核队列。

样本日志长什么样

好的日志至少要能回答四个问题：这个样本来自哪里，命中了哪条规则，分类器给了多少分，最后为什么被留、被删或被复核。只要这四个问题里有一个答不上来，后面的偏差审计就只能靠猜。

失败后果

如果日志里没有上下文，你看到的就只有一个删除结果；如果没有分桶，你看不到谁被删得更多；如果没有阈值版本，你也不知道今天的留存率和上周是不是同一个策略。这样改规则就很容易把原来的偏差藏起来。

日志字段 它帮你确认什么 缺了会怎样 下一步查什么
source 来源域名、站点类型、语言 看不出是不是某类来源被系统性删除 来源分桶删除率
matched signal 命中的长度、blocklist 或字符比例 不知道到底是哪条规则触发 规则本身是否写得太粗
classifier score 语义质量在标注定义下的大致位置 不知道边界样本为什么进了复核 边界分数段抽样

从日志回到动作

看到这张表，你应该能立刻做三件事：先判断删样本是规则删的还是分数压下去的，再看留存是否在某个来源上明显倾斜，最后再决定是改规则、改阈值，还是把边界样本送回人工复核。这个顺序比先调模型参数更稳，因为它先处理的是证据，而不是猜测。做完这三步以后，你才能把一次过滤故障真正收束成一个可修正的问题，而不是停留在“感觉不对”的层面。

一个样本怎么回查

样本 A 来自一个教程页，正文里出现了“登录”这个词，但它是在解释权限边界。规则层只看到这个词，就把整段打成了删除候选；分类器层并没有单独否定它，因为这段内容在语义上是清楚的。这个时候，真正需要修改的不是训练目标，而是规则解释范围和上下文读取方式。

把前后两句放回来以后，你会发现它是在讲“为什么不能把登录入口直接塞进正文”。这说明命中的不是有害内容，而是一个在教学里合法出现的词。若继续硬删，训练集会慢慢偏向更短、更模板化、上下文更少的文本，导致后面模型更难学到解释型内容。

如果你把这条样本继续往下追，还会看到一个常见连锁：同样的站点结构在其他页面上也会反复触发同一条规则，于是来源分桶会一起下滑。那时你需要的不是把删除率拉高，而是把规则限定得更具体，让它只打真正的模板噪声，而不是把教学正文一起卷进去。

练习钩子

  • 从删除桶里抽 3 条样本，把 source、matched_signal、context、score、threshold、action 写进同一张小表。
  • 如果某条样本只有词命中、没有语境命中，就把它标成 review，再补上下文窗口。
  • 如果某个来源的保留率低得异常，就同时看来源分桶和内容分桶，不要只看总删除率。
  • 如果同一规则在多个页面上重复触发，就把它升级成一轮规则修订，而不是先去动模型分数。
回查项 要写什么 缺了会怎样 下一步动作
source 来源、站点类型、语言 看不到是否存在来源偏差 做来源分桶删除率
matched_signal 命中的长度、词或格式规则 不知道为什么触发删除 检查规则写得是否太粗
context 前后句、段落位置、标题关系 会把教学解释当成噪声 加上下文窗口或改成 review

阈值扫例：同一批样本换三次阈值

如果你只看一次过滤结果，很容易把偏差误判成整体质量变化。更稳妥的做法，是把同一批样本固定住，只改阈值，然后比较整体留存和重点分桶留存。这样做的好处是，所有变化都来自阈值，不来自输入漂移。下面这组数字是一个很典型的形态：阈值越高，整体留存越低，但教育性来源的下降更快，说明被挤掉的是边界内容而不是纯噪声。

阈值 整体留存 教育性来源留存 解释
0.60 0.84 0.79 阈值宽，复核压力小
0.70 0.76 0.61 边界解释段开始被压掉
0.80 0.67 0.42 保守过头，长尾内容损失明显

for threshold in [0.60, 0.70, 0.80]:
    keep_rate = retained / total
    edu_rate = retained_edu / total_edu
    log(threshold, keep_rate, edu_rate)
    if edu_rate drops faster than keep_rate:
        route = "inspect boundary samples"

如果这一步已经看出教育性来源掉得更快，就不要继续把阈值往上推了。先把边界样本抽出来复核，再回看命中的规则是否过宽，最后才是考虑模型重训。这样你会得到一条稳定的修正路径：先保住该留的，再收紧真正的噪声。

复核时要把同一来源、同一时间段、同一类段落放在一起看，避免把来源差异误当成阈值差异。只要这一步做对了，你就能把“保留率变化”还原成一个能继续修正的工程问题，也能把复核样本重新分回来源桶和内容桶。这也方便你把同类样本按来源再切一刀，避免混桶。

失败后果也很具体：阈值太高会让复核队列被边界样本塞满，人工时间花得更多，却不一定留下更好的内容；阈值太低会让噪声混进训练预算，去重和配方阶段还得再付一次成本。你现在应该能做的，不是盯着一个总留存率下判断，而是拿一张分桶表，把这三档阈值复扫一遍，再决定改阈值还是改规则。
