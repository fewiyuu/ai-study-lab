数据配方：来源比例与阶段

这一页接在过滤、去重和泄漏控制后面，回答的是另一个更难的问题：当你已经有一批“能用”的数据时，应该怎么分来源、怎么定比例、什么时候换配方。这里先不追求万能答案，只追求一份能解释、能计算、能验证的数据配方。

评测反馈

先记住这一句

库存多，不等于训练多；训练多，也不等于一定更好。真正要看的，是某个来源在当前阶段为目标能力提供了多少有效 token，以及它把别的能力挤掉了多少。

术语与误区

terms
术语直觉精确定义常见误解
来源池一个可单独抽样的数据来源。如网页、代码、论文、百科、论坛、合成文本，每个池都记录规模、许可、过滤和版本。只记网站名，不记清洗和分桶规则。
采样权重抽得更频繁还是更少。对训练采样器的偏好设置，可能高于或低于库存占比。把权重当成来源本身的“质量事实”。
训练占比训练时真正消耗的 token 比例。模型实际看到的 token 分布，受库存、权重和阶段系数共同影响。把硬盘里有多少数据直接当成训练占比。
阶段系数某个阶段是否更偏向某类来源。按训练阶段对来源施加的额外权重，如主体、增强、退火。以为全程一套配方最稳。实际常常要随阶段调整。
消融看改动到底带来什么变化。在固定模型、预算和评测下，只改变来源、比例或阶段，观察结果。把最终大模型分数直接归因给某个小改动。

来源与能力

source

配方先从“来源能教什么”开始，而不是从“这个来源是不是干净”开始。干净只是门槛，能力贡献才是排位的依据。一个来源池如果只提供自然语言流畅度，那它就不该占满训练预算；如果它能带来代码、数学、结构化论证或稳定事实，那它就值得在某些阶段被提高权重。

来源表至少要写清什么

每个来源池都要能回到同一张记录：来源名称、采集时间、许可边界、语言、文档数、token 数、过滤规则、去重版本、质量分数、污染检查结果、采样权重。缺了这些字段，后面就很难解释模型为什么变了。

比例与权重

weights

比例先别写成库存表

库存占比只说明某个池在硬盘上有多少 token。训练占比才说明模型到底从这个池里看了多少 token。小而重要的池可以过采样，大而低效的池可以降采样。只要训练目标不同，比例就不该完全跟库存走。

一个常见错误是把“网页 80%、代码 10%、论文 10%”当成完整答案，却没有说这些比例按文档、样本还是 token 算，也没有说能不能重复抽样。这样的表格看起来整齐，实际上不能执行。

这里的 $n_i$ 是库存 token，$w_i$ 是来源权重，$s_i$ 是阶段系数。这个式子不是唯一方案，但它把“有多少”“想抽多少”“当前阶段要不要强化”三件事拆开了。

把目标写成配方

recipe

一份能执行的数据配方，通常一份能被平台读懂的配置。你至少要写清楚来源池、权重、阶段、以及每一段的回退条件。下面这个简化配置只演示结构，不声称它对所有模型都最优。

阶段与节奏

stages

主体预训练

这一段的目标是广覆盖和稳定学习。通用网页通常是主体，但还要混入代码、百科、书籍和论文，避免模型只学到一种语气或一种结构。

退火与收尾

训练后段常会同时降低学习率和噪声来源，把注意力放回更干净、更目标相关的数据上。这里减少最后一段对表示和输出风格的扰动。

配方追踪

trace

配方最容易出错的地方，写清楚了比例，却没写清楚这份比例在不同阶段会怎么变。下面这张表把“读什么、写什么、失效时看什么症状”放在一起，方便你回查。

从一份配方回到执行层

如果你在训练平台上维护配方，最少要能回答这四件事：哪个来源池被读到了、该池的权重是多少、这一阶段是否改过系数、结果变坏时应该回看哪张表。这个顺序比记住一个漂亮比例更重要。

完整走一遍

walkthrough

把前面的概念放回一个具体任务里，最容易看出它们到底是不是一回事。下面这份目标是“做一个技术文档理解模型”，总预算 100B token。它不需要极强的代码生成，但需要稳定的术语理解、结构化表达和一定的问题解决能力。这个目标很普通，所以最适合拿来练配方：它逼你同时考虑广覆盖、专业材料和阶段切换，而不是只盯着某一种来源。

第二步：把角色写成权重

假设你得到一个初版：网页 0.7、代码 2.0、论文教材 1.6、百科 1.0、论坛 0.8。这个版本的意思“在当前目标下，代码对能力贡献更高”。再加上阶段系数，网页在早段可以是 1.0，到了后段可能降到 0.5；论文教材在后段可能上升到 1.5，因为你要把前面学到的通用表达收束到更稳定的知识结构里。

这一层最容易犯的错，是把权重写成道德判断，好像“代码就是高级，网页就是低级”。不是这样。权重是对训练资源的分配决策，不是对数据身价的评语。它只对当前目标和当前阶段负责。

阶段来源怎么排为什么这样排错了以后最先坏什么
早段网页为主，少量代码和百科先学语言基本盘和事实分布，避免一开始就偏科模型会更快地学到某种风格，但通用覆盖不稳
中段提高代码、论文教材和领域文档开始塑形，补结构化表达和专业知识如果切得太猛，通用网页被压掉，很多日常能力会掉
后段更干净的高质量池 + 适量目标领域收尾时减少噪声，把输出风格压稳如果后段还沿用早段配方，模型会在噪声里收尾

第三步：识别失败症状，而不是只看均分

当你把网页压得太低，模型可能会显得更“专业”，但事实题、常识题和长尾表达会掉；当你把论坛问答放得太后，聊天会更顺，但事实保持和引用稳定性会掉；当你把代码过采样得太多，符号推理会变好，但自然语言风格会开始发硬。每一种退化都可以在任务簇里被具体看见的症状。

这就是为什么本页一直强调“验证方式”要跟配方绑在一起。你不是只在做一个表格，你是在做一个会改变模型行为的决策系统。没有反馈的配方，只能算初稿。

如果你现在要把这套方法迁移到新的模型，先别急着追最优。先复写一份这样的走读表：目标是什么、来源池有哪些、每段权重怎么变、最可能坏掉的是哪一簇任务。写完以后，再去看下一页的小模型消融。那一页会告诉你，今天这份配方到底是把能力做出来了，还是只是把某类分数抬高了。

案例推演

cases

案例 1：代码库存小，但目标是代码能力

如果代码库存只有 8%，训练占比却想给到 16%，这并不矛盾。矛盾的是你是否知道这 16% 会从哪里挤出来，以及重复暴露是否会伤到通用表达。做法通常是先保留通用网页，再把高质量代码池在中后段适度过采样，同时监控自然语言和数学任务是否被挤压。

案例 2：论坛问答放到后 10%，聊天更顺但事实题掉了

这时不要急着把论坛删掉。先看它的质量分布、是否和 benchmark 近似、是否在收尾阶段被放大过头，再看事实题下降是不是因为通用网页被压缩。问题可能出在阶段位置，也可能出在比例，而不是出在来源本身。

案例 3：法律领域继续预训练

如果目标改成法律领域，来源就不能只看通用网页了。要增加法规、判例、注释性材料和专业教材，同时把许可、时效、地区差异和隐私边界写进配方记录。这里的风险“来源对了但版本不对”。

边界说明

boundary

这里先不追的事

这页先不展开最优混合搜索、长期课程学习、动态重加权、最优控制和多阶段自动调度。它们会在更后面的实验设计和系统化流水线里回来。现在先把“来源-权重-阶段-验证”四件事写顺。

这页不承诺的事

这里不会给出一个放之四海而皆准的比例表。不同模型、预算、语言、任务和发布时间点，都会把配方改写。你要学的是怎么解释和验证配方，不是死记一组百分比。

下一页会把这份配方拿去做小模型消融和统一评测。到那时，才真正知道某个来源池是贡献能力，还是只是填满了 token。

怎么接到下一页

operation

这一页的任务，把它写到可以送去做实验。下一页你会看到的，更短的决策：哪些来源池进入了对照组，哪些阶段权重被固定，哪些变量被单独改动。换句话说，05 页负责把目标翻成配方，06 页负责把配方翻成对照实验。

常见第一失败

最常见的失败“你说不清哪一项导致了变化”。如果你没有先把来源、权重和阶段拆开，下一页的消融就会变成一次大杂烩训练，测到什么都像是对的，却什么都解释不清。

这页留下的作业

把你自己的配方写成三行：来源池、阶段系数、验证指标。三行都写完，再去看下一页的消融设计。只要能把这三行写清楚，你就已经比“凭感觉调比例”多走了一步。

如果阶段系数写错，会先坏哪里

最先坏掉的通常任务簇的方向：代码可能变强一点，但事实题和通用表达开始下滑；或者聊天更顺，但结构化理解不稳。这个时候别先改模型结构，先回到配方表，看是不是中后段把某个来源放得太重了。

给平台看的最短记录

如果你要把这页内容抄进训练平台，最少写四行就够：来源池名称、基础权重、阶段权重、验证指标。下面这段可以直接当作起点，再根据你的任务目标改数值。

四行记录模板复制

source_pools: web / code / papers_books / forums
base_weights: 0.7 / 2.0 / 1.6 / 0.8
stage_weights: early=1.0/0.8/0.6/0.7, middle=0.8/1.2/1.2/0.9, late=0.5/1.0/1.5/0.6
validation: task_cluster_score + contamination_check + language_regression

练习

diagnostic

一、概念判断

快速回看 库存是池里有多少，权重是想抽多少，占比是训练里真消耗多少。 -->
1choice

Q: Q1. 混合比例最准确的单位是什么？

参考答案 / 评分要点

训练中实际消耗的 token 占比。训练占比看的是训练时实际消耗的 token，不是库存文件数。

2true/false

Q: Q2. 小而高质量的数据池一定不能过采样。

参考答案 / 评分要点

false。过采样是常见手段，但要监控重复暴露和通用能力下滑。

3fill

Q: Q3. 填空：配方闭环的最后一步是用什么反馈修正配方？

参考答案 / 评分要点

评测;消融;污染检查。最后一步要看评测、消融或污染检查后的反馈，不能只盯着表格本身。

4true/false

Q: Q4. 质量权重等于数据本身的客观质量。

参考答案 / 评分要点

false。权重是采样决策，不是数据本身的客观事实。

5choice

Q: Q5. 哪个字段最能帮助复现一个数据池？

参考答案 / 评分要点

采集时间、过滤规则、去重版本和 token 数。复现要靠来源、时间、规则和规模，而不是只有一个好听的名字。

6short-fill

Q: Q6. 写出一个训练阶段名称和它的主要目标。

参考答案 / 评分要点

主体预训练;质量增强;退火;收尾。答案要同时写出阶段名和主要目标，例如“主体预训练：广覆盖和稳定学习”。

二、配方计算与解释

快速回看 比例要能解释，也要能被评测验证。 -->
7choice

Q: Q7. 计算：网页 900B 权重 0.5，代码 100B 权重 2.0，代码训练占比约是多少？

参考答案 / 评分要点

约 31%。加权后网页 450，代码 200，所以代码占比约 31%。

8synthesis

Q: Q8. 解释：为什么代码库存只有 8%，训练占比可以设到 16%？

参考答案 / 评分要点

。要点应包含：代码库存小不等于不能提高训练占比；过采样是为了在固定预算下提升代码能力；但要监控重复暴露、风格单一和通用能力下滑。回答里如果只写“代码更重要”，还不够。

9error-diagnosis

Q: Q9. 诊断：某团队说“我们把网页降到 40%，模型就一定更聪明”。这句话有什么问题？

参考答案 / 评分要点

。至少要指出三件事：没有说明“更聪明”对应什么评测；没有说明被网页让出的 token 去了哪里；没有控制其他变量；没有说明是否分阶段；没有考虑语言、领域和污染风险。只批评“太绝对”不算完整。

10scenario-transfer

Q: Q10. 设计：给一个技术文档理解模型分配 100B token 的三类来源比例。

参考答案 / 评分要点

。回答里要有三类来源、比例之和、以及为什么不能 100% 都给专业材料。最好能说明通用网页负责广覆盖，教材或论文负责结构化表达，代码或问答负责工具和问题解决能力。只写一串百分比不够。

11comparison

Q: Q11. 比较：质量权重和阶段系数有什么不同？

参考答案 / 评分要点

。质量权重更像全局偏好，说明某个来源池在整体训练里该被抽得多还是少；阶段系数则说明当前阶段是否要额外放大或缩小这个池。一个是“这池重要不重要”，一个是“现在这个阶段要不要更强调它”。

12synthesis

Q: Q12. 解释：为什么平均评测分不足以判断配方好坏？

参考答案 / 评分要点

。平均分会掩盖方向性变化。一个配方可能让代码和推理更强，却牺牲通用语言、某些语种或事实题；也可能是 benchmark 污染导致分数虚高。好答案要说明“平均值背后有哪些分组差异”。

三、阶段化方案

快速回看 阶段 = token 预算 + 目标 + 来源比例 + 监控指标。 -->
13scenario-transfer

Q: Q13. 设计一个 3 阶段配方。

参考答案 / 评分要点

。一个合格答案至少要写出：早段以广覆盖和低重复为主；中段提高代码、教材或论文来塑形；后段收尾时放大更干净、更目标相关的来源。还要说明每段要看什么指标来决定是否继续或回退。

14error-diagnosis

Q: Q14. 诊断：后 10% token 加入大量论坛问答，模型聊天更自然但事实题下降。你会怎么查？

参考答案 / 评分要点

。先查论坛问答本身是否质量稳定、是否和 benchmark 近似、是否在后段被放大过头；再查是否因为后段挤掉了高质量网页或百科；最后用对照实验拆成“来源问题、阶段问题、比例问题”三条线分别验证。

15scenario-transfer

Q: Q15. 迁移：如果目标是法律领域继续预训练，来源和风险会如何变化？

参考答案 / 评分要点

。应包含法律法规、判例、注释性材料、专业教材或合同文本等来源，同时说明地区差异、版本时效、隐私和许可边界。最好还能提到：领域数据要提高，但通用覆盖不能完全消失。

16synthesis

Q: Q16. 设计一个小模型消融实验来验证“论文教材占比提高有用”。

参考答案 / 评分要点

。要固定模型、token 总量、训练设置和评测协议，只改论文教材的占比或放置阶段；要有对照组；要选能体现结构化理解和事实保持的评测；结论不能只看单项，而要看任务簇和退化风险。

17debug-checklist

Q: Q17. 写一段审查意见：下面配方缺什么？

参考答案 / 评分要点

。这条审查意见至少要指出：比例单位不明；没有来源版本；没有阶段信息；没有质量或去重规则；没有污染和泄漏检查；没有评测和回退条件。它更像一句口号，不像一份配方。

Q: Q18. 用自己的话讲完整闭环。

参考答案 / 评分要点

。回答里要出现“来源”“比例”“质量权重”“阶段”“评估反馈”五个词，但更重要的是顺序：先盘来源，再定权重，再分阶段，再做评测回看，最后根据结果回调配方。
