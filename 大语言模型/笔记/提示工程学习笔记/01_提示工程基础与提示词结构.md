> [!question] 提示工程到底是什么？
> 提示工程是围绕 prompt 进行设计、优化和测试的方法。它的目标不是寻找一句万能话术，而是把任务、背景、材料和输出格式表达清楚，让模型更稳定地完成具体工作。

> [!question] prompt 是不是越长越好？
> 不是。好 prompt 的关键是相关、清楚、可检查。无关上下文会增加噪声，也会占用上下文窗口。

> [!question] 为什么同一个任务换种说法结果会变？
> 模型根据输入文本生成输出。指令、背景、示例、材料边界和输出格式都会改变模型的生成方向。

> [!abstract] 核心主线
> prompt 可以拆成 instruction、context、input data、output indicator。先写清任务动作，再补必要背景，用边界放入材料，最后规定输出形状。简单任务保持简单；复杂、长期复用或需要固定角色的任务，再使用 CRISPE 或结构化提示词。

## 概念精讲

### prompt

prompt 是交给模型的完整输入，不只是问题本身。它可以包含任务、背景、材料、示例、规则和输出格式。

### instruction

instruction 说明要模型做什么。好的 instruction 应该具体到动作和标准。

弱写法：总结一下。

强写法：用 5 个要点总结，每点不超过 30 字，保留关键数字。

### context

context 提供任务背景，让模型知道面向谁、为什么做、有什么限制。上下文要相关，不是越多越好。

### input data

input data 是要被处理的材料。长文本应使用分隔符包裹，避免和指令混淆。

### output indicator

output indicator 规定输出形式。它可以是 Markdown、JSON、表格、固定字段或长度限制。

### role prompting

role prompting 通过设定身份影响模型回答方式。角色应服务任务，不能替代具体任务要求。

### CRISPE

CRISPE 可用于组织复杂 prompt：

| 缩写 | 含义 | 用途 |
|---|---|---|
| CR | Capacity and Role | 能力与角色 |
| I | Insight | 背景与洞察 |
| S | Statement | 任务陈述 |
| P | Personality | 表达风格 |
| E | Experiment | 多方案实验 |

## 最小可操作样例

```text
任务：
请你{执行动作}。

背景：
{读者 / 使用场景 / 目标 / 限制}

输入材料：
"""
{input}
"""

输出要求：
1. {格式要求}
2. {长度或字段要求}
3. {边界规则}
4. {资料不足时的处理方式}
```

## 应用与迁移问答

### 做信息抽取时怎么迁移？

重点写清字段、格式、缺失值处理和只基于原文。需要程序解析时，要求只输出 JSON。

### 做知识库问答时怎么迁移？

重点写清资料范围，要求资料不足时说明“资料中未提及”，避免编造。

### 做客服机器人时怎么迁移？

重点写清角色、处理流程、安全边界、语气和升级条件。

### 做学习助手时怎么迁移？

重点写清学习者水平、讲解范围、练习题型和反馈方式。

> [!quote] 费曼解释润色版
> prompt 就是交给模型的一份任务说明。它不只是一个问题，还可以包括背景、材料和输出格式。结构化 prompt 的作用，是让模型少猜一点：先知道要做什么，再知道为什么做、处理哪段材料、结果要长成什么样。坏 prompt 常见问题是指令模糊、材料边界不清、只说不要做什么、没有输出格式。改 prompt 时，不需要一上来写很长，先把任务动作写准，再补必要背景和格式，用真实输出测试是否稳定。

> [!warning] 易错卡片：把 prompt 当成一句问题
> 误区：忽略背景、输入材料和输出格式。
> 正确说法：prompt 是完整任务输入。
> 提醒：复杂任务至少检查任务、背景、材料、格式四项。

> [!warning] 易错卡片：指令模糊
> 误区：使用“优化”“分析”“详细点”等不可检查表达。
> 正确说法：写清动作、对象、标准和限制。
> 提醒：形容词要尽量换成数量、结构或验收标准。

> [!warning] 易错卡片：上下文过多
> 误区：把所有资料都塞给模型。
> 正确说法：只保留与任务相关的背景，并说明优先级。
> 提醒：无关上下文也是噪声。

> [!warning] 易错卡片：输出格式缺失
> 误区：没有字段、模板或示例。
> 正确说法：提供固定格式和缺失值规则。
> 提醒：要给程序解析时，尤其要限制 JSON 外解释。

> [!warning] 易错卡片：角色提示空泛
> 误区：只写“你是专家”。
> 正确说法：补充角色能力、任务目标、输出标准和工作流程。
> 提醒：角色提示不能替代任务指令。
