机器学习基本概念简介

这一节先解决一个最基础的问题：拿到一个新任务时，先怎么把它翻译成机器学习语言。你会先判断输入和输出，再决定这是 regression、classification 还是 structured learning；接着把模型、参数、loss、optimization 和泛化放回同一条流程里。

如果前面这些词还没连起来，后面看深度学习、CNN、Transformer、GAN 都会散。P01 的作用就是把整门课的第一套坐标系搭好。

读完这一页要会做什么

  • 把一个任务先写成“输入是什么、输出是什么、函数要找什么”。
  • 分清 model、parameter、hyperparameter、loss、optimization。
  • 读懂最小线性回归代码，并说出哪一行在预测、哪一行在算 loss、哪一行在更新参数。
  • 知道为什么 training loss 低，不等于真正学会。

学习目标

goal

这一节不追复杂模型，先把最通用的判断顺序练熟。机器学习不是先背模型名，而是先问：输入是什么，输出是什么，候选函数长什么样，哪些值要从资料里学出来，怎样判断一组参数好不好，怎样把它调得更好。

先会翻译

看到“语音辨识”“垃圾邮件”“房价预测”这类任务，先把它翻成输入 / 输出 / 函数，而不是先找模型名。

先会分工

知道 model、parameter、hyperparameter、loss、optimization 各自负责什么，不把它们混成一个词。

先会停一下

training loss 低，只说明模型在见过的资料上做得不错。它还没回答泛化，也没回答是否真的可用。

先抓主线

route

这一页的主线很短：先把任务翻译成函数问题，再把函数拆成模型和参数，接着用 loss 给出评分方式，最后用 optimization 去找更好的参数。训练结束以后，还要回到未见资料上检查。前四步是“怎么学”，最后一步是“学得对不对”。

任务描述→ 输入 / 输出→ 函数 / model→ loss→ optimization→ 泛化

直觉

人不一定写得出复杂规则，所以先把规则写成一个可调函数，再让资料去调参数。

正式说法

学习算法在预设的模型集合中寻找参数，使训练资料上的损失函数尽量小。

边界

这一页先不追深度学习结构、卷积、注意力、batch norm。先把第一层语言说稳，再往后走。

现在先记住一句话

机器学习的核心，不是“让机器自己想办法”，而是“人在设定函数形式，机器在资料里找参数”。

任务先分类

output type

最先看的不是模型，而是输出长什么样。输出如果是连续数值，常常是 regression；输出如果是固定类别，常常是 classification；输出如果带长度、顺序或内部结构，就进入 structured learning。这个分类会直接影响你后面怎么选 loss 和评价指标。

Regression

输出是数值。房价、点阅数、温度、风险分数都在这类里。

Classification

输出是类别。垃圾邮件、影像辨识、情感分类都在这类里。

Structured learning

输出有内部结构。语音转文字、机器翻译、路线规划都不只是选一个标签。

Regression 房价 / 点阅数 / 温度

输出是一个数。先看数值型误差，再看任务本身的单位和量纲。

Classification 垃圾 / 正常

输出是固定选项中的一个。这里更关心选对没，而不是差了多少。

Structured 文字序列

输出不是一个标签，而是一串互相影响的内容，顺序不能乱。

Decision 棋盘上的一个动作

同一任务可能有不同建模方式，但先要说清楚你定义的输出是什么。

先别混的点

“输出是类别”不等于“模型一定很复杂”；“输出是数值”也不等于“回归一定简单”。先分任务类型，是为了选对后面的语言和指标。

完整例子：垃圾邮件分类

worked example

把一封邮件从头走到尾最能看清这门课的骨架。输入可以是邮件正文、发件人地址、是否带链接、是否有异常附件；输出是“垃圾”或“正常”。因为最后只要选一个类别，所以它通常是 classification，而不是 regression。

这件事看起来很小，但它其实把 P01 的所有概念都串起来了：先看输入和输出，再选 model 的形状，再定 loss，最后用 optimization 调参数。换句话说，这不是四个独立名词，而是一条连续链路。

步骤这一步在看什么机器学习里怎么说常见误区
读任务描述邮件里有什么信号，最后要判断什么先定输入 / 输出，再定任务类型先背模型名，结果把任务分类看错
选 model允许机器用什么特征组合去判断模型是带未知参数的函数把 model 当成训练完的答案，而不是搜索空间
定 loss错判一封邮件有多严重把“好不好”写成可优化数字把 loss 和准确率混成一件事
做 optimization参数怎么往更好的方向移动根据梯度和 learning rate 更新参数只看公式，不看更新方向和步幅
看泛化新邮件里还能不能判断对在未见资料上检查表现只看训练 loss 就宣布模型可上线

如果这一步看错了，会怎样

如果把垃圾邮件当回归问题，输出会变成一个没有意义的连续分数；如果只看训练资料上的 loss，模型可能把某些邮件模板背下来，却分不出新垃圾邮件。

如果这一步看对了，会怎样

你会先问输入是什么、输出是什么、评分规则是什么，再决定怎么训练。这样后面换到房价预测、语音转文字或围棋落子时，思路还能直接复用。

这一页的锚点

这封邮件就是本页的锚例子。后面每讲一个词，都可以回到它：它属于哪类任务，模型怎么写，loss 怎么算，参数怎么更新，最后怎么检查泛化。

现在你应该能做什么

你能拿一个新任务，先用完整一句话把它写成“输入 / 输出 / 任务类型 / loss / 泛化检查”的骨架，而不是急着找模型名。

模型和参数

model / parameter

model 可以先理解成“带未知参数的函数”。例如最小的房价预测模型可以写成 ŷ = b + w x。这里的 w 和 b 是未知的，训练要做的事，就是用资料把它们找出来。函数形式由人设定，具体数值由训练决定。

$$\hat{y}=b+w x$$

x 是输入特征，\hat{y} 是预测值，w 和 b 是要学出来的参数。这个模型很简单，但它把“预测”变成了可计算的问题。

model 负责什么

模型决定你允许机器在什么范围里找函数。范围太小，学不动；范围太大，容易把训练资料记得太细。

parameter 负责什么

参数是训练会改的值。这里的 w 和 b 不是手写答案，而是优化的对象。

hyperparameter 负责什么

hyperparameter 是人先设的设置，例如 learning rate、训练轮数、模型宽度。它们不会直接从资料里学出来。

最小代码：从模型形式到参数python
x = 3.0
w = 2.0
b = 1.0
y_hat = b + w * x

这四行只是在做预测，还没有训练。真正的训练要继续往下走：先定义 loss，再更新 w 和 b。

现在你应该能做什么

看到“模型”这个词时，能把它还原成“函数形式 + 待学习参数”，而不是把它当成已经训练好的成品。

Loss 怎么读

loss

只有模型还不够。机器还需要知道某一组参数好不好，这就要靠 loss。Loss 是人定义的评分方式：给定一组参数，先让模型做预测，再把预测和真实答案之间的差距汇总成一个数字。训练时希望这个数字越小越好。

Loss 不是准确率的同义词。准确率更像评价结果；loss 更像训练时用来推动参数更新的工具。不同任务会用不同 loss：回归常见平方误差或绝对误差，分类常见交叉熵，生成任务还会有更复杂的目标。

$$L=\frac{1}{N}\sum_{i=1}^{N} |\hat{y}_i-y_i|$$

这里先用平均绝对误差做玩具例子。L 越大，说明预测和真实答案差得越远；训练就会继续找更好的参数。

loss 的角色

把“好不好”变成数字。没有这个数字，优化器就不知道往哪里走。

为什么先定义它

因为训练不是凭感觉改参数，而是按一个明确目标去减小误差。

先别误会

loss 可以是训练目标，但不一定是最终业务指标。两者经常相关，但不必强行等号。

现在你应该能做什么

你能解释为什么训练前要先定义 loss，也能说出 loss 和准确率不是一回事。

Optimization

gradient descent

定义好模型和 loss 之后，下一步是 optimization，也就是找一组参数让 loss 变小。最经典的直觉是 gradient descent：先算当前点附近往哪边走会让 loss 下降，再朝那个方向走一步。重复很多次，参数就可能走到较低的 loss 区域。

Learning rate 控制每一步走多大。步子太小，训练会很慢；步子太大，可能越过低点，在 loss surface 上来回震荡，甚至越走越差。这里先把方向和步幅分开理解，不要急着卡在微分符号上。

$$\theta \leftarrow \theta - \eta \nabla_\theta L$$

\theta 表示参数，\eta 是 learning rate，\nabla_\theta L 是梯度。减号表示沿着让 loss 下降的方向走。

gradient 负责什么

告诉你当前参数该往哪边改，才更可能让 loss 下降。

learning rate 负责什么

控制一步走多远。方向错了再怎么走都不稳；步幅太大也会失控。

常见错法

最小化 loss 时写成加号，往往会把参数往更差的方向推。

Before 当前参数 θ

这一步只有一个当前位置。训练还没有开始改变参数。

Read loss 和 gradient

这里会读当前预测和真实答案，算出误差和梯度。

After 更新后的 θ

参数被写回。下一轮会在新的位置上再算一次 loss。

现在你应该能做什么

你能用一句话说明 gradient descent：先算下降方向，再按步幅更新参数。

训练与泛化

generalization

这一页最后要补的，不是更复杂的模型，而是更重要的检查：训练资料上的 loss 低，不等于未来数据上也做得好。训练资料是模型已经看过的例子；未见资料才更接近真实使用场景。

如果模型在训练资料上很好、在新资料上很差，常见情况是过拟合。模型可能把训练时的细节和噪声也记住了。反过来，如果训练和测试都差，可能是模型太简单、特征不够，或者训练本身没跑好。

训练资料

用来更新参数。模型在这批资料上表现好，只说明它会适配见过的例子。

验证 / 测试资料

用来检查泛化。模型在没见过的数据上也表现好，才更接近我们想要的“学会”。

重点判断

评价模型时，要同时看训练表现和未见资料表现。只看训练 loss 很容易把记忆当成能力。

现象可能原因先检查什么
训练 loss 高，新资料也高模型太简单、特征不够，或者训练没跑好。模型容量、learning rate、资料质量。
训练 loss 低，新资料高可能过拟合，模型记住了训练细节。验证集、正则化、模型复杂度。
训练 loss 和新资料都低模型当前较可用。继续看业务指标和分布变化。

现在你应该能做什么

听到“训练 loss 很低”时，你会立刻追问：在没见过的数据上呢？

状态账本

read / write

训练代码最容易让初学者糊掉的地方，不是公式本身，而是状态流转。上一行读了什么，下一行写了什么，哪一个值已经过期，哪一个值要传给下一步，这些都得分清。你只要把这一轮训练当成一个“读入—计算—写回”的账本，就会少掉很多迷糊。

下面这个账本就是 P01 最核心的机械部分。它把 x、y、y_hat、error、loss、dw、w、b 放在一张表里看。读这一表的目的，不是记住每个名词，而是知道训练为什么必须按这个顺序走。

对象更新前是什么更新后是什么读什么写什么出错时的症状
x输入特征不变数据本身不写回如果 x 取错，后面所有计算都跟着歪。
y真实答案不变监督信号不写回如果 y 乱了，loss 再小也没意义。
y_hat还没有预测当前预测值w、b、x预测结果如果没有这一步，loss 没有输入。
error还没有误差预测和真实值的差y_hat、y差值如果把 error 方向写反，梯度方向可能错。
loss还没有评分一轮误差的汇总error训练评分如果 loss 和任务不匹配，训练目标会跑偏。
dw、db还没有梯度参数的更新方向error、x梯度值如果梯度没算出来，参数不会动。
w、b当前参数更新后的参数dw、db、lr新参数如果这一步拿掉，训练每轮都在原地打转。

这一表在帮你看什么

它在帮你分清读和写。y_hat 是算出来的，loss 是评分数字，w 和 b 是被更新的对象。把这些对象混起来，训练逻辑就会乱。

最常见的两种坏情况

一种是漏掉 w -= lr * dw，模型根本没学；另一种是把 y 和 y_hat 混了，误差变成假象，loss 看起来像成功，其实只是算错。

回到代码时怎么用

先问这一行读什么，再问它写什么。如果你能把这两件事说清楚，P01 的代码阅读题基本就不会飘。

现在你应该能做什么

你能看着一段训练代码，把每个变量放进“读 / 写 / 过期 / 传给下一步”的账本里，而不是只会逐行翻译成中文。

完整追一轮训练

trace

这部分把“输入 → 预测 → loss → 梯度 → 更新”完整走一遍。你不需要先会推导，只要能看懂这一轮状态怎么流动，就已经抓到训练的主线了。

如果你把训练看成一条流水线，就更容易判断哪里出了问题：有些错误发生在输入阶段，有些发生在预测和 loss 之间，有些则发生在梯度更新之后。P01 里最重要的练习，就是学会把这些环节拆开。

时刻发生了什么你要盯住的对象这一刻能不能跳过
输入进来拿到 x 和 y输入、真实答案不能。没有输入，就没有后续计算。
先预测用当前 w、b 算出 y_hatmodel 当前形状不能。loss 依赖预测值。
再算 loss比较 y_hat 和 y评分数字不能。没有 loss，就不知道好坏。
求梯度看 loss 对参数怎么变dw、db不能。没有梯度，就不知道往哪边改。
写回参数按 learning rate 更新 w、b参数本身不能。跳过这一步，训练不会发生。

把这轮追踪想成什么

可以把它想成一条流水线：输入先经过当前函数得到输出，输出再被拿去算误差，误差再反过来告诉参数怎么改。训练就是不断重复这条线。

最容易出错的地方

很多人会把“算出预测”当成“训练完成”。其实预测只是中间一步。没有 loss、梯度和参数更新，训练还没开始。

和上面的账本怎么配合

账本看单个对象的读写，追踪表看整轮顺序。两个一起用，才能同时看懂局部状态和整体流程。

现在你应该能做什么

给你一段训练代码时，你可以先把它拆成“输入、预测、loss、梯度、更新”五个节点，再去看每个变量怎么流动。

代码与状态

code reading

下面这段代码把 P01 的三步串起来：先预测，再算 loss，再更新参数。你读的时候不要只看语法，要看每一行在读什么、写什么、依赖什么、如果删掉会坏在哪里。

最小示例：训练一条直线python
import numpy as np

x = np.array([1.0, 2.0, 3.0, 4.0])
y = np.array([2.0, 4.1, 6.0, 8.2])
w = 0.5
b = 0.0
lr = 0.05

y_hat = w * x + b
error = y_hat - y
loss = np.mean(error ** 2)
dw = np.mean(2 * error * x)
db = np.mean(2 * error)
w -= lr * dw
b -= lr * db
代码行读什么写什么如果错了会怎样
y_hat = w * x + b当前参数和输入预测值后面就没有误差可算。
loss = np.mean(error ** 2)预测和真实答案评分数字训练不知道自己好不好。
w -= lr * dw梯度和学习率更新后的参数参数不会变，训练会停在原地。

这个表的重点不是公式，而是状态。训练代码每一步都在做读入、计算、写回；如果顺序错了，训练逻辑就断了。

现在你应该能做什么

读训练代码时，你能把每一行标成 model、loss 或 optimization，并指出它读了什么、写了什么。

出错时怎么查

diagnosis

这一节不是为了多给你一个标题，而是为了把 P01 变成能排错的工具。新手最常见的问题，不是看不懂单个术语，而是看不清问题到底卡在流程的哪一层：任务定义、模型形式、loss、更新，还是泛化检查。

下面这张表就是一个最短排查顺序。你以后看到训练结果不对，不要先怀疑所有东西，先定位症状，再顺着流程往上查。

症状更像哪层出问题先看什么为什么先看这个
输出类型和题目不搭任务定义输入 / 输出 / 任务分类任务类型错了，后面所有选择都会跟着偏。
模型能预测，但训练不动更新链路loss、梯度、参数更新可能算了预测，却没有真正改参数。
训练集很好，新资料很差泛化验证集、测试集、过拟合这是最常见的“记住了，不代表会了”。
loss 一直不降优化learning rate、梯度方向、数据规模步幅或方向出了问题，训练很难往下走。
模型太简单，怎么都学不进去模型形式模型容量、特征、函数假设空间太小，优化器也找不到合适答案。

先查哪一层

先从任务定义查，再看模型形式，然后看 loss 和优化，最后才看泛化。这个顺序能减少“一个 bug 改成十个 bug”的情况。

为什么这张表有用

因为它把症状和层级绑在一起了。你不是在背诊断口诀，而是在判断问题更像从哪一步开始偏离。

遇到卡住时怎么问

先问“输入 / 输出定对了吗”，再问“loss 对吗”，再问“更新真的发生了吗”，最后问“在没见过的数据上还行吗”。

现在你应该能做什么

你能根据一个训练症状，先把它归到任务、模型、loss、优化或泛化中的某一层，再决定往哪边继续查。

常见误区

pitfalls
误区错因后果修法
把机器学习理解成机器“自己想办法”忽略了人要定义模型、loss 和资料。不知道哪里该人工设计，哪里该训练学习。先说清输入、输出、函数形式，再看参数怎么学。
把 model 当成训练好的成品没区分模型形式和参数值。写得出公式，却不知道参数从哪来。先把 model 当成“带未知参数的函数”。
把低 training loss 当成学会没区分训练资料和未见资料。模型可能上线就掉线。始终检查验证集或测试集表现。
把参数和超参数混在一起都叫“参数”，没有看谁自动学习。调试时不知道该训练还是手动改设置。参数由优化器学；超参数由人设或搜索。
以为 loss 必须等于准确率把训练目标和评价指标混同。分类、回归、生成任务的解释会乱。loss 是训练用的可优化目标，指标是评价角度。
把“预测出来了”当成“已经学会了”只看到 y_hat，没看 loss、梯度和更新。把推理流程误当训练流程。检查是否真的发生了参数更新，训练轮次有没有继续推进。
把学习率当成越大越好以为一步跨得大就能更快到低点。可能跨过低点，训练变得震荡。先把方向和步幅分开看，再调 lr。

9. 解释这四行训练代码各自做什么。

y_hat = w * x + b
loss = mean((y_hat - y) ** 2)
dw = mean(2 * (y_hat - y) * x)
w -= lr * dw
先说哪一行在预测，哪一行在算误差，哪一行在更新参数。
参考答案 / 评分要点

y_hat 是当前预测；loss 把预测和真实值差距汇总成一个数字；dw 估计 w 的更新方向；w -= lr * dw 用梯度和学习率更新参数。少说“做运算”这种空话，要说清读写关系。

error diagnosisgeneralization

10. “只在训练资料上 loss 很低，就宣布模型可上线。”这句话哪里不稳？

要同时说训练表现和未见资料表现。
参考答案 / 评分要点

错因是把训练表现当成泛化能力；后果是模型可能记住训练细节却在新资料上失效；修法是看验证集或测试集，并检查是否过拟合。

error diagnosisoptimization

11. 最小化 loss 时，把更新写成 w = w + lr * grad 可能有什么问题？

参考答案 / 评分要点

如果目标是最小化 loss，通常要沿着梯度的反方向更新。写成加号，容易把参数往更差的方向推，让 loss 变大。

short answerlearning rate

12. 为什么 learning rate 太大可能训练失败？

参考答案 / 评分要点

步子太大时，参数可能越过低点，在 loss surface 上来回震荡，甚至越走越差。答案里要同时提到方向和步幅。

18. 用 6-9 句话讲清楚：机器学习到底是在学什么？
