机器学习基本概念简介

这节课的核心句子很短：机器学习就是让机器具备找函数的能力。语音辨识是在找“声音到文字”的函数，影像辨识是在找“图片到类别”的函数，预测频道明天点阅数是在找“历史数据到未来数值”的函数。

你暂时不用急着背一堆模型名。先把“输入、输出、函数、模型、参数、损失、优化、训练资料”这几个词连起来。后面讲深度学习、CNN、Transformer、GAN，都会回到这条主线。

学完你应该能做什么

  1. 用自己的话说明“机器学习是在找函数”。
  2. 判断一个任务是 regression、classification 还是 structured learning。
  3. 解释 model、parameter、loss、optimization 各自负责什么。
  4. 看懂一段最小线性模型训练代码，不把训练资料上的低 loss 误认为真正学会。
  5. 说清为什么训练完成后还要看没见过的数据。

本节的概念地图

map

机器学习课程里会出现很多术语。第一节不是为了把所有术语背完，而是让你以后看到任何模型时都能问同一组问题：输入是什么，输出是什么，候选函数长什么样，哪些部分要从资料中学出来，用什么指标判断好坏，怎样把指标变小，最后在新资料上是否还做得好。

这组问题有很强的迁移性。预测 PM2.5、判断邮件是不是垃圾邮件、根据棋盘决定下一步落子，表面上很不一样；放进机器学习语言里，它们都可以写成“找一个函数”。差别在于输出的类型不同，资料长相不同，损失函数和模型结构也会跟着变。

任务→ 输入 x→ 模型 fθ→ 输出 ŷ→ Loss→ 优化参数 θ

直觉

先承认人手写不出那个复杂规则，所以把规则写成一个可调函数，让资料来调参数。

正式说法

学习算法在预设的模型集合中寻找参数，使训练资料上的损失函数尽量小。

判断句

看到新任务时，先说清输入和输出，再谈模型；输入输出说不清，后面的训练通常也会乱。

机器学习是在找函数

function

“函数”在这里不是高中数学里只会写成一行公式的东西。更宽一点说，函数就是一套输入到输出的对应规则。给它一段声音，它输出文字；给它一张图片，它输出图片内容；给它棋盘状态，它输出下一步落子位置。只要你能把任务说成“输入什么，输出什么”，就已经把机器学习问题往正确方向推进了一步。

为什么不用人手写这个函数？因为真实任务里的规则太复杂。人可以写出“如果 PM2.5 高就提醒戴口罩”这种规则，但很难手写出“任意一段声音准确转成文字”的规则。语速、口音、噪声、停顿、语境都会影响结果。机器学习做的是：先给出一个可调函数形式，再用大量例子调整它。

李宏毅在课里用了很多例子：语音辨识、影像辨识、AlphaGo、YouTube 点阅预测。它们不是随便凑热闹，而是在训练你把表面不同的任务翻译成同一个骨架。这个骨架一旦立住，后面你就不会被“深度学习”“生成模型”“自监督”这些名字吓住。

现在你应该能做什么

拿到一个 AI 任务时，不急着问用什么模型，先写一句“输入是……输出是……所以我要找的是……函数”。

三类任务：输出长什么样

task types

机器学习任务的分类，第一眼可以看输出。输出是连续数值，通常叫 regression；输出是在固定选项里选一个或多个类别，通常叫 classification；输出是一个有内部结构的对象，比如句子、树、序列、路线，就进入 structured learning。这个分类不是为了背英文，而是帮你选择合适的损失函数、评价指标和模型输出形式。

Regression 的重点是“差多少”。预测明天 PM2.5 或频道点阅数时，预测 101 和真实 100 很接近，预测 900 就差很远。Classification 的重点是“选哪类”。垃圾邮件判断只有“是/否”两个选项，图片分类可能有一千个选项。Structured learning 更难，因为输出不只是一个数字或一个标签，而是一整段有顺序和约束的内容。

不要把这三类看成互相隔绝。真实系统常常混在一起。一个语音辨识系统可能先做声学特征处理，再输出文字序列；一个聊天模型每一步是在词表里分类，但最终生成的是一串有结构的文本。第一节只需要你会做第一层判断。

Regression

输出是数值。重点看误差大小，例如 MAE、MSE。常见问题是预测价格、温度、点击量、风险分数。

Classification

输出是类别。重点看选项是否选对，例如准确率、交叉熵、召回率。垃圾邮件和图片分类都属于这里。

Structured learning

输出带结构。机器翻译、语音转文字、摘要、路线规划都不只是选一个标签。

易混点正确判断常见错法
“输出是数字”可能是 regression，也可能是类别编号。数字编号如果只是标签，不代表数值大小。把猫=0、狗=1 当成连续数值去回归。
“选项很多”仍然可以是 classification。AlphaGo 的 19×19 个落子位置可以看成很多选项。以为只有二分类才叫 classification。
“输出一串文字”通常是 structured learning，因为顺序、长度、语义约束都重要。只把它当成一次分类，忽略序列关系。

现在你应该能做什么

看到任务描述时，先根据输出形态做粗分类，并能解释为什么这样分类。

模型：带未知参数的函数

model

课程里说的 model，可以先理解成“带未知参数的函数”。比如点阅数预测可以从一个很简单的模型开始：明天点阅数 = b + w × 今天点阅数。这里的 w 和 b 还不知道，要靠资料找出来。这个模型可能很粗糙，但它把问题变成了可计算的问题。

这一步也不是完全交给机器。人要决定模型形式，也就是你愿意让机器在哪个范围里找函数。只看前一天点阅数，是一种模型假设；看过去七天点阅数，是另一种模型假设；用神经网络读入更多特征，又是更宽的模型集合。模型集合越大，表达能力越强，也越容易在训练资料上“记得太好”。

参数和超参数要分开。参数是训练过程会自动调整的值，例如 w 和 b。超参数是你先设的训练设置，例如 learning rate、训练轮数、模型结构宽度。很多初学者会把它们都叫“参数”，后面调模型时就容易说不清谁由机器学、谁由人设。

ŷ = b + w × x
ŷ：预测值　x：输入特征　w,b：需要从资料中学出来的参数

模型形式不是答案

写下 ŷ = b + w x 只是给机器一个搜索范围。真正训练完之前，w 和 b 还没有确定，这个函数还不能工作。

模型太简单会欠拟合

如果点阅数有七天周期，只看前一天可能抓不住规律。你会在训练资料和新资料上都表现一般。

最小示例：从函数形式到参数python
# x: 今天点阅数，y_hat: 预测明天点阅数
w = 0.97
b = 100
y_hat = b + w * x

这段代码里，真正决定函数形状的是 w 和 b。训练的目标不是把这三行背下来，而是找到一组让预测误差较小的 w,b。

现在你应该能做什么

看到“模型”这个词时，能把它还原成“函数形式 + 待学习参数”。

Loss：把“好不好”变成数字

loss

只有模型还不够。机器需要知道某一组参数好不好，否则它不知道往哪里调。Loss 就是这个评分规则。给定一组参数，用模型在训练资料上做预测，再把预测值和真实答案的差距汇总成一个数字。一般训练时希望 loss 越小越好。

Loss 是人定义的函数，不是宇宙自带的真理。点阅数预测可以用绝对误差，也可以用平方误差；分类问题常用交叉熵；生成任务还会有更复杂的目标。不同 loss 会鼓励模型学到不同的行为。第一节你只要记住：loss 的输入通常是模型参数或模型预测，输出是“这组参数有多不合适”的数字。

课程里有同学问 loss 能不能是负的。这个问题很好，因为它逼你区分“我们通常希望 loss 表示错误程度”和“loss 作为数学函数可以怎么定义”。如果你把 loss 定义成绝对误差，它不会是负数；如果你硬把绝对误差减 100，它就可能是负数。实践中我们更关心它是否能稳定引导训练。

L(w,b) = 1N Σ | ŷi - yi |
对每笔资料计算预测误差，再取平均。

Optimization：把 loss 往小处推

optimization

定义好模型和 loss 后，第三步是 optimization，也就是找一组参数让 loss 尽量小。最经典的直觉是 gradient descent。你先随机选一个参数位置，计算 loss 在这个位置附近往哪个方向变小，然后沿着变小的方向走一步。重复很多次，参数就可能走到较低的 loss 区域。

梯度可以先理解成“当前点附近的斜率”。如果某个方向会让 loss 变大，就往反方向走。Learning rate 控制每一步走多大。步子太小，训练很慢；步子太大，可能越过低点，在 loss surface 上来回震荡，甚至越跑越差。

这里先不要被微分符号卡住。深度学习框架会自动算梯度，你真正需要理解的是梯度在训练流程里的角色：它告诉参数应该朝哪个方向更新。后面写 PyTorch 时，loss.backward() 会负责把梯度算出来，optimizer.step() 会根据梯度更新参数。

w ← w - η × ∂L∂w
η 是 learning rate。减号表示朝让 loss 下降的方向走。

错误示例：把梯度方向写反python
# 错误：这会沿着 loss 增大的方向走
w = w + lr * grad

如果你在最小化 loss，常见更新是减去梯度方向。写成加号时，除非你本来就在最大化某个目标，否则很可能让 loss 变大。

用一句话说明 gradient descent：计算当前斜率，朝 loss 下降方向移动参数。

训练资料低 loss，不等于真正会了

generalization

课程后半段开始埋一个很重要的坑：训练资料上的 loss 变小，不一定代表模型在未来数据上表现好。训练资料是模型已经看过的例子。你可以把它当成开卷练习。真正关心的是模型面对没见过的数据时，是否还能给出可靠预测。

点阅数预测里，只看前一天数据可能太简单；看过去七天可能更合理，因为观看量有周期。但如果你继续加很多特征，模型可能在训练资料上越来越好，却把偶然噪声也记住。这个问题后面会正式讲过拟合。现在你先建立一个判断：评价模型时要分清训练表现和测试表现。

这也是为什么机器学习不是“loss 越低越好”这么单薄。你要问 loss 是在哪批资料上算的，资料是否代表未来场景，模型是否只是记住历史答案。很多项目失败，不是因为训练跑不动，而是因为训练指标好看，上线后数据分布变了，模型就失灵。

训练资料

用来调整参数。模型在这批数据上表现好，只说明它学会了适配这些例子。

未见资料

用来检查泛化。模型在没见过的数据上也表现好，才更接近我们想要的“学会”。

现象可能含义下一步检查
训练 loss 高，新资料 loss 也高模型可能太简单，或特征不够，或训练没跑好。检查模型容量、学习率、资料质量。
训练 loss 低，新资料 loss 高可能过拟合，模型记住训练细节。使用验证集、正则化、减少复杂度、补资料。
训练 loss 和新资料 loss 都低模型当前较可用。继续检查业务指标、边界样本、资料分布变化。

现在你应该能做什么

听到“训练 loss 很低”时，能马上追问：在没见过的数据上呢？

最小代码：训练一条直线

code reading

下面这段代码用 NumPy 做最小线性回归。它不负责展现真实项目的全部细节，只把第一节的三个步骤串起来：先定义模型形式，再定义 loss，再用梯度下降更新参数。读代码时盯住每一行读写了什么。

正确示例：模型、loss、优化三步python
import numpy as np

x = np.array([10, 20, 30, 40], dtype=float)
y = np.array([18, 31, 42, 53], dtype=float)

w, b = 0.0, 0.0
lr = 0.0005

for step in range(2000):
    y_hat = w * x + b              # model: 用当前参数做预测
    error = y_hat - y                 # 预测和真实答案的差
    loss = np.mean(error ** 2)       # loss: 平均平方误差
    dw = np.mean(2 * error * x)       # w 的梯度
    db = np.mean(2 * error)           # b 的梯度
    w -= lr * dw                     # optimization: 更新参数
    b -= lr * db

print(w, b, loss)

这段代码的关键顺序不能乱。先用当前参数预测，才能算误差；先有误差，才能算 loss；先算出梯度，才能更新参数。如果删掉更新参数的两行，loss 每轮都用同一组 w,b，训练不会真的发生。

错误示例：只算 loss，不更新参数python
for step in range(2000):
    y_hat = w * x + b
    loss = np.mean((y_hat - y) ** 2)
    # 少了梯度和参数更新，模型不会变好

常见误区和修法

pitfalls

第一节的误区往往不是公式难，而是语言混了。下面这些坑如果不早拆，后面学深度学习会反复卡住。

误区错因后果修法
把机器学习理解成机器“自己想办法”忽略人要定义模型、loss 和资料。不知道哪里该人工设计，哪里由训练学习。改成：人在设定搜索空间，机器在资料上找参数。
把 model 当成训练好的成品没区分模型形式和参数值。写出公式却不知道参数从哪来。说清：模型是带未知参数的函数，训练后才得到具体函数。
把 regression 和 classification 按模型名字区分没有先看输出形态。遇到新任务时无法分类。先看输出：连续数值、固定类别、结构化对象。
把低训练 loss 当成学会没区分训练资料和未见资料。模型可能上线就崩。始终检查验证集或测试集表现。
把参数和超参数混在一起都叫“参数”，没有看谁自动学习。调试时不知道该训练还是手动改设置。参数由优化器学；超参数由人设或搜索。
以为 loss 必须等于准确率把训练目标和评价指标混同。分类、回归、生成任务的训练解释会乱。loss 是训练用的可优化目标，指标是评价角度。
