深度学习基本概念简介

P1 说机器学习是在找函数。P2 接着问：如果一条直线太简单，怎么写出更有弹性的函数？答案不是神秘的“让机器像人脑思考”，而是把许多带参数的非线性组件加起来，再把这些组件一层一层接起来。

你会看到 sigmoid、ReLU、activation function、neuron、hidden layer、deep learning、batch、epoch 这些词。它们都可以接回同一条线：模型是带未知参数的函数，训练是用资料和 loss 调参数。

  4. 区分 parameter、hyperparameter、batch、epoch、update。

本节的概念地图

map

深度学习没有脱离 P1 的三步。第一步仍然是写出一个带未知参数的函数，只是这个函数比直线复杂得多；第二步仍然是定义 loss；第三步仍然是用 gradient descent 一类方法找参数。变化主要发生在“模型形式”这一栏。

线性模型只能表达直线或超平面。真实世界里的输入输出关系常常有转折、饱和、分段变化。深度学习用很多非线性组件组合出更复杂的函数。每个组件先把输入加权求和，再加 bias，再通过 activation function。很多组件排成一排是一个 layer，多层连接起来就是 neural network。

线性模型太硬→ 非线性组件→ Neuron→ Hidden layer→ Deep network→ 训练参数

直觉

一条直线不够，就用许多可弯曲的小组件拼出更复杂的曲线。

正式说法

神经网络通过线性变换和非线性激活函数的组合，表示一个高弹性的参数化函数族。

判断句

如果每层只有线性变换，没有 activation function，多层叠起来仍然等价于一个线性变换。

线性模型的限制：model bias

model bias

线性模型的形式很清楚：ŷ = b + w x。它能调斜率和截距，但不管怎么调，本质上还是一条直线。如果真实关系有多个转折，线性模型就会表达不出来。这个限制来自模型本身，不是训练算法偷懒。

这里的 model bias 不要和参数里的 b 混在一起。参数 b 是 bias term，是函数里的一个可学习数字；model bias 指模型集合太受限，表达不出真实关系。一个模型如果太简单，训练资料上怎么调参数都只能达到某个水平，这就是欠拟合的一种来源。

减少 model bias 的办法是扩大模型可以表达的函数集合。你可以加入更多特征，也可以把直线换成分段线性、sigmoid 叠加、ReLU 网络。深度学习就是沿着这个方向走得很远：用大量可学习参数和非线性组件，让函数形状足够灵活。

判断提示

如果模型无论怎么训练都抓不住主要趋势，先怀疑模型表达能力不够，而不只是学习率没调好。

后面讲过拟合时还会看到相反问题：模型太复杂，在训练资料上过度贴合噪声。

概念准确含义容易混淆点

Sigmoid 和 ReLU：可调的非线性组件

activation

为了拼出复杂曲线，课程先介绍了 sigmoid。一个 sigmoid 可以看成 S 型的小函数。它前面有可调参数：w 控制斜坡陡不陡，b 控制左右位置，c 控制高度。很多 sigmoid 加起来，就能拼出带转折的曲线。

后来课程又指出，不一定非用 sigmoid。ReLU 也很常见，它的形式是 max(0, z)：输入小于 0 时输出 0，输入大于 0 时按直线增长。Sigmoid 和 ReLU 都属于 activation function。Activation 的关键作用不是“看起来像神经元”，而是引入非线性。

为什么非线性这么重要？如果你只做线性变换，第一层是线性的，第二层还是线性的，多层合起来仍然可以折叠成一个线性变换。Activation function 把这件事打断，让多层组合能表达更复杂的形状。

sigmoid(z) = 11 + e-z， ReLU(z) = max(0, z)， z = b + Σ wjxj

多特征、多 neuron 和矩阵写法

matrix view

真实任务通常不只有一个输入特征。点阅数预测可以看前一天、前两天、前三天，甚至过去 56 天的数据。一个 neuron 会对多个 feature 做加权求和，再加 bias，再通过 activation function。写成公式就是 a_i = activation(b_i + Σ_j w_ij x_j)。

下标有含义：j 表示第几个 feature，i 表示第几个 neuron。w_ij 就是第 i 个 neuron 接收第 j 个 feature 时使用的 weight。多个 neuron 同时计算时，就可以把很多 w_ij 放进矩阵，用矩阵乘法一次写完。

矩阵写法不是为了把人吓退，而是为了简洁和高效。深度学习框架里的张量计算，本质上就是把大量 neuron 的重复计算组织成矩阵乘法。你先能读懂单个 neuron 的计算，再读矩阵版就不会迷路。

z = W x + b，a = activation(z)
x：输入特征向量　W：权重矩阵　b：bias 向量　a：这一层的输出
符号直觉在代码里常见形状
x一笔资料的特征[in_features] 或 batch 版 [B, in_features]
W每个输出 neuron 怎么读每个输入特征[out_features, in_features]
b每个输出 neuron 的平移项[out_features]
a通过 activation 后的输出[out_features] 或 [B, out_features]

看到 Wx+b 时，能说出它不是一条抽象公式，而是一层 neuron 同时读取输入特征。

神经网络：把层接起来

network

一个 layer 的输出 a 可以继续作为下一层的输入。下一层再做一次 Wx+b 和 activation。这样反复几次，就得到 neural network。每一排 neuron 通常叫一层；夹在输入和输出之间的层叫 hidden layer，因为它们不是直接给用户看的输入或答案。

“神经元”这个名字容易让人误会。课程里也调侃过，不要把它讲成真的在模拟人脑。对学习者更有用的说法是：neuron 是一个可学习的小计算单元，layer 是很多小单元并排算，network 是多层小单元串起来的大函数。

当 hidden layer 变多时，人们就把它叫 deep learning。Deep 不是玄学标签，只是层数更多。真正要问的是：更深的结构是否让模型更容易表达某些函数，是否更容易训练，是否在新资料上表现更好。不是层数越多就自动更好。

小图：从输入到 hidden layer 再到输出

Neuron

做一次加权求和、加 bias、过 activation 的小单元。

Hidden layer

输入层和输出层之间的中间表示。它把原始 feature 变成更适合预测的表示。

Deep

hidden layer 变多。层数是超参数，不是越深越稳。

训练神经网络：gradient、batch、epoch

training

神经网络参数很多。把所有参数排成一个很长的向量，loss 对每个参数的偏微分也能排成一个同样长度的向量，这个向量就是 gradient。Gradient descent 的更新式仍然是 θ ← θ - ηg，只是 θ 和 g 都变成很长的向量。

实践中通常不会每次都拿全部资料算 loss 和 gradient。我们会把训练资料切成很多 batch。拿一个 batch 算一次 loss，算一次 gradient，更新一次参数。所有 batch 都看过一遍，叫一个 epoch。每次参数更新叫一次 update。

Batch size 也是超参数。它影响训练速度、梯度噪声、显存占用和泛化表现。第一节已经说 learning rate 是超参数；这一节你再加上 batch size、hidden layer 数量、每层 neuron 数量。这些不是训练自动学出来的，而是人先设定或通过搜索选择。

Gradient

所有参数对 loss 的偏微分合成的向量，告诉参数朝哪里改。

Batch

一次用来计算 loss 和 gradient 的一小批资料。

Epoch

全部训练资料被模型看过一遍。一个 epoch 里通常会有多次 update。

为什么叫 deep：深不是越多越好

deep vs fat

课程里提出一个很好的追问：如果一层里放足够多 sigmoid/ReLU 理论上也能逼近很多连续函数，为什么还要把网络变深？这不是一句“deep 听起来厉害”就能解释完的问题。更深的网络可以用层级组合来复用中间表示，在某些任务上比单层超宽网络更有效。

不过，深也带来训练难度。层数增加后，梯度传播、数值稳定、优化路径都会变复杂。后续课程会讲为什么网络训练不起来、batch normalization 等技巧。现在你只需要保留一个克制判断：深度是模型设计选择，是否合适要看训练表现和未见资料表现。

选择三层还是四层，不应该只看训练 loss。课程最后又回到泛化：我们真正要预测的是未知资料。一个更深的网络如果只是在训练资料上更低 loss，却在新资料上更差，就不是更好的模型。

比较可能优势风险判断方式
更宽同一层放更多 neuron，表达能力增加。参数多，可能更难泛化。看验证集，不只看训练 loss。
更深多层组合中间表示，可能更高效。训练难度增加，梯度和数值问题更多。看训练是否稳定、未见资料是否更好。
更简单容易训练，解释更直观。可能 model bias 高。看训练 loss 是否也降不下来。

现在你应该能做什么

评价网络深浅时，不说“更深一定更强”，而是问它是否降低 bias、是否训练得动、是否泛化更好。

最小代码：一层 hidden layer

code reading

这段 NumPy 代码只展示前向传播。它把 P2 的概念连起来：输入 x 先经过第一层线性变换，再通过 ReLU，得到 hidden representation；然后经过第二层线性变换，得到输出。训练时还需要 loss、反向传播和参数更新，这里先读懂函数本体。

正确示例：一层 hidden layer 的前向传播python
import numpy as np

def relu(z):
    return np.maximum(0, z)

x = np.array([1200, 980, 1410])      # 3 个输入 feature
W1 = np.random.randn(4, 3) * 0.01      # 4 个 hidden neuron，每个读 3 个 feature
b1 = np.zeros(4)
W2 = np.random.randn(1, 4) * 0.01      # 输出层读取 4 个 hidden 值
b2 = np.zeros(1)

z1 = W1 @ x + b1                    # linear transform
a1 = relu(z1)                      # activation，加入非线性
y_hat = W2 @ a1 + b2                # output

关键 shape 是 W1 的行数等于 hidden neuron 数，列数等于输入 feature 数。W2 的列数要等于上一层输出的长度。如果这些形状对不上，矩阵乘法会直接报错。

错误示例：删掉 activationpython
z1 = W1 @ x + b1
a1 = z1                 # 错误：这里没有非线性
y_hat = W2 @ a1 + b2

如果每层都只是线性变换，多层可以合并成一层线性变换。这样网络看起来变深了，表达能力却没有按预期增加。

现在你应该能做什么

读神经网络代码时，先找每层的线性变换、activation 和输出 shape。

常见误区和修法

pitfalls
误区错因后果修法
把 neural network 解释成真正模拟大脑被命名带偏，忽略计算结构。无法读懂代码和公式。改成：很多可学习小函数连接起来的大函数。
以为深度学习脱离了 P1 的三步只看新术语，没有回到 model/loss/optimization。训练流程断裂。每次都问模型、loss、参数更新在哪里。
忘记 activation function以为多层线性变换自然更强。多层可折叠成一层线性模型。每个 hidden layer 后检查非线性。
把 bias term 和 model bias 混同两个词都叫 bias。调错方向，把表达能力问题当成某个 b 值问题。bias term 是参数，model bias 是模型形式限制。
把 batch、epoch、update 混在一起没有按训练循环数清楚。读训练日志时误判进度。一个 batch 一次 update；所有 batch 一遍是一个 epoch。
只看训练 loss 选择更深网络忽略未知资料表现。可能选中过拟合模型。用验证集或测试集比较泛化。
