线性模型的限制

model

线性模型的形式很清楚：ŷ = b + wx。它能调斜率和截距，但本质上还是一条直线。如果真实关系有多个转折，线性模型就会卡住。这个限制来自模型本身，不是训练算法偷懒。

现在你应该能做什么

看到模型训练不好时，能先判断是“参数没找到”还是“模型形式本来就不够”。

Activation

sigmoid

为了拼出复杂曲线，课程先介绍 sigmoid。一个 sigmoid 可以看成 S 型的小函数。它前面有可调参数：w 控制斜坡陡不陡，b 控制左右位置。后来课程又指出，不一定非用 sigmoid，ReLU 也很常见，它在 0 处有折点，形式是 max(0, z)。

$$ \operatorname{ReLU}(z)=\max(0,z) $$

两者都属于 activation function。它们的关键作用引入非线性，让多层结构不再等价于一层线性变换。

矩阵和 shape

matrix

真实任务通常不只有一个输入特征。一个 neuron 会对多个 feature 做加权求和，再加 bias，再通过 activation function。写成公式就是 a_i = activation(b_i + Σ_j w_ij x_j)。下标的意义要先读准：j 表示第几个 feature，i 表示第几个 neuron。

矩阵写法为了简洁和高效。深度学习框架里的张量计算，本质上就是把大量 neuron 的重复计算组织成矩阵乘法。

现在你应该能做什么

看到 Wx+b 时，能说出它一层 neuron 同时读取输入特征。

神经网络

hidden

一个 layer 的输出 a 可以继续作为下一层的输入。下一层再做一次 Wx+b 和 activation。这样反复几次，就得到 neural network。夹在输入和输出之间的层叫 hidden layer，因为它们不是直接给用户看的输入或答案。

训练循环

batch

神经网络参数很多。把所有参数排成一个长向量，loss 对每个参数的偏微分也会排成一个长向量，这个向量就是 gradient。Gradient descent 的更新式仍然是 θ ← θ - ηg，只是 θ 和 g 都变成了很长的向量。

现在你应该能做什么

听到“训练 10 个 epoch”时，能继续问 batch size 是多少、总共 update 了几次。

为什么叫 deep

efficiency

课程里提出的关键追问是：如果一层里放足够多 sigmoid 或 ReLU，也能逼近很多函数，为什么还要把网络变深？答案更深的网络可以在某些任务上复用中间表示，用更少参数拼出同样的目标。

这里的 k 是折叠次数或层数。它只是一个课程构造例子，用来说明“同一个目标，深层可能比浅层省参数”。

边界别拉太大

2k / 2^k 是课程里的构造例子，不能直接推广到所有任务。它的作用是让你先看到“深层为什么可能更省”。

代码与状态

shape

下面这段 NumPy 代码只展示前向传播。它把 P02 的概念连起来：输入 x 先经过第一层线性变换，再通过 ReLU，得到 hidden representation；然后经过第二层线性变换，得到输出。训练时还需要 loss、反向传播和参数更新，这里先读懂函数本体。

代码
import numpy as np

def relu(z):
 return np.maximum(0, z)

# x: [3]
x = np.array([1.0, -2.0, 0.5])
W1 = np.random.randn(4, 3) * 0.01
b1 = np.zeros(4)
W2 = np.random.randn(1, 4) * 0.01
b2 = np.zeros(1)

z1 = W1 @ x + b1
a1 = relu(z1)
y_hat = W2 @ a1 + b2

删掉 activation 会怎样

如果把 a1 = relu(z1) 删掉，第二层看到的还是线性输出。多层线性变换会被折叠成一层线性模型，深度就失去意义了。

现在你应该能做什么

读神经网络代码时，先找每层的线性变换、activation 和输出 shape。

完整追一轮

manual

这一段把前向传播拉成一条可以手算、可以互相检查的链。学习深度学习时，最容易把“概念会背”误当成“机制看懂了”；而真正稳的理解，是你能把输入、线性变换、激活函数和输出一层层追出来，知道每一步为什么要这样算。

坏掉时会发生什么

如果少了 activation，多层线性会折成一层线性，深度的意义就没了。这个结论你把上面的 trace 真正走一遍之后会直接看到的结果。

如果 shape 对不上，矩阵乘法会在前向阶段就报错。计算图本身断了。

如果把 batch 当成单笔样本来读日志，你会把更新次数、数据量和 epoch 关系全部看偏。那种错很隐蔽，因为程序还能跑。

现在你应该能做什么

拿到任何一段前向传播代码时，先把它翻译成“输入 → 线性变换 → activation → 输出”四步，再去找 shape、激活函数和层数是否一致。只要这一步能做，你对神经网络的理解就不只是概念层面的。

对照一层和两层

linear

同一个输入放进不同模型，差别会非常具体。纯线性模型只做一次投影，输出由一个线性关系决定；一层 hidden layer 会先制造一个中间表示，再让输出层在这个表示上做判断；两层 hidden layer 继续把上一层的结果重写一次，复用的表示更多，能拼出的形状也更细。这里真正重要的，中间表示有没有被下一层继续使用。

模型计算链能看到什么常见误判
纯线性y = wx + b一次投影就出结果，函数形状始终平滑。以为只要把训练做久一点，就能补上表达能力。
单 hidden layerx → Wx+b → ReLU → y中间会冒出折点，某些通道会被激活，某些通道会被压成 0。把 activation 当成可选装饰，忽略它在改函数形状。
双 hidden layerx → h1 → h2 → y上一层的表示继续被重组，局部特征可以被复用。把“更深”误写成“更神秘”，却说不出中间表示到底变了什么。
多层线性堆叠Wx+b 反复相乘表面层数很多，实质还是一个线性映射。把层数当成非线性的数量，结果把深度的意义看空。

同一输入的两种写法

对照最有用的地方，是把“有激活”和“没激活”直接并排。你会看到，两层网络之所以比一层线性模型更灵活，因为中间那一步把输入重新编码了。

# 有非线性
a1 = relu(W1 @ x + b1)
y1 = W2 @ a1 + b2

# 没有非线性
a1_linear = W1 @ x + b1
y2 = W2 @ a1_linear + b2

第二段代码看起来也像“网络”，但它没有真正的折点。继续把 W2 @ (W1 @ x + b1) + b2 展开，你会发现它仍然能收拢成一个线性形式。也就是说，层数增加了，表达方式却没变。

失败后果

如果把多层网络里的 activation 拿掉，实验里最常见的现象模型整个退回线性家族。你会得到看起来很深、实际上很平的计算图，然后在训练时误以为自己还在做深度学习。

如果再把 shape 和 batch 一起读错，问题会更乱：有些错误会直接在矩阵乘法那一步报出来，有些错误则会一直藏到验证集上。前者容易排查，后者更耗时间，所以 trace 的价值就在这里。

练习钩子

以后看到“更深的网络”，先不要问它是不是更大。先问三件事：中间表示在哪一层被改写、哪一层引入了非线性、如果删掉 activation 还剩下什么。只要这三个问题能答出来，你就能把深度和宽度分开看。

再往后做 P03、P10、P11 时，这个习惯会一直有用，因为很多后续问题其实都在问同一件事：模型现在到底在表示什么，代价是多少，失败会出在哪一步。

现在你应该能做什么

拿到一个神经网络结构时，先判断它有没有真正的非线性，再判断它是不是只是在堆线性层。能做到这一步，就已经能解释大部分“为什么这个模型看起来更深却不一定更强”的问题了。

比如二维分类里如果正负样本像棋盘一样交错，一条直线只能切一次，线性模型会卡住；一层带 ReLU 的网络可以先把输入拆成几个局部区间，再把这些区间重新拼起来。这个具体场景能帮助你把“表示能力”和“训练是否到位”分开看：前者决定边界能不能画出来，后者决定你有没有把这条边界学稳。

把这个场景记住，后面再看任何“更深的网络”，你都能先问：它究竟是在改边界，还是只是在加参数。

再做一次误差排查

checklist

如果你已经能看懂 trace 和 compare，就可以把它们接成一个检查顺序。先确认输入和权重 shape 是否对齐，再确认每个 hidden layer 后面是否真的有 activation，再确认训练日志里 batch、update、epoch 的关系是否被看错，最后才去判断“模型是不是太浅”或“是不是参数不够”。这个顺序很重要，因为它能帮你把最常见的错误挡在最前面。

这一步的意义

很多人会把“深度学习”先想成更复杂的名词集合，再去背 layer、neuron、activation。更稳的顺序是：先会追一轮，再会看一张表，再会查一次错。这样学到的内容才会落在手上，而不是只留在定义里。你现在应该已经能做的，就是把一次失败拆成“前向算不通”“表达不够”“训练读错”三类，再决定下一步查什么。很多时候你并不需要立刻扩大模型，只需要先把失败类型分清。
