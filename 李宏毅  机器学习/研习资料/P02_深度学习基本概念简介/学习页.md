# 本节的主线 {#goals}
<!-- tag: map -->

这节课没有离开 P01 的三步：先写模型，再定义 loss，再用梯度方法调参数。变化主要发生在模型形式上。线性模型只能表达直线或超平面，深度学习则把线性变换和非线性激活叠起来，让函数形状更灵活。

直觉一条直线不够，就用许多可弯曲的小组件拼出更复杂的曲线。
正式说法神经网络通过线性变换和非线性激活函数的组合，表示一个更大的参数化函数族。
先记住如果每层只有线性变换，没有 activation function，多层叠起来仍然等价于一层线性变换。

# 线性模型的限制 {#bias}
<!-- tag: model bias -->

线性模型的形式很清楚：`ŷ = b + wx`。它能调斜率和截距，但本质上还是一条直线。如果真实关系有多个转折，线性模型就会卡住。这个限制来自模型本身，不是训练算法偷懒。

小图：直线追不上有转折的关系


线性模型

## box warn
### 区分两种 bias
`bias term` 是参数 `b`，`model bias` 是模型形式太受限。前者是一个数字，后者是表达能力问题。
概念准确含义容易混淆点

bias term模型里的可学习常数项。它是参数。
model bias模型集合太受限，表达不出目标函数。它是模型形式带来的限制。
欠拟合训练资料和新资料都表现不好。常见原因之一是 model bias 太高。

### 现在你应该能做什么
看到模型训练不好时，能先判断是“参数没找到”还是“模型形式本来就不够”。

# Activation {#activation}
<!-- tag: sigmoid / ReLU -->

为了拼出复杂曲线，课程先介绍 sigmoid。一个 sigmoid 可以看成 S 型的小函数。它前面有可调参数：`w` 控制斜坡陡不陡，`b` 控制左右位置。后来课程又指出，不一定非用 sigmoid，ReLU 也很常见，它在 `0` 处有折点，形式是 `max(0, z)`。

$$\sigma(z)=\frac{1}{1+e^{-z}}$$
 $$\operatorname{ReLU}(z)=\max(0,z)$$
 两者都属于 activation function。它们的关键作用引入非线性，让多层结构不再等价于一层线性变换。

参数怎样改变 sigmoid
w 控制陡峭程度：1.0
b 控制左右平移：0.0

ReLU 参考


**参数观察**
拖动滑块，观察曲线变陡、变平、左右移动。你现在看的是一个可调小函数。
判断提示：activation function 不负责直接给最终答案，它负责把线性输入变成非线性表示。

### 现在你应该能做什么
解释为什么神经网络不能只堆线性层，并能说出 sigmoid 和 ReLU 的共同角色。

# 矩阵和 shape {#shape}
<!-- tag: matrix view -->

真实任务通常不只有一个输入特征。一个 neuron 会对多个 feature 做加权求和，再加 bias，再通过 activation function。写成公式就是 `a_i = activation(b_i + Σ_j w_ij x_j)`。下标的意义要先读准：`j` 表示第几个 feature，`i` 表示第几个 neuron。

$$z = Wx + b,\quad a = activation(z)$$
 矩阵写法为了简洁和高效。深度学习框架里的张量计算，本质上就是把大量 neuron 的重复计算组织成矩阵乘法。

| 对象 | 它代表什么 | 典型 shape |
| --- | --- | --- |
| x | 一笔资料的特征 | [in_features] 或 batch 版 [B, in_features] |
| W | 每个输出 neuron 怎么读每个输入特征 | [out_features, in_features] |
| b | 每个输出 neuron 的平移项 | [out_features] |
| a | 通过 activation 后的输出 | [out_features] 或 [B, out_features] |

## box soft
### 现在你应该能做什么
看到 `Wx+b` 时，能说出它一层 neuron 同时读取输入特征。

# 神经网络 {#network}
<!-- tag: hidden layer -->

一个 layer 的输出 `a` 可以继续作为下一层的输入。下一层再做一次 `Wx+b` 和 activation。这样反复几次，就得到 neural network。夹在输入和输出之间的层叫 hidden layer，因为它们不是直接给用户看的输入或答案。

<!-- interactive: ['shape-tracer'] → keep as HTML -->

## box warn
### 不要混掉的地方
“神经元”只是一个可学习的小计算单元，不是在真的模拟人脑。对学习者更有用的说法是：neuron 是一个小函数，layer 是很多小函数并排算，network 是多层小函数串起来的大函数。

# 训练循环 {#train}
<!-- tag: batch / epoch -->

神经网络参数很多。把所有参数排成一个长向量，loss 对每个参数的偏微分也会排成一个长向量，这个向量就是 gradient。Gradient descent 的更新式仍然是 `θ ← θ - ηg`，只是 `θ` 和 `g` 都变成了很长的向量。

## box info
### 读训练日志时先看这三件事
- `batch`：一次用来算 loss 和 gradient 的小批资料。
- `update`：参数真正改了一次。
- `epoch`：所有训练资料被模型看过一遍。

# 为什么叫 deep {#deep}
<!-- tag: efficiency -->

课程里提出的关键追问是：如果一层里放足够多 sigmoid 或 ReLU，也能逼近很多函数，为什么还要把网络变深？答案更深的网络可以在某些任务上复用中间表示，用更少参数拼出同样的目标。

$$\text{deep} \approx 2k,\qquad \text{shallow} \approx 2^k$$
 这里的 `k` 是折叠次数或层数。它只是一个课程构造例子，用来说明“同一个目标，深层可能比浅层省参数”。

| k | deep：约 2k 个 neuron | shallow：约 2^k 个 neuron | 看见什么差别 |
| --- | --- | --- | --- |
| 1 | 2 | 2 | 差别还看不出来 |
| 2 | 4 | 4 | 刚开始一样 |
| 3 | 6 | 8 | 浅层开始更快 |
| 4 | 8 | 16 | 差距已经明显 |
| 5 | 10 | 32 | 宽度开始追不上折叠速度 |

## box warn
### 边界别拉太大
`2k` / `2^k` 是课程里的构造例子，不能直接推广到所有任务。它的作用是让你先看到“深层为什么可能更省”。

# 代码与状态 {#code}
<!-- tag: shape / flow -->

下面这段 NumPy 代码只展示前向传播。它把 P02 的概念连起来：输入 `x` 先经过第一层线性变换，再通过 ReLU，得到 hidden representation；然后经过第二层线性变换，得到输出。训练时还需要 loss、反向传播和参数更新，这里先读懂函数本体。

P02 forward passpython

```python
import numpy as np

def relu(z):
 return np.maximum(0, z)

# x: [3]
x = np.array([1.0, -2.0, 0.5])
W1 = np.random.randn(4, 3) * 0.01
b1 = np.zeros(4)
W2 = np.random.randn(1, 4) * 0.01
b2 = np.zeros(1)

z1 = W1 @ x + b1
a1 = relu(z1)
y_hat = W2 @ a1 + b2
```

## box soft
### 逐行怎么读
- `W1` 的行数等于 hidden neuron 数，列数等于输入 feature 数。
- `W2` 的列数要等于上一层输出的长度。
- 如果这些 shape 对不上，矩阵乘法会直接报错。

## box info
### 删掉 activation 会怎样
如果把 `a1 = relu(z1)` 删掉，第二层看到的还是线性输出。多层线性变换会被折叠成一层线性模型，深度就失去意义了。

## box soft
### 现在你应该能做什么
读神经网络代码时，先找每层的线性变换、activation 和输出 shape。

# 完整追一轮 {#trace}
<!-- tag: manual trace -->

这一段把前向传播拉成一条可以手算、可以互相检查的链。学习深度学习时，最容易把“概念会背”误当成“机制看懂了”；而真正稳的理解，是你能把输入、线性变换、激活函数和输出一层层追出来，知道每一步为什么要这样算。

## box soft
### 先给一个能算完的例子
设输入 `x = [2, -1, 3]`，第一层参数为 `W1 = [[1, 0, 1], [0, 1, -1]]`、`b1 = [0, 1]`，第二层参数为 `W2 = [[0.4, -0.2]]`、`b2 = [0.4]`。那一轮前向传播会得到：
```python
z1 = W1 @ x + b1
# z1 = [5, -3]
a1 = relu(z1)
# a1 = [5, 0]
y_hat = W2 @ a1 + b2
# y_hat = [2.4]
```

这个例子刻意让第二个 hidden neuron 变成 0。这样你能直接看到 ReLU 的作用把一部分线性响应截掉，让后面一层看到更稀疏、更好组合的表示。

| 步骤 | 数值 | shape | 在做什么 |
| --- | --- | --- | --- |
| x | [2, -1, 3] | [3] | 原始输入特征。 |
| z1 = W1 @ x + b1 | [5, -3] | [2] | 第一层把三个特征压成两个 hidden 前激活值。 |
| a1 = relu(z1) | [5, 0] | [2] | 非线性把负值砍掉，留下可复用的中间表示。 |
| y_hat = W2 @ a1 + b2 | [2.4] | [1] | 输出层把表示收束成最终预测。 |

$$z^{(1)} = W^{(1)}x+b^{(1)}$$
 $$a^{(1)} = \operatorname{ReLU}(z^{(1)})$$
 $$\hat{y} = W^{(2)}a^{(1)}+b^{(2)}$$
 把式子写成带上标的形式，是为了看清“第几层”和“哪个张量”在对应哪一步。真正做代码审查时，你要盯的就是这条链有没有断。

## box warn
### 坏掉时会发生什么
如果少了 activation，多层线性会折成一层线性，深度的意义就没了。这个结论你把上面的 trace 真正走一遍之后会直接看到的结果。
如果 shape 对不上，矩阵乘法会在前向阶段就报错。计算图本身断了。
如果把 batch 当成单笔样本来读日志，你会把更新次数、数据量和 epoch 关系全部看偏。那种错很隐蔽，因为程序还能跑。

## box info
### 现在你应该能做什么
拿到任何一段前向传播代码时，先把它翻译成“输入 → 线性变换 → activation → 输出”四步，再去找 shape、激活函数和层数是否一致。只要这一步能做，你对神经网络的理解就不只是概念层面的。

# 对照一层和两层 {#compare}
<!-- tag: linear vs deep -->

同一个输入放进不同模型，差别会非常具体。纯线性模型只做一次投影，输出由一个线性关系决定；一层 hidden layer 会先制造一个中间表示，再让输出层在这个表示上做判断；两层 hidden layer 继续把上一层的结果重写一次，复用的表示更多，能拼出的形状也更细。这里真正重要的，中间表示有没有被下一层继续使用。

你可以把前面的 trace 当成一条主线，然后用下面的对照表去检查：每一种模型到底在哪一步引入了非线性，哪一种只是把线性算子多叠了几次，哪一种会在调参时把你带进“参数很多但表达方式没变”的坑里。能分清这个，后面看训练曲线和泛化误差时就不会只盯着层数。

| 模型 | 计算链 | 能看到什么 | 常见误判 |
| --- | --- | --- | --- |
| 纯线性 | y = wx + b | 一次投影就出结果，函数形状始终平滑。 | 以为只要把训练做久一点，就能补上表达能力。 |
| 单 hidden layer | x → Wx+b → ReLU → y | 中间会冒出折点，某些通道会被激活，某些通道会被压成 0。 | 把 activation 当成可选装饰，忽略它在改函数形状。 |
| 双 hidden layer | x → h1 → h2 → y | 上一层的表示继续被重组，局部特征可以被复用。 | 把“更深”误写成“更神秘”，却说不出中间表示到底变了什么。 |
| 多层线性堆叠 | Wx+b 反复相乘 | 表面层数很多，实质还是一个线性映射。 | 把层数当成非线性的数量，结果把深度的意义看空。 |

## box soft
### 同一输入的两种写法
对照最有用的地方，是把“有激活”和“没激活”直接并排。你会看到，两层网络之所以比一层线性模型更灵活，因为中间那一步把输入重新编码了。
```python
# 有非线性
a1 = relu(W1 @ x + b1)
y1 = W2 @ a1 + b2

# 没有非线性
a1_linear = W1 @ x + b1
y2 = W2 @ a1_linear + b2
```

第二段代码看起来也像“网络”，但它没有真正的折点。继续把 `W2 @ (W1 @ x + b1) + b2` 展开，你会发现它仍然能收拢成一个线性形式。也就是说，层数增加了，表达方式却没变。

## box warn
### 失败后果
如果把多层网络里的 activation 拿掉，实验里最常见的现象模型整个退回线性家族。你会得到看起来很深、实际上很平的计算图，然后在训练时误以为自己还在做深度学习。
如果再把 shape 和 batch 一起读错，问题会更乱：有些错误会直接在矩阵乘法那一步报出来，有些错误则会一直藏到验证集上。前者容易排查，后者更耗时间，所以 trace 的价值就在这里。

## box info
### 练习钩子
以后看到“更深的网络”，先不要问它是不是更大。先问三件事：中间表示在哪一层被改写、哪一层引入了非线性、如果删掉 activation 还剩下什么。只要这三个问题能答出来，你就能把深度和宽度分开看。
再往后做 P03、P10、P11 时，这个习惯会一直有用，因为很多后续问题其实都在问同一件事：模型现在到底在表示什么，代价是多少，失败会出在哪一步。

## box soft
### 现在你应该能做什么
拿到一个神经网络结构时，先判断它有没有真正的非线性，再判断它是不是只是在堆线性层。能做到这一步，就已经能解释大部分“为什么这个模型看起来更深却不一定更强”的问题了。
比如二维分类里如果正负样本像棋盘一样交错，一条直线只能切一次，线性模型会卡住；一层带 ReLU 的网络可以先把输入拆成几个局部区间，再把这些区间重新拼起来。这个具体场景能帮助你把“表示能力”和“训练是否到位”分开看：前者决定边界能不能画出来，后者决定你有没有把这条边界学稳。
把这个场景记住，后面再看任何“更深的网络”，你都能先问：它究竟是在改边界，还是只是在加参数。

# 再做一次误差排查 {#bridge-check}
<!-- tag: checklist -->

如果你已经能看懂 trace 和 compare，就可以把它们接成一个检查顺序。先确认输入和权重 shape 是否对齐，再确认每个 hidden layer 后面是否真的有 activation，再确认训练日志里 batch、update、epoch 的关系是否被看错，最后才去判断“模型是不是太浅”或“是不是参数不够”。这个顺序很重要，因为它能帮你把最常见的错误挡在最前面。

## box soft
### 从代码回到日志
当你读到 `z1 = W1 @ x + b1`，最先要问的是 `W1` 的行列数是不是和输入对应；当你读到 `a1 = relu(z1)`，最先要问的是非线性有没有真的发生；当你读到一串 loss 数字，最先要问的是这些数字属于哪一个 batch、哪一次 update、哪一个 epoch。只要这些问题问对，很多误判会直接消失。
这也是为什么课程把线性模型、activation、shape、network 和 training 放在一起讲，而不是拆成完全独立的零件。它们在实际排错时本来就是一条链，单独记词义不够，必须能把它们串回同一条计算路径。
你现在已经见到两种最常见的失败：一种是模型根本算不下去，另一种是模型算得下去但表达方式退回去了。前者是 shape 和连接关系的问题，后者是 activation 和结构设计的问题。只要能先分辨这两类，排错速度会快很多。

| 先查什么 | 看什么 | 为什么 |
| --- | --- | --- |
| shape | W 、 x 、 b 的维度 | 矩阵乘法不对齐时，模型连前向都跑不通。 |
| activation | 每个 hidden layer 后是否真有非线性 | 少了这一步，层数再多也可能只是线性堆叠。 |
| batch / update | 一次更新看的是多少样本 | 读错训练日志会让你误判学习速度。 |
| 泛化 | 验证集和测试集的表现 | 训练集好不等于真实任务好。 |

## box warn
### 这一步的意义
很多人会把“深度学习”先想成更复杂的名词集合，再去背 layer、neuron、activation。更稳的顺序是：先会追一轮，再会看一张表，再会查一次错。这样学到的内容才会落在手上，而不是只留在定义里。你现在应该已经能做的，就是把一次失败拆成“前向算不通”“表达不够”“训练读错”三类，再决定下一步查什么。很多时候你并不需要立刻扩大模型，只需要先把失败类型分清。

| 错因 | 后果 | 修法 |
| --- | --- | --- |
| hidden layer 后忘记 activation | 多层线性会折叠成一层线性模型 | 每个 hidden layer 后检查非线性 |
| 把 bias term 和 model bias 混同 | 调错方向，把表达能力问题当成某个 b 值问题 | bias term 是参数，model bias 是模型形式限制 |
| 把 batch、epoch、update 混在一起 | 读训练日志时误判进度 | 一个 batch 通常对应一次 update；所有 batch 一遍是一个 epoch |

# 常见误区 {#mistakes}
<!-- tag: pitfalls -->

| 误区 | 错因 | 后果 | 修正方式 |
| --- | --- | --- | --- |
| 把“深”理解成“更像大脑” | 被术语带跑 | 忽略真正重要的表示和训练问题 | 先把它当成一组可学习的小函数 |
| 一层能表示很多函数，所以深度没意义 | 把“能表示”直接等同于“表示得省” | 看不到参数效率 | 先问同一个函数要多少 neuron，再谈结构 |
| 深度学习只是把模型做大 | 只看参数数目，不看中间表示是否可复用 | 把“深”误解成“胖” | 对比深层和浅层在同样目标下的表达成本 |
| 只看训练 loss 选择更深网络 | 忽略未知资料表现 | 可能选中过拟合模型 | 用验证集或测试集比较泛化 |
| ReLU 就等于 piecewise linear | 把一种激活函数和一种函数形态混为一谈 | 概念边界变钝 | 说 ReLU 有利于拼出 piecewise linear，不要直接画等号 |

### p02-q5 排序

**Q:** 5. 排列一层 hidden layer 的计算顺序。

Explain: 正确顺序是：输入 x → 加权求和 Wx+b → activation → 输出 a 。写顺序时不要漏掉 activation。

### p02-q6 匹配

**Q:** 6. 匹配 W、b、a、activation 的作用。

Explain: W 负责加权， b 负责平移， activation 负责引入非线性， a 是经过激活后的输出。只要把“谁算什么”写清楚就行。

### p02-q8 短答

**Q:** 8. 为什么多个 neuron 能拼出更复杂的函数？

### p02-q13 代码阅读

**Q:** 13. 解释 `z1`、`a1`、`y_hat` 各自是什么。

Explain: z1 是第一层线性变换的输出， a1 是过 ReLU 后的 hidden representation， y_hat 是最后的预测值。回答时最好顺手指出第二层读的是上一层的输出。

### p02-q14 错误诊断

**Q:** 14. hidden layer 后没有 activation，会有什么问题？

Explain: 错因：把 hidden layer 当成只是叠更多线性层。后果：多层线性会折叠成一层线性模型，模型不再更灵活。修法：每个 hidden layer 后加上合适的 activation function。

### p02-q15 错误诊断

**Q:** 15. 只因为四层网络训练 loss 更低，就一定选四层，哪里不稳？

Explain: 错因：只看训练 loss，忽略泛化。后果：可能选中一个在训练资料上更低、但在新资料上更差的模型。修法：用验证集或测试集比较，并把 model bias、参数效率和泛化一起看。

### p02-q16 场景应用

**Q:** 16. 如果模型在训练资料和新资料上都表现差，优先怀疑什么？

Explain: 先怀疑 model bias 太高，或者模型设置、训练流程本身有问题。若训练集和新资料都差，往往不是“过拟合太严重”这一个解释就够了。
