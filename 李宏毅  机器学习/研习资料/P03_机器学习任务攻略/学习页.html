机器学习任务攻略

前两节把机器学习讲成“找函数”。这一节开始教你做作业和项目时怎么排查问题：结果不好时，不要一上来就换大模型，也不要只盯 leaderboard。先看 training loss，再看 testing 或 validation loss，沿着诊断树判断是 model bias、optimization，还是 overfitting。

这套攻略会贯穿后面的作业。你会经常遇到：训练资料学不好、训练资料很好但新资料很差、public 分数很好但 private 掉下去。每种现象背后的原因和下一步动作都不一样。

学完你应该能做什么

  1. 区分 training set、validation set、testing set 的角色。
  2. 用 training loss 判断模型是否至少学会训练资料。
  3. 区分 model bias、optimization failure 和 overfitting。
  4. 知道 data augmentation、限制模型、early stopping 各自适合什么问题。
  5. 解释为什么不能用 public leaderboard 反复挑模型。

本节的诊断地图

strategy

机器学习任务里，训练资料通常有 x 和对应的 y；测试资料常常只有 x，没有公开答案。训练完成后，你把学到的参数 θ* 放进模型 fθ(x)，对测试资料产生预测，再提交或部署。

真正困难的地方在于：分数不好时，原因不止一种。training loss 高，说明模型连见过的资料都没学好；这可能是模型太简单，也可能是优化没把参数找好。training loss 低但 validation/testing loss 高，通常更接近过拟合或资料分布问题。不同原因对应不同动作。

结果不好→ 先看 training loss→ 高：bias/optimization→ 低：看 validation→ 高：overfitting→ 检查 leaderboard

直觉

先确认模型会不会做练习题，再问它会不会做没见过的题。

正式说法

训练误差用于判断拟合训练集的能力，验证误差用于模型选择，测试误差用于最终泛化评估。

判断句

如果 training loss 还高，不要急着谈 overfitting；overfitting 通常要求训练表现已经很好。

训练资料、测试资料和要上传的预测

data split

训练资料里有一对一的 (x, y)。x 是 feature，y 是 label。比如语音辨识的 x 是声音，y 是文字；机器翻译的 x 是源语言句子，y 是目标语言句子；分类任务的 y 可能是类别编号。

测试资料通常只有 x，没有 y。这就是为什么你不能直接在真正 testing set 上算 loss。竞赛平台会用隐藏答案帮你算分，但你看不到完整答案。作业和竞赛会把 testing set 再分成 public 和 private，public 分数可以早看到，private 分数通常最后才公开。

这会带来一个重要后果：你不能把 public 分数当成无限可用的调参工具。你每根据 public 分数调整一次模型，就在把模型往 public set 上贴。调太多次后，public 分数会变得不可靠。

资料有什么用来做什么不能做什么
training setx 和 y训练参数单独证明泛化好
validation setx 和 y，从训练资料切出挑模型、调超参数、early stopping反复调到过度贴合后仍当无偏评估
public test通常只有 x，平台给公开分数粗看提交表现无限次用来选择模型
private test隐藏答案，最终评估最终泛化估计训练或调参时使用

诊断树：第一步永远先看 training loss

decision tree

作业分数不好，第一件事不是换模型，也不是多上传几次。先看 training loss。如果 training loss 高，说明模型在见过答案的资料上也没有学好。这时候谈 public/private 分数意义不大，因为基础训练还没过关。

如果 training loss 已经低，再看 validation 或 testing loss。如果新资料上的 loss 高，说明模型对训练资料适配得好，但泛化差。这通常会引向 overfitting、资料不足、资料分布不一致、模型选择方式错误等问题。

这条诊断树的价值在于减少乱试。training loss 高时，可能该增加模型弹性或改善优化；training loss 低但 validation 高时，可能该加资料、做 augmentation、限制模型、early stopping 或重新设计验证集。

拿到坏结果时，先按 training loss 和 validation loss 定位问题类型。

Training loss 高：model bias 还是 optimization？

bias vs optimization

training loss 高有两种常见原因。第一种是 model bias：你设定的模型集合里根本没有足够好的函数。就像在错误的海域捞针，针不在海里，再努力也捞不到。解决方向是增加 feature、使用更有弹性的模型、改模型结构。

第二种是 optimization failure：模型集合里其实有好函数，但优化算法没找到。针在海里，可是你没有捞起来。深层网络里这种问题很常见。一个更深、更有弹性的模型理论上应该至少能做到浅模型的训练 loss；如果它反而更高，就很可能是 optimization 没做好。

判断方法是比较模型。先跑简单模型，得到一个 training loss 基准；再跑更复杂模型。如果复杂模型的 training loss 明显更低，说明增加弹性有用，原模型可能 bias 高。如果复杂模型明明更有弹性，却训练 loss 更高，先怀疑优化、学习率、初始化、网络结构或训练技巧。

现象更像什么为什么下一步
简单模型和复杂模型 training loss 都高可能 model bias 或资料/目标问题模型集合都找不到好函数加 feature、改任务定义、换模型族
复杂模型 training loss 下降很多原模型 bias 高增加弹性后训练集能拟合继续观察 validation，防过拟合
更深模型 training loss 反而更高optimization failure更深模型理论上可模拟浅模型，却没训练到调学习率、初始化、归一化、残差、优化器

现象：training loss 高，validation loss 也高
错误结论：这是 overfitting
问题：模型连训练资料都没拟合好，不符合 overfitting 的典型特征
更合理：先查 model bias 或 optimization

用“针在不在海里 / 捞没捞起来”区分模型表达能力和优化失败。

Training 低、Testing 高：才开始谈 overfitting

overfitting

Overfitting 的典型现象是：training loss 很低，但 validation/testing loss 高。模型在训练资料上表现很好，却没有学到能迁移到新资料的规律。它可能记住了训练资料里的偶然细节、噪声或特殊分布。

模型复杂度增加时，training loss 往往会持续下降，因为模型越来越能贴合训练资料。但 testing loss 不一定一直下降。刚开始模型变复杂会降低 bias，testing loss 也下降；复杂到某个程度后，模型开始贴训练噪声，testing loss 反而上升。

判断提示

过拟合不是“模型表现不好”的同义词。它特指训练集好、新资料差。

如果训练集也差，先回到 model bias 或 optimization。

欠拟合

训练和新资料都差。常见原因是模型太简单、feature 不够、训练没跑好。

过拟合

训练好，新资料差。常见原因是模型太复杂、资料少、验证方式不可靠。

合适

训练和新资料都不错。仍要检查边界样本和分布变化。

现在你应该能做什么

准确使用 overfitting 这个词，不把所有坏结果都叫过拟合。

处理策略：加资料，或限制模型

fixes

面对 overfitting，常见方向有两个。第一是增加有效资料。作业里通常不能自己额外收集资料，但可以做 data augmentation：根据任务允许的变换，从已有资料生成合理的新样本。影像任务里水平翻转、裁剪、轻微缩放都很常见。

Data augmentation 不能乱做。你要确认变换后 label 仍然正确，且变换后的样本像真实世界会出现的资料。把猫狗图片水平翻转通常合理；把数字 6 上下翻转可能变成 9，label 就不一定还能保留。错误 augmentation 会教模型学错规律。

第二是限制模型弹性，也就是 regularization 的思想。方法包括减少 feature、使用更适合领域的结构、weight decay、dropout、early stopping、限制模型复杂度等。限制太少会过拟合，限制太多又回到 model bias。目标不是越限制越好，而是让模型复杂度和资料量匹配。

方法作用适合现象注意事项
data augmentation增加合理变体训练好、新资料差、资料少变换后 label 必须仍正确
减少无用 feature降低模型可钻空子的空间feature 很多、validation 差别删掉真正关键的信息
early stopping在 validation 变坏前停止训练训练 loss 继续降，validation 开始升需要可靠 validation set
更合适的模型结构用领域结构限制函数集合通用模型过拟合或效率差结构假设要符合任务

面对过拟合时，不只说“换模型”，而能给出加资料和限制模型两类动作。

Validation：用训练资料切出模型选择依据

validation

合理的模型选择流程是：从训练资料中切出一部分作为 validation set。模型只用 training set 学参数，再用 validation set 比较不同模型或超参数。最后把选好的模型拿去提交 testing set。

为什么不直接用 public testing score 挑模型？因为你会不自觉地把模型调到 public set 上。public set 虽然不是训练资料，但你反复看它的分数，它就参与了模型选择。次数多了，public 分数就不再是可靠的新资料估计。

Validation set 也不是万能。如果你根据同一个 validation set 调整太多轮，也会过拟合 validation。实践中可以用交叉验证、保留最终测试集、限制实验次数、记录每次改动理由来降低这个风险。P3 先记住最基本原则：模型选择用 validation，不要用 public leaderboard 当主要调参依据。

原始训练资料 → training set + validation set
training set：训练参数　validation set：挑模型和超参数
最小示例：切 validation setpython
from sklearn.model_selection import train_test_split

X_train, X_valid, y_train, y_valid = train_test_split(
    X, y, test_size=0.1, random_state=42
)

# 只用 X_train/y_train 更新参数
model.fit(X_train, y_train)

# 用 validation loss 选择模型，不用 public leaderboard 反复挑
valid_score = evaluate(model, X_valid, y_valid)

Public leaderboard 为什么会骗人

public/private

如果你上传一大堆随机模型，总会有某个模型碰巧在 public set 上分数不错。这个模型不一定学到了规律，可能只是撞上了 public set 的偶然性。把它当成最优模型，到了 private set 就可能掉得很惨。

这和公开 benchmark 也有关。一个公开测试集被很多人反复使用后，社区会逐渐对它“调参”。论文上某个公开测试集超强，不等于真实生活里一定同样强。P3 的提醒很朴素：不要把同一个公开测试集当成无限次试错的目标。

在作业里，public 分数当然可以参考，但更稳的做法是以 validation loss 为主要依据。看到 public 分数异常好，也不要太早开心；看到 public 分数一般，也先检查 validation 是否可靠、是否和 public/private 分布接近。

最小代码：按诊断树输出建议

code reading

下面这段代码不是训练模型，而是把本节诊断流程写成可读逻辑。它帮你检查：training loss 是否高；如果高，比较复杂模型是否更低；如果 training 低但 validation 高，指向过拟合；如果 public 和 validation 差异大，提醒你别只看 leaderboard。

正确示例：诊断训练结果python
def diagnose(train_loss, valid_loss, simple_train_loss, public_score=None):
    if train_loss > 1.0:
        if train_loss > simple_train_loss:
            return "复杂模型训练 loss 比简单模型还高：先查 optimization。"
        return "training loss 高：先查 model bias 或 feature 是否不足。"

    if valid_loss > 1.0:
        return "training 低但 validation 高：优先查 overfitting。"

    if public_score is not None and abs(public_score - valid_loss) > 0.5:
        return "public 和 validation 差异大：检查切分和分布。"

    return "当前结果较稳，继续做错误分析和边界检查。"
错误示例：只按 public 分数挑模型python
# 错误：把 public leaderboard 当成主要模型选择工具
best_model = min(models, key=lambda m: public_leaderboard_score(m))

# 更稳：先按 validation score 选择，public 只做有限参考
best_model = min(models, key=lambda m: validation_score(m))
