# 这页先解决什么 {#goals}
<!-- tag: overview -->

机器学习任务里，训练只是手段，分数也不是第一步。先把问题切成三层：模型会不会把训练资料学好，学好之后能不能迁移到没见过的数据，迁移不好时是模型太弱、优化没跟上，还是太容易记住训练细节。P03 就是把这条排查顺序讲清楚。

## box soft
### 最稳的判断顺序
1. 先看 training loss。
2. training loss 低，再看 validation / testing loss。
3. 新资料差，再区分 overfitting、资料切分问题和 public leaderboard 误导。

## box info
### 先别跑偏
这一页先不追更深的优化理论，也不讲 Hessian、saddle point、batch norm 或更后面的结构技巧。先把任务排查流程练熟，后面再接 P04 和后续内容。

结果不好→
先看 training loss→
再看 validation→
最后看 public / private

# 先抓主线 {#route}
<!-- tag: diagnosis route -->

直觉
先确认模型能不能做题，再确认它会不会只会做见过的题。


正式说法
training loss 用来判断拟合训练集的能力；validation set 用来挑模型和超参数；testing set 用来做最终泛化评估。


最容易错的顺序
training loss 还高时就谈 overfitting，通常会把问题看反。先判断模型有没有把训练资料学下来。

一条可执行的排查顺序


现象优先判断下一步动作


training loss 高model bias 或 optimization先比较简单模型和复杂模型，再看优化设置
training 低，validation 高overfitting做 augmentation、regularization、early stopping，或重切验证集
public 好，private 差leaderboard 过拟合回到 validation，减少对 public 的反复试错

# 资料怎么切 {#split}
<!-- tag: data split -->

训练资料通常有 `(x, y)`。`x` 是输入，`y` 是答案。测试资料通常只有 `x`，没有公开答案。于是训练、验证、测试这三份资料的角色就不能混。

## box info
### 这三份资料分别做什么
- `training set`：更新参数。
- `validation set`：挑模型、调超参数、做 early stopping。
- `testing set`：最终评估，不拿来反复试错。

## box warn
### public / private 的额外一层
竞赛或作业里的 testing set 往往又分成 public 和 private。public 分数先给你看，private 才是最后更稳的结果。public 不是免费验证集，反复按它挑模型，也是在训练它。

| 资料 | 包含什么 | 用途 | 常见误用 |
| --- | --- | --- | --- |
| training set | x, y | 训练参数 | 拿它证明泛化 |
| validation set | x, y | 选模型和超参数 | 无限次调到过拟合 |
| public test | 通常只有 x | 粗看提交表现 | 反复拿来挑模型 |
| private test | 隐藏答案 | 最终评估 | 训练或调参时使用 |

# 诊断树 {#diagnosis}
<!-- tag: decision tree -->

如果结果不好，先问 training loss。这个顺序很重要，因为它能立刻把问题拆成两类：模型连训练集都学不好，还是模型在训练集上很好、在新资料上不好。

根据现象选下一步

training 高
training 低，validation 高
public 好，private 差
两者都低


**training loss 高**
模型连训练资料都没学好。先区分 model bias 和 optimization，不要急着说 overfitting。
下一步：比较简单模型和复杂模型的 training loss，再看优化设置。

| 现象 | 更像什么 | 原因 | 动作 |
| --- | --- | --- | --- |
| training loss 高 | model bias 或 optimization | 模型太弱，或者参数没被找到 | 先比较不同模型，再调训练设置 |
| training 低，validation 高 | overfitting | 模型记住了训练细节 | augmentation、regularization、early stopping |
| public 好，private 差 | public 过拟合 | 反复看 public 分数，模型贴到了 public 集合 | 回到 validation，减少试错次数 |

# 读日志 {#logs}
<!-- tag: logs -->

结果不好时，日志比感觉可靠。先看 training loss 有没有持续下降，再看 validation loss 和 public score 有没有开始分叉。日志不是装饰，它是你判断“问题在哪一层”的第一现场。

## box soft
### 先盯三条线
- `train_loss`：模型有没有把见过的资料学下来。
- `valid_loss`：模型选择是不是开始失真。
- `public_score`：有没有把 leaderboard 当成了目标。

## box info
### 读错会怎样
只盯 training，会漏掉 overfitting；只盯 public，会把 public 试成训练信号；只盯单个 epoch 的波动，又会把正常噪声当成问题。日志要读趋势，不是读情绪。

训练日志样例复制

```text
epoch 01: train_loss=1.42 valid_loss=1.45 public=1.38
epoch 05: train_loss=0.88 valid_loss=1.02 public=0.97
epoch 10: train_loss=0.41 valid_loss=0.97 public=0.94
epoch 15: train_loss=0.19 valid_loss=1.24 public=0.88
```

<!-- interactive: ['terminal-lab'] → keep as HTML -->

$$
 gap = validation\_loss - training\_loss
 $$
 gap 持续变大，通常表示模型开始记细节了；public 和 private 的差值变大，则表示你可能把 public 试成了选择器。

| 读什么 | 读之前 | 读之后 | 出错时的症状 | 下一步 |
| --- | --- | --- | --- | --- |
| train_loss 曲线 | 只看到一个最终分数 | 知道模型有没有学会 | 训练还高却误判成过拟合 | 先查 bias / optimization |
| valid_loss 曲线 | 只有训练结果 | 知道何时开始分叉 | 把早停点选晚了 | 检查 overfitting 或切分 |
| public / private | 只有 leaderboard | 知道是不是试错过头 | public 好但 private 差 | 回到 validation |

# Training 高时，先分 bias 和 optimization {#biasopt}
<!-- tag: bias vs optimization -->

training loss 高，不代表一件事。第一类原因是 model bias：模型本身就太弱，函数集合里没有足够好的解。第二类原因是 optimization failure：模型其实能表示更好的解，但训练没有把它找到。

## box soft
### model bias
模型族太受限。就像拿一把很钝的刀去切复杂形状，不管切多久，都切不出目标边界。解决方向通常是改模型、加 feature、换更合适的结构。

## box info
### optimization failure
模型够强，但训练方法没把好解找出来。可能是学习率、初始化、结构、梯度传播或训练技巧的问题。更强的模型却训练得更差，这时优先怀疑优化，而不是先怀疑模型不够大。

$$\theta^*=\arg\min_{\theta} L(\theta)$$
 这条式子只是在说“找让 loss 最小的参数”。如果可选的模型族太窄，就会卡在 model bias；如果模型族够宽却没找到好点，就会落在 optimization failure。

| 比较项 | model bias | optimization failure |
| --- | --- | --- |
| 问题本质 | 函数集合太受限 | 函数集合里有好解，但没找到 |
| 常见现象 | 简单模型和复杂模型都学不好 | 更强模型反而更难训好 |
| 下一步 | 改模型、改特征、改任务定义 | 调学习率、初始化、优化器和结构 |

# 训练低、新资料高，才叫 overfitting {#overfit}
<!-- tag: overfitting -->

Overfitting 的关键训练集和新资料表现分裂。模型在训练资料上已经足够低 loss，但对 validation 或 testing 的表现开始变差。这表示它学到了训练中的偶然细节，没抓住能迁移的规律。

复杂度上去后，训练和验证会分叉


training loss
validation / testing loss


**判断句**
如果 training 也高，就先回到 bias / optimization。只有 training 已经低了，新资料才差，才开始认真谈 overfitting。

欠拟合
训练和新资料都差。常见原因是模型太简单、feature 不够，或者还没训好。


过拟合
训练好，新资料差。常见原因是模型太复杂、资料太少，或验证方式不稳。


合适
训练和新资料都不错，但仍要检查边界样本和分布变化，不要只看平均分。


---

# 遇到 overfitting，先想两条路 {#fix}
<!-- tag: fixes -->

第一条路是增加有效资料。很多任务里不一定能额外收集新数据，但可以做 data augmentation。第二条路是限制模型弹性，也就是 regularization 方向：让模型别把训练里的偶然细节都记下来。

## box soft
### data augmentation
data augmentation 要做任务允许的变换。影像里水平翻转、轻微裁剪、合理缩放常常可行，但前提是变换后 label 仍然成立。

## box warn
### regularization
包括 early stopping、weight decay、dropout、减少无用 feature 或改更合适的模型结构。限制太少会过拟合，限制太多又会回到 model bias。

| 方法 | 作用 | 适合什么现象 | 注意点 |
| --- | --- | --- | --- |
| data augmentation | 增加合理变体 | training 好、新资料差 | 变换后 label 必须还对 |
| early stopping | 在 validation 变差前停下 | 训练继续降、验证开始升 | 需要可靠 validation set |
| 限制模型 | 降低可钻空子的空间 | 模型太容易记住细节 | 别把关键特征也删掉 |


---

# Validation 的位置 {#validation}
<!-- tag: validation -->

最稳的模型选择方式，是从训练资料里切出 validation set。训练只看 training set，模型选择只看 validation set，最后再去碰 testing set。这样你才知道结果到底是模型真的更好，还是你刚好把某个测试集试顺了。

## box info
### 为什么不能直接拿 public 分数挑模型
因为你每看一次 public 分数，就在把它变成选择依据。次数一多，public 就不再像真正的新资料，最后很容易把模型调到 public 上，而不是调到真实目标上。

## box soft
### 更稳的做法
把 public 当反馈，把 validation 当主要依据，把 private 当最终检验。这样你在调参时不会把 leaderboard 当验证集用。

## box info
### 怎么选 early stopping 的点
看 validation loss 的最低点，不看 training loss 的最低点。训练集继续变好并不稀奇，关键是验证集什么时候开始转坏。只要 validation 开始连续上升，就该考虑停下，而不是继续期待它自己会回头。

validation 切分示例复制

```python
from sklearn.model_selection import train_test_split

X_train, X_valid, y_train, y_valid = train_test_split(
 X, y, test_size=0.1, random_state=42
)

# 训练只用 training set
model.fit(X_train, y_train)

# 选模型时看 validation loss
valid_score = evaluate(model, X_valid, y_valid)
```

# Leaderboard 为什么会骗人 {#leaderboard}
<!-- tag: public/private -->

如果你提交很多次，总会有某个模型碰巧在 public set 上看起来很好。这个模型未必真学到了规律，可能只是撞上了 public set 的偶然性。public 第一不等于 private 第一，更不等于真实部署最好。

尝试次数越多，碰巧挑中的概率越高
随机提交次数：100
碰巧好分数风险39%


**观察提示**
尝试越多，越容易选中一个只是在 public set 上碰巧好的模型。
public leaderboard 是反馈，不是免费验证集。

| 做法 | 看起来像什么 | 实际风险 |
| --- | --- | --- |
| 反复提交后只挑 public 最好的 | 积极调参 | 把模型调到 public 的偶然性上 |
| 只看一次 public 就定模型 | 省事 | 可能错过更稳的 validation 依据 |
| 用 validation 为主，public 为辅 | 慢一点 | 更稳地接近 private 和真实部署 |

# 代码与判断 {#code}
<!-- tag: code reading -->

下面这段代码把诊断顺序写成一个简单判断器。它的作用是提醒你：训练集、验证集和 public 分数应该分开读，别把它们混成一个信号。

诊断函数复制

```python
def diagnose(train_loss, valid_loss, simple_train_loss, public_score=None):
 if train_loss > 1.0:
 if train_loss > simple_train_loss:
 return "复杂模型训练 loss 比简单模型还高：先查 optimization。"
 return "training loss 高：先查 model bias 或 feature 是否不足。"

 if valid_loss > 1.0:
 return "training 低但 validation 高：优先查 overfitting。"

 if public_score is not None and abs(public_score - valid_loss) > 0.5:
 return "public 和 validation 差异大：检查切分和分布。"

 return "当前结果较稳，继续做错误分析和边界检查。"
```

反例代码复制

```python
# 错误：把 public leaderboard 当成主要模型选择工具
best_model = min(models, key=lambda m: public_leaderboard_score(m))

# 更稳：先按 validation score 选择，public 只做有限参考
best_model = min(models, key=lambda m: validation_score(m))
```

# 常见误区 {#mistakes}
<!-- tag: mistakes -->

把所有坏结果都叫 overfitting
training 也差时，先怀疑 bias 或 optimization，不要直接跳到过拟合。


把 public 当验证集
反复看 public 分数，最后会把模型调到 public 的偶然性上。


把 augmentation 当万能按钮
变换后 label 不对，或者分布和真实任务差太多，augmentation 反而会把模型带偏。

# 三步自检 {#audit}
<!-- tag: audit -->

每次改完一个模型，先把结果写成三句话：training 怎么变，validation 怎么变，public/private 有没有明显分叉。这样你不会把一个实验做成情绪判断，也不会把一个提交当成结论。

## box info
### 自检顺序
1. 看 training 是否真的下降。
2. 看 validation 有没有在某个点开始转坏。
3. 看 public 和 private 的差异是不是已经大到不能继续忽略。

## box warn
### 失败时怎么收尾
如果 training 和 validation 都差，先回到 bias / optimization；如果 validation 开始变坏，就停在最低点附近；如果 public 好 private 差，就减掉无效提交，回到自己的 validation 依据。

| 看到什么 | 说明 | 下一步 |
| --- | --- | --- |
| train 下降，valid 也下降 | 模型还在学习，暂时不用急着停 | 继续跑，盯住是否开始分叉 |
| train 下降，valid 上升 | 过拟合开始冒头 | early stopping、regularization、augmentation |
| public 好，private 差 | 你把 public 试得太熟了 | 回到 validation，减少提交次数 |

## box soft
### 练习接法
你可以把这张表当成每次实验的收尾模板。只要能把结果写进这三格，后面的第 18 题就不会只是空喊术语。

# 一个完整例子：日志怎么把问题看出来 {#example}
<!-- tag: worked case -->

假设你在做一个分类任务，前 5 个 epoch 里 train_loss 从 1.42 掉到 0.88，valid_loss 也跟着从 1.45 掉到 1.02。这个阶段说明模型至少开始学会了，training 和 validation 还在一起往下走，问题不算大。到了第 10 个 epoch，train_loss 已经到 0.41，但 valid_loss 还停在 0.97；第 15 个 epoch，train_loss 继续到 0.19，valid_loss 反而升到 1.24。这个分叉就是典型信号。

## box soft
### 怎么读
1. 前半段两条线一起下降，说明模型在学习。
2. 后半段 training 继续降、validation 开始升，说明开始记训练细节。
3. 这时不要再说 optimization 没做好，先把 overfitting 当首要假设。

## box warn
### 错了会怎样
如果你把这个分叉误判成 bias 或 optimization，就会继续加大模型或乱调学习率，结果只会让训练集更好看、验证集更难看。正确动作应该是回到 validation，做 early stopping、regularization，或者补 augmentation。

$$
 train\_loss \downarrow,\quad valid\_loss \uparrow
 $$
 这组方向相反的箭头，就是过拟合最常见的外观。它意味着两条曲线已经分叉。

## box info
### 练习接法
后面的第 18 题会要你把这条日志读法讲出来。能把这个例子说顺，才算真的会用 P03 的诊断顺序。

### 10 short-answer

**Q:** 10. 为什么 data augmentation 不能乱做？

Explain: 因为变换后 label 可能不再正确，或者样本已经偏离真实任务分布。augmentation 的前提是变换合理，且仍保留任务意义。

### 11 checklist

**Q:** 11. 列出 3 个处理 overfitting 的动作，并说明各自作用。

Explain: 可以写 data augmentation、early stopping、weight decay、dropout、减少无用 feature、改更合适的结构。说明时要写清“为什么能降低过拟合”。

### 12 scenario

**Q:** 12. 如果限制模型太多，训练和测试都变差，可能回到什么问题？

Explain: 可能回到 model bias。限制过度后，模型本身就不够表达目标函数。

### 13 error-diagnosis

**Q:** 13. “我把 100 个模型都提交，选 public 分数最好的。”错在哪里？

Explain: 错在把 public 当成调参目标。提交次数越多，越容易选中只是在 public 上碰巧好的模型，最后 private 可能掉分。

### 14 short-answer

**Q:** 14. public testing set 和 private testing set 为什么要分开？

Explain: 为了让一部分测试结果先给反馈，另一部分保留为更稳的最终检验，避免你把整份测试集都调成训练信号。

### 15 error-diagnosis

**Q:** 15. 有了 validation set 后，是否可以无限次按 validation 调模型？为什么？

Explain: 不行。validation 调太多次也会过拟合 validation。它还是应该是选择依据，不是无限次试错区。

### 16 workflow

**Q:** 16. 设计一个合理的模型选择流程。

Explain: 训练集训练参数，validation 选择模型和超参数，最后再用 testing 或提交系统做最终评估。public 只做有限参考，不反复试到顺手。

### 17 short-answer

**Q:** 17. 用一句话解释：为什么不能只看测试分数调参？

Explain: 因为测试分数会随着你的反复试错变成选择依据，最后它不再代表真正没见过的数据。

### 18 synthesis

**Q:** 18. 用 6-9 句话讲清楚：机器学习任务结果不好时怎么排查？

Explain: 先看 training loss。training 高时，先分 model bias 和 optimization；training 低时，再看 validation 是否变差。如果 validation 差，才认真谈 overfitting，并考虑 augmentation、regularization 或 early stopping。public 分数只做有限参考，不能反复拿来调参。最后把模型选择放回 validation，testing 只留给最终检验。
