P05｜批次与动量

P04 讨论了 loss 停住时，local minima 和 saddle point 的区别。P05 换一个角度：即使我们不直接计算 Hessian，训练过程本身也可以借助一些机制更不容易卡住。第一个机制是 mini-batch，它让每次更新只看一部分资料；第二个机制是 momentum，它让每次更新不只听当前 gradient，也保留前一步的移动方向。

本节的概念地图

map

神经网络训练不是每次都把全部训练资料拿出来算一次完整 loss。更常见的做法是先把资料分成若干个小批次，每次用一个小批次计算 loss 和 gradient，然后更新参数。这个小批次通常叫 mini-batch，课程里简称 batch。

Batch size 不是一个纯粹的工程参数。小 batch 带来更多 update 和更 noisy 的 gradient；大 batch 带来更稳定的 gradient 和更好的并行效率。真正的训练效果来自这些因素的合力，所以 batch size 是需要调的 hyperparameter。

直觉每次只抽一小包资料来估计整体下降方向，像用局部样本给参数指路。

正式说法Mini-batch SGD 用一个 batch 的 empirical loss 近似全训练集 loss 的 gradient。

关键边界Batch 的噪声不只是误差，它有时会让优化跳出不好的停滞区域。

公开速读与即时自测

quick

核心直觉Mini-batch 是在“方向稳定”和“更新频繁”之间折中；momentum 则让当前更新不只听这一批资料的 gradient。

术语速查batch 是一次算 loss 和 gradient 的样本包；epoch 是训练集大致看完一遍；velocity 保存 momentum 的过去方向。

易混点大 batch 不一定单步慢，但一个 epoch 的更新次数会变少；小 batch 的噪声可能帮助优化，也可能让曲线更抖。

Batch、epoch、shuffle：训练循环到底在做什么

mini-batch

假设训练集有 60,000 笔资料，batch size 设为 1,000。模型不会一次看完 60,000 笔再更新，而是先拿 1,000 笔算一个 batch loss，再算 gradient，更新一次参数；接着拿下一批 1,000 笔，再更新一次。60 个 batch 都用过后，才算完成一个 epoch。

Shuffle 则发生在分 batch 之前。常见做法是在每个 epoch 开始时重新打乱训练资料，再切成 batch。这样第一轮同一个 batch 里的样本，下一轮不一定还在一起。这个小动作能降低固定样本顺序带来的偏差，让训练路径更接近随机抽样。

效率反直觉：大 batch 单次更新不一定慢

parallelism

直觉上，batch size 从 1 变成 1,000，好像计算量应该增加 1,000 倍。这个直觉没有考虑 GPU 并行。对许多矩阵运算来说，GPU 可以同时处理大量样本，所以 1,000 笔资料一起算 loss 和 gradient，未必比 1 笔资料慢 1,000 倍。

因此要分清两个时间尺度。第一个是“更新一次参数要多久”，大 batch 在 GPU 上不一定吃亏。第二个是“跑完一个 epoch 要多久”，这取决于一个 epoch 需要多少次 update。batch size 小，update 次数多；batch size 大，update 次数少，所以一个 epoch 反而可能更快。

完整 epoch

还要乘上 update 次数。小 batch 的每次更新便宜，但次数可能非常多。

$$

epoch 时间 ≈ 每次 update 时间 × ceil(N / B)

B 增大时，每次 update 时间可能只缓慢增加，但 update 次数会明显减少。

$$

Noisy gradient：噪声为什么可能帮助 optimization

optimization

Full batch 用全部训练资料计算 gradient，方向更稳定，但它每次都沿着同一个全局训练 loss 更新。如果这个 loss surface 在某个 critical point 让 gradient 变成零，普通 gradient descent 就可能停住。

这就是 noisy gradient 的一个重要价值：它不是单纯“方向不准”，也可能是一种扰动。这个扰动能让训练路径不那么死板，有机会离开某些停滞点或窄小区域。课程里强调，实际结果常常小 batch 的 noisy update 在 optimization 上反而可能占优势。

小 batch 与泛化：sharp minima 和 flat minima 的直觉

generalization

Batch size 影响的不只是 training。即使通过调 learning rate 等方式，让大 batch 和小 batch 都达到相近的 training accuracy，testing 表现也可能不同。课程提到的大量实验中，小 batch 往往更容易得到较好的 testing 结果。

为什么小 batch 可能偏向 flat minima？因为 noisy update 让参数路径更容易从窄小峡谷里晃出去。一个很窄的 sharp minima 不容易困住带噪声的路径；更宽的 flat minima 才更容易让它停下来。这个解释不是绝对定理，但它提供了理解泛化差异的实用直觉。

Momentum：不只听当前 gradient，也保留过去方向

momentum

Vanilla gradient descent 每一步只看当前位置的 gradient，然后往反方向走。Momentum 加了一份“过去移动方向”的记忆：当前更新方向由当前 negative gradient 和上一轮 velocity 共同决定。这里的 velocity 优化算法保存的一组状态变量。

Momentum 还有一个需要小心的地方：它不是无条件越大越好。保留太多过去方向，可能在该转弯的时候冲过头；保留太少，则和普通 SGD 差别不大。所以 momentum 系数也是 hyperparameter，常见记号是 β 或 λ。

最小代码：mini-batch SGD 与 momentum update

code

下面的代码把训练循环的语义拆开。重点看三个位置：每个 epoch 先 shuffle；每个 batch 算一次 loss 和 gradient；momentum 版本额外保存 velocity，并把它加入参数更新。

代码
for epoch in range(num_epochs):
 indices = shuffle_indices(len(train_data)) # 每个 epoch 重新打乱
 for batch_ids in split(indices, batch_size):
 x, y = train_data[batch_ids], train_label[batch_ids]
 loss = model.loss(x, y) # 只看这个 batch 的 loss
 grad = compute_gradient(loss, model.params)
 model.params = model.params - lr * grad # 更新一次参数

这段代码里的 loss 当前 batch 的 loss。正因为每个 batch 不同，gradient 方向会有随机性。这个随机性可能让路径更曲折，也可能让优化更有弹性。

正确示例：加入 momentum 的更新骨架python

velocity 的初始值通常是 0。第一步时 momentum 还没有历史方向，更新和普通 SGD 很像；从第二步开始，上一轮移动方向会进入当前更新。把式子展开，会发现当前更新含有过去多个 gradient 的加权影响。

代码
# 错：这只是把步长放大，没有保存过去方向
model.params = model.params - (lr * beta) * grad

错误示例的问题在于没有状态变量。Momentum 的核心是保存并更新 velocity；如果每一步只把 lr 乘上某个常数，就没有“过去方向”的记忆，也无法体现穿过平坦区或短暂停滞点的作用。

常见误区和修法

pitfalls
误区错因后果修法
batch 只是为了省内存只看到一次不装全部资料的工程侧。忽略 batch size 对更新次数、噪声和泛化的影响。把它放回 optimization 过程理解。
batch size 越大越好以为 gradient 越稳定就一定训练越好。可能得到较差 training 或较差 testing。把 batch size 当 hyperparameter 调。
小 batch 一定更快只看单次更新，不看完整 epoch。低估小 batch 的大量 update 成本。同时比较单次 update 时间和 epoch 更新次数。
noisy gradient 只是坏事把噪声等同于错误。无法解释小 batch 训练和泛化优势。记住噪声有时能帮助离开停滞或窄谷。
training 好就代表 testing 一定好忽略 sharp / flat minima 的差异。在相同 training accuracy 下仍可能泛化差。关注地形宽度和 training/testing mismatch。
momentum 就是调大学习率没理解 velocity 状态变量。代码写不出真正的历史方向累积。保留 velocity，并用 β 控制过去方向影响。
