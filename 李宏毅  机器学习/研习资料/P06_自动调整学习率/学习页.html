本节的概念地图

map

训练卡住时，很多人第一反应是“卡在 local minima 或 saddle point”。但这个判断需要证据。Critical point 的特征是 gradient 很小；如果 loss 不降但 gradient norm 仍然很大，那就说明问题更新方向和步伐让 loss 没能有效下降。

这条线最后会汇合到常用 optimizer。AdaGrad 用过去所有 gradient 的 root mean square 作为分母；RMSProp 让最近 gradient 更有影响力；Adam 可以粗略理解为 RMSProp 加上 momentum；learning rate schedule 则让整体 learning rate 随训练时间变化。

直觉同一双鞋不适合所有地形；同一个 learning rate 也不适合所有参数方向。

正式说法Adaptive optimizer 根据历史 gradient 统计量，对每个参数的更新量做归一化。

关键边界自动调整不等于不用调参；初始 learning rate、α、β、schedule 仍是 hyperparameter。

公开速读与即时自测

quick

核心直觉固定 learning rate 像统一鞋码：陡峭方向需要小步，平坦方向需要大步，同一尺度很难两边都合适。

术语速查gradient norm 判断是否真没方向；σ 表示历史 gradient 尺度；schedule 控制整体 learning rate 随时间变化。

易混点Adaptive optimizer 不等于不用调参。Adam、RMSProp 仍需要初始 learning rate、β 或 α，以及可能的 decay / warm up。

Loss 不降，不一定是 critical point

gradient

Critical point 的定义来自 gradient：当 gradient 接近零，参数更新量也会接近零。可是训练曲线上的 loss 不再下降，只是一个表面现象。它可能来自 gradient 很小，也可能来自更新方向在地形里来回震荡，还可能来自 learning rate 让每一步都越过合适的位置。

这也是本节与 P04 的衔接。P04 教你 gradient 为零后如何分类 critical point；P06 提醒你，很多训练停滞甚至还没到 gradient 为零那一步。真正的障碍可能在更前面：优化器没有把参数带到一个可以顺利下降的路径上。

把“loss 停住”拆成两个问题：loss 现象是什么，gradient norm 证据是什么。

狭长谷地：固定 learning rate 的两难

ravine

想象一个只有两个参数的 error surface。它是凸的，理论上没有 local minima 陷阱，最低点清清楚楚就在谷底。但这个谷地很狭长：横向很平，纵向很陡。这样的地形会让普通 gradient descent 很痛苦。

这说明问题不是“只要把 learning rate 调小就好”。一个方向需要小步防止震荡，另一个方向需要大步加速前进。单一 learning rate 同时服务所有参数方向，本来就不够细。

minimum

解释为什么一个固定 learning rate 无法同时满足陡峭方向和平坦方向。

Adaptive learning rate：给每个参数自己的步伐

η

普通 gradient descent 对所有参数使用同一个 learning rate：θ_i ← θ_i - η g_i。Adaptive learning rate 把这个步伐改成 η / σ_i^t，也就是每个参数 i、每个时间 t 都可能有自己的分母。

注意这里的调整不是简单地“gradient 大就走大步”。恰好相反，gradient 本身已经乘在更新式里，adaptive 方法额外用 σ 去归一化它，避免陡峭方向走太猛，同时放大平坦方向的前进能力。

现在你应该能做什么

用 η / σ 解释 adaptive learning rate 的基本设计，而不是只背 optimizer 名字。

AdaGrad：用过去所有 gradient 的 RMS 当分母

AdaGrad

AdaGrad 的思路是用过去所有 gradient 的平方平均再开根号，作为 σ。这个量叫 root mean square，直译是均方根。它只看 gradient 的大小，不看正负方向，因为平方会把正负号去掉。

但 AdaGrad 也有问题。它把过去所有 gradient 都放进统计里，而且越积越多。某个方向如果长时间看到小 gradient，分母可能变得很小，导致后来在突然变陡的区域步伐过大；某些任务里也可能因为累计分母越来越大，使有效 learning rate 过早变小。

点解释训练含义
平方去掉方向，只保留大小σ 衡量尺度，不决定往左还是往右
平均把历史 gradient 汇总让每个参数有自己的历史尺度
开根号回到与 gradient 相近的单位避免平方尺度直接进入分母过大

解释 AdaGrad 的 σ 从哪里来，以及为什么它能让平坦方向走大步、陡峭方向走小步。

RMSProp 与 Adam：最近的 gradient 也要有话语权

RMSProp

AdaGrad 默认过去所有 gradient 同等重要。但同一个参数方向在训练早期和后期可能处在完全不同的地形里：前面平，后面陡；或者前面陡，后面平。此时旧统计如果影响太重，σ 的反应会变慢。

Adam 则可以作为本节的合体版理解：它一边像 momentum 那样累计 gradient 方向，用来决定往哪里走；一边像 RMSProp 那样累计 gradient 平方，用来决定步子多大。方向和尺度不是互相抵消，因为 momentum 看正负方向，RMS 分母只看大小。

Learning rate schedule：整体步伐也随时间变

decay

前面的 σ 是每个参数自己的缩放因子，η 则是整体基准 learning rate。Learning rate schedule 指的是让 η 也随着时间变化，而不是永远固定。最常见的做法是 learning rate decay：训练早期离目标远，步子可以大；训练后期接近较好区域，步子逐渐变小，避免在附近乱跳。

Warm up 在 Transformer、BERT 相关训练里很常见，也并不是新模型时代才出现的技巧。它的理论解释仍然有讨论空间，但实作上很常用。你需要掌握的是操作意图：decay 是后期刹车，warm up 是前期先稳住统计，再逐渐加速。

decaywarm up + decayconstant
Schedule 改的是整体 ηAdaptive optimizer 的 σ 是参数级缩放；schedule 是全局基准步伐随时间变化。两者可以同时存在，并不冲突。

区分 decay 和 warm up：前者是训练后期减速，后者是训练初期先小步稳定。

最小代码：从 SGD 到 Adam 的参数更新

code

下面的代码用一维参数写更新逻辑，重点读懂状态变量。grad 决定当前方向，v 保存 momentum 的方向记忆，s 保存 gradient 平方的尺度统计，lr_t 是 schedule 后的整体 learning rate。

代码
eps = 1e-8
s = 0.0

for t in range(num_steps):
 grad = compute_grad(theta)
 s = s + grad * grad # 累计过去所有梯度平方
 theta = theta - lr / (sqrt(s) + eps) * grad

代码
eps = 1e-8
v = 0.0 # 方向记忆
s = 0.0 # 尺度统计

for t in range(num_steps):
 grad = compute_grad(theta)
 lr_t = schedule(base_lr, t)

 v = beta * v + (1 - beta) * grad # 考虑方向，有正负
 s = alpha * s + (1 - alpha) * grad * grad # 只考虑大小
 theta = theta - lr_t * v / (sqrt(s) + eps)

常见误区和修法

pitfalls
误区错因后果修法
loss 不降就是 critical point没有检查 gradient norm。误判训练停滞原因。先测gradient，再判断是否接近 critical point。
learning rate 调小就一定解决震荡只看到陡峭方向，忽略平坦方向。训练可能慢到走不动。考虑不同参数方向需要不同有效步伐。
gradient 大就该走更大步混淆 gradient 本身和 adaptive 缩放。在陡峭方向更容易爆炸。用 σ 把陡峭方向的有效学习率压小。
AdaGrad 永远最好忽略它对历史 gradient 同等对待。统计反应慢，或有效学习率过早变小。理解 RMSProp 为什么加入指数移动平均。
Adam 的 momentum 和 σ 会互相抵消没区分方向和大小。无法理解 Adam 的设计。momentum 看正负方向，σ 看平方大小。
warm up 是随便的玄学只看形式，没看早期统计不稳定。遇到大模型训练时不知道为什么要小步开始。把 warm up 理解成先稳定统计，再逐步加速。
