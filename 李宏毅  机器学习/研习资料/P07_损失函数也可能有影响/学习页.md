# 本节的概念地图 {#goals}
<!-- tag: map -->

分类任务看起来只是“输出类别”，但类别不是普通数字。把 class 1、class 2、class 3 编成 1、2、3，再用回归去拟合，会偷偷加入一种距离假设：1 比 2 更接近，1 比 3 更远。对年级这种有顺序的类别可能勉强合理，但对猫、狗、车这种无序类别就很怪。

更常见的做法是 one-hot：如果有 3 类，正确类别分别写成 `[1,0,0]`、`[0,1,0]`、`[0,0,1]`。网络也输出 3 个数，这些原始分数叫 logits。Logits 可以任意大、任意小，不是概率，所以需要 softmax 把它们变成 0 到 1 之间且总和为 1 的分布。

最后才轮到 loss function。你可以用 MSE 比较预测概率和 one-hot，也可以用 cross entropy。两者在正确答案处都能最小，但训练地形不同。课程要强调的是：loss function 的选择会影响 optimization 难度，不只是评价方式不同。

直觉分类在多个互斥类别里分配概率。
正式说法Softmax 将 logits 映射到类别概率 simplex，cross entropy 衡量目标分布和预测分布的差异。
关键边界PyTorch 的 `CrossEntropyLoss` 直接吃 logits，内部包含 softmax 相关计算。

# 类别编号为什么不适合多数分类 {#label}
<!-- tag: label representation -->

回归任务输出一个数，目标也是一个数。分类任务如果硬套回归，就要把类别变成数字，比如 class 1 记作 1，class 2 记作 2，class 3 记作 3。这样做的问题是：数字之间自带距离和顺序。

如果任务是预测小学年级，一年级、二年级、三年级确实有某种顺序，编号可能还说得过去。但多数分类任务没有这种关系。把猫=1、狗=2、车=3，会暗示猫和狗比猫和车更接近；这个假设可能完全不是任务想表达的东西。

One-hot 的好处是取消这种虚假的顺序。每个类别对应一个独立维度，正确类别为 1，其余为 0。这样类别之间的距离在表示上是对称的，不会凭编号制造“谁更接近谁”。

小对比：编号 vs one-hot

编号1 2 3隐含顺序和距离。
One-hot100 010 001每类独立。
训练含义表示方式会影响模型被要求学习什么。


**类别不是天然数字**
只有当类别本来有顺序或连续含义时，编号才可能合理。
普通多类分类通常用 one-hot 或类别索引配合 cross entropy。

# 多输出网络：每个类别一个 logit {#logits}
<!-- tag: logits -->

如果 label 是三维 one-hot，网络输出也应该是三维。过去做 regression 时，最后一层可能只输出一个数；做三分类时，最后一层输出三个数。每个数可以理解为模型给某个类别的原始分数。

这些原始分数叫 logits。Logits 不是概率，可能是正数、负数，也不需要加起来等于 1。它们的作用是给 softmax 提供排序和相对强弱：某个 logit 越大，softmax 后对应类别概率越高。

为什么不直接把 logits 当概率？因为普通线性层输出没有概率约束。它可以输出 `[3, 1, -3]`，也可以输出 `[20, -7, 4]`。概率需要满足每一项在 0 到 1 之间，并且全部相加为 1。Softmax 正是把任意 logits 转成这种概率分布的函数。

$$
logits: z = [z₁, z₂, z₃]，可以是任意实数
 probability: p = softmax(z)，满足 0 ≤ pᵢ ≤ 1 且 Σ pᵢ = 1
$$

| 对象 | 范围 | 是否相加为 1 | 训练中位置 |
| --- | --- | --- | --- |
| one-hot label | 0 或 1 | 是 | 正确答案 |
| logits | 任意实数 | 不要求 | 网络原始输出 |
| softmax probability | 0 到 1 | 是 | 预测类别分布 |

# Softmax：把 logits 变成概率分布 {#softmax}
<!-- tag: softmax -->

Softmax 的步骤很直接：先对每个 logit 取指数，让它变成正数；再除以所有指数值的总和，让结果加起来等于 1。因为指数函数会放大差距，大的 logit 会得到更大的概率，小的 logit 会被压到更接近 0。

例如 logits 是 `[3, 1, -3]`。取指数后大约是 `[20.1, 2.7, 0.05]`，再归一化，概率就会接近 `[0.88, 0.12, 0.00]`。这把相对大小转成竞争关系。

二分类时，你可能更常见 sigmoid。两个类别的 softmax 和一个 logit 差值上的 sigmoid 是等价的表达方式。也就是说，二分类用 sigmoid 并softmax 在二类情形下的简化写法。

$$
softmax(z)i = exp(zᵢ)Σj exp(zⱼ)
 每个输出都是正数，所有输出相加为 1。
$$

小交互：改变第一个 logit 看概率变化
z₁**3**
p₁0.88
p₂0.12
p₃0.00


**z₁ 占优势**
当 z₁ 明显大于其他 logits，softmax 会把大部分概率分给第一个类别。
观察重点：softmax 输出的是竞争后的相对概率。

# MSE vs Cross Entropy：都能最小，但地形不同 {#loss}
<!-- tag: loss -->

有了 softmax probability 和 one-hot label，就可以定义距离。MSE 的做法是把每个维度的差平方再加总。Cross entropy 的做法是取正确类别概率的对数损失；对 one-hot label 来说，公式会简化成 `-log(p_correct)`。

如果预测概率和 one-hot 完全一样，MSE 会最小，cross entropy 也会最小。所以从“最终目标”看，两者似乎都合理。但优化关心的不只是终点，还关心从错误的初始位置出发时，gradient 是否能给出足够强的方向。

在分类问题里，cross entropy 更常用，因为它在模型非常自信但错得离谱时，会给出更强的惩罚和更有用的梯度。MSE 搭配 softmax 时，在一些错误很大的区域可能变得很平，gradient 很小，导致训练起步困难。

$$
MSE: L = Σi (pᵢ - yᵢ)²
 Cross Entropy: L = -Σi yᵢ log(pᵢ)
 若 y 是 one-hot，CE = -log(p正确类别)
$$

## box warn
### MSE 的风险
搭配 softmax 时，错得很远的区域可能出现梯度很小的问题，优化器不容易走出来。

## box soft
### Cross entropy 的优势
正确类别概率越低，损失越大，能更直接推动模型提高正确类别 logit。

# Loss function 会改变 optimization 难度 {#surface}
<!-- tag: error surface -->

课程用一个三分类例子来说明：正确答案是 `[1,0,0]`，让 `y₃` 固定为很小，只观察 `y₁` 和 `y₂` 两个 logits 的变化。如果 `y₁` 大、`y₂` 小，模型预测正确，loss 小；如果 `y₁` 小、`y₂` 大，模型错得很离谱，loss 大。

关键差异出现在错误区域。使用 cross entropy 时，错误区域仍然有明显斜率，gradient 可以带模型往正确方向走。使用 MSE 时，错误区域可能变得很平，gradient 很小。模型明明离目标很远，却得不到足够的更新信号。

这就是“loss function 也可能有影响”的含义。改变 loss function，不只是换一个评分公式，而是在改变训练看到的 error surface。一个更合适的 loss，可以让优化问题更容易；一个不合适的 loss，可能让同一个网络更难训练。

**优化角度的核心**
Cross entropy 更适合分类，不只是概率解释漂亮，也因为它让错误区域更容易产生有效梯度。
MSE 在分类上不是绝对不能用，但会让 training 更困难。

# PyTorch 里的重要细节：CrossEntropyLoss 直接吃 logits {#pytorch}
<!-- tag: PyTorch -->

在很多教材图里，分类模型写成 `network → softmax → cross entropy`。但在 PyTorch 里，`torch.nn.CrossEntropyLoss` 的常用写法是直接把 logits 传进去，不要在模型最后手动加 softmax。因为这个 loss 内部已经包含了 log-softmax 和负对数似然的稳定计算。

如果你在模型里先加 softmax，再把概率丢给 `CrossEntropyLoss`，就等于把 softmax 相关处理用错了位置，常见后果是数值不稳定、梯度变差、训练效果异常。框架这样设计，是为了把 softmax 和 cross entropy 作为一组稳定操作来实现。

还有一个细节：PyTorch 的 `CrossEntropyLoss` 默认 target 通常是类别索引，比如 `0`、`1`、`2`，而不是 one-hot 向量。它内部会按类别索引取出正确类别的 log probability。概念上仍然等价于 one-hot 的 cross entropy。

正确示例：PyTorch 分类训练常见写法python

```text
logits = model(x) # shape: [batch, num_classes]，没有手动 softmax
loss_fn = torch.nn.CrossEntropyLoss()
loss = loss_fn(logits, target) # target 是类别索引，如 0/1/2
```

错误示例：在 CrossEntropyLoss 前重复 softmaxpython

```text
prob = torch.softmax(model(x), dim=1)
loss = torch.nn.CrossEntropyLoss()(prob, target) # 错：CrossEntropyLoss 预期输入 logits
```

# 最小代码：手算 softmax 与 cross entropy {#code}
<!-- tag: code reading -->

下面的代码手动展示 softmax 和 cross entropy。真实训练时应使用框架的稳定实现，但手算版能帮你看清每一步：logits 先指数化并归一化，cross entropy 只关心正确类别的概率。

正确示例：从 logits 到概率再到 CEpython

```text
import numpy as np

logits = np.array([3.0, 1.0, -3.0])
target_class = 0

shifted = logits - logits.max() # 数值稳定：不改变 softmax 结果
exp_values = np.exp(shifted)
prob = exp_values / exp_values.sum()

loss = -np.log(prob[target_class])
print(prob, loss)
```

`shifted = logits - logits.max()` 是数值稳定技巧。Softmax 对所有 logits 同时减去同一个常数，输出概率不变，但可以避免指数爆炸。

对照示例：MSE 对概率做平方误差python

```text
one_hot = np.array([1.0, 0.0, 0.0])
mse = np.sum((prob - one_hot) ** 2)
print(mse)
```

MSE 计算的是概率向量和 one-hot 的欧氏距离；cross entropy 计算的是正确类别概率的 log penalty。两者都可以比较预测和标签，但给优化器的梯度地形不同。

# 常见误区和修法 {#mistakes}
<!-- tag: pitfalls -->

| 误区 | 错因 | 后果 | 修法 |
| --- | --- | --- | --- |
| 把类别编号直接做回归 | 忽略编号带来的顺序和距离假设。 | 让模型学习到任务没有要求的相似性。 | 无序分类用 one-hot 或类别索引配合 CE。 |
| 把 logits 当概率 | 不知道 logits 可为任意实数。 | 错误解释输出，也可能错用 loss。 | 用 softmax 后才得到概率分布。 |
| 认为 softmax 只是缩放到 0 到 1 | 忽略指数和归一化的竞争效果。 | 无法理解大 logit 如何压制其他类别。 | 记住 softmax 输出总和为 1 的相对概率。 |
| 分类用 MSE 和 CE 完全一样 | 只看终点，不看 gradient 地形。 | 可能让训练在错误区域起步困难。 | 分类优先使用 cross entropy。 |
| 在 PyTorch CrossEntropyLoss 前手动 softmax | 不了解 CE 内部包含稳定的 log-softmax。 | 训练数值和梯度可能变差。 | 模型输出 logits，直接传给 CrossEntropyLoss。 |
| 以为 likelihood 和 cross entropy 是两件相近但不同的事 | 没看到分类设定下的等价关系。 | 概念表达模糊。 | 理解最大化 likelihood 等价于最小化 cross entropy。 |
