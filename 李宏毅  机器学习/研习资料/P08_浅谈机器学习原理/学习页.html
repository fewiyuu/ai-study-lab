本节的概念地图

map

机器学习有三个基本步骤：先选一类有未知参数的函数，再定义 loss，最后做 optimization 找出参数。这个步骤你在深度学习里也见过，只是神经网络的参数很多，loss 可微，optimization 用 gradient descent。P08 用只有一个参数的阈值模型，把这些概念变得可数、可画、可推。

课程最后给出一个上界直觉：坏训练集出现的概率与训练资料数 N 下降相关，与模型可选择函数数量 |H| 上升相关。资料越多越稳，模型越复杂越容易挑到“只在训练集上刚好好”的函数。但模型太简单又会让理想性能本身变差。

直觉训练集是你抓到的一小批样本；真实世界是你没抓完的整体生态。

正式说法从 hypothesis set 里选出 empirical loss 最小的函数，希望它的 population loss 接近最优。

关键边界本节的上界是解释原理，不是实际项目里常直接拿来精算的工具。

公开速读与即时自测

quick

核心直觉训练集只是从真实世界抽到的一小份样本。模型在这小份样本上最优，不等于在整体分布上最优。

术语速查D_train 是抽样训练集；D_all 是理想整体；𝓗 是候选函数集合；|𝓗| 越大，可选择空间越大。

易混点模型复杂度太大容易挑中“只适合训练集”的函数；复杂度太小又可能连理想规律都表达不出来。

资料量 N 是否足够？越大越能降低抽样偶然性。

候选集合 |𝓗| 是否过大？越大越容易挑中偶然好函数。

模型是否太简单？太小的 𝓗 会带来 underfitting。

训练集是否代表未来资料？IID 假设不稳时，上界直觉也会失效。

案例：用边线复杂度分宝可梦和数码宝贝

threshold

宝可梦和数码宝贝有些角色很像，但画风也有明显差异。课程观察到：数码宝贝常常线条更复杂、细节更多；宝可梦通常更圆润、线条更简单。于是我们可以用 edge detection 把图片边线抽出来，再数白色像素数量，作为“线条复杂度”。

这个模型非常简单，未知参数只有一个 H。但它已经包含机器学习的核心：我们不知道 H 应该设多少，需要从资料中学出来。所有可能的 H 组成 hypothesis set，记作大写 𝓗。

机器学习三步：function、loss、optimization

three

第一步是定义带未知参数的 function。这里的 function family 是所有阈值分类器 f_H，参数是 H。所有可能参数组成 𝓗，|𝓗| 表示可选择函数的数量，也就是模型复杂度的一种度量。

第三步是 optimization。因为 H 只有 1 到 10000 共一万个可能，我们不需要 gradient descent，直接枚举所有 H，计算训练集错误率，选出错误率最低的 H。如果是神经网络，参数空间巨大且连续，才需要可微 loss 和 gradient descent。

步骤本例对应深度学习对应
Function阈值分类器 f_H神经网络结构和参数
Loss错误率，错 1 对 0cross entropy、MSE 等
Optimization枚举 10000 个 Hgradient descent / Adam

现在你应该能做什么

把一个简单规则模型也拆成机器学习三步，而不是只把机器学习等同于神经网络。

理想与现实：D_all、D_train、h_all、h_train

generalization

理想情况下，我们拥有全宇宙所有宝可梦和数码宝贝，记作 D_all。如果能在 D_all 上找出错误率最低的阈值，这个阈值就是 h_all，也就是在当前 hypothesis set 里真正最好的选择。

这就是泛化问题。训练集 loss 低只是经验表现好；population loss 低才是真实世界表现好。Overfitting 的典型情况是 L(h_train, D_train) 很低，但 L(h_train, D_all) 很高。

好抽样

D_train 分布像 D_all，选出的 h_train 在整体上也接近理想。

坏抽样

D_train 刚好偏掉，h_train 只适合训练集，在整体上表现差。

现在你应该能做什么

解释为什么训练错误率可能低于理想全集错误率，但这不代表模型真的更好。

什么叫“好的训练集”

representative

一个训练集好，说它对所有候选函数都足够有代表性。更正式地说，对每一个 H，在 D_train 上算出的 loss 和在 D_all 上算出的 loss 都接近。

课程给出的推导结论是：如果训练集对所有 H 都满足 |L(H,D_train)-L(H,D_all)| ≤ ε，那么用训练集选出的 h_train 在真实世界上的表现，就不会比理想 h_all 差太多。

坏训练集是指至少存在一个 H，让训练 loss 和真实 loss 差很多。

坏训练集概率上界：Hoeffding + union bound

bound

先固定一个候选函数 H。对这个固定 H 来说，L(H,D_train) 是从真实世界样本 loss 里抽一部分求平均，L(H,D_all) 是整体平均。Hoeffding inequality 告诉我们：如果每笔 loss 在 0 到 1 之间，训练样本数 N 越大，样本平均偏离整体平均很多的概率越小。

于是得到一个典型上界：坏训练集概率小于等于 |𝓗| × 2 exp(-2Nε²)。这个式子常常很松，实际算出来可能大于 1，项目里不一定可直接使用；但它清楚揭示了两个方向：增加资料数 N 会降低风险，增大模型复杂度 |𝓗| 会增加风险。

N 与 |𝓗| 的两难：过拟合和欠拟合同时存在

tradeoff

增加训练资料 N 通常有帮助，因为训练集更可能代表真实分布。问题是资料有成本，很多任务不能随便多收。课程里用很生活的方式说：你可能想先花钱收资料，但现实里预算、标注、业务约束都会卡住。

这就是机器学习经典两难：模型太复杂，现实和理想容易差很远，容易 overfitting；模型太简单，理想本身就坏，容易 underfitting。深度学习后面要讨论的神奇之处，就是它看起来在某些情况下能用很复杂的模型，同时仍取得不错泛化。

解释“参数越多越可能 overfitting”背后的理论直觉，同时说明模型太简单也会出问题。

最小代码：枚举阈值并比较 train/test error

code

这段代码模拟本节的阈值分类器。它先在训练集上枚举所有阈值，找出训练错误率最低的 h_train，再把这个阈值拿到测试集上评估。你要抓住两件事：optimization 用的是训练集，真正关心的是未见资料上的表现。

代码
import numpy as np

def predict(edge_count, threshold):
 return (edge_count > threshold).astype(int) # 1: 数码宝贝, 0: 宝可梦

def error_rate(edge_count, label, threshold):
 pred = predict(edge_count, threshold)
 return np.mean(pred != label)

thresholds = np.arange(1, 10001)
train_losses = [error_rate(train_edge, train_label, h) for h in thresholds]
h_train = thresholds[np.argmin(train_losses)]

train_error = error_rate(train_edge, train_label, h_train)
test_error = error_rate(test_edge, test_label, h_train)

常见误区和修法

pitfalls
误区错因后果修法
训练错误率低就代表模型好混淆 training loss 和 population loss。忽略 overfitting。始终评估未见资料或测试集。
h_all 在任何资料集上都最强忘记 h_all 只是在 D_all 上最优。无法理解训练 loss 可能低于理想全集 loss。明确 loss 的第二个输入是资料集。
一个好训练集只要让某个模型表现好忽略训练过程会挑 H。可能被某个偏差 H 误导。好训练集要对所有 H 都有代表性。
上界算出来大于 1 就说明推导错了不了解 union bound 是松上界。误解理论用途。把它当解释原理，不当精准实用估计。
模型复杂度越小越好只看泛化风险，不看表达能力。理想模型本身变差，欠拟合。在表达力和泛化之间取平衡。
连续参数让模型复杂度一定无限大且无从讨论忽略有限精度和 VC dimension 等复杂度概念。以为理论完全不能说神经网络。理解H是离散例子的入口，连续模型需更细复杂度度量。
