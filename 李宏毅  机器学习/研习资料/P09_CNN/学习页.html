卷积神经网络：把图像特性写进网络架构

P08 讲模型复杂度和泛化。P09 开始讨论 network architecture：同样是分类，为什么处理图像时不直接把图片拉直丢进 fully connected network？答案不是“CNN 比较高级”，而是图像有局部 pattern、同一 pattern 可出现在不同位置、适当下采样仍可能保留语义。

CNN 用三种设计回应这些观察：receptive field 让 neuron 只看局部；parameter sharing 让同一个 filter 扫过不同位置；pooling 或下采样减少空间大小和运算量。它牺牲一部分弹性，换来更适合图像任务的模型偏置和更少参数。

本节的概念地图

map

图片对电脑来说是一个三维 tensor：高、宽、channel。彩色图片通常有 RGB 三个 channel。把图片直接拉成向量再接全连接层当然可行，但参数量会非常大。例如 100×100×3 的输入接 1000 个 neuron，第一层就有 3000 万个 weight。

CNN 的核心是减少不必要的自由度。图像里的小 pattern 通常只需要局部范围就能侦测，所以 neuron 不必看整张图；同一个 pattern 可能出现在图像任何位置，所以不同位置可以共用一组参数；图片缩小一点通常仍能看出物体，所以可以用 pooling 减少空间大小。

这不是无条件的魔法。它是针对图像任务设计的 inductive bias，也就是模型预先带入的结构假设。用在围棋、语音、文字等任务时，必须确认这些任务是否也有类似局部性和共享结构。

image tensor→ local pattern→ receptive field→ parameter sharing→ filter / convolution→ feature map

直觉

一个鸟嘴 pattern 不需要看整张图片；同一个鸟嘴 detector 可以在不同位置重复使用。

正式说法

Convolutional layer 使用局部连接与权重共享，产生多个 feature map。

关键边界

CNN 对缩放、旋转不天然不变，常需 data augmentation 或专门架构。

图片输入：tensor、channel 和参数爆炸

image tensor

一张 100×100 彩色图片，不是一个抽象图案，而是 100×100×3 个数字。前两个维度是高和宽，第三个维度是 channel。RGB 图片的三个 channel 分别记录红、绿、蓝强度。

到目前为止学过的普通 network 接收向量，所以最直接做法是 flatten，把 100×100×3 个数字排成长度 30000 的向量。这个向量可以接 fully connected layer，但第一层如果有 1000 个 neuron，就需要 30000×1000 = 30000000 个 weight，还不算 bias。

参数很多意味着模型弹性强，但也意味着训练成本高、overfitting 风险高。P08 讲过，复杂度越大，越可能找到只适合训练集的函数。CNN 的设计目的之一，就是在不破坏图像任务关键能力的前提下，减少不必要的连接和参数。

输入长度 = H × W × C
fully connected 第一层参数量 ≈ H × W × C × neuron 数

局部感受野：pattern 通常只需要看一小块

receptive field

图像识别不是每个 neuron 都必须看整张图。很多有用特征是局部的：鸟嘴、眼睛、边缘、角点、纹理。要判断一小块区域有没有这些 pattern，通常只需要看附近像素。

Receptive field 就是一个 neuron 能看到的输入范围。如果 kernel size 是 3×3，且输入是 RGB，那么一个局部窗口实际包含 3×3×3=27 个数。这个 neuron 只对这 27 个数有 weight，而不是对整张图所有像素都有 weight。

Receptive field 可以重叠。Stride 决定窗口移动几格；stride 小，窗口重叠多，不容易漏掉刚好出现在边界上的 pattern。Padding 则是在边缘补值，常见是补 0，让边缘区域也能被窗口覆盖。

术语直觉常见设置注意点
kernel size局部窗口大小3×3 很常见深层叠加后能看到更大范围
stride窗口每次移动几格1 或 2 常见太大可能漏掉边界 pattern
padding边缘补值zero padding 常见用于保留边缘信息或控制输出大小

参数共享：同一个 pattern detector 可以到处用

parameter sharing

如果“鸟嘴”可以出现在左上角，也可以出现在图片中间，那么每个位置都重新学习一套鸟嘴 detector 是浪费的。它们要做的事情相同，只是看的位置不同。

Parameter sharing 的做法是：不同位置的 neuron 使用同一组 weights。因为输入区域不同，它们输出不会完全一样；但它们检测的是同一种局部 pattern。这组共享参数就是一个 filter，也常叫 kernel。

共享参数进一步降低了 network 的弹性。Fully connected layer 可以为每个位置学不同参数；CNN 强迫同一个 filter 在所有位置复用。这种限制就是 bias。Bias 不一定坏，只要它符合任务结构，就能减少过拟合并提高效率。

同权重，不同输入

两个位置共享同一个 filter，但输入 patch 不同，所以输出可以不同。

同输入，同权重

如果两个 neuron 看同一块区域又共享参数，输出就会重复，没有意义。

现在你应该能做什么

用“同一个 filter 扫不同位置”解释 parameter sharing，而不是只说减少参数。

卷积与 feature map：filter 扫过图片

convolution

Convolution 的常见讲法是：一个 filter 放到图片左上角，和局部 patch 做逐元素相乘再加总，得到一个数；然后 filter 按 stride 往右、往下移动，重复同样计算。所有位置的输出排成一张小图，这就是这个 filter 的 feature map。

如果一个 convolutional layer 有 64 个 filter，就会得到 64 个 feature map。你可以把输出看作一张新图片，只是它的 channel 不再是 RGB，而是 64 个 filter 的响应。下一层 convolution 的 filter 深度必须等于前一层输出 channel 数。

为什么 3×3 filter 也能抓大 pattern？因为层数叠起来后，高层的一个 3×3 区域对应回原图时已经覆盖更大范围。第一层看小边缘，第二层组合小边缘，越深越能组合出更大的结构。

Pooling：缩小 feature map，但不是必需品

pooling

Pooling 来自第三个图像观察：图片缩小一点，通常不影响我们判断里面是什么物体。Max pooling 会把 feature map 分成小块，比如 2×2，每块取最大值作为代表。这样空间大小变小，channel 数通常不变。

Pooling 没有可学习参数，它更像一个固定 operator。常见 max pooling 取最大值，也可以有 average pooling 等版本。它的主要作用是下采样，减少运算量，并让模型对小范围位置变化更不敏感。

但 pooling 不是永远好。如果任务对精确位置非常敏感，下采样可能伤害表现。课程用围棋提醒我们：棋盘不能随便拿掉行列，否则局势变了。所以 AlphaGo 可以用 CNN 的局部性和参数共享，但原始架构里没有使用 pooling。

2×2 max pooling: [[1, 3], [2, 0]] → 3
对每个 channel 分别做 pooling，空间大小变小，channel 数不变。

适合

物体识别中，小范围平移或缩小通常不改变语义。

小心

围棋、定位、分割等任务中，精确空间信息可能非常重要。

典型 CNN 架构：conv、pool、flatten、fully connected

architecture

经典图像分类网络常见流程是：输入图片经过几层 convolution，提取局部 pattern；中间可能穿插 pooling 降低空间大小；最后 flatten，把 feature map 拉直成向量；再接 fully connected layers 和 softmax 做分类。

Flatten 的作用是把空间结构输出转换成普通向量，让后面的分类层使用。注意 flatten 本身没有参数，只是改变形状。真正可学习的是 convolution filter 和 fully connected layer 的 weights。

如果前一层有 64 个 filter，输出就有 64 个 channel；下一层 filter 的深度就必须是 64。很多初学者只记 filter 是 3×3，却忘了它实际形状是 3×3×输入 channel 数。

image→ conv→ feature maps→ pool→ flatten→ fully connected→ softmax
模块是否有参数主要作用常见误区
Convolution有用 filter 提取局部 pattern只看 3×3，忘记 channel 深度
Pooling无下采样、减少运算以为 CNN 必须有 pooling
Flatten无把 feature map 拉直以为它会学习新特征
Fully connected有整合特征做分类忽略前面 conv 已经提取空间特征

适用边界：CNN 不天然解决所有图像变化

limits

CNN 的局部连接和参数共享适合图像，但它并不天然处理缩放、旋转等变化。对人眼来说，同一只狗放大、缩小、旋转后仍是狗；对网络来说，像素排列已经变了，输入向量数值也变了。

这就是 data augmentation 的意义。训练时随机裁切、缩放、旋转、翻转，让模型见过不同变形后的样子，才能在测试时更稳。也有专门设计的结构，例如 spatial transformer layer，用来处理空间变换，但本课程此处不展开。

把 CNN 用到围棋、语音、文字时，也要问同样的问题：这个任务是否有局部 pattern？同一个 pattern 是否会在不同位置复用？是否能下采样？AlphaGo 把棋盘看成 19×19×48 的“图片”，利用局部棋形和位置共享，但没有使用 pooling，因为棋盘位置不能随便丢。

图像

局部 pattern 和位置共享通常成立，pooling 常有用但不总是必须。

围棋

局部棋形和位置共享成立，但 pooling 可能破坏精确棋局。

语音/文字

可以用类似思想，但 receptive field 和共享方式要按任务重新设计。

最小代码：CNN 分类器骨架

code reading

下面是一个简化的 PyTorch CNN。重点看形状变化：输入是 [batch, channel, height, width]，卷积层输出更多 channel，pooling 缩小空间大小，flatten 后接全连接层。

正确示例：小型 CNN 前向流程python
import torch
import torch.nn as nn

class TinyCNN(nn.Module):
    def __init__(self, num_classes):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2)
        )
        self.classifier = nn.Linear(64 * 25 * 25, num_classes)

    def forward(self, x):
        x = self.features(x)
        x = torch.flatten(x, start_dim=1)
        logits = self.classifier(x)
        return logits

常见误区和修法

pitfalls
误区错因后果修法
图片就是二维矩阵忽略 channel。写错输入形状或 filter 深度。记住图片是 H×W×C，框架里常是 C×H×W。
CNN 一定比 fully connected 更灵活混淆任务适配和模型弹性。无法理解 CNN 的 bias。CNN 弹性更小，但 bias 适合图像。
3×3 filter 只能看 3×3 pattern忽略多层叠加扩大感受野。误以为必须用很大 kernel。理解深层 feature 对应原图更大范围。
共享参数会让输出一样忘记不同位置输入 patch 不同。误解 filter 扫描。同权重配不同输入，输出仍不同。
CNN 必须有 pooling把经典图像架构当硬规则。在精细空间任务中损失信息。根据任务判断是否下采样。
CNN 天然能处理旋转缩放用人眼直觉替代像素输入。测试时遇到变形表现差。使用 data augmentation 或专门空间变换结构。
