P09 / CNN

卷积神经网络 CNN

这一节只讲 CNN 的基础结构：图像为什么要保持 H×W×C 形状，局部连接和参数共享怎样减少不必要的参数，pooling 和 flatten 在前向流程里做什么，以及这些设计在哪些任务上会失手。batch norm、残差网络、检测和分割先放到后面。

读完本页应该能做什么

  • 估算把图片直接接全连接层时的大致参数量，并说出为什么会爆掉。
  • 沿着一段小型 CNN 前向流程追踪输入通道、输出通道、空间尺寸和 flatten 之后的向量长度。
  • 解释 receptive field、filter、feature map、pooling 的作用，并指出哪些任务不该随便下采样。

学习目标

goal

这一节先把 CNN 当成一套为图像写的结构规则来读，不把它当成“更高级的分类器”。你要能看见它在每一层到底读什么、写什么、删掉了什么。

先会算

遇到图片输入时，先估算直接全连接会带来多少参数，再判断 CNN 为什么值得加这些限制。

先会追

沿着一条前向路径追踪 image tensor、patch、filter、feature map、pooling、flatten。

先会停

知道什么时候不该硬套 CNN，尤其是对位置很敏感、不能随便下采样的任务。

边界 这里先不追 batch norm、残差网络、目标检测、语义分割和更复杂的 backbone。你只需要先把“局部 + 共享 + 下采样”这条线抓牢。

先抓主线

overview

P08 讲的是模型复杂度和泛化。P09 继续往前走，问题变成：同样是图像分类，为什么不直接把图片拉直丢给 fully connected network？答案不在“CNN 更高级”，而在图像本身有局部 pattern、同一 pattern 会在不同位置重复出现，而且适当缩小空间尺寸通常还够用。

1. 输入

把图片看成 H×W×C tensor，而不是一长串没结构的数字。

2. 局部

一个 neuron 先看附近一小块，先把局部关系学清楚。

3. 共享

同一个 filter 在不同位置重复使用，生成 feature map。

4. 汇总

pooling 缩小空间，flatten 后再交给分类头。

image tensor→ receptive field→ filter / feature map→ pooling→ flatten→ classifier
这里先别追的东西 先不要把注意力放到 batch norm、残差连接、检测头、分割头或更大的 backbone 上。现在只看 CNN 这四个动作怎么把图像结构写进网络。

图片与参数量

example

先看一个最容易算清楚的例子。假设输入是一张 100×100×3 的 RGB 图片，第一层直接接 1000 个 neuron。因为每个 neuron 都要和整张图的每个像素相连，参数会迅速膨胀。

$$\text{Params}_{FC} = H \times W \times C \times N + N$$

这里 H、W、C 分别是图片高、宽和 channel 数，N 是第一层 neuron 数。粗估时，bias 的 +N 可以先忽略；这时 100 \times 100 \times 3 \times 1000 大约就是 3000 万 个 weight。

做法连接方式参数规模直觉后果
直接全连接每个 neuron 看整张图随像素数和 neuron 数一起暴涨弹性大，但很容易过拟合
局部连接每个 neuron 先看一小块 patch先把“看全图”改成“看局部”更符合图像里的局部 pattern
局部 + 共享同一个 filter 到处复用再把重复参数收掉一大截更省参数，也更容易训练

这一题最该盯住什么

不是“CNN 会不会让模型更强”，而是“直接全连接为什么太浪费，而 CNN 怎样把不必要的自由度收回来”。

局部感受野

trace

receptive field 指的是一个 neuron 能看到的输入范围。kernel size 是这一层窗口本身有多大；stride 是窗口每次挪多远；padding 是在边缘补不补、补多少。先记住一句话：局部连接先把“看哪里”这件事定下来。

对象前后读什么写什么错了会怎样
输入图片5×5×1一小块局部响应附近的像素关系当前位置的激活值如果范围太大，局部 pattern 会被稀释
局部 patch3×3×1一个标量响应只读窗口内的像素当前 patch 的卷积结果如果 patch 太小，容易只看见碎片
更深一层上一层 feature map更大的有效视野前一层的局部响应更抽象的局部表示如果层数太浅，看到的范围不够大
整张图 5×5×1

这里先看成单通道小图。真实 RGB 图也可以先抽出一个通道来理解局部关系。

当前窗口 3×3×1

一个 neuron 只读这一小块。窗口大小就是这一层的 kernel size。

位置输出 3×3

同一个 filter 在不同位置重复计算，最后堆成一张局部响应图。

这一步在读什么、写什么

读的是局部 patch；写的是当前位置的响应。真正难的不是公式，而是先分清“这一层到底看了哪里”。

参数共享

sharing

局部连接解决“看哪里”，参数共享解决“谁来重复看”。同一个 filter 在图片不同位置复用同一组权重，所以它检测的是同一种局部 pattern，而不是每个位置单独学一套参数。

卷积单步伪代码text
for each position (r, c):
    patch = x[r:r+k, c:c+k, :]
    y[r, c] = sum(patch * w) + b
对象共享前共享后读什么写什么出错症状
filter每个位置都可学不同权重一组权重扫全图同一组 kernel 参数feature map 上各位置的值如果不共享，参数量会再次膨胀
feature map零散位置输出一张响应图同一 filter 的多次计算单个输出 channel如果把不同 filter 混在一起，通道含义会乱
bias可忽略的小修正给每个输出位置加同一个偏移当前响应修正后的激活值忘了 bias 不一定错，但会少一层微调

一句话抓手

共享参数不是让输出一样，而是让“检测什么”这件事在不同位置保持一致。输入不同，响应当然也不同。

多通道卷积

channel

单通道练习能把“窗口怎么滑”看清楚，但真实图片常常不是单通道。RGB 图片有 3 个 channel，所以一个卷积核也要有 3 层深度，先在每个 channel 上做局部乘加，再把三层结果加起来，才得到当前位置的一个输出值。

$$y[r,c] = \sum_{m=1}^{C}\sum_{i=1}^{k}\sum_{j=1}^{k} x[r+i,c+j,m]\,w[i,j,m] + b$$

这里 C 是输入 channel 数，k 是 kernel size。公式里多出来的 m 就是在说：每个 channel 都会贡献一部分，再把它们合并成一个标量。

对象输入前输出后读什么写什么错了会怎样
RGB 输入H×W×3一个局部响应值每个 channel 上的局部像素当前位置的卷积结果如果把深度写成 1，滤波器就接不上输入
单个 filterk×k×3一个输出 channel3 个 channel 的局部 patch一张 feature map如果忽略 channel 维度，颜色关系会被误读
多个 filter一组 k×k×3 kernel多个输出 channel同一个输入的不同局部模式多张 feature map如果 filter 数和输出通道数对不上，后面的层会接错

颜色不是附加物

在 CNN 里，channel 不是图像的装饰条，而是输入结构的一部分。卷积核必须沿着这个深度方向完整覆盖，才能把不同 channel 的关系一起学进去。

最常见的错法

把 3×3 看成完整卷积核，忘了 RGB 还要有深度；或者把输入误写成 1 channel，最后在 PyTorch 里遇到 expected ... to have 3 channels 的报错。

多通道卷积伪代码text
for each position (r, c):
    total = 0
    for each channel m:
        patch = x[r:r+k, c:c+k, m]
        total += sum(patch * w[:, :, m])
    y[r, c] = total + b

把这段代码和上一节接起来看

上一节的手算只看了单通道。这里多了 channel 维度，但流程没变：拿 patch、乘权重、加总、写输出。差别只是“加总”现在要先在深度方向加一轮。

实践中的检查顺序 先看 in_channels 是否等于输入的 channel 数，再看 out_channels 是否等于你想要的 feature map 数量。大多数 shape 错误都卡在这两步。

输出尺寸怎么算

shape

卷积一旦开始滑窗，最容易出错的就不是“有没有卷”，而是“会滑出多少格”。空间输出尺寸只看四个量：输入高宽 H/W、窗口大小 K、步长 S、边缘填充 P。高和宽各算一次，公式一样：

$$H_{out} = \left\lfloor \frac{H + 2P - K}{S} \right\rfloor + 1,\qquad W_{out} = \left\lfloor \frac{W + 2P - K}{S} \right\rfloor + 1$$

这里的 floor 不是装饰符号，而是在告诉你：最后一格窗口如果放不完整，就不会凭空多出一个输出位置。窗口能落下几次，输出就有几格。

情况怎么算输出尺寸直觉出错时常见症状
5×5 输入，3×3 kernel，S=1，P=0水平和竖直都能放 3 次3×3窗口从左上开始，还能往右、往下各挪两次如果还写成 5×5，通常是把 padding 忘了或把 kernel 当成输入尺寸
5×5 输入，3×3 kernel，S=1，P=1边缘补一圈后再滑5×5补零以后，最边上的窗口也能完整覆盖如果输出还写成 3×3，通常是没把 padding 算进去
4×4 feature map，2×2 max pooling，S=2每次跳两格2×2只保留每块里最强的信号如果把 pooling 当成换 channel，后面的 Linear 会接错

三个参数各管什么

K 决定窗口本身多大，也会影响 receptive field；S 决定窗口挪多快；P 决定边缘有没有机会被看见。三者一起决定输出到底缩不缩、缩多少。

先不要混在一起算

很多 shape 错误都不是公式记错，而是先把 Conv2d 的空间变化和 MaxPool2d 的空间变化混成一团。先分清是谁在改尺寸，再代公式。

接回代码 看到 Conv2d(3, 32, 3, padding=1)，先知道空间尺寸不变、channel 变成 32；看到 MaxPool2d(2)，先知道高宽减半。后面的 flatten 只是把这两个变化合成一条向量。

最小练法

先拿 5×5 输入、3×3 kernel、stride=1、padding=0 算出 3×3，再把 padding 改成 1 看输出回到 5×5。如果你把这两种情况混了，练习 13 就会直接暴露出来。

一层卷积怎么手算

walkthrough

把卷积先看成一个会移动的窗口，比直接记公式更稳。你每次只做同一件事：拿出当前 patch，和同一组 filter 做逐元素相乘再求和。窗口往右、往下移动，得到下一格输出。这个过程看起来长，其实每一格都在重复同一件事。

这个 filter 只关心 3×3 小块里的对角线模式。它不认识“图片整体”，只认识自己被训练出来的局部 pattern。

$$y[r,c] = \sum_{i=0}^{k-1}\sum_{j=0}^{k-1} x[r+i,c+j]\,w[i,j] + b$$

这条式子只是在说：取当前位置的 patch，和同一组权重做乘加，得到一个输出。r,c 变化以后，还是同一条公式，不会换规则。

如果你算出来不一样，先看这三件事

第一，窗口是不是从左上角开始滑；第二，stride 和 padding 有没有默认写对；第三，patch 和 filter 的位置有没有对齐。多数卷积错题，不是公式不会，而是窗口偏了一格、把边界算错，或者把通道维度漏掉了。

多 channel 时会多一层相加

单通道只是练习窗口移动。真实 RGB 或更深的 feature map 里，一个 filter 的深度要和输入通道数一致，然后在每个通道上分别做乘加，再把这些结果加起来。也就是说，卷积核不只在空间上滑动，还在 channel 维度上“对齐后汇总”。

接到练习 只要你能把这张 5×5 图手算成 3×3 输出，就已经能把卷积层的 shape 错、窗口错、边界错分开查了。后面代码里的 Conv2d 其实就是把这段循环交给框架去跑。

Pooling

downsample

Pooling 来自第三个图像观察：很多时候，图片缩小一点，语义还在。Max pooling 会把 feature map 切成小块，每块取最大值；空间尺寸变小，但 channel 数通常不变。

输入 feature map分块后取值输出 feature map
[[1, 3, 2, 0],
[4, 1, 6, 2],
[7, 5, 0, 4],
[3, 2, 8, 1]]		 每个 2×2 小块各取一个最大值：4、6、7、8 [[4, 6], [7, 8]]

Max pooling

保留局部最强响应，常用于压缩空间尺寸。

Average pooling

保留平均信息，效果更平滑，但有时会把强信号冲淡。

不要乱用

如果任务很怕位置丢失，过早下采样会伤到结果。棋盘、定位、精细边界都要先想清楚。

AlphaGo 这类例子 可以用 CNN 的局部性和共享性，但不能把 pooling 当成默认动作。棋盘位置本身就是信息，随便抹掉会改变局面。

代码与形状

code reading

下面这段 PyTorch 代码是一个简化版 CNN。你读它时不要先盯语法，先盯形状：每一层到底读什么、写什么，flatten 之前后尺寸怎么变。

正确示例：小型 CNN 前向流程python
import torch
import torch.nn as nn

class TinyCNN(nn.Module):
    def __init__(self, num_classes):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(3, 32, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(32, 64, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
        )
        self.classifier = nn.Linear(64 * 25 * 25, num_classes)

    def forward(self, x):
        x = self.features(x)
        x = torch.flatten(x, start_dim=1)
        return self.classifier(x)
层输入输出这层写了什么
输入B×3×100×100B×3×100×100先把 RGB 图当作 batch 里的样本
Conv2d(3, 32, 3, padding=1)B×3×100×100B×32×100×100把 32 个 filter 扫完整张图
MaxPool2d(2)B×32×100×100B×32×50×50空间尺寸减半，channel 不变
Conv2d(32, 64, 3, padding=1)B×32×50×50B×64×50×50通道变多，表示更多局部模式
MaxPool2d(2)B×64×50×50B×64×25×25再压缩一次空间尺寸
flatten + LinearB×64×25×25B×num_classes把 feature map 接到分类头

读代码时最容易漏的点

flatten 只改形状，不学参数。真正学参数的是 Conv2d 和 Linear。如果前面 pooling 改了，后面的 in_features 也要跟着改。

位置当前 shape会不会学参数为什么常见报错
Conv2dB×32×100×100会卷积核本身就是要学习的权重如果 in_channels 写错，输入接不上
MaxPool2dB×32×50×50不会它只做下采样，不新学参数如果把它当成分类层，shape 会算错
flattenB×64×25×25 变成 B×40000不会它只是把张量重新排成向量如果把它漏掉，Linear 会收到四维输入
LinearB×40000会这是分类头，负责把特征接到类别空间in_features 和前一层展平后的长度对不上

一个常见的接线错误

把 MaxPool2d 之后的输出误算成 64×50×50，然后直接写 nn.Linear(64 * 50 * 50, num_classes)，看起来很像，实际上会把 mat1 and mat2 shapes cannot be multiplied 这类报错直接引出来。排这种错时，先从最后一层往前倒，不要从输入往后猜。

接到练习 只要你能把 Conv2d -> MaxPool2d -> flatten -> Linear 的尺寸账算对，就能先把一半 CNN 报错排掉。剩下那一半多半是 in_channels、out_channels 或者 padding 写错。

常见误区

debug
误区错因后果先检查哪里
共享参数会让输出一样忘了输入 patch 不同误解 filter 扫描看同一组权重接了什么输入
CNN 必须有 pooling把经典架构当硬规则在精细位置任务里丢信息看任务是否怕空间尺寸缩小
flatten 本身会学习参数把形状变化和学习层混在一起调参时找错层看是哪一层在写权重
卷积层输出的 channel 数随便猜没盯住 filter 数后面的层接错看 Conv2d(in_channels, out_channels, ...)
 
RuntimeError: Given groups=1, weight of size [32, 1, 3, 3], expected input[8, 3, 100, 100] to have 1 channels, but got 3 channels instead

先读报错，再看层

通道错了通常先看 Conv2d(in_channels)；尺寸错了通常先看 pooling、stride、padding 和 flatten 前后的 shape；位置太敏感则要回到任务本身，看能不能接受下采样。

练习

diagnostic

怎么答才算会

先看输入形状，再看卷积层和 pooling 怎么改形状，最后回到任务边界。能说清“看哪里、谁复用、哪里下采样”才算过关。

检查重点

重点不是背术语，而是能用它们读代码、估参数量、解释报错，并说出什么时候不该继续压缩空间尺寸。

一、先把参数算清楚

先看输入形状，再看第一层会不会爆掉。

10. AlphaGo 为什么可以用 CNN，但不适合随便用 pooling？

先用自己的话说，再对照下面的评分要点。

答题时至少要提到：局部棋形、位置共享、棋盘位置不能随便丢。

参考答案 / 评分要点

好答案会说清楚：围棋盘上确实有局部棋形，而且同一种棋形会在不同位置重复出现，所以 CNN 的局部连接和参数共享很合适；但棋盘上的位置信息本身很重要，过早 pooling 可能把精确位置压掉，局势会被改写。答到“能用 CNN、但不能默认 pooling”就已经抓住重点。

14. 用 8-11 句话讲清楚：CNN 为什么适合图像？

必须串起：H×W×C、fully connected 参数爆炸、receptive field、parameter sharing、filter、feature map、pooling、flatten。也要说出一个不该直接套用 CNN 的边界。

参考答案 / 评分要点

高分答案会先说明图片是 H×W×C tensor，直接全连接会让参数量跟像素数一起暴涨；再说明 CNN 用局部连接让每个 neuron 只看一小块，用参数共享让同一个 filter 在不同位置重复使用，因此能把“检测局部 pattern”这件事做得更省参数；接着讲 filter 扫完整张图会形成 feature map，必要时再用 pooling 压缩空间尺寸，最后 flatten 接分类头。结尾要补一句边界，例如对位置特别敏感的任务不能随便 pooling，CNN 也不会天然解决旋转或缩放。
