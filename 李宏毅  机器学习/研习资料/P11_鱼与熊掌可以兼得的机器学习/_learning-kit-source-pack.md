## Source-To-Unit Notes

- 这一节的中心不是“深层一定更好”，而是“深层在某些有规律的函数族上，可能用更少参数表示同样的目标”。
- 旧页已经覆盖的主线很清楚：
  - 先从 P08 的两难出发：理想 loss 和现实泛化/表达效率不能只看一个。
  - 再说明一层 hidden layer 也能逼近很多函数，尤其是 piecewise linear function。
  - `ReLU(x) = max(0, x)` 能制造折点，所以很适合拼出折线。
  - 关键差别不在“能不能表示”，而在“表示同一个函数要多少参数”。
  - 深层结构的优势可以写成 `2k` 对 `2^k`：用层层折叠的方式，少量 neuron 就能产生很多线段。
  - parity check、程序 module、折纸/折叠这些例子都在说明中间表示可以复用。
  - `tent` / `deep_fold` 代码桥接了“每加一层就再折一次”的直觉。
  - 课程最后应把这个主题收束到边界：深度不是万能标签，问题是否有可复用结构才是关键。
- 这次重构把旧页里分散的片段改成一条完整教学链：
  1. 先把“鱼与熊掌”说成一个更精确的两难：低 loss、泛化和参数效率要一起看。
  2. 再把 universal approximation 和 `piecewise linear`、`ReLU` 接起来，先说明“能做到”。
  3. 然后说明为什么深层结构有机会更省参数，重点落在 `2k` / `2^k` 的对比。
  4. 接着用 parity、module、折纸/折叠把“中间表示复用”讲扎实。
  5. 再用 `tent` / `deep_fold` 代码和状态表把折叠过程手工走一遍。
  6. 最后补边界、误区和下一节衔接。
