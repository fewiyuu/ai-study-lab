学习目标

goal

这一节要先把三件事分开：一层网络能不能表示很多函数，深层结构怎样复用中间表示，以及什么时候“更深”确实能把参数花得更省。课程里说的“鱼与熊掌可以兼得”，指的就是低 loss 和参数效率可以同时讨论，但前提是任务里有可复用的规律。

先会算

能把课程里的锯齿例子写成 2k 对 2^k，看出线段数量为什么会被层层折叠放大。

边界先说清

这里先不追 batch norm、残差网络、优化器选择、检测和分割头，也不把深层网络说成万能模板。你只需要把“能表示”“表示得省”“深层在什么条件下更值”这三条线抓住。

现在你应该能做什么

如果别人只说“深层更好”，你能追问：好在哪里，是低 loss、更少参数，还是更容易复用中间表示？

先抓主线

overview

P08 讲的是理想目标和现实泛化之间的张力。P11 把这条线继续往前推：一层 hidden layer 其实已经能表示很多函数，但这只回答“能不能”，没有回答“要花多少 neuron”。课程要你看的，是当函数本身有规律、而且规律可以在中间层被复用时，深层结构怎样把表示成本压下来。

代码与边界

先会表示，再谈省不省

universal

课程先把一层 hidden layer 拉出来复习：只要 neuron 够多，浅层网络也能逼近很多函数。这里的意义是提醒你，深层优势主要落在“同一个函数要怎么表示得更省”这道题上，而非简单争论能否表示。ReLU 很适合做这件事，因为它天然带一个折点，拼起来就能得到 piecewise linear function。

这条式子很短，但它的作用很明确：负数被压成 0，正数原样通过，所以在 x=0 处有一个折点。多个 ReLU 组合起来，就能搭出很多折线段。

这里先别跳过的一个区分

“一层可以逼近很多函数”与“表示得省”是两件事。前者回答存在性，后者回答成本。P11 的后半段要把这个成本差讲清楚。

常见误读

看到 universal approximation，就直接得出“浅层已经够了”，这会漏掉参数规模；看到 ReLU，就以为它只是一个激活函数，不会想到它还能拼成很多折点。这里要把这两层意思都留住。

参数效率

2k

课程后半段最重要的数字对比是 2k 和 2^k。这个数字账需要带条件理解：课程用一个锯齿例子说明，如果目标函数可以通过层层折叠复用中间结构，那么深层网络往往能把复杂度藏进层数里，避免一次性摊平到宽度上。

这里的 k 是折叠次数或层数。课程强调的是一个能手算的直觉：每再折一次，线段数会翻倍；但如果不借助层层复用，要把同样的折线摊平，宽度会涨得更快。

这个对比到底在说什么

课程想让你看到的是：当函数有层次、有重复、有中间结果可以继续被加工时，深层结构会把“表达复杂度”变成“层数上的复用”，避免只靠加宽来堆。

边界别拉太大

2k / 2^k 属于课程里的构造例子，不能直接推广到所有任务。它的作用是让你先看到“深层为什么可能更省”，不能替所有模型下定论。

锯齿折叠

trace

这一段是 P11 最能手工追的部分。课程里的锯齿用意很务实：把“每加一层，就把上一层的输出再折一次”这件事看见。先拿一个最小锚点：x=0.30。第一次折叠后，输出会变成 0.60；第二次再折，变成 0.80；第三次继续折，得到 0.40。你会看到，深层会不断重用前一层结果，逐步形成复杂图案。

这个函数先上升再下降，所以像一个 V。课程把它当成“折一次”的最小模型。真正要记住的是它能把一个区间折出两段，再把两段继续折成更多段。

为什么这段例子有效

因为它把“层层复用”具体到一个能画出来的函数。你不用相信抽象口号，只要看着一个 V 被再折一次，就知道为什么层数会把折线数翻倍。

和上一节怎么接

universal approximation 只说明浅层能做出很多事；锯齿例子则补上另一半：如果你想让同一条函数曲线更省地出现，深层有机会把它做得更值。

不要把这个例子误用成总定理

课程没有说所有任务都能用锯齿分析，也没有说所有深层网络都一定更省。这个例子只是把“深层通过复用中间表示来省参数”这条线讲到你能自己算出来。

观察点deep 看到什么shallow 看到什么说明了什么
第 1 次折叠一个 V 形如果直接摊平，只能把局部变化写成很多小段深层先建立一个中间表示
第 2 次折叠V 继续变成 W要把同样的折痕摊开，宽度得继续增长中间结果被再次加工
第 3 次折叠折痕数再翻倍浅层要重新铺出更多分段复用让层数变成表达上的杠杆

结构复用

reuse

课程接着用三个例子把“复用中间表示”这件事讲稳。第一个是 parity check：做法不必直接枚举所有输入组合，可以把局部逻辑一层层合起来。第二个是程序里的 module 和 function：复杂程序通常不会把所有逻辑写在一个地方，而是拆成可以复用的中间部件。第三个是折纸/折叠：少量操作如果可以重复使用，就能让最后的形状变化得很丰富。

把三个例子合成一句话

深层结构的价值，在于它会把“中间结果可以继续被加工”这件事写进模型里；模型减少从头重算，让前一层学到的规律继续往下传。

对象别混

parity、module 和折纸当然差别很大。课程把它们放在一起，只是为了让你先看见“分层复用”这条共同的思路。

比较深浅层时先审计

audit

这一讲最容易被学成一句口号：“deep network 可以鱼与熊掌兼得。”真正该带走的是一套检查顺序。第一步看目标函数有没有可复用结构。锯齿、parity、module 这几个例子都有共同点：局部结果可以继续交给下一层加工。第二步看参数账怎么算。课程里的 2k 和 2^k 是为了说明同一类形状如果用层层折叠来表达，增长压力主要放在层数上；如果摊到单层宽网络里，压力会变成宽度快速膨胀。第三步看这个比较有没有离开构造条件。只要任务缺少这种可递归、可组合、可复用的结构，深层优势就需要重新验证。

检查项要问的问题支持 deep 更省的证据需要降级结论的情况
目标结构函数里有没有重复、组合或递归式的规律？能找到类似折叠、局部奇偶、模块调用的中间结果每个局部细节都像独立例外，无法被复用
参数账同样复杂度下，宽度和深度分别怎么增长？深层接近线性增加，浅层需要指数级或很快变宽比较只报最终 loss，没有说明模型规模
中间表示上一层输出是否会成为下一层的有效输入？代码或图形 trace 能看到状态被逐层加工每层像独立特征堆叠，看不出连续传递关系
验证边界这个构造能否迁移到当前任务？任务本身有层级、组合、局部到整体的结构只是把课程例子套到任何数据集上

小案例：先画三层，再下结论

假设目标曲线从一个简单的 V 形开始：第一层把区间折成两段，第二层把每一段再折一次，第三层继续处理第二层的输出。你可以把它当成一个审计练习：先在纸上记下每层输入和输出，再数最后有多少段直线。若每层都复用上一层的结果，pieces 会从 2、4 增到 8；若用单层网络一次性摊平，就要在同一层里准备足够多的局部折点。这个小案例不会证明所有深层网络都好，但它能帮你发现课程例子的核心：层数承担了“重复加工”的角色，宽度承担了“摊开所有局部变化”的角色。

把这个练习迁移到真实模型时，先问一个朴素问题：上一层到底给下一层留下了什么可用信息？如果只是把特征堆得更多，看不到状态传递，深层结构的参数效率优势就没有被说明。相反，如果你能写出类似 z = f(z) 的 trace，或者能在表格里标出每层都继续加工同一个中间表示，这个比较才开始有技术含义。练习题里的代码追踪就是为这个目的服务：它迫使你从“深层看起来更复杂”转到“每层怎样复用前一层输出”。

再做一个反向检查：如果把输入换成一串没有重复模式的离散标签，trace 表里就很难写出“第 2 层继续利用第 1 层结果”的稳定规则。此时深层结构也许还能训练出不错表现，但这页里的参数效率论证已经不够用。做练习时可以把这个反向案例写在答案末尾，用它提醒自己先验证结构，再引用 2k / 2^k。若写不出这条规则，答案就要停在观察层级，不能直接搬用课程里的指数差；能写出来时，也要说明它只支持这个函数族里的表示效率比较。

怎么把结论说稳

稳妥说法是：在某些有规律、可组合的函数族上，深层结构可以用更少参数表达同样复杂的变化。这个说法同时保留了三个限制：适用范围有限，深层不会自动更好，训练 loss 也不能单独定案。

怎么发现自己说过头

如果答案里出现“深层一定”“浅层不行”“参数越少越好”这类绝对句，就要补上条件。课程给的是表示效率的构造直觉，没有替模型选择、泛化能力和训练稳定性一次性盖章。

一句话自审

讲完这一页后，你应该能把“能表示”“表示得省”“实际表现好”拆开说：能表示靠 universal approximation，表示得省靠层间复用和参数账，实际表现好还要回到数据、优化和验证。

代码与折叠

code

课程最后把直觉压成一段很短的代码：tent(z) 负责把区间折成一个 V 形，deep_fold 则把“上一层输出作为下一层输入”这件事写成循环。你读这段代码时，不要先盯语法，先盯状态怎么传下去。每一层都在读上一层的 z，再把新的 z 写回去。

代码
def tent(z):
 if z <= 0.5:
 return 2 * z
 return 2 * (1 - z)

def deep_fold(x, layers):
 z = x
 for _ in range(layers):
 z = tent(z) # 下一层读取上一层的输出
 return z

这段代码在教你什么

它把“层层折叠”写成了一个最小状态机：读上一层输出，写下一层输入。课程真正想让你记住的，是深层网络为什么能把复杂图案藏进一个很短的循环里。

。可以这样答：折叠像把上一层输出继续加工，局部变化像中间表示的重复使用，展开后的图案像深层组合后形成的复杂函数。重点是“前一层的结果还能继续被下一层使用”，每一步都回到最初输入重算就失去了这个类比。

四、2k / 2^k 与代码追踪

公式 每折一次，线段数翻倍；层数往上加，输出继续被加工。 -->

deep 约 12，shallow 约 64。k=6 时 deep 约 2k=12，shallow 约 2^k=64。

p11-q14flow

Q: 14. 为什么每加一层类似 V 字的结构，线段数量会翻倍？

参考答案 / 评分要点

。好的答案会说清楚：上一层先把输入折成一个 V，下一层再把这个 V 当成输入重新折一次，于是原来的每一段又被分成两段，线段数自然会翻倍。核心在于“上一层输出继续被下一层加工”，层数的作用要通过这条传递链体现出来。

p11-q15code reading

Q: 15. 在代码 z = tent(z) 中，为什么说下一层使用了上一层的输出？

参考答案 / 评分要点

。因为这一行先读取当前的 z ，把它送进 tent ，再把返回值写回同一个变量 z 。下一次循环取到的已经是上一层折叠后的结果。这个写回动作就是层与层之间的传递。

Q: 16. “Deep network 一定比 shallow network 更需要大量资料。”这句话哪里不严谨？

参考答案 / 评分要点

。可以从三点答：第一，这句话把训练数据、任务规律和模型结构混成一件事；第二，参数多不一定总是更容易过拟合，关键还要看是否存在可复用结构；第三，更稳的说法是：深层结构是否划算，要看任务本身的规律和验证结果。

五、表达和边界

主线 复杂且有规律时，深层可能更省；换到没有可复用结构的任务，结论要重新验证。 -->

Q: 17. 用一句话回应：“deep learning 只是听起来比较厉害，其实 fat network 也可以。”

参考答案 / 评分要点

。可答成：一层网络当然能表示很多函数，但深层网络在有可复用结构的任务上，可能用更少参数表达同一个目标；所以要继续追问“代价多少”。

p11-q18Feynman

Q: 18. 用 8-12 句话讲清楚：深度学习如何让鱼与熊掌兼得？

参考答案 / 评分要点

。高分答案会先把 ideal 和 realistic 的张力说清楚：只看低 loss 不够，还要看模型空间和验证表现。然后再说一层网络能逼近很多函数，ReLU 可以制造折点，多个 ReLU 能拼出 piecewise linear function。接着用 2k / 2^k 说明深层结构在某些有规律的函数族上更省参数，再用 parity check、module、折叠这类例子说明中间表示可以复用。最后用审计链路收束：先确认任务有可复用结构，再追踪上一层输出如何被下一层加工，然后把 deep 的层数增长和 shallow 的宽度增长分开算。结尾要补一个边界：如果任务没有明显规律，或者位置细节很重要，深层也不会自动赢。
