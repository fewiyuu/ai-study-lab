# 本节的概念地图 {#goals}
<!-- tag: map -->

Self-attention 的输入是一排向量 `a₁, a₂, ..., aₙ`，输出是一排新向量 `b₁, b₂, ..., bₙ`。重点是：每个 `bᵢ` 都考虑整段 sequence 后产生。

为了产生 `b₁`，先把 `a₁` 变成 query，表示“我现在要找什么资讯”；把每个 `aⱼ` 变成 key，表示“我这里有什么索引”；query 与 key 做 dot product，得到 attention score。score 经过 softmax 变成权重，再用这些权重加权求和所有 value，就得到 `b₁`。

同样过程对每个位置都做一次，就得到一排带上下文的新向量。之后可以把这些 `b` 丢给 fully connected network 做分类或回归。课程也提到，self-attention 可以和 fully connected layer 交替堆叠；Transformer 的核心模块就是 self-attention。

sequence of vectors→
Q/K/V→
attention scores→
softmax weights→
weighted sum→
contextual vectors

直觉每个位置先向全句提问：哪些位置对判断我最重要？
正式说法`bᵢ = Σⱼ softmax(qᵢ · kⱼ) vⱼ`，其中 Q/K/V 来自输入向量的线性变换。
关键边界P12 先讲单层单头的核心机制；multi-head、positional encoding 和完整 Transformer 后面再展开。

# 输入变成一串长度可变的向量 {#sequence}
<!-- tag: sequence input -->

文字是最直观的例子。一个句子可以看成一串词，每个词用一个向量表示。最简单的表示是 one-hot encoding：词汇表有多大，向量就有多长；某个词对应的位置是 1，其余是 0。它的问题是看不出语义关系：cat 和 dog 都是动物，但 one-hot 向量之间并不会自然更接近。

更常见的做法是 word embedding。Embedding 也是向量，但它带有语义结构：动物词可能聚在一起，植物词可能聚在一起，动词也可能形成自己的区域。P12 不展开 embedding 怎么训练，只要知道：句子可以被表示成一排向量，而且每个句子的长度不同。

语音也是一排向量。一段声音会被切成很多短窗口，例如 25 ms 的 window，每次往右移动 10 ms。每个 window 被描述成一个 frame 向量，因此一分钟语音可能有几千个 frame。Graph 也是一堆向量：社交网络的节点、分子图里的原子，都可以各自带一个向量。

| 资料类型 | 一个向量代表什么 | 为什么长度会变 |
| --- | --- | --- |
| 文字 | 一个词或 token | 不同句子的词数不同。 |
| 语音 | 一个短时间 window 的 frame | 不同语音片段时长不同。 |
| Graph / 分子 | 一个节点或原子 | 不同图的节点数不同。 |

# 三种输出：这节先学 sequence labeling {#outputs}
<!-- tag: output types -->

当输入是一串向量时，输出不只有一种形式。第一种是输入几个向量，就输出几个 label。词性标注就是这样：句子里每个词都要标出名词、动词、形容词等；作业二中的音素预测也类似，每个 frame 要预测对应的声音类别。

第二种是整段 sequence 只输出一个 label。例如 sentiment analysis 给一段文字，判断整体正面或负面；说话人辨识给一段语音，判断是谁说的；分子性质预测给一个分子图，输出是否有毒或亲水性。

第三种是输出长度由模型自己决定，也就是 sequence-to-sequence。例如机器翻译输入中文句子，输出英文句子，词数未必相同；真正的语音识别输入一段语音，输出一串文字，也属于这种类型。P12 先只讲第一种，因为它最适合引出 self-attention 的基本结构。

many-to-many 同长度输入 N 个向量，输出 N 个 label。P12 重点：sequence labeling。
many-to-one整段输入只输出一个 label，如情感分析、说话人辨识。
seq2seq输出长度由模型决定，如翻译、完整语音识别。

# 为什么只用 Fully Connected 各个击破不够 {#why}
<!-- tag: context -->

最直觉的做法是：把每个向量单独丢进 fully connected network，分别输出 label。这样实现简单，但会失去上下文。词性标注里，两个完全相同的词，因为前后文不同，可能一个是动词、一个是名词；如果模型只看到当前词向量，它没有理由输出两个不同结果。

一个改良是开 window：不仅看当前向量，也把前后几个向量一起串起来丢进网络。语音任务里常用这种做法，例如看当前 frame 前后若干 frame，帮助判断当前音素。这能解决局部上下文问题。

但 window 有两个限制。第一，如果真正需要的是很远处的信息，固定小窗口看不到。第二，如果把 window 开到覆盖整段 sequence，长度可变会带来麻烦：你必须按最长 sequence 设计输入，参数量可能变得很大，计算成本和 overfitting 风险都上升。

小对照：三种看上下文的方式

**a₁**只看自己
**a₂**局部 window
**a₃**可能要看远处
**a₄**全序列可见


方法能看多远问题

单独 FC只看当前向量无法区分同词不同上下文。
固定 window看前后固定范围远距离依赖仍可能看不到。
Self-attention每个位置都能看全序列需要计算所有位置间的相关性。

# Q、K、V：怎么判断谁和谁相关 {#qkv}
<!-- tag: query key value -->

Self-attention 的第一步，是为每个输入向量制造三个版本：query、key、value。它们都是由输入向量乘上不同矩阵得到的：`qᵢ = Wq aᵢ`、`kᵢ = Wk aᵢ`、`vᵢ = Wv aᵢ`。同一个 `aᵢ` 会分出三种用途。

Query 可以理解为“我现在要找什么”。当我们要生成 `b₁` 时，就用 `a₁` 产生的 `q₁` 去问整段 sequence。Key 可以理解为“每个位置可被匹配的索引”。把 `q₁` 和每个 `kⱼ` 做 dot product，就得到 `a₁` 和 `aⱼ` 的 attention score。

Value 是真正会被汇总的内容。Score 只决定权重，最后输出 `b₁` 时，实际相加的是各个 `vⱼ`。因此 Q/K 负责“算相关性”，V 负责“提供内容”。

$$
qᵢ = Wqaᵢ,   kⱼ = Wkaⱼ,   vⱼ = Wvaⱼ
 αij = qᵢ · kⱼ
$$

Query当前要生成哪个位置的输出，就用哪个位置的 query 去提问。
Key每个位置都提供 key，让 query 可以计算相关性。
Value根据 softmax 权重被加权求和，成为输出向量的内容来源。

# Softmax 与 weighted sum：从分数到新向量 {#softmax}
<!-- tag: attention output -->

Dot product 得到的是一排原始 attention score，例如 `α₁₁, α₁₂, α₁₃, α₁₄`。这些分数还不能直接当比例使用，因为可能有正有负，总和也不一定是 1。常见做法是把它们送进 softmax，得到一排非负且总和为 1 的权重。

课程特别提醒：这里用 softmax 是最常见做法，但不是唯一选择。有些实验可能用 ReLU 或其他 activation 得到不同效果。重要的是理解它的功能：把相关性分数转成“要从各位置取多少信息”的权重。

最后，把每个 value 乘上对应权重再相加，就得到当前输出 `bᵢ`。如果某个位置的权重特别大，它的 value 就会主导输出。对每个 `i` 都重复这件事，就得到所有 `bᵢ`。

$$
α'ij = softmax(αij) = exp(αij) / Σt exp(αit)
 bᵢ = Σj α'ij vⱼ
$$

权重最大的位置会把自己的 value 更多贡献给输出。请注意：query 来自当前目标位置，但 key 和 value 来自所有位置。

# 最小代码：单头 self-attention 的核心 {#code}
<!-- tag: minimal code -->

下面代码只保留核心计算：输入矩阵 `A` 的每一行是一个向量；线性变换得到 `Q`、`K`、`V`；用 `Q @ K.T` 得到所有位置对所有位置的 attention score；softmax 后与 `V` 相乘得到 `B`。

读代码时要抓住 shape。假设序列长度是 `n`，每个向量维度是 `d`，那么 score 矩阵是 `n × n`：第 `i` 行表示第 `i` 个位置看所有位置的分数。输出 `B` 仍然是 `n` 个向量，因为 P12 讨论的是输入输出长度一样的 sequence labeling。

正确示例：单头 self-attentionpython

```text
import numpy as np

def softmax(x, axis=-1):
 x = x - x.max(axis=axis, keepdims=True)
 e = np.exp(x)
 return e / e.sum(axis=axis, keepdims=True)

def self_attention(A, Wq, Wk, Wv):
 Q = A @ Wq # 每个位置提出 query
 K = A @ Wk # 每个位置提供 key
 V = A @ Wv # 每个位置提供 value
 scores = Q @ K.T # scores[i, j] = q_i dot k_j
 weights = softmax(scores, axis=1)
 B = weights @ V # 每一行都是加权后的上下文向量
 return B, weights
```

这里没有写 Transformer 的其他部分，例如 multi-head、residual connection、layer normalization、positional encoding。它们很重要，但不是 P12 的最小目标。P12 的最小目标是：你能解释 `scores = Q @ K.T` 和 `B = weights @ V` 各自在做什么。

错误示例：把 score 当成最终输出pseudo

```text
# 错误说法：attention score 越高，score 本身就是输出向量。
scores = Q @ K.T
B = scores

# 问题：score 只是相关性分数；输出内容来自 value 的加权和。
```

# 常见误区与修正 {#mistakes}
<!-- tag: pitfalls -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：self-attention 只适合文字
错因：把课程开头的文字例子当成范围限制。后果：看不到语音、graph、分子等序列输入。修法：抓住本质是“一串向量”。

## box redsoft
### 误区 2：每个位置只看自己
错因：把输入输出长度相同误解成逐点独立。后果：无法解释上下文如何进入输出。修法：记住每个 `bᵢ` 都由所有 value 加权得到。

## box redsoft
### 误区 3：Q、K、V 是三份原始输入
错因：没有看到它们来自不同线性变换。后果：混淆匹配和内容汇总。修法：Q/K 用于算分数，V 用于被汇总。

## box redsoft
### 误区 4：softmax 后最大权重一定等于 1
错因：把 softmax 当成 argmax。后果：误以为只选择一个位置。修法：softmax 给一组总和为 1 的连续权重，可以同时混合多个 value。

## box redsoft
### 误区 5：attention score 就是最终输出
错因：停在相关性计算。后果：漏掉 value 的 weighted sum。修法：score → softmax weight → 加权 value → output。

## box redsoft
### 误区 6：window 开大就等于 self-attention
错因：只看到都使用上下文。后果：忽略可变长度和全局相关性。修法：window 固定范围拼接，self-attention 为每个位置动态计算全序列权重。

### 现在你应该能做什么
诊断一段解释是否把 Q/K/V、softmax、weighted sum 的角色说反或说漏。
