# 本节的概念地图 {#goals}
<!-- tag: map -->

P13 的第一件事，是把逐个位置的计算改写成矩阵计算。把所有输入向量拼成矩阵 `I`，分别乘上 `WQ`、`WK`、`WV`，得到 `Q`、`K`、`V`。接着计算所有 query 与所有 key 的内积，得到 attention matrix。

第二件事，是理解 attention matrix 的意义。它的每一列或每一行，取决于你采用的排布习惯，表示某个位置对所有位置的 attention score。通过 softmax 得到权重矩阵后，再与 `V` 相乘，就一次得到所有输出向量。

第三件事，是把 self-attention 放进真实模型设计里：multi-head 用多组相关性看同一段资料；位置编码告诉模型 token 的顺序；truncated attention 在长语音序列中只看局部范围；图像、RNN、graph 场景则展示它和其他架构的关系。

# 矩阵化：所有输出同时算出来 {#matrix}
<!-- tag: parallel -->

P12 是从 `b₁` 开始讲：拿 `q₁` 去和所有 key 算分数，再加权所有 value 得到 `b₁`。P13 强调，同样操作可以对所有位置并行做。你不需要先得到 `b₁`，再得到 `b₂`，它们没有依赖顺序。

把 `a₁` 到 `a₄` 拼成输入矩阵 `I`。每个位置都乘上同一个 `WQ`，得到所有 query；每个位置都乘上同一个 `WK`，得到所有 key；每个位置都乘上同一个 `WV`，得到所有 value。这就是并行的第一步。

接着，所有 query 与所有 key 的内积也能一次做完。课程投影片采用 column vector 的画法，所以会写成类似 `KᵀQ` 得到 attention matrix；很多代码会把序列放在 row 维度，于是写成 `QKᵀ`。不要被转置方向吓到，只要记住矩阵中的每个格子都是某个 `q_i` 与某个 `k_j` 的 dot product。

$$
Q = I WQ,   K = I WK,   V = I WV
 A = Q KT,   A' = softmax(A),   O = A' V
$$

小交互：矩阵尺寸检查
sequence 长度 L**6**
向量维度 d**8**


L=6, d=8
Attention matrix 的大小只由序列长度决定，是 L×L；向量维度影响 Q/K/V 和每次 dot product 的成本。

# 哪些是参数，哪些只是计算 {#params}
<!-- tag: learned weights -->

Self-attention 看起来步骤很多，但需要从训练资料学出来的，核心是 `WQ`、`WK`、`WV` 这些矩阵。它们决定了什么样的 query、key、value 表示更适合任务。

Dot product、softmax、weighted sum 这些固定运算。训练时，loss 会通过这些运算把梯度传回 `WQ/WK/WV`，让模型慢慢学会怎样产生更有用的 attention pattern。

这点很重要，因为很多初学者会误以为 attention score 是直接存起来的参数。不是。Attention score 是当前输入动态算出来的：换一个句子、换一段语音、换一张图片，score matrix 都会不同。

| 对象 | 是否学习参数 | 作用 |
| --- | --- | --- |
| WQ / WK / WV | 是 | 把输入映射成 query、key、value。 |
| Attention score A | 不是 | 由当前 Q/K 动态计算出来。 |
| Softmax / normalization | 不是 | 把分数变成可加权的比例。 |
| 输出 O | 不是参数 | 当前输入经过 attention 后的结果。 |

# Multi-head：同一段序列的多种相关性 {#multihead}
<!-- tag: multi-head -->

单头 attention 只有一种 query/key/value 空间，它会学习一种“相关”的定义。但现实任务里，相关性可能有很多种：词性标注可能要看语法关系，也要看词义关系；语音可能要看局部发音，也要看较长的节奏；翻译可能同时需要对齐词、短语和句法结构。

Multi-head self-attention 的直觉是：让不同 head 学不同类型的相关性。每个 head 有自己的 Q/K/V 子空间，分别计算 attention，得到多个输出片段。然后把这些片段 concat 起来，再乘上一个矩阵，合成当前层的输出。

Head 数目是 hyperparameter。更多 head 不一定永远更好，因为它会影响参数量、计算量和过拟合风险；有些作业或小任务可能少一点 head 就够，有些翻译或语音任务则可能需要更多 head 才能捕捉多种关系。

$$
headr = Attention(Qr, Kr, Vr)
 output = Linear(concat(head1, head2, ..., headh))
$$

一个 head只学习一种匹配视角，结构简单。
多个 head并行学习多种相关性，再合并。
调参Head 数不是越多越好，要看任务、资料量和计算预算。

# 位置编码：让模型知道谁在前、谁在后 {#position}
<!-- tag: position -->

到目前为止的 self-attention 本身没有顺序概念。投影片上写 `a₁`、`a₂`、`a₃` 是为了人类理解；对 attention 运算来说，每个位置执行的是同一套操作。位置 1 和位置 4 并不会天然更远，位置 2 和位置 3 也不会天然更近。

这会带来问题。许多任务需要位置资讯：词性标注里，句首、句中、句尾可能影响词性判断；语音和文字都强烈依赖顺序。如果模型不知道顺序，它只能根据内容向量本身做判断。

Positional encoding 的做法是给每个位置一个专属向量 `e_i`，再把它加到输入 `a_i` 上。这样模型看到的不只是词或 frame 的内容，也包含“我在第几个位置”的信息。最早 Transformer 使用 sin/cos 规则产生位置向量；后来也有可学习的位置向量、用网络生成的位置向量等许多变形。

$$
input at position i:   aᵢ + eᵢ
 eᵢ can be handcrafted, sinusoidal, learned, or generated by another model
$$

## box warn
### 别背死 sin/cos
课程强调 positional encoding 仍是研究问题。Sin/cos 是经典选择，不是唯一真理；你需要理解它解决“位置资讯缺失”，而不是只背公式长相。

### 现在你应该能做什么

# 长序列成本：L×L attention matrix {#complexity}
<!-- tag: complexity -->

Self-attention 的强项是每个位置都能看全序列；代价是要计算所有位置对所有位置的相关性。如果序列长度是 `L`，attention matrix 就是 `L × L`。当 `L` 很大时，计算量和记忆体都会变得可观。

语音特别容易遇到这个问题。一秒可能有 100 个 frame，五秒就 500 个 frame，一句话可能上千个 frame。完整 attention matrix 会迅速变大。因此语音任务中常见 truncated self-attention：不看整句，只看当前附近的一段范围。

这利用任务理解。如果判断某个音素主要依赖前后一定范围，没必要让每个 frame 都和全句所有 frame 计算 attention。更广泛地，efficient Transformer、Linformer、Performer、Reformer 等变形，都是在尝试减少 self-attention 的成本。

full attention**65,536**
window=64**16,384**

绿色表示完整 attention 的全矩阵；蓝色表示只看局部 band 的 truncated attention。


L=256
完整 attention 需要 L² 个位置对；局部窗口把计算限制在附近范围，速度更快但会牺牲远距离连接。

# 和 CNN、RNN、GNN 的关系 {#relations}
<!-- tag: architecture compare -->

Self-attention 可以用于图像。把一张图片看成很多 pixel 或 patch，每个 pixel/patch 是一个向量，就能套用 attention。与 CNN 相比，CNN 的 receptive field 是人工设定的局部范围；self-attention 则让模型自己用 attention 找相关位置。可以说 CNN 是一种受限制的 attention，attention 是更 flexible 的 CNN。

但更 flexible 不等于总是更好。P13 提到 Vision Transformer 相关实验：资料量少时，CNN 的限制反而是一种好归纳偏置，可能比 self-attention 更稳；资料量非常大时，self-attention 的弹性可以吃到更多资料带来的好处。这和前面讲的 overfitting 与模型弹性一致。

RNN 也处理 sequence，但它通常按时间一步步处理，当前 hidden state 依赖前一步 hidden state。即使双向 RNN 能看左右文，远距离信息也要被 memory 一路传递。Self-attention 则可以让远处位置直接通过 query/key 匹配，且所有输出并行产生。

Graph 上也能用 attention。Graph 的 edge 已经告诉我们哪些节点相关，因此可以只在有边连接的节点之间计算 attention score；没有边的 score 直接设为 0。这类受 graph 结构限制的 attention，就接近某些 Graph Neural Network 的做法。

| 架构 | 核心限制/优势 | 与 self-attention 的关系 |
| --- | --- | --- |
| CNN | 人工设定局部 receptive field | 可看成受限制的 attention；数据少时更稳。 |
| RNN | 按时间递推，较难并行 | 都处理 sequence，但 attention 可直接连远处并并行输出。 |
| GNN | 使用 graph edge 组织信息传播 | 只在相连节点算 attention 时，类似图上的 attention/GNN。 |

# 最小代码：矩阵化、多头、mask {#code}
<!-- tag: minimal code -->

下面代码用 row-major 的常见写法：`X` 的 shape 是 `[L, d]`，每一行一个位置。它展示三个关键点：一是所有输出并行算出；二是 multi-head 只是把投影后的维度拆成多个 head；三是可以用 mask 把某些位置对禁止掉，例如 graph edge mask 或 truncated attention mask。

正确示例：矩阵化 attention 与 maskpython

```text
def attention(X, Wq, Wk, Wv, mask=None):
 Q = X @ Wq # [L, d_head]
 K = X @ Wk # [L, d_head]
 V = X @ Wv # [L, d_head]
 scores = Q @ K.T # [L, L]
 if mask is not None:
 scores = scores + mask # 不允许看的格子加上很大的负数
 weights = softmax(scores, axis=-1)
 O = weights @ V # [L, d_head]，所有位置同时输出
 return O, weights
```

错误示例：以为必须按 b1、b2 顺序生成pseudo

```text
# 错误理解：self-attention 必须像 RNN 一样一步一步算。
b1 = attention_for_position(1)
b2 = attention_for_position(2, depends_on=b1)
b3 = attention_for_position(3, depends_on=b2)

# 修正：self-attention 中 b1、b2、b3 可以由同一个矩阵乘法并行得到。
```

注意：如果是自回归生成文本，常会额外加 causal mask，让当前位置不能看未来位置。那是另一类任务设置；P13 这里讲的是一般 self-attention 的并行能力和可加 mask 的结构。

# 常见误区与修正 {#mistakes}
<!-- tag: pitfalls -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：b1 到 bL 必须依序产生
错因：把 attention 想成 RNN。后果：看不懂它的并行优势。修法：用矩阵乘法一次生成所有输出。

## box redsoft
### 误区 2：attention matrix 是学习参数
错因：看到 A 就以为是权重表。后果：忽略它由当前输入动态生成。修法：WQ/WK/WV 是参数，A 是 QK 相乘结果。

## box redsoft
### 误区 3：multi-head 只是重复计算
错因：没有理解多种相关性。后果：无法解释 head 的意义。修法：每个 head 学不同 Q/K/V 子空间。

## box redsoft
### 误区 4：self-attention 天然懂顺序
错因：把投影片编号当成模型输入。后果：漏掉 positional encoding。修法：位置资讯需要额外加入。

## box redsoft
### 误区 5：越 flexible 永远越好
错因：只看表达能力。后果：忽略资料量少时 CNN 等结构先验可能更稳。修法：结合资料量、任务先验和 overfitting 风险判断。

## box redsoft
### 误区 6：长序列只是慢一点
错因：没看到 L² attention matrix。后果：低估显存和训练成本。修法：检查 full、truncated、efficient attention 的取舍。

## box soft
### 现在你应该能做什么
把 self-attention 的表达能力、并行能力、顺序缺口和计算成本分开讨论。

# 术语与检查：把 P13 讲成一张工程图 {#terms}
<!-- tag: terms / checklist -->

公开讲 self-attention 时，先给直觉：每个位置都拿自己的 query 去问全序列的 key，再按权重读取 value。矩阵化只是把这个动作对所有位置同时做；multi-head、位置编码和高效变体，是在“看什么、从哪里看、算不算得起”三个问题上做补强。

| 术语 | 一句话解释 | 最容易混的点 |
| --- | --- | --- |
| Q/K/V | Query 发问，key 负责被匹配，value 提供被汇总的内容。 | WQ/WK/WV 是参数； Q/K/V 是当前输入算出的表示。 |
| Attention matrix | 所有位置两两相关性的表，尺寸是 L×L 。 | 它不是固定参数，换一段输入就会重新计算。 |
| Multi-head | 让不同 head 在不同子空间学习不同关系。 | 不是把同一件事无意义地重复很多遍。 |
| Positional encoding | 给本来不懂顺序的 attention 补位置资讯。 | 投影片编号不是模型输入，位置必须显式加入。 |

## box soft
### Worked example
若 `L=4, d=8`，输入 `X` 可写成 `4×8`。投影后 `Q/K/V` 仍有 4 个位置；`QK^T` 得到 `4×4` attention matrix；softmax 后乘 `V`，输出仍是 4 个上下文化向量。

## box info
### 公开分享检查表
- 先讲“每个 token 查全局”这个直觉。
- 再讲 `L²` 成本，不把 attention 只说成万能架构。
- 最后用 CNN/RNN/GNN 对照，说明结构先验和并行性的取舍。

### 现在你应该能做什么
不用公式推导，也能把 P13 的 self-attention 讲成“动态查表 + 并行矩阵乘法 + 工程取舍”。
