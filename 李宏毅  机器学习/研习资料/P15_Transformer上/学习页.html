P15｜Transformer（上）

Transformer 是一种 sequence-to-sequence model，简称 seq2seq。它处理的任务是：输入是一段 sequence，输出也是一段 sequence，而且输出长度由模型自己决定。语音辨识、机器翻译、语音翻译、聊天机器人、问答、摘要，甚至一些看起来不像序列输出的任务，都可以被改写成 seq2seq。

本节的概念地图

map

Seq2seq 的核心是输出长度不固定。前面 self-attention 讲过 sequence labeling：输入 N 个向量，输出 N 个 label。而 seq2seq 不要求输出和输入一样长，模型需要自己决定什么时候继续输出、什么时候停止。

Encoder 的基本功能是输入一串向量、输出一串同长度向量。这个功能 RNN、CNN、self-attention 都可以做；Transformer encoder 选择 self-attention，并用 residual、LayerNorm、feed-forward 组成可堆叠的 block。

现在你应该能做什么

用“输入序列、输出序列、输出长度由模型决定”一句话抓住 seq2seq。

Seq2seq：输出多长由机器决定

sequence-to-sequence

语音辨识是典型 seq2seq。输入是一段声音讯号，可以表示成一串 frame 向量；输出是文字序列。声音长度和文字长度有关系，但没有固定比例。模型必须听内容，自己决定输出几个字。

语音翻译更直接：输入某种语言的声音，输出另一种语言的文字。课程用台语例子说明，很多语言文字系统不普及，先做语音辨识再翻译未必可行；直接训练声音到文字的 seq2seq 模型，虽然粗暴，却可能有效。

现在你应该能做什么

判断一个任务是否需要模型自己决定输出长度。

Seq2seq 的威力：很多任务可以硬改写

applications

聊天机器人可以看成 seq2seq：输入一句话，输出一句回应。训练资料可以来自大量对话，例如影视剧台词。模型学习“看到这句话，接下来应该输出什么回应”。

一些看起来不是序列输出的任务也能硬解。文法剖析输出的是树，但树可以用括号序列表示；multi-label classification 输出标签集合，标签数量不固定，也可以让模型依序输出标签；object detection 也可以把框和类别转成序列输出。课程强调：这不一定是每个任务的最佳专用模型，但说明 seq2seq 很通用。

现在你应该能做什么

把一个非典型任务改写成“输入序列 → 输出序列”，同时指出这种硬解的局限。

Encoder / Decoder：读进去，再生出来

architecture

一般 seq2seq model 会分成 encoder 和 decoder。Encoder 负责处理输入 sequence，把每个输入位置变成带上下文的表示。Decoder 负责根据 encoder 的结果生成输出 sequence，并决定输出多长。

在 Transformer 里，encoder 的核心模块是 self-attention。它输入一串向量，输出另一串同长度向量。多个 encoder block 堆叠后，输出的每个向量都可以看作对输入 sequence 的更深层表示。

现在你应该能做什么

解释为什么 P15 讲 encoder 还没有真正讲“如何生成输出”。

Transformer Encoder Block 里到底有什么

encoder

课程先给出简化版：一个 block 先做 self-attention，再做 fully connected feed-forward network。Self-attention 负责让每个位置看全序列；feed-forward network 负责对每个位置的向量做进一步非线性变换。

Residual connection 的直觉是保留原始信息和梯度通路。LayerNorm 则让每个向量内部的维度分布更稳定。多个这样的 block 重复 N 次，就形成 Transformer encoder。

现在你应该能做什么

按顺序复述一个 Transformer encoder block 的主要组件。

LayerNorm：不是 BatchNorm 换个名字

normalization

P14 刚讲过 BatchNorm：它对同一个 dimension，在一个 batch 的不同 examples 上计算 mean 和 standard deviation。LayerNorm 不看 batch，它对同一个 example 的一个向量内部，不同 dimensions 计算 mean 和 standard deviation。

课程还提到，为什么 LayerNorm 放在 residual 之后，或者能否放到 block 输入处，这不是绝对定论。原始 Transformer 是一种经典设计，但后续研究提出了不同顺序，有时效果更好。

现在你应该能做什么

说明 LayerNorm 和 BatchNorm 的统计对象差在哪里。

原始 Transformer 不是唯一最优答案

design

课程特别提醒：原始 Transformer 论文里的 encoder 结构很重要，但不代表它是永远最优的架构。比如 LayerNorm 放在哪里，就有后续论文专门研究。有些变体把 LayerNorm 放到 block input 处，也可能得到更好结果。

你应该把 Transformer encoder block 看成一个经过验证的强基线，而不是不能动的圣旨。学会原始结构之后，才有能力理解 Pre-LN、Post-LN、NormFormer 等后续设计。

现在你应该能做什么

回答“为什么 Transformer 长这样”时，能区分经典设计、经验选择和可研究的替代方案。

最小代码：Encoder Block 的骨架

minimal

下面是概念版代码，只表达 P15 的结构。真实实现还会包含 dropout、mask、multi-head 维度拆分、参数初始化等细节。关键是读出两个 residual 分支和两次 LayerNorm。

代码
class EncoderBlock:
 def forward(self, x):
 # x: [length, dim]，已经加上 positional encoding
 attn_out = self.self_attention(x)
 x = self.layer_norm1(x + attn_out) # residual + LayerNorm

 ff_out = self.feed_forward(x)
 x = self.layer_norm2(x + ff_out) # residual + LayerNorm
 return x

# 错误理解：Transformer encoder = self-attention 一层。

# 修正：还要考虑 positional encoding、residual、LayerNorm、feed-forward 等结构。

常见误区与修正

pitfalls

误区 1：seq2seq 输出长度等于输入长度

错因：混淆 sequence labeling。后果：无法理解翻译和语音辨识。修法：seq2seq 的输出长度由模型决定。

误区 2：所有任务都用 seq2seq 就最好

错因：只看到通用性。后果：忽略任务专用模型可能更强。修法：seq2seq 是强大通用框架，不一定是最佳专用解。

误区 3：P15 已经讲完 Transformer

错因：只看到 encoder。后果：不知道 decoder 如何生成输出。修法：P15 是 encoder，P16 继续 decoder。

误区 4：encoder block 只有 self-attention

错因：忽略真实 block。后果：看不懂论文图。修法：attention 后还有 residual、LayerNorm、FFN、residual、LayerNorm。

误区 5：LayerNorm 和 BatchNorm 一样

错因：只看名字都有 normalization。后果：统计维度说错。修法：BN 跨 batch，LN 在单个向量内部。

误区 6：原始论文结构不能改

错因：把经典设计当绝对最优。后果：无法理解后续 Transformer 变体。修法：先掌握原始结构，再理解改进动机。

现在你应该能做什么

诊断别人是否把 seq2seq、Transformer encoder、LayerNorm 的概念混在一起。

术语与检查：先分清 seq2seq 和 encoder

terms

P15 容易混在一起的点有两个：seq2seq 是任务框架，Transformer 是一种模型架构；encoder 是读入模块，decoder 是生成模块。本页只把 encoder block 打稳，decoder 的生成细节留到 P16。

Worked example

语音翻译可以写成 seq2seq：输入是很多声学 frame，输出是目标语言句子，长度通常不同。Transformer encoder 先读声音特征，产生上下文向量；后续 decoder 再决定输出多少 token。

小检查表

  • 任务是否真的需要长度可变输出？
  • 讲 Transformer encoder 时是否包含 attention、FFN、residual、LayerNorm？
  • 有没有把“通用可用”误说成“所有任务最优”？

现在你应该能做什么

面对一个任务，先判断它是不是 seq2seq，再决定是否需要完整 encoder-decoder，而不是默认套一种模型。
