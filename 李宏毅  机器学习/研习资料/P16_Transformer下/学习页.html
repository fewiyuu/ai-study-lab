Transformer（下）：Decoder、Cross-attention 与训练技巧

P15 讲 encoder：读入 sequence，输出一串上下文向量。P16 讲 decoder：它如何从特殊符号 BOS 开始，一步一步产生 token，把自己前一步的输出当作下一步输入，直到输出 END 停止。

Decoder 里有 masked self-attention，因为生成第 t 个 token 时，未来 token 还不存在，不能偷看右边。Decoder 还需要 cross-attention，用自己的 query 去 encoder 输出里抽取信息。训练时，每一步输出都像一次分类，用 cross entropy 对齐正确 token；但训练时常用 teacher forcing，测试时却要吃自己的输出，于是产生 exposure bias。

本节的概念地图

map

Autoregressive decoder 从 BOS 开始。它先预测第一个 token，再把这个 token 当作下一步输入；第二步预测第二个 token，再继续往后。模型需要学会在合适时机输出 END，否则序列会无限延续。

Decoder block 和 encoder block 很像：都有 attention、residual、normalization、feed-forward。关键差异有两处：第一，decoder 的 self-attention 是 masked，只能看左边已经生成的 token；第二，decoder 中间多了 cross-attention，用 decoder hidden state 去读取 encoder output。

训练时，我们知道正确答案序列，因此常把正确前缀喂给 decoder，这叫 teacher forcing。每一步都预测下一个 token，用 cross entropy 训练。但测试时没有正确前缀，decoder 只能吃自己的输出，这会带来训练/测试不一致。

BOS→ masked self-attention→ cross-attention→ softmax token→ feed back→ END

Autoregressive Decoder：一步一步吐 token

AR decoder

Autoregressive 的意思是“后一步依赖前一步”。语音辨识里，encoder 读声音，decoder 先吃 BOS，输出第一个文字的概率分布。分数最高的 token 被选出来，作为下一步输入。这个过程重复，直到输出 END。

Vocabulary 决定每一步 softmax 的长度。中文语音辨识可以把常用中文字作为输出单位；英文可以用字母、词、subword。每一步 decoder 输出的向量都是一个分布，选哪个 token 就是一次多类别分类。

AR decoder 的风险是 error propagation。如果第一步输出错了，下一步就会看到错误输入，可能一步错步步错。这也是后面 teacher forcing、scheduled sampling 等技巧要处理的问题。

BOS→机→器→学→习→END

Masked Self-attention：不能偷看未来

causal mask

Encoder 一次读完整个输入，所以每个位置可以看所有位置。Decoder 不一样：生成第 2 个 token 时，第 3、第 4 个 token 还没有产生。如果让它在训练时看到右边未来答案，测试时就做不到同样条件，训练会作弊。

Masked self-attention 的规则是：第 t 个位置只能 attend 到 1...t 的位置，不能 attend 到 t+1...T。产生 b₂ 时只看 a₁,a₂；产生 b₃ 时只看 a₁,a₂,a₃。

NAT Decoder：一次生成整句的诱惑与代价

non-autoregressive

Non-autoregressive decoder，简称 NAT，不是一字一字生成，而是一次产生整个输出序列。它可以吃一排 BOS token，同时输出多个 token，因此速度更快，也更容易显式控制长度。

长度问题需要额外处理。一种做法是训练一个 classifier，根据 encoder output 预测输出长度；另一种做法是给一个最大长度的 BOS 序列，让模型输出 END 后的部分作废。语音合成里，能控制输出长度有时很有用，例如调整说话速度。

但 NAT 通常 performance 不如 AR decoder，常需要许多技巧才能逼近 AR。课程提到 multi-modality 问题：同一个输入可能对应多种合理输出，如果并行独立地产生每个位置，协调这些位置会更困难。

Decoder生成方式优点常见问题
AR逐 token 生成质量通常较好速度慢，error propagation。
NAT并行生成整段快，可控长度质量较难追上 AR。

Cross-attention：Decoder 如何读取 Encoder

cross attention

Cross-attention 是 encoder 和 decoder 之间的桥。Self-attention 的 Q/K/V 都来自同一串向量；cross-attention 不同：query 来自 decoder 当前状态，key 和 value 来自 encoder 输出。

直觉是：decoder 现在要生成某个 token，于是拿自己的 query 去问 encoder：“输入里哪些位置对我现在最重要？”Encoder 输出提供 key 让它匹配，提供 value 让它抽取内容。加权求和后得到的向量再进入后面的 feed-forward network。

语音辨识中的 attention 图常呈现从左上到右下的斜线：输出第一个字母时关注前段声音，输出后面的字母时注意力逐渐往右移动。这说明 decoder 在不同生成时刻从 encoder 取不同位置的信息。

Q = decoder hidden state
K,V = encoder outputs
context = softmax(QKT)V

训练：每一步都是分类，但输入常偷看正确答案

teacher forcing

训练 seq2seq 需要成对资料。例如语音辨识要有声音讯号和正确文字。Decoder 每一步输出一个 vocabulary 分布，正确 token 用 one-hot 表示。我们计算输出分布和正确 one-hot 的 cross entropy，并把所有时间步的 loss 加总。

注意 END 也要训练。如果正确输出是“机器学习”四个字，第五步正确答案应该是 END，模型要学会停止。

训练时常用 teacher forcing：decoder 的输入不是自己前一步预测，而是正确答案前缀。例如要训练第三步输出“学”，就喂给它 BOS, 机, 器。这让训练稳定，但和测试时“只能看自己输出”的条件不同。

Training

输入正确前缀：BOS, 机, 器 → 目标输出 学。

Inference

输入模型自己前面输出的 token；如果前面错了，后面也会受影响。

常见技巧：copy、guided attention、beam search、scheduled sampling

tips

Copy mechanism 让 decoder 从输入复制词。对聊天机器人、摘要等任务很有用：人名、专有名词、文章中的关键短语，不一定要模型从 vocabulary 里“创造”，可以直接从输入拷贝。

Guided attention 用任务先验约束 attention 形状。语音辨识或语音合成常希望 attention 大致从左到右移动，如果乱跳，可能漏听或漏念。用 monotonic attention、location-aware attention 等方法，可以引导注意力更符合任务结构。

Beam search 不是每一步只选最高分 token，而是保留若干条候选路径，找较好的整体序列。对语音辨识、翻译等答案较明确的任务常有帮助；对故事续写、开放聊天、TTS 等需要随机性或多样性的任务，贪心或 beam search 不一定最好。

Evaluation mismatch 也要注意。训练常最小化 token-level cross entropy，但作业或任务可能看整句 BLEU score。BLEU 不容易直接微分，所以有人把句子级指标当 reinforcement learning 的 reward 来优化，但这通常更难。

Exposure bias 来自 teacher forcing：训练时看正确前缀，测试时看自己输出。Scheduled sampling 的思路是训练时偶尔喂模型自己的错误输出，让它更习惯测试情境；但在 Transformer 上会影响并行化，需要特别设计。

最小代码：训练与测试的 decoder 差异

minimal code

下面代码不是完整 Transformer，只是展示 P16 的关键差异：训练时使用正确前缀，测试时循环使用模型预测，并在预测到 END 时停止。

正确示例：teacher forcing 与 inferencepython
def train_step(encoder_out, target):
    # target = [机, 器, 学, 习, END]
    decoder_in = [BOS] + target[:-1]   # teacher forcing
    logits = decoder(decoder_in, encoder_out, causal_mask=True)
    loss = cross_entropy(logits, target)
    return loss

def infer(encoder_out, max_len):
    ys = [BOS]
    for _ in range(max_len):
        logits = decoder(ys, encoder_out, causal_mask=True)
        next_token = choose_token(logits[-1])
        ys.append(next_token)
        if next_token == END:
            break
    return ys[1:]

常见误区与修正

pitfalls

误区 1：decoder 可以看完整答案

错因：混淆训练并行和生成因果性。后果：模型偷看未来。修法：masked self-attention 只能看左侧前缀。

误区 2：不用 END，模型自然会停

错因：忽略输出长度由模型决定。后果：生成无法停止或靠外部硬截断。修法：把 END 当作特殊 token 训练。

误区 3：cross-attention 的 Q/K/V 都来自 decoder

错因：套用 self-attention。后果：说错 encoder-decoder 交互。修法：Q 来自 decoder，K/V 来自 encoder。

误区 4：teacher forcing 测试时也能用

错因：忘了测试没有正确答案。后果：低估 exposure bias。修法：区分训练输入和 inference 输入。

误区 5：beam search 永远最好

错因：只看全局高分路径。后果：开放生成可能重复、僵硬。修法：答案明确任务常用 beam，需要创造性的任务可加随机性。

误区 6：cross entropy 最小就等于 BLEU 最高

错因：混淆 token-level loss 和 sentence-level metric。后果：验证选择指标不匹配。修法：训练看 CE，模型选择可看任务指标。

术语与检查：Decoder 的三条边界

terms / checklist

讲 decoder 时最重要的边界有三条：生成时只能看左边，读取输入时通过 cross-attention，训练时看到正确前缀但测试时只能吃自己的输出。这三条边界决定了 mask、teacher forcing、exposure bias 和 beam search 为什么重要。

术语一句话解释易混点
Autoregressive一步一步生成，下一步依赖前面已生成 token。训练可并行算 loss，但因果 mask 不能取消。
Causal mask第 t 个位置只能看 1...t。不是为了省计算，而是为了防止偷看未来。
Cross-attentiondecoder 用 Q 读取 encoder 的 K/V。Q 来自 decoder，K/V 来自 encoder。
Teacher forcing训练时喂正确前缀，让每步预测下一个 token。测试时没有正确前缀，因此会产生 exposure bias。

Worked example

目标是“机器学习 END”时，训练第三步输出“学”，decoder 输入通常是 BOS, 机, 器；测试时如果前两步生成成 BOS, 机, 其，第三步就必须基于错误前缀继续预测。

小检查表

  • decoder self-attention 是否启用 causal mask？
  • END 是否作为真实目标训练？
  • 训练代码中 decoder input 和 target 是否错位一格？
  • 解码策略是 greedy、beam 还是 sampling，是否符合任务目标？
