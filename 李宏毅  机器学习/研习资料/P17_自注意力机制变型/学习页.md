# 本节的概念地图 {#goals}
<!-- tag: map -->

所有变体都围绕一个问题：标准 attention 的核心中间物是 `N × N` attention matrix。N 是 sequence 长度。N 越长，这张矩阵越贵。

第一类方法是稀疏化：不是所有 query-key pair 都算，只算局部、跳步、global token 或随机连接。第二类方法是让模型或算法决定哪些 pair 可能重要，例如 Reformer/Routing Transformer 用 clustering，Sinkhorn 用可学习排序网络。

第三类方法认为 attention matrix 可能是 low-rank 或有重复信息，于是只保留代表性的 key/value，例如 Linformer、Compressed Attention。第四类方法从公式下手，用 kernel/feature map 改写 softmax attention，先聚合 K/V，再让每个 Q 去读，避免显式 N×N。最后，Synthesizer 进一步问：attention matrix 一定要由 QK 算出来吗？

# N² 痛点：什么时候值得加速 attention {#pain}
<!-- tag: complexity -->

Transformer 里不只有 self-attention，还有 feed-forward network 等模块。如果序列不长，self-attention 不一定 dominate 总计算；这时换一个更快的 attention 变体，整体速度提升可能有限。

当 N 很大时，情况不同。图片、长文档、长语音都可能让 attention matrix 成为主要成本。比如 256×256 的图像如果每个 pixel 是 token，N 是 65,536，完整 attention matrix 有超过 40 亿个位置对。

# 稀疏规则：人先决定哪些格子不用算 {#sparse}
<!-- tag: sparse attention -->

**Local / truncated attention** 只看邻近位置。它适合局部信息足够的任务，但如果只看小窗口，就很像 CNN；远距离依赖可能被切断。

**Strided attention** 不只看相邻位置，而是隔几个位置看更远处。例如每隔 2 或 3 个 token 连接一次，扩大感受范围。

**Global attention** 加入特殊 token。Global token 会 attend 到所有 token，也被所有 token attend，像信息中转站。可以从原本序列中挑某些 token 当 global，也可以额外插入特殊 token。

**BigBird / Longformer** 常把多种规则混合到不同 head 里。一个 head 做 local，一个 head 做 global，一个 head 做 random 或 strided。课程的口径很直白：小孩子才做选择，很多好模型就是全部都用。

# 聚类与学习 mask：让相近的 Q/K 才互相看 {#learned}
<!-- tag: routing -->

人手写 local 或 global 规则不一定最好。另一个思路是先估计哪些 query/key 可能真的相关，只算那些 pair。Reformer 和 Routing Transformer 都使用类似 clustering 的想法：把相近的 query/key 分到同一群，同群才计算 attention。

Clustering 本身也要成本，所以实际会使用快速近似方法。目标足够快地把可能相关的向量放到一起。不同论文使用不同 routing 技术。

Sinkhorn sorting network 更进一步：用另一个 network 学出哪些位置要算 attention。它产生近似 binary mask，而且这个从连续输出到离散 mask 的过程要尽量可微，才能和主模型一起训练。真实实现还有分块、低解析度 mask 等细节，否则生成 mask 本身也会太贵。

Reformer用快速聚类或 hashing，把相近 token 放一起。
Routing Transformer用 routing/clustering 决定该看的群。
Sinkhorn让另一个网络学习排序或 mask。

# 低秩与代表性 key：矩阵本来就有重复 {#lowrank}
<!-- tag: low rank -->

有些论文观察到 attention matrix 往往接近 low-rank：很多 column 彼此相关，信息重复。既然完整 N×N 矩阵里有冗余，就可以只保留 K 个代表性 key/value，把矩阵从 N×N 变成 N×K。

Compressed Attention 可以用 CNN 等方式把长 key sequence 压短；Linformer 则用一个投影矩阵，把 N 个 key/value 线性组合成 K 个代表性向量。每个代表性 key 可以看成原始 key 的某种组合。

为什么通常压 key/value，不一定压 query？因为输出长度通常等于 query 数。如果任务需要每个输入位置都有输出，压 query 会让输出序列变短；但如果任务只需要一个整体分类 label，也可能压 query 或聚合 query。

$$
Full: attention matrix size N × N
 Representative keys: attention matrix size N × K, where K ≪ N
$$

# Linear Attention：换矩阵相乘顺序 {#linear}
<!-- tag: linear attention -->

标准 attention 可以粗略看成 `O = softmax(QKᵀ)V`。如果先算 `QKᵀ`，会显式产生 N×N 矩阵。Linear attention 的核心是用 feature map 把 softmax kernel 改写成 `φ(q)ᵀφ(k)`，于是可以先把所有 `φ(k)` 和 value 聚合成一个共享模板，再让每个 `q` 去读取。

忽略 softmax 细节时，矩阵乘法结合律告诉我们：`(QKᵀ)V` 和 `Q(KᵀV)` 结果形态类似，但计算成本不同。因为 N 往往远大于向量维度 D，先算 `KᵀV` 可以避免 N×N 中间矩阵。

Linear Transformer、Performer、Random Feature Attention、Efficient Attention 等方法差别在于如何选择或近似这个 feature map `φ`。核心共同点是：不要显式构造完整 attention matrix。

$$
Standard path: QKT → N×N matrix → multiply V
 Linear path: KTV → shared templates → multiply Q
$$

# Synthesizer 与 Attention-free：QK 一定必要吗 {#beyond}
<!-- tag: beyond QK -->

Synthesizer 问了一个很反直觉的问题：attention matrix 一定要由 query 和 key 算出来吗？它有版本直接由输入生成 attention matrix，也有更极端版本把 attention weights 当作模型参数的一部分。这样 attention matrix 不随输入改变，却仍然可以有不错表现。

这个结果迫使我们重新思考 attention 的价值。也许 attention 不只是“动态找相关位置”，还提供了某种混合 token 的结构偏置。再往后，还有一些 attention-free 方法尝试用 MLP 或其他结构处理 sequence，甚至不使用 attention。

Long Range Arena 这类 benchmark 会同时比较速度、performance 和 memory。课程的结论不同方法在速度和效果之间各有取舍：local 很快但效果可能差；BigBird 综合规则；linear/Performer 很快但表现可能下降；Synthesizer 则挑战我们对 attention 的理解。

| 方法 | 核心想法 | 主要取舍 |
| --- | --- | --- |
| BigBird / Longformer | 混合 local/global/random 等稀疏规则 | 结构先验强，适合长序列。 |
| Reformer / Routing | 相近 token 聚在一起才算 attention | 需要快速 routing，近似可能有误差。 |
| Linformer / Compressed | 只保留代表性 key/value | 依赖 low-rank 假设。 |
| Linear / Performer | 避免显式 N×N，用 kernel 近似 | 速度快，近似质量要看方法。 |
| Synthesizer | attention matrix 不一定来自 QK | 挑战直觉，解释空间更开放。 |

# 最小代码：几种 mask 的生成直觉 {#code}
<!-- tag: minimal code -->

下面的伪代码只展示 sparse attention 的 mask 思路。真实模型会用高效 kernel，不会真的用 Python loop 慢慢填矩阵。

正确示例：local + global + random maskpython

```text
def make_sparse_mask(N, window=2, global_ids=(0,), random_pairs=()):
 mask = zeros(N, N)
 for i in range(N):
 for j in range(max(0, i-window), min(N, i+window+1)):
 mask[i, j] = 1 # local attention
 for g in global_ids:
 mask[g, :] = 1 # global token attends to all
 mask[:, g] = 1 # all tokens attend to global token
 for i, j in random_pairs:
 mask[i, j] = 1 # random attention
 return mask
```

错误示例：以为 mask 一定免费pseudo

```text
# 错误理解：只要学习一个 mask，就一定比 full attention 快。
mask = another_network(x) # 如果这个网络本身也要 O(N^2)，就没有省到。

# 修正：mask 生成也必须足够便宜，或用低解析度/分块近似。
```

# 常见误区与修正 {#mistakes}
<!-- tag: pitfalls -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：任何序列都必须换高效 attention
错因：忽略 self-attention 可能不是总瓶颈。后果：换了变体却没明显加速。修法：先看 N 和模块耗时。

## box redsoft
### 误区 2：local attention 等于完美替代 full attention
错因：只看速度。后果：远距离依赖被切断。修法：结合 global、strided、random 或多层传播。

## box redsoft
### 误区 3：global token 只是普通 token
错因：没看 attention pattern。后果：无法解释信息中转。修法：global token attend all，也被 all attend。

## box redsoft
### 误区 4：学习 mask 一定省计算
错因：漏算 mask generator 成本。后果：加了网络反而更慢。修法：mask 生成要低成本、分块或近似。

## box redsoft
### 误区 5：Linformer 和 Linear Transformer 是同一个
错因：名字相近。后果：混淆代表性 key 与换乘法顺序。修法：Linformer 压 key/value；Linear attention 避免 N×N。

## box redsoft
### 误区 6：Synthesizer 表示 attention 没用
错因：过度解读。后果：忽略不同任务与结构偏置。修法：它挑战 QK 必要性，不等于所有 attention 都无价值。

# 术语与检查：先问它省了什么 {#terms}
<!-- tag: terms / checklist -->

高效 attention 的直觉减少 `N×N` attention matrix 的成本。公开讲解时可以先把方法归到四类：少算格子、压缩 key/value、换乘法顺序、甚至不用 QK 产生权重。

| 类别 | 代表方法 | 核心取舍 |
| --- | --- | --- |
| Sparse attention | Local、strided、global、BigBird、Longformer | 只算部分位置对，远距离连接需要额外设计。 |
| Routing / clustering | Reformer、Routing Transformer、Sinkhorn | 让相近 token 才互相看，但 mask 生成本身也有成本。 |
| Low-rank / projection | Linformer | 压缩 key/value 方向，假设 attention matrix 有冗余。 |
| Linear / beyond QK | Linear attention、Performer、Synthesizer | 避免显式形成 N×N ，但权重含义或近似条件会改变。 |

## box soft
### Worked example
若文本长度 `N=4096`，full attention 有约 1,677 万个位置对。Local window 只看每个 token 前后 64 个，位置对大约变成 `N×128` 级别，但跨章节信息需要 global token 或多层传播补回来。

## box info
### 小检查表
- 当前瓶颈真的是 attention，还是 FFN、IO 或显存碎片？
- 被删掉的位置对是否承载任务需要的远距离依赖？
- 加速方法是否引入额外排序、聚类或近似误差？

## box soft
### 现在你应该能做什么
读到一个新 X-former 时，先用“节省对象、额外成本、丢失连接、适用场景”四项做快速判断。
