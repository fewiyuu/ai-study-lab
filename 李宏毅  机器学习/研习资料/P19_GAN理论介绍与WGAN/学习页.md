# 本节的概念地图 {#goals}
<!-- tag: map -->

这一节不要先背公式，要先抓住角色变化。P18 里 discriminator 像一个真假判断器；P19 里它更像一个帮助估计分布距离的函数。我们只能从真实资料库取样、从 generator 取样。

# 生成目标：最小化两个分布的距离 {#objective}
<!-- tag: objective -->

在生成任务中，generator 不只是输出一张图片。给很多不同 `z`，它会输出很多图片，这些图片整体形成一个分布，记作 `p_g`。真实资料库里的图片也形成一个分布，记作 `p_data`。我们想要的 generator，是让 `p_g` 和 `p_data` 尽量像的 generator。

如果用普通训练语言说，loss 就是两个分布之间的 divergence。Divergence 可以先理解为“分布之间的差异度”：越大代表越不像，越小代表越接近。于是理想目标是找到参数 `θ`，让 `Div(p_g, p_data)` 最小。

困难在于：我们通常算不出这个 divergence。真实图片分布在高维空间里，没有一个简单公式；generator 输出分布也由深度网络隐式定义，写不出易算密度函数。GAN 的厉害之处是：只要能从两个分布取样，就能借助 discriminator 形成可优化的目标。

$$
Find Gθ such that
 Div(pg, pdata) is as small as possible.
 But we usually only have samples, not closed-form densities.
$$

解释为什么 GAN 理论要讨论 `p_g` 与 `p_data`，而不只讨论单张图片的像素误差。

# Discriminator 怎样和 JS divergence 扯上关系 {#discriminator}
<!-- tag: JS divergence -->

训练 discriminator 时，我们从 `p_data` 取真实样本，从 `p_g` 取生成样本。真实样本希望 `D(y)` 大，生成样本希望 `D(y)` 小。原始 GAN 常写成最大化下面的 objective：

$$
maxD V(D, G)
 V(D, G) = Ey~p_data[log D(y)] + Ey~p_g[log(1 - D(y))]
$$

这个式子看起来突然，其实它就是 binary classification 的 cross entropy 乘上负号。真实样本是 class 1，生成样本是 class 0。最大化这个 objective，等价于训练一个二分类器去分辨 real/fake。

关键结论是：如果给定 generator 后，discriminator 有足够能力并被训练到最优，那么这个最大值和 `p_g`、`p_data` 的 JS divergence 有关。课程没有展开完整证明，你可以先掌握直觉：两组样本越容易被分开，最优 discriminator 的 objective 越大；两组样本越混在一起，它越难分开，objective 越小。

## box soft
### 容易分开
说明 `p_g` 与 `p_data` 差异大，D 的最大值较大。

## box info
### 难以分开
说明两分布更接近，D 的最大值较小。

把 discriminator 的 objective 翻译成二分类任务，并说明它为何可用来估计分布差异。

# JS divergence 的问题：没重叠就看不出远近 {#jsproblem}
<!-- tag: failure mode -->

高维图片分布非常狭窄。随便在像素空间取一点，大概率不是有效图片。真实图片可能落在某个低维 manifold 上，generator 图片也落在另一个狭窄区域上。除非两者刚好重合，否则重叠部分很少。

更实际一点，就算两个真实分布有重叠，我们训练时也只取有限样本。例如取 256 张 real 和 256 张 fake，discriminator 可能轻易把这两堆点分开，甚至靠记忆样本就接近 100% 准确率。这样一来，D 的分类 loss 很难告诉我们 generator 是否真的在进步。

JS divergence 对没有重叠的分布会饱和成常数。两条线隔很远也是同一个值；两条线已经靠近但还没相交，也是同一个值。从优化角度看，这就像地形完全平坦：generator 往正确方向挪近一点，loss 也看不出来。

当两个分布没有重叠时，JS divergence 可以几乎不随距离变化。它知道“还没重叠”，但不知道“已经靠近多少”。

# Wasserstein distance：Earth Mover 的直觉 {#wasserstein}
<!-- tag: earth mover -->

Wasserstein distance 可以直觉地理解为 Earth Mover distance。把一个分布看成一堆土，把另一个分布看成目标形状。你要用推土机把第一堆土搬成第二堆土。所有可能搬法中，平均移动距离最小的那个值，就是 Wasserstein distance。

如果两个分布各自集中在一个点，那么 Wasserstein distance 就是两个点之间的距离。重点是：即使两个分布完全不重叠，只要它们离得更近，Wasserstein distance 就会变小。这给 generator 提供了连续的方向感。

这和眼睛演化的例子很像。如果只能从“完全不像”一步跳到“完全重合”，训练就很困难；但如果每往正确方向移动一点都有反馈，generator 就可以逐步改进。Wasserstein distance 的价值就在这里：分布还没重叠时，也能告诉你靠近了多少。

小交互：Earth Mover 距离
土堆距离**0.60**


怎么观察
红色 fake 分布越靠近蓝色 real 分布，移动土的平均距离越小。这个数值会连续变化，不需要等两者真正重叠。

# WGAN：把 JS 目标换成 Wasserstein 目标 {#wgan}
<!-- tag: WGAN -->

WGAN 的核心想法是：既然 JS divergence 在分布不重叠时给不出好信号，那就换成 Wasserstein distance。这样 generator 稍微靠近真实分布时，目标值也会变化。训练者不必只靠肉眼看图片，critic 的分数也更可能带有训练进展信息。

WGAN 中通常不再把 D 称作 discriminator，而称作 critic。原因是它不再输出真假概率，也不再用 sigmoid 后的二分类解释；它输出一个实数分数。我们希望 real 样本分数高，fake 样本分数低，两者期望分数差对应 Wasserstein distance 的估计。

$$
maxD ∈ 1-Lipschitz
 Ey~p_data[D(y)] - Ey~p_g[D(y)]
 Then train G to make this distance smaller.
$$

这个式子和原始 GAN 的 log 形式不同。它没有 `log D(y)` 或 `log(1-D(y))`，因为它不是二分类交叉熵。它的重点是比较 real 和 fake 的 critic score 差距。Generator 训练时希望 fake 的分数变高，从而缩小 real/fake 的差距。

| 项目 | 原始 GAN | WGAN |
| --- | --- | --- |
| D 的角色 | 二分类 discriminator | 打分 critic |
| 目标直觉 | 估计与 JS divergence 相关的值 | 估计 Wasserstein distance |
| 输出含义 | 真假概率或概率风格分数 | 没有概率意义的实数分数 |
| 关键约束 | 分类器尽量分开 real/fake | critic 必须足够平滑 |

# 为什么 WGAN 需要平滑约束 {#lipschitz}
<!-- tag: 1-Lipschitz -->

Wasserstein 的对偶形式要求 critic 属于 1-Lipschitz 函数。第一次看到这个词可以先理解成“函数不能变化得太剧烈”。输入稍微移动一点，输出不能突然从极大跳到极小。这个限制非常重要。

如果没有平滑约束，critic 可以给 real 样本无限大正分，给 fake 样本无限大负分。只要两组样本不重叠，目标值就能被推到无穷大。这又会失去“距离大小”的意义，和 JS 饱和问题一样让训练信号变坏。

有了平滑约束以后，real 和 fake 如果离得很近，critic 的分数不能差太多；如果离得很远，分数才允许差更多。这样 score gap 才能反映空间距离。早期 WGAN 用 weight clipping 粗糙限制参数范围；后来 WGAN-GP 用 gradient penalty；再后来也常见 spectral normalization。

Weight clipping把参数裁到固定区间，简单但粗糙，不保证真的满足理想约束。
Gradient penalty在 real/fake 连线附近惩罚梯度偏离目标，更贴近平滑需求。
Spectral normalization通过控制层的谱范数限制网络变化幅度，是常用稳定化技术。

### 现在你应该能做什么
解释 WGAN 里 critic 不能随便变成尖锐分类器，否则 Wasserstein 估计会失去意义。

# 理论与实作的缝隙 {#practice-note}
<!-- tag: practice note -->

理论上，给定 generator 后，我们应该把 discriminator 或 critic 训练到最优，然后用这个最优值估计分布距离，再更新 generator。实作上几乎没人每轮都从头初始化 D 并训练到完全收敛，因为太慢。

实际训练通常让 D 继承上一轮参数，只更新几步，然后更新 G。也有人会使用之前 generator 产生过的样本来增加稳定性或资料量。这些技巧不完全等于理论推导，但它们服务于同一个目标：让 critic 提供更可靠的训练信号。

所以学习这节时要同时保留两层视角。理论视角告诉你目标是什么：最小化两个分布距离。实作视角提醒你：我们只能近似这个目标，不能把每个公式都当成训练代码的逐行命令。

# 最小代码：WGAN 目标的形状 {#code}
<!-- tag: minimal code -->

下面伪代码只展示 WGAN 的目标方向。真实训练还要加上 gradient penalty、spectral normalization 或其他约束方法。

正确示例：WGAN critic 和 generator 的目标pseudo python

```text
def train_critic(G, C, real):
 z = sample_noise(len(real))
 fake = G(z).detach()
 score_real = C(real).mean()
 score_fake = C(fake).mean()
 loss_C = -(score_real - score_fake) # maximize the gap
 loss_C += gradient_penalty(C, real, fake)
 update(C, loss_C)

def train_generator(G, C):
 z = sample_noise(batch_size)
 fake = G(z)
 loss_G = -C(fake).mean() # make fake score higher
 update(G, loss_G)
```

错误示例：把 WGAN critic 当 sigmoid 分类器pseudo

```text
# 错误理解：WGAN 仍然用 BCE 训练 D，只是换个名字。
score = sigmoid(C(x))
loss = BCE(score, label)

# 修正：WGAN critic 输出实数分数，目标是 real/fake 期望分数差，
# 还必须加入平滑约束。
```

### 现在你应该能做什么
读到 WGAN 代码时，能检查它是否用了 critic score gap，以及是否有 Lipschitz 约束机制。

# 常见误区与修正 {#mistakes}
<!-- tag: pitfalls -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：GAN 只是让 G 骗 D
错因：停留在拟人化故事。后果：看不懂理论目标。修法：回到 `p_g` 接近 `p_data` 的分布距离。

## box redsoft
### 误区 2：能取样就等于知道分布公式
错因：混淆 sample 和 density。后果：以为 divergence 可以直接算。修法：样本可取，但密度函数通常不可得。

## box redsoft
### 误区 3：D 准确率越高越能说明 G 在进步
错因：把分类性能当生成质量指标。后果：D 可能只是轻易分开有限样本。修法：看训练信号是否反映分布距离。

## box redsoft
### 误区 4：JS divergence 总能衡量远近
错因：忽略不重叠时的饱和。后果：loss 对“靠近一点”没反应。修法：理解 WGAN 换距离的动机。

## box redsoft
### 误区 5：WGAN 不需要约束 critic
错因：只看 score gap。后果：critic 可把分数推到无穷，距离估计失效。修法：加入 weight clipping、gradient penalty 或 spectral normalization。

## box redsoft
### 误区 6：理论公式就是逐行训练代码
错因：忽略近似训练。后果：困惑为什么 D 没有每轮从头训到最优。修法：区分理论目标与实际优化过程。

## box soft
### 现在你应该能做什么
遇到 GAN 稳定化技巧时，先判断它是在改善分布距离估计，还是在限制 critic 的函数形状。

# 术语与检查：从骗过 D 到移动分布 {#terms}
<!-- tag: terms / checklist -->

P19 的核心升级是把 GAN 从拟人化故事改成分布距离问题。G 让 `p_g` 往 `p_data` 靠近；WGAN 的价值在于未重叠时仍能给出距离变化信号。

| 术语 | 一句话解释 | 易混点 |
| --- | --- | --- |
| p_g / p_data | 生成样本分布和真实资料分布。 | 能取样不等于知道密度函数，也不等于能直接算 divergence。 |
| JS divergence | 原始 GAN 最优 D 背后关联的分布差异。 | 两个分布没重叠时可能饱和，看不出远近。 |
| Wasserstein distance | 把一个分布搬成另一个分布的最小移动成本。 | 连续距离信号。 |
| Critic / 1-Lipschitz | WGAN 的实数打分函数必须足够平滑。 | 没有约束时 score gap 可以被放大到失去距离意义。 |

## box soft
### Worked example
想象真实分布集中在数轴 0，生成分布集中在 10。若两者完全不重叠，JS 可能已经饱和；G 从 10 移到 9 时，JS 仍看起来差不多，但 Wasserstein 距离会从 10 变成 9，给出“确实靠近了一点”的信号。

## box info
### 小检查表
- 有没有把 D 的准确率当成生成质量？
- WGAN 代码是否直接用 critic score gap，而不是 sigmoid+BCE？
- 是否有 weight clipping、gradient penalty 或 spectral normalization？

## box soft
### 现在你应该能做什么
看到一个 GAN loss 时，判断它是在做二分类目标、分布距离估计，还是在约束 critic 的函数形状。
