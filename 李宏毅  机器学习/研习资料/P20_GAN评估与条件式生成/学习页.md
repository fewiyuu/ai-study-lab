# 本节的概念地图 {#goals}
<!-- tag: map -->

P20 的第一部分是训练与评估现实。GAN 的两个网络不是独立训练：D 太弱，G 得不到好反馈；G 不进步，D 也没有新挑战。即使用 WGAN、spectral normalization 等技巧，GAN 仍然以难训练出名。

第二部分是评价 generator。人眼看图直观但不客观；分类器置信度可以衡量单张图片清楚不清楚；平均分类分布可以粗略看多样性；FID 则把真实图片和生成图片送进预训练网络，比较中间特征分布距离。每种指标都有边界。

第三部分是 conditional generation。以前的 G 只看 `z`，现在 G 看 `x` 和 `z`。如果 D 只看输出图片，G 可以无视条件；所以 D 也必须同时看条件和输出，并把“图像清楚但条件不匹配”的样本当作负例。

# 为什么 GAN 仍然难训练 {#training}
<!-- tag: training -->

GAN 的困难来自双网络互动。Discriminator 要分辨真实图片和生成图片，generator 要产生能骗过 discriminator 的图片。听起来像竞赛，但更准确地说，它们互相提供成长压力。只要其中一方在某个阶段训练不好，另一方就会失去合适目标。

普通 network 训练时，我们可以看训练 loss 是否下降，再调学习率、batch size、架构。GAN 中每个小阶段都在变化：D 的任务随着 G 的变化而变化，G 的训练信号又取决于 D 的状态。你无法保证每一次 D 或 G 的更新都刚好稳定有效。

Spectral normalization、WGAN、gradient penalty 等方法能缓解问题，但不代表 GAN 从此容易训练。网络架构、学习率、batch size、regularization、更新次数比例都可能影响结果。课程中特别提醒：GAN 是重要技术，也是真实会踩坑的技术。

D 太弱分不出 real/fake，G 不知道该往哪里改。
G 停住总生成类似东西，D 没有新挑战，训练容易僵住。
棋逢对手两者都能持续进步，训练才可能向好结果推进。

### 现在你应该能做什么
看到 GAN 崩掉时，能从 G/D 互动角度排查，而不是只盯单个 loss。

# 为什么用 GAN 生成文字特别难 {#sequence}
<!-- tag: sequence -->

图片输出是连续向量，像素值稍微变化，discriminator 的分数也能有可微变化。文字不同。文本生成通常要在每一步从 token 分布里选一个 token。Token 可以是中文字、英文词、subword 或其他单位；关键是最后会做离散选择。

如果 decoder 参数只发生很小变化，输出分布可能稍微变了，但最大概率 token 没变。对 discriminator 来说，输入句子完全一样，输出分数也完全一样。这样 generator 参数的小变化无法通过 discriminator 分数形成有效梯度。

可以把文本生成当 reinforcement learning 来做，因为离散动作也能用奖励信号训练。但 RL 本身难训练，GAN 也难训练，两者叠加更困难。课程提到曾经有从零训练 language GAN 的论文，但需要大量技巧、很大的 batch size、regularization、pretrain embedding 等工程条件。

错误示例：忽略离散 token 的梯度断点pseudo

```text
# 错误直觉：D 给句子一个分数，G 就能像图片一样直接反传。
probs = decoder(prefix)
token = argmax(probs) # 离散选择，微小参数变化常不改变 token
score = discriminator(token)
backprop(score)
```

# 质量与多样性：评价 generator 不能只看单张图 {#quality}
<!-- tag: evaluation -->

最直觉的评价方法是让人看图：像不像真实图片？这很有用，但不稳定、不客观，也难以大规模比较。于是人们借助预训练 image classifier。把一张生成图片丢进去，如果分类分布很集中，说明 classifier 很确定它看到了某个对象，这可以当作单张图片 quality 的信号。

但只看单张 quality 会被 mode collapse 骗过。Mode collapse 是 generator 来来去去只会产生少数几种样本；单张看可能不错，但多生成几张就发现都差不多。Mode dropping 更隐蔽：生成结果有一定多样性，但只覆盖真实分布的一部分，例如人脸生成只覆盖某些肤色或某些角度。

多样性可以也用 classifier 粗略估计：让 generator 产生很多图，把每张图的分类分布平均起来。如果平均分布很平坦，说明图片覆盖较多类别；如果平均分布集中在一类，可能多样性不足。注意 quality 看单张的分布是否集中，diversity 看许多图片平均后的分布是否分散。

小交互：quality 与 diversity 的拉扯

质量高且多样
mode collapse
质量差但分散
mode dropping


单张质量**0.80**
整体多样性**0.80**


质量高且多样
单张图片清楚，许多样本又覆盖多种类别，这是比较理想的情况。

# Inception Score 与 FID：常见但不完美的指标 {#fid}
<!-- tag: IS / FID -->

Inception Score 使用预训练 Inception network。它希望每张图片的分类分布集中，代表 quality 高；同时希望很多图片的平均类别分布分散，代表 diversity 大。两者结合成一个分数。但 IS 对任务依赖很强。二次元头像即使有不同发色、眼睛、表情，对 ImageNet 分类器来说可能都只是“人脸”或类似类别，多样性就被低估。

FID 的做法不同。它不拿最后类别，而是取 Inception network 进入 softmax 前的 hidden feature，把每张图片表示成一个向量。真实图片形成一组 feature，生成图片形成另一组 feature。再假设两组 feature 近似高斯分布，计算它们的 Fréchet distance。FID 是距离，所以越小越好。

FID 更常用，但也有假设与成本。它假设 feature 分布近似高斯，真实未必如此；还需要足够多样本才能估计稳定。课程作业中不会只看 FID，还会看动画人脸侦测数量，因为单个指标可能给出偏差结论。

| 指标 | 看什么 | 方向 | 常见问题 |
| --- | --- | --- | --- |
| 人眼 | 图片是否看起来真实 | 主观判断 | 不客观、不稳定。 |
| Inception Score | 单张分类集中 + 多张类别分散 | 越大越好 | 依赖分类器类别，不适合所有任务。 |
| FID | 真实与生成图片的 feature 分布距离 | 越小越好 | 高斯假设、需要大量样本。 |
| 任务专属指标 | 例如人脸侦测数量 | 看设计 | 只覆盖任务的一部分。 |

# 评估的边界：复制训练资料也可能拿高分 {#limits}
<!-- tag: limits -->

假设 generator 产生的图片和训练资料一模一样。FID 可能很低，人脸侦测也可能很高，但这不是我们真正想要的 generator。因为训练资料已经在手上，直接抽训练图就行；生成模型的价值在于产生新的、合理的样本。

你可能说，那就把生成图片和训练图片做相似度比对。但这也不够。Generator 可以学会把训练图片全部左右翻转、轻微裁剪或改颜色。它看起来生成了新图，但本质上仍然可能只是记忆训练资料的简单变换。

所以 generator evaluation 本身就是研究题目。好指标要同时考虑质量、多样性、覆盖率、新颖性、是否复制训练集、是否符合条件和任务需求。课程里不要求你掌握所有指标，但要知道：一个数字好看不等于模型真的好。

## box warn
### 低 FID 不保证新颖
如果模型复制训练集，分布距离可以很小，但生成价值很低。

## box info
### 相似度检查也不完美
翻转、裁剪、风格变化都可能绕过简单相似度。

## box soft
### 现在你应该能做什么
评价生成模型时，主动问“它是否只是复制训练资料”这个问题。

# Conditional Generation：把条件 x 加回来 {#conditional}
<!-- tag: conditional GAN -->

到目前为止，generator 只看随机向量 `z`。这叫 unconditional generation：你无法明确控制它输出红眼睛、蓝头发还是某种姿势。Conditional generation 加入条件 `x`，让 generator 接收 `x` 和 `z`，输出符合条件但仍有多样性的 `y`。

Text-to-image 是典型例子。`x` 是文字描述，例如 “red eyes, blue hair”；`z` 决定同一描述下的随机变化；`y` 是图片。文字可以用 RNN、Transformer encoder 或其他编码器变成向量，再送入 generator。训练通常需要 paired data，也就是图片和文字描述成对出现。

关键在 discriminator。若 D 只看图片 `y`，G 只要生成清楚图片就能骗过 D，完全可以无视文字。Conditional GAN 的 D 必须同时看 `x` 和 `y`，判断两件事：图片本身是否真实，图片与条件是否匹配。训练时还要加入 mismatch negative：真实图片配错误文字，也要判为不好。

小交互：D 是否检查条件匹配

D 看 x 和 y，并加入错配负例
D 只看 y
D 看 x 和 y，但没有错配负例


完整 conditional D
D 不只检查图片清楚，也检查文字和图片是否配得上。真实错配样本能逼 D 学会“条件一致性”。

$$
Unconditional: y = G(z)
 Conditional: y = G(x, z), and D sees (x, y)
$$

# Image Translation：从一张图生成另一张图 {#image2image}
<!-- tag: pix2pix -->

Conditional GAN 不只适用于文字到图片，也可以做 image-to-image translation。输入可以是房屋设计图、黑白照片、素描、白天街景、有雾图片；输出可以是真实房屋、彩色照片、实景图、夜晚街景、去雾结果。这类任务常叫 pix2pix。

为什么不用普通 supervised learning 就好？因为同一个输入可能有多种合理输出。例如素描转实景，不同材质、光照、颜色都可能合理。只用像素级监督 loss，模型容易把多种可能平均，结果模糊。GAN loss 能鼓励输出看起来像真实图片。

但只用 GAN 也会有问题：generator 可能想象力过度丰富，在输入没有的地方添东西。文献中常把 supervised loss 和 GAN loss 一起用。监督 loss 让输出贴近成对标准答案，GAN loss 让输出更清晰、更像真实图像分布。

只用监督 loss容易平均多种可能，输出模糊。
只用 GAN loss可能清楚但乱加输入没有的内容。
两者结合既保持条件对应，又提升真实感。

# 最小代码：conditional D 的三类样本 {#code}
<!-- tag: minimal code -->

下面伪代码展示 conditional GAN 的训练重点：D 要看条件和输出，而且要见过“真实但错配”的负例。

正确示例：conditional discriminatorpseudo python

```text
def train_conditional_discriminator(G, D, x, real_y):
 z = sample_noise(len(x))
 fake_y = G(x, z).detach()
 wrong_y = shuffle(real_y) # real image, wrong condition pair

 score_real = D(x, real_y) # matched real pair -> 1
 score_fake = D(x, fake_y) # generated image -> 0
 score_wrong = D(x, wrong_y) # mismatched real pair -> 0

 loss_D = BCE(score_real, 1) + BCE(score_fake, 0) + BCE(score_wrong, 0)
 update(D, loss_D)
```

错误示例：D 只看图片pseudo

```text
# 错误：D 不看文字条件 x。
score = D(y)

# 后果：G 只要生成清楚图片即可，可能完全无视 red eyes / blue hair 等条件。
```

## box soft
### 现在你应该能做什么
检查 conditional GAN 训练资料中是否包含 matched real、generated fake、mismatched real 三类样本。

# 常见误区与修正 {#mistakes}
<!-- tag: pitfalls -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：GAN 有 WGAN 后就容易训练
错因：把技巧当万能药。后果：忽略 G/D 互动失衡。修法：仍要调架构、学习率、正则化和更新比例。

## box redsoft
### 误区 2：单张图好看就代表 generator 好
错因：只看 quality。后果：mode collapse 可能被漏掉。修法：同时看多样性和覆盖率。

## box redsoft
### 误区 3：IS/FID 是绝对真理
错因：忘记指标假设。后果：在二次元、人脸等任务上误读分数。修法：结合任务专属指标与人工检查。

## box redsoft
### 误区 4：低 FID 一定代表模型有创造力
错因：忽略复制训练集。后果：模型可能只是记忆资料。修法：检查新颖性和与训练资料的关系。

## box redsoft
### 误区 5：conditional GAN 的 D 只需看 y
错因：把条件当作 G 的内部事。后果：G 可能无视条件。修法：D 同时看 `x,y` 并训练错配负例。

## box redsoft
### 误区 6：image-to-image 只用 GAN loss 就好
错因：只追求真实感。后果：输出可能乱加内容。修法：常结合监督重建 loss 和对抗 loss。

## box soft
### 现在你应该能做什么
评价 GAN 结果时，不只说“好看/不好看”，而是分解成质量、多样性、覆盖率、条件匹配和新颖性。

# 术语与检查：评价不只看单张图 {#terms}
<!-- tag: terms / checklist -->

P20 的核心是验收标准升级：生成模型既要单张样本像真的，也要覆盖真实分布，还要在条件式任务里匹配条件。一个好看的 cherry-pick 样本不能证明 generator 好。

| 术语 | 一句话解释 | 易混点 |
| --- | --- | --- |
| Quality | 单张样本是否清楚、可信。 | 单张质量高仍可能 mode collapse。 |
| Diversity / Coverage | 多张样本是否覆盖不同模式和真实分布区域。 | 多样不等于每张都好，覆盖不足就是 mode dropping。 |
| FID | 比较真实/生成样本在特征空间的分布距离，通常越小越好。 | 依赖预训练特征和样本量，不是绝对真理。 |
| Conditional D | 判别器同时看条件 x 和输出 y 。 | D 只看 y 时，G 可能无视条件。 |

## box soft
### Worked example
如果 text-to-image 条件是“蓝头发、红眼睛”，D 只看图片时，G 生成清楚的黑发头像也可能过关。让 D 看 `(x,y)`，再加入错配真实对，例如“蓝头发文字 + 黑发真实图”为 fake，才能迫使 G 学条件匹配。

## box info
### 小检查表
- 是否同时看单张质量和样本集合多样性？
- 是否检查复制训练集或过近邻样本？
- conditional GAN 是否有 matched real、generated fake、mismatched real 三类样本？
- image-to-image 是否平衡监督重建 loss 与 GAN loss？

## box soft
### 现在你应该能做什么
为一个生成任务写验收标准时，能把“像真、多样、覆盖、条件匹配、新颖”分开检查。
