# 本节的概念地图 {#goals}
<!-- tag: map -->

Cycle GAN 的目标是 unpaired translation。我们想学一个函数 `G_XY`，把 X 域样本转到 Y 域，例如真人照片转二次元头像。问题是没有配对答案，不能直接用监督学习，也不能训练 conditional discriminator 去判断“这个 X 是否对应这个 Y”。

单独套用 GAN 可以让 `G_XY(x)` 像 Y 域，但它不保证和 `x` 有关。Cycle GAN 加入反向函数 `G_YX`，要求 `G_YX(G_XY(x)) ≈ x`。同时也训练反方向 `G_XY(G_YX(y)) ≈ y`。两个 domain 各有一个 discriminator，分别检查输出是否像目标域。

X samples+
Y samples→
G_XY+
G_YX→
cycle consistency+
D_X / D_Y

# Unpaired data：有两堆资料，但没有对应关系 {#unpaired}
<!-- tag: unpaired -->

在普通监督学习里，我们有成对资料：输入 `x` 和标准答案 `y`。例如同一句英文和中文翻译、同一张素描和真实照片、同一张真人照片和人工绘制的二次元头像。模型可以直接学 `x → y`。

但很多风格转换任务很难收集成对资料。要让真人照片和二次元头像一一对应，可能需要艺术家为每个人画一张对应图，成本极高。我们更容易得到的是两堆独立资料：一堆真人照片，一堆二次元头像。它们都来自各自 domain，但不知道哪张对应哪张。

这种资料不等于完全没资料。我们仍然知道 X 域长什么样，也知道 Y 域长什么样。GAN 能利用这个信息：让输出落在 Y 域分布里。但还需要额外约束，防止模型只学会随机产生一个 Y 域样本。

| 资料类型 | 有什么 | 能直接做什么 |
| --- | --- | --- |
| Paired | 每个 x 都有对应 y | 监督学习、conditional GAN。 |
| Unpaired | 一堆 x，一堆 y，但无对应关系 | 需要 Cycle GAN 这类约束。 |
| 只有 X | 只有源域，没有目标域 | 无法定义要转成什么目标风格。 |

### 现在你应该能做什么
判断一个任务能不能直接用成对监督，还是需要 unpaired translation 方法。

# 单向 GAN 为什么不够 {#single}
<!-- tag: failure -->

一个自然想法是：把输入分布从高斯噪声换成 X 域图片。我们从真人照片中 sample 一个 `x`，送进 generator，输出一张图片。再训练一个 Y 域 discriminator，判断输出像不像二次元头像。这样看起来就能把 X 转成 Y。

问题是这个目标只要求输出像 Y 域，不要求输出和输入有关。Generator 完全可以把真人照片当作噪声来源，无论输入是谁，都输出一张好看的二次元头像。Discriminator 只看“像不像 Y 域”，不会惩罚“这张二次元头像和原真人无关”。

这和 P20 的 conditional GAN 问题类似：如果 D 不看条件，G 会无视条件。但在 unpaired 情况下，我们又没有成对资料来训练 D 判断某个 X/Y 是否匹配。因此需要另一种约束：让转换后的结果必须能转回原输入。

## box redsoft
### 只看目标域
D_Y 只要求输出像 Y。G_XY 可以无视 x。

## box soft
### 需要内容约束
输出不仅要像 Y，还要保留足够信息，能被反向生成器还原。

### 现在你应该能做什么
指出“只用一个目标域 discriminator”为什么不能保证风格转换保留原内容。

# Cycle Consistency：转过去，还要转得回来 {#cycle}
<!-- tag: cycle consistency -->

Cycle GAN 引入两个 generator。`G_XY` 把 X 域转成 Y 域；`G_YX` 把 Y 域转成 X 域。对任意 `x`，先做 `x → G_XY(x)`，再做 `G_YX(G_XY(x))`。如果第二次转换回来的结果和原来的 `x` 很接近，就说明中间的 Y 域图保留了足够的输入信息。

这个约束叫 cycle consistency。图片可以看成向量，所以还原图和原图之间可以用 L1/L2 距离衡量。它不直接告诉模型“这个真人应该对应哪张二次元脸”，但它限制模型不能随便输出和输入无关的目标域图。

# 完整 Cycle GAN：两个 G，两个 D，两个 cycle {#bidirectional}
<!-- tag: full model -->

完整训练是双向的。对 X 域样本，`G_XY` 产生 fake Y，`D_Y` 判断它像不像真实 Y；再用 `G_YX` 把 fake Y 还原成 X，并计算 cycle loss。对 Y 域样本，过程反过来：`G_YX` 产生 fake X，`D_X` 判断它像不像真实 X；再用 `G_XY` 还原成 Y。

所以 Cycle GAN 的目标由几部分组成：X→Y 的 adversarial loss、Y→X 的 adversarial loss、X cycle loss、Y cycle loss。Adversarial loss 保证风格像目标域；cycle loss 保证内容不会被随便丢掉。

| 组件 | 作用 | 如果缺少会怎样 |
| --- | --- | --- |
| G_XY | X 域转 Y 域 | 无法做 X→Y 风格转换。 |
| G_YX | Y 域转 X 域，并帮助还原 | 缺少 cycle 约束。 |
| D_Y | 检查 fake Y 是否像真实 Y | 输出可能不像目标 Y 域。 |
| D_X | 检查 fake X 是否像真实 X | 反向转换质量无法约束。 |

# 理论边界：可逆不等于语义正确 {#limits}
<!-- tag: limits -->

Cycle consistency 只保证“能转回来”，不保证中间语义一定是人想要的。理论上，`G_XY` 可以学到奇怪编码：把眼镜抹掉变成一颗痣，`G_YX` 再看到那颗痣还原成眼镜。甚至两个 generator 都学会左右翻转，也能满足 cycle。

课程提醒：这类奇怪转换理论上可能发生，没有完美保证。但实作中，网络往往偏向比较“懒”的转换。要把眼镜编码成痣再解码回来并不一定比直接保留眼镜容易。因此在很多图像风格转换任务里，Cycle GAN 输出确实会保留输入大结构，只改变风格。

这给我们一个重要判断：Cycle GAN 是强有力的实用约束，但不是语义正确性的数学保证。使用它时，仍需要看结果、设计额外约束、检查是否出现隐藏编码、结构扭曲或语义漂移。

## box warn
### 理论漏洞
只要可逆，就可能满足 cycle；可逆关系不一定是人想要的对应关系。

## box soft
### 实作经验
网络常倾向保留输入结构，改变风格，因此很多任务上效果不错。

## box soft
### 现在你应该能做什么
评价 Cycle GAN 时，同时承认它有用，也知道 cycle loss 不是语义保真的绝对证明。

# 应用迁移：图像、文字、语音都可套同一思路 {#applications}
<!-- tag: applications -->

Cycle GAN 常见于图像风格转换：真人照片转二次元、马和斑马转换、白天和夜晚转换、素描和照片转换。更进阶的 StarGAN 可以在多种风格之间转换，而不只限两域。

同样思路也能用于文字风格转换。收集一堆正面句子和一堆负面句子，但没有成对改写，就可以训练一个负→正 generator 和一个正→负 generator，再用 cycle consistency 约束。因为输出是离散文字，训练还会遇到 P20 提过的梯度问题，常要借助 reinforcement learning 等方法。

再往外扩展，非监督翻译、非监督语音识别也可以借鉴这种“两个 domain、两个方向、循环还原”的想法。不过不同模态的相似度和可微训练会更难。你现在要掌握的是抽象结构，而不是每个应用的完整实现。

Image style transfer真人↔二次元、白天↔夜晚、马↔斑马。
Text style transfer负面↔正面、长文↔摘要、不同语气风格。
Cross-modal声音↔文字等任务更难，需要额外设计。

# 最小代码：Cycle GAN 的 loss 骨架 {#code}
<!-- tag: minimal code -->

下面伪代码展示目标组成。真实实现还会有 identity loss、buffer、patch discriminator 等技巧，这里只保留本节主线。

正确示例：两个方向的 adversarial + cycle losspseudo python

```text
x = sample_X()
y = sample_Y()

fake_y = G_XY(x)
rec_x = G_YX(fake_y)
fake_x = G_YX(y)
rec_y = G_XY(fake_x)

loss_adv_xy = GAN_loss(D_Y(fake_y), target_real=True)
loss_adv_yx = GAN_loss(D_X(fake_x), target_real=True)
loss_cycle_x = L1(rec_x, x)
loss_cycle_y = L1(rec_y, y)

loss_G = loss_adv_xy + loss_adv_yx + lambda_cyc * (loss_cycle_x + loss_cycle_y)
```

错误示例：只有目标域 GAN losspseudo

```text
# 错误：只要求输出像 Y，没有要求和输入 x 有关系。
fake_y = G_XY(x)
loss_G = GAN_loss(D_Y(fake_y), target_real=True)

# 后果：G_XY 可能无视 x，只生成任意好看的 Y 域样本。
```

# 常见误区与修正 {#mistakes}
<!-- tag: pitfalls -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：Unpaired 就是没有任何资料
错因：把“无配对”误解成“无样本”。后果：看不懂 D_X/D_Y 怎么训练。修法：仍有 X 域和 Y 域各自的样本。

## box redsoft
### 误区 2：只要输出像目标域就完成转换
错因：忽略输入内容。后果：G 可以无视输入。修法：加入 cycle consistency 约束。

## box redsoft
### 误区 3：Cycle GAN 只需要一个 generator
错因：只看 X→Y。后果：无法做 X→Y→X 的还原约束。修法：需要 `G_XY` 与 `G_YX`。

## box redsoft
### 误区 4：Cycle loss 保证语义完全正确
错因：把可逆当语义一致。后果：忽略奇怪编码或左右翻转。修法：承认理论边界，结合额外检查。

## box redsoft
### 误区 5：文字风格转换和图片完全一样简单
错因：忽略离散 token。后果：忘记梯度问题。修法：文本输出常要额外方法，例如 RL。

## box redsoft
### 误区 6：Cycle GAN 等于 StarGAN
错因：混淆两域与多域。后果：模型能力判断错误。修法：Cycle GAN 主要处理两域，StarGAN 扩展到多域。

## box soft
### 现在你应该能做什么
用“目标域真实性 + 循环还原”两把尺子检查 Cycle GAN 任务设计。

# 术语与检查：无配对不等于无约束 {#terms}
<!-- tag: terms / checklist -->

Cycle GAN 的直觉是：没有一一配对标签时，仍然可以利用两个 domain 的分布，再用“转过去还能转回来”约束输入内容不被随便丢掉。它解决的是 unpaired translation，不是任意生成。

| 术语 | 一句话解释 | 易混点 |
| --- | --- | --- |
| Unpaired data | 有 X 域样本和 Y 域样本，但没有一一对应。 | 不是没有资料；两个域各自仍可训练 discriminator。 |
| G_XY / G_YX | 两个方向的转换器，分别做 X→Y 与 Y→X。 | 只做单向会缺少还原约束。 |
| Cycle consistency | X→Y→X 和 Y→X→Y 应该接近原样本。 | 保证可逆，不保证语义一定完美正确。 |
| D_X / D_Y | 分别检查 fake X 与 fake Y 是否像目标域。 | 它们检查域真实性，不检查具体配对关系。 |

## box soft
### Worked example
真人照片转二次元时，`G_XY` 让输出像二次元，`D_Y` 检查目标风格；再用 `G_YX` 把二次元结果还原成真人照片。如果还原后脸型、眼镜、姿势都完全丢失，cycle loss 会惩罚它。

## box info
### 小检查表
- 任务是否真的没有成对 `x,y`？
- 是否同时有两个 generator 和两个 discriminator？
- loss 是否同时包含 adversarial 和 cycle reconstruction？
- 是否检查隐藏编码、结构扭曲或语义漂移？

## box soft
### 现在你应该能做什么
面对风格转换任务时，判断它该用 paired supervision、conditional GAN，还是 Cycle GAN 这类 unpaired 约束。
