# 本节的概念地图 {#goals}
<!-- tag: map -->

自监督学习可以先理解为：从没有人工标签的资料里，自动制造一个可训练任务。语言资料特别适合这样做，因为一句话本身就有很多可遮住、可预测、可排序、可续写的部分。模型不需要人工告诉它“这句话的标签是什么”，也能从文本结构中得到训练信号。

这类模型通常先在大规模文本上预训练，得到通用语言表示；再拿去解决具体任务，例如分类、问答、翻译、摘要。预训练让模型学到词义、语法、上下文关系和世界知识的某些统计规律，下游任务只需要较少标注资料就能利用这些能力。

unlabeled data→

pretraining→
representation→
fine-tuning→
downstream task

# 芝麻街命名：记名字，但别停在名字 {#name}
<!-- tag: names -->

课程从模型名字开场：ELMo 来自 “Embeddings from Language Models”，BERT 来自 “Bidirectional Encoder Representations from Transformers”。后来还有 ERNIE、BigBird 等名字。许多名字明显有凑字母的趣味，甚至后来直接不凑了，直接叫 BigBird。

这些名字本身不是知识重点。它们的共同点是：都属于预训练语言表示或自监督学习模型。它们利用大量文本，训练出能表达上下文信息的神经网络，再迁移到很多 NLP 任务上。

| 模型名 | 直觉定位 | 这节先记什么 |
| --- | --- | --- |
| ELMo | 早期上下文词向量模型 | 同一个词在不同上下文里表示可以不同。 |
| BERT | Transformer encoder 预训练模型 | 双向读上下文，常用 masked prediction。 |
| GPT | 自回归语言模型路线 | 从左到右预测下一 token。 |
| BigBird | 长序列 Transformer 变体 | 名字延续芝麻街梗，也连接长序列建模。 |

### 现在你应该能做什么
把模型名字和大致路线对应起来，不被缩写梗带偏。

# 什么是自监督：标签从资料本身来 {#selfsup}
<!-- tag: self-supervised -->

监督学习需要人工标签。例如给句子标情感、给图片标类别、给语音标文字。自监督学习不依赖这种外部标签，而是把资料的一部分当输入，另一部分当答案。遮住句子里的词，让模型猜被遮住的词；给前文，让模型猜下一词；打乱句子顺序，让模型判断正确顺序。这些答案原本就在资料里。

自监督不等于无监督的“完全没有目标”。它仍然有明确 loss，只是目标自动构造。这个差别很重要：模型在大量自动生成的小任务中学习表示。

小交互：遮住一个 token，让资料自己出题

BERT 风格：看左右文猜中间
GPT 风格：看前文猜下一个


BERT 风格
模型看到左右上下文，用句子本身被遮住的部分当答案。这就是“自监督”的直觉。

# 预训练与下游任务：先练通用能力，再做具体题 {#pretrain}
<!-- tag: pretrain / downstream -->

预训练是先在大规模通用资料上训练模型。对语言模型来说，资料可以是网页、书籍、新闻、百科等文本。模型通过预测 token、填空或其他自监督任务，学习词义、句法、上下文依赖和一些事实知识。

下游任务是具体应用，例如情感分类、自然语言推理、问答、命名实体识别、摘要、翻译。预训练模型并先学通用表示，再通过 fine-tuning 或 prompt 等方式适配具体任务。

直觉上，预训练像先让模型读很多书，练出语言基本功；下游任务像给它一份具体试卷。真正术语是 representation learning 和 transfer learning：学到的表示可以迁移到别的任务。

$$
Stage 1: pretrain on large unlabeled corpus
 Stage 2: adapt to downstream task with labels, prompts, or task-specific heads
$$

### 现在你应该能做什么

# BERT 与 GPT：先抓方向差异 {#bertgpt}
<!-- tag: BERT / GPT -->

课程后面会正式讲 BERT 和 GPT。这里先建立入口直觉。BERT 使用 Transformer encoder，重点是双向上下文表示。它可以同时看一个 token 左边和右边的文字，因此很适合理解、分类、抽取式问答等需要“读懂整段”的任务。经典训练任务是 masked language modeling：遮住一些 token，让模型从上下文恢复。

GPT 路线更像语言模型的传统方向：给前面的 token，预测下一个 token。它通常使用 Transformer decoder 结构，只看过去，不看未来。这个训练目标天然适合生成：一直预测下一个 token，就能生成一段文字。

两者训练目标和结构偏向不同。BERT 更像强大的文本理解器，GPT 更像强大的文本续写器。现代模型已经有许多混合和扩展，但入门时先把这个方向差异抓稳。

## box info
### BERT
双向 encoder。常见任务：填空、理解、分类、抽取式问答。

## box soft
### GPT
自回归 decoder。常见任务：预测下一 token、续写、生成。

## box soft
### 现在你应该能做什么
用“BERT 读上下文，GPT 续下文”解释两条路线的入门差异。

# 进击的巨人：模型为什么越做越大 {#scale}
<!-- tag: scaling -->

课程用“进击的巨人”强调参数量增长。BERT 有数亿参数，GPT-2 有十几亿参数，GPT-3 到千亿级，后来还有上兆参数的模型。参数量是模型中可学习权重的数量。更多参数通常意味着更大的表达能力，但也意味着更高训练成本、推理成本、资料需求和部署难度。

为什么大模型有意义？因为自监督预训练可以利用海量未标注资料。资料足够多、计算足够强时，模型容量增大可能带来更强的语言表示、推理和生成能力。但“大”不是唯一因素。数据质量、训练目标、架构、优化方法和对齐方式也会影响结果。

小交互：模型参数量尺度感

**作业小模型**0.1M课堂练习级别。
**ELMo**94M早期上下文表示。
**BERT large**340M比课堂 baseline 大很多。
**GPT-2**1.5B生成能力更强。
**GPT-3**175B进入超大规模预训练。

### 现在你应该能做什么
解释参数量变大带来的能力、成本和风险，而不是只把“大”当成褒义词。

# 最小例子：自监督任务怎样被构造出来 {#code}
<!-- tag: minimal example -->

下面不是完整 BERT/GPT 代码，只是展示“从原始文本自动得到训练样本”的动作。

正确示例：masked language modeling 的样本构造pseudo python

```text
sentence = ["我", "喜欢", "机器", "学习"]
masked = ["我", "喜欢", "[MASK]", "学习"]
target = "机器"

loss = model.predict_mask(masked).cross_entropy(target)
```

```text

# 修正：遮住、预测下一词、排序等任务都能从 text 自动构造 target。
```

# 常见误区与修正 {#mistakes}
<!-- tag: pitfalls -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：自监督就是没有 loss
错因：把“无人工标签”误解成“无训练目标”。后果：看不懂预训练。修法：标签从资料自身自动构造。

## box redsoft
### 误区 2：BERT/GPT 只是名字梗
错因：停在芝麻街趣味。后果：忽略训练目标和结构差异。修法：记住 BERT 双向理解，GPT 自回归生成。

## box redsoft
### 误区 3：预训练等于完成所有任务
错因：忽略下游适配。后果：不知道 fine-tuning 或 prompt 的作用。修法：预训练学通用能力，下游任务再适配。

## box redsoft
### 误区 4：模型越大一定越好
错因：只看参数量。后果：忽略数据、成本、优化和任务需求。修法：把 scale 看成能力和成本的共同来源。

## box redsoft
### 误区 5：参数量等于知识量
错因：把权重数量当事实库大小。后果：误解模型能力。修法：参数是可学习权重，知识来自训练数据和目标共同塑造。

## box redsoft
### 误区 6：自监督只适用于文字
错因：被本节 NLP 开场限制。后果：忽略影像、语音中的遮挡预测、对比学习等。修法：抓住“从资料自身造任务”的通用定义。

## box soft
### 现在你应该能做什么
用定义判断一个学习方法是不是自监督，而不是看它名字是不是芝麻街角色。

# 术语与检查：没有人工标签，不等于没有答案 {#terms}
<!-- tag: terms / checklist -->

自监督学习的直觉是让资料自己出题。它不依赖人工标注，但会从原始资料自动构造输入和答案，所以仍然有明确 loss。预训练模型的名字只是入口，真正要看训练目标、模型结构和下游适配方式。

| 术语 | 一句话解释 | 易混点 |
| --- | --- | --- |
| Self-supervised learning | 从资料自身构造监督信号，例如遮住再预测。 | 不是没有 loss，也不是纯粹不训练。 |
| Pretraining | 先用大量通用资料学习可迁移表示。 | 预训练不会自动完成所有下游任务。 |
| Fine-tuning | 用具体任务资料把预训练模型适配到目标任务。 | 通常仍需要标注资料或任务反馈。 |
| BERT / GPT | BERT 偏双向理解，GPT 偏自回归生成。 | 不要只记名字梗，要看训练目标和结构。 |

## box soft
### Worked example
句子“我喜欢机器学习”可以构造成 MLM 样本：输入“我喜欢 [MASK] 学习”，target 是“机器”。也可以构造成 next-token 样本：输入“我喜欢机器”，target 是“学习”。两者都不需要人工另写标签。

## box info
### 小检查表
- target 是否能从原始资料自动恢复？
- 预训练目标和下游任务是否分清？
- 模型变大带来的效果、成本和风险是否一起讨论？
- 是否能迁移到图像 patch、语音片段等非文字资料？

## box soft
### 现在你应该能做什么
看到一个新预训练方法时，先问它如何造题、模型如何学、下游任务如何适配。
