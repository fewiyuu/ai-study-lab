P23｜BERT 简介

BERT 是一个 Transformer encoder。它输入一串 token，输出同样长度的一串向量。预训练时，BERT 主要做 masked language modeling：随机遮住一些 token，让模型从上下文预测原本是什么。这个目标来自文本自身，所以属于 self-supervised learning。

自监督回顾：把 X 分成输入和答案

self-supervised

监督学习需要外部 label。比如情感分析中，文章要人工标成正面或负面。自监督学习没有这种人工标签，而是把原始资料分成两部分：一部分当输入，另一部分当答案。BERT 的例子最直观：句子里被遮住的 token 就是答案。

$$ Given unlabeled data X, create X' as model input, create X'' as pseudo-label, train model to predict X'' from X'. $$

现在你应该能做什么

解释 BERT 为什么没有人工标签也能训练：答案来自被遮住前的原句。

BERT 架构：Transformer Encoder

encoder

BERT 的网络本体是 Transformer encoder。你输入一排 token embedding，它输出一排 contextual embedding。输入多长，输出多长。每个输出向量都通过 self-attention 看过整段上下文，所以同一个 token 在不同句子中可以得到不同表示。

输入Token 序列，加上特殊 token、位置编码和 segment 信息。

本体多层 Transformer encoder，包含 self-attention、残差和 normalization。

输出每个输入位置对应一个上下文化向量，可接不同任务头。

预训练任务：MLM、NSP 与 SOP

pretraining

Masked Language Modeling 是 BERT 的核心。随机选一些 token，把它们换成 [MASK] 或随机 token 等形式。BERT 输出对应位置的向量，接一个 linear layer 和 softmax，预测原本 token 是什么。这本质是一个大类别分类问题：类别数等于词表大小。

Sentence Order Prediction 是后来更有用的替代任务。两句本来相邻，但顺序可能被交换。模型要判断顺序是否正确。因为两句主题相关，只判断先后更难一些，ALBERT 等模型使用过这个方向。

Fine-tuning：BERT 像干细胞一样分化

fine-tuning

预训练后的 BERT 只被教过填空等自监督任务。它不会凭空完成情感分析、NLI 或 QA。要做具体任务，仍然需要一些标注资料。Fine-tuning 的做法是：把预训练 BERT 放到任务模型里，再加一个小任务头，然后用下游任务的 label 训练。

如果预训练阶段用大量无标签资料，下游微调阶段用少量有标签资料，整个流程也可以看成 semi-supervised 的味道：无标签资料和有标签资料都被使用了。

Fine-tuning

少量有标注下游数据，任务头随机初始化，BERT 和任务头一起更新。

回答“BERT 只学填空，为什么能做别的任务”：因为它提供了可迁移的初始表示。

抽取式问答：答案就是两个整数

extractive

抽取式 QA 假设答案一定出现在文章中。模型输入问题和文章，输出两个正整数：答案开始位置 S 和结束位置 E。例如答案是文章第 17 个 token，那么 S=17, E=17；答案是一段文字，就输出起点和终点。

答案是单个词

Start 和 end 都落在同一个 token 上，所以答案就是一个词。

成本与 GLUE：为什么自己预训练 BERT 很难

cost

BERT 的预训练资料量和计算量都很大。课程提到最早的 BERT 使用数十亿词汇级别资料，训练需要大型硬件长时间运行。普通学生环境可以 fine-tune BERT，但几乎不可能从头预训练同等级模型。

这也解释了课程作业常说“不要使用预训练模型”：预训练模型已经吸收大量外部资料和计算，直接拿来会让本来想练的模型训练问题失去意义。真正要学 BERT 时，才允许使用预训练模型做 fine-tuning。

补充：Seq2seq 也可以预训练

seq2seq

BERT 主要预训练 encoder。那 sequence-to-sequence model 的 encoder-decoder 能不能预训练？可以。一个常见想法是把输入句子弄坏，再让 decoder 还原原句。弄坏方法包括遮住片段、删除词、打乱顺序、旋转、插入 mask 等。

这一部分在 P23 只是补充，不需要抢先学完整论文。你现在要掌握的是：自监督预训练不只适用于 BERT encoder，也可以用于 encoder-decoder 架构。

最小代码：BERT fine-tuning 的参数关系

minimal

下面伪代码只展示参数初始化思路：BERT 本体来自预训练，任务头随机初始化，然后一起训练。

代码
bert = load_pretrained_bert()
head = Linear(hidden_size, num_labels) # random initialized

tokens = ["[CLS]"] + tokenize(sentence) + ["[SEP]"]
hidden = bert(tokens)
cls = hidden[0]
logits = head(cls)
loss = cross_entropy(logits, label)

update(bert.parameters + head.parameters, loss)

# 错误理解：BERT 永远冻结，只有 head 能更新。

常见误区与修正

pitfalls

误区 1：BERT 预训练时直接学所有下游任务

错因：把预训练和微调混在一起。后果：不理解为什么还需要标注资料。修法：预训练主要学 MLM，下游任务还要 fine-tune。

误区 2：MLM 没有 label

错因：只看到文本无人工标签。后果：看不懂 cross entropy。修法：被遮住前的 token 就是 target。

误区 3：NSP 是 BERT 最核心能力来源

错因：把原始论文任务都当同等重要。后果：忽略后续研究。修法：MLM 更核心，NSP 后来常被认为帮助有限，SOP 是替代之一。

误区 4：所有任务都只用 [CLS]

错因：只学了句子分类。后果：token 分类和 QA 接法错误。修法：token 级任务用每个位置输出，QA 用 start/end。

误区 5：BERT 输入长度无限

错因：理论上 encoder 可变长。后果：忽略 self-attention 记忆成本。修法：实践常限制长度，例如 512 左右，并切分长文章。

误区 6：自己预训练 BERT 和微调 BERT 差不多

错因：忽略资料和算力。后果：低估预训练成本。修法：预训练是大规模工程，微调才适合课堂实践。

术语与检查：BERT 的接法别混

terms

BERT 的直觉是“预训练 encoder + 任务头”。预训练阶段主要学填空；微调阶段根据任务输出粒度选择不同接法。公开讲解时，最好把“预训练任务”和“下游接法”分开讲。

Worked example

情感分类取 [CLS] 接分类头；词性标注给每个 token 接分类头；抽取式 QA 对文章 token 分别算 start score 和 end score。如果答案是第 77 到 79 个 token，就输出 S=77, E=79。

小检查表

  • BERT 本体是否来自预训练参数？
  • 任务头是否按输出粒度选择：句级、token 级、span 级？
  • 输入是否正确放入 [CLS]、[SEP] 和句段信息？
  • 长文本是否处理 self-attention 长度限制？
