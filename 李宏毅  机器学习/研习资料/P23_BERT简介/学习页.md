# 本节的概念地图 {#goals}
<!-- tag: map -->

BERT 的主线可以拆成三段。第一段是自监督预训练：没有人工标签，模型把文本中的一部分遮住，用剩下部分预测被遮住的 token。第二段是得到一个会输出上下文表示的 encoder。第三段是微调：给这个 encoder 接上小型任务头，用少量有标注资料训练具体任务。

课程特别强调：BERT 本体从预训练得到的参数开始；下游任务新加的 linear head 通常随机初始化。Fine-tuning 时，BERT 本体和任务头可以一起被梯度下降更新。

# 自监督回顾：把 X 分成输入和答案 {#selfsup}
<!-- tag: self-supervised -->

监督学习需要外部 label。比如情感分析中，文章要人工标成正面或负面。自监督学习没有这种人工标签，而是把原始资料分成两部分：一部分当输入，另一部分当答案。BERT 的例子最直观：句子里被遮住的 token 就是答案。

所以 self-supervised 可以看作 unsupervised learning 大家族里比较明确的一类方法。它没有用人工标注，但仍然有可优化的 loss。Cycle GAN 也没有成对标签，也属于更广义的 unsupervised 方法；但它和 self-supervised 的“从资料本身造预测目标”不是同一种机制。

$$
Given unlabeled data X,
 create X' as model input,
 create X'' as pseudo-label,
 train model to predict X'' from X'.
$$

### 现在你应该能做什么
解释 BERT 为什么没有人工标签也能训练：答案来自被遮住前的原句。

# BERT 架构：Transformer Encoder {#bert}
<!-- tag: encoder -->

BERT 的网络本体是 Transformer encoder。你输入一排 token embedding，它输出一排 contextual embedding。输入多长，输出多长。每个输出向量都通过 self-attention 看过整段上下文，所以同一个 token 在不同句子中可以得到不同表示。

BERT 最早用于文字，但同样思想也能迁移到语音和影像。只要资料可以表示成一串向量，就可以思考是否能遮住一部分再预测。后来的视觉 Transformer、自监督语音模型都沿着类似思路发展。

输入Token 序列，加上特殊 token、位置编码和 segment 信息。
本体多层 Transformer encoder，包含 self-attention、残差和 normalization。
输出每个输入位置对应一个上下文化向量，可接不同任务头。

### 现在你应该能做什么

# 预训练任务：MLM、NSP 与 SOP {#pretrain}
<!-- tag: pretraining tasks -->

**Masked Language Modeling** 是 BERT 的核心。随机选一些 token，把它们换成 `[MASK]` 或随机 token 等形式。BERT 输出对应位置的向量，接一个 linear layer 和 softmax，预测原本 token 是什么。这本质是一个大类别分类问题：类别数等于词表大小。

**Next Sentence Prediction** 是原始 BERT 里的另一个任务。输入两句，中间加 `[SEP]`，最前面加 `[CLS]`。模型看 `[CLS]` 输出，判断第二句是否真的是第一句的下一句。后续研究发现 NSP 帮助有限，可能因为任务太容易：随机抽来的句子通常和原句很不像。

**Sentence Order Prediction** 是后来更有用的替代任务。两句本来相邻，但顺序可能被交换。模型要判断顺序是否正确。因为两句主题相关，只判断先后更难一些，ALBERT 等模型使用过这个方向。

# Fine-tuning：BERT 像干细胞一样分化 {#finetune}
<!-- tag: fine-tuning -->

预训练后的 BERT 只被教过填空等自监督任务。它不会凭空完成情感分析、NLI 或 QA。要做具体任务，仍然需要一些标注资料。Fine-tuning 的做法是：把预训练 BERT 放到任务模型里，再加一个小任务头，然后用下游任务的 label 训练。

为什么比从头训练好？因为 BERT 已经从大量文本中学到语言表示。随机初始化模型要从零学词义、句法和任务规则；预训练模型已经有较好的起点，loss 下降更快，最后效果也常更好。课程展示的曲线就是这个意思：fine-tune 起点比 scratch 更有利。

如果预训练阶段用大量无标签资料，下游微调阶段用少量有标签资料，整个流程也可以看成 semi-supervised 的味道：无标签资料和有标签资料都被使用了。

## box soft
### Pretraining
大量无标注文本，学习填空等任务，得到 BERT 初始参数。

## box info
### Fine-tuning
少量有标注下游数据，任务头随机初始化，BERT 和任务头一起更新。

# 四类下游任务：同一个 BERT，不同接法 {#cases}
<!-- tag: downstream cases -->

**Case 1：输入一个 sequence，输出一个类别。** 例如 sentiment analysis。做法是在句子前加 `[CLS]`，取 `[CLS]` 的输出向量，接 linear + softmax 做分类。

**Case 2：输入一个 sequence，输出同长度标签序列。** 例如 POS tagging 或命名实体识别。做法是拿每个 token 的输出向量，各自接 linear layer，预测每个位置的类别。

**Case 3：输入两个 sequence，输出一个类别。** 例如 NLI、立场分析。把句子 A、`[SEP]`、句子 B 拼起来，前面加 `[CLS]`，取 `[CLS]` 输出预测句对关系。

**Case 4：输入文章和问题，输出答案片段。** 也就是抽取式问答。模型不直接生成答案，而是在文章里找 start 和 end 两个位置。

| 任务类型 | 用哪个输出 | 例子 |
| --- | --- | --- |
| Sequence → class | [CLS] | 情感分类。 |
| Sequence → tags | 每个 token 输出 | 词性标注、NER。 |
| Pair → class | [CLS] | NLI、立场判断。 |
| Passage + question → span | 文章 token 输出 | 抽取式问答。 |

# 抽取式问答：答案就是两个整数 {#qa}
<!-- tag: extractive QA -->

抽取式 QA 假设答案一定出现在文章中。模型输入问题和文章，输出两个正整数：答案开始位置 `S` 和结束位置 `E`。例如答案是文章第 17 个 token，那么 `S=17, E=17`；答案是一段文字，就输出起点和终点。

BERT 做法是把 `[CLS] question [SEP] passage [SEP]` 送进 encoder。对文章中的每个 token 输出向量，分别和两个可学习向量做 inner product。一个向量负责 start score，一个负责 end score。对所有文章位置做 softmax，取分数最高的位置作为 start/end。

# 成本与 GLUE：为什么自己预训练 BERT 很难 {#cost}
<!-- tag: cost / GLUE -->

BERT 的预训练资料量和计算量都很大。课程提到最早的 BERT 使用数十亿词汇级别资料，训练需要大型硬件长时间运行。普通学生环境可以 fine-tune BERT，但几乎不可能从头预训练同等级模型。

评估预训练语言模型时，常用多个下游任务组成 benchmark。GLUE 是经典任务集，包含多种自然语言理解任务。一个预训练模型通常会分别 fine-tune 到多个任务上，再看平均表现。后来模型逐渐把 GLUE 做得很高，甚至某些任务超过人类基准，于是出现更难的 SuperGLUE。

这也解释了课程作业常说“不要使用预训练模型”：预训练模型已经吸收大量外部资料和计算，直接拿来会让本来想练的模型训练问题失去意义。真正要学 BERT 时，才允许使用预训练模型做 fine-tuning。

从头预训练资料大、硬件贵、时间长。
Fine-tuning相对便宜，课堂和作业更可行。
BenchmarkGLUE / SuperGLUE 用多任务检验语言理解能力。

# 补充：Seq2seq 也可以预训练 {#seq2seq}
<!-- tag: seq2seq pretraining -->

BERT 主要预训练 encoder。那 sequence-to-sequence model 的 encoder-decoder 能不能预训练？可以。一个常见想法是把输入句子弄坏，再让 decoder 还原原句。弄坏方法包括遮住片段、删除词、打乱顺序、旋转、插入 mask 等。

MASS、BART、T5 等模型沿着这个方向发展。它们把“损坏输入 → 还原原文”作为自监督目标，适合摘要、翻译、生成等 seq2seq 任务。T5 还系统比较了很多文本到文本的预训练设定，并使用非常大的 C4 语料。

这一部分在 P23 只是补充，不需要抢先学完整论文。你现在要掌握的是：自监督预训练不只适用于 BERT encoder，也可以用于 encoder-decoder 架构。

# 最小代码：BERT fine-tuning 的参数关系 {#code}
<!-- tag: minimal code -->

下面伪代码只展示参数初始化思路：BERT 本体来自预训练，任务头随机初始化，然后一起训练。

正确示例：sequence classification fine-tuningpseudo python

```text
bert = load_pretrained_bert()
head = Linear(hidden_size, num_labels) # random initialized

tokens = ["[CLS]"] + tokenize(sentence) + ["[SEP]"]
hidden = bert(tokens)
cls = hidden[0]
logits = head(cls)
loss = cross_entropy(logits, label)

update(bert.parameters + head.parameters, loss)
```

错误示例：以为 fine-tuning 只训练 linear headpseudo

```text
# 错误理解：BERT 永远冻结，只有 head 能更新。
freeze(bert)
update(head, loss)

# 修正：有些场景会冻结 BERT，但典型 fine-tuning 是 BERT 和 head 一起更新。
```

# 常见误区与修正 {#mistakes}
<!-- tag: pitfalls -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：BERT 预训练时直接学所有下游任务
错因：把预训练和微调混在一起。后果：不理解为什么还需要标注资料。修法：预训练主要学 MLM，下游任务还要 fine-tune。

## box redsoft
### 误区 2：MLM 没有 label
错因：只看到文本无人工标签。后果：看不懂 cross entropy。修法：被遮住前的 token 就是 target。

## box redsoft
### 误区 3：NSP 是 BERT 最核心能力来源
错因：把原始论文任务都当同等重要。后果：忽略后续研究。修法：MLM 更核心，NSP 后来常被认为帮助有限，SOP 是替代之一。

## box redsoft
### 误区 4：所有任务都只用 `[CLS]`
错因：只学了句子分类。后果：token 分类和 QA 接法错误。修法：token 级任务用每个位置输出，QA 用 start/end。

## box redsoft
### 误区 5：BERT 输入长度无限
错因：理论上 encoder 可变长。后果：忽略 self-attention 记忆成本。修法：实践常限制长度，例如 512 左右，并切分长文章。

## box redsoft
### 误区 6：自己预训练 BERT 和微调 BERT 差不多
错因：忽略资料和算力。后果：低估预训练成本。修法：预训练是大规模工程，微调才适合课堂实践。

## box soft
### 现在你应该能做什么
面对一个 BERT 任务，先问：输入形式是什么，输出粒度是什么，使用 `[CLS]` 还是 token/span 输出。

# 术语与检查：BERT 的接法别混 {#terms}
<!-- tag: terms / checklist -->

BERT 的直觉是“预训练 encoder + 任务头”。预训练阶段主要学填空；微调阶段根据任务输出粒度选择不同接法。公开讲解时，最好把“预训练任务”和“下游接法”分开讲。

| 术语 | 一句话解释 | 易混点 |
| --- | --- | --- |
| MLM | 遮住 token，再从上下文预测原 token。 | 没有人工标签，但被遮住前的 token 就是 target。 |
| NSP / SOP | 判断两句关系；SOP 更关注相邻句顺序。 | MLM 更核心，NSP 不是全部能力来源。 |
| [CLS] | 常用于整句或句对分类的汇总输出。 | token tagging 和 QA 不应只取 [CLS] 。 |
| Extractive QA | 答案在文章中，模型预测 start/end 两个位置。 | 它不是让 decoder 生成一个新答案字符串。 |

## box soft
### Worked example
情感分类取 `[CLS]` 接分类头；词性标注给每个 token 接分类头；抽取式 QA 对文章 token 分别算 start score 和 end score。如果答案是第 77 到 79 个 token，就输出 `S=77, E=79`。

## box info
### 小检查表
- BERT 本体是否来自预训练参数？
- 任务头是否按输出粒度选择：句级、token 级、span 级？
- 输入是否正确放入 `[CLS]`、`[SEP]` 和句段信息？
- 长文本是否处理 self-attention 长度限制？

### 现在你应该能做什么
拿到一个 NLP 任务时，能决定 BERT 输入怎样拼、输出取哪里、任务头如何初始化和训练。
