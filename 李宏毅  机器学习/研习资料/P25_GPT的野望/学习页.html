P25｜GPT 的野望

BERT 的核心训练任务是填空。GPT 的核心训练任务更简单直接：给前面的 token，预测下一个 token。把这个动作在大量文本上重复，模型就学会了语言续写。使用时，只要把模型刚生成的 token 接回输入，再预测下一个，就能不断产生文章。

Next-token Prediction：每一步都是分类

autoregressive

假设训练句子是“台湾大学”。GPT 会构造多个训练步：给 BOS 预测“台”；给 BOS 台 预测“湾”；给 BOS 台湾 预测“大”；给 BOS 台湾大 预测“学”。每一步都用 hidden vector 经过 linear layer 和 softmax，得到下一个 token 的概率分布。

现在你应该能做什么

给任意一句话，拆出 GPT 的多个 next-token 训练样本。

GPT 架构：像 Decoder，但没有 Cross-attention

decoder-only

Transformer decoder 原本有 masked self-attention、encoder-decoder cross-attention 和 feed-forward 等部分。GPT 类模型更像 decoder-only：没有 encoder 输入，因此也不需要 cross-attention。它只在已经生成或已经给定的前文上做 masked self-attention。

保留Masked self-attention、feed-forward、残差、normalization。

拿掉Encoder-decoder cross-attention，因为没有 encoder 侧输入。

结果模型只能根据前文建模，天然适合生成。

生成能力：预测一次，再接回去预测下一次

generation

训练完成后，GPT 可以生成文字。给它一段 prompt，它输出下一个 token 的分布。选出一个 token 后，把这个 token 接到 prompt 后面，再预测下一个。不断重复，就能产生句子、段落甚至文章。

但生成不等于真实。模型学习的是文本分布，可能产生流畅但虚构的内容。理解 GPT 时要同时看到它的强大续写能力和事实可靠性的风险。

生成时

看 prompt 和自己生成的前文，继续预测；错误也可能被接着放大。

In-context Learning：不改参数，只改上下文

prompting

BERT 的典型用法是 fine-tuning：接任务头，使用标注资料更新参数。GPT 也可以这样做，但 GPT 论文中更强调一种更狂的用法：把任务说明和几个范例直接写进 prompt，不做梯度下降，不更新模型参数，让模型从上下文中推断任务格式。

小交互：zero / one / few-shot prompt

Zero-shot

只有任务说明，没有示范。模型要直接根据说明补出答案。

规模与效果：大模型让 in-context 更像样，但不万能

scaling

GPT-3 论文测试了不同大小模型在多任务上的 zero-shot、one-shot、few-shot 表现。总体趋势是模型越大，平均表现越好；few-shot 通常又比 zero-shot 更强。这个结果支持了一个直觉：大规模预训练模型更容易从 prompt 里读懂任务。

所以 GPT 的野望是：用同一个巨大语言模型，通过上下文完成许多任务。但入门时要记住边界：它不是传统意义上针对每个任务训练过，也不是所有任务都能可靠解决。

现在你应该能做什么

解释为什么模型变大和示范变多通常提升 in-context 表现，但不能保证所有任务成功。

文字之外：视觉和语音也能自监督

beyond

课程最后提醒：BERT 和 GPT 只是 self-supervised learning 的一部分，而且主要属于 prediction 路线。自监督学习不只用于文字，也用于 computer vision 和 speech。视觉里有 contrastive learning、BYOL 等方法；语音里可以做“语音版 BERT”：遮住声音片段再预测，也可以做“语音版 GPT”：预测接下来会出现的语音。

这一段的学习目标是扩展视野：自监督的核心不是文字，而是“能否从资料本身构造任务”。不同模态的任务形式不同，但理念相通。

公开分享速查：一句话看懂 GPT

quick

核心直觉

GPT 的训练任务很窄：永远只问“前文之后最可能出现什么 token”。但如果前文里放入任务说明、格式示范和问题，模型的续写动作就会表现得像在做翻译、分类、摘要或问答。

所以 GPT 的强项把许多任务都改写成“把后面的文字补出来”。

Worked example

Prompt：情绪分类。句子：这家店再也不去了。答案：

模型在预测“答案：”后最合理的 token。若训练中见过大量类似语言模式，它可能续写“负面”。这就是把分类任务转成语言续写。

术语精确定义容易混的点
Next-token prediction给定前文，预测词表中下一个 token 的概率分布。不是一次生成整篇文章；长文本来自逐步预测并接回上下文。
Masked self-attention当前位置只能看自己之前的 token，不能看未来。这里的 mask 是因果遮罩，不是 BERT 那种把词换成 [MASK]。
In-context learning通过 prompt 中的说明和示例让模型完成任务，不更新参数。看到 few-shot 不代表做了 fine-tuning；示例只是上下文。
Zero/one/few-shot分别对应不给示例、给一个示例、给多个示例。它们描述的是 prompt 里有多少示例，不是训练集大小。

小检查表

  • 解释 GPT 时，先说训练目标，再说生成循环。
  • 遇到 prompt 例子，先判断是否更新参数；不更新就是 in-context。
  • 评价能力时，区分“能续写出格式正确答案”和“答案事实可靠”。

最小代码：next-token loss 与 prompt

minimal

下面伪代码展示 GPT 训练和 in-context 使用的差别：训练会更新参数，prompt 使用不更新参数。

代码
tokens = [BOS, "台", "湾", "大", "学"]
hidden = gpt(tokens[:-1]) # cannot see future tokens
logits = linear(hidden)
loss = cross_entropy(logits, tokens[1:])
update(gpt.parameters, loss)

常见误区与修正

pitfalls

误区 1：GPT 和 BERT 训练目标一样

错因：都属于自监督。后果：混淆填空和续写。修法：BERT 做 MLM，GPT 做 next-token prediction。

误区 2：GPT 预测下一个 token 时可以看完整句子

错因：忘记 masked self-attention。后果：训练目标变成抄答案。修法：只能看当前位置之前的 token。

误区 3：In-context learning 等于 fine-tuning

错因：看到 few-shot 就以为训练。后果：误解参数没有更新。修法：prompt 示例只是上下文，模型参数不变。

误区 4：Zero-shot 表现不好代表 GPT 完全没学会语言

错因：把任务说明能力和语言模型能力混为一谈。后果：评价过度。修法：看任务、prompt、模型规模和评估方式。

误区 5：模型越大一定会解决所有推理任务

错因：只看 scaling trend。后果：忽略逻辑推理仍可能很差。修法：区分平均提升和具体任务可靠性。

误区 6：自监督学习只属于 NLP

错因：本节主要讲 BERT/GPT。后果：忽略视觉和语音应用。修法：抓住“从资料自身造任务”的通用核心。

用“训练目标、注意力可见范围、使用时是否更新参数”三点区分 GPT 和其他模型。
