Lee ML 2025 / P26

少一点标注，少一点参数，也能用好大模型

预训练语言模型已经有大量语言知识，但现实任务经常卡在两件事上。第一，某个下游任务的有标注资料很少，直接接分类器再微调容易过拟合。第二，模型本身太大，如果每个任务都完整保存一份微调后的模型，空间和推理成本都会很高。

本节把解决思路分成两条线。Data-efficient tuning 关心“标注少怎么办”，代表做法是把任务改写成 prompt，再配合 verbalizer、demonstration、半监督伪标签或多任务指令训练。Parameter-efficient tuning 关心“任务多、模型大怎么办”，代表做法是 Adapter、LoRA、prefix tuning、soft prompt 和 early exit。

本节的概念地图

MAP

把预训练模型用于下游任务时，最朴素的方法是 standard fine-tuning：拿掉原本的 language model head，接一个新的 classifier head，然后把整颗模型一起微调。这个方法简单，但在有标注资料很少时，新分类头需要从很少的样本里学映射；在任务很多时，每个任务都要保存一份完整参数。

prompt tuning 的想法是不要把任务改造成“新分类器问题”，而是把任务改写成语言模型熟悉的形式。例如自然语言推理可以写成“前提句。[MASK]，假设句”，再让模型填 yes、maybe、no。这个设计让下游任务更贴近预训练时的填空任务，也把人的任务理解放进 template 和 verbalizer 里。

参数高效微调继续往前走：如果整颗模型已经会很多东西，我们是否只训练一小块任务专属组件，让它改变 hidden representation 的方向，而不更新原始模型的大量参数？Adapter、LoRA、prefix tuning 和 soft prompt 都是在做这件事，只是插入位置和可训练对象不同。

两个现实问题：资料少，模型大

MOTIVATION

问题一：有标注资料太少

预训练阶段可以用海量未标注文本，但下游任务往往只有几十、几百或几千笔有标注资料。若直接训练一个新的分类头，分类头参数虽然看起来不大，却可能已经多过可用样本提供的信息量。模型容易记住训练样本，而不是学到稳健规则。

少数据路线的核心做法是降低任务和预训练任务之间的差距。与其让模型从零学“这个新分类头的输出 0/1/2 是什么”，不如把任务写成自然语言提示，让模型继续做它熟悉的词表预测。

问题二：每个任务保存一整颗模型

假设一个 BERT/RoBERTa 模型被微调用到四个任务上。standard fine-tuning 会得到四份完整模型参数。任务越多，存储成本越高；如果服务端要同时支持多个任务，加载和切换也会变麻烦。

少参数路线的核心做法是冻结共用预训练模型，只为每个任务保存很小的专属参数。这样部署多个任务时，可以共享同一个基础模型，再按任务切换 Adapter、LoRA 权重、prefix 或 soft prompt。

共同问题：不要让下游任务重新学习已经存在的语言知识。
少数据：减少任务形式差距；少参数：减少每个任务新增参数。

Prompt Tuning：保留 LM Head，用词表词表达标签

PROMPT

standard fine-tuning 会丢掉原本的 LM head，接上一个任务分类器。prompt tuning 不这么做。它保留 LM head，让模型继续预测词表里的 token；我们再用 verbalizer 把 token 映射回标签。比如 NLI 任务中，entailment、neutral、contradiction 可以映射成 yes、maybe、no。

这套做法需要三件东西。第一是 prompt template，也就是输入如何被改写成填空形式。第二是 pretrained LM，它负责根据上下文预测 mask 位置的词。第三是 verbalizer，它规定哪些词代表哪些标签。template 不是装饰，它把人的任务假设写进输入格式；verbalizer 也不是随便选词，因为不同词可能让模型调用到不同的预训练知识。

少数据路线：从 few-shot 到 zero-shot

DATA EFFICIENT

少数据路线不是一个单独算法，而是一组围绕 prompt 的使用策略。few-shot 场景下，资料可能只有十几笔。LM-BFF 这类方法会在 prompt tuning 之外加入 demonstration，也就是把少量示范例子放进输入，让模型看到“这个任务的输入和输出应该长什么样”。这和 GPT 的 in-context learning 形式相似，但这里可以配合较小的 RoBERTa 类模型做微调。

semi-supervised learning 场景下，我们有少量有标注资料和大量未标注资料。PET 的思路是设计多组 template 和 verbalizer，训练出多个 prompt-based 模型，再让这些模型给未标注资料产生伪标签。最后用原始标签和伪标签一起训练一个标准分类模型。这里的关键不是“伪标签一定正确”，而是用多个 prompt 视角把预训练知识转成更多监督信号。

zero-shot 场景下，甚至没有目标任务的有标注样本。GPT-3 展示过只靠任务说明做推断的能力。一种解释是：预训练文本里本来就混着 FAQ、摘要、翻译、问答等任务形态，模型在预测下一个 token 时隐约学到多任务模式。更主动的做法是把 T5 等模型用多任务 prompt 形式训练，让它显式练习很多任务，再测试未见过的任务。

few-shot少量标签 + demonstration
PET多种模板产生伪标签
semi-supervised少量标签 + 大量未标注
zero-shot只给任务说明
multi-task prompt显式训练跨任务能力

LM-BFF

给模型看 demonstration，让少量例子提供任务格式。它强调小一些的预训练模型也可以在 few-shot 上做得不错。

PET

用多组 prompt 模型给未标注资料产出伪标签，再训练一个标准分类模型。它把未标注资料转成可用信号。

Zero-shot

没有目标标签时，只能依赖模型在预训练或多任务训练中学到的任务理解能力。效果和任务、说明、模型规模都有关。

少参数直觉：只学一个小的 Δh

PARAMETER EFFICIENT

standard fine-tuning 更新整颗模型，本质上是在改变模型内部产生的 hidden representation。设原本预训练模型给出表示 h，下游任务想要的是更适合分类器或生成目标的 h'。如果可以让一个很小的任务模块产生变化量 Δh，使 h' = h + Δh，就不一定需要动到整颗模型。

这就是参数高效微调的共同直觉。原始 Transformer、embedding、attention 和 feed-forward 大多冻结；每个任务只训练插入的小模块或小向量。部署时，共用基础模型只存一份，任务 A 存自己的小参数，任务 B 存自己的小参数。任务切换时换小参数，而不是换整颗模型。

四种参数高效做法：插在哪里，训练什么

METHODS

Adapter、LoRA、prefix tuning、soft prompt 都在减少任务专属参数，但它们插入的位置不同。Adapter 是在 Transformer layer 内部串接一个小模块，通常放在 attention 或 feed-forward 后面。这个小模块先降维、非线性转换、再升维，然后用残差方式给原表示加一个变化量。

LoRA 也是产生变化量，但它常以平行分支的形式加在某些线性层旁边。它把一个大的权重更新近似成两个低秩矩阵的乘积，因此只需要训练很少的矩阵参数。和串接 Adapter 相比，LoRA 的一个卖点是平行计算更容易，不一定增加明显的推理深度。

prefix tuning 把可训练的 prefix 放到每一层 self-attention 的 key/value 前面。普通 token 有 query、key、value；prefix 不需要主动 query 别人，它主要被后续 token attend 到。训练完成后，每个任务保存对应层的 prefix key/value。soft prompt 更简单，只在输入 embedding 前面插入可训练向量，不在每一层都插。

方法插入位置训练对象容易混淆点
Adapter每层内部，常在 attention/FFN 后小型降维-升维模块和任务头会让路径更深一点
LoRA线性层旁边的低秩分支低秩矩阵 A/B 和任务头不是把原权重全量更新一遍
prefix tuning每层 attention 的 key/value 前缀每层 prefix 对应的向量prefix 通常不需要 query
soft prompt输入 embedding 前面少量连续向量不是人工写出的 hard prompt 词

hard prompt vs soft prompt

hard prompt 是真实 token，例如“请判断这句话的情感”。它经过词表 embedding 得到向量。soft prompt 直接训练连续向量，不一定对应任何可读文字，因此更灵活，也更难人工解释。

prompt tuning 名字容易乱

有些论文把用自然语言模板称为 prompt tuning，有些把训练连续 prompt 向量称为 prompt tuning。读论文时不要只看名字，要看它到底在训练整颗模型、训练 LM head，还是只训练若干连续向量。

Early Exit：容易的样本不用跑完整层数

INFERENCE

前面的做法主要减少每个任务需要保存或训练的参数。early exit 更关心推理时间。大模型有很多层，但不是每个样本都需要走到最后一层才能判断。有些输入在前几层已经足够明显，继续计算后面的层只是增加成本。

一种做法是在每一层后面都放一个小分类器，再训练 confidence predictor 判断当前层输出是否已经足够可靠。如果第 2 层已经很有把握，就直接输出结果，后面的层不再计算；如果还不够有把握，就继续往后走。它没有把模型结构真正删掉，而是按样本动态选择走多远。

常见误区与修正

PITFALLS

误区 1：prompt tuning 一定不更新模型

错因：把自然语言 prompt、soft prompt 和 in-context use 混在一起。后果：判断训练成本时出错。修法：看是否有梯度更新，更新的是整颗模型还是少量向量。

误区 2：verbalizer 只是翻译标签名

错因：以为 yes/no/maybe 随便选。后果：模型预测的词表概率和真实标签对不上。修法：把 verbalizer 当成标签到词表词的关键映射。

误区 3：few-shot 就等于完全不训练

错因：看到 few-shot prompt 就联想到 GPT 的上下文学习。后果：混淆 LM-BFF 一类微调方法。修法：few-shot 只描述样本少，不自动说明是否更新参数。

误区 4：Adapter、LoRA、prefix tuning 都只是换名字

错因：它们都训练少量参数。后果：无法判断插入位置和推理成本。修法：分别看串接模块、低秩分支、每层 key/value 前缀。

误区 5：soft prompt 是人工写出的提示词

错因：名字里有 prompt。后果：期待它可读、可解释。修法：soft prompt 是可训练连续向量，hard prompt 才是真实 token。

误区 6：early exit 会把大模型永久剪小

错因：把动态退出和模型压缩混淆。后果：误解部署方式。修法：模型层仍在，简单样本可提前输出，难样本继续往后走。
