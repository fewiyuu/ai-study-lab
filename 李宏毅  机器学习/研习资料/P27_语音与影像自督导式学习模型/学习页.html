语音和影像，也能自己给自己出题

文字里的 BERT 和 GPT 已经说明：只要能从资料本身构造训练目标，就可以不靠人工标签学到有用表示。语音和影像也能这样做。给模型大量未标注声音或图片，让它学出一排 representation，再用少量标注资料接下游模型，就能做语音辨识、唤醒词、说话人任务、影像辨识、物件侦测、分割等任务。

难点在于声音和图片不像文字 token 那样离散、干净。相邻语音 frame 太像，mask 一个 frame 只会学到内插；图片和声音包含大量细节，完整生成太难；对比学习需要选负例，负例太简单学不到东西，太难又可能把同类拉开。P27 的重点就是看这些方法如何绕过这些问题。

本节的概念地图

MAP

自督导式学习的共同核心是：没有人工标签，也要从资料内部造出监督信号。文字中，BERT 盖住 token 让模型填空，GPT 根据前文预测下一个 token。语音中，可以盖住一段声音让模型还原，也可以根据前面的声音预测较远之后的声音。影像中，可以盖住 patch、预测 pixel、判断旋转角度，或让同一张图不同增强版本的表示接近。

语音和影像的特殊之处让方法分化成五类。第一类 generative 试着还原原始讯号或被 mask 的部分。第二类 predictive 不一定还原原讯号，而是预测旋转、相对位置、cluster id 等更容易的目标。第三类 contrastive 不直接生成东西，而是让正样本表示接近、负样本表示远离。第四类 bootstrap 避开负例，用 teacher/student 和 stop-gradient 训练。第五类 regularization 用 variance/covariance 约束防止所有表示变成一样。

未标注声音/图片 自造任务 学 representation 少量标注适配 多任务评估

从文字迁移到语音和影像

TRANSFER

文字版 BERT 的输入是一串 token，输出是一串向量，每个 token 对应一个 representation。语音版 BERT 的输入是一段声音讯号，通常先变成一排 acoustic feature，然后输出一排向量。下游语音辨识模型可以吃这些向量，把它们转成文字。若没有这种预训练表示，端到端语音辨识往往需要成千上万小时的标注声音；有了语音版 BERT，研究者会挑战用十分钟、几十分钟标注资料做适配。

影像也类似。预训练模型先在大量未标注图片上学 representation，再把表示用于影像辨识、物件侦测、语义分割、视觉导航等任务。课程中特别强调一个惊人的现象：某些影像自督导模型在下游任务上可以超过用大规模标注资料监督训练得到的基准。这并不是说标签永远没用，而是说明未标注资料中的结构本身已经很有学习价值。

文字

输入 token 序列，遮住 token 或预测下一个 token，学语言表示。

语音

输入连续声音 feature，遮住一段时间或预测未来 frame，学内容、说话人、情感等表示。

影像

输入图片或 patch，预测变换、相对位置、cluster 或拉近增强视图，学视觉表示。

多任务评估：SUPERB 与影像下游任务

BENCHMARK

文字领域有 GLUE 这样的 benchmark，用多个自然语言处理任务评估预训练模型。语音领域也有类似思路：SUPERB 是 Speech processing Universal PERformance Benchmark，用一系列任务测试语音自督导模型的能力。任务不仅有语音辨识，也包括 keyword spotting、说话人辨识、说话人验证、情感辨识、语音翻译、语音增强、语音分离等。

这很重要，因为一个表示如果只在单一任务上好，可能只是学到了窄线索。比如只做语音辨识，模型可能偏向内容信息；若还要做说话人验证、情感辨识和语音增强，就能检查它是否保留了更多层次的信息。影像领域也是一样，不能只看 image classification，还要看 object detection、semantic segmentation、navigation 等任务。

模态典型任务评估意义
语音辨识、唤醒词、说话人、情绪、翻译、增强、分离检查表示是否覆盖内容、说话人、语气和语义
影像分类、侦测、分割、导航检查表示是否能迁移到不同视觉粒度
文字分类、推理、问答、相似度检查语言表示是否能支持不同文本任务

生成式方法：直接还原被遮住或未来的讯号

GENERATIVE

生成式方法最接近文字版 BERT/GPT。语音版 BERT 可以把声音某些片段 mask 掉，要求模型还原被盖住的部分；代表例子包括 Mockingjay。语音版 GPT 可以给前面的声音讯号，预测未来某个时间点的声音；代表例子包括 APC。影像也可以把 pixel 拉成序列，预测下一个 pixel，或 mask 一部分 pixel 再还原。

但照搬会出问题。语音 feature 相邻帧高度相似，如果只 mask 一个 frame，模型用左右邻居内插就能猜出来，学不到更高层的语音结构。所以语音中通常要 mask 一整段连续 feature，或者预测距离当前更远的未来 frame。影像和语音还包含大量低层细节，要求模型完整还原原始讯号很难，也不一定是下游任务最需要的能力。

预测式方法：不还原原讯号，预测更简单的目标

PREDICTIVE

如果还原声音或图片太复杂，可以设计一个更简单但有用的预测任务。影像中常见例子是 rotation prediction：把图片旋转 0、90、180、270 度，让模型判断旋转角度。另一个例子是 context prediction：从大图切出两个 patch，让模型判断第二个 patch 在第一个 patch 的哪个方向。这些任务迫使模型理解物体方向、局部结构和空间关系。

语音也可以设计类似任务，比如给同一句话里切出的两段声音，让模型判断它们相距多远。更通用的做法是先把连续 acoustic feature 做 clustering 或 quantization，把复杂向量变成 discrete cluster id，再让模型预测被 mask 位置的 cluster id。HuBERT、BEST-RQ、DeepCluster 都属于这类思路的代表：目标不是生成原始讯号，而是预测一个简化后的符号或聚类标签。

原始讯号声音/图片细节很多
变换或聚类旋转、相对位置、cluster
预测目标角度、方向、id
学习表示抽出结构信息
下游使用分类/辨识/分割

好处

目标比还原原始讯号更简单，训练更稳定，也可能更贴近下游任务需要的抽象结构。

风险

预测任务需要领域知识。太简单学不到东西，太偏门又可能学到和下游无关的线索。

对比学习：不生成，只拉近正样本、推远负样本

CONTRASTIVE

contrastive learning 的目标不是还原东西，而是塑造 representation 空间。对同一张图片做两次 data augmentation，得到两个视图，它们被当作 positive pair，表示要接近。来自不同图片的视图被当作 negative pair，表示要远离。SimCLR 就是这一路线的经典方法，常用 random crop、color distortion、Gaussian blur 等增强，其中 random crop 往往很关键。

语音里也有类似路线，例如 CPC、wav2vec、vq-wav2vec 和 wav2vec 2.0。wav2vec 2.0 看起来复杂：前端 encoder 把声音变成表示，中间有 quantization 产生 discrete target，后端 Transformer 吃 continuous representation，在 mask 位置预测正确的 discrete target，并把若干错误 target 当作负例压低。换个角度看，它和 BERT 很像：BERT 在 mask 位置预测正确 token；wav2vec 2.0 在 mask 位置预测正确的语音离散单位，只是负例太多时用抽样近似。

太简单的负例

如果猫的负例是天空或海，模型可能只学颜色差异就能分开，表示不一定学到物体结构。

对比学习的判断：positive pair 距离小；negative pair 距离大。
难点：负例不能太简单，也不能把真实同类误当成负例强行拉开。

Bootstrap：只用正样本，靠非对称路径避免 collapse

BOOTSTRAP

如果不想选负例，能不能只让 positive pair 接近？直接这样做会 collapse：模型可以对所有输入输出同一个零向量，正样本当然都很接近，但表示完全没用了。bootstrap 路线试图在不用负例的情况下避免 collapse，代表方法包括 BYOL、SimSiam，以及语音/影像/文字都可用的 data2vec。

BYOL 的结构像 teacher/student。两张增强视图分别走两条路径：一边通常 stop-gradient，另一边有 predictor，并且只更新 student 路径；然后用 moving average 慢慢更新 teacher。SimSiam 进一步说明 moving average 不一定必要，但 predictor 和 stop-gradient 很关键。课程里也提醒：这类方法为什么不 collapse 仍然不是初学者能用一句话完全解释的事，但实验上这些机制确实很重要。

小交互：是否有防 collapse 机制

容易 collapse

如果训练目标只要求正样本接近，模型可以把所有输入都输出同一个向量。

说清楚 BYOL 为什么不是“简单去掉负例”，而是加入了非对称网络和 stop-gradient。

正则化路线：直接约束表示不要塌掉

REGULARIZATION

另一条避开负例的路线是加正则化。VICReg 的名字来自 Variance-Invariance-Covariance Regularization。invariance 是让同一图片的不同增强版本表示接近；variance 是要求一个 batch 中每个表示维度都有足够变化，避免所有样本输出同一个向量；covariance 是让不同维度减少冗余，让 latent space 被更充分使用。

课程中特别强调 variance 很关键。只有 invariance 会 collapse；加上 variance，就能强制表示在不同样本之间保持差异。covariance 可以让不同维度不要学成重复信息，但没有 covariance 时模型仍可能训练起来。Barlow Twins 和 VICReg 都属于这个方向，语音里也有类似方法。

约束直觉防止什么问题
Invariance同一样本不同增强视图要接近学到增强不变的表示
Variance不同样本在每个维度上要有足够变化防止全部输出同一个向量
Covariance不同维度之间不要高度重复减少维度冗余，展开表示空间

公开分享速查：跨模态自监督怎么选

quick reference

核心直觉

文字、语音、影像都可以自监督，但任务不能机械照搬。文字 token 离散，next-token 很自然；语音和影像连续且细节多，直接重建可能太难，预测太近又可能太容易。

方法演化的主线是：先尝试生成或预测，再用对比学习关注表示相似性，最后用 bootstrap 或正则化绕开负例选择。

Worked example

语音若只预测下一个 10ms frame，模型可能学到平滑内插，而不是语义。更好的任务可以遮住一整段，或预测更远位置的离散单位，让模型必须利用上下文和内容结构。

影像若完整重建每个像素，模型可能沉迷纹理细节；对比或增强不变性目标更容易把注意力放在物体和语义表示上。

方法直觉主要风险
Generative重建被遮住或未来的原始信号。信号太细，任务太难或学到低层细节。
Predictive预测离散单位、未来表示或较简化目标。目标太近会变成平滑内插。
Contrastive拉近正样本，推远负样本。负例选错会把同类样本推开。
Bootstrap用 teacher/student 非对称路径只学正样本。若没有 stop-gradient、predictor 等设计，容易 collapse。
Regularization用 variance/covariance 等约束保持表示展开。只做 invariance 不够，仍可能所有样本输出同向量。

选择检查表

  • 先问资料是离散 token、连续语音，还是高维影像。
  • 再问训练目标是否太容易、太难，或会鼓励复制无关细节。
  • 使用无负例方法时，必须说明靠什么机制防止 collapse。

常见误区与修正

PITFALLS

误区 1：语音版 BERT 就是把文字 BERT 直接拿来用

错因：只看名字相似。后果：忽略声音是连续 feature。修法：说成“类似 BERT 的训练目标和 encoder 思路”。

误区 2：mask 一个语音 frame 就足够

错因：把语音当作离散 token。后果：模型可能只学内插。修法：mask 连续片段或预测更远未来。

误区 3：生成式方法总是最好

错因：文字中 BERT/GPT 很成功。后果：忽略图片和声音细节太多。修法：根据模态选择预测简化目标或表示空间目标。

误区 4：对比学习只要找不同图片当负例就行

错因：低估负例难度。后果：太简单学不到结构，太难可能误拉开同类。修法：关注增强策略和负例采样。

误区 5：没有负例一定会 collapse

错因：只看到朴素正样本目标。后果：无法理解 BYOL/VICReg。修法：补上 predictor、stop-gradient、variance 等机制。

误区 6：wav2vec 2.0 和 BERT 完全无关

错因：架构细节看起来复杂。后果：失去统一理解。修法：把它看作在 mask 位置预测正确语音离散单位的 BERT-like 目标。
