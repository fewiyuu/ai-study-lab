# 本节的概念地图 {#goals}
<!-- tag: MAP -->

自编码器是很早出现的无标签学习方法。它不需要人工标注类别，也不需要人告诉模型图片里是什么。它自己造的任务很简单：输入一张图片，先压缩成一个向量，再从这个向量重建回原图。若重建得好，说明中间向量保留了足够信息。

这个任务看起来像“复制输入”，但真正有价值的是瓶颈。只要中间 code 比输入窄，模型就不能把所有细节原封不动传过去，只能保留对重建最重要的变化。训练好的 encoder 可以单独拿出来，把高维输入转成低维表示，再给分类、聚类、可视化或其他下游模型使用。

input x
encoder
code z
decoder
reconstruction x'

# 它为什么可以算自监督学习 {#ssl}
<!-- tag: SELF-SUPERVISED -->

自监督学习的关键是：不用人工标签，也能从资料本身构造训练目标。BERT 的目标来自原句中被 mask 的 token，GPT 的目标来自句子里的下一个 token。自编码器的目标更直接：目标就是原输入。训练资料不需要类别标签，只需要原始样本。

当然，autoencoder 比“self-supervised learning”这个流行词早很多。有人会把它归入 unsupervised learning，有人会从“无需标签但有自造目标”的角度把它看成自监督学习的一种。学这节课不需要纠结术语归属，重点是看懂它如何从无标签资料里学习 representation。

BERT盖住 token，让模型还原被盖住的位置。
GPT根据前文，预测下一个 token。
Autoencoder把输入压缩后，再重建原输入。

# Encoder 与 Decoder：压缩，再重建 {#structure}
<!-- tag: STRUCTURE -->

encoder 可以是 CNN、MLP、Transformer 或其他网络。它读入输入 `x`，输出一个向量 `z`。decoder 再读入 `z`，输出重建结果 `x'`。训练目标是让 `x'` 和 `x` 尽量接近。对图片来说，可以把图片看成很长的像素向量，用均方误差或其他 reconstruction loss 衡量差距。

从功能上看，encoder 负责“编码”：把复杂输入变成可处理的 code。decoder 负责“解码”：用 code 还原输入。这个 code 有多个名字：embedding、representation、code。名字不同，都是在指中间那段表示。后续很多深度学习系统都会复用这种“先学表示，再做下游任务”的想法。

$$
z = Encoder(x)
 x' = Decoder(z)
 training objective: make x' close to x
$$

# 瓶颈：为什么中间要窄 {#bottleneck}
<!-- tag: BOTTLENECK -->

如果 code 的维度和输入一样大，甚至比输入还大，模型可能学到接近“复制通道”的行为：把输入细节直接传给 decoder。瓶颈变窄后，模型必须选择哪些信息值得保留。比如一张 100×100 RGB 图片有 30000 个数值，但 code 可能只有 10 或 100 维。这个差距迫使 encoder 做 dimension reduction。

降维尽量保留资料中真正会影响重建的变化。图片表面上维度很高，但真实图片并不会占满所有像素组合。随机噪声矩阵通常不像自然图片；自然图片有边缘、纹理、物体、局部连续性等结构。自编码器试图把这些结构压进低维 code。

小交互：code 维度与重建压力
code 维度**16**
压缩压力**高**
细节保留**中**


窄瓶颈
code 很小时，模型必须保留最重要的变化。它可能学到更抽象的表示，但重建细节会更难。

# 压缩为什么可能：资料的变化本来有限 {#why}
<!-- tag: INTUITION -->

课程用“复杂外观背后可能有少数控制因素”的思路解释 autoencoder。图片像素很多，但自然图片不是所有像素组合的集合。比如人脸图片的变化可能来自姿态、光照、表情、身份、背景等因素；手写数字的变化可能来自数字类别、倾斜程度、笔画粗细。像素空间很大，真实资料通常落在一个较低维的结构上。

encoder 做的事就是试着找到这些较少的控制因素。decoder 则学会根据这些因素重建出类似原输入的样本。若 code 保留的是有意义的变化，下游任务就能少学很多东西。比如分类器不必直接面对 30000 个像素，而是面对 100 维 code；聚类也可以在 code 空间做，而不是在原始像素空间里受噪声干扰。

## box info
### 高维外观
像素、声音波形、传感器读数看起来维度很高，包含很多细节。

## box soft
### 低维因素
真实变化往往来自少数结构性因素。找到这些因素，才是 representation 的价值。

## box soft
### 现在你应该能做什么
用“真实资料不占满整个高维空间”解释为什么降维可能有意义。

# 训练好以后怎么用 {#downstream}
<!-- tag: DOWNSTREAM -->

训练完成后，常见做法是拿 encoder 作为特征抽取器。输入图片先经过 encoder 变成 code，再把 code 交给下游模型，例如线性分类器、聚类算法、检索系统或可视化方法。下游模型面对的是低维表示，不是原始高维输入，因此可能需要更少标注资料，也更容易训练。

这和前面学过的 BERT 用法很像。BERT 预训练时做填空，真正使用时拿它的表示做分类、问答或推理。autoencoder 预训练时做重建，真正使用时可以拿 encoder 输出做其他任务。预训练任务本身不是终点，它的价值在于学到可迁移的 representation。

| 阶段 | 使用对象 | 目标 |
| --- | --- | --- |
| 预训练 | encoder + decoder | 让 reconstruction 接近 input |
| 下游任务 | 通常主要用 encoder | 把输入转成低维 code，再做分类/聚类/检索 |
| 生成或重建分析 | encoder + decoder | 观察重建质量或操作 code 空间 |

# 一点历史：它不是新概念 {#history}
<!-- tag: HISTORY -->

autoencoder 很早就出现了。课程提到 Hinton 2006 年左右的工作，当时人们还不太相信很深的网络可以直接端到端训练，因此会用 restricted Boltzmann machine 等方法逐层预训练，再把层接起来微调。今天我们通常不会用那套流程，但它说明 autoencoder 不是最近才出现的技巧。

历史部分还有一个容易混淆的词：pretraining。早期逐层训练 autoencoder 也叫 pretraining；现在 BERT/GPT 先在大规模未标注资料上训练，再迁移到下游任务，也叫 pretraining。两个词长得一样，但上下文不同。读论文时要看它预训练的是每一层、整个表示模型，还是某个下游任务前的初始化。

## box warn
### 早期语境
逐层训练深度网络，让后续整体训练更容易。

## box info
### 现在语境
在大规模无标签资料上学习通用表示，再用于下游任务。

## box soft
### 现在你应该能做什么
看到 pretraining 这个词时，先看上下文，不把所有时期的用法混成一个意思。

# Denoising Autoencoder：先加噪声，再还原干净输入 {#denoising}
<!-- tag: DENOISING -->

denoising autoencoder 是常见变形。它先给输入加噪声、遮挡或破坏，然后要求 decoder 重建原本干净的输入。这样 encoder 和 decoder 不只要学“复制”，还要学“去噪”：哪些变化是噪声，哪些结构需要保留。

这个概念和 BERT 很接近。BERT 把句子中的一些 token 盖掉，模型看到的是被破坏的句子，目标是还原原句中被盖住的位置。可以把 mask 看成一种噪声，把 BERT 看成 denoising autoencoder 的一种语言版本。decoder 不一定非要是线性层；如果把 BERT 中间层当作 code，也可以把后面的层看作 decoder。

### 现在你应该能做什么
解释“普通重建”和“去噪重建”的训练目标差别，并把 BERT mask 任务放进这个框架。

# 公开分享速查：Auto-encoder 到底学了什么 {#share}
<!-- tag: quick reference -->
<!-- layout: grid2 -->

## box soft
### 核心直觉
自编码器表面上是在复制输入，真正有价值的是中间的 code。只要 code 比输入窄，模型就被迫保留最能解释资料变化的因素，而不是逐像素保存全部细节。
因此它常被当成表示学习或降维工具：训练目标是重建，使用时常拿 encoder 输出给下游任务。

## box warn
### Worked example
输入是一张手写数字图片，encoder 把几百个像素压成 16 维 code，decoder 再重建图片。若 code 太窄，模型会优先保留“像 3 还是像 8”“笔画方向”等结构信息。
如果先给图片加噪声，再要求重建干净图片，模型就不能只复制噪点，必须学会哪些变化是稳定结构。

| 术语 | 精确定义 | 易混点 |
| --- | --- | --- |
| Encoder | 把输入压成 code / embedding / representation。 | 下游常用的是 encoder 输出，不是 decoder 输出。 |
| Decoder | 根据 code 重建输入或干净目标。 | 训练时重要，但部署下游分类时可能丢弃。 |
| Bottleneck | 容量受限的中间表示。 | 太宽可能直接复制，太窄可能丢掉任务需要的信息。 |
| Reconstruction loss | 衡量重建结果与目标输入的差距。 | 重建误差低不保证表示一定适合分类。 |
| Denoising AE | 输入加噪，目标是还原干净样本。 | 它学习稳定结构。 |

## box info
### 使用检查表
- 先确定 code 维度是否真的形成约束。
- 再检查重建目标是否会鼓励模型保留任务相关结构。
- 最后用下游任务验证表示，而不是只看重建图片好不好看。

# 最小代码：训练时哪些量互相接近 {#code}
<!-- tag: MINIMAL CODE -->

下面是伪代码，重点看数据流：encoder 输出 code，decoder 输出 reconstruction，loss 比较的是重建结果和目标输入。

正确示例：基础 autoencoderpseudo python

```text
x = load_images()
z = encoder(x) # compressed code / representation
x_hat = decoder(z) # reconstruction
loss = mse(x_hat, x)
update(encoder.parameters + decoder.parameters, loss)
```

正确示例：denoising autoencoderpseudo python

```text
clean = load_images()
noisy = add_noise(clean)
z = encoder(noisy)
recovered = decoder(z)
loss = mse(recovered, clean) # target is clean input, not noisy input
```

错误示例：瓶颈太宽导致复制通道pseudo

```text
# 错误理解：code 越大一定越好。
z = encoder(x, code_dim=input_dim)
x_hat = decoder(z)

# 修正：如果 code 太宽，模型可能绕过表示学习，只学近似复制。
```

# 常见误区与修正 {#mistakes}
<!-- tag: PITFALLS -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：autoencoder 只是复制输入，所以没意义
错因：忽略瓶颈。后果：看不到 representation 的价值。修法：重点看中间 code 是否被迫压缩并保留关键变化。

## box redsoft
### 误区 2：重建越完美，表示一定越好
错因：把重建质量和下游可用性画等号。后果：可能用太宽 code 复制细节。修法：同时看瓶颈、泛化和下游表现。

## box redsoft
### 误区 3：embedding、representation、code 是三种不同对象
错因：被命名差异干扰。后果：读图时混乱。修法：在本节它们通常都指 encoder 输出的中间向量。

## box redsoft
### 误区 4：decoder 下游任务一定要保留
错因：把训练结构和使用结构混淆。后果：部署复杂。修法：很多下游任务只拿 encoder 当特征抽取器。

## box redsoft
### 误区 5：denoising 的目标是还原带噪输入
错因：没分清输入和目标。后果：模型学不到去噪。修法：输入是 noisy，目标是 clean。

## box redsoft
### 误区 6：BERT 和 autoencoder 完全无关
错因：一个处理文字，一个处理图片。后果：错过统一视角。修法：把 mask 看成噪声，把还原 token 看成 denoising reconstruction。

## box soft
### 现在你应该能做什么
用“训练输入、训练目标、瓶颈宽度、下游使用”四点检查自编码器理解是否稳。
