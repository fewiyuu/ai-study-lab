# 本节的概念地图 {#goals}
<!-- tag: MAP -->

异常检测的核心句子是：让机器知道它不知道。更具体地说，我们给机器一批训练资料，让它学会什么算“熟悉”。当新样本进来时，如果它和训练资料相似，就说 normal；如果差异太大，就说 anomaly。这里的相似不是天然给定的，不同方法会用密度、距离、重建误差、分类置信度等不同方式定义。

这一节先不深入算法，而是搭问题框架。第一，要知道异常是相对训练资料而言的。训练资料是雷丘时，皮卡丘可能是异常；训练资料是皮卡丘时，雷丘可能是异常。第二，要知道异常类不是一个普通类别。不是宝可梦的东西太多了，数码宝贝、人物、茶壶都可能是异常，根本无法穷举。

训练资料
定义熟悉区域
新样本 x
像或不像
normal / anomaly

# 问题定义：找一个判断像不像的函数 {#formulation}
<!-- tag: FORMULATION -->

设训练资料为 `x¹, x², ..., xᴺ`。课程里常用上标表示一个完整样本，用下标表示样本内部的某个部分。异常检测要学习一个 detector `f(x)`。当 `x` 像训练资料时，`f(x)` 输出 normal；当 `x` 不像训练资料时，输出 anomaly。

这和一般机器学习一样，也是在找 function。不同之处是输出“是否属于我熟悉的范围”。这个范围可能很难画。比如正常交易并不是单一数值，可能包含金额、时间、商户、频率、地点等特征；一个新交易是否异常，要看它和过去正常交易模式的整体关系。

$$
Given training data: x¹, x², ..., xᴺ
 Learn detector f(x):
 if x is similar to training data → normal
 otherwise → anomaly
$$

# 异常不一定坏：它只是“不像训练资料” {#meaning}
<!-- tag: MEANING -->

“异常”这个中文词容易让人想到负面事件，比如诈欺、入侵、癌细胞。但 anomaly detection 找到的并不一定是坏东西。它也可以叫 novelty detection，强调新奇；也可以叫 outlier detection，强调离群；也可以叫 exception detection，强调例外。不同名字的情绪色彩不同，但共同点是和训练资料不一样。

所以，异常是相对的。若训练资料全是雷丘，皮卡丘就可能被判为异常；若训练资料全是皮卡丘，雷丘就可能是异常；若训练资料是各式宝可梦，那数码宝贝、人物、茶壶都可能是异常。不要把“异常”理解成全宇宙固定的坏类别，它由你给模型看的训练资料决定。

训练资料：雷丘皮卡丘可能不像训练资料，因此可被视为异常。
训练资料：皮卡丘雷丘可能变成异常。判断标准跟资料有关。
训练资料：宝可梦不是宝可梦的对象更可能被视为异常。

# 常见应用：诈欺、入侵、医疗 {#cases}
<!-- tag: APPLICATIONS -->

诈欺侦测中，训练资料通常是大量正常交易。新交易如果金额异常、频率异常、地点异常或行为组合异常，就可能被标记为可疑。网络入侵侦测中，训练资料是正常连线或正常系统行为；新的连线若不像正常模式，就可能是攻击。医疗场景中，训练资料可以是正常细胞或正常影像，新样本若不像正常组织，可能需要进一步检查。

这些应用有一个共同点：正常资料比较容易大量收集，异常资料稀少、昂贵、变化大，甚至事后才知道。也正因为这样，异常检测才有独立价值。如果每种异常都有大量标注样本，很多问题可以退化成监督分类；但现实里往往没有这么舒服。

| 场景 | 训练资料常是什么 | 新样本异常可能意味着 |
| --- | --- | --- |
| 信用卡诈欺 | 正常交易记录 | 盗刷或不寻常消费行为 |
| 网络入侵 | 正常连线或系统日志 | 攻击、扫描、异常流量 |
| 医疗检测 | 正常细胞或正常影像 | 病变、癌细胞、需要复查的区域 |

### 现在你应该能做什么
把一个业务问题改写成“正常资料是什么，新样本什么情况下不像它”。

# 为什么不是普通二分类 {#notbinary}
<!-- tag: NOT BINARY -->

最直觉的想法是：正常资料作为 class 1，异常资料作为 class 2，训练 binary classifier。这个想法只有在异常类别收集充分、边界稳定时才比较合适。异常检测真正困难的地方在于，异常类不是一个紧凑类别。正常是宝可梦时，异常可以是数码宝贝、人物、茶壶、车、噪声图片和无数其他东西。你无法穷举“不是宝可梦”的分布。

第二个问题是异常资料难收集。诈欺交易本来就少，而且有些交易一开始不知道是不是诈欺。网络攻击手法会更新，医疗异常也可能稀有。训练一个普通二分类器需要两边类别都有足够代表性；异常检测常常只有正常资料，或只有极少量、不完整的异常资料。

## box redsoft
### 异常类太大
“不是正常”的范围几乎无限，不能假设它像一个普通类别那样可以被完整采样。

## box redsoft
### 异常样本太少
现实里异常事件稀有、昂贵、标签不可靠，难以训练稳定的二分类器。

## box soft
### 现在你应该能做什么
说清楚异常检测和 binary classification 的根本差别：异常类不可穷举且常常缺标注。

# 三类设定：有标签、干净无标签、污染无标签 {#types}
<!-- tag: SETTINGS -->

第一类设定是有标签的 open set recognition。训练资料有已知类别标签，例如 class 1、class 2、class 3。模型可以训练成普通分类器，但测试时可能出现训练时没有的 unknown 类。我们希望模型不仅能分已知类别，也能在遇到没见过的东西时说“这是 unknown”。

第二类设定是无标签且训练资料干净。也就是训练资料都来自 normal，只是没有其他类别标签。模型要学习 normal 的范围，再把不像 normal 的新样本标出来。第三类更现实：无标签但训练资料被污染。你以为训练资料都是 normal，其实里面混了一点 anomaly，比如 0.1% 或 1%。这会让模型定义 normal 时被异常样本干扰。

| 设定 | 训练资料 | 目标 | 难点 |
| --- | --- | --- | --- |
| Open set recognition | 有已知类别标签 | 分类已知类，同时识别 unknown | 训练时没有 unknown 标签 |
| Clean one-class | 无标签，但全是正常资料 | 学正常范围 | 如何定义像不像 |
| Contaminated setting | 无标签，少量异常混在里面 | 在污染资料中仍识别正常模式 | 异常样本会污染正常边界 |

### 现在你应该能做什么
拿到一个异常检测任务时，先判断训练资料有没有类别标签、是否保证干净。

# 阈值取舍：太敏感会误报，太宽松会漏报 {#threshold}
<!-- tag: THRESHOLD -->

很多异常检测方法最后都会给一个 anomaly score 或 normal score。分数高不一定直接等于异常，通常还要设阈值。阈值越严格，越多样本会被标异常，可能抓到更多真正异常，也会误报更多正常样本。阈值越宽松，误报减少，但真正异常可能漏掉。

本节原稿还没进入具体算法，但阈值直觉很重要。无论后面用距离、密度、重建误差还是分类置信度，都会遇到“像不像到什么程度才算异常”。异常检测不是只训练模型，还要决定业务上能接受多少误报和漏报。

# 公开分享速查：异常检测先问“像不像” {#share}
<!-- tag: quick reference -->
<!-- layout: grid2 -->

## box soft
### 核心直觉
异常检测先定义“正常资料长什么样”。新样本如果离这个正常世界太远，就被标成 anomaly candidate。
因此异常是相对概念：换一批训练资料，normal 和 anomaly 的边界也会改变。

## box warn
### Worked example
如果训练资料只有皮卡丘，雷丘可能是异常；如果训练资料是“宝可梦电系角色”，雷丘可能是正常；如果训练资料是“所有宝可梦”，一张真实猫照片才更像异常。
同一个样本不会天然自带 anomaly 标签，关键是它相对哪批训练资料“不像”。

| 术语 | 精确定义 | 易混点 |
| --- | --- | --- |
| Anomaly / Outlier / Novelty | 相对训练资料不常见或不相似的样本。 | 不等于一定坏、恶意或错误。 |
| Normal data | 用来建立“熟悉世界”的训练样本。 | normal 不是道德判断，只是训练分布的参照。 |
| Anomaly score | 衡量不像正常资料的分数。 | 分数本身不是最终决策，还需要阈值。 |
| Open set recognition | 训练时有已知类标签，测试时可能出现未知类。 | 不是普通多分类，因为允许拒绝为 unknown。 |
| Contaminated training | 训练资料中混入少量异常样本。 | 比干净无标签设定更难，因为“正常模板”会被污染。 |

## box info
### 建模检查表
- 先写清训练资料代表什么正常范围。
- 再说明新样本如何被打分：距离、概率、重建误差或分类信心。
- 最后写清阈值取舍：误报和漏报哪个更贵。

# 最小代码：先写出抽象 detector {#code}
<!-- tag: MINIMAL CODE -->

下面伪代码先不指定具体算法，只表达异常检测的共通形式：训练阶段学习正常资料的某种描述，测试阶段把新样本转成分数，再和阈值比较。

正确示例：正常分数与阈值pseudo python

```text
normal_data = load_training_data()
model = fit_normal_pattern(normal_data)

score = anomaly_score(model, x_new) # larger means less like training data
if score > threshold:
 label = "anomaly"
else:
 label = "normal"
```

正确示例：open set recognition 的判断pseudo python

```text
probs = classifier(x_new)
if max(probs) < confidence_threshold:
 label = "unknown"
else:
 label = argmax(probs)
```

错误示例：把所有异常当成一个完整类别pseudo

```text
# 错误理解：只要收集几种异常，就代表异常类完整。
normal = pokemon_images
anomaly = digimon_images
train_binary_classifier(normal, anomaly)

# 修正：异常可能是任何不像训练资料的东西，不能假设已穷举。
```

# 常见误区与修正 {#mistakes}
<!-- tag: PITFALLS -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：异常一定是坏东西
错因：被“异常”这个词影响。后果：忽略 novelty detection 等正向场景。修法：异常只是和训练资料不一样。

## box redsoft
### 误区 2：异常有固定定义
错因：忘记训练资料决定 normal。后果：同一样本在不同训练资料下判断相反。修法：先问训练资料是什么。

## box redsoft
### 误区 3：异常检测就是二分类
错因：看到 normal/anomaly 两个输出。后果：低估异常类不可穷举。修法：区分输出形式和训练设定。

## box redsoft
### 误区 4：只要有几种异常样本，就覆盖了异常类
错因：把异常当普通类别。后果：新型异常会漏掉。修法：异常是“非正常”的开放集合。

## box redsoft
### 误区 5：无标签训练资料一定全正常
错因：过度相信数据收集过程。后果：模型边界被异常污染。修法：考虑 contaminated setting。

## box redsoft
### 误区 6：模型输出分数后就不用调阈值
错因：把分数当最终决策。后果：误报/漏报不可控。修法：根据任务代价设阈值。

## box soft
### 现在你应该能做什么
用“训练资料、异常集合、标签条件、阈值代价”四点检查一个异常检测方案。
