分类器也可以先学会说 unknown

P30 处理的是有标签的异常检测设定。训练资料有已知类别标签，例如辛普森家庭里每张图片都标了具体人物。我们可以先训练一个分类器，让它判断输入图片是 Bart、Lisa、Homer 还是其他已知角色。问题是：测试时如果给它一张训练世界之外的图片，它能不能不要硬分到某个已知角色，而是说 unknown？

一个直接做法是看分类器的 confidence score。若 softmax 分布非常尖，最高类别分数很高，模型可能比较有把握；若分布平均，模型可能不确定。设一个阈值：信心分数高于阈值就接受为已知类，低于阈值就拒绝为 unknown。这是简单 baseline，但在很多任务里并不弱。

本节的概念地图

MAP

open set recognition 的训练阶段像普通分类：训练资料里只有已知类，而且每个样本都有标签。模型学会在这些已知类之间分类。测试阶段不同：输入可能来自训练时没有出现过的 unknown 类。此时模型不能只输出一个已知标签，还要给出“我是否有足够信心”的判断。

这一节用辛普森家庭角色分类器做例子。若输入是已知角色，softmax 可能给某个角色很高分，例如 0.97。若输入是其他动画人物，分数可能更平均，最高分也较低。于是我们可以把最高 softmax 分数当作 confidence score，也可以用 entropy 衡量分布平均程度。分数再和阈值比较，得到 known 或 unknown 的决策。

已知类训练资料 训练分类器 softmax 分布 confidence score known / unknown

现在你应该能做什么

把 open set recognition 解释成“分类 + 拒绝未知输入”。

有标签设定：已知类内部可以正常训练分类器

SETTING

在 P29 的分类中，P30 属于“训练资料有标签”的 case。每张训练图片属于某个已知类别。以辛普森家庭为例，训练集中有 Bart、Lisa、Homer、Marge 等角色，并且每张图都标明是谁。用这些资料训练一个普通分类器没有问题，它可以在已知角色之间做多分类。

open set 的额外要求发生在测试时。测试样本可能是辛普森角色，也可能是其他动画人物、真人照片或完全不同的图。普通分类器的 softmax 一定会在已知类别中选一个最大值；open set recognition 要多加一步：如果信心不足，就拒绝把它分到任何已知类，输出 unknown。

Closed set classification

假设测试样本一定属于训练见过的类别。模型只需要在已知类中选一个。

Open set recognition

测试样本可能来自未知类别。模型除了分类，还要判断是否拒绝。

现在你应该能做什么

说明为什么 open set 比普通分类多了 unknown 判断。

信心分数：分类器输出不只拿来选类

CONFIDENCE

分类器通常输出一个 softmax 分布。这个分布给每个已知类别一个分数，所有分数加起来为 1。普通分类只关心分数最高的类别；open set recognition 还关心这个最高分到底有多高。如果最高分很高，模型可能比较肯定；如果最高分不高，模型可能只是被迫在已知类里选一个。

“比较肯定”不等于“一定正确”。softmax 分数不是完美概率，神经网络可能对未知输入过度自信。课程里的例子就出现过：某些非辛普森角色会被分类器以很高信心归到某个辛普森角色。这正是后续研究要改进的地方。但作为第一步，max softmax probability 是一个简单、便宜、常常值得先试的 baseline。

现在你应该能做什么

读 softmax 输出时，不只看 argmax 类别，也看最高分是否足够高。

方法一：Max Softmax Probability

MAX SCORE

最直觉的 confidence score 是 max p(y|x)，也就是 softmax 分布里最大的值。如果模型看到 Homer 的图片，输出 Homer=0.97，其他类别接近 0，那么 max score 是 0.97。若阈值是 0.7，就接受为 known，并把类别设为 Homer。

如果输入是未知人物，softmax 可能比较平均，例如最高分只有 0.26。此时 max score 小于阈值，就输出 unknown。这套做法不需要改训练流程，也不需要额外模型，只要已有分类器就能试。它特别适合做第一版系统或 sanity check：如果这个 baseline 都没有跑，直接上复杂方法容易失去参照。

confidence(x) = max_k p(y = k | x)
if confidence(x) ≥ λ → known class argmax_k p(y=k|x)
otherwise → unknown

给一组 softmax 分数，算出 max score，并按阈值判断 known/unknown。

方法二：Entropy 看分布是否平均

ENTROPY

另一个思路是看 entropy。entropy 衡量分布的平均程度。若某个类别分数接近 1，其他类别接近 0，entropy 低，表示分布尖锐；若各类别分数差不多，entropy 高，表示模型没有明显偏好。open set 中，可以把高 entropy 看作低信心。

max score 和 entropy 往往会给相近直觉：最高分高时，entropy 通常低；最高分低且分布平均时，entropy 通常高。但它们不是完全等价。max score 只看最大值，entropy 会看整个分布。实际使用时，文献里有多种 confidence 设计，课程强调这些简单方法的差异不一定特别大，关键是先有可解释的分数和阈值。

0.97, 0.01, 0.01, 0.01高低像已知类
0.26, 0.24, 0.23, 0.27低高模型犹豫
0.63, 0.08, 0.07, ...中高中可能过度自信，要小心

阈值与 development set：λ 不是凭空来的

THRESHOLD

课程里把阈值写成 λ。如果 confidence score 大于 λ，就认为输入来自已知类；若小于 λ，就拒绝为 unknown。阈值不能随便拍脑袋，它通常要根据 development set 调整。development set 应该模拟你关心的测试情况：既有已知类样本，也尽量有一些可代表未知情况的样本。

阈值越高，模型越谨慎，更多样本会被拒绝为 unknown。这会减少把未知样本误收为已知类的风险，但也可能把真正已知类误拒。阈值越低，模型更容易接受样本为已知类，误拒会少一些，但未知样本更容易混进来。open set recognition 的实际表现常常取决于这个取舍。

阈值太高

known 样本也可能被拒绝，使用体验会变保守。

阈值太低

unknown 样本更容易被硬塞进已知类，安全或质量风险升高。

实验观察：已知类通常高分，但会有例外

OBSERVATION

辛普森家庭例子里，分类器在已知角色测试集上准确率很高，且多数样本的 confidence score 接近 1。把其他动画人物丢进去时，多数未知样本的信心分数会低一些，于是 max score 的分布和已知类分布有明显差异。这说明简单信心分数确实能捕捉一部分 known/unknown 差异。

但也有例外。有些未知角色会被模型非常自信地归到某个已知角色，课程里提到某些非辛普森人物被高分判成 Carl。这通常与训练资料里的视觉偏差有关，例如某个角色肤色或发色不像典型黄色脸，导致未知人物容易被吸过去。这提醒我们：confidence score 有用，但不能迷信。

已知角色

分数常集中在高信心区域，分类器比较确定。

多数未知

分布更平均，最高分较低，容易被阈值拒绝。

例外未知

有些未知样本会触发过度自信，成为 open set 的难点。

简单 baseline 不等于弱方法

BASELINE

课程特别提醒：如果手上已经有分类器，先用 confidence score 做异常检测是很合理的第一步。它实现简单、解释清楚、成本低，而且在不少场景表现不差。工程上，简单方法可以帮你快速建立参照：后续复杂方法如果不能明显超过它，就未必值得引入。

当然，也有更专门的方法。比如训练分类器时就让它直接输出更可靠的 confidence，或者加入额外目标校准已知/未知边界。这类工作属于活跃研究方向。入门时先掌握本节的 baseline，再读更复杂论文会更稳，因为你知道复杂方法到底想改进什么。

遇到有分类器的 unknown 检测任务，先写出一个 max score baseline。

最小代码：从 softmax 到 unknown

MINIMAL CODE

下面伪代码展示 open set recognition 的基本数据流。重点是：先分类，再用信心分数决定是否拒绝。

正确示例：max probability 作为信心分数pseudo python
probs = softmax(classifier(image))
confidence = max(probs)
predicted_class = argmax(probs)

if confidence >= threshold:
    label = predicted_class
else:
    label = "unknown"
正确示例：entropy 作为不确定性pseudo python
entropy = -sum(p * log(p) for p in probs)
if entropy <= entropy_threshold:
    label = argmax(probs)
else:
    label = "unknown"
错误示例：永远强迫输出已知类pseudo
# 错误：测试样本可能来自未知类，但模型没有拒绝机制。
probs = softmax(classifier(image))
label = argmax(probs)

# 修正：加入 confidence score 和 threshold。

常见误区与修正

PITFALLS

误区 1：分类器最高分就是可靠概率

错因：把 softmax 分数当校准过的真实概率。后果：未知样本可能被高信心误收。修法：把它当 baseline 分数，并用验证集评估。

误区 2：open set 只要 argmax 就够

错因：沿用 closed set 分类习惯。后果：未知样本被强塞到已知类。修法：加入拒绝阈值。

误区 3：阈值可以随便定

错因：忽略 known/unknown 代价。后果：误拒或误收不可控。修法：用 development set 调整。

误区 4：unknown 分数低说明模型一定错

错因：混淆分类正确性和开放集拒绝。后果：把低信心已知样本都当异常。修法：看阈值取舍和验证结果。

误区 5：entropy 和 max score 完全一样

错因：都反映信心。后果：忽略 entropy 看整个分布。修法：理解一个看最大值，一个看分布形状。

误区 6：简单 baseline 没必要做

错因：以为简单就弱。后果：复杂方法缺少参照。修法：先跑 max score/entropy，再谈改进。
