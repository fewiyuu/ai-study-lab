P32｜异常检测 4：分类器过度自信

P30 的方法是用分类器信心分数做异常检测：信心高就接受为已知类，信心低就判为 unknown。P32 讲这个方法的关键失败模式：有些 unknown 并不会让分类器犹豫，反而会让它更自信。因为分类器只学会区分训练中的类别特征，不一定学会“训练世界之外”的边界。

MAP

P30 的信心分数方法隐含一个假设：未知样本会让分类器分布变平均，最高 softmax 分数变低。这个假设在很多时候有效，但不是定律。分类器训练时只见过已知类，它学到的是区分已知类的特征，而不是完整的“正常世界边界”。

P30 回顾：信心低就拒绝

RECAP

上一节的流程很简单。先训练一个已知类分类器，例如辛普森人物分类器。测试时，拿 softmax 最大值或 entropy 当作 confidence score。若最高分高于阈值，就输出某个已知类；若低于阈值，就输出 unknown。

$$ baseline assumption: unknown input → uncertain softmax → low confidence → reject $$

失败模式：unknown 可能比 known 更像某个类

FAILURE

想象一个猫狗分类器。它可能学到“尖耳朵、胡须、某种脸型”是猫特征，“长嘴、某种耳朵、某种毛发纹理”是狗特征。草泥马既不像猫也不像狗，可能低信心。但老虎有很多猫科特征，而且更大、更强烈；狼有很多犬科特征，也可能强烈触发狗类 logit。

远离已知特征模型可能低信心，baseline 有机会拒绝。

强化已知特征unknown 触发某个已知类 logit，模型高信心误收。

训练目标缺口分类器没被教过“强猫特征但不是训练猫”的样本。

小交互：特征越强，信心可能越高

INTERACTION

下面的简化模型只看两个 logit：猫分数和狗分数。真实分类器当然复杂得多，但这个交互能展示一种直觉：当某个输入强烈触发猫特征时，softmax 最高分会上升，即使这个输入根本不是训练中的猫。

高信心判为猫

如果 unknown 强烈触发猫特征，分类器会自信地输出猫。这训练目标只学过猫狗边界。

黄脸例子：模型可能抓到很脆的线索

SPURIOUS

课程里用辛普森家庭分类器做了一个很直观的实验。辛普森人物常见特征是黄脸、嘴型特殊。若把非辛普森人物的脸涂黄，分类器的 confidence 会明显上升；把头发也涂黄，甚至可能高信心判成某个辛普森角色。

黄脸强度

观察重点

黄色越强，简化分类器越容易把它当作辛普森线索。真实模型还会看更多特征，但脆弱线索仍可能造成过度自信。

方向一：拿真实异常资料训练低信心

OUTLIER

一个直接解法是：如果能收集到一些异常资料，就把它们放进训练流程。已知类样本仍然训练分类正确；异常样本则训练模型输出低 confidence，或训练它的 softmax entropy 变高。这样模型不只学会“已知类之间怎么分”，还学会“看到某些训练分布外样本时不要太自信”。

训练目标

known：正确分类且高信心；outlier：低 max score 或高 entropy。

限制

异常资料难收集、覆盖不完整，还可能和真实测试 unknown 不一致。

方向二：用生成模型造“像但不太像”的异常

SYNTHETIC

如果真实异常资料很难收集，可以尝试用 generative model 生成异常资料。直觉是：让生成模型产生一些有点像正常资料、但又不属于正常资料的样本，再用这些样本训练分类器低信心。这样模型可以在正常边界附近见到更多“不要太自信”的例子。

生成样本状态问题训练效果
太像正常可能混淆 known 类伤害分类或误教模型
太不像正常太容易拒绝对困难 unknown 帮助有限
有点像但不太像边界附近更有训练价值可教模型降低过度自信

公开分享速查：Unknown 也可能高信心

quick

核心直觉

分类器学到的是区分已知类的线索，不是完整的世界边界。Unknown 如果强烈触发某个已知类线索，max softmax 可能非常高。

因此 P32 指出：只看信心分数，会在边界附近和特征外推时失败。

Worked example

猫狗分类器见到老虎，可能因为“胡须、耳朵、脸型”强烈像猫而高信心输出 cat。它没有学过“老虎不是宠物猫”这个边界。

改进思路是让模型在训练时见到 outlier，并被要求对这些样本输出低 max score 或高 entropy。

术语精确定义易混点
Overconfidence模型对未知或错误样本给出很高信心。高 softmax 是模型状态，不等于客观正确。
Feature shortcut模型依赖少数强特征做判断。训练集上有效的线索，在 open set 中可能误导。
Outlier exposure用额外异常样本训练模型低信心。效果受异常样本覆盖范围限制。
Synthetic outlier用生成模型造边界附近异常。太像 normal 或太不像 normal 都会降低价值。

诊断检查表

  • 先找 unknown 为什么触发了已知类特征。
  • 再检查训练集中是否有类似 outlier 教模型降低信心。
  • 生成 outlier 时，要控制“像但不属于”的程度。

最小代码：让 outlier 高 entropy

MINIMAL

下面伪代码表达训练目标。known 样本用普通 cross entropy；outlier 样本不要求分到某个已知类，而是希望输出分布更平均，或 max score 更低。

代码
logits_known = classifier(x_known)
loss_cls = cross_entropy(logits_known, y_known)

logits_out = classifier(x_outlier)
probs_out = softmax(logits_out)
loss_out = -entropy(probs_out) # maximize entropy

loss = loss_cls + alpha * loss_out

常见误区与修正

PITFALLS

误区 1：unknown 一定低 confidence

错因：只看 P30 baseline 的理想情况。后果：忽略老虎/狼/黄脸这类触发特征。修法：考虑未知样本可能强化已知特征。

误区 2：高 softmax 代表样本来自训练分布

错因：把分类边界信心当分布内证明。后果：open set 误收。修法：softmax 高只说明某个已知类 logit 高。

误区 3：黄脸特征可靠代表辛普森

错因：把训练集相关线索当语义边界。后果：被涂黄 unknown 骗过。修法：检查 spurious feature。

误区 4：收集一点异常样本就覆盖 unknown

错因：把未知世界当固定小集合。后果：新型异常仍可能漏掉。修法：把 outlier exposure 当改进，不当完美解。

误区 5：生成异常越离谱越好

错因：只想让模型看到不同东西。后果：训练信号太容易，对边界无帮助。修法：生成有点像但不太像的困难样本。

误区 6：直接丢掉 confidence baseline

错因：看到失败例就否定简单方法。后果：失去参照。修法：保留 baseline，再用异常训练改进它。
