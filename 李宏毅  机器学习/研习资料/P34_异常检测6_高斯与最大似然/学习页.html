让资料自己告诉我们，高斯应该长什么样

上一节已经把无标签异常检测变成一个概率问题：学习 p(x)，再用新样本的概率密度判断它像不像训练资料。P34 进一步问：这个 p(x) 到底怎么学？如果我们决定用 Gaussian distribution，那么它的中心 mu 和形状 Sigma 应该怎么从训练资料中找出来？

答案是 likelihood。直觉上，我们假设训练资料是某个概率密度函数生成出来的；哪一组参数让“这些训练资料被生成出来”这件事最合理，我们就选哪一组参数。对高斯而言，最大似然估计会得到很熟悉的结果：mu 是训练向量的平均，Sigma 是围绕平均的协方差矩阵。

本节的概念地图

MAP

无标签异常检测的完整链路是：先收集多数正常资料，把每个对象变成向量；再选择一个概率密度函数家族，例如 Gaussian；然后用训练资料估计参数；最后把新样本放进这个密度函数，若密度低于阈值，就把它视为异常候选。

这一节最容易卡住的地方，是把 p(x) 和 likelihood 混成同一个东西。p(x) 是在参数已经固定时，对某个样本打分；likelihood 是把训练资料固定住，反过来比较不同参数哪一组更能解释这些资料。它们用同一个密度函数计算，但提问方向相反。

选密度家族 写出 p(x|theta) 计算 likelihood 找 theta* 用阈值判异常

现在你应该能做什么

把 P34 的主线概括为：用最大似然从训练资料估计高斯分布，再用密度阈值做异常检测。

Likelihood：不是先验相信谁，而是看谁更能解释资料

LIKELIHOOD

课程中先假设存在一个 probability density function，记作 p(x; theta)。这里 theta 是参数，决定密度函数的形状。如果 theta 不同，中心、宽窄、方向都可能不同，于是同一批训练资料得到的分数也不同。

假设训练资料是 x¹, x², ..., xᴺ。如果把每个样本看成独立从同一个密度函数抽出来，那么整批资料出现的合理程度可以写成每个样本密度的乘积。这个乘积就是 likelihood：

L(theta) = p(x¹; theta) × p(x²; theta) × ... × p(xᴺ; theta)
训练资料固定，参数 theta 变化；我们要找让 L(theta) 最大的 theta。

严格说，连续型资料里的 p(x) 是 probability density，不是一个介于 0 到 1 的概率值。密度可能大于 1，因为真正的概率来自某个区域下的面积或体积。不过入门理解时，可以暂时把它当作“这个位置有多常见”的分数。只要记得：密度越高，表示这个位置越符合模型认为的常见区域。

在计算时我们通常不会直接乘很多小数，因为乘积很容易下溢为 0。实际程序常用 log-likelihood，把乘法变成加法：

log L(theta) = Σᵢ log p(xⁱ; theta)
最大化 L(theta) 与最大化 log L(theta) 得到同一组参数，因为 log 是单调递增函数。

直觉

如果某个参数设定让训练点大多落在高密度区域，likelihood 就大；如果中心偏远、形状不对，训练点看起来像“很偶然才会出现”，likelihood 就低。

易错点

likelihood 不是“参数发生的概率”。在这里我们只是比较参数对资料的解释力，不是在说 theta 自己随机抽中的概率。

现在你应该能做什么：给定一批训练点和两组候选参数，判断哪组参数更可能有较大 likelihood，并说明理由。

多变量高斯：用中心和协方差描述一团资料

GAUSSIAN

课程接着选择一个常用的密度函数：Gaussian distribution。二维时你可以把它想成平面上的一座山。山顶是最高密度区域，越远离山顶，密度越低。若协方差允许不同方向有不同宽度，这座山的等高线会从圆形变成椭圆。

多变量高斯的两个核心参数是 mu 和 Sigma。mu 是平均向量，决定山顶的位置；Sigma 是 covariance matrix，决定资料云的伸展方向、宽窄和不同特征之间的相关性。若两个 feature 经常一起变大，协方差矩阵会把这种斜向关系表现出来。

p(x; mu, Sigma) = 1 / ((2π)D/2 |Sigma|1/2) · exp( -1/2 (x - mu)T Sigma-1 (x - mu) )
D 是 feature 维度；|Sigma| 是 determinant；指数项衡量 x 离中心有多远，而且会考虑不同方向的尺度。

这个公式看起来重，但拆开后并不神秘。前面的系数负责让整个密度函数正规化，使总面积或总体积为 1。后面的指数项负责给每个位置打分：当 x 很接近 mu 时，距离项小，密度高；当 x 沿着低方差方向偏离很远时，密度会快速下降。

符号直觉错用后果
x一个样本的 feature vector，例如玩家行为统计。feature 选得不合理，模型只能学到不相关的分布。
mu训练资料的中心位置。中心偏离资料群，正常样本也会被打低分。
Sigma资料的尺度、方向和 feature 相关性。忽略相关性时，斜向正常模式可能被误判。
p(x)样本在模型下的密度分数。把密度当绝对概率，会误解阈值和跨模型比较。

现在你应该能做什么：看到一个二维资料云时，说出高斯的 mu 大概在哪里、Sigma 大概是圆形还是椭圆，以及为什么。

最大似然估计：让训练资料的出现最合理

MLE

现在我们选择了高斯作为密度函数，参数从一般的 theta 变成 mu 与 Sigma。学习问题就是：

(mu*, Sigma*) = arg maxmu,Sigma Πᵢ p(xⁱ; mu, Sigma)
读法：在所有可能的 mu 与 Sigma 中，找出让训练资料 likelihood 最大的一组。

对单一高斯来说，这个最佳解有闭式形式，也就是不用反复梯度下降，可以直接套公式。平均向量就是所有训练向量的平均：

mu* = (1/N) Σᵢ xⁱ

协方差矩阵是每个样本减去平均后，外积再平均：

Sigma* = (1/N) Σᵢ (xⁱ - mu*) (xⁱ - mu*)T

为什么这个结果合理？如果 mu 离训练资料中心太远，许多训练点都会落在低密度区域，likelihood 变小。若 Sigma 太小，模型会认为资料应非常集中，稍微远一点的训练点就变成低密度；若 Sigma 太大，密度山被摊得太平，所有训练点的密度也会下降。最佳协方差要刚好描述资料的真实散布。

中心

平均向量让资料点围绕中心的总偏差处在最佳平衡。

尺度

方差越大，模型承认正常资料可以散得更开；方差越小，模型更严格。

相关性

完整协方差能表达两个 feature 的联动，斜向椭圆就是这种关系的可视化。

现在你应该能做什么：不用背复杂推导，也能解释为什么高斯最大似然的 mu* 是平均，Sigma* 是围绕平均的散布。

阈值与等高线：从密度分数变成决策

THRESHOLD

得到 mu* 和 Sigma* 后，每个新输入 x 都能算出 p(x; mu*, Sigma*)。密度本身只是一个连续分数，要做 anomaly detection，还需要设定阈值 lambda：

p(x; mu*, Sigma*) ≥ lambda → normal
p(x; mu*, Sigma*) < lambda → anomaly candidate

在二维图上，固定 lambda 会得到一条等高线。等高线里面是高密度区域，外面是低密度区域。课程提到这个阈值通常由 development set 决定：如果你有少量验证资料，就可以根据误报、漏报或业务代价来选。

阈值越高，模型越严格，更多样本会被标为异常，可能提高召回但增加误报。阈值越低，模型越宽松，误报较少，但真正异常可能被放过。异常检测的阈值从来不是纯数学问题，它总是和应用场景的代价相连。

阈值选择效果适合场景
较高低密度边界向中心收缩，更多点被判异常。宁可多检查，也不想漏掉风险。
中等在误报和漏报之间折中。验证集比较能代表真实使用情况。
较低只有非常罕见的点才被判异常。人工检查成本高，误报代价很大。

现在你应该能做什么：看到一个密度热力图时，把阈值想成某条等高线，并说出提高或降低阈值会改变什么。

高斯很常用，但它是强假设

ASSUMPTION

课程中特别提醒：选择 Gaussian 并不是因为世界上的资料都天然服从高斯，而是因为它常用、简单、可解释、还有闭式解。许多 feature 的真实分布可能根本不像高斯，例如只在 0 到 1 之间、明显偏斜、长尾、多峰，或者有奇怪边界。

实践中常见做法是做 feature transformation，让资料看起来更接近高斯。例如对长尾特征取 log，对计数特征取平方根，或者把比例类特征做合适的缩放。这样做不是魔法，而是在承认模型有假设：如果你非要用高斯，就尽量把输入表示整理到高斯比较能描述的形态。

如果资料分布非常复杂，单一高斯可能不够。你可以换更有弹性的概率模型，例如 Gaussian mixture model、由神经网络控制的生成模型，或后面课程会接触到的其他表示学习方法。核心思想仍然一样：学习正常资料的分布，再找低密度或低重构合理性的样本。

偏斜分布

大多数值很小、少数值极大时，直接套高斯会把尾部处理得很粗糙。log 变换常用来压缩长尾。

多峰分布

正常资料本来有多个群，例如不同类型的正常玩家。单一高斯只能给一座山，容易把群之间的空隙也当正常。

边界特征

比例永远在 0 到 1 之间，但高斯在整个实数空间都有密度。解释结果时要意识到这个近似。

现在你应该能做什么：说明“用高斯”是一种建模选择，并能说出至少两种资料不适合单一高斯的原因。

公开分享速查：高斯参数由资料决定

核心直觉

用高斯做无标签异常检测时，我们先假设正常资料像一团云。mu 是云的中心，Sigma 描述云往哪些方向拉长、不同特征是否一起变化。

最大似然估计不是凭感觉挑参数，而是选择让训练资料这团云看起来最合理的中心和形状。

Worked example

三位正常玩家的刷屏比例分别是 0.10、0.12、0.14，粗略中心就是 0.12。新玩家刷屏比例 0.13 很靠近中心，0.90 则远离中心。

多维时不能只看每个特征离中心多远，还要看特征是否相关。若两个特征总是一起升高，沿着这条方向偏移可能比垂直偏移更正常。

术语精确定义易混点
Probability density给定参数后，某个 x 所在位置的密度。连续空间里密度可以大于 1，不是单点概率。
Likelihood固定训练资料，反过来看哪组参数更能解释这些资料。不是模型“相信参数”的先验概率。
mu高斯中心，最大似然下通常是样本平均。中心靠近不代表一定正常，还要看协方差和阈值。
Sigma协方差矩阵，描述宽窄、方向和特征相关性。不是每个维度单独一个方差那么简单。
Log likelihoodlikelihood 的对数形式，方便数值计算。比较阈值时不能混用原始概率和 log 分数。

建模检查表

  • 先画或检查 feature 分布，确认单一高斯是否太粗糙。
  • 估计 mu 和 Sigma 后，用 development set 选择 log likelihood 阈值。
  • 若资料多峰、长尾或有边界，考虑变换特征或换更弹性的模型。

最小代码：从训练向量到异常分数

CODE

下面的代码只保留核心步骤：估计平均和协方差、计算高斯 log density、根据阈值做判断。为了避免数值问题，代码使用 log density；比较阈值时也应使用 log threshold。

正确示例：高斯异常检测最小流程Python / NumPy
import numpy as np

def fit_gaussian(X, eps=1e-6):
    # X: shape = (N, D)，每一列是一个 feature。
    mu = X.mean(axis=0)
    centered = X - mu
    sigma = centered.T @ centered / len(X)
    sigma = sigma + eps * np.eye(X.shape[1])
    return mu, sigma

def log_gaussian_density(x, mu, sigma):
    d = len(mu)
    diff = x - mu
    inv = np.linalg.inv(sigma)
    log_det = np.linalg.slogdet(sigma)[1]
    mahalanobis = diff.T @ inv @ diff
    return -0.5 * (d * np.log(2 * np.pi) + log_det + mahalanobis)

mu, sigma = fit_gaussian(train_X)
score = log_gaussian_density(new_player, mu, sigma)
is_anomaly = score < log_lambda

eps 是一个很小的稳定项。若 feature 之间高度相关，或样本数太少，协方差矩阵可能接近不可逆；加上小的对角项能让矩阵反演更稳定。它不改变主线，但能避免实际代码在数值上崩掉。

错误示例：把测试样本混进参数估计Data leakage
# 错误示例：新玩家被加入训练资料后再估计 mu 和 sigma。
all_X = np.vstack([train_X, new_player[None, :]])
mu, sigma = fit_gaussian(all_X)
score = log_gaussian_density(new_player, mu, sigma)

这段错在数据泄漏。异常检测要问的是“新样本像不像已知正常资料”，不能先让新样本影响正常资料的分布。如果异常点被加入训练，它可能把中心和协方差拉过去，让自己看起来没那么异常。

常见误区与修法

DIAGNOSIS

误区 1：把低密度直接等同于坏人

低密度只表示不像训练资料。可能是恶意行为，也可能是新手、特殊玩法、资料收集偏差或 feature 设计问题。修法是称为“异常候选”，再结合任务证据处理。

误区 2：把 density 当普通概率

连续密度可以大于 1，不能直接说“这个玩家出现概率是 1.2”。修法是把它作为相对分数或区域概率的基础。

误区 3：忘记取 log

很多小密度相乘会数值下溢。修法是使用 log-likelihood，把乘法转成加法。

误区 4：用训练集异常率直接定阈值

训练资料不一定干净，也不一定代表上线代价。修法是用 development set、人工审查预算或业务成本选择阈值。

误区 5：盲目相信单一高斯

多峰、长尾、有边界的资料可能不适合单一高斯。修法是做 feature transformation，或换更有弹性的模型。

误区 6：忽略 feature 尺度

某个 feature 数值范围特别大，会主导协方差和距离。修法是标准化、缩放，或重新设计 feature。

现在你应该能做什么：看到一个异常检测结论时，不只问“分数低不低”，还会追问 feature、分布假设、阈值来源和验证资料是否合理。
