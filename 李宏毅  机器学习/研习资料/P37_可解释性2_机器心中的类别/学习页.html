机器心中的猫，未必长得像人心中的猫

P36 讲 local explanation：给定一张图片，问模型为什么这样判断。P37 转到 global explanation：不针对某张图片，而是拿训练好的模型出来，问它内部某个 filter 想看什么 pattern，或者对这个模型而言，某个类别的理想输入长什么样。

方法听起来直接：把输入图片 X 当成要优化的变量，用 gradient ascent 找一张能让某个 filter 或某个类别分数最大化的图片。但课程强调：如果只最大化类别分数，常常得到一堆人看不懂的噪声。为了让图像像人能理解的对象，需要加限制、加先验，甚至通过 generator 只在真实图像流形附近搜索。

本节的概念地图

MAP

global explanation 的目标，是理解模型整体学到的概念或内部 pattern。它不问“这张图片哪一块让模型觉得是猫”，而问“对这个模型来说，什么样的输入最像猫”。这让我们可以检查 filter 是否学到笔画、边缘、纹理，或者类别输出是否依赖一堆人类看不懂的高频噪声。

本节有一条很重要的审美与真实性张力：我们希望生成出来的解释图像人能看懂，但模型真正的高分输入未必符合人类视觉习惯。很多方法通过额外限制让图像更像人类期待，这有用，但也意味着解释结果里混入了人的先验。

Global 解释：不看某张图，看模型的概念

GLOBAL

local explanation 有一个具体输入，例如一张猫图；global explanation 没有特定输入，而是分析模型参数本身。对 CNN 来说，我们可以问某个 convolutional filter 负责侦测什么 pattern；对分类器输出层来说，我们可以问某个类别的高分输入长什么样。

为什么这有价值？因为模型内部可能学到了人能理解的层次结构。低层 filter 可能侦测边缘、线条、角点；高层 filter 可能侦测纹理、部件或复杂形状。把这些 pattern 可视化出来，可以帮助我们确认模型是否在学习合理结构。

Local 问题

给定图片 X，为什么模型说它是猫？关注输入成分对当前决策的贡献。

Global 问题

不给定具体图片，对模型而言，猫、数字 7、某个 filter 的理想 pattern 是什么？

现在你应该能做什么：判断一个解释任务是在分析个案原因，还是在分析模型整体概念。

Filter 可视化：找最能激活它的图片

FILTER

在 CNN 中，一个 filter 对输入图片产生 feature map。若 feature map 上很多位置的值都很大，表示这张图片里有很多该 filter 喜欢的 pattern。global explanation 的做法是反过来：还没有图片时，我们创造一张图片 X，让某个 filter 的 feature map 总和越大越好。

X* = arg maxX ΣᵢΣⱼ Aᵢⱼ(X)
Aᵢⱼ(X) 是某个 filter 在 feature map 第 i,j 个位置的 activation。

注意这里优化的不是模型参数，而是输入图片本身。模型已经训练好并固定；我们把图片像素当成 unknown variable，用类似 gradient descent 的方法更新。因为目标是最大化 activation，所以更准确地说是 gradient ascent。

在 MNIST 手写数字模型中，如果把第二层卷积 filter 拿出来做这种可视化，可能会看到横线、斜线、竖线等笔画。这个结果合理，因为数字正是由笔画构成的。filter 可视化能把“卷积层学到局部 pattern”这件事变得可见。

现在你应该能做什么：用公式和文字说明：filter visualization 是固定 filter，优化输入图片，让 feature map activation 最大。

类别最大化：为什么直接问“数字 7 长什么样”会得到噪声

CLASS

如果不看中间 filter，而是看分类器最终输出，也可以做类似操作：找一张图片 X，让某个类别分数 y_c(X) 越大越好。例如找一张让模型最确信是数字 7 的图片。

X* = arg maxX y_c(X)
c 是目标类别，例如数字 7、猫、火山或蚂蚁。

但课程展示：如果只做这件事，结果常常是一堆噪声。原因并不神秘。我们在对像素空间直接优化，而像素空间里绝大多数点都不是自然图片。模型可能对某些人眼看不懂的高频 pattern 给出极高分，这和 adversarial attack 的直觉相通：机器的决策边界和人类视觉概念并不完全一致。

所以，“最大化类别分数”得到的图不一定是模型心中人类意义上的类别原型。它只表示：在模型当前数学函数里，这些像素能把目标输出推高。解释时必须把这两者分开。

关键警告

如果生成图像是一团噪声，不一定代表方法失败；它可能正暴露模型的高分区域和人类可理解图像之间有差距。

现在你应该能做什么：解释为什么直接最大化类别分数常得到噪声，并把它和 adversarial pattern 联系起来。

加限制：让解释图更像人能理解的对象

REGULARIZATION

如果我们希望生成图更像数字或自然物体，就需要在优化里加入限制。课程举了手写数字例子：数字通常由少量笔画构成，整张图里白色像素不会太多。于是可以加入一个 regularization term，惩罚太多白点、太强噪声或太不平滑的图像。

X* = arg maxX [ y_c(X) + R(X) ]
R(X) 表示对“像不像合理图片”的偏好；也可以写成最大化分数同时最小化惩罚。

限制能让图更符合人类期待，但也带来解释上的微妙问题：你看到的图不再只是模型自己最喜欢的输入，也包含了我们人为加入的图像先验。约束越强，图越漂亮，但也越可能把人的想象投射进去。

限制类型作用风险
稀疏让手写数字只保留少量笔画。可能压掉模型实际依赖的纹理。
平滑减少高频噪声，让图更自然。可能制造“看起来舒服”的假解释。
生成器先验限制搜索只产生像自然图像的样本。解释结果受 generator 学到的分布影响。

现在你应该能做什么：说明 regularization 为什么能让 global explanation 更可读，也为什么不能把可读图像当成纯粹真相。

Generator 先验：不直接找图片，而是找能生成图片的 z

GENERATOR

如果想生成更自然的图像，一个常见做法是先训练 image generator，例如 GAN 或 VAE。generator 输入低维向量 z，输出一张图像 X = G(z)。做 global explanation 时，我们不再直接优化每个像素，而是优化 z，让生成出来的图片经过分类器后目标类别分数最高。

z* = arg maxz y_c(G(z))
X* = G(z*)

这样做的好处是，搜索被限制在 generator 能产生的自然图像附近。若 generator 学过大量真实图像，它输出的蚂蚁、火山、鸟等会比直接像素优化更像人类熟悉的对象。坏处是，解释结果同时受分类器和 generator 影响：你看到的“模型心中的火山”，也可能是 generator 会画的火山。

局部替身模型：用简单模型模仿黑箱的一小块行为

LIME

P37 结尾还提到另一类解释思路：用一个比较简单、可分析的模型去模仿复杂黑箱模型。如果简单模型能模仿黑箱的行为，我们就分析简单模型，间接理解黑箱。

直接让 linear model 模仿整个 neural network 通常不现实，因为线性模型能力有限。但 LIME 的关键是 local：它不要求简单模型模仿整个黑箱，只要求在某个输入附近的小区域内模仿。局部范围足够小的时候，复杂函数常常可以被简单近似。

例如对一张图片，LIME 可以扰动不同区域，看黑箱输出怎么变，再训练一个局部线性模型去拟合这些扰动样本的输出。线性模型的权重就变成局部解释：哪些区域在这个样本附近最影响黑箱判断。

步骤做什么目的
选择样本固定要解释的输入。强调 local，而不是全局模仿。
产生扰动遮挡或改变局部成分。观察黑箱输出变化。
训练替身用简单模型拟合黑箱局部行为。获得可读权重或规则。
解释权重看哪些成分影响最大。把黑箱局部行为转成人能读的形式。

现在你应该能做什么：用一句话解释 LIME：在某个输入附近，用可解释的小模型近似黑箱模型。

最小代码：优化输入来最大化类别分数

CODE

下面是 PyTorch 风格的 activation maximization。注意优化器更新的是 x，不是模型参数。正则项用来鼓励图像更平滑、更稀疏或更接近人类期待。

正确示例：类别 activation maximizationPyTorch
def maximize_class(model, target_class, steps=300, lr=0.05, reg=1e-3):
    model.eval()
    x = torch.randn(1, 1, 28, 28, requires_grad=True)
    optimizer = torch.optim.Adam([x], lr=lr)
    for _ in range(steps):
        optimizer.zero_grad()
        logits = model(x)
        class_score = logits[0, target_class]
        penalty = reg * (x.abs().mean() + ((x[:, :, :, 1:] - x[:, :, :, :-1]) ** 2).mean())
        loss = -class_score + penalty
        loss.backward()
        optimizer.step()
        x.data.clamp_(0, 1)
    return x.detach()

loss = -class_score + penalty 是因为优化器默认做最小化。最小化负的类别分数，就等价于最大化类别分数；再加上 penalty，避免图片变成完全不可读的噪声。

错误示例：把模型参数也放进 optimizerOptimization bug
# 错误示例：这会改变模型，而不是解释固定模型。
optimizer = torch.optim.Adam(model.parameters(), lr=0.05)
loss.backward()
optimizer.step()

global explanation 要固定训练好的模型，优化输入或潜变量。如果你更新了模型参数，问题就变成重新训练模型，解释对象已经被你改掉了。

现在你应该能做什么：读懂 activation maximization 的最小代码，并能指出解释时优化的是输入，不是模型。

常见误区与修法

DIAGNOSIS

误区 1：把噪声图当失败

噪声图可能说明模型高分区域不符合人类视觉先验。修法是记录这一发现，再尝试加限制或换数据验证。

误区 2：把漂亮图当真相

漂亮图可能来自 regularization 或 generator prior。修法是说明加入了哪些先验，避免过度解读。

误区 3：优化了模型参数

解释应固定模型，只优化输入或 z。修法是检查 optimizer 参数列表。

误区 4：忽略类别分数与自然性的取舍

越自然不一定越高分，越高分不一定自然。修法是同时报告目标分数和正则设置。

误区 5：认为 generator 没有偏差

生成器也有训练分布和偏差。修法是把结果解释为分类器与生成器共同作用。

误区 6：让简单模型模仿整个黑箱

线性替身通常只能局部近似。修法是明确 local 范围和扰动方式。

现在你应该能做什么：评价 global explanation 结果时，追问目标函数、正则项、先验模型和优化对象。
