图片能加噪声，句子不能随便加噪声

图像对抗攻击常在像素上加很小的连续扰动，人眼看起来几乎一样，模型却预测错误。P38 问的是：自然语言处理里能不能做类似事情？文字输入由离散 token 组成，模型真正接收的是词、子词或字符序列，而不是可以随便加减的连续像素。

因此 NLP 对抗攻击的核心变成：如何改写句子，让人类看来任务答案不该变，但模型输出变坏。本节先讲 evasion attack 与 adversarial example 的定义，再用情感分类、dependency parsing、机器翻译三个例子建立直觉，最后拆解生成攻击样本的四个要素：goal、transformation、constraint、search method。

本节的概念地图

MAP

NLP adversarial attack 的目标，不是教人去攻击线上系统，而是理解模型有多脆弱，从而更谨慎地设计和防御。文本模型被公开后，真实用户可能每天用各种奇怪输入测试它；如果一个小改字、换词、拼写错误就能让模型崩掉，系统就不够 robust。

P38 讲到 constraint 前暂停，所以这一页的边界也放在 Part 1：重点是离散输入、evasion attack 的定义、攻击样本例子、目标设定和 transformation。constraint 与 search method 会作为完整流程的一部分先介绍，但细节留给下一节。

现在你应该能做什么

把 NLP 对抗攻击理解成“在离散文字空间中找保持人类语义、破坏模型预测的改写”。

为什么 NLP 不像图像：输入空间是离散的

DISCRETE

图像输入通常是连续数值，例如每个像素在 0 到 255 之间。对模型输出做 gradient 后，我们可以在像素上加一个小扰动。但文字不是这样。句子由 token 组成，token 是离散符号：你不能对单词 film 加上 0.07 的噪声。

你可能会想：模型内部不是会把 token 转成 word embedding 吗？能不能直接在 embedding 上加噪声？课程说明这不符合真实攻击设定。用户能控制的是输入文本，而不是模型内部 embedding lookup 后的向量。除非你在研究内部鲁棒性，否则攻击 NLP 模型时，通常要在 token、word 或 character 层面产生可读的改写。

图像攻击

输入是连续像素，小幅数值扰动仍是一张图片。gradient 可以直接指导像素变化。

NLP 攻击

输入是离散 token。扰动必须表现为换词、插入、删除、拼写变化等文本操作。

现在你应该能做什么：解释为什么“在 embedding 上加 noise”不能直接等同于攻击真实 NLP 输入。

Evasion Attack：推论时让模型预测变坏

EVASION

evasion attack 指的是在推论阶段给模型一个 adversarial example，让模型的 prediction 变坏。它不一定要修改训练资料，也不一定要植入后门；它关注的是模型上线后，攻击者输入一个经过扰动的样本，模型原本能答对，现在答错。

adversarial example 必须相对于某个 benign example 来理解。一个句子单独看，只是一个让模型判断错误的输入；当它和原本的正常句子相比，并且人类认为两者在任务标签上应该相同，而模型却从正确变错误时，我们才说它是这个正常样本的对抗样本。

benign x：人类标签 y，模型预测 y
adversarial x'：人类仍认为标签 y，但模型预测变成 y' ≠ y

这里的“人类看不出来差异”不是说字面完全一样，而是说对当前任务答案没有影响。在图像里，人类仍会认为是熊猫；在 dependency parsing 里，人类认为语法结构不该改变；在情感分类里，人类认为评论情绪不该从负面变正面。

现在你应该能做什么：用 benign example、perturbation、model prediction 三个词定义 adversarial example。

三个文本例子：从电影评论到机器翻译

EXAMPLES

第一个例子是 sentiment analysis。原句 skip the film and buy the soundtrack CD 是带讽刺意味的负面评论。只把 film 改成 films，人类仍会认为它是负面评论，但模型可能改判正面。这说明很小的词形变化也可能影响分类器。

第二个例子是 dependency parsing。把一个词换成语义上对句法结构不该产生本质影响的词，人类仍会给出相同或近似的 dependency tree，但模型输出可能完全不同。这里的“扰动小”要按任务定义：不是字面不变，而是目标结构不该变。

第三个例子是 machine translation。课程提到线上翻译系统可能把一句带有 sad 的话翻出相反情绪，且错误与特定地名组合有关。这提醒我们，NLP 模型上线后会被大量用户反复试探，奇怪案例并不罕见。

任务扰动人类预期模型问题
情感分类单复数、词形变化。情绪标签不变。负面变正面。
Dependency parsing替换某个词。句法结构不该大变。树结构大幅错误。
机器翻译特定词组或地名组合。语义和情绪应正确保留。翻译出相反含义。

现在你应该能做什么：判断“扰动是否小”时，不只看文字表面，而要看它是否改变当前任务的人类答案。

生成对抗样本的四要素

INGREDIENTS

课程把生成 NLP adversarial example 拆成四个 ingredient：goal、transformation、constraint、search method。这个框架很重要，因为它把攻击过程从“乱改句子”变成一个可分析的流程。

Goal

攻击想达到什么目标，例如只要分类错，或必须错到指定类别。

Transformation

如何产生候选扰动，例如换词、插词、删词、字符交换。

Constraint

扰动必须满足什么条件，例如语义相近、语法合理、改动数量有限。

Search method

如何在大量候选扰动中组合出成功样本，例如贪心、beam search、遗传算法。

流程上，攻击者先有任务、victim model 和 benign sample；接着 transformation 产生候选改写；constraint 过滤不合理改写；search method 尝试组合候选；最后把结果丢回模型，若达到 goal，就得到成功的 adversarial example。

现在你应该能做什么：看到一个文本攻击方法时，把它拆成目标、变换、限制、搜索四部分来分析。

Goal：只要错，还是错到指定类别

GOAL

最常见的目标之一是 untargeted classification：只要让模型预测错即可，不要求它错成哪一类。比如新闻标题原本是科技新闻，扰动后模型预测成体育或国际新闻都算成功。

另一种是 targeted classification：不仅要模型错，还要错到攻击者指定的类别。比如原本是科技新闻，攻击者指定要模型预测成体育新闻；如果扰动后预测成国际新闻，虽然错了，但不算达成 targeted goal。

不同任务的 goal 不同。dependency parsing 的目标可能是让输出树结构错误；机器翻译的目标可能是让某个关键词翻错、情绪翻反，或让输出质量下降。先定义 goal，才能判断攻击是否成功。

Goal 类型成功条件难度
Untargeted模型从正确变成任意错误。通常较容易。
Targeted模型输出攻击者指定的错误类别。通常更难。
结构/生成任务输出结构或生成文本满足特定失败条件。需要任务专属定义。

现在你应该能做什么：为一个 NLP 任务写出 untargeted 与 targeted 两种攻击目标。

Transformation：候选扰动从哪里来

TRANSFORM

transformation 是产生候选改写的方式。P38 重点讲了 word-level 与 character-level 的几类方法。word-level 可以做 substitution、insertion、deletion；character-level 可以交换相邻字符、替换字符、删除字符、插入字符。

同义词替换最直觉，但不可靠。词典里的 synonym 不一定在当前语境中等价，甚至可能引入反义或语气变化。embedding 邻近替换也类似：传统 GloVe embedding 会把出现在相似上下文的词放近，但相似上下文不等于同义，例如 east、west、north 都可能相近，却不能随便互换。

counter-fitted embedding 的思路，是用语言学限制把同义词拉近、反义词拉远，让邻近词更适合作为替换候选。BERT 类 masked language model 则可以根据上下文预测候选词，但直接 mask 原词可能生成语境合理但语义相反的词；不 mask 而做 reconstruction，候选通常更接近原词。

方法产生方式主要风险
词典同义词从 WordNet 等资源拿 synonym。上下文不合适，语义偏移。
Embedding 邻近找向量空间附近的词。相似上下文不等于同义。
BERT mask遮住词，让语言模型补词。补出语境合理但语义相反的词。
词形变化改时态、单复数、派生形。可能破坏语法或改变词性。
字符扰动swap、substitute、delete、insert。人类能读，模型可能 OOV；也可能太不自然。

现在你应该能做什么：解释为什么候选替换不能只靠“看起来相关”，还必须用 constraint 检查语义和语法。

Gradient 替换：用一阶近似估计换哪个词最能增大 loss

GRADIENT

如果攻击者知道模型结构和参数，就可以做 white-box attack，用 gradient 指导替换。虽然输入 token 是离散的，但 token 会映射到 embedding。我们可以计算 loss 对某个词 embedding 的梯度，再估计把当前词换成候选词后 loss 会增加多少。

ΔL ≈ ∇eL · (e_candidate - e_original)
这是 loss 在原 embedding 附近的一阶近似；值越大，表示替换后越可能让 loss 增加。

这不是直接在 embedding 上加连续噪声，而是用 gradient 给离散候选词排序。真正输出仍然是某个合法 token。课程中也提醒：如果某个词的所有候选替换都不能让 loss 增大，可以不换这个词，转而尝试句子里的其他词。

gradient 方法依赖 white-box 条件；如果只能查询黑箱模型，就需要其他策略，或者先做 model extraction 训练一个 proxy model，再在 proxy 上生成攻击样本。这部分只是建立概念，后续课程会继续展开。

现在你应该能做什么：说明 gradient 在 NLP 攻击里不是直接修改 token，而是给候选替换排序。

最小代码：生成候选替换并过滤

CODE

下面的伪代码展示四要素的骨架：先定义 goal，再由 transformation 产生候选，constraint 过滤，search method 选择并测试。P38 重点看前两步，后两步先保留结构。

正确示例：文本 evasion attack 流程骨架Python-like
def attack(model, sentence, true_label):
    candidates = []
    for position, token in enumerate(sentence.tokens):
        for new_token in transform(token, context=sentence):
            proposal = sentence.replace(position, new_token)
            if satisfies_constraints(sentence, proposal):
                candidates.append(proposal)

    for proposal in search(candidates):
        prediction = model.predict(proposal)
        if prediction != true_label:
            return proposal
    return None

真正实现会复杂得多：候选可以组合多个位置，constraint 可能要用语义相似度、语言模型分数、词性检查，search 也可能用贪心或 beam search。但这个骨架足以帮你定位每个组件的角色。

错误示例：只要模型错就接受Missing constraints
# 错误示例：完全不检查语义，可能把负面评论改成真正的正面评论。
proposal = "this movie is excellent"
if model.predict(proposal) != true_label:
    return proposal

这不一定是合格 adversarial example，因为它可能已经改变了人类标签。NLP 攻击最难的地方之一，就是同时让模型错、又让人类任务答案保持不变。

常见误区与修法

DIAGNOSIS

误区 1：把 embedding 噪声当真实文本攻击

用户不能直接改模型内部 embedding。修法是区分内部鲁棒性实验和真实 token 级攻击。

误区 2：只要字面变了就不是对抗样本

NLP 中字面变化不可避免，关键是人类任务标签是否保持。修法是按任务定义扰动可接受性。

误区 3：同义词一定保持语义

词典同义词脱离上下文可能不等价。修法是加入语义、词性、语言流畅度限制。

误区 4：BERT mask 候选一定合理

mask 预测的是上下文合理词，不保证原句语义不变。修法是结合 reconstruction 或语义过滤。

误区 5：字符扰动太小所以总是合法

拼写错误可能影响可读性或任务标签，也可能过于明显。修法是限制编辑距离和可读性。

误区 6：untargeted 与 targeted 混淆

任意错和错到指定类是不同目标。修法是在实验报告中明确 goal。

现在你应该能做什么：审查一个 NLP 攻击样本时，同时问：目标是什么、怎么改的、语义是否保持、模型为什么会错。
