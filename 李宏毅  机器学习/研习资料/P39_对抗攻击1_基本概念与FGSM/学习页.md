# 本节的概念地图 {#goals}
<!-- tag: MAP -->

对抗攻击把机器学习从“自然测试集表现好不好”推进到“有人故意找漏洞时会不会崩”。图像分类器在普通噪声下可能仍然认出猫，但某些专门优化出来的小扰动，却能让它以极高信心输出完全错误的类别。

这一节的数学主线是一个受限制优化问题：找一张新图片 `x`，让模型输出满足攻击目标，同时 `x` 和原图 `x0` 的距离小于阈值 `epsilon`。求解方法和训练模型很像，只是优化对象变成输入。

# 为什么要研究攻击：真实应用里有人会故意骗你 {#why}
<!-- tag: WHY -->

模型如果只在善意数据上准确，仍然可能无法上线。邮件过滤器会面对想绕过检测的垃圾邮件发送者；图像识别系统可能面对故意贴贴纸、加扰动、改变外观的人；安全系统更是天然处在对抗环境中。模型必须能应付来自人类的恶意。

课程用一个直观比喻：强不等于安全，因为人类可能不按常规方式正面对抗。对应到神经网络，普通测试准确率像是在常规条件下比强弱；对抗攻击则是有人专门研究你的弱点，再构造输入击穿你。

**现在你应该能做什么：**说明为什么 adversarial robustness 是部署模型时的安全需求，而不是学术玩具。

# Untargeted 与 Targeted：只要不是猫，还是必须变海星 {#types}
<!-- tag: TYPES -->

benign image 是没有被攻击的原图，attack image 是加了扰动后的图片。对于一张原本被模型判为猫的图片，untargeted attack 的目标很宽：只要模型输出不是猫就成功。targeted attack 更难：攻击者指定目标类别，例如必须让模型输出海星，才算成功。

targeted attack 通常更难，因为它不仅要降低正确类别分数，还要把特定错误类别分数推高。课程中猫图可以被变成海星、键盘等，说明现代网络的决策边界可能对精心构造的微小扰动非常敏感。

| 类型 | 成功条件 | 例子 |
| --- | --- | --- |
| Untargeted | 输出不是原正确类别。 | 猫变成任意非猫类别。 |
| Targeted | 输出为攻击者指定类别。 | 猫必须变成海星。 |

**现在你应该能做什么：**给定原类别和攻击后输出，判断攻击是 untargeted 成功、targeted 成功还是失败。

# 攻击目标函数：让正确答案远，或让目标答案近 {#loss}
<!-- tag: LOSS -->

设模型是固定函数 `f`，输入图片是 `x`，输出分布是 `y = f(x)`。对于 untargeted attack，我们希望输出和正确标签 `y_hat` 差距越大越好。如果用 cross entropy 衡量差距，就可以最大化 cross entropy，或者等价地最小化负的 cross entropy。

$$
L_untargeted(x) = - CE(f(x), y_hat)
最小化这个 L，就等于让 CE(f(x), y_hat) 变大，让模型远离正确答案。
$$

targeted attack 还要靠近指定目标 `y_target`。一种写法是同时让正确类远、目标类近：

$$
L_targeted(x) = - CE(f(x), y_hat) + CE(f(x), y_target)
第一项鼓励远离正确类，第二项鼓励靠近目标类。
$$

不同文献的符号和正负号可能不同，关键是看优化方向：到底是在最大化正确类 loss，还是最小化目标类 loss。写代码时一定要确认 loss 的方向，否则可能在帮模型变得更正确。

**现在你应该能做什么：**根据 untargeted/targeted 目标，解释 loss 中每一项想推高或压低什么。

# 距离限制：扰动要小到人不容易察觉 {#distance}
<!-- tag: DISTANCE -->

只优化攻击目标不够，因为我们希望 attack image 和 benign image 看起来几乎一样。于是加入限制：

$$
d(x, x0) ≤ epsilon
x0 是原图，x 是攻击图；epsilon 是允许扰动大小。
$$

常见距离包括 L2 norm 和 L∞ norm。L2 看整体平方和，许多像素都改一点时可能不大；L∞ 只看所有像素中最大改变量。课程用四像素例子说明：如果每个像素都小改，人可能看不出来；如果只有一个像素改很大，人可能更容易察觉。此时 L∞ 更能限制单个像素的最大变化。

| 距离 | 看什么 | 直觉 |
| --- | --- | --- |
| L2 | 整体扰动能量。 | 关心总变化量，适合整体大小比较。 |
| L∞ | 单一维度最大变化。 | 限制每个像素最多改多少，图像攻击常用。 |
| 领域距离 | 按人类感知定义。 | 语音等任务可能需要听觉模型，而不只是 Lp norm。 |

**现在你应该能做什么：**解释为什么图像作业常用 L∞，以及为什么距离定义需要 domain knowledge。

# 优化输入：像训练模型一样训练一张图 {#optimize}
<!-- tag: OPTIMIZE -->

如果暂时拿掉距离限制，攻击优化和训练模型非常像。训练模型时，我们固定输入，更新 network parameters；攻击时，我们固定 network parameters，更新输入图片 `x`。初始化通常设为原图 `x0`，因为我们希望最终图片和原图接近。

每一步计算 loss 对输入 `x` 的 gradient，然后更新 `x`。如果更新后超出限制集合，就把它拉回允许范围。这种“更新后投影回去”的思想，就是 projected gradient 的基本直觉。

$$
x ← x - η ∇xL(x)
η 是步长；对 untargeted loss 的具体正负号要按定义确认。
$$

**现在你应该能做什么：**说清楚攻击中的 gradient 是 loss 对输入的 gradient，而不是 loss 对模型参数的 gradient。

# FGSM：一拳超人的一步攻击 {#fgsm}
<!-- tag: FGSM -->

FGSM 是 Fast Gradient Sign Method。它不做很多次迭代，而是只更新一次。做法是取 loss 对输入的 gradient，再只保留每个维度的正负号。这样每个像素都沿着最能改变 loss 的方向移动 `epsilon`。

$$
x_adv = x0 + epsilon · sign(∇x loss)
不同目标和 loss 定义会影响加号/减号；重点是使用 gradient sign，并让扰动落在 L∞ 边界内。
$$

为什么取 sign？在 L∞ 限制下，每个维度最多能改 `epsilon`，那最直接的做法就是每个维度都走到允许范围的边界，方向由 gradient 正负决定。课程中说它像“一拳超人”：只攻击一次，很多时候就能让模型错。

iterative FGSM 则把一步改成多步。每一步走小一点，走完后投影回 `x0` 周围的 L∞ 方框。多步通常更强，但也更需要注意不要超出扰动限制。

## box info
### FGSM
一步到位，速度快，适合作为 baseline。

## box warn
### Iterative FGSM
多次小步更新加投影，攻击更强，但计算成本更高。

**现在你应该能做什么：**解释 FGSM 的公式，以及为什么 iterative 版本通常比一步版本更强。

# 最小代码：FGSM 与迭代版本 {#code}
<!-- tag: CODE -->

下面是 PyTorch 风格的 untargeted FGSM。为了让模型犯错，我们最大化正确标签的 cross entropy，所以代码里直接沿着 gradient sign 增大 loss。具体符号要和你的 loss 定义保持一致。

正确示例：untargeted FGSMPyTorch

```text
def fgsm(model, x0, y, epsilon):
 x = x0.clone().detach().requires_grad_(True)
 logits = model(x)
 loss = torch.nn.functional.cross_entropy(logits, y)
 model.zero_grad()
 loss.backward()
 x_adv = x + epsilon * x.grad.sign()
 x_adv = torch.clamp(x_adv, 0, 1)
 return x_adv.detach()
```

正确示例：iterative FGSM with projectionPyTorch

```text
def iter_fgsm(model, x0, y, epsilon, alpha, steps):
 x_adv = x0.clone().detach()
 for _ in range(steps):
 x_adv.requires_grad_(True)
 loss = torch.nn.functional.cross_entropy(model(x_adv), y)
 model.zero_grad()
 loss.backward()
 x_adv = x_adv + alpha * x_adv.grad.sign()
 delta = torch.clamp(x_adv - x0, min=-epsilon, max=epsilon)
 x_adv = torch.clamp(x0 + delta, 0, 1).detach()
 return x_adv
```

错误示例：更新模型参数Wrong target

```text
# 错误示例：攻击时不该训练 model.parameters()。
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
loss.backward()
optimizer.step()
```

攻击时模型参数固定。你要得到的是一张会骗过当前模型的输入，不是把模型训练坏或训练好。

**现在你应该能做什么：**读懂 FGSM 代码，并能指出投影步骤如何保证扰动不超过 `epsilon`。

# 常见误区与修法 {#mistakes}
<!-- tag: DIAGNOSIS -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：普通噪声有效，所以攻击只是加噪声
普通大噪声让模型错不奇怪；对抗扰动是专门优化的小扰动。修法是比较随机噪声和 gradient-based 扰动。

## box redsoft
### 误区 2：targeted 和 untargeted 混用
只要错和错到指定类别是不同目标。修法是先写清成功条件。

## box redsoft
### 误区 3：忘记距离限制
没有限制时可以把图改成任何东西。修法是明确 L2/L∞ 或感知距离阈值。

## box redsoft
### 误区 4：loss 符号写反
untargeted 要让正确类 loss 变大，targeted 要让目标类 loss 变小。修法是检查优化方向。

## box redsoft
### 误区 5：迭代后不投影
多步更新容易跑出 epsilon 限制。修法是每步 clamp delta。

## box redsoft
### 误区 6：所有领域都用同一种 norm
语音、人脸、文本的感知差异不同。修法是根据 domain knowledge 设计距离。

**现在你应该能做什么：**检查一个攻击实现时，逐项确认目标、loss 方向、距离限制、投影和模型参数是否固定。
