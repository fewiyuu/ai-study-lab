# 本节的概念地图 {#goals}
<!-- tag: MAP -->

如果攻击必须知道参数，防御似乎很简单：不公开模型就好。但黑箱攻击说明事情没那么轻松。只要攻击者能获得相似训练资料，或能反复查询线上服务，就可能训练出 proxy model；在 proxy 上优化出的扰动，可能迁移到真实模型。

防御也不是一劳永逸。模糊化、压缩等被动防御在攻击者不知道时可能有效；一旦攻击者知道防御流程，就能把它放进攻击优化里一起绕过。主动防御通过 adversarial training 把攻击样本加入训练，但计算成本高，也不一定挡得住新的攻击算法。

# 黑箱攻击：训练一个替身来攻击真正目标 {#blackbox}
<!-- tag: BLACK BOX -->

white-box attack 知道模型参数，可以直接算 gradient。black-box attack 不知道参数，只能看到模型输入输出，甚至可能只知道任务和训练资料来源。课程中的作业就是黑箱攻击：你不知道评测模型参数，只能在自己机器上攻击一个 proxy network，再把攻击图片交给评测系统。

proxy network 的来源有两种。第一，如果知道目标模型用什么训练资料，可以用同一资料训练自己的模型。第二，如果不知道训练资料，也可以大量查询线上服务，把输入和输出配对，用这些 pseudo-label 训练一个模仿模型。这类似 model extraction。

黑箱攻击中 untargeted 比 targeted 容易。骗目标模型“不要认成原类别”相对容易；要求它“必须认成指定类别”更难，因为不同模型的错误方向未必完全一致。

**现在你应该能做什么：**解释为什么隐藏参数只能提高攻击难度，不能直接保证安全。

# Transferability：骗过 A 模型的方向，也可能骗过 B 模型 {#transfer}
<!-- tag: TRANSFER -->

为什么 proxy 上的攻击能迁移？课程说这仍有研究空间，但一个常见观察是：不同模型在同一数据集上学到的决策边界可能有相似结构。某些方向会让图片快速离开“被认成小丑鱼”的区域，而这个方向对多个模型都有效。

有观点认为 adversarial examples 不只是模型 bug，也可能是 data feature：有限资料里的某些非鲁棒特征被不同模型共同学到。于是攻击样本在模型之间迁移，因为它们学到了相近的脆弱线索。

## box info
### 单 proxy
在一个替身模型上攻击，迁移到目标模型有一定概率成功。

## box warn
### Ensemble proxy
找一个能同时骗过多个替身模型的扰动，更可能骗过未知目标。

**现在你应该能做什么：**用“不同模型学到相似脆弱方向”解释黑箱攻击迁移性。

# 攻击变体：从一个像素到一个通用扰动 {#variants}
<!-- tag: VARIANTS -->

one-pixel attack 只改变图片中的一个像素，也能让某些模型出错。不过课程提醒，这类攻击有局限，错误往往不如“猫变海星”那么荒谬，攻击能力不一定非常强。

universal adversarial attack 更惊人：找一个通用 attack signal，加到很多不同图片上都能让模型出错。若这种信号能贴到摄像头或输入流程上，就不需要为每张图片重新计算扰动。

攻击也不局限于图像。语音合成检测器可以被人耳听不出的噪声骗过；问答系统也可能在文章末尾加一句无关内容后，所有问题都输出同一个错误答案。对抗脆弱性是跨模态问题。

| 攻击 | 核心想法 | 启示 |
| --- | --- | --- |
| One-pixel | 只改一个像素。 | 模型可能对局部变化敏感，但能力有限。 |
| Universal | 一个扰动攻击很多图片。 | 脆弱方向可能具有全局性。 |
| Audio/Text | 对语音检测、问答等任务构造扰动。 | 不只是影像模型会被攻击。 |

**现在你应该能做什么：**区分 per-sample attack 与 universal attack，并说明后者为什么更危险。

# 物理世界攻击：眼镜、路牌和真实约束 {#physical}
<!-- tag: PHYSICAL -->

攻击不一定只发生在电脑里的数字图片上。有人可以制作特殊眼镜，让人脸识别系统把佩戴者识别成另一个人；也有人在路牌上贴贴纸或轻微改变数字形状，让交通标志识别系统误判。

物理世界攻击比数字攻击多了很多约束。首先，摄像头从不同角度、距离、光照下拍摄，攻击必须在这些变化下仍有效。其次，摄像头分辨率有限，太小或太高频的扰动可能拍不出来。再次，打印颜色和屏幕颜色不同，设计出来的图案必须能真实制作。

视角变化从不同角度看仍要攻击成功。
传感器限制摄像头分辨率、模糊、压缩会改变扰动。
可制造性颜色、材质、打印误差都要考虑。

**现在你应该能做什么：**说明数字世界攻击迁移到物理世界时，必须额外考虑哪些真实约束。

# 被动防御：在模型前面加盾牌 {#passive}
<!-- tag: PASSIVE -->

被动防御不改模型参数，而是在输入进入模型前做处理。比如轻微模糊化、JPEG 压缩再解压、用 generator 重建图片。直觉是：攻击扰动很特殊，经过这些处理后会失去威力；正常图片受影响较小。

被动防御的问题是，一旦攻击者知道防御流程，就可以把防御当作网络第一层一起纳入优化，重新找到能穿过防御的扰动。randomization 试图缓解这个问题：输入图片随机缩放、随机 padding、随机位置放置，让攻击者更难精确模拟。但如果随机分布被知道，攻击者仍可能用期望上的攻击或 universal attack 绕过。

| 防御 | 作用 | 弱点 |
| --- | --- | --- |
| Blur | 削弱高频攻击信号。 | 过度模糊会伤害正常准确率；被知道后可绕过。 |
| JPEG | 压缩损失可能破坏扰动。 | 同样可被纳入攻击流程。 |
| Generator | 重建自然图像，丢掉小扰动。 | 依赖生成器质量，可能改变正常细节。 |
| Randomization | 让攻击者难以确定预处理。 | 分布已知时仍可被适配攻击。 |

**现在你应该能做什么：**解释为什么被动防御在“攻击者不知道”时有效，但一旦公开就可能变弱。

# 主动防御：Adversarial Training {#active}
<!-- tag: ACTIVE -->

主动防御在训练阶段就让模型见过攻击样本。流程是：先训练模型；用当前模型生成 adversarial examples；把这些攻击后的图片标回原正确标签；和原训练资料合并，再重新训练。这个过程可以反复进行，相当于不断找漏洞、补漏洞。

adversarial training 也可以看成 data augmentation。它生成更多“困难样本”，迫使模型在扰动附近也保持正确，从而提升鲁棒性，甚至有时能帮助缓解过拟合。

它的问题也明显。第一，计算成本高：每轮训练都要为大量样本生成攻击图。第二，它不一定挡得住训练时没考虑过的新攻击算法。如果训练阶段只用 A/B/C/D/E 方法，真实攻击者发明 F 方法，模型仍可能被攻破。

优点直接训练模型抵抗已知攻击，是经典强基线。
成本生成攻击样本很耗时，大数据集上尤其贵。
风险对未知新攻击不保证有效。

**现在你应该能做什么：**把 adversarial training 解释成“训练时主动暴露漏洞并补上”，同时说出它的计算与泛化限制。

# 最小代码：Proxy 与 Adversarial Training 骨架 {#code}
<!-- tag: CODE -->

下面的伪代码展示两件事：黑箱攻击中先训练 proxy，再在 proxy 上生成攻击；主动防御中把攻击样本加入训练。

正确示例：proxy black-box attack 骨架Python-like

```text
def black_box_attack(target_api, public_images):
 pseudo_labels = [target_api.predict(x) for x in public_images]
 proxy = train_model(public_images, pseudo_labels)
 x_adv = fgsm(proxy, x0, y0, epsilon)
 return x_adv
```

正确示例：adversarial training 骨架Python-like

```text
for epoch in range(num_epochs):
 for x, y in loader:
 x_adv = make_attack(model, x, y, epsilon)
 logits_clean = model(x)
 logits_adv = model(x_adv)
 loss = CE(logits_clean, y) + CE(logits_adv, y)
 update(model, loss)
```

错误示例：只防一种已知攻击就宣称安全Evaluation bug

```text
# 错误示例：只测 FGSM，就说模型已经 robust。
robust = accuracy_on_fgsm > 0.9
```

鲁棒性评估需要多种攻击、不同强度、白箱/黑箱、是否知道防御流程等设置。只过一种攻击，很可能只是对那种攻击过拟合。

**现在你应该能做什么：**读懂 proxy 攻击和 adversarial training 的流程骨架，并知道鲁棒性不能只用单一攻击证明。

# 常见误区与修法 {#mistakes}
<!-- tag: DIAGNOSIS -->
<!-- layout: grid2 -->

## box redsoft
### 误区 1：不公开参数就安全
proxy 和查询模仿可以支持黑箱攻击。修法是评估 black-box transfer。

## box redsoft
### 误区 2：只关注 targeted attack
黑箱 targeted 较难，但 untargeted 仍可能很容易。修法是分别报告两者。

## box redsoft
### 误区 3：被动防御公开后仍强
攻击者可把预处理纳入优化。修法是用 adaptive attack 测试。

## box redsoft
### 误区 4：随机化等于无法攻击
若随机分布已知，仍可优化期望或找通用扰动。修法是评估分布已知场景。

## box redsoft
### 误区 5：adversarial training 一劳永逸
它可能挡不住新算法。修法是持续用更强攻击回归测试。

## box redsoft
### 误区 6：忽略物理世界约束
数字扰动不一定能打印、拍摄、跨角度生效。修法是加入物理约束验证。

**现在你应该能做什么：**评价一个防御方案时，主动问攻击者知道什么、能查询什么、是否能适配你的防御。
