文本攻击最难的，不是改字，而是改得合理

P38 讲了 NLP 对抗样本的 goal 和 transformation。P41 续上剩下两件事：constraint 和 search method。constraint 决定一个改写是否仍然合理，例如字面改动不能太多、文法要对、语义要接近。search method 决定如何在大量候选改写中找出既满足限制、又能让模型失败的句子。

这一节还整合几个经典方法：TextFooler、BERT-Attack、universal adversarial trigger，并讨论防御。最关键的反思是：文本攻击的评估很依赖语义相似度模型、文法检查器等外部工具；如果这些工具不可靠，我们可能误以为攻击成功，其实只是生成了不合理句子。

本节的概念地图

MAP

一个合格 NLP adversarial example 不是“模型错了就好”。它必须满足任务相关限制：如果任务要求语义保持，改写后语义不能变；如果文法重要，句子不能明显不通；如果要求小改动，overlap 或 edit distance 要受控。

search method 则处理组合爆炸。一个句子有很多位置，每个位置有很多候选词，全部穷举不可行。于是常用 greedy、word importance ranking、beam search、genetic algorithm 等近似方法。

Constraint：什么样的改写才算合理

CONSTRAINT

constraint 和任务 goal 强相关。情感分类通常要求语义和情绪保持；句法任务要求结构不变；某些任务还要求文法正确、改动比例低、词性一致。没有 constraint，攻击可以把负面评论直接改成正面句子，那模型判断变化就不再说明脆弱性。

课程中 constraint 大致分三类：overlap，检查改动幅度；grammar，检查文法和词性；semantic similarity，检查语义相似。它们都只是近似，不能完全替代人类判断。

Overlap

改了多少字符或词，改动比例是否过大。

Grammar

词性是否一致，句子是否符合文法。

Semantics

原句和改写后句子语义是否足够接近。

现在你应该能做什么：看到一个攻击样本时，先问它满足哪些 constraint，而不是只看模型是否被攻破。

Overlap：改动幅度不能太大

OVERLAP

character-level 扰动常用 Levenshtein distance，也就是 edit distance。它衡量从一个字符串变到另一个字符串需要多少次插入、删除、替换等操作。例如 kitten 到另一个字符串的距离可以用递归或动态规划计算。

word-level 攻击常用更改词比例，例如总共 5 个词改了 1 个，就有 20% 的词被扰动。这个比例越低，通常越符合“小改动”的要求。但它也不保证语义保持：改掉一个关键否定词，比例很低，语义却可能完全反转。

指标适合场景局限
Levenshtein distance字符级 swap、delete、insert、substitute。只看字面编辑，不懂语义。
修改词比例word-level substitution。少量关键改动也可能改变标签。

现在你应该能做什么：解释为什么 overlap 是必要但不充分的限制。

Grammar：词性和文法要撑得住

GRAMMAR

POS consistency 检查替换前后词性是否一致。例如 recommend 是动词，替换成 advocate 仍是动词，通常比较合理；替换成 recommendation 变成名词，句子 I highly recommendation it 明显不通。

词性一致还可以更严格：都是动词是否还要保持时态一致？这取决于任务和 constraint 设计。若时态变化会造成文法错误，就应更严格。

另一类方法是使用 LanguageTool 之类文法检查器，或用 language model 的 perplexity 衡量句子自然度。perplexity 越高通常表示句子越不自然，但不同工具、阈值和语言模型都会影响结果。

现在你应该能做什么：判断一个替换是否违反文法，并能说出 POS 检查和 perplexity 检查各自的作用。

Semantic Similarity：最关键，也最危险的限制

SEMANTICS

许多任务要求改写后语义保持。简单做法是比较被替换词在 embedding space 的距离，例如 cosine similarity；更常见的是用 Universal Sentence Encoder 等 sentence encoder，把原句和改写句编码成向量，再计算句向量相似度。

难点在于阈值。阈值太低，会放过语义明显变化的句子；阈值太高，会过滤掉许多合理改写，攻击成功率下降。更深的问题是：sentence encoder 本身也可能被攻击或误判。若我们用一个不可靠的语义模型做 constraint，最后可能只是骗过了 constraint model。

关键警告

“语义相似度高”是模型判断，不是人类真理。报告 NLP 攻击时，最好结合人工评估或更严格的多指标检查。

现在你应该能做什么：解释为什么 semantic similarity 是 NLP 攻击评估中最重要、也最容易出问题的一环。

Search Method：在候选组合里找成功路径

SEARCH

最简单的 greedy search 会枚举候选 perturbation，测试每个候选让模型输出变化多少，选择最强的那个替换。如果攻击还没成功，就继续选择下一个候选。它直接、易实现，但可能陷入局部最优。

更常用的是 greedy search with word importance ranking。先估计每个词对模型判断的重要性，例如删掉或替换该词后目标类别概率变化多少；再优先攻击最重要的词。这样搜索空间更小，攻击效率更高。

genetic algorithm 则把一句改写看成一个个体，通过选择、交叉、突变逐步进化出成功样本。它搜索能力强，但通常很慢。beam search 也有人使用，可以保留多个候选路径，但 beam 越大计算成本越高。

方法直觉优缺点
Greedy每步选当前最强候选。快，但可能局部最优。
Importance ranking先攻击最重要的词。常用，效率好。
Beam search保留多个候选路径。更全面但更贵。
Genetic algorithm用进化方式搜索句子。慢，但能探索复杂组合。

现在你应该能做什么：解释为什么 NLP 攻击需要 search method，而不能只生成候选就结束。

经典方法：TextFooler 与 BERT-Attack

METHODS

TextFooler 的典型流程是：先计算词重要性；对重要词产生候选替换；用 POS、语义相似度等 constraint 过滤；再用 greedy search 逐步替换，直到模型预测改变。它代表了“word-level substitution + importance ranking + constraints”的经典路线。

BERT-Attack 则使用 BERT 产生候选替换。它会考虑上下文，让候选更自然；但仍需要 constraint 检查，避免语言模型给出语境合理但语义偏移的词。课程强调，BERT 不是魔法，候选生成和语义保持仍是两回事。

TextFooler

重点在词重要性排序、同义候选、POS 与句向量相似度过滤。

BERT-Attack

重点在上下文化候选生成，但仍需过滤和搜索。

现在你应该能做什么：把 TextFooler 和 BERT-Attack 都拆回四要素框架中说明。

Universal Trigger：一句固定触发词，攻击很多输入

TRIGGER

universal adversarial trigger 是一种固定字符串。把它加到很多不同输入前面或后面，都能让模型朝攻击者希望的方向输出。它像图像中的 universal perturbation，只是作用在文本上。

训练 trigger 时，可以初始化一串 token，然后把带 trigger 的样本丢进模型，计算目标 loss，再对 trigger token 的 embedding 做梯度，选择能让目标 loss 下降最多的替换 token。反复更新后，就得到一个通用触发字符串。

这个想法和 prompt/prefix tuning 有相似结构：固定模型，通过学习一个 prefix 改变模型行为。不同点是，prefix tuning 用于让模型完成好任务；universal trigger 则用于诱导不希望的输出。

现在你应该能做什么：解释 universal trigger 为什么危险，以及它和 prompt tuning 的结构相似处。

防御与评估：攻击有用，但评估要谨慎

DEFENSE

NLP defense 常见路线包括 adversarial training、data augmentation、拼写修正、输入检测、模型鲁棒性增强等。核心思想和图像类似：用攻击样本暴露 failure mode，再把这些困难样本用于训练或测试。

但 NLP 评估更依赖人类语义。一个攻击若把 fake news 改成真正不像原句的内容，即使骗过模型，也不能说明模型脆弱。课程问答中也强调：攻击样本最重要的价值之一，是帮助模型部署者发现 failure mode，例如模型依赖罕见词、拼写、伪相关或语义相似度工具的漏洞。

所以好的评估应该同时报告 attack success rate、修改比例、语义相似度、文法指标、人工评估结果，以及在防御后的 clean accuracy 和 robust accuracy。只报告攻击成功率是不够的。

指标看什么为什么重要
Attack success rate模型被攻破比例。衡量攻击强度。
Modification rate改动多少词或字符。衡量扰动大小。
Semantic similarity语义是否保持。避免伪攻击。
Human evaluation人类是否认可改写。弥补自动指标缺陷。
Clean / robust accuracy正常与攻击输入表现。评估防御是否牺牲正常性能。

现在你应该能做什么：设计 NLP 攻击评估时，不只看成功率，还会检查样本质量和防御后的准确率权衡。

下面是一个简化版 TextFooler 风格流程：先给词排序，再逐个尝试候选替换，过滤不合格样本，最后选择最能降低正确类概率的候选。

正确示例：importance ranking + constraintsPython-like
def attack_sentence(model, sentence, label):
    ranking = rank_words_by_importance(model, sentence, label)
    current = sentence
    for pos in ranking:
        candidates = generate_substitutes(current, pos)
        valid = []
        for cand in candidates:
            if same_pos(current, cand, pos) and semantic_sim(sentence, cand) > 0.82:
                valid.append(cand)
        current = pick_best_by_model_score(model, valid, label)
        if model.predict(current) != label:
            return current
    return None
