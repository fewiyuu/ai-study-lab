文本攻击也可以训练一个会改写的模型

P41 的路线是：先定义 transformation 和 constraint，再用 search method 在候选句子里寻找成功样本。P42 换一个角度：不再每次都手工搜索候选，而是训练一个 generator，让它读入正常句子，直接产出对抗样本。

这个方向的核心矛盾很清楚：输出句子要像原句，才能算合理扰动；输出又要让原 classifier 失败，才算攻击；训练时还要面对 token 采样不可微的问题。后半节则转向防御：训练期用 adversarial training 或 ASCC 增强鲁棒性，推理期用 DISP、FGWS 先侦测再修复输入。

从搜索式攻击到生成式攻击

SHIFT

搜索式攻击的典型流程是：为每个位置产生候选词，用 POS、语义相似度、改动比例等 constraint 过滤，再让模型打分，逐步选择最能让预测出错的替换。它的好处是直观，缺点是每个句子都要重新搜索，候选生成和过滤也很依赖外部工具。

生成式攻击的想法更像“训练一个改写器”。给 generator 一个正常输入 x，它输出 x*。这个 x* 不是任意乱写，而是要尽量保留原句 token 和语义，同时让原本的 classifier 输出变差。训练好之后，generator 可以快速把新句子改写成攻击样本。

这种做法和 GAN 有一点相似：一边有产生样本的模型，另一边有判断样本的分类器。但它不是经典 GAN，因为分类器不是判断“真样本还是生成样本”，而是判断情感、主题、标签等任务类别。也就是说，generator 不是为了骗过真假判别器，而是为了让任务 classifier 做错。

路线核心动作主要代价
搜索式攻击每个输入现找候选、过滤、试探模型。慢，依赖候选表和约束模型。
生成式攻击训练 generator，把输入直接改写成攻击样本。训练复杂，要处理离散采样和样本质量。

现在你应该能做什么：说明为什么生成式攻击不是“换一种写法”，而是把攻击过程本身变成一个可训练模型。

Autoencoder：输入一句话，输出一条扰动后的句子

GENERATOR

autoencoder 原本常用于“编码后再重建输入”。在这里，它被改造成文本攻击 generator：读入正常句子，输出一个看起来仍像原句的新句子。因为它带有重建目标，所以不会完全抛开原句乱生成；因为它还带有攻击目标，所以不会只做复制。

以电影评论情感分类为例，原句可能是 I really enjoyed this movie.，真实标签是正面。generator 可能输出 I truly enjoyed this movie. 这种语义接近的句子。如果目标 classifier 被这个替换弄错，就得到一个 adversarial example。若输出变成 I hated this movie.，模型错了也不算合格攻击，因为人类标签已经变了。

1. 输入

正常文本 x，带有任务标签。

2. 生成

autoencoder 输出 token distribution，再形成 x*。

3. 攻击

原 classifier 在 x* 上预测变差。

4. 检查

x* 仍要像原句，不能语义漂移。

这里的“像原句”有两个层级。token-level 相似，是说输出词序列不要离原句太远；sentence-level 相似，是说整体语义向量要接近。只做 token-level 可能错过同义替换，只做 sentence-level 又可能放过某些标签翻转，因此两者常一起出现。

现在你应该能做什么：看到 autoencoder 生成攻击样本时，能区分“复制输入”“合理改写”和“语义已经改变”的差别。

三类损失：像原句、像语义、能攻击

LOSSES

训练 generator 不能只看一个目标。若只要求它骗过 classifier，它可能直接生成和原句相反的句子；若只要求重建，它会学会复制输入，没有攻击能力。因此课程里把目标拆成三类 loss。

Reconstruction loss

要求输出 token 和输入 token 尽量接近。直觉是“不要把句子改得面目全非”。

Similarity loss

要求输入和输出的 embedding / sentence representation 接近。直觉是“整体语义要保留”。

Adversarial loss

要求原 classifier 在输出句子上表现变坏。直觉是“合理地让模型犯错”。

Lattack = λrecLrec + λsimLsim + λadvLadv L_rec 控制字面接近，L_sim 控制语义接近，L_adv 控制攻击效果。权重 λ 决定三者取舍。

权重很重要。λ_rec 和 λ_sim 太大，generator 会变保守，只会复制或微小替换；λ_adv 太大，它会为了攻击而牺牲语义。实际研究中，这些权重不是纯数学装饰，而是决定样本是否可信的实验条件。

一个好的回答方式是：重建和相似度让样本“仍是原任务中的同一个例子”，攻击损失让它“暴露模型脆弱性”。少任何一类，目标都会偏。

现在你应该能做什么：用三类 loss 解释生成式文本攻击为什么既要保守又要有攻击性。

Attack step 与 Defense step：一边制造困难，一边训练鲁棒分类器

TRAINING LOOP

课程里的训练不是只有 generator。还会训练一个 robust classifier，也就是希望它面对正常输入 x 和扰动输入 x* 都能正确分类。于是训练过程分成 attack step 和 defense step。

attack step 主要更新 generator。此时原来的 target classifier 通常固定不动，generator 学习如何输出能让它失败的 x*。这一步像是在制造更难的训练样本。

defense step 则更新 robust classifier，让它在正常输入和 generator 产生的扰动输入上都做对。有些设定也会继续约束 generator，让它生成的样本仍然合理，否则防御训练会被低质量样本拖偏。

步骤谁主要更新目标容易出错的地方
Attack stepgenerator产出像原句、但让旧 classifier 失败的文本。只追求攻击会造成语义漂移。
Defense steprobust classifier对 x 和 x* 都预测正确。若扰动样本不合理，训练信号会变脏。

这个交替过程的直觉是“出题者”和“答题者”一起变强。generator 不断制造更难的合理扰动，robust classifier 不断学习不要被这些扰动欺骗。最终目标不是让 generator 赢，而是让分类器在真实部署中更稳。

现在你应该能做什么：解释为什么训练期既要有攻击步骤，也要有防御步骤；并能指出两步各自更新谁。

关键困难：token 采样不可微

DISCRETE TOKENS

神经网络训练靠反向传播。问题是，generator 输出的不是直接的词，而是每个位置上的 token distribution。要变成一句真正文本，通常要 sample 或 argmax 选出某个 token。这个“选择动作”是离散的，梯度没法顺利穿过去。

可以把它想成：softmax 输出 [0.70, 0.20, 0.10] 还有连续变化；但 argmax 一做，就变成第一个 token。只要第一个概率仍最大，0.70 改到 0.65，输出 token 完全不变，后面的 loss 很难告诉前面的 generator “你应该怎么微调”。

错误示例：训练时直接 argmax 后再算 lossGradient break
# 错误示例：argmax 产生离散 token，loss 很难回传到 logits。
logits = generator(x)
token_ids = logits.argmax(dim=-1)
x_star = embedding_lookup(token_ids)
loss = classifier_loss(classifier(x_star), y)
loss.backward()

这段伪代码的问题不是“不能运行”，而是训练信号不理想。它在选择 token 的地方把连续概率压成离散编号，generator 很难从 classifier loss 学到细腻方向。于是需要一种近似：输出要接近 one-hot，方便代表某个词；同时又要保持可微，方便训练。

现在你应该能做什么：说清楚为什么 NLP 攻击比图像攻击多一个困难：文本必须选离散 token，而离散选择会阻断梯度。

Gumbel-Softmax：像抽样，又能让梯度通过

REPARAMETERIZATION

Gumbel-Softmax 的作用是把“从类别分布中抽一个 token”改写成一个可微近似。它会在 logits 上加入 Gumbel noise，再用带 temperature 的 softmax 生成一个接近 one-hot 的向量。temperature 越低，输出越尖锐，越像真正选了一个 token；temperature 越高，输出越平滑，梯度更容易传播，但不像离散词。

yi = exp((log αi + gi) / τ) / Σj exp((log αj + gj) / τ) α_i 是类别权重，g_i 是 Gumbel noise，τ 是 temperature。τ 小则更接近 one-hot，τ 大则更平滑。

调节 temperature，观察概率分布从平滑到尖锐的变化。

0.80

直觉：低温像做选择，高温像保留混合。训练时常希望在两者之间取得平衡。

注意，Gumbel-Softmax 是近似，不是把文本离散问题变没了。它让训练可以进行，但最后若要输出真实句子，仍要映射回具体 token。因此你在评估样本时，仍需要检查最终文本，而不是只看连续向量空间里的 loss。

现在你应该能做什么：用 temperature 解释“越像离散选择”和“越方便训练”之间的取舍。

ASCC：在同义词 convex hull 中做训练期防御

ASCC

ASCC 的核心想法是：如果同义词替换会攻击模型，那就不要只在原词 embedding 附近画一个球；可以直接使用同义词集合的 embedding，构造它们的 convex hull，然后在这个区域里寻找让 loss 最大的 adversarial embedding。

convex hull 可以理解为一组点能张成的最小凸集合。若有同义词 embedding e1, e2, e3, e4，convex hull 中的点可写成它们的加权平均，权重非负且总和为 1。这样做比随便选一个半径更贴近“这些同义词形成的区域”。

eadv = Σi wi ei, wi ≥ 0, Σiwi = 1 用 softmax 参数化权重可以自然满足非负和总和为 1。再加入 entropy 约束，可让权重更接近某个实际同义词。

为什么不用单纯半径

半径太小会漏掉同义词，半径太大又会包含非同义词。模型被迫对不合理替换也保持同一标签，会带来错误训练信号。

为什么加 entropy

convex hull 中间点未必是真实 token。让权重更尖锐，能让 adversarial embedding 更靠近某个具体同义词。

ASCC 的防御逻辑是：先在同义词 convex hull 中找最容易让模型出错的 embedding，再用这个困难点训练模型，使模型在同义词替换附近不要剧烈改变输出。这属于训练期增强，不是测试时修复输入。

现在你应该能做什么：解释 ASCC 为什么比简单 embedding ball 更针对同义词替换攻击，也能说出它仍依赖同义词集合质量。

DISP：推理期先找被扰动 token，再恢复

DISP

DISP 是推理期防御。它不直接要求分类器变得无所不能，而是在输入进分类器前先做“检查与修复”：判断哪些 token 可能被攻击者替换过，估计这些位置原本应该是什么 embedding，再找最接近的词换回去。

1. Detect

perturbation discriminator 判断每个 token 是否可疑。

2. Estimate

embedding estimator 估计原 token 的 embedding。

3. Lookup

在 embedding 表中找最近 token。

4. Predict

把恢复后的句子送入原 classifier。

它的优点是思路清楚：如果攻击主要来自局部替换，先修局部再分类可能有效。它也和一些 replaced token detection 预训练任务有相似直觉：判断某个位置是否被换过。

它的局限也很重要：你必须知道或模拟攻击样本长什么样，才能训练 detector 和 estimator。如果训练时只见过同义词替换，测试时攻击者改的是词形、字符、句法结构或更隐蔽的 paraphrase，检测器可能直接失效。

现在你应该能做什么：把 DISP 拆成“检测、估计、查表、分类”四步，并说明它为什么依赖攻击类型假设。

FGWS：利用低频替换词做修复

FGWS

FGWS 的观察是：某些 word substitution 攻击会把常见词替换成低频同义词。例如把常见的 clever 换成很少出现的词。低频词本身不一定错，但如果它突然出现在一个原本普通的句子里，就可能是攻击留下的痕迹。

FGWS 的流程是：先计算每个词在训练资料中的 frequency 或 log frequency，低于阈值 δ 的词被标记为可疑；再从同义词表中找高频同义词替换；最后比较替换前后 classifier 输出变化。如果原先类别概率变化超过阈值 γ，就把原输入视为对抗样本，并使用修复后的版本。

An ordinary movie with splendiferous acting.
An ordinary movie with excellent acting.
4.0
0.45

阈值决定哪些词会被修复，也决定误报和漏报。

FGWS 的边界很明显：如果攻击不是“低频同义词替换”，它就不一定有效。若攻击者使用高频词、字符扰动、词形变化或整句改写，低频线索会变弱。阈值 δ 和 γ 也需要开发集调参，不能保证换到新分布仍然好用。

现在你应该能做什么：解释 FGWS 为什么能防一类替换攻击，也能指出它为什么不是通用防御。

评估边界：防御不是把攻击样本清掉就结束

EVALUATION

P42 的后半节不断提醒一件事：防御方法都带有假设。ASCC 假设同义词集合和 embedding 区域有意义；DISP 假设你能模拟攻击样本并训练检测器；FGWS 假设攻击会留下低频替换词痕迹。假设越强，报告结果时越要说清实验场景。

评价防御时，至少要同时看 clean accuracy 和 robust accuracy。clean accuracy 是正常样本上的表现，robust accuracy 是攻击样本上的表现。如果防御让攻击样本更稳，却把正常样本准确率明显打坏，那不一定值得部署。

还要看攻击适配性。一个防御只对已知攻击有效，不代表对新攻击有效。更强的评估会让攻击者知道防御机制，再尝试绕过它；如果防御只能拦住没有针对它调整过的旧攻击，它的实际价值有限。

样本质量

扰动后是否语义保持、文法自然、标签不变。

性能权衡

正常样本和攻击样本都要报告，不能只挑好看的指标。

适配攻击

攻击者知道防御后是否还能绕过，是更现实的压力测试。

现在你应该能做什么：评价一个 NLP 防御实验时，不只问“有没有变准”，还会追问“对哪类攻击、牺牲了什么、能否被绕过”。

最小代码：生成式攻击训练与可微近似

CODE

下面的伪代码只保留核心语义。第一段展示 generator 的 attack step：输出 soft token，用可微近似喂给 classifier，再把重建、相似度和攻击目标加起来。

正确示例：attack step 的三类 lossPython-like
def attack_step(generator, frozen_clf, x, y, tau):
    logits = generator(x)
    soft_tokens = gumbel_softmax(logits, tau=tau)
    x_star_emb = soft_tokens @ embedding_table

    rec_loss = token_reconstruction_loss(logits, x)
    sim_loss = sentence_distance(encode(x), encode_soft(x_star_emb))
    adv_loss = -classification_loss(frozen_clf(x_star_emb), y)

    loss = lam_rec * rec_loss + lam_sim * sim_loss + lam_adv * adv_loss
    loss.backward()
    update(generator)

注意 adv_loss 的符号取决于实现：有些写法是最大化原 classifier 的错误，有些写法是最小化目标类别 loss。你不用死记符号，只要抓住目标：让 x* 在旧模型上更难被正确分类。

正确示例：defense step 同时看正常输入与扰动输入Python-like
def defense_step(robust_clf, generator, x, y):
    x_star = generator.make_soft_example(x)
    clean_loss = classification_loss(robust_clf(x), y)
    robust_loss = classification_loss(robust_clf(x_star), y)
    loss = clean_loss + beta * robust_loss
    loss.backward()
    update(robust_clf)
