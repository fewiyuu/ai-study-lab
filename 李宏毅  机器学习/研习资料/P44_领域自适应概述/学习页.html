模型不是学会任务，而是学会某个领域里的任务

在许多 benchmark 里，训练资料和测试资料来自同一分布，所以模型看起来很强。但真实部署常常不是这样：实验室里的手写数字是黑白的，用户上传的数字可能是彩色的；训练资料来自干净环境，实际输入来自噪声场景。训练和测试分布一变，准确率可能从 99% 直接掉到不及格。

领域自适应要处理的就是这个问题：我们有 source domain 的标注资料，希望模型能在 target domain 上工作。target domain 可能有少量标注资料，也可能只有大量无标注资料。P44 重点讲总览和最经典的想法：学习一个 domain-invariant feature，让 source 和 target 的表示看起来接近。

Domain Shift：训练资料和测试资料不再同分布

SHIFT

domain shift 指训练资料和测试资料的分布不同。最直观的例子是：训练时数字图片是黑白的，测试时变成彩色的。人类会说“数字形状没变”，但模型可能学到许多 source domain 的细节，例如背景、灰阶、笔画粗细、颜色通道统计。一旦这些统计变了，模型就可能失败。

这也是 benchmark 和真实部署之间的差距。benchmark 通常默认 train / test 来自同一分布；真实系统上线后，使用者、设备、场景、语言风格、光照、地区都会改变输入分布。模型在 source test 上高分，不代表它能处理 target。

Domain Shift 不只一种

TYPES

课程中提醒，分布变化有多种。P44 主要关注输入分布变化，也就是图片外观、文字风格、语音环境等 x 的分布变了。但还有其他可能：输出标签比例变了，或者输入和标签之间的关系变了。

为什么要区分？因为不同 shift 需要不同方法。若只是输入外观变了，学 domain-invariant feature 可能有帮助；若 target 的类别先验比例变了，你可能需要校准输出分布；若 x → y 的关系都变了，简单对齐 feature 可能反而有害。

变化类型直觉例子风险
输入分布变化黑白数字训练，彩色数字测试。模型依赖 source 外观统计。
输出分布变化训练集中 0-9 均衡，部署中某些数字更常见。预测校准和阈值可能失准。
输入-标签关系变化同样图案在两个场景中代表不同标签。source 规则本身可能不能迁移。

现在你应该能做什么：不要把所有分布变化都叫成同一种问题；先判断变的是输入、输出，还是关系。

Target 资料有多少，决定问题难度

SETTINGS

领域自适应不是一个固定场景。关键是 target domain 上有什么资料。若 target 有很多标注资料，那最直接的做法就是用 target 资料训练，不必绕远路。若 target 只有少量标注资料，可以从 source model 出发做 fine-tuning，但要防止 overfitting。

P44 主要关注的是：target 有大量资料，但没有标签。这很符合真实系统：模型上线后可以收集用户输入，但很难为每个输入标注正确答案。于是我们不能直接用 supervised loss，却可以利用 target 输入分布来调整 feature space。

很多 target 标注

直接用 target 训练或混合训练，问题相对简单。

少量 target 标注

fine-tuning；小学习率、少轮数、正则化，避免过拟合。

很多 target 无标注

无标注领域自适应：用 target 输入分布帮助学习表示。

如果 target 无标签且资料很少，甚至只有一个样本，常规对齐方法就很难工作；这会引出 testing-time training 等方向。如果 target 资料完全未知，就更接近 domain generalization。

现在你应该能做什么：拿到一个迁移问题时，先用“target 有多少资料、有没有标签”给它分类。

核心直觉：保留共同因素，去掉领域外观

INVARIANT FEATURE

在黑白数字到彩色数字的例子里，我们希望模型忽略颜色，只保留数字形状。更一般地，feature extractor 应该把 source 和 target 的表面差异过滤掉，抽取任务真正需要的共同因素。

如果 source image 和 target image 经过 feature extractor 后落在相似分布里，那么 source 上训练出的 label predictor 就更有机会应用到 target。注意这里说的是“更有机会”，不是保证。因为 feature 对齐还必须保留类别结构：0 还是要像 0，1 还是要像 1，不能为了对齐而把类别信息洗掉。

把分类器拆成两段：Feature Extractor + Label Predictor

MODEL PARTS

一个普通分类器可以拆成前半段和后半段。前半段叫 feature extractor，把输入图片或文本转成 feature vector；后半段叫 label predictor，把 feature vector 转成类别预测。CNN 里前几层输出可能是 feature map，拉平或池化后也可以看作 feature。

拆在哪里是超参数。你可以把浅层当 feature，也可以把更深层当 feature。浅层通常更接近边缘、颜色、纹理等低阶统计；深层更接近语义类别。领域自适应选择哪一层对齐，会影响最终效果。

输入 x

source 有标签，target 可能无标签。

Feature extractor

输出 feature vector 或 feature map。

Label predictor

用 source label 训练分类。

Domain classifier

判断 feature 来自 source 还是 target。

source data 可以训练整条分类链，因为它有标签。target data 没有标签，不能算分类 loss，但可以送过 feature extractor，再让 domain classifier 判断它来自哪个领域。这个二元分类任务就给了 target data 使用价值。

现在你应该能做什么：读懂领域对抗训练图时，知道每个模块吃什么、输出什么、用哪种资料训练。

Domain Adversarial Training：让领域分类器分不出来

DAT

domain adversarial training 的想法很像一个小型对抗游戏。domain classifier 想分辨 feature 来自 source 还是 target；feature extractor 则想骗过 domain classifier，让两个领域的 feature 混在一起。与此同时，label predictor 仍要在 source 上把类别分对。

如果只让 feature extractor 骗 domain classifier，它可以输出全零向量，让任何领域都不可分。但这样 label predictor 也无法分类。所以 source classification loss 是必要约束：feature 必须既能服务分类，又能消除领域差异。

minF,P Lcls(P(F(xs)), ys) + adversarial alignment F 是 feature extractor，P 是 label predictor。source 有标签，用来训练分类；source 和 target 都可用于领域对齐。
domain classifier: min Ld,   feature extractor: make domain hard to predict 实际实现常用 gradient reversal layer：前向不变，反向把 domain loss 的梯度乘上负系数。

这里容易误解的一点是：不是让 source 变成 target，也不是让 target 变成 source，而是让提取出的表示不再携带明显领域信息。理想状态下，domain classifier 只能瞎猜，但 label predictor 仍能正确分辨类别。

现在你应该能做什么：用“领域不可分、类别仍可分”概括 domain adversarial training。

动手观察：对齐太强，类别结构会被压坏

TRADEOFF

领域对齐不是越强越好。如果 alignment 权重太弱，target feature 仍和 source 分开，target 表现不好；如果权重太强，模型可能为了骗过 domain classifier，牺牲 label predictor 需要的类别结构。

方法限制：对齐分布不等于对齐类别

LIMITS

无标注 target data 的困难在于：我们知道它来自 target，但不知道每个点属于什么类别。若只要求 source feature 和 target feature 的整体分布接近，可能出现错误对齐。比如 target 的 1 被拉到 source 的 7 附近，整体看起来混合了，但分类会错。

一个常见补充想法是让 target 样本远离分类边界。虽然不知道 target label，但希望 label predictor 输出足够集中，不要每一类概率都差不多。这类思想和 entropy minimization、DIRT-T、maximum classifier discrepancy 等方法有关。P44 只做概述，后续可以再深入。

另一个限制是类别集合未必一致。source 可能有老虎、狮子、狗，target 可能只有狗和猫，或者两边各有独特类别。若强行把所有分布完全对齐，就可能把不同类别揉在一起。这会引出 universal domain adaptation 等更复杂设定。

错误示例

只看 source / target 的 feature 是否混合，就宣布适应成功。这样可能忽略 target 类别被错配到 source 类别的问题。

更稳的检查

观察 target pseudo-label、预测熵、少量人工标注验证、类别覆盖假设和不同对齐层的效果。

现在你应该能做什么：指出“分布对齐”与“分类正确”不是一回事。

Target 资料更少时：TTT 与 Domain Generalization

BEYOND DA

如果 target 有大量无标注资料，可以估计 target 分布并做对齐。但如果 target 资料很少，甚至测试时只看到一个样本，就很难靠分布对齐工作。testing-time training 试图在测试时利用单个或少量样本上的自监督目标，让模型临时适应当前输入。

如果训练时完全不知道 target domain，就不再是适应某个已知 target，而是 domain generalization。它希望模型训练完后，面对未知领域也能表现稳定。若训练资料包含多个领域，可以学习跨领域共同结构；若训练资料只有一个领域，常见思路是用 data augmentation 人造多种变化，让模型提前见过更多可能性。

设定target 信息代表思路
Domain adaptation训练时知道 target，有资料。fine-tuning、feature alignment、对抗训练。
Testing-time training测试时才看到少量 target 样本。用自监督任务临时更新或调整。
Domain generalization训练时不知道具体 target。多领域训练、增强、学习领域无关表示。

现在你应该能做什么：区分“已知 target 后适应”和“未知 target 前泛化”。

最小代码：Gradient Reversal 风格训练骨架

CODE

下面的伪代码展示 domain adversarial training 的核心。source batch 有标签，可以算分类 loss；source 和 target 都有领域标签，可以算 domain loss。feature extractor 在分类 loss 上正常下降，在 domain loss 上反向更新，让 domain classifier 更难分辨。

正确示例：source 分类 + domain 对齐Python-like
def train_step(source_x, source_y, target_x, lamb):
    xs = concat(source_x, target_x)
    domain_y = concat(zeros(len(source_x)), ones(len(target_x)))

    feat_source = feature_extractor(source_x)
    cls_logits = label_predictor(feat_source)
    cls_loss = cross_entropy(cls_logits, source_y)

    feat_all = feature_extractor(xs)
    reversed_feat = gradient_reversal(feat_all, lamb)
    domain_logits = domain_classifier(reversed_feat)
    domain_loss = binary_cross_entropy(domain_logits, domain_y)

    loss = cls_loss + domain_loss
    loss.backward()
    update_all_modules()

gradient_reversal 前向传播时像 identity，反向传播时把梯度乘上 -λ。这样 domain classifier 自己学会分 source / target，而 feature extractor 收到相反信号，学习隐藏领域信息。

# 错误示例：没有分类约束，feature 可以坍缩成常数。

domain_loss = confuse_domain_classifier(feat_all)
