# 本节的概念地图 {#goals}
<!-- tag: MAP -->

BERT 的预训练任务很简单：把部分 token mask 掉，让模型做填空题。但当这个填空题用很多语言、很多资料、很大的 Transformer 来做时，模型会得到一些出乎意料的迁移能力。

三个故事其实在问同一个问题：预训练到底学到什么？跨语言故事说明它能把不同语言的某些语义结构接起来；跨学科故事说明它可能学到更一般的序列归纳偏置；人造资料故事则提醒我们，预训练不是只要做就有效，资料中的规律类型会强烈影响下游能力。

# 故事一：multilingual BERT 可以跨语言裸考 {#story1}
<!-- tag: CROSS-LINGUAL -->

multilingual BERT 在许多语言上做 masked language modeling。它把多种语言的句子都喂进去做填空题。神奇之处在于：如果你用英文 QA 资料微调它，它可以直接在中文 QA 上测试，即使没有看过中文 QA 训练资料。

这叫 zero-shot cross-lingual transfer。zero-shot 是指目标语言的下游任务训练样本为零；cross-lingual 是指从一种语言训练，迁移到另一种语言。课程中的例子是：英文 SQuAD 微调，中文 DRCD 测试，仍能得到相当不错的 F1。

预训练多语言文本上做填空，学到跨语言共享结构。
微调只用英文 QA 资料训练问答能力。
测试直接在中文 QA 上作答，不提供中文 QA 标注。

直觉上，这说明模型内部可能把不同语言中相似语义的表达接到某种共享表示上。英文里学到“问题、文章、答案 span”的任务格式后，可以迁移到中文。但这只是直觉，不能直接证明它完全无视语言差异。

**现在你应该能做什么：**解释为什么“只用英文 QA 微调、中文 QA 测试”是跨语言能力的强证据。

# 表示是否真的对齐：用 MRR 做词义检索 {#align}
<!-- tag: MRR -->

为了检查 multilingual BERT 是否把同义的中英文词放近，可以做一个词典检索实验。先准备中英词典，例如 `year → 年`、`month → 月`。再为每个词找一个代表向量：因为 BERT 是 contextualized representation，同一个词在不同上下文的表示不同，所以可以把它在语料中多次出现的表示取平均。

接着，对每个英文词，计算它和所有中文词表示的相似度，看正确中文翻译排第几。若 `year` 和 `年` 的相似度最高，rank 就是 1；若 `month` 对应的 `月` 排第三，rank 就是 3。最后用 mean reciprocal rank，也就是 MRR，汇总结果。

$$
MRR = (1 / N) Σi=1N 1 / ranki
rank 越小，倒数越大；MRR 越高，说明正确翻译越常排在前面。
$$

实验发现：资料量足够、模型结构足够强时，multilingual BERT 的跨语言 alignment 比传统 word embedding 更好。资料少时，alignment 会明显变差。这说明跨语言能力不是凭空来的，资料和模型都重要。

**现在你应该能做什么：**说清 MRR 如何把“中英同义词是否靠近”变成可计算指标。

# 语言差异没有消失：它可能藏成一个方向 {#langvec}
<!-- tag: LANGUAGE VECTOR -->

如果 multilingual BERT 完全无视语言差异，就会出现一个矛盾：做填空时，它怎么知道英文句子该填英文词，中文句子该填中文词？事实是，它仍然保留语言信息，只是语义共享和语言差异以某种方式同时存在。

一个很直观的实验是计算“英文平均 embedding”和“中文平均 embedding”的差异向量。把英文词的表示加上这个差异方向，可能会靠近对应中文词；把整句英文表示往这个方向平移，再让模型还原，可能得到部分中文化的句子。它不是高质量翻译，但说明语言信息可以像一个方向一样被抽出来。

调节语言差异方向强度，观察英文 token 如何逐步靠近中文对应词。


平移强度45
直觉：共享语义让 fish 和 鱼 有联系；语言方向让模型仍知道当前更像英文还是中文。

这个发现也能指导跨语言微调：若模型在英文上微调，在中文上测试，中文 representation 可能和英文 representation 仍有偏差。若能把中文表示往英文方向修正一点，下游模型可能更容易使用。

**现在你应该能做什么：**解释为什么 multilingual BERT 既能跨语言迁移，又没有完全丢掉语言身份。

# 故事二：在人类语言上预训练，也能帮非语言任务 {#story2}
<!-- tag: CROSS-DISCIPLINE -->

第二个故事更反直觉：把在人类语言上预训练的 BERT，用到 DNA、蛋白质、音乐这类非自然语言任务上，也可能提升分类表现。做法是把非语言序列映射成 token sequence。例如 DNA 只有 A、T、C、G 四种符号，可以随机映射到一些英文 token，再送入 BERT。

这个实验的关键是：BERT 本体在英文或中文文字上预训练的。若它仍然能帮助非语言任务，说明它学到的不只是“英文语义”，还可能包含更一般的序列处理能力、初始化优势、注意力结构使用方式或优化路径。

| 任务 | 输入如何变成 token | 为什么令人意外 |
| --- | --- | --- |
| DNA 分类 | A/T/C/G 随机映射到词表 token。 | 模型从未在 DNA 上做过预训练。 |
| 蛋白质分类 | 氨基酸符号映射到 token。 | 生物序列和人类语言语义不同。 |
| 音乐分类 | 音符序列映射到 token。 | 任务结构不同，但仍是序列。 |

要注意，映射关系本身不一定有语义。A 对应哪个英文词不重要，重要的是这些符号形成序列，模型可以利用预训练参数提供的序列处理能力。这个结论也提醒我们：不能把预训练模型的能力简单归因于“懂语言”。

**现在你应该能做什么：**说明为什么 cross-discipline transfer 支持“预训练学到通用序列能力”的猜想。

# 它到底帮了什么：optimization 与 generalization {#whyhelp}
<!-- tag: WHY HELPS -->

预训练可能通过两个方向帮忙。第一是 optimization：有预训练初始化时，训练 loss 更容易下降，模型比较不容易卡在差的优化状态。第二是 generalization：即使训练集上都能做到很低 loss，预训练模型在 development set 上仍可能表现更好。

## box info
### Optimization
预训练参数给了更好的起点或更好的特征处理方式，使训练过程更顺。

## box soft
### Generalization
预训练带来的归纳偏置让模型不只是记住训练集，而更容易举一反三。

这两个概念要分清。optimization 看训练过程能不能把训练 loss 降下来；generalization 看训练好后能不能在未见过的样本上做对。预训练可以同时帮助两者，但实验设计要能区分：例如让 scratch model 训练更久，使训练 loss 也很低，再比较 development accuracy。

**现在你应该能做什么：**判断一个预训练实验是在证明“更好优化”，还是证明“更好泛化”。

# 应用：语音问答里接上文字 BERT {#speechqa}
<!-- tag: SPEECH QA -->

语音问答的梦想是：不先做语音辨识，直接听问题和文章音档，输出答案在音档中的位置。传统 cascade 方法先 ASR 转文字，再做文字 QA；问题是 ASR 错误会传给后面的 QA 模型。

直接用语音预训练模型如 HuBERT 做 end-to-end speech QA 并不容易，因为语音表示可能主要学到声音内容，还不够语义化。课程中的想法是：把 HuBERT 输出离散化成 token ID，再随机映射到文字 token，接上在人类文字上预训练过的 BERT / Transformer 层，借用文字预训练模型的理解能力。

**语音输入**问题和文章都是音档。
**HuBERT**抽出语音内容表示并离散化。
**文字 BERT**接收映射后的 token sequence。
**QA head**预测答案片段的位置。

结果显示，有文字预训练初始化后，end-to-end 模型显著变好，而且不受 ASR 错误率直接影响。ASR 错误率高时，end-to-end 路线甚至可能超过 cascade。这个故事把“跨学科能力”落到一个具体应用上。

**现在你应该能做什么：**解释为什么把语音预训练模型和文字预训练模型接起来，可能比只靠语音模型更强。

# 故事三：人造资料也能预训练，但必须有规律 {#story3}
<!-- tag: SYNTHETIC DATA -->

第三个故事问得更尖锐：如果预训练模型有这么强的迁移能力，是否在任何资料上做预训练都有用？答案是否定的。随机生成、没有规律的 token sequence 基本没有帮助；但某些有结构的人造资料，例如成对出现的 token、shuffle 后的连续数字序列，竟然能带来接近真实语言的一部分收益。

实验流程是：用规则生成 token sequence，在这些假资料上训练 BERT 做填空题；再把人类语言词汇随机映射到这些 token ID 上，拿这个模型去做 GLUE 等 NLP 下游任务。若下游表现比 train from scratch 好，就说明这种人造预训练规则给模型提供了有用能力。

随机序列没有可学习规律，预训练几乎无帮助。
成对结构某 token 出现后，另一个位置会出现关联 token。
长范围 shuffle连续编号打乱，迫使模型看更长上下文做填空。

这说明资料很重要，但重要的不只是“像不像英文”。某些结构性能力，例如捕捉长程依赖、配对关系、上下文补全，也可能是 NLP 任务需要的底层能力。

**现在你应该能做什么：**解释为什么“预训练动作本身”不够，资料必须包含能迁移的规律。

# 未解之谜：预训练到底学到了哪些可迁移能力 {#reflect}
<!-- tag: OPEN QUESTIONS -->

三个故事共同指向一个开放问题：自监督模型为什么这么能迁移？跨语言说明它能把语义跨符号系统连接起来；跨学科说明它可能学到更一般的序列处理归纳偏置；人造资料说明有效预训练不一定需要自然语言，但需要某种可迁移结构。

但这些故事也提醒我们不要过度神化。资料量不足，跨语言 alignment 会变弱；随机资料没有规律，预训练没有帮助；语音模型只抽声音内容，不一定足够理解语义。预训练有效，往往是资料规律、模型结构、下游任务三者碰巧对上。

学习这节后，你应该更会问实验问题：这个模型到底靠什么迁移？它是在学语义、位置、长程依赖、配对规律、优化路径，还是某种混合？这个问题没有一句话答案，但它正是自监督学习仍然有研究价值的地方。

**现在你应该能做什么：**用更精确的问题替代“BERT 很神”：它学到的能力能否在新任务中被利用？

# 最小代码：跨域 token 映射与人造预训练 {#code}
<!-- tag: CODE -->

下面的伪代码展示非语言序列如何被映射到 BERT token。映射可以是随机的，因为重点预训练模型提供的序列处理能力。

正确示例：DNA 符号随机映射到 tokenPython-like

```text
def map_dna_to_tokens(dna_seq, table):
 # table: {'A': 'we', 'T': 'he', 'C': 'is', 'G': 'she'}
 return [table[ch] for ch in dna_seq]

dna = "AGAC"
tokens = map_dna_to_tokens(dna, table)
features = bert(tokens)
logits = classifier(features)
```

下面是人造资料的简化生成。随机序列没有结构；成对结构和长范围 shuffle 则能迫使模型学习可预测关系。

正确示例：生成有规律的人造序列Python-like

```text
def make_pair_sequence(length):
 seq = []
 for _ in range(length // 2):
 token = random_token()
 seq.extend([token, token])
 shuffle(seq)
 return seq

def make_shuffle_sequence(start, span):
 seq = list(range(start, start + span))
 shuffle(seq)
 return seq
```

```text
pretrain_bert(fake_data)
claim("pretraining always helps")

# 应该检查资料是否有可迁移规律，以及和下游任务有什么关系。
```
