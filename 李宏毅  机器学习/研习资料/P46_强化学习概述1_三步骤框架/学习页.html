强化学习不是另一套玄学，它仍然是机器学习三步骤

强化学习常被 AlphaGo、打游戏、机器人控制这些例子包起来，看起来很神秘。但 P46 的目标是把它拆回机器学习的基本框架：找一个 function，定义好坏，再找参数让好坏指标变好。

不同点在于，监督学习有现成标签，告诉模型“这个输入应该输出什么”；强化学习没有逐步正确答案，只有 actor 和 environment 互动后得到的 reward。我们要找的 actor 会根据 observation 输出 action，目标是让整段互动得到的 total reward 最大。

本节的概念地图

MAP

P46 先不推导 policy gradient，而是建立 RL 的问题观。你要先能说清：actor 是要学习的函数，environment 是互动对象，observation 是输入，action 是输出，reward 是环境给的即时回馈，return 是一整场互动的 reward 总和。

本节最重要的桥接是：RL 也可以看成机器学习三步骤。第一步，写出带参数的 actor；第二步，定义目标，也就是最大化 total reward，等价于最小化负的 total reward；第三步，优化参数。困难主要出现在第三步，因为环境是黑盒、action 采样有随机性、reward 往往延迟。

没有逐步标签 Actor 选动作 环境回应 得到 Reward 累计 Return 优化 Policy

为什么需要强化学习

MOTIVATION

监督学习要求我们知道每个输入的正确输出。做图片分类时，图片是 input，类别是 label。问题是，很多任务中“正确动作”并不容易给出。例如围棋某个局面下一步该下哪里，人类棋谱可以提供参考，但不一定是最佳答案。

RL 适合这种场景：我们不知道每一步的最佳动作，但知道最后或过程中什么结果好。打游戏时，我们不一定知道当前画面最优动作是什么，但杀掉敌人会得分；下围棋时，中间很多落子没有即时分数，但最后赢棋就是好。

监督学习

有输入和正确输出；训练目标是模仿标签。

强化学习

没有逐步正确动作；训练目标是通过互动得到更高回馈。

现在你应该能做什么：判断一个任务为什么不能直接收集标准标签，以及是否适合考虑 RL。

互动框架：Actor 与 Environment

INTERACTION

RL 中的 actor 是我们要训练的模型。environment 是 actor 互动的世界，可以是游戏机、棋盘对手、机器人所在的物理世界或推荐系统用户。environment 给 actor 一个 observation，actor 根据 observation 输出 action。action 改变环境，环境再给新的 observation 和 reward。

两个例子：Space Invader 与围棋

EXAMPLES

在 Space Invader 中，observation 是游戏画面，action 可以是左移、右移、开火，reward 是游戏分数。向左或向右通常不会立刻得分，开火击中敌人才会得到 reward。actor 的目标不是每一步都开火，而是学到怎样移动和开火能让整局总分最高。

围棋也是同一个框架。observation 是棋盘状态，action 是选择落子位置，environment 是对手和棋局规则。多数中间落子没有即时 reward，只有最后赢或输才有回馈。这叫 delayed reward，也就是延迟回馈，是 RL 难点之一。

RL 也是机器学习三步骤

THREE STEPS

机器学习三步骤是：写出带未知参数的 function，定义 loss，做 optimization。RL 也一样，只是名字变了。function 是 actor / policy network；目标是最大化 total reward；如果非要写成 loss，就把 loss 定义成负的 total reward。

Step 1：函数

actor πθ(a|s)，输入 observation，输出 action 概率。

Step 2：好坏

一段互动得到 total reward，越大越好。

Step 3：优化

找 θ 让 expected total reward 最大。

loss(θ) = - total reward 普通训练最小化 loss；RL 想最大化 reward。因此把 reward 加负号，就能写成最小化问题。

现在你应该能做什么：用机器学习三步骤解释 RL，而不是把 RL 当作完全陌生的领域。

Actor / Policy Network：像分类器一样输出动作分数

POLICY

actor 通常是一个 network，也叫 policy network。它可以像分类器一样：输入一张游戏画面，输出每个 action 的分数，再用 softmax 变成概率。例如左移 0.7，右移 0.2，开火 0.1。

为什么常从概率分布采样，而不是永远 argmax？第一，随机性可以探索不同动作；第二，有些环境中固定动作模式容易被对手利用；第三，后面的 policy gradient 会利用“采样到某个 action 的概率”来更新模型。

Reward、Return、Episode、Trajectory

TERMS

reward 是某一步行动后得到的即时回馈。return 是一整个 episode 中 reward 的总和，也叫 total reward。episode 是一次从开始到终止的完整互动，例如一局游戏。trajectory 是这次互动中 observation 和 action 的序列。

R = r1 + r2 + ... + rT r_t 是第 t 步 reward，R 是整段 episode 的 total reward。

优化困难：随机、黑盒、延迟回馈

OPTIMIZATION

如果 actor、environment、reward 都是可微网络，RL 看起来像一个 recurrent network，直接反向传播就好。但真实 RL 难在：action 常由采样产生，environment 是黑盒，reward 可能是规则或真实世界反馈，并不一定可微。

此外，环境本身可能有随机性。同一个 actor 参数、同一个初始画面，采样动作不同，环境反应不同，最后 return 也不同。这个随机性不是“训练初始化没有固定 seed”的随机，而是部署互动时就存在的随机。

Action 采样

同一 observation 可能采样到不同动作。

Environment 黑盒

不知道环境内部如何从 action 产生新 observation。

Delayed reward

重要回馈可能很晚才出现，很难判断哪一步该负责。

这就是为什么 RL 的第三步 optimization 需要专门算法，例如下一节要讲的 policy gradient。RL 和普通机器学习框架相同，但优化方法要处理互动和随机性。

现在你应该能做什么：说明 RL 难点不在“找不到函数形式”，而在“如何用黑盒互动回馈优化参数”。

像分类一样控制 Actor 的输出

CLASSIFIER VIEW

在进入 policy gradient 前，课程先讲一个直觉：如果我们已经知道某个 observation 下应该采取某个 action，就可以把训练 actor 看成分类。输入是 s，label 是希望采取的 action a_hat，用 cross entropy 训练 actor 增加这个 action 的概率。

如果我们希望 actor 不要采取某个 action，就可以反过来，让这个 action 的 cross entropy 变大。更一般地，每个 (s, a) pair 可以带一个分数，表示我们多希望 actor 在 s 下采取 a。分数为正就鼓励，分数为负就抑制。

L = - A(s,a) · log πθ(a|s) A(s,a) 可以理解成这个动作的好坏分数。正数会提高动作概率，负数会降低动作概率。

通往 Policy Gradient：训练资料和动作分数从哪里来

BRIDGE

如果我们有一堆资料，说在 s1 要做 a1，在 s2 不要做 a2，就能像训练分类器一样训练 actor。问题是这些资料从哪里来？每个动作的好坏分数 A 又从哪里来？

答案会在 policy gradient 中展开：让 actor 自己和环境互动，产生 trajectory；根据整段 trajectory 的 return 判断哪些 action 倾向于带来好结果，再提高这些 action 的概率，降低坏结果相关 action 的概率。P46 先到这里，下一节才正式讲怎么估计这些分数。

已知标签时

像监督学习：给定 (s, action)，用 cross entropy 训练。

RL 实际场景

标签未知；只能从 sampled trajectory 和 reward 中估计动作好坏。

现在你应该能做什么：带着两个问题进入 P47：trajectory 怎么产生？动作好坏分数怎么估计？

最小代码：Actor 与一次 Episode

CODE

下面的伪代码展示 actor 如何和 environment 互动。注意，actor 输出的是 action distribution，再从中 sample 一个 action。环境返回下一步 observation、reward 和是否结束。

正确示例：收集一条 trajectoryPython-like
def run_episode(actor, env):
    s = env.reset()
    trajectory = []
    total_reward = 0

    while True:
        probs = actor(s)
        a = sample_from(probs)
        next_s, reward, done = env.step(a)
        trajectory.append((s, a, reward))
        total_reward += reward
        s = next_s
        if done:
            break

    return trajectory, total_reward

下面是“如果已经知道动作好坏分数”的训练形式。下一节 policy gradient 会解释这些分数如何从 return 中估计。

正确示例：用动作分数调整 actorPython-like
def train_actor_from_scores(actor, samples):
    loss = 0
    for s, a, score in samples:
        prob = actor(s)[a]
        loss += -score * log(prob)
    loss.backward()
    update(actor)

现在你应该能做什么：读懂 RL 互动循环，并看出 policy gradient 要补上的正是 score 的来源。
