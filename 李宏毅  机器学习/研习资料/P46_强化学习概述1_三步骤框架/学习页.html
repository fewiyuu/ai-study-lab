为什么需要强化学习

MOTIVATION

监督学习要求我们知道每个输入的正确输出。做图片分类时，图片是 input，类别是 label。问题是，很多任务中“正确动作”并不容易给出。例如围棋某个局面下一步该下哪里，人类棋谱可以提供参考，但不一定是最佳答案。

监督学习

有输入和正确输出；训练目标是模仿标签。

强化学习

没有逐步正确动作；训练目标是通过互动得到更高回馈。

现在你应该能做什么：判断一个任务为什么不能直接收集标准标签，以及是否适合考虑 RL。

互动框架：Actor 与 Environment

INTERACTION

RL 中的 actor 是我们要训练的模型。environment 是 actor 互动的世界，可以是游戏机、棋盘对手、机器人所在的物理世界或推荐系统用户。environment 给 actor 一个 observation，actor 根据 observation 输出 action。action 改变环境，环境再给新的 observation 和 reward。

actor 本质上就是一个 function：输入 observation，输出 action 或 action 的概率分布。若 actor 是神经网络，它的参数就是要学出来的 unknown variables。

两个例子：Space Invader 与围棋

EXAMPLES

在 Space Invader 中，observation 是游戏画面，action 可以是左移、右移、开火，reward 是游戏分数。向左或向右通常不会立刻得分，开火击中敌人才会得到 reward。actor 的目标学到怎样移动和开火能让整局总分最高。

RL 也是机器学习三步骤

THREE

机器学习三步骤是：写出带未知参数的 function，定义 loss，做 optimization。RL 也一样，只是名字变了。function 是 actor / policy network；目标是最大化 total reward；如果非要写成 loss，就把 loss 定义成负的 total reward。

$$ loss(θ) = - total reward 普通训练最小化 loss；RL 想最大化 reward。因此把 reward 加负号，就能写成最小化问题。 $$

现在你应该能做什么：用机器学习三步骤解释 RL，而不是把 RL 当作完全陌生的领域。

Actor / Policy Network：像分类器一样输出动作分数

POLICY

actor 通常是一个 network，也叫 policy network。它可以像分类器一样：输入一张游戏画面，输出每个 action 的分数，再用 softmax 变成概率。例如左移 0.7，右移 0.2，开火 0.1。

右移0.45

开火0.25

调节偏好，观察 softmax 后的动作概率。

现在你应该能做什么：把 actor 输出动作概率这件事，和普通分类器输出类别概率联系起来。

Reward、Return、Episode、Trajectory

TERMS

reward 是某一步行动后得到的即时回馈。return 是一整个 episode 中 reward 的总和，也叫 total reward。episode 是一次从开始到终止的完整互动，例如一局游戏。trajectory 是这次互动中 observation 和 action 的序列。

注意 reward 是一步一步的回馈，return 是整局累计。

优化困难：随机、黑盒、延迟回馈

OPTIMIZATION

如果 actor、environment、reward 都是可微网络，RL 看起来像一个 recurrent network，直接反向传播就好。但真实 RL 难在：action 常由采样产生，environment 是黑盒，reward 可能是规则或真实世界反馈，并不一定可微。

Action 采样同一 observation 可能采样到不同动作。

Environment 黑盒不知道环境内部如何从 action 产生新 observation。

Delayed reward重要回馈可能很晚才出现，很难判断哪一步该负责。

现在你应该能做什么：说明 RL 难点不在“找不到函数形式”，而在“如何用黑盒互动回馈优化参数”。

像分类一样控制 Actor 的输出

CLASSIFIER

在进入 policy gradient 前，课程先讲一个直觉：如果我们已经知道某个 observation 下应该采取某个 action，就可以把训练 actor 看成分类。输入是 s，label 是希望采取的 action a_hat，用 cross entropy 训练 actor 增加这个 action 的概率。

$$ L = - A(s,a) · log πθ(a|s) A(s,a) 可以理解成这个动作的好坏分数。正数会提高动作概率，负数会降低动作概率。 $$

错误示例：以为 RL 直接拥有正确 action 标签Concept bug

现在你应该能做什么：解释为什么“形式上像分类”不等于 RL 已经退化成监督学习。

通往 Policy Gradient：训练资料和动作分数从哪里来

BRIDGE

如果我们有一堆资料，说在 s1 要做 a1，在 s2 不要做 a2，就能像训练分类器一样训练 actor。问题是这些资料从哪里来？每个动作的好坏分数 A 又从哪里来？

已知标签时

像监督学习：给定 (s, action)，用 cross entropy 训练。

RL 实际场景

标签未知；只能从 sampled trajectory 和 reward 中估计动作好坏。

现在你应该能做什么：带着两个问题进入 P47：trajectory 怎么产生？动作好坏分数怎么估计？

最小代码：Actor 与一次 Episode

CODE

下面的伪代码展示 actor 如何和 environment 互动。注意，actor 输出的是 action distribution，再从中 sample 一个 action。环境返回下一步 observation、reward 和是否结束。

代码
def run_episode(actor, env):
 s = env.reset()
 trajectory = []
 total_reward = 0

 while True:
 probs = actor(s)
 a = sample_from(probs)
 next_s, reward, done = env.step(a)
 trajectory.append((s, a, reward))
 total_reward += reward
 s = next_s
 if done:
 break

 return trajectory, total_reward

下面是“如果已经知道动作好坏分数”的训练形式。下一节 policy gradient 会解释这些分数如何从 return 中估计。

正确示例：用动作分数调整 actorPython-like

现在你应该能做什么：读懂 RL 互动循环，并看出 policy gradient 要补上的正是 score 的来源。
