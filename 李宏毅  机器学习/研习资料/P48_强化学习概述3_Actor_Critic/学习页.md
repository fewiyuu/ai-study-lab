# 本节的概念地图 {#goals}
<!-- tag: MAP -->

Actor-Critic 由两类模型组成。actor 决定 action；critic 评价当前 state 或 state-action 的价值。它提供一个更稳定、更有信息量的评价基准。

本节先定义 value function，再讲两种训练方式：Monte Carlo 直接等 episode 结束后用真实 return 训练；TD 不等整局结束，而是利用 `V(s_t) ≈ r_t + γV(s_{t+1})` 的关系做一步更新。最后把 critic 放回 actor 更新，得到 advantage actor-critic 的直觉。

# Critic 是什么：评估 Actor 的好坏 {#critic}
<!-- tag: CRITIC -->

critic 的工作是评价某个 actor。给定 actor 和当前 observation，critic 预测接下来大概能得到多少 discounted cumulative reward。它像一个评论员：看当前局面，判断这个 actor 后面大概能打出多少分。

critic 有不同形式。有的只看 state，输出 `V(s)`；有的同时看 state 和 action，输出 `Q(s,a)`。P48 重点讲 value function，因为它会直接用作 baseline，并引出 Actor-Critic。

## box info
### Actor
输入 state，输出 action 分布；负责做决定。

## box warn
### Critic
输入 state 或 state-action，输出价值估计；负责评价。

**现在你应该能做什么：**解释 critic 和 actor 的分工，不把 critic 误解成另一个 actor。

# Value Function：某个 Actor 在这个 state 后能拿多少分 {#value}
<!-- tag: VALUE -->

value function 写作 `Vπ(s)` 或课程里的 `V^θ(s)`。上标表示它评估的是哪个 actor / policy。这个细节很重要：同一个游戏画面，对强 actor 可能价值很高；对弱 actor 可能价值很低，因为弱 actor 接下来可能马上失败。

$$
Vπ(s) = E[Gt | st=s, policy=π]
也就是：在 state s 开始，按照 policy π 继续行动，未来 discounted return 的期望值。
$$

value function 的能力是“游戏还没玩完，就估计未来”。如果当前画面还有很多可得分机会，且 actor 很会玩，value 应该高；如果局面快结束或 actor 很差，value 可能低。

**现在你应该能做什么：**说明为什么 value function 必须相对于某个 actor 来定义。


---

# 训练 Critic 方法一：Monte Carlo {#mc}
<!-- tag: MC -->

Monte Carlo 方法最直觉：让 actor 玩完整个 episode，记录它在每个 state 后实际得到的 return。然后把 state 当输入、实际 return 当目标，训练 value function。

**1. 玩一局**actor 与环境互动到结束。
**2. 算 return**对每个 state 算后续累计回馈。
**3. 形成资料**`(s, G)` 是 value 的训练样本。
**4. 回归训练**让 `V(s)` 接近 `G`。

优点是简单、目标来自真实完整结果。缺点是必须等 episode 结束；如果游戏很长，甚至没有自然终止，就很不方便。MC 的估计也可能方差很大，因为一次 sample 的结果可能很受运气影响。

**现在你应该能做什么：**用一句话解释 MC：玩到底，看真实 return，再训练 value。


---

# 训练 Critic 方法二：Temporal Difference {#td}
<!-- tag: TD -->

TD 方法不需要等整局结束。它利用 value function 的递推关系：当前 state 的价值，应该接近当前 reward 加上下一个 state 的折扣价值。

$$
V(st) ≈ rt + γ V(st+1)
TD target 是 r_t + γV(s_{t+1})。它用自己的估计 bootstrap 出当前估计。
$$

TD 的好处是只要看到一步 transition：`(s_t, a_t, r_t, s_{t+1})`，就能更新 value。它适合长 episode 或持续任务。代价是目标包含 `V(s_{t+1})` 这个估计值，可能带来偏差。

**现在你应该能做什么：**写出 TD target，并解释它为什么不用等游戏结束。

# 动手比较：MC 和 TD 可以给出不同答案 {#compare}
<!-- tag: MC VS TD -->

课程中有一个例子：观察到 8 次游戏，`S_B` 出现 8 次，其中 6 次后续 reward 为 1，2 次为 0，所以 `V(S_B)=3/4`。但 `S_A` 只出现一次，并且后面接 `S_B` 时刚好 reward 为 0。MC 会说 `V(S_A)=0`；TD 会借用 `V(S_B)`，说 `V(S_A)=0 + V(S_B)=3/4`。

Monte Carlo只看实际观察到的完整 return。V(SA)=0.00
Temporal Difference用下一状态价值估计当前状态。V(SA)=0.75


SB 成功次数6
γ1.00
TD 会把 SB 的估计传播给 SA；MC 只相信 SA 这次实际结果。

谁对？要看假设。MC 认为实际 sample 就是证据；TD 假设 `S_A` 转到 `S_B` 后，`S_B` 的平均价值仍适用。二者是 bias-variance 取舍的不同表现。

**现在你应该能做什么：**解释为什么 MC 和 TD 都合理，但背后假设不同。

# Critic 怎么帮助训练 Actor {#use}
<!-- tag: ACTOR UPDATE -->

P47 中我们用 `G_t - baseline` 当 action 分数。现在有了 critic，可以把 baseline 设成 `V(s_t)`。直觉是：`V(s_t)` 表示“在这个 state 按照当前 actor 随便 sample action，平均能拿多少”。如果实际采取 `a_t` 后得到的 return 比这个平均值高，就说明这个 action 比一般选择好。

$$
At = Gt - V(st)
这是用 critic 作为 baseline 的 advantage 直觉。
$$

这样比固定 baseline 更细，因为不同 state 的“平均表现”不同。一个危险局面里拿 2 分可能很好；一个大好局面里拿 2 分可能很差。`V(s)` 给每个 state 一个局部基准。

**现在你应该能做什么：**解释为什么 `V(s_t)` 是比常数 baseline 更合理的基准。


---

# Advantage Actor-Critic：用平均减平均 {#advantage}
<!-- tag: ADVANTAGE -->

`G_t - V(s_t)` 仍然使用一次 sampled return `G_t`。课程继续推一步：既然 `G_t` 是 sample，那可以用 `r_t + γV(s_{t+1})` 作为采取 `a_t` 后的期望回馈近似，再减掉 `V(s_t)`。

$$
At ≈ rt + γV(st+1) - V(st)
这个量也是 TD error 的形式。若大于 0，说明该 action 比当前 state 的平均选择好；若小于 0，说明较差。
$$

rt1.0
V(st+1)4.0
V(st)3.0
A = 1.00
调节数值，观察 advantage 正负如何改变动作评价。


r1.0
V next4.0
V current3.0

**现在你应该能做什么：**用 `r + γV(next) - V(current)` 判断某一步 action 是高于还是低于平均。

# 实现小技巧：Actor 和 Critic 共享前几层 {#share}
<!-- tag: SHARED BACKBONE -->

actor 和 critic 的输入常常一样，都是游戏画面或状态特征。若输入是图片，它们都需要前几层 CNN 来理解画面。因此实践中常让 actor 和 critic 共享前面的 backbone，只在最后分成两个 head：actor head 输出每个 action 的分数，critic head 输出一个 scalar value。

**Shared Backbone**抽取状态表示，例如 CNN / MLP / Transformer。
**Two Heads**policy head 输出 action logits；value head 输出 `V(s)`。

共享参数可以省计算，也能让两个任务互相提供表示学习信号。但要注意 actor loss 和 critic loss 的权重平衡；critic 学歪会影响 advantage，actor 学太快也会让 critic 的目标分布变化。

**现在你应该能做什么：**画出一个 Actor-Critic 网络结构：共享层 + policy head + value head。

# 补充：只用 Critic 行动的路线 Q / DQN {#dqn}
<!-- tag: Q LEARNING -->

critic 不一定只辅助 actor。还有一类方法直接学习 `Q(s,a)`，表示在 state `s` 做 action `a` 后的价值，然后选择 `Q` 最大的 action。Deep Q Network，也就是 DQN，就是这个方向的代表。

P48 不展开 DQN，但你要知道 RL 有两大直觉路线：policy-based 方法直接学 actor；value-based 方法学 value / Q，再由价值决定动作。Actor-Critic 则把二者结合：actor 负责策略，critic 提供评价。

| 路线 | 学什么 | 如何行动 |
| --- | --- | --- |
| Policy Gradient | 直接学 policy π(a|s) 。 | 按 actor 输出采样或选择动作。 |
| Actor-Critic | 学 actor，也学 critic。 | actor 行动，critic 评价并稳定训练。 |
| DQN | 学 Q(s,a) 。 | 选 Q 值最高的 action。 |

**现在你应该能做什么：**区分 value function 当 baseline 和 Q function 直接选 action 的差别。

# 最小代码：Actor-Critic 更新骨架 {#code}
<!-- tag: CODE -->

下面是 TD target 训练 critic 的伪代码。critic 学会让 `V(s)` 接近 `r + γV(next)`。

正确示例：TD 训练 criticPython-like

```text
def train_critic_td(critic, s, r, next_s, done, gamma):
 v = critic(s)
 next_v = 0 if done else critic(next_s).detach()
 target = r + gamma * next_v
 value_loss = mse(v, target)
 value_loss.backward()
 update(critic)
```

actor 使用 TD error / advantage 来更新。如果 advantage 为正，提高该 action 的 log probability；为负则降低。

正确示例：用 advantage 更新 actorPython-like

```text
def train_actor(actor, critic, s, a, r, next_s, done, gamma):
 v = critic(s).detach()
 next_v = 0 if done else critic(next_s).detach()
 advantage = r + gamma * next_v - v
 log_prob = log(actor(s)[a])
 policy_loss = -advantage * log_prob
 policy_loss.backward()
 update(actor)
```

错误示例：忘记 value 和 actor 相关Concept bug

```text
# 错误示例：把某个旧 actor 的 V(s) 永久拿来当新 actor 的 baseline。
baseline = old_critic(s)
advantage = current_return - baseline

# value function 评估的是某个 policy；actor 变了，critic 也需要跟着更新。
```

**现在你应该能做什么：**读懂 Actor-Critic 的最小更新，并指出 critic 的目标和 actor 的目标不同。
