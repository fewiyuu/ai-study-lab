# 本节的概念地图 {#goals}
<!-- tag: MAP -->

强化学习最怕的几乎永远没有 reward。延迟回馈至少还能在 episode 结束时拿到输赢；稀疏回馈则可能让早期随机 actor 什么有效事件都没碰到。机械手臂随便挥舞，永远没有把螺丝拴进去；游戏角色随便转圈，永远没有打倒敌人；这时无论 policy gradient 公式多漂亮，样本里都没有可学习的差异。

Reward Shaping 的主线很直接：保留原本目标，但额外给中间行为一些分数，让 agent 先学会“靠近成功的局部动作”。这些分数可能来自人类对任务的理解，例如扣血不好、拿补给好、站着不动不好；也可能来自通用探索原则，例如看到新状态就给一点内在奖励。

目标太远
多数 0 分
动作无差异
加辅助分
引导探索
防止带偏

# 稀疏回馈：没有信号 {#sparse}
<!-- tag: SPARSE REWARD -->

稀疏回馈的直觉是：大多数时间 reward 都是 0，只有非常小概率触发真正成功，才出现正分或负分。这样的问题比“reward 有延迟”更难。下棋至少完整对局后有赢或输；机械手臂把板子插到棍子上，如果一开始随机控制，它可能连正确接近都做不到，episode 结束时仍然全部 0 分。

为什么这会让学习停住？Policy Gradient 需要知道哪些 action 应该提高概率，哪些 action 应该降低概率。如果每条轨迹全是 0，算出的 return、advantage、TD error 都很接近 0，那么“挥手”“停住”“撞墙”“靠近目标”在数据里看起来都差不多。模型不是懒，它是真的没有证据。

即时回馈每一步都有分数，容易判断动作好坏。例如吃到豆子立刻加分。
延迟回馈中途没分，但结尾有输赢。仍然能把结果往前传播。
稀疏回馈很多局连成功事件都没有，数据里几乎没有可传播的结果。

**判断句：**如果早期随机 actor 几乎碰不到任何非零 reward，这不是简单调大训练轮数就一定能解决的问题。

**现在你应该能做什么：**看到一个任务时，先判断它是“有延迟”还是“几乎没有成功样本”。这会决定是否需要探索或额外奖励。

# Reward Shaping：在真正目标旁边加路标 {#shaping}
<!-- tag: SHAPING -->

Reward Shaping 指的是：除了任务原本的 reward，再定义一些辅助 reward 来帮助 agent 学习。任务原本的 reward 才是最后真正想最大化的目标；辅助 reward 是训练时的引导信号。它像把一个很远的终点拆成一段段可见的路标，但路标不能替代终点。

课程里用了“念博士”的例子：如果只有毕业那一刻才有 reward，过程会显得太漫长；中间通过修课、专题、会议论文等阶段性反馈，让人继续往前。机器也是类似：如果机械手臂只有插入成功才加分，它可能很久碰不到成功；如果靠近、对准、抓住、插入这些中间状态也有合理分数，就更容易学到局部动作。

$$
Rtrain(s,a,s') = Rtask(s,a,s') + λ · Raux(s,a,s')
训练时用的 reward 可以包含任务 reward 和辅助 reward。λ 控制辅助信号的强度。
$$

这里最容易犯的错，是把辅助 reward 设计成一个更容易优化、但和原目标不一致的目标。比如“靠近棍子就加分”听起来合理，但如果从侧面撞过去也能变近，机械手臂可能学会撞杆，而不是把板子插上去。

**现在你应该能做什么：**区分“我最终要什么”和“我训练时临时奖励什么”，并检查二者是否真的同向。

# 游戏例子：为什么“站着不动”有时会变成最优 {#game}
<!-- tag: GAME -->

课程中提到第一人称射击游戏的例子。游戏真正的分数通常来自击杀敌人或被敌人击杀：杀敌加分，死亡扣分。可是一开始的 agent 很弱，走出去很容易扣血甚至死亡；相比之下，躲在原地也许至少不会马上扣分。于是从短期数据看，“待在原地”可能比“出去探索”更安全。

这就是 shaping 的用处：扣血就给负分，让 agent 早一点知道“受伤会通往死亡”；捡补给给正分，让它知道资源重要；长时间不动扣分，让它不要把无所作为当成保守策略；适当移动给很小正分，让它愿意离开原地。甚至有些设计会让“每活一段时间扣一点分”，目的避免 agent 永远回避交战，只在边缘绕圈。

| 辅助信号 | 想解决的问题 | 潜在副作用 |
| --- | --- | --- |
| 扣血惩罚 | 死亡太晚才发生，提前提示危险。 | 可能让 agent 过度保守，不敢交战。 |
| 原地惩罚 | 避免待在安全角落什么都不做。 | 可能鼓励无意义移动。 |
| 补给奖励 | 提示药包和弹药对长期得分有帮助。 | 可能变成只捡东西不完成目标。 |
| 存活惩罚 | 逼 agent 不要无限拖延和回避。 | 权重过大可能导致鲁莽冲锋。 |

**现在你应该能做什么：**看到一个 shaping 项时，不只问“它鼓励了什么”，还要问“它会不会被单独最大化”。

# 机械手臂例子：距离近不等于任务完成 {#robot}
<!-- tag: ROBOT -->

机械手臂把板子插到棍子上，是一个典型的稀疏回馈任务。真正成功是板子按正确姿态插入；但随机动作很难碰到完整成功。因此很自然地会想到：让板子离棍子越近，reward 越高。这个想法有直觉价值，但不够。

问题在于“距离变近”不是“插入动作正确”。如果板子从侧面撞向棍子，距离也会变小，却不能完成插入；如果机械手臂把棍子撞歪，距离指标可能短期看起来变好，但长期破坏目标。只用距离作为辅助 reward，会把多维任务压扁成单一数字，容易漏掉姿态、接触方向、速度、稳定性等关键条件。

**好路标**靠近 + 对准 + 正确方向 + 稳定插入。每个中间量都和最终成功同向。
**坏路标**只看距离。侧面撞击、推歪目标、卡在附近也可能拿到分。

**错误示例：**“只要距离变小就加很多分”。错因是把一个几何任务简化成单维距离；后果是 agent 可能学会碰撞和卡位；修法是把距离、方向、姿态、接触稳定性和最终成功一起纳入评价，并让辅助项权重低于真正成功。

**现在你应该能做什么：**设计辅助 reward 时，至少检查它是否遗漏了任务里的关键约束。

# 好奇心奖励：没有外部得分，也鼓励探索新状态 {#curiosity}
<!-- tag: CURIOSITY -->

除了依赖人类手写规则，还有一类方法叫 curiosity-based 或 intrinsic reward。intrinsic reward 可以译作内在奖励：agent 自己因为“学到新东西、看到新状态、预测误差变小”等原因得到的训练信号。

课程里的直觉是：如果外部 reward 太稀疏，甚至完全不给破关分数，那就先奖励 agent 看到新东西。以横向卷轴游戏为例，想看到新画面，角色通常要往右走；于是即使没有告诉它“破关加分”，它也可能因为不断探索新画面而推进关卡。

$$
Rtrain = Rexternal + β · Rintrinsic
外部奖励来自任务；内在奖励来自探索价值，例如新颖性、可学习性或预测误差。
$$

好奇心奖励的价值是通用：人类不必为每个任务都写“扣血、补给、距离、姿态”等细规则。但它也有边界。不是所有“新”都值得追，有些新只是随机变化，不会让 agent 更接近目标。

**现在你应该能做什么：**解释为什么“看到新东西”可以在稀疏任务里提供探索信号，但它不能自动保证学到任务目标。


---

# 噪声陷阱：新东西必须有意义 {#noise}
<!-- tag: NOISY TV -->

好奇心奖励有一个经典问题：如果画面里有随机噪声，噪声每一帧都在变，朴素算法可能觉得它一直在看到新东西。于是 agent 可能站在原地看杂讯，而不是探索地图。这说明“新颖”还不够，真正有用的是可控、可学习、能扩展经验的变化。

更准确地说，好的内在奖励要区分两类未知：第一类是 agent 通过行动可以影响、学习后能预测得更好的未知，例如走进新房间、触发新机关；第二类是纯随机、不可控、学了也没用的未知，例如电视雪花或随机闪烁背景。前者值得探索，后者会消耗学习资源。

新颖性0.70
可控性0.80
随机噪声0.20
内在奖励 = 0.48
新颖且可控时，探索奖励较高。


新颖性0.70
可控性0.80
随机噪声0.20

**错误示例：**“只要画面和上一帧不一样就加分”。错因是把随机变化当成探索；后果是 agent 可能盯着不可控噪声；修法是奖励可控变化、状态覆盖、可学习预测误差下降，减少纯随机因素的吸引力。

**现在你应该能做什么：**判断一个 curiosity reward 是否会被随机噪声骗走。

# 完整知识链路：从没有分数到能学 {#chain}
<!-- tag: CHAIN -->

把 P46-P49 串起来看：actor 与环境互动，得到 reward；policy gradient 或 actor-critic 用 reward 计算动作好坏；如果 reward 太少，动作好坏就算不出来；因此 P49 在 reward 入口处加辅助信号，让后面的训练算法有东西可学。

Reward Shaping 不替代 actor-critic。它发生在更前面：改变训练时看到的 reward。critic 仍然可以学习 value，actor 仍然可以用 advantage 更新，只是 advantage 现在来自更密集的训练信号。好的 shaping 会让早期探索更快触及有效状态；坏的 shaping 会让 critic 认真学习一个错误目标，actor 也会认真优化错误行为。

**1. 原始任务**只有成功或失败才给分，早期全是 0。
**2. 辅助信号**加入靠近、探索、避免原地等中间评价。
**3. 训练算法**用更密集的 reward 算 return、value、advantage。
**4. 行为检查**确认 agent 没有只追辅助分数。

**现在你应该能做什么：**说明 shaping 是在 reward 设计层面解决探索和学习信号问题，而不是一种新的 policy update 公式。

# 最小代码：把辅助奖励加进训练循环 {#code}
<!-- tag: CODE -->

下面的伪代码展示一个基本写法：环境给出任务 reward；我们再根据 transition 计算辅助 reward；训练时使用二者相加。注意，评估最终能力时通常仍要看任务原本分数，否则无法知道 agent 是否真的完成目标。

正确示例：任务奖励加辅助奖励Python-like

```text
def train_step(actor, critic, env, beta):
 state = env.reset()
 done = False
 while not done:
 action = actor.sample(state)
 next_state, task_reward, done, info = env.step(action)

 aux_reward = shaping_reward(state, action, next_state, info)
 train_reward = task_reward + beta * aux_reward

 update_actor_critic(actor, critic, state, action, train_reward, next_state, done)
 state = next_state
```

辅助奖励函数应该读懂状态变化，而不是只看一个容易被钻空子的数。下面这个例子把靠近目标、姿态对准、稳定接触分开计算，并保留真正成功奖励。

正确示例：把中间条件拆开Python-like

```text
def shaping_reward(state, action, next_state, info):
 closer = distance(state.plate, state.peg) - distance(next_state.plate, next_state.peg)
 aligned = cosine_alignment(next_state.plate_axis, next_state.peg_axis)
 gentle_contact = 1.0 if info["stable_contact"] else 0.0
 stuck_penalty = -1.0 if info["hit_from_side"] else 0.0

 return 0.3 * closer + 0.2 * aligned + 0.5 * gentle_contact + stuck_penalty
```

错误示例：辅助奖励取代真正目标Reward bug

```text
# 错误示例：只最大化离目标近，不再关心是否正确插入。
train_reward = -distance(next_state.plate, next_state.peg)

# 后果：agent 可能把板子撞到棍子旁边，距离很近，但任务失败。
```

还有一种更有理论保证的思路叫 potential-based shaping。它给每个 state 一个势能 `Φ(s)`，奖励来自势能变化：往更好的状态移动加分，往更差状态移动扣分。在满足条件时，它更不容易改变原任务的最优策略。

$$
F(s,s') = γΦ(s') - Φ(s)
直觉：不是奖励某个静态状态本身，而是奖励“朝更有希望的状态移动”。
$$

**现在你应该能做什么：**读懂 shaping 在训练代码里的位置，并指出“训练分数”和“最终任务分数”不能混为一谈。
