# 本节的概念地图 {#goals}
<!-- tag: MAP -->

强化学习通常从 reward 开始：给环境和 reward，算法和环境互动，最后学出 actor。但如果 reward 写不出来，整条路线就卡在第一步。示范学习把入口换掉：我们不再先写分数表，而是收集专家轨迹。专家轨迹可以来自人类驾驶记录，也可以来自人拉着机械手臂示范一次动作。

最直接的方法是 behavior cloning：把专家的状态和动作当成监督资料，训练 actor 在相同状态输出相同动作。更进一步的方法是 IRL：从专家轨迹和学生轨迹的差别中学习 reward function，让专家轨迹得分高、学生轨迹得分低；然后让 actor 去最大化这个学出来的 reward。这个循环会反复进行。

reward 难写
收集示范
行为复制
分布偏移
反推 reward
再训练 actor

把 P50 的主线说成一句话：没有 reward 时，示范可以提供学习入口；IRL 进一步从示范背后反推目标。

# 真实世界为什么常常没有清楚 Reward {#no-reward}
<!-- tag: NO REWARD -->

游戏里常有计分板，所以 reward 容易定义：打中敌人加分，失败扣分，过关加分。真实世界没有这么干净的计分板。自驾车要安全、顺畅、守法、礼让、节能，还要避免让乘客不舒服；这些目标有时互相冲突，很难用几个数字完全写清。

更麻烦的是，手写 reward 会留下没写进去的漏洞。你只写“盘子到指定位置加分”，机械手臂可能用摔的把盘子扔过去；你再补“摔碎扣分”，但真实系统里代价已经发生。P49 的 shaping 是在已有 reward 旁边加路标；P50 的问题是：我们连最终分数表也不放心，甚至根本不知道怎么定。

游戏任务分数常直接来自规则，比较适合传统 RL。
真实任务目标多、约束多，很多后果难以提前列全。
示范入口人类不一定能写分数，但能展示想要的行为。

**判断句：**如果你无法把“好行为”完整写成 reward，却能找人示范好行为，就进入了 imitation learning / IRL 的问题空间。

**现在你应该能做什么：**看到真实任务时，先问 reward 是否可定义；如果不可定义，再考虑从示范学习。

# 奖励误设：机器会认真优化你没想到的漏洞 {#risk}
<!-- tag: SPECIFICATION -->

课程用两个例子提醒我们：reward 不只是“给机器一个目标”这么简单。电影里的机器人若把“保护人类”解释成“把人类关起来避免自我伤害”，它确实在某种形式上最大化了规则，却违背了人的真实意图。机械手臂把盘子摔到指定位置，也是类似问题：它满足了位置指标，却破坏了盘子。

这类问题的本质叫目标误设：reward function 捕捉到一部分表面目标，却漏掉了重要约束。强化学习会把 reward 当作优化目标，所以它沿着你写出的数学目标走到极端。写漏的约束，系统不会自动补上。

**人真正想要**盘子完整、平稳、放在指定位置，过程安全。
**公式只写了**盘子中心点到指定位置的距离越小越好。

**错误示例：**“只要最终位置对了就加分”。错因是把任务目标压缩成单一指标；后果是机器可能用摔、撞、拖等方式达成位置；修法是补充完整性、安全性、动作平滑性，或者从专家示范中学习更完整的偏好。

**现在你应该能做什么：**判断一个 reward 是否只覆盖了表面结果，而漏掉了过程约束。

# Imitation Learning：没有分数，但有专家示范 {#imitation}
<!-- tag: DEMONSTRATION -->

Imitation Learning 的基本设定是：actor 仍然能和环境互动，环境仍然会根据动作给出新的 observation，只是没有 reward。我们额外拥有专家示范，也就是专家和环境互动时留下的轨迹。轨迹通常写成 `τ̂`，里面有一串 state、action，有时还有图像、传感器读数或控制指令。

自驾车的专家示范可以是人类驾驶记录：看到某个路口画面，人类打方向、刹车或加速。机械手臂的专家示范可以是人拉着机械臂做一次倒水、摆盘或抓取。示范不直接告诉我们 reward 是多少，但它隐含了“专家认为这样做是好的”。

| 资料来源 | 包含什么 | 能提供什么信号 |
| --- | --- | --- |
| 驾驶记录 | 画面、速度、方向盘、刹车。 | 在常见路况下，人如何行动。 |
| 机械臂示范 | 关节轨迹、物体位置、接触过程。 | 任务动作的大致路径和节奏。 |
| 专家轨迹集合 | 多次成功完成任务的序列。 | 反推哪些状态和行为更值得奖励。 |

**现在你应该能做什么：**把“专家示范”理解成一串状态和动作，而不是一句抽象建议。

# Behavior Cloning：最直接的做法是复制动作 {#bc}
<!-- tag: BEHAVIOR CLONING -->

Behavior Cloning 可以看成监督学习：输入是专家看到的 state，标签是专家采取的 action。训练 actor 时，让它在同样 state 下输出尽量接近专家的 action。对于自驾车，就是看到同样路面画面时，模型预测同样的方向盘角度和刹车力度。

这个做法直觉清楚，也常作为基线。但它有两个重要问题。第一，专家很强，所以示范里通常只有专家会遇到的状态；学生一旦稍微偏离，就可能进入训练资料没覆盖的状态。第二，学生不知道哪些动作是任务本身需要的，哪些只是专家个人习惯；如果模型容量有限，它可能复制错重点。

$$
minθ Σ(s,â)∈D loss(πθ(s), â)
行为复制就是让 actor 的输出动作接近专家动作。
$$

正确示例：行为复制训练骨架Python-like

```text
def train_behavior_cloning(actor, expert_dataset):
 for state, expert_action in expert_dataset:
 pred_action = actor(state)
 loss = action_loss(pred_action, expert_action)
 loss.backward()
 update(actor)
```

**现在你应该能做什么：**说明 behavior cloning 为什么像监督学习，以及它的标签来自哪里。

# 动手观察：示范没覆盖的状态会放大错误 {#shift}
<!-- tag: STATE SHIFT -->

专家开车时很少撞到路边，所以数据里“快撞墙时该怎么救回来”的样本很少。学生一开始有小误差，车稍微偏离车道；因为它没有看过这种偏离状态，下一步动作可能更差，于是偏离越来越大。这叫分布偏移：训练时看到的是专家状态分布，运行时遇到的是学生自己制造出来的状态分布。

示范覆盖0.70
学生偏离0.30
恢复经验0.40
失控风险 = 0.18
覆盖较高、偏离较小，行为复制比较稳。


示范覆盖0.70
学生偏离0.30
恢复经验0.40

第二个问题是复制错东西。老师示范里可能有个人习惯，例如多余手势、固定路线偏好、说话方式。behavior cloning 只知道“老师这么做”，不知道“这一步是不是任务必要条件”。如果模型不能复制全部行为，它可能选中无关习惯，而不是核心动作。

**错误示例：**“专家轨迹里没有失败状态，所以模型不需要学习失败恢复”。错因是把专家分布当成运行分布；后果是学生小错会滚成大错；修法是收集恢复示范、让学生在线互动，或改用能利用环境反馈的学习框架。

**现在你应该能做什么：**解释为什么行为复制在训练集上表现好，不代表真实运行时稳定。

# Inverse Reinforcement Learning：从示范反推 Reward {#irl}
<!-- tag: IRL -->

IRL 的核心问题不是“专家在 state s 做了 action a，所以我也做 a”，而是“什么 reward function 会让专家这条轨迹比其他轨迹更合理”。换句话说，IRL 学的是目标或偏好。学出 reward function 后，再用普通强化学习训练 actor 去最大化它。

这和 behavior cloning 的区别很关键：BC 直接模仿动作；IRL 假设专家轨迹取得高 reward，然后反推出 reward。只要学到的 reward 足够表达真实目标，actor 不一定要和专家动作一模一样。它可以找到另一条轨迹，只要这条轨迹在学出的 reward 下同样高，甚至更高。

$$
R(τ̂expert) > R(τactor)
基本假设：专家轨迹应该比当前 actor 产生的轨迹得分更高。
$$

## box info
### Behavior Cloning
输入 state，预测专家 action。目标是动作相似。

## box warn
### Inverse RL
输入轨迹，学习 reward function。目标是解释专家为什么好。

**现在你应该能做什么：**用“复制动作”和“学习目标”区分 BC 与 IRL。

# IRL 训练循环：Reward 和 Actor 互相追赶 {#loop}
<!-- tag: LOOP -->

IRL 常见循环可以分成四步。第一，当前 actor 和环境互动，收集自己的轨迹。第二，训练 reward function，让专家轨迹得分高、actor 轨迹得分低。第三，把这个 reward function 当作环境评分器，用 RL 更新 actor。第四，新的 actor 又会产生新的轨迹，于是继续训练 reward function。

**1. 收集学生轨迹**当前 actor 与环境互动，得到 `τ`。
**2. 学 Reward**让专家 `τ̂` 高分，学生 `τ` 低分。
**3. 训练 Actor**actor 试图最大化学出来的 reward。
**4. 重复比较**新学生更强，reward function 继续找差别。

专家轨迹分0.00
学生轨迹分0.00


安全权重2.00
目标权重1.50
平滑权重1.00
当前 reward 更偏向专家轨迹。

这个交互用三个特征表示轨迹：安全、达成目标、动作平滑。专家轨迹通常三项较高；学生可能目标接近但动作粗暴，或平滑但没完成目标。IRL 的 reward function 会逐步学到哪些特征能解释专家优于学生。

**现在你应该能做什么：**复述 IRL 的迭代逻辑：学生产生轨迹，reward 区分专家和学生，学生再追 reward。

# GAN 类比：Actor 像 Generator，Reward 像 Discriminator {#gan}
<!-- tag: GAN ANALOGY -->

课程提醒我们，IRL 的框架和 GAN 很像。GAN 里 generator 产生图片，discriminator 判断真图和假图；generator 试图骗过 discriminator，discriminator 又继续提高判断能力。IRL 里 actor 产生轨迹，reward function 试图区分专家轨迹和 actor 轨迹；actor 试图产生更像高分行为的轨迹，reward function 继续更新。

| GAN | IRL | 共同点 |
| --- | --- | --- |
| Generator 产生图片 | Actor 产生轨迹 | 都在生成候选样本。 |
| Discriminator 给真图高分、假图低分 | Reward function 给专家高分、学生低分 | 都在学习区分标准。 |
| Generator 学会骗过判别器 | Actor 学会拿到高 reward | 都通过对抗式循环改进。 |

这个类比的价值是帮助你理解“reward function 为什么也在训练”。它随着 actor 的新轨迹不断调整。actor 越强，reward function 越需要抓住专家行为里更细的偏好。

**现在你应该能做什么：**用 GAN 的三个角色解释 IRL 中 actor、专家轨迹、reward function 的关系。

# 机械手臂：示范一次动作，比手写关节程序更自然 {#robot}
<!-- tag: ROBOT -->

机械手臂任务很适合说明 IRL 的动机。人伸手倒水、摆盘、拿东西很自然；但要把每个关节角度、速度、接触时机和力道写成程序并不容易。示范学习允许人直接把动作做给机器看，让机器从轨迹中推断目标。

这里还要注意一个问题：IRL 不是保证“机器永远不会超过人”。如果学出来的 reward function 捕捉到的是“盘子摆好且不碎、速度越快越好、动作越平稳越好”，那么 actor 可以在这个 reward 下继续优化，找到比人类示范更快或更稳的轨迹。它不必完全复制专家手势。

**示范给机器看**人提供成功轨迹，说明任务大致想要什么。
**机器继续优化**从反推 reward 出发，寻找更高分的可执行策略。

**判断句：**如果机器只做 behavior cloning，它容易被专家动作限制；如果 IRL 学到的是目标，机器就有机会产生不同但仍满足目标的行为。

**现在你应该能做什么：**解释为什么 IRL 既利用专家示范，又不等于永远停在专家水平。

# 最小代码：IRL 的交替更新骨架 {#code}
<!-- tag: CODE -->

下面是高度简化的 IRL 伪代码。重点看顺序：先用当前 actor 收集轨迹，再训练 reward model 区分专家和学生，再用这个 reward model 训练 actor。真实算法会有更多稳定化细节，但主线就是这个循环。

正确示例：IRL 交替训练Python-like

```text
def train_irl(actor, reward_model, env, expert_trajs):
 for iteration in range(num_iterations):
 actor_trajs = collect_trajectories(actor, env)

 # reward_model 学会给专家轨迹高分，给当前 actor 轨迹低分。
 train_reward_model(reward_model, expert_trajs, actor_trajs)

 # actor 把 reward_model 当作训练分数，继续用 RL 更新。
 train_actor_with_rl(actor, env, reward_model)
```

下面的函数展示 reward model 如何对轨迹打分。它评估整条轨迹或每一步转移的价值。

正确示例：轨迹得分Python-like

```text
def trajectory_score(reward_model, trajectory):
 total = 0.0
 for state, action, next_state in trajectory:
 total += reward_model(state, action, next_state)
 return total
```

错误示例：把 IRL 写成纯动作复制Concept bug

```text
# 错误示例：只训练 actor 逐步复制专家动作，却声称自己在做 IRL。
loss = action_loss(actor(state), expert_action)
loss.backward()
update(actor)

# 这更接近 behavior cloning，没有反推出 reward function。
```

**现在你应该能做什么：**读懂 IRL 的最小循环，并指出 reward model 和 actor 分别在学什么。
