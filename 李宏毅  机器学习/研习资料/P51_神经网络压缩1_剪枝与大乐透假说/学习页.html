P51｜神经网络压缩 1：剪枝与大乐透假说

大型模型效果好，但不一定适合部署在手表、手机、车载传感器和其他边缘设备上。设备内存有限，算力有限，还可能要求低延迟和本地隐私保护。Network Compression 的目标，是让模型更小、更快、更省资源，同时尽量保持原本效果。

为什么要做 Network Compression

MOTIVATION

Network Compression 是把神经网络变小、变快、变省资源的一组方法。课程强调的是软件层面的压缩，不讨论专门硬体加速。它的典型使用场景是 resource-constrained environment，也就是资源受限环境，例如智能手表、手机、IoT 设备、车载传感器。

Memory模型参数太多，设备放不下，或占用太多存储。

Computing Power设备算力有限，大模型推理太慢或耗电。

Latency / Privacy不能总靠云端，因为传输有延迟，也可能暴露数据。

Pruning 基本流程：先长大，再修剪

PRUNING

Pruning 的直觉来自修剪树枝：大网络里不是每个参数都重要，有些参数可能绝对值很小，或者对应神经元很少激活，对任务贡献不大。剪枝就是找到这些冗余部分，把它们移除或置零，让网络更小。

1. Train先训练一个表现好的大网络。

2. Score估计参数或神经元的重要性。

3. Prune剪掉低重要性的部分。

4. Fine-tune用训练资料微调，恢复效果。

现在你应该能做什么：按顺序复述剪枝流程，并解释为什么常常要多轮进行。

如何判断谁不重要

IMPORTANCE

最简单的重要性指标是权重绝对值。若一个权重接近 0，它对矩阵乘法输出的影响也可能较小，因此可以优先剪掉。这个方法简单、便宜、常见，但不是唯一方法。也可以用梯度、二阶信息、参数改变幅度、神经元激活频率等指标评估重要性。

指标直觉限制
绝对值大小权重越接近 0，单步影响越小。忽略数据分布和组合关系。
激活频率神经元很少输出非零，可能没参与任务。不同数据集上活跃度可能不同。
梯度或敏感度删除后 loss 变化小，说明影响小。计算成本更高，也可能依赖近似。

现在你应该能做什么：说出至少三种重要性估计方法，并知道它们只是剪枝候选，不是最终证明。

剪什么：参数级与神经元级

UNIT

剪枝可以以参数为单位，也可以以神经元、通道或整个结构为单位。参数级剪枝也叫 unstructured pruning：每个权重单独判断，可能剪掉矩阵中的任意元素。这样 sparsity 很高，理论上参数少很多，但网络结构会变得不规则。

参数级剪枝

优点：可以剪很多，sparsity 高。缺点：结构不规则，实际加速难。

神经元级剪枝

优点：结构规则，容易改维度和加速。缺点：剪枝粒度粗，可能更影响准确率。

现在你应该能做什么：区分“把矩阵里零散元素置零”和“把整个神经元/通道删掉”的差别。

动手观察：少参数不等于真的快

SPARSITY

课程中特别强调：如果你把 95% 的 weight 设为 0，模型参数看起来稀疏，但不一定真的加速。原因是通用 GPU 擅长规则矩阵乘法；不规则稀疏结构需要额外索引和特殊 kernel。如果框架没有有效支持，甚至可能比原来更慢。

稀疏率0.60

框架支持0.35

结构规则0.20

稀疏率高，但结构不规则、支持不足时，加速有限。

错误示例：“95% 参数为 0，所以推理一定快 20 倍”。错因是把数学稀疏率等同于硬体加速；后果是部署时延迟不降反升；修法是实际测量 latency、memory、throughput，并确认框架能利用稀疏结构。

为什么不一次剪完：损伤和恢复

ITERATIVE

剪枝会造成损伤。即使被剪的是低重要性参数，删除很多连接后，网络的函数也会变化，准确率通常会下降。Fine-tuning 的作用是让剩下的参数重新适应，弥补被剪掉的部分。小幅剪枝后微调，网络还有机会恢复；大幅剪枝后，功能路径可能被破坏，恢复就困难。

$$ Train → Prune p% → Fine-tune → Re-score → Prune p% → ... 每轮剪一点，给网络恢复机会，也让重要性评估跟着新结构更新。 $$

现在你应该能做什么：解释为什么剪枝比例、微调轮数和重新评估都是关键超参数。

大乐透假说：大网络像买了很多张彩票

LOTTERY

剪枝带来一个自然问题：既然剪完后的小网络也能表现很好，为什么不一开始就训练同样大小的小网络？常见观察是，直接训练小网络往往不如“先训练大网络再剪小”的结果。大乐透假说试图解释这一点。

直接训练小网络只买一张彩票。若初始化不好，可能训练不起来。

先大后剪大网络里有很多子网络，相当于买了很多张彩票。

实验思路：保留哪一种初始化

EVIDENCE

大乐透假说的经典实验流程是：先随机初始化大网络，训练它；然后剪枝得到一个小子网络。接下来比较两种重新训练方式。第一种是给这个小结构重新随机初始化；第二种是把它在大网络初始时对应的那些初始权重拿回来。观察结果通常是：保留原始初始化的子网络更容易训练好。

1. 初始大网络记录随机初始化 w0。

2. 训练并剪枝得到 surviving mask。

3. 随机小网络同结构但新随机初始化，可能较差。

4. 中奖小网络同结构且取回原初始值，可能较好。

争议边界：大乐透假说不是万能解释

DEBATE

课程也提醒：大乐透假说是 hypothesis，不是完全定论。另一些研究指出，直接训练小网络如果给更多 epoch、调整 learning rate，也可能达到或超过剪枝网络的表现。也就是说，“小网络训练不起来”有时可能只是训练设置不公平，而不是小网络本身不行。

结论要追问什么原因
剪枝后小网络很好直接小网络训练了多久？学习率一样吗？训练预算不公平会制造假象。
95% sparsity是否真正减少 latency 和 memory？不规则稀疏可能无法加速。
找到中奖子网络是否依赖某种初始化和剪枝策略？现象可能有条件，不是普遍定律。

现在你应该能做什么：对大乐透假说保持准确态度：它是重要解释，但需要看实验条件和对照。

最小代码：非结构化剪枝与结构化剪枝

CODE

下面是最简单的 magnitude pruning：按权重绝对值选阈值，把小权重置零。它容易实现，但如果只是 mask 掉权重，模型形状没有改变，部署时不一定变快。

代码
def unstructured_prune(weight, sparsity):
 scores = abs(weight)
 threshold = quantile(scores, sparsity)
 mask = scores > threshold
 pruned_weight = weight * mask
 return pruned_weight, mask

结构化剪枝会删掉整个神经元或通道。下面以线性层为例：保留部分输出神经元后，下一层的输入维度也要同步调整。这更麻烦，但更可能带来实际速度收益。
