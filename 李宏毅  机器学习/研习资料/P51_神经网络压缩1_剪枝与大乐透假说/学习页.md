# 本节的概念地图 {#goals}
<!-- tag: MAP -->

这一节从部署问题开始：为什么我们需要把模型变小？答案模型要进入资源受限环境。云端计算可以很强，但把资料传到云端会有延迟，也会带来隐私问题。边缘设备需要本地计算时，大模型就可能太慢或放不下。

剪枝的主线是：先训练一个大网络，然后估计每个参数或神经元的重要性，剪掉不重要的部分，再微调剩余网络恢复准确率。这个过程可以多轮进行。最后我们讨论一个有趣问题：既然剪完后小网络也能表现不错，为什么不一开始就训练小网络？大乐透假说给出一种解释：大网络里包含很多子网络，其中可能有“中奖”的初始化子网络。

# 为什么要做 Network Compression {#why}
<!-- tag: MOTIVATION -->

Network Compression 是把神经网络变小、变快、变省资源的一组方法。课程强调的是软件层面的压缩，不讨论专门硬体加速。它的典型使用场景是 resource-constrained environment，也就是资源受限环境，例如智能手表、手机、IoT 设备、车载传感器。

有人会问：为什么不把数据上传云端，让云端模型计算完再回传？第一个理由是 latency，延迟。自驾车传感器需要立刻反应，网络往返时间可能无法接受。第二个理由是 privacy，隐私。用户的声音、位置、健康数据或图像不一定适合传到云端。本地推理可以减少数据外泄。

Memory模型参数太多，设备放不下，或占用太多存储。
Computing Power设备算力有限，大模型推理太慢或耗电。
Latency / Privacy不能总靠云端，因为传输有延迟，也可能暴露数据。

**现在你应该能做什么：**看到“模型压缩”时，不只想到参数少，也能说出部署侧的真实动机。

# Pruning 基本流程：先长大，再修剪 {#pruning}
<!-- tag: PRUNING -->

Pruning 的直觉来自修剪树枝：大网络里不是每个参数都重要，有些参数可能绝对值很小，或者对应神经元很少激活，对任务贡献不大。剪枝就是找到这些冗余部分，把它们移除或置零，让网络更小。

典型流程：训练大网络，计算重要性，剪掉一部分，微调剩下参数，重新评估，再剪下一轮。原因很实际：一次剪太多可能造成准确率大幅下降，后续微调也救不回来；逐步剪枝能让网络有时间适应结构变化。

**1. Train**先训练一个表现好的大网络。
**2. Score**估计参数或神经元的重要性。
**3. Prune**剪掉低重要性的部分。
**4. Fine-tune**用训练资料微调，恢复效果。

**判断句：**剪枝不是“删完就交付”，删完后的微调和评估同样是方法的一部分。

**现在你应该能做什么：**按顺序复述剪枝流程，并解释为什么常常要多轮进行。

# 如何判断谁不重要 {#importance}
<!-- tag: IMPORTANCE -->

最简单的重要性指标是权重绝对值。若一个权重接近 0，它对矩阵乘法输出的影响也可能较小，因此可以优先剪掉。这个方法简单、便宜、常见，但不是唯一方法。也可以用梯度、二阶信息、参数改变幅度、神经元激活频率等指标评估重要性。

如果剪枝单位是神经元，可以观察它在数据上的输出。一个 ReLU 神经元若大多数时候输出 0，就可能是低活跃神经元；一个通道若对后续层贡献很小，也可能被剪掉。但要注意，神经网络有组合效应，单个参数小不一定绝对没用；两个小参数一起可能构成某个路径。因此剪枝后必须验证。

| 指标 | 直觉 | 限制 |
| --- | --- | --- |
| 绝对值大小 | 权重越接近 0，单步影响越小。 | 忽略数据分布和组合关系。 |
| 激活频率 | 神经元很少输出非零，可能没参与任务。 | 不同数据集上活跃度可能不同。 |
| 梯度或敏感度 | 删除后 loss 变化小，说明影响小。 | 计算成本更高，也可能依赖近似。 |

**现在你应该能做什么：**说出至少三种重要性估计方法，并知道它们只是剪枝候选，不是最终证明。

# 剪什么：参数级与神经元级 {#unit}
<!-- tag: UNIT -->

剪枝可以以参数为单位，也可以以神经元、通道或整个结构为单位。参数级剪枝也叫 unstructured pruning：每个权重单独判断，可能剪掉矩阵中的任意元素。这样 sparsity 很高，理论上参数少很多，但网络结构会变得不规则。

神经元级或通道级剪枝更接近 structured pruning：一次删掉整个神经元、整行、整列或整组通道。这样压缩比例可能没那么极端，但结构仍然规则。对 PyTorch 这类框架来说，规则结构更容易直接改 layer 的输入输出维度，也更容易被 GPU 的矩阵乘法利用。

## box info
### 参数级剪枝
优点：可以剪很多，sparsity 高。缺点：结构不规则，实际加速难。

## box warn
### 神经元级剪枝
优点：结构规则，容易改维度和加速。缺点：剪枝粒度粗，可能更影响准确率。

**现在你应该能做什么：**区分“把矩阵里零散元素置零”和“把整个神经元/通道删掉”的差别。

# 动手观察：少参数不等于真的快 {#speed}
<!-- tag: SPARSITY -->

课程中特别强调：如果你把 95% 的 weight 设为 0，模型参数看起来稀疏，但不一定真的加速。原因是通用 GPU 擅长规则矩阵乘法；不规则稀疏结构需要额外索引和特殊 kernel。如果框架没有有效支持，甚至可能比原来更慢。

理论参数减少0%
预期速度收益1.00x


稀疏率0.60
框架支持0.35
结构规则0.20
稀疏率高，但结构不规则、支持不足时，加速有限。

如果被剪掉的权重只是用 mask 乘 0，参数仍可能存在内存里，计算仍可能走密集矩阵乘法。这时你只是“逻辑上觉得模型小了”，实际部署可能没有减少内存或延迟。要得到真实收益，需要结构化剪枝、稀疏计算库，或专门硬体/框架支持。

**错误示例：**“95% 参数为 0，所以推理一定快 20 倍”。错因是把数学稀疏率等同于硬体加速；后果是部署时延迟不降反升；修法是实际测量 latency、memory、throughput，并确认框架能利用稀疏结构。

**现在你应该能做什么：**看到 sparsity 数字时，会追问是否有真实 speed-up，而不是直接相信参数比例。

# 为什么不一次剪完：损伤和恢复 {#iterative}
<!-- tag: ITERATIVE -->

剪枝会造成损伤。即使被剪的是低重要性参数，删除很多连接后，网络的函数也会变化，准确率通常会下降。Fine-tuning 的作用是让剩下的参数重新适应，弥补被剪掉的部分。小幅剪枝后微调，网络还有机会恢复；大幅剪枝后，功能路径可能被破坏，恢复就困难。

迭代剪枝的思路是控制每次损伤程度。比如每轮剪掉 10% 最不重要的参数，微调若干 epoch，再重新计算重要性。重新计算很重要，因为剪掉一部分后，剩余参数的角色会变化；原来不重要的参数可能变得重要，原来重要的路径可能被替代。

$$
Train → Prune p% → Fine-tune → Re-score → Prune p% → ...
每轮剪一点，给网络恢复机会，也让重要性评估跟着新结构更新。
$$

**现在你应该能做什么：**解释为什么剪枝比例、微调轮数和重新评估都是关键超参数。

# 大乐透假说：大网络像买了很多张彩票 {#lottery}
<!-- tag: LOTTERY -->

剪枝带来一个自然问题：既然剪完后的小网络也能表现很好，为什么不一开始就训练同样大小的小网络？常见观察是，直接训练小网络往往不如“先训练大网络再剪小”的结果。大乐透假说试图解释这一点。

大乐透假说的直觉是：一个大网络可以看成包含很多子网络。随机初始化就像抽彩票，不是每个子网络都能被训练好；但大网络里子网络数量多，出现“中奖子网络”的概率更高。训练大网络再剪枝，可能是在把那个幸运子网络找出来。这个子网络如果保留它原来的初始化，重新训练时也能成功。

**直接训练小网络**只买一张彩票。若初始化不好，可能训练不起来。
**先大后剪**大网络里有很多子网络，相当于买了很多张彩票。

**现在你应该能做什么：**用“中奖子网络”解释为什么大网络剪出来的小网络可能比随机小网络更好。

# 实验思路：保留哪一种初始化 {#evidence}
<!-- tag: EVIDENCE -->

大乐透假说的经典实验流程是：先随机初始化大网络，训练它；然后剪枝得到一个小子网络。接下来比较两种重新训练方式。第一种是给这个小结构重新随机初始化；第二种是把它在大网络初始时对应的那些初始权重拿回来。观察结果通常是：保留原始初始化的子网络更容易训练好。

这个结果说明，重要的不只是“结构小不小”，还包括“这组子网络的初始化是不是幸运”。后续研究甚至发现，在某些实验里，初始权重的正负号可能比具体绝对值更关键：只保留正负号，用固定大小替代数值，也能训练出不错结果。这也是课程提到的 sign 双关。

**1. 初始大网络**记录随机初始化 `w0`。
**2. 训练并剪枝**得到 surviving mask。
**3. 随机小网络**同结构但新随机初始化，可能较差。
**4. 中奖小网络**同结构且取回原初始值，可能较好。

**现在你应该能做什么：**说清楚“重新随机初始化小网络”和“用大网络原始初始化的小网络”为什么不是同一件事。


---

# 争议边界：大乐透假说不是万能解释 {#debate}
<!-- tag: DEBATE -->

课程也提醒：大乐透假说是 hypothesis，不是完全定论。另一些研究指出，直接训练小网络如果给更多 epoch、调整 learning rate，也可能达到或超过剪枝网络的表现。也就是说，“小网络训练不起来”有时可能只是训练设置不公平，而不是小网络本身不行。

还有研究认为，大乐透现象可能只在某些条件下明显，例如学习率较小、非结构化剪枝、特定数据集或模型规模。结构化剪枝、较大学习率或更充分训练时，结论可能变化。因此你在读剪枝论文或做实验时，不能只看最终 accuracy，还要看训练预算、学习率、剪枝单位、初始化方式、硬体加速和实验对照是否公平。

| 结论 | 要追问什么 | 原因 |
| --- | --- | --- |
| 剪枝后小网络很好 | 直接小网络训练了多久？学习率一样吗？ | 训练预算不公平会制造假象。 |
| 95% sparsity | 是否真正减少 latency 和 memory？ | 不规则稀疏可能无法加速。 |
| 找到中奖子网络 | 是否依赖某种初始化和剪枝策略？ | 现象可能有条件，不是普遍定律。 |

**现在你应该能做什么：**对大乐透假说保持准确态度：它是重要解释，但需要看实验条件和对照。

# 最小代码：非结构化剪枝与结构化剪枝 {#code}
<!-- tag: CODE -->

下面是最简单的 magnitude pruning：按权重绝对值选阈值，把小权重置零。它容易实现，但如果只是 mask 掉权重，模型形状没有改变，部署时不一定变快。

正确示例：按权重绝对值剪枝Python-like

```text
def unstructured_prune(weight, sparsity):
 scores = abs(weight)
 threshold = quantile(scores, sparsity)
 mask = scores > threshold
 pruned_weight = weight * mask
 return pruned_weight, mask
```

结构化剪枝会删掉整个神经元或通道。下面以线性层为例：保留部分输出神经元后，下一层的输入维度也要同步调整。这更麻烦，但更可能带来实际速度收益。

正确示例：按神经元保留输出维度Python-like

```text
def structured_prune_linear(linear, keep_indices):
 new_out = len(keep_indices)
 new_layer = Linear(linear.in_features, new_out)
 new_layer.weight = linear.weight[keep_indices, :]
 new_layer.bias = linear.bias[keep_indices]
 return new_layer
```

错误示例：只置零就宣称模型变小Deployment bug

```text
# 错误示例：参数仍保存在 dense tensor 中，推理仍走密集矩阵乘法。
weight = weight * mask
print("模型已经压缩 95%，一定快很多")

# 正确检查：保存大小、内存占用、真实延迟、硬体支持。
```

**现在你应该能做什么：**读懂剪枝代码里的 mask，并指出 mask 不等于真实部署压缩。
