P52｜神经网络压缩 2：多面向压缩

P51 讲了 pruning：先训练大模型，再剪掉不重要的部分。P52 继续补全模型压缩谱系：可以让小模型向大模型学，叫 Knowledge Distillation；可以减少每个参数占用的 bit，叫 Quantization；可以把卷积结构改成更省参数的 Depthwise Separable Convolution；也可以让模型根据设备或样本难度动态调整计算量。

Knowledge Distillation：小模型向大模型学

DISTILLATION

Knowledge Distillation 通常先训练一个大的 teacher network，再训练一个小的 student network。student 学习 teacher 的输出分布。以手写数字为例，老师看到某张图后可能输出：1 的概率 0.70，7 的概率 0.20，9 的概率 0.10。学生要逼近这个分布。

硬标签

[1,0,0]。只告诉学生正确类是哪一个。

软标签

[0.70,0.20,0.10]。同时告诉学生类别之间的相似性。

现在你应该能做什么：解释为什么学生学习老师对类别关系的判断。

Temperature Softmax：把老师输出变软

TEMPERATURE

如果老师模型非常自信，softmax 后最大类接近 1，其他类接近 0，那学生跟老师学和跟硬标签学差别不大。Temperature 的作用是在 softmax 前把 logits 除以一个温度 T。当 T>1 时，分布会变平滑，非最大类别也会保留一些概率。

class 10.00

class 70.00

class 90.00

T = 1.0

较低温度让分布尖锐，像硬标签。

错误示例：“T 越大越好”。错因是忽略温度过高会把所有类别推得太平均；后果是学生学不到老师真正区分；修法是把 T 当作超参数，配合验证集调。

把 Ensemble 压成一个 Student

ENSEMBLE

Teacher 不一定是单一大模型，也可以是多个模型的 ensemble。Ensemble 的直觉是训练多个模型，把输出平均或投票，通常可以提升比赛分数。但实际部署时，如果每次推理都跑 100 个模型，计算量太大。

做法优点代价
直接部署 ensemble效果可能最好。推理成本和存储成本高。
蒸馏到 student部署成本低，保留软信息。学生不一定完全复制 ensemble 能力。

现在你应该能做什么：解释为什么 ensemble 适合当老师，但不适合直接部署到资源受限设备。

Parameter Quantization：减少每个参数的存储位数

QUANTIZATION

量化关注的“每个参数用多少 bit 存”。如果原来用 32-bit float 存一个权重，改成 16-bit、8-bit，模型大小就会明显下降。量化还可能提高推理速度，因为某些硬体对低精度计算更友好。

原始模型大小0 MB

量化后大小0 MB

现在你应该能做什么：计算参数量固定时，bit 数减少会怎样影响模型存储空间。

Weight Clustering 与 Huffman Encoding

CLUSTER

Weight Clustering 是进一步压缩权重表示：把数值相近的权重分到同一群，每一群只用一个代表值。存储时不再保存每个参数的完整数值，而是保存两个东西：一个小表记录每个 cluster 的代表值；另一个表记录每个权重属于哪一群。

1. 训练权重得到连续数值的参数。

2. 分群相近数值放同一 cluster。

3. 代表值每群用平均值或中心表示。

4. 编码存群编号，必要时再做可变长编码。

Binary Network：每个权重只有两个可能

BINARY

压缩到极致时，可以让每个权重只有两个值，例如 +1 和 -1。这样每个权重只需要 1 bit。直觉上这听起来很粗糙，但一些 binary network 实验显示，它并不一定表现很差，在某些数据集上甚至可能因为容量受限而减少过拟合。

优点极低存储，某些硬体上乘法可简化。

风险表达能力受限，训练需要特殊技巧，复杂任务可能掉点。

Depthwise Separable Convolution：把空间和通道拆开

DEPTHWISE

普通卷积的 filter 是立方体：kernel 大小为 K×K，输入通道数为 I，输出通道数为 O，参数量是 K×K×I×O。它同时处理空间关系和通道混合。

$$ 普通卷积：K²IO　｜　Depthwise + Pointwise：K²I + IO 当 K=3、O 很大时，参数量约变成原来的 1/9 左右。 $$

普通卷积参数72

可分离卷积参数26

错误示例：“depthwise 卷积就是普通 CNN 的原始算法”。错因是把特殊变体当作标准卷积；标准卷积每个 filter 会跨所有输入通道，depthwise 则每个 filter 只负责一个通道，必须再用 pointwise 混合通道。

低秩近似：把一层拆成两层

LOW

低秩近似的直觉是：一个大的权重矩阵 W 参数太多，可以用两个较小矩阵 U 和 V 的乘积来近似。原来从 N 个输入到 M 个输出，需要 MN 个参数；若中间插入一个很窄的瓶颈维度 K，参数量变成 MK + KN。当 K 远小于 M,N 时，参数会少很多。

$$ WM×N ≈ UM×K VK×N，参数：MN → K(M+N) K 越小，压缩越强，但表达限制也越强。 $$

现在你应该能做什么：解释为什么“一层拆两层”有时反而参数更少。

Dynamic Computation：一个模型，多种计算预算

DYNAMIC

前面的方法主要是把模型本身变小。Dynamic Computation 的目标不同：希望同一个模型可以根据设备资源或样本难度，调整实际使用的计算量。电量少时少算一点，电量足时多算一点；简单图片早退出，困难图片走完整网络。

动态深度

中间层也能输出，简单样本早退出，难样本继续算。

动态宽度

同一个网络按预算使用不同数量的通道或神经元。

现在你应该能做什么：说明 dynamic computation 让推理成本随情境变化。

最小代码：蒸馏与可分离卷积

CODE

蒸馏训练通常同时包含两个 loss：一个让学生贴近老师软分布，一个让学生仍然尊重真实标签。下面只展示核心结构。

代码
def distill_step(teacher, student, x, y, T, alpha):
 teacher_prob = softmax(teacher(x) / T).detach()
 student_log_prob = log_softmax(student(x) / T)
 distill_loss = kl_div(student_log_prob, teacher_prob) * (T * T)
 hard_loss = cross_entropy(student(x), y)
 loss = alpha * distill_loss + (1 - alpha) * hard_loss
 loss.backward()
 update(student)

Depthwise separable convolution 在代码里通常就是一个 depthwise convolution 后接一个 1x1 convolution。

正确示例：Depthwise + PointwisePython-like

错误示例：只用 depthwise 不混合通道Architecture bug

现在你应该能做什么：读懂蒸馏 loss 的两个来源，以及可分离卷积为什么需要两步。
