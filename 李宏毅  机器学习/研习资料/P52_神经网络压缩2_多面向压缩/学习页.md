# 本节的概念地图 {#goals}
<!-- tag: MAP -->

模型压缩可以从多个面向下手。你可以换训练目标：让小模型跟大模型学；可以换数值表示：同样数量的参数，用更少 bit 存；可以换网络结构：从设计上减少参数和乘加；可以换推理策略：容易样本少算几层，困难样本多算几层。

这些方法各有代价。蒸馏依赖老师模型质量；量化会损失数值精度；二值网络限制更强，可能反而减少过拟合；depthwise separable 卷积把空间关系和通道关系拆开，参数大幅减少，但表达能力也被约束；动态计算需要训练多个出口或多种宽度，让同一个模型适应不同预算。

# Knowledge Distillation：小模型向大模型学 {#distill}
<!-- tag: DISTILLATION -->

Knowledge Distillation 通常先训练一个大的 teacher network，再训练一个小的 student network。student 学习 teacher 的输出分布。以手写数字为例，老师看到某张图后可能输出：1 的概率 0.70，7 的概率 0.20，9 的概率 0.10。学生要逼近这个分布。

为什么这比硬标签更有信息量？硬标签只说“这是一”，其他类别全是 0；老师分布会告诉学生“它主要像 1，但也有点像 7 和 9”。这叫 dark knowledge，暗知识，也就是类别之间的相似关系。小模型容量有限，直接学硬标签可能太难；跟老师学软分布，目标更平滑，训练也更有指导。

## box info
### 硬标签
`[1,0,0]`。只告诉学生正确类是哪一个。

## box warn
### 软标签
`[0.70,0.20,0.10]`。同时告诉学生类别之间的相似性。

**现在你应该能做什么：**解释为什么学生学习老师对类别关系的判断。

# Temperature Softmax：把老师输出变软 {#temperature}
<!-- tag: TEMPERATURE -->

如果老师模型非常自信，softmax 后最大类接近 1，其他类接近 0，那学生跟老师学和跟硬标签学差别不大。Temperature 的作用是在 softmax 前把 logits 除以一个温度 `T`。当 `T>1` 时，分布会变平滑，非最大类别也会保留一些概率。

$$
softmaxT(z)i = exp(zi/T) / Σj exp(zj/T)
T 越大，分布越平；T 太大时所有类别接近平均，信息也会被冲淡。
$$

class 10.00
class 70.00
class 90.00
T = 1.0
较低温度让分布尖锐，像硬标签。


温度 T1
使用 logits `[100,10,1]` 观察温度变化。排序不变，但分布形状会改变。

**错误示例：**“T 越大越好”。错因是忽略温度过高会把所有类别推得太平均；后果是学生学不到老师真正区分；修法是把 T 当作超参数，配合验证集调。

**现在你应该能做什么：**说明 temperature 不改变类别排序，但会改变学生能看到的软信息。

# 把 Ensemble 压成一个 Student {#ensemble}
<!-- tag: ENSEMBLE -->

Teacher 不一定是单一大模型，也可以是多个模型的 ensemble。Ensemble 的直觉是训练多个模型，把输出平均或投票，通常可以提升比赛分数。但实际部署时，如果每次推理都跑 100 个模型，计算量太大。

蒸馏可以把 ensemble 的输出当作 teacher distribution，让一个小 student 学会接近 ensemble 的行为。这样部署时只跑一个学生模型，却尽量保留多个老师平均后的效果。这个用法很常见：把难以部署的高精度系统压缩成可部署模型。

| 做法 | 优点 | 代价 |
| --- | --- | --- |
| 直接部署 ensemble | 效果可能最好。 | 推理成本和存储成本高。 |
| 蒸馏到 student | 部署成本低，保留软信息。 | 学生不一定完全复制 ensemble 能力。 |

**现在你应该能做什么：**解释为什么 ensemble 适合当老师，但不适合直接部署到资源受限设备。

# Parameter Quantization：减少每个参数的存储位数 {#quant}
<!-- tag: QUANTIZATION -->

量化关注的“每个参数用多少 bit 存”。如果原来用 32-bit float 存一个权重，改成 16-bit、8-bit，模型大小就会明显下降。量化还可能提高推理速度，因为某些硬体对低精度计算更友好。

但量化会牺牲数值精度。是否可接受取决于任务、模型、硬体和训练方式。有时低精度会像一种正则化，反而减轻过拟合；但过度量化也可能让模型表达能力不足。关键是测准确率和真实部署指标。

bit 数越少，存储越省，但数值精度越低。

**现在你应该能做什么：**计算参数量固定时，bit 数减少会怎样影响模型存储空间。

# Weight Clustering 与 Huffman Encoding {#cluster}
<!-- tag: CLUSTER -->

Weight Clustering 是进一步压缩权重表示：把数值相近的权重分到同一群，每一群只用一个代表值。存储时不再保存每个参数的完整数值，而是保存两个东西：一个小表记录每个 cluster 的代表值；另一个表记录每个权重属于哪一群。

如果只分 4 群，每个权重只需要 2 bit 表示群编号。再进一步，可以用 Huffman Encoding：出现频率高的群用更短编码，出现频率低的群用更长编码，从平均意义上继续减少存储。注意，这些方法主要节省存储；是否加速取决于解码和硬体支持。

**1. 训练权重**得到连续数值的参数。
**2. 分群**相近数值放同一 cluster。
**3. 代表值**每群用平均值或中心表示。
**4. 编码**存群编号，必要时再做可变长编码。

**现在你应该能做什么：**说明 weight clustering 节省的是“每个参数的数值表示”，不是减少参数个数。

# Binary Network：每个权重只有两个可能 {#binary}
<!-- tag: BINARY -->

压缩到极致时，可以让每个权重只有两个值，例如 +1 和 -1。这样每个权重只需要 1 bit。直觉上这听起来很粗糙，但一些 binary network 实验显示，它并不一定表现很差，在某些数据集上甚至可能因为容量受限而减少过拟合。

不过二值网络的训练并不等于直接把所有权重四舍五入。通常需要特殊训练技巧，让前向传播使用二值权重，同时保留可训练的实值影子权重或使用近似梯度。课程没有展开训练细节，你要记住的是：二值化是极端量化，它大幅减少存储，也强烈限制模型表达能力。

**优点**极低存储，某些硬体上乘法可简化。
**风险**表达能力受限，训练需要特殊技巧，复杂任务可能掉点。

**现在你应该能做什么：**解释为什么 binary weight 不一定总是很差，但也不能无脑替换所有模型。

# Depthwise Separable Convolution：把空间和通道拆开 {#depthwise}
<!-- tag: DEPTHWISE -->

普通卷积的 filter 是立方体：kernel 大小为 `K×K`，输入通道数为 `I`，输出通道数为 `O`，参数量是 `K×K×I×O`。它同时处理空间关系和通道混合。

Depthwise Separable Convolution 把这件事拆成两步。第一步 depthwise convolution：每个输入通道各自用一个 `K×K` filter，只处理同一通道内的空间关系，参数量是 `K×K×I`。第二步 pointwise convolution：用 `1×1` filter 混合通道，把 `I` 个通道变成 `O` 个通道，参数量是 `I×O`。

$$
普通卷积：K²IO　｜　Depthwise + Pointwise：K²I + IO
当 K=3、O 很大时，参数量约变成原来的 1/9 左右。
$$

普通卷积参数72
可分离卷积参数26


K3
输入通道 I2
输出通道 O4
调整 K、I、O，观察参数量比例。

**错误示例：**“depthwise 卷积就是普通 CNN 的原始算法”。错因是把特殊变体当作标准卷积；标准卷积每个 filter 会跨所有输入通道，depthwise 则每个 filter 只负责一个通道，必须再用 pointwise 混合通道。

**现在你应该能做什么：**给定 K、I、O，计算普通卷积和 depthwise separable 卷积的参数量。

# 低秩近似：把一层拆成两层 {#lowrank}
<!-- tag: LOW RANK -->

低秩近似的直觉是：一个大的权重矩阵 `W` 参数太多，可以用两个较小矩阵 `U` 和 `V` 的乘积来近似。原来从 `N` 个输入到 `M` 个输出，需要 `MN` 个参数；若中间插入一个很窄的瓶颈维度 `K`，参数量变成 `MK + KN`。当 `K` 远小于 `M,N` 时，参数会少很多。

代价是表达能力受限。原来的 `W` 可以是任意矩阵；拆成 `U,V` 后，能表达的矩阵范围变窄，rank 不超过 `K`。Depthwise separable convolution 也可以从这个角度理解：把普通卷积中“空间处理 + 通道混合”的大操作拆成两步，以减少参数。

$$
WM×N ≈ UM×K VK×N，参数：MN → K(M+N)
K 越小，压缩越强，但表达限制也越强。
$$

**现在你应该能做什么：**解释为什么“一层拆两层”有时反而参数更少。

# Dynamic Computation：一个模型，多种计算预算 {#dynamic}
<!-- tag: DYNAMIC -->

前面的方法主要是把模型本身变小。Dynamic Computation 的目标不同：希望同一个模型可以根据设备资源或样本难度，调整实际使用的计算量。电量少时少算一点，电量足时多算一点；简单图片早退出，困难图片走完整网络。

一种做法是动态深度：在中间层加分类出口，训练时让每个出口都能预测标签。推理时，如果前面出口已经足够自信，就提前输出；否则继续往后算。另一种做法是动态宽度：同一个网络预设多个宽度，低预算时只用部分神经元，高预算时用全部神经元。关键是这些共享参数的不同路径。

## box info
### 动态深度
中间层也能输出，简单样本早退出，难样本继续算。

## box warn
### 动态宽度
同一个网络按预算使用不同数量的通道或神经元。

**现在你应该能做什么：**说明 dynamic computation 让推理成本随情境变化。

# 最小代码：蒸馏与可分离卷积 {#code}
<!-- tag: CODE -->

蒸馏训练通常同时包含两个 loss：一个让学生贴近老师软分布，一个让学生仍然尊重真实标签。下面只展示核心结构。

正确示例：Temperature DistillationPython-like

```text
def distill_step(teacher, student, x, y, T, alpha):
 teacher_prob = softmax(teacher(x) / T).detach()
 student_log_prob = log_softmax(student(x) / T)
 distill_loss = kl_div(student_log_prob, teacher_prob) * (T * T)
 hard_loss = cross_entropy(student(x), y)
 loss = alpha * distill_loss + (1 - alpha) * hard_loss
 loss.backward()
 update(student)
```

Depthwise separable convolution 在代码里通常就是一个 depthwise convolution 后接一个 1x1 convolution。

正确示例：Depthwise + PointwisePython-like

```text
class DepthwiseSeparableConv:
 def __init__(self, in_ch, out_ch, kernel):
 self.depthwise = Conv2d(in_ch, in_ch, kernel_size=kernel, groups=in_ch)
 self.pointwise = Conv2d(in_ch, out_ch, kernel_size=1)

 def forward(self, x):
 x = self.depthwise(x)
 x = self.pointwise(x)
 return x
```

错误示例：只用 depthwise 不混合通道Architecture bug

```text
# 错误示例：每个通道各算各的，没有 pointwise 通道混合。
x = depthwise_conv(x)
return x

# 后果：跨通道 pattern 难以表达，输出通道数也无法自由改变。
```

**现在你应该能做什么：**读懂蒸馏 loss 的两个来源，以及可分离卷积为什么需要两步。
