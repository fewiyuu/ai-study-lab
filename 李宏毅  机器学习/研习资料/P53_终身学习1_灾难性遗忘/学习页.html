机器想活到老学到老，第一关是别忘光

人们常想象 AI 会不断学习新技能：先会语音辨识，再会图像辨识，再会翻译，最后技能越积越多。Lifelong Learning，也叫 Continual Learning 或 Incremental Learning，研究的就是让一个模型依序学习多个任务，并尽量保留过去能力。

问题是，神经网络常出现 Catastrophic Forgetting：学完新任务后，旧任务准确率大幅下降。它不是轻微遗忘，而是灾难性地把旧技能丢掉。P53 先定义这个问题，并讲清楚如何用准确率矩阵、最终平均准确率、Backward Transfer 和 Forward Transfer 评估。

本节的概念地图

MAP

Lifelong Learning 的目标很朴素：一个模型按顺序学习多个任务，学新任务时不要忘旧任务。现实应用中，模型上线后会持续收到用户反馈和新资料；如果每次更新都要把过去所有资料重训一遍，存储和计算都不现实。因此我们希望模型能用新资料更新，同时保住旧能力。

困难在于，神经网络参数会被新任务的 loss 推动。学新任务时，如果没有旧任务资料约束参数方向，参数可能离开旧任务需要的位置。模型容量并不一定不够，因为把任务资料一起训练时，它可能同时学会；问题是依序训练时旧任务没有被复习，所以旧能力被新更新覆盖。

什么是 Lifelong Learning

DEFINITION

Lifelong Learning 字面意思是终身学习。在机器学习里，它指一个模型持续接触新任务、新资料或新 domain，并持续更新自己。它还有很多名字：Continual Learning、Never-ending Learning、Incremental Learning。名字不同，核心问题类似：知识要随着时间累积，而不是每学一个新东西就把旧东西覆盖掉。

这和线上系统很相关。一个模型在实验室训练完成后上线，接着不断收到用户反馈。新反馈可以看成新任务或新 domain。理想情况下，模型用这些新资料变得更好，同时保留过去已经学到的能力。否则系统会出现更新后旧功能退化的问题。

任务序列

模型不是一次看完所有资料，而是依次看到任务 1、任务 2、任务 3。

单一模型

希望同一个模型积累能力，而不是每个任务都另开一个模型。

长期评估

学完后要回头测旧任务，确认没有忘掉。

现在你应该能做什么：解释为什么上线后持续更新模型，本质上也会遇到终身学习问题。

灾难性遗忘：新任务学会，旧任务掉光

FORGETTING

Catastrophic Forgetting 指模型依序学习任务时，学完后面的任务后，前面任务表现大幅下降。课程里的数字辨识例子很清楚：模型先学 noisy digit，任务一达到 90%；再继续学 clean digit，任务二提升到 97%，但任务一掉到 80%。如果把两个任务资料一起训练，模型能同时做好，说明模型容量不是主要问题。

这件事灾难在哪里？人也会遗忘，但神经网络可能忘得很极端。BABI QA 例子里，模型学完任务 5 后任务 5 正确率到 100%；一学任务 6，任务 5 准确率可能直接掉回 0。新任务像把旧技能从参数里挤出去。

Sequential Training：Task 1 → Task 2 → ... → Task T 关键检查：学完 Task T 后，Task 1...T-1 是否仍然会做。

错误示例：“旧任务掉分是因为模型太小”。错因是忽略了同时训练对照；如果同样模型同时训练多个任务能学好，说明问题不是容量不够，而是依序更新时旧任务约束消失。

现在你应该能做什么：判断一个实验是否真的展示了灾难性遗忘：必须比较依序训练后的旧任务表现。

两个例子：数字和 QA 都会忘

EXAMPLES

第一个例子是两个数字辨识任务：一个带噪声，一个不带噪声。它们甚至可以被看成同一任务的不同 domain，但依序训练仍会忘。这个例子说明，终身学习不一定已经做到跨语音、影像、翻译那种宏大任务；很多文献中的任务差异其实很小，仍然会出现遗忘。

第二个例子是 BABI QA。它是早期用于研究问答的简化任务集，文章和问题都由模板生成，比真实问答简单得多。即使这样，当模型按顺序学 20 个 QA 子任务时，学完后面的任务也会把前面的任务忘掉。这说明灾难性遗忘不是某个数据集偶然现象，而是依序训练的普遍风险。

例子现象说明
Noisy / clean digit学新 domain 后旧 domain 准确率下降。相似任务也会忘。
BABI QA学完任务 6 后任务 5 可能掉到 0。简单文本任务也会灾难性遗忘。

现在你应该能做什么：说明为什么“任务很相似”并不能保证不会遗忘。

Multi-task Training 是 Upper Bound，不是现实解法

UPPER BOUND

如果把所有任务资料倒在一起训练，模型通常能同时学会多个任务。这叫 multi-task training。它常被当成 lifelong learning 的 upper bound，也就是上限参考：如果你能看到所有资料并一起训练，大概能做到多好。

但它不是现实解法。假设模型要学第 1000 个任务，如果必须把前 999 个任务资料全拿出来一起训练，就意味着模型要保存一辈子所有资料，计算成本也会随任务数不断增长。这像一个人每上新课都必须把从小到大学过的所有教材重读一遍，显然低效。

Multi-task Training

所有任务资料同时可见，效果通常强，适合当上限。

Lifelong Learning

任务依次到来，旧资料可能不能保存或不能反复训练。

现在你应该能做什么：解释为什么多任务训练能作为对照，却不能直接解决终身学习。

为什么不每个任务一个模型

ONE MODEL

每个任务训练一个独立模型，确实可以避免遗忘：任务 1 模型永远不被任务 2 更新。但这有两个问题。第一，任务数量很多时，存储所有模型会变得不可行。第二，不同任务之间无法共享知识，模型不能从过去任务中抽取对新任务有帮助的表示。

人类只有一个脑，却能学习很多技能。终身学习也希望一个模型能逐步积累能力。它不是为了形式上“必须一个模型”，而是为了存储效率、知识共享和持续适应。

判断句：一个任务一个模型避免了遗忘，但也放弃了终身学习真正想要的知识累积。

现在你应该能做什么：说出独立模型方案的两个主要缺点。

Lifelong Learning 与 Transfer Learning 的关注点不同

TRANSFER

Transfer Learning 关心的是：在任务 1 学到的东西，能不能帮助任务 2。所以它通常主要看新任务表现。Lifelong Learning 也涉及多个任务，但它还要回头看旧任务：学完任务 2 后，任务 1 还会不会做？

Transfer Learning

重点：旧任务知识是否帮助新任务。

问题句：学完任务 1 再学任务 2，任务 2 是否更好？

Lifelong Learning

重点：学新任务后是否保留旧任务。

问题句：学完任务 2 后，任务 1 是否还好？

现在你应该能做什么：看到两个任务实验时，判断它是在问迁移能力，还是在问遗忘程度。

准确率矩阵 R_ij：读懂评估表

MATRIX

终身学习常用一个准确率矩阵记录结果。R_ij 表示：模型学完第 i 个任务后，在第 j 个任务测试集上的准确率。第一个下标是训练到哪里，第二个下标是测哪个任务。

如果 i>j，表示学完后面的任务后回头测旧任务，主要看遗忘。如果 i<j，表示还没学任务 j 就去测它，主要看 forward transfer。对角线 R_jj 是刚学完任务 j 后立刻测任务 j，通常接近该任务记忆最鲜的时候。

Final Accuracy、Backward Transfer、Forward Transfer

METRICS

最常见的整体指标是最后一行平均准确率：模型学完所有任务后，在所有任务上的平均表现。它直接回答“最终这个模型到底会多少任务”。如果灾难性遗忘严重，前面任务会拖低这个平均值。

Backward Transfer，缩写 BWT，比较的是某个任务刚学完时的表现和最终表现。直觉上它衡量旧任务被后续任务影响了多少。常见写法类似平均 R_Tj - R_jj。如果是负数，表示遗忘；如果是正数，表示后面任务反而帮助旧任务变好，这很难但很理想。

Forward Transfer，缩写 FWT，问的是还没学某个任务前，模型是否已经因为过去任务而对它有帮助。它不是终身学习最核心的指标，但能衡量任务间的正迁移。

Final Avg = average(RT,1 ... RT,T) BWT 关注旧任务掉多少；FWT 关注未学任务提前会多少。

最小代码：依序训练与评估矩阵

CODE

下面的伪代码展示如何生成 R_ij 矩阵：每学完一个任务，就在所有任务测试集上评估一次。

正确示例：构建准确率矩阵Python-like
def train_lifelong(model, train_loaders, test_loaders):
    R = zeros(len(train_loaders), len(test_loaders))
    for i, train_data in enumerate(train_loaders):
        train_on_task(model, train_data)
        for j, test_data in enumerate(test_loaders):
            R[i, j] = evaluate(model, test_data)
    return R

下面是指标计算。重点不是记某个常数项，而是知道每个指标在矩阵里取哪些位置。

正确示例：Final Avg 与 BWTPython-like
def metrics(R):
    T = R.shape[0]
    final_avg = mean(R[T - 1, :])
    bwt_terms = []
    for j in range(T - 1):
        bwt_terms.append(R[T - 1, j] - R[j, j])
    bwt = mean(bwt_terms)
    return final_avg, bwt
