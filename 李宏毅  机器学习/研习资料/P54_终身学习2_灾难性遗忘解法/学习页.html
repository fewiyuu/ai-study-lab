不忘旧知识的关键，是控制更新方向

P53 定义了灾难性遗忘：模型依序学习新任务时，旧任务能力大幅下降。P54 开始讲解法。核心思路有几类：对旧任务重要参数加限制；修改新任务梯度方向；给新任务分配额外参数；用少量旧资料或生成资料做回放。

这些方法都在处理同一个冲突：新任务需要改变参数，旧任务希望参数别乱动。如果完全不限制，会遗忘；如果全部锁死，又会变得顽固，新任务学不好。好的终身学习方法要在 plasticity 和 stability 之间找平衡。

本节的概念地图

MAP

如果任务 1 的好参数是 θ_B，接着训练任务 2 时，普通 gradient descent 只看任务 2 的 loss。它可能把参数推到 θ*，在任务 2 上很好，但回到任务 1 的 loss surface 上却很差。灾难性遗忘就是这样发生的：新任务的更新没有顾及旧任务。

P54 的几类方法可以按“保护旧知识的方式”来理解。Regularization-based 方法保护旧任务重要参数；GEM 保护旧任务 loss 不被当前梯度增加；Progressive / PackNet 给不同任务分资源；Memory Replay 让模型训练新任务时仍看到旧任务样本或生成样本。没有哪类方法免费，每类都要付出存储、计算、扩展性或新任务学习能力的代价。

遗忘为什么发生：两个任务的好位置不同

CAUSE

想象模型只有两个参数 θ1 和 θ2。任务 1 有自己的 loss surface，任务 2 也有自己的 loss surface。先在任务 1 上训练得到 θ_B，这个点对任务 1 好；接着把同一组参数拿去任务 2 上继续训练，任务 2 的梯度可能把它推到另一个区域。这个新点对任务 2 好，但对任务 1 不一定好。

解决思路不是“不准学任务 2”，而是在任务 2 的好区域中找一个对任务 1 伤害较小的位置。若任务 1 对 θ2 很敏感，就尽量别动 θ2；如果任务 1 对 θ1 不敏感，就让新任务多改 θ1。这就是选择性可塑性的直觉。

现在你应该能做什么：用“新任务梯度把参数推离旧任务低 loss 区”解释灾难性遗忘。

Selective Synaptic Plasticity：只让部分连接可塑

SSP

Selective Synaptic Plasticity 可以直译为选择性突触可塑性。突触是神经之间的连接；可塑性表示连接可以改变。放到神经网络里，就是某些参数可以继续改变，某些对旧任务重要的参数要尽量固定。

这类方法也叫 regularization-based 方法。训练新任务时，不直接最小化新任务 loss，而是在 loss 后面加一个惩罚项：如果某个参数偏离旧任务学到的值太多，就要扣分；偏离惩罚的强弱由重要性 B_i 决定。

L'(θ) = Lnew(θ) + Σi Bi(θi - θBi)² θ^B 是旧任务训练后的参数；B_i 越大，第 i 个参数越不该动。

现在你应该能做什么：读懂 regularization-based 方法的核心公式，并解释每一项的作用。

参数重要性 B_i：谁该锁住，谁能移动

IMPORTANCE

B_i 表示第 i 个参数对旧任务有多重要。如果改变某个参数对旧任务 loss 影响很小，它的 B_i 就应该小，新任务可以自由改它。如果改变某个参数会让旧任务 loss 大幅上升，它的 B_i 就应该大，新任务最好不要动它。

课程用二维例子说明：任务 1 对 θ1 不敏感，对 θ2 敏感；那么训练任务 2 时，应该允许参数主要沿 θ1 方向移动，避免沿 θ2 方向移动。不同方法的差别，就在于如何估计这些 B_i：EWC、SI、MAS、RWalk、SCP 等都在回答这个问题。

B_i 小

旧任务不太依赖这个参数，新任务可以改。

B_i 大

旧任务强烈依赖这个参数，新任务改动要付出代价。

现在你应该能做什么：解释为什么不能让所有参数受到同样强度的约束。

动手观察：遗忘与顽固是一对取舍

TRADEOFF

如果所有 B_i=0，就回到普通训练，新任务学得快，但旧任务容易忘。如果所有 B_i 都非常大，旧任务保住了，但新任务学不动，这叫 intransigence，可以理解成顽固、不肯让步。好的方法要让不同参数有不同约束。

EWC 类方法：Regularization-based 的代表

EWC

EWC，全名 Elastic Weight Consolidation，是这类方法中的代表。它的想法是，旧任务训练完后，估计每个参数对旧任务的重要性，并把旧参数值保存下来。训练新任务时，loss 里加入“不要偏离重要旧参数”的惩罚项。

课程没有逐个讲 EWC、SI、MAS 等方法如何算 B_i，但你要掌握共同骨架：它们都需要额外存一些信息，例如旧模型参数和每个参数的重要性。它们没有直接保存旧资料，但不代表没有额外存储成本。

方法族保护方式代价
EWC / SI / MAS 等给重要参数加偏离惩罚。要保存旧参数和重要性。
L2 全参数约束所有参数同等不准动。容易顽固，新任务学不好。
普通训练没有旧任务约束。容易灾难性遗忘。

现在你应该能做什么：说明 EWC 类方法为什么没有存旧资料，但仍然需要额外存储。

GEM：限制梯度方向，而不是限制参数值

GEM

GEM，全名 Gradient Episodic Memory。它不是直接约束参数离旧参数多远，而是约束新任务的梯度方向。训练任务 2 时，先算新任务梯度 g；再用少量旧任务样本算旧任务梯度 g_B。如果 g 和 g_B 方向冲突，也就是 inner product 小于 0，就把 g 修改成一个更安全的 g'。

直觉是：新更新不应该让旧任务 loss 变坏。GEM 需要存一些旧任务样本来计算旧任务梯度，所以它比 regularization-based 方法更接近 replay，但通常只需要少量样本。公平比较时，要考虑这些样本占用的 memory 是否比保存旧模型和 B_i 更大。

额外资源分配：给新任务新空间

RESOURCE

另一类方法叫 Additional Neural Resource Allocation。Progressive Neural Network 的做法是：任务 1 学到的模型固定住；任务 2 来了，就新增一块网络，只训练新参数，同时可以读取旧网络 hidden layer 的输出。这样旧任务参数完全不动，所以不会忘。

问题是模型会随着任务数不断变大，任务很多时 memory 会耗尽。PackNet 走另一个方向：一开始开一个大网络，每个任务只占用其中一部分参数，旧任务占用的部分固定，新任务使用剩余空间。它不会每次都增长，但本质上是提前准备大量空间。CPG 则结合 growing 和 packing 的思路。

方法做法风险
Progressive Network新任务新增网络列，旧参数冻结。任务越多模型越大。
PackNet大网络中为不同任务分配不同参数子集。预留空间有限，用完仍会受限。
CPG结合压缩、打包和增长。实现复杂，需要管理任务资源。

现在你应该能做什么：说明额外资源方法为什么能避免遗忘，以及为什么不能无限扩展。

Memory Replay：让旧任务资料回到训练中

REPLAY

最直觉的解法是复习旧资料。既然 multi-task training 能避免遗忘，那训练新任务时混入旧任务资料就能帮助保留旧能力。问题是不能保存全部旧资料，因此 replay 方法常保存少量样本，或训练一个 generator 生成旧任务的 pseudo data。

Generative Replay 的流程是：任务 1 训练 classifier，同时训练 generator 产生任务 1 资料；任务 2 来时，用 generator 生成任务 1 的 pseudo data，与任务 2 真实资料一起训练。之后再训练新的 generator，让它能产生任务 1 和任务 2 的资料。这样旧知识通过生成资料被复习。

Task 1

训练 classifier 和 generator。

Task 2

真实新资料 + 生成旧资料一起训练。

更新 Generator

学会产生更多任务的资料。

持续回放

逼近多任务训练效果。

Replay 的代价是存储样本或 generator。若 generator 比保存所有资料更小，而且生成质量足够好，它可能非常有效。课程提到生成资料的方法在经验上常能逼近 multi-task upper bound。

现在你应该能做什么：解释 replay 为什么像“复习”，以及 generative replay 为什么不直接存旧资料。

任务顺序也重要：Curriculum Learning

ORDER

课程最后提醒，任务顺序会显著影响遗忘。先学 noisy digit 再学 clean digit，可能忘；反过来先学 clean digit 再学 noisy digit，可能两个任务都做得不错。顺序改变，旧知识和新知识之间的覆盖关系也改变。

研究“什么顺序更适合学习”的问题叫 Curriculum Learning。做终身学习实验时，也要注意不能只跑一种任务顺序就下结论。常见做法是尝试多个顺序，甚至穷举小规模任务的所有顺序，再报告平均表现。

现在你应该能做什么：说明为什么任务顺序是终身学习实验中的重要变量。

最小代码：重要性惩罚与 GEM 检查

CODE

Regularization-based 方法的核心是在新任务 loss 后加一个按参数重要性加权的偏离惩罚。

正确示例：EWC 风格惩罚Python-like
def continual_loss(new_loss, params, old_params, importance, lam):
    penalty = 0.0
    for name, theta in params.items():
        diff = theta - old_params[name]
        penalty += (importance[name] * diff * diff).sum()
    return new_loss + lam * penalty

GEM 的核心是在更新前检查新梯度是否会伤害旧任务。下面只保留方向判断。

正确示例：GEM 梯度方向检查Python-like
def gem_update(grad_new, grad_old):
    dot = inner_product(grad_new, grad_old)
    if dot < 0:
        grad_new = project_to_safe_direction(grad_new, grad_old)
    apply_gradient(grad_new)

# 错误示例：把 B_i 当普通参数一起最小化。
loss = new_loss + sum(B_i * (theta_i - old_theta_i) ** 2)

# 若 B_i 可自由下降，最容易的做法是 B_i -> 0，旧任务保护消失。
