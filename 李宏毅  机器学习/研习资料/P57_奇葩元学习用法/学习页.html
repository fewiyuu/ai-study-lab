Self-supervised Learning 与 MAML：都在找初始化

SSL

BERT、BART 这类 self-supervised model 常被当成强初始化。它们先在大量无标注资料上做预设任务，比如遮住词再预测，得到一组对很多下游任务有用的参数。MAML 也在找初始化，但它找的是“少量标注资料更新几步后，在任务查询资料上表现好”的初始化。

这也解释了两者互补的原因。Self-supervised learning 的优点是能吃大量无标注资料，覆盖面广；弱点是预训练目标和真实下游任务之间有距离。MAML 的优点是外层目标直接看目标任务族的表现；弱点是需要构造有标注的训练任务，而且计算成本高。把二者接起来，常在低资源任务上更有优势。

错误示例：“BERT 和 MAML 都是找初始化，所以只能二选一。”错因是把目标相似误认为方法冲突；修法是把 BERT 看成 MAML 外层训练的起点，再让 meta 目标调整它。

BERT 加 Reptile：低资源任务为什么会受益

LOW

课程中提到的实验直觉很一致：在训练资料少的时候，self-supervised model 再接 meta 方法往往更有帮助。原因它利用少量任务结构告诉初始化“真正下游任务关心什么”。

无标注语料训练 BERT / BART，得到强起点。

训练任务构造多个少样本任务。

Meta 调整用 MAML / Reptile 贴近任务族。

新任务少量标注资料快速适应。

现在你应该能做什么：解释为什么“先大规模预训练，再用任务级目标微调初始化”常用于少样本 NLP。

Knowledge Distillation：老师强，不代表会教

DISTILLATION

知识蒸馏的普通做法是先训练一个大 teacher model，再让小 student model 去模仿 teacher 的输出。这个流程默认老师的分类能力越强，学生学得越好。但课程提醒我们：这件事不一定成立。一个在分类任务上正确率最高的老师，未必能给学生提供最容易吸收的软标签。

把元学习放进蒸馏，就是改外层目标：teacher 更新参数时，不只看自己做得怎样，而是看 student 经过蒸馏以后，在验证资料上的结果怎样。这样 teacher 学的是“怎么教”，不是单纯“怎么自己答题”。

普通训练teacher 追求自己的分类正确率。

普通蒸馏student 追求接近 teacher 输出。

Learn to Teachteacher 追求学生学完后的表现。

蒸馏温度也可以被学习

TEMPERATURE

实际做 learn to teach 时，不一定要更新整个 teacher model。一个更轻的做法是只学习输出温度 temperature。温度会改变 softmax 的尖锐程度：温度低，最大类别更接近 1；温度高，分布更平滑，非最大类别也会保留信息。

Domain Generalization：把源领域轮流假扮目标领域

DOMAIN

Domain adaptation 和 domain generalization 都关心分布差异。区别在于：domain adaptation 通常能看到目标领域的一些资料，可能有标注，也可能无标注；domain generalization 更严格，训练时完全看不到真正目标领域。我们只有多个源领域，希望训练出的模型能泛化到未知目标领域。

测试时，真正目标领域还是没见过。我们会把所有源领域资料用于训练，然后期待通过 leave-one-domain 学到的学习算法能迁移到新领域。这个想法的关键“反复练习从一些领域泛化到另一个未参与训练的领域”。

3 个源领域时，可以轮流让其中 1 个领域假扮目标领域。

错误示例：“domain generalization 训练时先看一点目标领域资料，再报告目标领域表现。”这更像 adaptation，不是严格的 generalization；修法是训练阶段不使用真正目标领域。

元学习自己也可能需要领域适应

TASK

前面说元学习可以帮助 domain generalization，但反过来，元学习本身也会遇到更高一层的分布差异。普通机器学习里，训练资料和测试资料可能分布不同；元学习里，训练任务和测试任务也可能来自不同分布。

层级分布差异发生在哪里会造成什么问题
样本层训练样本与测试样本不同分布普通模型在目标领域表现变差
任务层meta training tasks 与 meta testing tasks 不同分布学到的学习算法在新任务族上不稳

现在你应该能做什么：区分样本层 domain shift 和任务层 task shift。

Lifelong Learning：学习怎样更新而不忘旧任务

LIFELONG

终身学习关心的是连续任务。模型先学任务一，再学任务二，理想状态是新任务学会了，旧任务也保留。但普通训练常出现 catastrophic forgetting：参数被新任务更新后，旧任务表现大幅下降。

把元学习放进来，就是学习一个更新策略或正则化策略。外层目标不再只是新任务测试集 loss 低，而是“学完新任务以后，新任务能做，旧任务也别忘”。因此元学习可以优化一个更贴近终身学习需求的学习算法。

普通新任务训练只关心任务二 loss 下降。

正则化方法限制重要参数变化，减轻遗忘。

Meta Lifelong学习怎样限制、怎样更新、怎样平衡新旧任务。

元学习也会灾难性遗忘

META

元学习可以帮助普通模型减少遗忘，但元学习本身也可能遗忘。想象你在训练学习算法时，先给它一组任务分布 A，再给它任务分布 B。如果连续更新时只看 B，学到的 learning algorithm 可能只适合 B，不再适合 A。

普通层模型学新类别后忘旧类别。被更新的是 θ。

元层学习算法适应新任务族后忘旧任务族。被更新的是 φ。

最小代码：四种场景的共同骨架

CODE

下面的伪代码不追求可运行，而是帮助你读懂外层目标怎么写。共同点是：内层先让某个对象学习，外层再根据学习后的表现更新 φ。

代码
def meta_train_teacher(teacher_phi, tasks):
 meta_loss = 0.0
 for task in tasks:
 student = init_small_model()
 soft_targets = teacher_phi(task.train_x)
 student = distill(student, task.train_x, soft_targets)
 meta_loss += loss(student(task.valid_x), task.valid_y)
 update(teacher_phi, meta_loss)

在领域泛化里，关键是把一个源领域留出来当临时目标。

正确示例：leave-one-domain 任务构造Python-like

错误示例：把 teacher 自己准确率当作唯一目标Objective bug

现在你应该能做什么：从代码层面识别内层学习、外层评价和被更新的 φ。
