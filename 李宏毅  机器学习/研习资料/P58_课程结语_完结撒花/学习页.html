结语真正要复盘什么

WRAP-UP

一门覆盖很多主题的课程，很容易学完以后只留下“我听过很多名词”。P58 的重点不是让你再背一遍名词表，而是把每个主题放回问题类型：输入是什么、输出是什么、训练资料长什么样、评估看什么、实践会卡在哪里。

从这个角度看，课程的主线很清楚。最开始，机器学习被定义为找一个 function。深度学习把这个 function 写成多层可训练模型。CNN 处理矩阵形输入，self-attention 和 Transformer 处理序列与可变长度输入输出。后半段继续扩展：模型可以生成、可以用无标注资料预训练、可以面对分布不匹配、可以做决策、可以被攻击也需要解释、可以被压缩部署、可以连续学习，还可以学习如何学习。

课程路线：从找 function 到学习如何学习

ROUTE

最初的定义很朴素：机器学习就是从资料中找一个 function。这个 function 可以把图片映射到类别，把声音映射到文字，把一句话映射到另一种语言，把一个游戏状态映射到动作。深度学习不是把这个定义推翻，而是给 function 一个更强的表示形式。

课程前半段主要建立模型结构。Linear model 不够时，进入 deep learning；输入是图片这类矩阵时，用 convolutional neural network；输入是文字或声音这类 sequence 时，引入 self-attention；输入输出长度不一样时，用 Transformer 这种 encoder-decoder 或 attention-based 架构。

后半段把问题变复杂。生成模型问“机器能不能创造”；self-supervised learning 问“没有标注能不能先学表示”；domain adaptation 问“训练和测试不匹配怎么办”；reinforcement learning 问“没有固定答案、只有回馈时怎么办”；攻击防御问“人类恶意扰动怎么办”；可解释性问“模型到底学了什么”；模型压缩问“设备放不下怎么办”；终身学习问“不断学新东西时怎么不忘旧东西”；meta learning 问“能不能连学习算法也一起学”。

核心模型：先看输入输出，再选结构

MODELS

课程里出现很多模型，但选模型时先别从名字开始。先看输入输出。图片通常有局部空间结构，CNN 的卷积核能共享参数、捕捉局部模式。文字、语音、时间序列有顺序关系，self-attention 能让每个位置直接看其他位置。翻译、问答、摘要这类任务常常输入输出长度不同，Transformer 让 encoder 和 decoder 通过 attention 连接。

这不是说模型选择永远固定。很多现代系统会混合结构，例如影像也能用 Transformer，语音也能用卷积和 attention。但对入门学习来说，先用“输入是什么形状、输出是什么形式”做初步判断，会比直接背模型名更稳。

问题形态常见工具先问的问题
图片分类、检测、异常CNN、Vision Transformer输入是否有空间结构？输出是类别还是位置？

应用谱系：课程作业不是孤立练习

APPLICATIONS

结语里特别强调了很多应用：COVID-19 趋势预测、影像分类、影像调试、影像压缩、异常检测、生成二次元头像、机器翻译、机器问答、语音辨识、说话人辨识、强化学习小游戏。这些看似分散，其实都在训练同一种能力：把真实任务拆成资料、模型、loss、评估和调试。

如果你以后要做研究或项目，最有价值的不是“照抄某个作业”，而是识别哪个作业的结构接近你的问题。比如你的研究对象是医学影像，影像分类和异常检测的作业可能能改；你的问题是跨语言文本分析，翻译和 self-supervised 相关内容就可能有用；你的问题是设备端部署，压缩和蒸馏就会变重要。

真实摩擦：这不是 bug，是深度学习的一部分

REALITY

结语花了很大篇幅谈痛苦：训练时间长、Colab 不稳定、没有 GPU、public set 和 private set 不一致、每天提交次数有限、超参数像在猜。这些不只是课程管理问题，也不是故意为难学习者。真实深度学习项目里，这些摩擦会反复出现。

训练时间长会影响实验设计。你不能无限试所有参数，所以要先做小规模 sanity check，再扩大训练。public/private split 不一致提醒你不要只追公开榜单，而要关注泛化。提交次数有限迫使你在本地验证和错误分析上下功夫。超参数难调说明模型训练结果受初始化、学习率、资料切分、正则化等因素影响，不能只凭一次结果下结论。

错误示例

只要 private score 下降，就立刻认定模型架构错了，然后随机大改所有超参数。

更稳做法

先检查资料切分、过拟合迹象、随机种子、训练曲线和 public/private 分布差异，再决定改模型还是改验证。

下一步：找一个自己的应用来解决

PROJECT

课程结束后，最有价值的下一步是选一个真实问题，做一个最小版本。最小版本不是把目标变小到没有意义，而是把变量控制住：先确定输入输出，拿到一小份可用资料，选一个课程里见过的 baseline，跑通训练、验证和错误分析，再逐步加复杂度。

读论文：你已经有了入口

PAPER

结语里提到，如果课程大部分内容已经通顺，你应该能开始读顶会论文。这里的“能读懂”不是每个推导都秒懂，而是能抓住论文的问题、方法、实验和限制。课程给你的基础，是让你看到 CNN、Transformer、self-supervised、domain adaptation、RL、compression、meta learning 等关键词时，不至于完全没有方向。

读论文可以用四步。第一步，只读摘要、图一和实验表，判断它解决什么问题。第二步，找它和课程中哪个主题相连。第三步，读方法部分，把新符号翻译成已有概念。第四步，读实验设置，特别看 baseline、数据切分、消融实验和失败边界。

读论文问题对应检查
它要解决什么限制？资料少、分布变、算力小、解释难、连续任务、生成质量等。
它改了哪一块？模型结构、loss、训练流程、资料处理、评估方式。
它和谁比？baseline 是否公平，数据是否泄漏，任务是否太窄。
它什么时候会失败？论文有没有消融、限制、错误案例。

学习心态：知道困难，才算真的入门

MINDSET

结语里很重要的一点是：课程没有把困难藏起来。深度学习不是按按钮就自动解决所有问题。真正做项目时，你会遇到资料不干净、训练跑很久、结果不稳定、榜单分数不一致、模型难解释、资源不够、baseline 难超越。这些问题会让人不舒服，但它们让你对技术有真实认识。

知道困难，不等于劝退。相反，它能保护你以后面对真实项目时少受挫。你会知道老板或客户说“明天做个应用出来”时，背后缺了资料定义、评价指标、错误分析和资源预算。你也会知道，工程师做不出来不一定是能力差，可能是问题定义、资料质量或部署约束还没有被认真处理。

这门课的结尾不是说每个人都必须继续深度学习，而是让你有能力做选择。如果你继续做，你知道下一步怎么做；如果你不继续做，你也知道这个技术大概能做什么、难在哪里、什么时候值得请专业的人来做。

现在你应该能做什么：用更真实的标准评估一个机器学习项目，而不是被“看起来很简单”的演示误导。

最小流程：把课程知识用到真实项目

CODE

下面的伪代码把整门课压成一个项目流程。真正的难点不在某一行代码，而在每一步都要做判断：资料是否可信，指标是否对，baseline 是否跑通，错误是否被看见。

正确示例：最小机器学习项目流程Python-like
def run_minimal_ml_project(problem):
    task = define_input_output_metric(problem)
    data = collect_and_split_data(task)
    baseline = choose_course_baseline(task)
    result = train_and_validate(baseline, data)
    errors = inspect_failure_cases(result, data.valid)
    next_step = decide_model_data_or_metric_change(errors)
    return next_step
