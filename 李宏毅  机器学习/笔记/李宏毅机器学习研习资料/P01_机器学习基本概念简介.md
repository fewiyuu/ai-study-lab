## 值得记住的问题

- 机器学习不是先背模型名，而是先问：输入是什么，输出是什么，要找什么函数。
- `training loss` 低不等于模型已经会了；还要看验证集或测试集上的表现。
- 读训练代码时，不能只说“定义模型 / 计算 loss / 更新参数”，要说清每一行读什么、写什么。

## 核心主线

P01 的核心线索是：任务描述 → 输入 / 输出 → model → parameter → loss → optimization → generalization。

这次答题说明主线已经抓住了。基础概念、任务类型、loss 直觉、参数更新方向和训练 / 测试分工都比较稳。需要补的是开放题的完整度：不要只给一句结论，要把错因、后果和修法说出来。

## 关键概念

| 概念 | 稳定说法 | 这次要修正的点 |
| --- | --- | --- |
| model | 带未知参数的函数形式，也就是可搜索的函数骨架。 | 不要把 model 当成训练完的答案。 |
| parameter | 训练从资料里学出来的具体值。 | 不要说 parameter 构成 model，容易把骨架和值的关系说反。 |
| loss | 把“预测好不好”变成可优化的数字。 | 说 loss 时最好补一句：没有 loss，优化器不知道往哪里改。 |
| learning rate | 控制每次参数更新的步幅。 | 太大时不只是“找不到收敛点”，还可能越过低点、震荡或让 loss 变大。 |
| generalization | 模型在未见资料上的表现。 | 训练集表现好，只能说明它在见过的资料上做得好。 |

## 例子与操作模式

读最小训练代码时，按“读什么 / 写什么 / 如果跳过会怎样”走：

```python
y_hat = w * x + b
loss = mean((y_hat - y) ** 2)
dw = mean(2 * (y_hat - y) * x)
w -= lr * dw
```

- `y_hat = w * x + b`：读取当前 `w`、`x`、`b`，写出预测值。这里是在预测，不是在重新定义模型。
- `loss = mean((y_hat - y) ** 2)`：读取预测值和真实答案，写出评分数字。
- `dw = mean(2 * (y_hat - y) * x)`：读取误差和输入，写出 `w` 的更新方向。
- `w -= lr * dw`：读取梯度和 learning rate，写回新的 `w`。

## 常见误区

| 错法 | 后果 | 下次怎么识别 |
| --- | --- | --- |
| 把预测行说成“定义模型”。 | 会看不清训练循环里当前这一步到底在读写什么。 | 只要等号左边是 `y_hat`，通常先判断它是不是在生成预测。 |
| 只说训练 loss 低，但不提验证 / 测试。 | 容易把记住训练资料误判成真正学会。 | 凡是看到“上线”“可用”，立刻补问未见资料表现。 |
| 解释 learning rate 太大时只说“步子太大”。 | 没有讲清失败症状。 | 要补出震荡、越过低点、loss 变大这几个后果。 |
| 把聊天模型说成“一个字一次输出”。 | 粒度不准，也容易把停止条件说乱。 | 更稳的粒度是 token；停止通常和结束 token、最大长度或解码策略有关。 |
| 费曼解释只写“拟合函数”。 | 抓住核心但不够完整，没串起训练流程。 | 至少串起输入、输出、模型、参数、loss、优化、泛化。 |

## 迁移与使用

拿到一个新机器学习任务，先按这 5 个问题拆：

1. 输入是什么？
2. 输出是什么？
3. 任务更像 regression、classification，还是 structured learning？
4. loss 怎么定义，才和任务目标一致？
5. 训练完以后用什么资料和指标检查泛化？

生成式文本任务要额外记住：每一步可能像是在词表里选下一个 token，但整段回答不是一次固定类别分类，而是一串有顺序、受上下文约束的结构化输出。

## 修正后的解释

机器学习是在找一个从输入到输出的函数。人先决定 model，也就是函数的大致形式；训练资料再用来调整 parameter，让这个函数的预测更接近真实答案。loss 把预测好不好变成一个数字，optimization 根据这个数字和梯度去更新参数。训练 loss 低只说明模型在见过的资料上表现不错。要判断它是不是真的学到规律，还要看验证集或测试集上的表现。

## 复习卡片

- 问：`model` 和 `parameter` 的关系是什么？  
  答：model 是函数骨架，parameter 是训练从资料里学出来的具体值。

- 问：为什么不能只看 training loss？  
  答：training loss 只反映见过的资料；模型可能过拟合，所以要看验证集或测试集。

- 问：`y_hat = w * x + b` 在训练循环里做什么？  
  答：用当前参数和输入算预测值，不是在重新定义模型。

- 问：learning rate 太大有什么症状？  
  答：参数可能越过低点、来回震荡，甚至让 loss 变大。

- 问：为什么生成一段回答不是普通单次 classification？  
  答：输出是一串有顺序、受上下文约束的 token，而不是一次选一个固定标签。
