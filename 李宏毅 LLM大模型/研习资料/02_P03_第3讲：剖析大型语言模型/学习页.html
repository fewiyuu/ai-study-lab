剖开 LLM：从 token 向量到下一词概率

前一节把语言模型看成函数 F：输入未完成的句子 X，输出下一个 token 的分布。这一节把 F 拆开：先查 embedding table，再一层层通过 Transformer，最后用 LM head 得到 logits，并把它们转成便于采样的数值。

要特别注意：这节不训练模型，只观察已经训练好的模型。你看的不是“模型为什么学会”，而是“训练好的参数如何和输入互动，产生下一个 token 的分数”。

token id→embedding→layers→representation→logits→采样

概念骨架

MAP

它不是黑箱的一句话

“模型预测下一个 token”是外部行为；内部行为是很多矩阵、向量和非线性函数在层间反复变换。你现在应该能做什么：把一句“模型回答了”拆成一串可检查的中间对象。

参数不是一团雾

参数通常以矩阵或向量保存，也就是 tensor。embedding table、Q/K/V 投影、feed-forward 的 weight 和 bias 都是参数。你现在应该能做什么：看到 shape 时，先问“输入维度和输出维度分别是什么”。

解释不是等于读心

logit lens、attention 图、representation 加减都能帮助分析，但它们是工具和证据，不是模型“真实心理活动”的逐字记录。你现在应该能做什么：用这些工具提出假设，再谨慎验证。

术语表：从输入到 logits 的对象

TERMS

1. token id 只是查表用的编号

输入句子先被 tokenizer 切成 token，并转成 token id。课程为了投影片简洁，常假设一个中文字就是一个 token；真实模型里不一定如此，一个中文字可能被拆成多个 token，几个中文字也可能合成一个 token。

token id 本身没有语义距离：编号 540 和 541 不表示意思接近。它像图书馆书架索引，关键是用这个编号去 embedding table 里拿出那一行向量。

2. embedding table 把离散编号变成向量

embedding table 是一个矩阵。行数大约等于 vocabulary size，列数等于 embedding dimension。例如字幕里提到 Llama 的输入 embedding 形状接近 128256 × 3072：前者是 token 槽位数，后者是每个 token 用 3072 个数字表示。

这个矩阵不是临时算出来的字典，而是模型参数的一部分。训练之后，语义相近或经常以相似方式出现的 token，往往在 embedding 空间里更接近。

3. 每一层都保持序列长度，但改写每个位置的向量

输入有多少个 token，进一层还是多少个向量，出一层仍然是多少个向量。改变的是每个位置上的向量内容：它从“这个 token 自己的向量”，逐渐变成“考虑上下文后的表示”。

这就是 contextualized embedding，也常被叫作 hidden representation 或 latent representation。名字不同，核心直觉相同：它是模型在某一层、某个位置上对当前 token 及其左侧上下文的内部表示。

对象这一节要抓住的意思
token id离散编号，用来从矩阵取行；编号大小本身不代表语义。
embeddingtoken 对应的一排数字，是模型能继续计算的向量形式。
layer把一排向量变成另一排向量；长度不变，内容被改写。
representation某层某位置的内部向量，通常已经混入上下文信息。
LM head把最后位置的向量投到 vocabulary 大小的分数向量。
logits每个 token 的原始分数，可以正负，不是概率。

完整链路一句话

prompt 先变 token id，id 查 embedding table 变成向量，向量通过多层 Transformer 改写，最后一个位置的向量经过 LM head 得到每个 token 的 logits，logits 再被转成可采样的分布。

你现在应该能做什么：解释“LLM 内部都是矩阵运算”这句话具体指哪些矩阵。

讲义正文

LECTURE

LM head、logits、softmax 与 temperature

LOGITS

LM head 把最后一个位置投回 vocabulary

所有 layer 跑完以后，模型会取最后一个位置的向量。这个向量通常是 K 维，再乘上 LM head 这个矩阵，得到 V 维的向量。V 就是 vocabulary size：模型有多少候选 token，就有多少个分数。

很多模型会把开头的 embedding table 和结尾的 LM head 绑定成同一个矩阵。这时可以把 LM head 直觉理解为：最后向量和每个 token embedding 做相似度比较，越接近某个 token 的 embedding，该 token 的分数越高。

logits 不是概率

LM head 输出的分数叫 logits。它可以是正数、负数，也不要求加起来等于 1，所以不能直接当概率。softmax 的作用，是把这些原始分数变成非负、总和为 1 的数，方便后续采样。

p_i = exp(s_i / T) / Σ_j exp(s_j / T) s_i 是第 i 个 token 的 logit，T 是 temperature。T 越大，分布越平；T 越小，高分 token 越集中。

课程里特别提醒：softmax 后的数值不必被神化成“真实概率”。它更像一种方便掷骰子的权重。平台里的“创意模式”和“保守模式”，常常就是调 temperature 或类似采样参数。

中间表示：同一个 token 会因为上下文变成不同向量

REPRESENTATION

第 0 层只看 token 自己

在进入 Transformer 之前，同一个 token 查到的 embedding 是一样的。字幕里用 how about you、how are you、nice to meet you 做例子：第 0 层里，每个 you 的 token embedding 相同。

这是因为第 0 层只是查表。模型还没有通过 attention 读上下文，所以“你在什么句子里”还没有进入这个向量。

过了 layer 后，同一个 token 会被上下文改写

通过第一层以后，how about you 里的 you 和 how are you 里的 you 数字就不同了。因为 attention 会把左侧 token 的信息揉进当前位置。

但每个句子的第一个 token，例如两个句子开头的 how，在 causal attention 下仍然可能相同，因为它左边没有其他 token 可以看。这个例子很适合检查你是否真的理解“只看左边”的含义。

apple 例子：上下文能把词义拉开

I ate an apple for breakfast 里的 apple 更接近水果；Apple released a new iPad 里的 apple 更接近公司。随着层数加深，两个 apple 的 representation 相似度会下降。若两个上下文都指水果，或者都指公司，相似度会维持得更高。

观察：层数越深，词义分化越明显

归一化后更容易看出：水果 apple 与公司 Apple 随层数加深逐渐分开。

一个重要边界

不同层的向量分布可能不一样，直接跨层比较相似度会误导。课程里因此又计算平均相似度做 normalization，让趋势更可靠。

怎样解释 representation：观察、投影、扰动与读出

INTERPRET

相似度观察

拿出 apple 的 token embedding，和所有其他 token embedding 算相似度，可能会看到 Apple、苹果、Cupertino、iPhone、MacBook 等接近项。它说明模型把多种语义关系压进同一个高维空间。

你现在应该能做什么：用“相似”而不是“完全等价”描述 embedding 邻居。

低维投影

高维向量无法直接看。研究者会找一个二维平面，把 representation 投影上去。有论文发现某些中间层能投出文法树或世界地图，说明特定层可能承载文法或地理信息。

你现在应该能做什么：知道投影图是一个视角，不是唯一真相。

representation engineering

可以收集“模型会拒绝”的输入和“模型不会拒绝”的输入，分别取某层 representation 平均，再相减得到一个近似“拒绝方向”。把这个方向加进去，模型可能更容易拒绝；减掉则可能更容易放弃拒绝。

你现在应该能做什么：解释“加一个向量改变模型行为”为什么可能发生。

logit lens：每层都临时接上 LM head

正常只有最后一层会经过 LM head 产生 logits。logit lens 的做法是：把每一层的 representation 都拿去过一次 LM head，看看这一层“如果现在就输出”，最像哪个 token。

课程中用“天气”“天气预”“今天天气真好”做例子：模型中间层可能先出现英文 weather、forecast，最后再转成中文 token。这个工具可以帮助我们观察层间语义变化，但不能把它当成严格的心理读心术。

patch scope：让模型用一句话解释某个向量

如果一个 representation 不能用单个 token 描述，patch scope 会把它“补丁”到另一个 prompt 中。例如给模型“请简单介绍 X”，再把 X 对应位置的向量替换成某个中间 representation，看模型续写出什么。

这个方法的好处是能得到更完整的自然语言解释；风险是解释会被你选择的 prompt 影响。换成“请说出 X 的秘密”，输出可能就完全不同。

self-attention：先找相关 token，再融合信息

ATTENTION

大方向：当前位置要问“谁会影响我的意思”

以“两颗青苹果”的“果”为例，只看“果”不知道它是苹果、西瓜还是别的水果；看到“苹”和“青”，模型才知道它是青苹果。self-attention 的第一步，是计算其他 token 对当前位置的影响力。

Q/K/V：三个投影的分工

当前位置的向量乘上 WQ 得到 query，其他位置的向量乘上 WK 得到 key。query 和 key 做点积，点积越大，表示这个 token 对当前位置越重要。再经过 softmax，就得到 attention weights。

第二步，每个 token 还会乘上 WV 得到 value。模型按 attention weights 对 value 做加权和，把相关 token 的信息揉到当前位置里。

位置、多个头、残差与 causal mask

只看 token embedding 会丢失位置信息，所以模型还需要 positional information。课程用 positional embedding 说明，现代 Llama 常用 RoPE 这类方法处理位置和外推长度。

一个 attention head 只能看一种关系，例如形容词关系。多头 attention 可以让不同 head 分别看颜色、数量、前文实体、默认起始符号等。每个 head 产生一个向量，再用 WO 合并。

残差连接会把原来的当前位置向量加回来，避免模型在融合信息时忘掉“我原本是谁”。causal attention 则保证当前位置只看左边 token，适合自回归生成。

试一试：attention matrix 怎么读

横向是被看的 token，纵向是当前 token。右上角为 0 表示不能看未来。

浅色：权重低深色：权重高右上角：causal mask

读图提醒

attention 图常能提供线索，但不是每个 head 都容易解释。有些 head 大量看起始符，只表示“这一组关系暂时没什么可看”。

feed-forward：每个位置上的向量再被非线性改写

FFN

Transformer layer 里通常有两大块

一个 Transformer layer 先做 self-attention，让每个位置吸收上下文；再做 feed-forward network，让每个位置的向量通过矩阵、bias 和 activation function 被进一步改写。输入几个向量，输出仍然几个向量。

feed-forward 不直接在不同位置之间交换信息，它更像在每个位置内部做一次复杂变换。attention 负责“从别人那里拿信息”，feed-forward 负责“把拿到的信息再加工”。

神经元不是神秘器官，而是一段矩阵计算

如果输入向量是 x1, x2, x3，某一行权重是 w1, w2, w3，再加上 bias，就得到一个数；这个数经过 ReLU、GELU 等 activation function，就是常说的一个 neuron 输出。

把这些操作画成圆圈叫神经元，听起来很像人脑；但在这里你先把它理解成矩阵乘法、加 bias、过非线性函数就够了。

y = activation(xW + b) x 是当前位置向量，W 和 b 是参数，activation 可以是 ReLU、GELU 等人为设计的函数。

另一个看法：FFN 像 key-value memory

课程提到一篇论文把 Transformer feed-forward layer 看成另一种 attention 或 key-value memory。这个视角不要求本节完全掌握，但它提醒你：FFN 不是“无聊的两层线性层”，它可能存着很多模式和知识。

模块最小直觉常见错法
attention找上下文里谁重要，并融合 value。把 attention weight 当成完整因果解释。
FFN对每个位置的向量做非线性加工。以为它不重要，只是附属层。
residual保留原向量，降低信息丢失和训练困难。忘记它导致“模型会忘掉 token 本身”的误解。

最小例子：Hugging Face 实作解剖路线

EXAMPLE

这一节的实作重点不是训练，而是“能把代码输出对应回模型结构”。你看到参数名、shape、hidden states、attentions 时，要能知道它们来自课堂中的哪一块。

代码阅读：列出参数与形状inspect parameters
# model 是 Llama，model2 是 Gemma / 另一个模型
print(model.num_parameters())

for name, param in model.named_parameters():
    print(name, param.shape)

state = model.state_dict()
weight = state["model.layers.27.mlp.up_proj.weight"]
print(weight[:100, :100])

读法：num_parameters() 看参数总量；named_parameters() 列出每组参数的名字和 shape；state_dict() 可以取出某一组真实数值。直接看数值通常看不出语义，但 shape 和名字能帮你定位它属于 embedding、attention 还是 FFN。

代码阅读：拿出每层 representationhidden states
inputs = tokenizer("大家好", return_tensors="pt")
outputs = model(**inputs, output_hidden_states=True)
hidden_states = outputs.hidden_states

for i, h in enumerate(hidden_states):
    print(i, h.shape)

读法：第 0 个 hidden state 通常对应 token embedding；后面才是一层层 Transformer 的输出。shape 里的三个数字通常可以读成：batch 大小、sequence length、hidden dimension。

字幕中的几个实作数字

  • Llama 3B 级模型约 32 亿参数，输入 embedding 约 128256 × 3072，Transformer 约 28 层。
  • Gemma / 另一模型约 43 亿参数，文字部分 embedding 约 262208 × 2560，层数约 34 层。
  • Q/K/V 的 shape 不一定完全对称，可能涉及 grouped-query attention 等节省参数的设计。
  • 有些模型没有单独 LM head，因为它直接复用输入 embedding table。
代码阅读：导出 attention weightsattentions
model.config.attn_implementation = "eager"
outputs = model(**inputs, output_attentions=True)
attentions = outputs.attentions

layer = attentions[5]      # 第 6 层
head = layer[0, 6]      # 第 7 个 head: seq × seq
print(head.shape)

读法：attention 通常是四维：batch、head 数、query 位置、key 位置。字幕里 Llama 某层 attention 形状近似 1 × 24 × 12 × 12，表示 24 个 head，每个 head 都是一张 12×12 的关系表。

可视化读法

VIZ

temperature 条形图

看的是 logits 经 temperature 缩放后，各候选 token 的采样权重怎样变化。T 变小，最高分 token 更集中；T 变大，低分 token 也更容易被抽到。判断时不要把条形图读成“模型确信世界真相如此”，它只是采样权重。

曲线模拟的是不同上下文里的 apple representation 随层数分化。归一化曲线更适合看语义分离趋势；未归一化曲线可能被不同层的向量分布影响。判断时先问：比较的是同层、同位置、同一种归一化方式吗？

attention matrix

横向是被看的 token，纵向是当前 token；右上角空白表示 causal mask。深色格子说明某个 head 在这个位置给了更高权重。它能提示关系，但不能单独证明因果，最好结合输出、隐藏状态或干预实验一起看。

误区诊断

DIAGNOSIS

七个常见坑

  • 把 logits 当概率。错因：看到“分数高”就以为是概率；修法：先过 softmax 或说明只是 raw score。
  • 把 attention weight 当完整解释。错因：图很直观；后果：过度解读；修法：把它当线索，结合输出和干预验证。
  • 以为 token embedding 已经理解上下文。错因：embedding 本身有语义邻近；修法：区分第 0 层和通过 layer 后的 contextualized representation。
  • 忘记 causal attention 只能看左边。后果：误判第一个 token 为什么在不同句子中相同。
  • 看到参数矩阵数值就想直接读语义。后果：陷入无意义数字；修法：优先看名字、shape、相似度、层间变化和可视化。
  • 把 representation engineering 当成安全绕过教程。修法：理解机制和风险，研究用途是解释、评估与防护，不是鼓励滥用。
  • 死记 Llama/Gemma 的某个 shape。修法：记读法，不记答案；模型架构会变，读 shape 的能力更重要。
易混概念区别
embedding / representationembedding 常指查表得到的初始向量；representation 更泛，包含每一层输出。
logits / probabilitylogits 是原始分数；probability 是经过转换后用于采样的非负归一化数值。
attention / FFNattention 跨 token 融合上下文；FFN 在每个位置内部做非线性加工。
query / key / valuequery 是当前位置要找什么；key 是每个位置可被匹配的标签；value 是真正被加权汇入的信息。
logit lens / patch scopelogit lens 把向量读成 token；patch scope 用另一个 prompt 把向量读成更完整的文本。
