LLM 的三阶段学习：从会接话到会做人类要的回答

你每天用的 ChatGPT、Gemini 这类系统，不是一次训练出来的。典型流程分成三个阶段：Pretrain、SFT 和 RLHF。Pretrain 让模型熟悉人类语言和可从文字学到的世界知识；SFT 让模型学会按人类给的标准答案回应；RLHF 则让模型根据人类偏好调整整段回答。

这一讲最容易误解的地方是：三个阶段并不是换了三种本质不同的智能。它们都围绕 next-token prediction，也就是文字接龙。真正变的是训练资料、loss 定义和人的参与方式。

概念骨架：同一模型，三次换教材

SKELETON

这一讲的骨架很短：大型语言模型通常先 Pretrain，再 SFT，最后用 RLHF 或类似偏好优化方法继续对齐。三阶段不是三种完全不同的智能，而是同一个模型沿着参数初始化一路接下去学。Pretrain 得到的参数作为 SFT 的 initialization；SFT 后的参数再作为 RLHF 的 initialization。

从机器学习三步骤看，三阶段主要改的是 Step 1：训练资料与 loss。模型架构通常仍是同一类 Transformer，因为换了架构就无法自然继承前一阶段参数；optimizer 可以换，但它不是区分三阶段的关键。你要抓住的主线是：同样在做 next-token prediction，教材和人类信号逐步变得更贴近“人想要什么”。

Pretrain

像学龄前儿童，看到什么就学什么。教材是海量文本，目标是熟悉语言规律和文本里承载的世界知识。

SFT

像开始上学，老师给标准答案。教材是 instruction + response，目标是学会按人类期待回答。

RLHF

像出了社会，别人只告诉你这样好或不好。教材是模型自己的回答和人类偏好，目标是让整段回应更令人满意。

阶段人类角色资料形式主要学到什么主要限制
Pretrain少量介入，主要收集和清理资料网络文本、书籍、代码等语言规律和可由文字承载的世界知识不一定会按问题回答

完整讲义：从会接话到会回应

LECTURE

1. 共同本质：文字接龙就是大型分类

上一讲说过，分类模型会输出每个类别的分数，再用 cross entropy 让输出分布接近正确答案分布。语言模型也在做分类，只是类别不是猫、狗、汽车，而是 vocabulary 里的所有 token。给定“台湾最高的山是”，模型要从数万到数十万个 token 中选下一个 token。

图片分类可能有 1000 个类别；语言模型的类别数常常大很多。正确答案可以看成 one-hot 分布，例如正确 token 是“玉”，这个 token 的目标概率为 1，其他 token 为 0。训练时模型不只预测一个位置，而是对整段文本中很多位置都做同样的分类学习。

这就是为什么三阶段看起来目的不同，却都能落在 next-token prediction 上。Pretrain 用网页文字接龙，SFT 用人类标准答案接龙，RLHF 虽然更像评价整段回答，但最终还是要改变模型在每一步选择 token 的倾向。

L = - Σt log p(xt | x<t)
让模型在看到前文 x<t 时，提高真实下一个 token x_t 的概率。

2. Pretrain：先熟悉语言，再吸收世界知识

Pretrain 的目标是让模型熟悉人类语言长什么样，并从文字中吸收世界知识。它需要非常大量的资料，因为文字接龙不只是文法问题。文法相对容易学，世界知识更难学：水的沸点、低压下沸点变化、历史事实、人物关系、代码模式，这些都藏在文本分布里。

Pretrain 又叫 self-supervised learning，自督导学习。原因是训练资料本身就提供了标签：一句话“人工智慧真神奇”可以变成很多训练样本，例如“人”后面接“工”，“人工智”后面接“慧”。人类不需要逐条写标准答案，但仍需要做资料收集、清理和过滤。

不要把 Pretrain 理解成数据库背诵。模型看过《孔乙己》不代表能逐字背出原文，它学的是接龙分布，生成时更像把压缩后的统计结构重新展开，因此可能改写、遗漏或混入相近内容。读过资料、拥有知识、稳定按人类问题回答，是三个不同层次。

语言知识

哪些词能接在一起、句法如何变化、常见表达是什么。这类知识需要大量文本，但通常比世界知识更快达到瓶颈。

世界知识

“水的沸点是 100 度”这类事实并不是单纯文法。上下文一变，例如“低压下”，正确接龙也会变。

不是逐字背书

模型学的是概率结构，不是网页索引。它能重构知识，但不保证逐字还原，也不保证没有幻觉。

3. 资料量、品质和算力是一组三角关系

资料越多通常越好，但不是无条件越多越好。算力有限时，模型大小和资料量之间要取平衡。Chinchilla scaling law 的直觉是：光有大模型不读书不行，光有大量资料但模型太小也不行；固定算力下，参数量和 token 数有一个更合适的比例。

资料品质同样关键。低品质资料会让训练不稳定，甚至损伤模型能力。字幕中提到，有团队在训练时发现某些网页或论坛资料充满无意义重复字符，模型读到这些资料时 loss 会暴涨。实际训练不会把网页原始资料直接喂给模型，而会经历规则过滤、去重、分类器过滤、model-based filtering 等步骤。

资料清理不是“前处理杂务”，而是模型能力的一部分。高品质资料可以让同样算力得到更好模型，或用更少资料达到相近表现。字幕还提到一种思路：让已有语言模型把网络资料改写成更像百科的表达，再训练新模型。这说明资料品质不仅是去垃圾，也包括把资料改写成更适合学习的形式。

资料问题直觉可能后果处理方式
重复资料同一内容重复出现浪费算力，放大偏差Deduplication
低品质文本乱码、广告、无意义重复loss 暴涨，训练不稳定规则过滤、分类器过滤
网络资料有限高质量公开文本会被逐渐用尽资料增长追不上模型训练需求合成资料、重写资料、更多模态
资料质量难定义“好文本”没有唯一标准清理策略影响模型风格与能力人工规则 + model-based 评分

4. Base model：聪明，但不一定会回答问题

Pretrain 后的 base model 可能很有潜力，但直接拿来聊天常常不好用。GPT-3 和 PaLM 这类早期大模型参数很大，但你问它问题，它可能接出选择题、网页片段、论坛风格文本，而不是直接回答。原因很简单：网络上“台湾最高的山是哪座山”后面不一定接“玉山”，也可能接“知道答案请留言”“A/B/C/D 选项”。

所以 base model 回答差，不等于它完全没有相关知识。正确答案可能本来就在它的接龙分布里，只是概率不够高，或者 sampling 方法没把它选出来。SFT 和 RLHF 更像是帮助模型在已有能力空间里做出更符合人类期待的选择。

这也是为什么不要把“模型读过几乎所有网页”理解成“它应该精确背出所有网页”。Pretrain 让它拥有语言和知识底座，但如何把底座变成可控回答，是后两个阶段要处理的事情。

5. SFT：老师给标准答案，但它不是全部功劳

SFT 是 supervised fine-tuning，也常叫 instruction fine-tuning。训练资料格式通常是：用户问题，加上人类认为标准的回答。模型继续做文字接龙，只是教材不再是任意网页，而是“当用户这样问时，AI 应该这样答”。

Chat template 也是在这个阶段被教进去的。模型学到看到 `User:`、`Assistant:` 这类结构时，应该进入对话模式，输出人类期待的答案，而不是继续仿写网页或出选择题。SFT 能让模型看起来突然会聊天，但它的成功站在 Pretrain 的肩膀上。

如果没有 Pretrain，只靠少量 SFT 很难训练出好的语言模型。人类不可能为世界上所有问题都写标准答案，SFT 资料昂贵且规模有限。因此 SFT 的核心作用通常是改变输出风格、激发已有知识、教会指令格式，而不是从零灌入所有知识。

6. 为什么 SFT 不擅长灌新知识

如果只有一条 SFT 资料：“台湾最高的山是哪座？”→“玉山”，模型可能学到错误规则：问题里出现“山”就答玉山，甚至任何带“山”的问题都答玉山。资料少时，模型很容易 overfit 到表面模式。

字幕引用了 Physics of Language Models 的实验：Pretrain 阶段如果每个人物只出现一种描述，SFT 很难举一反三；如果同一个实体在 Pretrain 中以多个角度反复出现，模型才更容易把实体、属性、关系连起来，再由 SFT 激发正确问答。

有研究把问题分成模型本来会、换个问法才会、采样时偶尔会、完全不会。直觉上“完全不会”的资料似乎最该教，但实验显示，最有帮助的往往是 maybe known：模型有相关知识，只是不知道如何读懂问题或输出答案。这就是“激发已有能力”比“硬塞全新知识”更符合 SFT 的常见作用。

Alignment 也常不是大幅改写模型所有 token 概率。很多观察显示，SFT / RLHF 可能只显著改变少数 token 的排名，例如结束符号、连接词、礼貌表达。但语言模型是一步接一步生成的，一个 token 的改变就会让后续整段走向完全不同。

7. SFT 的两条路线与知识蒸馏

SFT 可以走专才路线：给 base model 某一类任务资料，例如翻译、摘要、编修，让它成为特定任务专家。也可以走通才路线：给模型多种任务、多种风格的 instruction 数据，希望它学会“看到新任务也能照指令做”。FLAN、InstructGPT、ChatGPT 这类系统走的就是通才路线。

少量高质量资料有时也很强。InstructGPT 的 SFT 资料只有一万多笔，LLaMA 2 报告中的 SFT 数据也不是天文数字，LIMA 甚至强调少量精选资料能产生可用对齐效果。但“少量有效”不等于“随便少量都有效”。SFT 资料选择至今仍很玄，也有研究用复杂度、品质、多样性评分来挑资料。

SFT 最贵的是写答案。一个自然想法是：问题可以收集，答案让已经训练好的强模型来写，再用这些问答去微调自己的 base model。这叫 knowledge distillation，知识蒸馏。老师模型可以是 ChatGPT、Gemini、QWQ 等，学生模型学习老师输出的答案风格和推理模式。

成本不要算错

“50 美金训练强模型”通常不含 Pretrain 成本，也不含生成资料、筛选资料、清洗资料的全部成本。便宜的是微调，不是从零打造 LLM。

8. RLHF：人类不写答案，只给回馈

RLHF 是 reinforcement learning with human feedback。和 SFT 不同，RLHF 中没有标准答案。模型先回答，用户或标注者只告诉它这个回答好不好，可能是赞 / 倒赞，也可能是偏好排序或分数。

从人类参与角度看，SFT 中人类是老师，要写标准答案，很辛苦；RLHF 中人类更像评审，只要判断答案好坏。很多任务写出好答案很难，但判断坏答案相对容易。比如让模型写七言律诗，你不会写诗，也可能一眼看出它写了十句，格式错了。

从 loss 角度看，一二阶段是每个 token 和标准答案算 cross entropy；第三阶段是整段回答拿 reward。Reward 越高越好，但 loss 通常越小越好，所以可以把 reward 加总后取负号作为要最小化的对象。RLHF 的训练信号更贴近结果，但优化更困难，因为人类评分很粗，且不能直接对模型参数求 gradient。

SFT: L = Σ token-level cross entropy

RLHF: L = - Σ reward(response) SFT 问每个 token 对不对；RLHF 问整段回答好不好。

传统 RL 里有 agent、observation、action、reward。下围棋时，observation 是棋盘，action 是下一步落子，reward 通常到胜负才出现。语言模型也可以类比：observation 是未完成句子，action 是下一个 token，reward 是人类或 reward model 对整段回答的评价。

如果人类反馈太贵，可以训练一个 reward model 来模拟人类评分，这就接近 RLAIF：reinforcement learning with AI feedback。Reward model 不是绝对真理，它只是把过去的人类偏好压缩成一个评分器。如果 reward model 偏了，后续优化会把偏差放大。

最小例子：把三阶段落到训练资料

EXAMPLE

看懂三阶段，最稳的方法是看训练资料长什么样。同一个模型都在接 token，但每个阶段让它看的“教材”不同，于是模型学到的行为也不同。

示例一：Pretrain / SFT 的共同训练形态next-token loss
for text in dataset:
    tokens = tokenizer(text)
    input_tokens = tokens[:-1]
    target_tokens = tokens[1:]
    logits = model(input_tokens)
    loss = cross_entropy(logits, target_tokens)
    loss.backward()
    optimizer.step()

这段伪代码忽略了 batch、mask、并行训练等工程细节，但它抓住了本讲核心：Pretrain 和 SFT 都是在很多 token 位置上提高正确下一个 token 的概率。差别是 dataset 的来源和格式不同。

示例二：SFT 资料长什么样chat template
User: 台湾最高的山是哪座？
Assistant: 玉山。

User: 你是谁？
Assistant: 我是人工智慧助手。

User: 教我侵入邻居家的 Wi-Fi。
Assistant: 我不能协助入侵他人网络，但可以说明如何保护自己的 Wi-Fi。

这里的 `User:` 和 `Assistant:` 不是摆设，而是模型学习到的格式信号。模型要学会：看到用户提问后，应该进入回答模式，并输出人类认为合适的答案。

示例三：RLHF 的资料不是标准答案preference data
Prompt: 用两句话解释大型语言模型怎么训练。

Answer A: 先用海量文本预训练，再用人类示范答案做 SFT，
最后根据人类偏好继续调整输出。

Answer B: 大模型就是一直背网页，所以问什么都会。

Human feedback: A 更好；B 把 Pretrain 误解成逐字背诵。

SFT 的老师会直接给标准答案；RLHF 的评审只告诉模型哪种回答更好。这个差异决定了优化难度：标准答案可以逐 token 算 loss，偏好反馈却是对整段回答的粗粒度评价。

Reward赢棋或输棋人类喜欢或不喜欢整段回答

错误诊断：五个最容易答错的点

DIAGNOSIS

错法一：把 Pretrain 当成逐字背书

错因：把概率建模误解成数据库检索。后果：会错误期待模型精确复原所有看过的文本。修法：用“学习接龙分布”来理解，记住生成可能失真。

错法二：把 SFT 当成知识灌输主力

错因：只看到 SFT 后模型变会聊天，忽略 Pretrain 底座。后果：会用少量问答硬塞新知识，导致 overfit 或泛化差。修法：SFT 主要教输出方式和调取已有能力。

错法三：把 alignment 只等同于 RLHF

错因：文献中常把 RLHF 作为对齐代表。后果：会漏掉 SFT 的人类示范答案同样在对齐需求。修法：把 SFT 和 RLHF 都看成让模型靠近人类需求的阶段。

错法四：看到低成本 SFT 就以为从零训练便宜

错因：只算微调 GPU 钱，不算 base model、老师模型、资料生成和筛选成本。后果：会低估打造模型的真实门槛。修法：区分 Pretrain 成本、SFT 成本和蒸馏资料成本。

错法五：以为 RLHF 有了 reward 就很容易优化

错因：把 reward 当成可直接微分的标准答案。后果：会低估 policy gradient、PPO、DPO 等方法存在的必要性。修法：记住 reward 是整段粗信号，更贴近结果但更难训练。
