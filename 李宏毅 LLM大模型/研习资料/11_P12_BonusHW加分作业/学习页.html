Bonus HW：NVIDIA DLI 加分作业导航

这节说明的是加分作业：课程提供 NVIDIA DLI 线上学习课程的 redemption code，让正式修课学生可以免费学习付费课程。完成课程后，需要把完课证明上传到 NTU COOL，才能获得加分。

这类作业的难点不是“会不会写代码”，而是“是否合规”。你要选对课程、用对账号命名、控制使用次数、截到必要信息、按时上传证明，并且不能转让或造假。

本块学完能做什么

  1. 说清 NVIDIA DLI 加分作业的完整流程。
  2. 知道账号姓名字段为什么要包含姓名和学号。
  3. 区分付费课程、免费课程、redemption code 和证书。
  4. 判断完课证明截图是否足够核验。
  5. 列出会导致失去加分资格或诚信违规的行为。

概念骨架：这不是“看完课”而是“可核验完成”

SKELETON

这份 Bonus HW 的核心不是 NVIDIA DLI 课程本身有多难，而是把一个外部学习平台的完成记录，转化成课程助教可以认列的加分证明。直觉上，你要同时满足三层要求：平台上真的完成、身份能对得上、证明能交到课程系统。

正式地说，本作业是一条合规链路。链路里任何一个环节断掉，都可能导致“你确实学了，但课程无法给你加分”。例如你完成了课程却忘记上传到 NTU COOL，助教仍然拿不到可认列的提交；你截图里没有姓名或学号，后台也可能无法确认完成记录属于你。

进入 NVIDIA DLI→ 注册账号→ 姓名含学号→ 选择付费课程→ 输入 redemption code→ 完成课程→ 截图证明→ 上传 NTU COOL

学习平台层

NVIDIA DLI 提供多种机器学习、生成式 AI 和 GPU 加速课程。你需要选择本次规则认可的付费课程，并用课程提供的 code 完成注册。

课程规则层

一门加五分，两门加十分，不能超过两门。免费课程、旁听生使用 code、转让 code 都不在规则允许范围内。

证明核验层

证书或 100% 完成截图必须能显示“谁、哪门课、什么时候、是否完成”，最后还要上传到 NTU COOL。

你现在应该能做什么：看到一个“我已经看完 DLI 课程”的说法时，不急着认定它可加分，而是检查它是否同时满足平台完成、身份可核验、证明已提交这三个条件。

术语表：先把规则词汇讲清楚

TERMS

这节课出现的关键词不多，但每个词都和“能不能被认列”有关。学习时不要只记中文翻译，要记它在作业流程里的作用和常见误解。

术语直觉解释正式作用常见误解
NVIDIA DLINVIDIA 的线上学习平台。本次 Bonus HW 的外部课程平台，提供机器学习、生成式 AI、GPU 加速等课程。以为只要在 DLI 学任意课程都能加分；字幕明确强调要选付费课程。
Redemption code抵用码，把原本收费的课程费用抵掉。正式修课学生通过 NTU COOL 小组公告取得，用来免费注册付费课程。把它当成无限使用券，或转给非正式修课者使用。
付费课程原本需要付款购买的 DLI 课程。本次加分作业认可的课程类型。30、60、90 美元等价格都可用 code 全额抵免。以为免费课也能提交；免费课可以自学，但不能当本作业证明。
完课证明能证明你完成课程的截图或证书。上传到 NTU COOL 后，助教据此核验身份、课程、完成状态和时间。只截课程标题或进度条，不截姓名、学号或日期。
证书某些课程完成后生成的正式完成页。若课程提供证书，截图完整证书页面作为证明。以为每门课都有证书；字幕说明不是每门课都有。

NTU COOL课程管理与作业提交系统。发布小组 code、接收完课证明、让助教核验加分。完成 DLI 课程后不上传，以为平台后台会自动给分。

你现在应该能做什么：把“我注册了课程”“我看完课程”“我拿到证书”“我上传证明”区分开。它们不是同一件事，只有最后形成可核验提交，才算作业链路完整。

讲义：把作业规则拆成可执行动作

LECTURE

1. 为什么要使用姓名加学号

字幕反复强调注册时要在姓名字段里写自己的名字，并把学号附在后面。直觉上，这是给后台和助教一个稳定的身份锚点：课程平台知道某个账号完成了课程，但学校课程系统需要知道这个账号对应哪位修课学生。

如果你只写英文名、昵称或常用网名，平台记录可能是真的，但助教无法稳妥地把它映射到课程名单。正确做法是在 first name 或姓名相关字段中保留姓名，并附上学号；其他职业、学校、角色字段可以自由填写，身份字段不能随意省略。

2. 为什么必须选择付费课程

DLI 网站上课程很多，有生成式 AI 应用方向，也有 CUDA/GPU 加速等主题。课程允许你按兴趣选择付费课程，时长长短和作业难度不会影响加分。

但免费课程不能用于本次 Bonus HW。原因很简单：这次作业的机制是课程方提供 redemption code，让正式修课学生免费学习原本要付费的课程。免费课可以作为额外自学资源，却不是这份加分作业的提交对象。

3. Redemption code 的边界

Redemption code 可以全额抵免付费课程。字幕里提到，不管课程原价是 30、60 或 90 美元，只要是本次认可的付费课，code 都能抵掉费用。

但 code 数量有限，所以课程设置了最多两门的限制。一门加五分，两门加十分；使用超过两次 code，不是多学多拿，而是会失去本次加分资格。这个限制体现的是资源公平：如果少数人多用，其他正式修课同学可能就没有 code 可用。

4. 完课证明的两条路径

完成课程后还要提交证明。第一条路径是截图证书：如果课程提供证书，就截完整证书页面，让姓名和完成时间都可见。第二条路径是截图 My Learning 或课程记录：如果没有证书，就截课程显示 100% 完成的页面。

两种路径的共同点是“可核验”。截图最好同时包含姓名或学号、课程名称、完成状态、完成时间。只截一个漂亮的证书边角，或者只截一个 100% 进度条，都可能缺少必要信息。

5. 上传 NTU COOL 才完成闭环

DLI 平台完成课程只是外部平台上的事实，课程加分发生在 NTU COOL 这边。字幕明确说明，如果完成了 DLI 课程但没有把完课证明上传到 NTU COOL，也没有办法拿到加分。

所以执行顺序应当是：完成课程后立刻截图，检查截图字段，再上传到 NTU COOL。不要把“我有截图”误当成“我已经提交”，也不要把“平台显示完成”误当成“课程系统已经知道”。

6. 问题处理和诚信边界

如果遇到网站操作问题、redemption code 无法使用等公共问题，字幕建议先在讨论区回复询问；如果是个人账号或个人 code 无法使用这类只涉及自己的问题，再发邮件给助教，并在标题中包含 NVIDIA DLI，方便助教辨识。

最后一页强调两条红线：不能转让 code 给没有正式修课的同学，不能伪造证书或完课记录。这不是普通格式错误，而是诚信违规。规则型作业最怕的不是难题，而是为了省事破坏核验链路。

完整知识链路可以这样复述：正式修课学生从 NTU COOL 小组公告取得 code，用符合身份核验要求的姓名加学号注册 DLI 账号，选择最多两门付费课程，用 code 抵免费用，完成课程后截取证书或 100% 完成记录，再把能显示身份、课程、时间和完成状态的证明上传到 NTU COOL。你现在应该能做什么：把任何一个同学的完成方案按这条链路逐项检查，指出哪里合规、哪里会出问题。

来源边界：哪些来自字幕，哪些需要回到课程公告

SOURCE

本页直接来自字幕的规则

  • NVIDIA DLI 提供线上课程，作为本学期 Bonus HW。
  • 正式修课学生可通过 NTU COOL 小组公告取得 redemption code。
  • 账号姓名字段要包含姓名和学号，方便后台核验。
  • 必须选择付费课程；免费课程不能用于加分。
  • 一门加五分，两门加十分，不能使用超过两次 code。
  • 有证书截证书；无证书截 100% 完成记录；最后上传 NTU COOL。
  • 不能转让 code，不能伪造证书或完课记录。

需要以实际公告为准的部分

字幕由语音识别而来，出现了少量识别噪声，例如 NTU COOL 被识别成 NQ cool，redemption code 被识别成 reading code，TA 信箱相关词也不完全清楚。

时间范围也应以课程公告或作业页面为准。学习页保留“按时上传证明”的规则重点，但不替代 NTU COOL 上的正式截止时间、具体 code、提交入口和助教联系方式。

你现在应该能做什么：遇到实际执行信息，例如截止日期、code 字串、分组公告、上传入口，不从学习页猜答案，而是回到 NTU COOL 的正式公告核对。

例题讲解：判断一个提交是否可认列

EXAMPLES

例 1：合格提交

学生用自己的姓名和学号注册 DLI，选择一门付费生成式 AI 课程，用小组公告中的 code 抵免费用。课程完成后，证书截图里能看到姓名、课程标题和完成日期，并上传到 NTU COOL。

判断：可认列。平台完成、身份核验、证明提交都成立。

例 2：学了但不能加分

学生选择了一门免费的 DLI 课程，认真看完并做完练习，也截了课程完成页面。截图本身清楚，但课程不是付费课程，也没有用本次 code。

判断：不能用于本次 Bonus HW。免费课可以学习，但不是本作业认可对象。

例 3：完成但无法核验

学生完成一门付费课，截图只显示 100% 进度条和课程标题，没有账号姓名、学号或完成时间。助教无法判断截图是否属于本人，也无法确认完成时间。

判断：高风险。应重新截图，保留账号姓名/学号、课程、完成状态和时间。

问
如果没有证书，是否一定没办法加分？

不是。字幕明确给了替代路径：截图 My Learning 或课程记录中显示 100% 完成的页面，但截图仍要包含身份和时间信息。

问
如果已经用了第三次 code，但只想提交两门呢？

字幕强调的是使用超过两次 code 会失去加分资格，而不是“提交超过两门才失格”。所以执行前就要控制注册次数。

问
如果网站或 code 出问题怎么办？

公共问题先在讨论区询问；个人账号或个人 code 问题再发邮件给助教，并在标题中标明 NVIDIA DLI，方便被及时识别。

错误诊断：把常见坑提前排掉

DIAGNOSIS

会让加分失败的错误

  • 没有使用自己的名字注册账号。
  • 使用 redemption code 超过两次。
  • 上传证明没有姓名、学号或完成时间。
  • 完成课程但没有上传证明到 NTU COOL。
  • 选择免费课程作为加分证明。

严重违规

  • 把 redemption code 转让给非正式修课同学。
  • 伪造证书或完课记录。
  • 用他人的账号或证明冒充自己的完成记录。
错误说法错因后果修法
“我用昵称注册也没关系，证书上有我的账号。”身份不能和课程名单稳定对应。助教可能无法核验本人。注册时在姓名字段加入姓名和学号。
“我先注册三门，最后只交两门。”字幕限制的是使用 code 次数。超过两次会失去加分资格。注册前控制在最多两门。
“免费课也是 DLI，应该能算。”本作业指定付费课程加 code 的机制。免费课证明不被认列。选择付费课程并用正式 code 抵免。
“DLI 完成了，系统会自动知道。”外部平台和课程系统不是同一个提交入口。未上传 NTU COOL 就无法给分。完成后截图并上传作业系统。
“我把 code 给朋友，他也想学。”code 只给正式修课学生，数量有限。属于严重违规，可能影响加分资格。不要转让；让对方走自己的课程渠道。

你现在应该能做什么：看到一个错误提交时，能说出它错在哪里、会造成什么核验后果，以及最小修正动作是什么。
