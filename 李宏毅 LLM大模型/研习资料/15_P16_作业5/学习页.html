概念骨架

MAP

输入与标签

训练资料有 3200 个 labeled samples，测试资料有 800 个 unlabeled samples。每个游戏有 45 个 feature 和一个 label，label 是 Metacritic score。数值 feature 可以直接标准化后喂模型；文字 feature 要先通过 sentence embedding 变成向量。

判断句：模型训练时看得到 train label，生成提交时 test.csv 没有 label。

训练循环

神经网络可以看成带未知参数的函数。训练时把 feature 喂进 model 得到预测值，用 MSE 衡量预测值和真实分数的差距，再用 backpropagation 计算梯度，由 optimizer 更新 weight 和 bias。

判断句：模型参数由训练自动更新；epoch、learning rate、batch size、optimizer、weight decay 是你手动设定的 hyperparameters。

泛化与提交

训练 loss 下降不代表模型一定好。validation loss 用来估计没见过的数据表现；如果 training loss 继续降而 validation loss 上升，就可能 overfitting。最后模型对 test.csv 预测 Metacritic score，写入 submission.csv 上传 Judge Boy。

判断句：选 final submission 时不要只看 public score，也要参考 validation score。

validation split

regression预测一个连续数值。本作业根据游戏 feature 预测 0-100 的 Metacritic score。把它当成分类问题，只关心游戏好/不好。
feature / label输入线索和正确答案。feature 是游戏 ID、发行日期、游玩时间、文字描述等输入；label 是训练时可见的 Metacritic score。在 test.csv 上寻找 label；测试资料本来就没有答案。
sentence embedding把文字特征压成向量。文字 feature 不能直接喂给普通数值模型，需要已训练好的句向量模型转成 fixed-length vector。以为文字列会自动被 linear model 理解。
MSE loss预测错得越远，惩罚越大。Mean Squared Error 是误差平方的平均值，训练时用于优化模型参数。只看误差绝对值，忽略平方会放大大错误。
RMSE把 MSE 开根号后的评估指标。Root Mean Squared Error 与原目标同量纲，越低代表预测越接近答案。以为 RMSE 越高越好；本作业越低越好。
optimizer帮模型更新参数的算法。SGD、Adam 都是梯度下降的变形，读取梯度后更新 weight 和 bias。把 optimizer 当成要预测的模型本身。
validation set训练中保留的小测试集。从 labeled data 切出来，不参与训练，只用来监控没见过资料上的表现。拿 validation set 继续训练，导致评估失真。
overfitting背训练资料，遇到新题变差。training loss 下降但 validation loss 上升，是常见警讯。只要训练 loss 很低就认为模型最好。
hyperparameter训练前由人设定的控制钮。epoch、learning rate、batch size、optimizer、weight decay、hidden dimension 都属于超参数。和模型自动学到的 weight/bias 混在一起。
feature selection挑真正有用的输入特征。可用 feature 与 label 的相关系数作参考，去掉无关或噪声特征。以为 feature 越多一定越好。

讲义主线

LECTURE

1. 回归任务：输入游戏特征，输出一个分数

本作业的目标是预测 Steam 游戏的 Metacritic score。这个分数是 0 到 100 的整数，反映游戏评价高低。模型输入不是一句 prompt，而是一组 feature：发行日期、价格、正负评价数、游玩时间、游戏名称、描述等。输出也不是类别，而是一个数值，所以它是 regression。

如果是分类，你会问“这个游戏属于哪一类”；如果是回归，你会问“这个游戏大概几分”。训练模型时，train.csv 提供 feature 和 label；test.csv 只有 feature，没有 label。最后你要把模型对 test.csv 的预测写成 submission.csv，上传到 Judge Boy。

你现在应该能做的是：看到一个机器学习作业，先说清楚输入是什么、目标值是什么、训练资料和测试资料有什么差别。

2. 数据预处理：数值可以标准化，文字要转向量

数据集共有 46 个 column，其中 45 个是 feature，1 个是 label。大部分 feature 是数值，例如 Steam ID、发行日期、平均游玩时间、价格、正负评价数。数值 feature 通常要做标准化，让不同量级的特征不会让训练变得不稳定。

另外有 4 个 text feature，例如游戏名称和描述。普通 linear model 或 MLP 不能直接吃一段文字；它们要吃 vector。课程提供的 Colab 可以用 sentence embedding model，把一句话压成一个向量，再和数值 feature 拼起来。这里的 embedding 不是为了生成文字，而是把文字特征变成回归模型能计算的输入。

你现在应该能做的是：解释为什么 train.csv 里的文字列不能直接喂给数值模型，以及为什么 test.csv 如果使用文字 feature，也必须做同样的 embedding 处理。

3. 带未知参数的函数

线性模型可以写成一组 weight 和 bias，对输入 feature 做加权组合。神经网络也是函数，只是参数更多、结构更复杂。训练不是手写规则，而是让算法调整这些未知参数，让预测分数靠近真实分数。

判断句：weight/bias 是模型学出来的参数，不是你手动调的超参数。

4. MSE loss 与 RMSE

MSE 是预测值和真实值误差平方的平均。平方会放大大误差，所以预测离答案越远，loss 增长越明显。RMSE 是 MSE 开根号，单位回到原来的分数量纲，Judge Boy 用类似 RMSE 的指标衡量回归表现。

判断句：MSE/RMSE 都是越低越好。

5. Optimizer 与 backpropagation

forward 得到预测，loss 衡量错多少，backpropagation 计算每个参数该往哪边改，optimizer 根据梯度更新参数。PyTorch 已经把这套流程封装好，你要理解每一步的意义，而不是从零手推梯度。

判断句：没有 zero_grad / backward / step，训练循环通常不完整。

6. Hyperparameter 怎么调

epoch 控制训练跑几轮；learning rate 控制每次更新步伐；batch size 控制每次用多少样本估梯度；optimizer 可试 Adam 或 SGD；weight decay 用来抑制参数过大，降低 overfitting 风险。

判断句：学习率太大可能震荡，太小可能收敛太慢。

7. Overfitting 怎么看

如果 training loss 不断下降，但 validation loss 开始上升，模型可能记住训练资料细节和噪声，泛化变差。常见处理包括减少 epoch、降低模型复杂度、加入 weight decay，或重新选择更有效的 feature。

判断句：validation curve 比 training curve 更接近你对未知资料的期待。

8. Baseline 推进路径

simple baseline 只要跑通 Colab 并提交；medium baseline 要看 training curve 调参；strong baseline 可换 MLP；boss baseline 要做 feature selection，挑和 Metacritic score 更相关的 feature，减少噪声。

判断句：更复杂的模型和更多 feature 不一定更好，仍要看 validation 表现。

完整知识链路

把本节串起来，可以这样讲：作业5 用 Steam 游戏资料做回归，目标是预测 Metacritic score。先读 train.csv 和 test.csv，选择数值 feature，必要时把文字 feature 用 sentence embedding 转成向量；再定义 linear model 或 MLP，把 feature 喂进去得到预测分数；用 MSE loss 衡量预测和真实 label 的距离，通过 backpropagation 和 optimizer 更新模型参数；训练过程中用 validation set 监控是否收敛和是否 overfitting；最后把 test.csv 的预测写成 submission.csv 上传 Judge Boy。冲 baseline 时，先跑通，再调 hyperparameter，再换 MLP，最后做 feature selection。

最小例子

EXAMPLE

例子只保留最小结构，目的是让你看懂每一步在读什么、写什么、改变什么状态。

MSE = 1nΣi=1..n(ŷi - yi)2 RMSE = √MSE
例：预测 92、84、76，真实 87、80、68，平方误差为 25、16、64，RMSE = sqrt((25+16+64)/3) ≈ 5.91。
最小示例：PyTorch 回归训练 looppython
model = LinearModel(input_dim=num_features)
loss_fn = torch.nn.MSELoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)

for epoch in range(num_epochs):
    model.train()
    for x, y in train_loader:
        pred = model(x)
        loss = loss_fn(pred.squeeze(), y.float())

        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

    model.eval()
    with torch.no_grad():
        val_pred = model(val_x).squeeze()
        val_rmse = torch.sqrt(loss_fn(val_pred, val_y.float()))

逐行理解

model.parameters() 是 optimizer 会更新的 weight 和 bias。pred = model(x) 是 forward，把 feature 转成预测分数。loss_fn 用 MSE 衡量预测和真实 label 的距离。zero_grad 清掉上一轮梯度；backward 计算新梯度；step 才真正更新参数。validation 阶段用 no_grad，因为它只评估，不更新参数。

你现在应该能做的事：看到训练 loop，指出哪一行计算预测、哪一行计算损失、哪一行更新模型、哪一段只做验证。

模型输入维度在训练时已经固定。如果训练时用了 4 个数值 feature，预测时却多拼了文字 embedding，维度会对不上；如果训练时用了文字 embedding，预测 test.csv 时忘了同样处理，模型看到的输入结构也不一致。你现在应该能做的是：改 feature list 时，同时检查训练、validation 和 test inference 三个位置。

误区诊断

DIAGNOSIS

常见坑

  1. 把 test.csv 当成训练资料。修法：test 没有 label，只能用于最后预测；训练和调参要用 train/validation。
  2. 只看 training loss 下降就认为模型变好。修法：同时看 validation loss；分歧时警惕 overfitting。
  3. 把 hyperparameter 当成模型学出来的参数。修法：weight/bias 是训练更新的参数；epoch、learning rate、batch size、weight decay 是训练前设定的超参数。
  4. 训练时用了 text embedding，预测 test 时忘了同样处理。修法：train、validation、test 的 feature pipeline 必须一致。
  5. feature 全部塞进去就以为更好。修法：用相关系数和 validation score 做 feature selection，去掉明显噪声。
  6. 手动修改 prediction CSV 或使用额外网络资料。修法：遵守作业规范，只用给定数据和模型流程产生提交文件。
