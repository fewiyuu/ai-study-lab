快速了解大型语言模型

这节课用 ChatGPT 作为代表，把大型语言模型从“看起来很聪明的应用”拆回四个层次：它的能力展示、它的文字接龙机制、它的训练与对齐流程，以及人类怎样通过提示、资料、范例、工具和任务拆解让它发挥更稳定。

下面不逐字复述 80 分钟视频，而是抓住可迁移的主线：机制、训练、使用方式和边界。读完后，你应该能解释为什么同一个问题每次答案不同、为什么模型会编造网址、为什么预训练之后少量人类示范就能大幅改善模型，以及为什么“让模型一步一步想”有时真的有效。

能力与失误→文字接龙→token 与采样→预训练 / SFT / RLHF→提示、工具与反省

学完后要能做四件事

  1. 用文字接龙解释大型语言模型的基本运作。
  2. 说明 token、机率分布和随机采样如何影响输出。
  3. 区分预训练、督导式学习、RLHF 和 alignment。
  4. 把清楚需求、提供资料、给范例、拆任务、用工具、自我检查放到正确使用场景里。

概念骨架

MAP

能力不是全知

模型能答线性代数、写题、改写文本、读文件、生成报告，也会在陷阱题、假网址、过度自信说明里犯错。

机制不是魔法

它输入未完成文本，输出下一 token 的机率分布，再按分布采样。多轮对话就是把历史内容也接进上下文。

训练不是一步完成

先在大量文本上预训练，再用人类示范做督导式学习，最后通过人类偏好和 reward model 做 RLHF。

阶段输入模型学到什么典型风险
预训练网络与书面文本、代码等大量语料下一个 token 怎么接，语言、知识、代码模式不知道人类真正想要什么
督导式学习人类写的好回答问题后面应该接出有用答案，而不是乱续写或出选择题示范成本高、覆盖有限
RLHF人类偏好比较与 reward model提高人类更喜欢答案的机率，降低不喜欢答案的机率偏好模型也可能有偏差
使用阶段清楚指令、资料、范例、工具结果、上下文按当前上下文继续文字接龙仍可能编造、误读工具结果或过早结束

术语表

TERMS
术语直觉解释准确说法常见误解
大型语言模型很大的文字接龙模型。用大量参数和大量文本训练，主要预测下一 token 的语言模型。以为它直接查事实或完整规划答案。
ChatGPT最有代表性的对话式 LLM 应用。OpenAI 开发的对话应用，名字包含 Chat、Generative、Pre-trained、Transformer。以为所有类似模型只有 ChatGPT 一种。
文字接龙接下一个最可能的符号。给定上下文，预测下一 token 的机率分布，并逐步生成。以为模型一次输出完整答案。
token模型接龙时可选择的符号单位。由 tokenizer 预先定义，可是字、子词、符号片段，不同模型不一样。把 token 等同于一个中文字或一个英文单词。
机率分布每个候选 token 被选中的可能性。模型输出对所有候选 token 的分数或机率，再用采样策略选 token。以为输出只有唯一正确 token。
随机采样按机率掷骰子。根据机率分布抽取下一个 token，因此同问可能有不同回答。以为不固定答案表示模型没规则。
幻觉模型编出看似合理但不真实的内容。模型只根据上下文续写顺畅文本，未必验证事实。以为接入搜索或工具后就绝不会错。
上下文当前对话里模型能看到的文字。包括用户历史输入、模型历史输出、文件抽取文本、工具返回结果等。以为模型有永久记忆或总能读完整资料。
预训练正式上课前先大量预习。在大规模语料上自我学习文字接龙，是 GPT 中 P 的含义。以为预训练只是不重要的准备步骤。
督导式学习人类老师给标准答案。supervised fine-tuning，用人类示范让模型学会如何回答。以为只要模型变大就不需要人类示范。
RLHF人类告诉模型哪个答案更好。reinforcement learning from human feedback，用人类偏好训练 reward model，再优化语言模型。以为人类必须为每个问题写完整答案。
reward model学会模仿人类打分的模型。输入回答，输出偏好分数，用来替代大量人工反馈。以为它就是最终聊天模型。
alignment把模型行为调到更符合人类需求。常指 SFT 与 RLHF 等对齐流程，让模型更有用、更安全、更听指令。以为 alignment 只是在加安全口号。
工具调用让模型把部分任务交给外部工具。模型生成工具需求，工具返回结果，模型再据此续写回答。以为工具结果能自动消灭所有错误。
自我反省让模型检查自己的上一版回答。把回答再放回上下文，请模型找错、修正或用原则重新判断。以为每次道歉都代表真的发现错误。

现在你应该能做什么：把“模型会回答”拆成“上下文中有什么、下一 token 怎么选、训练时学过什么、人类怎样对齐它”。

讲义主线

LECTURE

1. 先看能力，也先看失误

课程一开始用线性代数期中题测试 GPT-4。结果很典型：简单题能答对，陷阱题可能蒙对或答错，而且解释里会出现不该假设的条件。比如题目没有说矩阵一定是方阵或可逆，模型却可能在说明中默认了这些条件。

这比单纯说“模型很强”更重要。大型语言模型确实能解题、出题、改写文字、模仿文体、做课程助理，但它不是一台保证正确的定理机。判断它的输出时，要同时看答案和推理过程。答案对但理由错，仍然是风险。

现在你应该能做什么：评估 LLM 输出时，不只看最终选项，还要读它有没有引入题目没给的假设。

2. GPTs 类应用说明“专属助理”变便宜了

老师展示了把课程作业资料上传后，快速做出一个线性代数课程助理。它能回答“作业二什么时候交”这类普通 ChatGPT 不可能知道的问题，因为它的上下文或知识配置里有课程资料。

这个例子说明时代变化：过去做一个专属课程助理要写系统、做检索、做问答模型；现在很多场景下先上传资料、写清角色和行为说明，就能得到可用原型。但“可用”不等于“可靠上线”。资料过期、提取错误、模型误读仍然需要检查。

现在你应该能做什么：把“通用模型能力”和“给它专属资料后的能力”分开判断。

3. ChatGPT 名字里的四个部分

Chat 指对话，一问一答；G 是 Generative，表示它生成文字；P 是 Pre-trained，表示先在大量资料上预训练；T 是 Transformer，是背后的神经网络结构之一。

这四个字母对应了应用形态、任务类型、训练流程和模型结构。对初学者来说，不必马上展开 Transformer 的数学细节，但要知道它是输入上下文、输出下一 token 分布的复杂函数。

现在你应该能做什么：用 Chat / Generative / Pre-trained / Transformer 四个词解释 ChatGPT 名字。

4. 语言模型真正做的是文字接龙

输入“台湾最高的山是哪座？”，模型不会一次把完整答案从某个数据库里取出来。它先预测问题后面接哪个 token 合理，可能接“玉”；再把“玉”接回上下文，继续预测“山”；再判断是否结束。

能做文字接龙的模型叫语言模型。它看起来像回答问题，是因为问题加答案也可以被看成一段未完成文本。人问的问题只是上下文的一部分，模型输出的是这段上下文后面最可能继续出现的符号序列。

现在你应该能做什么：解释为什么“回答问题”可以被语言模型改写成“续写文本”。

5. token 不是一个固定自然语言单位

模型不是直接在“完整英文单词”或“固定中文方块字”上接龙，而是在 tokenizer 定义的 token 上接龙。一个英文词可能被拆成多个 token；中文在不同模型里也可能被拆成字、子词或更细片段。

token 必须可穷举，因为模型要给每个候选 token 一个分数或机率。如果直接用所有英文单词当候选，词表会不断增长，无法稳定穷举。所以 tokenizer 的设计会影响成本、长度、语言表现和收费。

现在你应该能做什么：看到 token 计费或 token 限制时，知道它不是简单等同字数。

6. 随机采样解释了“同问不同答”

模型输出的是下一 token 的机率分布，不是单一答案。系统会按这个分布采样，像按权重掷骰子。一个 token 机率高，更容易被选中；但只要不是 100%，其他 token 也可能被抽到。

为什么不永远选最高机率？因为研究和实践都发现，永远贪心选择最高机率容易让文本重复、僵硬或陷入循环。适度采样能让回答更自然，但也带来随机性和不稳定。

现在你应该能做什么：解释为什么同一个 prompt 可能生成多个不同版本。

7. 幻觉来自“顺畅续写”不等于“事实验证”

台大玫瑰花节的例子很典型。用户要求模型介绍一个不存在的活动并提供网址，模型会根据“台大、玫瑰花节、网址”这些词续写出看似合理的介绍和链接。它追求的是文字接得顺，不是主动证明活动存在。

即使用搜索或外部资料，模型仍然是在工具返回结果基础上继续文字接龙。工具能降低错误率，但不能保证零错误。它可能误读搜索结果，把“全台最大玫瑰园”误连成“台大玫瑰花节”。

现在你应该能做什么：遇到网址、论文、日期、事实清单时，把模型输出当待核验文本。

8. 多轮对话靠上下文拼接

你先让模型用 30 字介绍大型语言模型，再说“翻译成英文”，它知道要翻译上一轮答案，不是因为它有神秘记忆，而是同一段对话里的历史输入和历史输出会一起放进上下文。

上下文窗口越长，模型能参考的历史就越多。课程提到当时新版本已经能读很长的文本。但上下文不是无限记忆：太长会贵、会慢，也可能因截断或注意力分散而漏信息。

现在你应该能做什么：设计对话时，把需要模型参考的信息放进当前上下文，并检查它是否真的使用了这些信息。

9. 预训练：让模型先大量学习文字接龙

每句话都能成为训练资料。句子“人工智慧真神奇！”可以拆出很多学习信号：看到“人”后面该提高“工”的机率，看到“人工”后面该提高“智”的机率，看到完整句子后面该提高结束符号的机率。

网络上有大量文本和代码，这些都是语言模型学习接龙的材料。GPT-1、GPT-2、GPT-3 的历史显示，参数量和训练资料量大幅增长后，模型学到的语言模式、知识和代码能力也会提升。GPT-3 会写代码并不是凭空出现，而是训练资料里有大量注释和代码的对应模式。

现在你应该能做什么：解释为什么预训练模型读过大量代码后能根据注释续写程序。

10. 规模增长带来能力，但不自动带来“听话”

GPT-3 参数量高、训练语料规模庞大，但很多任务表现仍让人失望。原因之一是它只是在网络上自由学习文字接龙，并不知道人类希望它回答问题、遵守格式、不要乱出选择题。

这解释了为什么“模型大”不是全部。一个读过很多资料的模型，可能像在野外长大的孩子：知道很多文本模式，却不懂人类任务规则。让它变成可用助手，需要下一阶段的人类示范和偏好对齐。

现在你应该能做什么：区分“知识储备强”和“能按人类需求完成任务”。

11. 督导式学习：人类示范什么是好回答

督导式学习中，人类老师提供完整答案。比如有人问“台湾最高的山是哪座？”，人类告诉模型应该回答“玉山”，而不是出四个选项、写诗或说不知道。模型学到的是在这种上下文下提高正确回答 token 的机率。

课程强调，在大规模预训练之后，督导式学习不一定需要海量资料。InstructGPT 和 Llama 2 Chat 等案例都显示，几万笔高质量示范就可能显著改善行为。它像画龙点睛：不是重新教所有知识，而是教模型怎么服务人类需求。

现在你应该能做什么：说明 SFT 为什么能让小模型胜过没有人类指导的大模型。

12. RLHF：人类提供偏好，而不是完整答案

增强式学习阶段，人类不必写标准答案，只要比较两个回答哪个更好。模型随后学习提高好答案的机率、降低差答案的机率。这个流程比每题写完整示范更容易扩大。

但 RLHF 需要模型已经“偶有佳作”。如果模型所有答案都很差，人类只能从差中选不那么差的，优化效果有限。所以训练顺序通常是先预训练，再督导式学习，最后 RLHF。

现在你应该能做什么：解释为什么 RLHF 放在训练后段，而不是一开始就做。

13. reward model：先学会模仿人类偏好

实际 RLHF 通常会先训练一个 reward model。人类比较回答 A 和 B 后，这些偏好数据用来训练 reward model，让它学会给更符合人类偏好的回答高分。

有了 reward model，就不需要每次都请真人打分。语言模型生成回答后，reward model 打分；分数高的回答被强化，分数低的回答被压低。这个过程就是让模型行为更接近人类偏好。

现在你应该能做什么：把 RLHF 拆成“学偏好模型”和“用偏好模型优化语言模型”两步。

14. alignment：让模型对齐人类需求

督导式学习和 RLHF 合起来，常被放进 alignment 这个大概念里。对齐不是只让模型“礼貌”，而是让模型知道人类要的是有用、遵守指令、少害人、少胡说的回答。

GPT-3 经过人类示范和偏好学习后，才更接近 ChatGPT 这种可对话应用。GPT-4 的技术报告公开了大量能力评估，但没有完整公开训练细节，这也提醒我们：许多前沿系统的具体做法并不完全透明。

现在你应该能做什么：解释为什么 ChatGPT 不只是“更大的 GPT-3”。

15. 提示技巧一：把需求讲清楚

很多失败不是模型没能力，而是需求没说清。你让模型“润饰段落”，它可能重写整段；如果你只想修语法，就明确说“只修正文法错误，其他一字不改”。你让模型扩写英文段落，但用中文下指令，模型可能以为你要中文扩写。

清楚指令不是写玄学咒语，而是把目标、语言、长度、改动范围、格式和禁区写出来。模型按上下文续写，你不给约束，它就按常见模式自由发挥。

现在你应该能做什么：把模糊请求改成包含目标、范围、格式的可执行请求。

16. 提示技巧二：提供资料和范例

让模型写九份游记，如果只给题目，它只能自由编。你提供早上、下午、傍晚去了哪里，它就能按你的真实路线写。写报告也是一样：先收集资料，再让模型整理、改写、扩展，不要让它凭空编。

范例能教模型特殊风格。五条体、晶晶体这类网络文体，模型未必知道定义；给几条例子后，它就能模仿。范例不是答案泄漏，而是给模型说明“你希望的输入输出关系”。

现在你应该能做什么：遇到风格、格式、领域要求时，先给模型 2-3 个正例。

17. 提示技巧三：让模型展开中间步骤

鸡兔同笼例子说明，如果要求“不要列过程，直接给答案”，模型正确率可能很低；改成“请详细列出计算过程”，正确率显著提升。相关研究也发现 “Let's think step by step” 这类提示能改善某些推理题。

直觉上，逐步作答让模型在上下文中生成中间变量和约束，后续 token 可以参考这些中间结果。它不保证正确，但比直接跳答案更容易暴露错误，也更方便人检查。

现在你应该能做什么：遇到数学、规划、代码推理任务时，要求模型列步骤，并检查每一步。

18. 文件、工具和资料库能降低编造，但不能替你核对

模型可以读取上传文件、总结章节、生成投影片提纲，甚至通过工具查询论文资料库或搜索网页。文件上传的实质，通常是先把 PDF 或网页里的文字抽出来放进上下文；看到 analysis 之类的步骤时，模型还可能写一段小程序来处理文件或产出下载链接。

工具的价值是把事实检索、文件解析、代码执行交给更可靠的系统，再让模型整理结果。但工具返回结果仍会被模型续写、解释和压缩。它可能误读资料、遗漏限定条件、把相似结果当正确结果。语言模型加搜索不是万能保险，只是让它有更好的上下文。

现在你应该能做什么：把工具输出和模型解释分开检查。

19. 把大任务拆成小任务

写长篇小说、做学习地图、整理一批课程视频，这类任务太大。直接让模型一次完成，容易人设跑掉、结构混乱或卡住。更好的做法是先让模型做计划：背景设定、角色设定、章节规划，再逐章写。

模型也可以帮你把复杂任务拆成步骤。不过拆出来不等于都能执行。比如“观看每部影片并摘录内容”可能需要实际视频转写工具，语言模型本身做不到。拆任务后仍要检查每个子任务是否有可用输入和工具。

现在你应该能做什么：把“做一个大项目”改写成计划、资料收集、分块处理、检查合并四步。

20. 自我反省有用，但要看版本和题目

让 GPT-4 检查自己关于台大玫瑰花节的回答，它可能发现活动不存在并修正；再检查第二次，它也可能指出自己修正后的内容基本正确，只修小错。较旧模型则可能只是道歉，但更正内容不变。

所以自我反省不是神奇按钮。它的作用是把上一版回答重新放入上下文，让模型以检查者身份续写。强模型、清楚检查标准和外部资料会提高成功率；弱模型可能只学会“被质疑就道歉”。

现在你应该能做什么：用“请检查事实、列出不确定处、给出依据”替代单纯“你确定吗”。

21. 模型可以互相挑错，也可以接入现实动作

课程提到让一个模型回答，另一个模型挑毛病，再让原模型修正。这相当于把“批判者”放进流程。它能改善结果，但仍需要人或可靠标准检查。

再往前一步，模型可以把视觉或环境描述转成行动计划。例如看到桌上有笔电、耳机、笔记本和背包，模型能列出把物品放进背包的步骤；如果机械手臂只懂“移动、夹取、放下”，模型还可以把自然语言步骤转成更低层动作。这里的关键仍是：语言模型输出的是计划文本，执行环境必须能验证和约束动作。

现在你应该能做什么：区分“模型会写行动步骤”和“系统能安全执行动作”。

完整知识链路

大型语言模型的表面能力来自一个很朴素的机制：给定上下文，预测下一 token 的机率分布，再按采样策略逐步生成。它因此可以答题、写文、模仿风格、整理文件，也会因为只追求顺畅续写而编造事实。预训练让模型从大量文本和代码里学会语言与知识模式；督导式学习教它按人类期待回答；RLHF 通过 reward model 把人类偏好扩展到更多场景；alignment 让模型更接近“有用的助手”。使用时，人类也要配合：讲清需求、给资料和范例、要求步骤、接入工具、拆解任务、检查和反省。你现在应该能做什么：既不神化 LLM，也不把它当普通搜索框，而是按机制使用和检查它。

最小例子

EXAMPLE

这些例子不是要实现真实 LLM，而是把本节机制压成可读流程。

示例 1：下一 token 机率分布与采样python
context = "台湾最高的山是哪座？"
probs = model.next_token_distribution(context)

# probs 可能长这样：{"玉": 0.62, "阿": 0.08, "雪": 0.05, ...}
next_token = sample(probs)
context = context + next_token

next_token_distribution 表示模型输出所有候选 token 的分布。sample 不是固定选最高分，而是按机率抽样。抽到的 token 会接回上下文，影响下一步预测。

示例 2：训练三阶段的概念流程python
base_model = pretrain(text_corpus)
sft_model = supervised_finetune(base_model, human_answers)

reward_model = train_reward_model(human_preferences)
aligned_model = optimize_with_reward(sft_model, reward_model)

预训练让模型学会语言和知识模式；督导式学习让模型知道怎样回答人类问题；reward model 学人类偏好；最后模型根据偏好分数继续优化。删掉预训练，模型没有基础能力；删掉 SFT，模型可能不懂任务规则；删掉 RLHF，回答可能不够符合人类偏好。

示例 3：把模糊提示改成可执行提示text
模糊：
帮我润饰这段英文。

清楚：
请只修正文法错误和拼字错误，保持原句结构和语气。
不要扩写，不要重写整段。
请用表格列出：原句 / 修改后 / 修改原因。

模型按上下文续写。你不给范围，它会按常见“润饰”模式大改；你明确限制修改范围，它更容易输出你要的结果。

错误示例：工具结果未核对diagnostic
# 错误做法
answer = llm_with_search("介绍台大玫瑰花节，并给网址")
publish(answer)  # 没有检查活动是否存在，也没有打开网址

错因是以为接入搜索后模型一定正确。后果是模型可能把相似搜索结果误解成目标事实。修法是把事实、来源、网址逐项核验，并让模型标出“不确定”。

误区诊断

DIAGNOSIS

常见坑

  1. 答案对就认为模型懂了。错因：只看最终答案；后果：理由错误也被放过；修法：检查假设和推理。
  2. 把 token 当字数。错因：不了解 tokenizer；后果：误估成本和上下文长度；修法：按模型 token 规则估算。
  3. 以为同问不同答说明模型没规则。错因：忽略随机采样；后果：误解生成机制；修法：理解机率分布和采样。
  4. 以为工具能保证事实正确。错因：把工具结果和模型解释混为一谈；后果：发布错误来源；修法：逐项核验。
  5. 让模型自由写报告。错因：不给资料和约束；后果：内容编造或跑题；修法：提供资料、结构和格式。
  6. 把“让它一步一步想”当万能咒语。错因：把提示技巧神秘化；后果：步骤错了也照信；修法：让模型列步骤后逐步检查。
  7. 以为自我反省一定有效。错因：把道歉当成发现错误；后果：旧错误换一种说法；修法：给检查标准和外部依据。
