SEQ2SEQ · TRANSFORMER ENCODER · RESIDUAL + NORM

Transformer 上：从 seq2seq 到 encoder

这讲从作业五会用到的 Transformer 开始。Transformer 先被放在更大的框架里理解：它是一种 sequence to sequence model，也就是输入一段序列、输出另一段长度不固定的序列。语音辨识、机器翻译、语音翻译、聊天机器人、摘要、问答、文法剖析、多标签分类，很多任务都可以硬转成 seq2seq 问题。

本讲后半段聚焦 Transformer encoder。encoder 的职责是输入一排向量、输出另一排同长度向量。每个 encoder block 先做 self-attention，再做 feedforward network；真实 Transformer 还在这两步外面加 residual connection 和 layer normalization，并在最开始加入 positional encoding。decoder 留到下一讲。

seq2seq 任务→encoder / decoder→self-attention→residual + layer norm→encoder block 重复 N 次

  2. 能说清 encoder 和 decoder 的分工：encoder 处理输入序列，decoder 决定输出序列。
  3. 能画出 Transformer encoder block：self-attention → add & norm → feedforward → add & norm。
  4. 能解释 residual connection、layer normalization、positional encoding 的作用和常见误解。

概念骨架

MAP

1. seq2seq 是“输入序列到输出序列”

它处理的是输出长度由机器决定的任务。语音讯号长短和文字长度没有固定比例，中文句子和英文翻译长度也没有固定比例，所以不能预先规定输出几个 token。

判断句：只要输出长度不能由简单公式确定，就可以考虑 seq2seq 框架。

2. Transformer 通常拆成 encoder 和 decoder

encoder 读入输入序列，把它变成一排新的向量；decoder 再根据 encoder 的结果，逐步产生输出序列。本讲只讲 encoder，decoder 留到下一讲。

判断句：如果你把 encoder 说成“负责生成答案”，就把两半职责混了。

3. encoder block 不是只有 attention

一个 block 里有 self-attention、feedforward network、residual connection、layer normalization。原论文里还会把这个 block 重复 N 次，输入端还要加 positional encoding。

判断句：只说“Transformer 就是 attention”太薄，至少要补上 add & norm 和 feedforward。

Applications

语音辨识、翻译、语音翻译、聊天、QA、摘要、文法剖析。

Seq2seq

输入一段序列，机器自己决定输出序列长度。

Encoder

输入一排向量，输出一排同长度上下文化向量。

Block

self-attention + add & norm + feedforward + add & norm。

术语表

TERMS
术语直觉解释准确说法常见误解
sequence to sequence一段序列进，一段序列出。输入是 sequence，输出也是 sequence，且输出长度通常由模型自己决定；常简称 seq2seq。以为输入输出长度必须相同，或输出长度能提前固定。
encoder负责读懂输入序列。在 Transformer 中输入一排向量，经过多层 block 后输出另一排同长度向量。把 encoder 当成直接产生最终答案的模块。
decoder负责产生输出序列。接收 encoder 处理后的信息，并一步步决定输出什么 token；本讲只铺垫，不展开。把 decoder 和 encoder 混成一个“黑箱 Transformer”。
self-attention让序列内部各位置互相读取。每个位置根据整段输入序列更新自己的表示，适合 encoder 处理输入序列。以为 encoder block 只有 self-attention。
feedforward network每个位置各自再整理一次信息。fully connected network 接在 self-attention 之后，对每个位置的向量做非线性转换。忽略它，导致只会说“Transformer = attention”。
residual connection把输入直接加回输出。某层输出会和该层输入相加，形成新的向量；Transformer 图里的 Add 就对应这一步。以为它只是画图装饰，不影响深层网络训练。
layer normalization对单个向量内部做标准化。对同一个 example 的不同 dimension 计算 mean 和 standard deviation，再 normalize。把它和 batch normalization 混淆。
positional encoding给序列加入位置信息。由于 self-attention 本身不自带顺序概念，输入端要加入位置编码或位置向量。以为 attention 会自动知道 token 顺序。
task-specific model为特定任务定制的模型。seq2seq 像瑞士刀，能硬解很多问题；但语音辨识等任务常用 RNN Transducer 等专用架构表现更好。以为能用 seq2seq 解，就一定是最佳方案。

讲义主线

LECTURE

1. 为什么要先讲 seq2seq

作业五会用到 Transformer，而 Transformer 最早在课程脉络中常被放进 sequence to sequence 框架。seq2seq 的关键不是“输入输出都是文字”，而是“输入是一段序列，输出也是一段序列，且输出长度由模型决定”。

语音辨识里，输入是连续声学特征序列，输出是文字序列；机器翻译里，输入中文四个字“机器学习”，输出可能是英文两个词 “machine learning”；语音翻译甚至可以输入台语声音，直接输出中文句子。长度关系不是固定比例，所以需要模型自己决定什么时候停止。

你现在应该能做什么：判断一个任务是不是天然适合 seq2seq，关键看它是否需要从一段序列生成另一段不定长序列。

2. seq2seq 是通用瑞士刀，但不是总是最佳工具

很多 NLP 任务都能改写成 QA，再改写成 seq2seq。翻译可以问“这句英文的德文翻译是什么”，摘要可以问“这篇文章摘要是什么”，情感分析可以问“这段文字是正面还是负面”。文法剖析看起来输出是一棵树，但也可以用括号序列表示树结构，再让模型输出这串序列。

多标签分类也能硬转成 seq2seq：输入文章，输出它属于哪些 class，模型自己决定输出几个 class。甚至 object detection 也有研究把它改写成序列输出。这说明 seq2seq 很 powerful。

但老师也提醒：瑞士刀能做很多事，不代表每件事都最好用。语音辨识这类任务常有专门架构，例如 RNN Transducer，针对声学特性设计后可能表现更好。你现在应该能做什么：既能看见 seq2seq 的统一性，也不把它神化成所有任务的最优解。

3. encoder / decoder 的分工

一般 seq2seq model 分成 encoder 和 decoder。encoder 负责处理输入 sequence，decoder 负责根据 encoder 的结果决定输出 sequence。你可以把 encoder 想成“读题”，decoder 想成“作答”。

本讲只展开 encoder。decoder 为什么能一个 token 一个 token 生成、如何看 encoder 的结果，会在 Transformer 下继续讲。

你现在应该能做什么：在模型图上指出哪一半读输入，哪一半产生输出。

4. encoder 的基本输入输出

encoder 要做的是输入一排向量，输出另一排向量，长度相同。RNN、CNN、self-attention 都可以做到这件事；Transformer encoder 采用 self-attention。

每个输入位置经过处理后，会得到一个考虑整段序列的新向量。多个 block 叠起来，前一层输出会成为下一层输入。

你现在应该能做什么：把“输入声音或文字”抽象成“一排向量”，再说明 encoder 输出也是一排向量。

5. 一个 encoder block 的简化版

简化看，block 先做 self-attention：输入一排 vector，考虑整个 sequence 的信息，输出一排 vector。然后把这一排 vector 丢进 fully connected feedforward network，再输出一排 vector。

如果只记这个版本，容易以为 Transformer 就两步。但原始论文中的 block 更复杂，还包含 residual connection 和 layer normalization。

你现在应该能做什么：先画简化版 self-attention → feedforward，再在下一步补 Add & Norm。

6. residual connection：把输入加回来

在 self-attention 之后，Transformer 不是只拿 attention 的输出，而是把该层输入直接加到输出上。假设输入向量是 A，self-attention 输出是 B，新向量就是 A + B。

这种设计叫 residual connection，在深度学习中非常常见。直觉上，它让网络保留原始信息，也让深层网络更容易训练。

你现在应该能做什么：解释 Transformer 图里的 Add 不是“随便加个箭头”，而是输入和子层输出相加。

7. layer normalization：对同一个向量内部标准化

Add 之后还要做 normalization。这里不是 batch normalization，而是 layer normalization。它对同一个 example 的同一个向量，在不同 dimension 上计算 mean 和 standard deviation，再把每个维度标准化。

self-attention 后有一次 Add & Norm；feedforward 后还有一次 Add & Norm。图里的 “Add & Norm” 就是 residual connection 加 layer normalization。

你现在应该能做什么：说清 layer norm 和 batch norm 的计算对象不同。

8. 原始架构不是唯一最优

原论文的 encoder 顺序是一个重要基线，但不代表永远最优。后续论文讨论过 layer normalization 放在 block 输入处是否更好，也讨论过 batch norm 为什么在 Transformer 中不如 layer norm。

这提醒我们：学习架构时要先掌握原始设计，再知道它可以被改。背图不是终点，理解每个组件为什么在那里，才有能力读后续变体。

你现在应该能做什么：看到 pre-norm、post-norm 这类说法时，知道它们是在讨论 layer norm 的位置。

完整知识链路

把本节串起来，可以这样讲：Transformer 可以放在 sequence to sequence 框架中理解。seq2seq 的任务是读入一个 sequence，输出另一个长度由模型决定的 sequence；语音辨识、机器翻译、语音翻译、聊天、QA、摘要、文法剖析、多标签分类都可以被改写进去。一般 seq2seq 分 encoder 和 decoder，encoder 读输入，decoder 产输出。本讲讲 encoder：输入一排向量，先加 positional encoding，再通过 N 个 encoder block。每个 block 包含 multi-head self-attention、residual connection、layer normalization、feedforward network，再一次 residual 和 layer normalization。你现在应该能做什么：不看图也能复述 encoder block 的顺序。

SOURCE

线索 1：输出长度未知是 seq2seq 的入口

语音辨识、机器翻译、语音翻译都没有简单的输入输出长度对应关系。声音讯号多长、中文几个字、英文几个词之间没有固定比例，输出长度要由机器决定。

你现在应该能做什么：用“输出长度未知”解释为什么这些任务适合 seq2seq。

线索 2：台语声音直接转中文是端到端例子

老师提到用乡土剧资料训练台语声音到中文字幕的模型。即使资料有噪声、字幕未严格对齐，直接输入声音输出中文仍有可能做到，体现 seq2seq 的硬解能力。

你现在应该能做什么：说明端到端语音翻译为什么不一定需要先转台罗或文字。

线索 3：许多 NLP 可改写为 QA

翻译、摘要、情感分析都可被看成“文章 + 问题 → 答案”。再进一步，问题和文章拼成输入 sequence，答案就是输出 sequence。

你现在应该能做什么：把一个 NLP 任务改写成 QA 形式。

线索 4：seq2seq 能硬解树和多标签

文法树可以转成括号序列，多标签分类可以让模型输出多个 class，object detection 也能被序列化。关键是把非序列输出编码成 sequence。

你现在应该能做什么：给一个“看起来不是序列”的任务设计序列化输出。

线索 5：encoder block 的真实结构更复杂

简化版是 self-attention 再 feedforward；真实 Transformer 还要在每个子层后做 residual connection 和 layer normalization，也就是图里的 Add & Norm。

你现在应该能做什么：解释 Add & Norm 不是一个模糊名词，而是两步操作。

线索 6：架构设计可以被挑战

原始 Transformer 不是最终答案。后续研究讨论 layer norm 放在哪里、为什么不用 batch norm、有没有更好的 normalization。学习原图是起点，不是停止思考。

你现在应该能做什么：面对 Transformer 变体时，先比较它改了哪个组件或顺序。

最小例子

EXAMPLE

下面用伪代码把 Transformer encoder block 写成最小结构。它不追求运行，而是帮助你把图上的组件翻成步骤。

最小示例：Transformer encoder blockpython-like
x = token_embedding(input_tokens) + positional_encoding(input_tokens)

for block in encoder_blocks:
    attention_out = block.self_attention(x)
    x = layer_norm(x + attention_out)          # Add & Norm

    ff_out = block.feedforward(x)
    x = layer_norm(x + ff_out)                 # Add & Norm

encoder_output = x

逐行理解

第一行把 token 信息和位置信息合在一起。每个 block 有两个子层：self-attention 和 feedforward。每个子层后都不是直接结束，而是把子层输入加回输出，再做 layer normalization。

你现在应该能做什么：把原始 Transformer encoder 图中的 “Multi-Head Attention”“Add & Norm”“Feed Forward”对应到这段伪代码。

错误示例：漏掉残差和归一化diagnosis
# 错误想法：encoder block 只有两步
x = self_attention(x)
x = feedforward(x)

# 修正：每个子层后都要 Add & Norm
y = self_attention(x)
x = layer_norm(x + y)
z = feedforward(x)
x = layer_norm(x + z)

为什么这是错的

漏掉 residual connection 和 layer normalization，会把 Transformer encoder 简化得过头。真正读论文图时，Add & Norm 是稳定训练和保留信息的重要结构，不是可以忽略的装饰。

你现在应该能做什么：诊断“Transformer 就是 self-attention”这句话为什么不完整。

误区诊断

DIAGNOSIS

常见坑

  1. 以为 seq2seq 要求输入输出长度一样。错因：被“sequence 到 sequence”字面误导。后果：无法理解翻译、语音辨识等长度不固定任务。修法：抓住“输出长度由机器决定”。
  2. 以为所有任务用 seq2seq 就最好。错因：混淆“能硬解”和“最适合”。后果：忽略语音辨识等任务的专用模型。修法：把 seq2seq 当通用基线，不当万能最优。
  3. 把 encoder 和 decoder 分工混淆。错因：只记 Transformer 一个总名。后果：说不清哪部分读输入、哪部分生成输出。修法：encoder 处理输入序列，decoder 决定输出序列。
  4. 把 Transformer encoder 简化成 self-attention。错因：漏掉 feedforward、residual 和 layer norm。后果：读不懂原论文图里的 Add & Norm。修法：按 block 顺序复述。
  5. 把 layer normalization 当成 batch normalization。错因：都叫 normalization。后果：计算维度和使用场景说错。修法：layer norm 对同一向量不同 dimension 计算统计量。
  6. 以为原始 Transformer 架构不可改。错因：把论文图当定理。后果：读不懂 pre-norm、PowerNorm 等后续变体。修法：先掌握基线，再比较改动点。
