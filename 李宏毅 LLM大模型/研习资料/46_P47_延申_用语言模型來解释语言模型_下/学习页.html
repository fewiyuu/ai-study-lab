（延申）用语言模型來解释语言模型 (下)

上半段已经讲过一个基本想法：让一个语言模型观察 GPT-2 某个神经元在句子上的 activation，再写出这个神经元可能在侦测什么。下半段继续追问：这个解释怎么验收？如果给 explainer 的例子本身有偏差，它会不会写出听起来合理但实际错误的解释？

核心链路是 Subject model、Explainer、Simulator 三个角色。Subject model 是被解释的模型，视频里是 GPT-2；Explainer 读取神经元的高 activation 例子并写解释，视频里用 GPT-4；Simulator 根据解释扮演这个神经元，再用预测出的 activation 和真实 activation 比较。读到这里，你应能先画出三角色流程，再说明打分为什么不是一句“解释得好”就结束。

Subject model→activation pattern→Explainer→Simulator→score / revision

来源：https://www.bilibili.com/video/BV1JKoeBiEt4?p=47

学完后要能做四件事

  1. 说清 Subject model、Explainer、Simulator 各自读什么、写什么、不能替谁负责。
  2. 解释 correlation score 和替换神经元的 whole-model output 检查为什么会互相印证。
  3. 用 not all / all 的例子说明高 activation 样本为什么会让解释过宽。
  4. 指出这篇方法的边界：单一神经元未必有独立功能，语言未必能完整描述神经元，低分也可能是 simulator 太弱。

概念骨架

MAP

解释不是直接相信模型

Explainer 写出的句子只是一份候选解释。它可能准确，也可能只是抓住了高 activation 例子里最显眼的词。课程后半段一直在补一件事：把候选解释放回模型行为里检查。

判断句：只要一个解释没有经过真实 activation 或整网输出检验，它还不能算稳定解释。

打分要看两层行为

第一层是神经元级别：Simulator 根据解释预测某个神经元在新句子上的 activation，再和 GPT-2 真实 activation 做 correlation。第二层是模型级别：把 GPT-2 的一个神经元换成 GPT-4 扮演的神经元，看整个 GPT-2 输出是否变化。

两种分数正相关时，说明“像不像这个神经元”和“会不会影响整体输出”没有明显冲突。读到这里，你应能区分局部打分和整体打分。

样本选择会改变解释

Explainer 不可能读完上万个句子的 activation，只能看一小部分例子。通常会挑 target neuron 反应最强的句子，但这种选择会漏掉负例和反例。not all 的例子就是典型：只看正例时，模型容易把神经元解释成“看到 all 就启动”。

1. 观察

让 GPT-2 读大量句子，记录某个 neuron 的 activation。

2. 解释

把高 activation 片段给 GPT-4，让它写出神经元的自然语言解释。

3. 扮演

让 GPT-4 根据解释预测同一神经元在新句子上的反应。

4. 修正

用分数和反例检查解释，必要时生成新句子再 revision。

Simulator照着解释扮演神经元的模型。根据 explanation 和新句子，预测 target neuron 应该给出怎样的 activation。分数低时只怪解释，其实也可能是 simulator 扮演能力不够。
correlation score看扮演结果像不像真实神经元。比较 simulator 预测 activation 与真实 activation 的相关程度，越高说明解释更能指导扮演。以为分数高就等于人一定看懂解释；分数只说明这套系统内的匹配较好。
revision用新例子修正解释。先让模型按当前解释生成可能触发神经元的句子，再送回 Subject model 标注 activation，用新证据改写解释。把 revision 当成随便扩充数据；关键是生成能区分旧解释和新解释的例子。

讲义主线

LECTURE

1. 为什么还要检查解释

让 GPT-4 给 GPT-2 神经元写解释，这件事最容易让人误会成“模型自己说了算”。课程里没有这样处理。解释被写出来以后，还要交给 simulator 使用：如果 simulator 只看这段解释，就能预测出真实 target neuron 在不同句子上的 activation，解释才算有实际约束力。

这里的直觉很简单：好的说明书应该能指导别人复现行为。若 explanation 写成“这个神经元在寻找 all”，但 simulator 按这句话预测所有含 all 的句子都会高 activation，而真实 GPT-2 只在 not all 这种结构上高反应，那么说明书太宽了。

正式说法是，解释质量通过 simulation 任务来间接衡量。Explainer 产生一段自然语言解释，Simulator 把这段语言转成 activation 预测，最后用 correlation score 比较预测值与真实值。读到这里，你应能把“自然语言解释”看成中间变量，而不是最终答案。

2. 局部像不像，还要问整体有没有变

只比较单个 neuron 的 activation 仍然有一个问题：某些神经元很关键，数值差一点就会影响整个 GPT-2 输出；某些神经元不重要，activation 差很多也不会影响最终输出。单看局部相关性，可能把这两类神经元混在一起。

视频里提到另一种检查：用 GPT-4 扮演的神经元替换 GPT-2 里的真实神经元，再观察 GPT-2 的整体输出有没有改变。这个实验听起来很怪，一个有许多参数的大模型被要求扮演一个神经元，但它抓住了更现实的问题：这个 neuron 的行为差异是否真的传到整网输出。

OpenAI 的结果显示，activation correlation score 和这种替换后的整体输出分数呈正相关。因此实际分析时可以优先看其中一个分数，不必每次都做代价更高的整体替换。读到这里，你应能解释为什么“像不像神经元”和“影响不影响模型输出”是两个不同层次的问题。

3. 高 activation 样本为什么必要

GPT-4 的输入长度有限，不可能把 target neuron 在所有句子上的反应都塞进去。研究者通常先让 GPT-2 跑过大量文本，再挑出这个神经元 activation 最大的片段给 explainer。

这样做能把最有信号的例子放到 explainer 面前。判断句：如果一个样本从来没有触发目标神经元，它对“这个神经元喜欢什么”帮助有限。

4. 高 activation 样本也会误导

问题在于，正例只告诉你“这些句子会触发”，不告诉你“哪些相似句子不会触发”。如果高 activation 片段都包含 not all，explainer 可能注意到 all，却没有证据知道没有 not 的 all 不会触发。

这就是解释过宽。判断句：看到一个解释时，要问它有没有被负例或对照样本挑战过。

5. revision 的作用

revision 不是单纯多喂一点资料，而是让模型按当前解释生成新句子，再让 Subject model 给这些句子标 activation。若当前解释说“侦测 all”，就生成更多含 all 的句子；若 GPT-2 对 “All the students ...” 没有高反应，旧解释就站不稳。

这样得到的新资料能逼 explainer 收窄解释，从“all”改成“not all”。读到这里，你应能说明 revision 为什么比随机加样本更有针对性。

6. random revision 为什么也有用

视频里提到，随机多找一些句子或从数据库抽一些句子，解释力分数也可能提升。原因不是随机本身有魔法，而是更多样本会补上一些原来没见过的边界。

但随机样本未必撞到关键反例。模型生成的样本如果围绕当前解释制造对照，通常能更快暴露漏洞。

7. 最后的质疑不能跳过

这套方法确实漂亮，但它没有解决所有问题。单一神经元可能没有独立语义；某个神经元的真实功能也许无法用人类语言描述；explainer 和 simulator 的分工还会混淆责任。分数低时，你很难判断到底是 explanation 不好，还是 simulator 不会扮演。

判断句：这节课不是要你相信“AI 已经能解释 AI”，而是要你会拆解这套验证框架。

完整知识链路

把本节串起来，可以这样讲：研究者先从 Subject model 取出 target neuron 的 activation pattern，让 Explainer 写出自然语言解释；再让 Simulator 按这段解释去预测 activation，用 correlation score 衡量预测和真实反应是否一致。为了避免只看高 activation 正例造成过宽解释，可以用当前解释生成新句子，再送回 Subject model 标注，利用反例做 revision。最后还要记住，这个流程解释的是被选中的神经元，不一定解释了整个网络；语言解释也不一定覆盖神经元真正做的复杂运算。你应能从“观察、解释、扮演、打分、修正、质疑”六步复述整节课。

材料熔炼

SOURCE

下面按字幕推进顺序整理。它们不是逐字摘录，而是把视频口头讲法压成可复习的知识块。

线索 1：三种模型角色

字幕开头先回顾整体流程：被解释的是 Subject model，课程例子里是 GPT-2；Explainer 和 Simulator 都可以由 GPT-4 扮演，但它们不必和被解释模型相同，也不必固定是 GPT-4。关键是角色分工，不是模型名字。

读到这里，你应能判断：哪个角色产出解释，哪个角色产出 activation 预测。

线索 2：两种评分角度

第一种角度比较 simulator 输出和真实神经元输出，得到 correlation score。第二种角度把 GPT-2 的真实神经元换成 GPT-4 扮演的神经元，检查 GPT-2 整体输出改变多少。前者看局部拟合，后者看整网影响。

读到这里，你应能说明为什么“局部相似”不自动等于“整体无影响”。

线索 3：分数和人类判断的关系

视频提到 OpenAI 做了 human study：人觉得比较好的解释，通常在自动解释力分数上也更高。这一点缓解了一个担心：分数不只是模型内部自娱自乐，至少和人类读到的解释质量有正相关。

但它不是最终证明。human study 说明有关联，不说明所有高分解释都可靠。

线索 4：为什么要选 activation 最大的句子

Explainer 的上下文有限，只能看一小批例子。比较自然的做法是让 GPT-2 先读大量句子，再挑出 target neuron 输出特别大的句子。这样能让 explainer 看到最明显的触发模式。

读到这里，你应能说出这一步的必要性：不是偷懒，而是受上下文长度限制。

线索 5：not all 反例

字幕里的例子很关键：某个神经元看起来在看到 all 时启动，但真实功能可能是寻找 not all。只给正例时，Explainer 没有证据排除普通 all，于是会写出过宽解释。

你应能用这个例子解释什么叫“缺少对照样本”。

线索 6：用 GPT-4 生成关键样本

如果当前解释是“侦测 all”，就让 GPT-4 生成更多含 all 的句子，再让 GPT-2 给这些句子标 activation。若普通 all 不触发，Explainer 就能把解释修成“not all”。这种 revision 比随机抽样更容易打到旧解释的弱点。

读到这里，你应能区分随机补样本和有目标地造反例。

线索 7：方法本身的质疑

结尾集中列出几个边界：用 AI 解释 AI 是否可靠；解释单一神经元是否有意义；神经元行为是否能被人类语言描述；explainer 与 simulator 的合作分数是否会掩盖责任归属。

你应能把这些质疑归类为“对象边界、语言边界、评估边界”。

最小例子

EXAMPLE

下面用几行伪代码重建课程里的 not all 例子。重点不是运行代码，而是看懂样本、解释、预测和修正之间的关系。

最小示例：用反例修正神经元解释pseudo-python
case = {
  "positive_examples": [
    ("not all of them agreed", 0.96),
    ("not all answers are correct", 0.91)
  ],
  "draft_explanation": "This neuron activates on the word all.",
  "generated_tests": [
    "All the students arrived early.",
    "Not all the students arrived early."
  ],
  "subject_model_labels": [0.08, 0.94],
  "revised_explanation": "This neuron activates on the phrase not all."
}
correlation_before = score_simulator(case["draft_explanation"])
correlation_after = score_simulator(case["revised_explanation"])

逐行理解

前两条正例只告诉我们：含 not all 的句子会让 target neuron 有高 activation。它们还不能证明 all 本身就是触发条件，因为这两条正例里 all 总是和 not 一起出现。

revision 的关键动作是主动造对照。把只有 all 的句子和含 not all 的句子一起送回 Subject model，观察真实 activation 差异。若只有 not all 高，解释就必须从“all”收窄到“not all”。读完这段，你应能指出一个解释缺少哪些反例。

代码里的关键对象

positive_examples 是原本挑出来的高 activation 片段，它们都支持“这个神经元会启动”的事实，但不一定能区分 all 与 not all。draft_explanation 是 Explainer 的初稿，它抓到了 all，却漏掉了前面的 not。

generated_tests 是 revision 的关键。第一句只有 all，第二句有 not all；送回 Subject model 后，真实 activation 标签显示第一句低、第二句高。这样 Explainer 才有证据把解释收窄。

删掉哪一行会出问题

删掉 subject_model_labels，你就只有 GPT-4 自己生成的新句子，却不知道 GPT-2 真实神经元怎样反应。删掉 generated_tests，你就只能在旧正例里打转，无法发现“all”这个解释过宽。

判断句：revision 的价值来自“生成候选反例 + 回到 Subject model 标注”，两步缺一不可。

误区诊断

DIAGNOSIS

常见坑

  1. 把 Explainer 的解释当成事实。错因：忽略解释只是候选假设。后果：遇到反例时仍相信过宽解释。修法：用 simulator 和真实 activation 打分。
  2. 只看高 activation 正例。错因：没有负例或对照样本。后果：把 not all 误解释成 all。修法：生成相似但可能不触发的句子，再送回 Subject model 标注。
  3. 把 correlation score 理解成整网影响。错因：局部分数只比较神经元输出。后果：无法判断这个 neuron 的误差是否影响 GPT-2 最终输出。修法：必要时做替换神经元的整体输出检查。
  4. 分数低就断定解释差。错因：忽略 simulator 也可能很弱。后果：把责任全推给 explanation。修法：同时检查 simulator 能力、给 simulator 的说明和样本质量。
  5. 以为单一神经元一定有清晰语义。错因：把神经网络内部表示想得太模块化。后果：解释结果可能碎片化或误导。修法：保留“功能可能分布在神经元组或方向上”的判断。
  6. 以为自然语言能描述所有行为。错因：语言词汇有限。后果：高维复杂特征被压成粗糙标签。修法：把语言解释看成可用近似，不当成完整还原。
