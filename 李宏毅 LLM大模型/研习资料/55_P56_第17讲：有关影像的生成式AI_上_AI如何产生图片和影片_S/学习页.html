第17讲：有关影像的生成式 AI（上）：AI 如何产生图片和影片

这一讲先把“跟影像有关的生成式 AI”分成两类：一类是把图片或影片当作 condition，让模型产生文字或其他回应；另一类是把图片或影片本身当作目标输出。本页聚焦第二类，也就是文字、草图、姿态、录音、旧影片等条件如何驱动模型生成新的图片或影片。

课程的关键不是记住 SORA 很厉害，而是看懂它背后的工程抽象：图片由 pixel 组成，影片由 frame 组成，但现代生成模型通常不会直接处理像素，而是先用 encoder 把影像压成 patch，再让 Transformer 等模型在 patch 序列上生成、对齐和建模。影片只是“更多 patch”，但更多到足以把普通 attention 变成天文数字，所以才需要空间、时间、伪 3D attention 与分阶段流水线。

condition→pixel / frame→encoder 压成 patch→Transformer + attention→decoder 还原影像

学完后要能做五件事

  1. 区分“影像作为输入条件”和“影像作为生成目标”。
  2. 解释 SORA、talking head、ControlNet、个人化生图各自用了什么 condition。
  3. 说明 pixel、frame、FPS、patch、encoder、decoder 在影像生成链路里的角色。
  4. 比较文字生图中的自回归 patch 接龙、非自回归并行生成和 CLIP score 评估。
  5. 用数量级解释为什么文字生影片难，并说出 spatial、temporal、伪 3D attention 和流水线如何降成本。

概念骨架

MAP

先看输入条件

生成式 AI 总是在某个 condition 下生成东西。图片理解任务里，图片或影片是 condition；影像生成任务里，文字、草图、姿态、录音、低清影片或少量参考图都可以成为 condition。

判断句：看到一个影像生成应用，先问“它拿什么当条件”，不要急着问模型名字。

再看影像表示

人看到的是完整图片和连续影片，模型常看到的是一排 patch。encoder 把高维 pixel 或 frame 压缩成较短的 patch 表示，Transformer 在这些 patch 上建模，decoder 再把 patch 还原成图片或影片。

判断句：patch 不是简单把解析度调低；它通常来自神经网络压缩，保留了可还原的语义和局部信息。

最后看规模瓶颈

一张图的 patch 已经不少，一分钟影片会变成数百万个 patch。若所有 patch 两两做 attention，复杂度按平方增长，这就是文字生影片比文字生图难很多的核心原因。

判断句：影片生成的关键问题不是“会不会画”，而是“怎样在可承受计算量内保持空间一致和时间连贯”。

应用层

SORA 文字生影片、影片续写、老片修复、talking head、ControlNet 控制生成。

资料层

文字-图片配对、视频-文字配对、少量个人化参考图，以及可能有版权风险的大规模网络资料。

表示层

pixel 与 frame 先被 encoder 压成 patch，二维图片和三维影片都变成序列。

计算层

attention 负责让 patch 彼此参考；影片中要在空间一致和时间连贯之间降成本。

非自回归生成很多 patch 同时生成。同一个 Transformer 可同时接收多个位置请求，让各位置通过 attention 互相参考。以为并行生成就意味着 patch 之间完全没关系。
CLIP / CLIP score让另一个模型判断图文是否匹配。CLIP 由正确和错误图文配对训练而来；生成图片后可用输入文字和图片的匹配分数做自动评估。以为 CLIP score 高就等于人类一定满意。
spatial / temporal attention分别看空间关系和时间关系。spatial attention 看同一 frame 内 patch 关系；temporal attention 看不同 frame 同位置 patch 的时间关系。只用其中一种就宣称能解决影片一致性。

讲义主线

LECTURE

1. 影像相关生成式 AI 有两条路

课程开头先拆概念：跟影像有关，不代表都是“生成图片”。第一条路是让影像当输入条件，例如把照片交给 GPT-4o，让它描述人物穿着、猜职业、回答图片相关问题。这时输出可能是文字，影像只是 condition。

第二条路才是本讲重点：输出目标本身是图片或影片。你给模型一段文字、一个草图、一张参考照片、一段录音、一段低清影片或一个姿态骨架，模型要生成对应的影像。后面所有机制都围绕这条路展开。

区分这两条路很重要，因为“看图说话”和“凭条件造图”对模型能力的要求不同。前者侧重理解和描述，后者要补出大量输入没有明说的视觉细节，还要让结果在空间和时间上自洽。

2. SORA 示例说明“能生成”不等于“完全懂物理”

SORA 的 demo 展示了文字生影片的震撼效果：例如小怪物看着红色蜡烛融化，或者纽约街道沉在海底。这些场景现实中不太会出现，所以比普通街景更能说明模型不是只在训练集里复制一段现成影片。

但课程也特别提醒：SORA 的影片仍有瑕疵。墙面可能不自然地移动，水生生物可能从海龟变成鱼，狼崽可能越变越多，考古学家挖出的塑胶椅还会跑掉。这些错误说明模型可以产生很拟真的局部画面，却仍可能在物体恒定性、时间连续性和因果关系上犯错。

学习到这里，你应该能做一件事：评价影像生成模型时，不能只看单帧清晰度，还要看同一物体在时间中是否保持身份、位置、数量和运动逻辑。

3. 应用不是只有文字生图

影片生影片可以做续写、风格转换、黑白影片上色、老电影修复和画质提升。talking head 则把录音和照片结合起来，预测人物说话时的嘴形和脸部变化。

ControlNet 代表另一类思路：给模型更多结构化条件，例如草图、姿态、边缘或深度，让它不是只靠文字猜画面。你现在应该能按 condition 类型整理应用。

4. pixel 和 frame 是人的直观单位

图片由 pixel 组成，像素越多通常越清楚。影片由一张张 frame 组成，FPS 表示每秒有多少张图。YouTube 常见影片约 24 FPS，游戏则常希望 60 FPS 才更流畅。

这些概念看似基础，却是后面数量级计算的起点：影片不是“一个文件”，而是大量图片按时间排列。

5. patch 是模型真正处理的影像单位

直接处理像素太复杂，所以现代模型常先用 encoder 把图片切成 patch 并压缩。patch 可以是离散符号，也可以是低维向量。图片原本是 2D，但送进 Transformer 前常被拉直成一排 patch。

decoder 不是简单把 4x4 小图放大，而是神经网络，会参考邻近甚至全局 patch 还原出清晰图片。

6. 影片 patch 还压缩时间维度

影片比图片多了时间维度。除了高和宽，encoder 还可以把相邻 frame 同位置的 patch 合并，形成 video patch。一个 video patch 可能对应一小段宽、一小段高和一小段时间。

SORA 官方图示虽然没有公开细节，但从图中可以看出类似 patch 的影像表示：影片经过 encoder 变成 patch，再进入生成模型。

7. 文字生图训练依赖图文配对

要训练模型读文字生图片，需要大量“图片 + 文字叙述”的配对资料。课程提到 LAION 这类大规模数据集，规模可到数十亿张图，并可能来自网络爬取和后处理。

这里要同时看到能力和风险：大规模资料让模型见过各种文字-影像对应关系，但资料来源、版权、成人内容过滤和图文是否真的匹配，都会影响训练和使用边界。

8. 影片生成靠降成本而不是硬算到底

文字生影片可视为生成更多 patch，但完整 3D attention 会让所有 patch 两两交互，成本随 patch 数平方爆炸。课程用一分钟、24 FPS、每帧 64x64 patch 的例子算出约 600 万 patch，完整 attention 约 36 兆次。

因此常见做法是把 spatial attention 和 temporal attention 交替叠加，形成伪 3D attention；再用多个模型组成流水线，逐步提高解析度和 FPS。

完整知识链路

把本节串起来，可以这样讲：影像生成模型先接受某种 condition，例如文字、草图、姿态、录音或参考图。图片和影片本身先由 pixel、frame 和 FPS 描述，但模型通常用 encoder 把它们压成 patch 序列，再用 Transformer、attention 或其他生成模型在 patch 表示上工作，最后交给 decoder 还原。文字生图可以想象成 patch 接龙，但主流生成常常更偏向并行或多步生成；生成结果可以用人类评价，也可以用 CLIP score 这类模型评价。文字生影片在原理上是生成更多 patch，但难点在 attention 计算量和时间一致性，所以需要 spatial / temporal / 伪 3D attention，以及先低清低帧率再逐步超分辨率、补帧的流水线。你现在应该能把任意影像生成 demo 拆成 condition、表示、生成、评估、降成本五层。

线索 1：condition 是总入口

字幕反复说生成式 AI 有输入和 condition，并根据 condition 生成东西。本讲把 condition 扩展到文字、影片、录音、照片、草图、姿态、低清影像和少量参考图。

你现在应该能先识别一个系统的 condition，再判断它是看图生成文字，还是生成图片或影片。

线索 2：SORA 的震撼和瑕疵要一起看

课程选了现实中少见的场景说明 SORA 可能创造新画面，也选了墙面移动、生物变形、狼崽增殖、椅子跑掉等瑕疵说明视频生成还会失去时间一致性。

你现在应该能评价一个视频 demo 的空间质量、时间连贯和物体恒定性。

线索 3：图片不是直接丢像素

字幕强调现代模型通常不会直接处理像素，而是用 encoder 把图片切成 patch 并压缩，再把二维 patch 拉直成序列给 Transformer 处理。

你现在应该能解释为什么 patch 不是“低解析度小格子”。

线索 4：文字生图可以类比文字接龙，但不止接龙

用图文配对资料训练时，可以想象 Transformer 看到文字后一个 patch 接一个 patch 生成；但课程也说明文字生图很少真的靠这种自回归方式，更多时候会同时生成多个 patch，让 attention 处理相互关系。

你现在应该能区分“帮助理解的自回归类比”和“实际常用的并行或多步生成”。

线索 5：CLIP 把图文匹配变成分数

CLIP 用正确图文配对给高分、随机错配给低分的方式训练。生成图像后，把原始文字和生成图一起交给 CLIP，可得到 CLIP score 作为自动评估参考。

你现在应该能说明 CLIP score 是方便指标，但不能完全替代人类判断。

线索 6：影片难在 attention 成本

一分钟影片在 24 FPS、每帧 64x64 patch 的假设下约有 600 万 patch。若做完整 3D attention，需要约 36 兆次两两关系计算，所以研究会改造 attention 或拆成多阶段流水线。

你现在应该能用数量级解释为什么文字生影片比文字生图难。

最小例子

EXAMPLE

这里用伪代码把课程主线压到最小。它不是某个真实模型的源码，而是帮你看清“条件、patch、attention、评估、解码”的顺序。

最小示例：文字生图的 patch 流程python
text = "一只奔跑的狗，在草地上"
condition = text_encoder(text)

positions = make_patch_positions(height=4, width=4)
patches = transformer_generate(condition, positions)

image = image_decoder(patches)
score = clip_score(text, image)
review = human_check(image, checks=["是否像狗", "是否在奔跑", "背景是否合理"])

逐行理解

text_encoder 把文字条件变成模型可用的表示；make_patch_positions 告诉模型要生成哪些位置的 patch；transformer_generate 不是画像素，而是在 patch 表示上工作；image_decoder 再把 patch 还原成图片。

clip_score 代表自动评估：看生成图片和文字是否匹配。最后还保留 human_check，因为 CLIP 分数无法完全判断美感、细节、物体数量和真实使用风险。

和字幕的对应

字幕先用“文字接龙”帮你类比 patch 接龙，再提醒影像生成常常并不一个 patch 一个 patch 等待，而是同时生成许多 patch，并靠 attention 让不同位置互相参考。

你现在应该能做的事：把这个流程改成 talking head 或 ControlNet 的版本，只替换 condition，不改“表示、生成、解码、评估”的骨架。

最小示例：文字生影片的规模估算python
fps = 24
seconds = 60
patches_per_frame = 64 * 64

frames = fps * seconds
video_patches = frames * patches_per_frame
full_3d_attention = video_patches ** 2

spatial_attention = frames * (patches_per_frame ** 2)
temporal_attention = patches_per_frame * (frames ** 2)
pseudo_3d_attention = spatial_attention + temporal_attention

读懂计算

在字幕设定下，一分钟影片有 1440 个 frame，每个 frame 有 4096 个 patch，总数约 600 万。完整 3D attention 要让所有 patch 两两相看，约 36 兆次。把 spatial attention 和 temporal attention 分开做，约为 240 亿加 85 亿，仍然很大，但比完整 3D 小了约千倍。

你现在应该能解释：影片生成的“难”不是抽象地难，而是 patch 数量和 attention 平方复杂度把成本推高。

误区诊断

DIAGNOSIS

常见坑

  1. 把“影像作为 condition”和“生成影像”混为一谈。错因：只看输入里有没有图片。后果：解释不清任务目标。修法：先问输出是不是图片或影片。
  2. 把 patch 当成简单马赛克。错因：忽略 encoder / decoder 是神经网络。后果：误以为还原结果只能是低清小格子。修法：说明 patch 是压缩表示，decoder 会参考上下文还原。
  3. 以为文字生图主流就是 patch 接龙。错因：把类比当事实。后果：无法解释并行生成和 attention 的作用。修法：区分自回归类比与非自回归、多步生成。
  4. 只用单帧清晰度评价影片。错因：把影片当图片集合。后果：漏掉物体变形、数量变化和运动不连续。修法：加入时间一致性检查。
  5. 把 CLIP score 当最终真理。错因：忘记它也是模型评估。后果：图文匹配分高但细节、版权、审美或安全仍有问题。修法：CLIP 作为筛选，人类和任务指标做最终验收。
  6. 说“Transformer 很强所以能直接生成长视频”。错因：忽略 attention 平方复杂度。后果：看不懂为什么需要 attention 简化和流水线。修法：用 patch 数量级重新估算。
