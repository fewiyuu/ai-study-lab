作业十：Stable Diffusion Fine-tuning

这一节讲的是作业十的完整实作：把一个会根据文字画图的 Stable Diffusion 模型，调成更会画同一个人的个性化模型。课程里的例子使用 100 张 Brad Pitt 照片和对应文字描述，训练后希望模型既能维持同一张脸，又不能忘掉文字条件，比如白色 T 恤、护士服、牛仔裤这些 prompt 约束。

主线很清楚：用 LoRA 只训练少量新增参数，接着观察 checkpoint 里的图片变化，再用 testing 生成 25 张最终图片。评分不只看像不像人脸，还会检查图片和 prompt 的关系，以及有没有生成没有脸的图片。读完后，你应该能独立解释“怎样训练”“怎样选参数”“怎样看分数”“怎样避免无效提交”。

概念骨架

MAP

先看任务

基础 Stable Diffusion 已经会按照文字生成图片，但它不知道“固定生成某一个人”这件事。本作业要做 personalization：给模型一组目标人物照片，让它学会在不同 prompt 下保持同一个人的脸。

判断句：如果只说“模型会画 Brad Pitt”，还不够；还要说它是否继续听文字条件。

再看方法

完整微调整个 Stable Diffusion 成本高，作业改用 LoRA。LoRA 的做法是在大模型里加少量可训练层，只更新这些小层，让模型行为发生偏移，同时保留原模型的大部分能力。

判断句：LoRA 不是重新训练一个大模型，而是用小参数去调大模型的生成方向。

最后看评分

评分分三类风险：人脸是否像训练集、图片是否符合 prompt、图片里有没有可检测的人脸。face distance 越小越好，CLIP score 只要过门槛，no-face 超过 5 张会让提交无效。

判断句：只追求人脸相似会牺牲文字对齐；只看文字对齐又可能不像目标人物。

资料

100 张目标人物图片、对应 captions、评分用 prompts。

训练

从约 1000 step 的预置 checkpoint 继续训练 LoRA。

生成

validation 看中途三张图，testing 生成 25 张最终图。

提交

确认图片目录，压成 zip，上传到 NTU Cool。

术语表

TERMS
术语直觉解释准确说法常见误解
Stable Diffusion给文字条件后生成图片的扩散模型。它在潜空间中逐步去噪，并用文字编码条件引导图像生成。以为它天然会稳定生成同一个人的脸。
text-to-image输入一句文字，输出一张图片。模型需要同时满足图像质量、文字条件和随机采样带来的多样性。只看图片像不像，不看 prompt 说了什么。
personalization让模型记住一个特定人物或风格。用少量目标样本微调，让不同 prompt 下仍出现同一身份特征。把它当成单纯复制训练照片。
LoRA只训练小模块来影响大模型。冻结原模型大部分权重，在部分层加入低秩可训练参数。以为 LoRA 会完整重训 Stable Diffusion。
learning rate每一步更新参数的幅度。学习率越大，LoRA 参数变化越快，但过大容易覆盖原有 text-to-image 能力。以为越大越像目标人物就一定越好。
train steps继续训练多少步。步数越多，训练越久，身份相似度可能上升，也更容易过拟合。只追求最高步数，忽略一天能跑几次和模型退化。
validation ratio每隔多少训练步生成验证图。例如总训练 800 步、ratio 0.5，约每 400 步做一次 validation。把 validation 的三张图分数当成最后成绩。
checkpoint训练到某一步保存下来的模型状态。作业会保存中途 checkpoint，也会记录 last 或 best 供 testing 使用。不知道最后 testing 选择的是 last 还是 best。
face distance生成脸和训练集脸之间的距离。GhostFaceNet 抽取人脸特征，两两比较后取平均；越小越像。以为分数越大越好。
CLIP score文字和图片是否对得上的分数。用 prompt 和生成图像计算相似度，本作业主要作为门槛，防止只交目标人物照片。以为 CLIP score 会直接决定所有分段。
no-face图片里没有可检测的人脸。少量 no-face 可以容忍，超过 5 张提交会被判无效。以为衣服或姿势对了就可以没有脸。

讲义主线

LECTURE

1. 从 text-to-image 到 personalization

普通 text-to-image 模型擅长把“一个有胡子的男人，穿白色 T 恤”这类文字变成图像，但它生成的是一个泛化出来的人，不会固定成某个具体身份。作业的目标是把这个泛化能力往一个人物身份上拉，让同一批 prompts 生成的脸都像训练集中同一个人。

这里的难点不是“让图片好看”这么简单。模型必须同时满足两个要求：脸要像目标人物，prompt 里的衣服、职业、姿态等文字条件也要尽量保留。如果训练把模型推得太狠，脸可能越来越像，但 prompt 对齐会变差，白色 T 恤可能被画成黑色西装。

你读到任何作业结果时，都可以先问一个判断问题：这张图是在“复制训练人物”，还是在“按照新的文字条件生成同一身份”？前者容易投机，后者才是本作业想要的能力。

2. LoRA 为什么适合这次作业

完整微调整个 Stable Diffusion 很费时间、显存和钱。LoRA 的直觉是：不要动大模型的所有参数，只在若干层旁边加小的可训练参数，让这些小参数改变模型行为。大模型原本的 text-to-image 能力被保留下来，训练成本也降到学生能在 Colab 里完成的程度。

课程里提到，本作业大约 30 分钟可以继续训练出一个可用模型。这个时间并不是因为 Stable Diffusion 本身很小，而是因为训练范围被压缩到 LoRA。你以后看到“低成本微调”时，要追问它具体冻结了哪些参数、训练了哪些新增模块。

判断句：如果一个方案说要 fine-tune，但没有说明训练的是全模型、LoRA、adapter 还是别的参数高效方法，你还不能判断它的成本和风险。

3. 作业资料从哪里来

Notebook 第一步会连接 Google Drive，并建立作业资料夹。里面有一个 dataset 资料夹，存放 100 张目标人物图片、对应文字描述，以及最后评分会用到的 prompts。

另一个以你命名的工作资料夹会放训练输出和 checkpoint。课程示例里还会预先下载约 1000 step 的模型，让你从这个点继续训练，而不是从零开始。

判断句：如果找不到 dataset、captions 或 prompts，后面训练和 testing 都会变成空跑。

4. learning rate 的取舍

learning rate 控制每一步更新有多大。调大之后，模型可能更快学到目标人物的脸，face distance 有机会下降。

代价是模型可能忘掉原本的文字生成能力。它仍然画同一张脸，却不再认真听 prompt，这会让 CLIP score 或人工观察变差。

判断句：learning rate 不是越大越好，它是在身份相似和文字对齐之间移动平衡点。

5. train steps 的取舍

train steps 决定继续训练多少步。步数少时，模型可能还不像目标人物；步数多时，可能更像，但训练时间更长，也更容易过拟合。

课程提醒：一般设置下一天大约能跑两次，若把 train steps 调到很高，一天可能只能跑一次。作业不是无限试错，参数选择也要考虑时间预算。

判断句：如果你没有时间再跑 testing，就算参数理论上更好，也可能交不出完整 25 张图。

6. validation ratio 看的是中途样子

validation ratio 用来决定训练过程中每隔多少步生成一次图片。例子里总训练 800 步、ratio 0.5，就会约每 400 步做一次 validation。

validation 会在 checkpoint report 里留下图片和分数，帮助你观察“同一个 prompt 在训练中怎样越来越像目标人物”。它很适合选方向，但不能直接替代最终 testing。

判断句：validation 通常样本少，只能做训练观察，不是最终成绩。

7. last 与 best checkpoint

训练结束后，代码可能保存最后一步 checkpoint，也可能根据验证分数保存 checkpoint-best。testing 时可以选择用 last 或 best。

last 的含义是训练走到最后的模型；best 的含义是验证过程中某个评分更好的模型。二者谁更适合提交，要看生成图是否既像目标人物，又没有破坏 prompt 对齐和人脸检测。

判断句：不要只凭文件名选模型，要看它在 validation 和最终 testing 中实际生成了什么。

8. testing 才生成最终提交图片

testing 或 inference 阶段会使用评分 prompts 生成 25 张图片。课程里说这一段大约 15 分钟，生成后会在 inference 资料夹里看到需要提交的图片。

最后提交时，不是交 checkpoint，也不是交 report，而是把这 25 张图片所在资料夹压成 zip。路径错、图片数量错、压错文件夹，都会让本来训练好的结果变成提交问题。

判断句：最终成绩看的是 testing 生成的完整 25 张图，不是你在中途随手看到的几张图。

完整知识链路

本作业可以串成一句话：给 Stable Diffusion 一个由 100 张目标人物照片和文字描述组成的数据集，用 LoRA 继续训练少量参数，使模型在新的 prompts 下生成同一身份的人脸；训练中用 validation 图片观察 checkpoint 变化，最终用 testing 生成 25 张图片，再由 face distance、CLIP score 和 no-face 数量判断成绩。这个链路里每一步都有常见错法：数据放错会跑不起来，learning rate 太大可能让文字对齐变差，train steps 太少可能不像目标人物，validation 分数不能代表最终分数，no-face 超过阈值会让提交无效。读到这里，你应该能从一张生成图反推它可能是哪一类问题。

WORKFLOW

2. 下载资料和预置模型

dataset 里有 100 张训练照片、对应 captions 和评分 prompts。工作目录里会放已经训练到约 1000 step 的 checkpoint，让学生从这里继续训练。中途生成的图片用于看同一 prompt 如何逐步靠近目标人物。

3. 安装 package 并 import

依赖安装大约 5 分钟。这里主要是让 Colab 环境具备训练和生成需要的库。安装阶段不是调参阶段，出错时先看环境、路径和版本，不要急着改 learning rate。

你现在应该能区分：环境错误和模型效果错误不是同一类问题。

4. 设置 learning rate 与 train steps

课程希望学生主要调这两个参数。learning rate 决定每步更新幅度，train steps 决定总训练量。二者都能影响 face similarity，也都可能让 text-to-image 能力退化。

你现在应该能说明：为什么“更像脸”和“更听 prompt”会互相拉扯。

5. 训练与 validation report

训练过程中会按 validation ratio 生成图片，并在 report 里留下 checkpoint 记录。这个阶段适合看趋势：脸有没有靠近目标人物，衣服和场景有没有开始偏离 prompt。

你现在应该能判断：validation 分数小样本、可参考，但不是最后分数。

6. testing、压缩和上传

最后 testing 会生成完整 25 张评分图片，课程强调这个分数才是最终成绩依据。确认 inference 目录里的图片数量和内容后，把图片资料夹压成 zip，再上传到 NTU Cool。

你现在应该能检查：提交物是 25 张生成图的 zip，不是 checkpoint 或 notebook。

最小例子

EXAMPLE

例子只保留作业逻辑，不复刻 Notebook 里的长代码。你要看懂每一步读什么、写什么、最后会影响哪一类评分。

最小示例：一次 LoRA 作业流程python
config = {
    "base_model": "stable-diffusion",
    "method": "LoRA",
    "dataset": "100 target-face images + captions",
    "learning_rate": 1e-4,
    "extra_train_steps": 200,
    "validation_ratio": 0.5,
}

checkpoint = continue_train(config)      # 只更新 LoRA 参数
preview = validate(checkpoint, n_images=3)
chosen = pick_checkpoint(preview, mode="last_or_best")

final_images = testing(chosen, prompts="grading_prompts", n_images=25)
submit_zip = zip_folder(final_images)

逐行理解

config 把作业设置集中在一起：基础模型、LoRA、数据集、学习率、训练步数和验证频率。continue_train 只更新 LoRA 参数；validate 生成少量中途图片；testing 才生成最终 25 张评分图。

你现在应该能做的事：看到一份训练结果时，先判断它来自 validation 还是 testing，再决定能不能拿来当提交依据。

这段代码在读什么

dataset 读的是目标人物照片和 captions。prompts 读的是评分用文字条件。两者不能混淆：训练资料负责让模型学身份，评分 prompts 负责检查模型能否把身份放进新文字条件。

这段代码在写什么

continue_train 写出 checkpoint，testing 写出最终图片，zip_folder 写出提交压缩包。删掉 validation 还能生成结果，但你会失去中途观察；删掉 testing 就没有最终 25 张图。

误区诊断

DIAGNOSIS

常见坑

  1. 只追求脸像。错因是把 personalization 当成复制照片；后果是 prompt 对齐下降；修法是同时看 CLIP score 和不同 prompts 下的图片。
  2. learning rate 调太大。错因是把更新幅度当成提升速度；后果是模型忘掉原本 text-to-image 能力；修法是降低学习率或减少训练步数。
  3. train steps 调太少。错因是为了省时间忽略身份学习；后果是生成图仍像普通陌生人；修法是增加步数并看 validation 趋势。
  4. 把 validation 当最终成绩。错因是没区分三张中途图和 25 张 testing 图；后果是提交前误判；修法是以 testing 输出和最终评分为准。
  5. 没检查 no-face。错因是只看衣服或姿势；后果是超过 5 张无脸图会无效；修法是提交前逐张确认有人脸。
  6. 压错资料夹。错因是不知道提交物是 inference 生成图片；后果是 zip 内没有 25 张有效图；修法是先数图，再压缩正确目录。
