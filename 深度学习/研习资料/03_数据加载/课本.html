数据加载：Dataset 与 DataLoader

直觉：Dataset 像“数据索引器”，回答怎么取第 i 条；DataLoader 像“批处理器”，回答一次取几条、要不要打乱、最后不满一批怎么办。

学习目标

  • 能写一个最小自定义 Dataset：实现 __len__ 和 __getitem__。
  • 能用 DataLoader 生成 batch，并解释 batch_size、shuffle、drop_last。
  • 能把 DataLoader 接进最小训练循环。

你现在应该能做什么：知道本节不是“学一个新模型”，而是学数据如何一批一批送进模型。

1. Dataset：定义怎么取一条

Dataset 是 PyTorch 对“数据集”的统一接口。它不关心你的数据来自 CSV、图片文件、数据库还是内存张量，只要求你回答两个问题：

方法要回答的问题例子
__len__一共有多少条数据return len(self.x)
__getitem__(idx)第 idx 条数据是什么return self.x[idx], self.y[idx]
import torch
from torch.utils.data import Dataset

class TinyPairDataset(Dataset):
    def __init__(self):
        self.x = torch.tensor([[1.0], [2.0], [3.0]])
        self.y = torch.tensor([[2.0], [4.0], [6.0]])

    def __len__(self):
        return len(self.x)

    def __getitem__(self, idx):
        return self.x[idx], self.y[idx]

ds = TinyPairDataset()
print(len(ds))      # 3
print(ds[0])        # (tensor([1.]), tensor([2.]))
理解关键：Dataset 不负责“取一批”，只负责“取一条”。这样图片、文本、表格都能被统一包装。

你现在应该能做什么：看到一个数据来源时，能先想清楚“第 i 条样本应该返回什么”。

2. DataLoader：定义怎么取一批

DataLoader 接收 Dataset，然后按你的设置把单条样本组装成 batch。训练时，模型通常不是一次看完整数据，而是一批一批看。

from torch.utils.data import DataLoader

loader = DataLoader(ds, batch_size=2, shuffle=False)

for xb, yb in loader:
    print(xb.shape, yb.shape)
    print(xb, yb)
参数含义训练常见设置
batch_size每批几条样本16、32、64 起步
shuffle每个 epoch 是否打乱顺序训练 True，验证 False
drop_last最后不满一批是否丢弃多数先用 False
num_workers几个子进程并行读数据学习阶段 Windows 先用 0
Windows 注意：num_workers > 0 通常需要 if __name__ == "__main__": 保护。学习阶段先保持 num_workers=0，少引入额外问题。

你现在应该能做什么：能用 DataLoader 拿到一个 batch，并检查 xb.shape 是否符合预期。

3. batch / epoch / iteration

这三个词经常一起出现，先按训练循环直觉理解：

术语直觉例子
batch一次喂给模型的一小批数据batch_size=32 表示一次 32 条
iteration处理一个 batch 的一次参数更新机会100 条数据、batch_size=20，有 5 次 iteration
epoch完整看完一遍训练集训练 3 个 epoch，就是完整看 3 遍
手算对照：10 条数据，batch_size=3

默认 drop_last=False：批大小分别是 3、3、3、1，所以一共 4 个 batch。

如果 drop_last=True：最后 1 条被丢弃，所以只剩 3 个 batch。

你现在应该能做什么：给定样本数和 batch_size，能算出每个 epoch 有多少个 batch。

4. 接入最小训练循环

DataLoader 的最终用途是给训练循环提供 xb 和 yb。一轮最小训练循环通常是：

model = torch.nn.Linear(1, 1)
criterion = torch.nn.MSELoss()
optimizer = torch.optim.SGD(model.parameters(), lr=0.01)

for xb, yb in loader:
    pred = model(xb)          # forward
    loss = criterion(pred, yb)
    optimizer.zero_grad()     # 清掉旧梯度
    loss.backward()           # 反向传播，计算新梯度
    optimizer.step()          # 用梯度更新参数
理解关键：DataLoader 不训练模型，它只提供 batch；真正改变参数的是 loss.backward() 和 optimizer.step()。

你现在应该能做什么：读懂一段 PyTorch 训练循环时，能指出数据、预测、loss、梯度、参数更新分别在哪一步发生。
