从线性模型到多层感知机

多层感知机，常写作 MLP，是很多神经网络的基本积木。它不像 CNN 那样专门处理图像局部结构，也不像 Transformer 那样处理序列注意力。它最核心的任务很朴素：把输入特征经过若干线性层和非线性激活，变成适合任务的输出。

这一节要把 MLP 讲成能写代码、能查错的模型，而不是只记“输入层、隐藏层、输出层”。你会看到线性层到底在做什么，为什么只堆线性层没有意义，激活函数为什么放在隐藏层后面，输出层为什么经常不加 softmax，以及隐藏层宽度、深度、参数量会怎样影响训练。

学完你应该能做什么

  1. 用 nn.Sequential 或自定义 nn.Module 写一个 MLP。
  2. 解释 nn.Linear(in, out) 的输入输出形状和参数量。
  3. 说清为什么 MLP 需要非线性激活函数。
  4. 根据回归、多分类、二分类任务选择输出层和 loss。
  5. 诊断常见错误：shape 不匹配、最后一层维度错、错误 softmax、过拟合、激活函数位置不对。

MLP 的完整主线

mental model

MLP 可以先理解成一个可学习的函数。输入是一批特征，形状通常是 [batch, features]；隐藏层把特征投影到新的表示空间；激活函数引入非线性；输出层把最后一层隐藏表示变成任务需要的输出。训练循环负责根据 loss 调整每一层的权重和偏置。

这条主线里最容易混的是两个问题。第一，线性层不是“神经元个数”这么粗略，它有明确的矩阵乘法：y = xW^T + b。第二，激活函数不是装饰。没有激活函数，多层线性层可以合并成一个线性层，模型并没有真的变强。MLP 能处理非线性关系，关键就在“线性变换 + 非线性激活”反复组合。

输入特征[B, F]
Linear仿射变换
Activation引入非线性
Output匹配任务
Loss驱动训练

直觉

MLP 不是记住样本，而是学习一个从输入特征到输出目标的函数。隐藏层越多、越宽，函数空间越灵活，但也更容易过拟合或训练不稳定。

正式说法

MLP 是由多个全连接层和逐元素非线性激活函数组成的前馈神经网络，常用于表格特征、简单分类回归和其他模型里的前馈子网络。

判断句

如果一个网络里只有多个 Linear，中间没有非线性激活，它在表达能力上仍然等价于一个线性层。

你现在应该能做什么看到 MLP 代码时，先找输入形状、每层输出维度、激活函数、最后输出维度和 loss 是否匹配。

线性层：不是黑盒，是矩阵乘法加偏置

linear layer

nn.Linear(in_features, out_features) 做的是一个仿射变换。给定输入 x，它内部有权重 weight 和偏置 bias，输出是 x @ weight.T + bias。如果输入形状是 [B, in_features]，输出形状就是 [B, out_features]。

这里的 B 是 batch size。线性层不会关心 batch 里有多少条样本，它只关心每条样本最后一维是不是 in_features。所以很多 shape 报错都可以翻译成一句话：你喂给这一层的每条样本特征数，和它声明的输入特征数对不上。

y=xWT + b,W 的形状是[out_features, in_features]
最小示例：Linear 的形状linear_shape.py
import torch
from torch import nn

layer = nn.Linear(4, 3)
x = torch.randn(8, 4)
y = layer(x)

print(layer.weight.shape)  # torch.Size([3, 4])
print(layer.bias.shape)    # torch.Size([3])
print(y.shape)             # torch.Size([8, 3])

参数量怎么数

一个 Linear(in, out) 的权重有 out × in 个，偏置有 out 个。总参数量是 out × in + out。例如 Linear(4, 3) 有 3 × 4 + 3 = 15 个参数。

参数量不是越多越好。参数多，模型更灵活，也更容易记住训练集噪声；数据少、特征简单时，过宽的隐藏层可能很快过拟合。

常见错法

错法 1：把 in_features 写成 batch size。batch size 不属于层参数。

错法 2：忘记展平图片或高维输入，直接把 [B, C, H, W] 喂给期望 [B, F] 的 MLP。

错法 3：前一层输出维度和后一层输入维度对不上。

你现在应该能做什么给定一串 Linear 层，能算出每层输出形状和参数量；遇到矩阵乘法 shape 报错时，先查最后一维。

非线性：为什么只堆 Linear 没有意义

activation

如果两个线性层中间没有激活函数，那么它们可以合并成一个线性层。比如第一层做 xA + a，第二层再做 (xA + a)B + b，整理后仍然是 x(AB) + (aB + b)。层数变多了，但表达能力没有真正越过线性模型。

激活函数的作用是打断这种“可合并性”。ReLU、Sigmoid、Tanh 等函数会对每个元素做非线性变换，让网络可以组合出弯曲的决策边界。初学 PyTorch 时，隐藏层最常见的选择是 ReLU，因为它简单、计算快，也比较不容易像 Sigmoid 那样在极端输入处梯度很小。

切换结构，观察激活函数放在哪里。重点不是图好看，而是判断哪些层有参数，哪些层只改变非线性形状。
激活函数常见位置直观作用注意点
ReLU隐藏层后负数变 0，正数保留入门最常见，简单稳定
Sigmoid少用于隐藏层；二分类推理可用压到 0 到 1BCEWithLogitsLoss 前不要手动 sigmoid
Tanh部分旧模型或特定结构压到 -1 到 1输入过大时梯度可能变小

结构设计：输入、隐藏层、输出层各自管什么

architecture

MLP 的结构可以按职责拆。输入层不是真正的一层代码，它表示输入特征的维度。隐藏层负责逐步提取和重组特征，常见写法是 Linear → ReLU → Linear → ReLU。输出层负责把隐藏表示映射到任务需要的维度，例如回归输出 1 个数，多分类输出类别数个 logits。

隐藏层宽度和深度是模型容量的主要来源。宽度表示每层有多少隐藏单元，深度表示有多少层变换。容量太小，模型可能学不会；容量太大，在小数据上可能很快把训练集记住。入门阶段不要一上来堆很深，先用一到两层隐藏层跑通，再根据训练和验证表现调整。

正确示例：Sequential 写 MLPmlp_sequential.py
from torch import nn

model = nn.Sequential(
    nn.Linear(20, 64),
    nn.ReLU(),
    nn.Linear(64, 32),
    nn.ReLU(),
    nn.Linear(32, 3),
)
正确示例：自定义 Module 写 MLPmlp_module.py
class MLP(nn.Module):
    def __init__(self, in_dim, hidden_dim, out_dim):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(in_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, out_dim),
        )

    def forward(self, x):
        return self.net(x)

输入维度

由每条样本的特征数决定。表格数据是列数；展平图片是 C × H × W；不能把 batch size 写进去。

隐藏维度

是你设计的模型容量。常见从 32、64、128 这类规模试起，不要在小数据上盲目加大。

输出维度

由任务决定。三分类输出 3 个 logits；回归一个目标输出 1 个数；多标签分类输出标签个数个 logits。

你现在应该能做什么根据任务写出 MLP 的输入维度、隐藏维度、输出维度，并解释每一层为什么接得上。

shape 与参数量：读模型先读维度

shape

读 MLP 代码时，最实用的习惯是把每一层输出 shape 写在旁边。假设输入是 [B, 20]，经过 Linear(20, 64) 之后变成 [B, 64]，ReLU 不改变 shape，下一层 Linear(64, 32) 变成 [B, 32]，最后 Linear(32, 3) 变成 [B, 3]。

参数量也一样按层算。ReLU 没有参数，Linear 有权重和偏置。知道参数量不是为了炫技，而是为了判断模型是否过大、错误层是否被注册、输出层是否设置合理。如果参数量明显不符合预期，通常说明维度写错或层没有按你想的方式接起来。

Linear(64, 32)[B, 64][B, 32]64×32 + 32 = 2080
Linear(32, 3)[B, 32][B, 3]32×3 + 3 = 99

拖动滑块观察参数量变化。隐藏维度变大时，参数量通常比类别数变化更明显。

输出层与 loss：最后一层必须服务任务

output & loss

MLP 的最后一层不是固定写法。它要看任务。多分类任务通常输出 [B, num_classes] 的 logits，配 CrossEntropyLoss，标签是 [B] 的类别编号。二分类或多标签任务常输出一个或多个 logits，配 BCEWithLogitsLoss。回归任务输出连续值，常配 MSELoss 或 L1Loss。

一个很常见的错误是在输出层后面手动加 Softmax，再把结果传给 CrossEntropyLoss。训练代码看起来更像“概率”，但 loss 期望的是 logits。类似地，使用 BCEWithLogitsLoss 时也不要先手动 sigmoid。你可以在推理或展示结果时再把 logits 转成概率，但训练 loss 前不要乱加。

任务最后一层loss标签要求不要做什么
多分类Linear(hidden, C)CrossEntropyLosslong 类别编号 [B]不要先 softmax
二分类Linear(hidden, 1)BCEWithLogitsLossfloat 0/1，形状匹配输出不要先 sigmoid
回归Linear(hidden, target_dim)MSELossfloat 连续值不要把标签当类别编号
错误示例：输出层和 loss 不匹配bad_output.py
model = nn.Sequential(
    nn.Linear(20, 64),
    nn.ReLU(),
    nn.Linear(64, 3),
    nn.Softmax(dim=1),        # CrossEntropyLoss 前不该加
)
criterion = nn.CrossEntropyLoss()
你现在应该能做什么换任务时先改输出层和 loss，而不是只改数据。最后一层输出维度、标签格式、loss 函数必须三者一致。

训练 MLP：容量、过拟合和正则化

training

MLP 的训练仍然使用上一节的训练循环：前向、loss、反向、更新、验证。新增的判断是模型容量。隐藏层越宽、越深，模型可以表达的函数越复杂；但数据不够时，它可能只是把训练集记住。过拟合的典型迹象是训练 loss 持续下降，而验证 loss 上升或验证指标变差。

常见的控制方法包括减小模型、加权重衰减、加 Dropout、早停、增加数据或数据增强。入门阶段先理解最小的两个：权重衰减通过 optimizer 的 weight_decay 惩罚过大的权重，Dropout 在训练时随机丢掉一部分隐藏单元，让模型不要过度依赖某几个特征。注意 Dropout 在训练和评估模式下行为不同，所以 model.train() / model.eval() 又变得很重要。

示例：带 Dropout 和 weight_decay 的 MLPregularized_mlp.py
model = nn.Sequential(
    nn.Linear(20, 128),
    nn.ReLU(),
    nn.Dropout(p=0.3),
    nn.Linear(128, 64),
    nn.ReLU(),
    nn.Linear(64, 3),
)
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3, weight_decay=1e-4)

欠拟合

训练集也学不好。可能是模型太小、训练不够、学习率不合适、特征不足或 loss/标签设置错误。

过拟合

训练集表现很好，验证集变差。可能是模型容量过大、数据太少、正则化不足或训练太久。

先查代码

不要把所有问题都归为过拟合。先确认训练/验证数据分开、验证用 eval() 和 no_grad()。

你现在应该能做什么根据 train/valid loss 的走势，初步判断欠拟合、过拟合或代码问题，并知道先检查哪些训练边界。

MLP 常见错误：多数都能从 shape 和任务匹配查起

diagnosis

MLP 的错误往往比看起来朴素。shape 不匹配、输出层维度不对、激活函数放错位置、loss 和标签格式不一致、训练验证模式忘切，都可能让训练结果变得很怪。排查时不要从“是不是模型太差”开始，而是从最硬的契约开始：输入最后一维、每层输出维度、最后输出、标签 dtype、loss 期望。

另一个常见问题是把 MLP 当成万能结构。表格数据、小型特征向量、简单 demo 用 MLP 很合适；图像任务直接把图片展平给 MLP 也能跑，但会丢掉空间结构，通常不如 CNN 合适。理解 MLP 是为了打好基础，不是说所有任务都应该用 MLP。

完整模板：用 MLP 跑一个最小分类任务

template

下面的模板把 MLP 接入上一节的最小训练循环。数据仍然是合成数据，目的是让你看清结构：输入是 2 维特征，隐藏层是 32 和 16，输出是 3 个类别 logits，loss 是 CrossEntropyLoss。真实任务里你主要替换数据集、输入维度、输出类别数和指标。

完整模板：MLP 三分类mlp_train.py
import torch
from torch import nn
from torch.utils.data import TensorDataset, DataLoader

torch.manual_seed(0)
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

X = torch.randn(800, 2)
y = ((X[:, 0] ** 2 + X[:, 1] > 0.8).long() + (X[:, 0] > 0).long()).clamp(max=2)

train_ds = TensorDataset(X[:650], y[:650])
valid_ds = TensorDataset(X[650:], y[650:])
train_loader = DataLoader(train_ds, batch_size=32, shuffle=True)
valid_loader = DataLoader(valid_ds, batch_size=64, shuffle=False)

class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(2, 32),
            nn.ReLU(),
            nn.Linear(32, 16),
            nn.ReLU(),
            nn.Linear(16, 3),
        )

    def forward(self, x):
        return self.net(x)

model = MLP().to(device)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-2)

def train_one_epoch():
    model.train()
    total_loss, total_seen = 0.0, 0
    for x, y in train_loader:
        x, y = x.to(device), y.to(device)
        optimizer.zero_grad()
        logits = model(x)
        loss = criterion(logits, y)
        loss.backward()
        optimizer.step()
        total_loss += loss.item() * x.size(0)
        total_seen += x.size(0)
    return total_loss / total_seen

def evaluate():
    model.eval()
    total_loss, total_correct, total_seen = 0.0, 0, 0
    with torch.no_grad():
        for x, y in valid_loader:
            x, y = x.to(device), y.to(device)
            logits = model(x)
            loss = criterion(logits, y)
            pred = logits.argmax(dim=1)
            total_loss += loss.item() * x.size(0)
            total_correct += (pred == y).sum().item()
            total_seen += x.size(0)
    return total_loss / total_seen, total_correct / total_seen

for epoch in range(1, 11):
    train_loss = train_one_epoch()
    valid_loss, valid_acc = evaluate()
    print(f"epoch={epoch} train_loss={train_loss:.4f} valid_loss={valid_loss:.4f} valid_acc={valid_acc:.3f}")
你现在应该能做什么把这份模板改成自己的表格分类或回归任务：先改输入维度和输出维度，再改 loss 和 metric。
