卷积网络最小理解

MLP 可以处理展平后的图片，但它会把二维空间结构打散。对图像来说，相邻像素之间有局部关系：边缘、角点、纹理通常只需要看一小块区域。卷积网络利用这个事实，用小卷积核在图像上滑动，提取局部特征，再逐层组合成更高层的表示。

这一节只讲最小 CNN 必须知道的部分：输入形状 [B, C, H, W]，Conv2d 的参数含义，kernel/stride/padding 如何决定输出尺寸，通道如何从灰度图扩展到多特征图，池化和 Flatten 为什么常出现在分类网络里。学完你应该能读懂一份小型 CNN，并能定位最常见的 shape 错误。

学完你应该能做什么

  1. 解释 Conv2d(in_channels, out_channels, kernel_size) 的含义。
  2. 说清图片张量为什么通常是 [B, C, H, W]。
  3. 计算给定 kernel、stride、padding 下的输出高度和宽度。
  4. 读懂 Conv → ReLU → Pool → Flatten → Linear 的最小 CNN。
  5. 诊断通道数不匹配、Flatten 维度错误、padding/stride 导致尺寸不对等问题。

CNN 的完整主线

mental model

CNN 的主线可以从输入形状开始。图片进入模型时通常是四维张量：batch、channel、height、width。卷积层用多个卷积核在空间维度上滑动，每个卷积核会产生一张 feature map。ReLU 引入非线性，池化降低空间尺寸，Flatten 把二维特征图拉成一维向量，最后用 Linear 做分类或回归。

卷积网络比纯 MLP 更适合图像，不是因为参数更多，而是因为它更有约束。卷积核只看局部区域，参数在不同位置共享。这样模型能更自然地捕捉局部模式，也比把整张图片展平后全连接更省参数。你学这节的重点不是背网络名字，而是能追踪每一层如何改变 shape。

直觉

卷积核像一个小窗口，在图像上看局部区域。不同卷积核学不同模式，例如边缘、亮暗变化或纹理组合。

正式说法

卷积层通过局部连接和参数共享，从输入特征图中产生输出特征图。每个输出通道对应一组可学习卷积核。

判断句

读 CNN 先追 [B,C,H,W]。只要通道或空间尺寸没算清，后面的 Flatten 和 Linear 很容易写错。

你现在应该能做什么看到 CNN 代码时，先写出每层输出形状，而不是先纠结网络名字。

为什么图片不直接展平给 MLP

spatial structure

把一张 28×28 灰度图展平成 784 个数字，MLP 当然能吃。但展平之后，模型只看到一串特征，原来上下左右相邻的像素关系不再显式存在。一个边缘、一个局部纹理、一个小角点，本来是空间结构里的模式，展平后会变成许多独立位置上的权重。

卷积层保留空间结构。它用小窗口扫描图片，同一个卷积核在不同位置复用参数。这样做有两个结果：第一，模型更容易学到局部模式；第二，参数量通常更少。例如对 28×28 输入直接接 Linear(784, 128)，参数已经超过十万；而 Conv2d(1, 16, 3) 只有 16×1×3×3 + 16 个参数。

这个差异很重要。MLP 的每个输出神经元都要连接所有像素，所以它会为“左上角像素”和“右下角像素”分别学权重。卷积核只学一个小窗口里的权重，然后把这组权重复用到整张图上。也就是说，CNN 不是偷懒少学参数，而是把“同一种局部模式可能出现在不同位置”这个假设写进结构里。

层参数量它意味着什么
Linear(784,128)784×128 + 128 = 100480每个像素位置都和每个隐藏单元单独相连
Conv2d(1,16,3)16×1×3×3 + 16 = 16016 个 3×3 小窗口在所有位置共享
Conv2d(3,16,3)16×3×3×3 + 16 = 448RGB 三个通道都被同一组输出特征同时查看

MLP 的问题

展平会丢掉显式空间位置关系。全连接层还会给每个输入位置单独一套权重，参数量容易变大。

CNN 的约束

局部连接让每次只看小区域；参数共享让同一个模式可以在不同位置被检测到。这不是万能，但很适合图像。

你现在应该能做什么解释 CNN 适合图像的两个关键词：局部连接、参数共享。

Conv2d：卷积核在图像上滑动

conv2d

nn.Conv2d(in_channels, out_channels, kernel_size) 是 PyTorch 里最常用的二维卷积层。in_channels 是输入通道数，灰度图通常是 1，RGB 图通常是 3；out_channels 是输出通道数，也就是卷积层会学习多少组卷积核；kernel_size 是卷积核窗口大小，例如 3 表示 3×3。

卷积核在图像的高宽维度上滑动。每到一个位置，它拿窗口里的数字和自己的权重做加权求和，再加偏置，得到输出 feature map 上的一个位置。这个过程对每个输出通道重复一次，所以一个卷积层可以同时提取多种局部模式。

切换卷积核位置。高亮区域表示一个 3×3 kernel 当前看到的局部区域。
最小示例：Conv2d 输入输出conv_shape.py
import torch
from torch import nn

conv = nn.Conv2d(in_channels=1, out_channels=8, kernel_size=3)
x = torch.randn(4, 1, 28, 28)
y = conv(x)

print(conv.weight.shape)  # [8, 1, 3, 3]
print(conv.bias.shape)    # [8]
print(y.shape)            # [4, 8, 26, 26]

输出尺寸：kernel、stride、padding 一起决定

output size

卷积层最常见的 shape 问题来自输出高宽计算。kernel_size 决定窗口有多大，stride 决定窗口每次移动几格，padding 决定输入周围补多少圈 0。kernel 越大，输出通常越小；stride 越大，输出更小；padding 可以保留边缘信息并控制输出尺寸。

H_out=H_in + 2 × padding - kernel_sizestride+ 1

上面是没有 dilation 时的一维公式，高和宽各算一次。实际结果要求能整除；如果不能整除，PyTorch 会按整数位置滑动，得到向下取整后的有效输出尺寸。初学时先掌握常见配置：kernel=3, stride=1, padding=1 会保持高宽不变；kernel=3, stride=1, padding=0 会让高宽各少 2。

通道：out_channels 是学出来的特征图数量

channels

图片通道最开始来自数据。灰度图通常是 1 个通道，RGB 图通常是 3 个通道。经过卷积层后，通道数不再只是颜色，而是特征图数量。比如 Conv2d(3, 16, 3) 会把 3 通道输入变成 16 通道输出，每个输出通道都对应一组卷积核。

权重形状可以帮助你理解通道关系。Conv2d(in_channels, out_channels, k) 的权重形状是 [out_channels, in_channels, k, k]。每个输出通道都要看所有输入通道，然后合成一张 feature map。所以下一层的 in_channels 必须等于上一层的 out_channels。

这里容易误会：RGB 图片不是“分别跑三遍灰度卷积再拼起来”。对某一个输出通道来说，它有一整组形状为 [3, k, k] 的权重，会同时看 R、G、B 三个输入通道，然后把三个通道上的响应加起来，得到一张输出 feature map。换成灰度图时，第一层 in_channels 要写 1；换成 RGB 图时，第一层 in_channels 要写 3。后面的通道数由上一层 out_channels 决定，已经不再等同于颜色。

错误示例：通道数接不上bad_channels.py
model = nn.Sequential(
    nn.Conv2d(1, 8, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.Conv2d(1, 16, kernel_size=3, padding=1),  # 错：上一层输出是 8 通道
)
正确示例：通道数接上channels_ok.py
model = nn.Sequential(
    nn.Conv2d(1, 8, kernel_size=3, padding=1),
    nn.ReLU(),
    nn.Conv2d(8, 16, kernel_size=3, padding=1),
    nn.ReLU(),
)
你现在应该能做什么检查 CNN 时，沿着通道数读：第一层输入通道来自数据，后面每层输入通道等于上一层输出通道。

池化与 Flatten：从特征图走向分类器

pool & flatten

池化层通常用来降低空间尺寸。最常见的是 MaxPool2d(2)，它把每个 2×2 区域压成一个值，高宽通常减半。池化没有可训练参数，但会改变 shape。它让后面的计算更省，也让模型对小范围位置变化更不敏感。

卷积和池化之后，张量还是 [B, C, H, W]。如果后面接 Linear，需要先 Flatten 成 [B, C×H×W]。这里最常见的错误是手算错 C×H×W，导致 Linear 的 in_features 不匹配。稳妥做法是先用一次假输入跑前向，打印 Flatten 后的 shape，或者使用 nn.LazyLinear 在入门 demo 中减少手算负担。

降低空间尺寸不只有池化一种办法。stride=2 的卷积也会让高宽变小，而且它有可训练参数；池化是固定规则，通常没有参数。初学阶段可以先记住：池化更像一个明确的降采样步骤，stride 卷积把“提特征”和“降尺寸”合在一起。无论哪种写法，只要空间尺寸变了，后面的 Flatten 维度就要跟着重新计算。

示例：池化和 Flattenpool_flatten.py
x = torch.randn(8, 16, 14, 14)
pool = nn.MaxPool2d(2)
flat = nn.Flatten()

y = pool(x)
z = flat(y)

print(y.shape)  # [8, 16, 7, 7]
print(z.shape)  # [8, 784]

池化

改变空间尺寸，不改变通道数。MaxPool2d(2) 常让高宽减半。

Flatten

保留 batch 维，把 C,H,W 合并成一个特征维度，方便接 Linear。

判断句

Linear 报 mat1 and mat2 shapes 错误时，优先检查 Flatten 后维度。

最小 CNN 结构：Conv → ReLU → Pool → Linear

architecture

最小 CNN 分类器通常由特征提取器和分类器组成。特征提取器用卷积层、激活函数和池化层把图片变成多通道低分辨率特征图；分类器把这些特征展平后，用 Linear 输出类别 logits。这个结构很简单，但已经包含了 CNN 的关键路径。

随着网络变深，常见模式会重复出现：多个 Conv2d + ReLU 提取特征，间隔使用池化或 stride 卷积降低空间尺寸，最后用全连接层或全局平均池化做任务输出。当前阶段先别急着背 ResNet、VGG、AlexNet，先能看懂最小结构的 shape。

最小结构：小型 CNNtiny_cnn.py
class TinyCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(1, 8, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),              # 28 -> 14
            nn.Conv2d(8, 16, kernel_size=3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),              # 14 -> 7
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(16 * 7 * 7, 10),
        )

    def forward(self, x):
        x = self.features(x)
        return self.classifier(x)
你现在应该能做什么解释上面每一层如何改变 shape，并指出 16 * 7 * 7 是怎么来的。

CNN 常见错误：通道和 Flatten 最容易错

diagnosis

CNN 入门最常见的报错集中在两个地方。第一个是通道数不匹配：某层期望输入 8 通道，但实际输入只有 1 或 16 通道。第二个是 Flatten 后接 Linear 时维度算错：卷积和池化改变了高宽，但 Linear 的 in_features 仍按旧尺寸写。

排查时不要只看错误最后一行。先打印每层输出 shape，确认 [B,C,H,W] 怎么变，再确认 Flatten 后是多少。对于图片输入，还要确认数据排列是 PyTorch 需要的 [B,C,H,W]，不是很多图像库常见的 [B,H,W,C]。

调试片段：打印 CNN 每层 shapecnn_shape_trace.py

for layer in model.features:
    x = layer(x)
    print(layer.__class__.__name__, x.shape)

完整模板：小型 CNN 分类器

template

下面模板用随机图片模拟 1×28×28 灰度输入，重点是完整结构和 shape。真实任务里你会换成真正的 Dataset，但模型结构、训练循环和验证循环可以沿用。分类任务仍然输出 logits，配 CrossEntropyLoss。

完整模板：CNN 三分类 democnn_train.py
import torch
from torch import nn
from torch.utils.data import TensorDataset, DataLoader

torch.manual_seed(0)
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

X = torch.randn(600, 1, 28, 28)
y = torch.randint(0, 3, (600,))
train_loader = DataLoader(TensorDataset(X[:500], y[:500]), batch_size=32, shuffle=True)
valid_loader = DataLoader(TensorDataset(X[500:], y[500:]), batch_size=64, shuffle=False)

class TinyCNN(nn.Module):
    def __init__(self):
        super().__init__()
        self.features = nn.Sequential(
            nn.Conv2d(1, 8, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
            nn.Conv2d(8, 16, 3, padding=1),
            nn.ReLU(),
            nn.MaxPool2d(2),
        )
        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(16 * 7 * 7, 3),
        )

    def forward(self, x):
        x = self.features(x)
        return self.classifier(x)

model = TinyCNN().to(device)
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
