Transformer / LLM Foundation 02

Transformer 与 Attention 的直观理解

上一节里，文本已经被 tokenizer 变成 token id，又被 embedding 层变成形状为 [B, L, D] 的向量序列。Transformer 接下来要解决的问题是：每个位置怎样根据上下文更新自己。一个 token 不能只看自己的 embedding，否则它不知道前后词、语法关系和指代关系。

Attention 的核心是让每个 token 去“读取”同一序列里的其他 token。Q 和 K 决定读谁，V 提供被读到的内容，softmax 把相关性分数变成权重，多头注意力让模型用多组视角同时读取。Transformer block 再加上残差、LayerNorm 和前馈网络，让这个过程可以堆很多层。

Transformer 的最小主线

mental model

Transformer 的输入是一串 token 向量。每一层 Transformer block 都会把这串向量更新一遍：先用 self-attention 让每个位置读取其他位置，再用前馈网络对每个位置的表示做非线性变换。残差连接负责把原信息保留下来，LayerNorm 负责让数值更稳定。

这一节只讲最小理解，不直接进入完整论文细节。你要抓住三个层次：第一，attention 是“按权重混合 value”的计算；第二，self-attention 是同一序列内部互相读取；第三，Transformer block 是 attention、残差归一化和 FFN 的组合。后续的 decoder-only LLM、encoder-only BERT、encoder-decoder 翻译模型，都是在这个基础上改结构和 mask。

Q/K/V三种投影
Attention读上下文
Multi-Head多组视角
Block残差归一化 FFN
Logits预测 token

直觉

每个 token 先带着自己的向量进入层内，然后根据其他 token 重新更新自己的表示。

正式说法

self-attention 根据 token 间的点积分数生成注意力权重，并用这些权重对 value 向量加权求和。

判断句

attention 层通常不改变序列长度；它改变的是每个位置向量里包含的上下文信息。

为什么需要 Attention

why attention

如果每个 token 只使用自己的 embedding，模型就只能看到孤立片段。比如“苹果发布了新手机”和“我吃了苹果”，同一个“苹果”在不同上下文里含义不同。模型必须让当前 token 的表示结合周围 token，才能处理指代、语序、修饰关系和长距离依赖。

RNN 通过按顺序传递隐藏状态处理上下文，但长距离信息要一步步传。CNN 通过局部窗口扩大感受野，但也需要多层叠加才能覆盖远位置。Self-attention 的做法更直接：在一层里，每个位置都能和所有位置建立关系。代价是长度为 L 时，关系表大致是 L×L。

RNN

按顺序处理，天然适合序列，但并行性较差，远距离路径长。

CNN

局部窗口高效，但远距离关系需要堆层或大窗口。

Attention

同层内所有位置互相打分，远位置可以直接被读取。

你现在应该能做什么解释 attention 解决的不是“生成文字”本身，而是“每个位置如何使用上下文”。

Self-Attention：同一序列内部互相读取

self attention

self-attention 里的 self 很容易误解。它不是“只看自己”，而是 Q、K、V 都来自同一条序列。每个 token 作为 query，去和序列里所有 token 的 key 打分，然后根据 softmax 权重读取所有 token 的 value。输出仍然是一串向量，只是每个位置已经混入了上下文信息。

假设输入 X 形状是 [B,L,D]。线性投影得到 Q,K,V，形状先理解为 [B,L,D]。Q @ K^T 得到 [B,L,L] 的 score 矩阵：第一个 L 是发问位置，第二个 L 是被读取位置。对最后一维做 softmax 后，每一行是一组权重。再用权重乘 V，输出回到 [B,L,D]。

Attention(Q,K,V)=softmax(QKTsqrt(dk))V

Q、K、V：读谁与读到什么

qkv

Q/K/V 不是三份不同的原始输入，而是同一份输入 X 经过三个不同线性层得到的三种工作视角。Query 表示当前位置拿什么条件去找信息；Key 表示每个位置怎样被匹配；Value 表示如果这个位置被读到，它提供什么内容。Q 和 K 负责产生权重，V 负责被混合。

为什么不直接用同一个向量做所有事？因为“匹配条件”和“输出内容”可以是不同视角。一个 token 在判断相关性时可能看语法角色，在提供内容时可能提供语义信息。模型不用人工指定这些维度，训练会把三个投影矩阵学出来。

Q = XWQ,K = XWK,V = XWV
对象职责常见误区
Query当前位置发出的读取条件不是自然语言问题，而是向量
Key每个位置用于被匹配的索引Key 不提供最终被加权输出的内容
Value真正被权重混合的内容标准 attention 是权重乘 V，不是乘 K
你现在应该能做什么用一句话区分：Q/K 决定“读谁”，V 决定“读到什么”。

缩放与 Mask：让权重稳定且不偷看

scale & mask

点积会把多个维度的乘积加起来。维度越大，分数波动越容易变大。分数太大时，softmax 会变得很尖，最大项接近 1，其他项接近 0，梯度可能变差。除以 sqrt(d_k) 的作用是把分数尺度拉回更温和的范围。它不是为了改变哪个位置最相关，而是为了让训练更稳定。

mask 用来挡住不该看的位置。padding mask 挡住填充位置，causal mask 挡住未来位置。decoder-only 语言模型生成第 t 个 token 时，只能看之前的 token，不能看答案后面的 token。mask 通常在 softmax 前加到 score 上，把非法位置设成很小的负数，让 softmax 后概率接近 0。

正确示例：mask 在 softmax 之前attention_mask.py
scores = q @ k.transpose(-2, -1)
scores = scores / math.sqrt(d_k)

if mask is not None:
    scores = scores.masked_fill(mask == 0, -1e9)

weights = torch.softmax(scores, dim=-1)
out = weights @ v
错误示例：softmax 维度写错bad_softmax.py
weights = torch.softmax(scores, dim=1)  # 错：通常应该沿 key 维度，也就是最后一维

多头注意力：多组读取视角

multi-head

单头注意力只有一套 Q/K/V 投影和一套注意力权重。多头注意力会把表示维度拆成多个 head，每个 head 独立算一套 attention，然后把结果拼接起来，再通过输出投影混合。这样模型可以同时学习多种关系：有的 head 可能偏向局部邻近，有的偏向句法依赖，有的偏向特殊符号或段落边界。

形状上，常见做法是把 [B,L,D] 投影后 reshape 成 [B,H,L,D_head]，其中 D = H × D_head。每个 head 在自己的 D_head 维度里算 attention，输出再合回 [B,L,D]。多头不是把模型复制很多份，而是在同一层里并行使用多组注意力子空间。

输入[B,L,D]
拆头[B,H,L,Dh]
各头 attention每个头一套权重
拼接回到 [B,L,D]
输出投影混合各头结果

Transformer Block：Attention 只是其中一半

block

一个 Transformer block 通常不只有 attention。典型结构包含 self-attention、残差连接、LayerNorm、前馈网络 FFN，再加一次残差和 LayerNorm。Attention 负责混合不同位置的信息；FFN 对每个位置单独做非线性变换；残差让信息和梯度更容易穿过很多层；LayerNorm 让数值分布更稳定。

不同模型会有 Pre-LN 和 Post-LN 的差异。入门阶段先记住职责：attention 是跨位置通信，FFN 是逐位置加工。FFN 不会让 token 之间交流，它对每个位置独立应用同一个小 MLP；跨 token 的信息交换主要发生在 attention 里。

伪代码：一个 Transformer block 的职责block.py
def transformer_block(x, attention, ffn, norm1, norm2):
    x = x + attention(norm1(x))  # 跨位置读取上下文
    x = x + ffn(norm2(x))        # 每个位置独立非线性加工
    return x
Attention不同位置之间交换信息。
FFN每个位置单独加工表示。
Residual保留原信息，改善深层训练。
LayerNorm稳定每个 token 表示的数值尺度。

代价与边界：Attention 不是免费全局视野

cost

Self-attention 的优势是同一层内全局互看，代价是注意力矩阵随序列长度平方增长。长度 L 翻倍，L×L 关系表大约变成四倍。长上下文模型需要更多显存、计算和工程优化。RAG、摘要、分块、压缩上下文，很多时候都是为了控制这个长度压力。

注意力权重也不要直接当成完整因果解释。某个位置权重高，只说明这一层、这个 head、这个样本里它参与了加权混合。模型最后的预测还经过多层 attention、FFN、残差和输出头。看 attention map 可以帮助理解，但不能简单说“权重最高所以它就是原因”。

Attention matrix size≈L × L

优势

远距离 token 可以直接建立关系，训练时并行性好。

代价

长序列下注意力矩阵很大，计算和显存压力上升。

完整链路：从输入向量到下一个 token

workflow

把前两节连起来：文本经过 tokenizer 得到 input_ids，embedding 查表得到 [B,L,D]，加上位置相关信息后进入 Transformer blocks。每个 block 先让 token 通过 self-attention 读取上下文，再通过 FFN 做逐位置加工。堆叠多层后，模型得到最后一层 hidden states。语言模型头把 hidden state 映射到词表大小的 logits，用来预测下一个 token。

训练和推理的关键差异在于可见范围和目标。训练 decoder-only 语言模型时，常用 causal mask 防止当前位置看到未来 token；推理时模型一步步生成，每次把已生成 token 放回上下文。无论哪种情况，attention 的基本动作都是：当前 token 根据 Q/K 权重读取 V。
