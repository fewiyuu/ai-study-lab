论文主线

SOURCE

这篇论文解决的不是“再加一个注意力模块”

在 Transformer 之前，机器翻译常见结构是 encoder-decoder：encoder 把源语言句子读成表示，decoder 再逐步生成目标语言。RNN 会按顺序处理 token，CNN 会用卷积窗口扩大感受野，注意力通常只是连接 encoder 和 decoder 的辅助机制。Vaswani 等人的论文把顺序递归和卷积都拿掉，只保留注意力和前馈网络。这就是标题里 “Attention Is All You Need” 的准确含义。

第一层直觉很简单：如果翻译一个词需要参考句子里很远的位置，RNN 要把信息一步步传过来，路径变长；自注意力让任意两个位置直接建立关系。句子长度是 n 时，一个 self-attention 层会形成 n × n 的关系表，每个位置都能一次看到其他位置。代价是关系表会随长度平方增长，所以长上下文模型后来才会不断改 attention 的计算方式。

本页不急着讲完整 Transformer。先把“一个位置如何读取其他位置”学清楚。后面的 multi-head、encoder-decoder attention、位置编码、残差和层归一化都建立在这里。

到这里，你应该能把论文主张说成一句朴素的话：Transformer 用注意力替代序列递归，让 token 之间直接通信。

概念骨架

MAP

先问：这个位置想找什么？

Q 是 query，可以理解为当前位置发出的检索条件。它不一定是自然语言里的问题，而是一个向量。

再比：其他位置能匹配什么？

K 是 key，用来和 query 做点积。点积大，表示这个位置更值得被当前 query 读取。

最后拿：真正汇总的信息

V 是 value。softmax 给出权重后，模型对所有 value 做加权求和，得到新的 token 表示。

注意力公式

FORMULA

注意力不是“模型灵感来了所以关注某个词”。在 Transformer 里，它是一套明确的矩阵计算。先用 Q 和 K 做相似度，得到一个 n × n 的分数矩阵；再对每一行做 softmax，变成权重；最后用这些权重去混合 V。每一行代表一个当前位置如何读取整句。

Attention(Q,K,V) = softmax ( QKT dk ) V
其中 d_k 是 key/query 的维度。除以 sqrt(d_k) 是为了让点积分数不要随着维度变大而过度膨胀。

形状先过一遍

假设一个 batch 里有 B 条句子，每条长度 N，每个 token 的模型维度是 D。为了先把概念讲清，可以暂时不考虑多头，令 Q,K,V 的形状都是 [B, N, D]。

K^T 指的是把最后两个维度交换，所以 K.transpose(-2, -1) 的形状是 [B, D, N]。于是 Q @ K^T 的结果是 [B, N, N]。第一个 N 表示“谁在发问”，第二个 N 表示“被问的是谁”。

softmax 要沿最后一个维度做，因为对每个 query 位置来说，它要在所有 key 位置之间分配注意力权重。然后 [B, N, N] @ [B, N, D] 变回 [B, N, D]。这说明注意力层输入输出长度不变，但每个位置的内容被上下文重写了。

到这里，你应该能用形状判断：score 矩阵为什么是 N × N，输出为什么又回到 N × D。

一个小例子

句子是“猫 坐 在 垫子”。如果当前位置是“坐”，它的 query 会和所有 key 比较：猫、坐、在、垫子。模型可能给“猫”和“垫子”较高权重，因为它们帮助判断动作主体和位置。注意力权重不是人工规则，训练会让这些权重服务最终任务。

这也解释了为什么注意力图有时好读，有时不好读。权重高不一定等于人类语言学意义上的“解释”，它只是模型当前层当前头用于计算的混合系数。把它当作可视化线索可以，直接当作因果解释就容易过度解读。

QKV 读法

QKV

Q/K/V 最容易被生活化比喻带歪。更稳的读法是：同一个 token 表示 X 被投影成三种工作状态。作为 query，它表达“我现在需要什么信息”；作为 key，它表达“我能被怎样匹配”；作为 value，它表达“如果别人读我，我能提供什么内容”。这三种状态都由训练学出来，不需要人为规定哪个维度对应语法、语义或位置。

为什么不直接用 X @ X^T？因为“发问方式”和“被匹配方式”未必应该共享同一套坐标。一个词作为当前预测目标时关心的信息，和它被其他词检索时提供的索引，可能不是同一种特征。三套投影给模型更多自由度：它可以学出专门用于匹配的空间，也可以学出专门用于内容汇总的空间。

Query：当前位置的检索条件

比如 decoder 正在生成一个动词，它可能需要寻找主语、时态和上下文约束。query 表示这次读取想匹配什么。

Key：每个位置的匹配索引

key 不直接输出内容，它负责和 query 点积。点积越大，softmax 后权重越高。

Value：真正被混合的内容

value 是加权求和的对象。注意力权重决定从哪些 value 中拿信息。

Q=XWQ , K=XWK , V=XWV
这三个 W 都是可学习参数。多头注意力会把这些投影拆成多个头，每个头在更小的维度里算一套 attention。
读完这一节，你应该能解释：Q 和 K 决定“读谁”，V 决定“读到什么”。

缩放与 mask

SCALE

为什么要除以 sqrt(d_k)

点积会把多个维度上的乘积加起来。如果 query 和 key 的每个维度大致独立，维度越多，点积的波动通常越大。分数一旦很大或很小，softmax 会变得很尖：最大项接近 1，其他项接近 0。这样梯度容易变小，训练会更难。

缩放的作用是把分数拉回温和区间。它不是为了改变哪个 key 最相关，而是让 softmax 不要过早饱和。论文里把这种形式称为 scaled dot-product attention，这个小小的除法在实现里很容易漏掉。

mask 挡住不该看的位置

encoder self-attention 可以看完整源句，因为源句已经全部给出。decoder 自注意力不同：生成第 t 个 token 时，不能提前偷看第 t+1 个目标词。否则训练时答案泄漏，推理时模型又看不到未来，训练和生成条件不一致。

正确做法是在 softmax 前加 mask，把未来位置的 score 设成一个很小的负数，例如 -1e9。softmax 后这些位置的概率接近 0。注意顺序很重要：先 mask，再 softmax。softmax 后再把概率清零，会破坏概率归一化，除非你重新归一化，但标准实现不会这么绕。

读完 mask，你应该能指出 decoder 里为什么必须有“上三角遮挡”。

最小代码

PYTORCH

下面的代码只实现单头注意力，故意不加入多头、dropout、缓存和工程优化。先把顺序读清楚：投影 Q/K/V，计算 score，缩放，加 mask，softmax，乘 V。任何一步放错，模型都可能还能跑，但语义已经变了。

正确示例：单头 scaled dot-product attentionPyTorch
import torch
import torch.nn.functional as F

def scaled_dot_product_attention(x, w_q, w_k, w_v, mask=None):
    # x: [batch, seq_len, d_model]
    q = x @ w_q                         # [batch, seq_len, d_k]
    k = x @ w_k                         # [batch, seq_len, d_k]
    v = x @ w_v                         # [batch, seq_len, d_v]

    d_k = q.size(-1)
    scores = q @ k.transpose(-2, -1)    # [batch, seq_len, seq_len]
    scores = scores / (d_k ** 0.5)

    if mask is not None:
        scores = scores.masked_fill(mask == 0, -1e9)

    weights = F.softmax(scores, dim=-1)
    out = weights @ v                   # [batch, seq_len, d_v]
    return out, weights

逐行拆解

q = x @ w_q 不是复制输入，而是把同一个输入映射到 query 空间。w_k 和 w_v 同理。实际模型里这些常由 nn.Linear 实现。

k.transpose(-2, -1) 只交换最后两个维度，保留 batch 维。这里如果写成普通转置，很容易把 batch 也搅进去。

F.softmax(scores, dim=-1) 表示每个 query 位置在所有 key 位置上分配权重。若写成 dim=1，归一化方向就变成“所有 query 争一个 key”，语义不对。

读完代码，你应该能用形状和 softmax 维度验收一段最小 attention 实现。

错误示例：能运行，但语义错

错误示例：softmax 维度写错PyTorch
scores = q @ k.transpose(-2, -1)
weights = F.softmax(scores, dim=1)   # 错：应该沿最后一维归一化
out = weights @ v

这段代码的危险在于它可能不报错。输出形状仍然像对的，但每一行注意力权重不再表示“当前 token 如何读取所有 token”。调试 attention 时，不能只看 shape 通过，还要检查每行权重是否沿 key 维度和为 1。

易混概念对照

容易混的两件事区别
self-attention / encoder-decoder attention前者 Q/K/V 来自同一序列；后者 decoder 的 Q 去读 encoder 的 K/V。
attention score / attention weightscore 是 softmax 前的原始分数；weight 是 softmax 后的概率式权重。
mask / dropoutmask 挡住非法位置；dropout 是训练时随机丢部分连接或激活来正则化。
embedding 维度 / 序列长度embedding 维度是每个 token 向量有多宽；序列长度是有多少个 token。
value 加权求和 / key 加权求和标准 attention 用权重乘 V，不是乘 K。K 用来匹配，V 用来输出内容。

五个坑

  1. 把 self-attention 理解成“每个词只看自己”。正确说法是同一序列内部互相读取。
  2. 忘记除以 sqrt(d_k)。后果是 softmax 分布可能太尖，训练更不稳。
  3. mask 放在 softmax 后。后果是概率归一化被破坏，未来信息可能没有真正被挡住。
  4. softmax 维度写错。输出形状可能没问题，但每行权重语义变了。
  5. 把注意力权重当成完整解释。它能显示一次混合，不等于严格因果解释。
