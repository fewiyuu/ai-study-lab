Encoder 与 Decoder：Transformer 的两条信息流

前两页讲的是 attention 本身。这一页把 attention 放回完整架构里：encoder 由若干相同层堆叠，decoder 也由若干相同层堆叠。真正容易混的不是“有几层”，而是每一层里哪一段看源句、哪一段看目标句、哪一段必须遮住未来。

源序列→Encoder 堆叠→memory→Decoder 交叉读取→目标词概率

学完后要能做三件事

  1. 能画出 encoder layer 和 decoder layer 的子层顺序。
  2. 能区分 encoder self-attention、masked decoder self-attention、encoder-decoder attention。
  3. 能读懂残差、LayerNorm、FFN 在每层里的位置和作用。

论文原版采用 6 层 encoder、6 层 decoder。这里先学结构和数据流，不纠结每个超参数。

整体结构先看懂

OVERVIEW

Encoder：把源句变成可被读取的表示

机器翻译里，源句是输入语言，目标句是要生成的语言。encoder 的工作不是直接输出最终翻译，而是把源句每个位置变成上下文表示。这里的上下文表示已经通过 self-attention 读过同一句子里的其他词，所以每个位置不再只是孤立词向量。

论文里的 encoder 由 N 个相同层堆叠。每个 encoder layer 有两个子层：multi-head self-attention 和 position-wise feed-forward network。每个子层外面都有 residual connection 和 layer normalization。你看到“Add & Norm”时，不要把它当成装饰框，它负责把子层输出和输入合回去，再稳定数值分布。

到这里你应该能判断：encoder 的 self-attention 里，Q/K/V 都来自源序列当前层表示。它通常不需要 causal mask，因为源句在编码时已经完整给出。

Decoder：按生成顺序使用两类信息

decoder 要生成目标句。训练时它能拿到已经右移的目标序列，推理时它只能拿到已经生成的前缀。所以 decoder 的第一段 self-attention 必须 masked：当前位置不能看未来目标词。

decoder 每层比 encoder 多一个注意力子层：先做 masked self-attention，读取目标端已知前缀；再做 encoder-decoder attention，用目标端表示作为 Query，去读取 encoder 输出作为 Key/Value；最后接 FFN。这样 decoder 既能维护目标语言内部顺序，又能对齐源句信息。

到这里你应该能判断：cross-attention 不是“decoder 又自己看自己一次”，而是 decoder 用自己的 Q 去查 encoder 的 K/V。

Encoder 层逐块拆开

ENCODER

Self-attention 解决源句内部通信

encoder self-attention 的 Q、K、V 都来自同一条源序列。它让每个源词位置读取其他源词位置，形成带上下文的表示。例如翻译 “bank” 时，它可以通过上下文判断更接近河岸还是银行。这里不是最后决策，只是把源端信息组织得更适合 decoder 读取。

判断句：如果 Q 来自源句，K/V 也来自源句，这就是 encoder self-attention。

Add & Norm 不是可有可无

残差连接把子层输入直接加到子层输出上。它提供一条信息和梯度的直通路径，减少深层堆叠时信息被反复变换后丢失的问题。LayerNorm 再把每个 token 的隐藏维做归一化，让数值尺度更稳定。

判断句：看到 x + Sublayer(x)，你要意识到它保留了原表示，同时让子层学习增量修改。

FFN 在每个位置独立工作

position-wise FFN 对每个 token 位置使用同一套两层 MLP。它不在位置之间通信，位置之间的通信已经由 attention 完成。FFN 的作用是在每个位置内部做非线性特征变换，扩展维度再压回 d_model。

判断句：如果一个模块只沿隐藏维变换，不混合不同 token 位置，它更像 FFN，不是 attention。

Decoder 层为什么更复杂

DECODER

第一段：masked self-attention

decoder 的目标端输入通常是右移后的目标序列。训练时我们知道完整答案，但模型在位置 i 只能使用 i 之前的词，否则它会偷看未来。masked self-attention 就是在 score 矩阵里挡住未来位置，让 softmax 后未来 token 的权重为 0。

这一步只处理目标端内部关系。它回答的是“当前已经生成的目标前缀里，哪些词对下一个表示有用”。如果你把 causal mask 放到 encoder self-attention 上，通常就是错用；如果你忘了 decoder mask，训练损失可能很好看，但推理会崩。

读完这一段，你应该能解释 teacher forcing 下为什么仍要 mask。

第二段：encoder-decoder attention

cross-attention 发生在 decoder 中间。此时 Query 来自 decoder 当前层的目标端表示，Key 和 Value 来自 encoder 的最终输出。也就是说，decoder 拿着“我现在要生成的目标位置需要什么信息”去源句 memory 里检索。

它和 encoder self-attention 的区别不在公式，而在 Q/K/V 来源。公式仍是 scaled dot-product attention；变的是 Q 来自目标端，K/V 来自源端。翻译任务里的对齐关系主要在这里被学习：目标端某个位置可以对源端多个位置分配注意力。

读完这一段，你应该能用一句话说清 cross-attention：目标端 query 去读源端 memory。

切换 Q/K/V 来源

判断时只看两件事

第一，看 Q/K/V 来源是不是同一条序列。Q/K/V 同源，就是 self-attention；Q 来自 decoder、K/V 来自 encoder，就是 cross-attention。

第二，看当前位置能不能读未来目标词。只要是自回归生成目标序列，decoder self-attention 就要挡未来。

残差、LayerNorm 和顺序问题

ADD NORM

论文原版是 Post-LN 结构

《Attention Is All You Need》图里写的是每个子层后接 Add & Norm，常见形式可以写成 LayerNorm(x + Sublayer(x))。这通常被称为 Post-LN，因为归一化发生在残差相加之后。很多现代 Transformer 变体会改成 Pre-LN，也就是 x + Sublayer(LayerNorm(x))，以改善更深模型训练稳定性。

学习这篇论文时，不需要把 Pre-LN/Post-LN 混成一件事。原论文结构先按 Post-LN 记；读现代代码时再检查具体实现。面试里如果被问到，你可以说“原论文图是 Add & Norm 在子层之后，后续很多实现会用 Pre-LN”。

Post-LN: y = LayerNorm(x + Sublayer(x))Pre-LN: y = x + Sublayer(LayerNorm(x))
两者都用了残差和归一化，但顺序不同，训练行为也会不同。

LayerNorm 和 BatchNorm 不一样

LayerNorm 通常对每个 token 的隐藏维做归一化，不依赖 batch 内其他样本。序列任务里 batch 长度、padding、生成步数经常变化，LayerNorm 比 BatchNorm 更自然。它不会在不同样本之间统计均值方差，而是在同一个 token 表示内部稳定尺度。

判断句：如果归一化依赖 batch 统计，那更像 BatchNorm；如果对单个样本的特征维归一化，那是 LayerNorm 的思路。

Position-wise FFN 做什么

FFN

同一套 MLP 应用于每个位置

论文里的 FFN 是两层线性变换，中间接 ReLU：max(0, xW1 + b1)W2 + b2。它对每个位置独立应用，参数在所有位置共享。注意力负责“位置之间读信息”，FFN 负责“每个位置内部加工信息”。

论文配置里，隐藏层维度常写作 d_ff=2048，输入输出仍是 d_model=512。先扩维再压回去，让每个 token 的表示经过更强的非线性变换。这个模块看起来朴素，但在 Transformer 层里占了相当多参数。

FFN(x) = max(0, xW1 + b1) W2 + b2
这里的 x 是某个位置的隐藏向量。每个位置独立算，但共享同一套 W1/W2。

别把 FFN 当成卷积或 attention

FFN 不看相邻 token，也不计算注意力权重。它对序列中每个位置做同样的通道变换。位置之间的关系已经在 attention 子层里混合过，所以 FFN 接收到的每个位置表示，本身已经包含上下文信息。

判断句：如果代码是 Linear -> ReLU -> Linear，并且输入输出形状都是 [B,N,D]，它大概率是在做 position-wise FFN。

最小 PyTorch 骨架

CODE

下面代码用 PyTorch 的 nn.MultiheadAttention 表达结构重点。真实工程还会处理 padding mask、dropout、embedding、位置编码和输出层，这里先把一层 encoder/decoder 的数据流看清楚。

正确示例：Post-LN 风格 EncoderLayerPyTorch
import torch
import torch.nn as nn

class EncoderLayer(nn.Module):
    def __init__(self, d_model=512, heads=8, d_ff=2048):
        super().__init__()
        self.self_attn = nn.MultiheadAttention(d_model, heads, batch_first=True)
        self.ffn = nn.Sequential(
            nn.Linear(d_model, d_ff),
            nn.ReLU(),
            nn.Linear(d_ff, d_model),
        )
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)

    def forward(self, x, src_key_padding_mask=None):
        attn_out, _ = self.self_attn(
            x, x, x,
            key_padding_mask=src_key_padding_mask,
            need_weights=False,
        )
        x = self.norm1(x + attn_out)     # Add & Norm
        ffn_out = self.ffn(x)
        x = self.norm2(x + ffn_out)      # Add & Norm
        return x
正确示例：DecoderLayer 的三段子层PyTorch
class DecoderLayer(nn.Module):
    def __init__(self, d_model=512, heads=8, d_ff=2048):
        super().__init__()
        self.self_attn = nn.MultiheadAttention(d_model, heads, batch_first=True)
        self.cross_attn = nn.MultiheadAttention(d_model, heads, batch_first=True)
        self.ffn = nn.Sequential(nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model))
        self.norm1 = nn.LayerNorm(d_model)
        self.norm2 = nn.LayerNorm(d_model)
        self.norm3 = nn.LayerNorm(d_model)

    def forward(self, tgt, memory, causal_mask=None, memory_padding_mask=None):
        self_out, _ = self.self_attn(tgt, tgt, tgt, attn_mask=causal_mask, need_weights=False)
        tgt = self.norm1(tgt + self_out)

        cross_out, _ = self.cross_attn(
            query=tgt,
            key=memory,
            value=memory,
            key_padding_mask=memory_padding_mask,
            need_weights=False,
        )
        tgt = self.norm2(tgt + cross_out)
        tgt = self.norm3(tgt + self.ffn(tgt))
        return tgt

代码读法

EncoderLayer 的 self-attention 传入 x,x,x，说明 Q/K/V 都来自源端当前表示。DecoderLayer 的第一段 self-attention 传入 tgt,tgt,tgt，但额外带 causal mask。第二段 cross-attention 的 query 是 tgt，key/value 是 memory，这就是 decoder 读取 encoder 输出。

LayerNorm 的数量也能帮你判断结构：encoder layer 有两个子层，所以常见两个 norm；decoder layer 有三个子层，所以常见三个 norm。不同实现会把 dropout 和 norm 顺序写得略有差异，但三段主线不变。

错误示例：cross-attention 来源写错

错误示例：decoder 没有读取 encoder memoryPyTorch
# 错：第二段又让 decoder 自己看自己
cross_out, _ = self.cross_attn(
    query=tgt,
    key=tgt,
    value=tgt,
    need_weights=False,
)

这段代码可能能跑，但它把 cross-attention 写成了第二次 self-attention。后果是 decoder 没有在这一层读取 encoder 的源句表示，翻译模型会失去关键源端信息入口。修法是让 key 和 value 来自 memory。

常见误区和修法

PITFALLS

1. 把 decoder 的两种 attention 混成一种

decoder self-attention 读目标前缀，cross-attention 读 encoder memory。它们公式相似，但 Q/K/V 来源不同。修法：每次判断 attention 类型，先问 Q 来自哪里，K/V 来自哪里。

2. 忘记 causal mask

训练时目标答案完整存在，如果不遮未来，模型会看到不该看的 token。损失可能下降很快，但推理时没有未来 token 可看。修法：decoder self-attention 必须使用上三角遮挡。

3. 把 LayerNorm 当成 BatchNorm

LayerNorm 通常对单个 token 的隐藏维归一化，不依赖 batch 统计。修法：解释归一化时说清统计维度，不要只说“归一化让训练稳定”。

4. 以为 FFN 会混合位置

FFN 是 position-wise 的，同一套 MLP 用在每个位置上。位置之间通信靠 attention。修法：看到 [B,N,D] 进出 FFN，确认它只变换 D 维。

5. 混淆原论文结构和现代变体

原论文图是 Add & Norm 在子层之后，很多现代代码用 Pre-LN。修法：读论文按原图记，读代码按实际顺序判断，不硬套。

6. 以为 encoder 输出只有一个向量

Transformer encoder 输出通常仍是每个源 token 一个向量，形状类似 [B,src_len,D]。decoder cross-attention 读取的是这整段 memory。修法：不要把它想成 RNN 最后的单个 hidden state。
