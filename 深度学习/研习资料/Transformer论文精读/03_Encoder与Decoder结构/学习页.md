# 主线位置 {#overview}
<!-- tag: OVERVIEW -->
<!-- layout: grid2 -->

## box
### Encoder：把源句变成可被读取的表示
encoder 的工作是把源句每个位置变成上下文表示。它不直接输出翻译；这些表示已经通过 self-attention 读过同一句子里的其他词，所以每个位置不再只是孤立词向量。
原论文里的 encoder 由 `N` 个相同层堆叠。每层包含 multi-head self-attention 和 position-wise feed-forward network，两个子层外面都包着 residual connection 和 layer normalization。你看到 “Add & Norm” 时，不要把它当成装饰框，它负责把子层输出和输入合回去，再稳定数值分布。
$$
 \text{EncoderLayer}(x)=\mathrm{LN}(x+\mathrm{FFN}(\mathrm{LN}(x+\mathrm{SelfAttn}(x))))
 $$
 这里写的是原论文的 Post-LN 读法。先记住子层顺序，再把实现细节和变体分开看。

## box soft
### Decoder：按生成顺序使用两类信息
decoder 要生成目标句。训练时它能拿到已经右移的目标序列，推理时它只能拿到已经生成的前缀。所以 decoder 的第一段 self-attention 必须 masked，当前位置不能看未来目标词。
每层 decoder 比 encoder 多一个注意力子层：先做 masked self-attention，读取目标端已知前缀；再做 encoder-decoder attention，用目标端表示作为 Query，去读取 encoder 输出作为 Key/Value；最后接 FFN。这样 decoder 既能维护目标语言内部顺序，又能对齐源句信息。
到这里你应该能判断：cross-attention 的关键是 decoder 用自己的 Q 去查 encoder 的 K/V。

# 术语表 {#terms}
<!-- tag: TERMS -->

| 术语 | 直觉解释 | 精确定义 | 常见误解 |
| --- | --- | --- | --- |
| Encoder | 把源序列读成一段 memory。 | 由多层 encoder layer 堆叠而成，输出通常保持为 [B,src_len,d_model] 的序列表示。 | 会保留整段序列表示，decoder 读取的是整段 memory。 |
| Decoder | 按目标端前缀逐步生成。 | 每层包含 masked self-attention、encoder-decoder attention 和 FFN 三个主子层。 | decoder 既要看 encoder 输出，也要建模已生成目标词。 |
| Memory | decoder 可读取的源端表示。 | encoder 最后一层输出，作为 cross-attention 的 K/V。 | memory 是 encoder 最后一层输出，不是训练缓存，也不是单个 hidden state。 |
| Cross-Attention | 目标端拿问题去源端找答案。 | Q 来自 decoder 当前表示，K/V 来自 encoder memory 的 attention。 | 公式像 self-attention，但 Q/K/V 来源不同，不能写成 tgt,tgt,tgt 。 |
| Add & Norm | 保留原输入，再稳定尺度。 | 残差连接加子层输出，再做 LayerNorm；原论文图中是 Post-LN。 | Add 和 Norm 两部分都重要，它们一起构成残差后的稳定路径。 |
| Position-wise FFN | 每个位置单独做通道加工。 | 同一套两层 MLP 应用于每个 token 表示，输入输出形状仍是 [B,N,d_model] 。 | 它不负责跨 token 通信，跨位置通信靠 attention。 |
| Teacher Forcing | 训练时喂真实目标前缀。 | decoder 训练时输入右移后的目标序列，用真实前缀预测下一个 token。 | 即使训练时有完整目标句，也不能让当前位置通过 self-attention 偷看未来词。 |

# Encoder 层 {#encoder}
<!-- tag: ENCODER -->

Self-attention 解决源句内部通信
encoder self-attention 的 Q、K、V 都来自同一条源序列。它让每个源词位置读取其他源词位置，形成带上下文的表示。例如翻译 “bank” 时，它可以通过上下文判断更接近河岸还是银行。这里只是把源端信息组织得更适合 decoder 读取。
判断句：如果 Q 来自源句，K/V 也来自源句，这就是 encoder self-attention。


Add & Norm 的作用
残差连接把子层输入直接加到子层输出上。它提供一条信息和梯度的直通路径，减少深层堆叠时信息被反复变换后丢失的问题。LayerNorm 再把每个 token 的隐藏维做归一化，让数值尺度更稳定。
判断句：看到 `x + Sublayer(x)`，你要意识到它保留了原表示，同时让子层学习增量修改。


FFN 在每个位置独立工作
position-wise FFN 对每个 token 位置使用同一套两层 MLP。它不在位置之间通信，位置之间的通信已经由 attention 完成。FFN 的作用是在每个位置内部做非线性特征变换，扩展维度再压回 `d_model`。
判断句：如果一个模块只沿隐藏维变换，不混合不同 token 位置，它更像 FFN。

步骤读什么写什么失败症状


Self-attention源序列当前表示更强的上下文表示读成别的序列就会把源端语义写歪
Add & Norm原输入和子层输出稳定后的残差结果漏残差时深层信息更容易断
FFN每个位置的隐藏向量通道变换后的表示把 FFN 当成跨位置通信会误解职责


这张表的目标是把“读、写、错了会怎样”拆开。你能把它讲清楚，就说明 encoder layer 不只是会画图了。

# Decoder 层 {#decoder}
<!-- tag: DECODER -->
<!-- layout: grid2 -->

## box
### 第一段：masked self-attention
decoder 的目标端输入通常是右移后的目标序列。训练时我们知道完整答案，但模型在位置 `i` 只能使用 `i` 之前的词，否则它会偷看未来。masked self-attention 就是在 score 矩阵里挡住未来位置，让 softmax 后未来 token 的权重为 0。
这一步只处理目标端内部关系。它回答的是“当前已经生成的目标前缀里，哪些词对下一个表示有用”。如果你把 causal mask 放到 encoder self-attention 上，通常就是错用；如果你忘了 decoder mask，训练损失可能很好看，但推理会崩。
$$
 \text{MaskedSelfAttn}(Q,K,V)=\mathrm{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}+\text{mask}\right)V
 $$
 mask 先于 softmax 生效。这样未来位置在概率上直接被压成 0。

## box
### 第二段：encoder-decoder attention
cross-attention 发生在 decoder 中间。此时 Query 来自 decoder 当前层的目标端表示，Key 和 Value 来自 encoder 的最终输出。也就是说，decoder 拿着“我现在要生成的目标位置需要什么信息”去源句 memory 里检索。
它和 encoder self-attention 的区别不在公式，而在 Q/K/V 来源。公式仍是 scaled dot-product attention；变的是 Q 来自目标端，K/V 来自源端。翻译任务里的对齐关系主要在这里被学习：目标端某个位置可以对源端多个位置分配注意力。
源句有 5 个 token，目标端当前前缀有 3 个 token 时，decoder self-attention 的 score 是 `[B,h,3,3]`，cross-attention 的 score 是 `[B,h,3,5]`。后者通常不需要 causal mask，但源句里的 padding 仍要挡住。

位置Q 来自哪里K/V 来自哪里是否 causal mask主要作用


Encoder self-attention源序列源序列通常不需要源端 token 互相读取
Decoder masked self-attention目标前缀目标前缀需要目标端按生成顺序建模
Encoder-decoder attentiondecoder 表示encoder 输出通常不用 causal mask，但要处理源端 padding目标端读取源端信息

# 残差与归一化 {#norm}
<!-- tag: ADD NORM -->
<!-- layout: grid2 -->

## box
### 论文原版是 Post-LN 结构
《Attention Is All You Need》图里写的是每个子层后接 Add & Norm，常见形式可以写成 `LayerNorm(x + Sublayer(x))`。这通常被称为 Post-LN，因为归一化发生在残差相加之后。很多现代 Transformer 变体会改成 Pre-LN，也就是 `x + Sublayer(LayerNorm(x))`，以改善更深模型训练稳定性。
$$
 y_{\text{post}}=\mathrm{LN}(x+\mathrm{Sublayer}(x))
 \qquad
 y_{\text{pre}}=x+\mathrm{Sublayer}(\mathrm{LN}(x))
 $$
 两者都用了残差和归一化，但顺序不同，训练行为也会不同。读论文时先按 Post-LN 记，读现代代码时再看实现顺序。

## box soft
### LayerNorm 和 BatchNorm 不一样
LayerNorm 只看每个 token 的隐藏维，不看 batch 内其他样本。序列任务里 batch 长度、padding、生成步数经常变化，LayerNorm 比 BatchNorm 更自然。它在同一个 token 表示内部稳定尺度，不在不同样本之间统计均值方差。
判断句：如果归一化依赖 batch 统计，那更像 BatchNorm；如果对单个样本的特征维归一化，那是 LayerNorm 的思路。

# Position-wise FFN {#ffn}
<!-- tag: FFN -->
<!-- layout: grid2 -->

## box
### 同一套 MLP 应用于每个位置
论文里的 FFN 是两层线性变换，中间接 ReLU：`max(0, xW1 + b1)W2 + b2`。它对每个位置独立应用，参数在所有位置共享。注意力负责“位置之间读信息”，FFN 负责“每个位置内部加工信息”。
原论文配置里，隐藏层维度常写作 `d_ff=2048`，输入输出仍是 `d_model=512`。先扩维再压回去，让每个 token 的表示经过更强的非线性变换。这个模块看起来朴素，但在 Transformer 层里占了相当多参数。

## box warn
### 别把 FFN 当成卷积或 attention
FFN 不看相邻 token，也不计算注意力权重。它对序列中每个位置做同样的通道变换。位置之间的关系已经在 attention 子层里混合过，所以 FFN 接收到的每个位置表示，本身已经包含上下文信息。
判断句：如果代码是 `Linear -> ReLU -> Linear`，并且输入输出形状都是 `[B,N,D]`，它大概率是在做 position-wise FFN。

# 最小实现 {#code}
<!-- tag: CODE -->

下面代码用 PyTorch 的 `nn.MultiheadAttention` 表达结构重点。真实工程还会处理 padding mask、dropout、embedding、位置编码和输出层，这里先把一层 encoder / decoder 的数据流看清楚。

EncoderLayer：Post-LN 风格复制

```python
import torch
import torch.nn as nn

class EncoderLayer(nn.Module):
 def __init__(self, d_model=512, heads=8, d_ff=2048):
 super().__init__()
 self.self_attn = nn.MultiheadAttention(d_model, heads, batch_first=True)
 self.ffn = nn.Sequential(
 nn.Linear(d_model, d_ff),
 nn.ReLU(),
 nn.Linear(d_ff, d_model),
 )
 self.norm1 = nn.LayerNorm(d_model)
 self.norm2 = nn.LayerNorm(d_model)

 def forward(self, x, src_key_padding_mask=None):
 attn_out, _ = self.self_attn(
 x, x, x,
 key_padding_mask=src_key_padding_mask,
 need_weights=False,
 )
 x = self.norm1(x + attn_out)
 ffn_out = self.ffn(x)
 x = self.norm2(x + ffn_out)
 return x
```

DecoderLayer：三段子层复制

```python
class DecoderLayer(nn.Module):
 def __init__(self, d_model=512, heads=8, d_ff=2048):
 super().__init__()
 self.self_attn = nn.MultiheadAttention(d_model, heads, batch_first=True)
 self.cross_attn = nn.MultiheadAttention(d_model, heads, batch_first=True)
 self.ffn = nn.Sequential(nn.Linear(d_model, d_ff), nn.ReLU(), nn.Linear(d_ff, d_model))
 self.norm1 = nn.LayerNorm(d_model)
 self.norm2 = nn.LayerNorm(d_model)
 self.norm3 = nn.LayerNorm(d_model)

 def forward(self, tgt, memory, causal_mask=None, memory_padding_mask=None):
 self_out, _ = self.self_attn(tgt, tgt, tgt, attn_mask=causal_mask, need_weights=False)
 tgt = self.norm1(tgt + self_out)

 cross_out, _ = self.cross_attn(
 query=tgt,
 key=memory,
 value=memory,
 key_padding_mask=memory_padding_mask,
 need_weights=False,
 )
 tgt = self.norm2(tgt + cross_out)
 tgt = self.norm3(tgt + self.ffn(tgt))
 return tgt
```

代码行读什么写什么错了会怎样


`self_attn(tgt, tgt, tgt)`目标前缀目标端上下文漏掉 causal mask 会偷看未来
`cross_attn(query=tgt, key=memory, value=memory)`目标端 query 和源端 memory源句对齐结果写成 `tgt,tgt,tgt` 就失去 encoder 信息
`tgt + cross_out`残差输入稳定后的 decoder 表示漏残差时深层训练更脆

### 8_padding_mask boundary

**Q:** padding mask

### 9_cross_bug diagnosis

**Q:** cross-attention 写成 self-attention

Explain: 错因是把 cross-attention 的 K/V 写成了 decoder 自己的表示。后果是 decoder 没有真正读取 encoder memory，跨端对齐能力会丢掉。修法是把 key 和 value 改成 memory ，即 encoder 最后一层输出。

### 10_mask_bug diagnosis

**Q:** 漏 causal mask

Explain: 训练时模型会偷看未来 token，所以损失可能异常低；但推理时未来 token 根本不存在，训练和推理的行为不一致，生成质量会明显变差。mask 的作用就是避免这种信息泄漏。

### 11_ffn_mistake diagnosis

**Q:** FFN 混位置

Explain: 不稳的地方在于 FFN 不负责跨 token 通信。不同 token 之间的信息交换主要由 attention 完成；FFN 只是对每个位置单独做通道变换，所以更像位置内的非线性加工，也不负责位置之间的通信。

### 12_norm_count code-reading

**Q:** norm 数量

Explain: 因为 encoder layer 有两个主子层，所以常见两个 norm；decoder layer 有三个主子层（masked self-attention、cross-attention、FFN），所以常见三个 norm。norm 的数量和子层数量对应。

### 13_encoder_use interview

**Q:** encoder 做什么

Explain: encoder 输出是一整段源序列的上下文表示，也就是 memory。decoder 在 cross-attention 里把自己的 query 送进去，读取这段 memory 的 key/value，从而把源句信息带进目标端生成过程。

### 14_decoder_three explanation

**Q:** decoder 三段

Explain: 第一段 masked self-attention 只看目标前缀，避免偷看未来；第二段 cross-attention 让目标端读取 encoder memory；第三段 FFN 在每个位置内部做通道加工。三段外面都要配残差和 LayerNorm。

### 15_residual explanation

**Q:** 残差作用

Explain: 残差连接给输入提供直通路径，避免每层都必须重新构造所有信息。深层堆叠时，子层只需要学增量修改，优化更容易，梯度也更容易往前传。

### 16_ffn_width explanation

**Q:** FFN 参数

Explain: 先扩到更大的中间维度，是为了让每个位置有更强的非线性变换空间；再压回 d_model ，是为了把结果重新接回主干表示。这样既提高通道加工能力，又不改变序列表示的外部接口。

### 17_feynman synthesis

**Q:** 费曼解释

Explain: 合格答案至少应包含：encoder 先把源句读成 memory；decoder 的 masked self-attention 只读目标前缀；cross-attention 用目标端 query 读取 encoder memory；每个子层外面都有残差和 LayerNorm；FFN 在每个位置内部做通道加工。再补一句：这样 decoder 既保持自回归顺序，又能读取源句信息，就能完成序列到序列建模。

Explain: 清单至少应覆盖：1）encoder self-attention 的 Q/K/V 是否同源；2）decoder self-attention 是否用了 causal mask；3）cross-attention 的 K/V 是否来自 encoder memory；4）encoder / decoder 的 norm 数量是否和子层数一致；5）FFN 是否只做位置内通道加工；6）输出是否仍保持序列形状。

# 参考资料 {#sources}
<!-- tag: source -->

- [Vaswani et al., *Attention Is All You Need*](https://arxiv.org/abs/1706.03762)。本页的 encoder / decoder 层顺序、三种 attention 的来源边界和 Post-LN 读法都以原论文为主。访问日期：2026-06-21。
- [PyTorch `torch.nn.MultiheadAttention` 文档](https://docs.pytorch.org/docs/stable/generated/torch.nn.MultiheadAttention.html)。用于理解 encoder self-attention、decoder self-attention 和 cross-attention 的最小实现接口。访问日期：2026-06-21。
- [PyTorch `torch.nn.LayerNorm` 文档](https://docs.pytorch.org/docs/stable/generated/torch.nn.LayerNorm.html)。用于理解 Add & Norm 里 Norm 的实现含义。访问日期：2026-06-21。
- [PyTorch `torch.nn.Linear` 文档](https://docs.pytorch.org/docs/stable/generated/torch.nn.Linear.html)。用于理解 FFN 的两层线性变换。访问日期：2026-06-21。
- [PyTorch `torch.transpose` 文档](https://docs.pytorch.org/docs/stable/generated/torch.Tensor.transpose.html)、[`torch.Tensor.masked_fill`](https://docs.pytorch.org/docs/stable/generated/torch.Tensor.masked_fill.html)、[`torch.softmax`](https://docs.pytorch.org/docs/stable/generated/torch.softmax.html)。用于检查 attention 里的轴、mask 和 softmax 顺序。访问日期：2026-06-21。
