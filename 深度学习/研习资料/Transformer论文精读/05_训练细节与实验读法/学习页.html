Noam warmup 学习率

schedule

论文的学习率会随训练步数变化。公式看起来像一个技巧，但它的直觉很清楚：训练初期模型表示还不稳，先用较小步子逐渐放大；过了 warmup 之后，学习率按步数平方根衰减，让后期更新更细。

$$ lrate = d_{model}^{-0.5} \cdot \min(step\_num^{-0.5},\ step\_num \cdot warmup\_steps^{-1.5}) $$

原论文使用 warmup_steps=4000。当 step_num 很小时，线性 warmup 项更小，学习率随 step 上升；当 step 过大后，step_num^{-0.5} 更小，学习率开始衰减。

分段读公式

这个公式的关键是 min。优化器真正使用的是两个候选学习率里较小的那个。warmup 前，step_num * warmup_steps^{-1.5} 这项随 step 线性增加；warmup 后，step_num^{-0.5} 这项变成较小项，于是曲线缓慢下降。

d_model^{-0.5} 是尺度因子。模型维度越大，初始学习率规模会被压低。这样读公式时，你会发现它把模型宽度、训练步数和 warmup 长度绑在一起。

为什么不直接用默认 Adam

Adam 本身会根据梯度一阶、二阶矩调整每个参数的步长，但它仍需要一个全局学习率。原论文把 optimizer 的 lr 当成外部 schedule 的载体，每一步重写参数组里的学习率。只写 torch.optim.Adam(model.parameters())，没有复现这条曲线。

这件事在复现实验时很实际：相同架构、相同数据，如果学习率计划不同，训练速度、稳定性和最终 BLEU 都可能变。论文读法里，训练配方就是证据的一部分。

阶段较小项学习率趋势训练含义
训练刚开始step_num * warmup_steps^{-1.5}随 step 线性升高让早期更新从小步开始，减少震荡。
warmup 附近两项接近达到峰值附近模型已经积累了一段稳定梯度历史。
warmup 之后step_num^{-0.5}按平方根衰减后期逐渐收小步长，细化参数。
改变 d_model整体乘 d_model^{-0.5}曲线整体缩放模型宽度进入学习率尺度。

目标分布与正则化

targets

训练配方里最容易被粗读的是 dropout 和 label smoothing，因为它们都常被概括成“防过拟合”。更好的读法是分清它们作用的对象：dropout 在中间表示或 attention 权重上制造扰动；label smoothing 改的是监督目标分布，让模型不要把训练标签当成绝对概率 1。

$$ q(k) = \begin{cases} 1-\epsilon_{ls}, & k = y \\ \epsilon_{ls}/(V-2), & k \ne y,\ k \ne \langle pad \rangle \\ 0, & k = \langle pad \rangle \end{cases} $$

这里用的是训练时常见的实现读法：正确类保留主要概率，非 padding 的其他类分到少量概率，padding 类不参与目标分布。原论文使用 ε_ls=0.1。

label smoothing 为什么可能让 PPL 变差

perplexity 来自模型给真实 token 的概率。label smoothing 把训练目标从 one-hot 改成软分布，会主动降低模型对正确类的极端置信度。于是它可能让 perplexity 看起来不如没有 smoothing 时漂亮，但生成译文的 BLEU 更好。

这类现象通常来自测量对象不同。PPL 更贴近概率校准和似然，BLEU 看生成结果与参考译文的 n-gram 匹配。论文里这类现象提醒你：不要把一个指标当成全部结论。

dropout 要问“放在哪里”

原论文在多个位置使用 dropout，包括每个子层输出后、attention 权重上，以及 embedding 与 positional encoding 相加后。读实现时应检查 dropout 是否作用在这些路径上，并确认训练/评估模式切换是否正确。

如果只在结果表里看到 dropout 比例，不看它落在哪些张量上，你只能得到很粗的结论。训练复现时，这些位置差异会变成行为差异。

组件作用对象训练后果读代码时先查
dropout子层输出、attention 权重、输入表示。减少对某些通道或连接的过度依赖。model.train() 下是否启用，位置是否贴近论文。
label smoothing交叉熵目标分布。降低过度自信，可能改善 BLEU。正确类概率、非 padding 类分配、padding 是否排除。
padding ignoreloss 中的无效位置。避免模型学习预测补齐 token。ignore_index 或手动 mask 是否生效。
causal maskdecoder self-attention 的未来位置。避免训练时偷看答案。mask 是否加在 softmax 前，方向是否正确。

你现在应该能做什么：看到 “label smoothing=0.1” 时，能说出它改的是目标分布，并解释为什么它和 BLEU/PPL 的关系不能简单等同。

指标读法

metrics

实验表要按测量对象读。BLEU 近似衡量译文与参考译文的 n-gram 重叠，perplexity 衡量概率模型对真实序列的困惑程度，训练成本衡量达到结果所花的硬件与时间。只看最高分，会把这些对象混成一个“好”字。

BLEU 读成“翻译基准上的自动指标”

BLEU 在机器翻译论文里常用，但它不能完整替代人类评价。它奖励和参考译文重叠的 n-gram，并带有长度惩罚。原论文报告 WMT 2014 En-De 和 En-Fr 上的 BLEU，是为了和当时机器翻译系统在同一基准上比较。

因此，读到 28.4 BLEU 时，合格表达应包括任务、数据、模型是否单模型、对比对象和训练成本。只说“BLEU 高，所以模型全面更好”太粗。

PPL 读成“概率模型困惑度”

perplexity 越低，通常表示模型给真实序列更高概率。但 label smoothing 改了目标分布，所以 PPL 与 BLEU 可能方向不同。读这种差异时先问：指标测的是概率、译文重叠，还是训练成本。

这条经验可以迁移到其他论文：当两个指标方向不一致，不要急着说实验矛盾，先拆开测量对象。

指标或条件它在测什么能支持的结论不能直接支持的结论
BLEU生成译文与参考译文的 n-gram 重叠。在该翻译基准上输出质量强。所有语义、事实性、风格都无问题。
perplexity概率模型对真实序列的困惑度。模型给真实 token 的概率情况。生成文本一定更符合人类偏好。
single model是否不用模型集成。单个模型本身就能达到该分数。完全排除训练技巧或数据处理影响。
training cost硬件数量和训练时间。并行训练效率是贡献之一。任何硬件和实现都能复现同样时间。
base / big模型规模和容量。规模增加是否带来更好结果。架构变化和规模变化可以自动分离。

你现在应该能做什么：读实验表时至少说出“指标测量对象 + 对比条件 + 结论边界”三件事。

实验结果怎么读

results

原论文摘要里的关键结果很有辨识度：Transformer 在 WMT 2014 English-to-German 上达到 28.4 BLEU，比当时已有最好结果包括 ensembles 高出 2 BLEU 以上；在 WMT 2014 English-to-French 上，单模型达到 41.8 BLEU，并用 8 块 GPU 训练约 3.5 天。要把这几句话一起读，不要拆成孤立口号。

结果点应该读成什么需要同时检查什么不要读成什么
WMT14 En-De 28.4 BLEU在该翻译基准上超过当时强基线。对比对象、是否 ensemble、数据处理。所有语言任务都已被证明最优。
WMT14 En-Fr 41.8 BLEU单模型在该任务上达到很强结果。模型规模、训练设置、解码设置。靠多个模型集成堆出的数字。
8 GPUs，约 3.5 天并行训练效率是论文贡献的一部分。硬件、实现、batching、模型大小。任何环境都能复现同样时间。
base 和 big两档模型展示容量与效果关系。参数量、训练成本、表格行是否对应同一设置。big 的分数可以代表所有 Transformer。

“效果强”之外还有“训练快”

论文标题强调 attention，但实验叙述还强调并行训练。RNN 类模型按时间步递推，训练并行度受序列顺序限制；Transformer 用 attention 和前馈网络堆叠，更容易把训练工作摊到矩阵计算上。BLEU 和训练成本一起出现，是为了说明架构在质量和效率上都占优势。

所以，当你复述贡献时，最好不要只说“分数高”。更准确的说法是：在当时机器翻译基准上，Transformer 用较低训练成本达到或超过强基线，并通过消融支持了 attention-only 架构的设计选择。

历史位置要说清

这篇论文的实验背景是 2017 年前后的机器翻译和解析任务，不包含后来大规模预训练、长上下文推理、KV cache、RoPE、FlashAttention 等生态。经典论文可以提供机制和读法，但不能把后来所有进展都反向塞回原文。

迁移到新任务时，结论需要重新验证。比如在长上下文模型里，标准 attention 的 N^2 成本会重新成为问题；在生成式大模型里，解码成本和缓存机制又会成为新主线。

你现在应该能做什么：复述 Transformer 实验贡献时，同时说质量、成本、对比对象和历史边界。

消融表读法

ablation

消融实验的任务是回答设计变量是否有用。原论文的消融表改变 head 数、key/value 维度、模型宽度、FFN 维度、dropout、attention 类型、位置表示等设置。每一行都应该读成一个小问题：改了什么，指标怎么变，变化能归因到哪里，边界是什么。

变量它在问什么读表时看什么结论边界
head 数多头是否比少头或单头更有效。BLEU/PPL 是否随 head 数变化，参数量是否同步变化。head 模式来自训练，不由人工固定语法功能。
d_k / d_v每个 head 的子空间维度是否足够。点积维度、输出容量、计算成本。维度变化可能同时影响参数量和优化。
d_model / d_ff模型宽度和 FFN 容量是否限制效果。效果提升是否伴随明显成本增加。容量提升不等于结构贡献本身。
dropout正则强度是否影响泛化。训练/验证指标是否出现过拟合迹象。最佳比例和数据规模、模型大小相关。
attention 类型替换 attention 机制后表现如何。是否控制其他条件，复杂度是否改变。单个任务上的结果不能覆盖所有场景。
位置表示固定正弦与可学习位置表示差别如何。长度外推、训练长度、指标变化。不等于后来所有位置机制的最终结论。
 

          
          
          
        

        

          

            
一行消融的合格表达

            
合格表达一般长这样：作者改变了某个变量，在同一任务和相近训练设置下观察到某个指标变化，因此这个变量可能贡献了某种效果；但如果参数量、计算成本或训练稳定性也改变了，结论要保留边界。

            
这个句式很实用。它迫使你把“变量、现象、解释、边界”都说出来，也避免把整张表读成“作者调了很多参数”。

          

          

            
消融和复现的关系

            
消融表能告诉你哪个设计值得优先保留。复现训练时，先保住 optimizer、schedule、mask、loss、模型规模这些基础项，再讨论是否改 head 数或 dropout。否则你看到的指标变化可能来自训练配方漂移，未必来自结构变量。

          

        

        
你现在应该能做什么：拿任意一行消融结果，写出变量、对照、指标变化和不能外推的范围。

Noam schedule 与 Adam 参数

        
import math
import torch


def noam_lr(step: int, d_model: int = 512, warmup: int = 4000) -> float:
    step = max(step, 1)
    scale = d_model ** -0.5
    warm = step * (warmup ** -1.5)
    decay = step ** -0.5
    return scale * min(decay, warm)


optimizer = torch.optim.Adam(
    model.parameters(),
    lr=1.0,
    betas=(0.9, 0.98),
    eps=1e-9,
)

for step, batch in enumerate(loader, start=1):
    lr = noam_lr(step, d_model=512, warmup=4000)

        group["lr"] = lr

    loss = compute_smoothed_loss(model, batch)
    loss.backward()

Label smoothing 与 padding 排除

        
import torch
import torch.nn.functional as F


def smooth_targets(target, vocab_size: int, smoothing: float = 0.1, pad_id: int = 0):
    confidence = 1.0 - smoothing
    low = smoothing / (vocab_size - 2)
    flat = target.reshape(-1)

    dist = torch.full((flat.numel(), vocab_size), low, device=target.device)
    dist.scatter_(1, flat[:, None], confidence)
    dist[:, pad_id] = 0.0
    dist[flat == pad_id] = 0.0
    return dist


def smoothed_kl_loss(logits, target, pad_id: int = 0, smoothing: float = 0.1):
    vocab_size = logits.size(-1)
    log_probs = F.log_softmax(logits.reshape(-1, vocab_size), dim=-1)
    gold = smooth_targets(target, vocab_size, smoothing, pad_id)
    token_loss = F.kl_div(log_probs, gold, reduction="none").sum(dim=-1)
    valid = target.reshape(-1).ne(pad_id)
    return token_loss[valid].mean()


        

          

            
代码验收顺序

            
先看 noam_lr 是否真的每一步更新参数组学习率，再看 Adam 的 betas 和 eps 是否贴近论文。接着检查 decoder 输入是否右移，target 是否是下一个 token，最后看 padding 是否同时从 attention 和 loss 中排除。

            
label smoothing 里最容易漏的是 padding。padding 类不应该分到概率，padding 位置也不应该贡献 loss。否则模型会被训练去解释补齐 token，指标看起来还能跑，但学习目标已经偏了。

固定学习率与 padding 漏处理

            
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
dist = torch.full((target.numel(), vocab_size), smoothing / vocab_size)
dist.scatter_(1, target.reshape(-1, 1), 1.0 - smoothing)
loss = F.kl_div(log_probs, dist, reduction="batchmean")

            
这段代码可能仍能训练，但它不能声称复现了论文配方：学习率没有使用 Noam schedule，Adam 参数也偏离论文设置；padding 类和 padding 位置都被放进了目标和损失。

          

        

        
你现在应该能做什么：用一张检查表判断训练 loop 是否接近原论文，别只看模型类名是不是 Transformer。

      


      

        

          
常见误区和修法

          pitfalls
        

        

          

            
只记 BLEU，不看条件

            
BLEU 必须和任务、数据、单模型/集成、模型规模、训练成本一起读。修法：复述结果时至少补上 WMT14、En-De/En-Fr、single model 或 ensemble、硬件时间。

          

          

            
把 warmup 当成一直升高

            
Noam schedule 是先升后降。修法：用公式里的 min 解释两个阶段，不要只背 warmup_steps=4000。

          

          

            
忽略 Adam 参数

            
原论文使用 β2=0.98、ε=1e-9，和许多默认设置不同。修法：复现时把 optimizer、schedule、warmup、batching 一起列出。

          

          

            
把 label smoothing 当成随机改标签

            
label smoothing 改的是目标概率分布，并不会随便把正确标签换掉。修法：写出正确类、其他类、padding 类的概率分配。

          

          

            
把 teacher forcing 当成答案泄漏

            
训练用真实前缀，但 decoder self-attention 仍要 causal mask。修法：检查右移目标和 causal mask 同时存在。

          

          

            
把消融表读成调参记录

            
消融是为了回答变量作用。修法：每一行按“变量、对照、指标、边界”写一句解释。

short answermask

              
teacher forcing 和 mask

              
训练时用了真实目标前缀，为什么 decoder self-attention 仍然必须有 causal mask？

              
答题目标：说清右移输入不等于允许当前位置看未来 token。

              
              
              

                参考答案 / 评分要点
                
合格答案要包含：teacher forcing 只是把真实前缀作为 decoder 输入；每个位置仍只能利用之前位置预测下一个 token；如果没有 causal mask，当前位置可以直接读取未来答案，训练指标会虚高，推理时会崩。

short answermetric

              
PPL 与 BLEU

              
为什么 label smoothing 可能让 perplexity 不那么好看，但 BLEU 变好？

              
答题目标：分别说出 PPL 和 BLEU 的测量对象。

              
              
              

                参考答案 / 评分要点
                
PPL 来自模型给真实 token 的概率，label smoothing 会降低模型对正确类的极端置信度；BLEU 看生成译文与参考译文的 n-gram 重叠，较好的泛化或置信度控制可能提升生成质量。两者测量对象不同，所以方向可以不一致。

              

            


            

              
diagnosisloss

              
padding 处理

              
做 label smoothing 时，padding 类和 padding 位置为什么要特殊处理？

              
答题目标：写出不处理的后果和修法。

              
              
              

                参考答案 / 评分要点
                
padding 是无效占位。若 padding 类分到概率，或 padding 位置进入 loss，模型会被训练去解释补齐 token，损失和梯度都会偏。修法是把 padding 类概率置 0，并用 mask 或 ignore_index 排除 padding 位置。

base 与 big

              
为什么读结果表时要区分 Transformer base 和 Transformer big？

              
答题目标：说出模型规模、效果和成本的关系。

              
              
              

                参考答案 / 评分要点
                
base 和 big 是不同规模模型。big 往往效果更强，但参数量、训练成本和优化行为也不同。读表时不能拿 big 的分数代表所有 Transformer，也不能用 base 的成本解释 big 的结果。

              

            


            

              
short answerwriting

              
结果复述

              
用 2 句话复述原论文实验贡献，必须包含质量、训练成本和对比对象。

              
答题目标：避免只写“BLEU 高”。

              
              
              

                参考答案 / 评分要点
                
可以写成：Transformer 在 WMT14 En-De 和 En-Fr 机器翻译基准上达到强 BLEU，且 En-Fr 单模型在 8 块 GPU 约 3.5 天训练后达到 41.8 BLEU。这个结果要和当时强基线、是否单模型、训练成本一起读，说明论文强调质量和并行训练效率。

diagnosisschedule

              
固定学习率诊断

              
有人用固定 lr=1e-3 加默认 Adam，说复现了 Transformer 原论文训练配方。哪里不稳？

              
答题目标：写出错因、后果和修法。

              
              
              

                参考答案 / 评分要点
                
错因：没有使用论文的 Adam 参数，也没有 Noam schedule 和 warmup。后果：训练早期稳定性、收敛速度和最终指标都可能变，不能说复现论文配方。修法：设置 betas=(0.9,0.98)、eps=1e-9，并逐步更新 Noam 学习率。

              

            


            

              
diagnosismetric

              
只看 BLEU 诊断

              
只说“BLEU 高，所以模型全面更好”哪里不稳？怎么改成合格表述？

              
答题目标：补上任务、指标边界、成本和对比对象。

              
              
              

                参考答案 / 评分要点
                
BLEU 只是在特定翻译基准上的自动指标，不能覆盖所有语言质量。合格表述要包含 WMT14 任务、BLEU 分数、是否单模型或 ensemble、训练成本和当时基线，并说明结论只在这些实验条件内成立。

              

            


            

              
diagnosisablation

              
消融混杂诊断

              
某一行同时增加 d_model、d_ff 和 head 数，结果变好。为什么不能只说“head 数增加带来提升”？

              
答题目标：指出变量混杂。

              
              
              

                参考答案 / 评分要点
                
因为多个变量同时改变，效果可能来自模型容量、FFN 宽度、head 数、优化稳定性或它们的交互。要归因给 head 数，最好控制其他条件，只改变 head 数，或至少在结论里保留“变量混杂”的边界。

              

            

          

        


        

          

            

              
五、完整讲清楚

              
最后把训练配方和实验读法串成完整解释。

            

            
复盘提示：费曼解释至少包含 Adam、Noam、dropout、label smoothing、BLEU、成本、base/big、消融边界。

          

          

            

              
synthesislong answer

              
费曼解释

              
用 6-9 句话解释：Transformer 原论文的训练配方和实验结果应该怎么读。

              
至少包含 teacher forcing 和 mask、Adam 参数、Noam 学习率、dropout、label smoothing、PPL 与 BLEU、base/big、消融和训练成本。

              
              
              

                参考答案 / 评分要点
                
合格答案应覆盖：训练时目标右移并保留 causal mask；Adam 使用论文给定参数；Noam 学习率先 warmup 后衰减；dropout 和 label smoothing 控制泛化与置信度；PPL 与 BLEU 测量对象不同；WMT14 En-De/En-Fr 结果要和单模型、训练成本、对比对象一起读；base/big 不能混讲；消融表用于判断变量贡献，但结论只在实验设置内成立。
