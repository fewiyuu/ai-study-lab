Attention Is All You Need · Block 05

训练细节与实验读法：别只记住 28.4 BLEU

Transformer 论文的贡献不只是提出架构，还展示了它能在机器翻译任务上训练得快、效果强。读实验部分时，要把训练配方、指标、对比对象和消融结果分开看。这样你才知道论文到底证明了什么，没有证明什么。

Adam + warmup→dropout / label smoothing→WMT BLEU→base / big 对比→消融表

学完后要能做三件事

  1. 能解释论文学习率公式的 warmup 和衰减逻辑。
  2. 能说明 label smoothing、dropout、Adam 参数在训练里各自管什么。
  3. 能读实验表：BLEU、训练成本、base/big 和消融结果分别说明什么。

这页会引用论文摘要中的关键结果：WMT14 En-De 28.4 BLEU，WMT14 En-Fr 41.8 BLEU，En-Fr 单模型训练约 3.5 天、8 块 GPU。

训练设置先分层看

SETUP

模型结构和训练配方要分开

论文提出的是 Transformer 架构，但实验效果来自一整套训练配方：Adam 优化器、特殊学习率计划、dropout、label smoothing、batching 策略，以及 base/big 两档模型规模。读论文时不能把所有提升都归因到“用了 attention”。

更稳的读法是先问：模型结构改了什么？训练设置用了什么？对比对象是谁？指标怎么计算？如果这些问题不分开，看到 BLEU 提升就很容易误读成某个单点技巧的功劳。

判断句：实验部分不是宣传数字列表，而是“结构假设 + 训练配方 + 对比证据”的组合。

关键训练组件

组件论文读法常见误解
Adam自适应优化器，论文使用特定 beta 和 epsilon。以为只要写 Adam 就等于复现论文。
warmup LR先升后降，避免训练初期步子太猛。以为学习率一直增大。
dropout用于残差、attention 权重等位置的正则化。以为只在最终分类层用。
label smoothing让目标分布不要是极端 one-hot。以为它一定提高困惑度。
BLEU机器翻译自动评价指标。以为 BLEU 高就代表所有语言质量都无争议。

warmup 学习率公式

LR

论文使用 Adam，并配了一个和模型维度、训练步数、warmup 步数相关的学习率计划。公式看起来长，但直觉只有两段：训练初期线性升高，过了 warmup 后按步数的平方根衰减。

lrate = d_model-0.5 · min(step_num-0.5, step_num · warmup_steps-1.5)
论文使用 warmup_steps=4000。前半段由 step_num · warmup_steps^-1.5 主导，后半段由 step_num^-0.5 主导。

4000

为什么不一开始就用大学习率

Transformer 训练早期参数还没形成稳定表示，attention 分数、残差路径和 LayerNorm 都在调整。如果一开始学习率太大，模型容易震荡。warmup 先把步子放小，再逐渐放大；过了峰值后逐步衰减，让训练后期更细。

判断句：warmup 不是为了让学习率永远变大，而是为了让训练早期更稳。

正则化和目标分布

REGULARIZE

dropout 降低共适应

论文在多个位置使用 dropout，包括子层输出、attention 权重和 embedding/position encoding 后。它的作用不是让模型“随机忘词”，而是在训练中减少某些通道或连接过度依赖，提升泛化。

判断句：看到 dropout 时，要问它作用在哪个张量上，而不是只记一个比例。

label smoothing 不让目标过分尖锐

普通交叉熵把正确类别当成概率 1，其他类别概率 0。label smoothing 会把一点概率质量分给其他类别，降低模型对训练标签的过度自信。论文使用了 label smoothing，说明他们关心的不只是拟合训练集。

它可能让模型的困惑度变差，但 BLEU 变好。原因是困惑度直接看目标概率，label smoothing 改了训练目标；BLEU 看生成结果的 n-gram 匹配，两者不是同一个指标。

Adam 参数也是配方的一部分

论文使用 Adam，并给出 β1=0.9、β2=0.98、ε=10^-9。这和很多默认 Adam 设置不同。复现论文时，只写“用了 Adam”不够，还要检查这些超参数和学习率计划。

判断句：优化器名字相同，不代表训练配方相同。

label smoothing 小计算

0.10

读指标时别混

困惑度关心模型给正确 token 的概率，BLEU 关心生成文本和参考译文的 n-gram 重叠。label smoothing 会影响概率校准和置信度，所以它可能让困惑度读起来不那么漂亮，但生成质量指标更好。

判断句：当两个指标方向不一致，先问它们各自测的是什么，不要急着说实验矛盾。

实验结果怎么读

RESULTS

摘要里的两个关键 BLEU

arXiv 摘要写到，Transformer 在 WMT 2014 English-to-German 上达到 28.4 BLEU，比当时已有最好结果包括 ensembles 高出 2 BLEU 以上；在 WMT 2014 English-to-French 上，单模型达到 41.8 BLEU，并且使用 8 块 GPU 训练约 3.5 天。

这几个数字要一起读。28.4 和 41.8 是任务指标；“single-model”说明不是靠多个模型集成；“3.5 days on eight GPUs”说明论文强调训练成本也低。它不是只说效果好，还在说并行训练带来的效率优势。

判断句：读结果时至少同时看质量、是否集成、训练成本和对比对象。

base 和 big

论文区分 Transformer base 和 Transformer big。big 模型更大，通常效果更强，但训练成本也更高。读表时不要把 big 的结果直接拿来代表所有 Transformer，也不要用 base 的成本去解释 big 的结果。

模型规模是实验变量。更大模型带来的提升不等于架构本身的全部贡献，所以论文才需要和其他模型、不同设置、消融结果一起读。

结果点应该读成什么不要读成什么
En-De 28.4 BLEU在该机器翻译基准上质量强所有语言任务都必然同等提升
En-Fr 41.8 BLEU单模型达到很高结果靠 ensemble 堆出来的数字
8 GPU 3.5 天并行训练效率是卖点之一任何硬件和实现都能复现这个时间
解析任务泛化架构不只限于翻译证明了所有 NLP 任务都最优

消融表读法

ABLATION

消融不是装饰，它回答“哪个设计真的有用”

论文里的消融实验会改变 head 数、key/value 维度、模型宽度、FFN 维度、attention 类型、dropout 等设置。读消融时不要只找最高分，而要看“改动了什么”和“指标怎样变化”。如果 head 数变少效果下降，你能说多头设计有帮助；如果参数量变化同时发生，就要小心不要把规模效应误当成结构效应。

消融表最适合训练你的论文阅读习惯：每一行只问一个问题。改变 head 数是在问多头数量；改变 d_ff 是在问 FFN 容量；改变 dropout 是在问正则强度。不要把整张表读成“作者试了很多参数”。

读完这一节，你应该能拿一行消融结果说清：变量、现象、可能解释和边界。

实验结论的边界

论文证明 Transformer 在当时机器翻译基准上非常强，也展示了并行训练效率。但它没有证明 attention 在所有任务上永远替代一切结构，也没有讨论后来长上下文、大规模预训练、推理缓存、位置机制变化这些问题。读经典论文要尊重它的历史位置，不把后来的知识硬塞回原文。

判断句：论文结论只在它的实验设置、数据、硬件和对比对象内成立；迁移到新场景要重新验证。

训练配方最小代码

CODE

下面代码只写训练配方骨架：Noam 学习率、label smoothing 损失和一次训练 step。它不是完整翻译系统，但能把论文实验细节落到可检查代码。

正确示例：Noam warmup 学习率PyTorch
import math
import torch

def noam_lr(step, d_model=512, warmup=4000):
    step = max(step, 1)
    return (d_model ** -0.5) * min(step ** -0.5, step * (warmup ** -1.5))

optimizer = torch.optim.Adam(
    model.parameters(),
    lr=1.0,
    betas=(0.9, 0.98),
    eps=1e-9,
)

for step, batch in enumerate(loader, start=1):
    lr = noam_lr(step)
    for group in optimizer.param_groups:
        group["lr"] = lr
    loss = compute_loss(model, batch)
    loss.backward()
    optimizer.step()
    optimizer.zero_grad(set_to_none=True)
正确示例：label smoothing 目标分布PyTorch
def smooth_targets(target, vocab_size, smoothing=0.1, pad_id=0):
    confidence = 1.0 - smoothing
    low = smoothing / (vocab_size - 2)
    dist = torch.full((target.numel(), vocab_size), low, device=target.device)
    dist.scatter_(1, target.view(-1, 1), confidence)
    dist[:, pad_id] = 0
    dist[target.view(-1) == pad_id] = 0
    return dist

代码读法

noam_lr 里的 min 决定了先升后降。step 很小时，线性 warmup 项更小，所以学习率上升；step 大到超过 warmup 后，step^-0.5 更小，学习率开始下降。

label smoothing 不是把正确类变错，而是把正确类概率从 1 降到 1-smoothing，剩余概率分给其他非 padding 类。padding 类和 padding token 位置要特殊处理，否则损失会把无效位置也纳入训练。

错误示例：scheduler 顺序和 padding 都漏

optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)

常见误区和修法

PITFALLS

1. 只记 BLEU，不看对比对象

BLEU 数字必须和数据集、是否单模型、训练成本、当时基线一起看。修法：读结果时至少说出任务和对比条件。

2. 把 warmup 当成一直升高

Noam 学习率先升后降。修法：用公式里的两个项解释前后阶段。

3. 以为 label smoothing 只会让所有指标更好

它可能让困惑度变差但 BLEU 变好。修法：区分概率指标和生成质量指标。

4. 忽略 Adam 参数

论文 Adam 的 β2=0.98、ε=1e-9 和很多默认值不同。修法：复现时把优化器参数、学习率计划一起列出。

5. 把 big 模型结果当成 base 成本

模型规模影响效果和成本。修法：读表时分清 Transformer base 和 Transformer big。

6. 把历史论文结论无限外推

论文实验主要围绕机器翻译和解析任务。修法：迁移到长上下文、预训练大模型或新硬件时重新验证。
