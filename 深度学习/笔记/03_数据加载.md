# 03 · 数据加载（Dataset & DataLoader）

## 两个类的职责

| | Dataset | DataLoader |
|---|---|---|
| 职责 | 定义"怎么取一条" | 负责"怎么取一批" |
| 必实现 | `__len__` + `__getitem__` | 不需要继承，直接用 |
| 管什么 | 数据来源（文件/内存/数据库） | 批量大小、随机打乱、并行加载 |

Dataset 是一个抽象接口，不管数据格式——里面可以是 NumPy、Tensor、图片文件路径、数据库查询结果。PyTorch 只通过 `__getitem__` 取数据。

## 最小模板

```python
from torch.utils.data import Dataset, DataLoader
import torch

class MyDataset(Dataset):
    def __init__(self, features, labels):
        self.features = features
        self.labels = labels

    def __len__(self):
        return len(self.features)

    def __getitem__(self, idx):
        return self.features[idx], self.labels[idx]

ds = MyDataset(X, y)
loader = DataLoader(ds, batch_size=32, shuffle=True)

for X_batch, y_batch in loader:   # 每次取一批
    pred = model(X_batch)
    loss = criterion(pred, y_batch)
```

## 三个关键参数

| 参数 | 作用 | 默认 |
|------|------|------|
| `batch_size` | 每批取几条 | 1 |
| `shuffle` | 每个 epoch 是否打乱顺序 | False |
| `num_workers` | 用几个子进程并行读数据 | 0 |

## 三个术语

| 术语 | 含义 | 计算公式 |
|------|------|---------|
| batch | 一次前向+反向用的数据量 | — |
| epoch | 全部训练数据被过一遍 | — |
| iteration | 一个 batch 完成一次前向+反向 | `样本数 ÷ batch_size` |

> [!warning] 最后一个 batch 可能不满
> 总数据 1000 条，batch_size=33，前面 30 个 batch 各 33 条，最后一批只有 10 条（`1000 - 33×30`）。
> DataLoader 默认不丢弃不满的 batch（`drop_last=False`）。

## epoch / batch / iteration 的关系

```
1 epoch = (样本数 ÷ batch_size) 次 iteration
总 iterations = epoch 数 × 每 epoch 的 iteration 数

例：1000 条数据，batch_size=50，训练 3 个 epoch → 一共 20×3 = 60 次 iteration
```

> [!warning] 批卷 + 课后作业踩坑
> 第 2 题：shuffle 不改变 iteration 次数，100 条数据，batch_size=20 → 5 次 iteration，选 B 不是 D。
> 第 11 题：除不尽时最后一批是 10 条（`1000 - 33×30`），不是 1 条。

## 训练 vs 验证

| | 训练 | 验证 |
|---|---|---|
| `shuffle` | True（防模型学到数据顺序假关联） | False（保证结果可比） |
| `no_grad` | 不需要（梯度正常算） | 必须加（不计算梯度，省显存） |

## Windows 注意

`num_workers > 0` 在 Windows 下如果没写 `if __name__ == "__main__":` 保护，多进程会递归创建导致卡死。学习阶段 `num_workers=0` 完全够用。

## 费曼解释

> Dataset 和 DataLoader 负责把数据喂给模型，但分工不同。**Dataset 是"菜单"**——它定义怎么读出每一条数据（不管数据是从硬盘、网络还是内存来的）。**DataLoader 是"服务员"**——按你的要求（一次上几盘、要不要随机顺序、叫几个人同时上菜）从菜单取数据，组装好送到模型面前。分成两个类的意义是：不管你的数据是图片、CSV 还是数据库，只需要写 Dataset 告诉 PyTorch "怎么读一条"；批量和并行不用你操心，DataLoader 包了。
