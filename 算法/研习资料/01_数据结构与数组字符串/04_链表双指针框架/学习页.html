LINKED LIST 04

用两个指针补上链表不能随机访问的短板

链表没有数组下标，不能直接跳到第 k 个位置。双指针就是在链表上制造位置关系：一个走得快、一个走得慢；一个先走 n 步、另一个再一起走；或者两个链表的指针通过换头对齐长度。

链表双指针为什么必要

why

数组能用 `nums[i]` 直接访问第 i 个元素，链表不行。链表只能从某个节点沿 `next` 一步一步走。双指针的作用，就是让两个节点在遍历过程中保持某种关系，弥补不能随机跳转的问题。

这种关系可以是速度差：fast 比 slow 快；也可以是距离差：fast 先走 n 步；还可以是同步移动：两个链表指针一起走，靠换头抵消长度差。

写链表双指针题时，变量名只是表面。真正要维护的是关系。只要关系在每轮循环后仍成立，代码就有稳定性。

判断句

链表题只要问位置、倒数、相遇、中点，先考虑用指针关系替代下标。

快慢指针：用速度差定位

fast slow

快慢指针最常见写法是 slow 每次走一步，fast 每次走两步。fast 先到尾部，slow 就在中点附近；如果链表有环，fast 迟早会追上 slow。

这个技巧背后的直觉是速度差。两个指针在同一条链上跑，fast 每轮比 slow 多走一步。如果没有环，fast 会撞到空；如果有环，fast 会在环里不断追近 slow。

边界要写稳。循环条件常见是 `while fast and fast.next:`，因为 fast 要走两步，必须确认当前和下一步都存在。少一个判空，就可能访问空节点的 next。

判断句

需要中点或判断环时，快慢指针比先计算长度更直接。

前后指针：保持固定距离

gap

倒数第 n 个节点很适合前后指针。让 fast 先走 n 步，再让 slow 和 fast 一起走。fast 到末尾时，slow 距离末尾正好还剩 n 步。

删除倒数节点时通常需要 slow 停在被删节点的前一个位置。这时 dummy 很有用：让 slow 从 dummy 出发，fast 也从 dummy 出发，距离关系更统一。

前后指针的边界是 n 是否超过长度、删除头节点是否特殊。真实面试题通常保证 n 合法，但你仍要知道 dummy 如何消除头节点特殊情况。

判断句

看到倒数第 k 个、删除倒数节点，先让一个指针领先 k 步。

虚拟头结点：把头节点当普通节点处理

dummy

虚拟头结点 dummy 不保存真实数据，它的 next 指向原链表头。它最大的价值是：当头节点也可能被删除或替换时，你不用写一堆特殊判断。

82 删除排序链表中的重复元素 II 会删除所有重复值，只保留没有重复的节点。如果重复段在链表头，比如 `1->1->2`，没有 dummy 就要特别处理 head；有 dummy 时，只需要维护 prev 指向结果链表的尾部。

dummy 的返回也要记住：最终返回 `dummy.next`，不是原来的 head。因为原 head 可能已经被删掉或改接。

判断句

只要头节点可能改变，就先考虑 dummy。

两数相加：顺着链表模拟逐位处理

add

2 两数相加把数字逆序存在链表里。每个节点是一位，顺着链表从低位到高位加，维护一个 `carry` 进位。这里不是典型快慢指针，但依然是链表多指针：p、q 分别扫两个输入链表，cur 负责接结果链表。

445 两数相加 II 是正序链表，不能直接从低位开始。常见做法是先用栈把节点值倒过来，或者反转链表再加。它把链表和栈联系起来：链表顺序不方便从尾部访问，就借助后进先出的结构。

这类题要注意循环条件：只要 p、q 或 carry 还有一个存在，就要继续处理。否则最后进位会丢。

判断句

链表逐位题要明确每个指针扫哪条链，结果链由哪个指针负责追加。

复杂度与断链检查

debug

链表双指针通常只扫一遍，所以时间复杂度多为 `O(n)`，额外空间常是 `O(1)`。但如果你用了栈或新建结果链，空间要按实际情况说明。

链表错误最常见的是断链、漏节点、死循环。断链来自覆盖 next 前没保存后继；漏节点来自推进顺序错；死循环来自让某个节点指回前面后没有正确移动当前指针。

调试时不要只看输出。拿 3 到 5 个节点的小链表手动走，每次写出 prev、cur、fast、slow 指向哪里。你能追踪状态，才能改对边界。

判断句

链表代码的正确性，靠指针关系和更新顺序一起保证。

渐进例子与代码

code reading

这几段代码按从小到大的顺序安排。读代码时先看状态放在哪里，再看每一步读什么、写什么。

1. 快慢指针找中点

fast 每次走两步，slow 每次走一步。循环结束时 slow 在中点附近。

正确示例：快慢指针找中点Python
def middle_node(head):
    slow = head
    fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow

2. 删除倒数第 n 个节点

dummy 消除删除头节点的特殊情况，fast 先走 n+1 步让 slow 停在被删节点前面。

正确示例：删除倒数第 n 个节点Python
def remove_nth_from_end(head, n):
    dummy = ListNode(0, head)
    slow = dummy
    fast = dummy
    for _ in range(n + 1):
        fast = fast.next
    while fast:
        slow = slow.next
        fast = fast.next
    slow.next = slow.next.next
    return dummy.next

3. 两数相加的结果链

p、q 扫输入，cur 接结果。carry 不能漏，它可能在两个链表都空后继续产生新节点。

正确示例：两数相加的结果链Python
def add_two_numbers(l1, l2):
    dummy = ListNode(0)
    cur = dummy
    carry = 0
    p, q = l1, l2
    while p or q or carry:
        x = p.val if p else 0
        y = q.val if q else 0
        s = x + y + carry
        carry = s // 10
        cur.next = ListNode(s % 10)
        cur = cur.next
        p = p.next if p else None
        q = q.next if q else None
    return dummy.next

4. 错误示例：反转链表前没保存后继

这段代码改掉 cur.next 后，再想从 cur.next 继续往后走就已经回头了。

错误示例：错误示例：反转链表前没保存后继Python
def reverse(head):
    prev = None
    cur = head
    while cur:
        cur.next = prev
        prev = cur
        cur = cur.next  # 错：这里会回到 prev，后面的链丢了
    return prev

常见误区

debug

fast 判空条件少写一层

错因：fast 要走两步，却只判断 fast 存在。

后果：访问 fast.next.next 时可能空指针。

修法：循环条件写 `while fast and fast.next`。

删除头节点时忘记 dummy

错因：默认 head 不会变。

后果：头部重复或头节点删除时返回错误链表。

修法：用 dummy 指向 head，最后返回 dummy.next。

反转前不保存 nxt

错因：改 next 太早。

后果：后续节点丢失或形成死循环。

修法：先 `nxt = cur.next`，再改 `cur.next`。

前后指针距离差一

错因：没有明确 slow 要停在目标还是目标前一个。

后果：删除错节点。

修法：先定义 slow 的最终位置，再决定 fast 先走 n 还是 n+1。

两数相加漏掉最后 carry

错因：循环条件只写 p or q。

后果：比如 5+5 得不到 10 的高位 1。

修法：循环条件加上 `or carry`。

返回原 head

错因：忘记链表头可能改变。

后果：删除或重建链表后返回旧头。

修法：涉及 dummy 或结果链时返回 dummy.next。
