数组题的指针不是装饰。每个指针都要有职责：读、写、缩小范围、跳过重复、维护答案。职责说不清，代码很快就会变成乱移动。

把两个下标变成稳定的扫描策略

数组有下标，所以双指针在数组里非常自然。一个指针从左走，一个从右走，可以夹逼答案；一个指针快扫，一个指针慢写，可以原地删除；两个数组从尾部合并，可以避免覆盖还没读的元素。

观察什么

读图时先找状态变量，再判断下一步移动哪个指针或修改哪个位置。

左右夹逼：移动依据来自比较

two ends

左右夹逼用两个指针从数组两端向中间走。它适合有序数组、回文判断、两端贡献不同的题。核心是每次比较后能确定哪一边可以移动。

比如验证回文串，左边和右边字符相等，就一起向中间；不相等就失败。比如有序数组的平方，最大平方一定来自绝对值更大的端点，所以可以从结果数组末尾往前写。

左右夹逼不是两个指针随便移动。移动必须有理由：因为当前比较已经排除了某一侧，或者已经把某一侧写入答案。

判断句

只有当端点比较能决定下一步时，左右夹逼才稳。

快慢指针：读写职责分离

slow fast

快慢覆盖里，fast 负责从左到右读取每个元素，slow 负责指向下一次写入位置。这个模板常用来原地删除、保留、移动元素。

80 删除有序数组中的重复项 II 的关键是有序。因为相同元素相邻，所以只要比较当前值和 `nums[slow - 2]`，就能判断保留当前值后是否超过两次。没有有序条件，这个判断就不成立。

快慢指针的好处是避免真正删除。数组中间删除会移动大量元素，而覆盖只是把应保留的元素写到前面，最后返回新长度。

判断句

题目要求原地保留部分元素时，优先想 fast 读、slow 写。

逆向合并：从尾部写避免覆盖

merge

88 合并两个有序数组的陷阱是 nums1 前面还存着未读元素。如果从前往后写，可能覆盖掉还没比较的值。

从尾部写就简单：p1 指向 nums1 的有效尾部，p2 指向 nums2 尾部，write 指向 nums1 最末端。每次把较大的数写到 write，然后对应指针左移。

这种写法依赖两个数组本来有序。因为最大值一定在两个尾部之一，所以写完最大值后，剩下问题仍然是同样的合并问题。

判断句

合并到第一个数组且后面有空位时，优先从尾部写。

重复跳过：有序条件减少判断成本

duplicates

数组双指针很多题依赖有序。因为有序后，相同值挨在一起，大小关系单调，指针移动就有依据。三数之和、删除重复、合并有序数组都吃这个条件。

跳过重复时，要注意跳过的时机。比如 nSum 题常在固定一个数后跳过相同值，防止答案重复；删除重复题则是根据保留次数判断是否写入。

不要把“跳过重复”当成固定代码片段。先说清重复会造成什么问题：是答案重复、保留次数超限，还是无意义比较。不同问题的跳过位置不同。

判断句

只要使用重复跳过，先说明重复会破坏哪条规则。

边界：相遇、空数组和写指针范围

boundary

左右指针通常以 `left < right` 或 `left <= right` 控制，区别在于中间元素是否需要处理。回文判断常用 `left < right`，有序平方从结果尾部写时要覆盖所有位置，常用 `while left <= right`。

快慢覆盖要小心 slow 的初始值。80 允许最多两个重复，所以前两个元素天然可保留，常从 slow=0 统一处理，或从 slow=2 开始。不同写法都可以，但解释必须一致。

空数组、单元素、全重复、没有重复、两个数组其中一个为空，都是数组双指针常见边界。写完模板后用这些用例手动跑一遍。

判断句

循环条件决定中间元素是否被处理，不能凭感觉写。

复杂度：一趟扫描但要看排序前提

complexity

数组双指针常见时间复杂度是 `O(n)`，因为每个指针单调移动，不会反复回头。空间如果原地写，通常是 `O(1)`。

但如果题目需要先排序，比如三数之和，排序成本是 `O(n log n)`，后续双指针才是 `O(n^2)` 或 `O(n)` 的一部分。不要把整个题只说成双指针扫描成本。

判断复杂度时，看每个元素最多被指针经过几次。如果指针只前进不后退，通常就能说明线性扫描。

判断句

双指针快的前提是指针单调移动，不重复枚举同一段。

渐进例子与代码

code reading

这几段代码按从小到大的顺序安排。读代码时先看状态放在哪里，再看每一步读什么、写什么。

1. 验证回文串的左右夹逼

忽略非字母数字后，左右字符必须相同。每轮比较后两个指针都向中间移动。

正确示例：验证回文串的左右夹逼Python
def is_palindrome(s):
    left, right = 0, len(s) - 1
    while left < right:
        while left < right and not s[left].isalnum():
            left += 1
        while left < right and not s[right].isalnum():
            right -= 1
        if s[left].lower() != s[right].lower():
            return False
        left += 1
        right -= 1
    return True

2. 删除有序数组重复项 II

有序数组中，当前 x 如果大于 nums[slow-2]，保留它不会让同一个数超过两次。

正确示例：删除有序数组重复项 IIPython
def remove_duplicates(nums):
    slow = 0
    for x in nums:
        if slow < 2 or x > nums[slow - 2]:
            nums[slow] = x
            slow += 1
    return slow

3. 合并两个有序数组

从尾部写最大值，避免覆盖 nums1 前面还没读的有效元素。

正确示例：合并两个有序数组Python
def merge(nums1, m, nums2, n):
    p1, p2 = m - 1, n - 1
    write = m + n - 1
    while p2 >= 0:
        if p1 >= 0 and nums1[p1] > nums2[p2]:
            nums1[write] = nums1[p1]
            p1 -= 1
        else:
            nums1[write] = nums2[p2]
            p2 -= 1
        write -= 1

4. 错误示例：从前往后合并覆盖未读元素

如果 nums1 前面还有有效元素，从前写会覆盖它们，后面就无法比较。

错误示例：错误示例：从前往后合并覆盖未读元素Python
def bad_merge(nums1, m, nums2, n):
    i = j = 0
    while i < m and j < n:
        if nums2[j] < nums1[i]:
            nums1[i] = nums2[j]  # 错：覆盖了 nums1[i] 原值
            j += 1
        i += 1

常见误区

debug

指针职责不清

错因：left、right、slow、fast 都只是随手命名。

后果：循环里不知道该移动谁。

修法：写代码前先定义每个指针读什么、写什么。

左右夹逼没有移动依据

错因：不管比较结果都随便移动。

后果：可能漏答案或死循环。

修法：每次移动都要能解释排除了哪一侧。

快慢覆盖后还返回原长度

错因：忘记 slow 才是新数组长度。

后果：结果前缀正确，但返回长度错误。

修法：返回 slow，并只看 nums[:slow]。

合并从前往后写

错因：忽略 nums1 前面还有未读元素。

后果：覆盖有效值。

修法：从尾部 write 开始写。

跳过重复越界

错因：while 跳重复时不检查边界。

后果：访问越界或漏中间元素。

修法：跳重复前加 `left < right` 等边界条件。

把有序前提用到无序数组

错因：只记住模板，没看输入条件。

后果：比较 slow-2 或左右夹逼失去依据。

修法：先确认数组有序，或先排序并计入复杂度。
