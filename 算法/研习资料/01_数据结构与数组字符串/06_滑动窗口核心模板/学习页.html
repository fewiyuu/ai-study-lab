用左右边界维护一个正在变化的区间

滑动窗口的直觉是：不要为每个起点重新扫描一遍区间，而是让窗口在数组或字符串上向右滑动。右边界加入新元素，左边界移除旧元素，窗口里的状态同步更新。

滑动窗口只管连续区间

scope

滑动窗口处理的是连续子数组或子串。窗口从左到右滑动，内部元素始终是一段连续区间。子序列可以跳着选，通常不适合直接用滑动窗口。

题目里出现“最长子串”“最短子数组”“连续 1”“长度为 k 的窗口”时，可以考虑窗口。题目如果问任意子集、排列组合、非连续选择，就不要硬套。

连续性带来一个好处：右边加入一个元素，左边移除一个元素，窗口状态可以增量更新，不需要重算整个区间。

判断句

先确认答案必须连续，再考虑滑动窗口。

基本骨架：右扩张，左收缩

template

可变窗口的常见骨架是：右指针不断加入新元素；当窗口不满足条件或已经满足可收缩条件时，左指针移动；每次在合适时机更新答案。

不同题的区别在于收缩条件。209 最小长度子数组是 `sum >= target` 时收缩；无重复子串是出现重复时收缩；713 乘积小于 K 是乘积过大时收缩。

窗口状态必须和边界一致。加入右端后要更新状态，移除左端前后也要同步。状态和边界一旦错位，答案会非常隐蔽地错。

判断句

写窗口代码前，先写清“什么时候扩大、什么时候缩小、什么时候更新答案”。

计数窗口：字符和频次要同步

count

字符串窗口常维护字符计数。3 无重复字符最长子串里，窗口内每个字符次数不能超过 1；438 找异位词里，窗口频次要和目标频次匹配。

计数窗口的难点是删除。左边界移动时，离开的字符必须从计数表里减掉。计数减到 0 后是否删除 key，要看你的判断方式。如果用 key 数量或 distinct 计数，就要谨慎处理。

不要把计数表和字符串下标分开想。计数表就是当前窗口的影子。窗口变了，表必须同步变。

判断句

字符窗口每移动一端，都要同步更新频次。

固定窗口和可变窗口

fixed variable

固定窗口长度已知，比如长度为 k 的子数组、最近 k 个元素。右边加入一个元素后，如果窗口超过 k，就从左边移除一个元素。

可变窗口长度不固定，长度由条件决定。比如最小长度子数组要在满足 sum 条件后尽量缩短；最大连续 1 III 要保证窗口内 0 的数量不超过 k。

固定窗口更像节拍器，可变窗口更像调节器。两者都维护连续区间，但收缩触发条件不同。

判断句

窗口长度由题目直接给定就是固定窗口；由合法条件决定就是可变窗口。

答案更新时机

answer

滑动窗口题很容易错在答案更新时机。求最长合法窗口，通常在窗口已经合法时更新最大长度；求最短满足条件窗口，通常在满足条件后先更新最短长度，再收缩。

比如 209 中，`sum >= target` 时当前窗口已经满足要求，可以更新答案，然后尝试移除左端寻找更短。无重复子串则要先处理重复，让窗口恢复合法，再更新最大长度。

更新答案前要明确当前窗口是不是你想统计的窗口。不要在非法窗口上更新最长，也不要在还没满足条件时更新最短。

判断句

最长题通常合法后更新；最短题通常满足条件时边更新边收缩。

复杂度：左右指针都只前进

complexity

滑动窗口常是 `O(n)`，因为 left 和 right 都只向右走，每个元素最多进入窗口一次、离开窗口一次。即使里面有 while，整体也不是简单乘成 `O(n^2)`。

但如果窗口内部每次都重新扫描或排序，复杂度就会变坏。239 最大值窗口如果每次求 max 都扫描窗口，复杂度会变成 `O(nk)`，所以需要单调队列维护候选最大值。

空间复杂度取决于状态。整数 sum 是 `O(1)`，字符计数表通常是字符集大小，哈希表可能到 `O(k)` 或 `O(n)`。

判断句

只要每个元素进出窗口次数有限，窗口主循环就是线性的。

渐进例子与代码

code reading

这几段代码按从小到大的顺序安排。读代码时先看状态放在哪里，再看每一步读什么、写什么。

1. 最小长度子数组

右边扩张增加 sum，满足目标后不断收缩并更新最短长度。

正确示例：最小长度子数组Python
def min_sub_array_len(target, nums):
    left = 0
    total = 0
    ans = float("inf")
    for right, x in enumerate(nums):
        total += x
        while total >= target:
            ans = min(ans, right - left + 1)
            total -= nums[left]
            left += 1
    return 0 if ans == float("inf") else ans

2. 无重复字符的最长子串

右端加入字符后，如果重复，就移动左端直到重复消失。合法后更新最大长度。

正确示例：无重复字符的最长子串Python
def length_of_longest_substring(s):
    left = 0
    count = {}
    ans = 0
    for right, ch in enumerate(s):
        count[ch] = count.get(ch, 0) + 1
        while count[ch] > 1:
            out = s[left]
            count[out] -= 1
            left += 1
        ans = max(ans, right - left + 1)
    return ans

3. 乘积小于 K 的子数组

窗口乘积过大就左移。每个 right 固定时，以 right 结尾的合法子数组数量是窗口长度。

正确示例：乘积小于 K 的子数组Python
def num_subarray_product_less_than_k(nums, k):
    if k <= 1:
        return 0
    left = 0
    prod = 1
    ans = 0
    for right, x in enumerate(nums):
        prod *= x
        while prod >= k:
            prod //= nums[left]
            left += 1
        ans += right - left + 1
    return ans

4. 错误示例：窗口移动但计数不删

左边界移动后，如果不减少离开字符的次数，计数表就不再代表当前窗口。

错误示例：错误示例：窗口移动但计数不删Python
while count[ch] > 1:
    left += 1  # 错：忘记 count[s[left]] -= 1

常见误区

debug

把非连续问题硬套窗口

错因：只看到最长/最短，没有确认连续性。

后果：漏掉可跳选答案或逻辑不成立。

修法：先确认题目问子数组/子串而不是子序列。

窗口状态和边界不同步

错因：移动 left/right 后没更新 sum/count。

后果：条件判断基于旧状态，答案错。

修法：每次元素进入或离开窗口，都同步修改状态。

答案更新时机错

错因：非法窗口也更新最长，或未满足条件就更新最短。

后果：结果偏大或偏小。

修法：先定义当前窗口何时合法，再决定更新位置。

while 条件写成 if

错因：以为收缩一次就够。

后果：窗口可能仍然非法。

修法：只要条件仍不满足，就继续收缩。

239 每次扫描窗口最大值

错因：不知道最大值不能靠 sum/count 增量维护。

后果：复杂度变成 O(nk)。

修法：使用单调队列保存候选最大值。

边界长度差一

错因：混用 [left,right] 和 [left,right)。

后果：长度计算差 1。

修法：固定一种窗口定义，代码和长度公式保持一致。
