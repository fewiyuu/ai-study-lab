ARRAY 07

用辅助数组把区间问题降维

数组区间题有两类高频需求：反复查询某段和，或者反复给某段加一个值。直接每次扫描区间会重复做很多工作。前缀和和差分数组就是把重复工作提前或延后。

前缀和先累计，查询时相减；差分数组先记录边界变化，最后一次还原。你之前接触过的 560 和为 K 的子数组，就是前缀和 + 哈希表的经典题。差分数组常出现在航班预订、区间加法、拼车这类“很多次区间更新”题中。

前缀和：把区间求和变成相减

prefix

前缀和数组 `pre` 的定义是：`pre[i]` 表示原数组前 i 个元素的和。注意是前 i 个，不是到下标 i。这样 `pre[0] = 0`，`pre[1] = nums[0]`，`pre[2] = nums[0] + nums[1]`。

为什么要多一个 0？因为这样查询从下标 0 开始的区间也不用特殊处理。闭区间 `[l, r]` 的和统一写成 `pre[r + 1] - pre[l]`。

这个公式的直觉是：`pre[r+1]` 包含 0 到 r 的所有元素，`pre[l]` 包含 0 到 l-1 的元素。相减后，前面多余部分被抵消，只剩 l 到 r。

判断句

只要有多次静态区间求和，先考虑前缀和。

下标边界：r+1 不是随便加的

index

前缀和最常见错误是下标差一。因为 `pre` 比 `nums` 长 1，所以 nums 的下标 r 对应到 pre 的右边界是 r+1。

如果你把 `[l,r]` 写成 `pre[r] - pre[l]`，实际上少算了 `nums[r]`。如果写成 `pre[r] - pre[l-1]`，又会让 l=0 时需要特殊判断。

固定使用左闭右开区间能减少错误。把原数组区间 `[l,r]` 变成 `[l,r+1)`，前缀和差值就是右边界前的累计减左边界前的累计。

判断句

闭区间 `[l,r]` 求和时，右边界一定去 pre 的 r+1。

560：前缀和为什么要配哈希表

hash prefix

560 和为 K 的子数组不是只查询一次区间和，而是要统计有多少个连续子数组和为 k。暴力枚举所有 l、r 会是 `O(n^2)`。

设当前前缀和为 s。如果某个旧前缀和为 `s-k`，那么旧前缀之后到当前位置这一段的和就是 k。于是问题变成：到目前为止，前缀和 `s-k` 出现过几次。

这就需要哈希计数表。扫描数组时先更新当前前缀 s，再把 `count[s-k]` 加到答案上，最后记录当前 s 出现一次。初始要有 `count[0]=1`，表示从数组开头开始的子数组。

判断句

统计和为 k 的子数组时，当前前缀需要找旧前缀 s-k。

二维前缀和：矩形也能相减

2d

二维前缀和把一维区间扩展到矩形。`pre[i][j]` 表示原矩阵左上角到 `(i-1,j-1)` 的矩形和。它同样多一行和一列 0，用来统一边界。

查询某个矩形时，用大矩形减掉上方区域、左方区域，再把左上角被减了两次的交叉区域加回来。公式看起来长，但本质还是“去掉不想要的部分”。

二维前缀和适合矩阵静态多次区域求和。如果矩阵频繁更新，就要考虑树状数组、线段树或差分等其他结构。

判断句

二维矩形求和也是前缀和，只是相减对象从线段变成矩形。

差分数组：把区间更新变成边界标记

diff

差分数组可以看作前缀和的反方向。`diff[i]` 表示 nums[i] 相对 nums[i-1] 变化了多少。只要知道 diff，从左到右累计，就能还原 nums。

如果要给 `[l,r]` 每个元素都加 val，直接改每个位置是 `O(n)`。差分数组只改两个边界：`diff[l] += val` 表示从 l 开始增加，`diff[r+1] -= val` 表示 r 后面停止增加。

最后对 diff 做一次前缀累计，得到每个位置叠加后的变化。多次区间更新时，每次都是 `O(1)`，最后还原 `O(n)`。

判断句

很多次区间加减、最后才要结果时，先考虑差分数组。

前缀和与差分的取舍

choice

前缀和适合“数组不变、查询很多”。你先花 `O(n)` 建表，之后每个区间和 `O(1)`。差分适合“更新很多、最后统一还原”。你每次更新 `O(1)`，最后一次 `O(n)` 得到结果。

如果既要频繁更新，又要频繁查询，简单前缀和和差分都不够，需要树状数组或线段树。但在速成阶段，先把前缀和与差分的静态场景掌握牢。

复杂度表达时要写清任务数量。比如 m 次区间更新、数组长度 n，差分总复杂度是 `O(m+n)`，不是每次更新都扫区间的 `O(mn)`。

判断句

问区间和用前缀和；问批量区间加减用差分；两者都频繁时再升级结构。

1. 一维前缀和查询

pre 多一个 0，闭区间 [l,r] 的和统一用 pre[r+1]-pre[l]。

正确示例：一维前缀和查询Python
def build_prefix(nums):
    pre = [0] * (len(nums) + 1)
    for i, x in enumerate(nums):
        pre[i + 1] = pre[i] + x
    return pre

def range_sum(pre, l, r):
    return pre[r + 1] - pre[l]

2. 560 和为 K 的子数组

扫描到当前前缀 s，查以前有多少个 s-k。初始 count[0]=1 处理从开头开始的子数组。

正确示例：560 和为 K 的子数组Python
def subarray_sum(nums, k):
    count = {0: 1}
    s = 0
    ans = 0
    for x in nums:
        s += x
        ans += count.get(s - k, 0)
        count[s] = count.get(s, 0) + 1
    return ans

3. 差分数组区间更新

每次区间加法只动两个边界，最后累计还原。注意 r+1 可能越界，要先判断。

正确示例：差分数组区间更新Python
def apply_updates(n, updates):
    diff = [0] * n
    for l, r, val in updates:
        diff[l] += val
        if r + 1 < n:
            diff[r + 1] -= val
    ans = [0] * n
    running = 0
    for i in range(n):
        running += diff[i]
        ans[i] = running
    return ans

4. 错误示例：560 更新顺序错

如果先记录当前前缀，再查 s-k，当 k=0 时可能把空区间也算进去。

错误示例：错误示例：560 更新顺序错Python
count[s] = count.get(s, 0) + 1
ans += count.get(s - k, 0)  # 错：当前前缀已经进表，可能误计

常见误区

debug

前缀和少建一个 0

错因：把 pre[i] 定义成到下标 i 的和。

后果：查询从 0 开始的区间要特殊处理，容易差一。

修法：统一用长度 n+1 的 pre，pre[0]=0。

闭区间公式写成 pre[r]-pre[l]

错因：混淆 nums 下标和 pre 下标。

后果：少算右端元素。

修法：闭区间 [l,r] 写 `pre[r+1]-pre[l]`。

560 忘记 count[0]=1

错因：没考虑从数组开头开始的子数组。

后果：漏掉前缀和本身等于 k 的情况。

修法：初始化 `{0:1}`。

560 先记录当前前缀再查

错因：更新时机错。

后果：k=0 时可能把空区间算进去。

修法：先查旧前缀，再记录当前前缀。

差分 r+1 越界

错因：区间右端在最后一个位置还照样减。

后果：数组越界。

修法：只有 `r+1 < n` 时才写 `diff[r+1]-=val`。

差分更新后忘记还原

错因：把 diff 当最终数组。

后果：输出的是变化边界，不是最终结果。

修法：对 diff 做前缀累计得到答案。
