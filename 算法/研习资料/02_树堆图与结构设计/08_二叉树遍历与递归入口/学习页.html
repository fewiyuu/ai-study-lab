力扣 257 / 129 / 199

把递归写在正确入口上

二叉树遍历题的直觉很简单：沿着树走一遍，在合适的位置记录信息。难点不是“递归会调用自己”，而是你要决定记录发生在进入节点时、左右子树之间，还是离开节点时。

这页先讲遍历思维。你会把每个节点当成一次小任务：处理当前节点、进入左子树、进入右子树。力扣 257、129、199、1022、1457 这些题看起来输出不同，但入口都来自这套顺序判断。

学完应该能做什么

  1. 能区分前序、中序、后序和层序各自适合记录什么信息。
  2. 能写出带路径参数的 DFS 模板，并知道什么时候恢复路径。
  3. 能解释 257 和 129 为什么适合从根到叶递归。
  4. 能判断 199 右视图为什么常用层序或“深度第一次出现”。
  5. 能说清递归深度、空节点、叶子节点的边界。

主线地图

mental model

遍历思维不是“把所有节点看一遍”这么粗。你要把访问顺序和题目要的答案对上。题目要从根到叶的路径，就在路径进入和退出时维护状态；题目要每层最右边，就要让深度或队列层次成为状态。

前序像“进入房间先记录”

前序在处理左右子树之前先处理当前节点。路径题常用它，因为当前节点必须先加入路径，孩子节点才能继承这条路径。

判断句：答案依赖从根到当前节点的历史时，通常要在进入节点时更新状态。

后序像“孩子做完再合并”

后序适合需要左右子树结果之后再决定当前节点的问题。比如高度、直径、是否平衡，当前节点的答案依赖孩子的返回值。

遍历题也会用后序恢复路径：离开节点时把刚加入的值弹出，避免影响别的分支。

层序按“距离根的步数”组织

层序遍历用队列，一层一层处理。右视图、每层最大值、平均值、宽度这类题，题目天然按层发问。

判断句：题目反复出现“每一层”“最浅”“最深”，优先想队列。

术语表

terms

先把词说清楚，后面的代码才不会变成背模板。

术语直觉准确说法常见误解
递归入口每次进入一个节点时先做的事。函数参数给出当前节点和必要状态，函数体处理当前节点并递归处理子树。以为递归入口只是一句 if root is None。
叶子节点路径走到头的节点。没有左孩子也没有右孩子的节点。把空节点当叶子，导致路径多出 None。
路径状态从根走到当前位置已经经过了什么。递归参数或外部列表记录当前分支上的节点值、字符串或累计数字。多个分支共享同一个列表却不恢复。
层序遍历先看离根近的，再看离根远的。用队列保存当前层或下一层节点，每轮处理一层。把普通 DFS 结果误当成层序。
递归深度当前在树的第几层。从根到当前节点的调用层数，可作为右视图等题的记录条件。深度从 0 还是 1 混乱，导致数组下标错。

知识链路

chain

二叉树题的最小链路是：先处理空节点，再处理当前节点，然后把任务交给左子树和右子树。这个链路能跑通，是因为每个子树仍然是一棵树，处理方式和整棵树相同。

257「二叉树的所有路径」把状态设计成路径列表。进入节点时加入节点值，到了叶子就把路径转成字符串，离开节点时弹出。129「求根到叶数字之和」把状态设计成累计数字，每走一步做 `cur = cur * 10 + node.val`。

199「二叉树的右视图」提醒你：遍历顺序可以服务答案。如果 DFS 先走右子树，那么某个深度第一次出现的节点就是右视图；如果用 BFS，每层最后一个节点就是右视图。

本页的判断句

  • 看到“根到叶”，先想 DFS 携带路径或累计值。
  • 看到“每层”，先想队列；如果用 DFS，就必须显式带深度。
  • 共享列表传下去时，进入节点 push，离开节点 pop。
  • 空节点是终止条件，叶子节点是收集答案的位置。
递归函数的职责：处理当前节点 + 把同一任务交给左右子树 + 在正确边界收集答案

核心模板

template

遍历模板要先写清函数参数。参数不是越多越好，只放“孩子节点继续工作必须知道”的状态。路径题需要 path；累计数字题需要 cur；右视图需要 depth。

模板里的顺序不能随便换。你先判断空节点，是为了避免访问 `node.val` 报错；你在叶子处收集答案，是因为只有走到头时路径才完整；你在递归后恢复 path，是因为下一条分支不能继承上一条分支的尾巴。

正确示例：根到叶路径 DFSPython
def binaryTreePaths(root):
    ans = []
    path = []

    def dfs(node):
        if node is None:
            return
        path.append(str(node.val))
        if node.left is None and node.right is None:
            ans.append("->".join(path))
        else:
            dfs(node.left)
            dfs(node.right)
        path.pop()

    dfs(root)
    return ans

`path.append` 写在进入节点时，`path.pop` 写在离开节点时。这样同一份列表可以被不同分支复用，又不会互相污染。

正确示例：累计数字不用恢复列表Python
def sumNumbers(root):
    def dfs(node, cur):
        if node is None:
            return 0
        cur = cur * 10 + node.val
        if node.left is None and node.right is None:
            return cur
        return dfs(node.left, cur) + dfs(node.right, cur)

    return dfs(root, 0)

累计数字是不可变整数，每次调用拿到自己的 cur，不需要手动回退。这里的返回值是“当前子树贡献的总和”。

进入节点维护路径

1、2、5 形成一条根到叶路径。到 5 时收集答案，回到 2 后要弹出 5，再去别的分支。

你要能看出来：路径题维护的是分支状态，层级题维护的是深度或队列层。

逐行拆解：右视图 DFSPython
def rightSideView(root):
    ans = []

    def dfs(node, depth):
        if node is None:
            return
        if depth == len(ans):
            ans.append(node.val)
        dfs(node.right, depth + 1)
        dfs(node.left, depth + 1)

    dfs(root, 0)
    return ans

先访问右子树，所以某个 depth 第一次被填入的节点就是最右边能看到的节点。若改成先左后右，这个判断会变成左视图。

例题：伪回文路径的状态Python
def pseudoPalindromicPaths(root):
    ans = 0

    def dfs(node, mask):
        nonlocal ans
        if node is None:
            return
        mask ^= 1 << node.val
        if node.left is None and node.right is None:
            if mask & (mask - 1) == 0:
                ans += 1
            return
        dfs(node.left, mask)
        dfs(node.right, mask)

    dfs(root, 0)
    return ans

mask 记录每个数字出现次数的奇偶性。到叶子时，最多只能有一个数字出现奇数次，路径才可能重排成回文。

边界与复杂度

limits

空树

空树通常返回空列表、0 或 None，取决于题目问什么。别把空节点当成叶子节点。

单节点

单节点既是根也是叶子。路径题应返回只含一个节点的路径，累计数字题返回该节点值。

递归栈

最坏情况下树退化成链，递归深度是 O(n)。Python 里极深的树可能触发递归深度限制。

做法时间空间什么时候用
DFS 遍历O(n)O(h)根到叶路径、累计值、前中后序
BFS 层序O(n)O(w)每层统计、最浅层、右视图
路径字符串输出O(n·h) 视输出而定O(h)需要保存完整路径文本

常见错误：错因、后果和修法

debug

把空节点当叶子

错因：终止条件和收集条件混在一起。

后果：答案里多出非法路径，或访问 None 报错。

修法：先判断 `node is None`，再判断左右孩子是否都为空。

路径列表不回退

错因：共享同一个 path，却没有在离开节点时 pop。

后果：右分支会带上左分支的节点。

修法：进入节点 append，递归结束后 pop。

只写递归不写职责

错因：没有明确 dfs 参数和返回值代表什么。

后果：代码能写几行但无法处理变形题。

修法：先用一句话写出函数职责。

右视图遍历顺序反了

错因：使用“深度第一次出现”却先走左子树。

后果：得到左视图。

修法：右视图 DFS 要先右后左，或改用 BFS 每层最后一个。

复杂度漏掉输出长度

错因：只数节点访问次数。

后果：路径字符串很多时，输出成本被低估。

修法：基础遍历 O(n)，涉及构造路径字符串时说明输出成本。

错误示例：共享路径没有恢复Python
def dfs(node):
    if node is None:
        return
    path.append(str(node.val))
    if node.left is None and node.right is None:
        ans.append("->".join(path))
    dfs(node.left)
    dfs(node.right)
    # 少了 path.pop()，下一条分支会继承当前路径

这段代码的结果可能混入别的分支节点。修法是在函数结束前恢复 path。

易混概念

compare

遍历思维 vs 分解思维

遍历思维：我亲自走到每个节点，沿路维护状态。

分解思维：我向左右子树要答案，再合并。

判断：状态来自祖先时多用遍历；答案依赖孩子返回值时多用分解。

空节点 vs 叶子节点

空节点：没有节点，通常直接返回。

叶子节点：有节点但没有孩子，常是收集答案的位置。

判断：访问 node.val 前先排除空节点。

路径参数 vs 返回值

路径参数：把祖先信息带给孩子。

返回值：把子树结果交回父节点。

判断：信息向下传用参数，信息向上传用返回值。
