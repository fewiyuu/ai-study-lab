二叉堆
与优先队列二叉堆是优先队列的常见实现。重点不是画树，而是理解 push、pop 为什么只改一条路径。

遇到“动态最小、动态最大、第 K 个、每次取当前最优”，先想优先队列。它不是排序的替代品，而是反复取最值的工具。
heap / priority queue

让最值自己浮到队首

二叉堆解决的是动态最值问题。数组整体排序适合一次性得到全局顺序；优先队列适合一边加入元素，一边反复拿当前最小或最大。

完全二叉树让堆可以放在数组里。父子下标有固定关系，`swim` 负责新元素向上调整，`sink` 负责堆顶删除后向下调整。理解这两步，`O(log n)` 的插入和删除就不再是死记。

学完应该能做什么

  1. 能说清完全二叉树、堆序性质和优先队列 API 的关系。
  2. 能解释 `heappush`、`heappop` 为什么是 O(log n)。
  3. 能区分“小根堆取最小”和“用负数模拟大根堆”。
  4. 能把 Top K、合并 K 个有序链表、数据流中位数迁移到堆模型。
  5. 能识别堆不适合随机删除和全量有序遍历。

主线地图

mental model

堆的主线是“局部有序”。它只保证堆顶是当前最值，不保证整个数组有序。这个性质刚好适合很多题：你不关心全部顺序，只关心下一次应该处理谁。

完全二叉树数组表示
堆序性质父节点优先级更高
push向上 swim
pop向下 sink
应用动态取最值

完全二叉树提供结构

堆不是随便一棵树，而是尽量从左到右填满的完全二叉树。因为形状固定，所以可以用数组保存，不需要显式 TreeNode。

下标关系让父子访问很快：父节点 `(i-1)//2`，左右孩子 `2*i+1` 和 `2*i+2`。

堆序性质提供最值

小根堆要求父节点不大于孩子，所以根节点一定是最小值。它不保证左孩子小于右孩子，也不保证数组从小到大。

判断句：如果你只需要当前最小，不需要完整排序，堆就合适。

调整只走一条路径

插入时，新元素先放到末尾，再向上交换到合适位置。删除堆顶时，把末尾元素放到堆顶，再向下交换。

树高是 `log n`，所以 push/pop 都是 `O(log n)`。

优先队列每次取出优先级最高的元素。支持插入元素和删除/查看最高优先级元素的抽象数据结构。以为它必须按插入顺序出队。

swim新元素太小，就往上浮。插入后沿父指针比较并交换，直到堆序恢复。以为要和所有节点比较。
sink堆顶元素太大，就往下沉。删除堆顶后沿较小孩子方向交换，直到堆序恢复。忽略两个孩子中更优先的那个。

知识链路

chain

优先队列是接口，二叉堆是实现。接口告诉你能 `push` 和 `pop`；实现告诉你为什么这些操作能快。把二者分开，很多题会更清楚：题目要的是反复拿最小/最大，你选优先队列；至于底层用二叉堆，是语言库帮你做的事。

Top K 问题常见两种堆法。要找最大的 K 个元素，可以维护一个大小为 K 的小根堆，堆顶是当前 K 个候选里最小的。新元素比堆顶大才替换，这样堆里始终保留较大的 K 个。

合并有序结构时，堆里放每条链或数组的当前头部。每次弹出最小元素，再把同一来源的下一个元素放进去。这个过程把“K 条路谁先走”交给堆判断。

本页的判断句

  • 题目需要反复取当前最小/最大，优先考虑堆。
  • 只要需要完整有序结果，排序可能更直接。
  • 维护 Top K 时，堆大小通常固定为 K。
  • Python `heapq` 是小根堆，大根堆常用负数或反向比较。
数组下标关系：parent=(i-1)//2，left=2*i+1，right=2*i+2；树高约为 log₂ n

核心模板

template

Python 的 `heapq` 直接操作列表。它默认是小根堆，`heap[0]` 是最小值。不要把 `heap` 当作已排序数组打印出来看，那个顺序只满足堆序，不满足全局有序。

Top K 模板的关键是控制堆大小。堆里不是存所有元素，而是存当前最有希望的 K 个候选。这样每个元素只做一次 `O(log k)` 调整。

正确示例：维护最大的 K 个元素Python
import heapq

def top_k_largest(nums, k):
    heap = []
    for x in nums:
        if len(heap) < k:
            heapq.heappush(heap, x)
        elif x > heap[0]:
            heapq.heapreplace(heap, x)
    return heap

堆顶是这 K 个候选里最小的。只有新元素比堆顶大，它才有资格进入最大的 K 个。

正确示例：合并 K 个有序链表的堆思路Python
import heapq

def merge_k_lists(lists):
    heap = []
    for i, node in enumerate(lists):
        if node:
            heapq.heappush(heap, (node.val, i, node))

    dummy = ListNode(0)
    cur = dummy
    while heap:
        _, i, node = heapq.heappop(heap)
        cur.next = node
        cur = cur.next
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next

元组里的 i 用来打破相同 val 时的比较冲突。每次弹出当前最小节点，再补入同一链表的下一个节点。

状态演示

切换场景，观察关键状态怎么变。看懂状态，比背下一行代码更重要。

新元素向上浮

插入 2 后，它先在数组末尾，再和父节点比较。如果它更小，就一路向上交换直到堆序恢复。

你要能判断：堆只保证“当前最值”可靠，不保证数组里的其他元素全局有序。

例题：第 K 大元素Python
import heapq

def findKthLargest(nums, k):
    heap = []
    for x in nums:
        heapq.heappush(heap, x)
        if len(heap) > k:
            heapq.heappop(heap)
    return heap[0]

堆里保留最大的 k 个数，堆顶就是这 k 个数中最小的，也就是全局第 k 大。

例题：数据流中位数的双堆框架Python
import heapq

small = []  # 大根堆，用负数模拟
large = []  # 小根堆

def add_num(x):
    heapq.heappush(small, -x)
    heapq.heappush(large, -heapq.heappop(small))
    if len(large) > len(small):
        heapq.heappush(small, -heapq.heappop(large))

两个堆分别维护较小一半和较大一半。平衡大小后，中位数只看堆顶。

边界与复杂度

limits

堆不等于排序

`heap[0]` 是最小值，但 `heap[1:]` 没有全局顺序。需要完整升序输出时，要连续 pop 或直接排序。

随机删除不擅长

标准堆擅长删除堆顶，不擅长删除任意元素。需要频繁删任意元素时，可能要配哈希表做延迟删除。

K 很小时优势明显

Top K 用大小为 K 的堆，复杂度是 `O(n log k)`。当 K 远小于 n，比全量排序更省。

做法时间空间什么时候用
heappush / heappopO(log n)O(1) 额外动态插入和取最值
Top K 小根堆O(n log k)O(k)只要最大的 K 个或第 K 大
一次性排序O(n log n)视实现而定需要完整有序列表

常见错误：错因、后果和修法

debug

把堆数组当排序结果

错因：看到堆顶最小，以为整个列表有序。

后果：直接返回 heap 可能顺序不符合题意。

修法：需要有序输出就连续 pop 或排序结果。

Top K 堆方向反了

错因：找最大 K 个却维护大根堆并随便弹。

后果：可能把真正的大元素弹掉。

修法：最大 K 个用小根堆守门，最小 K 个用大根堆守门。

相同值元组比较报错

错因：堆元素写成 `(val, node)`，val 相同会比较 node。

后果：Python 抛出对象不可比较错误。

修法：加入唯一序号 `(val, i, node)`。

忘记限制堆大小

错因：Top K 把所有元素都 push 进去。

后果：复杂度退回 `O(n log n)`，空间也变大。

修法：超过 K 就 pop，或用 heapreplace。

大根堆负号漏还原

错因：用负数模拟大根堆，但取出后忘记取反。

后果：答案符号错误。

修法：push 时取负，pop 后再取负还原。

错误示例：Top K 没有限制堆大小Python
def top_k(nums, k):
    heap = []
    for x in nums:
        heapq.heappush(heap, x)
    return heap[:k]  # heap 不是排序数组，这样也不是最大的 k 个

堆里放了全部元素，而且切片不是按大小取。修法是维护大小为 K 的候选堆。

易混概念

compare

优先队列 vs 普通队列

优先队列：按优先级出队。

普通队列：按进入顺序先进先出。

判断：题目说“当前最小/最大/代价最低”，用优先队列；说“按层扩散”，多用普通队列。

堆顶有序 vs 全局有序

堆顶有序：只保证根节点是最值。

全局有序：每个位置都满足排序关系。

判断：要完整顺序就别直接返回堆数组。

小根堆 vs 大根堆

小根堆：堆顶是最小值。

大根堆：堆顶是最大值，Python 常用负数模拟。

判断：Top K 先问“谁是门槛”，再选堆方向。
