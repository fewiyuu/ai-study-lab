# 概念地图：接口、结构和不变量 {#overview}
<!-- tag: map -->

先把三个层次拆开：优先队列是接口，二叉堆是常见实现，堆序性质是不变量。接口回答“能做什么”，实现回答“怎么做到”，不变量回答“什么时候算没坏”。

层次
直觉
准确说法
读题时怎么用


优先队列
每次取优先级最高的元素。
支持插入、查看或删除最高优先级元素的抽象数据结构。
题目说“当前最小 / 最大 / 代价最低”，先考虑它。


二叉堆
存在数组里的局部有序完全二叉树。
同时满足完全二叉树形状和堆序性质。
解释 `O(log n)` 调整路径时使用。


小根堆
最小值在根。
每个父节点的 key 不大于孩子节点的 key。
Python `heapq` 默认就是小根堆。


候选集
堆里当前还可能成为答案的元素。
算法主动维护的一小部分状态，不一定包含所有输入。
Top K、合并链表、数据流题都先问候选是什么。

$$
 parent(i)=\left\lfloor\frac{i-1}{2}\right\rfloor,\quad left(i)=2i+1,\quad right(i)=2i+2,\quad height=O(\log n)
 $$
 完全二叉树让堆可以压进数组。`push` 和 `pop` 最多沿树高走一条路，所以单次调整是 `O(log n)`。

# 堆的局部调整：只修一条路径 {#mechanism}
<!-- tag: mechanism -->

# Top K：堆顶是候选门槛 {#topk}
<!-- tag: top k -->

找最大的 K 个元素时，常用大小为 K 的小根堆。堆里保存“当前见过的最大 K 个候选”，堆顶是这 K 个候选里最小的，也就是门槛。新元素只有超过门槛，才有资格进入候选集。

手算锚点
数组 `[3,2,1,5,6,4]` 找第 2 大，维护容量为 2 的小根堆。看到 `3,2` 后堆顶是 `2`；`1` 进不来；`5` 进来后弹出 `2`；`6` 进来后弹出 `3`；`4` 不超过门槛 `5`。最后堆顶 `5` 就是第 2 大。


观察点
堆里不一定按降序保存答案。它只保证门槛在堆顶。你要返回完整 Top K 集合时，可以再排序；只问第 K 大时，直接返回堆顶。

215 第 K 大元素
复制

```python
import heapq

def findKthLargest(nums, k):
 heap = []
 for x in nums:
 if len(heap) < k:
 heapq.heappush(heap, x)
 elif x > heap[0]:
 heapq.heapreplace(heap, x)
 return heap[0]
```

$$
 n\text{ 个元素，每次调整大小为 }k\text{ 的堆： }T(n,k)=O(n\log k),\quad S(k)=O(k)
 $$
 这里的 `log k` 来自堆的最大大小。若把所有元素都放进堆，复杂度会变成 `O(n log n)`，也失去了 Top K 的空间优势。

# 合并有序结构：每条路只放当前头部 {#merge}
<!-- tag: merge -->

合并 K 个有序链表时，不需要把所有节点一次性展开。每条链表已经内部有序，所以堆里只放每条链当前头部。弹出最小节点后，再补入同一条链表的下一个节点。

23 合并 K 个有序链表
复制

```python
import heapq
from itertools import count

def mergeKLists(lists):
 heap = []
 seq = count()
 for node in lists:
 if node:
 heapq.heappush(heap, (node.val, next(seq), node))

 dummy = ListNode(0)
 cur = dummy
 while heap:
 _, _, node = heapq.heappop(heap)
 cur.next = node
 cur = cur.next
 if node.next:
 heapq.heappush(heap, (node.next.val, next(seq), node.next))
 return dummy.next
```

# 双堆中位数：两个堆维护左右两半 {#median}
<!-- tag: two heaps -->

数据流中位数通常要两个堆配合。一个大根堆保存较小一半，一个小根堆保存较大一半。两个堆的大小差不超过 1，较小一半的最大值和较大一半的最小值就能给出中位数。

295 数据流中位数
复制

```python
import heapq

class MedianFinder:
 def __init__(self):
 self.small = [] # 大根堆，用负数模拟，保存较小一半
 self.large = [] # 小根堆，保存较大一半

 def addNum(self, num):
 heapq.heappush(self.small, -num)
 heapq.heappush(self.large, -heapq.heappop(self.small))
 if len(self.large) > len(self.small):
 heapq.heappush(self.small, -heapq.heappop(self.large))

 def findMedian(self):
 if len(self.small) > len(self.large):
 return -self.small[0]
 return (-self.small[0] + self.large[0]) / 2
```

对象
保存什么
堆顶含义
坏了的症状


`small`
较小一半，用负数模拟大根堆。
`-small[0]` 是较小一半最大值。
忘记取负，结果符号或顺序错。


`large`
较大一半，小根堆。
`large[0]` 是较大一半最小值。
元素没有平衡，两个堆大小差太大。


平衡动作
先从 `small` 转到 `large`，再按大小转回来。
保证 `small` 至少和 `large` 一样多。
偶数个元素时中位数取错两个堆顶。

# 实现一致性：堆里放的对象必须和问题目标一致 {#consistency}
<!-- tag: implementation -->

堆模板最容易写错的地方通常堆元素的含义和题目目标没有对齐。你可以把每道堆题都翻译成四个问题：比较字段是什么，堆顶代表什么，弹出后会改变哪一部分状态，下一次应该补入哪个候选。

这四个问题能帮你发现很多隐蔽错误。例如 Top K 里堆顶是门槛，合并链表里堆顶是当前全局最小节点，双堆中位数里两个堆顶共同决定答案。它们都用堆，但堆顶的语义不一样。语义没说清，代码看着像模板，运行结果会很难调。

问题
堆元素设计
堆顶语义
弹出 / 替换后状态
调试时先看


最大 K 个
直接放数值 `x`，堆大小固定为 K。
当前最大 K 个候选里的最小值。
新元素超过堆顶才替换门槛。
堆大小是否超过 K；是否把小根堆误当大根堆。


Top K 高频
`(freq, value)` 或 `(freq, seq, value)`。
当前候选里频率最低的元素。
新频率超过门槛才进入。
比较字段应为频率，别误用元素值。


合并 K 个链表
`(node.val, seq, node)`。
K 条链当前头部里的最小节点。
弹出节点接入结果，再补同链下一个。
是否记录来源；相同值是否有 `seq`。


有序矩阵第 K 小
`(matrix[r][c], r, c)`。
每行当前候选中的最小值。
弹出 `(r,c)` 后补 `(r,c+1)`。
是否补同一行；是否越界；是否重复入堆。


数据流中位数
`small` 放负数，`large` 放正数。
`-small[0]` 和 `large[0]` 夹住中位数。
插入后在两个堆之间搬一次或两次。
两个堆大小差；负号是否还原。

## box
### 把代码读成状态机
读堆代码时，先不要急着模拟每一次数组交换。更稳的做法是读状态：堆里最多有多少元素，堆顶对应哪个判断，弹出后有没有补候选。比如合并链表的 `while heap` 循环，每轮都做同一件事：拿出当前最小节点，接到答案尾部，再把这条链的新头部放回堆。
如果你发现代码里没有来源信息，却需要“弹出后补同来源下一个”，这就是缺状态。此时继续调 `heappush` 的顺序没有用，应该先把堆元素从 `(val, node)` 改成 `(val, seq, node)` 或 `(val, row, col)`。

## box warn
### 别把局部最优误读成全局结果
堆每次给你的是“当前候选集里的最优”，完整答案还没有自动排好。Top K 只维护 K 个候选时，堆外还有很多已经被排除的元素；合并链表只放每条链当前头部时，链表后面的节点还没进入候选集。这个边界说不清，常见后果就是提前返回堆数组，或者弹出后补错来源。
调试时可以打印三列：堆顶、堆大小、下一次补入对象。比打印整个堆数组更有用，因为内部数组顺序本来就不是你要看的答案。

调试堆题时打印语义状态
复制

```python
# Top K 调试时，看门槛和候选数量，不看 heap 是否整体有序。
print("threshold =", heap[0], "size =", len(heap))

# 合并链表调试时，看弹出节点和将要补入的同来源节点。
value, seq, node = heapq.heappop(heap)
print("pop =", value, "next =", node.next.val if node.next else None)
```

这段调试代码只用于定位问题。它把“堆顶语义”暴露出来，让你确认候选集没有跑偏。

def kth_largest_debug(nums, k):
 heap = []
 for x in nums:
 if len(heap) < k:
 heapq.heappush(heap, x)

 heapq.heapreplace(heap, x)

# 边界与复杂度：什么时候别强上堆 {#boundary}
<!-- tag: limits -->

要完整排序
如果最终必须输出完整升序或降序列表，直接排序通常更清楚。堆可以连续 `pop` 得到顺序，但代码意图不一定更好。


要随机删除
标准二叉堆擅长删除堆顶，不擅长删除任意元素。频繁删除任意值时，通常要配哈希表做延迟删除，或换别的数据结构。


K 接近 n
Top K 的优势来自 `k` 小。如果 `k` 接近 `n`，`O(n log k)` 和排序差距不大，代码复杂度反而可能更高。

# 常见错误：错因、后果和修法 {#mistakes}
<!-- tag: debug -->

把堆数组当排序结果
错因：只记住了“堆顶最小”，误以为整个数组都按升序排列。
后果：直接返回 `heap` 或 `heap[:k]` 会得到顺序不可靠的结果。
修法：只问第 K 个就读堆顶；要完整顺序就 `sorted(heap)` 或连续弹出。


Top K 堆方向反了
错因：找最大 K 个时用了大根堆守门。
后果：你容易弹掉真正应该保留的大元素。
修法：最大 K 个用小根堆保存候选，堆顶是门槛；最小 K 个反过来。


相同值比较报错
错因：堆元素写成 `(val, node)`，当 `val` 相同时 Python 会继续比较 `node`。
后果：节点对象不可比较，程序直接抛异常。
修法：加唯一序号，写成 `(val, seq, node)`。


忘记限制堆大小
错因：Top K 中把所有元素都 `push` 进堆。
后果：空间变成 O(n)，时间也接近全量堆处理。
修法：超过 K 立刻弹出门槛外元素，或用 `heapreplace`。


大根堆负号漏还原
错因：用负数模拟大根堆，取出后忘了再取负。
后果：答案符号错，中位数或最大值读反。
修法：写清楚变量名，例如 `max_value = -small[0]`。


弹出后补错来源
错因：合并有序结构时，只补“某个下一个”，没有记录它来自哪条链或哪一行。
后果：候选集漏项或重复，最后顺序错。
修法：堆元素带上来源下标，弹出后只补同来源的下一个。

# 题目迁移：先说候选集，再选堆方向 {#transfer}
<!-- tag: transfer -->

堆题读题时可以按三问走：候选集是什么？每次弹出谁？弹出后补什么？这三问比背题号更可靠。

题型
候选集
堆方向
弹出后动作
边界提醒


215 第 K 大
当前最大的 K 个候选。
小根堆。
超过 K 就弹出最小候选。
堆顶是第 K 大，最大值只是候选之一。


347 Top K 高频
当前频率最高的 K 个元素。
按频率的小根堆。
新频率超过门槛才替换。
比较字段是频率，元素值只用于返回答案。


23 合并 K 个链表
每条链的当前头部。
小根堆。
补同一条链的下一个节点。
相同值要加 `seq`。


378 有序矩阵第 K 小
每行当前最小未取元素。
小根堆。
补同一行下一列。
不要一次性生成所有元素。


295 数据流中位数
左右两半数据。
大根堆 + 小根堆。
插入后平衡大小。
查询中位数只看堆顶。

### 1 选择

**Q:** Q1 Python 小根堆中，`heap[0]` 保证是什么？

- [ ] . 当前堆内最小元素。
- [ ] . 整个数组已经升序。
- [ ] . 最后插入的元素。

Answer: . 当前堆内最小元素。
Explain: 小根堆只保证堆顶是当前堆内最小值。

### 2 判断

**Q:** Q2 小根堆的内部数组可以直接当升序数组返回。

- [ ] 对
- [ ] 错

Answer: false
Explain: 堆数组只满足父子堆序，不能直接当排序结果。

### 3 选择

**Q:** Q3 `heappush` / `heappop` 为什么是 `O(log n)`？

- [ ] . 因为每次都会重排整个数组。
- [ ] . 因为最多沿完全二叉树高度调整一条路径。
- [ ] . 因为堆里只能放 10 个元素。

Answer: . 因为最多沿完全二叉树高度调整一条路径。
Explain: 插入或删除堆顶后，只需要沿树高修一条路径。

### 4 选择

**Q:** Q4 Python `heapq` 想模拟大根堆，常见做法是什么？

- [ ] . 写入 `-x`，取出后再取负。
- [ ] . 把列表倒序打印。
- [ ] . 先随机打乱数组。

Answer: . 写入 `-x`，取出后再取负。
Explain: 写入负数后，小根堆弹出的最小负数对应原值最大。

### 5 选择

**Q:** Q5 n 个数找最大 K 个，维护容量 K 的小根堆，常见时间复杂度写作什么？

- [ ] . O(n log k)
- [ ] . O(k log n)
- [ ] . O(n²)

Answer: . O(n log k)
Explain: 每个元素最多触发一次大小为 K 的堆调整。

### 6 走读

**Q:** Q6 手算 `[3,2,1,5,6,4]` 找第 2 大时，容量 2 的小根堆如何变化？

Explain: 放入 3、2 后，候选是 2 和 3，堆顶 2。看到 1，不超过门槛 2，丢掉。看到 5，替换 2，候选变成 3 和 5。看到 6，替换 3，候选变成 5 和 6。看到 4，不超过门槛 5，丢掉。最后堆顶 5 是第 2 大。

### 7 诊断

**Q:** Q7 合并 K 个有序链表时，`(val, seq, node)` 三个字段分别解决什么问题？

Explain: `val` 是优先级，决定哪个节点先出堆；`seq` 是唯一序号，用来在 `val` 相同时继续比较，避免 Python 比较 `node` 对象；`node` 是真正要接到结果链表上的节点。弹出后只补这个节点所在链表的下一个节点。

### 8 选择

**Q:** Q8 合并有序链表或有序矩阵时，弹出一个元素后最应该补入什么？

- [ ] . 同一来源的下一个元素。
- [ ] . 任意一个还没看过的元素。
- [ ] . 已经弹出的元素本身。

### 9 诊断

**Q:** Q9 找最大 K 个却维护了错误方向的堆，会怎样？怎么修？

Explain: 错因是没有把堆顶定义成“门槛”。找最大 K 个时应该用小根堆，堆顶是候选里最小的那个；如果方向反了，可能把应该保留的大元素弹走。修法是重新定义候选集：最大 K 个用容量 K 的小根堆，最小 K 个用容量 K 的大根堆。

### 10 诊断

**Q:** Q10 Python 堆元素写成 `(val, node)`，在 `val` 相同时为什么可能报错？

Explain: Python 会先比较元组第一项，第一项相同时继续比较第二项。`node` 对象通常没有定义大小关系，所以会报不可比较。修法是在中间加唯一序号，写成 `(val, seq, node)`。

### 11 诊断

**Q:** Q11 Top K 中如果把所有元素都放进堆，复杂度和空间会怎样变化？

Explain: 堆大小从 K 变成 n 后，每次调整可能是 `log n`，空间也变成 O(n)。这相当于放弃了 Top K 只维护少量候选的优势。修法是堆超过 K 就弹出门槛外元素，或在新元素超过堆顶时用 `heapreplace`。

### 12 诊断

**Q:** Q12 数据流中位数里，`small` 和 `large` 分别维护什么不变量？

Explain: `small` 用负数模拟大根堆，保存较小一半，`-small[0]` 是较小一半最大值；`large` 是小根堆，保存较大一半，`large[0]` 是较大一半最小值。两个堆大小差不超过 1，通常让 `small` 至少和 `large` 一样多。

### 13 迁移

**Q:** Q13 有序矩阵第 K 小中，为什么弹出一个元素后要补同一行下一个？

Explain: 每一行内部已经有序，所以当前列元素弹出后，同一行的下一列才成为这条来源的新候选。堆里保持每行一个候选，就能在 K 次弹出里逐步得到第 K 小，而不必一次性展开整个矩阵。

### 14 迁移

**Q:** Q14 查找和最小的 K 对数字中，怎样避免一次性生成所有组合？

Explain: 可以先把每个起点的最小配对放入堆，例如固定一个数组下标，另一个数组从 0 开始。每次弹出当前和最小的数对，再推进同一路径上的下一个配对。关键是只扩展可能成为下一个最小和的邻居，并用下标避免重复。

### 15 表达

**Q:** Q15 用 2-3 句话解释堆题中 `log k` 和 `log n` 的区别。

Explain: `log` 后面的量取决于堆里最多有多少元素。Top K 只维护 K 个候选，所以调整成本是 `log k`；如果把全部 n 个元素都放进堆，调整成本就是 `log n`。复杂度解释先写堆大小，再写总共处理多少次。

### 16 总结

**Q:** Q16 用 6-9 句话讲清堆、优先队列、Top K、复杂度和常见误区。

Explain: 一份稳定答案至少说清：优先队列按优先级出队；二叉堆是常见实现；小根堆堆顶是当前最小值；`push` 和 `pop` 只沿一条路径调整，所以是 `O(log n)`；Top K 用容量 K 的堆时复杂度是 `O(n log k)`；合并有序结构弹出后要补同来源下一个；堆数组只满足堆序；Python 相同值比较要加 `seq`。
