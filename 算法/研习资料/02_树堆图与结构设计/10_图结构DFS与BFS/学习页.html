学习目标

起步

图题的难点不在“会不会 DFS/BFS”，而在“你先把什么当节点、什么当边”。只要建模错了，后面模板写得再像也会跑偏。

这页先把三件事分开：图怎么表示，visited 什么时候写，什么时候该用 DFS，什么时候该用 BFS。

先盯住这三步

  1. 把输入改写成邻接表。
  2. 给 visited 选对时机。
  3. 根据题目目标选 DFS 或 BFS。
你要会的动作具体表现错了会怎样
建模把课程依赖、钥匙、网格、变量关系写成节点和边。visited 会存错对象，整个搜索方向都偏。
去重图有环时，第一次见到状态就做标记。同一个点会被反复走进，甚至递归爆栈。
选法连通性看 DFS，等权最短步数看 BFS。把 BFS 用到带权问题上，结论会错。

主线地图

图建模

图题最常见的顺序是：先读输入，再把关系写成邻接表，然后才轮到 DFS 或 BFS。状态可以是节点编号、网格坐标、字符串、变量名，关键在于它能不能作为“下一步能去哪里”的起点。

对象读什么写什么错了的症状
邻接表原始关系对每个节点的邻居列表边方向写反，或者漏掉反向边
visited当前节点是否已经见过进入节点时或入队时立刻标记重复走、重复入队、递归回环
DFS 递归栈当前路径上的节点按深度往下追邻居有环时不断回到旧节点
BFS 队列当前层边界按层扩散新状态出队才标记会把队列撑大

状态账本

visited

图遍历最容易写坏的地方在状态顺序：什么时候写、写给谁看。把这个账本想清楚，模板才会稳定。

$$ O(V + E) $$

V 是节点数，E 是边数。邻接表里，每个节点和每条边都只会被常数次检查；网格题里可以把 V 理解成格子数，把复杂度写成 O(mn)。

from collections import defaultdict

def build_graph(pairs, directed=True):
 graph = defaultdict(list)

 for a, b in pairs:
 graph[a].append(b)
 nodes.add(a)
 nodes.add(b)
 if not directed:
 graph[b].append(a)
 for node in nodes:
 graph.setdefault(node, [])
 return graph

例子与走读

案例

图题最容易学会的方式，把一个小例子完整走完。这里用两个锚点：841 看“房间与钥匙”，1306 看“数组下标如何变成图节点”。它们的共同点是都需要先定义节点和邻居，再决定 visited 什么时候写。

841 钥匙和房间

把每个房间看成节点，把钥匙列表看成邻居。起点是房间 0，visited 里存的是已经访问过的房间编号。题目的关键你能不能在写代码前先把“房间 0 能到 1 和 3，1 和 3 都能到 2”这件事说清楚。

走一遍就会发现，DFS 只要进入节点时先标记，顺着钥匙往下就能扫完；BFS 也能做，但它的优势在这道题里不明显，因为这题没有“最少几步”的要求。

步骤当前点visited下一步看什么
10{0}房间 0 的钥匙指向 1 和 3
21{0,1}房间 1 继续给出 2
33{0,1,3}房间 3 也给出 2，但 2 还没被处理
42{0,1,2,3}全部房间都进过 visited

841 的 DFS 写法复制

def canVisitAllRooms(rooms):
 visited = set()

 def dfs(room):
 if room in visited:
 return
 visited.add(room)
 for key in rooms[room]:
 dfs(key)

 dfs(0)
 return len(visited) == len(rooms)

这里最重要的一行是 visited.add(room)。如果它挪到后面，房间 0 可能沿着 1 回到 0，或者沿着 3 回到 2 再碰到别的分支，图一旦有回边，递归就不再安全。你学到的知道这行为什么必须在递归之前。

1306 跳跃游戏 III

这题看起来像数组，实际上是图。下标是节点，两个邻居是 i + arr[i] 和 i - arr[i]。因为邻居不是输入里直接给你的，所以你必须自己生成它们，再做边界判断。

这题里最常见的错，把越界位置当成合法节点，或者把相同下标重复塞进队列。前者会直接错，后者会让搜索多做很多无意义工作。BFS 在这里更直观，因为题目又有“第一次到 0 就结束”的味道，但 DFS 也能做，只要 visited 放对位置。

步骤当前下标候选邻居保留哪些
1startstart ± arr[start]只保留在数组范围内的下标
2合法邻居继续算它们的 ± arr[i]已经见过的下标直接跳过
3arr[i] = 0不再往外扩直接返回成功
4越界非法节点丢弃

1306 的 BFS 写法复制

from collections import deque

def canReach(arr, start):
 q = deque([start])
 visited = {start}
 while q:
 i = q.popleft()
 if arr[i] == 0:
 return True
 for j in (i + arr[i], i - arr[i]):
 if 0 <= j < len(arr) and j not in visited:
 visited.add(j)
 q.append(j)
 return False

这段代码里，0 <= j < len(arr) 和 j not in visited 是两个不同的门。前者挡住非法节点，后者挡住重复节点。很多人会只记住“BFS 用队列”，但真正决定结果的是这两道门有没有同时立住。

417 和 399 的边界感

417 太平洋大西洋水流问题仍然是图题，但它更适合反向思考：水是从高处流向低处的，直接正向搜会在很多点上分叉得很散；从海岸线反向搜，邻居规则反过来之后，判断会更直接。你只需要记住这一点：有些题的图是“题目写给你的”，有些题的图是你自己换个方向才看清的。

399 除法求值也能改成图，但它已经不只是“走到没走过的节点”那么简单，因为边上还带着比例。这个页面先不展开带权乘积的传递规则，只把它当成一个提醒：如果题目里答案依赖路径上的累计量，普通 BFS 的层数就不再代表结果本身。

所以你现在看到题面时，要先问三件事：节点是不是显式给出，邻居是不是要你自己生成，答案是不是只依赖可达性还是还依赖路径上的累计量。前两件事这页能解决，第三件事需要把问题送到后面的单元。

题目特征你先做什么最常见的错下一步动作
邻居在输入里直接给出先建邻接表没把关系写成图再决定 DFS 还是 BFS
邻居需要你自己算先写生成规则越界或重复节点把 visited 和边界一起写进循环
只问能不能到DFS 和 BFS 都能试把“最短”错当成默认目标用最容易写对的一种
问最少几步先看是否等权把带权问题硬塞给 BFS去后续最短路框架

721 的 DFS 收集组件复制

from collections import defaultdict

def merge_accounts(accounts):
 graph = defaultdict(set)
 owner = {}

 for row in accounts:
 name, *emails = row
 first = emails[0]
 for email in emails:
 owner[email] = name
 graph.setdefault(email, set())
 for email in emails[1:]:
 graph[first].add(email)
 graph[email].add(first)

 seen = set()
 merged = []

 def dfs(email, bucket):
 seen.add(email)
 bucket.append(email)
 for nei in graph[email]:
 if nei not in seen:
 dfs(nei, bucket)

 for email in graph:
 if email in seen:
 continue
 bucket = []
 dfs(email, bucket)
 merged.append([owner[email]] + sorted(bucket))

 return merged

这段代码里，first 是这一行的锚点。它把同一行的邮箱绑成一团，后面的 DFS 只负责把这一团扫出来。真正常见的错，是只连相邻邮箱，或者把名字当节点键。前者会把本来该合并的邮箱拆散，后者会把不同人的同名账户混到一起。

2101 引爆最多的炸弹

这题的图是有方向的。炸弹 A 能引爆炸弹 B，不代表 B 也能回头引爆 A，所以边不能默认双向。节点还是炸弹本身，边则由“距离是否落在半径内”决定。

步骤炸弹能指向谁为什么
1(0,0,r=4)0→1, 0→2起点半径足够大，能先把后两颗炸弹覆盖住
2(3,0,r=1)1→2半径更小，只能继续推到第三个炸弹
3(4,0,r=1)2→1方向已经不是同一回事，不能偷写成无向边
4统计从每个起点做 DFS / BFS看哪个起点能覆盖最多炸弹

2101 的有向建图与 DFS复制

def max_detonation(bombs):
 n = len(bombs)
 graph = [[] for _ in bombs]
 for i, (x1, y1, r1) in enumerate(bombs):
 r1_sq = r1 * r1
 for j, (x2, y2, _) in enumerate(bombs):
 if i == j:
 continue
 dx = x1 - x2
 dy = y1 - y2
 if dx * dx + dy * dy <= r1_sq:
 graph[i].append(j)

 def dfs(start):
 seen = {start}
 stack = [start]
 while stack:
 x = stack.pop()
 for nei in graph[x]:
 if nei not in seen:
 seen.add(nei)
 stack.append(nei)
 return len(seen)

 return max(dfs(i) for i in range(n))

这里最容易错的是把 graph[i].append(j) 理解成“只要两点挨得近就互相连边”。不是这样。方向来自爆炸半径本身，i 能不能到 j，只看 i 的半径够不够，不看 j 会不会反过来够到 i。一旦写成无向边，统计结果通常会被抬高。

这两题怎么回到模板

721 提醒你：连通块题，先把同团的点用 DFS 或 BFS 收起来，再做排序和输出。2101 提醒你：图边有方向时，先把边建对，再统计从每个起点能走到多少点。它们共同逼你记住同一件事：模板只是最后一步，前面的建图如果错了，模板写得再熟也会错。

如果你做练习时卡住，先别急着改代码，先画出三样东西：节点、边、visited。能把这三样说清楚，代码往往只剩下十几行；说不清楚的时候，代码行数反而会更多，但正确率更低。

200 岛屿数量：网格也是图

如果题面给的是二维网格，别把它当成和树题不一样的东西。每个格子都是节点，四个方向的上下左右就是边。visited 里存的是坐标，不是格子的值；只要一个格子已经进过队列或递归，就不要再回头算第二次。

步骤当前格子邻居visited会发生什么
1(0,0)上下左右四个候选位置先标记 (0,0)开始扩展这一块陆地
2(0,1)继续看四邻域已访问坐标跳过整块岛屿只会被数一次
3水格不扩展可不记把岛和海分开

200 的 DFS 计数复制

def numIslands(grid):
 rows, cols = len(grid), len(grid[0])
 seen = set()

 def dfs(r, c):
 if not (0 <= r < rows and 0 <= c < cols):
 return
 if grid[r][c] == "0" or (r, c) in seen:
 return
 seen.add((r, c))
 for dr, dc in ((1, 0), (-1, 0), (0, 1), (0, -1)):
 dfs(r + dr, c + dc)

 islands = 0
 for r in range(rows):
 for c in range(cols):
 if grid[r][c] == "1" and (r, c) not in seen:
 islands += 1
 dfs(r, c)
 return islands

这段代码里，边界判断先于递归，seen.add((r, c)) 也必须先于扩展邻居。少了前者会越界，少了后者会把同一片岛屿反复数成多片。网格题和钥匙房间题本质一样，只是节点从“编号”换成了“坐标”。

你做练习时，看到 m x n 的输入就先问一句：这是图还是矩阵。只要题目允许上下左右走、四邻域扩散，九成以上都会落回“先定节点，再定边，再定 visited”的图模型。

模板

代码

图题写模板时，真正要盯的是两句：visited.add(x) 放在什么地方，和邻居从哪里来。其余代码只是把这两句包起来。

代码
def dfs_graph(graph, start):
 visited = set()
 order = []

 def dfs(x):
 if x in visited:
 return
 visited.add(x)
 order.append(x)
 for nei in graph[x]:
 dfs(nei)

 dfs(start)
 return order, visited

这一版的关键是先标记、再递归。顺序一旦换掉，遇到环时就可能绕回旧节点。

图 BFS 模板复制

这一版的关键是入队时标记。这样同一层的多个父节点不会把同一个邻居重复塞进队列。

怎么选 DFS / BFS

判断

先别急着背“DFS 看连通、BFS 看最短”这句口号。更稳的判断是：题目到底要你回答“能不能到、是否连通、这一团有多大”，还是要你回答“最少几步、最短层数、最近一次到达”。

先选 BFS 的场景

  • 每条边代价一样，题目问最少步数。
  • 起点到目标的第一次到达就有意义。
  • 多源同时扩散更自然。
题目在问什么优先思路理由
能不能到DFS 或 BFS 都行只要把图建对，任一遍历都能把可达状态扫出来。
这一团有多大DFS 更顺手递归或栈能很直接地把连通块扩完。
最少几步BFS队列按层推进，第一次到达就是当前最短层。
边权不同先停在这页外面这时该看 15 号单元之后的最短路框架。

第一次到达不再保证最短距离。边权不同的时候，层数不等于真实代价。

13debug-checklist

Q: 13. 按“错因 / 后果 / 修法”写出忘记 visited 的诊断。

参考答案 / 评分要点

。错因通常是把图误当成树，默认不会回到旧节点；后果是重复搜索、队列膨胀或无限递归；修法是在 DFS 进入节点时、BFS 入队时立刻标记 visited。三个部分都写到就算完整。
