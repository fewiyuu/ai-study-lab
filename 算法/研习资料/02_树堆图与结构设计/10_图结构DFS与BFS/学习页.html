图结构 DFS 与 BFS

读完本页应该能做什么

  • 能把房间、下标、网格坐标或变量关系改写成节点和边。
  • 能说清 DFS 进入时标记、BFS 入队时标记为什么更稳。

学习目标

起步

图题的难点不在“会不会 DFS/BFS”，而在“你先把什么当节点、什么当边”。只要建模错了，后面模板写得再像也会跑偏。

这页先把三件事分开：图怎么表示，visited 什么时候写，什么时候该用 DFS，什么时候该用 BFS。

先盯住这三步

  1. 把输入改写成邻接表。
  2. 给 visited 选对时机。
  3. 根据题目目标选 DFS 或 BFS。
你要会的动作 具体表现 错了会怎样
建模 把课程依赖、钥匙、网格、变量关系写成节点和边。 visited 会存错对象，整个搜索方向都偏。
去重 图有环时，第一次见到状态就做标记。 同一个点会被反复走进，甚至递归爆栈。
选法 连通性看 DFS，等权最短步数看 BFS。 把 BFS 用到带权问题上，结论会错。

主线地图

图建模

图题最常见的顺序是：先读输入，再把关系写成邻接表，然后才轮到 DFS 或 BFS。状态可以是节点编号、网格坐标、字符串、变量名，关键在于它能不能作为“下一步能去哪里”的起点。

visited 的作用也很具体：它负责记录“这个状态已经处理过，不要再重复走进来”。图里一旦有环，这个标记就是防线。

输入 [[0,1],[0,3],[1,2],[3,2]]

先看到的是关系对，还没有邻居列表。

邻接表 0:[1,3] / 1:[2] / 3:[2]

这一步把“谁能去谁”固定下来，后面的搜索只读它。

DFS 进入节点时标记

先记 visited，再递归邻居，环才不会把栈绕爆。

BFS 入队时标记

同一层的多个父节点不会把同一个邻居重复塞进队列。

对象 读什么 写什么 错了的症状
邻接表 原始关系对 每个节点的邻居列表 边方向写反，或者漏掉反向边
visited 当前节点是否已经见过 进入节点时或入队时立刻标记 重复走、重复入队、递归回环
DFS 递归栈 当前路径上的节点 按深度往下追邻居 有环时不断回到旧节点
BFS 队列 当前层边界 按层扩散新状态 出队才标记会把队列撑大

状态账本

visited

图遍历最容易写坏的地方在状态顺序：什么时候写、写给谁看。把这个账本想清楚，模板才会稳定。

环节 进入前 进入后 读什么 写什么 如果顺序错了
DFS 节点还没见过 先标记，再递归邻居 graph[x] visited.add(x) 遇到环时会一直回到旧节点
BFS 节点还没进队 先标记，再入队 当前层的队列 visited.add(nei) 同一个邻居会被多个父节点重复加入
邻接表 只有原始关系对 每个点都能读到自己的邻居 边列表 邻居列表 方向漏掉，图就变了
$$ O(V + E) $$

V 是节点数，E 是边数。邻接表里，每个节点和每条边都只会被常数次检查；网格题里可以把 V 理解成格子数，把复杂度写成 O(mn)。

走读示意

trace

用同一个小图看两遍：先看怎么把关系写成图，再看 DFS 和 BFS 的 visited 时机为什么不同。这里故意选一个会重复指向同一节点的例子，这样你能看见“入队时标记”到底在防什么。

建图：把关系对变成邻接表
from collections import defaultdict

def build_graph(pairs, directed=True):
    graph = defaultdict(list)
    nodes = set()
    for a, b in pairs:
        graph[a].append(b)
        nodes.add(a)
        nodes.add(b)
        if not directed:
            graph[b].append(a)
    for node in nodes:
        graph.setdefault(node, [])
    return graph

pairs = [(0, 1), (0, 3), (1, 2), (3, 2)]
graph = build_graph(pairs)
步骤 队列 / 栈 visited 观察
原始关系 还没有搜索 空 0 指向 1 和 3，1 和 3 都指向 2。
写成邻接表 graph[0] = [1,3] 空 现在每个节点都能直接读到自己的邻居。
DFS 从 0 开始 0 → 1 → 2 回退 → 3 {0,1,2,3} 进入节点时标记，2 被第二次遇到时直接跳过。
BFS 从 0 开始 [0] → [1,3] → [2] {0,1,3,2} 如果 2 在入队时不标记，它会被 1 和 3 重复塞进队列。
DFS 读法 0 先进入 1，再回到 3

DFS 更像沿着一条路往深处追，适合连通性和可达性。

BFS 读法 先处理 0 的下一层，再处理下一层

队列保持的是层边界，所以第一次到达通常就是最短层数。

DFS 标记 进入节点时写 visited

先把当前点锁住，再往后看邻居，环才不会把你拉回去。

BFS 标记 生成邻居时就写 visited

这样同一个邻居不会从两个父节点同时进队。

例子与走读

案例

图题最容易学会的方式，不是先背定义，而是把一个小例子完整走完。这里用两个锚点：841 看“房间与钥匙”，1306 看“数组下标如何变成图节点”。它们的共同点是都需要先定义节点和邻居，再决定 visited 什么时候写。

这两个例子也正好把边界感带出来。841 只问能不能到，所以 DFS 和 BFS 都能用；1306 还是只问能不能到，但它需要更谨慎的边界检查，因为邻居不是现成列表，而是你根据当前下标算出来的两个候选位置。

841 钥匙和房间

把每个房间看成节点，把钥匙列表看成邻居。起点是房间 0，visited 里存的是已经访问过的房间编号。题目的关键不是“能不能写 DFS”，而是你能不能在写代码前先把“房间 0 能到 1 和 3，1 和 3 都能到 2”这件事说清楚。

走一遍就会发现，DFS 只要进入节点时先标记，顺着钥匙往下就能扫完；BFS 也能做，但它的优势在这道题里不明显，因为这题没有“最少几步”的要求。

步骤 当前点 visited 下一步看什么
1 0 {0} 房间 0 的钥匙指向 1 和 3
2 1 {0,1} 房间 1 继续给出 2
3 3 {0,1,3} 房间 3 也给出 2，但 2 还没被处理
4 2 {0,1,2,3} 全部房间都进过 visited
841 的 DFS 写法
def canVisitAllRooms(rooms):
    visited = set()

    def dfs(room):
        if room in visited:
            return
        visited.add(room)
        for key in rooms[room]:
            dfs(key)

    dfs(0)
    return len(visited) == len(rooms)

这里最重要的一行是 visited.add(room)。如果它挪到后面，房间 0 可能沿着 1 回到 0，或者沿着 3 回到 2 再碰到别的分支，图一旦有回边，递归就不再安全。你学到的不是“把这段代码背下来”，而是知道这行为什么必须在递归之前。

1306 跳跃游戏 III

这题看起来像数组，实际上是图。下标是节点，两个邻居是 i + arr[i] 和 i - arr[i]。因为邻居不是输入里直接给你的，所以你必须自己生成它们，再做边界判断。

这题里最常见的错，不是搜索方向，而是把越界位置当成合法节点，或者把相同下标重复塞进队列。前者会直接错，后者会让搜索多做很多无意义工作。BFS 在这里更直观，因为题目又有“第一次到 0 就结束”的味道，但 DFS 也能做，只要 visited 放对位置。

步骤 当前下标 候选邻居 保留哪些
1 start start ± arr[start] 只保留在数组范围内的下标
2 合法邻居 继续算它们的 ± arr[i] 已经见过的下标直接跳过
3 arr[i] = 0 不再往外扩 直接返回成功
4 越界 非法节点 丢弃
1306 的 BFS 写法
from collections import deque

def canReach(arr, start):
    q = deque([start])
    visited = {start}
    while q:
        i = q.popleft()
        if arr[i] == 0:
            return True
        for j in (i + arr[i], i - arr[i]):
            if 0 <= j < len(arr) and j not in visited:
                visited.add(j)
                q.append(j)
    return False

这段代码里，0 <= j < len(arr) 和 j not in visited 是两个不同的门。前者挡住非法节点，后者挡住重复节点。很多人会只记住“BFS 用队列”，但真正决定结果的是这两道门有没有同时立住。

417 和 399 的边界感

417 太平洋大西洋水流问题仍然是图题，但它更适合反向思考：水是从高处流向低处的，直接正向搜会在很多点上分叉得很散；从海岸线反向搜，邻居规则反过来之后，判断会更直接。你只需要记住这一点：有些题的图是“题目写给你的”，有些题的图是你自己换个方向才看清的。

399 除法求值也能改成图，但它已经不只是“走到没走过的节点”那么简单，因为边上还带着比例。这个页面先不展开带权乘积的传递规则，只把它当成一个提醒：如果题目里答案依赖路径上的累计量，普通 BFS 的层数就不再代表结果本身。

所以你现在看到题面时，要先问三件事：节点是不是显式给出，邻居是不是要你自己生成，答案是不是只依赖可达性还是还依赖路径上的累计量。前两件事这页能解决，第三件事需要把问题送到后面的单元。

题目特征 你先做什么 最常见的错 下一步动作
邻居在输入里直接给出 先建邻接表 没把关系写成图 再决定 DFS 还是 BFS
邻居需要你自己算 先写生成规则 越界或重复节点 把 visited 和边界一起写进循环
只问能不能到 DFS 和 BFS 都能试 把“最短”错当成默认目标 用最容易写对的一种
问最少几步 先看是否等权 把带权问题硬塞给 BFS 去后续最短路框架

补一轮

前面两个锚点已经把“图怎么建”和“visited 什么时候写”讲过一遍。这里再补两道更像实战的题：721 让你看到“同一组邮箱是一团连通块”，2101 让你看到“边是有方向的，能引爆不等于能被反向引爆”。

这两题的共同点是，节点都不是天然的数字下标。你先把题面里的实体换成节点，再把关系写成边，最后才轮到 DFS 或 BFS。差别在于 721 只关心连通块，2101 关心从某个起点出发能覆盖多少点，所以前者更像“收集一团”，后者更像“统计可达范围”。

题号 节点 边怎么来 遍历目标 最容易错的点
721 账户合并 邮箱 同一账户里的邮箱互相连通 把同一连通块收拢成一组 把名字当节点，或者只连相邻邮箱

721 账户合并

把每个邮箱当成节点，同一行里出现的邮箱之间连边。姓名只用于最后输出，不适合作为 visited 的键，因为同名不同人会被误合并，同人不同写法又会被拆开。

步骤 输入行 图里发生什么 visited / 结果
1 [John, a, b] a 与 b 互相连通 {a, b}
2 [John, b, c] b 与 c 连起来，三者落到同一团 {a, b, c}
3 [Mary, d] 单独成团 {d}
4 输出 每个连通块内排序后再带上名字 [[John, a, b, c], [Mary, d]]
721 的 DFS 收集组件
from collections import defaultdict

def merge_accounts(accounts):
    graph = defaultdict(set)
    owner = {}

    for row in accounts:
        name, *emails = row
        first = emails[0]
        for email in emails:
            owner[email] = name
            graph.setdefault(email, set())
        for email in emails[1:]:
            graph[first].add(email)
            graph[email].add(first)

    seen = set()
    merged = []

    def dfs(email, bucket):
        seen.add(email)
        bucket.append(email)
        for nei in graph[email]:
            if nei not in seen:
                dfs(nei, bucket)

    for email in graph:
        if email in seen:
            continue
        bucket = []
        dfs(email, bucket)
        merged.append([owner[email]] + sorted(bucket))

    return merged

这段代码里，first 是这一行的锚点。它把同一行的邮箱绑成一团，后面的 DFS 只负责把这一团扫出来。真正常见的错，是只连相邻邮箱，或者把名字当节点键。前者会把本来该合并的邮箱拆散，后者会把不同人的同名账户混到一起。

2101 引爆最多的炸弹

这题的图是有方向的。炸弹 A 能引爆炸弹 B，不代表 B 也能回头引爆 A，所以边不能默认双向。节点还是炸弹本身，边则由“距离是否落在半径内”决定。

步骤 炸弹 能指向谁 为什么
1 (0,0,r=4) 0→1, 0→2 起点半径足够大，能先把后两颗炸弹覆盖住
2 (3,0,r=1) 1→2 半径更小，只能继续推到第三个炸弹
3 (4,0,r=1) 2→1 方向已经不是同一回事，不能偷写成无向边
4 统计 从每个起点做 DFS / BFS 看哪个起点能覆盖最多炸弹
2101 的有向建图与 DFS
def max_detonation(bombs):
    n = len(bombs)
    graph = [[] for _ in bombs]
    for i, (x1, y1, r1) in enumerate(bombs):
        r1_sq = r1 * r1
        for j, (x2, y2, _) in enumerate(bombs):
            if i == j:
                continue
            dx = x1 - x2
            dy = y1 - y2
            if dx * dx + dy * dy <= r1_sq:
                graph[i].append(j)

    def dfs(start):
        seen = {start}
        stack = [start]
        while stack:
            x = stack.pop()
            for nei in graph[x]:
                if nei not in seen:
                    seen.add(nei)
                    stack.append(nei)
        return len(seen)

    return max(dfs(i) for i in range(n))

这里最容易错的是把 graph[i].append(j) 理解成“只要两点挨得近就互相连边”。不是这样。方向来自爆炸半径本身，i 能不能到 j，只看 i 的半径够不够，不看 j 会不会反过来够到 i。一旦写成无向边，统计结果通常会被抬高。

这两题怎么回到模板

721 提醒你：连通块题，先把同团的点用 DFS 或 BFS 收起来，再做排序和输出。2101 提醒你：图边有方向时，先把边建对，再统计从每个起点能走到多少点。它们共同逼你记住同一件事：模板只是最后一步，前面的建图如果错了，模板写得再熟也会错。

如果你做练习时卡住，先别急着改代码，先画出三样东西：节点、边、visited。能把这三样说清楚，代码往往只剩下十几行；说不清楚的时候，代码行数反而会更多，但正确率更低。

200 岛屿数量：网格也是图

如果题面给的是二维网格，别把它当成和树题不一样的东西。每个格子都是节点，四个方向的上下左右就是边。visited 里存的是坐标，不是格子的值；只要一个格子已经进过队列或递归，就不要再回头算第二次。

步骤 当前格子 邻居 visited 会发生什么
1 (0,0) 上下左右四个候选位置 先标记 (0,0) 开始扩展这一块陆地
2 (0,1) 继续看四邻域 已访问坐标跳过 整块岛屿只会被数一次
3 水格 不扩展 可不记 把岛和海分开
200 的 DFS 计数
def numIslands(grid):
    rows, cols = len(grid), len(grid[0])
    seen = set()

    def dfs(r, c):
        if not (0 <= r < rows and 0 <= c < cols):
            return
        if grid[r][c] == "0" or (r, c) in seen:
            return
        seen.add((r, c))
        for dr, dc in ((1, 0), (-1, 0), (0, 1), (0, -1)):
            dfs(r + dr, c + dc)

    islands = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "1" and (r, c) not in seen:
                islands += 1
                dfs(r, c)
    return islands

这段代码里，边界判断先于递归，seen.add((r, c)) 也必须先于扩展邻居。少了前者会越界，少了后者会把同一片岛屿反复数成多片。网格题和钥匙房间题本质一样，只是节点从“编号”换成了“坐标”。

你做练习时，看到 m x n 的输入就先问一句：这是图还是矩阵。只要题目允许上下左右走、四邻域扩散，九成以上都会落回“先定节点，再定边，再定 visited”的图模型。

模板

代码

图题写模板时，真正要盯的是两句：visited.add(x) 放在什么地方，和邻居从哪里来。其余代码只是把这两句包起来。

图 DFS 模板
def dfs_graph(graph, start):
    visited = set()
    order = []

    def dfs(x):
        if x in visited:
            return
        visited.add(x)
        order.append(x)
        for nei in graph[x]:
            dfs(nei)

    dfs(start)
    return order, visited

这一版的关键是先标记、再递归。顺序一旦换掉，遇到环时就可能绕回旧节点。

图 BFS 模板
from collections import deque

def bfs_graph(graph, start):
    q = deque([start])
    visited = {start}
    order = []

    while q:
        x = q.popleft()
        order.append(x)
        for nei in graph[x]:
            if nei not in visited:
                visited.add(nei)
                q.append(nei)

    return order, visited

这一版的关键是入队时标记。这样同一层的多个父节点不会把同一个邻居重复塞进队列。

写法 推荐位置 理由
DFS 进入节点时 先把当前节点锁住，后面的邻居才不会把你绕回去。
BFS 生成邻居时 同一节点只进队一次，队列不会被重复状态撑大。
邻接表 读输入时 先把图整理好，再考虑遍历顺序。

怎么选 DFS / BFS

判断

先别急着背“DFS 看连通、BFS 看最短”这句口号。更稳的判断是：题目到底要你回答“能不能到、是否连通、这一团有多大”，还是要你回答“最少几步、最短层数、最近一次到达”。

先选 DFS 的场景

  • 只关心可达性或连通分量。
  • 要沿着一条路径持续往深处看。
  • 问题里有递归返回值或回溯痕迹。

先选 BFS 的场景

  • 每条边代价一样，题目问最少步数。
  • 起点到目标的第一次到达就有意义。
  • 多源同时扩散更自然。
题目在问什么 优先思路 理由
能不能到 DFS 或 BFS 都行 只要把图建对，任一遍历都能把可达状态扫出来。
这一团有多大 DFS 更顺手 递归或栈能很直接地把连通块扩完。
最少几步 BFS 队列按层推进，第一次到达就是当前最短层。
边权不同 先停在这页外面 这时该看 15 号单元之后的最短路框架。

常见误区

调试

图题的错法通常不花哨，基本都落在节点、边、visited、边权这四个地方。你要学会从症状倒推错因。

错因 后果 修法
把节点默认成数字编号 visited 存错对象，字符串或坐标题直接乱掉。 先写一句“节点是什么”。
无向图只加单向边 反向可达性丢失，连通分量变小。 建表时双向都 append。
DFS 递归后才标记 visited 有环时会不断回到旧节点。 进入节点时先标记。
BFS 出队时才标记 visited 同一节点会被多个父节点重复入队。 生成邻居时就标记。
普通 BFS 套到带权最短路 第一次到达不再保证最短距离。 换成后续的最短路框架。
错误示例：visited 放晚了
def dfs(x):
    for nei in graph[x]:
        dfs(nei)  # 图有环时会绕回旧节点

这段代码看起来像树递归，但图里允许有环。只要有环，就必须先做 visited 保护。

10. 用 2-3 句说明 DFS 和 BFS 各适合什么问题，别只写口号。

答题时至少要提到一种“能不能到 / 连通”场景和一种“最少几步”场景。

参考答案 / 评分要点

DFS 更适合可达性、连通块、路径枚举这类问题，因为它沿着一条路往深处走，回溯也自然。BFS 更适合等权最短步数，因为队列按层推进，第一次到达目标就有最短层数的含义。边权不同的最短路不属于这一页。
