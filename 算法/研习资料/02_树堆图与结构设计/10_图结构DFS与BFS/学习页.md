# 主线地图 {#overview}
<!-- tag: 图建模 -->

图题最常见的顺序是：先读输入，再把关系写成邻接表，然后才轮到 DFS 或 BFS。状态可以是节点编号、网格坐标、字符串、变量名，关键在于它能不能作为“下一步能去哪里”的起点。

visited 的作用也很具体：它负责记录“这个状态已经处理过，不要再重复走进来”。图里一旦有环，这个标记就是防线。

from collections import defaultdict

def build_graph(pairs, directed=True):

841 的 DFS 写法复制
```python
def canVisitAllRooms(rooms):
 visited = set()

 def dfs(room):
 if room in visited:
 return
 visited.add(room)
 for key in rooms[room]:
 dfs(key)

 dfs(0)
 return len(visited) == len(rooms)
```

这里最重要的一行是 `visited.add(room)`。如果它挪到后面，房间 0 可能沿着 1 回到 0，或者沿着 3 回到 2 再碰到别的分支，图一旦有回边，递归就不再安全。你学到的知道这行为什么必须在递归之前。

### 1306 跳跃游戏 III
这题看起来像数组，实际上是图。下标是节点，两个邻居是 `i + arr[i]` 和 `i - arr[i]`。因为邻居不是输入里直接给你的，所以你必须自己生成它们，再做边界判断。

1306 的 BFS 写法复制
```python
from collections import deque

def canReach(arr, start):
 q = deque([start])
 visited = {start}
 while q:
 i = q.popleft()
 if arr[i] == 0:
 return True
 for j in (i + arr[i], i - arr[i]):
 if 0 <= j < len(arr) and j not in visited:
 visited.add(j)
 q.append(j)
 return False
```

这段代码里，`0 <= j < len(arr)` 和 `j not in visited` 是两个不同的门。前者挡住非法节点，后者挡住重复节点。很多人会只记住“BFS 用队列”，但真正决定结果的是这两道门有没有同时立住。

### 417 和 399 的边界感
417 太平洋大西洋水流问题仍然是图题，但它更适合反向思考：水是从高处流向低处的，直接正向搜会在很多点上分叉得很散；从海岸线反向搜，邻居规则反过来之后，判断会更直接。你只需要记住这一点：有些题的图是“题目写给你的”，有些题的图是你自己换个方向才看清的。
399 除法求值也能改成图，但它已经不只是“走到没走过的节点”那么简单，因为边上还带着比例。这个页面先不展开带权乘积的传递规则，只把它当成一个提醒：如果题目里答案依赖路径上的累计量，普通 BFS 的层数就不再代表结果本身。

721 的 DFS 收集组件复制
```python
from collections import defaultdict

def merge_accounts(accounts):
 graph = defaultdict(set)
 owner = {}

 for row in accounts:
 name, *emails = row
 first = emails[0]
 for email in emails:
 owner[email] = name
 graph.setdefault(email, set())
 for email in emails[1:]:
 graph[first].add(email)
 graph[email].add(first)

 seen = set()
 merged = []

 def dfs(email, bucket):
 seen.add(email)
 bucket.append(email)
 for nei in graph[email]:
 if nei not in seen:
 dfs(nei, bucket)

 for email in graph:
 if email in seen:
 continue
 bucket = []
 dfs(email, bucket)
 merged.append([owner[email]] + sorted(bucket))

 return merged
```

这段代码里，`first` 是这一行的锚点。它把同一行的邮箱绑成一团，后面的 DFS 只负责把这一团扫出来。真正常见的错，是只连相邻邮箱，或者把名字当节点键。前者会把本来该合并的邮箱拆散，后者会把不同人的同名账户混到一起。

2101 的有向建图与 DFS复制
```python
def max_detonation(bombs):
 n = len(bombs)
 graph = [[] for _ in bombs]
 for i, (x1, y1, r1) in enumerate(bombs):
 r1_sq = r1 * r1
 for j, (x2, y2, _) in enumerate(bombs):
 if i == j:
 continue
 dx = x1 - x2
 dy = y1 - y2
 if dx * dx + dy * dy <= r1_sq:
 graph[i].append(j)

 def dfs(start):
 seen = {start}
 stack = [start]
 while stack:
 x = stack.pop()
 for nei in graph[x]:
 if nei not in seen:
 seen.add(nei)
 stack.append(nei)
 return len(seen)

 return max(dfs(i) for i in range(n))
```

这里最容易错的是把 `graph[i].append(j)` 理解成“只要两点挨得近就互相连边”。不是这样。方向来自爆炸半径本身，`i` 能不能到 `j`，只看 `i` 的半径够不够，不看 `j` 会不会反过来够到 `i`。一旦写成无向边，统计结果通常会被抬高。

## box
### 这两题怎么回到模板
721 提醒你：连通块题，先把同团的点用 DFS 或 BFS 收起来，再做排序和输出。2101 提醒你：图边有方向时，先把边建对，再统计从每个起点能走到多少点。它们共同逼你记住同一件事：模板只是最后一步，前面的建图如果错了，模板写得再熟也会错。
如果你做练习时卡住，先别急着改代码，先画出三样东西：节点、边、visited。能把这三样说清楚，代码往往只剩下十几行；说不清楚的时候，代码行数反而会更多，但正确率更低。

200 的 DFS 计数复制
```python
def numIslands(grid):
 rows, cols = len(grid), len(grid[0])
 seen = set()

 def dfs(r, c):
 if not (0 <= r < rows and 0 <= c < cols):
 return
 if grid[r][c] == "0" or (r, c) in seen:
 return
 seen.add((r, c))
 for dr, dc in ((1, 0), (-1, 0), (0, 1), (0, -1)):
 dfs(r + dr, c + dc)

 islands = 0
 for r in range(rows):
 for c in range(cols):
 if grid[r][c] == "1" and (r, c) not in seen:
 islands += 1
 dfs(r, c)
 return islands
```

这段代码里，边界判断先于递归，`seen.add((r, c))` 也必须先于扩展邻居。少了前者会越界，少了后者会把同一片岛屿反复数成多片。网格题和钥匙房间题本质一样，只是节点从“编号”换成了“坐标”。
你做练习时，看到 `m x n` 的输入就先问一句：这是图还是矩阵。只要题目允许上下左右走、四邻域扩散，九成以上都会落回“先定节点，再定边，再定 visited”的图模型。

# 模板 {#template}
<!-- tag: 代码 -->

图题写模板时，真正要盯的是两句：`visited.add(x)` 放在什么地方，和邻居从哪里来。其余代码只是把这两句包起来。

图 DFS 模板复制

```python
def dfs_graph(graph, start):
 visited = set()
 order = []

 def dfs(x):
 if x in visited:
 return
 visited.add(x)
 order.append(x)
 for nei in graph[x]:
 dfs(nei)

 dfs(start)
 return order, visited
```

这一版的关键是先标记、再递归。顺序一旦换掉，遇到环时就可能绕回旧节点。

图 BFS 模板复制

```python
from collections import deque

def bfs_graph(graph, start):
 q = deque([start])
 visited = {start}
 order = []

 while q:
 x = q.popleft()
 order.append(x)
 for nei in graph[x]:
 if nei not in visited:
 visited.add(nei)
 q.append(nei)

 return order, visited
```

这一版的关键是入队时标记。这样同一层的多个父节点不会把同一个邻居重复塞进队列。

写法
推荐位置
理由


DFS
进入节点时
先把当前节点固定住，后面的邻居才不会把你绕回去。


BFS
生成邻居时
同一节点只进队一次，队列不会被重复状态撑大。


邻接表
读输入时
先把图整理好，再考虑遍历顺序。

# 怎么选 DFS / BFS {#choose}
<!-- tag: 判断 -->

先别急着背“DFS 看连通、BFS 看最短”这句口号。更稳的判断是：题目到底要你回答“能不能到、是否连通、这一团有多大”，还是要你回答“最少几步、最短层数、最近一次到达”。

## box soft
### 先选 DFS 的场景
- 只关心可达性或连通分量。
- 要沿着一条路径持续往深处看。
- 问题里有递归返回值或回溯痕迹。

## box warn
### 先选 BFS 的场景
- 每条边代价一样，题目问最少步数。
- 起点到目标的第一次到达就有意义。
- 多源同时扩散更自然。

# 常见误区 {#mistakes}
<!-- tag: 调试 -->

图题的错法通常不花哨，基本都落在节点、边、visited、边权这四个地方。你要学会从症状倒推错因。

错因
后果
修法


把节点默认成数字编号
visited 存错对象，字符串或坐标题直接乱掉。
先写一句“节点是什么”。


无向图只加单向边
反向可达性丢失，连通分量变小。
建表时双向都 append。


DFS 递归后才标记 visited
有环时会不断回到旧节点。
进入节点时先标记。


BFS 出队时才标记 visited
同一节点会被多个父节点重复入队。
生成邻居时就标记。


普通 BFS 套到带权最短路
第一次到达不再保证最短距离。
换成后续的最短路框架。

错误示例：visited 放晚了复制

```python
def dfs(x):
 for nei in graph[x]:
 dfs(nei) # 图有环时会绕回旧节点
```

这段代码看起来像树递归，但图里允许有环。只要有环，就必须先做 visited 保护。

# 练习 {#practice}
<!-- tag: diagnostic -->

<!-- practice-intro -->
<!-- end-practice-intro -->

**建模**先说节点是什么，再说边是什么。
**去重**图有环时，visited 不能拖后。
**选择**连通性偏 DFS，等权最短步数偏 BFS。
**边界**带权最短路先停在这页外面。

### 1 choice

**Q:** 1. 开始做图题时，最先要说清什么？

- [ ] 节点是什么，边是什么
- [ ] 先把所有点排成有序数组
- [ ] 先写 BFS 模板

Answer: 节点是什么，边是什么
Explain: 先定节点和边，visited 才知道该存什么。

### 2 true/false

**Q:** 2. 节点一定要是数字编号。

- [ ] 对
- [ ] 错

Answer: 错
Explain: 节点可以是坐标、字符串、变量名，不一定是数字编号。

### 3 fill

**Q:** 3. 邻接表遍历的时间复杂度是 ______。

Answer: O(V+E),O(v+e),O(n+m)
Explain: 邻接表里每个点和每条边都只被常数次检查。

### 4 choice

**Q:** 4. 无向图的建表方式应该是：

- [ ] a→b 和 b→a 都要写
- [ ] 只保留一条边
- [ ] 先排成数组再说

Answer: a->b 和 b->a 都要写
Explain: 无向图的边要双向加入邻接表。

### 5 debug-checklist

**Q:** 5. 下面哪些说法正确？

- [ ] DFS 进入节点时标记 visited
- [ ] BFS 入队时标记 visited
- [ ] 出队后再标记会让队列更短

Explain: 正确的是前两项。第三项正好相反：出队才标记通常会让队列里出现重复状态，问题越大越明显。

### 6 short-answer

**Q:** 6. 841 中节点和边分别是什么？visited 里应该存什么？

Explain: 节点是房间，边是钥匙指向的房间，visited 里存已经访问过的房间编号。答到这三点就够。

### 7 choice

**Q:** 7. BFS 为什么通常在入队时标记 visited？

- [ ] 防止同一邻居被多个父节点重复入队
- [ ] 这样就不需要邻接表
- [ ] 因为 BFS 不会遇到重复状态

Answer: 防止同一邻居被多个父节点重复入队
Explain: BFS 的重复问题出在同层多个父节点同时看到同一个邻居。

### 8 code-reading

**Q:** 8. 如果把 DFS 里的 `visited.add(x)` 放到递归之后，最直接的风险是什么？

- [ ] 遇到环时可能无限递归
- [ ] 时间复杂度必定变成 O(1)
- [ ] 邻接表自动变成无向图

Answer: 遇到环时可能无限递归
Explain: visited 放晚了，递归会重新走回旧节点。

### 9 fill

**Q:** 9. BFS 按层推进时，`step` 代表 ______。

Answer: 层数
Explain: BFS 的 step 代表当前扩散到第几层，也就是离起点多远。

### 10 synthesis

**Q:** 10. 用 2-3 句说明 DFS 和 BFS 各适合什么问题，别只写口号。

Explain: DFS 更适合可达性、连通块、路径枚举这类问题，因为它沿着一条路往深处走，回溯也自然。BFS 更适合等权最短步数，因为队列按层推进，第一次到达目标就有最短层数的含义。边权不同的最短路不属于这一页。

### 11 choice

**Q:** 11. 无向图只加单向边会带来什么问题？

- [ ] 反向可达性丢失，连通分量判断会错
- [ ] visited 会自动清空
- [ ] DFS 立刻变成 BFS

Answer: 反向可达性丢失，连通分量判断会错
Explain: 无向图必须双向建边。

### 12 choice

**Q:** 12. 普通 BFS 不适合边权不同的最短路，因为：

- [ ] 第一次到达不再保证最短距离
- [ ] BFS 只能处理字符串
- [ ] 队列不支持出队

Answer: 第一次到达不再保证最短距离
Explain: 边权不同的时候，层数不等于真实代价。

### 13 debug-checklist

**Q:** 13. 按“错因 / 后果 / 修法”写出忘记 visited 的诊断。

Explain: 错因通常是把图误当成树，默认不会回到旧节点；后果是重复搜索、队列膨胀或无限递归；修法是在 DFS 进入节点时、BFS 入队时立刻标记 visited。三个部分都写到就算完整。

### 14 synthesis

**Q:** 14. 用 6 句左右讲清图建模、visited、不变量、复杂度和迁移，像是在教别人做题。

Explain: 高分答案通常会先说节点和边怎么定义，再说 visited 的写入时机，接着说 DFS / BFS 各适合什么目标，最后给出 O(V+E) 的复杂度口径，并提醒带权最短路不在这一页。能把 841、1306、417、399 里至少一题说成图，就说明迁移到位了。
