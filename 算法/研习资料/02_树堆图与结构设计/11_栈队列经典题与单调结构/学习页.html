用受限操作换稳定状态

栈和队列都很朴素：栈后进先出，队列先进先出。它们有用，是因为很多问题本来就有这样的顺序约束。括号匹配要看最近打开的括号，路径简化要处理最近一段目录，BFS 要按先到的状态扩散。

单调栈和单调队列是在普通栈队列上加一个条件：里面的元素按大小保持单调。它们不是为了“排序”，而是为了快速找到下一个更大元素、窗口最大值、最近更小边界。

学完应该能做什么

  1. 能说清栈、队列、单调栈、单调队列各自维护什么顺序。
  2. 能写出括号匹配、路径简化、逆波兰表达式的栈模板。
  3. 能解释单调栈如何找下一个更大元素。
  4. 能解释单调队列如何维护滑动窗口最大值。
  5. 能诊断“过期元素没删”“比较方向反了”等常见错误。

这一组题的共同点是：答案依赖“最近的未解决元素”或“当前窗口的最优元素”。栈负责最近，队列负责先后，单调结构负责在保持先后关系的同时留下更有价值的候选。

题目括号 / 路径 / 窗口

栈处理最近未匹配

有效括号里，右括号只应该匹配最近的左括号。这个“最近”就是栈的后进先出。

路径简化里，`..` 要撤销最近一个目录，也适合栈。

队列处理先到先走

BFS、最近请求次数、循环队列都依赖先进先出。队列的重点是头尾指针和容量边界。

如果你发现题目按时间顺序到达、按时间顺序离开，多半有队列影子。

单调结构保留候选

单调栈弹掉那些不可能成为答案的元素。单调队列弹掉窗口里更旧且更差的候选。

判断句：如果每个元素都想找左边或右边第一个更大/更小，想单调栈；如果窗口移动还要取最值，想单调队列。

术语表

terms

术语直觉准确说法常见误解
栈最后放进去的先拿出来。只允许在同一端插入和删除的线性结构。把栈当成可以随便查删的列表。
队列先来先服务。一端入队、另一端出队的线性结构。用列表头部 pop 导致 O(n)。
单调栈栈内元素保持递增或递减。入栈前弹出破坏单调性的元素，常用于找最近更大/更小关系。以为单调栈保存所有元素。
单调队列窗口里只保留可能成为最大/最小的候选。队头是当前窗口最值，队尾入新元素时删除更差候选。忘记删除窗口外元素。
过期元素已经不属于当前窗口的候选。下标落在窗口左边界之前的元素。只比较值不保存下标，无法判断是否过期。

知识链路

chain

栈题先看“最近关系”。20「有效的括号」中，当前右括号必须和最近的左括号匹配；71「简化路径」中，`..` 取消最近一级目录；150「逆波兰表达式」中，运算符取最近两个数字计算。

单调栈题先看“谁被谁解决”。例如下一个更大元素，当前元素比栈顶大时，当前元素就是栈顶等待的答案。被弹出的元素已经得到答案，当前元素再入栈等待未来更大的数。

单调队列题先看窗口。新元素进入时，队尾那些比它小的旧元素再也不可能成为最大值，可以删掉；窗口左边界移动时，队头如果过期也要删掉。

本页的判断句

  • 右括号匹配最近左括号，用栈。
  • 时间顺序进入和离开，用队列。
  • 找第一个更大/更小，用单调栈。
  • 滑动窗口取最值，用单调队列，队列里最好存下标。
单调结构的摊还复杂度：每个元素最多入一次、出一次，所以整体 O(n)

核心模板

template

栈模板的核心是：遇到开放状态就入栈，遇到关闭状态就检查栈顶。错误通常来自空栈没判、匹配表写反、处理完后忘记检查栈是否为空。

单调栈模板的核心是 while。只要当前元素能解决栈顶，就持续弹出。弹出时写答案，循环结束后当前元素入栈等待别人解决它。

正确示例：括号匹配Python
def isValid(s):
    pairs = {")": "(", "]": "[", "}": "{"}
    stack = []
    for ch in s:
        if ch in pairs.values():
            stack.append(ch)
        else:
            if not stack or stack[-1] != pairs[ch]:
                return False
            stack.pop()
    return not stack

右括号来时必须看最近的左括号。最后 `not stack` 防止还有未关闭的左括号。

正确示例：下一个更大元素Python
def nextGreater(nums):
    ans = [-1] * len(nums)
    stack = []  # 存下标，栈内对应值递减
    for i, x in enumerate(nums):
        while stack and nums[stack[-1]] < x:
            j = stack.pop()
            ans[j] = x
        stack.append(i)
    return ans

当前 x 比栈顶元素大，就解决栈顶的“下一个更大”。栈里存下标，方便写回答案。

切换场景，观察关键状态怎么变。看懂状态，比背下一行代码更重要。

右括号匹配栈顶

读到 `]` 时，只能匹配最近的 `[`。如果栈顶不是 `[`，就说明嵌套顺序错了。

你要能判断：容器里保存的不是所有历史，而是未来还可能有用的状态。

例题：简化路径Python
def simplifyPath(path):
    stack = []
    for part in path.split("/"):
        if part == "" or part == ".":
            continue
        if part == "..":
            if stack:
                stack.pop()
        else:
            stack.append(part)
    return "/" + "/".join(stack)

普通目录入栈，`..` 弹出最近目录。根目录处再遇到 `..` 不应继续弹。

例题：滑动窗口最大值的单调队列Python
from collections import deque

def maxSlidingWindow(nums, k):
    q = deque()  # 存下标，对应值递减
    ans = []
    for i, x in enumerate(nums):
        while q and nums[q[-1]] <= x:
            q.pop()
        q.append(i)
        if q[0] <= i - k:
            q.popleft()
        if i >= k - 1:
            ans.append(nums[q[0]])
    return ans

队头是最大值下标。用下标而不是值，是为了判断是否已经离开窗口。

边界与复杂度

limits

空栈访问

右括号到来时，如果栈为空，直接访问 `stack[-1]` 会报错。要先判空。

重复值比较

单调队列里用 `<` 还是 `<=` 会影响保留旧值还是新值。多数最大值题用 `<=` 弹掉旧的相等值，队列更短。

循环队列容量

设计循环队列时，要明确用一个空位区分满和空，还是额外维护 size。不要只靠 head==tail。

做法时间空间什么时候用
普通栈题O(n)O(n)括号、路径、表达式
单调栈O(n)O(n)下一个更大/更小、柱状图边界
单调队列O(n)O(k)滑动窗口最大/最小

常见错误：错因、后果和修法

debug

空栈不判

错因：默认右括号前一定有左括号。

后果：非法输入时报错或误判。

修法：访问栈顶前先判断 `not stack`。

运算符顺序反了

错因：逆波兰表达式弹出两个数后随便相减相除。

后果：`a-b` 写成 `b-a`。

修法：先弹出的是右操作数，后弹出的是左操作数。

单调方向写反

错因：没明确找更大还是更小。

后果：弹出了应该保留的候选。

修法：先写出栈内要递增还是递减。

窗口过期不删

错因：单调队列只维护大小关系，不维护窗口边界。

后果：队头可能是窗口外元素。

修法：队列存下标，每轮检查 `q[0] <= i-k`。

把单调栈当排序

错因：以为栈里必须保存所有元素。

后果：不敢弹出“没用”的候选，模板失效。

修法：理解被弹出的元素已经得到答案或不可能成为答案。

错误示例：窗口最大值只存值不存下标Python
q = deque()
for x in nums:
    while q and q[-1] <= x:
        q.pop()
    q.append(x)
    # 无法判断队头是不是已经离开窗口

只存值会丢失位置。滑动窗口题通常要存下标。

易混概念

compare

栈 vs 队列

栈：最近进入的先处理。

队列：最早进入的先处理。

判断：括号、撤销、嵌套用栈；层序、时间窗口用队列。

单调栈 vs 单调队列

单调栈：解决左右第一个更大/更小。

单调队列：维护滑动窗口最值。

判断：没有窗口移动，多想栈；有窗口边界，多想队列。

存值 vs 存下标

存值：只关心数值，不需要位置。

存下标：需要写回答案或判断过期。

判断：单调结构默认优先考虑存下标。
