让子树自己交回答案

二叉树的另一种核心思路是分解问题。你不一定要亲自维护从根到当前节点的路径，而是把整棵树拆成根、左子树、右子树。只要能确定根在哪里，左右子树范围在哪里，构造题就能写出来。

序列化和删点成林也一样。序列化要把当前节点和左右子树的表示拼起来；反序列化要按同一规则读回树；删点成林要先让左右子树处理完，再决定当前节点是否保留。这页会把“返回值代表什么”讲清楚。

学完应该能做什么

  1. 能区分遍历思维和分解思维。
  2. 能用前序+中序、中序+后序构造二叉树。
  3. 能解释为什么只给前序+后序通常无法唯一确定普通二叉树。
  4. 能写出包含空节点标记的序列化/反序列化框架。
  5. 能用后序思想处理删点成林。

主线地图

mental model

分解题的关键不是访问顺序本身，而是“当前函数返回什么”。构造题返回当前范围对应的子树根；序列化返回当前子树的字符串；删点成林返回删除后的当前子树根。返回值说清楚，递归就稳。

变形序列化 / 删除

构造题先找根

前序遍历的第一个元素是根，后序遍历的最后一个元素是根。中序遍历负责告诉你左子树和右子树的边界。

判断句：构造二叉树时，先问根从哪来，再问左右范围怎么切。

序列化必须记录空节点

如果只记录非空节点，很多不同结构会得到同一个序列。空节点标记能保留形状信息。

前序序列化常写成 `root,left,right`，空节点用 `#`。

删除题常用后序

删点成林中，你必须先处理左右子树，拿到它们删除后的结果，再决定当前节点是否被删除。

如果当前节点被删，未删除的孩子会成为新树根。

术语直觉准确说法常见误解
分解思维把整棵树的问题拆成根、左子树、右子树。递归函数向子树要结果，再在当前节点合并或返回。以为分解思维就是换一种遍历顺序。
前序遍历根在最前面。访问顺序为根、左、右。只会背顺序，不知道构造题用它找根。
中序遍历根夹在左子树和右子树中间。访问顺序为左、根、右，可用于划分左右子树范围。没有唯一值时仍直接用哈希定位。
序列化把树写成字符串。按约定遍历顺序和空节点标记，将树转为可恢复的文本或列表。省略空节点，以为还能完整还原结构。
反序列化按同一规则把字符串读回树。消费序列中的标记，递归构造当前节点及其左右子树。读写规则不一致，导致还原错位。

知识链路

chain

105「从前序与中序遍历序列构造二叉树」的链路是：前序首元素给出根；在中序中找到根的位置；位置左边是左子树，右边是右子树；左右子树长度反过来决定前序里的切分范围。

106「从中序与后序遍历序列构造二叉树」只是根的位置从前序首部换成后序尾部。思路没有变：根负责切分中序，长度负责切分另一条序列。

331「验证二叉树的前序序列化」不要求真正建树。它考的是槽位：一个非空节点消耗一个槽位，并产生两个新槽位；空节点只消耗一个槽位。槽位不能提前变负，最后必须刚好为 0。

本页的判断句

  • 构造题先找根，再用中序切左右范围。
  • 递归函数的返回值通常是当前范围构造出的子树根。
  • 序列化若要可逆，必须记录空节点。
  • 删点成林要先处理孩子，再决定当前节点是否保留。
构造题核心：根的位置 + 左子树长度 + 右子树长度 决定递归范围

核心模板

template

构造题最容易写乱的是切片边界。为了少犯错，可以用下标范围而不是每次切新数组。先建立 `inorder` 值到下标的映射，定位根就能 `O(1)`。

序列化/反序列化必须成对设计。你用前序写，就要按前序读；你写了空节点标记，读的时候遇到标记就返回 None。

正确示例：前序 + 中序构造树Python
def buildTree(preorder, inorder):
    pos = {v: i for i, v in enumerate(inorder)}

    def build(pre_l, pre_r, in_l, in_r):
        if pre_l > pre_r:
            return None
        root_val = preorder[pre_l]
        root = TreeNode(root_val)
        mid = pos[root_val]
        left_size = mid - in_l
        root.left = build(pre_l + 1, pre_l + left_size, in_l, mid - 1)
        root.right = build(pre_l + left_size + 1, pre_r, mid + 1, in_r)
        return root

    return build(0, len(preorder) - 1, 0, len(inorder) - 1)

`left_size` 是桥梁：它把中序里的左子树长度映射到前序范围。

正确示例：前序序列化和反序列化Python
def serialize(root):
    vals = []
    def dfs(node):
        if node is None:
            vals.append("#")
            return
        vals.append(str(node.val))
        dfs(node.left)
        dfs(node.right)
    dfs(root)
    return ",".join(vals)

def deserialize(data):
    vals = iter(data.split(","))
    def build():
        x = next(vals)
        if x == "#":
            return None
        node = TreeNode(int(x))
        node.left = build()
        node.right = build()
        return node
    return build()

写入顺序是根、左、右；读取也必须按根、左、右消费标记。`#` 保存了结构信息。

前序首位是根

preorder[0] 找到根，再去 inorder 中切开左右子树。切开的长度决定前序范围。

你要能判断：当前函数到底返回子树根、字符串，还是删除后的新根。

例题：验证前序序列化的槽位法Python
def isValidSerialization(preorder):
    slots = 1
    for x in preorder.split(","):
        slots -= 1
        if slots < 0:
            return False
        if x != "#":
            slots += 2
    return slots == 0

每个节点先消耗一个父节点给它的槽位。非空节点还能给孩子提供两个新槽位，空节点不能提供。

例题：删点成林Python
def delNodes(root, to_delete):
    delete = set(to_delete)
    forest = []

    def dfs(node, is_root):
        if node is None:
            return None
        removed = node.val in delete
        if is_root and not removed:
            forest.append(node)
        node.left = dfs(node.left, removed)
        node.right = dfs(node.right, removed)
        return None if removed else node

    dfs(root, True)
    return forest

`is_root` 表示当前节点是否可能成为森林中新树的根。父节点被删时，孩子就获得这个机会。

边界与复杂度

limits

节点值重复

构造题通常默认节点值唯一。若不唯一，单靠值到下标的哈希表会失效。

前序 + 后序不唯一

普通二叉树只给前序和后序，往往无法唯一确定左右子树。真二叉树等额外条件会改变这个结论。

序列化格式一致

读写必须使用同一遍历顺序、同一分隔符、同一空节点标记。格式不一致，树会错位。

做法时间空间什么时候用
构造树O(n)O(n)前序/中序或中序/后序
序列化/反序列化O(n)O(n)持久化树结构
槽位验证O(n)O(1)只验证序列是否合法

常见错误：错因、后果和修法

debug

切片边界错一位

错因：没有用 left_size 统一前序和中序范围。

后果：左右子树节点数不一致，构造错树或递归死循环。

修法：先算 `left_size = mid - in_l`，再写范围。

序列化漏空节点

错因：只记录非空节点值。

后果：不同结构变成同一个字符串，无法还原。

修法：用 `#` 或 null 标记空孩子。

反序列化顺序不一致

错因：前序写入，却按层序读取。

后果：节点消费错位，还原结构错误。

修法：读写使用同一协议。

删点先处理当前再丢孩子

错因：当前节点被删后没有把未删除孩子加入森林。

后果：子树丢失。

修法：用 `is_root` 或父节点删除标记传给孩子。

把返回值写成全局答案

错因：没明确 dfs 返回当前子树根。

后果：父节点无法接上处理后的孩子。

修法：每个递归函数先写职责：返回删除后的当前根。

错误示例：序列化不记录空节点Python
def serialize(root):
    vals = []
    def dfs(node):
        if node is None:
            return
        vals.append(str(node.val))
        dfs(node.left)
        dfs(node.right)
    dfs(root)
    return ",".join(vals)

这段代码丢掉了结构。只有节点值顺序，没有空孩子位置，反序列化无法唯一还原。

易混概念

compare

遍历思维 vs 分解思维

遍历思维：沿路维护状态，走到节点时处理。

分解思维：拿左右子树结果，在当前节点合并。

判断：构造、删除、序列化常要先定义返回值。

前序 vs 中序

前序：根在开头，帮你找根。

中序：根在中间，帮你切左右。

判断：构造题里二者分工不同，不能互换。

序列化 vs 遍历结果

序列化：要能还原结构。

普通遍历结果：只记录访问顺序。

判断：能不能反序列化，取决于是否保留空节点或其他结构信息。
