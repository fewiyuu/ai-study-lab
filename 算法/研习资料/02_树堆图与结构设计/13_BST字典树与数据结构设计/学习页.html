BST / Trie / LRU

把操作需求变成结构约束

BST、Trie、LRU 看起来分散，其实都在做同一件事：让结构贴合操作需求。BST 用“左小右大”加速有序查找；Trie 用字符路径加速前缀匹配；LRU 用哈希表加双向链表同时满足 O(1) 查找和 O(1) 更新顺序。

做设计题时，先不要急着写类。把每个操作列成表：它要查什么？要更新什么？目标复杂度是多少？如果单一结构做不到，就组合结构。LRU 就是最典型的组合。

学完应该能做什么

  1. 能用 BST 的有序性质优化查找和遍历。
  2. 能写出 Trie 插入、搜索、前缀匹配的核心逻辑。
  3. 能解释 LRU 为什么需要哈希表 + 双向链表。
  4. 能区分 MinStack、FreqStack、Deque 等设计题的状态维护方式。
  5. 能按 API 反推数据结构组合。

主线地图

mental model

结构设计的主线是“操作反推结构”。如果要按 key 快速查，哈希表合适；如果要维护最近使用顺序，链表合适；如果要查前缀，Trie 合适；如果要按有序范围查，BST 合适。

操作需求get / put / search
复杂度O(1) / O(log n)
结构性质有序 / 前缀 / 顺序
组合结构哈希 + 链表
实现细节边界和更新

BST 的价值是有序剪枝

二叉搜索树要求左子树都小于当前节点，右子树都大于当前节点。查找时，每一步可以丢掉一半方向。

中序遍历 BST 会得到升序结果，这是很多题的切入点。

Trie 的价值是共享前缀

字典树把字符串按字符一层层存。相同前缀只存一份，所以搜索单词和前缀都能按字符长度处理。

判断句：看到前缀、通配符、词典匹配，先想 Trie。

LRU 的价值是组合

哈希表能 O(1) 找到节点，双向链表能 O(1) 移动节点到最新位置，也能 O(1) 删除最旧节点。

单用哈希表不知道谁最久没用，单用链表又找不到 key。

术语表

terms

先把词说清楚，后面的代码才不会变成背模板。

术语直觉准确说法常见误解
BST每个节点把小的放左边，大的放右边。任意节点的左子树所有值小于节点值，右子树所有值大于节点值。只检查当前节点和左右孩子，没检查整棵子树范围。
Trie把单词按字符路径存成树。多叉树节点表示前缀，边或子节点对应字符，终止标记表示完整单词。前缀存在就等于单词存在。
LRU容量满时删最久没用的。Least Recently Used 缓存策略，get 和 put 都会刷新使用顺序。只在 put 时刷新，不在 get 时刷新。
双向链表一个节点能向前也能向后。每个节点保存 prev 和 next，支持 O(1) 删除已知节点。用单链表也想 O(1) 删除任意节点。
哨兵节点固定头尾，减少空链表特判。不存真实数据的 dummy head/tail，用于统一插入删除逻辑。把哨兵当成缓存数据返回。

知识链路

chain

BST 题先问能不能利用有序性。查找某个值时，小了往右，大了往左；验证 BST 时，要传上下界，而不是只比较父子。因为一个节点必须满足来自所有祖先的范围约束。

Trie 的节点通常包含 `children` 和 `is_word`。插入时沿字符创建节点；搜索完整单词时，最后必须检查 `is_word`；搜索前缀时，只要路径存在即可。通配符匹配会在某一层分叉搜索。

LRU 的 API 是 `get(key)` 和 `put(key, value)`，目标都是 O(1)。哈希表负责 key 到链表节点的定位；双向链表负责维护最近使用顺序。每次访问后，把节点移到链表头；容量超出时，删除尾部最旧节点。

本页的判断句

  • BST 查找利用上下界和有序剪枝。
  • Trie 搜完整单词要看终止标记，搜前缀只看路径。
  • LRU 中 get 也算使用，必须刷新顺序。
  • 设计题先列 API，再反推每个 API 需要的状态。
LRU 组合：dict: key → node + 双向链表: 最近使用顺序，目标 get/put O(1)

核心模板

template

BST 模板里最重要的是范围。验证时每个节点都有合法区间 `(low, high)`，左子树收紧上界，右子树收紧下界。只比较当前节点和左右孩子是不够的。

Trie 模板里最重要的是终止标记。路径 `app` 存在，只能说明有单词以 `app` 为前缀；如果 `is_word` 为假，就不能说单词 `app` 存在。

正确示例：验证 BST 的上下界Python
def isValidBST(root):
    def dfs(node, low, high):
        if node is None:
            return True
        if not (low < node.val < high):
            return False
        return dfs(node.left, low, node.val) and dfs(node.right, node.val, high)

    return dfs(root, float("-inf"), float("inf"))

上下界来自所有祖先。右子树里的每个节点都必须大于根，不只是大于父节点。

正确示例：Trie 的插入和查询Python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_word = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for ch in word:
            node = node.children.setdefault(ch, TrieNode())
        node.is_word = True

    def search(self, word):
        node = self.root
        for ch in word:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return node.is_word

`search` 最后返回 `is_word`，不是简单返回路径是否存在。`startsWith` 才只需要路径存在。

例题：LRU 的核心节点操作Python
class Node:
    def __init__(self, key=0, val=0):
        self.key = key
        self.val = val
        self.prev = None
        self.next = None

def remove(node):
    node.prev.next = node.next
    node.next.prev = node.prev

def add_to_front(head, node):
    node.prev = head
    node.next = head.next
    head.next.prev = node
    head.next = node

双向链表删除已知节点只改四条指针。哨兵 head/tail 能减少空链表特判。

例题：MinStack 保存同步最小值Python
class MinStack:
    def __init__(self):
        self.stack = []
        self.min_stack = []

    def push(self, x):
        self.stack.append(x)
        if not self.min_stack:
            self.min_stack.append(x)
        else:
            self.min_stack.append(min(x, self.min_stack[-1]))

    def pop(self):
        self.min_stack.pop()
        return self.stack.pop()

    def getMin(self):
        return self.min_stack[-1]

辅助栈和主栈同步增长。每一层都保存“到当前位置为止的最小值”。

常见错误：错因、后果和修法

debug

BST 只比较父子

错因：没有传祖先上下界。

后果：右子树里小于根的节点可能漏判。

修法：递归传 low/high。

Trie 搜索忘记 is_word

错因：把路径存在当作单词存在。

后果：`app` 会被误判为存在。

修法：完整搜索最后返回终止标记。

LRU get 不刷新

错因：以为只有 put 算使用。

后果：最近访问过的节点仍可能被淘汰。

修法：get 命中后 move_to_front。

双向链表指针漏改

错因：删除或插入时只改一侧指针。

后果：链表断裂或形成错误环。

修法：删除和插入都按四条指针检查。

设计题不列 API

错因：直接凭印象选结构。

后果：某个操作复杂度达不到要求。

修法：先列每个方法的读写状态和目标复杂度。

错误示例：验证 BST 只看左右孩子Python
def isValidBST(root):
    if root is None:
        return True
    if root.left and root.left.val >= root.val:
        return False
    if root.right and root.right.val <= root.val:
        return False
    return isValidBST(root.left) and isValidBST(root.right)

这段代码没检查祖先范围。右子树深处出现小于根的值时会漏判。

易混概念

compare

BST vs 普通二叉树

BST：有左小右大的全局约束。

普通二叉树：只表示父子结构，不保证大小关系。

判断：题目没有给有序性质，就不能擅自用 BST 剪枝。

Trie search vs startsWith

search：路径存在且终止标记为真。

startsWith：只要求前缀路径存在。

判断：完整单词看 is_word，前缀查询不看。

哈希表 vs 链表

哈希表：快速按 key 找节点。

链表：维护顺序并 O(1) 移动已知节点。

判断：LRU 同时需要二者，单独一个不够。
