拓扑排序处理依赖

课程 A 必须在课程 B 前完成，这就是有向边。入度为 0 的节点表示当前没有前置依赖，可以先处理。

如果最后处理的节点数少于总数，说明有环，依赖无法完成。

并查集处理连通

并查集不关心路径细节，只关心两个点是否在同一集合。它适合动态合并、判断连通、统计连通分量。

路径压缩和按秩合并让操作接近常数时间。

Dijkstra 处理非负权最短路

Dijkstra 每次从优先队列里取当前距离最小的节点，并尝试松弛它的邻居。

边权不能为负。遇到负权边，要考虑 Bellman-Ford 等算法。

术语表

terms

先把词说清楚，后面的代码才不会变成背模板。

术语直觉准确说法常见误解
拓扑排序按依赖关系排一个可执行顺序。有向无环图中，若存在边 u→v，则 u 在序列中出现在 v 前。以为无向图也能拓扑排序。
入度还有多少前置任务没完成。指向某个节点的边数。把出度当入度更新。
并查集快速判断两个点是否属于同一组。维护若干不相交集合，支持 find 和 union。以为它能给出两点之间的具体路径。
松弛发现一条更便宜的路，就更新距离。若 `dist[u] + w < dist[v]`，则更新 `dist[v]`。只要看到邻居就更新，不比较是否更优。
非负权每条边代价不能为负。Dijkstra 正确性的前提之一。有负权边也直接套 Dijkstra。

知识链路

chain

拓扑排序的链路是：建有向图，统计每个节点入度，把入度为 0 的节点入队；每处理一个节点，就把它指向的邻居入度减一；新的入度为 0 节点再入队。处理完数量不足，说明存在环。

并查集的链路是：每个节点开始时父亲是自己；`find(x)` 找到集合代表；`union(a,b)` 把两个代表接起来。路径压缩让以后查同一集合更快。

Dijkstra 的链路是：`dist[start]=0`，其他为无穷大；优先队列按当前距离取节点；如果弹出的距离已经过期就跳过；否则用它松弛所有邻居。最终 `dist` 保存从起点到各点的最短距离。

本页的判断句

  • 有向依赖顺序，想拓扑排序。
  • 只问是否连通或合并集合，想并查集。
  • 非负边权最短路，想 Dijkstra。
  • 等权最短路用 BFS 更轻，不必上 Dijkstra。
Dijkstra 松弛条件：if dist[u] + w < dist[v]: dist[v] = dist[u] + w

核心模板

template

拓扑排序的核心状态是入度。不要只建邻接表不维护入度，否则你不知道当前谁可以先处理。

并查集的核心状态是 parent。`find` 不是简单返回 parent[x]，而是一路找到根，并把沿途节点直接接到根上。

Dijkstra 的核心状态是 dist 和优先队列。队列里可能有旧距离，所以弹出后要判断 `if d > dist[u]: continue`。

正确示例：拓扑排序Python
from collections import deque

def topo(n, edges):
    graph = [[] for _ in range(n)]
    indeg = [0] * n
    for a, b in edges:
        graph[a].append(b)
        indeg[b] += 1
    q = deque([i for i in range(n) if indeg[i] == 0])
    order = []
    while q:
        x = q.popleft()
        order.append(x)
        for y in graph[x]:
            indeg[y] -= 1
            if indeg[y] == 0:
                q.append(y)
    return order if len(order) == n else []

入度为 0 表示没有未完成前置依赖。处理数量不足，就是有环。

正确示例：DijkstraPython
import heapq

def dijkstra(graph, start):
    dist = {start: 0}
    pq = [(0, start)]
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist.get(u, float("inf")):
            continue
        for v, w in graph[u]:
            nd = d + w
            if nd < dist.get(v, float("inf")):
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    return dist

每次只用当前最短的弹出状态扩展。过期状态跳过，避免重复扩展旧路径。

入度为 0 先处理

A 完成后，B 的入度减一。谁的入度变成 0，谁就进入可处理队列。

你要能根据输出目标选择算法，而不是看到图就套同一个遍历。

正确示例：并查集Python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.size = [1] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return False
        if self.size[ra] < self.size[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        self.size[ra] += self.size[rb]
        return True

`find` 带路径压缩，`union` 按 size 把小集合挂到大集合上。

例题：概率最大路径的 Dijkstra 变形Python
import heapq

def maxProbability(n, edges, succProb, start, end):
    graph = [[] for _ in range(n)]
    for (a, b), p in zip(edges, succProb):
        graph[a].append((b, p))
        graph[b].append((a, p))
    prob = [0.0] * n
    prob[start] = 1.0
    pq = [(-1.0, start)]
    while pq:
        neg, u = heapq.heappop(pq)
        cur = -neg
        if u == end:
            return cur
        if cur < prob[u]:
            continue
        for v, p in graph[u]:
            np = cur * p
            if np > prob[v]:
                prob[v] = np
                heapq.heappush(pq, (-np, v))
    return 0.0

把“距离越小越好”换成“概率越大越好”。Python 小根堆用负数模拟最大概率优先。

边界与复杂度

limits

拓扑只适合有向图

无向图没有“先后依赖”的方向，不能直接做拓扑排序。

并查集不回答路径

并查集能说是否连通，但不能告诉你具体怎么走，也不能处理带权最短路径。

Dijkstra 不处理负权

负权边会破坏“当前最小出队即确定”的前提。遇到负权要换算法。

做法时间空间什么时候用
拓扑排序O(V+E)O(V+E)依赖顺序、环检测
并查集近似 O(1)O(V)动态连通、合并集合
DijkstraO(E log V)O(V+E)非负边权最短路

常见错误：错因、后果和修法

debug

拓扑边方向反了

错因：把“学 A 才能学 B”建成 B→A。

后果：顺序反转，入度含义错。

修法：明确边表示先修指向后续。

并查集 find 不压缩

错因：只返回父节点，不递归找根。

后果：链很长时性能变差，连通判断可能不稳。

修法：`parent[x] = find(parent[x])`。

Dijkstra 不跳过旧状态

错因：优先队列里有多份同一节点距离。

后果：重复扩展，性能变差。

修法：弹出后检查 `d > dist[u]`。

负权边套 Dijkstra

错因：只看到最短路就套模板。

后果：可能得到错误答案。

修法：确认边权非负。

等权图过度使用 Dijkstra

错因：不知道 BFS 已经足够。

后果：代码更复杂，调试成本更高。

修法：等权最短步数先用 BFS。

错误示例：Dijkstra 看到邻居就更新Python
for v, w in graph[u]:
    dist[v] = dist[u] + w
    heapq.heappush(pq, (dist[v], v))

少了“是否更短”的比较，会用更差路径覆盖更好路径。松弛必须比较。

易混概念

compare

BFS vs Dijkstra

BFS：每条边代价相同，按层就是最短步数。

Dijkstra：边权非负但不相同，按当前最小距离扩展。

判断：等权用 BFS，非负权用 Dijkstra。

拓扑排序 vs DFS 遍历

拓扑排序：输出满足依赖的全局顺序。

DFS 遍历：从起点访问可达节点。

判断：题目问依赖能否完成或顺序，才用拓扑。

并查集 vs DFS 连通

并查集：多次合并和查询很方便。

DFS 连通：一次性遍历图也能找连通分量。

判断：动态合并多次查询，用并查集更顺手。
