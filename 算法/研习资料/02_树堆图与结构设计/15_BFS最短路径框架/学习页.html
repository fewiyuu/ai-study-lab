BFS 最短路径框架

这一页只讲等权图里的最少步数。题目问“走几步能到”，先看 BFS；题目一开始给了多个起点，就用多源 BFS；题目起点和终点都明确、分支又大，再考虑双向 BFS。上一页的 Dijkstra 管代价，这一页只管每条边都算一步时的层序推进。

读完先会的三件事

  • 能把“最少步数、每步代价相同、第一次到达目标”翻成 BFS。

  • 能把多源 BFS 和双向 BFS 分开，给 1091、994、127、433、542 这类题先选入口。

术语表

terms

BFS 的词不多，但每个词都要对应到一个动作。你要知道队列里装什么，层数什么时候加，visited 什么时候写，才不会把层序走成乱序。

对象 它表示什么 入队前后怎么变 读什么 写什么 错了的症状
queue 当前层和下一层的候选点 先入先出，层序推进 当前层剩余哪些点 把邻居放到队尾 层顺序乱，最短步数失真
step / dist 从起点到这个点的步数 下一层 = 当前层 + 1 父节点的步数 新节点的步数 全局答案差一层
visited 这个点有没有被安排过 通常在入队时就写下 是否已经入过队 避免重复入队 队列被同一点塞满
layer 当前这一轮弹出的所有点 每轮处理完再加一 这一层共有多少点 下一层的起点 第一次到目标时层数算错
$$ step[nxt] = step[cur] + 1 $$

这里的 `cur` 是当前层弹出的点，`nxt` 是它扩展出的下一层点。BFS 能直接给出最少步数，靠的就是这一句按层递增。

把这句和上面的表一起看，会更清楚：queue 负责顺序，visited 负责去重，step 负责记层号。它们分工不同，但目标一样，都是让“第一次到达”对应到最短步数。

单源 BFS：按层走

chain

还是看同一张 3x3 迷宫图：从左上角 `S` 出发，目标是右下角 `T`。只要每一步都能走上下左右，BFS 就会先把近的点处理完，再碰到远的点。你不用猜路线，队列已经把路线顺序排好了。

层数 队列内容 这一层确认了什么 visited 这时起什么作用
0 [(0,0)] 起点步数是 0 先把起点标掉，避免回头再进来
1 [(0,1), (1,0)] 离起点 1 步的格子被找出来 两个邻居都只入队一次
2 [(1,1)] 中间的交汇点出现了 同一个点从两个父节点来，只收一次
3 [(1,2), (2,1)] 离目标更近的边缘层展开 把会重复出现的格子挡在外面
4 [(2,2)] 第一次碰到目标，答案就是 4 不用再往更远层跑

这一轮走读要看清一件事：BFS 先把当前层处理完，再进入下一层。第一次碰到目标时，层数才是最短步数。

一轮走读

walkthrough

把同一张图放进三种入口里走一遍，差异就会很直观。单源 BFS 看近层，接力的是队列；多源 BFS 看最近源点，接力的是同一层的所有起点；双向 BFS 看两边何时碰头，接力的是两个前沿。入口换了，层序规则本身不变。

步骤 单源 BFS 多源 BFS 双向 BFS
初始化 一个 `S` 入队，步数 0 所有源点一起入队，步数都 0 起点队列和终点队列都放好
第一轮 先处理与 `S` 相邻的一层 所有源点同时向外扩一层 两边都各扩一步，优先扩小的一边
第二轮 更远的一层才开始进队 最近的格子先收到距离 两边的访问集合可能开始重叠
停止 第一次弹出 `T` 最后一个未覆盖点被填上 两个前沿相遇
层序主循环
from collections import deque

def bfs_levels(start):
    q = deque([start])
    visited = {start}
    step = 0

    while q:
        size = len(q)
        for _ in range(size):
            cur = q.popleft()
            if cur == target:
                return step
            for nxt in neighbors(cur):
                if nxt in visited:
                    continue
                visited.add(nxt)
                q.append(nxt)
        step += 1

这段循环里最重要的是 `size = len(q)`。它把当前层锁住，让这一轮只处理同一层的点；如果少了这句，层数和队列就会互相干扰，最短步数会算歪。

看见什么 先问什么 先写什么 下一步
一个起点 是否是普通最短步数 单源队列 按层扩散
一批源点 是否同时扩散 把所有源点放入第一层 看最近源点到各处的距离
起点和终点都明确 分支是否很多 双向前沿 等两边相遇后收口

核心模板

template

把 BFS 写稳，靠的是四件事：队列、visited、层数、邻居生成。只要这四件事的顺序对了，网格题、图题、字符串变换题都能接上。

BFS 最短步数模板
from collections import deque

def shortest_steps(start, is_goal, neighbors):
    q = deque([start])
    visited = {start}
    steps = {start: 0}

    while q:
        cur = q.popleft()
        if is_goal(cur):
            return steps[cur]

        for nxt in neighbors(cur):
            if nxt in visited:
                continue
            visited.add(nxt)
            steps[nxt] = steps[cur] + 1
            q.append(nxt)

    return -1
代码片段 读什么 写什么 如果顺序错了会怎样
`q = deque([start])` 起点是谁 队列初值 没有入口，整轮 BFS 起不来
`visited = {start}` 起点是否已经占位 先把起点记住 起点可能被邻居反复塞回队列
`steps[nxt] = steps[cur] + 1` 父节点步数 下一层步数 答案少一层或多一层
`if is_goal(cur): return ...` 当前点是否为目标 直接返回最短步数 如果拖到后面才判断，容易多跑一层

`neighbors(cur)` 可以换成四方向网格、字母替换、字符串改一位、图邻接表。核心没变，变的是邻居怎么生成。

在等权 BFS 里，`steps` 和 `dist` 的角色几乎一样：都在记录从起点到当前点的层数。它们只该在第一次确认这个点时写入，后来的父节点如果给出更晚的路径，就直接跳过。

渐进例子

examples

先从单源扩散看清“层”的意思，再把起点加多、把搜索改成两头同时走。这样你会知道 BFS 的变化只在入口，不在核心规则。

多源 BFS：一开始就把所有源点放进去

像 994、542 这类题，题目一开始就告诉你有一批源点。它们本来就在同一层，所以队列里也要一起排在第一层。你只要把所有源点都设成步数 0，后面的扩散就自然会按层推进。

题型 初始队列 第一轮之后 最常见用途
994 腐烂橘子 所有已经腐烂的格子 同时向周围扩散一层 算最少分钟数
542 01 矩阵 所有 0 每个 1 都收到最近 0 的距离 算最近距离

for s in sources:

双向 BFS：两头都明确时，先让波前相遇

像 127、433 这类题，起点和终点都很明确，分支又多。双向 BFS 的做法是从两边同时长波前，优先扩小的那一边，等两个前沿相遇就停。这样比单头一直往外铺更省。

它把搜索切成两半，让两边的前沿朝中间长。哪一边更小，就先扩哪一边；这样每轮新增的状态更少，碰头也更早。某个点第一次同时出现在两边的访问集合里，最短层数就已经接上了。

前向队列 后向队列 会合条件 为什么省
从 start 出发 从 target 出发 某个点同时出现在两边已访问集合里 每边都只铺一半路程

双向 BFS 只是把入口换成了两个端点。它仍然靠层数推进，只不过中间的会合点比单头终点更早出现。

一轮追踪

trace

把 BFS 写稳，先追一轮纸面状态。下面还是同一张 3x3 迷宫图，只看一条最短层序：先记队列里是谁，再看 visited 什么时候落字，最后看 step 什么时候往前走。

时刻 queue visited step 这一刻确认了什么
初始化 [S] {S} S = 0 起点已经占位，第一层准备开始
弹出 S 后 [右、下] {S, 右, 下} 右 = 1， 下 = 1 第一圈邻居都只进队一次
再扩一层 [中间点、边缘点] 继续增加新点 新点 = 2 同一层的父节点可以不同，层号仍然统一
碰到 T 目标被弹出 目标已记过 当前层就是答案 第一次出队时，最短步数已经定了

为什么先写 visited

同一个点可能从两个父节点同时被看见。只要它还没入队，就还没有被这一层真正占住；一旦入队并标记，后面的父节点就应该直接跳过。这样做，队列只保存一份候选，层数也不会被重复节点拖歪。

这轮追踪有个固定顺序：先看 queue，确认这一层是谁；再看 visited，确认去重有没有发生；最后看 step，确认层号是不是跟着父节点往下走。三列对齐后，最短步数就容易手算出来。

收口点

stop

同样是 BFS，不同入口的收口点不一样。单源 BFS 看目标什么时候第一次出队；多源 BFS 看扩散什么时候没有新点可写；双向 BFS 看两边前沿什么时候碰上。收口点写错，最短层数就会晚一拍。

入口 什么时候停 停下来时读什么 最容易写错什么
单源 BFS 目标第一次出队 当前层号 等整轮结束再返回
多源 BFS 队列为空或没有新点可写 最后一次扩散的层号 漏记最后一层
双向 BFS 两边访问集合出现交点 两边各自的层数 只看一边的深度

为什么收口点很重要

很多人写 BFS 时，前面的扩散是对的，错在最后一步。比如单源题如果等整轮跑完才停，答案会多一层；多源题如果没把最后一批新点记进结果，答案会少一层。把收口点单独拎出来，这种错误就能提前挡住。

单源 BFS 里，“第一次看见目标”和“第一次弹出目标”不是同一个动作。前者只是说明目标已经进了下一层候选，后者才说明更短的层都处理完了。多源题也一样，最后一层的点可以先进入队列，但答案要等这一层处理完才算稳定。

如果你把停止条件写早了，迷宫题会少算一层，橘子题会漏掉最后一次扩散，双向 BFS 则可能在两边还没真正接上时就提前收口。把停点和入口绑在一起看，比单独背“BFS 最短”更可靠。

典型案例

cases

下面这一段专门把多源 BFS 讲透。你在 994 腐烂橘子和 542 01 矩阵里看到的是同一个扩散框架，只是输入和输出不同。

先把两个题放在一起看，会更清楚。994 关注的是“多久全部腐烂”，542 关注的是“每个位置离最近 0 有多远”。共同点是：第一层由一批点组成；不同点是：一个返回时间，一个返回整张距离表。这样一来，队列、层数和 visited 的写法就会完全对应到你要交的答案。

案例一：腐烂橘子

初始状态里，所有已经腐烂的橘子就是源点。它们同时发出扩散，下一分钟里能感染的格子都在第二层。你关心的是最后一个新腐烂的橘子是在第几分钟被染上。

分钟 当前队列 新变化 答案在记录什么
0 所有坏橘子 准备向四周扩散 初始源点数
1 第一圈好橘子 一批一起变坏 第一层扩散结果
2 第二圈好橘子 继续向外一层 第二层扩散结果
结束 队列空了 没有新橘子可染 最后一层的分钟数
腐烂橘子模板
from collections import deque

def orangesRotting(grid):
    m, n = len(grid), len(grid[0])
    q = deque()
    fresh = 0
    for i in range(m):
        for j in range(n):
            if grid[i][j] == 2:
                q.append((i, j))
            elif grid[i][j] == 1:
                fresh += 1

    minutes = 0
    while q and fresh:
        for _ in range(len(q)):
            x, y = q.popleft()
            for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
                nx, ny = x + dx, y + dy
                if 0 <= nx < m and 0 <= ny < n and grid[nx][ny] == 1:
                    grid[nx][ny] = 2
                    fresh -= 1
                    q.append((nx, ny))
        minutes += 1
    return minutes if fresh == 0 else -1

这里有两个关键动作：一是把所有坏橘子都放进第一层，二是每一轮处理完再把分钟数加一。少了前者，多源入口就没立住；少了后者，分钟数就会错。

案例二：01 矩阵

542 01 矩阵也是同一件事。这一次源点换成了所有 `0`，不再是坏橘子。你要给每个 `1` 填上它到最近 `0` 的距离，所以从一开始就要把所有 `0` 当成第一层。

状态 队列里是谁 距离数组怎么长 为什么不会乱
初始 所有 0 0 的位置步数都为 0 源点先占位
第一层 离 0 最近的 1 第一次收到距离 1 离得最近的先被写入
第二层 更远的 1 第一次收到距离 2 后面的层只能更大
01 矩阵模板
from collections import deque

def updateMatrix(mat):
    m, n = len(mat), len(mat[0])
    q = deque()
    dist = [[-1] * n for _ in range(m)]
    for i in range(m):
        for j in range(n):
            if mat[i][j] == 0:
                q.append((i, j))
                dist[i][j] = 0

    while q:
        x, y = q.popleft()
        for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
            nx, ny = x + dx, y + dy
            if 0 <= nx < m and 0 <= ny < n and dist[nx][ny] == -1:
                dist[nx][ny] = dist[x][y] + 1
                q.append((nx, ny))
    return dist

这两个题的共性

这两题都从一批点同时出发。你把它们看成“同时扩散”，就不会再把它们和单源最短步数混在一起。

这两个题的差别

腐烂橘子最后要返回时间，01 矩阵最后要返回每个位置的距离表。前者关注最后一层什么时候出现，后者关注每个点第一次收到多源距离时是多少。

如果你把这两个案例想明白了，多源 BFS 就不再只是“多个起点一起入队”这句话，而会变成你手里能直接写出的队列、层数和距离表。练习时先手推一轮：源点是谁，第一层是谁，第二层是谁，最后答案记的是时间还是距离。这个动作做顺了，后面的模板只是把你已经想明白的过程翻成代码。

精修补充

refine

真正写题时，BFS 出错常常出在三个小点：起点放错、visited 标晚、目标停得太晚。把这三个点逐个固定，代码就会稳很多。

检查点 正确写法 常见错法 会出现的症状
起点 队列一开始就放进所有源点 先空队列，等循环里再补 第一层丢失，答案整体偏一层
visited 在入队时标记 出队后才标记 重复节点塞满队列
目标 到目标就返回当前层数 等整轮都跑完才停 多跑一层或错过最短答案
网格邻居生成
def neighbors(grid, x, y):
    m, n = len(grid), len(grid[0])
    for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
        nx, ny = x + dx, y + dy
        if 0 <= nx < m and 0 <= ny < n and grid[nx][ny] != '#':
            yield nx, ny

这段邻居生成是网格 BFS 的底座。你只要保证它只吐出合法格子，后面的队列、层数、visited 就能继续工作；如果这里把墙也放进去，整条搜索会被脏节点带偏。

边界与复杂度

boundary

这一页先锁等权步数，不把别的最短路框架混进来。只要题目一变成“每条边代价不同”，就先停下来看别的算法。

算法 边权条件 核心状态 适合什么 现在先不追什么
BFS 每一步都算 1 queue + layer + visited 最少步数、最少层数 加权最短路
Dijkstra 边权非负且不相同也行 dist + 小根堆 最小代价 等权扩散的细节
0-1 BFS 只有 0 或 1 deque + 双端入队 特殊的二值权值 这一页不展开
DFS 不关心层序 栈 / 递归 枚举、连通块、回溯 最短步数
$$ \text{最短步数} = \text{第一次从队列里弹出目标时的层数} $$

这个式子只在“每一步代价相同”时成立。只要代价不一样，这一页就该停住，换别的框架。

边界提示

0-1 BFS、Bellman-Ford、A*、最小生成树都先放下。它们和 BFS 都在图上工作，但今天只追“等权最短步数”这条线。

BFS 和加权最短路的分叉

compare

这一段专门做一个对照：同一张小图，如果边权都一样，BFS 看层数就够了；如果边权不一样，层数和代价就会分家，必须换到 Dijkstra。先看明白分叉点，后面才不会把“最短步数”和“最小代价”混成一件事。

看这张图：`0→1(1), 1→2(1), 2→3(1), 0→3(10)`。如果你只按步数，BFS 会先弹出 `3`，因为它是一条 1 步直达路；如果你问的是代价，`0→1→2→3` 才是总权重 3 的最短路。这里代码本身未必错，问题在于题目要的是代价，不是步数。

路径 步数 总代价 BFS 会怎么想 Dijkstra 会怎么想
0 → 3 1 10 先到，先出队，层数最短 代价太大，不该抢最优
0 → 1 → 2 → 3 3 3 层数更长，不会先于直达路出现 代价更小，应当被保留
错误的 BFS 代价写法
# 错：把“边走一条”当成“代价加 1”
for nxt, w in graph[cur]:
    if nxt not in visited:
        visited.add(nxt)
        dist[nxt] = dist[cur] + 1
        q.append(nxt)
正确的停手信号
# 对：如果题目要的是代价，就把问题交给上一页的 Dijkstra
if any(w != 1 for _, w in graph[cur]):
    return "换到 Dijkstra"

# 只有每步都一样时，BFS 的层数才等于最短步数

再做一个最小判题：`0→1(1), 1→2(1), 2→3(1), 0→3(10)`。按 BFS 记层数，`0→3` 这条路会先到；按代价记，`0→1→2→3` 才是最优。这个对照一写出来，你就知道题目问的是层数还是总代价。

草稿记录 先用什么 你会先看到什么 如果写错会怎样
只看步数 BFS 队列 1 步直达路先弹出 最短代价被误判
看总代价 Dijkstra / 小根堆 3 代价路径保留 不会被 10 代价直达路带偏

做题时先把“层数”和“代价”写成两列，再决定框架。看到 `time / cost / fee` 这些词，先停在这一格，不要直接把 BFS 当默认答案。

这就是这页和上一页的分界线：上一页管非负权代价，这一页管等权层数。看到 `time / fee / cost / probability` 这类词时，先确认自己是在算步数，还是在算总代价。

最后的判断句

层数解决的是“走几步”，代价解决的是“花多少”。题目如果只问前者，BFS 很轻；只要它开始问后者，就该换框架。把这句话刻在入口上，很多题就不会走偏。

如果你拿这张图去做练习，先不要写完整代码，只做一个纸面判断：第一步会不会先弹出直达路，最后答案要的是步数还是代价。这个判断做对了，后面就不会把 BFS 当成万能模板。

做纸面验算时，顺手把“先出队的是哪条路、为什么它先出队、代价是否更小”写成三行。BFS 只看层数，Dijkstra 只看代价；这三行能把两者分开，也能直接告诉你练习题应该先看哪一条边。

真正的练习动作很简单：拿出 1091、994、127、433、542 中任意一题，先在草稿上写出队列初值、第一层、停止条件三件事，再决定要不要继续。你只要把这三件事写出来，题目是单源、多源，还是双向，就会自己跳出来。

你先写什么 对应检查点 为什么值得先写
队列初值 入口是不是单源、多源，还是双向 起点放对了，后面层序才不会漂
第一层 这一轮到底在扩谁 能直接看出 BFS 会不会先碰到直达路
停止条件 什么时候该收口 可以防止把最短答案多跑一层

失败实验

lab

把常见故障做成可切换的实验，比只看错误描述更快。先看症状，再回到自己的代码里找同类问题。

同一个点会反复进队列，队列长度突然变大。
症状 常见错误 先看什么 下一步怎么修
队列突然爆长 visited 放晚了 是不是出队后才标记 在入队时就记住
答案偏一层 层数和轮次没对齐 这一轮处理完有没有加 1 按层计数，不按点计数
多源题离最近源点距离不对 初始化时漏放源点 第一层是不是所有源点 把所有源点一起入队
加权题过不了 把非等权图当等权图 题目是否明确每一步都一样 改用 Dijkstra 或其他框架
错误版与修正版
# 错：出队后才标记，邻居会把同一个点重复塞进队列
cur = q.popleft()
if cur in visited:
    continue
visited.add(cur)

# 对：入队时就标记，队列里永远只保留一次
for nxt in neighbors(cur):
    if nxt not in visited:
        visited.add(nxt)
        q.append(nxt)

这个实验区的目标很直接：你以后只要看到队列爆长、层数偏一层，或者多源题像单源题，就先回到这里检查 visited、层数和初始队列。

常见错误

mistakes

这页最常见的错，主要出在 visited、层数和边权判断上。错一步，后面整条层序都会偏。

症状 常见原因 先检查什么 修法
队列里同一个点反复出现 visited 标记太晚 是不是出队后才补 visited 在入队时就标记
答案比标准值多一层或少一层 层数没和队列轮次同步 一轮处理完后有没有加 1 按层处理，按层加数
加权题写成 BFS 还过不了 把非等权图当成等权图 题目是不是明确说每步代价一样 换 Dijkstra 或别的框架
多源题只从一个点开始扩散 初始队列少放了源点 题目是不是一开始就给了一批源点 把所有源点一起放进第一层
visited 标记时机对照
# 对：入队时就标记，避免同一个点重复进队列
if nxt not in visited:
    visited.add(nxt)
    q.append(nxt)

# 错：出队后才补 visited，队列容易被重复节点塞满
cur = q.popleft()
if cur in visited:
    continue
visited.add(cur)

易混概念

confusions

这页先不追 0-1 BFS、Bellman-Ford、A*、最小生成树。它们都和图有关，但今天只要把单源、多源、双向这三种 BFS 入口分清，下一步就够用了。

入口 队列初值 核心判断 适合解决什么 最容易混成谁
单源 BFS 一个起点 第一次到目标的层数 最少步数 DFS / Dijkstra
多源 BFS 多个起点一起入队 最近源点的层数 最近距离、同时扩散 单源 BFS
双向 BFS 起点队列 + 终点队列 两边前沿有没有碰上 分支大、起终点都明确 普通单源 BFS
BFS vs Dijkstra 同样都能找最短路 边权是不是全一样 一个看层数，一个看代价 把所有最短路都当 BFS

真正要记的是入口。起点一批就多源，两头都明确就双向，边都算一步就单源 BFS；一旦代价变了，今天这一页就停。

题目迁移

transfer

把题号当落点，不把题面原文搬进来。你现在要做的是先圈起点、终点、是否多源，再决定队列初值和停止条件。

题号 更像哪类 BFS 第一步先写什么 最常见的错位
1091 单源 BFS 先把一个起点放进队列 把它写成 Dijkstra 或 DFS
994 多源 BFS 把所有初始源点一起入队 只从一个源点开始扩散
127 双向 BFS 从起点和终点两边同时开波前 只开一头，分支爆掉
433 双向 BFS 先准备两边的访问集合 把它当成普通单源 BFS
542 多源 BFS 先把所有 0 放进第一层 只记住一个最近源点

做题时先写三件事：一个源点、多个源点、两头都明确。把题面翻成这三种入口，再决定队列初值、层数和停止条件，BFS 的框架就不会乱。
