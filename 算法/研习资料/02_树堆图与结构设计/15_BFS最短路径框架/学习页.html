BFS 最短路径框架

这一页只讲等权图里的最少步数。题目问“走几步能到”，先看 BFS；题目一开始给了多个起点，就用多源 BFS；题目起点和终点都明确、分支又大，再考虑双向 BFS。上一页的 Dijkstra 管代价，这一页只管每条边都算一步时的层序推进。

学习目标

goal

主线地图

map

先把一张小迷宫固定住，再看三种扩散方式怎么换入口。这样你会先看队列里放的是谁，再决定怎么扩散。

入口队列初值什么时候停典型落点

术语表

terms

BFS 的词不多，但每个词都要对应到一个动作。你要知道队列里装什么，层数什么时候加，visited 什么时候写，才不会把层序走成乱序。

$$ step[nxt] = step[cur] + 1 $$

这里的 cur 是当前层弹出的点，nxt 是它扩展出的下一层点。BFS 能直接给出最少步数，靠的就是这一句按层递增。

单源 BFS：按层走

chain

还是看同一张 3x3 迷宫图：从左上角 S 出发，目标是右下角 T。只要每一步都能走上下左右，BFS 就会先把近的点处理完，再碰到远的点。你不用猜路线，队列已经把路线顺序排好了。

一轮走读

walkthrough

把同一张图放进三种入口里走一遍，差异就会很直观。单源 BFS 看近层，接力的是队列；多源 BFS 看最近源点，接力的是同一层的所有起点；双向 BFS 看两边何时碰头，接力的是两个前沿。入口换了，层序规则本身不变。

代码
from collections import deque

def bfs_levels(start):
 q = deque([start])
 visited = {start}
 step = 0

 while q:
 size = len(q)
 for _ in range(size):
 cur = q.popleft()
 if cur == target:
 return step
 for nxt in neighbors(cur):
 if nxt in visited:
 continue
 visited.add(nxt)
 q.append(nxt)
 step += 1

这段循环里最重要的是 size = len(q)。它把当前层固定住，让这一轮只处理同一层的点；如果少了这句，层数和队列就会互相干扰，最短步数会算歪。

看见什么先问什么先写什么下一步
一个起点是否是普通最短步数单源队列按层扩散
一批源点是否同时扩散把所有源点放入第一层看最近源点到各处的距离
起点和终点都明确分支是否很多双向前沿等两边相遇后收尾

核心模板

template

把 BFS 写稳，靠的是四件事：队列、visited、层数、邻居生成。只要这四件事的顺序对了，网格题、图题、字符串变换题都能接上。

代码
from collections import deque

def shortest_steps(start, is_goal, neighbors):
 q = deque([start])
 visited = {start}
 steps = {start: 0}

 while q:
 cur = q.popleft()
 if is_goal(cur):
 return steps[cur]

 for nxt in neighbors(cur):
 if nxt in visited:
 continue
 visited.add(nxt)
 steps[nxt] = steps[cur] + 1
 q.append(nxt)

 return -1

在等权 BFS 里，steps 和 dist 的角色几乎一样：都在记录从起点到当前点的层数。它们只该在第一次确认这个点时写入，后来的父节点如果给出更晚的路径，就直接跳过。

渐进例子

examples

先从单源扩散看清“层”的意思，再把起点加多、把搜索改成两头同时走。这样你会知道 BFS 的变化只在入口，不在核心规则。

像 994、542 这类题，题目一开始就告诉你有一批源点。它们本来就在同一层，所以队列里也要一起排在第一层。你只要把所有源点都设成步数 0，后面的扩散就自然会按层推进。

代码
q = deque()
visited = set()
steps = {}

for s in sources:

 steps[s] = 0

双向 BFS：两头都明确时，先让波前相遇

像 127、433 这类题，起点和终点都很明确，分支又多。双向 BFS 的做法是从两边同时长波前，优先扩小的那一边，等两个前沿相遇就停。这样比单头一直往外铺更省。

前向队列后向队列会合条件为什么省
从 start 出发从 target 出发某个点同时出现在两边已访问集合里每边都只铺一半路程

一轮追踪

trace

把 BFS 写稳，先追一轮纸面状态。下面还是同一张 3x3 迷宫图，只看一条最短层序：先记队列里是谁，再看 visited 什么时候落字，最后看 step 什么时候往前走。

为什么先写 visited

同一个点可能从两个父节点同时被看见。只要它还没入队，就还没有被这一层真正占住；一旦入队并标记，后面的父节点就应该直接跳过。这样做，队列只保存一份候选，层数也不会被重复节点拖歪。

这轮追踪有个固定顺序：先看 queue，确认这一层是谁；再看 visited，确认去重有没有发生；最后看 step，确认层号是不是跟着父节点往下走。三列对齐后，最短步数就容易手算出来。

收尾点

stop

同样是 BFS，不同入口的收尾点不一样。单源 BFS 看目标什么时候第一次出队；多源 BFS 看扩散什么时候没有新点可写；双向 BFS 看两边前沿什么时候碰上。收尾点写错，最短层数就会晚一拍。

为什么收尾点很重要

很多人写 BFS 时，前面的扩散是对的，错在最后一步。比如单源题如果等整轮跑完才停，答案会多一层；多源题如果没把最后一批新点记进结果，答案会少一层。把收尾点单独拎出来，这种错误就能提前挡住。

单源 BFS 里，“第一次看见目标”和“第一次弹出目标”不是同一个动作。前者只是说明目标已经进了下一层候选，后者才说明更短的层都处理完了。多源题也一样，最后一层的点可以先进入队列，但答案要等这一层处理完才算稳定。

如果你把停止条件写早了，迷宫题会少算一层，橘子题会漏掉最后一次扩散，双向 BFS 则可能在两边还没真正接上时就提前收尾。把停点和入口绑在一起看，比单独背“BFS 最短”更可靠。

典型案例

cases

下面这一段专门把多源 BFS 讲透。你在 994 腐烂橘子和 542 01 矩阵里看到的是同一个扩散框架，只是输入和输出不同。

案例一：腐烂橘子

分钟当前队列新变化答案在记录什么
0所有坏橘子准备向四周扩散初始源点数
1第一圈好橘子一批一起变坏第一层扩散结果
2第二圈好橘子继续向外一层第二层扩散结果
结束队列空了没有新橘子可染最后一层的分钟数
代码
from collections import deque

def orangesRotting(grid):
 m, n = len(grid), len(grid[0])
 q = deque()
 fresh = 0
 for i in range(m):
 for j in range(n):
 if grid[i][j] == 2:
 q.append((i, j))
 elif grid[i][j] == 1:
 fresh += 1

 minutes = 0
 while q and fresh:
 for _ in range(len(q)):
 x, y = q.popleft()
 for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
 nx, ny = x + dx, y + dy
 if 0 <= nx < m and 0 <= ny < n and grid[nx][ny] == 1:
 grid[nx][ny] = 2
 fresh -= 1
 q.append((nx, ny))
 minutes += 1
 return minutes if fresh == 0 else -1

这里有两个关键动作：一是把所有坏橘子都放进第一层，二是每一轮处理完再把分钟数加一。少了前者，多源入口就没立住；少了后者，分钟数就会错。

案例二：01 矩阵

状态队列里是谁距离数组怎么长为什么不会乱
初始所有 00 的位置步数都为 0源点先占位
第一层离 0 最近的 1第一次收到距离 1离得最近的先被写入
第二层更远的 1第一次收到距离 2后面的层只能更大
代码
from collections import deque

def updateMatrix(mat):
 m, n = len(mat), len(mat[0])
 q = deque()
 dist = [[-1] * n for _ in range(m)]
 for i in range(m):
 for j in range(n):
 if mat[i][j] == 0:
 q.append((i, j))
 dist[i][j] = 0

 while q:
 x, y = q.popleft()
 for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
 nx, ny = x + dx, y + dy
 if 0 <= nx < m and 0 <= ny < n and dist[nx][ny] == -1:
 dist[nx][ny] = dist[x][y] + 1
 q.append((nx, ny))
 return dist

这两个题的差别

腐烂橘子最后要返回时间，01 矩阵最后要返回每个位置的距离表。前者关注最后一层什么时候出现，后者关注每个点第一次收到多源距离时是多少。

如果你把这两个案例想明白了，多源 BFS 就不再只是“多个起点一起入队”这句话，而会变成你手里能直接写出的队列、层数和距离表。练习时先手推一轮：源点是谁，第一层是谁，第二层是谁，最后答案记的是时间还是距离。这个动作做顺了，后面的模板只是把你已经想明白的过程翻成代码。

精修补充

refine

真正写题时，BFS 出错常常出在三个小点：起点放错、visited 标晚、目标停得太晚。把这三个点逐个固定，代码就会稳很多。

代码
def neighbors(grid, x, y):
 m, n = len(grid), len(grid[0])
 for dx, dy in ((1,0), (-1,0), (0,1), (0,-1)):
 nx, ny = x + dx, y + dy
 if 0 <= nx < m and 0 <= ny < n and grid[nx][ny] != '#':
 yield nx, ny

这段邻居生成是网格 BFS 的底座。你只要保证它只吐出合法格子，后面的队列、层数、visited 就能继续工作；如果这里把墙也放进去，整条搜索会被脏节点带偏。

边界与复杂度

boundary

这一页先锁等权步数，不把别的最短路框架混进来。只要题目一变成“每条边代价不同”，就先停下来看别的算法。

$$ \text{最短步数} = \text{第一次从队列里弹出目标时的层数} $$

这个式子只在“每一步代价相同”时成立。只要代价不一样，这一页就该停住，换别的框架。

BFS 和加权最短路的分叉

compare

这一段专门做一个对照：同一张小图，如果边权都一样，BFS 看层数就够了；如果边权不一样，层数和代价就会分家，必须换到 Dijkstra。先看明白分叉点，后面才不会把“最短步数”和“最小代价”混成一件事。

路径步数总代价BFS 会怎么想Dijkstra 会怎么想
0 → 3110先到，先出队，层数最短代价太大，不该抢最优
0 → 1 → 2 → 333层数更长，不会先于直达路出现代价更小，应当被保留

for nxt, w in graph[cur]:

代码
# 对：如果题目要的是代价，就把问题交给上一页的 Dijkstra
if any(w != 1 for _, w in graph[cur]):
 return "换到 Dijkstra"

# 只有每步都一样时，BFS 的层数才等于最短步数

再做一个最小判题：0→1(1), 1→2(1), 2→3(1), 0→3(10)。按 BFS 记层数，0→3 这条路会先到；按代价记，0→1→2→3 才是最优。这个对照一写出来，你就知道题目问的是层数还是总代价。

草稿记录先用什么你会先看到什么如果写错会怎样
只看步数BFS 队列1 步直达路先弹出最短代价被误判
看总代价Dijkstra / 小根堆3 代价路径保留不会被 10 代价直达路带偏

这就是这页和上一页的分界线：上一页管非负权代价，这一页管等权层数。看到 time / fee / cost / probability 这类词时，先确认自己是在算步数，还是在算总代价。

失败实验

lab

把常见故障做成可切换的实验，比只看错误描述更快。先看症状，再回到自己的代码里找同类问题。

代码

这个实验区的目标很直接：你以后只要看到队列爆长、层数偏一层，或者多源题像单源题，就先回到这里检查 visited、层数和初始队列。

常见错误

mistakes

这页最常见的错，主要出在 visited、层数和边权判断上。错一步，后面整条层序都会偏。

代码

 visited.add(nxt)
 q.append(nxt)

# 错：出队后才补 visited，队列容易被重复节点塞满

visited.add(cur)
