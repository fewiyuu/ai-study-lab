DFS 回溯
排列组合子集回溯不是神秘写法，而是把一棵选择树走完：做选择、进入下一层、撤销选择。

遇到“列出所有方案”的题，先问三件事：路径存什么，下一层从哪里选，什么时候收集答案。
dfs backtracking

把选择树走稳，而不是背题型

排列、组合、子集都在枚举方案，但它们枚举的对象不同。子集关心“选或不选”，组合关心“选哪些但不关心顺序”，排列关心“顺序也算不同”。这三个差别会直接落到代码里的 start、used 和收集答案的位置。

回溯代码的核心动作只有三步：把当前选择加入路径，递归处理下一层，再把选择撤销。撤销不是多余动作，它保证下一条分支看到的是干净状态。学完本页，你应能读懂回溯模板，能判断一道题该用子集、组合还是排列模型，也能定位重复答案和漏答案的原因。

主线地图

mental model

回溯题可以先画成选择树。树上的一条根到某个节点的路径，就是当前已经做出的选择；一个节点的孩子，就是下一步可选的候选项。DFS 负责走到树的深处，回溯负责从一个分支退回去，继续尝试旁边的分支。

撤销选择还原现场

子集

每个元素都可以选或不选。常见写法是在每个节点都收集一次路径，因为任意长度都能成为一个子集。

判断：结果里空集、单元素、多元素都要出现。

组合

只关心选出来的元素集合，不关心顺序。为了避免重复，下一层只能从当前元素后面继续选，通常使用 start。

判断：[1,2] 和 [2,1] 不能同时出现。

排列

顺序不同就是不同答案。每一层都可以从头扫描所有元素，但已经放进路径的元素不能再用，通常使用 used。

判断：[1,2] 和 [2,1] 都是合法答案。

术语表

terms
术语直觉准确说法容易误解的地方
DFS沿着一条路先走到底。深度优先遍历，可以遍历树、图或隐式状态树。DFS 不一定会撤销状态；回溯才强调选择后的还原。
回溯试一个选择，不行或试完就退回。在 DFS 枚举中维护路径状态，进入分支前修改，离开分支后恢复。把回溯当成固定代码片段，忘记先定义状态。
路径 track当前已经选了什么。从根到当前递归节点的选择序列。直接把同一个列表放进答案，后续撤销会把答案也改掉。
选择列表下一步还能试什么。当前递归层允许尝试的候选集合。组合题和排列题的选择列表不同，不能混用。
start从哪个下标继续向后选。控制组合和子集题不回头选，避免顺序重复。把 start 用在排列题里会漏掉合法顺序。
used哪些元素已经放进路径。排列题中防止同一个位置被重复使用。把 used 当成全局答案去重集合，会误删分支。

知识链路

chain

一道回溯题开始时，不要急着写递归函数。先把题目翻译成三个问题：路径里放什么，下一层能从哪些候选里选，什么时候把路径加入答案。只要这三个问题清楚，代码通常就是模板的变形。

子集题的收集点最宽松。因为任意长度的路径都算一个答案，进入每个节点时都可以把当前路径复制进结果。组合题会多一个长度限制，比如选满 k 个才收集。排列题则要求路径长度等于数组长度，且每个元素位置只能用一次。

回溯里最容易被忽略的是“撤销选择”。递归调用会把控制权交给下一层，下一层可能继续修改同一个 track。如果离开分支时不撤销，旁边分支会继承错误路径，最终出现脏答案、漏答案或重复答案。

本页的判断句

  • 问所有方案，先画选择树。
  • 组合和子集常用 start，排列常用 used。
  • 答案里要复制路径，不要保存同一个列表引用。
  • 有重复元素时，先排序，再在同一层跳过重复选择。

回溯模板

template

模板不是要背的句子，而是一套状态更新顺序。每次循环选择一个候选，把它加入路径，再递归到下一层。递归回来后，必须撤销这次选择。你以后检查回溯代码，可以按这个顺序逐行问：这一行读取什么，修改什么，下一层依赖什么。

正确示例：通用回溯骨架Python
def backtrack(路径, 选择列表):
    if 满足收集条件:
        res.append(路径.copy())
        # 是否 return 取决于后面还要不要继续向下找

    for 选择 in 选择列表:
        if 这个选择不合法:
            continue
        路径.append(选择)
        backtrack(路径, 下一层选择列表)
        路径.pop()

路径.copy() 很重要。路径 是递归过程中反复修改的列表，不复制就会让结果指向同一个可变对象。

错误示例：忘记复制路径Python
res.append(track)   # 错误：后续 track.pop() 会影响已经放进 res 的内容

三类渐进例题

examples
选择树观察

切换题型，看同样的数组 [1,2,3] 在不同模型下怎样展开。重点看下一层还能选谁。

子集：每个节点都可收集

路径 []、[1]、[1,2]、[1,2,3] 都是答案。下一层只向后选，避免 [2,1] 这种重复路径。

读完这个图，你应能说明为什么子集题常把收集答案放在递归函数开头。

例题一：子集Python
def subsets(nums):
    res, track = [], []

    def backtrack(start):
        res.append(track.copy())
        for i in range(start, len(nums)):
            track.append(nums[i])
            backtrack(i + 1)
            track.pop()

    backtrack(0)
    return res

这段代码里，start 表示下一层只能从哪个位置往后选。假设当前路径是 [1]，下一层从 2 开始选，就不会再回头选 1，也不会制造 [2,1]。答案在函数开头收集，是因为空路径和所有中间路径都算子集。

例题二：组合Python
def combine(n, k):
    res, track = [], []

    def backtrack(start):
        if len(track) == k:
            res.append(track.copy())
            return
        for x in range(start, n + 1):
            track.append(x)
            backtrack(x + 1)
            track.pop()

    backtrack(1)
    return res

组合题只在选满 k 个元素时收集答案。这里的 x + 1 表示后面的选择必须比当前数字大，所以 [1,2] 会出现，[2,1] 不会出现。读到组合题时，你应能先判断“不回头选”这件事是否成立。

例题三：排列Python
def permute(nums):
    res, track = [], []
    used = [False] * len(nums)

    def backtrack():
        if len(track) == len(nums):
            res.append(track.copy())
            return
        for i, x in enumerate(nums):
            if used[i]:
                continue
            used[i] = True
            track.append(x)
            backtrack()
            track.pop()
            used[i] = False

    backtrack()
    return res

排列题每一层都从头扫描，因为 [1,2] 和 [2,1] 都可能合法。used 只禁止同一个位置在同一条路径里重复使用。离开分支时既要 track.pop()，也要把 used[i] 改回 False。

去重与剪枝

dedupe

当输入里有重复元素时，答案重复通常不是因为递归多走了一层，而是同一层尝试了等价选择。解决办法是先排序，让相同元素挨在一起，然后在同一层跳过已经试过的值。注意，这里的“同一层”很重要，不能把所有重复值都粗暴删除。

正确示例：子集 II 的同层去重Python
def subsetsWithDup(nums):
    nums.sort()
    res, track = [], []

    def backtrack(start):
        res.append(track.copy())
        for i in range(start, len(nums)):
            if i > start and nums[i] == nums[i - 1]:
                continue
            track.append(nums[i])
            backtrack(i + 1)
            track.pop()

    backtrack(0)
    return res

i > start 表示“这不是本层第一个候选”。如果当前值和前一个值相同，说明本层已经用前一个相同值开过分支，再用当前值开分支会得到重复结构。这个判断不会禁止下一层使用重复值，所以 [1,2,2] 仍然能出现。

剪枝

提前发现某个分支不可能产生答案，就不继续递归。组合题里，如果剩余数字不够凑满 k 个，可以提前停。

去重

不同分支会生成同一个答案时，跳过等价分支。去重通常依赖排序和同层判断。

合法性检查

约束题常在进入下一层前检查选择是否可用。排列中的 used[i] 就是最小合法性检查。

常见错误：错因、后果和修法

debug

忘记撤销选择

错因：只看到递归向下，没有处理返回后的状态。

后果：旁边分支继承旧路径，答案混乱。

修法：每个 append 后面都要能对应一个 pop。

答案保存同一个列表

错因：不了解列表是可变对象。

后果：结果里很多答案最后都变成空列表或同一个内容。

修法：收集时使用 track.copy()。

组合题从 0 重新扫描

错因：把组合写成排列。

后果：出现 [1,2] 和 [2,1] 这种重复。

修法：用 start 控制只能向后选。

排列题误用 start

错因：看到回溯就套组合模板。

后果：漏掉顺序不同的合法排列。

修法：每层从头扫描，使用 used 防止同一位置重复。

重复元素去重过度

错因：把“同层跳过”误写成“所有重复都不能用”。

后果：漏掉 [1,2,2] 这类合法答案。

修法：排序后用 i > start and nums[i] == nums[i-1]。

结束条件写得太早或太晚

错因：没先确认什么才算一个答案。

后果：子集漏空集，组合收集未满长度的路径，排列漏最后一层。

修法：先用一句话写清收集条件，再写代码。

易混概念

compare

DFS vs 回溯

DFS 是遍历顺序，强调先深入。

回溯 是 DFS 枚举中的状态管理，强调修改后恢复。

判断：如果代码有“选择、递归、撤销”，它就是回溯味道很重的 DFS。

组合 vs 排列

组合 不关心顺序，常用 start。

排列 关心顺序，常用 used。

判断：[a,b] 和 [b,a] 是否都算答案。

剪枝 vs 去重

剪枝 是这个分支不可能有答案。

去重 是这个分支会得到已经出现过的答案。

判断：你跳过它是因为没用，还是因为重复。
