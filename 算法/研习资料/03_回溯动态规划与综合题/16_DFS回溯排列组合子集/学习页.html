主线地图

mental

子集

每个元素都可以选或不选，任意长度的路径都可能成为答案，所以常在每一层开头收集路径。

排列

顺序不同就是不同答案。每一层都要重新扫描候选，但已经在路径里的下标不能再用，所以常用 used。

子集每一层都可以收集从当前下标往后忘记复制路径，或把空集漏掉

组合选满 k 个再收集从 i + 1 往后把顺序当成答案的一部分

排列路径长度等于 n每一层都从头扫误把 start 套进排列

术语表

terms
术语直觉准确说法错了会怎样
track当前已经选了什么。从根到当前节点的选择序列。保存同一个列表引用，后面撤销会把答案一起改掉。
start下一层从哪里往后选。控制组合和子集不回头选。把 start 套到排列里，会漏掉顺序不同的答案。
used哪些下标已经在路径里。排列题里阻止同一位置重复使用。把它当全局去重集合，会误删合法分支。
同层去重这一层已经试过相同值了。排序后在同一层跳过重复候选。把所有重复都禁掉，会丢掉 [1,2,2] 这类合法答案。
撤销选择试完一个分支，回到干净状态。递归返回前恢复路径和标记。旁边分支会看到旧状态，答案会脏。

回溯模板

template

模板看的是状态顺序

先做选择，再递归下一层，返回后撤销选择。看清每一行读什么、写什么、依赖什么，就能判断这段代码是否在做回溯。

真正要盯的是状态是否连续：append 改了路径，pop 还原路径，copy 保存快照。少了任一环，兄弟分支看到的就是错的现场。

一眼判断这行在做什么

  • append 把当前候选写进路径。
  • backtrack(...) 把当前状态交给下一层。
  • pop 把路径恢复干净。
  • copy 把可变对象变成答案快照。

通用回溯骨架复制

def backtrack(path, choices):
 if should_collect(path):
 res.append(path.copy())

 for choice in choices:
 if not valid(choice, path):
 continue
 path.append(choice)
 backtrack(path, next_choices(choice))
 path.pop()

代码行读什么写什么错了会怎样

path.append(choice)当前候选路径后面的分支看不到新选择

backtrack(...)当前状态递归深一层不会继续展开方案树

path.pop()刚刚加入的候选还原路径兄弟分支继承脏状态

path.copy()当前快照结果数组答案会跟着后续修改一起变

状态账本

read

把状态分开记，递归就清楚了

回溯页最容易混的，是当前路径、下一层起点、已用下标和答案快照。它们看起来都只是变量，但职责不同：有的控制还能选谁，有的控制能不能回退，有的只是把结果定格下来。

先分清这四件事，看到代码时就能知道这一行是在改搜索范围，还是在改路径内容，还是在给结果拍快照。

先记这四个动作

  • track 记录当前走到哪条分支。
  • start 决定下一层从哪里开始扫。
  • used 标记某个下标是否已经进过路径。
  • copy 把当前路径定格成答案。

对象进入下一层前离开这一层后读什么写什么错了会怎样

track当前路径恢复成父层路径当前选择序列append / pop兄弟分支看到脏路径

start决定下一层从哪里扫传给下一层当前下标i + 1 或 i组合题和排列题会互相串模板

used标记某个下标是否进过路径回溯时取消标记当前位置占用情况true / false排列题漏解或重复用位

res.append(track.copy())收到完整快照与后续回溯解耦当前路径内容结果数组答案会跟着后续修改一起变

手算追踪

trace

拿 [1,2,2] 看同层去重

这个例子最适合看“同层重复”和“下一层重复”有什么区别。根节点试第一个 2 可以，回到根再试第二个 2 就该跳过；但在路径 [2] 的下一层，第二个 2 仍然可以用。

把“这一层已经试过等价值”记牢，就能分清同层去重和下一层合法重复。去重只挡住等价分支，不挡住更深一层的正常选择。

先看三个观察点

  • 子集题会在每一层收集路径。
  • 组合题会把下一层往右挪。
  • 排列题会用 used 阻止同一下标重复进入路径。

层级starttrack发生了什么结果

根0[]先试第一个 2进入 [2] 分支

根0[]再试第二个 2同层重复，跳过

[2]2[2]下一层继续试第二个 2得到 [2,2]

回到根0[]跳过第二个 2 后再往后看避免重复的 [2] 分支

同层去重

dedupe

先排序，再在同层跳过

重复元素要先排序，才能把相同值挨在一起。然后在同一层里，如果当前值和前一个值相同，就跳过当前值，避免生成结构完全一样的分支。

注意这个判断只管同层，不管更深一层。也就是说，在路径 [2] 的下一层，第二个 2 仍然可以进入，因为它不是“同层重复”。

去重和剪枝不是一回事

  • 去重：这个分支会得到已经出现过的答案。
  • 剪枝：这个分支继续走也不可能得到答案。
  • 同层去重依赖排序，剪枝依赖题目约束。

子集 II 的同层去重复制

def subsetsWithDup(nums):
 nums.sort()
 res, track = [], []

 def backtrack(start):
 res.append(track.copy())
 for i in range(start, len(nums)):
 if i > start and nums[i] == nums[i - 1]:
 continue
 track.append(nums[i])
 backtrack(i + 1)
 track.pop()

 backtrack(0)
 return res

模型是否排序关键判断它保护了什么

子集 / 组合通常先排序i > start and nums[i] == nums[i-1]同层不会重复开同样的分支

排列通常也先排序used[i] + 同层跳过既不重复用下标，也不把顺序当重复

题面落点

cases

先把四种常见题面翻成同一套语言

电话号码组合、组合总和、全排列、子集 II，看起来像四个不同世界，其实都可以先翻成“路径里放什么、下一层从哪里选、什么时候收集”。区别只在于题目对顺序、重复和合法性的要求不同。

电话号码组合里，路径是已经拼出来的字母，下一层的候选是当前数字对应的字母。组合总和更像“在一个有序集合里往右挑数”，所以用了 start 以后，顺序重复就没了。全排列把顺序当成答案的一部分，因此每一层都要重新扫候选，但 used 会挡住已经在路径里的下标。子集 II 则多了一个同层去重：同一层已经试过一个 2，另一个 2 就别再开同样的分支。

把这四种题放在同一张图里看，会发现它们都在回答同一个问题：下一步候选是否还合法，合法的话会不会和已经走过的分支重复。

每个题面先问一句

  • 电话号码组合：每一层要补一个字符，答案长度固定。
  • 组合总和：顺序不重要，但每个数的使用次数可能有限制。
  • 全排列：顺序本身就是答案的一部分。
  • 子集 II：顺序不重要，但相同值的分支要在同层去掉。

一个完整例子：组合总和里的“只往右选”

假设候选是 [2,3,6,7]，目标和是 7。如果你先选了 2，下一层还可以继续选 2、3、6、7 这些右侧候选中的一部分；如果你把下一层又从 0 开始扫，就会把 [2,3] 和 [3,2] 都搜出来，答案变成排列式重复。

这类题最有用的检查方式是先问自己：这一层结束后，答案中是否只关心集合，不关心顺序。如果答案只关心集合，下一层就必须从右侧继续，不然同一批数会按不同顺序被重复构造。

把这个例子换成子集 II，逻辑也一样。根节点上两个 2 的分支是等价的，所以同层只保留一个；但在路径 [2] 之后，第二个 2 是更深一层的合法选择，不能因为“值重复”就一起删掉。

题面路径代表什么选择列表怎么来关键控制最容易写错的地方

电话号码组合已经拼出的字母当前数字对应的字母每层补一个字符忘记在最后一层收集

组合总和已经选中的数剩余且允许继续试的数start 只往右把顺序也算进答案

全排列当前顺序没用过的下标used 阻止重复用同一位误用 start

子集 II当前集合排序后的候选同层去重把重复值一刀切掉

题面落点检查清单复制

路径里放什么？下一层从哪里选？什么时候收集？是否需要 start？是否需要 used？有没有同层去重？

症状通常是哪一行错了会漏什么修法

组合里出现 [1,2] 和 [2,1]下一层又从 0 开始顺序重复把下一层改成 i + 1

答案保存后一起变了把 track 直接塞进结果所有快照收集时写 track.copy()

合法的 [1,2,2] 消失了把重复值全都禁掉更深一层的合法重复只做同层去重

排列漏掉 [2,1]误用了组合模板顺序不同的答案每层从头扫，并用 used

完整走法

walkthrough

拿组合总和看一轮真正的递归

还是看 [2,3,6,7] 和目标和 7。先选 2，路径变成 [2]，下一层继续从右侧开始扫；此时再选 2 或 3 都可以，因为组合总和允许重复使用同一个数，只是不允许把同一组数按不同顺序重复构造。

如果你把下一层写成从 0 开始，就会在同一棵树里制造出 [2,3] 和 [3,2] 两条等价分支。它们最后得到的是同一个集合，但代码会把它们当成两个答案，这就是“组合题写成排列题”的典型症状。

这类题真正要看的，是“当前层的候选还剩哪些”和“这层算出来的答案是不是已经足够唯一”。一旦答案唯一性靠顺序来区分，题就变成排列；一旦顺序不该区分，下一层就必须只往右选。

这轮走法里读写了什么

  • 读取：当前路径 track，以及下一层的起点 start。
  • 写入：把候选数放进路径，再把新路径递归下去。
  • 恢复：回到当前层后，把刚才放进去的数弹出去。
  • 判断：剩余和是否还够，候选是否还能继续用。

当前层trackstart当前动作为什么这样做

根[]0先试 2从最小候选开始，先把可行分支展开出来

第一层[2]0继续试 2组合总和允许重复使用同一个数

第一层[2]1试 3继续往右选，避免顺序重复

第二层[2,3]1收集或回退如果剩余和正好为 0，就把这条路径当答案

回到根[]0试 3 再试 6兄弟分支看到的是干净路径，不会继承 [2]

正确与错误的分界复制

def backtrack(start, remain):
 if remain == 0:
 res.append(track.copy())
 return
 for i in range(start, len(nums)):
 x = nums[i]
 if x > remain:
 break
 track.append(x)
 backtrack(i, remain - x)
 track.pop()

错法表面上看起来像什么真正后果改哪一行

下一层从 0 开始好像能多试几个分支重复构造顺序不同的同一答案把递归入口改成 i + 1 或当前题允许的规则

收集时不写 copy()结果数组里先看不出问题所有答案最后一起变形在收集时保存快照

不判断剩余和递归树看着更完整会把明显超和的分支继续走下去在进入下一层前先做剩余和检查

排列边界

boundary

排列题为什么不能套组合模板

拿 [1,2,3] 做一个最小例子。若题目要的是长度为 2 的组合，[1,2]、[1,3]、[2,3] 就够了；若题目要的是长度为 2 的排列，[1,2]、[2,1]、[1,3]、[3,1] 都要出现。差别不在数据，而在题目是否把顺序算成答案的一部分。

这意味着排列题每一层都得重新扫候选，因为上一层放过的值，下一层可能要换个位置继续放。组合题不需要这么做，它只关心一组数有没有被选中，不关心同一组数换个顺序会不会变成另一个答案。

如果你把排列题写成“只往右选”，你得到的是一个被压扁的答案树：所有以 2 开头的排列、以 3 开头的排列，都会被你提前切掉。代码看起来很规整，结果却少了一大半。

排列和组合的分界点

  • 组合看集合，排列看顺序。
  • 组合常用 start，排列常用 used。
  • 组合的下一层往右，排列的下一层从头扫。
  • 如果顺序不同也算不同答案，不能把它写成组合。

题型根节点先选 1 之后第二层还能看见谁会不会把 [2,1] 漏掉

组合路径是 [1]只看右侧候选 2,3会，因顺序不算不同答案

排列路径是 [1]重新看 2,3，也要允许 2 开头的分支不会，[2,1] 必须保留

症状你大概写错了什么会少掉哪类答案下一步怎么修

总是少一半左右把排列写成只往右选以 2、3 开头的分支改成每层从头扫描

顺序不同却只留一种把顺序误当成重复本该保留的排列用 used 区分位置和数值

答案树看着很整齐，但结果太少过早剪掉分支所有逆序分支回到题意，确认顺序是不是答案的一部分

实践入口：做第 4、7、11、15、18 题时，先判断题面有没有把顺序算进答案。只要顺序重要，下一层就不能再用组合模板那种“只往右看”的方式。

复杂度与剪枝

complexity

复杂度由方案树决定

子集、组合、排列都会在方案树上展开，只是分叉方式不同。子集大致是 2^n 量级，排列会走到阶乘量级，组合介于两者之间。这里最重要的，是看懂为什么路径越深，候选会越来越少或越来越多。

什么时候该提前停

  • 组合题里，剩余元素不够凑满 k 个，就可以剪掉。
  • 重复值已经在同层试过，就不要再开等价分支。
  • 排列题里，已经进过路径的下标不能再选。

模型时间复杂度直觉递归空间常用剪枝

子集O(n · 2^n)O(n)同层去重

组合约 O(k · C(n, k))O(k)剩余元素不够时提前停

排列O(n · n!)O(n)used 和同层去重

常见错误

debug

忘记撤销选择错因：只看到向下递归，没处理返回后的状态。后果：兄弟分支继承旧路径，答案变脏。修法：每个 append 后面都配一个 pop。

答案保存同一个列表错因：把可变对象直接丢进结果。后果：后面每次撤销都会改写已经保存的答案。修法：收集时写 track.copy()。

组合题从 0 重新扫描错因：把组合写成排列。后果：出现 [1,2] 和 [2,1] 这种重复结果。修法：用 start 控制只能向右选。

排列题误用 start错因：看到回溯就套组合模板。后果：漏掉顺序不同但合法的排列。修法：每层从头扫描，并用 used 防止同一下标重复。

同层去重过度错因：把“同层跳过”误写成“所有重复都不准用”。后果：像 [1,2,2] 这样的合法答案会被删掉。修法：只跳过本层已经试过的重复值。

结束条件写错错因：没先确认什么才算一个答案。后果：子集漏空集，组合收集太早，排列漏最后一层。修法：先用一句话写清收集条件，再写代码。

10synthesis

Q: 10. 如果输入里有 [1,2,2]，但你把所有重复值都一刀切地禁止使用，会漏掉什么？

参考答案 / 评分要点

。错因：把同层去重写成了全局禁止。后果： [1,2,2] 这类合法答案会被误删。修法：先排序，再只在同层跳过重复候选，别把下一层的合法重复也挡住。

11choice

Q: 11. 如果同层去重写成 if i > 0 and nums[i] == nums[i - 1]，最危险的结果是什么？

参考答案 / 评分要点

会把更深一层的合法重复也挡住。同层去重只挡住等价分支，不该挡住更深一层的合法重复。

12synthesis

Q: 12. 为什么同层去重要先排序？如果不排序，会卡在哪一步？

参考答案 / 评分要点

。先排序，重复值才会挨在一起，同层跳过时才能靠“和前一个相同”来认出等价分支。不排序的话，相同值分散在数组里，你就看不出当前候选是不是本层已经试过的同类分支。

17synthesis

Q: 17. 用 2-3 句话说明回溯模板为什么同时需要 track.copy() 和 track.pop()。

参考答案 / 评分要点

。track.copy() 是把当前路径的快照保存进答案，避免后续修改把已经收集的结果改掉。 track.pop() 是在递归返回时恢复现场，让兄弟分支看到干净状态。两者一个负责保存结果，一个负责还原状态，缺一个都不行。

18synthesis

Q: 18. 用 6-8 句话讲清楚：子集、组合、排列三者的区别，以及它们如何落到 start、used 和同层去重上。

参考答案 / 评分要点

。好答案至少要说清：子集看选不选，组合不看顺序，排列看顺序；子集和组合通常用 start 限制下一层只往右走，排列要用 used 防止同一下标重复进入路径；有重复元素时，还要先排序，再在同层跳过重复值。只要能把这三件事连起来，就说明你真的把题型和代码对应上了。
