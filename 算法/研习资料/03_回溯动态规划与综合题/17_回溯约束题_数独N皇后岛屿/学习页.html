回溯约束题：数独、N 皇后与岛屿 DFS

这一页只讲三类会被约束卡住的 DFS：N 皇后、数独、岛屿。前两类是在试一个合法方案，岛屿 DFS 是在把一整块连通区域走完。读完后，你应该能把新题拆成“当前状态、候选、合法性检查、进入下一层后的动作”四件事，并判断该回退还是该沉没。位运算优化、数独最少候选位和 BFS flood fill 先放后面。

学习目标

start

这一页不再重复回溯怎么写。前一页已经把路径、选择和恢复讲过了，这一页只盯住约束：什么能放，什么不能放，放进去以后该撤销还是该留下痕迹。N 皇后和数独是在试完整方案，岛屿 DFS 是在走完整个连通块。

先抓主线

map

三类题的共同骨架是判断和写回，递归只是外层形状。先读当前状态，再列出候选，再做合法性检查，再决定要不要改状态。区别只在最后一步：构造答案时要撤销，遍历连通块时要保留访问痕迹。

状态账本

mechanism

先把三类题压成一个通用流程：找到当前位置，枚举候选，过合法性检查，做选择，递归，回退或沉没。N 皇后和数独在离开分支时要恢复现场，岛屿 DFS 在进入格子时就标记，不再还原。

代码
def dfs(state):
 if finished(state):
 record_answer()
 return

 for choice in candidates(state):
 if not legal(state, choice):
 continue

 apply(state, choice)
 dfs(next_state(state))
 undo_or_keep(state, choice)

数独找到一个完整解时要一路返回 True；N 皇后如果要列出全部方案，就继续把其他列也试完；岛屿题只关心连通块，不需要把访问痕迹擦掉。

N 皇后：一行只放一个

trace

N 皇后的行已经被题目固定了，所以递归层只负责当前行选哪一列。要记的是三个占用集合：列、主对角线、副对角线。这样你一看候选列，就能马上知道它会不会撞上之前放过的皇后。

$$ d_1 = r - c,\quad d_2 = r + c $$

这里的 r 是行号，c 是列号。主对角线上的格子 r-c 相同，副对角线上的格子 r+c 相同。

代码
def solve_n_queens(n):
 cols = set()
 diag1 = set()
 diag2 = set()
 board = [["."] * n for _ in range(n)]
 answers = []

 def dfs(row):
 if row == n:
 answers.append(["".join(line) for line in board])
 return

 for col in range(n):
 d1 = row - col
 d2 = row + col
 if col in cols or d1 in diag1 or d2 in diag2:
 continue

 cols.add(col)
 diag1.add(d1)
 diag2.add(d2)
 board[row][col] = "Q"

 dfs(row + 1)

 board[row][col] = "."
 cols.remove(col)
 diag1.remove(d1)
 diag2.remove(d2)

 dfs(0)
 return answers

数独：空格、候选和回退

trace

数独和 N 皇后一样在构造答案，但合法性检查换成了行、列和 3×3 宫。递归通常先找一个空格，再从 1 到 9 试数。只要某条分支已经填完所有空格，就立刻返回 True，让上层停止继续回退。

$$ \text{box\_r} = (r // 3) \times 3,\quad \text{box\_c} = (c // 3) \times 3 $$

r 和 c 是当前空格的行列号。先算出 3×3 宫的左上角，再扫这个宫里的 9 个格子。

代码
def solve_sudoku(board):
 def find_empty():
 for r in range(9):
 for c in range(9):
 if board[r][c] == ".":
 return r, c
 return None

 def legal(r, c, ch):
 for i in range(9):
 if board[r][i] == ch:
 return False
 if board[i][c] == ch:
 return False

 br = (r // 3) * 3
 bc = (c // 3) * 3
 for rr in range(br, br + 3):
 for cc in range(bc, bc + 3):
 if board[rr][cc] == ch:
 return False
 return True

 def dfs():
 empty = find_empty()
 if empty is None:
 return True

 r, c = empty
 for ch in "123456789":
 if not legal(r, c, ch):
 continue
 board[r][c] = ch
 if dfs():
 return True
 board[r][c] = "."
 return False

 dfs()

岛屿 DFS：走完连通块

fill

岛屿题的 DFS 在走连通块，不是在试方案。你进到一个 1，就立刻把它标记成 0 或写进 visited，这样四个方向再绕回来时就会停住。数量题和面积题都依赖这个动作。

对象进入前进入后读什么写什么错了会怎样
grid[r][c]10边界和水访问痕迹重复计数或无限递归
visited[r][c]FalseTrue当前位置是否已经处理打标记回头路会再次进入

(0,0)

右

1

继续 DFS

同一块岛被接上

(1,2)

四周

越界或 0

停止

连通块边界收住

代码
def num_islands(grid):
 rows, cols = len(grid), len(grid[0])

 def dfs(r, c):
 if r < 0 or r >= rows or c < 0 or c >= cols:
 return
 if grid[r][c] != "1":
 return

 grid[r][c] = "0"
 dfs(r + 1, c)
 dfs(r - 1, c)
 dfs(r, c + 1)
 dfs(r, c - 1)

 count = 0
 for r in range(rows):
 for c in range(cols):
 if grid[r][c] == "1":
 count += 1
 dfs(r, c)
 return count

题面到代码

operation

这页真正要落地的看到题面后能迅速决定入口状态和第一步动作。公开题里最常见的三条线就是：LeetCode 51 N-Queens、37 Sudoku Solver、200 Number of Islands；同一页里提到的 695 Max Area of Island、79 Word Search、130 Surrounded Regions，也都沿着同一条思路走。

公开题先看什么第一次写什么第一类常见失败回来看本页哪一段
51 N-Queens当前行和三个占用集合列、主对角线、副对角线一起更新只查列，漏掉斜线冲突N 皇后状态账本和追踪表
37 Sudoku Solver一个空格和它周围的行、列、宫先试一个数字，再决定是否继续找到解以后还继续回退，或者宫格起点算错数独一格试数追踪和回退账本
200 Number of Islands一个陆地格子先把它沉没，再向四周扩散不标记访问，导致重复计数或死循环岛屿 DFS 状态账本和连通块追踪表
695 Max Area of Island当前连通块的面积贡献返回当前块能累加多少把面积题写成只计数题，返回值丢掉岛屿数量和最大面积的返回值差别
79 Word Search当前字符和已访问路径进入格子后标记，回溯时恢复把路径当成可重复访问的普通网格“撤销”与“保留访问痕迹”的区别
130 Surrounded Regions先从边界找活路先标记边界连通块，再反转内部区域从中间乱搜，最后把边界关系弄丢岛屿 DFS 的边界意识
代码
# N 皇后：row -> col
if col not in cols and (row - col) not in diag1 and (row + col) not in diag2:
 place_queen()

# 数独：空格 -> 数字
if legal(r, c, ch):
 board[r][c] = ch
 if dfs():
 return True
 board[r][c] = "."

# 岛屿：陆地 -> 访问痕迹
if grid[r][c] == "1":
 grid[r][c] = "0"
 dfs(r + 1, c)
 dfs(r - 1, c)
 dfs(r, c + 1)
 dfs(r, c - 1)

这三段代码看起来像不同模板，实际上只是在同一个动作模型上换了状态。你要做的事很简单：先看入口对象，再看候选，再看合法性，最后决定是撤销还是保留痕迹。这样读新题时，你不会先被代码形式吓住，而是先知道自己在处理哪一种状态变化。

完整追踪

walkthrough

这一段把三类题放在同一张追踪表里看。你可以把它当成答题前的最后一遍确认：先找状态，再看候选，再看合法性检查读了什么，最后看失败后是回退还是保留痕迹。这样做的好处很直接，遇到新题时不会先找“像哪道旧题”，而会先找“这一层到底在改哪个对象”。

代码
# N 皇后：写集合，再递归，再撤销
cols.add(col); diag1.add(row - col); diag2.add(row + col)
dfs(row + 1)
cols.remove(col); diag1.remove(row - col); diag2.remove(row + col)

# 数独：写数字，再递归，再回退
board[r][c] = ch
if dfs():
 return True
board[r][c] = "."

# 岛屿：先沉没，再扩散，不恢复
grid[r][c] = "0"
dfs(r + 1, c)
dfs(r - 1, c)
dfs(r, c + 1)
dfs(r, c - 1)

看完这张表，再回头看 51、37、200，会更容易定位第一行该写什么。你只要把“起点状态、第一步动作、失败后的处理”这三项说清楚，后面的实现就不会漂。练习题里如果卡住，优先回这一段查：你到底是在回退，还是在保留访问痕迹。

起手实验

lab

这一段只回答一个问题：拿到题面后，第一步到底写什么。先把题目翻成状态，再试一个候选，再看合法性检查有没有通过。后面的递归只是把这三步重复多次。

代码
# N 皇后
row = 0
for col in range(n):
 if col not in cols and (row - col) not in diag1 and (row + col) not in diag2:
 place_and_recurse()

# 数独
r, c = find_empty()
for ch in "123456789":
 if legal(r, c, ch):
 board[r][c] = ch
 if dfs():
 return True
 board[r][c] = "."

# 岛屿 DFS

 grid[r][c] = "0"

 dfs(r - 1, c)
 dfs(r, c + 1)
 dfs(r, c - 1)

这三题都可以先写出第一步，再补递归和回退。卡住时先回看 1、5、13 题，确认你是不是已经能把状态、候选和动作对应起来。

这里先不追的变体

boundary

同一条主线能覆盖很多题，但不是所有变体都该在这一页里展开。你现在要先学会“选对模板”，不是先把模板堆到最省时间的版本。太早追优化，通常只会把状态写法弄乱，然后把调试难度抬高。

常见误区

debug

看症状时先分两类：答案少，多半是合法性检查漏了；答案重复或死循环，多半是状态没有被正确改写。下面这张表把最常见的几处卡点直接连到修法。

下一步怎么接

transfer

这页的判断顺序可以直接带去做新题。只要题目在“试一个合法方案”这边，N 皇后和数独的模板就有用；只要题目在“把一个连通块走完”这边，岛屿 DFS、被围绕的区域、单词搜索就能接上。下一页进入 DP 后，重心会换成“当前状态依赖谁”，不再主要看合法性。

练习

diagnostic

状态先说这一步处理的是行、空格还是格子。

候选再说这一层能试哪些选择。

合法性然后看冲突、越界和重复访问。

动作最后决定回退、早停还是沉没。
