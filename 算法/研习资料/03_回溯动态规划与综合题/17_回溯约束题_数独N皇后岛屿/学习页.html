回溯约束题：数独、N 皇后与岛屿 DFS

这一页只讲三类会被约束卡住的 DFS：N 皇后、数独、岛屿。前两类是在试一个合法方案，岛屿 DFS 是在把一整块连通区域走完。读完后，你应该能把新题拆成“当前状态、候选、合法性检查、进入下一层后的动作”四件事，并判断该回退还是该沉没。位运算优化、数独最少候选位和 BFS flood fill 先放后面。

读完后能做什么

  • 能把 N 皇后、数独、岛屿分别翻成 state、candidate、legal 和 action。
  • 能写出 cols、diag1、diag2 和数独的行、列、宫检查。
  • 能说清数独要回退，岛屿 DFS 为什么通常不回退。

学习目标

start here

这一页不再重复回溯怎么写。前一页已经把路径、选择和恢复讲过了，这一页只盯住约束：什么能放，什么不能放，放进去以后该撤销还是该留下痕迹。N 皇后和数独是在试完整方案，岛屿 DFS 是在走完整个连通块。

先抓主线

map

三类题的共同骨架是判断和写回，递归只是外层形状。先读当前状态，再列出候选，再做合法性检查，再决定要不要改状态。区别只在最后一步：构造答案时要撤销，遍历连通块时要保留访问痕迹。

对象 进入前 进入后 读什么 写什么 错了会怎样
board[row][col] in N 皇后 . Q 列和两条对角线是否已占 棋盘与三个占用集合 忘记撤销会污染兄弟分支
board[r][c] in 数独 . 一个数字 行、列、宫是否冲突 这个格子的数字 不回退会把假答案带到后面
grid[r][c] in 岛屿 DFS 1 0 或 visited 边界和水 访问痕迹 不先标记就会绕回去重复走

把这张账本写清楚，代码就不会乱。你只要能回答“谁被读了、谁被改了、谁需要恢复”，就已经比只背模板稳了。

状态账本

mechanism

先把三类题压成一个通用流程：找到当前位置，枚举候选，过合法性检查，做选择，递归，回退或沉没。N 皇后和数独在离开分支时要恢复现场，岛屿 DFS 在进入格子时就标记，不再还原。

N 皇后 row = 2

进入这一层时，前两行已经各放了一个皇后；候选只看当前行每一列能不能被三个集合挡住。

数独 空格 (0, 2)

先找空格，再按 1 到 9 试；每次试之前先看同行、同列、同宫是否冲突。

岛屿 DFS grid[r][c] = "0"

一旦进入陆地就改成 0，四方向再回来时会直接停住，连通块只会被走一次。

题型 进入时写什么 退出时做什么 为什么
N 皇后 棋盘格、列集合、两条对角线集合 全部撤销 还要继续试同一行的别的列
数独 这个空格的数字，必要时同步记录行列宫 写回 . 只要当前分支失败，就要换数字
岛屿 DFS 把陆地改成 0 或写进 visited 不恢复 访问过的格子不应该再被当成新陆地
通用骨架：构造方案和遍历连通块Python
def dfs(state):
    if finished(state):
        record_answer()
        return

    for choice in candidates(state):
        if not legal(state, choice):
            continue

        apply(state, choice)
        dfs(next_state(state))
        undo_or_keep(state, choice)

数独找到一个完整解时要一路返回 True；N 皇后如果要列出全部方案，就继续把其他列也试完；岛屿题只关心连通块，不需要把访问痕迹擦掉。

N 皇后：一行只放一个

trace

N 皇后的行已经被题目固定了，所以递归层只负责当前行选哪一列。要记的是三个占用集合：列、主对角线、副对角线。这样你一看候选列，就能马上知道它会不会撞上之前放过的皇后。

先看一个最小但不省略机制的例子：n = 4，先在 (0, 1) 放一个皇后。这个选择会把 cols 写成 {1}，把 diag1 写成 {-1}，把 diag2 写成 {1}。下一层来到第 1 行时，(1, 1) 会因为列冲突直接跳过，(1, 3) 才值得继续试。这个例子里最重要的不是答案，而是你能看到“写了什么，下一层就挡住什么”。

$$d_1 = r - c,\quad d_2 = r + c$$

这里的 r 是行号，c 是列号。主对角线上的格子 r-c 相同，副对角线上的格子 r+c 相同。

N 皇后回溯代码Python
def solve_n_queens(n):
    cols = set()
    diag1 = set()
    diag2 = set()
    board = [["."] * n for _ in range(n)]
    answers = []

    def dfs(row):
        if row == n:
            answers.append(["".join(line) for line in board])
            return

        for col in range(n):
            d1 = row - col
            d2 = row + col
            if col in cols or d1 in diag1 or d2 in diag2:
                continue

            cols.add(col)
            diag1.add(d1)
            diag2.add(d2)
            board[row][col] = "Q"

            dfs(row + 1)

            board[row][col] = "."
            cols.remove(col)
            diag1.remove(d1)
            diag2.remove(d2)

    dfs(0)
    return answers
当前行 试的列 冲突检查 结果
row = 0 col = 1 列、r-c、r+c 都空 放下第一个皇后
row = 1 col = 1 列已占 跳过
row = 1 col = 3 三个集合都不冲突 继续向下
row = 2 col = 2 r+c 与上一层冲突 跳过
步骤 board cols diag1 diag2 下一步应该观察什么
放下 (0,1) Q 写进去 {1} {-1} {1} 第 1 行里哪些列立刻失效
试 (1,1) 不变 列冲突 不看 不看 这个候选直接跳过，不进入下一层
回退 (0,1) . 写回去 {} {} {} 兄弟分支能不能看到干净现场

如果你只检查列，row = 2 的这个候选会被误放进去。撤销时也要把三个集合一起恢复，少一个都会把后面的判断带偏。

数独：空格、候选和回退

trace

数独和 N 皇后一样在构造答案，但合法性检查换成了行、列和 3×3 宫。递归通常先找一个空格，再从 1 到 9 试数。只要某条分支已经填完所有空格，就立刻返回 True，让上层停止继续回退。

还是看一个具体空格。假设你现在要填 (0, 2)，这一行里已有 {5, 3, 7}，这一列里已有 {8}，这个宫里已有 {1, 9}。5 会在同行被挡住，6 会在同宫被挡住，4 先能写进去。填进去以后不是结束，而是给下一层制造新的局面；如果后面卡住，就把这个位置恢复成 . 再换下一个数字。

$$\text{box\_r} = (r // 3) \times 3,\quad \text{box\_c} = (c // 3) \times 3$$

r 和 c 是当前空格的行列号。先算出 3×3 宫的左上角，再扫这个宫里的 9 个格子。

数独求解代码Python
def solve_sudoku(board):
    def find_empty():
        for r in range(9):
            for c in range(9):
                if board[r][c] == ".":
                    return r, c
        return None

    def legal(r, c, ch):
        for i in range(9):
            if board[r][i] == ch:
                return False
            if board[i][c] == ch:
                return False

        br = (r // 3) * 3
        bc = (c // 3) * 3
        for rr in range(br, br + 3):
            for cc in range(bc, bc + 3):
                if board[rr][cc] == ch:
                    return False
        return True

    def dfs():
        empty = find_empty()
        if empty is None:
            return True

        r, c = empty
        for ch in "123456789":
            if not legal(r, c, ch):
                continue
            board[r][c] = ch
            if dfs():
                return True
            board[r][c] = "."
        return False

    dfs()
空格 row 里已有 col 里已有 box 里已有 候选 5 候选 6 候选 4
(0, 2) {5, 3, 7} {8} {1, 9} 行冲突 宫冲突 可放
动作 board 行列宫状态 为什么要这样做
写入 4 . 变成 4 行、列、宫都暂时占住 下一层在这个局面上继续试别的格子
递归失败 恢复成 . 把这次试数撤掉 不撤销的话，后面的候选会看到假的填法
找到完整解 保持当前填法 一路返回 True 这题只要一个解，不需要继续试别的分支

这里先按一个解来写，不追多解计数、最少候选位和 DLX。先把回退和早停写对，后面的优化才有意义。

岛屿 DFS：走完连通块

fill

岛屿题的 DFS 在走连通块，不是在试方案。你进到一个 1，就立刻把它标记成 0 或写进 visited，这样四个方向再绕回来时就会停住。数量题和面积题都依赖这个动作。

如果地图是 1 1 0 / 1 0 1，从左上角进来后，右边和下边都还会继续扩散，碰到 0 就停。这里要做的是把同一块岛上的每个格子都收进来。等你把这块区域改成 0，外层扫描再看到它，就知道它已经处理过了。

对象 进入前 进入后 读什么 写什么 错了会怎样
grid[r][c] 1 0 边界和水 访问痕迹 重复计数或无限递归
visited[r][c] False True 当前位置是否已经处理 打标记 回头路会再次进入
起点 方向 遇到什么 动作 结果
(0,0) 右 1 继续 DFS 同一块岛被接上
(0,1) 下 1 继续 DFS 面积继续累加
(1,2) 四周 越界或 0 停止 连通块边界收住
岛屿数量代码Python
def num_islands(grid):
    rows, cols = len(grid), len(grid[0])

    def dfs(r, c):
        if r < 0 or r >= rows or c < 0 or c >= cols:
            return
        if grid[r][c] != "1":
            return

        grid[r][c] = "0"
        dfs(r + 1, c)
        dfs(r - 1, c)
        dfs(r, c + 1)
        dfs(r, c - 1)

    count = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == "1":
                count += 1
                dfs(r, c)
    return count
题目 DFS 进入时 DFS 返回什么 遍历完后怎么处理
岛屿数量 把 1 改成 0 通常不返回值 外层计数 + 1
岛屿最大面积 把 1 改成 0 当前连通块贡献的面积 外层取最大值

如果题目不允许改输入，就换 visited。岛屿数量、岛屿最大面积、被围绕的区域、单词搜索都在这条线上；最短路和层数不在这页。

题面到代码

operation bridge

这页真正要落地的不是“我知道它们都用 DFS”，而是看到题面后能迅速决定入口状态和第一步动作。公开题里最常见的三条线就是：LeetCode 51 N-Queens、37 Sudoku Solver、200 Number of Islands；同一页里提到的 695 Max Area of Island、79 Word Search、130 Surrounded Regions，也都沿着同一条思路走。

动手时先别急着写完整代码。先把题目翻成一句人话：N 皇后是在第 row 行试列；数独是在某个空格试数字；岛屿题是在一个陆地格子上做四方向扩散。只要这句话说不顺，你后面的递归层、返回值和回退方式大概率都会乱。

公开题 先看什么 第一次写什么 第一类常见失败 回来看本页哪一段

完整追踪

walkthrough

起手实验

lab

这一段只回答一个问题：拿到题面后，第一步到底写什么。先把题目翻成状态，再试一个候选，再看合法性检查有没有通过。后面的递归只是把这三步重复多次。

这里先不追的变体

boundary

同一条主线能覆盖很多题，但不是所有变体都该在这一页里展开。你现在要先学会“选对模板”，不是先把模板堆到最省时间的版本。太早追优化，通常只会把状态写法弄乱，然后把调试难度抬高。

变体 这一页现在怎么处理 为什么先不展开 以后回到哪里
N 皇后位运算 先用集合写清楚状态 位运算只是更快的占用编码，不改主逻辑 先把列和对角线判断写顺，再看优化
数独最少候选位 先按“找一个空格、试一个数字”写 MRV 的核心是挑最难的空格，先追它只会分散注意力 当你已经能稳定写出回退后再加
DLX / 精确覆盖 不展开 这已经是另一套建模方式了，不是本页的入门层 等你能把数独按行列宫写稳，再看它
BFS flood fill 只提思路，不改主线 这页要先把“标记访问”和“是否恢复”讲透 后面遇到网格题再比较队列写法
并查集 / 连通块预处理 不展开 它解决的是另一种批量连通性问题，不是单次 DFS 入口 等你已经知道何时该沉没、何时该回退再接

如果你现在一上来就想把这些优化一起写，最可能出现的问题不是慢，而是状态读写顺序先错了。先把能跑通的版本写出来，再决定要不要压缩状态或者换更强的搜索顺序。

常见误区

debug

看症状时先分两类：答案少，多半是合法性检查漏了；答案重复或死循环，多半是状态没有被正确改写。下面这张表把最常见的几处卡点直接连到修法。

题型 常见错法 外在症状 修法
N 皇后 只检查列 同斜线皇后被漏掉 同时维护 cols、r-c 和 r+c
N 皇后 回退时只改棋盘，不改集合 后续合法位置被误判 做了几个状态改写，就恢复几个状态
数独 找到一个解后还继续回退 答案被撤掉，或者最后棋盘不完整 成功分支一路 return True
数独 宫格起点算错 检查的不是同一个 3×3 宫 (r // 3) * 3 和 (c // 3) * 3 先算对
岛屿 DFS 不先标记访问 四方向绕回去，甚至递归栈爆掉 进入格子后立刻沉没或打标记
岛屿 DFS 访问后又恢复成 1 同一块陆地被重复计数 计数题保持访问痕迹，不回退

位运算、候选数最少优先、DLX、BFS flood fill 都可以提速，但它们不会替你补这五个判断。先把检查顺序和改写时机写对，再谈优化。

下一步怎么接

transfer

这页的判断顺序可以直接带去做新题。只要题目在“试一个合法方案”这边，N 皇后和数独的模板就有用；只要题目在“把一个连通块走完”这边，岛屿 DFS、被围绕的区域、单词搜索就能接上。下一页进入 DP 后，重心会换成“当前状态依赖谁”，不再主要看合法性。

先问什么 如果答案是“是” 对应动作
要不要列出所有合法方案？ 是 按回溯写，成功后再决定是否继续搜
题目只要连通块数量或面积？ 是 按 flood fill 写，访问后保留痕迹
当前选择会不会破坏已有状态？ 会 先做合法性检查，再进入下一层
走完后要不要恢复？ 要 构造方案就回退，遍历区域就不回退

如果你做完这一页还是会把回溯写成一堆模板，回来看状态账本；如果你已经能分出恢复和标记，就可以去 18 看 DP 怎么记状态。
