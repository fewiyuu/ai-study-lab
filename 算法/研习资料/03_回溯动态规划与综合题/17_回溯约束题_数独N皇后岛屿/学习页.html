约束回溯
与岛屿 DFS带约束的题不是只把模板写长，而是在每次选择前判断它能不能放。

N 皇后和数独是在“造方案”，岛屿题常是在“走完连通块”。它们都用 DFS，但目标不同。
constraint backtracking

先验合法，再进入下一层

前一页的排列、组合、子集更像“把所有选择树走完”。这一页的题会多出约束：N 皇后不能同行、同列、同斜线；数独不能在同行、同列、同宫重复；岛屿 DFS 不能越界、不能走水、不能重复访问。

带约束的 DFS 可以先抓住一句话：每个递归层负责一个位置或一个状态，只有合法选择才进入下一层。N 皇后和数独常需要撤销选择，因为它们在尝试构造一个完整方案；岛屿题常直接把访问过的陆地改掉，因为目标是计数或标记，不需要恢复原地图。

学完应能完成的事

  1. 能把约束题拆成状态、候选和合法性检查。
  2. 写出 N 皇后按行放置的回溯模板。
  3. 读懂数独中“找空格、试数字、失败回退”的流程。
  4. 用岛屿 DFS 处理连通块计数、面积和沉没标记。
  5. 区分需要恢复现场的回溯和不需要恢复的 flood fill。

主线地图

mental model

带约束的 DFS 本质上还是选择树，只是每条边都要先过一层门禁。普通枚举问“下一步能选谁”，约束枚举还要问“这个选择放到当前局面里是否合法”。如果合法性检查写错，模板再像也会得到错答案。

当前状态行/格子/坐标
候选选择列/数字/方向
合法性检查冲突/越界
递归推进下一行/下一格
恢复或标记看任务目标

N 皇后

每一行放一个皇后，候选是这一行的每一列。合法性检查是列和两条斜线是否已有皇后。

判断：构造所有摆法，要撤销棋盘和占用集合。

数独

每次找一个空格，尝试 1 到 9。合法性检查是同行、同列、同 3×3 宫没有重复。

判断：通常只需要找到一个解，成功后一路返回 true。

岛屿

每个陆地格子是一个节点，四方向相邻是边。DFS 会把一个连通块全部访问完。

判断：计数题常把访问过的陆地沉没，不必恢复。

知识链路

chain

N 皇后和数独是“构造合法方案”。每次放入一个符号后，棋盘状态改变，下一层会基于这个状态继续判断。如果下一层失败，当前选择必须撤销，否则下一次尝试会看到错误局面。这类题的代码通常是：检查合法、做选择、递归、撤销选择。

岛屿题的目标不同。它通常不是要列出所有路径，而是要知道有多少块陆地、最大面积是多少，或某块陆地是否被边界包围。这里的 DFS 更像遍历连通块。访问过的陆地可以直接改成水，表示这块已经处理过。因为后面不需要还原地图来尝试别的方案，所以它常常不撤销。

你做题时可以先判断任务目标：如果是在多个候选方案里试出合法解，多半需要回溯恢复；如果只是把一个连通区域走完并计数，多半只需要访问标记。这个判断能帮你避免把岛屿题写成复杂回溯，也能避免把数独题写成无法回退的单路贪心。

本页的判断句

  • 构造方案：做选择后要能撤销。
  • 遍历连通块：访问过就标记，通常不恢复。
  • N 皇后按行推进，每行放一个。
  • 数独常用布尔返回值让成功解一路返回。
约束检查示意

切换场景，看“候选选择”为什么会被放行或拦下。

N 皇后：列和斜线冲突

绿色是已放皇后，黄色是当前尝试，红色是同列或同斜线不能放的位置。

你应能判断一个候选列是否被 columns、diag1、diag2 三个集合挡住。

N 皇后：每行放一个

n-queens

N 皇后最稳的建模方式是按行递归。第 row 层只负责给这一行找一个列位置。因为每行只放一个皇后，同行冲突天然避免；剩下只需要检查列冲突和两条斜线冲突。

斜线编号是这题的关键。左上到右下的主对角线，格子的 row - col 相同；右上到左下的副对角线，格子的 row + col 相同。用集合记录这些编号，合法性检查就能从扫描棋盘变成 O(1) 查询。

正确示例：N 皇后集合剪枝Python
def solveNQueens(n):
    board = [["."] * n for _ in range(n)]
    res = []
    cols, diag1, diag2 = set(), set(), set()

    def backtrack(row):
        if row == n:
            res.append(["".join(r) for r in board])
            return
        for col in range(n):
            if col in cols or row - col in diag1 or row + col in diag2:
                continue
            board[row][col] = "Q"
            cols.add(col)
            diag1.add(row - col)
            diag2.add(row + col)

            backtrack(row + 1)

            board[row][col] = "."
            cols.remove(col)
            diag1.remove(row - col)
            diag2.remove(row + col)

    backtrack(0)
    return res

这段代码里，row 是递归层数，也是当前要填写的行。每次尝试列 col 前先查三个集合。通过检查后，棋盘和三个集合都要更新；递归回来后，也要按相反顺序恢复。删掉任意一个恢复动作，后续分支都会被错误拦住。

数独：找空格，试数字，失败回退

sudoku

数独的状态不是“第几行放皇后”，而是“当前还要填哪个空格”。最直观的写法是按行列扫描，遇到已经填好的格子就跳过，遇到空格就尝试 1 到 9。某个数字如果不违反行、列、宫格，就先放进去，再递归处理下一个空格。

数独通常只要求求出一个可行解，所以递归函数可以返回布尔值。下一层如果返回 True，说明后面的所有空格都填好了，当前层不必继续试别的数字，直接把成功信号往上传。若所有数字都试完仍不行，就返回 False，让上一层撤销并换数字。

正确示例：数独回溯Python
def solveSudoku(board):
    def valid(r, c, ch):
        for k in range(9):
            if board[r][k] == ch:
                return False
            if board[k][c] == ch:
                return False
            br = (r // 3) * 3 + k // 3
            bc = (c // 3) * 3 + k % 3
            if board[br][bc] == ch:
                return False
        return True

    def backtrack(pos=0):
        if pos == 81:
            return True
        r, c = divmod(pos, 9)
        if board[r][c] != ".":
            return backtrack(pos + 1)
        for ch in "123456789":
            if not valid(r, c, ch):
                continue
            board[r][c] = ch
            if backtrack(pos + 1):
                return True
            board[r][c] = "."
        return False

    backtrack()

divmod(pos, 9) 把 0 到 80 的线性位置转成行列坐标。valid 会同时检查同行、同列和同宫。注意 board[r][c] = "." 只在后续失败时执行；如果后续成功，函数直接返回，棋盘保留解。

岛屿 DFS：走完整个连通块

islands

岛屿题的核心对象是网格连通块。格子为 1 表示陆地，四方向相邻的陆地属于同一个岛。DFS 从一块陆地出发，把能连到的陆地全部访问并标记，这样外层扫描再次遇到它们时就不会重复计数。

这类题常用“沉没”写法：把访问过的 1 改成 0。它改变了输入，但让访问标记更简单。若题目不允许修改输入，可以额外维护 visited 集合或布尔矩阵。两种写法目标一样：保证每个陆地格子只处理一次。

正确示例：岛屿数量Python
def numIslands(grid):
    m, n = len(grid), len(grid[0])

    def dfs(r, c):
        if r < 0 or r >= m or c < 0 or c >= n:
            return
        if grid[r][c] != "1":
            return
        grid[r][c] = "0"
        dfs(r + 1, c)
        dfs(r - 1, c)
        dfs(r, c + 1)
        dfs(r, c - 1)

    count = 0
    for r in range(m):
        for c in range(n):
            if grid[r][c] == "1":
                count += 1
                dfs(r, c)
    return count

这里没有撤销 grid[r][c] = "0"，因为它不是在试方案，而是在标记“这块陆地已经属于某个岛”。如果恢复成 1，外层扫描会再次计数，答案反而错。

常见错误：错因、后果和修法

debug

N 皇后只检查列

错因：按行递归后误以为只剩列冲突。

后果：同斜线皇后会互相攻击。

修法：同时检查 row-col 和 row+col。

撤销集合不完整

错因：只恢复棋盘，忘记恢复列或斜线集合。

后果：后续合法位置被误判为冲突。

修法：做选择改了几个状态，撤销时就恢复几个状态。

数独成功后继续回退

错因：没有用布尔返回值终止搜索。

后果：找到解后又撤销，最终棋盘可能为空或不完整。

修法：后续成功就返回 True，一路向上停止。

数独宫格下标算错

错因：只用 r % 3、c % 3，没定位宫格左上角。

后果：检查的不是同一个 3×3 宫。

修法：宫格起点是 (r//3)*3 和 (c//3)*3。

岛屿 DFS 不标记访问

错因：以为边界检查够了。

后果：相邻陆地互相递归，可能无限循环。

修法：进入陆地后立刻沉没或标记 visited。

岛屿题错误恢复地图

错因：把 flood fill 误当成构造方案回溯。

后果：同一岛被重复计数。

修法：计数类岛屿题访问后不恢复。

错误示例：岛屿访问后恢复Python
grid[r][c] = "0"
dfs(r + 1, c)
dfs(r - 1, c)
grid[r][c] = "1"   # 错误：外层扫描会再次把它当作新岛

易混概念

compare

约束回溯 vs 普通枚举

普通枚举 主要关心选或不选。

约束回溯 每次选择前要验证当前局面。

判断：题目是否有“不能冲突、不能重复、必须合法”的规则。

回溯恢复 vs 访问标记

恢复 用在试方案，失败后要换选择。

标记 用在遍历区域，处理过就不再处理。

判断：后续分支是否需要看到原状态。

岛屿 DFS vs BFS

DFS 递归或栈走完整块陆地。

BFS 队列层层扩散，也能遍历同一连通块。

判断：只要不是求最短层数，DFS 写法通常更短。
