def dfs(state):
 if finished(state):
 record_answer()

# N 皇后：一行只放一个 {#nqueen}
<!-- tag: trace -->

N 皇后的行已经被题目固定了，所以递归层只负责当前行选哪一列。要记的是三个占用集合：列、主对角线、副对角线。这样你一看候选列，就能马上知道它会不会撞上之前放过的皇后。

先看一个最小但不省略机制的例子：`n = 4`，先在 `(0, 1)` 放一个皇后。这个选择会把 `cols` 写成 `{1}`，把 `diag1` 写成 `{-1}`，把 `diag2` 写成 `{1}`。下一层来到第 1 行时，`(1, 1)` 会因为列冲突直接跳过，`(1, 3)` 才值得继续试。这个例子里最重要的不是答案，而是你能看到“写了什么，下一层就挡住什么”。

$$d_1 = r - c,\quad d_2 = r + c$$
 这里的 `r` 是行号，`c` 是列号。主对角线上的格子 `r-c` 相同，副对角线上的格子 `r+c` 相同。

N 皇后回溯代码Python

```python
def solve_n_queens(n):
 cols = set()
 diag1 = set()
 diag2 = set()
 board = [["."] * n for _ in range(n)]
 answers = []

 def dfs(row):
 if row == n:
 answers.append(["".join(line) for line in board])
 return

 for col in range(n):
 d1 = row - col
 d2 = row + col
 if col in cols or d1 in diag1 or d2 in diag2:
 continue

 cols.add(col)
 diag1.add(d1)
 diag2.add(d2)
 board[row][col] = "Q"

 dfs(row + 1)

 board[row][col] = "."
 cols.remove(col)
 diag1.remove(d1)
 diag2.remove(d2)

 dfs(0)
 return answers
```

# 数独：空格、候选和回退 {#sudoku}
<!-- tag: trace -->

数独和 N 皇后一样在构造答案，但合法性检查换成了行、列和 3×3 宫。递归通常先找一个空格，再从 1 到 9 试数。只要某条分支已经填完所有空格，就立刻返回 True，让上层停止继续回退。

还是看一个具体空格。假设你现在要填 `(0, 2)`，这一行里已有 `{5, 3, 7}`，这一列里已有 `{8}`，这个宫里已有 `{1, 9}`。`5` 会在同行被挡住，`6` 会在同宫被挡住，`4` 先能写进去。填进去以后不是结束，而是给下一层制造新的局面；如果后面卡住，就把这个位置恢复成 `.` 再换下一个数字。

$$\text{box\_r} = (r // 3) \times 3,\quad \text{box\_c} = (c // 3) \times 3$$
 `r` 和 `c` 是当前空格的行列号。先算出 3×3 宫的左上角，再扫这个宫里的 9 个格子。

数独求解代码Python

```python
def solve_sudoku(board):
 def find_empty():
 for r in range(9):
 for c in range(9):
 if board[r][c] == ".":
 return r, c
 return None

 def legal(r, c, ch):
 for i in range(9):
 if board[r][i] == ch:
 return False
 if board[i][c] == ch:
 return False

 br = (r // 3) * 3
 bc = (c // 3) * 3
 for rr in range(br, br + 3):
 for cc in range(bc, bc + 3):
 if board[rr][cc] == ch:
 return False
 return True

 def dfs():
 empty = find_empty()
 if empty is None:
 return True

 r, c = empty
 for ch in "123456789":
 if not legal(r, c, ch):
 continue
 board[r][c] = ch
 if dfs():
 return True
 board[r][c] = "."
 return False

 dfs()
```

# 岛屿 DFS：走完连通块 {#island}
<!-- tag: fill -->

岛屿题的 DFS 在走连通块，不是在试方案。你进到一个 `1`，就立刻把它标记成 `0` 或写进 visited，这样四个方向再绕回来时就会停住。数量题和面积题都依赖这个动作。

如果地图是 `1 1 0 / 1 0 1`，从左上角进来后，右边和下边都还会继续扩散，碰到 `0` 就停。这里要做的是把同一块岛上的每个格子都收进来。等你把这块区域改成 `0`，外层扫描再看到它，就知道它已经处理过了。

岛屿数量代码Python

```python
def num_islands(grid):
 rows, cols = len(grid), len(grid[0])

 def dfs(r, c):
 if r < 0 or r >= rows or c < 0 or c >= cols:
 return
 if grid[r][c] != "1":
 return

 grid[r][c] = "0"
 dfs(r + 1, c)
 dfs(r - 1, c)
 dfs(r, c + 1)
 dfs(r, c - 1)

 count = 0
 for r in range(rows):
 for c in range(cols):
 if grid[r][c] == "1":
 count += 1
 dfs(r, c)
 return count
```

# 这里先不追的变体 {#boundary}

同一条主线能覆盖很多题，但不是所有变体都该在这一页里展开。你现在要先学会“选对模板”，不是先把模板堆到最省时间的版本。太早追优化，通常只会把状态写法弄乱，然后把调试难度抬高。

这一页现在怎么处理
为什么先不展开
以后回到哪里


N 皇后位运算
先用集合写清楚状态
位运算只是更快的占用编码，不改主逻辑
先把列和对角线判断写顺，再看优化


数独最少候选位
先按“找一个空格、试一个数字”写
MRV 的核心是挑最难的空格，先追它只会分散注意力
当你已经能稳定写出回退后再加


DLX / 精确覆盖
不展开
这已经是另一套建模方式了，不是本页的入门层
等你能把数独按行列宫写稳，再看它


BFS flood fill
只提思路，不改主线
这页要先把“标记访问”和“是否恢复”讲透
后面遇到网格题再比较队列写法


并查集 / 连通块预处理
不展开
它解决的是另一种批量连通性问题，不是单次 DFS 入口
等你已经知道何时该沉没、何时该回退再接

如果你现在一上来就想把这些优化一起写，最可能出现的问题不是慢，而是状态读写顺序先错了。先把能跑通的版本写出来，再决定要不要压缩状态或者换更强的搜索顺序。

# 常见误区 {#mistakes}
<!-- tag: debug -->

看症状时先分两类：答案少，多半是合法性检查漏了；答案重复或死循环，多半是状态没有被正确改写。下面这张表把最常见的几处卡点直接连到修法。

题型
常见错法
外在症状
修法


N 皇后
只检查列
同斜线皇后被漏掉
同时维护 `cols`、`r-c` 和 `r+c`


N 皇后
回退时只改棋盘，不改集合
后续合法位置被误判
做了几个状态改写，就恢复几个状态


数独
找到一个解后还继续回退
答案被撤掉，或者最后棋盘不完整
成功分支一路 `return True`


数独
宫格起点算错
检查的不是同一个 3×3 宫
`(r // 3) * 3` 和 `(c // 3) * 3` 先算对


岛屿 DFS
不先标记访问
四方向绕回去，甚至递归栈爆掉
进入格子后立刻沉没或打标记


岛屿 DFS
访问后又恢复成 `1`
同一块陆地被重复计数
计数题保持访问痕迹，不回退

位运算、候选数最少优先、DLX、BFS flood fill 都可以提速，但它们不会替你补这五个判断。先把检查顺序和改写时机写对，再谈优化。

# 下一步怎么接 {#transfer}
<!-- tag: transfer -->

这页的判断顺序可以直接带去做新题。只要题目在“试一个合法方案”这边，N 皇后和数独的模板就有用；只要题目在“把一个连通块走完”这边，岛屿 DFS、被围绕的区域、单词搜索就能接上。下一页进入 DP 后，重心会换成“当前状态依赖谁”，不再主要看合法性。

如果答案是“是”

是
按回溯写，成功后再决定是否继续搜


题目只要连通块数量或面积？
是
按 flood fill 写，访问后保留痕迹


当前选择会不会破坏已有状态？
会
先做合法性检查，再进入下一层


走完后要不要恢复？
要
构造方案就回退，遍历区域就不回退

如果你做完这一页还是会把回溯写成一堆模板，回来看状态账本；如果你已经能分出恢复和标记，就可以去 18 看 DP 怎么记状态。

# 练习 {#practice}
<!-- tag: diagnostic -->

<!-- practice-intro -->
<!-- end-practice-intro -->

**状态**先说这一步处理的是行、空格还是格子。
**候选**再说这一层能试哪些选择。
**合法性**然后看冲突、越界和重复访问。
**动作**最后决定回退、早停还是沉没。

<!-- practice-group: 一、基础判断 -->
<!-- quick-review: 看这一组 先别写代码，先判断当前层负责什么。 -->

### 1 choice

**Q:** 1. N 皇后按行递归时，每一层最适合负责什么？

- [ ] 给当前行选一个列位置
- [ ] 把整张棋盘重新扫描一遍
- [ ] 寻找最短路径

Answer: 给当前行选一个列位置
Explain: 按行递归时，每层只需要决定这一行放在哪一列。

### 2 true/false

**Q:** 2. 数独只检查同行就够了。

- [ ] 对
- [ ] 错

Answer: 错
Explain: 数独要同时看同行、同列和同宫，单看同行不够。

### 3 choice

**Q:** 3. N 皇后主对角线对应哪条表达式？

- [ ] r-c
- [ ] r+c
- [ ] r*c

Answer: r-c
Explain: 主对角线上的格子 r-c 相同。

### 4 choice

**Q:** 4. 岛屿 DFS 进入一个陆地格子后通常怎么做？

- [ ] 立刻改成 0 或标记 visited
- [ ] 等四个方向都走完再改
- [ ] 把它再改回 1

Answer: 立刻改成 0 或标记 visited
Explain: 进入陆地后立刻写访问痕迹，才能防止四方向绕回去。

### 5 choice

**Q:** 5. 数独里，宫格左上角的行号应该怎么写？

- [ ] 先把 r 除以 3 并向下取整，再乘 3
- [ ] 先把 r 取余 3，再乘 3
- [ ] 直接用 r // 3

Answer: 先把 r 除以 3 并向下取整，再乘 3
Explain: 先定位 3×3 宫的左上角，再扫描这个宫。

### 6 choice

**Q:** 6. 数独找到一个完整解后为什么要立刻 `return True`？

- [ ] 让上层停止继续回退
- [ ] 为了多试几个答案
- [ ] 为了把 board 变空

Answer: 让上层停止继续回退
Explain: 数独找到一个解后要一路返回 True，不再继续试别的分支。

### 7 fill

**Q:** 7. 岛屿 DFS 的沉没写法通常先把格子改成 ______。

Answer: 0
Explain: 岛屿数量题里，把访问过的陆地改成 0，外层扫描才不会重复计数。

### 13 choice

**Q:** 13. 题目要求“把一整块相邻陆地都处理完”，更像哪种动作？
