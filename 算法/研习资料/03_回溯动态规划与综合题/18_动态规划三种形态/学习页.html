dynamic programming

从重复递归到按表计算

动态规划的入口不是公式，而是重复子问题。暴力递归会把同一个状态算很多遍；备忘录把算过的状态存起来；迭代 DP 再把这些状态按照依赖顺序从小到大填出来。三种形态解决的是同一个问题，只是计算方式不同。

本页先用斐波那契和爬楼梯建立最小直觉，再把重点放到状态定义、base case、选择、转移方程和遍历顺序。你应能看出一个题能不能用 DP，也能解释为什么状态转移写反、初始化漏掉或遍历顺序不对会让答案错。

学完应能完成的事

  1. 说清暴力递归、备忘录、迭代 DP 的关系。
  2. 能从“函数返回什么”定义状态。
  3. 能写出爬楼梯的三种形态代码。
  4. 能解释 base case 和遍历顺序为什么不能随便写。
  5. 为 LIS、编辑距离、背包等后续模型建立入口。

主线地图

mental model

动态规划常从“穷举所有选择”开始。只要一个状态的答案能由更小状态推出来，并且同一个小状态会被反复用到，就有机会用 DP。你可以把 DP 看成对穷举过程的压缩：先保留正确的状态含义，再减少重复计算。

暴力递归

直接按问题定义写函数，逻辑最接近人脑穷举，但会重复计算。

用途：先确认状态和转移是不是对的。

备忘录

递归不变，增加缓存。每个状态第一次算，之后直接查。

用途：最容易从暴力递归改成可通过的写法。

迭代 DP

把递归调用改成按依赖顺序填表。适合减少栈深度，也更容易做空间压缩。

用途：面试和提交时常见的最终形态。

术语表

terms
术语直觉准确说法常见误解
状态子问题是谁。能唯一描述当前子问题所需信息的变量。把状态写成一堆变量，却不能回答它表示什么。
选择当前能怎么走。从当前状态通向其他状态的决策。把题目给的每个数字都叫状态，忽略选择。
base case最小问题的答案。递归终止或 DP 表初始化的已知答案。只写循环，不处理 0、1、空串等边界。
状态转移大问题怎么用小问题算。描述状态之间依赖关系的方程或代码。记住公式但说不出每一项的含义。
重叠子问题同一小问题被反复问。递归树中多个分支会计算相同状态。所有递归题都以为能 DP。
最优子结构小问题最优能组成大问题最优。问题的最优解可由子问题最优解组合得到。只要有最大最小字样就套 DP。

知识链路

chain

DP 的第一步是定义状态。一个好的状态定义，应该能用一句话讲清楚。例如 dp[i] 表示“到达第 i 阶的方法数”，或者 dp[i] 表示“以 nums[i] 结尾的最长递增子序列长度”。这句话写不清，后面的转移通常也会飘。

第二步是找选择。爬楼梯里，最后一步可能从第 i-1 阶走 1 步，也可能从第 i-2 阶走 2 步，所以 dp[i] = dp[i-1] + dp[i-2]。这里的加法不是凭空来的，而是两类来源互不重叠，方法数可以相加。

第三步是决定计算形态。暴力递归最接近问题定义，但会重复；备忘录保留递归结构，避免重复；迭代 DP 需要明确依赖顺序，先算小状态，再算大状态。三种形态之间能互相翻译，说明你真的理解了状态和转移。

本页的判断句

  • 先说状态含义，再写转移。
  • 递归重复算同一状态，才值得加备忘录。
  • 填表顺序必须保证依赖项已知。
  • base case 是 DP 表的地基，不能靠样例凑。
爬楼梯状态：dp[i] = 到达第 i 阶的方法数
转移：dp[i] = dp[i - 1] + dp[i - 2]，因为最后一步只能来自 i-1 或 i-2

暴力递归：先把问题讲对

recursion

暴力递归的价值是清楚。它不一定高效，但能逼你回答“这个函数返回什么”。以爬楼梯为例，ways(n) 表示到达第 n 阶有多少种走法。最后一步要么从 n-1 来，要么从 n-2 来，于是函数可以写成两个子问题相加。

例题一：暴力递归Python
def climbStairs(n):
    def ways(i):
        if i == 0:
            return 1
        if i < 0:
            return 0
        return ways(i - 1) + ways(i - 2)

    return ways(n)

ways(0) = 1 的含义是“刚好走到目标，有一种完成方式”。i < 0 返回 0，表示走过头不是合法方案。这两个 base case 决定了递归如何停下。它的问题是 ways(3)、ways(2) 这类状态会被多次计算。

唯一状态只有 0 到 8，但暴力递归会反复进入相同状态。备忘录要解决的正是这件事。

你应能判断：如果递归树里没有重复状态，缓存就不是主要矛盾。

备忘录：递归结构不变，结果记住

memo

备忘录写法保留了暴力递归的思考方式，只是在每次计算前先查缓存。如果状态已经算过，直接返回缓存值；如果没算过，按递归公式算出结果再存起来。它的好处是改动小，尤其适合刚从暴力递归推完状态转移时使用。

例题二：备忘录递归Python
def climbStairs(n):
    memo = {}

    def ways(i):
        if i == 0:
            return 1
        if i < 0:
            return 0
        if i in memo:
            return memo[i]
        memo[i] = ways(i - 1) + ways(i - 2)
        return memo[i]

    return ways(n)

memo 的键必须能唯一表示状态。这里状态只有 i，所以键就是整数。如果后续学到编辑距离，状态会变成两个指针 (i, j)；背包问题里可能是 (i, capacity)。缓存键和状态定义不一致，是 DP 题里很隐蔽的错误。

迭代 DP：按依赖顺序填表

bottom-up

迭代 DP 把递归调用反过来：既然 dp[i] 依赖 dp[i-1] 和 dp[i-2]，那就先填小的，再填大的。这里最重要的是遍历顺序。你不能先算 dp[10]，因为它依赖的两个状态还没有值。

例题三：迭代 DP 表Python
def climbStairs(n):
    dp = [0] * (n + 1)
    dp[0] = 1
    if n >= 1:
        dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]

这个表里，dp[0] 和 dp[1] 是 base case。循环从 2 开始，因为 0 和 1 已知。若把循环从 0 开始，代码不是越界，就是用未定义的值推后面的状态。DP 表并不神秘，它只是把递归状态按顺序摆出来。

正确示例：空间压缩Python
def climbStairs(n):
    prev2, prev1 = 1, 1
    for i in range(2, n + 1):
        cur = prev1 + prev2
        prev2, prev1 = prev1, cur
    return prev1

空间压缩的前提是当前状态只依赖有限几个旧状态。这里 dp[i] 只依赖前两项，所以可以用两个变量替代表。若一个问题需要整行、整列或多个维度的历史状态，就不能随便压缩。

状态与转移：后续模型的入口

state design

后续会遇到 LIS、编辑距离、最大子数组、LCS、背包等经典 DP。它们看起来题面差很多，但入口仍然是状态定义。LIS 常用 dp[i] 表示以 nums[i] 结尾的最长递增子序列长度；编辑距离常用 dp[i][j] 表示两个前缀之间的最少操作次数；背包常用 dp[i][w] 表示只看前 i 个物品、容量为 w 时的最优值。

题型状态常见写法选择最容易错的点
LISdp[i]：以 i 结尾的长度接在哪个 j 后面把“以 i 结尾”误写成“前 i 个整体最优”
编辑距离dp[i][j]：两个前缀的距离插入、删除、替换或不动空串初始化漏掉
最大子数组dp[i]：以 i 结尾的最大和接前面或从当前重开把子数组和子序列混淆
背包dp[i][w]：前 i 个物品容量 w选或不选当前物品一维压缩时遍历方向写反

你不需要在这一页掌握所有模型，但要记住检查顺序：先定义状态，再找选择，再写转移，再定 base case，最后确定遍历顺序。跳过状态定义直接套公式，后面遇到变形题会很快失控。

常见错误：错因、后果和修法

debug

状态定义含糊

错因：只写 dp[i]，不说它代表什么。

后果：转移方程看似能跑，遇到变形题就错。

修法：先写一句“dp[i] 表示……”。

base case 漏掉

错因：只看一般转移，没处理最小问题。

后果：越界、答案偏 1、空输入错误。

修法：单独检查 0、1、空串、空数组。

备忘录键不完整

错因：状态有两个变量，只缓存其中一个。

后果：不同子问题互相覆盖，答案随机错。

修法：缓存键必须覆盖完整状态，如 (i, j)。

填表顺序反了

错因：没看状态依赖谁。

后果：用未计算的值推出当前状态。

修法：让依赖项先出现；二维表要检查行列方向。

把贪心当 DP

错因：只凭局部最大或局部最小做决定。

后果：局部最优不能保证整体最优。

修法：说明状态如何覆盖未来影响。

空间压缩过早

错因：还没弄清依赖关系就改成一维。

后果：覆盖仍要使用的旧状态。

修法：先写二维或完整表，再根据依赖压缩。

错误示例：base case 不稳Python
def climbStairs(n):
    dp = [0] * (n + 1)
    dp[1] = 1
    dp[2] = 2     # n = 1 时这里会越界
    for i in range(3, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]

易混概念

compare

递归 vs DP

递归 是函数调用自己，解决问题的表达方式。

DP 是利用状态转移和缓存减少重复计算。

判断：只有递归不代表一定是 DP。

备忘录 vs 迭代

备忘录 从目标往下问，需要就算。

迭代 从 base case 往上填，按顺序算。

判断：两者状态和转移应当一致。

状态 vs 选择

状态 描述当前子问题。

选择 描述从这个子问题可以怎么转移。

判断：状态回答“我是谁”，选择回答“我从哪来或到哪去”。
