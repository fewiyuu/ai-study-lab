lis / edit distance / lcs

把序列问题写成可推的状态

LIS、编辑距离、LCS 都是序列 DP。它们看起来题面不同，一个问递增子序列，一个问字符串修改次数，一个问两个字符串共同保留多少字符；但它们都在回答同一个问题：当前答案能不能由更小的前缀或更早的结尾推出。

这页不要求你把公式硬背下来。重点是学会定义状态、找选择、写 base case、确定遍历顺序。只要这四步稳，换成最长回文子序列、最大子数组、两个字符串匹配类问题，也能沿着同一条路推。

学完后能做的事

  1. 能解释 LIS 的 `dp[i]` 为什么表示“以 i 结尾”。
  2. 能写出编辑距离二维 DP，并说清插入、删除、替换对应哪个格子。
  3. 能写出 LCS 的二维 DP，并区分它和最长公共子串。
  4. 能判断 DP 遍历方向是否满足依赖关系。
  5. 能用状态定义检查参考代码有没有漏初始化或越界。

主线地图

mental model

经典序列 DP 有两种常见视角：一种是“以某个位置结尾”，另一种是“处理到某个前缀”。LIS 多用结尾视角，因为要知道当前数能接在哪些更小的数后面；编辑距离和 LCS 多用前缀视角，因为每次只看两个字符串当前最后一个字符。

状态定义格子含义
选择来源从哪转移
base case空前缀 / 单点
遍历顺序先算依赖
答案位置max 或右下角

LIS：以位置结尾

`dp[i]` 表示必须选择 `nums[i]`，并且以它作为最后一个数时，最长递增子序列长度是多少。它不表示前 `i` 个数的全局最优。

判断句：题目要求子序列不连续，只要相对顺序保留，就可以跳过中间元素。

编辑距离：前缀变前缀

`dp[i][j]` 表示 `word1[:i]` 变成 `word2[:j]` 的最少操作数。插入、删除、替换分别来自左、上、左上三个方向。

判断句：空串变成长度 `j` 的串，需要插入 `j` 次。

LCS：两个前缀共同保留

`dp[i][j]` 表示 `text1[:i]` 和 `text2[:j]` 的最长公共子序列长度。字符相等时接上左上角，不等时取上方或左方较大值。

判断句：LCS 允许跳字符，不要求共同部分连续。

术语表

terms
术语直觉准确说法容易错在哪
状态一个格子保存一个子问题答案。`dp[i]` 或 `dp[i][j]` 的精确定义，必须包含边界。只写“前 i 个”却不知道是否必须选第 i 个。
转移从已经算好的小问题推出当前格子。根据选择关系写出的递推式。当前格子依赖还没算出来的格子。
base case最小问题的答案。数组初值、空串行列、单元素长度等。二维表忘记初始化第 0 行和第 0 列。
子序列可以删掉一些元素，剩下的顺序不变。不要求连续，只要求相对顺序。把子序列当成子数组或子串。
答案位置最后从哪里读答案。LIS 常取 `max(dp)`，编辑距离和 LCS 常看右下角。误以为所有 DP 都返回最后一个格子。

知识链路

chain

做 LIS 时，先问“如果我必须把 `nums[i]` 放进答案，它前面能接谁？”能接的前驱是所有 `j < i` 且 `nums[j] < nums[i]` 的位置。每个前驱都给出一个候选长度 `dp[j] + 1`，取最大值就是 `dp[i]`。因为每个 `dp[i]` 只描述以 `i` 结尾，所以最终答案要在所有 `dp[i]` 中取最大。

做编辑距离时，状态不是“到第几个字符”，而是“一个前缀变成另一个前缀”。如果当前两个字符相同，右下角可以直接继承左上角；如果不同，就要从插入、删除、替换三种操作里选代价最小的那条。二维表的第 0 行和第 0 列不是装饰，它们负责处理空串。

做 LCS 时，两个前缀的最后字符相等，就说明这个字符可以被共同保留，答案来自 `dp[i-1][j-1] + 1`。如果不相等，至少有一个最后字符不能同时进入公共子序列，于是比较“丢掉 text1 最后一个”和“丢掉 text2 最后一个”的结果。

本页的判断句

  • `dp[i]` 若表示以 `i` 结尾，答案通常要取最大值。
  • `dp[i][j]` 若表示两个前缀，空前缀那一行一列要先想清。
  • 子序列允许跳过元素，子数组和子串要求连续。
  • 遍历顺序只看依赖方向，不看题目文字顺序。
LCS 转移：若 a[i-1] = b[j-1]，则 dp[i][j] = dp[i-1][j-1] + 1；否则 dp[i][j] = max(dp[i-1][j], dp[i][j-1])。

核心模板

templates

先看 LIS 的朴素写法。它慢一点，但最适合理解“以 i 结尾”的状态定义。只要当前数大于前驱，就能接在前驱的递增子序列后面。

正确示例：LIS O(n²) 状态定义Python
def lengthOfLIS(nums):
    if not nums:
        return 0
    dp = [1] * len(nums)
    for i in range(len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)

`dp[i]` 初始为 1，因为只选 `nums[i]` 自己也能形成长度为 1 的递增子序列。返回 `max(dp)`，不是 `dp[-1]`。

编辑距离的二维表多一行一列。第 0 行表示空串变成目标前缀，第 0 列表示源前缀变成空串。少了这层，后面的 `i-1`、`j-1` 会很难写稳。

正确示例：编辑距离Python
def minDistance(a, b):
    m, n = len(a), len(b)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                insert = dp[i][j - 1] + 1
                delete = dp[i - 1][j] + 1
                replace = dp[i - 1][j - 1] + 1
                dp[i][j] = min(insert, delete, replace)
    return dp[m][n]

插入看左边，删除看上边，替换看左上角。名字可以按当前字符串视角理解，但格子方向要固定。

渐进例子

examples
状态视角切换

切换模型，观察答案为什么从不同位置读取。

以某个位置结尾

LIS 的每个格子只承诺“我必须以这里结尾”。所以最后要在所有结尾里找最大值。

看见一个 DP 表，先问格子承诺了什么。

LIS 还有一个常见优化写法，维护一个 `tails` 数组：`tails[k]` 表示长度为 `k+1` 的递增子序列，当前能做到的最小结尾值。它不保存真实序列，但能保存足够判断长度的信息。

进阶示例：LIS 二分优化Python
from bisect import bisect_left

def lengthOfLIS(nums):
    tails = []
    for x in nums:
        i = bisect_left(tails, x)
        if i == len(tails):
            tails.append(x)
        else:
            tails[i] = x
    return len(tails)

替换 `tails[i]` 不是删掉答案，而是把同长度序列的结尾变小，给后续数字留下更多接上的机会。

正确示例：最长公共子序列Python
def longestCommonSubsequence(a, b):
    m, n = len(a), len(b)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[m][n]

不相等时，不是把两个字符都扔掉，而是分别尝试丢掉其中一个，保留更好的结果。

错误示例：把 LIS 答案写成 dp[-1]Python
def lengthOfLIS(nums):
    dp = [1] * len(nums)
    for i in range(len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return dp[-1]   # 错：最长递增子序列不一定以最后一个元素结尾

边界与复杂度

limits

空输入

LIS 如果直接 `dp = [1] * len(nums)` 后返回 `max(dp)`，空数组会报错。先判断空输入。

重复字符和重复数字

严格递增用 `<`，非递减才用 `<=`。LCS 可以处理重复字符，但只能按顺序匹配，不能乱配。

空间压缩

二维 DP 可以压缩，但必须确认当前格子需要左、上、左上哪个旧值。没想清前不要急着压成一维。

模型状态常见复杂度答案位置
LIS 朴素`dp[i]` 以 i 结尾O(n²) / O(n)`max(dp)`
LIS 二分`tails[k]` 长度 k+1 的最小结尾O(n log n) / O(n)`len(tails)`
编辑距离`dp[i][j]` 前缀变前缀O(mn) / O(mn)`dp[m][n]`
LCS`dp[i][j]` 两前缀公共子序列O(mn) / O(mn)`dp[m][n]`

常见错误：错因、后果和修法

debug

状态定义含糊

错因：只说 `dp[i]` 是“最长长度”，没说是否必须选 `i`。

后果：转移来源和答案位置都会错。

修法：把格子含义写成一句完整的话。

LIS 用 `dp[-1]`

错因：把“以最后一位结尾”当成全局最优。

后果：`[10,9,2,5,3,7,101,18]` 这类输入会错。

修法：朴素 LIS 返回 `max(dp)`。

编辑距离漏初始化

错因：没有处理空串到非空串。

后果：第一行第一列错误，后续格子全被污染。

修法：`dp[i][0]=i`，`dp[0][j]=j`。

LCS 写成公共子串

错因：字符不等时把当前长度清零。

后果：只能找连续匹配，跳字符的答案丢失。

修法：不等时取上方和左方最大值。

二分 LIS 用错边界

错因：严格递增和非递减混用 `bisect_left` / `bisect_right`。

后果：重复数字时长度偏大或偏小。

修法：严格递增用 `bisect_left` 替换第一个大于等于的位置。

易混概念

compare

子序列 vs 子数组 / 子串

子序列可以跳过元素，只保留顺序。

子数组 / 子串必须连续。

判断：题目写 subsequence，一般允许跳；写 subarray / substring，就要连续。

LCS vs 最长公共子串

LCS不连续，不等时取上或左。

公共子串必须连续，不等时当前格子清零。

判断：能不能跳字符，是两者的分界。

编辑距离 vs 汉明距离

编辑距离允许插入、删除、替换，长度可以变化。

汉明距离只比较等长字符串对应位置不同的数量。

判断：允许改长度，就不是汉明距离。
