序列 DP

把 LIS、编辑距离与 LCS 放进同一张状态图

这三道题表面不同：一个看结尾，一个看两个前缀的编辑成本，一个看两个前缀能共同保留多少字符。真正要练的是同一件事，先把状态句子写对，再按依赖把表填出来。

读这页时，先看格子在描述什么，再看它从哪里读值，最后才看代码长什么样。状态、转移、base case、答案位置一旦对齐，公式就不容易漂。

状态地图

map

这三类题的分界，不在题面长什么样，而在状态能不能一句话说清。LIS 是单位置结尾；编辑距离和 LCS 是双前缀；只要这层看准了，后面的递推式就会自己落到对应的格子上。

LIS 单位置结尾

当前格子只承诺“我以 nums[i] 结尾”。它读的是前面的更小数字，写进去的是当前能接到的最长长度，最后从所有结尾里取最大值。

编辑距离 前缀变前缀

dp[i][j] 读左、上、左上三个旧格子。它们分别对应插入、删除、替换或保持不动，答案从右下角读。

LCS 两个前缀共同保留

字符相等时接左上角，不等时保留更大的那一边。这里算的是“共同保留了多长”，不是“连续对上了多长”。

模型 状态视角 依赖方向 答案位置 一句提醒
LIS 以位置结尾 看前驱，筛更小的数 max(dp) 最后一个位置不一定是最优结尾。
编辑距离 前缀到前缀 左、上、左上 dp[m][n] 空串那一行和一列是地基。
LCS 共同保留 相等接左上，不等取最大 dp[m][n] 它允许跳字符，不要求连续。

术语表

terms
术语 直觉 准确说法 错了会怎样
状态 一个格子保存一个子问题答案。 dp[i] 或 dp[i][j] 的完整含义。 状态句子漂了，转移和答案位置就会跟着错。
转移 从旧答案推出当前答案。 当前格子读哪些旧格子，怎么合并它们。 依赖没算完就写当前格子，表面能跑，结果会偏。
base case 最小问题的已知答案。 空串、单元素、第一行、第一列这些起点。 起点没立住，后面的归纳也站不稳。
依赖顺序 先算被别人要用的格子。 填表顺序必须覆盖当前格子的所有来源。 顺序反了，读到的会是旧值或空值。
答案位置 最后从哪里读结果。 LIS 常读 max(dp)，二维前缀常读右下角。 把局部状态当成全局答案，常见就是 dp[-1] 误用。
子序列 可以跳过元素，只保留顺序。 不要求连续，只要求相对顺序不变。 把它当成子串，会把题意缩窄一截。

状态账本

ledger

先把“读、写、错”写成账本

DP 不只是公式，还包括数据从哪里来、写到哪里去、哪些格子必须先算好。把这三件事写清，错误会比直接背模板更容易定位。

如果你只能记住一件事，就记住这句：先说格子表示什么，再说它读谁，最后说它写谁。

调试时先看这三层

  • 状态句子有没有把边界说进去。
  • 当前格子读到的旧值是不是已经算好。
  • 答案是不是从真正的全局位置读出来。
模型 进入前 写入后 读什么 写什么 错位症状
LIS 只知道前面的若干结尾长度 得到以 i 结尾的长度 所有更小的前驱 dp[j] dp[i] 最后一个元素不一定在最优序列里。
编辑距离 左上方的前缀代价已算完 得到当前两个前缀的最少编辑数 左、上、左上 dp[i][j] 第一行第一列一旦漏掉，整张表都会偏。
LCS 两个前缀的更小状态已算完 得到当前共同保留长度 左上、上、左 dp[i][j] 把“不等”写成清零，会变成子串问题。

小例子

examples

LIS：[4, 10, 4, 3, 8, 9]

朴素 DP 看的是“以当前数结尾时，最长能接多长”。这组数据里，最后一个 9 不是全局最优的起点，但它能接在 8 后面，所以能把长度推到 3。

$$ dp[i] = 1 + \max\{dp[j] \mid 0 \le j < i,\ nums[j] < nums[i]\} $$

如果没有符合条件的前驱，dp[i] 就是 1。这里的 i 不是“前 i 个”的总答案，而是“以 i 结尾”的局部答案。

i nums[i] 最好的前驱 dp[i] 提醒
04无1单独成段
110424 可以接到 10 前面
24无1相等不能接，严格递增要用 <
33无1前面没有更小的数
484 或 32接在更长的前驱后面
5983全局答案不一定在最后一格

编辑距离与 LCS：同一张二维表，方向不同

horse -> ros 说明编辑距离里左、上、左上分别对应插入、删除、替换；abcde -> ace 说明 LCS 允许跳字符，不要求连续。

$$ dp[i][j] = \begin{cases} dp[i-1][j-1], & a[i-1] = b[j-1] \\ 1 + \min(dp[i][j-1], dp[i-1][j], dp[i-1][j-1]), & a[i-1] \ne b[j-1] \end{cases} $$

左边像插入，上边像删除，左上像替换。空串那一行和一列不是附属行，它们是整张表的起点。

$$ dp[i][j] = \begin{cases} dp[i-1][j-1] + 1, & a[i-1] = b[j-1] \\ \max(dp[i-1][j], dp[i][j-1]), & a[i-1] \ne b[j-1] \end{cases} $$

LCS 只在字符相等时接上去；不相等时只是在“丢掉左边最后一个”与“丢掉上边最后一个”之间选更好的一边。

模型 左 上 左上 一眼识别法
编辑距离 插入 删除 替换或匹配 要把一个串变成另一个串。
LCS 丢掉 text1 的最后一个字符 丢掉 text2 的最后一个字符 字符相等时一起保留 只问共同保留了多少。

手算追踪

trace

这一节不追公式外形，只看三个小输入是怎么把状态一点点推出来的。你会发现，表格里最重要的从来不是数字多漂亮，而是每个数字都能回到一个明确的来源。

LIS：[4, 10, 4, 3, 8, 9]

朴素 LIS 的关键，是把每个位置都当成一个“结尾候选”。4、10、8、9 这些数分别给出不同长度的结尾链，最后要比较的是所有结尾，而不是最后一个位置本身。

编辑距离：ab -> acb

这个例子足够小，能把左、上、左上三种来源都看见。ab 变到 acb 的最短路只有 1 步，但你要先把空串的那一圈补好，表才会长得起来。

LCS：abc -> ac

这里最容易卡住的点，是看到 b 以后以为当前格子要清零。实际该做的，是把上方和左方的结果保留下来，再让后面的 c 把长度接上去。

i nums[i] 可接的前驱 dp[i] 这一步在提醒什么
04无1单个元素本身就是长度 1 的答案。
11042当前数只接比它小的前驱。
24无1相等不算严格递增，不能接。
33无1更小的前驱已经没有了。
484、32局部最优不一定在最右边。
5983最后一个位置能延长，但全局答案还是要看所有结尾。
前缀 状态值 来源 为什么这样写
dp[0][0]0空串到空串起点先立住，后面才有依赖。
dp[1][0]1删掉 a 的第 1 个字符前缀变空串，只能一路删除。
dp[0][1]1往空串里插入 a空串变前缀，只能一路插入。
dp[1][1]0a == a末尾字符已经对上，直接继承左上角。
dp[2][2]1把 b 换成 c一次替换就够，不必绕远路。
dp[2][3]1在 ab 后插入 c当前格子会读左边，而不是凭空生成。
前缀 状态值 来源 为什么这样写
dp[1][1]1a == a相等就把左上角接上去。
dp[2][1]1丢掉 b末尾不同的时候，先看上方和左方。
dp[2][2]1继续保留 a当前格子仍然延续之前的最好结果。
dp[3][1]1前面的 a 已经够了c 还没出现时，长度不能乱涨。
dp[3][2]2c == c最后一个字符相等，长度往左上角加 1。

这三张表连起来看

如果你在 LIS 里坚持看 dp[-1]，会把“最后一个结尾”和“全局最优”混在一起；如果你在编辑距离里漏掉第一行和第一列，空串状态就会断；如果你在 LCS 不等时清零，模型会立刻掉成最长公共子串。三类错法都能在这三张表里找到自己的影子。

练习时别急着写整段代码，先把这一页里的三张表拆开复述一遍。只要你能讲清“读什么、写什么、为什么这样写”，代码通常就只剩下排版问题。

核心模板

templates

三份代码只差状态句子。LIS 盯一个位置，编辑距离和 LCS 盯两个前缀。代码读起来像是在做三件不同的事，实际上都在做“读旧值、写当前格子、把答案从正确的位置取出来”。

LIS：朴素状态Python
def lengthOfLIS(nums):
    if not nums:
        return 0
    dp = [1] * len(nums)
    for i in range(len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)
编辑距离：前缀表Python
def minDistance(a, b):
    m, n = len(a), len(b)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1):
        dp[i][0] = i
    for j in range(n + 1):
        dp[0][j] = j
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = 1 + min(dp[i][j - 1], dp[i - 1][j], dp[i - 1][j - 1])
    return dp[m][n]
LCS：共同保留Python
def longestCommonSubsequence(a, b):
    m, n = len(a), len(b)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
    return dp[m][n]
错误示例：把局部结尾当全局答案Python
def lengthOfLIS_bad(nums):
    if not nums:
        return 0
    dp = [1] * len(nums)
    for i in range(len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return dp[-1]   # 错：最后一个位置只是最后一个结尾，不是全局最优

手算走法

walkthrough

这一节只做一件事：跟着一个很小的输入，把 DP 表从左到右、从上到下推出来。你会看到，表格里的每一格都能回到一个明确的前驱，而不是凭感觉填一个“看起来差不多”的数。

编辑距离：ab 到 acb

先放边界，再填内部格。空串到前缀的那一圈已经告诉你，插入和删除的成本是多少；真正要看的，是字符相等时怎么沿着左上角继承，不相等时怎么从左、上、左上里取最小值。

∅ a ac acb
∅ 0 1 2 3
a 1 0 1 2
ab 2 1 1 1

看最后一格 dp[2][3]。它的来源是 ab 和 acb 的最后一个字符都已经对上，所以可以沿着左上角继承，得到 1。这个例子能把“插入、删除、替换”三种动作都压进同一张表里。

如果你把第一行和第一列省掉，ab 变成空串、空串变成 acb 这两类起点就没有地方写，后面所有格子都会失去来源。这就是编辑距离里最先该检查的地方。

LCS：abc 到 ac

LCS 的手算顺序和编辑距离差不多，差别在于它只关心共同保留的长度。末尾字符相等时加 1，不相等时保留上方和左方里更大的那个值。你只要在这一步把“连续”误加进去，表就会立刻走偏。

∅ a ac
∅ 0 0 0
a 0 1 1
ab 0 1 1
abc 0 1 2

这张表里，b 没有让任何格子清零。它只是被跳过去了。到最后一格时，c 和 c 对上，长度从左上角接到 2。这个例子足够小，能直接看出 LCS 和最长公共子串的分界。

出错时会长什么样

编辑距离漏掉 base case，第一圈就断；LCS 在不等时清零，表格会变成公共子串；LIS 把最后一格当答案，最优结尾一旦落在中间就会丢。三类错误都能通过这一节的追踪表直接看出来。

对应的填表顺序Python
for i in range(1, m + 1):
    for j in range(1, n + 1):
        # 先确保 dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1] 已经算好
        if a[i - 1] == b[j - 1]:
            dp[i][j] = dp[i - 1][j - 1] + 1
        else:
            dp[i][j] = min(dp[i][j - 1] + 1, dp[i - 1][j] + 1, dp[i - 1][j - 1] + 1)

这一节和练习怎么接

做题时先把这两张表中的一种复写出来，再去看代码。只要你能说清楚“这一格从哪来、这一格写完后代表什么”，练习里的代码阅读和错误诊断就会轻松很多。后面的题不要求你背完整张表，但会要求你在几秒内判断应该看左、上、左上，还是看所有结尾。

题面翻译

selection

这一节只训练一件事：看到题面以后，先把它翻成状态句子，再决定读哪一侧的旧值。你如果先被代码样子带走，很容易把“最长递增子序列”写成“前 i 个的最优值”，把“最长公共子序列”写成“最长公共子串”，或者把“编辑距离”漏成只会替换的一维版本。

题面一：LIS

题目说的是“递增”而不是“连续”，这意味着状态要盯住一个结尾。你在看的是“当前数能接到谁后面”，不是“前面所有数加起来有多长”。

如果题面给的是一个数组，而你必须回答“以某个位置结尾的最优值”，那就先往 dp[i] 的方向想。

题面二：编辑距离

题目说的是“把一个串改成另一个串”，这时状态天然是两个前缀之间的关系。你要先看最后一个字符的三种去向，再决定当前格子怎么写。

如果题面在问“最少几步”，当前格子通常是在描述代价，不是在描述长度。

题面三：LCS

题目说的是“公共子序列”，这四个字已经把“连续”排除了。末尾字符相等时才接上去，不等时就跳过一边继续看。

如果题面写的是 common subsequence，你就不能把不等分支写成清零。

题面信号 先翻成什么 常见误判 纠正方式
“以某个位置结尾” 单位置状态 把全局答案塞进局部格子 先问自己：这格子只描述当前结尾还是整个数组。
“把 A 改成 B” 前缀到前缀 只保留一个维度 把两串的最后一个字符都纳入状态句子。
“公共子序列” 共同保留长度 误写成公共子串 不等时保留上方和左方，不要清零。
“最少操作次数” 代价状态 只看长度变化 把插入、删除、替换都算进同一张表。
一个很短的选型顺序text
先看题面最后一步在说什么
如果只盯一个位置的结尾，就先想 dp[i]
如果要比较两个串，就先想 dp[i][j]
如果题目写了 subsequence，就不要把它改成连续
如果题目写了 minimum operations，就把状态翻成代价

为什么这一段值得单独看

这三道题最常见的错，不是公式记错，而是状态选错。LIS 里一旦把结尾视角写成全局视角，你会被 dp[-1] 误导；编辑距离里一旦漏掉前缀视角，空串那圈就没法落地；LCS 里一旦把 subsequence 误看成 substring，整个不等分支都会被你清空。

做后面的练习时，先拿这一节的表做一次自检，再动手写代码。你只要能把题面翻成状态，很多题就已经走完一半了。

修法路线

repair

这节把常见错法直接改回可用版本。别把它当成补充说明，它其实是从“看懂题”走到“写对代码”的最后一公里。你只要会看这张修法表，练习区里的错题就能很快定位。

LIS：只看最后一格

错法是把 dp[-1] 当答案。修回去的方式很简单：先问所有结尾，再取最大值。你在检查时要看的是“当前位置是否只是一个候选结尾”。

编辑距离：只写对角线

错法是看到字符相等就直接从左上角抄，看到字符不等就不知道该怎么走。修回去要补上左、上、左上三种来源，并且先把空串那一圈写完整。

LCS：把不等写成清零

错法是把公共子序列误写成公共子串。修回去时，不等就保留上方和左方中更大的那个结果，不要把已经算好的长度扔掉。

看到的错误 先查哪里 修回去后要看到什么 对应练习
return dp[-1] 答案位置 所有结尾里取最大值 题 7、题 10
没有第 0 行和第 0 列 base case 空串到前缀、前缀到空串都能落地 题 3、题 9
不等时写 0 转移分支 保留上方和左方更大的结果 题 8、题 12
把 < 改成 <= 却没改题意 题目类型 从严格递增换成非递减，或者明确题意不允许这么改 题 5、题 11

一个很短的修错流程

  1. 先圈出状态句子。
  2. 再找当前格子的依赖方向。
  3. 接着看 base case 和答案位置。
  4. 最后才动手改代码。

为什么这一段够用

因为这页真正要教的，不是某个固定解法，而是你看到错代码后该怎么往回找。LIS、编辑距离和 LCS 的错误几乎都能落在状态、边界、顺序和答案位置四个点上。只要你能把这四个点说清，后面的练习就有检查标准，不会只剩“感觉对了”。

最小反例

counterexample

如果你想快速判断自己有没有把状态说对，先拿一个最小反例去撞。反例不需要大，够把错误暴露出来就行。LIS 里最常见的 dp[-1] 误用，用一个三元素数组就能看出来。

LIS 反例：[2, 5, 1]

朴素 DP 的结果是 dp = [1, 2, 1]。如果你直接返回 dp[-1]，拿到的是 1；如果你返回 max(dp)，拿到的是 2。这个反例说明，局部结尾和全局答案不是一回事。

位置 数值 dp 解释
021单独成段。
1522 可以接到 5 前面。
211最后一位自己成段，但不是全局最优。

编辑距离反例：空串开头

如果你把 dp[0][j] 忘了，"" -> "acb" 这一类输入就没有起点。表里最先坏掉的不是中间格，而是第一行。

如果你把 LCS 的不等分支写成 0，abc -> ac 会被误写成公共子串。反例会直接告诉你：跳字符的能力还在不在。

怎么用这类反例

做练习时，先拿一个最小反例去试你自己的状态句子：它能不能把错误暴露出来。如果暴露不出来，说明状态定义还不够精确，或者答案位置还没对齐。

反例 暴露的错法 修回去的动作 回到哪一节
[2, 5, 1]把 dp[-1] 当答案改成读所有结尾的最大值修法路线 / 练习 7
"" -> "acb"漏掉首行首列先写 base case手算走法 / 练习 3、9
abc -> ac把 LCS 写成子串不等时保留上方和左方更大值手算走法 / 练习 8

先别追

boundary

LIS 的二分优化

tails 只保存“长度为 k 的最小结尾”，它比朴素 DP 更快，但先把 end-state 的意思弄稳，再去看这个技巧。

状态压缩

空间能不能压，先看依赖有没有被下一轮再次使用。没把依赖方向想清楚之前，别急着把二维表压成一维。

区间 DP、树形 DP、位运算优化

这些模型后面都会回来，但它们的状态空间更大，今天先不追。先把“状态句子 + 依赖方向 + 答案位置”这三件事练稳。

现在先做 现在先放下 以后回来时看什么
把状态句子说完整 先不追更快的优化 依赖有没有被更小的表达式覆盖
把 base case 和答案位置对齐 先不追复杂的状态压缩 哪些旧值还会被下一轮再读一次
把连续、子序列、前缀分开 先不追相近但不同的模型名 题面到底允许不允许跳字符

常见错误

debug

LIS 直接返回 dp[-1]

错因是把“最后一个结尾”误当成“全局最优”。后果是只要最优序列没落在最后一个元素上，答案就会错。

LCS 不等就清零

这会把题变成最长公共子串。LCS 允许跳字符，不等时应该比较上方和左方，而不是把当前格子抹成 0。

编辑距离漏掉第一行和第一列

空串到前缀、前缀到空串就是 base case。少了这两条，后面的 i-1、j-1 会失去落点。

Trace:
LIS returns dp[-1].
Symptom: the last element is only one end-state.
Next check: read max(dp), not the final cell.

迁移到后续

transfer

遇到新题时先问这三句

  1. 状态是在看一个位置，还是两个前缀。
  2. 最后一步是接前驱，还是删掉一个字符，还是把两个前缀对齐。
  3. 答案是读所有结尾、读右下角，还是别的位置。

下一节为什么是背包

背包会继续复用这套问法，只是状态从“位置/前缀”换成“容量/选择”。你如果已经能把 LIS、编辑距离、LCS 的状态句子说稳，下一节会更像换坐标，而不是换思路。

题面信号 先选视角 本页对应模型 下一步
要求严格递增、只看一个结尾 结尾视角 LIS 找前驱，取最大值。
要把一个串改成另一个串 前缀视角 编辑距离 看左、上、左上。
要保留共同部分，但允许跳字符 前缀视角 LCS 相等接左上，不等取更大。
题面强调连续 先别套 LCS 公共子串 / 子数组 确认题目到底允不允许跳过字符。

7. 为什么朴素 LIS 不能直接 return dp[-1]？

参考答案

要点： dp[-1] 只表示“以最后一个元素结尾”的那条链，不代表所有结尾里的最优值。

判断标准： 必须提到“最后一个元素不一定在最优序列里”，并指出正确答案位置是 max(dp)。

error diagnosis错误诊断

8. 为什么 LCS 在字符不等时写成 dp[i][j] = 0 会变成另一个问题？

参考答案

要点： 这会把题变成最长公共子串。LCS 允许跳字符，所以不等时要在上方和左方之间选更好的一边。

判断标准： 需要明确说出“连续”这个限制是被你误加进去的。

error diagnosis错误诊断

9. 编辑距离如果没有第 0 行和第 0 列，会先在哪里出错？

参考答案

要点： 第 0 行和第 0 列是空串和空前缀的 base case。没有它们，i - 1、j - 1 的来源会失去落点，表的第一圈就会断掉。

判断标准： 至少说出“空前缀状态缺失”与“依赖无法落地”中的一项。

11. 遇到一个新题时，什么时候先想“以位置结尾”，什么时候先想“前缀对前缀”？

参考答案

要点： 如果题目只关心一个位置的结尾、最后一个元素能不能接上去，就先想 dp[i] 的结尾视角；如果题目在比较两个串、两个前缀或改写字符串，就先想 dp[i][j] 的前缀视角。

判断标准： 需要提到“题目最后一步在说什么”，而不是只背某个模型名。

synthesis迁移表达

12. 用 5-7 句话解释 LIS、编辑距离、LCS 的状态、转移、答案位置和常见坑。

参考答案

参考方向： 先说 LIS 是“以位置结尾”的一维状态，答案读 max(dp)；再说编辑距离和 LCS 都是双前缀，但一个看最少操作，一个看共同保留长度；然后点出编辑距离的左、上、左上，和 LCS 的相等接左上、不等取最大；最后收一个坑，比如 dp[-1]、空串初始化或把 LCS 写成子串。

判断标准： 四个要素都要出现：状态、转移、答案位置、一个具体坑。
