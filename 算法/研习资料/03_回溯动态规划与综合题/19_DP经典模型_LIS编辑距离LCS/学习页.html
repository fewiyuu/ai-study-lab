DP 经典模型：LIS、编辑距离与 LCS

这三道题表面不同：一个看结尾，一个看两个前缀的编辑成本，一个看两个前缀能共同保留多少字符。真正要练的是同一件事，先把状态句子写对，再按依赖把表填出来。

状态地图

这三类题的分界，不在题面长什么样，而在状态能不能一句话说清。LIS 是单位置结尾；编辑距离和 LCS 是双前缀；只要这层看准了，后面的递推式就会自己落到对应的格子上。

术语表

terms
术语直觉准确说法错了会怎样
状态一个格子保存一个子问题答案。dp[i] 或 dp[i][j] 的完整含义。状态句子漂了，转移和答案位置就会跟着错。
转移从旧答案推出当前答案。当前格子读哪些旧格子，怎么合并它们。依赖没算完就写当前格子，表面能跑，结果会偏。
base case最小问题的已知答案。空串、单元素、第一行、第一列这些起点。起点没立住，后面的归纳也站不稳。
依赖顺序先算被别人要用的格子。填表顺序必须覆盖当前格子的所有来源。顺序反了，读到的会是旧值或空值。
答案位置最后从哪里读结果。LIS 常读 max(dp) ，二维前缀常读右下角。把局部状态当成全局答案，常见就是 dp[-1] 误用。
子序列可以跳过元素，只保留顺序。不要求连续，只要求相对顺序不变。把它当成子串，会把题意缩窄一截。

状态账本

ledger

先把“读、写、错”写成账本

DP 不只是公式，还包括数据从哪里来、写到哪里去、哪些格子必须先算好。把这三件事写清，错误会比直接背模板更容易定位。

如果你只能记住一件事，就记住这句：先说格子表示什么，再说它读谁，最后说它写谁。

调试时先看这三层

  • 状态句子有没有把边界说进去。
  • 当前格子读到的旧值是不是已经算好。
  • 答案是不是从真正的全局位置读出来。
模型进入前写入后读什么写什么错位症状
LIS只知道前面的若干结尾长度得到以 i 结尾的长度所有更小的前驱 dp[j]dp[i]最后一个元素不一定在最优序列里。
编辑距离左上方的前缀代价已算完得到当前两个前缀的最少编辑数左、上、左上dp[i][j]第一行第一列一旦漏掉，整张表都会偏。

小例子

examples

LIS：[4, 10, 4, 3, 8, 9]

朴素 DP 看的是“以当前数结尾时，最长能接多长”。这组数据里，最后一个 9 不是全局最优的起点，但它能接在 8 后面，所以能把长度推到 3。

$$

dp[i] = 1 + \max\{dp[j] \mid 0 \le j < i,\ nums[j] < nums[i]\}

$$

如果没有符合条件的前驱，dp[i] 就是 1。这里的 i “以 i 结尾”的局部答案。

编辑距离与 LCS：同一张二维表，方向不同

horse -> ros 说明编辑距离里左、上、左上分别对应插入、删除、替换；abcde -> ace 说明 LCS 允许跳字符，不要求连续。

$$

dp[i][j] =

\begin{cases}

dp[i-1][j-1], & a[i-1] = b[j-1] \\

1 + \min(dp[i][j-1], dp[i-1][j], dp[i-1][j-1]), & a[i-1] \ne b[j-1]

\end{cases}

$$

左边像插入，上边像删除，左上像替换。空串那一行和一列不是附属行，它们是整张表的起点。

$$

dp[i][j] =

\begin{cases}

dp[i-1][j-1] + 1, & a[i-1] = b[j-1] \\

\max(dp[i-1][j], dp[i][j-1]), & a[i-1] \ne b[j-1]

\end{cases}

$$

LCS 只在字符相等时接上去；不相等时只是在“丢掉左边最后一个”与“丢掉上边最后一个”之间选更好的一边。

模型左上左上一眼识别法
编辑距离插入删除替换或匹配要把一个串变成另一个串。
LCS丢掉 text1 的最后一个字符丢掉 text2 的最后一个字符字符相等时一起保留只问共同保留了多少。

核心模板

templates

三份代码只差状态句子。LIS 盯一个位置，编辑距离和 LCS 盯两个前缀。代码读起来像是在做三件不同的事，实际上都在做“读旧值、写当前格子、把答案从正确的位置取出来”。

代码
def lengthOfLIS(nums):
 if not nums:
 return 0
 dp = [1] * len(nums)
 for i in range(len(nums)):
 for j in range(i):
 if nums[j] < nums[i]:
 dp[i] = max(dp[i], dp[j] + 1)
 return max(dp)

编辑距离：前缀表Python复制

代码
def minDistance(a, b):
 m, n = len(a), len(b)
 dp = [[0] * (n + 1) for _ in range(m + 1)]
 for i in range(m + 1):
 dp[i][0] = i
 for j in range(n + 1):
 dp[0][j] = j
 for i in range(1, m + 1):
 for j in range(1, n + 1):
 if a[i - 1] == b[j - 1]:
 dp[i][j] = dp[i - 1][j - 1]
 else:
 dp[i][j] = 1 + min(dp[i][j - 1], dp[i - 1][j], dp[i - 1][j - 1])
 return dp[m][n]

LCS：共同保留Python复制

代码
def longestCommonSubsequence(a, b):
 m, n = len(a), len(b)
 dp = [[0] * (n + 1) for _ in range(m + 1)]
 for i in range(1, m + 1):
 for j in range(1, n + 1):
 if a[i - 1] == b[j - 1]:
 dp[i][j] = dp[i - 1][j - 1] + 1
 else:
 dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
 return dp[m][n]

手算走法

walkthrough

这一节只做一件事：跟着一个很小的输入，把 DP 表从左到右、从上到下推出来。你会看到，表格里的每一格都能回到一个明确的前驱，而不是凭感觉填一个“看起来差不多”的数。

LCS：abc 到 ac

LCS 的手算顺序和编辑距离差不多，差别在于它只关心共同保留的长度。末尾字符相等时加 1，不相等时保留上方和左方里更大的那个值。你只要在这一步把“连续”误加进去，表就会立刻走偏。

这张表里，b 没有让任何格子清零。它只是被跳过去了。到最后一格时，c 和 c 对上，长度从左上角接到 2。这个例子足够小，能直接看出 LCS 和最长公共子串的分界。

出错时会长什么样

编辑距离漏掉 base case，第一圈就断；LCS 在不等时清零，表格会变成公共子串；LIS 把最后一格当答案，最优结尾一旦落在中间就会丢。三类错误都能通过这一节的追踪表直接看出来。

对应的填表顺序Python复制

for i in range(1, m + 1):
 for j in range(1, n + 1):
 # 先确保 dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1] 已经算好
 if a[i - 1] == b[j - 1]:
 dp[i][j] = dp[i - 1][j - 1] + 1
 else:
 dp[i][j] = min(dp[i][j - 1] + 1, dp[i - 1][j] + 1, dp[i - 1][j - 1] + 1)

这一节和练习怎么接

做题时先把这两张表中的一种复写出来，再去看代码。只要你能说清楚“这一格从哪来、这一格写完后代表什么”，练习里的代码阅读和错误诊断就会轻松很多。后面的题不要求你背完整张表，但会要求你在几秒内判断应该看左、上、左上，还是看所有结尾。

题面翻译

selection

这一节只训练一件事：看到题面以后，先把它翻成状态句子，再决定读哪一侧的旧值。你如果先被代码样子带走，很容易把“最长递增子序列”写成“前 i 个的最优值”，把“最长公共子序列”写成“最长公共子串”，或者把“编辑距离”漏成只会替换的一维版本。

题面二：编辑距离

题目说的是“把一个串改成另一个串”，这时状态天然是两个前缀之间的关系。你要先看最后一个字符的三种去向，再决定当前格子怎么写。

如果题面在问“最少几步”，当前格子通常是在描述代价，不是在描述长度。

题面信号先翻成什么常见误判纠正方式
“以某个位置结尾”单位置状态把全局答案塞进局部格子先问自己：这格子只描述当前结尾还是整个数组。
“把 A 改成 B”前缀到前缀只保留一个维度把两串的最后一个字符都纳入状态句子。
“公共子序列”共同保留长度误写成公共子串不等时保留上方和左方，不要清零。
“最少操作次数”代价状态只看长度变化把插入、删除、替换都算进同一张表。
代码
先看题面最后一步在说什么
如果只盯一个位置的结尾，就先想 dp[i]
如果要比较两个串，就先想 dp[i][j]
如果题目写了 subsequence，就不要把它改成连续
如果题目写了 minimum operations，就把状态翻成代价

修法路线

repair

这节把常见错法直接改回可用版本。别把它当成补充说明，它其实是从“看懂题”走到“写对代码”的最后一公里。你只要会看这张修法表，练习区里的错题就能很快定位。

编辑距离：只写对角线

错法是看到字符相等就直接从左上角抄，看到字符不等就不知道该怎么走。修回去要补上左、上、左上三种来源，并且先把空串那一圈写完整。

看到的错误先查哪里修回去后要看到什么对应练习
return dp[-1]答案位置所有结尾里取最大值题 7、题 10
没有第 0 行和第 0 列base case空串到前缀、前缀到空串都能落地题 3、题 9
不等时写 0转移分支保留上方和左方更大的结果题 8、题 12
把 < 改成 <= 却没改题意题目类型从严格递增换成非递减，或者明确题意不允许这么改题 5、题 11

一个很短的修错流程

  1. 先圈出状态句子。
  2. 再找当前格子的依赖方向。
  3. 接着看 base case 和答案位置。
  4. 最后才动手改代码。

为什么这一段够用

因为这页真正要教的，你看到错代码后该怎么往回找。LIS、编辑距离和 LCS 的错误几乎都能落在状态、边界、顺序和答案位置四个点上。只要你能把这四个点说清，后面的练习就有检查标准，不会只剩“感觉对了”。

最小反例

counterexample

如果你想快速判断自己有没有把状态说对，先拿一个最小反例去撞。反例不需要大，够把错误暴露出来就行。LIS 里最常见的 dp[-1] 误用，用一个三元素数组就能看出来。

编辑距离反例：空串开头

如果你把 dp[0][j] 忘了，"" -> "acb" 这一类输入就没有起点。表里最先坏掉的第一行。

如果你把 LCS 的不等分支写成 0，abc -> ac 会被误写成公共子串。反例会直接告诉你：跳字符的能力还在不在。

怎么用这类反例

做练习时，先拿一个最小反例去试你自己的状态句子：它能不能把错误暴露出来。如果暴露不出来，说明状态定义还不够精确，或者答案位置还没对齐。

tails 只保存“长度为 k 的最小结尾”，它比朴素 DP 更快，但先把 end-state 的意思弄稳，再去看这个技巧。

区间 DP、树形 DP、位运算优化

这些模型后面都会回来，但它们的状态空间更大，今天先不追。先把“状态句子 + 依赖方向 + 答案位置”这三件事练稳。

常见错误

debug

LIS 直接返回 dp[-1]

错因是把“最后一个结尾”误当成“全局最优”。后果是只要最优序列没落在最后一个元素上，答案就会错。

编辑距离漏掉第一行和第一列

空串到前缀、前缀到空串就是 base case。少了这两条，后面的 i-1、j-1 会失去落点。

迁移到后续

transfer

遇到新题时先问这三句

  1. 状态是在看一个位置，还是两个前缀。
  2. 最后一步是接前驱，还是删掉一个字符，还是把两个前缀对齐。
  3. 答案是读所有结尾、读右下角，还是别的位置。

下一节为什么是背包

背包会继续复用这套问法，只是状态从“位置/前缀”换成“容量/选择”。你如果已经能把 LIS、编辑距离、LCS 的状态句子说稳，下一节会更像换坐标，而不是换思路。

题面信号先选视角本页对应模型下一步
要求严格递增、只看一个结尾结尾视角LIS找前驱，取最大值。
要把一个串改成另一个串前缀视角编辑距离看左、上、左上。
要保留共同部分，但允许跳字符前缀视角LCS相等接左上，不等取更大。
题面强调连续先别套 LCS公共子串 / 子数组确认题目到底允不允许跳过字符。
