# 术语表 {#terms}
<!-- tag: terms -->

| 术语 | 直觉 | 准确说法 | 错了会怎样 |
| --- | --- | --- | --- |
| 状态 | 一个格子保存一个子问题答案。 | dp[i] 或 dp[i][j] 的完整含义。 | 状态句子漂了，转移和答案位置就会跟着错。 |
| 转移 | 从旧答案推出当前答案。 | 当前格子读哪些旧格子，怎么合并它们。 | 依赖没算完就写当前格子，表面能跑，结果会偏。 |
| base case | 最小问题的已知答案。 | 空串、单元素、第一行、第一列这些起点。 | 起点没立住，后面的归纳也站不稳。 |
| 依赖顺序 | 先算被别人要用的格子。 | 填表顺序必须覆盖当前格子的所有来源。 | 顺序反了，读到的会是旧值或空值。 |

dp[i][j] =
\begin{cases}

<!-- tag: trace -->

这一节不追公式外形，只看三个小输入是怎么把状态一点点推出来的。你会发现，表格里最重要的从来每个数字都能回到一个明确的来源。

LIS：`[4, 10, 4, 3, 8, 9]`
朴素 LIS 的关键，是把每个位置都当成一个“结尾候选”。`4`、`10`、`8`、`9` 这些数分别给出不同长度的结尾链，最后要比较的是所有结尾，而不是最后一个位置本身。

# 核心模板 {#templates}
<!-- tag: templates -->

三份代码只差状态句子。LIS 盯一个位置，编辑距离和 LCS 盯两个前缀。代码读起来像是在做三件不同的事，实际上都在做“读旧值、写当前格子、把答案从正确的位置取出来”。

LIS：朴素状态Python复制

```python
def lengthOfLIS(nums):
 if not nums:
 return 0
 dp = [1] * len(nums)
 for i in range(len(nums)):
 for j in range(i):
 if nums[j] < nums[i]:
 dp[i] = max(dp[i], dp[j] + 1)
 return max(dp)
```

编辑距离：前缀表Python复制

```python
def minDistance(a, b):
 m, n = len(a), len(b)
 dp = [[0] * (n + 1) for _ in range(m + 1)]
 for i in range(m + 1):
 dp[i][0] = i
 for j in range(n + 1):
 dp[0][j] = j
 for i in range(1, m + 1):
 for j in range(1, n + 1):
 if a[i - 1] == b[j - 1]:
 dp[i][j] = dp[i - 1][j - 1]
 else:
 dp[i][j] = 1 + min(dp[i][j - 1], dp[i - 1][j], dp[i - 1][j - 1])
 return dp[m][n]
```

LCS：共同保留Python复制

```python
def longestCommonSubsequence(a, b):
 m, n = len(a), len(b)
 dp = [[0] * (n + 1) for _ in range(m + 1)]
 for i in range(1, m + 1):
 for j in range(1, n + 1):
 if a[i - 1] == b[j - 1]:
 dp[i][j] = dp[i - 1][j - 1] + 1
 else:
 dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])
 return dp[m][n]
```

错误示例：把局部结尾当全局答案Python复制

```python
def lengthOfLIS_bad(nums):
 if not nums:
 return 0
 dp = [1] * len(nums)
 for i in range(len(nums)):
 for j in range(i):
 if nums[j] < nums[i]:
 dp[i] = max(dp[i], dp[j] + 1)
 return dp[-1] # 错：最后一个位置只是最后一个结尾，不是全局最优
```

# 题面翻译 {#selection}
<!-- tag: selection -->

这一节只训练一件事：看到题面以后，先把它翻成状态句子，再决定读哪一侧的旧值。你如果先被代码样子带走，很容易把“最长递增子序列”写成“前 i 个的最优值”，把“最长公共子序列”写成“最长公共子串”，或者把“编辑距离”漏成只会替换的一维版本。

题面一：LIS
题目说的是“递增”而不是“连续”，这意味着状态要盯住一个结尾。你在看的是“当前数能接到谁后面”，不是“前面所有数加起来有多长”。
如果题面给的是一个数组，而你必须回答“以某个位置结尾的最优值”，那就先往 `dp[i]` 的方向想。


题面二：编辑距离
题目说的是“把一个串改成另一个串”，这时状态天然是两个前缀之间的关系。你要先看最后一个字符的三种去向，再决定当前格子怎么写。
如果题面在问“最少几步”，当前格子通常是在描述代价，不是在描述长度。


题面三：LCS
题目说的是“公共子序列”，这四个字已经把“连续”排除了。末尾字符相等时才接上去，不等时就跳过一边继续看。
如果题面写的是 common subsequence，你就不能把不等分支写成清零。

题面信号
先翻成什么
常见误判
纠正方式


“以某个位置结尾”
单位置状态
把全局答案塞进局部格子
先问自己：这格子只描述当前结尾还是整个数组。


“把 A 改成 B”
前缀到前缀
只保留一个维度
把两串的最后一个字符都纳入状态句子。


“公共子序列”
共同保留长度
误写成公共子串
不等时保留上方和左方，不要清零。


“最少操作次数”
代价状态
只看长度变化
把插入、删除、替换都算进同一张表。

一个很短的选型顺序text复制

```text
先看题面最后一步在说什么
如果只盯一个位置的结尾，就先想 dp[i]
如果要比较两个串，就先想 dp[i][j]
如果题目写了 subsequence，就不要把它改成连续
如果题目写了 minimum operations，就把状态翻成代价
```

## box info
### 为什么这一段值得单独看
这三道题最常见的错，状态选错。LIS 里一旦把结尾视角写成全局视角，你会被 `dp[-1]` 误导；编辑距离里一旦漏掉前缀视角，空串那圈就没法落地；LCS 里一旦把 subsequence 误看成 substring，整个不等分支都会被你清空。
做后面的练习时，先拿这一节的表做一次自检，再动手写代码。你只要能把题面翻成状态，很多题就已经走完一半了。

# 修法路线 {#repair}
<!-- tag: repair -->

这节把常见错法直接改回可用版本。别把它当成补充说明，它其实是从“看懂题”走到“写对代码”的最后一公里。你只要会看这张修法表，练习区里的错题就能很快定位。

LIS：只看最后一格
错法是把 `dp[-1]` 当答案。修回去的方式很简单：先问所有结尾，再取最大值。你在检查时要看的是“当前位置是否只是一个候选结尾”。


编辑距离：只写对角线
错法是看到字符相等就直接从左上角抄，看到字符不等就不知道该怎么走。修回去要补上左、上、左上三种来源，并且先把空串那一圈写完整。


LCS：把不等写成清零
错法是把公共子序列误写成公共子串。修回去时，不等就保留上方和左方中更大的那个结果，不要把已经算好的长度扔掉。

# 常见错误 {#mistakes}
<!-- tag: debug -->

LIS 直接返回 `dp[-1]`
错因是把“最后一个结尾”误当成“全局最优”。后果是只要最优序列没落在最后一个元素上，答案就会错。


LCS 不等就清零
这会把题变成最长公共子串。LCS 允许跳字符，不等时应该比较上方和左方，而不是把当前格子抹成 0。


编辑距离漏掉第一行和第一列
空串到前缀、前缀到空串就是 base case。少了这两条，后面的 `i-1`、`j-1` 会失去落点。

### 1 choice

**Q:** 1. 朴素 LIS 中 `dp[i]` 的准确含义是哪一个？

- [ ] 前 i 个数的最长递增子序列长度
- [ ] 以 nums[i] 结尾的最长递增子序列长度
- [ ] 从 i 开始的最长递增子序列长度

Answer: 以 nums[i] 结尾的最长递增子序列长度
Explain: LIS 的朴素状态必须以 i 结尾的局部答案。

### 2 true/false

**Q:** 2. LCS 要求匹配的字符在两个字符串里都连续出现。

- [ ] 对
- [ ] 错

Answer: 错
Explain: LCS 允许跳字符，不要求连续；要求连续的是最长公共子串。

### 3 fill

**Q:** 3. 空串变成长度 `j` 的前缀，`dp[0][j]` 应该写成什么？

Answer: j,j 次,插入 j 次
Explain: 空串变成长度为 j 的前缀，需要插入 j 次。

### 4 code-reading

**Q:** 4. 在 LIS 朴素写法里，`dp[i] = max(dp[i], dp[j] + 1)` 主要读的是哪个旧值？

Answer: dp[j],前驱的长度,前一个状态
Explain: 这句读的是前驱的结尾长度。

### 5 choice

**Q:** 5. 把 `nums[j] < nums[i]` 改成 `nums[j] <= nums[i]`，更接近哪一类题？

- [ ] 最长公共子串
- [ ] 非递减子序列
- [ ] 编辑距离

Answer: 非递减子序列
Explain: 把 <code><</code> 改成 <code><=</code> 后，模型从严格递增变成非递减。

### 6 choice

**Q:** 6. 当 `a[i - 1] == b[j - 1]` 时，编辑距离里当前格子最直接继承哪个位置？

- [ ] 左边
- [ ] 左上角
- [ ] 右下角

Answer: 左上角,dp[i - 1][j - 1]
Explain: 字符相等时，当前格子可以直接继承左上角。

### 7 error-diagnosis

**Q:** 7. 为什么朴素 LIS 不能直接 `return dp[-1]`？

Explain: LIS 的正确答案要看所有结尾，而不是最后一个结尾。

### 8 error-diagnosis

**Q:** 8. 为什么 LCS 在字符不等时写成 `dp[i][j] = 0` 会变成另一个问题？

Explain: LCS 的不等分支应该保留两边更大的结果，而不是抹成 0。

### 9 error-diagnosis

**Q:** 9. 编辑距离如果没有第 0 行和第 0 列，会先在哪里出错？

Explain: 空串是编辑距离的起点，少掉这一层，整张表的依赖都会断。
