0-1 / subset / complete

先分清物品能用几次

背包问题的题面经常换皮：装物品、凑金额、分割数组、硬币兑换、目标和。别急着套公式，先问三个问题：每个物品能用几次？答案问最大价值、是否可行，还是方案数量？容量维度是重量、金额、和，还是差值转换后的目标？

0-1 背包每个物品最多选一次；子集背包通常是 0-1 背包的可行性或计数版本；完全背包允许同一种物品重复使用。循环方向正是由这句话决定的。

学完后能做的事

  1. 能写出二维 0-1 背包的状态定义和转移。
  2. 能解释一维 0-1 背包为什么容量倒序。
  3. 能把分割等和子集、目标和转成子集背包。
  4. 能区分完全背包求最优值、组合数和排列数。
  5. 能诊断容量初始化、循环方向、方案数口径错误。

主线地图

mental model

背包题的主线不是“背包”两个字，而是选择集合受到容量限制。每个物品都有代价，可能还有价值。你要在容量不超的前提下，回答最优值、可行性或方案数。题目一变，`dp[j]` 的含义就要跟着变。

物品次数一次 / 无限
答案类型最优 / 可行 / 计数
容量定义重量 / 金额 / 目标和
循环方向倒序 / 正序
初始化0 / inf / true

0-1 背包

每个物品只有选或不选。二维状态常写成 `dp[i][j]`：前 `i` 个物品，在容量 `j` 内能得到的最大价值。

压缩到一维后，容量要倒序，避免当前物品在同一轮被重复使用。

子集背包

子集背包不一定有“价值”，常问能不能凑出某个和，或凑出目标和有几种选法。

布尔可行性用 `or`，方案数用加法。两者长得像，但含义完全不同。

完全背包

完全背包允许同一物品重复选。求最优值时容量正序，让当前轮更新后的 `dp[j-w]` 能继续参与本轮。

硬币兑换就是典型完全背包。

知识链路

chain

二维 0-1 背包最直观。处理第 `i` 个物品时，如果容量不够，只能不选；如果容量够，可以在“不选当前物品”和“选当前物品”之间取更大值。这个状态表把物品维度和容量维度分开，所以不容易误用当前物品。

一维压缩后，物品维度被滚动掉。为了让第 `i` 个物品只用一次，`j` 必须从大到小遍历。这样读到的 `dp[j-w]` 还是上一轮物品的结果，不会刚刚被当前物品更新过。

完全背包正好相反。它希望同一个物品可以重复参与，所以容量从小到大遍历。这样 `dp[j-w]` 可能已经在当前物品这一轮被更新过，表示“再拿一个当前物品”。

本页的判断句

  • 每个物品最多一次，一维容量倒序。
  • 物品无限次，一维容量正序。
  • 问能不能凑出目标，用布尔 DP。
  • 问几种凑法，用计数 DP，组合和排列要分清。
0-1 背包：dp[j] = max(dp[j], dp[j - w] + v)，其中 j 从容量上限倒序到 w。

核心模板

templates

先看二维 0-1 背包。它最适合用来理解选择关系：当前物品不选，答案来自上一行同容量；当前物品选，答案来自上一行剩余容量再加价值。

正确示例：二维 0-1 背包Python
def knapsack(weights, values, cap):
    n = len(weights)
    dp = [[0] * (cap + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        w, v = weights[i - 1], values[i - 1]
        for j in range(cap + 1):
            dp[i][j] = dp[i - 1][j]
            if j >= w:
                dp[i][j] = max(dp[i][j], dp[i - 1][j - w] + v)
    return dp[n][cap]

读 `i-1` 行，是为了保证当前物品只被考虑一次。

正确示例：一维 0-1 背包Python
def knapsack(weights, values, cap):
    dp = [0] * (cap + 1)
    for w, v in zip(weights, values):
        for j in range(cap, w - 1, -1):
            dp[j] = max(dp[j], dp[j - w] + v)
    return dp[cap]

倒序让 `dp[j-w]` 保持上一轮状态。若改成正序，同一个物品会被本轮反复拿。

完整追踪：同一组物品，方向一变结果就变

trace

这一段不再停在“记住倒序”这句话上，而是直接拿一个小例子把状态跑出来。物品只有三个：`(2,3)`、`(3,4)`、`(4,5)`，容量上限是 `7`。如果你先做 0-1 背包，再看完全背包，就会发现同一个转移式为什么会因为循环方向不同而变成两种题。

先看 0-1 背包的一维写法。第一件物品是重量 `2`、价值 `3`。如果你按正确方式倒序，`dp[4]` 读到的还是“上一轮的 `dp[2]`”，这个值仍然是 0，所以只能得到 `3` 一次。可是一旦你把容量写成正序，`dp[2]` 在刚刚这一轮就已经被改成 `3`，接着 `dp[4]` 再去读它，就会算出 `6`。这不是“更优”，而是把同一件物品在同一轮里拿了两次，题意已经被你改掉了。

0-1 背包的正确节奏

倒序写法的关键，不是“习惯上要倒序”，而是你希望 `dp[j-w]` 还保留前一轮的旧值。只要旧值还在，当前物品就只能参与一次。这个旧值是你在同一轮里最需要保护的东西。

把三件物品都跑完，正确答案会停在 `9`。它对应的是容量 `7` 下选第二件和第三件：`3 + 4 = 7`，价值 `4 + 5 = 9`。如果第一件物品被重复拿，`dp` 会在中间阶段就出现 `6`、`9` 这种看似很高、其实不合法的数。

为什么同一段代码在完全背包里又是对的

因为完全背包本来就允许重复拿同一种物品。你让容量正序，本质上是在说：刚刚这一轮更新出来的 `dp[j-w]`，本轮还可以继续拿来用。对于硬币兑换、最少硬币数、组合数这类题，这恰好就是题目要的语义。

所以“正序/倒序”不是一个纯粹的编码细节，而是你在明确题意：这道题到底允不允许把同一个物品连着拿几次。题意没看清，代码就会看起来很像，结果却完全跑偏。

再换成子集可行性，你看到的其实还是同一套骨架，只是状态值从“最大价值”变成了布尔值。比如分割等和子集，`dp[j]` 表示能不能凑出和为 `j`。如果数组是 `[1, 5, 11, 5]`，总和 `22`，目标就变成 `11`。初始化时先把 `dp[0] = True`，再把每个数按倒序塞进去。跑到最后，`dp[11]` 为真，说明确实存在一个子集把左半边凑出来。这个时候你如果把它写成 `max`，程序不会直接报错，但语义已经全错了，因为你要的不是“最大值更大”，而是“能不能成立”。

同样的道理也适用于方案数。完全背包里的组合数通常把物品放在外层，容量放在内层正序；排列数常常把容量放在外层，再枚举物品。它们都在做加法，但计数口径不一样。比如硬币 1、2、5 凑 5，组合只看“选了哪些币”，排列还要看“顺序怎么排”。如果你把循环顺序写反，结果看似还是一个整数，实际上统计口径已经换了。很多人做题时会以为自己只是少了一个小循环，最后却是在统计另一个问题。

0-1 背包的错误与修正Python
# 错：正序会把当前物品重复读进来
for w, v in zip(weights, values):
    for j in range(w, cap + 1):
        dp[j] = max(dp[j], dp[j - w] + v)

# 对：倒序保住上一轮的旧值
for w, v in zip(weights, values):
    for j in range(cap, w - 1, -1):
        dp[j] = max(dp[j], dp[j - w] + v)
完全背包的正确写法Python
# 允许重复使用同一种物品，所以要正序
for coin in coins:
    for j in range(coin, amount + 1):
        dp[j] = min(dp[j], dp[j - coin] + 1)

你可以把这整段追踪理解成一个判断顺序：先问“这道题能不能重复拿同一种物品”，再问“状态是最优值、可行性还是方案数”，最后才决定循环怎么写。只要这个顺序没乱，代码就不会乱。反过来，若你先抄模板再倒推题意，最容易出现的就是“代码看着对，答案始终差一点”。

最后给一个很实用的自检方法：只要你写出了背包循环，马上拿一件重量为 `2`、价值为 `3` 的物品做手算，看看容量 `4` 时到底应该是 `3` 还是 `6`。如果题意说每件物品只能用一次，那答案只能是 `3`；如果题意说可以无限次使用，那答案才可能是 `6`。这一步很短，但能把一半的方向错误直接筛掉。

练习钩子

如果你能不看代码，直接说出“为什么这个例子里 0-1 背包不能得到 6，而完全背包可以得到 6”，说明你已经把循环方向和题意绑定起来了。接下来做题时，就不要再死记“倒序=0-1，正序=完全”这句口令，而是先把题目翻译成“能不能重复用”。

题面翻译：先把句子变成状态

bridge

背包题最容易卡住的地方，不是转移式，而是第一步翻译。题目会把“能不能选”“选几次”“有几种选法”“最少要多少个”写成很像的话，但它们在 DP 里的含义完全不同。你要先把题面压缩成三件事：物品能不能重复用、`dp[j]` 代表什么、答案到底是在问最优值、可行性还是方案数。只要这三件事没钉住，后面的循环方向就只是模仿，不是理解。

以目标和为例。题目给你一串正整数，要求你在每个数前面放 `+` 或 `-`，让总和等于目标值。很多人会直接问“怎么写代码”，但更稳的问法是“我能不能把它翻译成子集和”。设正号集合和为 `P`，负号集合和为 `N`，那么 `P - N = target`，同时 `P + N = sum`。两式相加以后就得到 `P = (sum + target) / 2`。这一步的意义很直接：它把一个看起来像符号推理的问题，变成了一个标准背包：从若干数里选出一部分，使它们的和恰好等于一个容量。

先写三句，再写循环

第一句，`dp[j]` 到底表示什么。第二句，一个物品能用几次。第三句，答案是最优值、可行性，还是方案数。你只要把这三句写对，循环方向通常就已经被限定住了。

比如分割等和子集里，`dp[j]` 表示“能不能凑出和为 `j`”。这时转移必须是布尔逻辑，不是最大值；初始化必须让 `dp[0] = True`，不是 0；每个数只能用一次，所以容量要倒序。你如果把它写成“更大的价值”，程序虽然能跑，问题却已经被换掉了。

错法通常长什么样

最常见的错不是完全不会写，而是少了一层语义判断：看见 `dp[j] = dp[j] or dp[j-x]` 就默认所有子集题都能这么写；看见 `dp[j] = min(dp[j], dp[j-coin] + 1)` 就默认所有凑金额题都要正序。实际上，只有在“允许重复使用”时，正序才成立；只有在“每个数只能用一次”时，倒序才成立。

换句话说，循环方向不是记忆题，是题意翻译题。你如果能把题面先翻成“是否重复用、状态是什么、答案问什么”，代码就会自然落到正确模板里。

题目表述状态语义循环顺序初值常见失败点
能不能凑出目标和布尔可行性倒序dp[0] = True把它写成最值题。
有几种选法方案数通常倒序dp[0] = 1忘了空方案入口。
最少硬币数最小值正序无穷大把可行性初始化搬过来。
每个数只能用一次0-1 选择倒序看答案口径正序导致重复使用。
同一种物品可以重复用完全选择正序看答案口径倒序把题意写没了。

再看零钱兑换 II。它问的是组合数，不是排列数。组合只关心“选了哪些硬币”，顺序不同不算新方案；排列才关心“先拿谁后拿谁”。这意味着你不能只盯着转移式，还得盯着循环顺序。外层硬币、内层容量，通常是在统计组合；外层容量、内层硬币，才更接近排列。两个答案都可能是正整数，但它们根本不是同一个问题。

这也是为什么背包题一旦进入综合题，就特别容易让人误判。题目有时不会直接写“背包”，而是写成分割数组、硬币兑换、目标和、子集计数、路径压缩后的最值题。你若只记住模板，看到题目就急着往里塞，往往会在第一步就把语义塞错。反过来，如果你先把题面翻译成“物品次数、状态语义、答案口径”，模板就会变得很安静，不会自己出错。

目标和转换骨架Python
def target_sum(nums, target):
    total = sum(nums)
    if abs(target) > total:
        return 0
    if (total + target) % 2:
        return 0
    cap = (total + target) // 2
    dp = [0] * (cap + 1)
    dp[0] = 1
    for x in nums:
        for j in range(cap, x - 1, -1):
            dp[j] += dp[j - x]
    return dp[cap]

这段代码背后的关键不是语法，而是等价变形。你先把“正负号”翻译成“选子集”，再把“计数”翻译成“方案数 DP”，最后才有了 `dp[0] = 1` 和倒序循环。这个顺序一旦倒过来，程序往往还能运行，但统计对象已经变了。

再看一个更短的手算场景：物品只有 `(2,3)` 和 `(3,4)`，容量是 `5`。0-1 背包里，正确答案是 `4`，因为第二件物品单独拿一次就已经是最优；如果你误把一维循环写成正序，`dp[4]` 会先被第一件物品错误地抬成 `6`，接着更大的容量还会继续沿着这个错误结果扩散。这个偏差不是“少一点点”，而是状态语义已经被污染了。

完全背包的情况正好相反。若题目允许重复使用重量 `2` 的物品，那么容量 `4` 就应该得到 `6`，容量 `6` 还能继续叠到 `9`。也就是说，能否得到 `6` 不是绝对对错，而是你要先确认题目是否允许重复使用。把这个判断记成手算规则，比死背“倒序/正序”更稳。

练习钩子

你现在可以用一句话检查自己：如果题目说“顺序不同算不同方案”，那我就不能按组合数的外层顺序去写；如果题目说“每个数只能用一次”，那我就不能用完全背包的正序。把这两句和 `dp[j]` 的语义对上，背包题的大半坑就已经被提前排除了。

完全背包正序走容量

先看物品次数，再看循环方向

0-1、子集和完全背包都在改同一件事，只是“一个物品能用几次”不同。你看到题目时，先把次数和答案口径钉住，再决定一维表是倒序还是正序。

子集背包常见题是分割等和子集。总和如果为奇数，直接不可行；如果为偶数，问题变成能否选出若干数凑到总和一半。

正确示例：分割等和子集Python
def canPartition(nums):
    s = sum(nums)
    if s % 2:
        return False
    target = s // 2
    dp = [False] * (target + 1)
    dp[0] = True
    for x in nums:
        for j in range(target, x - 1, -1):
            dp[j] = dp[j] or dp[j - x]
    return dp[target]

`dp[0] = True` 表示什么都不选可以凑出 0。倒序仍然是因为每个数只能用一次。

正确示例：完全背包最少硬币数Python
def coinChange(coins, amount):
    inf = amount + 1
    dp = [inf] * (amount + 1)
    dp[0] = 0
    for coin in coins:
        for j in range(coin, amount + 1):
            dp[j] = min(dp[j], dp[j - coin] + 1)
    return -1 if dp[amount] == inf else dp[amount]

正序让同一枚面值可以反复使用。`inf` 初始化表示还凑不出来。

边界与复杂度

limits

目标转换

目标和题常要把正负号问题转成子集和：选正数集合 P，负数集合 N，有 P - N = target 且 P + N = sum，所以 P = (sum + target) / 2。

先查 |target| <= sum，再查 sum + target 是否为偶数，否则没有合法子集。

初始化含义

最大值问题常初始化 0；最小硬币数要用无穷大；可行性要用 False 并让 dp[0]=True；方案数要让 dp[0]=1。

初始化不是格式，而是在给“空选择”或“不可达状态”定义含义。

组合与排列

硬币组合数通常外层硬币、内层容量；排列数通常外层容量、内层硬币。循环顺序会改变计数口径。

同一个转移式，换循环顺序可能就不是同一道题。

为什么说 O(nW) 是伪多项式

这里的 W 是容量的数值大小，不是输入文本占用的位数。若容量写成十进制整数 1000000，它在输入里只需要大约 7 个数字字符，但 DP 表要开到一百万格，并且每个物品都要扫这些格。

所以 O(nW) 对“数值 W”是多项式，但对“输入长度”不一定是多项式。严格复杂度理论里，整数输入的规模通常按位数计，W 的位数约为 log W；算法却和 W 本身成正比，因此叫伪多项式。

刷题时仍然常接受 O(nW)，因为题目通常限制容量或总和不大。看到 W 或数组总和很大时，要警惕表格 DP 可能爆内存或超时。

复杂度判断顺序

  1. 先确定容量上限：背包容量 W、金额 amount、子集目标 T。
  2. 再数物品个数 n，看是否每个物品都要扫容量。
  3. 最后看表的维度：二维表占 nW，一维压缩占 W。
  4. 如果容量来自数组和，别只看数组长度，还要看数值总和。
背包复杂度的核心：状态数通常是“物品前缀 * 容量数值”。
O(nW) 里的 W 是数值容量，所以它是伪多项式复杂度，不是按输入位数衡量的真正多项式保证。

常见错误：错因、后果和修法

debug

0-1 背包正序循环

错因：没意识到当前轮更新会被再次读取。

后果：一个物品被重复选，答案偏大。

修法：容量从 cap 倒序到 w。

完全背包倒序循环

错因：沿用了 0-1 背包写法。

后果：同一物品只能用一次，凑不出本应可行的金额。

修法：容量从 w 正序到上限。

方案数初始化为 0 后忘记 dp[0]=1

错因：没有给“凑出 0 的空方案”留入口。

后果：所有计数都加不起来。

修法：计数题通常设置 dp[0]=1。

目标和转换漏边界

错因：直接算 (sum + target) // 2。

后果：目标绝对值超过总和、或奇偶不匹配时仍然误建 DP 表。

修法：先检查 |target| <= sum，再检查 sum + target 是否为偶数。

组合数和排列数混淆

错因：只看转移式，不看循环顺序。

后果：同一组硬币不同顺序被重复计数或漏计。

修法：组合外层物品，排列外层容量。

易混概念

compare

0-1 背包 vs 完全背包

0-1：每个物品最多一次，一维容量倒序。

完全：每个物品无限次，一维容量正序。

判断：题目说每种可重复使用，就不要倒序。

可行性 vs 方案数

可行性：`dp[j]` 是 True/False。

方案数：`dp[j]` 是数量，转移累加。

判断：问“能否”用布尔，问“多少种”用计数。

组合 vs 排列

组合：顺序不重要，例如 1+2 和 2+1 算一种。

排列：顺序重要，二者算两种。

判断：题目是否把不同顺序当不同方案。

16. 322 零钱兑换问最少硬币数，`dp[j]` 应表示什么？初值怎么设？

dp[j] 表示凑出金额 j 所需的最少硬币数。dp[0]=0，其余初始化为无穷大；每个硬币正序更新容量，转移为 dp[j]=min(dp[j], dp[j-coin]+1)。

面试

18. 用 2-3 句话解释 0-1 背包一维优化为什么倒序。

参考答案

0-1 背包每个物品只能用一次，一维优化时 dp[j] 必须从上一轮的 dp[j-w] 转移。容量倒序可以保留上一轮状态，防止当前物品在同一轮被反复使用。

面试

19. 用 2-3 句话解释完全背包为什么正序。

参考答案

完全背包允许同一物品用多次，所以一维 DP 要让本轮更新后的 dp[j-w] 继续参与转移。容量正序正好表达这种重复使用。

复杂度

20. 写出 0-1 背包、完全背包和子集背包一维 DP 的时间、空间复杂度，并说明容量上限如何影响它。

参考答案

三类一维 DP 常见时间复杂度都是 O(nW)，空间复杂度都是 O(W)，其中 W 是容量上限。容量越大，数组越长、转移次数越多；子集背包只是价值维度换成可达性或方案数。

复杂度

21. 为什么 O(nW) 中的 W 不是输入位数意义下的多项式？请用“容量数值”和“输入位数”解释。

参考答案

W 是容量的数值大小，而输入里写下 W 只需要约 log W 位。O(nW) 对容量数值是线性的，但对输入位数可能是指数级，所以叫伪多项式。

22. 用 6-9 句话讲清 0-1 背包、子集背包和完全背包的区别、循环方向、初始化、复杂度和常见错误。

参考答案

0-1 背包的物品只能选一次，dp[j] 通常表示容量 j 下的最优值、可达性或方案数，一维更新要倒序。子集背包是 0-1 背包的一类，常把目标转成容量，再用布尔或计数状态判断可行性。完全背包允许物品重复使用，所以容量正序，让本轮新状态继续参与转移。初始化要按问题含义来：最值题给不可达哨兵，方案数题通常 dp[0]=1。复杂度常写 O(nW) 时间和 O(W) 空间，但 W 是容量数值，因此是伪多项式。常见错误是 0-1 写正序、完全背包写倒序、方案数忘记 dp[0]，或把组合数循环顺序写成排列数。
