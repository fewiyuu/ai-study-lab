# 主线地图 {#map}
<!-- tag: mental model -->

这组题的主线是“结构识别”。贪心不是 DP 的简化版，分治也不是普通递归的别名。排序常常不是终点，而是为了制造单调性、相邻关系或扫描顺序。数学题则需要把题目约束转成能计算的性质。

贪心选择局部不伤全局分治递归拆分与合并排序不变量单调 / 相邻数学性质整除 / 取模 / 随机反例检查小样本推翻

贪心要能解释为什么每次选当前看起来最好的方案，不等于贪心正确。你要能说明：把某个最优解调整成包含当前选择，不会让答案变差。判断句：如果当前选择会影响未来可选空间，就必须小心证明或回到 DP。分治要有合并逻辑分治把问题拆成更小的同类问题，再把结果合并。归并排序的核心两个有序数组的线性合并。判断句：如果子问题答案之间没有清晰合并方式，分治可能不合适。排序制造可利用顺序排序后，相邻元素、前缀最大、区间端点、双指针移动都变得有意义。很多题排序为了让贪心或扫描成立。判断句：排序后如果没有新不变量，只是习惯性排序，可能是在浪费复杂度。

# 术语表 {#terms}
<!-- tag: terms -->

| 术语 | 直觉 | 准确说法 | 常见误解 |
| --- | --- | --- | --- |
| 贪心选择性质 | 当前选法不会挡住最优解。 | 存在某个全局最优解包含当前局部选择。 | 以为样例通过就说明贪心正确。 |
| 交换论证 | 把最优解中的一部分换成贪心选择，结果不变差。 | 证明贪心正确的常用方法。 | 只描述算法，不解释为什么能换。 |
| 分治 | 拆开算，再合起来。 | 递归解决同类子问题，并用合并过程得到原问题答案。 | 把所有递归都叫分治。 |
| 排序稳定性 | 相等元素原有顺序是否保留。 | 稳定排序不会改变相等键的相对顺序。 | 只看时间复杂度，忽略稳定性要求。 |
| 取模 | 只保留余数。 | 常用于防溢出、周期、同余关系和哈希。 | 在除法时随意取模，破坏等价关系。 |

# 知识链路 {#chain}
<!-- tag: chain -->
<!-- layout: grid2 -->

## box
贪心题的完整链路是：确定排序或选择标准，说明每一步选什么，再说明这个选择不会让后续最优解变差。区间调度按结束时间早排序，就是因为越早结束，给后面留下的空间越大；如果某个最优解第一段不是最早结束的区间，可以把它换成最早结束的区间，后续仍然可行。
分治题的完整链路是：定义递归函数负责的区间，写出终止条件，拆成左右子问题，再写合并过程。归并排序合并两个有序区间；快速排序通过 partition 把小于 pivot 和大于 pivot 的元素分开。两者都是排序，但递归后的工作不同。
数学杂题的链路是：先找约束里的不变量。素数题看整除范围，快速幂看指数二进制拆分，随机权重选择看前缀和和二分，最大公约数看欧几里得算法，位运算看每一位独立贡献。

## box soft
### 本页的判断句
- 贪心要能找反例，也要能说出交换理由。
- 分治必须有明确合并步骤。
- 排序后要产生可利用的单调性或相邻关系。
- 数学题先找不变量、范围和取模规则。

$$
快速幂思想：a^n = (a^(n/2))^2；若 n 为奇数，再乘一次 a。每次把指数减半。
$$

## box soft
### 核心不变量
- 区间贪心扫描到某一步时，`end` 始终是已选区间中最后一个区间的结束点，后续只接不冲突区间。
- 归并排序合并前，左右两段各自有序；合并后，输出前缀始终是当前可取元素中的最小有序前缀。
- 快速排序 partition 后，pivot 左侧不大于它、右侧不小于它，递归只负责继续整理左右内部。
- 快速幂每轮保持 `ans * base^exp` 等于原始幂值的剩余贡献。

## box
### worked example：区间调度反例
区间 `[1,4]`、`[2,3]`、`[3,5]` 如果按开始时间选，会先拿 `[1,4]`，最多只能选 1 个；按结束时间选，先拿 `[2,3]`，再接 `[3,5]`，能选 2 个。这个例子说明贪心标准必须服务于可证明的不变量。
复杂度通常由排序主导为 O(n log n)，扫描本身是 O(n)。

## box
### 同一套判断，换几种题面都能用
如果你把“看排序信号、看合并信号、看数字信号”当成固定动作，很多综合题其实会变得很安静。区间调度、快速排序、快速幂、取模、比较器这些题看起来差得很远，实际上只是把信号换了壳。
比如区间调度的信号是“结束得早更好”，快速排序的信号是“先把 pivot 放到边界上”，快速幂的信号是“指数可以二进制拆分”，而取模题的信号是“运算能否保持同余关系”。你要做的先把这个信号说出来。

# 核心模板 {#template}
<!-- tag: templates -->

区间调度是最经典的贪心例子。按结束时间排序，每次选择不和已选区间冲突的最早结束区间。这里的重点“结束越早，留给后面的空间越大”。

正确示例：区间调度贪心Python

```text
def maxNonOverlap(intervals):
 intervals.sort(key=lambda x: x[1])
 count = 0
 end = float("-inf")
 for s, e in intervals:
 if s >= end:
 count += 1
 end = e
 return count
```

排序后只看当前区间能不能接在已选区间后面。`end` 保存当前选择集合的最后结束时间。

正确示例：归并排序Python

```text
def merge_sort(nums):
 if len(nums) <= 1:
 return nums
 mid = len(nums) // 2
 left = merge_sort(nums[:mid])
 right = merge_sort(nums[mid:])
 ans = []
 i = j = 0
 while i < len(left) and j < len(right):
 if left[i] <= right[j]:
 ans.append(left[i]); i += 1
 else:
 ans.append(right[j]); j += 1
 ans.extend(left[i:])
 ans.extend(right[j:])
 return ans
```

递归负责把左右排好，合并负责把两个有序数组变成一个有序数组。

# 渐进例子 {#examples}
<!-- tag: examples -->

工具入口切换切换场景，观察先看哪条线索。贪心分治数学按结束时间留空间区间题里，越早结束越不妨碍后续选择。先选结束早的区间，通常更容易证明。工具选择要落到不变量，不要只靠题目分类名。

快速排序的核心是 partition：把一个 pivot 放到最终位置附近，再递归处理两边。面试里常问平均复杂度和最坏复杂度，也会问为什么随机 pivot 能降低退化概率。

正确示例：快速排序 partitionPython

```text
def quick_sort(nums, lo=0, hi=None):
 if hi is None:
 hi = len(nums) - 1
 if lo >= hi:
 return
 pivot = nums[hi]
 i = lo
 for j in range(lo, hi):
 if nums[j] < pivot:
 nums[i], nums[j] = nums[j], nums[i]
 i += 1
 nums[i], nums[hi] = nums[hi], nums[i]
 quick_sort(nums, lo, i - 1)
 quick_sort(nums, i + 1, hi)
```

partition 后，pivot 左边都小于它，右边都不小于它。左右内部还没排好，所以继续递归。

正确示例：快速幂Python

```text
def fast_pow(a, n, mod=None):
 ans = 1
 while n > 0:
 if n & 1:
 ans = ans * a if mod is None else ans * a % mod
 a = a * a if mod is None else a * a % mod
 n >>= 1
 return ans
```

指数每轮右移一位。当前二进制位为 1 时，把当前底数乘进答案。

错误示例：没有证明的贪心Python

```text
def choose(intervals):
 intervals.sort(key=lambda x: x[0]) # 只按开始时间早选
 # 错：开始早不代表留下的后续空间最大，容易被长区间卡住
```

# 完整追踪 {#trace}
<!-- tag: trace -->

<!-- interactive: ['shape-tracer'] → keep as HTML -->

拿区间调度做一次完整手算：把所有区间按结束时间排好，设 `end` 是当前选中集合里最后一个区间的结束点。每遇到一个新区间，只要它的开始点不小于 `end`，就把它接上。这个过程看起来像扫描，真正起作用的是不变量：`end` 只会向右移动，不会倒回来。

再拿快速排序看一遍。partition 之前，数组只是一堆数；partition 之后，pivot 左侧都不大于它，右侧都不小于它。这里把“全局排序”拆成两个更小的排序问题。你如果在这里忘了合并，只会得到两个局部正确、整体错误的结果。

快速幂的意义也类似。指数并每轮把问题规模减半。偶数位不需要乘进答案，奇数位才乘一次当前底数；然后底数平方，指数右移。它和贪心、分治不同，但都在做同一件事：找到一个跨轮次成立的不变量，让每轮只处理必要的信息。

判断式：如果一个题能先排序或先分治，并且你能说明“当前一步怎么做不会破坏后续最优”，就优先从贪心或分治切入。
快速幂的式子也可写成：a^n = (a^(n/2))^2，n 为奇数时再乘一次 a。

| 场景 | 你要盯住的状态 | 一旦看错，最常见的后果 | 修法 |
| --- | --- | --- | --- |
| 区间调度 | 最后结束时间 end | 把长区间先选进去，后面没空间 | 按结束时间排序，先保证“留空间” |
| 归并排序 | 左右两段都已排好 | 合并时误读未就绪的值 | 先让左右各自成立，再线性合并 |
| 快速排序 | pivot 的最终相对位置 | partition 做完却忘了两侧还要递归 | 把 pivot 当成边界，不是答案本身 |
| 快速幂 | 当前底数与剩余指数 | 指数减半了，底数却没平方 | 每轮同时更新底数和指数 |

区间调度反例检验Python

```text
def greedy_check(intervals):
 intervals = sorted(intervals, key=lambda x: x[1])
 end = float('-inf')
 picked = []
 for s, e in intervals:
 if s >= end:
 picked.append((s, e))
 end = e
 return picked

# 先用 [1,4], [2,3], [3,5] 这样的样例手算
# 如果按开始时间选会卡住，说明策略本身就错了
```

如果你不能很快写出一个会把策略推翻的小样本，那通常你还没找到真正的判据。

# 迁移到新题 {#transfer}
<!-- tag: transfer -->
<!-- layout: grid2 -->

## box
### 看到新题先问哪三个问题
先问题目能不能排序。排序本身不是答案，但它常常把“前后关系、相邻关系、区间关系”暴露出来。若排序以后出现了明显的局部规则，比如结束早的区间更容易被接上，或者相同权重可以按前缀区间落点来选，那题目大概率已经露出了可用的不变量。
再问子问题能不能合并。只要题目拆成两半以后还能把答案拼回来，分治就有入口。若左右两边完全独立，那就可以把合并写成线性过程；若合并时还要回看很多历史信息，问题通常已经更像 DP 或搜索。
最后问数字性质能不能直接算。能整除、能取模、能二进制拆分、能把概率转成前缀区间的题，往往都不需要复杂结构，只需要把约束翻成可计算的形式。这个动作一旦做对，代码通常就短了。

## box soft
### 反例优先级
- 先找一个小得足够手算的反例，检查你选的排序标准是否会挡住后面的选择。
- 再找一个边界值，检查比较器是否满足传递性，或随机数的边界是否和前缀区间对齐。
- 最后看合并步骤，判断你拿到的子答案是否能稳定拼回总答案。
工具判断顺序pseudo
```text
def pick_tool(problem):
 if has_ordering_signal(problem):
 return 'sort + greedy'
 if can_split_and_merge(problem):
 return 'divide and conquer'
 if has_numeric_invariant(problem):
 return 'math transform'
 return 'need more modeling'

# 先找信号，再选模板。
# 只记模板会在变体题里失效。
```


如果一个题的核心信号能被压缩成“局部最优可交换”“子问题可合并”或“数字性质可直接计算”，就分别考虑贪心、分治或数学化。
如果三者都不成立，先别套模板，回到题意重新建模。

| 信号 | 你要验证的句子 | 常见落点 | 失败后果 |
| --- | --- | --- | --- |
| 排序信号 | 排序后相邻关系能不能被利用 | 区间调度、nSum、比较器题 | 排序了半天，没得到新信息 |
| 合并信号 | 左右子问题能不能线性拼接 | 归并、区间统计、合并区间 | 递归写完却不知道怎么返回 |
| 数值信号 | 整除、取模、概率区间是否可直接计算 | 快速幂、筛法、随机映射 | 边界一变结果就偏 |

这一步的目标很朴素：让你在见到新题时，先把题意翻成状态，再决定该走哪条线，而不是先拿一个模板硬往里塞。

# 边界与复杂度 {#boundaries}
<!-- tag: limits -->

贪心边界如果局部选择会改变未来选择集合，并且找不到交换论证，先怀疑贪心。小反例是排查贪心的好工具。排序复杂度比较排序平均下界是 O(n log n)。计数排序、桶排序利用值域，不属于普通比较排序。数学溢出和取模乘法、幂和组合数题要注意取模位置。加乘可按同余规则取模，除法不能随便直接取模。

| 工具 | 典型复杂度 | 适用信号 | 风险 |
| --- | --- | --- | --- |
| 贪心 | 常见 O(n log n) | 排序后逐步选择 | 缺证明容易错 |
| 归并排序 | O(n log n) / O(n) | 稳定排序、逆序对 | 需要额外空间 |
| 快速排序 | 平均 O(n log n) | 原地排序、快速选择 | 最坏 O(n²) |
| 快速幂 | O(log n) | 大指数幂、取模幂 | 负指数和取模除法要另处理 |

# 常见修法 {#repair}
<!-- tag: repair -->

只记结论，不记理由错法是背“按结束时间排”“先分治”“先取模”这些口令，却说不出为什么。这样遇到变体时就会迷路。修法是把每一步都翻成一句判据：它在保护什么不变量，或者在减少什么未来冲突。
只会写模板，不会找反例错法是看到区间题就直接写贪心，看到排序题就默认比较器没问题。结果样例能过，边界一来就垮。修法是给自己留一条固定动作：先手算一个小样本，再用它去验证“当前选择是不是会堵住后面”。
把分治当成递归同义词错法是只写 `solve(left)`、`solve(right)`，却没有合并过程。这样写出来的代码只是递归展开，不是完整算法。修法是先写合并，再写递归边界；如果想不出怎么合并，就先停下，说明题目可能不是分治入口。

一个很实用的判断方式，是把“错误答案会怎么坏掉”说完整：会多选、少选、重复算、还是把顺序搞乱。你一旦把失败方式说具体，修法通常也就出来了。比如快排退化递归深度变成线性；取模除法整个等价关系已经被你破坏了。

区间题里，最容易丢的是“留空间”的意识。你如果按开始时间先拿长区间，后面就会被挤掉；如果按结束时间先拿短区间，剩下的选择空间最多。这个判断不依赖记忆，依赖的是你能不能把“当前一步会给未来留下什么”说清楚。

分治题里，最容易丢的是“子问题答案如何拼回去”。归并排序之所以经典，是因为两个子数组已经分别有序，合并能线性完成。快排之所以能快，是因为 partition 以后左右两边互不干扰。你只要能把这句话说完整，很多“看起来像分治”的题就能被排掉。

| 问题 | 先问什么 | 你要得到的答案 | 不合格信号 |
| --- | --- | --- | --- |
| 区间贪心 | 当前选择会不会挡住后面 | 交换论证或反例 | 只说“看起来最优” |
| 分治 | 拆完以后怎么合并 | 明确的合并过程 | 只有递归，没有返回值语义 |
| 数学题 | 约束能不能变成不变量 | 整除、取模、概率、位运算的可计算形式 | 只会套公式，不会解释边界 |

如果你把这三条都能落到具体例子上，这一节就“可以自己重新推出来”。

# 常见错误：错因、后果和修法 {#mistakes}
<!-- tag: debug -->

贪心只凭直觉错因：当前最小、最大、最早看起来顺眼。后果：隐藏反例会推翻算法。修法：找交换论证或构造小反例验证。分治没有合并错因：只会递归拆分，没定义子答案如何组成总答案。后果：递归结果无法返回正确值。修法：先写清合并函数或跨区间统计逻辑。快速排序退化错因：固定 pivot 遇到已有序或重复数据。后果：递归深度接近 n，时间 O(n²)。修法：随机 pivot、三路 partition 或改用归并/堆排序。比较函数不自洽错因：排序比较规则不满足传递性。后果：排序结果不稳定，甚至不同语言行为异常。修法：用明确 key，或保证比较规则可传递。取模除法乱写错因：把 `(a / b) % mod` 当成 `a % mod / b % mod`。后果：结果错误。修法：需要模逆元时先确认模数和 b 互质。

# 易混概念 {#confusions}
<!-- tag: compare -->

贪心 vs 动态规划**贪心**：局部选择能安全固定。**DP**：需要比较多个历史状态，局部选择不能直接定。判断：能否证明当前选择不伤害未来。分治 vs 回溯**分治**：子问题相对独立，最后合并。**回溯**：在选择树里试探和撤销。判断：是否需要遍历所有选择路径。归并排序 vs 快速排序**归并**：先排左右，再线性合并，稳定但需额外空间。**快排**：先 partition，再排左右，平均快但可能退化。判断：稳定性和最坏复杂度是否重要。

### 3 填空

**Q:** 3. 稳定排序会保留相等元素的 ______。

Explain: 稳定排序会保留相等元素的相对顺序。判断是否需要稳定性时，要看相等键后面是否还带有原始顺序或次级信息。

### 4 填空

**Q:** 4. 快速幂把指数每次减半，时间复杂度通常是 ______。

Explain: 快速幂每次把指数减半，时间复杂度通常是 O(log n)。如果带取模，乘法和取模都要放在每次更新里，避免中间值膨胀。

- [ ] 按结束时间从早到晚
- [ ] 按区间长度从长到短
- [ ] 按输入顺序

### 6 代码阅读

**Q:** 5. 区间调度中 `end` 保存什么？为什么按结束时间排序？

Explain: end 表示当前已选择区间的最早结束边界。按结束时间排序，是为了让每次选择留下最多后续空间，这是区间调度贪心的关键不变量。

### 7 代码阅读

**Q:** 6. 归并排序中，合并阶段利用了左右数组的什么性质？

Explain: 合并阶段利用左右两半都已经有序这一性质。比较两个头部元素即可决定下一个输出，同时也能顺手统计逆序对等跨区间信息。

### 8 代码阅读

**Q:** 7. 快排 partition 后，pivot 左右分别满足什么条件？

Explain: partition 后，pivot 左边元素通常不大于它，右边元素通常不小于它，具体取决于模板口径。pivot 的最终位置确定后，只需要递归或查找相关一侧。

### 9 代码阅读

**Q:** 8. 快速幂里 `if n & 1` 处理的是什么情况？

Explain: if n & 1 处理指数当前最低位为 1 的情况，要把当前底数乘进答案。随后底数平方、指数右移，等价于按二进制展开指数。

### 10 诊断

**Q:** 9. 有人区间调度按开始时间最早选。错因、后果、修法是什么？

Explain: 错因是开始早不代表结束早，可能占用很长时间。后果是少选很多本可兼容的短区间；修法是按结束时间从早到晚选，并用 end 维护已选区间的右边界。

### 11 诊断

**Q:** 10. 分治只递归左右，不写合并，会导致什么问题？

Explain: 分治只递归左右会得到两个局部结果，但跨左右的关系没有处理。后果是排序、逆序对、最大跨区间子数组等问题都缺结果；修法是补合并逻辑，明确合并阶段产出什么。

### 12 诊断

**Q:** 11. 快排固定选最后一个元素为 pivot，在有序数组上可能怎样退化？怎么修？

Explain: 有序数组上固定选最后一个 pivot，partition 会极度不平衡，递归深度接近 n，时间退化到 O(n^2)。常见修法是随机 pivot、三数取中，或在重复值多时用三路划分。

### 13 诊断

**Q:** 12. 排序比较规则不满足传递性，会有什么风险？

Explain: 比较器不满足传递性会让排序结果不稳定甚至违反算法假设，后续二分、去重、扫描都会失真。修法是把比较规则写成严格弱序，并用小样例检查 a<b、b<c 时 a<c 是否成立。

### 14 诊断

**Q:** 13. 为什么取模运算里不能随便做除法？需要什么条件才可用模逆元？

Explain: 模意义下不能直接除法，因为除法需要乘以逆元。只有除数和模数互质时逆元才存在；常见质数模下可用费马小定理求逆元。

### 15 迁移

**Q:** 14. 跳跃游戏为什么常用贪心维护最远可达位置？这个不变量是什么？

Explain: 跳跃游戏维护 farthest，表示扫描到当前位置前能到达的最远下标。只要当前位置不超过 farthest，就可以继续扩展；若某处超过 farthest，说明前面所有选择都到不了这里。

### 16 迁移

**Q:** 15. 逆序对为什么适合用归并排序的合并阶段统计？

Explain: 逆序对是左边元素大于右边元素的跨区间关系。归并时左右两半已有序，一旦左指针元素大于右指针元素，就能一次性计算左侧剩余元素形成的逆序对数量。

### 17 迁移

**Q:** 16. 第 K 大元素如何联系到快排 partition 或快速选择？

Explain: 第 K 大可以转成找排序后某个下标的元素。快速选择复用 partition，每次 pivot 定位后只递归目标所在一侧，平均时间 O(n)。

### 18 迁移

**Q:** 17. 计数小于 n 的素数时，埃氏筛利用了什么性质？

Explain: 埃氏筛利用的是：如果一个数是素数，那么它的倍数一定不是素数。从小到大筛时，可以从 p*p 开始划掉，因为更小的倍数已经被更小因子处理过。

### 19 迁移

**Q:** 17a. 加油站问题常用贪心重置起点。你会用什么不变量解释“当前累计油量为负时，前面的起点都不可能成功”？

Explain: 当从候选起点到当前位置累计油量为负，说明这段路内部任一点作为起点都会在当前位置前耗尽。总油量足够时，可以把起点重置到下一站，并继续维护当前段累计油量。

### 20 面试

**Q:** 18. 用 2-3 句话解释什么样的题可以尝试贪心。

Explain: 当题目具有局部选择，并且能证明这个选择不会破坏全局最优时，可以尝试贪心。常见证明方式是交换论证、不变量或反证；没有证明时要先找小反例。

### 21 面试

**Q:** 19. 用 2-3 句话比较归并排序和快速排序。

Explain: 归并排序稳定，时间稳定 O(n log n)，需要 O(n) 额外空间，适合顺便统计跨区间信息。快速排序原地性好、平均 O(n log n)，但 pivot 不佳时会退化，通常配合随机化。

### 22 复杂度

**Q:** 20. 写出区间贪心、归并排序、快速排序、快速幂的常见复杂度。

Explain: 区间调度按结束时间排序后线性扫描，通常 O(n log n) 时间、O(1) 或 O(n) 空间。归并排序 O(n log n) 时间、O(n) 空间；快速排序平均 O(n log n)、最坏 O(n^2)、递归栈平均 O(log n)；快速幂 O(log n) 时间、O(1) 空间。

### 23 费曼解释

**Q:** 21. 用 6-9 句话讲清贪心、分治、排序、数学杂题分别看什么信号，以及常见错误。

Explain: 贪心题先看是否存在可靠的局部选择，再用交换论证或不变量排除反例。分治题要确认拆分后的左右结果如何合并，不能只递归不收尾。排序题要看排序不变量、稳定性和比较器是否满足传递性。数学杂题要特别小心取模、除法和边界条件，模除法需要逆元条件。遇到综合题时，先拿小反例测试策略，再决定用贪心、分治、排序还是数学工具。

### 24 清单

**Q:** 22. 写出你以后遇到综合杂题时会检查的 5 件事。

Explain: 先判断题目有没有可证明的贪心选择；再看是否能拆成子问题并合并；然后检查排序是否只是预处理还是核心不变量；接着确认复杂度、边界和重复值处理；最后用小反例验证比较规则、取模操作和局部选择是否可靠。
