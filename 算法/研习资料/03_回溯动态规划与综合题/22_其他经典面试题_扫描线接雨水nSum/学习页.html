把题面变成可维护的状态

扫描线、接雨水、丑数、带权随机选择、nSum 都是面试里常见的综合题。它们没有统一的数据结构，但有共同做法：先把题面改造成一种可维护状态，再按顺序推进。

扫描线把区间变成事件；接雨水维护左右最大高度；丑数用多个指针生成有序候选；带权随机选择用前缀和把权重映射到区间；nSum 通过排序和双指针把组合搜索压缩到可控范围。

学完后能做的事

  1. 能把区间重叠题转成开始/结束事件。
  2. 能解释接雨水的左右最大值、双指针和单调栈三种思路。
  3. 能写出丑数多指针生成过程。
  4. 能用前缀权重和二分实现按权重随机选择。
  5. 能写出 nSum 递归框架并处理重复答案。

主线地图

mental model

这些题的共同点是“把难题改写成顺序维护”。区间题按时间顺序维护活跃数量；雨水题按高度边界维护可接水量；随机题按权重前缀维护区间长度；nSum 按排序后的单调性移动指针。

扫描线：区间变事件

区间 `[start, end)` 可以变成两个事件：开始时活跃数量加一，结束时活跃数量减一。按时间排序后，一边扫描一边维护当前重叠数量。

判断句：题目问区间重叠、会议室数量、天际线轮廓，先想事件顺序。

接雨水：水由短板决定

某个柱子能接多少水，取决于它左侧最高柱和右侧最高柱中较小的那一个。双指针写法利用这一点，谁的边界低就先结算谁。

判断句：当前位置水量是 `min(leftMax, rightMax) - height[i]`。

nSum：排序后递归降维

排序让 twoSum 可以双指针。nSum 固定一个数后，把问题递归降成 n-1 Sum，最终落到 twoSum。

判断句：去重必须和排序绑定，否则同一组合会重复出现。

术语直觉准确说法常见误解
扫描线按坐标或时间从左到右扫。把对象转成有序事件，扫描时维护活跃状态。结束事件和开始事件同点时顺序处理错。
左/右最大值当前位置两边能挡水的最高墙。`leftMax[i]` 和 `rightMax[i]` 分别是两侧最高高度。只看相邻柱子，漏掉远处边界。
前缀权重把每个权重变成一段连续区间。权重数组的累积和，用随机数落点二分查索引。随机范围取错开闭区间。
丑数只含 2、3、5 质因子的数。可由前一个丑数乘 2、3、5 得到，并按升序去重生成。遇到相同候选只移动一个指针，产生重复。
去重同一组合只保留一次。排序后跳过相同固定值和相同左右指针值。用 set 兜底却没理解重复来源。

知识链路

chain

扫描线的链路是：把每个区间拆成事件，给开始和结束标记不同增量；按时间排序；扫描时维护活跃数量或活跃集合；在每个事件点更新答案。会议室 II 可以用开始 +1、结束 -1 求最大活跃数量。天际线更复杂，需要维护当前最高建筑。

接雨水的链路是：水位由左右最大值的较小者决定。预处理数组法最直观，先算每个位置左边最高和右边最高，再累加水量。双指针法省空间，它利用“较低一侧已经能确定水位”的事实。

nSum 的链路是：排序，递归固定当前数，缩小目标，直到 twoSum；twoSum 用左右指针根据和与目标的大小移动。每一层都要跳过重复固定值，twoSum 找到答案后也要跳过重复左右值。

本页的判断句

  • 区间重叠，先事件化。
  • 柱状雨水，看左右边界。
  • 按权重随机，前缀和加二分。
  • nSum 先排序，再递归降到 twoSum。
接雨水单点公式：water[i] = max(0, min(leftMax[i], rightMax[i]) - height[i])。

核心不变量

  • 扫描线处理到某个事件点时，active 等于当前仍覆盖该位置的区间数量或集合。
  • 接雨水双指针中，较低一侧的最大边界已经足以决定该侧水位，另一侧只会更高或不影响结算。
  • 权重随机中，前缀区间长度正比于权重，随机落点必须与二分边界保持同一套开闭区间。
  • nSum 每层固定一个数后，剩余问题仍是在排序数组右侧寻找不重复组合。

worked example：雨水与 nSum

高度 [3,0,2] 中间位置的左最大是 3，右最大是 2，所以接水 min(3,2)-0=2。三数之和 [-1,0,1,2,-1,-4] 排序后先固定第一个 -1，左右指针找到 [-1,0,1] 后，两侧重复值要跳过；第二个 -1 如果在同层再次固定，也要跳过。

扫描线和 nSum 都靠排序制造可维护顺序，接雨水靠边界不变量减少重复计算。

接雨水和扫描线的共同陷阱

这两类题最怕的都是“把局部当成全部”。扫描线里，你不能只看当前点附近的几个事件，而要看整个事件序列里的同点顺序；接雨水里，你不能只看当前柱子左右的邻居，而要看左侧最高和右侧最高这两个边界。

所以这页里最该记住的不是某个代码片段，而是两个判断句：扫描线看事件顺序，接雨水看边界高度。只要你把这两句说准，很多题的第一层建模就已经过了。

核心模板

templates

扫描线的关键是事件排序。同一时刻开始和结束如何排序，要按题目区间口径决定。会议区间通常是半开区间，某会议结束的时刻可以立刻开始另一个会议。

正确示例：会议室数量扫描线Python
def minMeetingRooms(intervals):
    events = []
    for s, e in intervals:
        events.append((s, 1))
        events.append((e, -1))
    events.sort(key=lambda x: (x[0], x[1]))
    active = ans = 0
    for _, delta in events:
        active += delta
        ans = max(ans, active)
    return ans

同一时刻结束事件 `-1` 先处理，表示会议室可以复用。

正确示例：接雨水双指针Python
def trap(height):
    l, r = 0, len(height) - 1
    leftMax = rightMax = 0
    ans = 0
    while l <= r:
        leftMax = max(leftMax, height[l])
        rightMax = max(rightMax, height[r])
        if leftMax < rightMax:
            ans += leftMax - height[l]
            l += 1
        else:
            ans += rightMax - height[r]
            r -= 1
    return ans

较低一侧的最大边界已经决定水位，因为另一侧至少不低于它。

事件按时间推进

开始事件让活跃数量加一，结束事件让活跃数量减一。最大活跃数量就是会议室需求。

先把题面转换成可维护状态，再写循环。

带权随机选择的直觉是：权重越大，占据的前缀区间越长，随机数落到它的概率就越高。实现时不要真的展开数组，用前缀和和二分即可。

正确示例：按权重随机选择Python
import bisect, random

class Solution:
    def __init__(self, w):
        self.pre = []
        total = 0
        for x in w:
            total += x
            self.pre.append(total)
        self.total = total

    def pickIndex(self):
        x = random.randint(1, self.total)
        return bisect.bisect_left(self.pre, x)

随机整数取 `[1, total]`，前缀和用第一个大于等于落点的位置。

正确示例：三数之和Python
def threeSum(nums):
    nums.sort()
    ans = []
    n = len(nums)
    for i in range(n - 2):
        if i > 0 and nums[i] == nums[i - 1]:
            continue
        l, r = i + 1, n - 1
        while l < r:
            s = nums[i] + nums[l] + nums[r]
            if s == 0:
                ans.append([nums[i], nums[l], nums[r]])
                l += 1; r -= 1
                while l < r and nums[l] == nums[l - 1]:
                    l += 1
                while l < r and nums[r] == nums[r + 1]:
                    r -= 1
            elif s < 0:
                l += 1
            else:
                r -= 1
    return ans

排序后，和小了移动左指针，和大了移动右指针。去重发生在固定 `i` 和找到答案之后。

正确示例：丑数 II 多指针Python
def nthUglyNumber(n):
    ugly = [1]
    i2 = i3 = i5 = 0
    while len(ugly) < n:
        nxt = min(ugly[i2] * 2, ugly[i3] * 3, ugly[i5] * 5)
        ugly.append(nxt)
        if ugly[i2] * 2 == nxt:
            i2 += 1
        if ugly[i3] * 3 == nxt:
            i3 += 1
        if ugly[i5] * 5 == nxt:
            i5 += 1
    return ugly[-1]

候选相同时，多个指针都要移动，否则会生成重复丑数。

完整追踪

trace
扫描线开始 +1，结束 -1

会议室数量题把区间转成事件以后，真正要维护的不是会议本身，而是当前活跃数量。排序后按时间扫过去，最大活跃数就是答案。

接雨水较低边界先结算

水位不是由当前柱子单独决定，而是由两侧最高墙里较低的那边决定。双指针写法把这个事实变成循环方向，所以它能用 O(1) 空间。

随机权重权重映射成区间长度

把权重累积成前缀和以后，随机数落在哪一段，哪一个索引就被选中。你维护的是区间边界，而不是把数组真的展开。

nSum先排序，再降维

nSum 把多层选择压成固定一个数、再在剩余数组里找更低维的组合。去重一定要跟排序配合，否则同一组合会因为顺序不同被算多次。

先看扫描线。比如会议室区间 [0,30]、[5,10]、[15,20]，如果把开始记成 +1、结束记成 -1，排序后扫一遍就能知道任意时刻有多少会议同时活跃。这里的关键不是“数组里放了什么”，而是同一时刻的事件顺序：半开区间通常要先处理结束，再处理开始，不然同一时间点会被你多算一次。

再看接雨水。高度数组 [4,2,0,3,2,5] 里，中间的水位不是看左右相邻柱，而是看左右最高墙。你如果只盯着局部，就会漏掉远处的高墙；你如果用双指针，就会把“较低一侧的边界已经足够决定水位”这件事写进循环。这个思路和扫描线很像，都是先建立状态，再顺着状态推进。

权重随机和 nSum 也是同一类问题。前者把权重变成前缀区间，再用随机数和二分锁定索引；后者先排序，再用固定值 + 双指针把组合问题降成两个数的查找。你在这两类题里都不是在“猜答案”，而是在维护一个可检索的顺序结构：一个是区间边界，一个是有序数组。

接雨水：water[i] = max(0, min(leftMax[i], rightMax[i]) - height[i])。
权重随机：若前缀和为 pre，随机整数落在 (pre[i-1], pre[i]]，就选索引 i。
nSum：排序后，kSum 固定一个数，剩余问题继续变成 (k-1)Sum。
题型你维护的状态边界在哪一步决定最常见的坏结果
扫描线会议室active 活跃数同点事件的排序多算/少算一次重叠
接雨水leftMax / rightMax较低一侧先结算远处高墙被漏掉
权重随机prefix 前缀区间随机数的闭开区间概率偏移或越界
nSum固定值 + 双指针固定层去重和收缩指针重复组合进入答案
nSum 排序去重骨架Python
def ksum(nums, target, k):
    nums.sort()
    ans = []
    def dfs(start, k, target, path):
        if k == 2:
            l, r = start, len(nums) - 1
            while l < r:
                s = nums[l] + nums[r]
                if s == target:
                    ans.append(path + [nums[l], nums[r]])
                    l += 1; r -= 1
                    while l < r and nums[l] == nums[l - 1]:
                        l += 1
                    while l < r and nums[r] == nums[r + 1]:
                        r -= 1
                elif s < target:
                    l += 1
                else:
                    r -= 1
            return
        for i in range(start, len(nums) - k + 1):
            if i > start and nums[i] == nums[i - 1]:
                continue
            dfs(i + 1, k - 1, target - nums[i], path + [nums[i]])
    dfs(0, k, target, [])
    return ans

这一段代码背后其实只有两个动作：固定当前层，然后把问题缩小一层。只要你还记得什么时候该去重、什么时候该缩指针，nSum 的变体基本都能接住。

迁移到新题

transfer

综合题的四个壳

扫描线、接雨水、前缀随机和 nSum 看起来像四类题，但它们都在做同一件事：把原题改造成一种顺序可维护的状态。扫描线维护活跃数量，接雨水维护左右边界，随机选择维护前缀区间，nSum 维护已经固定的前缀和剩余的有序搜索范围。

一旦你能把题目压成“状态 + 推进 + 边界”，很多题就不再难。比如会议室数量不需要关心每个会议的内容，只关心活跃数；三数之和也不需要关心每个排列，只关心排序后的去重组合。你要看的永远是状态是否稳定，而不是题面名词长什么样。

这也是为什么面试里经常考这几类题：它们表面是算法题，实际上在考你能不能把输入改写成可控结构。只要建模对了，代码通常不会太长；建模错了，代码写得再多也只是把错误铺开。

两个实战桥梁

天际线和会议室数量都属于扫描线，只是前者维护的是最高轮廓，后者维护的是活跃数量。前者更像“状态集合”的变化，后者更像“计数器”的变化；但两者都依赖事件排序。

航班预订、区间加法、范围统计这类题，若坐标是连续的而且上限可控，差分数组会比扫描线更直接。差分维护的是“每个位置被加了几次”，前缀和再把它还原出来。坐标很稀疏时，扫描线更灵活；坐标很密集时，差分更省事。

差分数组骨架Python
def diff_range_updates(n, updates):
    diff = [0] * (n + 1)
    for l, r, val in updates:
        diff[l] += val
        if r + 1 < n:
            diff[r + 1] -= val
    ans = [0] * n
    cur = 0
    for i in range(n):
        cur += diff[i]
        ans[i] = cur
    return ans
扫描线维护的是“某一刻有哪些事件还活着”；差分数组维护的是“某个位置被影响了多少次”；nSum 维护的是“固定前缀之后，剩余数组里如何找不重复组合”。
它们的共同点是：都先把问题改写成状态，再按一个稳定顺序推进。
题型输入改写成什么你要维护的状态常见错法
扫描线开始/结束事件活跃数量或活跃集合同点顺序和题意不一致
接雨水左右边界leftMax / rightMax把局部邻居当成全部边界
随机权重前缀区间落点与索引映射随机区间和二分边界口径不一致
nSum排序后的搜索空间固定层 + 双指针只去重一层，重复组合漏进答案

一个短手算

高度 [4,2,0,3,2,5] 的第 2 个位置，左边最高是 4，右边最高是 5，所以能接的水是 min(4,5)-2=2；如果你只看相邻柱子，就会误以为它只能接 0 或 1。

这类例子很有用，因为它会逼你把“局部邻居”改成“全局边界”。扫描线和 nSum 也是同样的逻辑：你不能只看眼前一小段，而要看整个有序状态是否已经准备好。

如果你能在纸上先把输入改写成事件、边界、前缀或有序搜索空间，这一页就已经会了大半。

一眼看穿题型的信号

如果题面里出现“区间重叠”“同时发生”“求最少会议室”，大概率是扫描线；如果出现“高度图”“接多少水”，大概率是边界题；如果出现“按权重抽样”，就去想前缀和；如果出现“n 个数求和”，就去想排序、双指针和去重。

这个判断很短，但它会直接缩小搜索空间。你不用先想所有可能，只要先把明显不对的方向排掉，剩下的模板通常就已经很少了。

一句话总结

扫描线看事件顺序，接雨水看左右边界，随机权重看前缀区间，nSum 看排序后的去重搜索。只要你能先把题面翻成状态，这一页的大多数题就已经走对方向了。

再补一句更实用的判断：如果你在面试里把这四类题讲给别人听，最好别只说“它们都要排序”。更准确的说法是，扫描线靠事件顺序，接雨水靠边界，随机权重靠区间映射，nSum 靠排序后的去重搜索。这个区别一旦说清，后面的代码模板就会非常自然。

边界与复杂度

limits

事件同点顺序

扫描线同一坐标的开始和结束事件，顺序要按题目含义处理。半开区间通常先结束后开始。

雨水空数组

高度数组长度不足 3 时接不到水。双指针写法也要能自然处理空数组和单元素。

nSum 去重

去重不是最后用 set 草草处理。排序后在每层固定值和 twoSum 指针移动时跳过重复，复杂度和输出更稳定。

题型核心状态复杂度主要风险
扫描线会议室活跃区间数O(n log n)同点排序
接雨水双指针左右最大高度O(n) / O(1)水位结算方向
权重随机前缀权重数组初始化 O(n)，选择 O(log n)随机范围
nSum排序 + 递归 + 双指针约 O(n^(k-1))去重和剪枝

常见修法

repair

只盯着局部相邻值

错法是把接雨水想成“看左右邻居的高低”，或者把扫描线想成“看当前事件附近的几项”。这样会把远处边界和事件顺序都漏掉。

修法是强迫自己先写状态名：active、leftMax、prefix、path。只要状态一命名，哪些信息必须保留就会清楚很多。

随机和二分开闭区间不一致

错法是随机数取一个区间，前缀数组却按另一个区间定义。这样会让某些索引概率偏大，或者把边界点漏掉。

修法是先写一行“随机落点落在哪”，再写一行“前缀边界如何比较”。两边只要不是同一套开闭区间，结果就一定会歪。

nSum 去重只做一半

错法是只跳过固定值，却忘了找到答案以后左右指针还要继续跳。也有人反过来，只在 twoSum 去重，固定层却重复了。

修法是把去重拆成两层：固定层去重保证不会重复开分支，指针层去重保证同一对不会重复进答案。

如果你在答题时容易写得散，就按“状态是什么、状态怎么变、边界在哪里、错了会怎样”这四句回答。扫描线题里说状态时先讲 active，接雨水时先讲左右最大，随机题先讲前缀区间，nSum 先讲排序后的双指针。你要把题目翻成状态，而不是把状态硬套到题目上。

会议室数量和天际线这类题，最常见的失败是同点事件顺序错了。你可以用一个极小的反例把它钉死：如果结束和开始都落在同一时刻，先开始会让活跃数多算一次。接雨水则相反，它更常见的错误不是顺序，而是把“当前点的水量”误认为“左右邻居决定”，结果远处的高墙完全没被算进来。

权重随机的错误，往往不在代码长度，而在区间口径。你如果随机数取的是 [0, total)，前缀判断却按 [1, total] 写，边界就已经错了一半。最稳的做法，是先把随机范围和前缀区间各写成一句中文，再把它们翻成代码。只要这两句中文不一致，代码就一定不一致。

常见错误：错因、后果和修法

debug

扫描线同点顺序错

错因：开始和结束事件排序没按区间含义设计。

后果：会议室数量多算或少算。

修法：半开区间同点先处理结束事件。

接雨水只看相邻柱

错因：误以为水只由左右邻居决定。

后果：远处高墙形成的水量被漏算。

修法：维护左侧最高和右侧最高。

随机落点从 0 到 total

错因：开闭区间和前缀和二分不匹配。

后果：概率偏移或索引越界。

修法：使用 `[1,total]` 搭配 `bisect_left`，或统一改成 `[0,total)` 搭配 `bisect_right`。

丑数重复候选只移动一个指针

错因：`6` 可以来自 `2*3` 和 `3*2`，只移动一个会重复生成。

后果：序列里出现重复，位置答案错。

修法：所有等于当前候选的指针都移动。

nSum 去重位置错

错因：只跳过固定值，忘记 twoSum 找到答案后的左右重复。

后果：重复组合进入答案。

修法：固定值和双指针两处都处理重复。

易混概念

compare

扫描线 vs 差分数组

扫描线适合坐标稀疏或事件数量有限。

差分数组适合坐标范围可控、需要区间加减。

判断：坐标很大但区间不多，优先事件排序。

接雨水双指针 vs 单调栈

双指针按左右边界结算。

单调栈在遇到右边界时结算凹槽。

判断：双指针更省空间，单调栈更像按坑结算。

twoSum 哈希 vs 双指针

哈希适合无序找一对数。

双指针适合排序后去重并扩展到 nSum。

判断：要输出不重复组合，排序双指针更稳。

诊断

9. 会议室扫描线同一时刻先处理开始事件，可能会怎样？

参考答案

如果半开区间 [start, end) 下同一时刻先处理开始事件，会把刚结束和刚开始的会议误判为重叠。后果是会议室数量可能多算；修法是同点先处理结束事件。

诊断

10. 接雨水只看左右相邻柱子会漏掉什么？举一个形状说明。

参考答案

接雨水取决于当前位置左侧最高柱和右侧最高柱的较小值，不是左右相邻柱子。比如 [3,0,0,2] 中间能接水，虽然相邻局部信息不足以直接判断总水位。

诊断

11. 权重随机选择中，随机范围和二分方式不匹配会有什么后果？

参考答案

随机范围和二分方式不匹配会造成某些区间永远抽不到，或边界点概率偏移。修法是统一使用 1..total 配 bisect_left，或 0..total-1 配 bisect_right/严格大于口径。

诊断

12. 丑数候选相同但只移动一个指针，会发生什么？

参考答案

候选值相同时只移动一个指针，会把同一个丑数重复加入序列。修法是所有等于当前最小候选的指针都移动，保证序列去重且递增。

诊断

13. 三数之和只跳过固定值，不跳过左右指针重复值，会怎样？

参考答案

只跳过固定值会避免一部分重复三元组，但左右指针在找到答案后仍可能扫到相同值。需要同时跳过 left 和 right 的重复值，保证输出集合不重复。

19. 用 2-3 句话解释接雨水双指针为什么能 O(1) 空间。

双指针接雨水只维护 leftMax 和 rightMax 两个边界最大值。每次结算较小最大值一侧，所以不需要额外数组保存每个位置的左右最大值，空间可降到 O(1)。
