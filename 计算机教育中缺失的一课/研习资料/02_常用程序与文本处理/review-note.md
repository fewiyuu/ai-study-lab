## 值得记住的问题

这次答题整体扎实：客观题全对，多选日志排查也选中了有用的 `grep`、`tail`、`less`，没有把 `mkdir` 混进来。开放题基本覆盖了参考要点，尤其能说清 `grep -n` 的输入、参数、模式、文件和输出形状。

需要继续巩固的是“输出形状变了以后，后续命令还能不能把它当原始行”：`wc` 和 `uniq -c` 已经把文本变成计数结果，不能再当作原日志行理解。

## 核心主线

处理陌生文本时，先不要急着写复杂管道。先按这条线走：

1. 查看样本：`head` 看格式，`tail` 看最近内容，长文件用 `less`。
2. 筛选行：`grep` 输出匹配到的原始行；`-n` 只是在前面加行号。
3. 统计规模：`wc -l` 输出文本行数，不等于错误数、用户数或业务记录数。
4. 整理重复：`sort` 先让相同行相邻，再交给 `uniq` 或 `uniq -c`。
5. 检查边界：压缩包、二进制、复杂 CSV/JSON 不要直接当普通文本行处理。

## 关键概念

- `head` / `tail` 保留原行，只是截取文件的一部分。
- `grep` 仍输出原始匹配行；加 `-n` 后会多一个行号前缀。
- `wc` 输出的是计数，不再保留原始行内容。
- `sort` 会改变原始顺序；如果时间顺序有意义，排序前要先想清楚。
- `uniq` 只压缩相邻重复行。要做全局去重，通常先 `sort`。
- `file archive.zip` 这类检查可以先判断输入是不是文本。

## 例子与操作模式

日志起手式：

```bash
head -n 3 app.log
wc -l app.log
tail -n 20 app.log
grep -n "timeout" app.log
grep -C 1 "timeout" app.log
```

这组命令分别回答：格式长什么样、文件多大、最近发生了什么、目标词在哪几行、目标词前后发生了什么。

全局去重计数：

```bash
sort names.txt | uniq -c
```

`sort` 的作用不是“好看”，而是让相同名字相邻；`uniq -c` 只统计相邻重复块。

## 常见误区

- 把 `wc -l app.log` 的数字当成错误次数。它只是文本行数。
- 忘记 `grep -n` 的行号是工具加的，不是文件原本字段。
- 没排序就用 `uniq -c`，导致分散重复被拆成多个块。
- 用默认 `sort` 排数字，结果 `10` 可能排在 `2` 前面；数字排序要用 `sort -n`。
- 对压缩包或二进制文件直接 `grep`，看到 `Binary file matches` 后还把它当日志行分析。
- 搜短语时忘记加引号，Shell 会先把短语切成多个参数。

## 迁移与使用

组合文本工具前，先问三个问题：

1. 这一步输出还是原始行吗？
2. 这一步丢掉了哪些信息？
3. 下一步命令需要的是原始行、计数，还是重新排序后的行？

如果拿到陌生 CSV，起手可以是：

```bash
head -n 5 data.csv
wc -l data.csv
less data.csv
```

`head` 用来看表头、分隔符和前几行结构；`wc -l` 只估算文本行数；文件长的时候用 `less` 搜索或抽看。正式清洗复杂 CSV 时，不要只靠普通文本行假设。

## 修正后的解释

查看、搜索、统计日志可以这样讲给零基础同学：先用 `head` 看格式，用 `tail` 看最新内容；文件长时用 `less` 分页和搜索。要找错误行，用 `grep -n "ERROR" app.log`，它会输出匹配整行，并在前面加行号。要估文件规模，用 `wc -l app.log` 看文本行数。短语模式要加引号，压缩包或二进制文件要先确认类型或解压，不能直接当普通文本日志处理。

## 复习卡片

- Q：`grep -n "ERROR" app.log` 输出的 `2:` 是什么？  
  A：`grep -n` 添加的行号前缀，不是日志原本字段。

- Q：为什么 `sort names.txt | uniq -c` 里要先 `sort`？  
  A：`uniq` 只处理相邻重复行，`sort` 先把相同行放到一起。

- Q：`wc -l app.log` 能说明什么，不能说明什么？  
  A：能说明文本行数；不能直接说明错误次数、用户数或业务事件数。

- Q：看到 `Binary file matches` 时先做什么？  
  A：先用 `file` 或来源信息确认文件类型，必要时解压或换格式工具，再处理里面的文本。

- Q：搜索 `user login` 为什么要写成 `grep "user login" app.log`？  
  A：引号让 Shell 把带空格的短语作为一个模式参数传给 `grep`。
